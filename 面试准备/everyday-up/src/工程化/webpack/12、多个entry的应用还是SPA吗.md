webpack 允许我们配置多个入口文件,每个入口文件会生成一个对应的 bundle 文件。但是这些 bundle 文件并不代表应用必须是多页应用。

在单页应用中,我们通常会将应用的核心代码、公共库、路由模块等打包到一个 bundle 中作为主入口,而将一些功能模块作为代码分割点异步加载。或者为了缓存和并行加载的目的,将某些模块拆分到独立的 bundle 中。这种情况下就需要配置多个入口文件。

举个例子,一个 SPA 可以这样配置入口:

```js
module.exports = {
  entry: {
    main: './src/main.js',  // 主入口, 包含核心代码
    vendor: './src/vendor.js', // 单独打包第三方库
    async: './src/async.js' // 异步加载的功能模块
  }
}
```

这样生成的 bundle 文件都将通过 HTML 中的 `script` 标签引入,通过 SPA 的路由系统控制加载和渲染不同的模块视图。

所以,webpack 多入口的配置只是为了打包优化和代码分割的目的,并不会将单页应用改变为多页应用。关键是在前端应用中如何利用路由和组件加载机制来实现 SPA。
