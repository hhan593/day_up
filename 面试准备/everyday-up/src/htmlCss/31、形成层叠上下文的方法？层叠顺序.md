### 31、形成层叠上下文的方法？层叠顺序？
层叠上下文（Stacking Context）是 CSS 中的一个三维概念，决定了文档中的元素如何在层叠方向上相互覆盖。形成层叠上下文的元素会在自己内部处理层叠，独立于外部的层叠顺序。这意味着层叠上下文内部的元素不会与外部的元素混合层叠。

### 形成层叠上下文的条件

层叠上下文可以通过以下方式形成：

1. **根元素**（HTML）自身就是一个层叠上下文。
2. **CSS定位**：对于 `position` 属性为 `absolute` 或 `relative` 的元素，当 `z-index` 值不是 `auto` 时，会创建层叠上下文。
3. **CSS Flexbox 和 Grid**：设置为 Flexbox 或 Grid 容器的子元素，当其 `z-index` 值不是 `auto` 时，也会创建层叠上下文。
4. **CSS变换和过滤**：应用 `transform`、`filter`、`perspective` 等属性的元素（不是 `none` 值）会创建层叠上下文。
5. **CSS混合模式**：使用 `mix-blend-mode` 属性（不是 `normal` 值）的元素会创建层叠上下文。
6. **CSS透明度**：`opacity` 属性值小于 1 的元素会创建层叠上下文。
7. **CSS剪裁和遮罩**：`clip-path`（不是 `none`）、`mask`（不是 `none`）等属性的元素会创建层叠上下文。
8. **will-change**：指定了任何会创建层叠上下文的属性作为值的 `will-change` 属性。
9. **-webkit-overflow-scrolling**：应用 `-webkit-overflow-scrolling: touch;` 的元素。

### 层叠顺序

在一个层叠上下文中，元素的层叠顺序如下（从下到上排序）：

1. **层叠上下文背景和边框**：层叠上下文元素的背景和边框。
2. **负z-index**：具有负 `z-index` 值的子元素。
3. **块级盒子**：在正常文档流中，非定位的块级盒子，包括 `float` 元素。
4. **块级盒子内的行内级元素**：非定位的行内级元素。
5. **z-index: auto 或 z-index: 0**：`z-index` 为 `auto` 或 `0` 的定位元素。
6. **正z-index**：具有正 `z-index` 值的子元素。

了解层叠上下文和层叠顺序对于管理复杂布局中元素的视觉顺序至关重要，尤其是在涉及到定位和视觉效果叠加时。
