在 HTML5 页面中，防止页面放大或缩小主要是通过在`<meta>`标签中设置视口（viewport）相关的属性来实现的。具体来说，可以在`<head>`部分添加一个名为`viewport`的`<meta>`标签，并通过设置`user-scalable=no`来禁止用户缩放页面。此外，还可以通过指定`maximum-scale`和`minimum-scale`的值为 1 来进一步限制缩放。

下面是一个示例代码：

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>无法缩放的页面</title>
    <meta
      name="viewport"
      content="width=device-width, initial-scale=1, maximum-scale=1, minimum-scale=1, user-scalable=no"
    />
  </head>
  <body>
    <h1>这个页面不能缩放</h1>
    <p>试图缩放这个页面将不会有任何效果。</p>
  </body>
</html>
```

这里的`viewport`元素有几个关键的属性：

- `width=device-width`：将页面宽度设置为跟随设备屏幕宽度变化。
- `initial-scale=1`：页面最初被显示时的缩放级别。
- `maximum-scale=1`和`minimum-scale=1`：定义了页面可以被缩放的最大和最小程度，这里都设置为 1，意味着页面将不能被放大或缩小。
- `user-scalable=no`：禁止用户通过手势在页面上进行缩放。

虽然这种方法在很多情况下是有效的，但需要注意的是，某些浏览器或设备可能不完全支持这些设置。此外，从可访问性的角度来看，禁止页面缩放可能会对一些需要放大内容以便更好阅读的用户产生不利影响。因此，在决定使用这种方法之前，请仔细考虑是否真的需要限制页面缩放。
