##### vue 中的修饰符：

事件修饰符：stop、prevent、capture、self、once、passive

Vue 中的事件修饰符提供了一种简洁的方式，允许在模板中直接声明事件应该如何被处理，而无需在事件处理函数内部编写逻辑来实现这些常见的 JavaScript 功能。这些修饰符可以单独使用，也可以组合使用。下面是 Vue 中事件修饰符的作用说明：

### `.stop`

- **作用**：调用`event.stopPropagation()`，阻止事件冒泡。
- **应用场景**：当你不希望事件继续传递给父元素时使用。

### `.prevent`

- **作用**：调用`event.preventDefault()`，阻止事件的默认行为。
- **应用场景**：常用于表单提交事件中，防止页面重新加载；或者链接的点击事件中，阻止跳转。

### `.capture`

- **作用**：添加事件监听器时使用事件捕获模式，即内部元素触发的事件会先在此处处理，然后才是冒泡阶段。
- **应用场景**：当你想在事件到达目标元素前先捕获它时使用。

### `.self`

- **作用**：只当在 event.target 是当前元素自身时触发处理函数，即事件不是从内部元素冒泡上来的。
- **应用场景**：用于限制事件处理器只响应由元素本身触发的事件。

### `.once`

- **作用**：事件将只会触发一次。
- **应用场景**：当你想要事件处理器只执行一次时，如一些性能开销大的处理函数。

### `.passive`

- **作用**：以`{ passive: true }`模式添加事件监听器。这意味着你不能在处理函数中调用`event.preventDefault()`。这个修饰符主要用来提高滚动性能。
- **应用场景**：常用于滚动事件的监听器（`scroll`），尤其是移动端 Web 应用，可以提高滑动的流畅性。

这些修饰符可以很方便地控制事件的行为，减少了在事件处理函数中手动调用`preventDefault()`、`stopPropagation()`等方法的需要，让模板的意图更加明确，代码也更清晰。

鼠标修饰符：left、right、middle
sync

Vue 2.3.0+ 引入了 `.sync` 修饰符，这是一个语法糖，用于在父子组件之间双向绑定一个 prop。它简化了子组件改变由父组件传递的 prop 值，并通知父组件更新该值的流程。

### 如何工作

在没有`.sync`修饰符的情况下，为了实现类似的双向绑定效果，子组件需要：

1. 接收一个 prop。
2. 监听该 prop 的变化。
3. 当需要改变该 prop 的值时，子组件通过事件通知父组件，然后由父组件修改这个 prop 的值。

使用`.sync`修饰符后，这个过程被简化了。当子组件需要更新这个 prop 的值时，它可以直接触发一个更新事件（默认为`update:myPropName`），Vue 会自动更新父组件中相应的 prop。

### 示例

假设有一个`<child-component>`，它有一个`title` prop。

#### 父组件

在父组件中，你可以这样使用`.sync`修饰符：

```html
<child-component :title.sync="pageTitle"></child-component>
```

#### 子组件

当子组件需要更新`title`的值时，它会这样做：

```javascript
this.$emit("update:title", newTitle);
```

这个`$emit`调用触发了一个事件，Vue 监听这个事件，并相应地更新`pageTitle`的值为`newTitle`。这样，父组件中的`pageTitle`和子组件中的`title` prop 就被同步了。

### 使用场景

`.sync`修饰符适合于那些需要双向绑定的场景，特别是在开发可重用的组件时。它使得父组件和子组件之间的数据流更加直观，易于管理。不过，应当注意，过度使用双向绑定可能会使应用的数据流变得难以追踪和理解，特别是在大型应用中，因此推荐在确实需要时才使用`.sync`修饰符。

### 注意

Vue 3 中，`.sync`修饰符被移除，推荐使用`v-model`来替代在 Vue 2 中`.sync`的用法，`v-model`在 Vue 3 中被增强，支持对组件 props 的双向绑定。

表单修饰符：lazy、number、trim
