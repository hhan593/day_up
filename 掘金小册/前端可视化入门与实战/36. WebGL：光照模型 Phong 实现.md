## 相关概念

### 法向量

法线（Normal）是指在三维空间中与曲面**垂直**的向量。

法向量是指与曲面上某一点的法线方向相同的向量。它是通过计算曲面的切线向量并进行**归一化**处理得到的，也就是**单位向量**。

它们用于计算光照模型中的漫反射和镜面反射分量，以及进行阴影计算和几何处理等任务。通过使用法线和法向量，可以实现更真实和逼真的渲染效果。


### 单位向量

单位向量：指其长度（或称为模）为 1 的向量。

normalize 就是归一化将向量转为单位向量的方法。

这里我们依照 three.js 的源码中封装的代码理解：

```js
length() {
    return Math.sqrt( this.x * this.x + this.y * this.y + this.z * this.z );
}
normalize() {
    return this.divideScalar( this.length() || 1 );
}
divideScalar( scalar ) {
    return this.multiplyScalar( 1 / scalar );
}

multiplyScalar( scalar ) {
    this.x *= scalar;
    this.y *= scalar;
    this.z *= scalar;
    return this;
}
```

上面源码的理解困难主要在 length 这个数学方法，**其实它是欧几里得范数（Euclidean norm），也称为 L2 范数（L2 norm）或向量的模（magnitude），用于求出向量的长度**。

欧几里得范数可以通过以下公式计算`：||v|| = √(x² + y² + z²)`，其中 v 是当前向量。因此，length() 方法计算当前向量的每个分量的平方之和的平方根，以确定它的长度。

那么我们可以假设长度为 0.5，那么 multiplyScalar 中传入的 scalar 就为 2，对向量相乘也就是将原有向量的长度增加了一倍，那么就是 1 的长度。

一般法线有个问题，比如 x 轴缩放、y 轴缩放等这个时候意味着物体不是等比例的缩放，那么计算光线的法向量这个时候长度和方向也会对应改变，很显然是不符合单位向量，同时方向也是不应该变化的而是垂直的。

这个时候我们就需要通过模型视图矩阵的逆矩阵并转置来解决这个问题。

```js
mat4.invert(normalMatrix, mvMatrix);
mat4.transpose(normalMatrix, normalMatrix);
```

在光照计算中，法线向量通常是在模型空间中定义的。当模型进行平移、旋转和缩放等变换时，法线向量也需要相应地进行变换，以保持其正确的方向。为了实现这一点，我们需要使用模型视图矩阵（mvMatrix）来变换法线向量。然而，由于法线向量是一个方向向量，而不是位置向量，所以在进行变换时，不需要进行平移变换。

因此，在进行法线变换时，我们只关心旋转和缩放部分。这就是为什么我们使用模型视图矩阵的逆矩阵（invert）来进行法线的变换。

然后，为了确保变换后的法线向量仍然是垂直于曲面的，我们需要对逆矩阵进行转置操作。这是因为逆矩阵的转置矩阵实际上是原始矩阵的转置矩阵的逆矩阵。通过转置操作，我们可以保持法线向量的方向不变，同时确保它们与变换后的顶点位置保持一致。所以，使用 `mat4.invert` 函数获取模型视图矩阵的逆矩阵，然后使用 `mat4.transpose` 函数将逆矩阵的行转为列，从而得到正确的法线矩阵。这样，我们就可以将法线向量正确地应用于光照计算中，以获得正确的光照效果。


 
### transpose 函数

transpose 函数是一个矩阵转置的操作。在线性代数中，矩阵的转置是将矩阵的行和列互换得到的新矩阵。

在 WebGL 中，transpose 函数是 gl-matrix 库中的一个函数，用于将给定的矩阵进行转置操作。它接受一个矩阵作为参数，并返回转置后的矩阵。

转置操作的原理是将矩阵的行和列互换。例如，对于一个 3x3 的矩阵：

```js
a b c
d e f
g h i
```

转置后的矩阵为：

```js
a d g
b e h
c f i
```

通过转置操作，矩阵的行变成了列，列变成了行。

在计算机图形学中，转置操作常用于将模型视图矩阵和投影矩阵进行转置，以便正确地进行矩阵乘法和变换。转置后的矩阵可以更方便地用于计算顶点的变换和光照计算。

> 
> 在数学中，给定两个 n 维向量 A 和 B，它们的点积可以通过将两个向量的对应分量相乘，并将乘积相加得到。点积的计算公式为：dot(A, B) = A1 * B1 + A2 * B2 + ... + An * Bn

## 光照模型分类

WebGL 中常用的光照模型有以下几种：

1. **平面光照模型**：对每个面片进行光照计算，然后在面片上使用相同的颜色。这种模型忽略面片内部的光照变化，使得模型看起来平坦。

2. **Gouraud 着色模型**：对每个顶点进行光照计算，然后在面片内插值得到每个像素的颜色。这种模型可以产生更平滑的光照效果，但在细节较多的模型上可能会出现明显的颜色过渡。
3. **Lambert 漫反射光照模型**：用于计算物体表面的漫反射光照。它是基于兰伯特定律的简化版本，也被称为 Lambertian 反射。 Lambert 光照模型假设光线照射到物体表面后，光线在各个方向上均匀地散射，而不考虑光线的反射方向。它认为物体表面的亮度只取决于入射光线和法线之间的夹角余弦值。夹角越大，光线越弱，物体表面越暗；夹角越小，光线越强，物体表面越亮。就像 three.js 封装的一样也有 Lambert 来模拟环境光。
4. **Phong 光照模型**：在每个像素上进行光照计算，得到更准确的光照效果。它结合了 Gouraud 模型的平滑性和平面模型的计算效率，是目前使用最广泛的光照模型之一。
5. **Blinn-Phong 光照模型**：类似于 Phong 模型，但使用了 Blinn-Phong 的半程向量来计算镜面反射。这种模型在计算上比 Phong 模型更高效，同时能够产生类似的光照效果。

这些光照模型都是基于光照方程和材质属性进行计算的。它们可以通过设置光源的位置、光源的颜色和强度、材质的反射属性等参数来调整光照效果。我们将讲解其中之一的 Phong。



### Phong 介绍

在计算机图形学中，最常用的光照模型是基于 Phong 光照模型的。Phong 光照模型是一种经典的光照模型，它包括环境光、漫反射光和镜面反射光三个成分。

1. **环境光**：指在场景中无处不在的光线，它对物体的每个部分都有相同的影响。环境光使得物体在整体上都有一定的明暗效果。

2. **漫反射光**：指光线照射到物体表面后，根据表面法线的方向，使得光线在各个方向上均匀地散射。漫反射光决定了物体的明暗和阴影效果。可简单理解为物体本身的颜色。

3. **镜面反射光**：指光线照射到物体表面后，根据表面法线的方向，使得光线在某个特定方向上反射。镜面反射光决定了物体的高光效果，即光线直接反射的亮点。

Phong 光照模型通过计算环境光、漫反射光和镜面反射光的组合，来模拟真实世界中的光照效果。

```js
    varying vec3 v_viewDirection;
    void main() {
        gl_Position = u_projectionMatrix * u_modelViewMatrix * a_position;
        v_normal = normalize(u_normalMatrix * a_normal);
        v_viewDirection = normalize(vec3(0.0, 0.0, 1.0) - vec3(u_modelViewMatrix * a_position));
    }
`;
```

gl_Position 是一个最终的坐标，通过法线矩阵和立方体的法线计算归一化得到最终法线向量 v_normal。最后，计算顶点到视点的方向向量，并将其赋值给 `v_viewDirection` 变量。v_normal 和 v_viewDirection 传递给片元着色器。

```js
const fragmentShaderSource = `
    precision mediump float;
    varying vec3 v_normal;
    varying vec3 v_viewDirection;
    void main() {
    // 光线方向
    vec3 lightDirection = normalize(vec3(0.0, 0.0, 1.0));
    // 环境光颜色
    vec3 ambientColor = vec3(0.3, 0.3, 0.3);
    // 漫反射颜色
    vec3 diffuseColor = vec3(.3, .3, .3);
    // 镜面反射颜色
    vec3 specularColor = vec3(.2, .2, .2);
    // 光泽度
    float shininess = 33.;
    // 顶点法线向量进行归一化
    vec3 normal = normalize(v_normal);
    // 计算漫反射
    float diffuse = max(dot(normal, lightDirection), 0.0);
    // 使用了光照方向向量和顶点法线向量的反射得到反射方向向量
    vec3 reflectDirection = reflect(-lightDirection, normal);
    // 视点方向向量归一化
    vec3 viewDirection = normalize(v_viewDirection);
    // 通过反射向量和视点向量得到最终的镜面反射强度
    float specular = pow(max(dot(reflectDirection, viewDirection), 0.0), shininess);
    vec3 finalColor = ambientColor + diffuse * diffuseColor + specular * specularColor;
    gl_FragColor = vec4(finalColor, 1.0);
    }
`;
```

首先定义了光照的方向向量 `lightDirection` ，这里假设光源的方向是从正 Z 轴方向照射。然后定义了环境光的颜色 `ambientColor` 、漫反射光的颜色 `diffuseColor` 和镜面反射光的颜色 `specularColor`。`shininess` 表示镜面反射的光泽度。

最后，根据环境光、漫反射和镜面反射的分量，计算了最终的颜色 `finalColor`。将最终的颜色赋值给 `gl_FragColor` ，这个变量表示片元的最终颜色。

这个时候你会明白原来 three 封装的 MeshPhongMaterial 材质是这么实现的。

## 光源原理

上文的光照方式其实是一个点光源的实现，下面我们简单分析下各个光源的实现原理。

### 平行光原理

平行光表示单个方向的矩阵平面的光，我们一般用于模拟太阳光。

```js
vec3 normal = normalize(v_normal); 
float light = dot(normal, u_reverseLightDirection);
```

使用 `dot` 函数计算法线向量和光线方向向量的点积。 `normal` 是归一化后的法线向量， `u_reverseLightDirection` 是光线方向的反方向向量，通常是平行光的方向向量的负值。点积的结果表示法线向量与光线方向的夹角的余弦值，也可以看作是光照强度的一部分。较大的点积值表示法线向量与光线方向更接近，光照强度更强。


### 点光源原理

点光源我们可以简单理解为我们房间中四周发射光线的射灯。

```js
v_surfaceToLight = u_lightWorldPosition - surfaceWorldPosition;

vec3 surfaceToLightDirection = normalize(v_surfaceToLight); //转为法向量
light = dot(normal, surfaceToLightDirection);
```

`v_surfaceToLight` 是一个从表面世界位置到光源世界位置的向量。这个向量表示从表面到光源的方向和距离。 接下来，使用 `normalize` 函数将 `v_surfaceToLight` 向量归一化，得到一个单位向量 `surfaceToLightDirection` 。这个单位向量表示了从表面到光源的方向。 然后，使用 `dot` 函数计算了法线向量 `normal` 与表面到光源方向向量 `surfaceToLightDirection` 的点积。点积的结果是两个向量之间的夹角的余弦值。这个值表示了光线与表面法线之间的夹角的余弦值，可以用来计算光照强度。


### 聚光灯原理

点光源和聚光灯很像，但是聚光灯更强调**衰减的范围**，会有明显的聚集光束效果，我们可以用来模拟晚上舞会的光束。

```js
  // 计算光源方向
  float dotFromDirection = dot(surfaceToLightDirection, -u_lightDirection);
   // 如果光线在聚光灯范围内 inLight 就为 1，否则为 0
  float inLight = step(u_limit, dotFromDirection);
  // 计算光照强度
  float light = inLight * dot(normal, surfaceToLightDirection);
```

### 烘培贴图

在 three 的高级材质章节我们已经讲解了很多贴图，其实它们就是烘焙贴图。

实际大部分 3D 游戏中，真实的物理场景，光线会不断二次反弹，灯光的计算是非常耗时的，尤其是白天的效果，相反夜晚更容易模拟真实，而我们可以将这些计算结果提前通过烘培贴图的方式加载。

烘培贴图（Baked Texture）是指在计算机图形学中，通过一些预先计算的过程将光照信息或其他场景信息嵌入到纹理中，然后将这个贴图应用到物体表面，以模拟光照效果。这个过程通常是在建模软件或者专用的烘培工具中进行的。

就像上图 blender 中建模的物体烘培出的法线贴图，我们可以将贴图渲染在平面快速模拟出特殊纹理。

常见的烘培贴图包括：

1.  **光照贴图（Lightmap）**：储存场景中的光照信息，可以包括直接光照、间接光照等。这些信息在烘培时被计算并嵌入到纹理中。
1.  **环境贴图（Environment Map）**：储存场景的环境信息，例如天空盒、周围的景物等。用于模拟环境光照。
1.  **阴影贴图（Shadow Map）**：储存场景中物体对光源产生的阴影信息。通过烘培计算阴影，可以减少实时渲染中的阴影计算。

使用烘培贴图的好处包括提高渲染性能、降低实时计算的负担，但也有限制，因为烘培贴图是静态的，不适用于动态场景或动态光照变化。



## 总结

Phong 光照模型是一种用于模拟光照效果的经典模型，通常分为环境光照（Ambient）、漫反射光照（Diffuse）和镜面光照（Specular）三个分量。

理解 Phong，我们只需要理解下面核心的片元着色器的代码：

```js
precision mediump float;

uniform vec3 u_lightDirection; // 光照方向
uniform vec3 u_lightColor;     // 光照颜色
uniform vec3 u_ambientColor;   // 环境光颜色
uniform vec3 u_diffuseColor;   // 漫反射颜色
uniform vec3 u_specularColor;  // 镜面光颜色
uniform float u_shininess;     // 镜面反射指数

varying vec3 v_normal;
varying vec3 v_position;

void main() {
    vec3 normal = normalize(v_normal);
    vec3 lightDirection = normalize(u_lightDirection);
    vec3 viewDirection = normalize(-v_position); // 视点方向

    // 计算漫反射光照
    float diffuse = max(dot(normal, lightDirection), 0.0);
    vec3 diffuseLight = u_diffuseColor * u_lightColor * diffuse;

    // 计算镜面光照
    vec3 reflectionDirection = reflect(-lightDirection, normal);
    float specular = pow(max(dot(reflectionDirection, viewDirection), 0.0), u_shininess);
    vec3 specularLight = u_specularColor * u_lightColor * specular;

    // 计算环境光照
    vec3 ambientLight = u_ambientColor * u_lightColor;

    // 组合所有光照分量
    vec3 finalColor = ambientLight + diffuseLight + specularLight;

    gl_FragColor = vec4(finalColor, 1.0);
}
```
