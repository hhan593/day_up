
如果想要深入理解 three.js，WebGL 是绕不过去的门槛。

WebGL 其实是在 Web 端的 JS 的图形标准接口，使用 GLSL 着色器语言执行渲染。所以，**WebGL 的核心在于着色器语言 GLSL**，该语言特性类似于 C 语言。接下来我们来深入学习着色器及其语法特性。



## 着色器发展

着色器语言 GLSL 的发展最早从 1992 年开始，然后慢慢发展适应新的时代需求，为了兼容移动端、web 端等，分割出了 OpenGL ES，然后到 OpenGL ES 2.0、3.0 的规范。最后在 2011 年才正式适应了 Web 端，也就是我们今天要学习的 WebGL 的着色器语言 GLSL。

OpenGL ES 为了适应 Web 端对 OpenGL 做了很多优化和删减，所以本质是 **OpenGL 的阉割版**。但是慢慢随着新时代的显卡发展，比如光线追踪、各种高特性。相比 OpenGL 一些老旧包袱的 API 已经无法适应新的时代，而谷歌开发的 **WebGPU** 在浏览器端做出了新的挑战和性能的提升，在后续的章节会详述。

学好 WebGL，首先要学好 GLSL。GLSL 的着色器分为顶点着色器和片元着色器（也称为片段着色器），分别对应坐标点和颜色。



## 片元着色器起步

接下来我们先从片元着色器入手：

```js
#ifdef GL_ES
precision mediump float;
#endif

uniform float u_time;

void main(){
    gl_FragColor=vec4(0.,0.,1.,1.);
}
```

我们可以在 vscode 中新建一个 `index.glsl` 文件，并复制粘贴上面的代码，接着安装 `glsl lint`、`glsl-canvas` 等插件，ctrl + shift + P 输入 `Show glslCanvas` 我们就可以实时预览显示效果了。

### 结构 Structures

在 GLSL 中，结构是一种自定义的数据类型，包含多个不同类型的成员变量，可以通过结构类型的名称和成员变量的名称来访问结构中的数据。

```js
struct Person {
     int age;
     float height;
     vec3 position;
   };

void main() {
     Person person;
     person.age = 25;
     person.height = 1.75;
     person.position = vec3(0.0, 0.0, 0.0);
     // 使用person结构中的数据进行操作
     // ...
}
```

Struct 初始化：

```js
struct light {
    float intensity;
    vec3 position;
};
light lightVar = light(3.0, vec3(1.0, 2.0, 3.0));
```

初始化结构体时，参数的顺序和类型必须与结构体定义中成员变量的顺序和类型相匹配。



### 数组 Array

在 GLSL 中，数组是一种可以存储**多个相同类型**的元素的数据结构。

数组使用方括号 `[]` 来定义，并在方括号内指定数组的大小。数组的索引从 0 开始，可以使用索引来访问和修改数组中的元素。

```js
float[5] numbers; // 定义一个包含5个浮点数的数组

void main() {
 numbers[0] = 1.0;
 numbers[1] = 2.0;
 numbers[2] = 3.0;
 // ...
}
```

同时，我们可以用类似对象的结构类型和数组创建一个复杂的数据结构。

```js
struct Point {
     float x;
     float y;
};

Point[3] points; // 定义一个包含3个点的结构数组

void main() {
 points[0].x = 1.0;
 points[0].y = 2.0;
 // ...
}
```

注意：glsl 只支持 1 维数组，数组可以是结构体的成员。



### 向量 Vector

向量是一种用于表示多个数值的数据类型，它可以存储`位置、颜色、方向`等。

GLSL 提供了不同维度的向量类型，包括 2 维、3 维和 4 维向量。每个维度都对应一个浮点数（float）值。

```js
vec4 position = vec4(1.0,2.0,3.0,1.0)
```

这是一个简单的四维向量的示例，但是我们如何访问某个分量的值呢？首先每个分量可以有多个不同固定的属性访问，在 glsl 中已经定义了空间坐标（x, y, z, w）、颜色坐标（r, g, b, a）、纹理坐标 （s, t, p, q）。

```js
float a = position.x  // 1.0
float a = position.b  // 3.0

vec2 xy = position.xy // vec2(1.0, 2.0)
vec3 xy1 = vec(position[0], position[1], position[3]) // vec3(1.0, 2.0, 3.0)
```


### 计算规则

接下来讲解我们开发中最主要的基本类型、向量、矩阵的相互计算。

#### 基本类型间运算

由于在 glsl 中没有像 js 一样的隐式类型转换规则，比如：

```js
int a = 2.0
```

浮点数类型是不能直接赋予左侧的整数类型，所以我们需要手动类型转换：

```js
int a = int(2.0)
float b = float(2)
```

#### 浮点数与向量的计算

```js
// 浮点数与向量的计算
vec3 result = vec3(1.0, 2.0, 3.0) + 2.0 // vec3(3.0, 4.0, 5.0)
```

#### 向量与向量的计算

```js
// 浮点数与向量的计算
vec3 result = vec3(1.0, 2.0, 3.0) + 2.0 // vec3(3.0, 4.0, 5.0)
// 向量与向量的计算
vec3 result = vec3(1.0, 2.0, 8.0) - vec3(4.0, 5.0, 6.0)  // vec3(-3.0, -3.0, 2.0)
```

以上都是简单的逐分量运算，根据对应位置做计算即可。但是向量和另一个向量必须保持阶数（维度）相同。

> **逐分量运算是指对两个向量或矩阵的对应元素进行逐个操作的运算**。逐分量运算可以是加法、减法、乘法或除法等。
> 
> 在逐分量运算中，对应位置的元素进行相同的运算操作，得到一个新的向量或矩阵。逐分量运算常用于对图像、颜色或向量进行处理，例如调整亮度、对比度或颜色校正等。

#### 向量与矩阵的计算

向量和矩阵的计算**只存在乘法运算**，同时要保证**阶数相同**，向量乘以矩阵通过**矩阵乘法**得到新的向量。

```js
vec2 v = vec2(10., 20.);
mat2 m = mat2(1., 2.,  3., 4.);
vec2 w = v * m   // = vec2(1. * 10. + 3. * 20., 2. * 10. + 4. * 20.)
```

#### 矩阵与矩阵的计算

前提保证**阶数相同**，如果乘法就用矩阵相乘，但如果是其他运算将采用逐分量运算。

```js
mat2 a = mat2(1., 2.,  3., 4.);
mat2 b = mat2(10., 20.,  30., 40.);
mat2 c = a * b; //mat2(1.*10.+3.*20.,2.*10.+4.*20.,1.* 30.+3.*40.,2.* 30.+4.*40.);

mat2 d = a+b; //mat2(1.+10.,2.+20.,3.+30.,4.+40);
```

> **矩阵乘法是指两个矩阵相乘的运算**。在矩阵乘法中，两个矩阵的对应元素进行乘法运算，并将结果相加。
> 
> 矩阵乘法的结果是一个新的矩阵，其维度由两个原始矩阵的维度决定。具体而言，如果第一个矩阵的维度为 m×n，第二个矩阵的维度为 n×p，那么它们的乘积矩阵的维度为 m×p。矩阵乘法在计算机图形学、计算机视觉和物理模拟等领域中广泛应用。




### 行主序和列主序

我们之前采用的是行主序来计算，这是符合数学约定的。但是在 GLSL 中，默认情况下使用的是列主序（column-major）的矩阵表示方式。这意味着矩阵的元素按列存储，而不是按行存储。

然而，需要注意的是，GLSL 允许使用 `transpose()` 函数来转置矩阵，从而在需要时可以进行行主序（row-major）的矩阵操作。通过使用 `transpose()` 函数，你可以在需要时将列主序的矩阵转换为行主序，以便进行相应的操作。

所以如果是列主序的话，那么结果就不一样了：

```js
mat2 a = mat2(1., 2.,
         3., 4.)
mat2 b = mat2(10., 20.,
         30., 40.)
         
=>

c = mat2(1.*10. + 3.*30., 1.*20. + 3.*40.,
         2.*10. + 4.*30., 2.*20. + 4.*40.)
   = mat2(70., 100.,
          90., 140.)
          
=>

c = mat2(70., 100.,
          90., 140.)
```



### 变量限定符

在 GLSL 中，变量限定符用于指定变量在着色器中的作用域和存储方式。如 uniform、varing、attribute、in、out、const。


#### uniform

GLSL 中的 uniform 是一种特殊类型的变量限定符，用于在顶点着色器和片段着色器之间传递全局数据。**uniform 变量是只读的，对于每个绘制调用都是常量，其值在渲染过程中保持不变**。

uniform 变量通常用于在着色器中传递应用程序的全局参数，如变换矩阵、光照参数、纹理等。通过将 uniform 变量定义在顶点着色器和片段着色器中，我们可以在不同的着色器阶段共享这些数据。

下面是一个使用 uniform 变量的简单示例。

片段着色器代码：

```js
uniform vec4 uColor;  // 定义一个uniform变量用于传递颜色值

void main() {
    gl_FragColor = uColor;
}
```

通过使用 uniform 变量，我们可以在应用程序中更改这些全局数据的值，并在每个绘制调用中传递给着色器，从而实现动态的渲染效果。


#### attribute

attribute 只存在于顶点着色器中定义顶点属性。它用于传递顶点相关的数据，例如顶点位置、法线、纹理坐标等。

```js
attribute vec3 aPosition;

void main() {
    gl_Position = vec4(aPosition, 1.0);
}
```


#### varying

varying 用于在顶点着色器和片段着色器之间传递插值数据。它允许我们在顶点着色器中计算出的数据在后续的片段着色器中进行插值，以便实现平滑的渲染效果。

在顶点着色器中，我们可以使用 varying 变量来传递需要在片段着色器中进行插值的数据。这些数据可以是法线向量、纹理坐标、颜色等。

下面是一个简单的示例，演示如何使用 varying 变量来传递顶点颜色数据并在片段着色器中进行插值。

顶点着色器代码：

```js
attribute vec3 aPosition;
attribute vec3 aColor;
varying vec3 vColor;

void main() {
    gl_Position = vec4(aPosition, 1.0);
    vColor = aColor;
}
```

片段着色器代码：

```js
varying vec3 vColor;

void main() {
    gl_FragColor = vec4(vColor, 1.0);
}
```

通过使用 varying 变量，我们可以在顶点着色器和片段着色器之间传递需要进行插值的数据，实现平滑的渲染效果。


#### in

in 用于在顶点着色器和片段着色器中定义输入变量，例如顶点着色器中接收的顶点属性数据。

我们可以使用 in 变量来接收从前一阶段传递过来的数据，并进行相应的处理。

```js
in vec3 inPosition;

void main() {
    gl_Position = vec4(inPosition, 1.0);
}
```

#### out

out 用于在顶点着色器和片段着色器中定义输出变量。它用于将数据从顶点着色器传递到后续的阶段。

顶点着色器代码：

```js
in vec3 inPosition;
in vec3 inColor;
out vec3 outColor;

void main() {
    gl_Position = vec4(inPosition, 1.0);
    outColor = inColor;
}
```

片元着色器代码：

```js
in vec3 outColor;
void main() {
    gl_FragColor = vec4(outColor, 1.0);
}
```


#### const

使用 const 关键字，我们可以在着色器中定义一个在编译时确定且不可修改的常量值。

```js
const float PI = 3.14159;

void main() {
    float radius = 2.0;
    float circumference = 2.0 * PI * radius;
    // 其他计算和操作
}
```



### 函数

GLSL 会默认先执行主函数 main，主要需要注意函数返回值的类型定义在函数名的前面。

```js
float square(float x) {
    return x * x;
}

void main() {
    float result = square(3.0);
}
```


GLSL 函数库包括数学函数、向量和矩阵函数、纹理函数等，可以在顶点着色器和片段着色器中使用。

1. **数学函数库**：GLSL 提供了许多数学函数，用于执行常见的数学操作，例如三角函数（sin、cos、tan）、指数函数（exp、log）、幂函数（pow、sqrt）、取整函数（floor、ceil）等。这些函数可以用于处理顶点位置、颜色、纹理坐标等数据。

2. **向量和矩阵函数库**：GLSL 提供了一系列用于处理向量和矩阵的函数。这些函数包括向量的归一化（normalize）、点积（dot）、叉积（cross）、矩阵乘法（matrix multiplication）、转置（transpose）等。这些函数在进行向量和矩阵运算时非常有用，例如变换顶点位置、计算光照等。

3. **纹理函数库**：GLSL 提供了一些用于处理纹理数据的函数，例如采样纹理（texture sampling）、纹理过滤（texture filtering）等。这些函数可以用于在片段着色器中对纹理进行采样和处理，实现纹理映射和纹理效果。

4. **几何函数库**：GLSL 还提供了一些用于处理几何图形的函数，例如计算法线（normal calculation）、计算切线（tangent calculation）等。这些函数在进行几何计算和渲染时非常有用。

通过使用这些内置函数库，我们可以在 GLSL 中进行各种数学和图形操作，实现各种复杂的渲染效果和计算。当然，我们也可以通过 GLSL 函数来自定义封装工具库。


### 精度限定

GLSL 的精度限定符用于控制浮点数的精度。它们可以用于声明浮点数变量的精度级别，以便在着色器中控制计算的精度和性能。

在 GLSL 中，有三种精度限定符可用：

1.  `highp` ：表示高精度。这是最高精度级别，适用于需要高精度计算的情况，但可能会牺牲一些性能。

2.  `mediump` ：表示中等精度。这是默认的精度级别，适用于大多数情况下，提供了较好的精度和性能平衡。

3.  `lowp` ：表示低精度。这是最低精度级别，适用于对精度要求不高的情况，可以提供更高的性能。
 
```bash
highp float highPrecisionValue;
mediump float mediumPrecisionValue;
lowp float lowPrecisionValue;
```

精度限定符的实际效果可能会受到硬件和驱动程序的影响，在不同的平台上可能会有所差异。因此，在使用精度限定符时，应该仔细考虑所需的精度和性能要求。


### 预编译指令

GLSL 的预编译指令（Preprocessor Directive）是一些特殊的指令，用于在编译阶段对着色器代码进行预处理。它们以 `#` 符号开头，并在编译之前对代码进行处理。

下面是一些常见的 GLSL 预编译指令。

1.  `#version` ：用于指定 GLSL 的版本号。例如， `#version 330 core` 表示使用 OpenGL 3.3 的核心着色器语言版本。

2.  `#ifdef`  和  `#ifndef` ：用于条件编译。 `#ifdef` 用于检查某个宏是否已经定义，`#ifndef` 用于检查某个宏是否未定义。例如：

```js
#ifdef USE_TEXTURE
        // 如果USE_TEXTURE宏已定义，则执行这段代码
        // ...
#endif
```
3.  `#define` ：用于定义宏。宏是一种标识符，可以在代码中用来代替一段常量或代码片段。例如：

```js
#define PI 3.14159
```
4.  `#undef` ：用于取消宏的定义。例如：

```js
#undef PI
```
5.  `#include` ：用于包含其他文件的内容。例如：

```js
#include "common.glsl"
```

这些预编译指令可以用于控制编译过程中的条件分支、宏定义和代码复用。它们在 GLSL 中起到了很重要的作用，使得代码的编写和维护更加灵活和高效。



### 宏

在 GLSL 中，宏（Macro）是一种预处理指令，用于在编译之前替换代码中的标识符。宏可以用来定义常量、函数、代码片段等，以便在着色器中进行代码复用和简化。

GLSL 中的宏使用 `#define` 指令来定义，它将一个标识符和一个值或一段代码关联起来。当编译器**遇到宏的引用**时，它会将宏的定义替换为相应的值或代码。

下面是一个示例，演示如何使用宏来定义常量和简化代码：

```js
#define PI 3.14159
#define DOUBLE(x) (2.0 * (x))

void main() {
    float radius = 2.0;
    float circumference = DOUBLE(PI) * radius;
    // 其他代码
}
```

除此之外还有一些常见的内置宏：

1.  `GL_ES` ：如果代码是在 OpenGL ES 上运行，则定义为 1。

2.  `GL_FRAGMENT_PRECISION_HIGH` ：如果片段着色器支持高精度浮点数计算，则定义为 1。

3.  `GL_FRAGMENT_SHADER` ：表示当前着色器是片段着色器。

4.  `GL_VERTEX_SHADER` ：表示当前着色器是顶点着色器。

5.  `GL_MAX_VERTEX_ATTRIBS` ：表示顶点着色器中可用的最大顶点属性数量。

6.  `GL_MAX_VERTEX_UNIFORM_VECTORS` ：表示顶点着色器中可用的最大 uniform 向量数量。

7.  `GL_MAX_VARYING_VECTORS` ：表示顶点着色器和片段着色器之间可用的最大 varying 向量数量。

8.  `GL_MAX_VERTEX_TEXTURE_IMAGE_UNITS` ：表示顶点着色器中可用的最大纹理图像单元数量。

### 内置常量

GLSL 中有一些内置的常量，它们是预定义的常量值，可以在着色器中直接使用。以下是一些常见的内置常量：

1.  `gl_MaxVertexAttribs` ：表示顶点着色器中可用的最大顶点属性数量。

2.  `gl_MaxVertexUniformVectors` ：表示顶点着色器中可用的最大 uniform 向量数量。

3.  `gl_MaxVaryingVectors` ：表示可在顶点着色器和片段着色器之间传递的最大 varying 向量数量。

4.  `gl_MaxVertexTextureImageUnits` ：表示顶点着色器中可用的最大纹理图像单元数量。

5.  `gl_MaxCombinedTextureImageUnits` ：表示可在顶点着色器和片段着色器中使用的最大纹理图像单元数量的总和。

6.  `gl_MaxTextureImageUnits` ：表示片段着色器中可用的最大纹理图像单元数量。

7.  `gl_MaxDrawBuffers` ：表示可同时写入的最大绘制缓冲区数量。



### 流控制的 discard

在 GLSL 中， `discard` 是一个关键字，用于在片段着色器中丢弃当前片段的处理，即提前终止片段的渲染。当满足某些条件时，我们可以使用 `discard` 关键字来跳过当前片段的处理，从而达到一些特定的效果。

以下是一个简单的示例：

```js
void main() {
    if (someCondition) {
        discard;  // 如果满足条件，丢弃当前片段的处理
    }
}
```

在这个示例中，如果满足 `someCondition` 条件， `discard` 关键字将会丢弃当前片段的处理，不会对其进行任何操作。这样可以实现一些特定的效果，例如根据某些条件来裁剪或隐藏特定的片段。

使用 `discard` 关键字会导致片段被丢弃，不会对深度缓冲区进行更新。这可能会导致一些深度相关的渲染问题。因此，在使用 `discard` 关键字时，需要谨慎考虑其影响，并确保在需要丢弃片段时进行适当的处理。



### 在 GLSL 中为什么变量的前缀都是 `a_`、 `u_` 或 `v_` ？

1. a_ 前缀：属性（Attribute），用于表示从顶点数据传递到顶点着色器的输入变量。这些属性通常与顶点缓冲对象中的数据关联，例如顶点位置、颜色、法线等。 
2. u_ 前缀：统一变量（Uniform），用于表示在整个渲染过程中保持不变的全局变量。这些变量在渲染管线的各个阶段中都可以访问，例如投影矩阵、视图矩阵、光照参数等。 
3. v_ 前缀：变量（Varying），用于表示从顶点着色器传递到片段着色器的插值变量。

这些前缀的使用是一种命名约定，有助于更清晰地理解和组织着色器代码。
