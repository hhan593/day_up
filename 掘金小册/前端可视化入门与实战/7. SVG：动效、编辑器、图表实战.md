我们在 `btn__progress` 下定义了三个元素，`btn__progress-track` 代表圆环的背景，我们设置了描边样式和填充样式，`btn__progress-fill` 代表填充的环形，同时我们定义 stroke-dasharray 和 stroke-dashoffset 为描边动画做准备。

> `stroke-dasharray` 和 `stroke-dashoffset` 是 SVG 中的两个属性，用于创建描边的虚线效果。 
> - `stroke-dasharray` 属性用于定义虚线的样式。它接收一个由数字组成的列表，用逗号或空格分隔。每个数字表示一个实线段的长度，后面紧跟着一个空白段的长度。列表中的值将循环应用于整个描边。例如， `stroke-dasharray="5,2"` 表示每个实线段的长度为 5 个单位，每个空白段的长度为 2 个单位。 
> 
> - `stroke-dashoffset` 属性用于调整虚线的起始位置。它定义了虚线模式的偏移量，即虚线从路径的起点开始的位置。较大的值将使虚线向路径的起点方向移动，较小的值将使虚线向路径的终点方向移动。 
> 
> 这两个属性通常一起使用，可以创建出各种不同的虚线效果。通过调整 `stroke-dasharray` 中实线段和空白段的长度以及 `stroke-dashoffset` 的值，可以实现不同的虚线样式和动画效果。

> **`技巧说明`**：在代码第 17 行，我们在全局定义了 hue 变量用于 HSL 颜色模型。 
> 
> HSL 是根据颜色的色相（Hue）、饱和度（Saturation）和亮度（Lightness）来进行定义的。
> - 色相（Hue）：表示颜色的基本属性，用角度值（0～360 度）表示，对应于不同的颜色。例如，红色对应的色相值为 0 度，绿色对应的色相值为 120 度，蓝色对应的色相值为 240 度。 
> - 饱和度（Saturation）：表示颜色的鲜艳程度，取值范围为 0% 到 100%。值为 0% 时，颜色变为灰色；值为 100% 时，颜色最鲜艳。 
> - 亮度（Lightness）：表示颜色的明暗程度，取值范围为 0% 到 100%。值为 0% 时，颜色最暗；值为 100% 时，颜色最亮。  
> 
> **为什么要用 HSL 颜色模型呢？** 
> 
> HSL 可以让我们很方便定义颜色规范，或者说色调基础。对于色调来说最重要的是色相，所以我们用 hue 来控制色相，其他的颜色我们可以用饱和度和亮度来调控。所以，以后一旦主题色变了，我们只需要修改色相值，其他的细节颜色样式都会同步变化，而不用费心费力调整大量的基础颜色变量。


这里需要注意延迟时间的衔接，进度条展示动画、宽度动画一共 0.6s 之后才正式开始进度条的运行动画并持续 2s，也就是 2.6s 后就是完成状态，此时需要展示成功动画。但是此时是有进度条结束动画 0.3s，所以我们要延迟 0.3s，然后持续 0.3s 的成功动画。

大功告成，接下来我们书写下 js 代码逻辑：


```js
btn && btn.addEventListener("click", function () {
    const runClass = "btn--running";
    const doneClass = "btn--done";

    const submitDuration = 2000;
    const resetDuration = 1500; 

    this.disabled = true;

    this.classList.add(runClass);

    clearTimeout(doneTimeout);
    clearTimeout(resetTimeout);

    doneTimeout = setTimeout(() => {
        this.classList.remove(runClass);
        this.classList.add(doneClass);

        // 重置
        resetTimeout = setTimeout(() => {
            this.disabled = false;
            this.classList.remove(doneClass);
        }, resetDuration);

    }, 600 + submitDuration);
});
```

第一个定时器代表进度条完成的状态，刚刚好是 2.6s，我们动态添加了 btn-done 的完成状态，重置动画倒计时其实 0.6s 就结束了，0.6s 由进度条的完成 0.3s 和成功动画的 0.3s 组成。为了在虚线动画走完时有个停留的效果，我们故意延长到了 1.5s。

### 完善按钮交互体验：涟漪动画

我们发现在点击按钮的时候是有点生硬的，尤其是长按的时候是没有视觉反馈的。接下来我们将用 CSS 来实现复杂的涟漪动画：


```js
function handleClick(e) {
    const { layerX, layerY } = e;
    const { width, height } = this.getBoundingClientRect();

    this.style.setProperty("--top", `${(layerY / height) * 100}%`);
    this.style.setProperty("--left", `${(layerX / width) * 100}%`);

    const dx = layerX > width / 2 ? layerX : width - layerX;
    const dy = layerY > height / 2 ? layerY : height - layerY;
    const size = Math.sqrt(dx ** 2 + dy ** 2) * 2;
    this.style.setProperty("--size", `${size}px`);
}

btn.addEventListener("mousedown", handleClick);
```

实现思路：我们监听鼠标下按事件，通过 layerX、layerY 来得到相对位置；接下来通过 `getBoundingClientRect` 来得到按钮的宽高，就可以计算出点击位置的百分比，百分比的好处就是可以适配；接着我们需要根据点击位置离按钮中心位置距离来判断涟漪的初始大小，通过简单的勾股定理计算出涟漪的宽高。

下面我们处理 CSS 样式：

```css

button:after {
```

## 纹理可视化编辑器

这里有必要提一嘴，SVG 和 CSS 的过滤器都提供了图形和视觉效果处理的功能，但它们的语法和应用方式有所不同。具体使用哪种过滤器取决于你的需求和使用环境。

SVG 过滤器提供了一系列的效果，如颜色矩阵、高斯模糊、阴影等。它还支持自定义过滤器效果，相比 CSS 过滤器功能更强大。这个功能是非常强大的，我们可以设计出大量体积小并且令人惊讶的纹理图案。

### 基础噪声纹理

在上面图一效果中，我们通过 baseFrequency 来定义 x 和 y 轴向的拉伸效果，值范围在 0.001～1 之间，值越小特征越明显。类型 turbulence 最直接区别就是显示效果更清晰，numOctaves 一般在 1～5 之间，用于处理噪声的频率。seed 种子代表不同的噪声实例。


### 花岗岩纹理

在上面图二效果中，我们应用了`颜色矩阵`，主要用于对图像进行颜色转换和调整。这个矩阵是一个 4x5 的矩阵，每行包含 5 个值，用空格分隔。每个值代表一个颜色通道（红、绿、蓝、透明度和偏移量）的转换系数。

具体来说，这个矩阵的作用是对图像进行颜色转换，使图像变暗并增加一些颜色偏移。

下面是矩阵中每个值的含义： 

- 第一行：红色通道转换系数。这里的值为 `0 0 0 9 -4` ，表示将红色通道的值乘以 0，不对绿色和蓝色通道进行转换，将透明度乘以 9，再减去 4。 
- 第二行：绿色通道转换系数。与红色通道类似，这里的值也为 `0 0 0 9 -4` ，表示对绿色通道进行相同的转换。 
- 第三行：蓝色通道转换系数。同样，这里的值为 `0 0 0 9 -4` ，表示对蓝色通道进行相同的转换。 
- 第四行：透明度通道转换系数。这里的值为 `0 0 0 0 1` ，表示不对透明度进行转换，保持原始值为不变。 

通过调整这个矩阵中的值，你可以实现不同的颜色转换效果。例如，增加或减少转换系数可以改变图像的亮度、对比度和饱和度等属性。

### 实现路径动画

我们仔细观察之前的动画效果，进度条会先快速递进然后缓慢下来，我们可以用过渡实现。但是如果要根据这种更新状态实现中间数值的同步，显然不合适，这里我们将用定时器实现。

```js
 // 根据设定的百分比值开始动画效果。 
animateStart() {
    let v = 0;
    let self = this;
    let intervalOne = setInterval(function () {
        let p = +(v / self.value).toFixed(2);
        // 定义速率
        let a = (p < 0.95) ? 2 - (2 * p) : 0.05;
        v += a;
        // 停止状态
        if (v >= +self.value) {
            v = self.value;
            clearInterval(intervalOne);
        }
        self.setValue(v);
    }, 10);
}

```

animateStart 方法通过不断更新当前值并调用 `setValue` 方法，动画将逐渐过渡到目标值，实现动态渲染的效果。


```js
// 根据给定的百分比值更新仪表盘的显示。 
setValue(value) {
    let c = (value / 100) * 360;
    if (c === 360) c = 359.99;
    let xy = this.size / 2 - this.strokeWidth / 2;
    let d = this.describeArc(xy, xy, xy, 180, 180 + c);
    this.slice.setAttribute('d', d);
    let tspanSize = (this.size / 3.5) / 3;
    this.text.innerHTML = Math.floor(value) + '<tspan font-size="' + tspanSize + '" dy="' + -tspanSize * 1.2 + '">%</tspan>';
}
```

在 setValue 中，我们根据给定的百分比值 `value` 计算出对应的角度 `c` 。角度值是根据百分比值与 360 度的比例计算得出的，如果角度值 `c` 等于 360 度，我们将其设为 359.99 度。这是因为 SVG 中的路径描述中，完整的圆弧是无法正确显示的。

接下来，我们计算出圆弧路径的起点和终点的坐标。我们使用 `this.size` 表示仪表盘的尺寸， `this.strokeWidth` 表示路径的宽度。使用 `this.describeArc` 方法，我们根据起点和终点的坐标以及角度值来生成描述圆弧的路径字符串 `d`，将路径字符串 `d` 设置为仪表盘的 `slice` 元素的 `d` 属性，从而更新仪表盘的显示。

这里我们看下 `describeArc` 的实现：

```js
// 根据起始角度和终止角度描述一个弧
describeArc(x, y, radius, startAngle, endAngle) {
    let start = this.polarToCartesian(x, y, radius, endAngle);
    let end = this.polarToCartesian(x, y, radius, startAngle);
    let largeArcFlag = endAngle - startAngle <= 180 ? "0" : "1";
    let d = [
        "M", start.x, start.y,
        "A", radius, radius, 0, largeArcFlag, 0, end.x, end.y
    ].join(" ");
    return d;
}
```

这里作用是根据给定的圆心、半径、起始角度和终止角度，生成描述弧的路径字符串。这里需要注意路径字符串的 d 中的 A 弧线的参数：

- `radius`：表示弧线的半径，它有两个值，分别表示弧线的水平半径和垂直半径。例如，如果 `radius` 的值为 `50 30` ，则表示水平半径为 50，垂直半径为 30。 
- `0`：表示弧线的旋转角度。这个参数一般设置为 0。
-  `largeArcFlag`：表示是否使用大弧标志。它有两个值，分别为 `0` 和 `1`。当终止角度与起始角度之间的弧度小于等于 180 度时，使用小弧标志（ `0` ）；当终止角度与起始角度之间的弧度大于 180 度时，使用大弧标志（ `1` ）。 
-  `sweepFlag`：表示弧线的方向。它有两个值，分别为 `0` 和 `1`。当终止角度大于起始角度时，使用 `1` 表示顺时针方向；当终止角度小于起始角度时，使用 `0` 表示逆时针方向。 
-  `end.x` 和 `end.y`：表示弧线的终点坐标。




### 极坐标与笛卡尔坐标

极坐标和笛卡尔坐标系是两种常用的坐标系统，用于描述平面上的点的位置。 


<p align=center><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/301d16a043a24364b3b9b6596864c388~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=535&h=543&s=316459&e=png&b=a3e1f1" alt="image.png"  width="70%"/></p>

- **极坐标**（Polar Coordinates），使用一个点到原点的距离（半径）和该点与参考方向之间的角度来表示点的位置。在极坐标中，一个点的坐标由两个值组成：极径（r）和极角（θ）。极径表示点到原点的距离，极角表示点与参考方向（通常是正右方向）之间的角度。 

- **笛卡尔坐标系**（Cartesian Coordinates），使用一个点在水平轴（X 轴）和垂直轴（Y 轴）上的投影来表示点的位置。在笛卡尔坐标系中，一个点的坐标由两个值组成：横坐标（X）和纵坐标（Y）。横坐标表示点在水平轴上的位置，纵坐标表示点在垂直轴上的位置。 

这两种坐标系统在不同的应用中有不同的优势。**极坐标适用于描述点相对于中心点的距离和方向**，例如极坐标可用于描述天文学中的星体位置。**笛卡尔坐标系适用于描述点在平面上的精确位置**，例如在几何学和计算机图形学中常用笛卡尔坐标系。 

需要注意的是，极坐标和笛卡尔坐标系是可以相互转换的。通过一些数学公式和转换关系，可以将一个点的极坐标转换为笛卡尔坐标，或将一个点的笛卡尔坐标转换为极坐标。这在进行坐标转换和计算时非常有用。

接下来我们看下 `极坐标转换为笛卡尔坐标` 的方法：

```js
// 将极坐标转换为笛卡尔坐标。
polarToCartesian(centerX, centerY, radius, angleInDegrees) {
    let angleInRadians = (angleInDegrees - 90) * Math.PI / 180.0;
    return {
        x: centerX + (radius * Math.cos(angleInRadians)),
        y: centerY + (radius * Math.sin(angleInRadians))
    };
}
```

这个函数接收四个参数： `centerX` 、 `centerY` 、 `radius` 和 `angleInDegrees`。它将极坐标的中心点坐标 `(centerX, centerY)` 、半径 `radius` 和角度 `angleInDegrees` 转换为笛卡尔坐标系中的点坐标。 

在函数内部，首先将角度 `angleInDegrees` 转换为弧度 `angleInRadians` 。这是因为 JavaScript 中的三角函数（ `Math.cos()` 和 `Math.sin()` ）接收的是弧度值而不是角度值。转换公式是 `(angleInDegrees - 90) * Math.PI / 180.0` 。 

然后，使用转换后的弧度值计算笛卡尔坐标 `(x, y)`。`x` 坐标的计算公式是 `centerX + (radius * Math.cos(angleInRadians))`，表示 x 坐标在极坐标中心点的基础上，沿着半径方向乘以余弦值。同样，`y` 坐标的计算公式是 `centerY + (radius * Math.sin(angleInRadians))`，表示 y 坐标在极坐标中心点的基础上，沿着半径方向乘以正弦值。 

最后，函数返回一个包含转换后的笛卡尔坐标 `(x, y)` 的对象。 

通过调用这个函数，我们将极坐标转换为了笛卡尔坐标，得到了 d 路径字符串，就可以动态渲染了。
