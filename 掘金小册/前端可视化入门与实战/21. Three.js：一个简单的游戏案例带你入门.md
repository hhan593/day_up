本文将通过 `TS + Webpack + Three.js` 的基础入门来实现一个 Only Up 小游戏。目标是完成一个场景的空间模型来慢慢掌握 Three.js，并在开发的过程中不断完善一些新的功能特性。



## Three.js 简介

**Three 是一个基于 WebGL 的开源 JavaScript 库，用于在 Web 浏览器中创建和渲染三维图形**。它提供了丰富的功能和工具，使开发者能够轻松地构建交互式的 3D 场景、动画和效果。

Three 的功能模块组成可以分为如下：

1.  **场景**（Scene）：用于组织和管理 3D 对象的容器。
1.  **渲染器**（Renderer）：使用 WebGL 技术进行硬件加速的图形渲染。渲染器负责将场景和相机中的内容渲染到绘图上下文中。
1.  **相机**（Camera）：决定了场景中哪些部分可见，并控制了视图的投影方式。
1.  **几何体**（Geometry）：定义了 3D 对象的形状和结构。
1.  **材质**（Material）：决定了 3D 对象的外观和如何对光进行反射。
1.  **光源**（Light）：用于照亮场景中的对象。
1.  **动画**（Animation）：用于创建和控制 3D 对象的动画效果。
1.  **控制器**（Controller）：用于交互式控制和操作 3D 场景。

## 创建 View 视图类

```js
import * as THREE from "three";

export default class View {
    private renderer: THREE.WebGLRenderer;
    private scene: THREE.Scene;
    private camera: THREE.PerspectiveCamera;

    constructor(canvasElem: HTMLCanvasElement) {
        // 渲染器
        this.renderer = new THREE.WebGLRenderer({
                canvas: canvasElem,
                antialias: true,
        });
        // 生成相机的45度角度范围、渲染区域宽高比屏幕的宽高比，近裁剪代表相机离最近物体的距离为0.1，最远为100
        this.camera = new THREE.PerspectiveCamera(
          75,
          window.innerWidth / window.innerHeight,
          1,
          1000
        )
        // 设置相机高度
        this.camera.position.y = 10
        // 场景
        this.scene = new THREE.Scene();
        // 加载场景背景
        // this.scene.background = new THREE.TextureLoader().load("./textures/bgnd.png");
       this.scene.background = new THREE.Color(0x6699ff)

        // 适配缩放
        this.onWindowResize(window.innerWidth, window.innerHeight);
	}

    public onWindowResize(vpW: number, vpH: number): void {
        // 确保渲染器的输出和窗口大小匹配
        this.renderer.setSize(vpW, vpH);
        // 更新相机的纵横比
        this.camera.aspect = vpW / vpH;
        // 更新投影矩阵
        this.camera.updateProjectionMatrix();
    }

    // 所有子物体的更新
    public update(secs: number): void {
            this.renderer.render(this.scene, this.camera);
    }
}
```

我们首先创建了一个 WebGL 渲染器实例，并配置了绘图上下文。然后我们创建一个透视相机 `PerspectiveCamera`，生成相机的 45 度角度范围、渲染区域宽高比屏幕的宽高比，近裁剪代表相机离最近物体的距离为 1，最远为 1000 的相机。

我们也可以通过 `this.camera.position.set(0, 0, 15);`来设置相机的位置，注意参数分别对应 x、y、z。z 轴为 15，也就是距离屏幕为 15 个单位。该坐标系的规则遵循右手坐标系。

然后配置高度、生成场景，加载场景图片、适配缩放。

在适配缩放中要考虑更新相机的纵横比，一旦视图区域大小变化，也意味着相机的位置、方向、视场角以及近平面和远平面的距离会变化。这个时候我们需要使用 `camera.updateProjectionMatrix` 更新投影矩阵。

那么，什么是投影矩阵呢？

投影矩阵，我们一般用于将三维场景投影到二维屏幕上。它是一个 4x4 的矩阵，通过将三维坐标转换为二维坐标，实现透视或正交投影的效果。

投影可以分为两种主要类型：`透视投影` 和 `正交投影`。

-   **透视投影**（Perspective Projection）：透视投影模拟了人眼观察远近物体时的透视效果。远处的物体看起来较小，近处的物体看起来较大。简称`近大远小`。
-   **正交投影**（Orthographic Projection）：正交投影保持了物体的透视线都是一个方向，不会因为远近而变化。比如左视图、俯视图等。应用场景例如 CAD 绘图或平面图形的渲染。

说完投影矩阵，我们基础的渲染模板已经构建差不多了，接下来我们开始用 Three 来实现我们的想法。

import * as THREE from 'three'

export default class PointerLockControl {
  controls: PointerLockControls
  // 键盘的移动指令
  moveForward = false
  moveBackward = false
  moveLeft = false
  moveRight = false
  canJump = false
  // 用于控制速度
  prevTime = 0
  // 速度
  velocity = new THREE.Vector3()
  // 方向
  direction = new THREE.Vector3()
  // 射线用于碰撞检测
  raycaster: THREE.Raycaster

  constructor (
    camera: THREE.Camera,
    domElement: HTMLCanvasElement,
    scene: THREE.Scene
  ) {
    this.controls = new PointerLockControls(camera, domElement)
    this.prevTime = performance.now()
    scene.add(this.controls.getObject())
    this.watchEvent()
    this.raycaster = new THREE.Raycaster(
      new THREE.Vector3(),
      new THREE.Vector3(0, -1, 0),
      0,
      10
    )
  }

  watchEvent () {
    document.addEventListener('keydown', this.onKeyDown.bind(this))
    document.addEventListener('keyup', this.onKeyUp.bind(this))
    document.addEventListener('click', () => {
      this.controls.lock()
    })
  }
  onKeyDown (event) {
    switch (event.code) {
      case 'ArrowUp':
      case 'KeyW':
        this.moveForward = true
        break

      case 'ArrowLeft':
      case 'KeyA':
        this.moveLeft = true
        break

      case 'ArrowDown':
      case 'KeyS':
        this.moveBackward = true
        break

      case 'ArrowRight':
      case 'KeyD':
        this.moveRight = true
        break

      case 'Space':
        if (this.canJump === true) this.velocity.y += 350
        this.canJump = false
        break
    }
  }

  onKeyUp (event) {
    switch (event.code) {
      case 'ArrowUp':
      case 'KeyW':
        this.moveForward = false
        break

      case 'ArrowLeft':
      case 'KeyA':
        this.moveLeft = false
        break

      case 'ArrowDown':
      case 'KeyS':

## 监听事件

为了可以控制人物视角，我们需要监听点击事件，并 lock 锁定相机的位置。这个时候鼠标的位置会修改相机的朝向。

```js
document.addEventListener('click', () => {
  this.controls.lock()
})
```

为了通过键盘事件监听对应的处理，我们监听 keydown 按下会一直触发，一旦 keyUp 即为触发结束。为了能够移动，我们需要在 Update 更新中做相机的移动逻辑处理。

```js
  update (secs, { objects }) {
    const time = performance.now()
    const {
      velocity,
      direction,
      moveForward,
      moveBackward,
      moveLeft,
      moveRight,
      controls,
      raycaster
    } = this

    if (controls.isLocked) {
      // 通过射线来处理碰撞检测
      raycaster.ray.origin.copy(controls.getObject().position)
      // 防止相机在底部
      raycaster.ray.origin.y -= 10

      const intersections = raycaster.intersectObjects(objects, false)

      // 是否存在碰撞物体
      const onObject = intersections.length > 0
      console.log(intersections)

      // 根据时间间隔和速度来计算物体的位置和速度变化
      const delta = (time - this.prevTime) / 1000

      // 欧拉积分法来模拟物体运动的速度变化，使用负数可以通过方向来偏移
      velocity.x -= velocity.x * 10.0 * delta
      velocity.z -= velocity.z * 10.0 * delta

      velocity.y -= 9.8 * 100.0 * delta

      // 计算方向
      direction.z = Number(moveForward) - Number(moveBackward)
      direction.x = Number(moveRight) - Number(moveLeft)
      direction.normalize()

      // 根据按键输入方向、时间、系数来计算新的速度
      if (moveForward || moveBackward) velocity.z -= direction.z * 400.0 * delta
      if (moveLeft || moveRight) velocity.x -= direction.x * 400.0 * delta

      if (onObject === true) {
        velocity.y = Math.max(0, velocity.y)
        this.canJump = true
      }
      // 移动
      controls.moveRight(-velocity.x * delta)
      controls.moveForward(-velocity.z * delta)

      controls.getObject().position.y += velocity.y * delta // new behavior
      // 限制最小的 y 值
      if (controls.getObject().position.y < 10) {
        velocity.y = 0
        controls.getObject().position.y = 10

        this.canJump = true
      }
    }
    // 做时间差
    this.prevTime = time
  }
```

如果控制器已锁定（即指针已锁定），则会使用射线进行碰撞检测。射线的起点设置为相机的位置，以避免相机陷入底部。通过射线与物体进行相交检测，并将结果保存在 `intersections` 数组中。判断是否存在碰撞物体，若存在，则将 `onObject` 设置为 `true` 。

然后使用`欧拉积分法`根据时间间隔和速度模拟物体运动的速度变化。最后根据按键输入的方向、时间和系数计算新的速度。如果物体在 Y 轴有碰撞发生，则将垂直速度设置为非负值，此时可跳跃。若相机位置低于 10 个单位，则将垂直速度设置为 0，并将相机位置的 y 坐标设置为 10，同时将 `canJump` 设置为 `true` 。最后更新 `prevTime` 方便后续做时间差。



## 欧拉积分法模拟运动

这里有必要解释下欧拉积分法：

```js
velocity.x -= velocity.x * 10.0 * delta

// 使用欧拉积分法来更新速度的公式
v(n+1) = v(n) - v(n) * c * Δt
```

欧拉积分法用于模拟物体的运动。它基于离散化的时间步长，通过不断迭代来逼近连续的运动过程。`delta` 表示时间间隔的秒数。通过将速度乘以 `delta` ，可以根据时间间隔来更新速度。这里使用了欧拉积分法的一种形式，通过减去速度与自身乘以某个系数和时间间隔的乘积，来模拟速度的变化。使用负数可以通过方向来偏移速度。这样，根据时间间隔和速度的变化，可以计算物体的新位置和速度。

这样我们就可以根据键盘做移动操作了。

## 创建材质

但是为什么立方体是黑色的呢？这里有必要讲一讲 Three 的材质问题。

1.  **MeshBasicMaterial**（基础网格材质）：没有光照效果，只显示基本颜色或贴图。
2.  **MeshLambertMaterial**（兰伯特网格材质）：这种材质对光照非常敏感，可以产生漫反射效果。可以设置基本颜色、贴图、光照颜色等属性。
3.  **MeshPhongMaterial**（冯氏网格材质）：这种材质对光照更加真实，可以设置基本颜色、贴图、光照颜色、镜面高光颜色、反射率等属性。
4.  **MeshStandardMaterial**（标准网格材质）：一种基于物理的材质模型，可以产生更真实的光照效果。它结合了漫反射、高光、环境光、法线贴图和粗糙度等属性。
5.  **MeshPhysicalMaterial**（网格物理材质）：继承自 MeshStandardMaterial，可以实现清漆效果和更真实的玻璃。

同时 MeshStandardMaterial 和 MeshPhysicalMaterial 都是 PBR 材质，PBR 材质它考虑了光照的物理特性，使用基于物理的反射模型，如 Cook-Torrance 或 GGX，来计算光线的反射和散射。它还使用环境光遮蔽、间接光照和全局光照等技术来增强渲染效果。这块内容在后续章节会专门讲解。

由于立方体是 MeshPhysicalMaterial 材质，我们当前还没有引入光照，而 MeshBasicMaterial 材质的地面是不用接受光照就可以显示，而天空背景的加载不受光照影响。

// 渲染器开启阴影

this.plane.receiveShadow = true
