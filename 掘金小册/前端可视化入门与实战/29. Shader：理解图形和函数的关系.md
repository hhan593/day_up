
有了前面的 glsl 的语法基础，我们将通过实际案例来学习 Shader 的图形处理思想。

## 渐变

我们先看下面的代码：

```js
#ifdef GL_ES
precision mediump float;
#endif

uniform vec2 u_resolution;
uniform vec2 u_mouse;
uniform float u_time;

float plot(vec2 st){
    return smoothstep(.02,0.,abs(st.y-st.x));
}

void main(){
    // 坐标归一化
    vec2 st=gl_FragCoord.xy/u_resolution;
    
    vec3 color=vec3(st.x);
    
    color=color+vec3(0.,1.,0.);
    
    gl_FragColor=vec4(color,1.);
}
```

代码使用了条件编译指令 `#ifdef GL_ES`用于检查是否在 OpenGL ES 环境下编译的语句以及中等精度的限制。接下来，定义了由外部程序传递给着色器的 `u_resolution`。

我们继续看这行代码 `vec2 st=gl_FragCoord.xy/u_resolution;`，这行代码的意思是将当前片段的坐标 `gl_FragCoord` 转换为一个在 0.0 到 1.0 范围内的**纹理坐标** `st` 。

> 在计算机图形学中，纹理坐标通常用 u 和 v 表示，代表水平和垂直方向上的坐标。在 GLSL 中，使用 st 作为纹理坐标的命名更为常见。虽然在某些情况下也可以使用 uv 作为纹理坐标的命名，但为了遵循惯例和提高代码的可读性，建议在 GLSL 中使用 st 作为纹理坐标的命名。

`gl_FragCoord` 是一个内置变量，它表示当前片元的屏幕坐标，即片元的像素位置。它是一个二维向量，包含了片元在屏幕上的 x 和 y 坐标。

`u_resolution` 是一个 uniform 变量，它表示渲染目标的分辨率，即屏幕的宽度和高度。它是一个二维向量，包含了渲染目标的宽度和高度。

本质这个操作就是**归一化坐标**。当 glsl 在执行的过程中， gl_FragCoord 会并发处理屏幕的每个像素，越右边的点的像素的 x 值会更大，我们知道 x 的范围是`[0, 1]`。对应的颜色是黑色到白色。

为什么要归一化坐标?

GLSL（OpenGL Shading Language）中的显示范围是 -1 到 1，是因为它使用了标准化设备坐标（Normalized Device Coordinates，NDC）。NDC 会将屏幕的左下角映射为 (-1, -1)，右上角映射为 (1, 1)。这种规范化的范围方便了跨不同硬件和平台的一致性。

对于颜色通道范围 0 到 255，对应 0 到 1，这是因为通常在计算机图形中，颜色通道的范围是 8 位（8-bit），即 0 到 255。然而，在进行图形渲染时，通常需要将颜色值标准化到 0 ~ 1 的范围内，以便与其他计算和处理步骤兼容。

所以，归一化坐标就是为了让屏幕尺寸的坐标映射到 NDC 的可视范围。



### 模拟渐变

处理渐变的核心代码是`color=color+vec3(0.,1.,0.)`，我们希望有一种线性关系来表示这种颜色渐变，我们可以通过一个模拟 x、y 变化的函数来模拟不同的渐变效果。

<p align=center><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e2d7ce6e73834b30b8f17db0a6c02ffc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=480&h=298&s=14888&e=png&b=d5ffd5" alt="image.png"  /></p>

我们接着来让线性关系更明显些，我们手动绘制这个线段：

```js
// 绘制线段
float plot(vec2 st){
    return smoothstep(.02,0.,abs(st.y-st.x));
}

void main(){
    vec2 st=gl_FragCoord.xy/u_resolution;
    
    float y=st.x;
    
    vec3 color=vec3(y);
    
    float pct=plot(st);
    
    color=(1.-pct)*color+pct*vec3(0.,1.,0.);
    
    gl_FragColor=vec4(color,1.);
}
```

plot 函数主要用于绘制线段，该函数传入了屏幕的归一化坐标，下面我们来解析下 glsl 的插值函数 `smoothstep`。

#### 插值函数 smoothstep

```js
float smoothstep(float edge0, float edge1, float x);
```

`smoothstep`  函数的计算过程如下：
1. 如果  `x`  小于  `edge0` ，则返回 0.0。
2. 如果  `x`  大于等于  `edge1` ，则返回 1.0。
3. 如果  `x`  在  `edge0`  和  `edge1`  之间，它会根据  `x`  在这个范围内的相对位置进行插值计算。具体计算方式为将  `x`  归一化到 0.0 到 1.0 之间，然后使用三次方程  `3x^2 - 2x^3`  进行插值。

 `smoothstep`  函数的作用是在两个阈值之间生成一个平滑的过渡值。它常用于图形渲染中，用于实现平滑的颜色过渡、形状过渡等效果。通过调整  `edge0`  和  `edge1`  的值，可以控制过渡的范围和形状。
 
所以 `smoothstep(.02,0.,abs(st.y-st.x));`， st.y - st.x 当是 0 的时候，也就是 y 和 x 相等的时候，此时即斜边位置。通过设置 edge 值限制线段的粗细。

```js
color=(1.-pct)*color+pct*vec3(0.,1.,0.);
```

vec3(0.,1.,0.) 传入的 rgb 通道对应显示的是绿色，但是我们因为 pct 的线性函数控制了显示为`[0.2, 1]`的颜色通道范围。所以最终显示的是黑白渐变。而 `(1.-pct)` 是  为了保持颜色一致。

#### 插值函数 step

`step`  函数是一个用于生成阶跃函数的函数。

```js
float step(float edge, float x);
vec2 step(vec2 edge, vec2 x);
```

 `step`  函数的计算过程如下：
1. 如果  `x`  小于  `edge` ，则返回 0.0。
2. 如果  `x`  大于等于  `edge` ，则返回 1.0。

`step`  函数的作用是将输入值与阈值进行比较，如果输入值大于等于阈值，则返回 1.0，否则返回 0.0。它可以用于创建阶跃效果，例如将一个值分为两个区域，或者用于控制条件的开关。

在绘制图形时， `step`  函数常用于控制颜色或形状的边界，根据输入值与阈值的关系来确定是否绘制或如何绘制。

简单总结：`smoothstep` 用于**过渡**效果， `step` 用于**阶梯**效果。

### 径向渐变

我们首先实现一个径向渐变的效果：

```js
void mainImage(out vec4 fragColor,in vec2 fragCoord)
{
    vec2 uv= fragCoord / iResolution.xy;
    
    // 将uv坐标转为中心点坐标0，0
    uv = uv * 2.0 - 1.0;
       
    // 计算向量的长度
    float d = length(uv);
    
    fragColor=vec4(d, d, d,1.);
}
```

我们得到的 uv 坐标，其实对应的左下角为 0，0 的坐标，此时中心点的坐标为 （0.5，0.5），但是我们将实现一个以中心点做对称的效果，所以中心点的坐标我们必须设置为（0., 0.）。接着我们就可以使用 length 拿到向量的长度，d 的范围为 0 到 1。很显然最外围是白色，内部为黑色。

## 实现三角形动画效果

我们已经有了基础的认知，接下来我们在此基础上做一个三角形动画效果，首先将使用 SDF 符号距离函数来改变其造型。

### 符号距离函数 SDF

SDF 是符号距离函数（Signed Distance Function）的缩写。SDF 是一种用于表示几何形状的函数，它返回从给定点到最近表面的有符号距离。这个距离可以是正数、负数或零，表示点在表面之外、之内或在表面上。

- [Inigo Quilez ：： 计算机图形学、数学、着色器、分形、演示场景等 (iquilezles.org)](https://iquilezles.org/articles/distfunctions2d/)


在上面的网站，我们可以直接使用 SDF 函数来实现符号效果：

<p align=center><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6500df581604a31ab0733c8c6629056~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1062&h=319&s=110283&e=png&b=202020" alt="image.png"  /></p>


```js
float sdEquilateralTriangle( in vec2 p, in float r )
{
    const float k = sqrt(3.0);
    p.x = abs(p.x) - r;
    p.y = p.y + r/k;
    if( p.x+k*p.y>0.0 ) p = vec2(p.x-k*p.y,-k*p.x-p.y)/2.0;
    p.x -= clamp( p.x, -2.0*r, 0.0 );
    return -length(p)*sign(p.y);
}

void mainImage(out vec4 fragColor,in vec2 fragCoord)
{
    vec2 uv= (fragCoord * 2.0 - iResolution.xy) / iResolution.y;
    
    float d = sdEquilateralTriangle(uv, 0.2);
    
    fragColor=vec4(d, d, d,1.);
}
```

```js
    d -= 0.5;
    d = abs(d);
    
    d = smoothstep(0.0, .1, d);

}
```

### 径向重复

实现径向重复，核心在于 sin 正弦函数和 cos 余弦函数的振浮效果实现。

```js
void mainImage(out vec4 fragColor,in vec2 fragCoord)
```

### 反函数模拟霓虹灯衰减效果

反函数（Inverse function）是指与给定函数具有相反操作的函数。如果函数 f 将一个集合 A 的元素映射到另一个集合 B 的元素，那么它的反函数 f^(-1) 将 B 中的元素映射回 A 中的元素。

可以使用反函数来使光源在离光源较远的地方逐渐减弱，从而模拟出霓虹灯效果中的明亮中心和逐渐减弱的边缘。此外，还可以使用反函数来调整颜色的渐变和闪烁效果，以增强霓虹灯效果的真实感。


```js
void mainImage(out vec4 fragColor,in vec2 fragCoord)
{
    vec2 uv= (fragCoord * 2.0 - iResolution.xy) / iResolution.y;
    
    float d = sdEquilateralTriangle(uv, 0.8);
    
    d = sin(d * 8. + iTime) / 8.;
    d = abs(d);
    
    d = smoothstep(0.0, .5, d);
    
    d = 0.02 / d;
    fragColor=vec4(d, d, d,1.);
}
```

### 三角函数实现颜色渐变

具体来说，公式为：color(t) = a + b * cos[2π(c*t+d)]

在这个公式中，t 代表时间，范围从 0 到 1，可以看作是归一化的调色板索引或域。余弦函数以相位 d 和频率 c 振荡，结果通过缩放和偏置 a 和 b 来调整亮度和颜色。

为了实现色相变化，参数 a、b、c 和 d 需要是包含红、绿、蓝三个分量的向量。如果希望确保调色板在 0 到 1 的范围内循环，参数 c 需要是半整数（0.0、0.5、1.0、1.5、2.0 等）。如果担心参数 c 的连续性，可以将其设为整数，这样可以获得无限的连续性。


- [dev.thi.ng/gradients/](http://dev.thi.ng/gradients/)  自定义可视化调配颜色  
- [Inigo Quilez ：计算机图形学、数学、着色器、分形、演示场景等 (iquilezles.org)](https://iquilezles.org/articles/palettes/)

代码演示：


```js
vec3 palette(float t){
    vec3 a=vec3(.5,.5,.5);
    vec3 b=vec3(.5,.5,.5);
    vec3 c=vec3(2.0, 1.0, 0.0);
    vec3 d=vec3(0.50, 0.20, 0.25);
    
    return a+b*cos(6.28318*(c*t+d));
}

float sdEquilateralTriangle( in vec2 p, in float r )
{
    const float k = sqrt(3.0);
    p.x = abs(p.x) - r;
    p.y = p.y + r/k;
    if( p.x+k*p.y>0.0 ) p = vec2(p.x-k*p.y,-k*p.x-p.y)/2.0;
    p.x -= clamp( p.x, -2.0*r, 0.0 );
    return -length(p)*sign(p.y);
}

void mainImage(out vec4 fragColor,in vec2 fragCoord)
{
    vec2 uv= (fragCoord * 2.0 - iResolution.xy) / iResolution.y;
    
    float d = sdEquilateralTriangle(uv, 0.8);
    vec3 col = palette(d);
    
    d = sin(d * 8. + iTime) / 8.;
    d = abs(d);
    
    d = smoothstep(0.0, .5, d);
    
    d = 0.02 / d;
    
    col *= d;
    fragColor=vec4(col,1.);
}
```

### Fract 实现重复

在 GLSL 中，"fract" 是一个内置函数，用于计算一个浮点数的小数部分。它返回一个在 0 到 1 之间的值，表示输入值的小数部分。

例如，如果输入值为 2.7，则 "fract(2.7)" 将返回 0.7。如果输入值为 -1.5，则 "fract(-1.5)" 将返回 0.5。这样，你可以使用 "fract" 函数来处理浮点数的小数部分，以满足特定的计算需求。

我们通过图会更容易理解：

### 叠加

```js
void mainImage(out vec4 fragColor,in vec2 fragCoord)
{
    vec2 uv= (fragCoord * 2.0 - iResolution.xy) / iResolution.y;
    vec2 uv2 = uv;    
    vec3 finalColor = vec3(0.0);
    
    for(float i = 0.0; i < 3.0; i++) {
         uv = fract(uv * 2.) - .5;
        float d = sdEquilateralTriangle(uv, 0.8);
        vec3 col = palette(length(uv2) + iTime * .4);

        d = sin(d * 8. + iTime) / 8.;
        d = abs(d);

        d = smoothstep(0.0, .5, d);

        d = 0.02 / d;

        finalColor += col * d;
    }
    
    fragColor = vec4(finalColor, 1.);
} 
```

通过 for 循环，不断叠加 finalColor 向量，可以生成多个图形的层次渐变。

```js
float sdStar5(in vec2 p,in float r,in float rf)
{
    const vec2 k1=vec2(.809016994375,-.587785252292);
    const vec2 k2=vec2(-k1.x,k1.y);
    p.x=abs(p.x);
    p-=2.*max(dot(k1,p),0.)*k1;
    p-=2.*max(dot(k2,p),0.)*k2;
    p.x=abs(p.x);
    p.y-=r;
    vec2 ba=rf*vec2(-k1.y,k1.x)-vec2(0,1);
    float h=clamp(dot(p,ba)/dot(ba,ba),0.,r);
    return length(p-ba*h)*sign(p.y*ba.x-p.x*ba.y);
}
```
