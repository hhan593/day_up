
现代前端开发日新月异地发展，从以往的 HTML 已经不能满足时代的需求到 Web 端能做的事也越来越多。HTML5 的发展始于 2004 年，至如今已经发展了二十年，大量的新特性出现和各浏览器厂商的兼容，我们可以做出更优秀的视觉化效果，同时随着移动设备的普及，HTML5 也能轻松应用移动场景。

在 HTML5 中，Canvas 提供了一个基于 JavaScript 的 API，开发者可以使用它来绘制 2D 和 3D 图形、创建动画、处理图像等。

接下来，我们将深入学习 HTML5 的 Canvas。

##  Canvas 起步

开发 Canvas 的第一步我们需要获取上下文：

```ts
const canvas = document.getElementById('myCanvas') as HTMLCanvasElement;
const ctx = canvas.getContext('2d') as CanvasRenderingContext2D
canvas.width = 1000
canvas.height = 600
```


对于 Canvas 元素我们需要知道，它有`两个属性` width 和 Height，分别代表 Canvas 的宽和高；它有`三个方法`，分别是 getContext、toDataURL 和 toBlob。

getContext 返回一个绘图上下文对象，用于在 Canvas 上绘制。还有两个不太常见的 API，在下载传输的时候我们会用到：

- `toDataURL`：用于将 Canvas 图像数据转换为数据 URL。它接收一个参数，表示图像的格式（默认为 `image/png` ）。该方法返回一个包含图像数据的 Base64 编码的数据 URL，可以使用这个数据 URL 来显示图像或在 HTML 中嵌入图像。

- `toBlob`：用于将 Canvas 图像数据转换为 Blob 对象。它接收两个参数，第一个参数是一个回调函数，用于接收生成的 Blob 对象，第二个参数是图像的格式（默认为 `image/png` ）。该方法将生成的 Blob 对象作为回调函数的参数传递。

Canvas 上下文容器不仅仅可以获取 2D 的上下文，还可以获取 3D 的上下文，比如 WebGL。

这里有个细节，对于 Canvas 的尺寸默认是 300 * 150，大部分人会想当然地通过 CSS 的 style 来设置尺寸，但是这样就无法缩放适配画布。我们可以通过下面这个案例看出明显的尺寸区别，注意 Canvas 的尺寸分为元素尺寸和画布尺寸，通过 CSS 的修改只能处理元素，对于画布是无效的。

```html
<body>
    <canvas id="myCanvas">
        // 一旦浏览器不兼容canvas，会默认渲染内部结构
        浏览器不支持canvas
    </canvas>
    <script>
        var canvas = document.getElementById("myCanvas");
        var ctx = canvas.getContext("2d");
        // 尺寸一直是固定的
        canvas.width = window.innerWidth
        canvas.height = window.innerHeight
    
        // 浏览器尺寸一旦变化，画布也变化
        canvas.style.width = window.innerWidth
        canvas.style.height = window.innerHeight
        // 设置文本样式
        ctx.font = "30px Arial";
        ctx.fillStyle = "red";

        // 在Canvas上绘制文本
        ctx.fillText("Hello, World!", canvas.width / 2, canvas.height / 2);
    </script>
</body>
```


## 绘图上下文 CanvasRenderingContext2D 对象

这里我简单总结下所有 API 的大致用法（虽琐碎但很重要），具体详细的使用可以参考文档 [canvas API中文网 - Canvas API中文文档首页地图](https://www.canvasapi.cn/) 。

**属性**：
- `fillStyle` ：设置绘制的填充样式。可以是颜色值、渐变对象或图案对象。 
-  `filter` ：设置绘制的滤镜效果。可以使用 CSS 滤镜函数来指定滤镜效果，如模糊、亮度调整等。
-  `font` ：设置绘制文本的字体样式。可以是字体大小和字体族名的组合，如 "16px Arial"。

-  `globalAlpha` ：设置绘制的全局透明度。取值范围为 0（完全透明）到 1（完全不透明）之间。

-  `globalCompositeOperation` ：设置绘制的全局合成操作。用于控制新绘制的图形如何与已有的图形混合。常见的操作包括源覆盖、目标覆盖、叠加等。

-  `lineCap` ：设置线条的端点样式。可以是 "butt"（平直端点，默认值）、"round"（圆形端点）或 "square"（方形端点）。

-  `lineDashOffset` ：设置虚线模式的起始偏移量。用于控制虚线的起始位置。

-  `lineJoin` ：设置线条的连接样式。可以是 "round"（圆角连接）、"bevel"（斜角连接）或 "miter"（尖角连接，默认值）。

-  `lineWidth` ：设置线条的宽度。可以是任意正数值，单位为像素。

-  `miterLimit` ：设置尖角连接的限制。当线条连接为尖角时，通过限制尖角的长度来控制连接的外观。

-  `shadowBlur` ：设置阴影的模糊程度。可以是任意正数值，值越大阴影越模糊。

-  `shadowColor` ：设置阴影的颜色。可以是颜色值，用于指定阴影的颜色。

-  `shadowOffsetX` ：设置阴影在水平方向的偏移量。可以是任意数值，单位为像素。
-  `shadowOffsetY` ：设置阴影在垂直方向的偏移量。可以是任意数值，单位为像素。  
-  `strokeStyle` ：设置线条的样式。可以是颜色值、渐变对象或图案对象，用于指定线条的颜色或样式。  
- `textAlign` 的默认值为 `"start"` ，表示文本内容的起始位置与绘制位置对齐。
- `textBaseline` 的默认值为 `"alphabetic"` ，表示文本内容的基线与绘制位置对齐。



**方法**：

-  `arc(x, y, radius, startAngle, endAngle, anticlockwise)` ：绘制一段圆弧或部分圆。参数  `x`  和  `y`  是圆心的坐标， `radius`  是半径，`startAngle`  和  `endAngle`  是起始角度和结束角度， `anticlockwise`  是一个布尔值，表示是否按逆时针方向绘制圆弧。

-  `arcTo(x1, y1, x2, y2, radius)` ：根据给定的控制点和半径绘制一段圆弧，使其与前一个点和给定的终点相切。 `x1`  和  `y1`  是第一个控制点的坐标， `x2`  和  `y2`  是第二个控制点的坐标， `radius`  是半径。

-  `beginPath()` ：创建一个新的路径，用于绘制图形。

-  `bezierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)` ：绘制贝塞尔曲线，通过控制点来确定曲线的形状。 `cp1x`  和  `cp1y`  是第一个控制点的坐标， `cp2x`  和  `cp2y`  是第二个控制点的坐标， `x`  和  `y`  是曲线的结束点坐标。

-  `clearRect(x, y, width, height)` ：清除指定矩形区域内的像素，使其变为透明。 `x`  和  `y`  是矩形区域的起始坐标， `width`  和  `height`  是矩形区域的宽度和高度。

-  `clip()` ：根据当前路径创建一个剪切区域，只有在剪切区域内的图形才会被绘制。

-  `closePath()` ：将当前路径的起点和终点连接，形成一个闭合路径。

-  `createImageData(width, height)` ：创建一个与给定尺寸相匹配的空白图像数据对象。 `width`  和  `height`  是图像数据的宽度和高度。

-  `createLinearGradient(x0, y0, x1, y1)` ：创建一个线性渐变对象，用于绘制线性渐变效果。 `x0`  和  `y0`  是起始点的坐标， `x1`  和  `y1`  是结束点的坐标。

-  `createPattern(image, repetition)` ：创建一个图案对象，用于绘制重复的图案。 `image`  是一个图像、画布或视频元素， `repetition`  是一个字符串，表示图案的重复方式。

-  `createRadialGradient(x0, y0, r0, x1, y1, r1)` ：创建一个径向渐变对象，用于绘制径向渐变效果。 `x0`  和  `y0`  是内圆的圆心坐标， `r0`  是内圆的半径， `x1`  和  `y1`  是外圆的圆心坐标， `r1`  是外圆的半径。

-  `drawFocusIfNeeded(element)` ：如果指定元素具有焦点，则绘制指定元素。

-  `drawImage(image, dx, dy)` ：将图像绘制到 Canvas 上。 `image`  是一个图像、画布或视频元素， `dx`  和  `dy`  是图像在 Canvas 上的绘制起始位置的坐标。

-  `ellipse(x, y, radiusX, radiusY, rotation, startAngle, endAngle, anticlockwise)` ：绘制一个椭圆或部分椭圆。 `x`  和  `y`  是椭圆中心的坐标， `radiusX`  和  `radiusY`  是椭圆的水平和垂直半径， `rotation`  是椭圆的旋转角度， `startAngle`  和  `endAngle`  是起始角度和结束角度， `anticlockwise`  是一个布尔值，表示是否按逆时针方向绘制椭圆。

-  `fill()` ：填充当前路径的内容。

-  `fillRect(x, y, width, height)` ：填充指定矩形区域内的像素。 `x`  和  `y`  是矩形区域的起始坐标， `width`  和  `height`  是矩形区域的宽度和高度。

-  `fillText(text, x, y [, maxWidth])` ：在指定位置绘制填充的文本。 `text`  是要绘制的文本， `x`  和  `y`  是文本的起始位置的坐标， `maxWidth`  是可选参数，表示文本的最大宽度。

-  `getImageData(x, y, width, height)` ：获取指定区域的像素数据。 `x`  和  `y`  是要获取像素数据的区域的起始坐标， `width`  和  `height`  是要获取像素数据的区域的宽度和高度。

-  `getLineDash()` ：获取当前虚线样式的线段和间隔序列。

-  `isPointInPath(x, y)` ：检查指定的点是否在当前路径中。 `x`  和  `y`  是要检查的点的坐标。

-  `isPointInStroke(x, y)` ：检查指定的点是否在当前路径的描边上。 `x`  和  `y`  是要检查的点的坐标。

-  `lineTo(x, y)` ：添加一条直线到当前路径的子路径中。 `x`  和  `y`  是直线的结束点坐标。

-  `measureText(text)` ：测量指定文本的宽度。 `text`  是要测量的文本。

-  `moveTo(x, y)` ：将路径的起始点移动到指定的坐标。 `x`  和  `y`  是起始点的坐标。

-  `putImageData(imageData, dx, dy)` ：将像素数据绘制到 Canvas 上。`imageData`  是要绘制的像素数据， `dx`  和  `dy`  是图像在 Canvas 上的绘制起始位置的坐标。

-  `quadraticCurveTo(cp1x, cp1y, x, y)` ：绘制二次贝塞尔曲线，通过控制点来确定曲线的形状。 `cp1x`  和  `cp1y`  是控制点的坐标， `x`  和  `y`  是曲线的结束点坐标。

-  `rect(x, y, width, height)` ：创建一个矩形路径。 `x`  和  `y`  是矩形的起始坐标， `width`  和  `height`  是矩形的宽度和高度。

-  `restore()` ：恢复之前保存的绘图状态和变换矩阵。

-  `rotate(angle)` ：旋转当前绘图。 `angle`  是旋转的角度，以弧度为单位。

-  `save()` ：保存当前的绘图状态和变换矩阵。

-  `scale(scaleX, scaleY)` ：缩放当前绘图。 `scaleX`  和  `scaleY`  是水平和垂直方向上的缩放因子。

-  `setLineDash(segments)` ：设置虚线样式的线段和间隔序列。 `segments`  是一个数组，表示线段和间隔的长度。

-  `setTransform(a, b, c, d, e, f)` ：设置绘图的变换矩阵。

-  `stroke()` ：绘制当前路径的描边。

-  `strokeRect(x, y, width, height)` ：绘制指定矩形区域的描边。 `x`  和  `y`  是矩形区域的起始坐标， `width`  和  `height`  是矩形区域的宽度和高度。

-  `strokeText(text, x, y [, maxWidth])` ：在指定位置绘制文本的描边。 `text`  是要绘制的文本， `x`  和  `y`  是文本的起始位置的坐标， `maxWidth`  是可选参数，表示文本的最大宽度。

-  `transform(a, b, c, d, e, f)` ：对当前绘图进行变换。

-  `translate(x, y)` ：平移当前绘图。




接下来我们再总结下**特殊的 Canvas 接口**：

- `CanvasGradient`：用于创建渐变效果。它可以用作绘制图形的填充或描边样式。CanvasGradient 对象是通过调用 Canvas 上下文对象的  `createLinearGradient()`  或  `createRadialGradient()`  方法创建的。通过方法  `addColorStop(offset, color)` 用于指定渐变中的颜色和位置。 `offset`  参数表示颜色在渐变中的位置，范围从 0 到 1，表示渐变的起始到结束位置。 `color`  参数表示要添加的颜色，可以是 CSS 颜色字符串或 CanvasGradient、CanvasPattern 对象。

- `ImageBitmap`：一个用于表示位图图像的对象，可以通过调用  `createImageBitmap()`  方法创建。它通常用于异步加载图像并在 Canvas 上绘制图像。

- `ImageData`：一个对象，用于表示 Canvas 上指定区域的像素数据。可以通过调用  `createImageData()`  或  `getImageData()`  方法创建 ImageData 对象。返回的 data 属性是一个 Uint8ClampedArray 类型的数组，包含了指定区域内每个像素的颜色信息。

- `TextMetrics`：一个对象，用于表示文本的测量信息，包括文本的宽度等。可以通过调用  `measureText()`  方法获取 TextMetrics 对象。

- `CanvasImageSource`：一个接口，表示可以作为 Canvas 的源的对象，包括 HTMLImageElement、HTMLVideoElement、HTMLCanvasElement 和 ImageBitmap。

- `CanvasPattern`：一个对象，结合矩阵用于创建图案样式，可以用作绘制图形的填充或描边样式。

- `RenderingContext`：表示 Canvas 上下文的基类。它是 Canvas 2D 上下文和 WebGL 上下文的父级接口。





## 屏幕坐标系转为 Canvas 坐标系

当我们想要得到浏览器页面点击 Canvas 画布相对的坐标位置的时候，我们往往需要做一层转换，比如这样：

```js
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

function handleMouseMove(event) {
  const screenX = event.clientX; // 鼠标事件的屏幕X坐标
  const screenY = event.clientY; // 鼠标事件的屏幕Y坐标

  const canvasRect = canvas.getBoundingClientRect(); // 获取Canvas元素在页面上的位置
  const canvasX = screenX - canvasRect.left; // 相对于Canvas元素的X坐标
  const canvasY = screenY - canvasRect.top; // 相对于Canvas元素的Y坐标
    // 兼容移动端的高dpr设备
  const scale = window.devicePixelRatio; // 获取Canvas的缩放比例
  const canvasScaledX = canvasX / scale; // 考虑缩放后的Canvas X坐标
  const canvasScaledY = canvasY / scale; // 考虑缩放后的Canvas Y坐标

  console.log('Canvas坐标系 X:', canvasScaledX);
  console.log('Canvas坐标系 Y:', canvasScaledY);
}

canvas.addEventListener('mousemove', handleMouseMove);
```

我们通过 `event.clientX` 和 `event.clientY` 获取鼠标事件的屏幕坐标。然后，通过 `canvas.getBoundingClientRect()` 获取 Canvas 元素在页面上的位置，计算出相对于 Canvas 元素的坐标。最后，考虑到 Canvas 的缩放比例，将相对于 Canvas 元素的坐标除以缩放比例，得到最终的 Canvas 坐标系坐标。




## Canvas 坐标系转为笛卡尔坐标系

对于开发大量对称图形或者习惯用笛卡尔坐标系的开发者来说，我们需要再做一套兼容的坐标转换。

```js
const canvas = document.getElementById('myCanvas');
const width = canvas.width;
const height = canvas.height;

const ctx = canvas.getContext('2d');

// 将原点移动到Canvas的中心点
ctx.translate(width / 2, height / 2);

// 反转Y轴方向
ctx.scale(1, -1);

// 在以中心点为原点的笛卡尔坐标系中绘制一个矩形
ctx.fillStyle = 'red';
ctx.fillRect(-50, -50, 100, 100);
```

在这个示例中，我们首先获取 Canvas 的宽度和高度，然后使用 `ctx.translate()` 将原点移动到 Canvas 的中心点。接下来，使用 `ctx.scale()` 反转 Y 轴方向，使得 Y 轴向上为正方向。最后，我们在以中心点为原点的笛卡尔坐标系中绘制一个红色的矩形。

## Canvas 绘图的保存与恢复

Canvas 提供了两个方法来保存和恢复绘图状态： `save()` 和 `restore()` 。 保存状态的容器我们可以看作是一个栈的结构，先进后出的出入栈顺序。

- `save()` 方法用于保存当前的绘图状态，包括当前的变换矩阵、剪切区域、线条样式、填充样式等。可以多次调用 `save()` 方法来保存多个绘图状态。 

- `restore()` 方法用于恢复最近保存的绘图状态。每次调用 `restore()` 方法时，Canvas 都会将当前的绘图状态替换为最近保存的状态。 

我们来看下代码示例：

```js
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// 绘制第一个矩形
ctx.fillStyle = 'red';
ctx.fillRect(50, 50, 100, 100);

// 保存绘图状态
ctx.save();

// 修改绘图状态
ctx.fillStyle = 'blue';
ctx.fillRect(200, 50, 100, 100);

// 恢复最近保存的绘图状态
ctx.restore();

// 绘制第二个矩形，颜色恢复为红色
ctx.fillRect(350, 50, 100, 100);
```

在这个示例中，我们首先绘制了一个红色的矩形。然后，使用 `save()` 方法保存了当前的绘图状态。接着，修改绘图状态，将填充样式修改为蓝色，并绘制了一个蓝色的矩形。最后，使用 `restore()` 方法恢复了最近保存的绘图状态，使得填充样式恢复为红色。然后，我们绘制了第二个矩形，它的颜色恢复为红色。

> 这里有必要讲下 Canvas 和 SVG 的两种绘图模式。
> - **Canvas 的立即模式（Immediate Mode）：** Canvas 使用立即模式进行绘图，这意味着绘图操作是即时执行的。在每次绘制时，你需要通过 Canvas API 直接绘制图形、设置样式、处理交互等。立即模式适合于动态、实时的图形绘制，例如游戏、数据可视化等场景。绘制的结果不会被保留，不会以独立的图形对象存在。
> 
> - **SVG 的保留模式（Retained Mode）：** SVG 使用保留模式进行绘图，这意味着绘图操作创建了可保留的图形对象。你可以使用 SVG 标记语言定义图形、属性和样式，然后将其插入到 HTML 文档中。SVG 图形对象可以通过 CSS 样式和 JavaScript 进行操作和修改。保留模式适合于静态、可缩放的矢量图形，例如图表、图标等。绘制的结果以独立的图形对象存在，可以随时进行修改和交互。 




## Canvas 转换为 Image

在我们平常开发中，会遇到需要将 Canvas 海报转为图片、方便下载上传，这个时候我们就需要将 Canvas 转为 Image。

```js
    window.onload = () => {
        const canvas = document.getElementById('myCanvas');
        const ctx = canvas.getContext('2d');

        // 在Canvas上绘制内容
        ctx.fillStyle = 'red';
        ctx.fillRect(0, 0, canvas.width, canvas.height);

        // 将Canvas内容转换为图像数据URL
        const dataURL = canvas.toDataURL();

        // 创建一个新的Image元素
        const img = new Image();
        img.src = dataURL;

        // 在新窗口中显示图像并打印
        const printWindow = window.open('', '_blank');
        printWindow.document.write('<img src="' + dataURL + '" onload="window.print();">');
    }
```


这里我们使用 `toDataURL` 方法将 Canvas 内容转换为图像数据 URL（base64 编码形式）。然后，我们创建一个新的 `Image` 元素，并将图像数据 URL 赋值给 `src` 属性。最后，我们创建一个新的浏览器窗口，并在该窗口中写入一个包含图像的 HTML 字符串。

此外，我们可以用 `toBlob` 通过 Blob 二进制来传输实现：

```js
const canvas = document.getElementById('mycanvas');
const context = canvas.getContext('2d');

// 在 canvas 上绘制图形
context.fillStyle = 'blue';
context.fillRect(0, 0, canvas.width, canvas.height);

// 将 canvas 转换为 Blob
canvas.toBlob(function (blob) {
    // 在这里处理 Blob 对象
    var img = new Image();
    img.src = URL.createObjectURL(blob);
    document.body.appendChild(img);
}, 'image/jpeg', 0.95);
```

`toBlob()` 生成的 Blob 对象更适合处理大型图像，因为它不会将图像数据编码为字符串，而是以二进制形式保存。这样可以减少内存消耗，并且可以设置图像的格式和质量。而 `toDataURL` 更简单方便，对于小型图像我们可以使用它。

## Canvas 的像素处理

Canvas 是如何处理图片的呢？比如我想让图片变灰色或者饱和度变高，或者我想处理颜色体块做高亮，又或者我想扣除背景的操作，这个时候我们就需要通过 Canvas 的 `getImageData()` 和 `putImageData()` 来得到像素和绘制像素。

```js
const imageData = context.getImageData(x, y, width, height);
context.putImageData(imageData, x, y);
```



### 图像马赛克化

这里我们做一个简单的基础案例：

```html
<canvas id="myCanvas" width="400" height="400"></canvas>
<script>
    var canvas = document.getElementById("myCanvas");
    var ctx = canvas.getContext("2d");

    var img = new Image();
    img.src = "../1b4044dac82289dc7c1d878f4183c1b7.jpg";
    img.onload = function () {
        // 将图片绘制到 Canvas 上
        ctx.drawImage(img, 0, 0, canvas.width, canvas.height);

        // 获取图片的像素数据
        var imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
        var pixels = imageData.data;

        // 马赛克化处理
        var blockSize = 10; // 马赛克块的大小
        for (var y = 0; y < canvas.height; y += blockSize) {
            for (var x = 0; x < canvas.width; x += blockSize) {
                var index = (y * canvas.width + x) * 4;

                // 获取当前块的颜色
                var red = pixels[index];
                var green = pixels[index + 1];
                var blue = pixels[index + 2];

                // 将当前块的颜色应用到整个块
                for (var i = y; i < y + blockSize; i++) {
                    for (var j = x; j < x + blockSize; j++) {
                        var idx = (i * canvas.width + j) * 4;
                        pixels[idx] = red;
                        pixels[idx + 1] = green;
                        pixels[idx + 2] = blue;
                    }
                }
            }
        }

        // 将处理后的像素数据绘制回 Canvas
        ctx.putImageData(imageData, 0, 0);
    };
</script>
```

我们首先将图片绘制到 Canvas 上。然后，使用 `getImageData()` 方法获取整个 Canvas 区域的像素数据。接下来，我们对每个马赛克块进行处理，将当前块的颜色应用到整个块上。最后，使用 `putImageData()` 方法将处理后的像素数据绘制回 Canvas 上。

### ImageData

`ImageData` 是一个用于表示 Canvas 上像素数据的对象。它提供了对像素级别的访问和修改能力。

`createImageData()` 是一个用于创建新的空白 `ImageData` 对象的方法。你可以使用它来创建一个与指定尺寸相匹配的空白像素数据对象，然后通过修改像素数据来实现各种图像处理效果。

```js
const imageData = context.createImageData(width, height);
```

创建了一个空白的 `ImageData` 对象后，你可以使用 `imageData.data` 属性来访问像素数据。`imageData.data` 是一个包含每个像素的颜色信息的一维`Uint8ClampedArray` 类型数组。

> `Uint8ClampedArray` 是 JavaScript 中的一种`类型化数组`（Typed Array），它表示一个包含了 8 位无符号整数（0 到 255）的固定长度数组。在 `ImageData` 对象中，`.data` 属性返回的 `Uint8ClampedArray` 数组用于存储每个像素的颜色信息。

为什么要用类型化数组呢？我们知道 JavaScript 是动态的脚本语言，数组的长度是动态的，而类型化数组以固定的内存结构能更高效地处理二进制数据，比如像素、音频等。在与一些底层的 API 交互的时候，大部分需要以二进制做数据传递，比如 WebGL。



## Canvas 的性能优化

Canvas 的性能优化可以通过以下几种方式来实现。

1. **减少绘制区域**：只绘制需要更新的区域。
2. **使用离屏渲染**：使用离屏 Canvas 进行复杂的图形处理，然后将处理结果绘制到主 Canvas 上。这样可以减少对主 Canvas 的直接绘制操作，提高性能。 
3. **批量绘制**：将多个绘制操作合并为一个，减少绘制次数。例如，可以将多个矩形的绘制合并为一个 `context.fillRect()` 调用。 
4. **使用图像缓存**：将经常使用的图像绘制到一个隐藏的 Canvas 上，并将其作为图像资源重复使用。这样可以减少重复绘制的开销，提高性能。 
5. **减少图像大小**：如果图像过大，可以将其缩放到合适的尺寸，以减少绘制的像素数量。可以使用 `context.drawImage()` 方法的缩放参数来实现。 
6. **避免频繁的状态更改**：在绘制过程中，避免频繁地修改 Canvas 的属性和状态。例如，可以将颜色、线宽等属性设置为常量，避免在每次绘制前都进行设置。 
7. **使用硬件加速**：在支持的浏览器中，可以使用 CSS 属性 `transform` 或 `opacity` 来触发硬件加速，提高绘制性能。 
8. **使用 requestAnimationFrame**：可以优化动画的绘制，确保在浏览器的刷新间隔内进行绘制，以获得更流畅的动画效果。 
9. **避免频繁的像素操作**：对于大量的像素级操作，如使用 `getImageData()` 和 `putImageData()` ，尽量减少其使用频率，因为这些操作会涉及大量的像素数据复制和粘贴。
10. **Web Worker**：将复杂的 JavaScript 的计算放在工作线程中执行，这样可以避免在主线程上进行大量的计算，保持页面的流畅性。

接下来我们将深入讲解其中几个重要的性能优化。


### drawImage 和 putImageData 的渲染效率对比

由于 `drawImage()` 内部会进行一些优化和硬件加速的操作，因此在大多数情况下， `drawImage()` 的渲染效率会更高。尤其是在处理大量图像或需要频繁更新的场景下， `drawImage()` 通常能够更好地利用浏览器的硬件加速功能，提供更高的性能和更平滑的动画效果。 

然而，对于一些特定的图像处理需求，`putImageData()` 可能会更适合，例如需要直接修改像素数据、实现自定义的图像处理算法等。在这些情况下，`putImageData()` 提供了更高的灵活性和精确度。


### Web Worker 工作线程

对于 Canvas，使用 Web Worker 可以提供性能优化的可能性。

Web Worker 是在后台运行的 JavaScript 线程，可以在独立于主线程的环境中执行计算密集型任务，从而避免阻塞主线程，提高页面的响应性能。

接下来我们通过一个简单的案例来快速理解：


```html
 <script>
    const canvas = document.getElementById("mainCanvas");
    const ctx = canvas.getContext("2d");
    ctx.fillStyle = "red";
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    // 创建一个新的 Worker
    const worker = new Worker("worker.js");

    // 监听 Web Worker 发送的消息
    worker.onmessage = function (event) {
        // 获取处理后的像素数据
        const processedData = event.data;
        // 将处理后的像素数据绘制回 Canvas
        const imageData = new ImageData(processedData, canvas.width, canvas.height);
        ctx.putImageData(imageData, 0, 0);
    };

    // 获取 Canvas 上的像素数据
    const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
    const pixels = imageData.data;

    // 将像素数据发送给 Web Worker 进行处理
    worker.postMessage(pixels);
</script>
```


Web Worker 代码（worker.js）：

```js
self.onmessage = function (event) {
    var pixels = event.data;

    // 将像素点的颜色改为蓝色
    for (var i = 0; i < pixels.length; i += 4) {
        pixels[i] = 0;       // 红色通道设为0
        pixels[i + 1] = 0;   // 绿色通道设为0
        pixels[i + 2] = 255; // 蓝色通道设为255
    }

    // 将处理后的像素数据发送回主线程
    self.postMessage(pixels);
};
```

我们首先在主线程中创建了一个新的 Web Worker，并监听其发送的消息。然后，我们使用 `getImageData()` 方法获取 Canvas 上的像素数据，并将其发送给 Web Worker 进行处理。Web Worker 接收到像素数据后，将像素点的颜色改为蓝色，并将处理后的像素数据发送回主线程。

> 注意：Web Worker 是在后台线程中运行的脚本，它与主线程是分离的，并且没有直接访问 DOM 的权限。因此，`不能在 Web Worker 中直接操作 Canvas 元素或使用与 DOM 相关的方法`。

### Canvas 的离屏渲染

Canvas 的离屏渲染是指在 Canvas 外部创建一个隐藏的 Canvas 元素，并将图形绘制到隐藏的 Canvas 上，然后再将隐藏 Canvas 上的内容绘制到可见的 Canvas 上。

离屏渲染通常用于以下情况：

1. **复杂的图形处理**：如果需要进行复杂的图形处理，可能需要多次绘制和修改图像。使用离屏渲染可以将这些操作分离到一个隐藏的 Canvas 上，减少对可见 Canvas 的直接绘制，提高性能。 
2. **图像缓存**：如果需要频繁地绘制相同的图像或图形，可以使用离屏渲染将其绘制到隐藏的 Canvas 上，并在需要时将其绘制到可见的 Canvas 上。这样可以减少重复绘制的开销，提高性能。 
3. **特效和动画**：某些特效和动画需要对图像进行复杂的处理和计算，离屏渲染可以提供更多的灵活性和控制，使得实现这些效果更加方便。

```js
var mainCanvas = document.getElementById("mainCanvas");
var mainCtx = mainCanvas.getContext("2d");

// 创建离屏Canvas
var offscreenCanvas = document.createElement("canvas");
var offscreenCtx = offscreenCanvas.getContext("2d");

// 设置离屏Canvas的尺寸
offscreenCanvas.width = mainCanvas.width;
offscreenCanvas.height = mainCanvas.height;

// 在离屏Canvas上绘制图形
offscreenCtx.fillStyle = "blue";
offscreenCtx.fillRect(50, 50, 100, 100);

// 将离屏Canvas绘制到主Canvas上
mainCtx.drawImage(offscreenCanvas, 0, 0);
```

### OffscreenCanvas

除了上文用 worker 线程来处理复杂的 js 逻辑，我们还可以将离屏渲染的 canvas 上下文对象放入其中。

而 `OffscreenCanvas` 刚好为我们提供了一个可以脱离屏幕渲染的 Canvas 对象，可以运行在`窗口环境和 web worker 环境`。但是该 API 已知具有兼容性问题（比如 Safari 和 IE），需要考虑不兼容情况下的降级方案。

OffscreenCanvas 的优势在于：当主线程繁忙时，依然可以通过 OffscreenCanvas 在 worker 中更新画布内容，避免给用户造成页面卡顿的体验。

接下来讲解 OffscreenCanvas 的三种常见用法：

1. 在窗口环境下离屏渲染

```html
<canvas id="onscreenCanvas" width="200" height="200"></canvas>
<script>
  // 获取屏幕上的 <canvas> 元素的绘图上下文
  const onscreenCanvas = document.getElementById('onscreenCanvas')
  const onscreenCtx = onscreenCanvas.getContext('2d')

  // 创建一个 OffscreenCanvas 对象
  const offscreenCanvas = new OffscreenCanvas(200, 200)
  const offscreenCtx = offscreenCanvas.getContext('2d')

  // 在 OffscreenCanvas 上绘制
  offscreenCtx.fillStyle = 'red'
  offscreenCtx.fillRect(50, 50, 100, 100)

  // 将离屏缓冲区的内容转移到屏幕上的 <canvas> 元素
  onscreenCtx.drawImage(offscreenCanvas, 0, 0)
</script>
```

2. 通过 transferToImageBitmap 实现离屏渲染
```js
const canvas = document.getElementById('canvas').getContext('bitmaprenderer')

// 创建一个新的 OffscreenCanvas 对象
const offscreen = new OffscreenCanvas(256, 256)

// 获取 WebGL 上下文进行绘制
const gl = offscreen.getContext('webgl')

// 设置 WebGL 绘制的一些状态
gl.clearColor(0.0, 0.0, 1.0, 1.0) // 设置清除颜色为蓝色
gl.clear(gl.COLOR_BUFFER_BIT) // 清除颜色缓冲区

// 将离屏缓冲区的内容转换为 ImageBitmap
const bitmap = offscreen.transferToImageBitmap()

// 将 ImageBitmap 应用到 canvas 画布
canvas.transferFromImageBitmap(bitmap)
```

3. 在 worker 环境下的离屏渲染

```js
// main.js
var htmlCanvas = document.getElementById("canvas");
var offscreen = htmlCanvas.transferControlToOffscreen();

var worker = new Worker("offscreencanvas.js");
worker.postMessage({ canvas: offscreen }, [offscreen]);
```

```js
// offscereencanvas.js
onmessage = function (evt) {
  var canvas = evt.data.canvas;
  var gl = canvas.getContext("webgl");
  gl.clearColor(0.0, 0.0, 1.0, 1.0)
  gl.clear(gl.COLOR_BUFFER_BIT) 
  gl.commit();
};
```


## 动画渲染

对于动画或者在 Canvas 上进行绘制的场景，使用 requestAnimationFrame 是更好的选择。这是因为 requestAnimationFrame 会在浏览器的`下一次重绘之前执行函数`，`这样可以保证动画的更新频率与浏览器的刷新率相匹配，避免了过度绘制和资源浪费`。同时，使用 requestAnimationFrame 还可以获得更好的性能和更平滑的动画效果。

相比之下，`setTimeout` 和 `setInterval` 的时间间隔参数只是一个指示，不能保证准确的执行时间。特别是在处理高性能动画时，由于 JavaScript 的单线程执行机制和其他任务的干扰，可能会导致间隔时间不准确，从而导致动画不流畅或不稳定。

在动画游戏方面，这里有非常多的细节值得探索，会在后续的二维游戏开发中详细讲解。



## Canvas 的双缓存技术

双缓冲技术是一种绘图技术，用于解决`图像闪烁`问题。

在双缓冲技术中，使用两个缓冲区，一个用于绘制图像，另一个用于显示图像。绘制操作在后备缓冲区进行，绘制完成后将整个后备缓冲区的图像复制到前景缓冲区，然后显示在屏幕上。这样可以避免绘制过程中的闪烁问题，提供平滑的图像显示。

这里需要注意和离屏渲染的区分，离屏渲染主要通过另外隐藏的 Canvas 上下文计算绘制，然后通过 drawImage 来渲染到显示的 Canvas 上下文中。`所以，核心区别在于是否在同一个绘图上下文中。而双缓冲很显然会有两个队列来异步更新渲染。`

现代浏览器已经实现了双缓冲技术。通常使用 `<canvas>` 元素进行绘图操作。当你在 `<canvas>` 上进行绘制时，浏览器会自动使用双缓冲技术来处理图像的绘制和显示。
