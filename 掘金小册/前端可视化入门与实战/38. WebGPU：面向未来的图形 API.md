
WebGPU 是一套基于浏览器的图形 API，封装了基于现代图形 API（Dx12、Vulkan、Metal），提供给 Web 3D 程序员，为 Web 释放了更多的 GPU 硬件的功能。


为什么 WebGPU 取代 WebGL 是必然？

WebGL 是 Web 上图形处理的事实标准，但它已经存在了 10 多年，这么多年 GPU 显卡的迅猛发展让厂商开始慢慢抛弃 OpenGL 的老旧存在缺陷的 API。为了适应新的时代发展，谷歌公司开发了 WebGPU，它的底层基于 Vulkan（Khronos Group 开发的跨平台的图形 API）所实现。

下面是 WebGPU 取代 WebGL 的一些原因：

* **性能**。WebGPU 使用现代图形 API 和硬件，提供比 WebGL 更高的性能。
* **可移植性**。WebGPU 是跨平台的，可以在任何支持 WebGPU 的浏览器上运行。
* **灵活性**。WebGPU 提供了更灵活的 API，允许开发人员更轻松地创建复杂的图形。
* **可扩展性**。WebGPU 可以扩展到支持更高分辨率的图像和更复杂的场景。


## WebGL 与 WebGPU 区别

WebGPU 是 WebGL 的升级版，它提供了更高的性能和更强的功能。但是，由于 WebGPU 的兼容性较差，目前还没有广泛使用。

以下是 WebGL 和 WebGPU 的大致对比：

| 指标 | WebGL | WebGPU |
|---|---|---|
| 基础 | 基于 OpenGL ES 2.0 | 基于 Vulkan | 
| 着色器语言 | GLSL | WGSL |
| 着色器语言类比 | C/C++ | Rust |
| 管线 | 固定管线 | 可编程管线 |
| 性能 | 较低 | 较高 |
| 兼容性 | 较好 | 较差 |

除此之外，还有一些区别。

- **状态**：在 WebGL 中会有大量保存的状态，这无疑会占用电脑的资源，而在 WebGPU 中并没有保存的状态，比如缓存、纹理等，所有的管线状态都不会改变。

- **通信的数据结构**：在 WebGL 中数据的传递我们可以通过传入对象通信，而在 WebGPU 中我们将使用数组来存储在不同索引上存储哪些内容，比如颜色、顶点、法线等。

- **画布**：WebGL 的画布是固定的，而 WebGPU 的画布可以动态创建和销毁。WebGL 的画布只能使用一个纹理，而 WebGPU 的画布可以使用多个纹理。总的来说就是 WebGPU 比 WebGL 更加灵活和强大。

- **Mipmap 生成**：在 WebGL 中，可以通过调用  `gl.generateMipmap`  自动生成纹理的所有 mipmap 级别。但是，在 WebGPU 中没有内置的自动生成 mipmap 的功能，需要手动生成它们。

- **采样器的要求**：在 WebGL1 中，采样器是由 WebGL 内部处理的，不需要显式定义。在 WebGL2 中，使用采样器是可选的。而在 WebGPU 中，采样器是必需的，需要显式定义和使用采样器。

- **缓冲区和纹理的调整大小**：在 WebGL 中，可以随时更改缓冲区或纹理的大小，通过调用  `gl.bufferData`  或  `gl.texImage2D`  等函数来重新分配它们的大小。通常的做法是创建一个大小为 1x1 像素的占位纹理或缓冲区，然后异步加载图像或数据，并在加载完成后更新纹理或缓冲区的内容。但是，在 WebGPU 中，缓冲区和纹理的大小是固定的，无法动态调整。

## 获取并配置 WebGPU 上下文

```js
async function main() {
    // 获取适配器
    const adapter = await navigator.gpu?.requestAdapter();
    const device = await adapter?.requestDevice();
    if (!device) {
        alert('need a browser that supports WebGPU');
        return;
    }

    const canvas = document.querySelector('canvas');
    
    canvas.width = window.innerWidth;
    canvas.height = window.innerHeight;
    // 获取上下文
    const context = canvas.getContext('webgpu');
    const presentationFormat = navigator.gpu.getPreferredCanvasFormat();
    // 配置上下文
    context.configure({
        device,
        format: presentationFormat,
    });
}

main();
```

我们使用  `navigator.gpu.requestAdapter()`  方法异步来获取一个适配器。适配器是与 GPU 通信的接口。然后，我们使用  `adapter.requestDevice()`  方法来获取一个设备。设备是与 GPU 进行实际通信的对象。

如果没有找到支持 WebGPU 的浏览器，我们会弹出一个警告。否则，我们会获取到一个 canvas 元素，并使用  `canvas.getContext('webgpu')`  方法来获取一个 WebGPU 上下文。

最后，我们使用  `context.configure()`  方法来配置 WebGPU 上下文。同时我们需要指定设备和格式。


## Render

```js
function render() {
    // 配置渲染通道，设置颜色附件的视图为当前纹理的视图
    renderPassDescriptor.colorAttachments[0].view =
        context.getCurrentTexture().createView();

    // 创建命令编码器
    const encoder = device.createCommandEncoder({ label: 'our encoder' });

    // 开启渲染通道
    const pass = encoder.beginRenderPass(renderPassDescriptor);

    // 设置渲染管线
    pass.setPipeline(pipeline);

    // 绘制三角形
    pass.draw(3);

    // 结束渲染通道
    pass.end();

    // 完成命令编码器并生成命令缓冲区
    const commandBuffer = encoder.finish();

    // 提交命令缓冲区至设备队列进行执行
    device.queue.submit([commandBuffer]);
}
```

首先，我们需要创建一个渲染通道描述符，并将其颜色附件设置为当前纹理。然后，我们创建一个命令编码器，并开始一个渲染通道。 接下来，我们设置渲染管线，并绘制一个三角形。最后，我们结束渲染通道，并提交命令缓冲区。


## renderPassDescriptor 渲染通道

 `renderPassDescriptor` 是一个用于描述渲染通道的对象。它包含了渲染通道的配置信息，如颜色附件、深度附件、模板附件等。

`renderPassDescriptor` 包含以下信息：

- 颜色附件（colorAttachments）：用于指定渲染目标的颜色附件的配置，如颜色附件的视图（view）、加载操作（loadOp）、存储操作（storeOp）等。

- 深度附件（depthAttachment）：用于指定渲染目标的深度附件的配置，如深度附件的视图（view）、加载操作（loadOp）、存储操作（storeOp）等。

- 模板附件（stencilAttachment）：用于指定渲染目标的模板附件的配置，如模板附件的视图（view）、加载操作（loadOp）、存储操作（storeOp）等。

- 清除颜色（clearColor）：用于指定在每次渲染通道开始时是否需要清除颜色附件的颜色值。

- 清除深度（clearDepth）：用于指定在每次渲染通道开始时是否需要清除深度附件的深度值。

- 清除模板（clearStencil）：用于指定在每次渲染通道开始时是否需要清除模板附件的模板值。

通过配置  `renderPassDescriptor` ，我们可以定义渲染通道的输入和输出，并设置相应的操作和清除值，以实现所需的渲染效果。

比如，我们可以这么配置：

```js
const renderPassDescriptor = {
    label: 'our basic canvas renderPass',
    colorAttachments: [
        {
            clearValue: [0, 0, 0, 1],
            loadOp: 'clear',
            storeOp: 'store',
        },
    ],
};
```

-  `label` ：渲染通道描述符的标签，用于标识该渲染通道。
-  `colorAttachments` ：颜色附件的配置数组，用于指定渲染目标的颜色附件的配置。
-  `clearValue` ：清除颜色的值，这里设置为  `[0, 0, 0, 1]` ，表示清除为黑色（RGBA 值为 0, 0, 0, 1）。
-  `loadOp` ：加载操作，设置为  `'clear'` ，表示在渲染通道开始之前清除颜色附件。
-  `storeOp` ：存储操作，设置为  `'store'` ，表示在渲染通道结束后将颜色附件的内容保留。


在我们通过编码器开启渲染通道 pass 后，我们需要处理最重要的环节设置渲染管线。


## pipeline 渲染管线

```js
// 创建一个渲染管线（Render Pipeline）对象
const pipeline = device.createRenderPipeline({
    label: 'our hardcoded red triangle pipeline',
    layout: 'auto',
    vertex: {
        module,
        entryPoint: 'vs',
    },
    fragment: {
        module,
        entryPoint: 'fs',
        targets: [{ format: presentationFormat }],
    },
});
```

-  `label` ：渲染管线的标签，用于标识该渲染管线（在后续很多配置中都会存在 label 选项，主要用于 bug 的定位）。
-  `layout` ：渲染管线的布局，可以设置为  `'auto'`  或自定义布局。
-  `vertex` ：顶点着色器的配置，包括着色器模块（ `module` ）和入口点（`entryPoint`）。
-  `fragment` ：片段着色器的配置，包括着色器模块（ `module` ）、入口点（`entryPoint`）和渲染目标（ `targets` ）。

我们创建了一个渲染管线，其中包括了一个顶点着色器和一个片段着色器。着色器模块（`module`）是我们之前定义的模块对象，入口点（`entryPoint`）指定了着色器代码的入口点函数。 片段着色器还指定了一个渲染目标，通过 `targets` 属性设置了目标的格式（`format`），这里是 `presentationFormat`。

接着我们讲解着色器模块 module，也就是对应的 WGSL 代码：

```js
const module = device.createShaderModule({
```

## 渲染流程

<p align=center><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2c8b8285c82f46d48a91f455d5d10f8c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=555&h=635&s=22565&e=png&b=e5f6da" alt="image.png"  /></p>

WebGPU 的渲染流程可以概括为以下几个步骤：

1. **获取适配器**（Adapter）：使用  `navigator.gpu.requestAdapter()`  方法获取适配器。适配器是与 GPU 通信的接口。

2. **获取设备**（Device）：通过适配器使用  `adapter.requestDevice()`  方法获取设备。设备是与 GPU 进行实际通信的对象。

3. **创建上下文**（Context）：使用  `canvas.getContext('webgpu')`  方法获取 WebGPU 上下文。上下文是用于与 GPU 进行交互的接口。

4. **配置上下文**（Context Configuration）：使用  `context.configure()`  方法配置上下文。需要指定设备和颜色格式等参数。

5. **创建渲染通道描述符**（Render Pass Descriptor）：创建一个渲染通道描述符对象，用于配置渲染通道的参数，如颜色附件、深度附件等。

6. **创建命令编码器**（Command Encoder）：使用  `device.createCommandEncoder()`  方法创建命令编码器。命令编码器用于记录和生成一系列 GPU 命令。

7. **开始渲染通道**（Begin Render Pass）：使用命令编码器的  `beginRenderPass()`  方法开始一个渲染通道，并传入渲染通道描述符。

8. **设置渲染管线**（Set Pipeline）：使用渲染通道的  `setPipeline()`  方法设置渲染管线。渲染管线定义了渲染操作的着色器程序和其他配置。

9. **执行绘制命令**（Draw Commands）：使用渲染通道的  `draw()`  方法执行绘制命令，绘制图形或模型。

10. **结束渲染通道**（End Render Pass）：使用渲染通道的  `end()`  方法结束渲染通道。

11. **完成命令编码器**（Finish Command Encoder）：使用命令编码器的  `finish()`  方法完成命令编码器，并生成一个命令缓冲区。

12. **提交命令缓冲区**（Submit Command Buffer）：使用设备的队列（Queue）的  `submit()`  方法提交命令缓冲区，将命令发送给 GPU 执行渲染操作。

以上是 WebGPU 的基本渲染流程，通过这些步骤，我们可以配置渲染通道、执行绘制命令，并将结果提交给 GPU 进行渲染。

## 修复模糊

这个原因是在浏览器中 `原始尺寸 = 样式尺寸 * dpr`，一旦放大屏幕意味着 dpr 变大。而原始尺寸不变，样式尺寸会变小，也就变模糊了。这个时候我们只要再乘以 dpr 还原原来的尺寸即可。相同的道理，在 Canvas 我们也是一样的处理方法。

```js
// 获取设备的 DPR
const dpr = window.devicePixelRatio || 1;
// 根据设备的 DPR 设置画布的实际分辨率
canvas.width = canvas.clientWidth * dpr;
canvas.height = canvas.clientHeight * dpr;
```

你也可以更直接的方式，这样也就不用设置 dpr。

```js
canvas.width = window.innderWidth
canvas.height = window.innerHeight
```



## 阶段性变量实现渐变三角形

在 glsl 中，我们可以通过 varing 与 attribute 传递到片段着色器来修改 gl_fragment 变量。而在 wgsl 中会相对复杂些，阶段性变量主要在 wgsl 中用于着色器不同阶段传递数据的变量。

```js
const module = device.createShaderModule({
    label: '通过阶段性变量vsOutput实现渐变三角形',
    code: `
    // outVS 传递的阶段性结构体变量
  struct outVS {
    @builtin(position) position: vec4f,
    @location(0) color: vec4f,
  };

  @vertex fn vs(
    @builtin(vertex_index) vertexIndex : u32
  ) -> outVS {
    let pos = array(
      vec2f( 0.0,  0.5),  // top center
      vec2f(-0.5, -0.5),  // bottom left
      vec2f( 0.5, -0.5)   // bottom right
    );
     var color = array<vec4f, 3>(
      vec4f(1, 0, 0, 1), // red
      vec4f(0, 1, 0, 1), // green
      vec4f(0, 0, 1, 1), // blue
    );

    var vsOutput: outVS;
    vsOutput.position = vec4f(pos[vertexIndex], 0.0, 1.0);
    vsOutput.color = color[vertexIndex];
    return vsOutput;
  }

  @fragment fn fs(@location(0) color: vec4f) ->   @location(0) vec4f {
    return color;
  }
`,
});
```

其实这段代码理解起来很简单，我们想把阶段性变量传递到片段着色器，首先我们要改造输出的结构体变量，注意要保留输出的顶点变量 position，比如 outVS 结构体（注意属性的定义 @location），我们将 color 颜色坐标数组传入。这样在片段着色器中入参中，就可以定义传入`@location(0) color: vec4f`。

## buffer 缓冲区创建渐变三角形

buffer 其实就是连续的字节单元的内存空间，我们可以存储顶点、纹理等，同时我们需要指定 size、offset 来配置空间。和 webgl 也是一样的思路，主要是 API 的不同。

```js
   const module = device.createShaderModule({
    label: 'buffer demo',
    code: `
      struct OutBufferStruct {
        color: array<vec4f, 3>,
        scale: vec2f,
        offset: vec2f,
      };
      struct OutVS {
        @builtin(position) position: vec4f,
        @location(0) color: vec4f
      };
      @group(0) @binding(0) var<uniform> outBufferStruct: OutBufferStruct;

      @vertex fn vs(
        @builtin(vertex_index) vertexIndex : u32
      ) -> OutVS {
        let pos = array(
          vec2f( 0.0,  0.5),  // top center
          vec2f(-0.5, -0.5),  // bottom left
          vec2f( 0.5, -0.5)   // bottom right
        );
        var outVS: OutVS;
        outVS.color = outBufferStruct.color[vertexIndex];
        outVS.position = vec4f(pos[vertexIndex] * outBufferStruct.scale + outBufferStruct.offset, 0.0, 1.0);
        return outVS;
      }
      
      @fragment fn fs(@location(0) color: vec4f) -> @location(0) vec4f {
        return color;
      }
    `,
  });
```

理解这个代码核心在于 fs 的 color 是通过顶点的内置 vertex_index 得到不同顶点，通过 buffer 的配置传入 vs 并输出 OutVS，注意 @builtin(position)、@location(0) 的属性使用表示了输出的位置信息以及颜色存储位置，然后管线在执行到 fs 的时候就可以得到每个顶点的 color。

那么，如果在 WebGPU 外部创建对应的 buffer 属性呢？


## 创建 buffer

```js
// 当前buffer需要的位数
const uniformBufferSize = 12 * 4 + 2 * 4 + 2 * 4
// 创建buffer
const uniformBuffer = device.createBuffer({
    label: 'create buffer',
    size: uniformBufferSize,
    usage: GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST,
});
```

我们首先要对 buffer 做空间的配置，1 字节等于 8 位，这样就能得到 uniformBufferSize 的大小。然后 `device.createBuffer()` 创建缓冲区对象，接着配置 buffer 数据 `uniformValues`。

这里讲解下 uniformBuffer 的 `usage`，`usage`  是用于创建 GPU 缓冲区时的一个选项，用于**指定缓冲区的用途**。在这里，`usage`  被设置为  `GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST`，表示缓冲区既可以用于 UNIFORM 数据，也可以用于数据拷贝目的。

下面是  `usage`  中的常见标志：

- `GPUBufferUsage.VERTEX` ：指定缓冲区将用于存储顶点数据，用于图形渲染管线中的顶点输入。 
- `GPUBufferUsage.INDEX` ：指定缓冲区将用于存储索引数据，用于图形渲染管线中的索引缓冲区。 
- `GPUBufferUsage.STORAGE` ：指定缓冲区将用于存储可读写的通用数据，用于计算着色器中的计算任务。 
- `GPUBufferUsage.INDIRECT` ：指定缓冲区将用于存储间接绘制命令，用于执行间接绘制操作。
- `GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST` ：指定缓冲区既可以用于存储 UNIFORM 数据，也可以用作数据拷贝的源。 
- `GPUBufferUsage.STORAGE | GPUBufferUsage.COPY_DST` ：指定缓冲区既可以用于**存储可读写**的通用数据，也可以用作数据拷贝的源。

通过正确设置  `usage` ，可以帮助 GPU 优化数据的存储和访问，从而提高性能和效率。

## 循环生成随机三角形

我们先封装随机函数 rend：

```js
const rand = (min, max) => {
    if (min === undefined) {
      min = 0;
      max = 1;
    } else if (max === undefined) {
      max = min;
      min = 0;
    }
    return min + Math.random() * (max - min);
};
```

接下来我们只要不断创建和绑定 buffer，将 bindGroup、uniformValues、uniformBuffer 传入数组中，在 render 中提交 buffer 即可。

```js
  // 当前buffer需要的位数
  const uniformBufferSize = 12 * 4 + 2 * 4 + 2 * 4
  const kColorOffset = 0;
  const kScaleOffset = 12;
  const kOffsetOffset = 14;
  const objectNum = 1000;
  const objectList = []

  for (let i = 0; i < objectNum; i++) {
    // 创建buffer
    const uniformBuffer = device.createBuffer({
      label: 'our uniform buffer',
      size: uniformBufferSize,
      usage: GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST,
    });
    // 当前字节创建空间大小的类型化数组
    const uniformValues = new Float32Array(uniformBufferSize / 4);

    uniformValues.set([rand(), rand(), rand(), 1, rand(), rand(), rand(), 1, rand(), rand(), rand(), 1], kColorOffset);
    uniformValues.set([rand(-0.9, 0.9), rand(-0.9, 0.9)], kScaleOffset);
    uniformValues.set([rand(-0.9, 0.9), rand(-0.9, 0.9)], kOffsetOffset);

    // 绑定buffer
    const bindGroup = device.createBindGroup({
      label: 'triangle bind group',
      layout: pipeline.getBindGroupLayout(0),
      entries: [
        { binding: 0, resource: { buffer: uniformBuffer } },
      ],
    });

    objectList.push({
      bindGroup,
      uniformValues,
      uniformBuffer
    })
  }
```

render 中循环绘制：

```js
function render() {
    //...
    for (let i = 0; i < objectList.length; i++) {
      const item = objectList[i]
      device.queue.writeBuffer(item.uniformBuffer, 0, item.uniformValues);
      pass.setBindGroup(0, item.bindGroup);
      pass.draw(3);
    }
    
    //...
}
```
