## Three 是如何打包的

在 `utils/build/rollup.config.js`文件中，我们可以看到不到几百行的打包代码，这当中有一些技巧值得学习。

```js
export function glsl() {
    return {
        transform( code, id ) {
            if ( /.glsl.js$/.test( id ) === false ) return;
            code = new MagicString( code );
            code.replace( //* glsl */`(.*?)`/sg, function ( match, p1 ) {
                return JSON.stringify(
                        p1.trim()
                        .replace( /\r/g, '' )
                        .replace( /[ \t]*//.*\n/g, '' ) // remove //
                        .replace( /[ \t]*/*[\s\S]*?*//g, '' ) // remove /* */
                        .replace( /\n{2,}/g, '\n' ) // # \n+ to \n
                );
            } );
            return {
                    code: code.toString(),
                    map: code.generateMap()
            };
        }
    };
}

function header() {
    return {
        renderChunk( code ) {
                code = new MagicString( code );
                code.prepend( `/**
                     * @license
                     * Copyright 2010-2023 Three.js Authors
                     * SPDX-License-Identifier: MIT
                     */\n` );
                return {
                        code: code.toString(),
                        map: code.generateMap()
                };
        }
    };
}

function deprecationWarning() {
    return {
        renderChunk( code ) {
            code = new MagicString( code );
            code.prepend( `console.warn( 'Scripts "build/three.js" and "build/three.min.js" are deprecated with r150+, and will be removed with r160. Please use ES Modules or alternatives: https://threejs.org/docs/index.html#manual/en/introduction/Installation' );\n` );
            return {
                    code: code.toString(),

            };
        }
    };
}
```

首先，导入了 `@rollup/plugin-terser` 压缩混淆工具和 `magic-string` 。

magic-string 主要用于轻微地替换代码的字符，比如页眉、页脚的位置。同时能够非常快速、方便地生成简单的 sourcemap 来定位代码。

紧接着定义了三个插件函数： `glsl` 、 `header` 和 `deprecationWarning` 。

- `glsl` 函数用于将代码中的 GLSL 片段转换为字符串，并进行一些处理，如移除注释和空行。

- `header` 函数用于在代码开头添加版权声明和许可证信息的注释。

- `deprecationWarning` 函数用于在代码开头添加一个警告，提示旧版本的脚本已被弃用，并提供了替代方案的链接。

最后，导出的函数接受一个参数 `args` ，根据参数的值返回相应的构建选项。如果 `args.configOnlyModule` 为 `true` ，则返回第一个构建选项，否则返回整个 `builds` 构建配置数组。这能让我们更灵活通过命令行解析的方式执行打包逻辑。

使用 Rollup 根据不同的构建选项构建 Three.js 库的不同版本，包括 ES 模块、CommonJS 模块和 UMD 模块。同时，通过插件函数对代码进行转换和添加额外的内容，如 GLSL 片段转换、版权声明和警告信息的添加。

CubicInterpolant 是三次插值，一个快速且简单的**三次样条插值**算法。它基于 Hermite 构造，通过在每个样本位置设置一阶导数为相邻位置之间的线性斜率来计算样本值。LinearInterpolant 是线性插值，通过计算**线性函数**来计算样本值。DiscreteInterpolant 是离散插值，离散插值是指在给定参数位置，**直接返回**离散样本值。相关插值算法可以直接在 Math 模块中阅读。

```js
// 根据形变目标序列创建动画片段
static CreateFromMorphTargetSequence(name, morphTargetSequence, fps, noLoop) {
        // 获取形变目标序列的长度
        const numMorphTargets = morphTargetSequence.length;
        // 创建一个数组用于存储轨道
        const tracks = [];
        // 遍历形变目标序列
        for (let i = 0; i < numMorphTargets; i++) {
                // 创建空数组用于存储关键帧的时间和值
                let times = [];
                let values = [];
                // 添加三个关键帧的时间和值，实现循环播放效果
                times.push(
                        (i + numMorphTargets - 1) % numMorphTargets,
                        i,
                        (i + 1) % numMorphTargets
                );
                values.push(0, 1, 0);
                // 根据关键帧的时间顺序进行排序
                const order = AnimationUtils.getKeyframeOrder(times);
                times = AnimationUtils.sortedArray(times, 1, order);
                values = AnimationUtils.sortedArray(values, 1, order);
                // 如果第一个关键帧的时间为0且不禁止循环播放，则将第一个关键帧的时间和值复制到最后一个关键帧
                if (!noLoop && times[0] === 0) {
                        times.push(numMorphTargets);
                        values.push(values[0]);
                }
                // 创建形变目标的关键帧轨道，并将时间缩放为 1.0 / fps
                tracks.push(
                        new NumberKeyframeTrack(
                                '.morphTargetInfluences[' + morphTargetSequence[i].name + ']',
                                times, values
                        ).scale(1.0 / fps)
                );
        }
        // 使用构造函数创建新的 AnimationClip 对象，传入名称、持续时间（-1 表示根据轨道自动计算）、tracks 数组作为参数，并返回新创建的 AnimationClip 对象
        return new this(name, -1, tracks);
}
```

```js
_lendAction(action) {
    // [ active actions |  inactive actions  ]
    // [  active actions >| inactive actions ]
    //                 s        a
    //                  <-swap->
    //                 a        s

    const actions = this._actions,
            prevIndex = action._cacheIndex,

            lastActiveIndex = this._nActiveActions++,

            firstInactiveAction = actions[lastActiveIndex];

    action._cacheIndex = lastActiveIndex;
    actions[lastActiveIndex] = action;

    firstInactiveAction._cacheIndex = prevIndex;
    actions[prevIndex] = firstInactiveAction;
}
```

```js
update(deltaTime) {
    // 时间缩放因子
    deltaTime *= this.timeScale;
    // 活动的动画片段数组 
    const actions = this._actions,
            // 动画片段的数量 
            nActions = this._nActiveActions,

            time = this.time += deltaTime,
            // timeDirection（根据  deltaTime  的正负确定）
            timeDirection = Math.sign(deltaTime),
            // 累积索引  accuIndex（通过异或操作进行切换）
            accuIndex = this._accuIndex ^= 1;

    // 更新动画
    for (let i = 0; i !== nActions; ++i) {

            const action = actions[i];

            action._update(time, deltaTime, timeDirection, accuIndex);

    }

    // 更新场景属性
    const bindings = this._bindings,
            nBindings = this._nActiveBindings;

    for (let i = 0; i !== nBindings; ++i) {
            bindings[i].apply(accuIndex);
    }
    return this;
}
```

```js
		return this;

	}
	// 用于更新音频的世界矩阵，并根据位置和方向信息更新 panner 对象。 
	updateMatrixWorld(force) {
		super.updateMatrixWorld(force);

		// 如果音频对象没有播放控制权限或者当前没有在播放，则直接返回
		if (this.hasPlaybackControl === true && this.isPlaying === false) return;

		// 将音频对象的世界矩阵分解为位置、四元数和缩放
		this.matrixWorld.decompose(_position, _quaternion, _scale);

		// 根据四元数计算出音频对象的方向向量
		_orientation.set(0, 0, 1).applyQuaternion(_quaternion);

		const panner = this.panner;

		// 如果浏览器支持 positionX 属性（例如 Chrome 和 Firefox），则采用线性插值的方式设置位置和方向，以实现平滑的声音过渡效果
		if (panner.positionX) {
			// 获取当前时间和监听器的时间差，用于计算线性插值的结束时间
			const endTime = this.context.currentTime + this.listener.timeDelta;

			// 使用线性插值将音频对象的位置在指定的时间内过渡到目标位置
			panner.positionX.linearRampToValueAtTime(_position.x, endTime);
			panner.positionY.linearRampToValueAtTime(_position.y, endTime);
			panner.positionZ.linearRampToValueAtTime(_position.z, endTime);

			// 使用线性插值将音频对象的方向在指定的时间内过渡到目标方向
			panner.orientationX.linearRampToValueAtTime(_orientation.x, endTime);
			panner.orientationY.linearRampToValueAtTime(_orientation.y, endTime);
			panner.orientationZ.linearRampToValueAtTime(_orientation.z, endTime);
		} else {
			// 如果浏览器不支持 positionX 属性，则直接设置位置和方向
			panner.setPosition(_position.x, _position.y, _position.z);
			panner.setOrientation(_orientation.x, _orientation.y, _orientation.z);
		}
	}

}
```

```js
import { Matrix4 } from '../math/Matrix4.js';
import { Object3D } from '../core/Object3D.js';

class Camera extends Object3D {
	constructor() {
		super();

		// 设置相机特有的属性
		this.isCamera = true; // 标记为相机对象
		this.type = 'Camera'; // 相机类型
		this.matrixWorldInverse = new Matrix4(); // 世界矩阵的逆矩阵
		this.projectionMatrix = new Matrix4(); // 投影矩阵
		this.projectionMatrixInverse = new Matrix4(); // 投影矩阵的逆矩阵
		this.coordinateSystem = WebGLCoordinateSystem; // 坐标系类型，默认为 WebGLCoordinateSystem
	}

	copy(source, recursive) {
		super.copy(source, recursive);

		// 复制相机特有的属性
		this.matrixWorldInverse.copy(source.matrixWorldInverse);
		this.projectionMatrix.copy(source.projectionMatrix);
		this.projectionMatrixInverse.copy(source.projectionMatrixInverse);
		this.coordinateSystem = source.coordinateSystem;

		return this;
	}

	getWorldDirection(target) {
		// 获取相机在世界坐标系中的方向，并取反
		return super.getWorldDirection(target).negate();
	}

	// 强制更新
	updateMatrixWorld(force) {
		super.updateMatrixWorld(force);

		// 更新世界矩阵的逆矩阵
		this.matrixWorldInverse.copy(this.matrixWorld).invert();
	}

	// 指定更新对象
	updateWorldMatrix(updateParents, updateChildren) {
		super.updateWorldMatrix(updateParents, updateChildren);

		// 更新世界矩阵的逆矩阵
		this.matrixWorldInverse.copy(this.matrixWorld).invert();
	}

	clone() {
		// 克隆相机对象
		return new this.constructor().copy(this);
	}
```

## core 核心库

- **Clock**：跟踪时间的流逝，用于动画和其他需要时间控制的操作。
- **EventDispatcher**：实现事件的分发和监听。
- **Object3D**：所有 3D 对象的基类。它提供了一些方法来管理和操作 3D 对象的属性、位置和旋转等。

## geometries 几何体

<p align=center><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c1f51653f0ca4b71b317bb4418487c98~tplv-k3u1fbpfcp-watermark.image?" alt="image.png"  /></p>

在该文件夹下的所有 Geomary 子类最终都是继承于 BufferGeometry， BufferGeometry 封装了公用的方法，比如索引、顶点、变换、法线、分组。

我们先来看 BoxGeometry 的源码：

```js
import { BufferGeometry } from '../core/BufferGeometry.js';
import { Float32BufferAttribute } from '../core/BufferAttribute.js';
import { Vector3 } from '../math/Vector3.js';

class BoxGeometry extends BufferGeometry {

	constructor(width = 1, height = 1, depth = 1, widthSegments = 1, heightSegments = 1, depthSegments = 1) {

		super();

		this.type = 'BoxGeometry';

		// 存储参数
		this.parameters = {
			width: width,
			height: height,
			depth: depth,
			widthSegments: widthSegments,
			heightSegments: heightSegments,
			depthSegments: depthSegments
		};
		// 存储作用域
		const scope = this;

		// 宽度、长度、高度的分段数量
		widthSegments = Math.floor(widthSegments);
		heightSegments = Math.floor(heightSegments);
		depthSegments = Math.floor(depthSegments);

		// buffers 配置（索引、顶点、法线、纹理）
		const indices = [];
		const vertices = [];
		const normals = [];
		const uvs = [];

		// 每个顶点包含的数量，用于开启新的顶点计算
		let numberOfVertices = 0;

		// 实际绘制顶点的开始（索引）
		let groupStart = 0;

		// 构建几何体的每一个面
		buildPlane('z', 'y', 'x', - 1, - 1, depth, height, width, depthSegments, heightSegments, 0); // px
		buildPlane('z', 'y', 'x', 1, - 1, depth, height, - width, depthSegments, heightSegments, 1); // nx
		buildPlane('x', 'z', 'y', 1, 1, width, depth, height, widthSegments, depthSegments, 2); // py
		buildPlane('x', 'z', 'y', 1, - 1, width, depth, - height, widthSegments, depthSegments, 3); // ny
		buildPlane('x', 'y', 'z', 1, - 1, width, height, depth, widthSegments, heightSegments, 4); // pz
		buildPlane('x', 'y', 'z', - 1, - 1, width, height, - depth, widthSegments, heightSegments, 5); // nz

		// 开始构建
		this.setIndex(indices);
		this.setAttribute('position', new Float32BufferAttribute(vertices, 3));
		this.setAttribute('normal', new Float32BufferAttribute(normals, 3));
		this.setAttribute('uv', new Float32BufferAttribute(uvs, 2));

		/**
		 * 构建平面 
		 *  
		 * @param {string} u 纹理坐标的横向分量 
		 * @param {string} v 纹理坐标的纵向分量 
		 * @param {string} w 纹理坐标的深度分量 
		 * @param {number} udir u方向的正负标志 
		 * @param {number} vdir v方向的正负标志 
		 * @param {number} width 平面的宽度 
		 * @param {number} height 平面的高度 
		 * @param {number} depth 平面的深度 
		 * @param {number} gridX 横向分段数 
		 * @param {number} gridY 纵向分段数 
		 * @param {number} materialIndex 材质索引 
		 */
		function buildPlane(u, v, w, udir, vdir, width, height, depth, gridX, gridY, materialIndex) {

			// 细分面的宽度和高度
			const segmentWidth = width / gridX;
			const segmentHeight = height / gridY;

			// 细分面的中心位置
			const widthHalf = width / 2;
			const heightHalf = height / 2;
			const depthHalf = depth / 2;

			const gridX1 = gridX + 1;
			const gridY1 = gridY + 1;

			// 当前坐标起始（包含顶点、法线、uv）
			let vertexCounter = 0;

			// 当前计算的细分面的顶点开始位置
			let groupCount = 0;

			// 实例化向量
			const vector = new Vector3();

			// 循环生成顶点、法线、uv
			for (let iy = 0; iy < gridY1; iy++) {

				const y = iy * segmentHeight - heightHalf;

				for (let ix = 0; ix < gridX1; ix++) {

					const x = ix * segmentWidth - widthHalf;

					// 根据uv的正负朝向来得到实际的坐标
					vector[u] = x * udir;
					vector[v] = y * vdir;
					vector[w] = depthHalf;

					// 生成顶点坐标
					vertices.push(vector.x, vector.y, vector.z);

					// 根据depth生成实际的法线朝向
					vector[u] = 0;
					vector[v] = 0;
					vector[w] = depth > 0 ? 1 : - 1;

					// 生成法线坐标
					normals.push(vector.x, vector.y, vector.z);

					// uvs
					uvs.push(ix / gridX);
					uvs.push(1 - (iy / gridY));

					// counters
					vertexCounter += 1;
				}

			}

			// 创建索引，实际1个细分面由两个三角面组成，每个三角面对应3个索引，所以每组6个顶点的索引
			for (let iy = 0; iy < gridY; iy++) {

				for (let ix = 0; ix < gridX; ix++) {

					const a = numberOfVertices + ix + gridX1 * iy;
					const b = numberOfVertices + ix + gridX1 * (iy + 1);
					const c = numberOfVertices + (ix + 1) + gridX1 * (iy + 1);
					const d = numberOfVertices + (ix + 1) + gridX1 * iy;

					// 注意这6个坐标，b和d是细分面的相交线，所以b和d已经通过索引被复用
					indices.push(a, b, d);
					indices.push(b, c, d);

					groupCount += 6;
				}
			}

			// 用来支持每个细分面实现不同材质的功能
			scope.addGroup(groupStart, groupCount, materialIndex);

			groupStart += groupCount;
			numberOfVertices += vertexCounter;

		}

	  }
        //...
}
```

该核心通过调用 buildPlane 函数来构建立方体的每个面。该函数根据传入的参数生成顶点坐标、法线和纹理坐标，并将它们存储在对应的数组中。同时，根据顶点坐标生成顶点索引，以便后续绘制面。最后，通过调用父类 BufferGeometry 的方法，将顶点索引、顶点坐标、法线和纹理坐标设置为几何体的属性。 

而其他的几何体原理也相似，区别无非是顶点坐标、法线、uv、索引的计算不同。

```js
import { Color } from '../math/Color.js';

class GridHelper extends LineSegments {

        constructor(size = 10, divisions = 10, color1 = 0x444444, color2 = 0x888888) {

                color1 = new Color(color1);
                color2 = new Color(color2);

                const center = divisions / 2;
                const step = size / divisions;
                const halfSize = size / 2;

                const vertices = [], colors = [];

                for (let i = 0, j = 0, k = - halfSize; i <= divisions; i++, k += step) {

                        vertices.push(- halfSize, 0, k, halfSize, 0, k);
                        vertices.push(k, 0, - halfSize, k, 0, halfSize);

                        const color = i === center ? color1 : color2;

                        color.toArray(colors, j); j += 3;
                        color.toArray(colors, j); j += 3;
                        color.toArray(colors, j); j += 3;
                        color.toArray(colors, j); j += 3;

                }

                const geometry = new BufferGeometry();
                geometry.setAttribute('position', new Float32BufferAttribute(vertices, 3));
                geometry.setAttribute('color', new Float32BufferAttribute(colors, 3));

                const material = new LineBasicMaterial({ vertexColors: true, toneMapped: false });

                super(geometry, material);

                this.type = 'GridHelper';
        }
        // 销毁几何体、材料
        dispose() {
                this.geometry.dispose();
                this.material.dispose();
        }
}
```

GridHelper 实现原理根据 size 和 divisions 计算出网格的顶点坐标和颜色数组，创建一个 BufferGeometry 对象，并使用顶点坐标和颜色数组设置其属性，最后附加上 `LineBasicMaterial` 材料来修改 color 颜色坐标值。

这里需要注意 GridHelper 的继承关系是 `GridHelper -> LineSegments -> Line -> Object3D`（线细分图元），所以我们可以调用父类的方法，比如计算线的距离、计算线与射线的交点、形变、矩阵应用等。

当然还包括其他几何体的 helper，主要用于查看如何分面。它们的实现原理也是通过传入的 Object3D，得到所有细分面属性后，在外层绘制线框类型的 Object3D。

```js
class ImageLoader extends Loader {

	constructor( manager ) {

		super( manager );

	}

	load( url, onLoad, onProgress, onError ) {

		if ( this.path !== undefined ) url = this.path + url;

		url = this.manager.resolveURL( url );

		const scope = this;

		const cached = Cache.get( url );

		if ( cached !== undefined ) {

			scope.manager.itemStart( url );

			setTimeout( function () {

				if ( onLoad ) onLoad( cached );

				scope.manager.itemEnd( url );

			}, 0 );

			return cached;

		}

		const image = createElementNS( 'img' );

		function onImageLoad() {

			removeEventListeners();

			Cache.add( url, this );

			if ( onLoad ) onLoad( this );

			scope.manager.itemEnd( url );

		}

		function onImageError( event ) {

			removeEventListeners();

			if ( onError ) onError( event );

			scope.manager.itemError( url );
			scope.manager.itemEnd( url );

		}

		function removeEventListeners() {
			image.removeEventListener( 'load', onImageLoad, false );
			image.removeEventListener( 'error', onImageError, false );
		}

		image.addEventListener( 'load', onImageLoad, false );
		image.addEventListener( 'error', onImageError, false );

		if ( url.slice( 0, 5 ) !== 'data:' ) {
			if ( this.crossOrigin !== undefined ) image.crossOrigin = this.crossOrigin;
		}

		scope.manager.itemStart( url );
		image.src = url;
		return image;
	}

}
```

首先，该类继承自 Loader 类，用于加载资源。我们可以调用父类的构造函数，传入 manager 给父类来实现自定义的不同资源的统一加载管理。

load 方法用于加载图像资源。它接收 url、onLoad、onProgress 和 onError 等参数。首先判断是否设置了 this.path 属性，如果设置了，则将 url 与 this.path 拼接起来。然后通过 this.manager.resolveURL 方法解析 url，得到最终的资源 URL。

接下来，通过 Cache.get 方法尝试从缓存中获取已加载的图像资源。如果缓存中存在该资源，则直接调用 onLoad 回调函数，并返回缓存中的图像资源。

如果缓存中不存在该资源，则创建一个 img 元素，并定义 onImageLoad 和 onImageError 两个回调函数。onImageLoad 函数在图像加载成功时被调用，将图像添加到缓存中，并调用 onLoad 回调函数。onImageError 函数在图像加载失败时被调用，调用 onError 回调函数。

然后，通过 addEventListener 方法给 img 元素添加 load 和 error 事件的监听器，并在事件发生时执行对应的回调函数。如果 url 不是以 "data:" 开头的，且设置了 this.crossOrigin 属性，则将 img 元素的 crossOrigin 属性设置为 this.crossOrigin。

最后调用 `this.manager.itemStart` 方法开始加载资源，并将 img 元素的 src 属性设置为 url，开始加载图像并返回 img 元素。

另外还有一个重要的 loader 就是 ObjectLoader，我们都可以通过 FileLoader 解析出字符串，然后转换成 json 格式，注意这里只支持 JSON、GLTF、OBJ 格式。其他比如 3DS、SKP、FBX 等我们需要用其他 loader 做转换。

有了 json 后，我们只需要解析出属性与顶点坐标、颜色、uv、索引、法线的映射关系后，设置到 bufferAttribute 中即可，最后通过渲染器渲染，这就是模型导入的原理。

```js
		const x = this.x - center.x;
		const y = this.y - center.y;
		this.x = x * c - y * s + center.x;
		this.y = x * s + y * c + center.y;
		return this;
	}

		this.y = Math.random();
		return this;
```

```js
    if (camera !== undefined && camera.isCamera !== true) {
        console.error('THREE.WebGLRenderer.render: camera is not an instance of THREE.Camera.');
        return;
    }

    if (_isContextLost === true) return;

    // 更新场景图
    if (scene.matrixWorldAutoUpdate === true) scene.updateMatrixWorld();

    // 更新相机矩阵和视锥体
    if (camera.parent === null && camera.matrixWorldAutoUpdate === true) camera.updateMatrixWorld();

    if (xr.enabled === true && xr.isPresenting === true) {
        if (xr.cameraAutoUpdate === true) xr.updateCamera(camera);
        camera = xr.getCamera(); // 使用 XR 相机进行渲染
    }

    if (scene.isScene === true) scene.onBeforeRender(_this, scene, camera, _currentRenderTarget);

    currentRenderState = renderStates.get(scene, renderStateStack.length);
    currentRenderState.init();
    renderStateStack.push(currentRenderState);

    _projScreenMatrix.multiplyMatrices(camera.projectionMatrix, camera.matrixWorldInverse);
    _frustum.setFromProjectionMatrix(_projScreenMatrix);

    _localClippingEnabled = this.localClippingEnabled;
    _clippingEnabled = clipping.init(this.clippingPlanes, _localClippingEnabled);

    currentRenderList = renderLists.get(scene, renderListStack.length);
    currentRenderList.init();
    renderListStack.push(currentRenderList);

    projectObject(scene, camera, 0, _this.sortObjects);
    currentRenderList.finish();

    if (_this.sortObjects === true) {
        currentRenderList.sort(_opaqueSort, _transparentSort);
    }

    this.info.render.frame++;

    if (_clippingEnabled === true) clipping.beginShadows();
    const shadowsArray = currentRenderState.state.shadowsArray;
    shadowMap.render(shadowsArray, scene, camera);
    if (_clippingEnabled === true) clipping.endShadows();

    if (this.info.autoReset === true) this.info.reset();

    background.render(currentRenderList, scene);

    currentRenderState.setupLights(_this._useLegacyLights);

    if (camera.isArrayCamera) {
        const cameras = camera.cameras;
        for (let i = 0, l = cameras.length; i < l; i++) {
            const camera2 = cameras[i];
            renderScene(currentRenderList, scene, camera2, camera2.viewport);
        }
    } else {
        renderScene(currentRenderList, scene, camera);
    }

    if (_currentRenderTarget !== null) {
        // 如果使用了任何类型的 mipmap 过滤，将多样本渲染缓冲区解析为单样本纹理

        textures.updateRenderTargetMipmap(_currentRenderTarget);

    bindingStates.resetDefaultState();
    _currentMaterialId = -1;
    _currentCamera = null;

        currentRenderList = renderListStack[renderListStack.length - 1];
    } else {
        currentRenderList = null;
    }
```

CompressedTexture 是一种用于存储和渲染压缩图像的纹理类型。它的实现原理也就是 minmaps。Mipmaps 是一种预先生成的纹理图像序列，每个级别的纹理图像都是原始图像的一半大小。这种级别的纹理图像被存储在压缩纹理中，以便在渲染时使用。

在渲染过程中，当物体离相机较远或较小时，较低层级的 mipmaps 会被使用，这样可以节省计算资源和存储带宽。而当物体靠近相机或变大时，较高层级的 mipmaps 会被使用，以提供更多的细节和质量。这对于移动设备和实时渲染应用程序特别有用，可以提高性能和效率。



## 总结

最后，我们总结下该章节 three.js 的源码知识点：

- **animation**：动画核心在于对值和时间的分割成满足不同插值算法的关键帧序列，在 Mixer 中做注册一个或多个的 clip 并管理新的序列，然后在 Mixer 中执行 action 方法更新状态，在 update 中更新渲染。

- **audio**：音频的核心在于原生的 AudioContext 对象的逻辑控制。为了能实现位置衰减，AudioListener 和 PositionalAudio 需要继承 Object3D 基类。
- **cameras**：相机的实现核心在于模型视图投影矩阵。
- **core**：我们需要理解 WebGL 是如何处理 Buffer 以及为什么 Three 需要封装实例化类。
- **extras**：封装基础的图元坐标点的计算和辅助方法。
- **geometries**：绘制不同几何体的核心在于计算不同的顶点坐标、法线、uv、索引。
- **helpers**：在一些无法看到灯光最大范围、尺寸、衰减范围的情况下，我们会使用对应的 helpers 来帮助我们观察，另外还有几何体 helper、辅助线 helper 等。
- **lights**：灯光实现的核心在于法线向量和光源方向向量的点乘计算得到的光强度值来修改片元着色器的颜色坐标值。
- **loaders**：loader 核心在于文件的导入异步加载后状态处理以及对文件内容的解析。比如模型我们需要 objectLoader 解析出 json 然后将属性解析不同的法线、顶点等，然后赋值到 bufferAttribute，最后在 Renderers 中渲染显示在屏幕上。
- **materials**：材质类主要做了基础属性的封装，核心在于渲染器解析其中的材质配置然后赋值于 uniform 中，在 glsl 中传入的 unifrom 常量最终和 textures 结合形成材质纹理。
- **math**：在 math 中主要封装了我们常用的 Color 类的封装、MathUtils 工具方法、射线类、矩阵类、插值计算类等。
- **objects**：图元本质就是基础的封装，而具体的形体的坐标都放在了 geometries 中。
- **renderers**：渲染器的核心在于 WebGLRenderer 的初始化（屏幕尺寸、渲染堆栈、读取渲染器配置、webgl 模块、webxr 模块等）以及其中的 render 方法。
- **scenes**：scenes 相当于最顶层的 Object3D，我们可以配置背景、环境、雾等全局属性。
- **textures**：textures 是其他纹理的基类，其他子类包括 canvas 纹理、视频纹理、压缩纹理等。

通过 Three 源码的学习，我们对 Three 底层的运转逻辑有了基础的认知，这样当我们遇到报错问题的时候，我们能快速定位到源码的具体位置并能分析错误原因。另外，我们在后续 WebGL 的深入学习后再回过头来反复看的时候，就能更容易理解那些复杂的运算、向量、矩阵、光照等为什么要这么做、为什么要这么封装。又或者当我们想要封装一个我们自己的 3d 渲染引擎的时候，我们可以借鉴 three 的架构思想。
