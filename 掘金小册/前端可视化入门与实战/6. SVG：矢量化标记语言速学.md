
大部分人只知道 SVG 是基于 XML 的标记语言，然后可以绘制一些简单的矢量图形，但是掌握这些是远远不够的，本节会以`概念`、`高级特性`、`开发技巧`等维度来帮助你深入理解 SVG。

## 为什么使用 SVG

**SVG 是一种基于 XML 的可扩展、轻量级、基于文本且高效的图形标记语言**。它易于编辑、动画和集成。重要的是，除了 Internet Explorer 8 和 Android 2.3 之外，几乎所有浏览器都支持它们。

作为矢量图像，SVG 由数学方程式组成，这些方程式定义了构成 XML 格式图形形状和文本的线条和曲线的位置和颜色。在设计领域，SVG 用于图标、徽标、图形设计和字体。

SVG 最大的特征就是**矢量化**。

 -   **无损放缩**：矢量图形可以无限放大或缩小而不失真，因为它们是基于数学公式计算的。
 -   **小文件大小**：相对于像素图形，矢量图形通常具有较小的文件大小，因为它们只需存储图形的数学描述。
 -   **可编辑性**：由于矢量图形是由数学公式定义的，因此可以轻松地编辑和修改其形状、颜色和属性。
 -   **清晰度**：矢量图形具有高分辨率和平滑的边缘，无论其显示尺寸如何，都能保持清晰度。

接着让我们再说明一下 SVG 的一些`优点`。

1. **基于文本的格式**。SVG 元素包含文本，这大大提高了网站的可访问性。但主要优点是此文本被搜索引擎索引。用户可以通过谷歌找到 SVG 文件。

2. **可扩展性**。SVG 图像的质量不取决于分辨率。与其他格式或图标字体的图像不同，SVG 在任何屏幕尺寸的任何设备上看起来都非常清晰。可扩展性还意味着，如果你在整个网站中使用相同的图像但大小不同，则使用单个 SVG。你不必像 PNG 那样创建它的多个副本。相反，你可以嵌入相同的图像并直接在 SVG 代码中定义它的大小。

3. **高性能**。如果优先考虑性能，则应使用 SVG。使用 SVG，无需在图像文件中加载 HTTP 请求。页面加载速度更快，因为它没有要下载的文件。更快的加载时间转化为更好的网页性能和更高的搜索引擎排名。反过来，它改善了用户体验。

4. **文件大小**。简单 SVG 文件的大小由它包含的颜色、图层、渐变、效果和蒙版定义。PNG 或任何其他光栅图形文件的大小由它所包含的像素数定义。PNG 图像越大，其大小就越重。但是，SVG 图标的情况并非如此。此外，SVG 可以优化，我将在本文后面介绍如何优化。

5. **更好的集成**。与光栅图像不同，矢量图像既可以在特殊的矢量绘图程序中编辑，也可以直接在文本编辑器中编辑。还可以直接通过 CSS 编辑 SVG 图标的颜色或大小。至于 SVG 的动画，可以在 SMIL、Web Animations API、WebGL 或 CSS 动画的帮助下完成。由于 SVG 天然跟 Web 相关技术集成，你可以用 JavaScript 来处理逻辑，使用 CSS 来修改大小、样式等。

当然，SVG 也是有`缺点`的。

图像包含的细节部分越多，它的图形体积就越大。

如果图片是线性的并且包含几种颜色，SVG 是一个解决方案。但是，如果细节很重要并且有很多细节，PNG 或 JPEG 可能更合适。

另外需要注意，SVG 不能用于照片。如果你在网站上使用照片，SVG 不是最佳选择。你应该使用位图。

> **位图**是一种图像表示方法，也被称为栅格图像或点阵图像。 
> 
> 它由一个矩形网格（像素阵列）组成，每个像素都包含一系列颜色值。每个像素的颜色值决定了图像在该位置上的颜色和亮度，这使得位图图像在放大或缩小时会出现像素化或失真的问题，因为像素的大小和位置是固定的。
> 
> 位图图像通常用于表示照片、复杂的图形和图像细节较多的情况。



## SVG 的使用

要创建 SVG，我们可以在文本编辑器中创建一个新文件并为其指定扩展名 “.svg”。

```html
<svg  width="256" height="256" viewBox="0 0 256 256"xmlns="http://www.w3.org/2000/svg"> <!-- Write all the SVG code in here --> </svg>
```

这里需要注意的是，`xmlns` 是 XML 命名空间属性，用于指定所使用的 XML 命名空间。命名空间是一种用于区分不同 XML 元素和属性的机制，以确保不同来源的 XML 数据可以正确解析和处理。但是，如果嵌入在 HTML 内部，xmlns 属性是可以被省略的。

下面我们分别绘制矩形和圆形：

```html
<svg width="256" height="256" viewBox="0 0 256 256" xmlns="http://www.w3.org/2000/svg">       <rect x="10" y="10" width="300" height="300" />
    <circle cx="175" cy="100" r="75" fill="red" />
</svg>
```

x 和 y 代表坐标位置，width 和 height 代表宽高，r 代表半径，cx 和 cy 代表圆心。除此之外，我们还可以绘制椭圆、曲线、折线、多边形等。

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7ce7d9e8f8144a5d8248667af25877a0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=342&h=338&s=1848&e=webp&b=000000" alt="image.png"  /></p>

当我们仔细观察绘制效果，在 SVG 中也像 Canvas 一样是绘画模式的，即每一次绘制操作，都会覆盖早期的操作。所以，在后续的开发中我们要极其注意结构顺序。

紧接着如果要修改颜色，我们可以直接增加 fill 属性，但其实我们可以直接在 CSS 上修改，比如这样：

```js
<svg width="256" height="256" viewBox="0 0 256 256" xmlns="http://www.w3.org/2000/svg">
  <style>
    rect {
      fill: #f33;
    }
  </style>
  <rect x="10" y="10" width="75" height="50" />
</svg>
```

> 这里有一个比较细节的地方，就是 CSS 属性的权重是比 SVG 的属性大的。

我们看到 CSS 能影响 SVG，那么除此之外呢，比如添加悬停效果、添加过渡、添加动画等等，我们可以在 [Property Index — SVG 2 (w3.org)](https://link.juejin.cn/?target=https%3A%2F%2Fwww.w3.org%2FTR%2FSVG%2Fpropidx.html "https://www.w3.org/TR/SVG/propidx.html") 中查看所有可影响 SVG 的 CSS 属性。

```css
.my-rect { fill: #f33; } .my-rect:hover { fill: #000; }
```


### SVG 过滤器

另外，我们在 SVG 中也可以像 CSS 一样使用过滤器，比如下面通过 id 自定义的高斯模糊滤镜，然后在 SVG 元素中通过 filter 的 url 来引用。

```html
<svg width="500" height="500" viewBox="0 0 500 500">
    <filter id="blur">
        <feGaussianBlur in="SourceGraphic" stdDeviation="5" />
    </filter>
    <circle cx="170" cy="60" r="50" fill="green" filter="url(#blur)" />
</svg>
```

### SVG 遮罩

同时，我们也可以使用遮罩 Mask，通过 mask 标签，引用 id 为 mask，然后在 image 标签通过 `xlink:href` 来引用遮罩图片，黑色是将隐藏的区域，白色是将显示的区域，通过这个特性我们可以实现修图软件的蒙版效果。

比如，下面这两张图：

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de2665046ded4534a76b0ee9579e9b9e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1121&h=433&s=19050&e=webp&b=faefee" alt="image.png"  /></p>

代码如下：

```html
<svg width="500" height="500" viewBox="0 0 500 500">
    <defs>
        <mask id="mask">
            <image width="500" height="500" xlink:href="./mask.png" />
        </mask>
    </defs>
    <image mask="url(#mask)" width="500" height="500" xlink:href="./img.png" />
</svg>
```

效果如下：

> **学到这里，我们有必要讲清楚容易搞混的 xmlns、xmlns:xlink、xlink:href**。
> 
> xmlns 和 xmlns:xlink 是 XML 命名空间的属性，而 xlink:href 是 XLink 规范中的属性。它们之间的区别如下：
>
> 1.  xmlns 属性用于定义 XML 文档中的默认命名空间。它指定了 XML 元素和属性的命名空间，如果没有指定其他命名空间，则默认使用该命名空间。
> 
> 1.  xmlns:xlink 属性也是用于定义 XML 文档中的命名空间，但它是为了支持 XLink（XML 链接）规范而定义的。xmlns:xlink 属性指定了用于解析和处理 XLink 链接的命名空间。
> 1.  xlink:href 是 XLink 规范中的属性，用于指定链接的目标。它是在具有 XLink 命名空间的 XML 文档中使用的属性，通过该属性可以指定链接的目标资源。
>   
>     **综上所述，xmlns 属性用于定义默认命名空间，xmlns:xlink 属性用于定义 XLink 命名空间，而 xlink:href 属性用于指定 XLink 链接的目标资源。**



## 如何优化 SVG 图像

渲染矢量格式时，我们必须编写一些额外的 SVG 代码。最终结果应使用不同的服务进行优化。

大多数情况下，为了优化 SVG，我们可以使用 [SVGO](https://link.juejin.cn/?target=https%3A%2F%2Fgithub.com%2Fsvg%2Fsvgo "https://github.com/svg/svgo")。

`SVG Optimizer `是一个基于 Node.js 的工具，用于优化 SVG 矢量图形文件，可以针对元数据、注释、隐藏元素、默认值或非最佳值以及其他可以安全删除或转换的内容，而不会影响 SVG 渲染结果。



## Web 端引入 SVG 的不同方式

1. **img**。这个是将 SVG 图像添加到网页的最简单方法。若要使用此方法，请将元素添加到 HTML 文档中并在属性中引用它，如下所示：

> `<img>`尽管我们可以更改通过标签添加的 SVG 图像的大小，但如果你想对 SVG 图像进行重大样式更改，仍然存在一些限制。

```js
<img src = "happy.svg" alt="My Happy SVG"/>
```


2. **background-image**。如下：

```js
body {
  background-image: url(happy.svg);
}
```

3. **内联 SVG**。SVG 图像可以使用标签直接写入 HTML 文档。`<svg> </svg>`。

4. **object**。你还可以通过 HTML 元素使用以下代码语法将 SVG 图像添加到网页：

`<object>`

```html
<object data="happy.svg" width="300" height="300"> </object>
```

5. **iframe**。使用此格式添加的 SVG 图像不可缩放，同时 SEO 也会有问题。

6. **embed**。我们在 HTML 中可以使用嵌入的方式引用，比如这样：`<embed><embed src="happy.svg" />`，但是该标签已经在大多数浏览器弃用了，不再推荐使用。




## 高级应用：结构、分组和引用

SVG 带有自己的方法来通过某些 SVG 元素构建文档，这些元素允许我们在文档中定义、分组和引用对象。这些元素使重用元素变得容易，同时保持干净和可读的代码。在本文中，我们将介绍这些元素，强调它们之间的区别以及每个元素的优点。

我们首先来定义一个由多个元素组成的图形——笑脸。如下：

```html
<circle cx="100" cy="100" r="50" fill="yellow" stroke="black" stroke-width="10" />
<!-- 绘制眼睛 -->
<circle cx="80" cy="85" r="7" fill="black" />
<circle cx="120" cy="85" r="7" fill="black" />

<!-- 绘制嘴巴 -->
<path d="M80 120 Q100 140 120 120" fill="none" stroke="black" stroke-width="8" stroke-linecap="round" />
```

### 元素分组

上面的笑脸 icon，比如在项目肯定会有很多地方需要被引用，这个时候我们可以通过 g 标签来做分组，在 defs 标签下代表被引用的资源，这个时候我们在 g 标签上定义引用的 id，接着通过 use 标签的`xlink:href="#myCircle"`，来引用对应的资源。

比如下面这样：

```html
<use xlink:href="#myCircle" />
<use xlink:href="#myCircle" x="200" /> // 为了让实例有单独的位置属性等可以单独修改
<use xlink:href="#myCircle" y="200" />
<use xlink:href="#myCircle" x="200" y="200" />
```

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/538f5ba228a24e878c325a03b0a61657~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=449&h=438&s=14158&e=webp&b=fefefe" alt="image.png"  /></p>


对元素进行分组非常有用，不仅用于组织和结构目的。当你想要向由多个项目组成的 SVG 图形添加交互性或转换时，它特别有用。你可以将这些项目关联到一个组中，然后定义变换以一起移动、缩放或旋转所有项目，以便保持它们彼此之间的空间关系。

```css
    #myCircle {
        transform: scale(1.4);
    }
```


但是分组有个缺点，我们无法对每个实例覆盖原始元素的样式，比如填充和描边。但是我们可以像下面这么做，再定义一个 defs 模板被引用，在每个实例上单独传递 props。

```js
<svg xmlns="http://www.w3.org/2000/svg" width="800" height="800">
    <defs>
        <!-- 定义一个圆形 -->
        <g id="myCircle">
            <circle cx="100" cy="100" r="50" stroke="black" stroke-width="10" />
            <!-- 绘制眼睛 -->
            <circle cx="80" cy="85" r="7" fill="black" />
            <circle cx="120" cy="85" r="7" fill="black" />

            <!-- 绘制嘴巴 -->
            <path d="M80 120 Q100 140 120 120" fill="none" stroke="black" stroke-width="8" stroke-linecap="round" />
        </g>

    </defs>
    <defs>
        <linearGradient id="gradient">
            <stop offset="0%" style="stop-color: rgb(255, 93, 12)" />
            <stop offset="100%" style="stop-color: #ff0000" />
        </linearGradient>
    </defs>
    <!-- 使用定义的圆形 -->
    <use xlink:href="#myCircle" fill="url(#gradient)" />
    <use xlink:href="#myCircle" x="200" fill="yellow" />
    <use xlink:href="#myCircle" y="200" fill="yellow" />
    <use xlink:href="#myCircle" x="200" y="200" fill="yellow" />
</svg>
```

### Symbol 视口引用标签

symbol 和 g 一样可以用于元素的分组，区别在于它可以缩放以适合由引用元素定义的矩形视口。比如，我们可以使用 viewBox。

下面是具体的代码示例：

```js

    <use xlink:href="#myShape" x="50" y="50" />
    <use xlink:href="#myShape" x="150" y="50" />
</svg>
```




## 高级应用：SVG 适配

SVG 适配是一种高级应用，用于实现可伸缩矢量图形在不同屏幕尺寸和设备上的自适应和响应式布局。下面将深入讲解视窗、视图框、preserveAspectRatio。

### 视窗

视窗一般我们视为 SVG 画布的宽高尺寸，如果未定义这些值，则视口的尺寸通常由 SVG 中的其他元素确定，例如最外层 SVG 元素的宽度。

### 视图框 ViewBox

使用 `viewBox` 属性可以控制 SVG 元素的视图范围，使其适应不同的显示设备和尺寸。通过调整 `viewBox` 的值，可以缩放、平移和裁剪 SVG 内容，以适应不同的需求和场景。

`viewBox` 中包含四个值的字符串，分别表示：`x` 、 `y` 、 `width` 和 `height` 。

-   `x` 是可视区域左上角的横坐标。
-   `y` 是可视区域左上角的纵坐标。
-   `width` 是可视区域的宽度。
-   `height` 是可视区域的高度。

```html
<body>
    <svg width="100" height="100" viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
        <circle cx="100" cy="100" r="100" fill="red" />
    </svg>
</body>
```

`viewBox="0 0 100 100"` 表示可视区域的起点坐标为 `(0, 0)` ，宽度为 `100` ，高度为 `100` 。这意味着 SVG 元素的内容将在一个 `100x100` 的矩形区域内显示，而实际圆形是 `200x200`。

### preserveAspectRatio 宽高比

默认情况下，SVG 画布的纵横比将被保留，所以会导致视口内图形被裁剪。但是，我们可以通过 [preserveAspectRatio](https://link.juejin.cn/?target=https%3A%2F%2Fsvgwg.org%2Fsvg2-draft%2Fcoords.html%23PreserveAspectRatioAttribute "https://svgwg.org/svg2-draft/coords.html#PreserveAspectRatioAttribute") 属性指定不同的行为。

`preserveAspectRatio` 是 SVG（可缩放矢量图形）中的属性，用于控制元素在视口中的缩放和对齐方式，以保持其纵横比。

`preserveAspectRatio` 属性的值由两部分组成，用空格分隔。

1.  **对齐方式（Alignment）**：指定元素在视口中的对齐方式，可以是以下值之一。

    -   `none` ：不保持纵横比，直接拉伸元素以填充整个视口。
    -   `xMinYMin` ：保持纵横比，将元素缩放到视口内并保持在左上角。
    -   `xMidYMid` ：保持纵横比，将元素缩放到视口内并居中显示。
    -   `xMaxYMax` ：保持纵横比，将元素缩放到视口内并保持在右下角。
    -   `xMinYMid` ：保持纵横比，将元素缩放到视口内并保持在左边垂直居中。

2.  **缩放方式（Meet or Slice）**：指定元素在视口中的缩放方式，可以是以下值之一。

    -   `meet` ：缩放元素以适应视口，并保持元素完全可见，可能会留有空白区域。
    -   `slice` ：缩放元素以填充视口，可能会裁剪掉部分元素。例如， `preserveAspectRatio="xMidYMid meet"` 表示元素将保持纵横比，缩放到视口内并在视口中居中显示，同时保持元素完全可见。 `preserveAspectRatio` 属性通常应用于 `<svg>` 元素或 `<image>` 元素，以控制其在视口中的显示方式。它允许响应式地调整 SVG 元素，以适应不同的屏幕尺寸和显示环境。

通过调整 `preserveAspectRatio` 的值，可以根据需要来控制 SVG 元素的缩放和对齐方式，以实现所需的显示效果。比如，我们对图三的 SVG 加上属性 `preserveAspectRatio="xMinYMin meet"`，此时效果就跟图一一样了。

## 高级应用：SVG 动画

在我们平常开发中，SVG 动画在图标动效设计中是十分重要的，比如加载进度条时候的 icon、菜单栏各种 icon、还有骨架屏我们会用 SVG 的渐变实现。我会分别从动画实现技术方案、各标签功能解析、实战案例来讲解。


### SVG 动画的不同实现

1.  **SMIL**：一种 XML 语言，用于描述多媒体元素的时间和空间关系。在 SVG 中，可以使用 SMIL 动画来创建平移、旋转、缩放、颜色变化等效果。通过在元素上添加 `<animate>` 元素，设置属性的起始值和结束值，以及动画的持续时间和重复次数。

2.  **JavaScript**：使用 JavaScript 可以通过操作 SVG 元素的属性和样式来创建动画效果。可以使用 `requestAnimationFrame` 或 `setInterval` 函数来更新元素的属性值，从而实现平滑的动画效果。
3.  **CSS 动画**：与 HTML 中的 CSS 动画类似，SVG 也可以使用 CSS 来创建动画效果。通过在 SVG 元素上应用 CSS 类或样式，使用 `@keyframes` 规则定义动画的关键帧，然后使用 `animation` 属性来指定动画的名称、持续时间、重复次数等。
4.  **SMIL 动画的替代方案**：由于浏览器对 SMIL 动画的支持逐渐减弱，可以使用一些 JavaScript 库，如 GreenSock Animation Platform（GSAP），来创建更复杂和兼容性更好的 SVG 动画。


### `<animate>、<animateTransform>、<animateMotion>` 的区别

1.   `<animate>`：用于在 SVG 中创建`基本的属性动画`。它可以用于对元素的位置、大小、颜色等属性进行动画处理。通过指定起始值和结束值，以及动画的持续时间和重复次数，可以实现属性的渐变过渡效果。

2.  `<animateTransform>`：用于对 SVG 元素的`变换属性`进行动画处理，例如平移、旋转、缩放等。它可以通过指定起始值和结束值，以及动画的持续时间和重复次数，实现变换属性的平滑过渡动画。
3.  `<animateMotion>`：用于在 SVG 中`创建沿路径移动的动画效果`。它可以使元素沿着指定的路径进行运动，并可以控制运动的速度、方向和重复次数。通过指定路径和动画的持续时间，可以实现元素沿路径移动的动画效果。

animateMotion 在之前的路径动画已经演示过，接下来我将演示 animate 和 animateTransform。


### animate 播放动画

播放动画我们只要定义 5 个 rect 矩形，通过`attributeName`定义为 height 的过渡变化，`dur` 定义 1s 持续时间，`repeatCount` 定义循环播放，`begin`做不同的延迟播放来实现动态律动的效果，`values` 属性定义了动画的值序列，这里是一个循环的序列，从 30 增加到 100 再减少到 30。

## 总结

最后，我们总结下本节学到的知识点：

1.  **SVG 是什么**：SVG 是可缩放矢量图形，用于描述二维图形和动画的基于 XML 标记语言。
1.  **SVG 的优势**：体积小、不失真、Web 端高度集成、兼容性好。
1.  **SVG 使用方法**：可以使用文本编辑器创建 SVG 代码，或使用图形编辑软件导出为 SVG 格式。使用`<svg>`标签包裹图形元素，设置属性和样式。
1.  **SVG 和 CSS 的关系**：CSS 可以应用于 SVG 元素，通过样式属性或外部样式表进行样式控制，包括颜色、填充、边框、动画等。
1.  **如何优化 SVG**：压缩 SVG 代码、合并路径、减少节点数量、使用符号和引用、避免过度渲染等，以减小文件大小和提高性能。我们可以使用 `SVGO` 解决。
1.  **SVG 适配**：使用响应式设计和媒体查询，设置视口和`viewBox`属性，使用百分比单位和媒体查询来适应不同设备和屏幕大小。
1.  **SVG 文本应用**：使用`<text>`元素嵌入文本内容，应用字体、大小、颜色和对齐方式，可以沿路径排列、换行、旋转或动画。
1.  **SVG 结构分组**：使用`<g>、<symbols>`进行分组，`<defs>、<use>`来复用，方便管理和操作相关元素，可以应用变换、样式和事件处理。
1.  **SVG 动画**：使用`<animate>、<animateTransform>、<animateMotion>`等元素，通过设置属性值或路径，实现平移、旋转、缩放、颜色变化等动画效果。
