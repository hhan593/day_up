有了之前 Shader 的 2D 编码基础，接下来我们深入 Shader 的三维世界。



## Ray Marching 介绍

Ray Marching 是一种计算机图形技术，用于渲染三维场景。它是一种基于光线追踪的算法，通过追踪光线在场景中的**传播路径**来生成图像。与传统的光栅化渲染技术相比，Ray Marching 更适合处理复杂的几何形状和光照效果。使用光线追踪我们可以惊人的几 kb 的文件大小生成可能需要建模几百 M 的三维场景。

Ray Marching 的基本原理：通过从相机位置发射光线，并在场景中追踪光线的路径，直到光线与场景中的**物体相交或达到最大迭代次数**。通过根据相交点的法线、材质属性和光照条件等信息，计算出像素的颜色值，从而生成最终的图像。

Ray Marching 的优势：在于它能够处理复杂的几何形状，如曲面、体积和分形等，而不需要对几何形状进行三角剖分。它还可以实现全局光照效果，包括阴影、反射和折射等，使得渲染结果更加真实和细致。

## SDF 符号距离函数

在之前的文章我们已经提过 SDF，SDF 是一种**表示几何形状的方法**，**它使用每个点到形状表面的有向距离作为表示**。SDF 可以表示各种几何形状，如球体、立方体、圆柱体等。对于给定的点，SDF 函数返回该点到最近表面的距离，并根据点的位置和距离值决定点在形状内部还是外部。


### SDF 和 Ray Marching 的关系

Ray Marching 是一种用于**渲染 SDF 几何的算法**。它通过沿光线进行迭代，从相机位置开始，依次采样沿光线的点，并使用 SDF 函数计算每个点到最近表面的距离。然后，根据距离值来确定光线是否击中了几何形状，并进行光照计算。

简单来说，SDF 用于生成几何形状，而 Ray Marching 用于渲染计算这些几何形状。

之前我们提到的 SDF 是二维的，那么转变三维后，对应的 SDF 函数是怎么样的呢？

比如球体 SDF：

```js
float sdSphere( vec3 p, float s ) { 
    return length(p) - s;
}
```

这段代码是一个用于计算点到球体表面的有向距离的函数。通过 `length(p)` 计算点 p 到原点的距离，然后减去球体的半径 s，就可以得到点 p 到球体表面的距离。

其他更多的 SDF 我们可以参阅这个网站:   [Inigo Quilez :: computer graphics, mathematics, shaders, fractals, demoscene and more (iquilezles.org)](https://iquilezles.org/articles/distfunctions/)


### 不同形状的三维 SDF


```js
// 胶囊 SDF
// p 表示计算距离的点的位置
// a 胶囊的一个端点位置
// b 胶囊的另一个端点位置
// r 胶囊的半径
float sdCapsule(vec3 p,vec3 a,vec3 b,float r){
    vec3 ab=b-a;
    vec3 ap=p-a;
    
    float t=dot(ab,ap)/dot(ab,ab);
    t=clamp(t,0.,1.);
    
    vec3 c=a+t*ab;
    
    return length(p-c)-r;
}

// 圆柱 SDF
// p 表示计算距离的点的位置
// a 圆柱的一个端点位置
// b 圆柱的另一个端点位置
// r 圆柱的半径
float sdCylinder(vec3 p,vec3 a,vec3 b,float r){
    vec3 ab=b-a;
    vec3 ap=p-a;
    
    float t=dot(ab,ap)/dot(ab,ab);
    
    vec3 c=a+t*ab;
    
    float x=length(p-c)-r;
    float y=(abs(t-.5)-.5)*length(ab);
    float e=length(max(vec2(x,y),0.));
    float i=min(max(x,y),0.);
    
    return e+i;
}

// 圆环 SDF
// p 表示计算距离的点的位置
// r 一个包含两个值的向量，分别表示圆环的主半径和管道半径
float sdTorus(vec3 p,vec2 r){
    float x=length(p.xz)-r.x;
    return length(vec2(x,p.y))-r.y;
}

// 立方体 SDF
// p 表示计算距离的点的位置
// s 一个表示立方体尺寸的向量
float dBox(vec3 p,vec3 s){
    return length(max(abs(p)-s,0.));
}
```

### 布尔计算

布尔计算的 Signed Distance Function（SDF）可以用于计算几何体之间的布尔操作结果。以下是一些常见的布尔计算的 SDF 函数。

1. 并集（Union）：取两个几何体的最小距离。

```js
float sdfUnion(float d1, float d2) {
  return min(d1, d2);
}
```
2. 交集（Intersection）：取两个几何体的最大距离。

```js
float sdfIntersection(float d1, float d2) {
  return max(d1, d2);
}
```
3. 差集（Difference）：取第一个几何体的距离，并将其与第二个几何体的距离取负。

```js
float sdfDifference(float d1, float d2) {
  return max(d1, -d2);
}
```

这些函数可以根据需要在着色器代码中使用，通过组合不同的 SDF 函数，可以实现复杂的布尔操作，例如组合多个几何体、创建复杂的形状等。注意，这些 SDF 函数的参数通常是点到几何体的距离，因此需要先计算出点与几何体的距离函数。





## 光线追踪原理实现

### 1. 计算距离

```js
// 计算距离
float GetDist(vec3 p){
    vec4 s=vec4(0,1,6,1);
    
    float sphereDist=length(p-s.xyz)-s.w;
    float planeDist=p.y;
    
    float cd=sdCapsule(p,vec3(3,.5,6),vec3(3,2.5,6),.5);
    float td=sdTorus(p-vec3(0,.5,6),vec2(1.5,.4));
    float bd=dBox(p-vec3(-3.5,1,6),vec3(1,.75,1));
    float cyld=sdCylinder(p,vec3(0,.3,3),vec3(3,.3,5),.3);
    
    float d=min(cd,planeDist);
    d=min(d,td);
    d=min(d,bd);
    
    d=min(d,cyld);
    
    return d;
}
```

在函数内部，首先定义了一个 vec4 类型的变量 s，表示一个球体的位置和半径。然后，通过 `length(p - s.xyz) - s.w` 计算点 p 到球体表面的有向距离。

接下来，定义了一个表示平面距离的变量 `planeDist` ，即点 p 的 y 坐标。

然后，分别计算了点 p 到胶囊、圆环、立方体和圆柱体的有向距离，分别存储在变量 `cd` 、 `td` 、 `bd` 和 `cyld` 中。

最后，通过多次调用 `min()` 函数，将点 p 到各个几何形状的有向距离与平面距离进行比较，得到最小的有向距离，并将其存储在变量 d 中。

函数返回变量 d，即点 p 到多个几何形状的最小有向距离。

这个函数可以用于基于体积的渲染和几何建模，通过计算点到多个几何形状的有向距离，可以实现复杂的形状组合和碰撞检测。

> **有向距离**（Signed Distance）是指从一个点到几何形状表面的距离，并且带有方向信息。它可以用于描述点相对于几何形状的位置关系，即点在几何形状内部、外部还是在表面上。


### 2. 光线追踪核心算法

```js
float RayMarch(vec3 ro,vec3 rd){
    float dO=0.;
    
    for(int i=0;i<MAX_STEPS;i++){
        vec3 p=ro+rd*dO;
        float dS=GetDist(p);
        dO+=dS;
        // 此时发生碰撞
        if(dO>MAX_DIST||dS<SURF_DIST)break;
    }
    
    return dO;
}
```

`RayMarch(vec3 ro, vec3 rd)` 函数用于执行光线步进算法，以从摄像机位置 `ro` 发射光线 `rd` 进行渲染。在每一步迭代中，通过 `GetDist()` 函数获取当前点 `p` 到最近表面的有向距离，并将其累加到 `dO` 变量上。如果累加的距离超过了最大距离 `MAX_DIST` ，或者当前点到最近表面的距离小于表面距离阈值 `SURF_DIST` ，则终止迭代。最后，返回累加的距离 `dO` 。


### 3. 生成法线

```js
vec3 GetNormal (vec3 p) {
    float d=GetDist(p);
    vec2 e=vec2(.001,0);
    
    vec3 n= d - vec3(
        GetDist(p-e.xyy),
        GetDist(p-e.yxy),
        GetDist(p-e.yyx));
        
        return normalize(n);
    }
```

`GetNormal(vec3 p)` 函数用于获取点 `p` 处的法线向量。首先，通过 `GetDist()` 函数获取当前点 `p` 到最近表面的有向距离，并存储在变量 `d` 中。然后，通过微小的偏移 `e` ，分别计算点 `p` 在 x、y、z 方向上的偏移点的有向距离，然后将其与 `d` 相减得到法线向量 `n` 。最后，使用 `normalize()` 函数将法线向量归一化，并返回法线。

当我们有了法线向量，即表面每个点的法向量时，就可以利用法线与光照方向的关系来计算光照效果。光照的计算通常基于光线与表面的交互，而法线则提供了表面的方向信息。

通过计算光线方向与法线的夹角，可以确定光线照射到表面上的角度。根据光照模型（例如，冯氏光照模型），我们可以使用法线与光线方向的夹角来计算漫反射和镜面反射的强度，从而获得光照效果。


### 4. 模拟灯光

```js
float GetLight(vec3 p) {
    vec3 lightPos = vec3(0, 5, 6);
    lightPos.xz += vec2(sin(iTime), cos(iTime)) * 2.0;
    vec3 l = normalize(lightPos - p);
    vec3 n = GetNormal(p);
    float dif = clamp(dot(n, l), 0.0, 1.0);
    float d = RayMarch(p + n * SURF_DIST * 2.0, l);
    if (d < length(lightPos - p))
        dif *= 0.1;
    return dif;
}
```

首先定义了一个表示光源位置的  `lightPos`  变量。通过  `vec2(sin(iTime), cos(iTime)) * 2.0`  的计算，将光源位置在 x 和 z 方向上根据时间进行周期性的移动。

然后通过  `normalize(lightPos - p)`  计算光线的方向向量  `l` ，即从点  `p`  指向光源的方向。通过调用  `GetNormal(p)`  函数获取点  `p`  处的法线向量，并存储在变量  `n`  中。

通过  `dot(n, l)`  计算法线向量  `n`  和光线方向向量  `l`  的点积，然后使用  `clamp()`  函数将其限制在 0 到 1 之间，得到漫反射光照的强度  `dif` 。

> 点积的计算公式为： dot(n, l) = n.x * l.x + n.y * l.y + n.z * l.z
> 
> 在光照计算中，**点积可以用于计算光线的入射角度和法向量之间的关系**。当两个向量趋向于同一方向时，点积的结果会接近于 1，表示光线与法向量之间的夹角较小，光线更接近于垂直于表面的方向，光照强度较高。当两个向量趋向于相互垂直时，点积的结果会接近于 0，表示光线与法向量之间的夹角较大，光线的入射角度较大，光照强度较低。


接着，通过  `RayMarch(p + n * SURF_DIST * 2.0, l)`  调用光线步进函数  `RayMarch()` ，以点  `p`  加上法线向量  `n`  的偏移，并沿着光线方向  `l`  进行渲染。返回的距离  `d`  用于判断是否有遮挡物。

最后，通过判断  `d`  是否小于光源到点  `p`  的距离，如果有遮挡物，则将漫反射光照强度  `dif`  乘以 0.1。

函数返回最终的光照强度  `dif` 。


### 5. 主逻辑

这段代码是一个基本的片段着色器函数，用于渲染场景并进行光照计算。以下是对代码的解析：

```js
void mainImage(out vec4 fragColor, in vec2 fragCoord) {
    vec2 uv = (fragCoord - 0.5 * iResolution.xy) / iResolution.y;
    vec3 col = vec3(0);
    vec3 ro = vec3(0, 2, 0);
    vec3 rd = normalize(vec3(uv.x - 0.15, uv.y - 0.2, 1));
    float d = RayMarch(ro, rd);
    vec3 p = ro + rd * d;
    float dif = GetLight(p);
    col = vec3(dif);
    col = pow(col, vec3(0.4545)); 
    fragColor = vec4(col, 1.0);
}
```
这是一个基本的光线追踪片段着色器函数，用于在屏幕上渲染场景的每个像素。

首先第一行代码就是我们之前做过的 uv 适配代码。col 模拟黑色，ro 模拟光线的起始坐标，rd 就是相机的朝向的单位向量，d 就是代表光线追踪后的有向距离。

通过 d * rd 得到有大小、方向的向量，然后加上坐标的初始位置得到最终的结果 p 向量，然后附加上光照强度。


### 6. 伽马校正

```js
// 伽马矫正
col = pow(col, vec3(0.4545));
```

**伽马校正**是一种用于调整颜色显示的技术。它用于纠正由于显示设备和图像处理管道的非线性响应而引起的颜色偏差。

伽马校正的目的是使图像的显示与人眼感知的亮度更加一致。它通过对颜色值进行幂次运算来实现。在这段代码中，通过  `pow(col, vec3(0.4545))`  对变量  `col`  的每个分量进行了 0.4545 次方的运算，以实现伽马校正。

最终，将颜色值  `col`  和不透明度 1 打包成  `vec4`  类型的  `fragColor` ，作为输出的片段颜色。

这段代码实现了基本的光线追踪渲染，通过光线与场景中的几何体的交互来计算每个像素的颜色。

## 总结

- Ray Marching 光线追踪以传播路径的方式渲染可见区域，相比光栅化能极大地提高 GPU 的性能。
- 三维的 SDF 函数能表示三维的几何体，创建球体非常容易，我们可以试着推敲更复杂的形体。
- 想要使用 Ray Marching 创建一个三维场景，核心步骤可以简化为：**创建相机、光线追踪算法得到有向距离、计算灯光和法线、控制片元着色器。**
- 对于复杂的建模，我们可以使用布尔运算的 SDF 来实现交集、差集等。
