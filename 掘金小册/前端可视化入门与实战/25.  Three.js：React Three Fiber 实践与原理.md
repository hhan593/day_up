你是否发现 Three 和主流框架 React 结合中我们往往要为了适应组件化、生态等，我们需要很麻烦地封装一个个组件，但是使用 R3F 可以让你更轻松地在 React 项目中创建和操作 Three.js 的 3D 场景和对象，提高开发效率和代码可维护性。如果你的技术栈是 vue，你也可以使用 trois 来实现组件化开发。

本节内容将从 R3F 介绍、组件化场景开发、交互动画、物理库、源码来讲解。



## R3F 简介

**R3F 是指 React Three Fiber，是一个用于在 React 项目中使用的 Three.js 渲染库**。它提供了一套简化的 hooks 和组件，R3F 充分利用了 React 的组件化开发模式和声明式编程的特性，使得在创建和管理 3D 元素时更加高效和易于理解。

下面我列举了要使用 R3F 的原因：

1. **更简洁的语法**：使用 R3F 我们可以享受 JSX 的语法，使得创建和操作 Three.js 的 3D 元素更加直观和简单。 
2. **声明式组件化开发**：R3F 充分利用了 React 的组件化开发模式，可以将 3D 场景和对象封装成可重用的组件，方便管理和复用。 
3. **React生态系统**：R3F 与 React 无缝集成，可以享受到 React 生态系统中丰富的工具和库的支持。
4. **性能优化**：`loader、材质等缓存，组件式分层渲染、渐进式加载、LOD、帧同步渲染、React 18 的 fiber 异步调度渲染`。

接下来我们将实现一个场景渲染的综合案例。

- `<RandomizedLight>` ：创建了一个随机化的光源。

- `<OrbitControls>` ：创建了一个轨道控制器，用于交互式控制场景的视角。

- `<Backdrop>` ：创建了一个背景板的组件，用于模拟场景中的地面或背景。

- `<Monkey>` ：这是一个 URL 加载的 GLTF 模型文件。

```js
function Monkey(props) {
    const { scene, materials } = useGLTF('https://vazxmixjsiawhamofees.supabase.co/storage/v1/object/public/models/suzanne-high-poly/model.gltf')
    useLayoutEffect(() => {
        scene.traverse((obj) => obj.isMesh && (obj.receiveShadow = obj.castShadow = true))
        materials.default.color.set('#fff')
        materials.default.roughness = 0
        materials.default.normalMap = new THREE.CanvasTexture(new FlakesTexture(), THREE.UVMapping, THREE.RepeatWrapping, THREE.RepeatWrapping)
        materials.default.normalMap.repeat.set(40, 40)
        materials.default.normalScale.set(0.1, 0.1)
    })
    return <primitive object={scene} {...props} />
}
```

我们通过 useGLTF 加载得到加载的场景对象，我们通过 traverse 遍历所有对象，对于是网格对象的部分，设置其接收阴影和投射阴影的属性为 `true` 。材质配置我们设置了白色、法线贴图、颗粒纹理及尺寸配置等，使用 `<primitive>` 组件将场景对象作为原始对象进行渲染，并传递其他属性。

接下来我们讲解下 Three 的模型文件格式：

1. **OBJ**（Wavefront OBJ）：这是一种基本的文本文件格式，用于描述模型的几何形状和材质信息。OBJ 文件可以包含顶点、面、纹理坐标和法线等数据。

2. **GLTF**（GL Transmission Format）：GLTF 是一种基于 JSON 的文件格式，它支持几何形状、材质、纹理、动画和其他相关数据的存储和交换。GLTF 的优点是文件尺寸较小、加载速度快，并且能够保留模型的大部分细节和特性。由于其广泛的支持和高效性能，GLTF 成为了 Three.js 中最常用的模型文件格式之一。

    GLTF 使用二进制格式的 .bin 文件来存储模型的原始二进制数据，例如顶点坐标、法线、纹理坐标、颜色等。这种二进制格式相比于文本格式，可以更有效地存储和传输大量的数据，因为它不需要进行字符编码和解码的过程。

- **如何加载：**

    ```js
    const loader = new THREE.GLTFLoader();
    loader.load('path/to/your/model.gltf', function (gltf) {
      const model = gltf.scene;
      scene.add(model);
    });
    ```

- **GLB：**

    GLB（GLTF Binary）是一种**基于 glTF**（GL Transmission Format）的文件格式，用于在 3D 图形应用程序中存储和交换三维模型和场景数据。GLB 文件是**二进制**格式的，其中**包含了模型的几何信息、材质、纹理、动画和其他相关数据**。

    GLB 文件的优势在于它是一种自包含的格式，所有的模型数据都被打包在一个文件中，**方便传输和加载**。同时，GLB 文件还支持**压缩和加密**，可以保护模型数据的安全性。GLB 材质是指在 GLB 文件中定义的三维模型的外观和纹理信息，**它是 GLTF 文件格式的一部分**，用于存储和交换三维图形数据。

- **bin 文件**

    GLTF 使用**二进制**格式的 .bin 文件来存储模型的原始二进制数据，例如顶点坐标、法线、纹理坐标、颜色等。这种二进制格式相比于文本格式，可以更有效地存储和传输大量的数据，因为它不需要进行字符编码和解码的过程。如下图 GLTF 文件的 uri 引用了 1.bin 文件。

3. **FBX**（Filmbox）：这是一种由 Autodesk 公司开发的二进制文件格式，用于存储三维模型、动画和其他相关数据。FBX 文件在游戏开发和动画制作中广泛使用。

## R3F 原生组件

在React Three Fiber中， `<primitive>` 是一个用于渲染原始的 Three.js 对象的组件。它允许您在 r3f 中直接使用 Three.js 中的任何对象，而不需要进行组件封装。 默认接受一个 `object` 属性，用于指定要渲染的 Three.js 对象。其实就是任何继承于三维的基类Object3D的对象。比如下面代码：

```js
import { Canvas } from "@react-three/fiber";
import { BoxGeometry, MeshBasicMaterial, Mesh } from "three";

function App() {
  const geometry = new BoxGeometry(1, 1, 1);
  const material = new MeshBasicMaterial({ color: 0xff0000 });
  const cube = new Mesh(geometry, material);

  return (
    <Canvas>
      <primitive object={cube} />
    </Canvas>
  );
}

export default App;
```

## R3F 交互动画

接下来我们模拟下运动的小球交互：


<p align=center><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b83d950905184f668289137ecd65a756~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=776&h=556&s=10423430&e=apng&f=84&b=e0e2e5" alt="动画.apng"  /></p>

```js
function Sphere() {
    const ref = useRef()
    const [active, setActive] = useState(false) // 是否hover
    const [zoom, set] = useState(true) // 是否视口缩放
    const [up, setUp] = useState(true) // 是否上升
    const [v, setV] = useState(0) //速度

    // 实现悬停在球体上时的交互效果
    useCursor(active)

    // 每帧更新时执行动画和交互逻辑
    useFrame((state) => {
        let v2 = v
        if (up) {
            if (v2 < 0.5) {
                v2 += 0.01
            } else {
                setUp(false)
            }
        } else {
            if (v2 <= 0) {
                setUp(true)
            } else {
                v2 -= 0.01
            }
        }
        setV(v2)
        const position = ref.current.position;

        position.y = v2
        state.camera.position.lerp({ x: 0, y: 10, z: zoom ? 22 : 14 }, 0.03)
        state.camera.lookAt(0, 0, 0)
    })
    return (
        <mesh ref={ref} receiveShadow castShadow onClick={() => set(!zoom)
        } onPointerOver={() => setActive(true)} onPointerOut={() => setActive(false)}>
            <sphereGeometry args={[0.25, 64, 64]} />
            <meshStandardMaterial color={active ? 'hotpink' : 'lightblue'} clearcoat={1} clearcoatRoughness={0} roughness={0} metalness={0.25} />
        </ mesh>
    )
}
```
通过 `onPointerOver` 和 `onPointerOut` 事件来处理是否激活，然后修改 color 实现样式变化。

在 R3F 中我们可以通过 `useFrame` 钩子来获取每一帧的渲染回调，通过控制 y 值的循环变化实现小球的上下移动。

为了实现点击后相机的偏移动画，我们监听点击 zoom 的状态，使用了 `lerp()` 方法来平滑地过渡相机的位置，并控制过渡速度为 0.03，最后将相机的焦点位置一直定位在中心点。

> `onPointerOver` 事件是一种更通用的事件，可以用于处理不同类型的输入设备的交互，而 `onMouse` 事件则更专注于鼠标设备的交互。




## R3F 物理库

`@react-three/rapier` 是一个用于在 React Three Fiber 中集成物理引擎的库。它是基于 Rust 编写的 Rapier 物理引擎的 JavaScript/TypeScript 绑定。Rapier 是一个高性能的、2D 和 3D 的物理引擎，可用于模拟和处理刚体、碰撞检测、关节和约束等物理效果。通过集成 @react-three/rapier，你可以在 React Three Fiber 中轻松地实现物理模拟和交互性的场景。

### 刚体和碰撞体


如果之前做过 Phaser.js 实战你会很容易理解碰撞体、刚体。同时也很容易总结出大部分物理库的用法规律。

**刚体**是指一个在物理空间中具有质量、位置和速度的物体，它可以受到力和碰撞的影响，从而改变它的位置和速度。在物理引擎中，刚体通常被用来表示实体对象，如球体、方体等。

**碰撞体**是指一个用于**检测碰撞**的几何形状，它可以与其他碰撞体进行碰撞检测，从而确定它们是否相交。在物理引擎中，碰撞体通常与刚体相关联，用于检测刚体之间的碰撞。

刚体用于表示物体的运动状态，而碰撞体用于检测碰撞和计算碰撞响应。通常，一个刚体可以有多个碰撞体，以便更准确地表示物体的形状和大小。

而在 rapier 中，我们会很常用这三个组件：`Collider、 Physics、 RigidBody`。

- **Physics**：用于创建物理世界的**容器组件**。在 Physics 组件内部放置的物体可以受到物理引擎的模拟和影响。 

- **RigidBody**：会对组件内**所有网格 Mesh 默认自动创建**刚体物体，并将其添加到**物理世界**中。刚体物体是具有质量、位置和速度的物体，可以受到力和碰撞的影响。RigidBody 组件可以与不同类型的碰撞体（如 CuboidCollider、SphereCollider 等）一起使用，以定义物体的形状和碰撞属性。RigidBody 组件的参数包括 `position` （刚体的初始位置）、 `mass`（刚体的质量）和 `colliders` （与该刚体关联的碰撞体）等。
     - RigidBody 的 type 属性将其分为 4 种不同的类型刚体：
         - `fixed` 不受外力影响，如场景；
         - `dynamic` 真实动态物体，受外力影响，适用于大多数物体；
         - `kinematicPosition` 部分受外力影响，比如弹簧；
         - `kinematicVelocity` 速度恒定，但是位置受影响，比如子弹。
- **Collider**：在 RigidBody 组件内部使用 Collider，用于创建碰撞体的组件。这对于**创建更复杂的形状、出于性能原因创建简化形状或检测网格特定部分的碰撞**非常有用。就像下面的代码一样，我们只有特定大小的球形碰撞体。

```js
<RigidBody position={[0, 10, 0]}>
    <Sphere />
    <BallCollider args={[0.5]} />
    <BallCollider args={[0.5]} position={[1, 0, 0]} />
</RigidBody>
```

function PhysicsSphere({ position }) {
    const ref = useRef();
    return (
        <RigidBody position={position} ref={ref} colliders="ball" mass={1} restitution={0.5}>
            <mesh ref={ref} castShadow>

对于球体，我们可以通过 colliders 设置 ball 来模拟刚体的碰撞形体结构。但是对于复杂的模型来说，我们往往更需要使用 trimesh（大量的三角面计算会带来性能问题），如果棱角分明一些那么我们可以使用 hull。

下面是 RigidBody 组件的 colliders 属性可传参数：

- `ball`：表示使用球体作为自动碰撞体。

- `cuboid`：表示使用长方体作为自动碰撞体。

- `hull`：表示使用凸包作为自动碰撞体。凸包是一个封闭的凸多边形，它会将物体的形状包围起来。

- `trimesh`：表示使用三角网格作为自动碰撞体。这意味着物体会被近似为一个由三角形构成的网格形状的碰撞体。

-  `false` ：表示禁用自动碰撞体，即不使用自动生成的碰撞体。

  extends Omit<RenderProps<HTMLCanvasElement>, 'size'>,

## R3F 组件化实现原理

大部分人可能认为 R3F 的组件化实现是直接封装一个个 Three 元素的 JSX 组件，就像下面的球体组件：

```jsx
class SphereGeometry extends React.Component {
  constructor(props) {
    super(props);
    this.geometryRef = React.createRef();
  }

  componentDidMount() {
    // 在组件挂载后创建SphereGeometry实例并添加到场景中
    const geometry = new THREE.SphereGeometry(this.props.radius, this.props.segments, this.props.rings);
    this.geometryRef.current.geometry = geometry;
    this.props.scene.add(this.geometryRef.current);
  }

  componentWillUnmount() {
    // 在组件卸载前从场景中移除SphereGeometry实例
    this.props.scene.remove(this.geometryRef.current);
  }
  
  //...
  render() {
    return <mesh ref={this.geometryRef} />;
  }
}
```

但实际在源码中我们并没有看到这种处理。我们可以仔细体会下面从 R3F 转变为 three 代码的过程。

```js
import { Canvas } from '@react-three/fiber'

function MyApp() {
  return (
    <Canvas>
      <group>
        <mesh>
          <meshNormalMaterial />
          <boxGeometry args={[2, 2, 2]} />
        </mesh>
      </group>
    </Canvas>
  )
}
```

实际转换后：

```js
import * as THREE from 'three'

const scene = new THREE.Scene() // <Canvas>

const group = new THREE.Group() // <group>

const mesh = new THREE.Mesh() // <mesh />
const material = new THREE.MeshNormalMaterial() // <meshNormalMaterial />
const geometry = new THREE.BoxGeometry(2, 2, 2) // <boxGeometry />

mesh.material = material
mesh.geometry = geometry

group.add(mesh)
scene.add(group)
```

其实核心的处理就是 r3f 在通过 fiber 解析到每个元素名，然后去 three 对象上直接匹配属性然后调用，r3f 大部分 api 的实现能有属性查找的理念。比如：

```js
<mesh>
  <meshNormalMaterial />
  <boxGeometry  />
</mesh>
```

上面的 mesh 我们直接匹配了 `new THREE.Mesh()`，meshNormalMaterial 和 boxGeometry 也是如此，但是很明显 mesh 与两个子组件是父子关系，所以 r3f 设计了 `attach` 来引用 mesh，有没有发现 attach 的值还是属性绑定，`mesh.material = material`。所以，你可以完全自己实现不同的元素组件（但必须满足 three 的传参规则）。

```js
<mesh>
  <meshNormalMaterial attach="material" />
  <boxGeometry attach="geometry" />
</mesh>
```
