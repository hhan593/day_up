有了之前的基础，我们接下来学习 WebGL 的纹理，也就是 UVs 纹理系统。



## 相关概念

### UV

UV 纹理是一种在计算机图形学中常用的纹理映射技术。**它将一个二维图像或纹理映射到三维模型的表面上。**

UV 纹理的名称来自于其使用的二维坐标系统，其中 U 和 V 表示在纹理坐标空间中的水平和垂直方向。

右侧就是我们常见的屏幕坐标系，左侧为 uv 坐标系。很明显 uv 的原点在左下角。我们可以把 uv 的 v 看作屏幕坐标系的 x，而 y 的负值对应 uv 正方向的 u。



### 纹理对象

WebGL 中的纹理对象是用于在渲染过程中将纹理映射到三维模型上的对象。

在 WebGL 中，纹理对象由 `WebGLTexture` 类型表示。要创建纹理对象，我们需要执行以下步骤：

1. 创建纹理对象：使用 `gl.createTexture()` 函数创建一个空的纹理对象。

2. 绑定纹理对象：使用 `gl.bindTexture()` 函数将纹理对象绑定到目标纹理单元上。目标纹理单元可以是 `gl.TEXTURE_2D` 表示二维纹理，或者是 `gl.TEXTURE_CUBE_MAP` 表示立方体纹理。

3. 设置纹理参数：使用 `gl.texParameteri(target, pname, param)` 函数设置纹理的一些参数，例如放大和缩小过滤器、纹理环绕方式等。

4. 加载纹理图像：使用 `gl.texImage2D()` 函数将图像数据加载到纹理对象中。可以从图像元素或像素数据中加载纹理。

5. 在着色器中使用纹理：将纹理对象绑定到纹理单元上，并在着色器中使用相应的纹理单元进行**纹理采样**，例如 `texture2D(textureID, uv)`。

> 在 WebGL 中，`texture` 和 `texture2D` 实际上是等价的。在 WebGL 2.0 版本中，`texture2D` 被废弃，而是使用 `texture` 来进行纹理采样。这是因为 WebGL 2.0 引入了纹理函数的重构，将通用的纹理函数命名为 `texture`。


### gl.texParameteri 纹理配置

```js
gl.texParameteri(target, pname, param);
```
-   `target`：指定目标纹理，常用的值包括 `gl.TEXTURE_2D` 表示二维纹理。

-   `pname`：指定要设置的参数名称，可以是以下之一：

    -   `gl.TEXTURE_MIN_FILTER`：设置纹理缩小过滤器，决定当纹理被缩小时的采样方式。param 可取 `gl.NEAREST` 或 `gl.LINEAR`。
    -   `gl.TEXTURE_MAG_FILTER`：设置纹理放大过滤器，决定当纹理被放大时的采样方式，param 可取 `gl.NEAREST` 或 `gl.LINEAR`。
    -   `gl.TEXTURE_WRAP_S` 和 `gl.TEXTURE_WRAP_T`：设置纹理在 S 和 T 方向上的环绕方式，param 可取 `gl.CLAMP_TO_EDGE`、`gl.REPEAT` 或 `gl.MIRRORED_REPEAT`。
        -   `gl.CLAMP_TO_EDGE`：纹理坐标超出范围时，会被截断到边缘的纹理坐标。这意味着超过纹理边界的部分将使用纹理图像的边缘颜色。
        -   `gl.REPEAT`：纹理坐标会在整个范围内重复。当超过 1.0 时，纹理坐标将回到 0.0。
        -   `gl.MIRRORED_REPEAT`：纹理坐标会在整个范围内镜像重复。当超过 1.0 时，纹理坐标将反向回到 0.0，然后继续镜像重复。
    -   `gl.TEXTURE_MIN_LOD` 和 `gl.TEXTURE_MAX_LOD`：设置最小和最大 Level of Detail（LOD）值，用于控制 mipmap 的级别。

纹理的过滤器说白了就是将过大或过小的纹理缩放到实际尺寸，在这个过程中我们要考虑如何采样，所以有了 NEAREST 和 LINEAR 两种不同的采样方式。

其他参数具体取决于 WebGL 的版本和扩展。



### gl.texImage2D 纹理函数

`texImage2D`  是 WebGL 中用于将图像数据加载到纹理对象的函数，即  `texImage2D(target, level, internalFormat, format, type, source)` 。

-  `target` ：指定纹理目标，可以是  `gl.TEXTURE_2D`  表示二维纹理或  `gl.TEXTURE_CUBE_MAP`  表示立方体纹理。
-  `level` ：指定纹理的级别，通常为 0。如果使用纹理金字塔（mipmap），可以指定不同的级别。
-  `internalFormat` ：指定纹理的内部格式，表示纹理数据在 GPU 中的存储格式。常见的值有  `gl.RGBA` 、 `gl.RGB` 、 `gl.LUMINANCE_ALPHA`  等。
-  `format` ：指定源图像数据的格式，即图像数据的像素格式。常见的值有  `gl.RGBA` 、 `gl.RGB` 、 `gl.LUMINANCE_ALPHA`  等，要与  `internalFormat`  保持一致。
-  `type` ：指定源图像数据的数据类型，表示每个像素元素的数据类型。常见的值有  `gl.UNSIGNED_BYTE` 、 `gl.FLOAT` 、 `gl.HALF_FLOAT`  等。
-  `source` ：指定源图像数据，可以是图像元素（如  `<img>`、`<canvas>`  元素），也可以是像素数据数组（如  `Uint8Array` 、`Float32Array` ）。

通过调用  `texImage2D` ，我们可以将图像数据加载到纹理对象中，并为纹理对象设置相应的参数。



### 纹理单元

WebGL 的纹理单元是用于存储和处理纹理数据的单元。每个纹理单元可以存储一个或多个纹理对象。我们可以从以下四点理解：

1.  **纹理单元编号：** 纹理单元通过整数编号来标识，通常从 0 开始一般至少有 8 个纹理单元的空间可使用。我们可从`gl.getParameter(gl.MAX_COMBINED_TEXTURE_IMAGE_UNITS)` 来查询可用的纹理单元数量。
2.  **激活纹理单元：** 在使用纹理之前，需要激活一个纹理单元。使用 `gl.activeTexture(unit)` 方法选择要激活的纹理单元，其中 `unit` 是纹理单元的编号，通常是 `gl.TEXTURE0` 到 `gl.TEXTURE7`。
3.  **绑定纹理对象：** 一旦激活了纹理单元，就可以将纹理对象绑定到该单元。使用 `gl.bindTexture(target, texture)` 方法，其中 `target` 是纹理目标（如 `gl.TEXTURE_2D`），而 `texture` 是要绑定的纹理对象。
4.  **在着色器中使用纹理单元：** 在着色器中使用纹理时，需要将纹理单元的编号传递给 uniform 变量。

```js
// 在JavaScript中传递纹理单元到着色器
const textureLocation = gl.getUniformLocation(program, 'u_texture');
gl.uniform1i(textureLocation, 0); // 将纹理单元0传递给着色器中的 u_texture 变量
```

着色器中的对应代码：

```js
uniform sampler2D u_texture; // 着色器中的纹理变量
```

### 定义着色器源码、buffer 处理

```js
// 顶点着色器
const vertexShader = gl.createShader(gl.VERTEX_SHADER);
gl.shaderSource(vertexShader, `
    precision mediump float;

    attribute vec3 position;
    attribute vec2 uv;

    varying vec2 vUV;

    uniform mat4 matrix;

    void main() {
        vUV = uv;
        gl_Position = matrix * vec4(position, 1);
    }
`);
gl.compileShader(vertexShader);

// 片元着色器
const fragmentShader = gl.createShader(gl.FRAGMENT_SHADER);
gl.shaderSource(fragmentShader, `
    precision mediump float;

    varying vec2 vUV;
    uniform sampler2D textureID;

    void main() {
        gl_FragColor = texture2D(textureID, vUV);
    }
`);
gl.compileShader(fragmentShader);

const program = gl.createProgram();
gl.attachShader(program, vertexShader);
gl.attachShader(program, fragmentShader);

gl.linkProgram(program);

// 初始化buffer
const positionBuffer = gl.createBuffer();
gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);
gl.bufferData(gl.ARRAY_BUFFER, new Float32Array(vertexData), gl.STATIC_DRAW);

const uvBuffer = gl.createBuffer();
gl.bindBuffer(gl.ARRAY_BUFFER, uvBuffer);
gl.bufferData(gl.ARRAY_BUFFER, new Float32Array(uvData), gl.STATIC_DRAW);

// 将position 和 uv 绑定到 buffer 缓冲区上
const positionLocation = gl.getAttribLocation(program, `position`);
gl.enableVertexAttribArray(positionLocation);
gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);
gl.vertexAttribPointer(positionLocation, 3, gl.FLOAT, false, 0, 0);

const uvLocation = gl.getAttribLocation(program, `uv`);
gl.enableVertexAttribArray(uvLocation);
gl.bindBuffer(gl.ARRAY_BUFFER, uvBuffer);
gl.vertexAttribPointer(uvLocation, 2, gl.FLOAT, false, 0, 0);

gl.useProgram(program);
gl.enable(gl.DEPTH_TEST);

// 存储uniform变量
uniformLocations = {
    matrix: gl.getUniformLocation(program, `matrix`),
    textureID: gl.getUniformLocation(program, 'textureID'),
};

// 默认纹理id为0
gl.uniform1i(uniformLocations.textureID, 0);
```

整个流程就是定义片元、顶点着色器，然后将我们需要传入的变量抛出做 buffer 的缓冲处理，然后赋默认值给 glsl 变量。这里我们注意下片元着色器的纹理核心代码：

```js
gl_FragColor = texture2D(textureID, vUV);
//...
gl.uniform1i(uniformLocations.textureID, 0);
```

`texture2D()` 是一个内置函数，用于从纹理中获取特定纹理坐标处的颜色值。它接收两个参数：纹理对象（textureID）和纹理坐标（vUV）。通过 textureID 来绑定不同的纹理对象，默认初始化是 0。

#### 纹理加载器

```js
function loadTexture(url) {
    const texture = gl.createTexture();
    const image = new Image();

    image.onload = e => {
        gl.bindTexture(gl.TEXTURE_2D, texture);
        // 使用 `texImage2D` 方法将图片数据上传到纹理对象。
        gl.texImage2D(gl.TEXTURE_2D, 0, gl.RGBA, gl.RGBA, gl.UNSIGNED_BYTE, image);
        gl.generateMipmap(gl.TEXTURE_2D);
    };

    image.src = url;
    return texture;
}
```

所以，生成图像的纹理核心步骤也就是在着色器通过 texture2D 传入对应 id（它会默认引用对应的纹理对象），那么在 webgl 的外部我们需要创建并传入一个 id 为 0 的纹理对象。那么纹理对象的资源，我们需要定义一个纹理加载器，在其中我们做对应的纹理配置。最后做纹理的激活和绑定。


这里最关键的在于 `gl.texImage2D` 方法，将 img 图片元素节点存储到纹理单元：

```js
gl.texImage2D(target, level, internalformat, format, type, source);
```
-   `target`：指定目标纹理，常用的值包括 `gl.TEXTURE_2D` 表示二维纹理。
-   `level`：表示纹理的层级，通常为 0 表示基本图像级别。
-   `internalformat`：表示纹理存储的内部格式，例如 `gl.RGBA` 表示使用红、绿、蓝和 alpha 通道。
-   `format`：表示 source 数据的格式，通常也是 `gl.RGBA`。
-   `type`：表示 source 数据的类型，常用的值包括 `gl.UNSIGNED_BYTE`，表示每个颜色通道占用一个字节。
-   `source`：表示要上传的图像数据，可以是 `Image` 对象、`Canvas` 对象或 `ArrayBufferView`。

#### 卷积实现滤镜

卷积通常用于图像处理任务，例如图像滤波、边缘检测和特征提取等。通过将一个滤波器（也称为卷积核）应用于输入图像的每个像素，来生成输出图像。卷积核是通常我们用一个 3 * 3 的卷积矩阵，矩阵中的每一项代表当前处理的像素和周围 8 个像素的乘法之和，然后除以内核权重之和。

矩阵计算过程：

```js
// 像素的矩阵
[
    10, 20, 30,
    20, 40, 60,  // 这里的40就是内核   
    10, 20, 30,
]

// 乘以

// 卷积矩阵
[
    0, -1, 0,
    -1,  6, -1,     
    0, -1, 0,
]

// => 得到最终的像素矩阵
[
    x, x, x,
    x, y, x, 
    x, x, x,
]
```

那么，如何得到最终像素矩阵的 y 值呢？

```js
// 像素乘法之和 
const pixelSum = 10 * 0 + 20 * -1 + 30 * 0 + 20 * -1 +  40 * 6 + 60 * -1 + 0 * 10 + 20 * -1 + 30 * 0 
// 内核权重之和
const weight == 0 + -1 + 0 + -1 + 6 + -1 + 0 + -1 + 0
// y = 120 / 2 = 60
const y  = pixelSum / weight
```

所以，最终 y 值是 `像素乘法之和 / 内核权重之和`，其他坐标值也是同理计算，我们可以发现中间与周围的差值会变大，反映到纹理上就是会有锐化效果。调整不同的卷积矩阵，我们可以实现更多强大的滤镜效果，比如反相、高斯模糊等。

那么对应到 webgl 的着色器代码中：

```js
void main() {
   vec2 onePixel = vec2(1.0, 1.0) / u_textureSize;
   vec4 colorSum =
       texture2D(u_image, v_texCoord + onePixel * vec2(-1, -1)) * u_kernel[0] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 0, -1)) * u_kernel[1] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 1, -1)) * u_kernel[2] +
       texture2D(u_image, v_texCoord + onePixel * vec2(-1,  0)) * u_kernel[3] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 0,  0)) * u_kernel[4] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 1,  0)) * u_kernel[5] +
       texture2D(u_image, v_texCoord + onePixel * vec2(-1,  1)) * u_kernel[6] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 0,  1)) * u_kernel[7] +
       texture2D(u_image, v_texCoord + onePixel * vec2( 1,  1)) * u_kernel[8] ;
   gl_FragColor = vec4((colorSum / u_kernelWeight).rgb, 1);
}
```

注意，假设 v_texCoord 当前是`(0, 0)` ， onePixel 这里是最小单位的偏移量，而后面的 vec2 向量代表偏移的方向，刚好对应 3 * 3 的像素矩阵。然后传入我们之前的锐利效果的卷积矩阵，最终效果反映到纹理：

## 总结

- webgl 的纹理核心流程：创建纹理对象、将纹理对象绑定到纹理单元、设置纹理参数并加载纹理，最后在片元着色器 texture 函数中获取纹理单元和 uv 做纹理采样。

- 纹理加载器的核心在于 texImage2D 将图片数据上传到纹理对象，注意确保 Image 加载完毕。封装一个 TextureLoader 可以简化我们的开发。

- 在实际开发中，我们会使用一个贴图通过分割 uv 的方式渲染不同面的纹理，在处理滤镜效果中，我们可以用卷积算法实现锐化、高斯模糊、反相等高级效果。
