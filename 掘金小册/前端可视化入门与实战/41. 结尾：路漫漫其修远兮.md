下面将从 `趋势`（把握未来技术方向）和 `成长`（推动个人成长方向）两方面来总结小册。



## 趋势 📈

可视化的技术发展一直不断演进，自从 CSS3 和 H5 的标准诞生，新的功能就一直在不断涌现，比如 @scope、嵌套、color-mix、text-warp、:has、容器查询等等；SVG 1.x 从 2001 年开始漫长发展到在 23 年 3 月发布 2.x 草案，SVG 2 删除或弃用了 SVG 1.1 的某些功能，并合并了 HTML5 和 Web Open Font Format 的新功能，未来 SVG 会与 H5C3 更加紧密；而 Canvas 提供了通用的图形接口方便我们接入 WebGL、WebGPU。

WebGL 在 17 年从 1.0 升级为 2.0，引入了更多数据类型、纹理和渲染目标支持、实例渲染、新的着色器语言、同步操作等功能；WebGPU 除了在功能和兼容性的改变，未来将与 WebAssembly 密切发展，以快速绑定到 Blink。还会有与 WebVR/WebXR 和 HTML 的 `<img>` 和 `<video>` 元素交互等等。



## 成长 🌱

路漫漫其修远兮，下面我会从`体系、感悟、能力`三个方面说明，希望对你未来的学习成长有所帮助。

### 体系化的养成

小册包含的内容其实已经十分全面，每个章节环环相扣，知识领域也很宽阔，与之诞生的形形色色的框架也非常多，看似很难，但也万变不离其宗，那就是可视化的体系。

体系怎么理解？就是它的本源是怎么样的，为什么会诞生，它的目的又是什么，诞生之后它是如何演进的，我们需要理解它。这样后续任何技术的发展我们都能跟之前所学关联，并启发思考。技术有所沉淀，人才有所成长。

怎么检验自己是否掌握了呢？当一个新的技术出来，我希望你能在你的体系树上快速找到对应的位置，并延展出概念、用法以及其他技术的优缺点的对比等。此时，你就掌握了体系化的精髓，学任何东西也会考虑更全面，也更容易记住。



### 我的四点感悟 🤔️

**专业性**

我相信大部分人觉得前端可视化的学习路线很凌乱繁杂，可视化一直没有很专业的视角展开学习，前端仿佛就是一个写页面的存在。但是深入后会发现可视化背后的知识是极其广阔，是能让人研究一生的。相比后端各种林立的抽象概念堆砌，我们也是可以用很专业的术语、概念、技术来表达前端的专业性。

**坚持与热爱**

小册从策划开始到结束总共花了 3 个月的空闲时间，刚开始更多的是兴奋，而之后更多的是困惑不解，做永远比说难，只有做了才会发现之前自己觉得对的可能并不正确。很多章节翻来覆去改了又改，也是希望能让读者有更好的阅读体验，也有很多时候也会想着放弃，但是当我发现困难不断被战胜，自豪感扑面而来的时候想想还是值得的，所以也是坚持过来了。希望你也能保持坚持与热爱。

**勇于质疑**

掘金上有很多很好的技术文章，但也有一些文章存在问题或者改进的空间，很多人也并没有提出质疑，任何技术永远不要觉得对方写出来的就是对的，我们要敢于做不同的尝试和讨论。希望读者也能以审阅的视角看待本小册并给出意见。

**热爱分享**

很多时候分享沟通的越多，同事和伙伴也会更依赖你、更能达成共识，项目推进也会更顺利。就像写文章这件事情，通过流量也有更多的人知道你在做什么，你能为大家解决什么问题，大家有什么更好的解决方案，互相学习与成长。

### 如何提高学习能力

- 多阅读和实践高质量的`开源项目`，可以先从你实际项目中使用的库中入手，多问为什么这么做，多尝试用新的角度来审视代码；
- 当开源库中存在问题，你也可以贡献你的 pr 或者 issue 提出问题，更多在于`参与社区`的技术氛围，当然顺道结交一些志同道合的朋友还是很不错的；
- 每当你的脑袋里有新的 idea，有分享的欲望无处宣泄的时候，你可以快速写个demo 或者在掘金写一篇文章`分享`给大家。
