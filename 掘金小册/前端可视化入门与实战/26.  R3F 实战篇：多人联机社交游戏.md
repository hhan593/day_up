## 多人联机的通信方案

多人联机场景能实现，最主要在于高效的通信方案同步多个 Web 端连接的数据，每个连接的 Web 端保存有其他联机用户的数据。

接下来我们聊聊多人联机的通信方案的选择。

### WebSocket

WebSocket 协议是用于在**浏览器和服务器**之间建立**全双工通信**的协议。它使用 TCP 协议，通过二进制帧传输。我们一般使用 WebSocket 进行信令传输和房间管理，而 WebRTC 进行音视频传输。

在 Node 服务中我们经常会使用 Socket.io 来实现前后端的 socket 连接，比如下面这样：

```js
// 服务端
import { Server } from "socket.io";

const io = new Server({
  cors: {
    origin: "http://127.0.0.1:5173",
  },
});

io.listen(3001);

const generateRandomUserName = () => {
  return "player_" + Math.random().toString(36).substr(2, 10);
};

io.on("connection", (socket) => {
  console.log("user connected");
  
  socket.emit("hello");
  
  socket.on("disconnect", () => {
    console.log("user disconnected");
  });
})
```

而在 Web 端中我们可以直接调用 `socket.io-client` 这个包来实现连接：

```js
//  web 端
import { useEffect } from "react";
import { io } from "socket.io-client";

export const socket = io("http://localhost:3001");

export const SocketManager = () => {
    useEffect(() => {
        function onConnect() {
            console.log("connected");
        }
        function onDisconnect() {
            console.log("disconnected");
        }

        function onHello() {
            console.log("hello");
        }

        socket.on("connect", onConnect);
        socket.on("disconnect", onDisconnect);
        socket.on("hello", onHello);

        return () => {
            socket.off("connect", onConnect);
            socket.off("disconnect", onDisconnect);
            socket.off("hello", onHello);
        };
    }, []);
};
```

### WebRTC

webRTC 是 Web 端解决实时的音视频的通信技术（内部封装了音视频的采集、处理、编码、解码、传输、连接等），同时可以**支持浏览器之间进行数据传输**，所以我们可以脱离服务器实现点对点的数据传输，更偏向于**去中心化思想**。注意，并不意味着没有服务器的介入，浏览器之间的连接建立要依赖服务器，连接建立的过程中可以使用 websocket 传输协商的数据。

此外，webRTC 是使用 UDP 的通信协议。UDP 是一种无连接的协议，意味着它不建立连接，也就意味着相比 TCP 传输更快、数据不可靠。WebRTC 主要用于视频聊天、网络会议、游戏和虚拟现实等场景。

下面是简单的代码演示：

```js
// 获取媒体流
var video = document.getElementById('video');
var audio = document.getElementById('audio');

navigator.mediaDevices.getUserMedia({video: true, audio: true})
  .then(function(stream) {
    video.srcObject = stream;
    audio.srcObject = stream;
  })
  .catch(function(error) {
    console.log(error);
  });

// 建立连接
var peerConnection = new RTCPeerConnection();

// 交换信令
peerConnection.onicecandidate = function(event) {
  if (event.candidate) {
    // 将本地端的网络候选者发送到远程端
    sendICECandidate(event.candidate);
  }
};

// 处理信令
function sendICECandidate(candidate) {
  // 将本地端的网络候选者发送到远程端
  var message = {
    type: 'candidate',
    candidate: candidate
  };

  // 通过信令服务器或其他信道将消息发送到远程端
  sendMessage(message);
}

// 建立数据通道
var dataChannel = peerConnection.createDataChannel('chat');

// 传输媒体流
peerConnection.addStream(stream);

// 处理远程媒体流
peerConnection.ontrack = function(event) {
  // 将远程端的媒体流添加到本地端
  var remoteVideo = document.getElementById('remoteVideo');
  remoteVideo.srcObject = event.streams[0];
};

// 关闭连接
function closeConnection() {
  peerConnection.close();
}
```

本文并不会讲解如何封装这些通信技术来服务于游戏，我们将快速使用现成的框架实现多人联机游戏，比如 playroom kit。

    // 等待 playroomkit 初始化 开始游戏的核心
    await insertCoin();

    // 一旦在ui界面点击加入游戏的回调（自己 + 其他玩家）
    onPlayerJoin((state) => {
      // 创建操纵杆
      const joystick = new Joystick(state, {
        type: "angular",
        buttons: [
          { id: "wave", label: "打招呼" },

      state.setState("deaths", 0);
      state.setState("kills", 0);

      // 用户数据是保存在主机，其他联机的浏览器将同步数据
      setPlayers((players) => [...players, newPlayer]);

      // 断开连接后删除当前退出的玩家 （自己 + 其他玩家）
      state.onQuit(() => {
        setPlayers((players) => players.filter((p) => p.state.id !== state.id));
      });
    });
  };

  // 初始化
  useEffect(() => {
    startGame();
  }, []);

  return (
    <>
      {/* 地图场景组件 */}
      <StreetMap />

      {/* 遍历所有联机的玩家 */}
      {players.map(({ state, joystick }, index) => (
        <CharacterController
          key={state.id}
          state={state}
          userPlayer={state.id === myPlayer()?.id}

export function Character({

  ...props

  const group = useRef();

    "/models/SciFiCharacter.glb"

        child.receiveShadow = true;

#### 跳跃与移动

之前我们直接使用 playroomkit 的 joystick 操纵杆实现监听用户的手指的按压方向，在 useFrame 中 `joystick.angle()` 可以得到当前角度、 `joystick.isJoystickPressed()` 得到按压状态。 此时，我们就可以使用 `@react-three/rapier` 这个库提供的 `applyImpulse` 方法来**生成一次性的力推动刚体**。

> 注意：由于物理世界，我们操控的一定是力的推动，而不是修改节点的 position，这样我们才能享受 rapier 提供的物理配置和碰撞检测。

代码如下：

```js
  useFrame((_, delta) => {
    // 根据操纵杆修改角度
    const angle = joystick.angle();
    const jump = joystick.isPressed("jump");

    const move = joystick.isJoystickPressed() && angle;
    if (move || jump) {
      setAnimation("CharacterArmature|Run");
      character.current.rotation.y = angle;

      // 生成一次性的力推动玩家
      const impulse = {
        x: !jump && Math.sin(angle) * MOVEMENT_SPEED * delta,
        y: jump && !jumplock ? 60 : 0,
        z: !jump && Math.cos(angle) * MOVEMENT_SPEED * delta,
      };

      rigidbody.current.applyImpulse(impulse, true);
    } else {
      // 默认静止
      setAnimation("CharacterArmature|Idle");
    }

    if (jump && !jumplock) setJumplock(true);

    // 主机玩家可以控制所有玩家状态
    if (isHost()) {
      // 每帧保存所有玩家位置
      state.setState("pos", rigidbody.current.translation());
    } else {
      // 更新当前玩家的位置信息
      const pos = state.getState("pos");
      if (pos) {
        rigidbody.current.setTranslation(pos);
      }
    }
  });
```

注意一旦跳跃，需要锁定跳跃键（无法连续触发），一旦触碰到碰撞体后才解锁。

#### 聊天功能

联机交互存在一个核心问题，是需要**实时区分主机的数据和网络其他玩家的数据**，因为会存在不同的逻辑处理。在 playroomkit 中，我们只需要使用 `useMultiplayerState` 这个钩子获得联机玩家的数据即可。

const [networkChats, setNetworkChats] = useMultiplayerState("chatList", []);
 
// 广播到其他玩家
useEffect(() => {
   setNetworkChats(chatList);
}, [chatList]);

// 组件变更
<PlayerInfo
  player={state}
  state={state.state}
  chatList={isHost() ? chatList : networkChats}
/>

/**
 * 玩家信息面板
 */
const PlayerInfo = ({ state, chatList }) => {

    return chatList.length ? chatList.at(-1).message : "";

#### 射击

射击的数据（子弹等）与我们的聊天交互一样，需要区分主机数据和网络数据，主机来维护数据同步到其他联机玩家中，来实现数据一致性。

接着就是子弹的实现，我们需要实现子弹的组件，它也是一个刚体，并且我们也知道当前玩家的角度方向，只不过一旦初始化他会不断生成推力，并且重力在 Y 轴的方向上值近似为 0，一旦 onIntersectionEnter 检测到相交碰撞就销毁该实例即可，如果是对方玩家我们使用 `state.setState` 削减玩家的生命值即可。

##### 实现子弹组件

```js
export const Bullet = ({ player, angle, position, onHit }) => {
  const rigidbody = useRef();
    
  // 核心逻辑
  useEffect(() => {
    const audio = new Audio("/audios/rifle.mp3");
    audio.play();
    const velocity = {
      x: Math.sin(angle) * BULLET_SPEED,
      y: 0,
      z: Math.cos(angle) * BULLET_SPEED,
    };
    
    // 不间断的力
    rigidbody.current.setLinvel(velocity, true);
  }, []);

  return (
    <group position={[position.x, position.y, position.z]} rotation-y={angle}>
      <group >
        <RigidBody
          ref={rigidbody}
          gravityScale={0}
          onIntersectionEnter={(e) => {
            // 保证子弹间无影响
            if (isHost() && e.other.rigidBody.userData?.type !== "bullet") {
              // 防止多次触碰到玩家
              rigidbody.current.setEnabled(false);
              onHit(vec3(rigidbody.current.translation()));
            }
          }}
          sensor
          // 和 three 一样在 userData 挂载用户的数据
          userData={{
            type: "bullet",
            player,
            damage: 10,
          }}
        >
          <mesh position-z={0.25} material={bulletMaterial} castShadow>
            <boxGeometry args={[0.05, 0.05, 0.5]} />
          </mesh>
        </RigidBody>
      </group>
    </group>
  );
};
```

接着我们就可以在玩家的刚体中监听碰撞事件，处理生命值和死亡的逻辑，并且播放相应的中弹动画、死亡动画等。

```js
<RigidBody
    gravityScale={jumplock ? 6 : 1}
    ref={rigidbody}
    colliders={false}
    // 控制线性阻尼
    linearDamping={12}
    // 锁定y轴旋转
    lockRotations
    // 默认 dynamic 受物理影响 kinematicPosition 不受物理影响
    type={isHost() ? "dynamic" : "kinematicPosition"}
    // 每个相交都会触发
    onIntersectionEnter={({ other }) => {
      if (
        isHost() &&
        other.rigidBody.userData.type === "bullet" &&
        state.state.health > 0
      ) {
        // 计算新的健康值
        const newHealth =
          state.state.health - other.rigidBody.userData.damage;

        if (newHealth <= 0) {
          state.setState("deaths", state.state.deaths + 1);
          state.setState("dead", true);
          state.setState("health", 0);
          rigidbody.current.setEnabled(false);
          setTimeout(() => {
            spawnRandomly();
            rigidbody.current.setEnabled(true);
            state.setState("health", 100);
            state.setState("dead", false);
          }, 2000);

          // 更新死亡状态数据
          onKilled(state.id, other.rigidBody.userData.player);
        } else {
          state.setState("health", newHealth);
        }
      }
    }}
    onCollisionEnter={() => {
      // 锁定跳跃
      setJumplock(false);
    }}
  >
```

## 其他问题

### 获取当前玩家状态

我们会在项目开发中通过 websocket 维护所有玩家状态，所以我们需要遍历所有玩家，那么如何判断当前联机的是哪个玩家呢？我们并不能直接通过框架外部存储的变量来判断，因为这里的变量并不是联机的数据。

所以如果需要通用的状态，我们可以用 React 来定义 `useState` 状态等。如果需要独立的状态，我们需要使用 playroomkit 封装的状态机来设置状态`state.setState('xxx', xx)` ，获取状态我们可以使用函数 `myPlayer()`、`me()`。



### 数据一致性

如果实际游玩下来，我们发现一旦弱网、卡死等情况下，玩家自身位置在其他玩家屏幕里面是不一致的，也就是`数据不一致`的情况，所以需要有一个主机来同步数据到全局，通过识别主机我们也能控制房间的权限、其他玩家的状态等。

在 `playroomkit` 中，我们可以使用 `isHost()` 来判断当前客户端是否为主机（也就是第一个开房间的房主，如果离线将自动转为另外一个玩家），如果是主机我们可以主动通过 websocket 同步主机的全局数据，而其他联机的服务将被动地接收数据。

简而言之，一般在多人游戏中，通常都会有一个主机负责同步和处理游戏状态，其他玩家根据主机发送的状态进行更新。

## 总结

最后，总结本节的核心内容：

- 在维护房间状态、玩家状态，并且要求对数据可信度要求高的时候，我们用 Websocket 来同步数据，如果数据要求不高、并且要求响应快可以使用 WebRTC 来同步数据。
- 玩家的自身状态我们需要从 playroomkit 为我们封装的 state 中获取，`state.setState('xxx', xx)` 设置状态。
- 一旦涉及联机数据维护，我们都需要使用 `isHost` 来区分主机数据和联机方的数据，`useMultiplayerState` 来得到联机数据，主机方需要广播该更新数据。简而言之，主机负责同步和处理游戏状态，其他玩家根据主机发送的状态进行更新。
- 个人建议组件的分层：基础环境组件 -> 游戏相关组件 -> 控制组件 -> 实体组件。
- 在物理世界中，我们不再考虑 three 中的修改 position 来更新物体的状态，而是修改刚体的position，如果想要实现力的效果，我们可以使用 `setLinvel` 持续力或者 `applyImpulse` 推力。
