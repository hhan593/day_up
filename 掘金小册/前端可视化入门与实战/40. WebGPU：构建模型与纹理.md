## WebGPU Samples 是如何搭建的？

WebGPU Samples 是一个全栈框架 Next.js（基于React 框架）构建的 WebGPU 的示例仓库。从这个仓库我们能快速习得很多知识。

首先从项目的 Next 服务配置文件中可以看到核心代码：

```js
  basePath: BASE_PATH,

  compress: true,
```

在之前我们写 WGSL 的时候，都是混杂在 js 中以字符串的形式传入，这种情况下没有格式化、校验的体验是很差的，所以我们首要目标是将用 WGSL 文件中写代码，我们可以用 node 监听转换，也可以像他这里的做法，直接用 Webpack 的 raw-loader、file-loader，这样我们就可以用 ESM 的模块方法直接导入。

```js
import { makeSample, SampleInit } from '../../components/SampleLayout';

// 模块化导入
import triangleVertWGSL from '../../shaders/triangle.vert.wgsl';
import redFragWGSL from '../../shaders/red.frag.wgsl';

const init: SampleInit = async ({ canvas, pageState }) => {
  const adapter = await navigator.gpu.requestAdapter();
  const device = await adapter.requestDevice();
    
  // 外部变量通过pageState控制，active控制是否渲染
  if (!pageState.active) return;
  const context = canvas.getContext('webgpu') as GPUCanvasContext;

  const devicePixelRatio = window.devicePixelRatio;
  canvas.width = canvas.clientWidth * devicePixelRatio;
  canvas.height = canvas.clientHeight * devicePixelRatio;
  const presentationFormat = navigator.gpu.getPreferredCanvasFormat();

  context.configure({
    device,
    format: presentationFormat,
    alphaMode: 'premultiplied',
  });

  const pipeline = device.createRenderPipeline({
    layout: 'auto',
    vertex: {
      module: device.createShaderModule({
        code: triangleVertWGSL,
      }),
      entryPoint: 'main',
    },
    fragment: {
      module: device.createShaderModule({
        code: redFragWGSL,
      }),
      entryPoint: 'main',
      targets: [
        {
          format: presentationFormat,
        },
      ],
    },
    primitive: {
      topology: 'triangle-list',
    },
  });

  function frame() {
    // Sample is no longer the active page.
    if (!pageState.active) return;

    const commandEncoder = device.createCommandEncoder();
    const textureView = context.getCurrentTexture().createView();

    const renderPassDescriptor: GPURenderPassDescriptor = {
      colorAttachments: [
        {
          view: textureView,
          clearValue: { r: 0.0, g: 0.0, b: 0.0, a: 1.0 },
          loadOp: 'clear',
          storeOp: 'store',
        },
      ],
    };

    const passEncoder = commandEncoder.beginRenderPass(renderPassDescriptor);
    passEncoder.setPipeline(pipeline);
    passEncoder.draw(3);
    passEncoder.end();

    device.queue.submit([commandEncoder.finish()]);
    requestAnimationFrame(frame);
  }

  requestAnimationFrame(frame);
};
```

在 Float32Array 中，一个浮点数通常由 4 个字节（32 位）来表示。每四个字节表示一个浮点数。每个字节由 8 个位组成，所以每四个字节就可以表示一个 32 位的浮点数。

另外，我们之前都是用 GPUBufferUsage.UNIFORM 的方式处理 buffer，而在这里我们使用 GPUBufferUsage.VERTEX 指定缓冲区将用于存储顶点数据，这种方式可以很方便我们写入 buffer。

```js
  new Float32Array(verticesBuffer.getMappedRange()).set(cubeVertexArray);
  verticesBuffer.unmap();
```

使用 getMappedRange 方法获取缓冲区的映射范围，并创建一个 Float32Array 数组来表示该范围。然后，我们使用 set 方法将 cubeVertexArray 的数据复制到这个映射范围中，从而将顶点数据写入缓冲区。在最后 `verticesBuffer.unmap();` 取消映射缓冲区的数据，表示我们已经完成了对缓冲区的写入操作。

另外，我们还需要定义一个 UNIFORM 方式的 buffer，用于传递矩阵数据。

接着我们创建深度纹理（存储场景的深度信息。GPU 使用这些信息来决定绘制哪些片段，丢弃哪些片段）：

```js

```

WebGPU 纹理对象的 format 属性可以是以下值之一：

- depth24plus：带有 8 位模板缓冲的 24 位深度纹理。
- depth32float：32 位浮点深度纹理。
- depth24unorm：一个 24 位无符号规范化深度纹理。
- depth32floatsnorm：32 位浮点符号归一化深度纹理。


usage 属性可以是以下值之一：

- COPY_SRC：纹理可以用作复制操作的源。
- COPY_DST：纹理可以用作复制操作的目标。
- SAMPLED：纹理可以用作采样纹理。
- STORAGE：纹理可以用作存储纹理。
- RENDER_ATTACHMENT：附加渲染的纹理。
- PRESENT：纹理可以用作呈现目标。
- MAP_READ：纹理可以映射为读访问。
- MAP_WRITE：纹理可以映射为写访问。


然后我们开始配置 renderPassDescriptor（配置渲染通道的内容）：

```js
  const renderPassDescriptor: GPURenderPassDescriptor = {
    colorAttachments: [
      {
        view: undefined, 
        clearValue: { r: 0.5, g: 0.5, b: 0.5, a: 1.0 },
        loadOp: 'clear',
        storeOp: 'store',
      },
    ],
    depthStencilAttachment: {
      view: depthTexture.createView(),

      depthClearValue: 1.0,
      depthLoadOp: 'clear',
      depthStoreOp: 'store',
    },
  };
```

colorAttachments 数组描述了渲染通道中的颜色附件。每个颜色附件都有一个 view 属性，它引用一个纹理视图，该纹理视图将用于渲染通道中的颜色。colorAttachments 数组中的第一个颜色附件是默认颜色附件，它将用于渲染通道中的所有颜色。

clearValue 属性指定了渲染通道中的颜色附件在渲染之前被清除时的颜色。

loadOp 属性指定了渲染通道中的颜色附件在渲染之前如何被处理。'clear' 表示将颜色附件清除为 clearValue 指定的颜色。'load' 表示将颜色附件从帧缓冲区中加载。

storeOp 属性指定了渲染通道中的颜色附件在渲染之后如何被处理。'store' 表示将颜色附件存储到帧缓冲区中。'discard' 表示将颜色附件丢弃。

depthStencilAttachment 属性描述了渲染通道中的深度/模板附件（内部属性配置同上）。

最后就是用 `requestAnimationFrame` 动画渲染，但是在这之前我们需要定义初始的矩阵，而在实际开发中我们会使用现成封装的矩阵库，比如 wgpu-matrix。具体用法：[greggman/wgpu-matrix: Fast WebGPU 3d math library (github.com)](https://github.com/greggman/wgpu-matrix)。

```js
  const aspect = canvas.width / canvas.height;
  const projectionMatrix = mat4.perspective(
    (2 * Math.PI) / 5,
    aspect,
    1,
    100.0
  );
  const modelViewProjectionMatrix = mat4.create();

  function getTransformationMatrix() {
    const viewMatrix = mat4.identity();
    mat4.translate(viewMatrix, vec3.fromValues(0, 0, -4), viewMatrix);
    const now = Date.now() / 1000;
    mat4.rotate(
      viewMatrix,
      vec3.fromValues(Math.sin(now), Math.cos(now), 0),
      1,
      viewMatrix
    );

    mat4.multiply(projectionMatrix, viewMatrix, modelViewProjectionMatrix);

    return modelViewProjectionMatrix as Float32Array;
  }
```
getTransformationMatrix() 函数返回模型视图投影矩阵。这个函数首先创建一个单位矩阵，然后将它平移到 (0, 0, -4) 的位置。然后，它使用 sin() 和 cos() 函数计算当前时间，并将这个值作为旋转角度。最后，它将投影矩阵和模型视图矩阵相乘，在更新函数中我们会使用这个返回的复合矩阵 `modelViewProjectionMatrix`。

```js
  function frame() {
    if (!pageState.active) return;
    // 复合矩阵
    const transformationMatrix = getTransformationMatrix();
    // 将复合矩阵的buffer写入 uniformBuffer
    device.queue.writeBuffer(
      uniformBuffer,
      //uniformBuffer的起始偏移位置
      0,
      // Float32Array 对象存在buffer属性
      transformationMatrix.buffer,
      // 缓冲区偏移量
      transformationMatrix.byteOffset,
      // 缓冲区字节长度
      transformationMatrix.byteLength
    );
    renderPassDescriptor.colorAttachments[0].view = context
      .getCurrentTexture()
      .createView();

    const commandEncoder = device.createCommandEncoder();
    const passEncoder = commandEncoder.beginRenderPass(renderPassDescriptor);
    passEncoder.setPipeline(pipeline);
    // uniform 的方式使用 setBindGroup 设置矩阵数据的buffer
    passEncoder.setBindGroup(0, uniformBindGroup);
    // vertex 的方式使用 setVertexBuffer 设置顶点buffer
    passEncoder.setVertexBuffer(0, verticesBuffer);
    passEncoder.draw(cubeVertexCount);
    passEncoder.end();
    device.queue.submit([commandEncoder.finish()]);

    requestAnimationFrame(frame);
  }
```

## Web Worker 立方体

我们知道 WebGPU 是天然支持多线程的，我们可以应用 Web Worker 来实现性能的提升，web worker 是一种在后台运行的脚本，它可以独立于主线程执行任务。web worker 可以用于执行耗时或计算密集型任务，而不会阻塞主线程。

`Worker` 对象有一个 `postMessage()` 方法，可以将消息发送到工作线程。工作线程可以通过 `addEventListener()` 方法注册事件监听器，以接收来自主线程的消息。

比如这样：

```js
// index.js

// 监听 worker里面传出来的消息
worker.addEventListener('message', (event) => {
  // my name is worker线程
  console.log(event.data);
});

// 发送消息
worker.postMessage('my name is 主线程');
```

而在 Worker.js 文件中：

```js
// worker.js（worker线程）
self.addEventListener('message', e => {
    // my name is 主线程
    console.log(e.data); 
    // 向主线程发送消息
    self.postMessage('my name is worker线程'); 
});
```

而我们想对立方体的渲染用 Web Worker 实现也很容易。核心原理就是使用 ` canvas.transferControlToOffscreen() ` 方法来创建一个 offscreen canvas，然后将其传递给 Worker。Worker 使用 offscreen canvas 来初始化 WebGPU 并绘制场景。场景被绘制到 offscreen canvas 中，然后自动呈现在主线程上的源 canvas 上。

```js

      default: {

      }

//...
```


通过采样器，我们可以定义纹理的解析行为，比如上面代码我们用线性过滤来缩放或缩小纹理（和 WebGL 类似），这里我来总结下所有的采样器配置。

-  `magFilter` ：指定纹理放大时的插值方式。可选值包括：
      -  `'nearest'` ：最近邻插值，使用最近的纹理像素。
      -  `'linear'` ：线性插值，对相邻的纹理像素进行插值。
-  `minFilter` ：指定纹理缩小时的插值方式。可选值包括：
      -  `'nearest'` ：最近邻插值，使用最近的纹理像素。
      -  `'linear'` ：线性插值，对相邻的纹理像素进行插值。
-  `mipmap` ：指定是否启用纹理的 mipmap。默认值为  `false` 。如果启用，将使用纹理的不同层级来提供细节级别的纹理。
-  `addressModeU` ：指定纹理在 U 轴方向上的寻址模式。可选值包括：
      -  `'clamp-to-edge'` ：超出纹理范围的坐标将被固定为边界上的最近纹理像素。
      -  `'repeat'` ：超出纹理范围的坐标将被循环重复。
      -  `'mirror-repeat'` ：超出纹理范围的坐标将被镜像重复。
-  `addressModeV` ：指定纹理在 V 轴方向上的寻址模式。可选值与  `addressModeU`  相同。
-  `addressModeW` ：指定纹理在 W 轴方向上的寻址模式。可选值与  `addressModeU`  相同。
-  `lodMinClamp` ：指定纹理的最小 Level of Detail (LOD) 值的最小值。默认值为  `-1000.0` 。
-  `lodMaxClamp` ：指定纹理的最大 Level of Detail (LOD) 值的最大值。默认值为  `1000.0` 。

## normalMap 法线贴图

我们先从核心的 wgsl 看起，它将顶点和片元着色器放在了一起，接着在顶点着色器中核心传入了三个 buffer，两个 unifrom 方式生成了两个 group 对应 spaceTransformsBuffer 和 mapInfoBuffer，分别管理矩阵、光照和纹理，另外一个是 vertex 的方式生成 buffer，用于处理不同类型的坐标。

```js
// 漫反射纹理模式
const modeAlbedoTexture = 0;
// 法线纹理模式
const modeNormalTexture = 1;
// 深度纹理模式
const modeDepthTexture = 2;
// 法线贴图模式
const modeNormalMap = 3;
// 视差缩放的模式（通过修改纹理坐标来模拟物体表面的凹凸效果）
const modeParallaxScale = 4;
// 陡峭视差的模式 （是一种视差缩放技术的变体，它更加强调凹凸效果 ）
const modeSteepParallax = 5;

struct SpaceTransforms {
  // 世界坐标系下的视图投影矩阵
  worldViewProjMatrix: mat4x4f,
  // 世界坐标系下的视图矩阵（无透视效果）
  worldViewMatrix: mat4x4f,
}

// 光源信息
struct MapInfo {
  // 光源位置
  lightPosVS: vec3f, 
  mode: u32,
  lightIntensity: f32,
  depthScale: f32,
  depthLayers: f32,
}

struct VertexInput {
  @location(0) position : vec4f,
  @location(1) normal : vec3f,
  @location(2) uv : vec2f,
  // 切线（切线通常与曲面的法线垂直）
  @location(3) vert_tan: vec3f,
  // 副切线（与切线和法线都垂直的向量，它与切线和法线构成一个正交坐标系。 ）
  @location(4) vert_bitan: vec3f,
}

struct VertexOutput {
  // 顶点在裁剪空间中的位置
  @builtin(position) posCS : vec4f,  
  // 顶点在视图空间中的位置
  @location(0) posVS : vec3f, 
  // 顶点在视图空间中的切线
  @location(1) tangentVS: vec3f,       
  // 顶点在视图空间中的双切线
  @location(2) bitangentVS: vec3f,    
  // 顶点在视图空间中的法线
  @location(3) normalVS: vec3f,  
  // 顶点的纹理坐标
  @location(5) uv : vec2f,     
}

// 矩阵相关信息
@group(0) @binding(0) var<uniform> spaceTransform : SpaceTransforms;
// 光照相关信息
@group(0) @binding(1) var<uniform> mapInfo: MapInfo;

// 采样器
@group(1) @binding(0) var textureSampler: sampler;
// 漫反射纹理
@group(1) @binding(1) var albedoTexture: texture_2d<f32>;
// 法线纹理
@group(1) @binding(2) var normalTexture: texture_2d<f32>;
// 深度纹理
@group(1) @binding(3) var depthTexture: texture_2d<f32>;

@vertex
fn vertexMain(input: VertexInput) -> VertexOutput {
  var output : VertexOutput;
  // 透视状态下世界坐标
  output.posCS = spaceTransform.worldViewProjMatrix * input.position;

  // 无透视下世界坐标
  output.posVS = (spaceTransform.worldViewMatrix * input.position).xyz;

  // 切线坐标
  output.tangentVS = (spaceTransform.worldViewMatrix * vec4(input.vert_tan, 0)).xyz;

  // 副切线坐标
  output.bitangentVS = (spaceTransform.worldViewMatrix * vec4(input.vert_bitan, 0)).xyz;

  // 法线坐标
  output.normalVS = (spaceTransform.worldViewMatrix * vec4(input.normal, 0)).xyz;

  //纹理坐标
  output.uv = input.uv;

  // 输出
  return output;
}

@fragment
fn fragmentMain(input: VertexOutput) -> @location(0) vec4f {
  // 切换转视图的矩阵
  let tangentToView = mat3x3f(
    input.tangentVS,
    input.bitangentVS,
    input.normalVS,
  );

  // 视图转切线的矩阵
  let viewToTangent = transpose(tangentToView);

  // 相机视角的单位向量
  let viewDirTS = normalize(viewToTangent * input.posVS);

  // 根据不同的模式转换不同的纹理（视差映射、陡峭视差）
  var uv : vec2f;
  switch (mapInfo.mode) {
    case modeParallaxScale: {
      uv = parallaxScale(input.uv, viewDirTS);
      break;
    }
    case modeSteepParallax: {
      uv = parallaxSteep(input.uv, viewDirTS);
      break;
    }
    default: {
      uv = input.uv;
      break;
    }
  }

  // 漫反射纹理采样
  let albedoSample = textureSample(albedoTexture, textureSampler, uv);

  // 法线纹理采样
  let normalSample = textureSample(normalTexture, textureSampler, uv);

  switch (mapInfo.mode) {
    case modeAlbedoTexture: { 
      return albedoSample;
    }
    case modeNormalTexture: { 
      return normalSample;
    }
    case modeDepthTexture: { 
      return textureSample(depthTexture, textureSampler, input.uv);
    }
    default: {
      // 得到相机视口中心发出的法向量
      let normalTS = normalSample.xyz * 2 - 1;
      let normalVS = normalize(tangentToView * normalTS);

      // 光源方向向量
      let fragToLightVS = mapInfo.lightPosVS - input.posVS;

      // 光源方向向量的平方模长
      let lightSqrDist = dot(fragToLightVS, fragToLightVS);

      let lightDirVS = fragToLightVS * inverseSqrt(lightSqrDist);

      // 计算光照强度
      let diffuseLight = mapInfo.lightIntensity * max(dot(lightDirVS, normalVS), 0) / lightSqrDist;

      // 漫反射附加光照强度得到最终的颜色
      let diffuse = albedoSample.rgb * diffuseLight;

      return vec4f(diffuse, 1.0);
    }
  }
}

// 用于计算在视图方向上由深度纹理采样和表面法线与视图方向之间的角度计算出的位移量来偏移 UV 坐标。 
fn parallaxScale(uv: vec2f, viewDirTS: vec3f) -> vec2f {
  let depthSample = textureSample(depthTexture, textureSampler, uv).r;
  return uv + viewDirTS.xy * (depthSample * mapInfo.depthScale) / -viewDirTS.z;
}

// 用于在视图方向上沿着深度纹理行走，直到射线与深度图相交，并返回相交点的 UV 坐标。类似于我们在shader学的 Ray Marching的算法 .
fn parallaxSteep(startUV: vec2f, viewDirTS: vec3f) -> vec2f {
  let ddx = dpdx(startUV);
  let ddy = dpdy(startUV);

  let uvDelta = viewDirTS.xy * mapInfo.depthScale / (-viewDirTS.z * mapInfo.depthLayers);
  let depthDelta = 1.0 / f32(mapInfo.depthLayers);
  let posDelta = vec3(uvDelta, depthDelta);

  var pos = vec3(startUV, 0);
  for (var i = 0; i < 32; i++) {
    if (pos.z >= textureSampleGrad(depthTexture, textureSampler, pos.xy, ddx, ddy).r) {
      break;
    }
    pos += posDelta;
  }

  return pos.xy;
}
```

从总的逻辑来说，就是先得到顶点的切线、法线、副切线生成视图转切线的矩阵，然后反推出视图转切线的矩阵，我们就可以将坐标相乘得到视角投射的向量，有了这个向量我们可以根据 mode 做一些比如视差映射、陡峭视差的 uv 纹理偏移来计算光照效果，又或者通过外部加载的图片用漫反射纹理采样、法线纹理采样来模拟光照效果，更又或者我们默认计算漫反射的规则，它的核心算法就是这一句 `let diffuseLight = mapInfo.lightIntensity * max(dot(lightDirVS, normalVS), 0) / lightSqrDist;` 。

在 `max(dot(lightDirVS, normalVS), 0)` 我们就可以得到 0 到 1 的光照强度的值，那么 lightIntensity 就是一个外部控制强度因子，除以 `lightSqrDist` 帮助我们来实现一个衰减的效果。

## 总结

最后，我们总结本节的知识点：

- **WebGPU Samples 是如何搭建的**？使用 Next.js 的打包配置将 node 变量混入到 jsx 组件中输出对应的源码到__source__变量上并动态编译，当客户端打开会请求这些源码 js 并渲染；
- **旋转立方体是如何实现的**？核心在于矩阵和顶点的 buffer 配置并输出到顶点着色器中；
- **如何用 Web Worker 实现立方体**？核心在于`canvas.transferControlToOffscreen` 将 canvas 传递到 Worker 线程，在 Worker 线程接收到 message 事件后执行 init 方法渲染；
- **如何实现纹理立方体？** 核心在于采样器和纹理对象的处理；
- **如何实现 normalMap 法线贴图**？核心在于光照方向向量与通过法线贴图得到法线向量的点乘计算得到光照强度，最后与漫反射（物体本身的颜色）坐标相乘即可。
