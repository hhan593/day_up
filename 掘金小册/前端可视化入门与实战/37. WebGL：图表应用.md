每当我们图表渲染的数据量巨大的时候，WebGL 可以用于突破 Web 端的图形渲染的性能瓶颈，这使我们可以做对性能要求高的渲染编辑器、特效、游戏开发等等，在本节我们将探讨 WebGL 在图表上的应用：

## 万级数据量试试 WebGL 渲染

在以往我们使用 Canvas 绘制破万数据量图表的时候，在一些性能不好的硬件上渲染还是会存在掉帧的现象，我们可以通过缓存、离屏等技术手段来优化。但是我们也可以尝试更直接的手段，用 WebGL 渲染数据点，这种方式可以最少提升10倍的性能，如果 WebGL 渲染遇到了瓶颈，我们也可以试试提升5倍多性能的 WebGPU 技术。

```js
      gl_PointSize = 3.5;

      gl_FragColor = vec4(0.0, 0.0, 1.0, 1.0);
    }

    gl.shaderSource(fragmentShader, fragmentShaderSource);
    gl.compileShader(fragmentShader);

    gl.attachShader(program, vertexShader);
    gl.attachShader(program, fragmentShader);
    gl.linkProgram(program);
    gl.useProgram(program);

    const positionBuffer = gl.createBuffer();
    gl.bindBuffer(gl.ARRAY_BUFFER, positionBuffer);

    for (let i = 0; i < numPoints; i++) {

        dataPoints.push(x, y);
    }
```

那么到底是什么原因带来了如此强大的提升呢？

1.  **并行计算与硬件加速：** WebGL 是基于 OpenGL ES 的，它使用图形硬件进行加速。WebGL 可以利用图形处理单元（GPU）并行处理大量数据
2.  **缓冲区：** 在 WebGL 中，你使用了顶点缓冲区来存储大量顶点数据，并通过一次性将所有顶点数据传递给 GPU。这样可以减少 CPU 到 GPU 的数据传输次数。而在 Canvas 2D 上下文中，每个点都需要通过 API 调用进行绘制，导致了更多的 CPU 到 GPU 数据传输。
3.  **图形管线：** WebGL 使用了图形管线来处理图形数据，这是一种高度优化的渲染管道。这个管线包括顶点着色器、片元着色器等阶段，可以高效地处理图形数据。Canvas 2D 上下文也有一定的图形管线，但通常没有 WebGL 的灵活性和效率高。

所以当你在项目中对海量数据已经尝试了各种优化，还希望有更直接的性能提升，不妨试试用 WebGL 渲染。

实现饼图也是一样的做法，只不过我们需要用三角函数计算每个细分线段对应顶点的坐标。

```js
function createSphereVertices(
    innerRadians, // 内环半径
    outerRadians,// 外环半径
    xGrids = 20, // 弧形的精度
    startPI = 0,
    endPI = Math.PI * 1,
    y = 1
) {
    const numVertices = xGrids * 2 + 2; //顶点数量
    const positions = webglUtils.createAugmentedTypedArray(3, numVertices);
    const normals = webglUtils.createAugmentedTypedArray(3, numVertices);
    const texCoords = webglUtils.createAugmentedTypedArray(2, numVertices);
    const indices = webglUtils.createAugmentedTypedArray(3, xGrids * 2, Uint16Array); //索引数

    let ringTextList = []
    let mid = xGrids >> 1

    for (let i = 0; i <= xGrids; i++) {
        let angle = (endPI - startPI) * (i / xGrids) + startPI;
        if (mid === i && y > 1) {
            let midRadians = innerRadians + (outerRadians - innerRadians) / 2
            textPointList.push([midRadians * Math.sin(angle), y, midRadians * Math.cos(angle), 1])
        }
        // 内环
        positions.push(
            innerRadians * Math.sin(angle),
            y,
            innerRadians * Math.cos(angle),
        );
        // 外环
        positions.push(
            outerRadians * Math.sin(angle),
            y,
            outerRadians * Math.cos(angle),
        )
        normals.push(1, 1, 1)
        texCoords.push(1, 1)
    }

    // 生成索引
    for (let i = 0; i < xGrids; i++) {
        let p0 = i * 2; // 0
        let p1 = i * 2 + 1; // 1
        let p2 = (i + 1) * 2 + 1; // 3
        let p3 = (i + 1) * 2; // 2
        if (i == xGrids) {
            p2 = 1;
            p3 = 0;
        }
        // 索引的顶点引用顺序
        indices.push(p0, p1, p2, p2, p3, p0)
    }

    return {
        position: positions,
        normal: normals, // 法线
        texcoord: texCoords, // 纹理
        indices: indices, // 索引
    };
}
```

## 总结

- 当场景的绘制元素数量超过万级或者容易存在卡顿的情况下，我们可以利用其缓冲区和更高效的 GPU 渲染能力的 WebGL、 WebGPU来解决性能问题；
- 在做任何三维图表前，我们需要知道核心的逻辑顺序：定义图表数据结构 -> 创建 webgl 的上下文和程序对象 -> 创建每个图元的顶点、颜色信息以及绑定到缓冲区 -> 创建着色器代码并接收坐标数据 -> 渲染。
