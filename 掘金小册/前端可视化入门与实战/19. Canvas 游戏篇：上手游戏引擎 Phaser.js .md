在我们平常开发中，或多或少会遇到各种活动，这个时候身为前端会接到各种游戏需求，比如抢红包、跳一跳、跑一跑等。对于普通的 H5 游戏来说，我们确实没必要用 H5 的游戏框架，但是通过游戏框架的学习，我们能知道游戏框架的设计思路及原理、为什么要这么封装，以及以后我们在写的时候是不是能借鉴其中的精华。

本文将采用 TS + 2D 游戏引擎 `Phaser.js` 来实现 2D 横板 H5 闯关游戏，不过首先我们得来聊聊什么是 Phaser.js。



## 为什么要用 Phaser.js

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5e0c2b6ff52b4ce883918e5a8b9b86b4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1209&h=595&s=1258129&e=png&b=c6eff7" alt="image.png"  /></p>

**Phaser 是一个用于创建 2D 游戏和交互式应用程序的开源框架**。它是基于 JavaScript 的，并且具有强大的功能和丰富的工具集，使开发者能够快速构建游戏和应用程序。

Phaser 提供了许多功能，包括场景管理、精灵动画、物理引擎、输入处理、音频管理、粒子效果等，使开发者能够轻松创建游戏中的角色、场景、动画和交互。

同时我们不免需要跟其他 2D 游戏引擎做比较：

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/042e05dde6014da4a7c98631ecd3a484~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=400&h=474&s=30369&e=png&b=fefefe" alt="image.png"  /></p>

很明显 Phaser 缺点体积会大点，因为功能多但本身也按需加载问题不大。Pixi 相比体积小，动画内置的能力更强。总体而言，Phaser 能力更强，同时文档和社区更容易帮助新手入门。另外，Phaser 也被很多大公司使用开发游戏。

> 你可以先在官网看看 [Phaser - A fast, fun and free open source HTML5 game framework](https://phaser.io/)。

> 然后在开始本文之前请手动实现里面的教学案例，当然也十分简单：
> 
> [Phaser 3 入门实例教程: 第一节 - 前言 - Learn - Phaser](https://phaser.io/tutorials/making-your-first-phaser-3-game-chinese)



我们会用 `Phaser 3.5 版本`，注意当前版本跟之前版本存在较大差别，比如多纹理支持，而不用一个纹理单元反复更新、更高级的后期处理能力、六边形等复杂瓦片地图支持、sprite 容器更新优化、更高效的点光源等。

## Parcel 打包器

![image.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ebb3f865dcaa4f858d2ad33b8be373a7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1302&h=652&s=422339&e=png&b=4c46e3)

**Parcel 是一个现代的 `零配置` 打包工具，用于构建 Web 应用程序。**

优势在于：

1. **零配置**：Parcel 提供了默认的配置，无需手动配置即可进行打包。它会自动检测和处理项目中的资源。 
2. **快速构建**：Parcel 使用多核编译和缓存来提高构建速度，从而加快开发迭代周期。
3. **模块热替换**（HMR）：Parcel 支持模块热替换，可以在开发过程中实时更新修改的模块，无需刷新整个页面。   
4. **插件系统**：Parcel 提供了插件系统，可以扩展其功能，满足特定项目的需求。

首先安装全局依赖：

```node
npm i parcel -g
```

然后安装对应的 parcel 的插件、ts、eslint、phaser 等。紧接着我们配置好 parcel 的两个脚本命令，来启动本地环境和生产环境。

## TexturePacker 纹理打包器

![image.png](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0641a481e2df41f890849b153f9192c1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1309&h=546&s=636295&e=png&b=f3f2f2)

为什么要用 TexturePacker？因为它可以快速处理精灵图尺寸、压缩体积，帮助我们导出 json，在 json 中有定义好了的位置信息等，我们可以直接用而不是自己设置。

![image.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5bc2e1766e247809593a51b27fbfe94~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1299&h=770&s=313246&e=png&b=f9f9f9)

### 图块层和对象层

我们知道图层上面的会覆盖下面的，我们将图层分为`图像层`、`对象层`和`图块层`。简单说，对象层我们做墙体，图块层我们做动态的东西，比如金币、碰撞物、敌人。

接下来会讲解几个核心关键点，也是容易忽视的地方。

第一个关键点，设置碰撞属性。

<p align=center><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1ee7922016c94ba8b690fa715f7740f6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1350&h=888&s=193411&e=png&b=e4e3e3" alt="image.png"  /></p>

为了设置墙体，我们需要设置碰撞属性，在编辑按钮打开后，左下角的加号创建。这样我们就可以在 matter.js 物理引擎中直接模拟真实的物体。

```js
const ground = map.createLayer(
  "ground",
  tileset
) as Phaser.Tilemaps.TilemapLayer;
 ground.setCollisionByProperty({ collides: true });
 //...
 this.matter.world.convertTilemapLayer(ground);
```

```js
         "type":"map",
         "version":"1.10",
         "width":50
}
```
以下是对配置文件中核心字段的解释： 

- `compressionlevel` ：压缩级别，此处为 -1 表示未压缩。 
- `height` ：地图的高度，以瓦片为单位。 
- `infinite` ：地图是否为无限大小。 
- `layers` ：地图的图层数组，包含了地图的各个图层的信息。 
- `data` ：图层的数据数组，表示每个瓦片的 ID。 
- `id` ：图层的唯一标识符。 
- `name` ：图层的名称。 
- `nextlayerid` ：下一个图层的 ID。 
- `nextobjectid` ：下一个对象的 ID。 
- `orientation` ：地图的方向，此处为 "orthogonal" 表示正交方向。 
- `renderorder` ：渲染顺序，此处为 "right-down" 表示从右到左、从上到下的渲染顺序。 
- `tiledversion` ：Tiled 编辑器的版本号。 
- `tilesets` ：瓦片集的数组，包含了地图使用的瓦片集的信息。 
- `columns` ：瓦片集中的列数。 
- `firstgid` ：瓦片集的第一个全局 ID。 
- `margin` ：瓦片集图像的边距。 
- `spacing` ：瓦片之间的间距。 
- `tilecount` ：瓦片集中的瓦片数量。 
- `tileheight` ：瓦片集中瓦片的高度。 
- `tiles` ：瓦片集中每个瓦片的属性数组。 
- `tilewidth` ：瓦片集中瓦片的宽度。 
- `version` ：地图的版本号。
