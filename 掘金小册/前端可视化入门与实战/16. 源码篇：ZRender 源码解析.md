## ZRender

从 echarts 的包文件中，我们可以发现其核心依赖是 ZRender，如下图：

![image.png](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a36ca26affc048b4890d02906eeef502~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=789&h=555&s=66243&e=png&b=212121)

**ZRender 是一个基于 Canvas 和 SVG 的渲染引擎，用于在 Web 端绘制图形和动画**。它是百度 **ECharts** 的底层渲染工具，提供了丰富的图形绘制和交互功能。

ZRender 具有高性能、轻量级和可扩展的特点，能够快速渲染大规模的图形和动画效果。使用它，我们可以方便地创建各种交互式的数据可视化图表和图形应用。

在阅读源码过程中，我会忽视并移除目前看起来无关紧要的代码，希望你能慢慢体会这种设计思想，并将其应用到实际的项目封装中。

具体代码如下：

```js
// zrender.ts
class ZRender {
  dom?: HTMLElement;

  id: number;
    
  // 数据层
  storage: Storage;
  // 绘制层
  painter: PainterBase;
  // 事件层
  handler: Handler;
  // 动画层
  animation: Animation;
    
    
  // 实例id、实例dom、配置
  constructor(id: number, dom?: HTMLElement, opts?: ZRenderInitOpt) {
    opts = opts || {};
    this.dom = dom;
    this.id = id;
    
    const storage = new Storage();
    
    // 选择的渲染器，默认canvas
    let rendererType = opts.renderer || "canvas";
    
    // 存在不兼容情况，使用第一次注册的渲染器
    if (!painterCtors[rendererType]) {
      rendererType = zrUtil.keys(painterCtors)[0];
    }
    
    if (process.env.NODE_ENV !== "production") {
      if (!painterCtors[rendererType]) {
        throw new Error(
          `Renderer '${rendererType}' is not imported. Please import it first.`
        );
      }
    }
    
    // 缓存处理
    opts.useDirtyRect = opts.useDirtyRect == null ? false : opts.useDirtyRect;
    
    // 是否是服务端渲染
    const ssrMode = opts.ssr || painter.ssrOnly;
    
    const painter = new painterCtors[rendererType](dom, storage, opts, id);
    this.storage = storage;
    this.painter = painter;
    
    // 对数据做代理
    const handerProxy =
      !env.node && !env.worker && !ssrMode
        ? new HandlerProxy(painter.getViewportRoot(), painter.root)
        : null;
    
    // 元素响应范围的处理
    const useCoarsePointer = opts.useCoarsePointer;
    const usePointerSize =
      useCoarsePointer == null || useCoarsePointer === "auto"
        ? env.touchEventsSupported
        : !!useCoarsePointer;
    const defaultPointerSize = 44;
    let pointerSize;
    if (usePointerSize) {
      pointerSize = zrUtil.retrieve2(opts.pointerSize, defaultPointerSize);
    }
    
    // 事件层
    this.handler = new Handler(
      storage,
      painter,
      handerProxy,
      painter.root,
      pointerSize
    );
    
    // 如果是服务端渲染就不能实时更新
    this.animation = new Animation({
      stage: {
        update: ssrMode ? null : () => this._flush(true),
      },
    });
    // 客户端开始渲染
    if (!ssrMode) {
      this.animation.start();
    }
  }
```

上述代码首先得到绘图的 dom 区域、配置项，我们可以对此做校验。接着我们需要获取相应渲染器、渲染环境等处理。最后按照绘制顺序：数据 -> 绘图 api -> 鼠标交互 -> 动画更新，我们逐一挂载数据层、视图层、事件层、动画层。

下图是 ZRender 的一个 MVC 架构模式，用于组织和管理应用程序的代码。它将应用程序分为三个主要组件：模型（Model）、视图（View）和控制器（Controller）。

ZRender 的核心入口是 Zrender 类，为了初始化的入口实例化对象，所以有了 init 方法；为了能够维护不同的实例对象，我们可以通过 id 查找，所以有了 getInstance。为了销毁对象，所以也有了 dispose 方法，disposeAll 来销毁全部。 

同时也考虑到了多个绘图器的注册，统一挂载在 `painterCtors` 中。

## Storage 数据层

该类实现了一个简单的存储和绘制队列，用于处理图形元素的更新和绘制操作。它提供了添加、删除、获取、遍历根元素的方法，并通过 `updateDisplayList` 方法更新绘制队列。

```js
export default class Storage {
  private _roots: Element[] = [];

  private _displayList: Displayable[] = [];

  private _displayListLen = 0;

  // 递归遍历元素
  traverse<T>(cb: (this: T, el: Element) => void, context?: T) {
    for (let i = 0; i < this._roots.length; i++) {
      this._roots[i].traverse(cb, context);
    }
  }
  
  // ...
  
  /**
   * 更新图形的绘制队列。
   * 每次绘制前都会调用，该方法会先深度优先遍历整个树，更新所有Group和Shape的变换并且把所有可见的Shape保存到数组中，
   * 最后根据绘制的优先级（zlevel > z > 插入顺序）排序得到绘制队列
   */
  updateDisplayList(includeIgnore?: boolean) {
    this._displayListLen = 0;

    const roots = this._roots;
    const displayList = this._displayList;
    for (let i = 0, len = roots.length; i < len; i++) {
      // 深度优先遍历每个树的根节点
      this._updateAndAddDisplayable(roots[i], null, includeIgnore);
    }

    displayList.length = this._displayListLen;
    
    // 在shapeCompareFunc定义了根据层级zlevel排序
    timsort(displayList, shapeCompareFunc);
  }
  
  // 添加图形(Displayable)或者组(Group)到根节点
  addRoot(el: Element) {
    if (el.__zr && el.__zr.storage === this) {
      return;
    }
    this._roots.push(el);
  }

  // 删除指定的图形(Displayable)或者组(Group)
  delRoot(el: Element | Element[]) {
    if (el instanceof Array) {
      for (let i = 0, l = el.length; i < l; i++) {
        this.delRoot(el[i]);
      }
      return;
    }

    const idx = util.indexOf(this._roots, el);
    if (idx >= 0) {
      this._roots.splice(idx, 1);
    }
  }
  //  清空并且释放Storage
  dispose() {
    this._displayList = null;
    this._roots = null;
  }
  //...
}
```

为什么要区分 `_roots` 和 `_displayList` 呢？

因为一个场景中会有多个组（树结构）和元素构成，需要通过`_roots`存储来管理场景。当然，我们在场景中要考虑 zlevel 的层级因素，它会直接影响前后的渲染顺序。`_displayList`是一个无嵌套结构的数组，在这里没有组和元素的区分，所有组会以深度优先遍历 push 到`_displayList`，同时 zlevel 越大越往后置。

接下来我们讲解下 updateDisplayList 中的 `_updateAndAddDisplayable`。代码如下：



```js 
/**
* 更新并添加可绘制元素到绘制队列
* @param el 元素
* @param clipPaths 剪裁路径
* @param includeIgnore 是否包含被忽略的元素
*/
private _updateAndAddDisplayable(
  el: Element,
  clipPaths: Path[],
  includeIgnore?: boolean
) {
  // 如果元素被忽略且不包含被忽略的元素，则直接返回
  if (el.ignore && !includeIgnore) {
    return;
  }
  // 在更新前执行一些操作
  el.beforeUpdate();
  // 更新元素的状态和属性
  el.update();
  // 在更新后执行一些操作
  el.afterUpdate();

  // 获取元素的剪裁路径
  const userSetClipPath = el.getClipPath();
  if (el.ignoreClip) {
    // 如果元素设置了忽略剪裁，则将剪裁路径设置为null
    clipPaths = null;
  } else if (userSetClipPath) {
    // 如果元素设置了剪裁路径
    // FIXME 效率影响
    if (clipPaths) {
      // 复制剪裁路径数组，避免对原数组的修改
      clipPaths = clipPaths.slice();
    } else {
      clipPaths = [];
    }
    let currentClipPath = userSetClipPath;
    let parentClipPath = el;
    // 递归添加剪裁路径
    while (currentClipPath) {
      // 设置剪裁路径的父级元素
      currentClipPath.parent = parentClipPath as Group;
      // 更新剪裁路径的变换
      currentClipPath.updateTransform();
      // 将剪裁路径添加到数组中
      clipPaths.push(currentClipPath);
      parentClipPath = currentClipPath;
      // 获取剪裁路径的父级剪裁路径
      currentClipPath = currentClipPath.getClipPath();
    }
  }

  // 如果元素是一个组（Group）或具有子元素（childrenRef存在）
  if ((el as GroupLike).childrenRef) {
    const children = (el as GroupLike).childrenRef();
    for (let i = 0; i < children.length; i++) {
      const child = children[i];
      // 如果组元素（el）被标记为脏（dirty），则将子元素的脏标记（dirty）设置为REDRAW_BIT，表示需要重新绘制
      if (el.__dirty) {
        child.__dirty |= REDRAW_BIT;
      }
      // 递归调用_updateAndAddDisplayable方法，更新并添加子元素到绘制队列中
      this._updateAndAddDisplayable(child, clipPaths, includeIgnore);
    }
    // 在此处标记组元素为干净（clean）
    el.__dirty = 0;
  } else {
    const disp = el as Displayable;
    // 如果元素是可绘制元素（Displayable）
    if (clipPaths && clipPaths.length) {
      // 如果剪裁路径存在且长度大于0，则将剪裁路径赋值给元素的__clipPaths属性
      disp.__clipPaths = clipPaths;
    } else if (disp.__clipPaths && disp.__clipPaths.length > 0) {
      // 否则，如果元素的__clipPaths属性存在且长度大于0，则将其重置为空数组
      disp.__clipPaths = [];
    }
    // 避免无效的z、z2和zlevel属性，如果这些属性为NaN，则将其设置为0
    if (isNaN(disp.z)) {
      logInvalidZError();
      disp.z = 0;
    }
    if (isNaN(disp.z2)) {
      logInvalidZError();
      disp.z2 = 0;
    }
    if (isNaN(disp.zlevel)) {
      logInvalidZError();
      disp.zlevel = 0;
    }
    // 将元素添加到绘制队列中
    this._displayList[this._displayListLen++] = disp;
  }

  // 添加装饰元素（decalEl）到绘制队列
  const decalEl =
    (el as Path).getDecalElement && (el as Path).getDecalElement();
  if (decalEl) {
    this._updateAndAddDisplayable(decalEl, clipPaths, includeIgnore);
  }

  // 添加附加的文本元素（textGuide）和引导线（textEl）到绘制队列
  const textGuide = el.getTextGuideLine();
  if (textGuide) {
    this._updateAndAddDisplayable(textGuide, clipPaths, includeIgnore);
  }
  const textEl = el.getTextContent();
  if (textEl) {
    this._updateAndAddDisplayable(textEl, clipPaths, includeIgnore);
  }
}

```

```js
            this._height = height;

            mainLayer.__builtin__ = true;
            mainLayer.initContext();
            layers[CANVAS_ZLEVEL] = mainLayer;
            mainLayer.zlevel = CANVAS_ZLEVEL;
            zlevelList.push(CANVAS_ZLEVEL);

            this._domRoot = root;
        }
    }

    /**
     * 刷新
     * @param paintAll 强制绘制所有displayable
     */
    refresh(paintAll?: boolean) {
        const list = this.storage.getDisplayList(true);
        const prevList = this._prevDisplayList;

        const zlevelList = this._zlevelList;
        // 生成一个随机的重绘ID，用于标识当前的重绘操作。
        this._redrawId = Math.random();

        this._paintList(list, prevList, paintAll, this._redrawId);

        // Paint custum layers
        for (let i = 0; i < zlevelList.length; i++) {
            const z = zlevelList[i];
            const layer = this._layers[z];
            if (!layer.__builtin__ && layer.refresh) {
                const clearColor = i === 0 ? this._backgroundColor : null;
                layer.refresh(clearColor);
            }
        }

        if (this._opts.useDirtyRect) {
            this._prevDisplayList = list.slice();
        }
    
        // 返回当前 `Painter` 实例，以便进行链式调用
        return this;
    }
```

### _paintList 绘制列表

```js
 private _paintList(list: Displayable[], prevList: Displayable[], paintAll: boolean, redrawId?: number) {
        if (this._redrawId !== redrawId) {
            return;
        }
        paintAll = paintAll || false;

        this._updateLayerStatus(list);

        const {finished, needsRefreshHover} = this._doPaintList(list, prevList, paintAll);
    
        // 是否需要手动合成
        if (this._needsManuallyCompositing) {
            this._compositeManually();
        }
        // 处理悬停状态的list
        if (needsRefreshHover) {
            this._paintHoverList(list);
        }
        //如果没有绘制完，接着在下一帧渲染
        if (!finished) {
            const self = this;
            requestAnimationFrame(function () {
                self._paintList(list, prevList, paintAll, redrawId);
            });
        }
        else {
           // 更新图层的绘制状态
            this.eachLayer(layer => {
                layer.afterBrush && layer.afterBrush();
            });
        }
    }
```

`_paintList` 主要是渲染的入口，在 `_doPaintList` 中一直会有 finished 状态能获取到，我们很清楚当前是否已渲染，如未渲染完图形会继续 requestAnimationFrame，而不是在 JavaScript 执行中阻塞渲染，更准确的来说是 `渐进式更新`。



### _doPaintList

根据绘制需求，对存储中的可绘制元素列表进行绘制操作。它会遍历图层、处理脏矩形优化、进行增量绘制，并返回绘制是否完成以及是否需要刷新悬停元素的信息。

```js
private _doPaintList(
        list: Displayable[],
        prevList: Displayable[],
        paintAll?: boolean
    ): {
        finished: boolean
        needsRefreshHover: boolean
    } {
        const layerList = [];
        const useDirtyRect = this._opts.useDirtyRect;
        //...

        let finished = true;
        let needsRefreshHover = false;

        for (let k = 0; k < layerList.length; k++) {
            const layer = layerList[k];
            const ctx = layer.ctx;

            // ...
        
            let i: number;
            const repaint = (repaintRect?: BoundingRect) => {
                const scope: BrushScope = {
                    inHover: false,
                    allClipped: false,
                    prevEl: null,
                    viewWidth: this._width,
                    viewHeight: this._height
                };

                for (i = start; i < layer.__endIndex; i++) {
                    const el = list[i];

                    if (el.__inHover) {
                        needsRefreshHover = true;
                    }

                    this._doPaintEl(el, layer, useDirtyRect, repaintRect, scope, i === layer.__endIndex - 1);
                    
                    // 如果使用了定时器，根据时间判断是否需要中断绘制
                    if (useTimer) {
                        const dTime = Date.now() - startTime;
                        if (dTime > 15) {
                            break;
                        }
                    }
                }
                 // 如果存在之前保存的剪裁路径，则恢复上下文状态。
                if (scope.prevElClipPaths) {
                    ctx.restore();
                }
            };
               
            // 是否存在重绘区域
            if (repaintRects) {
                if (repaintRects.length === 0) {
                    // Nothing to repaint, mark as finished
                    i = layer.__endIndex;
                }
                else {
                    const dpr = this.dpr;
                    // Set repaintRect as clipPath
                    for (var r = 0; r < repaintRects.length; ++r) {
                        const rect = repaintRects[r];

                        ctx.save();
                        ctx.beginPath();
                        ctx.rect(
                            rect.x * dpr,
                            rect.y * dpr,
                            rect.width * dpr,
                            rect.height * dpr
                        );
                        ctx.clip();

                        repaint(rect);
                        ctx.restore();
                    }
                }
            }
            else {
                // Paint all once
                ctx.save();
                repaint();
                ctx.restore();
            }

            layer.__drawIndex = i;

            if (layer.__drawIndex < layer.__endIndex) {
                finished = false;
            }
        }
        
        // 如果运行环境是微信小程序，则刷新所有图层的绘制。
        if (env.wxa) {
            // Flush for weixin application

            });

            finished,
            needsRefreshHover
```

```js

    el: Displayable,
    currentLayer: Layer,
    useDirtyRect: boolean,
    repaintRect: BoundingRect,

    isLast: boolean
```

### _updateLayerStatus 更新图层状态

```js
_updateLayerStatus(list: Displayable[]) {
        // 遍历已构建的图层，此时状态未更新
        this.eachBuiltinLayer(function (layer, z) {
            layer.__dirty = layer.__used = false;
        });
    
        // 更新后（一定要考虑图层的生命周期）
        function updatePrevLayer(idx: number) {
            if (prevLayer) {
                if (prevLayer.__endIndex !== idx) {
                    prevLayer.__dirty = true;
                }
                prevLayer.__endIndex = idx;
            }
        }

        if (this._singleCanvas) {
            for (let i = 1; i < list.length; i++) {
                const el = list[i];
                // 如果是只有一个绘图上下文，如果该图层被修改或增量渲染就手动更新
                if (el.zlevel !== list[i - 1].zlevel || el.incremental) {
                    this._needsManuallyCompositing = true;
                    break;
                }
            }
        }
        //...
        for (i = 0; i < list.length; i++) {
            const el = list[i];
            const zlevel = el.zlevel;
            let layer;

            if (prevZlevel !== zlevel) {
                prevZlevel = zlevel;
                incrementalLayerCount = 0;
            }
            
            // 在我们之前使用number处理图层的key，但是存在局限性，所以另外通过3种不同的方式来定义增量更新的状态，1. 增量元素后绘制的其他元素 2.增量元素 3. 在增量元素之前绘制的元素
            // TODO Not use magic number on zlevel.

            // Each layer with increment element can be separated to 3 layers.
            //          (Other Element drawn after incremental element)
            // -----------------zlevel + EL_AFTER_INCREMENTAL_INC--------------------
            //                      (Incremental element)
            // ----------------------zlevel + INCREMENTAL_INC------------------------
            //              (Element drawn before incremental element)
            // --------------------------------zlevel--------------------------------
            if (el.incremental) {
                layer = this.getLayer(zlevel + INCREMENTAL_INC, this._needsManuallyCompositing);
                layer.incremental = true;
                incrementalLayerCount = 1;
            }
            else {
                layer = this.getLayer(
                    zlevel + (incrementalLayerCount > 0 ? EL_AFTER_INCREMENTAL_INC : 0),
                    this._needsManuallyCompositing
                );
            }
    
            if (!layer.__builtin__) {
                util.logError('ZLevel ' + zlevel + ' has been used by unkown layer ' + layer.id);
            }
            // 如果当前图层与前一个图层不同，则更新前一个图层的状态，并将 `prevLayer` 更新为当前图层
            if (layer !== prevLayer) {
                layer.__used = true;
                if (layer.__startIndex !== i) {
                    layer.__dirty = true;
                }
                layer.__startIndex = i;
                if (!layer.incremental) {
                    layer.__drawIndex = i;
                }
                else {
                    // Mark layer draw index needs to update.
                    layer.__drawIndex = -1;
                }
                updatePrevLayer(i);
                prevLayer = layer;
            }
            if ((el.__dirty & REDRAW_BIT) && !el.__inHover) {  // Ignore dirty elements in hover layer.
                layer.__dirty = true;
                if (layer.incremental && layer.__drawIndex < 0) {
                    // Start draw from the first dirty element.
                    layer.__drawIndex = i;
                }
            }
        }

        updatePrevLayer(i);
    
        // 遍历图层
        this.eachBuiltinLayer(function (layer, z) {
            //如果图层未被使用且图层中有元素，则将图层的状态重置为初始状态。
            if (!layer.__used && layer.getElementCount() > 0) {
                layer.__dirty = true;
                layer.__startIndex = layer.__endIndex = layer.__drawIndex = 0;
            }
            // 对于增量渲染图层，如果图层的状态为 `__dirty` 且 `__drawIndex` 小于 0，则将 `__drawIndex` 设置为图层的 `__startIndex`.
            if (layer.__dirty && layer.__drawIndex < 0) {
                layer.__drawIndex = layer.__startIndex;
            }
        });
    }
```
从该源码我们能发现，ZRender 之所以性能好，很大程度在于它使用了图层来区分不同的绘制，最大限度地按需更新渲染。

ZRender 的更新策略：一个是对增量渲染的图形，通过 el.incremental 来标识更新，区分增量渲染前图层、增量渲染开始图层、增量渲染后的图层，来按需更新；另外一个是只要 zLevel 不同就会更新。

对于基础图元类，我们无需添加额外的 id、事件、状态等。我们最主要在生成 buildPath 方法上，同时注意 type 的定义。

在基础图元的基础上，我们可以继承通用的父类，比如自定义的事件类、动画类等。

而在 ZRender 中，shape 的类关系是：

`Circle -> Path -> Displayable -> Element`

在 ZRender 中，Shape、Path、Displayable 和 Element 是表示图形元素的四个重要类。

- **Shape**（形状）类有 Circle、Rect、Arc 等，代表了基础的图元，图元内包含了基础的图形属性定义。

- **Path**（路径）类代表对 Shape 的更新。上层的属性封装，如控制描边、分段、样式、优化配置等。

- **Displayable**（可绘制对象）类是一个抽象类，提供了一些共享的方法和属性，如绘制、更新、剪裁等。

- **Element**（元素）类是一个更高层次的抽象类，表示图形元素的基本属性和状态。它是 Displayable 类的基类，提供了一些通用的方法和属性，如坐标变换、事件处理等。

通过这样的设计，ZRender 可以实现对各种图形元素的创建、绘制和交互等操作。



## Animation 动画层

接下来我们看看动画层是如何处理的：

```js
export default class Animation extends Eventful {
  stage: Stage;
  // 指向动画剪辑链表头部的引用
  private _head: Clip;
  //指向动画剪辑链表尾部的引用
  private _tail: Clip;

  private _running = false;

  private _time = 0;
  private _pausedTime = 0;
  private _pauseStart = 0;

  private _paused = false;

  constructor(opts?: AnimationOption) {
    super();

    opts = opts || {};

    this.stage = opts.stage || {};
  }

  addClip(clip: Clip) {
    if (clip.animation) {
      this.removeClip(clip);
    }

    if (!this._head) {
      this._head = this._tail = clip;
    } else {
      this._tail.next = clip;
      clip.prev = this._tail;
      clip.next = null;
      this._tail = clip;
    }
    clip.animation = this;
  }
  
  addAnimator(animator: Animator<any>) {
    animator.animation = this;
    const clip = animator.getClip();
    if (clip) {
      this.addClip(clip);
    }
  }
 
  removeClip(clip: Clip) {
    if (!clip.animation) {
      return;
    }
    const prev = clip.prev;
    const next = clip.next;
    if (prev) {
      prev.next = next;
    } else {
      this._head = next;
    }
    if (next) {
      next.prev = prev;
    } else {
      this._tail = prev;
    }
    clip.next = clip.prev = clip.animation = null;
  }

 
  removeAnimator(animator: Animator<any>) {
    const clip = animator.getClip();
    if (clip) {
      this.removeClip(clip);
    }
    animator.animation = null;
  }

  update(notTriggerFrameAndStageUpdate?: boolean) {
    const time = getTime() - this._pausedTime;
    const delta = time - this._time;
    let clip = this._head;

    while (clip) {
      const nextClip = clip.next;
      let finished = clip.step(time, delta);
      if (finished) {
        clip.ondestroy();
        this.removeClip(clip);
        clip = nextClip;
      } else {
        clip = nextClip;
      }
    }

    this._time = time;

    if (!notTriggerFrameAndStageUpdate) {
      this.trigger("frame", delta);

      this.stage.update && this.stage.update();
    }
  }

  _startLoop() {
    const self = this;

    this._running = true;

    function step() {
      if (self._running) {
        requestAnimationFrame(step);
        !self._paused && self.update();
      }
    }

    requestAnimationFrame(step);
  }

  start() {
    if (this._running) {
      return;
    }

    this._time = getTime();
    this._pausedTime = 0;

    this._startLoop();
  }

  stop() {
    this._running = false;
  }

  pause() {
    if (!this._paused) {
      this._pauseStart = getTime();
      this._paused = true;
    }
  }

  resume() {
    if (this._paused) {
      this._pausedTime += getTime() - this._pauseStart;
      this._paused = false;
    }
  }

  clear() {
    let clip = this._head;

    while (clip) {
      let nextClip = clip.next;
      clip.prev = clip.next = clip.animation = null;
      clip = nextClip;
    }

    this._head = this._tail = null;
  }

  isFinished() {
    return this._head == null;
  }

  animate<T>(
    target: T,
    options: {
      loop?: boolean; 
    }
  ) {
    options = options || {};

    this.start();

    const animator = new Animator(target, options.loop);
    
    // 添加一个动画器
    this.addAnimator(animator);

    return animator;
  }
}
```

对于上述源码我们并不难理解，上文实现了一个简单的动画管理类，用于管理动画效果和动画器。它提供了添加、移除剪辑和动画器的方法，以及控制动画的开始、停止、暂停和恢复等功能。通过更新和触发事件，可以实现动画效果的展示和交互。

值得注意的是，对于动画帧与帧的前后关系，我们需要使用链表来表述这种前后关系，来方便我们后续的剪辑更新、移除、增加等。

这里我们要讲下两个关键的类：

- **`Animator` 类用于管理和控制动画的播放过程**。它可以设置动画的目标对象，并通过设置动画的属性、持续时间、缓动函数等参数来定义动画的行为。 `Animator` 类提供了方法来启动、停止、暂停和恢复动画，以及设置循环播放等功能。它还可以与 `Clip` 剪辑类结合使用，将动画效果应用到指定的剪辑上。

 - **`Clip` 剪辑类是动画的基本单元，表示一个独立的动画片段**。它包含了描述动画的关键帧和过渡效果的属性。 `Clip` 类提供了方法来计算和应用动画的状态，包括在指定的时间点计算属性值、应用属性到目标对象等。通过将多个 `Clip` 剪辑组合在一起，可以创建复杂的动画效果。
 
它们的关系是： `Clip （帧） -> Animator (帧组成的一个动画) -> Animation (多个动画管理)`。

成熟的图形库要兼容不同浏览器，标准的 `requestAnimationFrame` 方法，兼容大部分现代浏览器。`msRequestAnimationFrame` 适用于 IE10 及以上版本。`mozRequestAnimationFrame` 适用于旧版 Firefox 浏览器。`webkitRequestAnimationFrame` 适用于旧版 WebKit 内核浏览器。如果不可用则使用 `setTimeout` 模拟。

```js
export default function transformPath(path: PathProxy, m: MatrixArray) {
  if (!m) {
    return;
  }

  let data = path.data;
  const len = path.len();
  let cmd;
  let nPoint: number;
  let i: number;
  let j: number;
  let k: number;
  let p: VectorArray;

  const M = CMD.M;
  const C = CMD.C;
  const L = CMD.L;
  const R = CMD.R;
  const A = CMD.A;
  const Q = CMD.Q;

  for (i = 0, j = 0; i < len; ) {
    cmd = data[i++];
    j = i;
    nPoint = 0;

    switch (cmd) {
      case M:
        nPoint = 1;
        break;
      case L:
        nPoint = 1;
        break;
      case C:
        nPoint = 3;
        break;
      case Q:
        nPoint = 2;
        break;
      case A:
        const x = m[4];
        const y = m[5];
        const sx = mathSqrt(m[0] * m[0] + m[1] * m[1]);
        const sy = mathSqrt(m[2] * m[2] + m[3] * m[3]);
        const angle = mathAtan2(-m[1] / sy, m[0] / sx);
        // cx
        data[i] *= sx;
        data[i++] += x;
        // cy
        data[i] *= sy;
        data[i++] += y;
        // Scale rx and ry
        // FIXME Assume psi is 0 here
        data[i++] *= sx;
        data[i++] *= sy;

        // Start angle
        data[i++] += angle;
        // end angle
        data[i++] += angle;
        // FIXME psi
        i += 2;
        j = i;
        break;
      case R:
        // x0, y0
        p[0] = data[i++];
        p[1] = data[i++];
        v2ApplyTransform(p, p, m);
        data[j++] = p[0];
        data[j++] = p[1];
        // x1, y1
        p[0] += data[i++];
        p[1] += data[i++];
        v2ApplyTransform(p, p, m);
        data[j++] = p[0];
        data[j++] = p[1];
    }

    for (k = 0; k < nPoint; k++) {
      let p = points[k];
      p[0] = data[i++];
      p[1] = data[i++];

      v2ApplyTransform(p, p, m);
      // Write back
      data[j++] = p[0];
      data[j++] = p[1];
    }
  }

  path.increaseVersion();
}
```

**transformPath 是一个将路径应用矩阵的方法**。对于 SVG，我们直接用 transfrom 确实挺方便的。但是对于 Canvas 来说，我们往往就需要涉及大量坐标的计算。

使用矩阵进行路径变换的好处是可以一次性对整个路径对象进行变换，而不需要逐个操作路径的各个点，这样可以提高性能并简化代码。同时，使用矩阵进行变换也更符合数学上的矩阵变换概念，可以方便地进行组合变换和逆变换等操作。

```js
                        if (out.target) {
                            return out;
                        }
                    }
                }
            }
        }

        return out;
    }
    
    // 处理手势事件
    processGesture(event: ZRRawEvent, stage?: 'start' | 'end' | 'change') {
        if (!this._gestureMgr) {
            this._gestureMgr = new GestureMgr();
        }
        const gestureMgr = this._gestureMgr;

        stage === 'start' && gestureMgr.clear();

        const gestureInfo = gestureMgr.recognize(
            event as ZRRawTouchEvent,
            this.findHover(event.zrX, event.zrY, null).target,
            (this.proxy as HandlerDomProxy).dom
        );

        stage === 'end' && gestureMgr.clear();

        if (gestureInfo) {
            const type = gestureInfo.type;
            (event as ZRPinchEvent).gestureEvent = type;

            let res = new HoveredResult();
            res.target = gestureInfo.target;
            this.dispatchToElement(res, type as ElementEventName, gestureInfo.event as ZRRawEvent);
        }
    }

    click: (event: ZRRawEvent) => void
    mousedown: (event: ZRRawEvent) => void
    mouseup: (event: ZRRawEvent) => void
    mousewheel: (event: ZRRawEvent) => void
    dblclick: (event: ZRRawEvent) => void
    contextmenu: (event: ZRRawEvent) => void
}
```

**`Handler` 类，用于处理图形元素的交互事件**。它包含了鼠标移动、点击、拖拽、手势等多种事件的处理方法，以及事件的分发和代理等功能。通过这些方法，可以实现对图形元素的交互操作和事件处理。

接下来，我们看下它是如何实现事件分发代理的：

```js
/**
 * 事件分发代理
 * @private
 * @param {Object} targetInfo {target, topTarget} 目标图形元素
 * @param {string} eventName 事件名称
 * @param {Object} event 事件对象
 */
dispatchToElement(targetInfo: {
    target?: Element
    topTarget?: Element
}, eventName: ElementEventName, event: ZRRawEvent) {

    targetInfo = targetInfo || {};

    let el = targetInfo.target as Element;
    if (el && el.silent) {
        return;
    }
    const eventKey = ('on' + eventName) as ElementEventNameWithOn;
    const eventPacket = makeEventPacket(eventName, targetInfo, event);

    while (el) {
    // 如果元素具有对应的事件处理函数，则调用该函数，并将返回值赋值给 `eventPacket.cancelBubble` ，用于判断是否取消事件冒泡。
        el[eventKey]
            && (eventPacket.cancelBubble = !!el[eventKey].call(el, eventPacket));
        //  触发元素的事件
        el.trigger(eventName, eventPacket);
        
        // 将 `el` 更新为元素的宿主目标（如果存在）或者父级元素。
        el = el.__hostTarget ? el.__hostTarget : el.parent;
        
        // 如果事件被取消冒泡，则跳出循环。
        if (eventPacket.cancelBubble) {
            break;
        }
    }
    
    // 如果无冒泡
    if (!eventPacket.cancelBubble) {
        // 冒泡到顶级 zrender 对象
        this.trigger(eventName, eventPacket);
        // 分发事件到用户自定义层
        // 用户有可能在全局 click 事件中 dispose，所以需要判断下 painter 是否存在
        if (this.painter && (this.painter as CanvasPainter).eachOtherLayer) {
                (this.painter as CanvasPainter).eachOtherLayer(function (layer) {
                // 如果图层具有对应的事件处理函数。调用图层的事件处理函数。
                if (typeof (layer[eventKey]) === 'function') {
                    layer[eventKey].call(layer, eventPacket);
                }
                // 触发图层的自身的事件
                if (layer.trigger) {
                    layer.trigger(eventName, eventPacket);
                }
            });
        }
    }
}
```

事件的分发代理功能，它会根据事件的目标元素以及事件类型，逐级向上冒泡触发事件，并在每个元素上调用对应的事件处理函数。同时，还会触发 `Handler` 对象和绘图器的事件，并将事件分发到用户自定义的图层中。这样可以实现事件的传递和处理。

接下来我们将通用的事件挂载到 Hanlder 中：

```js
util.each(['click', 'mousedown', 'mouseup', 'mousewheel', 'dblclick', 'contextmenu'], function (name: HandlerName) {
    Handler.prototype[name] = function (event) {
        const x = event.zrX;
        const y = event.zrY;
        const isOutside = isOutsideBoundary(this, x, y);

        let hovered;
        let hoveredTarget;

        if (name !== 'mouseup' || !isOutside) {
            hovered = this.findHover(x, y);
            hoveredTarget = hovered.target;
        }

        if (name === 'mousedown') {
            this._downEl = hoveredTarget;
            this._downPoint = [event.zrX, event.zrY];
            this._upEl = hoveredTarget;
        }
        else if (name === 'mouseup') {
            this._upEl = hoveredTarget;
        }
        else if (name === 'click') {
            if (this._downEl !== this._upEl
                || !this._downPoint
                || vec2.dist(this._downPoint, [event.zrX, event.zrY]) > 4
            ) {
                return;
            }
            this._downPoint = null;
        }

        this.dispatchToElement(hovered, name, event);
    };
});
```

ZRender 的事件发布流程：

1. 当原生 DOM 事件发生时，事件管理器会接收到该事件。 
2. 事件管理器会根据事件的类型和坐标信息，确定事件发生在哪个图形元素或组件上。 
3. 事件管理器会调用该图形元素或组件注册的事件处理函数来处理事件。 
4. 事件处理函数可以进行一些操作，如修改图形元素的状态、触发其他事件或更新 UI 等。




## ZRender 的性能优化

ZRender 在性能优化方面做了很多工作，以提供高性能的图形绘制能力。下面列举了一些 ZRender 的性能优化技术和策略：

1. **硬件加速**：ZRender 利用基于 WebGL 的硬件加速渲染，可以充分利用 GPU 的计算能力，提高绘制效率和性能。

2. **脏矩形优化**：ZRender 使用脏矩形（dirty rectangle）的概念，只对发生变化的区域进行重新绘制，减少了不必要的绘制操作，提高了性能。

3. **分层渲染**：ZRender 将图形元素分为多个层级，每个层级使用独立的 Canvas 进行绘制，可以减少绘制的区域和元素数量，提高渲染效率。

4. **批量渲染**：ZRender 将相同类型的图形元素进行批量渲染，减少了绘制调用的次数，提高了绘制效率。

5. **图形剪裁和裁剪路径优化**：ZRender 使用剪裁（clip）功能来限制绘制区域，避免对不可见的元素进行绘制，提高了性能。同时，ZRender 还对剪裁路径进行了优化，减少了剪裁路径的计算和应用的开销。

6. **可见性检测**：ZRender 对元素的可见性进行检测，只对可见的元素进行绘制，避免了对隐藏元素的绘制操作，提高了性能。

7. **缓存和复用**：ZRender 使用缓存和对象复用的技术，减少了内存的分配和释放，提高了性能和资源利用率。

8. **惰性更新**：ZRender 采用惰性更新的策略，只在需要重新绘制时才进行更新操作，避免了不必要的计算和绘制，提高了性能。

9. **事件优化**：ZRender 对事件的处理进行了优化，通过事件委托和事件合并等技术，减少了事件的处理次数和开销，提高了性能。

ZRender 通过多种优化技术和策略，提高了图形绘制的性能和效率，使得在大规模图形绘制和交互中能够保持较高的性能和流畅度。
