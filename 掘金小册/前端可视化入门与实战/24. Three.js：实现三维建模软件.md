为什么需要用 Zustand？因为它`解决了状态丢失的问题`，尤其是引入图形库会存在多个 React 实例出现的时候，使用 React 会无法获取状态。至于其他的优势，就是性能、简单等。

另外，如果我们直接在 react 组件内写 three 的逻辑是要提前封装 three 的组件，社区有成熟的比如 R3F 来解决这个问题，它帮助我们封装并优化了各种渲染更新、物体、灯光的组件和 hooks。在这里我们还是先将 three 从 react 中抽离出来单独维护。由于 Three.js 动态渲染需要使用 requestFrameAnimation，所以 `useStore.getState` 可以得到最新的状态值。

```js

    const { scene } = useStore.getState()
    // React 组件中使用 zustand 获取最新状态
    const frame = useStore((state) => state.frame)
    const targetInfo = useStore((state) => state.targetInfo)
 }
 
 // 在threejs目录，注意在requestAnimition作用域内执行获取到的state是最新的
 const { scene, camera, renderer, setControls } = useStore.getState();
```

### 实现网格平面

在 3D 建模中我们都会有一个网格平面， 而在 Three 中我们只需要调用 GridHelper 插件的方法。

GridHelper（网格帮助器）是 Three 中一种可视化工具，用于在 3D 场景中生成坐标轴、网格等辅助线，以便于用户进行调试、演示等操作。

```js
import * as THREE from "three";
import { OrbitControls } from "three/examples/jsm/controls/OrbitControls.js";
import useStore from "../store";
import { render } from "./render";
import { watchEvent } from "./eventWatch";
import { objectControls } from "./objectControls";
function sceneRender() {
  const { scene, camera, renderer, setControls } = useStore.getState();
  const container = document.getElementById("container") as HTMLDivElement;
  // 创建渲染器
  renderer.setSize(container.offsetWidth, container.offsetHeight);
  container.appendChild(renderer.domElement);

  camera.position.set(0, 5, 10);
  camera.lookAt(scene.position);

  // 创建网格平面
  const gridHelper = new THREE.GridHelper(30, 30, 0xaaaaaa, 0x444444);
  scene.add(gridHelper);

  // 物体操纵轴
  objectControls();
  watchEvent();

  // 创建控制器
  const controls = new OrbitControls(camera, renderer.domElement);
  controls.enableDamping = true;

  controls.update();
  setControls(controls);
  render();
}

export { sceneRender };
```

同时，我们这里使用了 Three.js 的 `OrbitControls 插件`，用于在三维场景中控制相机的旋转、缩放和平移。它允许用户通过鼠标或触摸屏幕进行交互，自由地浏览和观察场景中的对象。OrbitControls 还支持手动设置相机视角的距离、注视点和旋转轴，以及打开或关闭相机的自旋转功能。这样，使用者可以非常方便地在三维场景中进行导航和视差的调整。

创建几何体的方法：

```js
function createGeomary() {
  const { scene, geoList } = useStore.getState();

  if (geoList.length) {
    // 剔除重复的几何体
    const diff = geoList.filter(
      (obj1) => !readlyRenderList.some((obj2) => obj1.gid === obj2.gid)
    );
    diff.length &&
      diff.forEach((el) => {
        const { BoxGeometry, ConeGeometry, SphereGeometry, type } = el;
        let geometry = null,
          h = 0;
        if (type === GEOMARY.CUBE && BoxGeometry) {
          geometry = new THREE.BoxGeometry(...BoxGeometry);
          h = BoxGeometry[1] / 2;
        }
        if (type === GEOMARY.CONE && ConeGeometry) {
          geometry = new THREE.ConeGeometry(...ConeGeometry);
          h = ConeGeometry[1] / 2;
        }
        if (type === GEOMARY.SPHERE && SphereGeometry) {
          geometry = new THREE.SphereGeometry(...SphereGeometry);
          h = SphereGeometry[0];
        }

        if (geometry) {
          const material = new THREE.MeshNormalMaterial();
          const renderGeometry = new THREE.Mesh(geometry, material);
          renderGeometry.userData.el = el;
           
          // 封装的射线方法
          const { x, y, z } = getRayPos(el.x, el.y);
          renderGeometry.position.set(x, y + h, z);
          scene.add(renderGeometry);
          readlyRenderList.push(el);
        }
      });
  }
}

export { createGeomary };
```

在 30 行我们可以看到使用了 `userData`，在所有几何体中，我们都能看到对象内存在 `userData`，我们可以将外部属性统一挂载到此，因为这样 three 内部状态可以共享，更方便我们获取。


在 33 行的 `getRayPos` 是一个**射线算法**，我们传入屏幕坐标系的鼠标坐标后，可以映射到 WebGL 的 0 到 1 的范围。通过这个，我们能判断拖拽在 WebGL 中的实际位置。然后，我们只要根据之前定义的 `geoList` 遍历创建对应的几何体。这里要注意我们的射线是以鼠标位置与网格面的相交点，而 threejs 几何体的创建是以坐标为中心创建的，当拖拽生成的时候我们会发现有错误的高度：

### 射线算法

在 Three.js 中，射线算法常用于实现鼠标拾取（mouse picking）和碰撞检测（collision detection）等功能。在 Three.js 中，可以使用 `THREE.Raycaster类`来创建射线，并使用它来进行鼠标拾取和碰撞检测。

射线由起点和方向向量组成，始于三维空间中的某一点，朝着某一方向延伸。

射线与场景中的物体的交点可以通过以下步骤计算：

1.  将射线转换到物体所在的局部坐标系中，这可以通过物体的世界矩阵的逆变换实现。
1.  对于每个物体三角形，计算该三角形的平面方程，确定射线是否与该三角形相交。
1.  如果存在相交三角形，则找到其中与射线相交的最近的三角形。这可以通过比较交点的距离来实现。
1.  将交点的坐标从局部坐标系转换回全局坐标系，以在场景中得到正确的位置。

从硬件加速的角度来看，射线算法基于 BVH（Bounding Volume Hierarchy，包围盒层次结构）树实现，以减少射线与场景中所有物体的检查次数。在这种情况下，每个场景物体会被划分成一组包围盒，以方便快速判断射线与物体的相对位置。

### 高亮效果

高亮的核心原理是通过射线算法以屏幕坐标系的鼠标坐标来计算，然后得到相交的 `intersects`组，我们只要获取第一个对象 `selected`，然后将纹理等需要后期复用的数据挂载到 userData 中。这样在取消高亮的时候，我们可以从 `userData` 获取。当鼠标点击其他物体，我们可以提前执行 `disableHighlight` 将所有物体都移除高亮，然后应用 `EdgesGeometry` 用于生成网格线的几何体，并生成 `LineSegments`，应用世界矩阵后物体就附着到了点击的物体上。

```js
import * as THREE from "three";
import useStore from "../store";
const disableHighlight = (MeshList: any[]) => {
  const { scene } = useStore.getState();
  // 取消所有原来被选中的几何体的高亮
  for (let i = 0; i < MeshList.length; i++) {
    const mesh = MeshList[i] as THREE.Mesh;
    if (mesh.userData.isSelected) {
      mesh.material = mesh.userData.originalMaterial;
      mesh.userData.isSelected = false;

      // 移除外轮廓几何体
      scene.remove(mesh.userData.highlightEdge);
    }
  }
};

function clickLight(e: MouseEvent) {
  const { camera, scene, setTargetInfo, transformControls, setEditStatus } =
    useStore.getState();
  setEditStatus("NONE");

  const raycaster = new THREE.Raycaster();
  const mouse = new THREE.Vector2();
  // 更新射线投射器的起始点和方向
  const x = e.clientX;
  const y = e.clientY;

  // 获取屏幕坐标
  mouse.x = (x / window.innerWidth) * 2 - 1;
  mouse.y = -(y / window.innerHeight) * 2 + 1;

  raycaster.setFromCamera(mouse, camera);

  // 计算射线和几何体的交点
  const MeshList = scene.children.filter((item) => item.type === "Mesh");
  if (MeshList.length) {
    const intersects = raycaster.intersectObjects(MeshList);
    console.log("intersects.length", intersects.length);

    // 如果有交点，则对几何体进行高亮处理
    if (intersects.length > 0) {
      // 取消原来的高亮
      disableHighlight(MeshList);

      // 获取被选中的几何体
      const selected = intersects[0].object as THREE.Mesh;

      // 记录原来的材质和缩放值
      selected.userData.originalMaterial = selected.material;
      selected.userData.originalScale = selected.scale.clone();
      // 创建外轮廓几何体
      const edge = new THREE.EdgesGeometry(selected.geometry);
      const edgeMesh = new THREE.LineSegments(edge);
      edgeMesh.position.setFromMatrixPosition(selected.matrixWorld);

      selected.userData.highlightEdge = edgeMesh;

      setTargetInfo(selected);

      // 物体操作
      transformControls.attach(selected);

      // 设置选中标志
      selected.userData.isSelected = true;
      // 在选中的几何体周围添加边框几何体
      scene.add(edgeMesh);
    } else {
      // 取消所有原来被选中的几何体的高亮
      disableHighlight(MeshList);
    }
  }
}

export { clickLight };
```

### 键盘事件

在 3 维软件中，键盘的 W 代表操作轴移动，S 代表操作轴缩放，E 代表操作轴旋转，所以我们需要定义下键盘事件以方便我们编辑：

```js
const onKeydown = (event) => {
  const { transformControls } = useStore.getState();

  switch (event.keyCode) {
    case 69: // E - rotate
      transformControls.setMode("rotate"); // 切换到旋转模式
      break;
    case 83: // S - scale
      transformControls.setMode("scale"); // 切换到缩放模式
      break;
    case 87: // W - translate
      transformControls.setMode("translate"); // 切换到平移模式
      break;
    default:
      break;
  }
};
```

```js
import React from "react";
import { GUI } from "three/examples/jsm/libs/lil-gui.module.min.js";
import useStore from "../store";

function GuiComp() {
  const { setGUI } = useStore.getState();

        .name("z轴坐标")
```

```js
import { GLTFLoader } from "three/examples/jsm/loaders/GLTFLoader.js";
export const onModelImport = () => {
  const { scene } = useStore.getState();
  const fileInput = document.getElementById("download")
  // 当用户选择文件时，触发文件选择事件
```

### 导出模型

既然模型可以编辑，那么对应也需要保存导出不同的模型格式的文件，这样我们可以共享模型给其他人打开和备份。

## 场景设置区：画布的交互实现


在我们编辑场景的时候会需要删除几何体、灯光等，这个时候我们可以清理画布。


清理画布的实现，我们要注意，由于场景其实是有 GridHelper 网格体需要避免删除，我们要筛选出所有的 Mesh 面，通过 scene 的 remove 删除，同时其他的第三方的插件如 transformControls 等我们也需要移除。

此外，**移除对象并不会对该对象的其他引用进行清理**，比如 Three.js 的材质或纹理等引用，因此需要手动释放这些资源以避免内存泄漏。


```js
import useStore from "../../store";
export const clear = () => {
  const { renderer, scene, transformControls } = useStore.getState();

  scene.remove(transformControls);
  // 从场景中移除所有几何体
  const MeshList = scene.children.filter((item) => item.type === "Mesh");

  if (MeshList.length) {
    MeshList.forEach((child) => {
      if (child.type === "Mesh") {
        if (child.userData) {
          scene.remove(child.userData.highlightEdge);
        }
        scene.remove(child);
      }
    });
  }

  // 清理画布
  renderer.clear();

  // 清除颜色缓冲区和深度缓冲区
  renderer.clearColor();
  renderer.clearDepth();
};
```

重置的实现我们只要在清理画布后，重新执行渲染。

```js
import { sceneRender } from "..";
import { clear } from "./clear";
export const reset = () => {
  clear();
  sceneRender();
};
```

  


## 状态区：计算帧率


在状态区，我们会展示帧率、几何体信息，比如多少个几何体，一共有多少个面，选中的几何体网格面细分状态等，下面主要说下如何实现帧率计算。

由于 requestAnimation 函数会得到时间戳，我们只要计算每一帧最新的时间戳和上一次的时间戳的差值，就是一帧的渲染时间，` Math.round(1000 / elapsed)  `获取到帧率，然后通过定时器间隔去计算，这样防止了频繁更新。

```js
import useStore from "../../store";
let fps = 0;
let lastTimestamp = 0;
let frameCount = 0;
let isFirst = true;
let s = 2;

function updateFPS() {
  const { setFrame } = useStore.getState();
  fps = Math.round(frameCount / s);
  setFrame(fps);
  frameCount = 0;
}

export function clacFrame(timestamp: number, newS: number) {
  if (newS) s = newS;
  if (isFirst) {
    setInterval(updateFPS, s * 1000);
    isFirst = false;
  }

  if (lastTimestamp !== 0) {
    const elapsed = timestamp - lastTimestamp;
    fps = Math.round(1000 / elapsed);
    frameCount++;
  }
  lastTimestamp = timestamp;
}
```

>  计算帧率等性能指标以及图形界面我们可以使用开源的 stat.js 库。

接下来我们回顾下每步的实现。

-  **项目搭建**：使用偏向原生的 React 框架，并使用 Vite 方便我们开发环境的体验，通过 Zustand 可以方便我们 Three.js 管理状态。
-  **可视化与编辑区**：
   - 使用 OrbitControls 类实现**网格平面**；
   - **拖拽生成**后通过监听鼠标拖拽的结束位置，射线算法得到最终的 WebGL 世界坐标系的坐标，然后调用 three 的几何体 position 方法移动位置生成；
   - **高亮效果**的实现也是通过**射线算法**，根据是否相交的结果判断是否选中，将结果挂载到 userData 内，当未选中或其他物体的时候根据 userData 的结果移除对应的网格线；
   - 使用 **TransformControls** 类实现物体的操作轴；
   - 为了更方便地操作操作轴，我们需要监听 w、s、e 等**键盘事件**；
   - **GUI** 我们选用 lil-gui 实现，注意 listen 的数据同步更新；
   - **模型导入**核心在于 input 的 file 字段实现文件上传，解析不同格式调用不同的 loader；
   - **模型导出**核心在于先转为 blob、createObjectURL 将 blob 转为 url，最后通过 a 标签下载；
   - **导出效果**核心在于渲染器 preserveDrawingBuffer 属性的配置，然后我们转为 base64 格式通过 a 标签下载即可。
-  **通用交互区**：国际化的实现我们只要从 Zustand 获取最新状态，然后在需要的地方引入组件，在组件内部根据语言的属性值在对应的语言 json 中匹配即可。
-  **场景设置区**：主要处理场景的记录、重置等，注意在重置的时候移除对象并不会自动清理与该对象相关的其他引用，比如材质、纹理等。我们需要手动释放这些资源，以避免内存泄漏。
-  **状态区**：帧率可以通过`fps = Math.round(1000 / elapsed)` 获取，这里需要注意避免频繁更新。
