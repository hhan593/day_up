## 1. 前言

**响应式网页设计（Responsive Web Design，RWD）** 指的是 Web 页面根据不同屏幕宽度等因素，进行布局和外观调整的一系列实践。它不是单独的技术，是描述 Web 设计的一种方式、或者是一组最佳实践的一个词。

随着时代发展，屏幕尺寸的种类越来越多，如今响应式设计已经成为了众多网站的默认做法，本篇我们就带大家快速掌握响应式设计。

## 2. 早期实现

“响应式设计”这个词是 [Ethan Marcotte 在 2010 年首度提出的](https://alistapart.com/article/responsive-web-design/)，他将其描述为三种技术的混合使用：

1.  流动布局：简单来说，就是百分比布局，比如将页面分为 30%宽度的侧边栏、10% 的分隔区域、60% 的内容栏，元素宽度便会随着浏览器宽度自动进行调整。[点击查看示例 Demo](https://alistapart.github.io/code-samples/fluidgrids/examples/grid/final.html)。
2.  流体图片：为图片设置 `max-width:100%`，这样当包含图片的元素宽度小于图片原始宽度时，图片会自动缩放。
3.  媒体查询：根据设备特征和参数是否存在以及对应的值来应用 CSS 样式，通常是根据视口大小来应用媒体查询，比如 `@media screen and (min-width: 600px) { ... }`。

这是一个[综合使用流动布局和媒体查询的示例](https://mdn.github.io/css-examples/learn/rwd/float-based-rwd.html)，[点击查看源码](https://github.com/mdn/css-examples/blob/main/learn/rwd/float-based-rwd.html)，浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e74fee23a204d829eef67e4474089ea~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=951\&h=526\&s=2985910\&e=gif\&f=85\&b=f6f6f6)

最核心的代码是这段：

```css
@media screen and (min-width: 600px) {
  .col1 {
    width: 31.24999999%;
    float: left;
  }

  .col2 {
    width: 64.58333331%;
    float: right;
  }
}
```

页面内容分为 2 列，这是因为设置了媒体查询，当屏幕宽度大于等于 600 时，使用浮动并设置百分比宽度，由此实现了左右两列布局。当屏幕宽度小于 600 时，回归最原始的样式，分为上下两部分。

这就是早期的响应式布局实现。

## 3. 现代实现

### 3.1. 新型布局方式

随着 CSS 的发展，现在已经提供了多种实现响应式布局的方式，常见的比如弹性布局（Flexbox）、网格布局（Grid）、多列布局（Multi-column）。

#### 3.1.1. 多列布局

多列布局实现的是类似于报纸上的效果。通过指定父元素的 `column-count`属性，将内容分成几列，浏览器会随着屏幕尺寸变化自动计算这些列的大小。

#### 3.1.3. 弹性布局

弹性布局就是我们常用的 Flexbox，我们来写一个效果：

```xml
<div class="wrapper">
    <div class="col1">
      <p>
        This layout is responsive. See what happens if you make the browser
        window wider or narrow.
      </p>
    </div>
    <div class="col2">
      <p>
        One November night in the year 1782, so the story runs, two brothers sat
        over their winter fire in the little French town of Annonay, watching
        the grey smoke-wreaths from the hearth curl up the wide chimney. Their
        names were Stephen and Joseph Montgolfier, they were papermakers by
        trade, and were noted as possessing thoughtful minds and a deep interest
        in all scientific knowledge and new discovery.
      </p>
      <p>
        Before that night—a memorable night, as it was to prove—hundreds of
        millions of people had watched the rising smoke-wreaths of their fires
        without drawing any special inspiration from the fact.
      </p>
    </div>
</div>

<style>
  body {
    margin: 20px;
    padding: 0;
    background-color: #eee;
  }
  .wrapper {
    max-width: 960px;
    margin: 2em auto;
  }

  .col1,
  .col2 {
    background-color: #fff;
  }

  @media screen and (min-width: 600px) {
    .wrapper {
      display: flex;
    }

    .col1 {
      flex: 1;
      margin-right: 5%;
    }

    .col2 {
      flex: 2;
    }
  }
</style>
```

### 3.2. 响应式图片

早期实现中的流体图片，虽然能够正确缩放，但有两个问题：

1.  移动端和 PC 端要下载同样尺寸的图片，浪费带宽
2.  移动端尺寸更小，有的时候会希望展示一张更凸显主体或者完全不同的图像

这两个问题不能简单的通过缩放图像来解决。当然现在，HTML5 中 `<picture>`和 `<img>`的 `srcset` 和 `sizes` 属性解决了这两个问题。

在现代框架中，基本都提供了类似 `<Image>` 这样的图片组件专门处理图像，开发的时候只用使用一张图片，框架会自动构建该图片不同尺寸的版本，并通过设置 `srcset`和 `sizes`属性从而保证在不同屏幕尺寸下加载不同大小的图片。

关于响应式图片，可以参考：

1.  [响应式图片 - 学习 Web 开发 | MDN](https://developer.mozilla.org/zh-CN/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images)
2.  [《Svelte 小册 SvelteKit | Images 中关于 srcset 和 sizes 的介绍》](https://juejin.cn/book/7407681878888562723/section/7408068654681554978#heading-6)

### 3.3. 响应式排版

响应式排版指的是根据屏幕真实使用范围，在媒体查询时改变字体大小。比如在 PC 端屏幕空间很多，标题可以很大，但到了移动端，屏幕空间小，则更适合将标题调小。

常用的单位一个是 `rem`：

```css
html {
  font-size: 1em;
}

h1 {
  font-size: 2rem;
}

@media screen and (min-width: 600px) {
  h1 {
    font-size: 4rem;
  }
}
```

[点击查看 Demo](https://mdn.github.io/css-examples/learn/rwd/type-vw.html)，浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0c7adec38564455b8ace6ee5265dcfd2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1105\&h=663\&s=1375823\&e=gif\&f=35\&b=f8f8f8)

在这个例子中，标题正常是 `2rem` 大小，过了断点 600px 后，标题变为 `4rem` 大小。

另一个常用单位是 `vw`，因为 `1vw`表示视口宽度的百分之一，所以如果使用 `vw`，字体的大小将总是随视口的大小进行改变：

```css
h1 {
  font-size: calc(1.5rem + 3vw);
}
```

[点击查看 Demo](https://mdn.github.io/css-examples/learn/rwd/type-vw.html)，浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3557ede9b42d44ccba303688512bf758~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1327\&h=596\&s=3656537\&e=gif\&f=68\&b=f8f8f8)

可以看到随着视口的变化，标题的 font-size 一直在变化。

> 注意：虽然学习响应式设计的时候，经常会看到有文章在讲 rem 布局、vw 布局之类的内容，但这并不能算是响应式设计，一般我们会用自适应布局来称呼。
>
> 在自适应布局中，元素会统一用 rem 或者 vw 单位，这样实现的效果是页面可以在不同尺寸的设备屏幕上按比例缩放，常用于适配移动端的各种设备。使用自适应布局会有一个问题，比如手机淘宝页面，一排有 2 个商品，到了更大的手机上，还是一排 2 个商品，只是商品和文字的图片都更大了。对于大部分的手机并没有什么问题，但如果到了比如 ipad、 PC 端，其实应该一排展示更多的商品，而非将以前的页面等比缩放。
>
> 所以响应式和自适应一个很大的不同就在于多列的处理方式不同。

最后总结一下，随着响应式设计的发展，早期的百分比+浮动布局转变为现在的多列布局、网格布局、弹性布局，结合媒体查询，实现了不同尺寸下不同的布局效果。流体图片新增了 srcset 和 sizes 属性，可以根据不同的屏幕尺寸加载不同版本的图片。此外，又多了一个响应式排版的概念，主要是调整字体的大小、边距等元素让排版更加舒适。

## 4. Tailwind CSS

Tailwind CSS 下该如何实现响应式呢？

### 4.1. 断点

Tailwind CSS 默认[提供了 5 个断点](https://tailwindcss.com/docs/responsive-design)：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/855afda846fe400ea7ba5575e7b83197~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2228\&h=734\&s=168300\&e=png\&b=101729)

比如一张图片：

```html
<img class="w-16 md:w-32 lg:w-48" src="...">
```

这句代码的意思是图片正常宽度是 `w-16`，在屏幕宽度 >= 768 的时候为 `w-32`，在屏幕宽度大于 >= 1024 的时候为 `w-48`。

> 注意：断点是基于 min-width 的，这也就是说背后遵循的 Mobile First 的设计原则，即默认设计（不使用断点时）的是小屏的样式，然后在此基础上增加断点时的样式。本篇的例子都是遵循这样的原则。

你可以在 `tailwind.config.js`中自定义断点逻辑：

```javascript
/** @type {import('tailwindcss').Config} */
module.exports = {
  theme: {
    screens: {
      'tablet': '640px',
      // => @media (min-width: 640px) { ... }

      'laptop': '1024px',
      // => @media (min-width: 1024px) { ... }

      'desktop': '1280px',
      // => @media (min-width: 1280px) { ... }
    },
  }
}
```
