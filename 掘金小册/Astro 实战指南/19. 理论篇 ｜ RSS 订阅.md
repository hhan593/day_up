## 1. 前言

本篇我们讲解 Astro 博客如何进行 RSS 订阅。

## 2. RSS 介绍

RSS（英文全称：Really Simple Syndication，中文译为：简易信息聚合）是一种基于 XML 的标准、在互联网上被广泛采用的内容包装和投递协议。

简单来说，它是一种描述和同步网站内容的格式，网站按照这个规范提供包含文章标题、摘要、链接、内容等信息，用户在客户端可以使用 RSS 软件，订阅不同网站的 RSS 链接，从而将不同的信息源进行聚合，甚至可以在不打开网站内容页面的情况下阅读网站内容。

相对于各大 APP 的“猜你喜欢”，RSS 是一种主动获取信息的方法，让你自己决定看什么，所以虽然小众，但备受一部分用户的喜爱。

RSS 常用于订阅新闻资讯、博客文章等等。所以做个人博客的时候非常适合顺便做一个 RSS 订阅。

那在 Astro 中该如何实现 RSS 订阅呢？

## 3. RSS 配置

Astro 提供了 RSS 处理包 [@astrojs/rss](https://github.com/withastro/astro/tree/main/packages/astro-rss)，安装此依赖项：

```bash
npm install @astrojs/rss
```

新建 `src/pages/rss.xml.js`，代码如下：

```javascript
import rss from "@astrojs/rss";

export function GET(context) {
  return rss({
    title: "我的技术博客",
    description: "我的技术博客描述",
    site: context.site,
    items: [],
  });
}
```

关于在于 items 的生成，它是 RSS 摘要对象列表，RSS 订阅软件正是根据这个数据展示、管理内容以及通知用户等。

根据 RSS 的规范，每个发布条目应该有这些元数据：

1.  `title`: 条目的标题。只有在设置了 description 时才是可选的，否则是必需的。
2.  `description`: 条目的简短摘录或描述。只有在设置了 title 时才是可选的，否则是必需的。
3.  `link`: 指向条目原始来源的 URL。（可选）
4.  `pubDate`: 条目发布日期。（可选）
5.  `content`: 你的帖子的全部内容。（可选）

更多的字段参考 [@astrojs/rss 中的 items 配置项](https://github.com/withastro/astro/tree/main/packages/astro-rss#items)。

### 3.1. 自动生成 items

items 如果手动添加就太累了，最好是自动生成，我们可以使用内容集合遍历生成 items 数据：

```javascript
import rss from "@astrojs/rss";
import { getCollection } from "astro:content";

export async function GET(context) {
  const posts = await getCollection("posts");
  return rss({
    title: "我的技术博客",
    description: "我的技术博客描述",
    site: context.site,
    items: posts.map((post) => ({
      title: post.data.title,
      description: post.data.description,
      link: `/post/${post.slug}/`,
      pubDate: dayjs(published, "YYYY-MM-DD HH:mm:ss", "zh-cn").toDate(),
    })),
  });
}
```

为了保证每篇文章都写了有效的 RSS 数据，可以导入使用 rssSchema，修改 `blog/src/content/config.ts`，代码如下：

```javascript
import { z, defineCollection } from "astro:content";
import { rssSchema } from "@astrojs/rss";

const postsCollection = defineCollection({
  type: "content",
  schema: rssSchema.extend({
    columns: z.array(z.string()).default(["uncategory"]),
    published: z.string().refine((arg) => arg.match(/^(\d{4})-(\d{2})-(\d{2}) (\d{2}):(\d{2}):(\d{2}(?:\.\d*)?)$/)),
  }),
});

export const collections = {
  posts: postsCollection,
};
```

此外，如果我们已经写了一些自定义的 schema 字段，那可以使用 `rssSchema.extend`将这些自定义的 schema 字段加入其中。

### 3.2. 添加完整文章内容

因为内容集合并没有返回完整编译好的内容，所以如果要添加完整内容，需要安装 Markdown 解析器：

```bash
npm i markdown-it sanitize-html
```

修改 `src/pages/rss.xml.js`，代码如下：

```javascript
import rss from "@astrojs/rss";
import { getCollection } from "astro:content";
import dayjs from "dayjs";
import customParseFormat from "dayjs/plugin/customParseFormat";
import sanitizeHtml from "sanitize-html";
import MarkdownIt from "markdown-it";
const parser = new MarkdownIt();

dayjs.extend(customParseFormat);
export async function GET(context) {
  const posts = await getCollection("posts");
  return rss({
    title: "我的技术博客",
    description: "我的技术博客描述",
    site: context.site,
    trailingSlash: false,
    items: posts
      .sort((a, b) => Date.parse(b.data.published) - Date.parse(a.data.published))
      .map(({ data: { published, title, description }, slug, body }) => ({
        title,
        ...(description && { description }),
        link: `/post/${slug}/`,
        pubDate: dayjs(`${published}+08:00`, "YYYY-MM-DD HH:mm:ssZ").toDate(),
        // 注意：这不会处理 MDX 文件中的组件或 JSX 表达式。
        content: sanitizeHtml(parser.render(body), {
          allowedTags: sanitizeHtml.defaults.allowedTags.concat(["img"]),
        }),
      })),
  });
}
```

注意：因为这里我们改为使用 Markdown-it 解析 Markdown 文件，之前的那些插件和 MDX 都不会生效，所以 Markdown 渲染效果会有差别。Astro 官网给出的就是这个方案，目前没有看到更好的解决方案。

### 3.3. 添加自动发现

[RSS 自动发现](RSS%20自动发现)允许浏览器和其他软件自动从主 URL 找到网站的 RSS 摘要。在 Layout 中添加代码：

```html
<link rel="alternate" type="application/rss+xml" title="你的网站标题" href={new URL("rss.xml", Astro.site)} />
```

### 3.4. 部署并配置 site

现在我们将代码部署到服务器或者三方托管平台上，注意要为 `astro.config.mjs`添加 site 配置项：

```javascript
export default defineConfig({
  site: "https://blog.vercel.app/",
});
```

因为 `src/pages/rss.xml.js`中，我们的 `site`配置的是`context.site`，这个 context 就来自于我们的 config，@astrojs/rss 会拼接出每篇文章的最终 URL。

本地运行 `npm run build`，构建完成后就可以访问 <http://localhost:4321/rss.xml> 查看构建的 `rss.xml`：
