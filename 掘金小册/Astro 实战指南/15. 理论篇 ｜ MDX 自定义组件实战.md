### 3.1. Astro 组件与自定义元素

Astro 推荐使用[自定义元素](https://developer.mozilla.org/zh-CN/docs/Web/API/Web_components/Using_custom_elements)的方式构建交互式组件。

所谓自定义元素，出自于 Web Components，比如这就是一个独立自定义元素的最小实现：

```javascript
class PopupInfo extends HTMLElement {
  constructor() {
    super();
  }
  // 此处编写元素功能
}

// 注册自定义元素
customElements.define("popup-info", PopupInfo);
```

此时我们就可以在 HTML 中使用该自定义元素：

```html
<popup-info>
  <!-- 元素的内容 -->
</popup-info>
```

那如何在 Astro 中使用自定义元素构建组件呢？我们以[官网的例子](https://docs.astro.build/zh-cn/guides/client-side-scripts/#%E5%85%B7%E6%9C%89%E8%87%AA%E5%AE%9A%E4%B9%89%E5%85%83%E7%B4%A0%E7%9A%84-web-%E7%BB%84%E4%BB%B6)为例：

```jsx
<!-- 将按钮包装在我们的自定义元素“astro-heart”中。 -->
<astro-heart> <button aria-label="Heart">💜</button> × <span>0</span> </astro-heart>

<script>
  class AstroHeart extends HTMLElement {
    connectedCallback() {
      let count = 0;

      const heartButton = this.querySelector("button");
      const countSpan = this.querySelector("span");

      // 每次单击按钮时，更新计数。
      heartButton.addEventListener("click", () => {
        count++;
        countSpan.textContent = count.toString();
      });
    }
  }

  // 注册自定义元素
  customElements.define("astro-heart", AstroHeart);
</script>
```

在这个例子中，我们定义了一个 `<astro-heart>` 自定义元素，用于记录喜欢按钮的点击次数。

在这里使用自定义元素有两个优点：

1.  可以使用 `this.querySelector()` ，它只在当前自定义元素实例中搜索。这使得一次只处理一个组件实例的子实例变得更容易。
2.  虽然 `<script>` 只运行一次，但浏览器每次在页面上找到 `<astro-heart>` 时都会运行我们自定义元素的 `constructor()` 方法。这意味着你可以安全地一次为一个组件编写代码，即使你打算在一个页面上多次使用该组件。

简单来说就是：如果组件在一个页面中多次使用，使用自定义元素，处理会变得更加简单。

### 3.2. Astro.slots.render()

[Astro.slots.render()](https://docs.astro.build/zh-cn/reference/api-reference/#astroslotsrender) 是 Astro 的运行时 API，用于将插槽的内容异步渲染为 HTML 字符串。

通常我们获取组件内容只用使用 `<slot />`插槽即可，但有的时候需要以编程的方式用到（比如我们现在这个例子），使用方式如下：

```jsx
---
const html = await Astro.slots.render('default');
---
<Fragment set:html={html} />
```

### 3.3. Rehype 插件

我们在《理论篇 | 如何写 remark 插件？》和《理论篇 | 如何写 rehype 插件？》介绍过相关体系。这次我们单独讲讲 [rehype](https://github.com/rehypejs/rehype/tree/main/packages/rehype)。

rehype 是一个 unified 处理器，专门用于处理 HTML，如果你以 HTML 作为输入并希望 HTML 作为输出时，就适合使用 rehype。

举个例子：

```javascript
import { rehype } from "rehype";
import rehypeFormat from "rehype-format";

const file = await rehype().use(rehypeFormat).process(`<!doctype html>
        <html lang=en>
<head>
    <title>Hi!</title>
  </head>
  <body>
    <h1>Hello!</h1>

</body></html>`);

console.error(String(file));
```

这段代码也可以拆分为：

```javascript
import { rehype } from "rehype";
import rehypeFormat from "rehype-format";

const formatProcessor = rehype().use(rehypeFormat);

const file = formatProcessor.processSync(`<!doctype html>
        <html lang=en>
<head>
    <title>Hi!</title>
  </head>
  <body>
    <h1>Hello!</h1>

</body></html>`);

console.error(String(file));
```

### 3.4. 实战开发

基础知识已经铺垫好了，我们来实现下 Tabs 组件。

我们想要实现的效果是这样的：

```jsx
import Tabs, { TabItem } from "@/components/Tabs.astro";

<Tabs>
  <TabItem label="选项卡 1">内容 1</TabItem>
  <TabItem label="选项卡 2">内容 2</TabItem>
</Tabs>;
```

对应 HTML 输出为：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/999072807f294e968fbe53b53d061741~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1432\&h=352\&s=122319\&e=png\&b=282828)

每个 `<TabItem>` 组件渲染为 `<div>`是很容易实现的，但是 `<Tabs>` 组件需要知道有几个 `<TabItem>`组件，并获取 `label` 内容，才能正确渲染出 `<ul>`。

`<Tabs>` 组件如何知道有几个 `<TabItem>` 组件呢？这个时候就需要写 rehype 插件，也就是先获取子组件正常渲染的 HTML，然后对 HTML 做解析，最后输出新的 HTML。

新建 `src/components/TabItem.astro`，完整代码如下：

```jsx
---
const { label } = Astro.props;

if (!label) {
throw new Error('Missing prop `label` on `<TabItem>` component.');
}
---

<custom-tab-item data-label={label} >
  <slot /> 
</custom-tab-item>
```

在这个例子中，我们将 `<TabItem>`渲染为 `<custom-tab-item>`元素，并将 `label` 写入到 `data-label `属性中。之所以渲染为 `<custom-tab-item>`是为了方便在 rehype 插件中通过 tagName 判断出这是一个 TabItem，理论上这里的名字可以随便乱起，反正在 rehype 插件中最终会转为 `<div>`。

新建 `src/components/Tabs.astro`，代码如下：

```jsx
---
import type { Element } from 'hast';
import { rehype } from 'rehype';
import { CONTINUE, SKIP, visit } from 'unist-util-visit';
import TabItem from './TabItem.astro';
export {
  TabItem
}

interface Panel {
  panelId: string;
  tabId: string;
  label: string;
}

let count = 0;
const getIDs = () => {
  const id = count++;
  return { panelId: 'tab-panel-' + id, tabId: 'tab-' + id };
};

const tabsProcessor = rehype()
  .data('settings', { fragment: true })
  .use(function tabs() {
    return (tree: Element, file) => {
      file.data.panels = [];
      let isFirst = true;
      visit(tree, 'element', (node) => {
        if (node.tagName !== "custom-tab-item" || !node.properties) {
          return CONTINUE;
        }

        const { dataLabel } = node.properties;
        const ids = getIDs();
        const panel = {
          ...ids,
          label: String(dataLabel),
        };
        (file.data.panels as Panel[])?.push(panel);

        node.tagName = 'div';
        node.properties.id = ids.panelId;
        node.properties['aria-labelledby'] = ids.tabId;
        node.properties.role = 'tabpanel';

        if (isFirst) {
          isFirst = false;
        } else {
          node.properties.hidden = true;
        }

        return SKIP;

      });
    }
  });

function processPanels(html: string) {
  const file = tabsProcessor.processSync({ value: html });
  return {
    panels: file.data.panels as Panel[],
    html: file.toString(),
  };
}

const panelHtml = await Astro.slots.render('default');
const { html, panels } = processPanels(panelHtml);

---

<custom-tabs>
  <div class="tablist-container">
    {
      <ul role="tablist">
        {panels.map(({ label, panelId, tabId }, idx) => (
          <li role="presentation" class="tab">
            <a
              role="tab"
              href={'#' + panelId}
              id={tabId}
              aria-selected={idx === 0 ? 'true' : 'false'}
              tabindex={idx !== 0 ? -1 : 0}
            >
              {label}
            </a>
          </li>
        ))}
      </ul>
    }
    <Fragment set:html={html} />
  </div>
</custom-tabs>

<style>
  .tablist-container {
    --color-gray-5: hsl(224, 6%, 77%);
    --color-gray-3: hsl(224, 7%, 36%);
    --outline-offset-inside:-.1875rem;
    --color-white: hsl(224, 10%, 10%);
    --color-text-accent:hsl(234, 90%, 60%);
  }
  .tablist-container {
    display: block;
    overflow-x: auto;
  }

  [role='tablist'] {
    display: flex;
    list-style: none;
    border-bottom: 2px solid var(--color-gray-5);
    padding: 0;
  }

  .tab {
    margin-bottom: -2px;
  }
  .tab > [role='tab'] {
    display: flex;
    align-items: center;
    gap: 0.5rem;
    padding: 0 1.25rem;
    text-decoration: none;
    border-bottom: 2px solid var(--color-gray-5);
    color: var(--color-gray-3);
    outline-offset: var(--outline-offset-inside);
    overflow-wrap: initial;
  }
  .tab [role='tab'][aria-selected='true'] {
    color: var(--color-white);
    border-color: var(--color-text-accent);
    font-weight: 600;
  }

  .tablist-wrapper ~ :global([role='tabpanel']) {
    margin-top: 1rem;
  }
</style>

<script>
class Tabs extends HTMLElement {

    tabs: HTMLAnchorElement[];
    panels: HTMLElement[];

    constructor() {
      super();
      const tablist = this.querySelector<HTMLUListElement>('[role="tablist"]')!;
      this.tabs = [...tablist.querySelectorAll<HTMLAnchorElement>('[role="tab"]')];
      this.panels = [...this.querySelectorAll<HTMLElement>('[role="tabpanel"]')];
      this.tabs.forEach((tab, i) => {
        tab.addEventListener('click', (e) => {
          e.preventDefault();
          const currentTab = tablist.querySelector('[aria-selected="true"]');
          if (e.currentTarget !== currentTab) {
            this.switchTab(e.currentTarget as HTMLAnchorElement, i);
          }
        });
      });
    }

    switchTab(newTab: HTMLAnchorElement | null | undefined, index: number, shouldSync = true) {
      if (!newTab) return;

      this.tabs.forEach((tab) => {
        tab.setAttribute('aria-selected', 'false');
        tab.setAttribute('tabindex', '-1');
      });

      this.panels.forEach((oldPanel) => {
        oldPanel.hidden = true;
      });

      const newPanel = this.panels[index];
      if (newPanel) newPanel.hidden = false;

      newTab.removeAttribute('tabindex');
      newTab.setAttribute('aria-selected', 'true');
    }
  }

  customElements.define('custom-tabs', Tabs);
</script>
```

这里就是核心代码了，代码主要分为 2 部分，一部分是 rehype 插件处理，用于渲染出最终的 HTML 结构，一部分是 Astro 组件开发，用于实现交互效果。

首先，这段代码使用 `await Astro.slots.render('default')`，将 `<Tabs>` 组件的插槽内容异步渲染为 HTML 字符串。此时返回的结果为：

```html
<custom-tab-item data-label="选项卡 1">
  <p>内容 1</p>
</custom-tab-item>
<custom-tab-item data-label="选项卡 2">
  <p>内容 2</p>
</custom-tab-item>
```

然后我们将返回的结果传入 `processPanels` 函数中，`processPanels` 就是一个 rehype 插件。在该函数中，我们通过 unist-util-visit 这个库（这个库我们在 《理论篇 | 如何写 rehype 插件？》中使用过，简单来说，它提供了一个便捷的访问节点的方式）遍历 tree，找到其中 tagName 名为 `custom-tab-item` 的节点，然后修改节点数据，将其最终转为 `<div>`，并将需要的数据放入到 `file.data.panels` 中，如此我们就知道了 panels 的数量和相关数据。

接下来就是正常渲染 HTML：

```html
<div class="tablist-container">
  {
  <ul role="tablist">
    {panels.map(({ label, panelId, tabId }, idx) => (
    <li role="presentation" class="tab">// ...</li>
    ))}
  </ul>
  }
  <Fragment set:html="{html}" />
</div>
```

如此我们获得了最终的 HTML。

最后是 `<script>` 中的脚本，我们通过自定义 HTML 元素的方式开发组件，实现代码并不复杂，点击的时候修改类名和属性，以此实现样式的修改。
