## 1. 前言

如果感觉做出的页面有些慢，那就来看看本篇介绍的这些优化方法。

## 2. 渲染模式

查看 `astro.config.mjs`的 `output` 配置项：

```javascript
import { defineConfig } from "astro/config";

export default defineConfig({
  output: "static" | "server" | "hybrid",
});
```

默认的渲染模式是 `output: 'static'`，它会在构建时将所有页面渲染为 HTML。

在这种模式下，你的整个站点将被预渲染，服务器将提前构建所有页面，并准备好发送给浏览器。对于每个访问者，都会发送相同的 HTML 文档，而更新页面内容需要全站重新构建。这种方法也被称为静态站点生成（SSG）。

默认情况下，所有 Astro 项目都被配置为在构建时预渲染（静态生成），以提供最轻量级的浏览器体验。浏览器无需等待任何 HTML 构建，因为服务器无需按需生成任何页面。你的网站不依赖后端数据源的性能，一旦构建完成，只要你的服务器正常运行，它就会作为静态站点一直对访问者开放。

其他两种渲染模式则会开启服务端渲染：

1. `output: 'server'`：默认按需渲染。当你的网站或应用的大部分或全部应该在请求时进行服务器渲染时使用这个选项。任何单独的页面或端点都可以选择加入预渲染。
2. `output: 'hybrid'`：默认预渲染为 HTML。在你的网站大部分应该是静态的时候使用这个选项。任何单独的页面或端点都可以选择退出预渲染。

所以此时的 output 选项代表了你的大部分页面和路由将如何被渲染。

当你选择了 `output: 'server'`时，能预渲染的页面应该尽可能设置为预渲染模式：

```jsx
---
export const prerender = true;
// ...
---

<html>
  <!-- 这里是静态的、预渲染的页面... -->
</html>
```

## 3. 预获取

预获取对于性能的提升也很明显。使用 prefetch 配置来启用预获取：

```javascript
import { defineConfig } from "astro/config";

export default defineConfig({
  prefetch: true,
});
```

此时一个预获取脚本将被添加到你网站的所有页面。

然后你可以在你网站的任何 `<a />` 链接上添加 data-astro-prefetch 属性来选择预获取：

```html
<a href="/about" data-astro-prefetch></a>
```

当你将鼠标悬停在链接上时，脚本将在后台获取页面。

但是这样做太麻烦了，还要一个一个加属性，你也可以直接预获取所有链接：

```javascript
import { defineConfig } from "astro/config";

export default defineConfig({
  prefetch: {
    prefetchAll: true,
  },
});
```

然后你可以通过设置 `data-astro-prefetch="false"` 来为单个链接选择不进行预获取：

```javascript
<a href="/about" data-astro-prefetch="false">
  About
</a>
```

此外预获取又分为 4 种策略：

1. `hover`（默认）：当你将鼠标悬停在链接上或聚焦在链接上时预获取。
2. `tap`：在你点击链接之前预获取。
3. `viewport`：当链接进入视口时预获取。
4. `load`：当页面加载后预获取当前页面的所有链接。

默认预获取策略是 hover，可以通过 `prefetch.defaultStrategy` 配置项修改：

```javascript
import { defineConfig } from "astro/config";

export default defineConfig({
  prefetch: {
    defaultStrategy: "viewport",
  },
});
```

也可以通过属性在单个链接上修改：

```html
<a href="/about" data-astro-prefetch="tap">
  About
</a>
```

## 4. 流式渲染

### 4.1. 问题描述

当使用服务端渲染的时候，要避免数据获取造成的阻塞。举个例子：

```jsx
---
function sleep(time: any) {
  return new Promise((resolve) => setTimeout(resolve, time));
}

async function fetchColumns() {
  await sleep(3000);
  return new Response(JSON.stringify(["专辑 1", "专辑 2", "专辑 3"]));
}

async function fetchSingers() {
  await sleep(5000);
  return new Response(JSON.stringify(["歌手 1", "歌手 2", "歌手 3"]));
}

const columnsResponse = await fetchColumns();
const columnsData = await columnsResponse.json();
const signersResponse = await fetchSingers();
const signersData = await signersResponse.json();
---

<html>
  <head>
    <meta charset="UTF-8" />
    <title>A name and a fact</title>
  </head>
  <body>
    <h2>热门专辑</h2>
    <p>{columnsData.join(" | ")}</p>
    <h2>热门歌手</h2>
    <p>{signersData.join(" | ")}</p>
  </body>
</html>
```

浏览器效果如下：

### 4.2. 解决方案一：拆分组件

为了防止串行，我们将请求拆分到组件中。

新建 `src/components/Columns.astro`，代码如下：

```jsx
---
function sleep(time: any) {
  return new Promise((resolve) => setTimeout(resolve, time));
}

async function fetchColumns() {
  await sleep(3000);
  return new Response(JSON.stringify(["专辑 1", "专辑 2", "专辑 3"]));
}

const columnsResponse = await fetchColumns();
const columnsData = await columnsResponse.json();
---

<p>{columnsData.join(" | ")}</p>
```

新建 `src/components/Singers.astro`，代码如下：

```jsx
---
function sleep(time: any) {
  return new Promise((resolve) => setTimeout(resolve, time));
}

async function fetchSingers() {
  await sleep(5000);
  return new Response(JSON.stringify(["歌手 1", "歌手 2", "歌手 3"]));
}

const signersResponse = await fetchSingers();
const signersData = await signersResponse.json();
---

<p>{signersData.join(" | ")}</p>
```

页面代码如下：

```jsx
---
import Columns from "../../components/Columns.astro";
import Singers from "../../components/Singers.astro";
---

<html>
  <head>
    <meta charset="UTF-8" />
    <title>A name and a fact</title>
  </head>
  <body>
    <h2>热门专辑</h2>
    <Columns />
    <h2>热门歌手</h2>
    <Singers />
  </body>
</html>
```

浏览器效果如下：

![astro-2.gif](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2eeef08e01ad4758a019f19b5c27f5a0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=858&h=309&s=129401&e=gif&f=23&b=f5f3ff)

当我们输入网址的时候，页面会立刻渲染出来，服务器并行获取 Columns 和 Singers 的数据，然后将生成的 HTML 流式传输到浏览器，所以最后的加载时间为 5s：


![image (1).png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d67e3d96daeb4bcf94a88f3224d08958~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2056&h=484&s=164437&e=png&b=2d2d2d)

不过这种方式请求会阻塞其后的标记，所以页面加载时，热门专辑标题会立刻展现，等待 3s 后，专辑信息和热门歌手一起展现，再等待 2s 后，歌手信息展现。

### 4.3. 解决方案二：直接使用 Promise

此外，你也可以不使用拆分组件的形式，而是直接使用 Promise：

```jsx
---
function sleep(time: any) {
  return new Promise((resolve) => setTimeout(resolve, time));
}

async function fetchColumns() {
  await sleep(3000);
  return new Response(JSON.stringify(["专辑 1", "专辑 2", "专辑 3"]));
}

async function fetchSingers() {
  await sleep(5000);
  return new Response(JSON.stringify(["歌手 1", "歌手 2", "歌手 3"]));
}

const columnsPromise = fetchColumns().then((response) => response.json());
const signersPromise = fetchSingers().then((response) => response.json());
---

<html>
  <head>
    <meta charset="UTF-8" />
    <title>音乐列表</title>
  </head>
  <body>
    <h2>热门专辑</h2>
    <p>{columnsPromise}</p>
    <h2>热门歌手</h2>
    <p>{signersPromise}</p>
  </body>
</html>
```

使用这种方式，浏览器效果同拆分组件。

### 4.4. 解决方案三：使用 Suspense

但我们真正想要的效果是类似于 React 的 Suspnse 效果。请求同时发出，但请求并不会阻塞之后的标记内容。

正好有一个 [Simple stream](https://simple-stack.dev/stream) 集成可以实现这个功能：

```bash
npx astro add simple-stack-stream
```

在解决方案一的基础上，修改页面代码：

```jsx
---
import { Suspense } from "simple-stack-stream/components";
import Columns from "../../components/Columns.astro";
import Singers from "../../components/Singers.astro";
---

<html>
  <head>
    <meta charset="UTF-8" />
    <title>A name and a fact</title>
  </head>
  <body>
    <h2>热门专辑</h2>
    <Suspense>
      <Columns />
      <p slot="fallback">Loading...</p>
    </Suspense>
    <h2>热门歌手</h2>
    <Suspense>
      <Singers />
      <p slot="fallback">Loading...</p>
    </Suspense>
  </body>
</html>
```

此时浏览器效果如下：


![astro-3.gif](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34614c4453ac4d0aa6308bd99f3c0470~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=858&h=309&s=207378&e=gif&f=22&b=f5f3ff)

当我们输入网址的时候，页面会立刻渲染出来，服务器并行获取 Columns 和 Singers 的数据，然后将生成的 HTML 流式传输到浏览器，最后的加载时间依然是 5s，但请求不会阻塞之后的标记内容。算是目前比较好的实现方案了。

## 5. 图片优化

### 5.1. 图片占比

根据 [Web Almanac](https://almanac.httparchive.org/en/2022/media) 中的介绍，图片大小占典型网站页面大小的很大一部分。根据统计，2021 年 6 月网站的总大小中位数是 2019 KB（移动端），其中 881 KB 是图像。这比 HTML（30 KB），CSS（72 KB），JavaScript（461 KB）和字体（97 KB）的总和还要多。

在绝大多数页面上（70% 移动设备，80% 桌面），最有影响的就是图片。Largest Contentful Paint（最大内容绘制，简写：[LCP](https://web.dev/articles/lcp?hl=zh-cn)） 是一种 Web 性能指标，可以标识首屏中最大的内容元素。大部分时候，该元素都有图片。

### 5.2. LCP 背景

考虑到 LCP 并不算是一个常为大家熟知的概念，所以我们单独介绍下 LCP。

对于 Web 开发者而言，衡量网页主要内容的加载速度一直是一个挑战。

传统我们会使用 load、DOMContentLoaded 等方法，但它们并不表示用户在屏幕上看到的内容的时间。

而像首次内容渲染（FCP），如果页面有 loading 效果，那获取的时间也是不准确的。

当然也有首次有效绘制（FMP）等指标，但是这些指标非常复杂，往往是错误的。所以也不能用来确定主要内容的加载时间。

根据 W3C Web 性能工作组中的讨论和 Google 的研究，要衡量网页主要内容的加载时间，更为准确的方法是查看最大元素的呈现时间。这就是 LCP。

### 5.3. LCP 概念与标准

Largest Contentful Paint (LCP) 指标会报告视口内可见的最大图片或文本块的呈现时间（相对于网页首次开始加载的时间）。

为了提供良好的用户体验，网站应尽力将 Largest Contentful Paint 设置为 **2.5 秒**或更短。

让我们看一些 LCP 的例子：

![d45d5ac94a014e66a4b791e61e2a41dc~tplv-k3u1fbpfcp-jj-mark_0_0_0_0_q75.jpg](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8870937e3294e97b2775ea9dc64229f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=621&s=410256&e=png&b=fbfbfb)

那么问题来了，页面往往是分阶段加载的，网页中最大元素可能是在不断变化的，LCP 是怎么计算出来的呢？

首先，浏览器会将 LCP 的元素限定在一些特定的元素类型内，比如`<img>` 元素、包含文本节点或其他内嵌级别文本元素的子项的块级元素、为自动播放 `<video>` 元素而绘制的第一帧、动画图片格式（例如 GIF 动画）的第一帧等等（这是为了简化这个问题，如果什么元素都计算一遍大小，就太复杂了而且没必要）。

然后浏览器在绘制完第一帧后，就会立即分派 largest-contentful-paint 类型的 PerformanceEntry，用于标识最大的内容元素。在渲染后续帧后，只要最大内容元素发生变化，该 API 就会再分派另一个 PerformanceEntry。简单的来说，每一帧绘制的时候，浏览器都会标示出最大内容元素。

当用户与页面发生交互（通过点按、滚动或按键），浏览器就会停止报告新条目。（因为用户交互通常会改变向用户显示的内容，就比如滚动操作）。一般来说，发出的最后一个条目的 startTime 值是 LCP 值。

### 5.4. 内置图片组件

讲解 LCP，只是为了帮助大家认识到图片优化的重要性（毕竟最大内容元素往往是图片）。回到 Astro 上，最佳实践就是尽可能使用 `<Image>`、`<Picture>` 组件：

> <Image /> 可以转换本地或授权的远程图像的尺寸、文件类型和质量，以便控制显示的图像。生成的 <img> 标签包括 alt、loading 和 decoding 属性，并推断图像尺寸以避免累积布局偏移（CLS）。

```jsx
---
// 导入 Image 组件和图像
import { Image } from 'astro:assets';
import myImage from '../assets/my_image.png'; // Image is 1600x900
---

<!-- 在 Image 组件上 `alt` 是必须的  -->
<Image src={myImage} alt="A description of my image." />
```

输出结果为：

```jsx
<!-- 输出 -->
<!-- 优化图像，强制实施适当的属性 -->
<img
  src="/_astro/my_image.hash.webp"
  width="1600"
  height="900"
  decoding="async"
  loading="lazy"
  alt="A description of my image."
/>
```

如果需要动态导入图片，则借助 Vite 的 `import.meta.glob` 函数：

```jsx
---
import type { ImageMetadata } from 'astro';
import { Image } from 'astro:assets';
interface Props {
   imagePath: string;
   altText: string;
   name: string;
   age: number;
}

const { imagePath, altText, name, age } = Astro.props;
const images = import.meta.glob<{ default: ImageMetadata }>('/src/assets/*.{jpeg,jpg,png,gif}');
if (!images[imagePath]) throw new Error(`"${imagePath}" does not exist in glob: "src/assets/*.{jpeg,jpg,png,gif}"`);
---

<div class="card">
    <h2>{name}</h2>
    <p>Age: {age}</p>
    <Image src={images[imagePath]()} alt={altText} />
</div>
```
