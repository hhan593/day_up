## 1. 前言

本篇我们讲讲如何使用插件扩展 Markdown 的功能以及常添加的一些插件。

## 2. Astro 的插件体系

根据 Astro 官网的介绍：

> Astro 中的 Markdown 支持由 [remark](https://remark.js.org/) 提供，remark 是一个强大的解析和处理工具，拥有一个活跃的生态系统。目前不支持其他 Markdown 解析器，如 Pandoc 和 markdown-it。
>
> Astro 默认应用 [GitHub 风格的 Markdown](https://github.com/remarkjs/remark-gfm) 和 [SmartyPants](https://github.com/silvenon/remark-smartypants) 插件。这带来了一些便利，例如从文本生成可点击的链接，并格式化[引用和 em-dashes](https://daringfireball.net/projects/smartypants/)。
>
> Astro 支持添加第三方 [remark](https://github.com/remarkjs/remark) 和 [rehype](https://github.com/rehypejs/rehype) 插件来解析 Markdown。这些插件允许你扩展你的 Markdown，以获得新的功能。

简单的来说，MD 的编译分为两步，一步处理 Markdown，一步处理 HTML。处理的伪代码如下：

```javascript
import { unified } from "unified";
import remarkParse from "remark-parse";
import remarkRehype from "remark-rehype";
import rehypeSanitize from "rehype-sanitize";
import rehypeStringify from "rehype-stringify";

main();

async function main() {
  const file = await unified()
    .use(remarkParse) // 将 markdown 转换为 markdown AST
    .use(remarkRehype) // 转换为 HTML AST
    .use(rehypeSanitize) // HTML 消毒，处理不安全的内容，防止 XSS 攻击
    .use(rehypeStringify) // 将 AST 转换为 HTML
    .process("Hello, Astro!");

  console.log(String(file)); // <p>Hello, Astro!</p>
}
```

处理 Markdown 的这部分工具体系统称为 [Remark](https://github.com/remarkjs/remark)，处理 HTML 的这部分工具体系统称为 [Rehype](https://github.com/rehypejs/rehype)。Remark 和 Rehype 目前已经有不少的生态插件，比如语法高亮（rehype-pretty-code）、标题自动链接（rehype-autolink-headings）、生成目录（remark-toc）等。

如果你想找 Remark 相关的插件：

1.  <https://github.com/remarkjs/remark/blob/main/doc/plugins.md>
2.  <https://github.com/remarkjs/awesome-remark>

如果你想找 Rehype 相关的插件：

1.  <https://github.com/rehypejs/rehype/blob/main/doc/plugins.md>
2.  <https://github.com/rehypejs/awesome-rehype>

我们来说说大家基本都会用到的几个插件。

## 3. 标题自动链接（rehype-autolink-headings）

[rehype-autolink-headings](https://github.com/rehypejs/rehype-autolink-headings) 插件用于为标题自动生成链接，当点击标题的时候就会在当前网址上添加锚点。当分享的文章内容比较长的时候，用户可以通过锚点快速定位到需要的位置。

注意，因为生成链接需要 headings 有 id 属性，所以该插件需要搭配其他插件如 [rehype-slug](https://github.com/rehypejs/rehype-slug) 一起使用。所以只使用 rehype-autolink-headings 是没有效果的。

但你会发现，生成的标题本身是有 id 属性的，那为什么会没有效果呢？

其实这是执行顺序的问题，默认情况下，Astro 会在你的 rehype 插件运行后注入 id 属性。所以要想让 rehype-autolink-headings 生效有 2 个解决方法。

第一种：使用 Astro 自带的注入 id 属性的插件 rehypeHeadingIds 并将其放到 rehype-autolink-headings 前执行。修改 `astro.config.mjs`，代码如下：

```javascript
// @ts-check
import { defineConfig } from "astro/config";

// ...
import rehypeAutolinkHeadings from "rehype-autolink-headings";
import { rehypeHeadingIds } from "@astrojs/markdown-remark";
// ...

export default defineConfig({
  // ...
  markdown: {
    // ...
    rehypePlugins: [rehypeHeadingIds, rehypeAutolinkHeadings],
  },
});
```

第二种：使用 [rehype-slug](https://github.com/rehypejs/rehype-slug)。修改 `astro.config.mjs`，代码如下：

```javascript
// @ts-check
import { defineConfig } from "astro/config";

// ...
import rehypeAutolinkHeadings from "rehype-autolink-headings";
import rehypeSlug from "rehype-slug";
// ...

export default defineConfig({
  // ...
  markdown: {
    // ...
    rehypePlugins: [rehypeSlug, rehypeAutolinkHeadings],
  },
});
```

这两种都是可以的。此时浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a040562e1a92471c8841abe9cb5796d5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3110\&h=358\&s=162375\&e=png\&b=ffffff)

虽然标题没有变蓝，点击也没有效果，但查看 DOM 元素，确实在标题前面生成了一个链接元素。

为了让该链接元素生效，需要自己定义样式，比如在 `global.css`中添加：

```css
span.icon-link::before {
  content: "#";
  font-size: 1em;
  margin-right: 6px;
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/00b9d116f70a4f3189d6e792f3f57833~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=857\&h=290\&s=48195\&e=gif\&f=20\&b=fefefe)

如果我想让 `#`的下划线消失呢？你会发现，这个下划线样式是加在外层的 `<a>`标签上，而且没有特殊的类名：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d034b47e27cd4df49a24222e55bfb71e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1514\&h=246\&s=53415\&e=png\&b=282828)

如果你想修改，可以通过配置项中的 `properties`属性，修改 `astro.config.mjs`，添加代码如下：

```javascript
// ...

export default defineConfig({
  markdown: {
    rehypePlugins: [
      rehypeSlug,
      [
        rehypeAutolinkHeadings,
        {
          properties: {
            ariaHidden: "true",
            tabIndex: -1,
            class: "link-anchor",
          },
        },
      ],
    ],
  },
});
```

此时再在 `global.css`中添加：

```css
.link-anchor {
  text-decoration: none;
}
```

就可以去除下换线样式：

那现在我不想要 `#`这个效果，我想仿照 GitHub 的自动标题链接效果该如何实现呢？

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df0caf5a1bd541d680fb7a9bbece32cd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=850\&h=336\&s=70223\&e=gif\&f=14\&b=0c1216)

当鼠标悬浮在标题的时候，标题出现一个链接按钮，点击链接按钮，跳转锚点链接，那我们就来下实现这个效果。

运行：

```bash
npm i hast-util-from-html-isomorphic
```

修改 `astro.config.mjs`，添加代码如下：

```javascript
// @ts-check

export default defineConfig({
  markdown: {
    // ...
    rehypePlugins: [
      rehypeSlug,
      [
        rehypeAutolinkHeadings,
        {
          content: fromHtmlIsomorphic(
            '<svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="20" height="20" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .751.751 0 0 1 .018-1.042.751.751 0 0 1 1.042-.018 1.998 1.998 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.751.751 0 0 1-1.042-.018.751.751 0 0 1-.018-1.042Zm-4.69 9.64a1.998 1.998 0 0 0 2.83 0l1.25-1.25a.751.751 0 0 1 1.042.018.751.751 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .751.751 0 0 1-.018 1.042.751.751 0 0 1-1.042.018 1.998 1.998 0 0 0-2.83 0l-2.5 2.5a1.998 1.998 0 0 0 0 2.83Z"></path></svg>',
            { fragment: true }
          ).children,
          headingProperties: {
            class: "anchor",
          },
          properties: {
            ariaHidden: "true",
            tabIndex: -1,
            class: "link-anchor",
          },
        },
      ],
    ],
  },
});
```

然后添加全局样式：

```css
.anchor {
  position: relative;
}

.link-anchor {
  position: absolute;
  top: 50%;
  left: -28px;
  display: flex;
  width: 28px;
  height: 28px;
  margin: auto;
  opacity: 0;
  justify-content: center;
  align-items: center;
  transform: translateY(-50%);
  border: none;
}

@media (prefers-color-scheme: dark) {
  .link-anchor {
    fill: #fff;
  }
}

.anchor:hover > .link-anchor {
  opacity: 1;
}
```

## 4. emojis 添加可访问性（rehype-accessible-emojis）

[rehype-accessible-emojis](https://www.npmjs.com/package/rehype-accessible-emojis) 插件会为 emojis 添加 `aria-label`属性，使其具有可访问性。修改 `astro.config.mjs`，代码如下：

```javascript
import { defineConfig } from "astro/config";
import { rehypeAccessibleEmojis } from "rehype-accessible-emojis";

export default defineConfig({
  markdown: {
    rehypePlugins: [rehypeAccessibleEmojis],
  },
});
```

## 6. 警告（remark-alerts）

[remark-alerts](https://github.com/7nohe/remark-alerts) 用于实现警告信息，参照的是 GitHub Markdown 的语法：

```markdown
> [!NOTE]
> Useful information that users should know, even when skimming content.

> [!TIP]
> Helpful advice for doing things better or more easily.

> [!IMPORTANT]
> Key information users need to know to achieve their goal.

> [!WARNING]
> Urgent info that needs immediate user attention to avoid problems.

> [!CAUTION]
> Advises about risks or negative outcomes of certain actions.
```

效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f3d5af7bfced48fa84fa76b50b55b92f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600\&h=1010\&s=24696\&e=png\&b=ffffff)

那么我们该如何使用呢？首先安装插件：

```bash
npm i remark-alerts
```

修改 `astro.config.mjs`，代码如下：

```javascript
import remarkAlerts from "remark-alerts";

export default defineConfig({
  markdown: {
    remarkPlugins: [remarkAlerts],
  },
});
```

添加样式，需要导入三个样式文件：

```css
import "remark-alerts/styles/github-colors-light.css";
/* 如果你用的是 media query 判断暗黑模式，导入下面这个文件 */
import "remark-alerts/styles/github-colors-dark-media.css";
/* 如果你用的是 .dark 切换暗黑模式，导入下面这个文件 */
/* import "remark-alerts/styles/github-colors-dark-class.css"; */
import "remark-alerts/styles/github-base.css";
```

现在你再写：

```css
> [!NOTE] 注意
> The custom title will replace the regular title.

> [!TIP] 技巧
> Helpful advice for doing things better or more easily.

> [!IMPORTANT] 重要
> Key information users need to know to achieve their goal.

> [!WARNING] 警告
> Urgent info that needs immediate user attention to avoid problems.

> [!CAUTION] 谨慎
> Advises about risks or negative outcomes of certain actions.
```

## 7. 数学表达式（remark-math rehype-katex）

运行：

```css
npm i remark-math rehype-katex
```

修改 `astro.config.mjs`，代码如下：

```javascript
import remarkMath from "remark-math";
import rehypeKatex from "rehype-katex";

export default defineConfig({
  markdown: {
    remarkPlugins: [remarkMath],
    rehypePlugins: [rehypeKatex],
  },
});
```

在页面的 `<head>`中引入样式文件：

```css
---
// ..
---

<html lang="zh-cn">
  <head>
    <link
      rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css"
      crossorigin="anonymous"
    />
  </head>
  // ..
</html>
```

在 Markdown 中写入：

```css
## 4.4. 数学表达式

**The Cauchy-Schwarz Inequality**

$$\left( \sum_{k=1}^n a_k b_k \right)^2 \leq \left( \sum_{k=1}^n a_k^2 \right) \left( \sum_{k=1}^n b_k^2 \right)$$
```
