## 3. Typography

但是目前的 Markdown 渲染后的样式并不算好看。如何优化这个样式呢？

我们可以在 `global.css`中慢慢写每个元素的样式，但如果你想更快捷的实现，可以使用 Tailwind CSS 的 Typography 插件，它专门用于处理文章的排版样式，只用添加一个类名，就可以让文章有个不错的排版样式。

首先集成 Tailwind CSS：

```bash
npx astro add tailwind
```

然后安装 `@tailwindcss/typography`：

```bash
npm install -D @tailwindcss/typography
```

修改 `tailwind.config.js`，代码如下：

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: ["./src/**/*.{astro,html,js,jsx,md,mdx,svelte,ts,tsx,vue}"],
  theme: {
    extend: {},
  },
  plugins: [require("@tailwindcss/typography")],
};
```

新建 `src/components/Prose.astro`，代码如下：

```jsx
---
---
<div
  class="prose dark:prose-invert">
  <slot />
</div>
```

修改 `src/pages/blog/[...slug].astro`，使用该 Prose 组件包裹 Content 组件，代码如下：

```jsx
---
import type {
  InferGetStaticParamsType,
  InferGetStaticPropsType,
  GetStaticPaths,
} from "astro";
import { getCollection } from "astro:content";
import Prose from "../../components/Prose.astro";

export const getStaticPaths = (async () => {
  const posts = await getCollection("blog");
  return posts.map((post) => ({
    params: { slug: post.slug },
    props: { post },
  }));
}) satisfies GetStaticPaths;

type Props = InferGetStaticPropsType<typeof getStaticPaths>;

const { post } = Astro.props;
const { Content } = await post.render();
---

<html lang="zh-cn">
  <head>
    <meta charset="utf-8" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <meta name="viewport" content="width=device-width" />
    <meta name="generator" content={Astro.generator} />
    <title>{post.data.title}</title>
  </head>
  <body style="padding: 30px;">
    <Prose>
      <Content />
    </Prose>
  </body>
</html>

<style>
  @media (prefers-color-scheme: dark) {
    body {
      background-color: #0d1117;
    }
  }
</style>
```

```

如果样式修改比较多，我会更加推荐使用第二种方式，通过 `tailwind.config.js`的 Typography 插件配置项，直接修改默认的样式效果。

也是参考 tailwindcss-typography 的[使用说明](https://github.com/tailwindlabs/tailwindcss-typography?tab=readme-ov-file#customizing-the-css)，删除掉上一节设置的 `src/components/Prose.astro`新增的类名，然后修改 `tailwind.config.mjs`，完整代码如下：

/** @type {import('tailwindcss').Config} */
export default {
  content: ["./src/**/*.{astro,html,js,jsx,md,mdx,svelte,ts,tsx,vue}"],
  theme: {
    extend: {
      typography: {
        DEFAULT: {
          css: {
            blockquote: {
              color: "#666",
              fontStyle: "normal",
              backgroundColor: "#f8f8f8",
              padding: "1px 20px",
            },
          },
        },
        invert: {
          css: {
            blockquote: {
              color: "#f0f6fc",
              backgroundColor: "#151b23",
            },
          },
        },
      },
    },
  },
  plugins: [require("@tailwindcss/typography")],
};
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/541d2e03d2034b6abaa277dc8aca00d7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1476\&h=420\&s=45225\&e=png\&b=25292e)

这是为什么呢？

这是因为对于语法高亮，Astro 内置支持 [Shiki](https://shiki.style/) 和 [Prism](https://prismjs.com/)。Shiki 默认启用，预配置为 `github-dark` 主题。编译输出将限于内联 `style`，而不会包含任何冗余的 CSS 类、样式表或客户端 JS。



当然都使用暗黑代码样式也是可以的，但假如你希望能够切换代码主题样式，首先你需要先修改 `shikiConfig`的配置。



修改 `astro.config.mjs`，完整代码如下：



```jsx
// @ts-check
import { defineConfig } from "astro/config";

import tailwind from "@astrojs/tailwind";

// https://astro.build/config
export default defineConfig({
  integrations: [tailwind()],
  markdown: {
    shikiConfig: {
      themes: {
        light: "github-light",
        dark: "github-dark",
      },
    },
  },
});
```

这里我们选择了 `github-light`和 `github-dark`，如果你要选择主题，到这里选择： <https://shiki.style/themes>。

此时浅色代码样式确实生效了：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/051f8ba893004e1ba0c812fa31594f05~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1462\&h=426\&s=43897\&e=png\&b=ffffff)

但是有两个问题：

1.  暗黑模式下，代码还是浅色主题。也就是代码主题没有跟随外观模式而切换
2.  浅色模式下，代码背景和页面背景同色，需要自定义背景颜色

我们一个一个看。

第一个问题，其实[官方文档](https://docs.astro.build/zh-cn/guides/markdown-content/#shiki-%E9%85%8D%E7%BD%AE)里有讲到：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbae80ea8af04cc4ac4de531ed303677~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2314\&h=424\&s=129629\&e=png\&b=ced2f8)

修改 `src/components/Prose.astro`，完整代码如下：

```jsx

---

<div class="prose dark:prose-invert">
  <slot />
</div>

<style is:global>
  @media (prefers-color-scheme: dark) {
    .astro-code,
    .astro-code span {
      color: var(--shiki-dark) !important;
      background-color: var(--shiki-dark-bg) !important;
      /* Optional, if you also want font styles */
      font-style: var(--shiki-dark-font-style) !important;
      font-weight: var(--shiki-dark-font-weight) !important;

```

+            },

      text-decoration: var(--shiki-dark-text-decoration) !important;
    }
  }
</style>

## 6. 暗黑模式与样式设置影响

本篇之前的例子中，我们都是使用 `@media (prefers-color-scheme: dark)`来判断的暗黑主题，这样做法需要让网页页面跟随系统设置，而不能主动修改。如果页面自定义了外观模式，这种定义暗黑主题的方式会导致页面效果无法跟随自定义的外观模式而变化，如果你使用了诸如为 `html` 添加了 `dark`类名的方式来定义暗黑主题，则需要进行一些调整。这里说一下，最少有两处需要修改：

一处是 `tailwind.config.mjs`，需要为 Tailwind CSS 添加 dark 模式的判断：

```diff
export default {
  // ...
+ darkMode: "selector",
  // ...
};
```

一处是代码主题的定义，本篇我们将代码放到了 `src/components/Prose.astro`：

```jsx
---

---

<div class="prose dark:prose-invert">
  <slot />
</div>

<style is:global>
  pre.astro-code {
    background-color: #e6eaef !important;
  }

  html.dark {
    pre.astro-code {
      background-color: var(--shiki-dark-bg) !important;
    }
  }

  html.dark .astro-code,
  html.dark .astro-code span {
    color: var(--shiki-dark) !important;
    background-color: var(--shiki-dark-bg) !important;
    /* Optional, if you also want font styles */
    font-style: var(--shiki-dark-font-style) !important;
    font-weight: var(--shiki-dark-font-weight) !important;
    text-decoration: var(--shiki-dark-text-decoration) !important;
  }
</style>
```

如此我们就可以实现根据 html 的 class 类名切换文章的外观模式。
