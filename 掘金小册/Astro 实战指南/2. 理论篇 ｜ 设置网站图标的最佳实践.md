## 1. 前言

如何设置网站的图标呢？你可能以为设置一个 `favicon.ico`就可以了，但实际上要设置好多个。我们肯定经常看到这种设置图标的代码，以掘金网站为例：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/82268b7b5419405b90efebe8e251864b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1788\&h=392\&s=207470\&e=png\&b=282828)

怎么有这么多尺寸需要设置的？16x16、32x32、180x180，还有其他需要的吗？`site.manifest` 又是什么鬼？也需要吗？

本篇我们就来详细介绍一下如何设置一个网站的图标。

## 2. 最佳实践

为了节约时间，我先直接给出最佳实践：

找到需要的图片，然后借助平台生成不同尺寸和格式的图标。目前有两个推荐网站：

1.  <https://www.logosc.cn/favicon-generator>
2.  <https://realfavicongenerator.net/>

两个网站生成的文件和推荐使用代码非常类似，我们简单介绍一下。

### 2.1. [favicon-generator](https://www.logosc.cn/favicon-generator)

该网站既可以根据输入的字母生成图标，也可以上传图片生成图标：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3622e29f1af549489ef343995945fb46~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2262\&h=1218\&s=176262\&e=png\&b=fdfdfd)

下载生成的图标文件，解压后查看，一共有 7 个文件：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/103cc7ba2c9245908d3a6fd879481bc4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1286\&h=382\&s=83133\&e=png\&b=211d26)

分别是：

*   android-chrome-192×192.png
*   android-chrome-512×512.png
*   apple-touch-icon.png
*   favicon-16×16.png
*   favicon-32×32.png
*   favicon.ico
*   site.webmanifest

将这些文件都上传到网站根目录（Astro 中就是都放到 `public`目录下），然后设置 HTML 的 `<head>` 中代码为：

```html
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="manifest" href="/site.webmanifest">
```

查看 `site.webmanifest`的代码：

```json
{
  "background_color": "#ffffff",
  "display": "standalone",
  "icons": [
    {
      "sizes": "192x192",
      "src": "/android-chrome-192x192.png",
      "type": "image/png"
    },
    {
      "sizes": "512x512",
      "src": "/android-chrome-512x512.png",
      "type": "image/png"
    }
  ],
  "name": "",
  "short_name": "",
  "theme_color": "#ffffff"
}
```

可以看到这里使用了 192x192 和 512x512 这两张图标。

### 2.2. [realfavicongenerator](https://realfavicongenerator.net/)

这个网站也是类似：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e4424725a74d4a3aa60f40704981811f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2830\&h=1222\&s=262063\&e=png\&b=ffffff)

上传图片后下载文件，也是 7 个文件：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/80cc9a69c1ba4265b0ec9cc30d3c1ee7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1110\&h=394\&s=81375\&e=png\&b=201d26)

分别是：

*   web-app-manifest-192x192
*   web-app-manifest-512x512.png
*   apple-touch-icon.png
*   favicon-48x48.png
*   favicon.svg
*   favicon.ico
*   site.webmanifest

与上一个网站生成的文件的主要区别在于：

favicon-generator 生成的是 16x16.png 和 32x32.png，realfavicongenerator 生成的是 favicon.svg 和 48x48.png

将这些文件都上传到网站根目录（Astro 中就是都放到 `public`目录下），然后 HTML 的 head 中代码为：

```html
<link rel="icon" type="image/png" href="/favicon-48x48.png" sizes="48x48" />
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
<link rel="shortcut icon" href="/favicon.ico" />
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
<link rel="manifest" href="/site.webmanifest" />
```

使用哪个网站都是可以的，我们接下来介绍为什么是这几个文件，以及每个文件的作用。

## 3. 图标介绍

`favicon`是 `“favorite icon”`的缩写，从 2000 年就开始存在。我们经常在浏览器的标签栏、收藏栏、书签页面看到它：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a3b7d513dedc4462ad13e721fed6eb50~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=838\&h=216\&s=39081\&e=png\&b=363636)

这些图标往往非常小，但却可以帮助我们区分不同的网站，让用户轻松的识别并打开你的网站。

最早定义 favicon 的方式是将一个名为 `favicon.ico`的文件放在服务器根目录下，浏览器会在加载网页的时候，请求 `/favicon.ico`文件作为图标。后来才出现了使用 `<link>` 标签这种更为灵活的方法：

```html
<link rel="icon" href="/favicon.ico" />
```

目前大多数浏览器都支持这两种方法。

当我们设置网站的图标时：

首先要确保网站根目录有一个 `favicon.ico`，也就是 `https://example.com/favicon.ico`应该是一个图标。因为有些工具（例如 RSS 阅读器）仅从服务器请求 `/favicon.ico`，而不会再去其他地方查找。

其次，Chrome 中的浏览器操作图标宽和高均为 16 像素，这就是为什么有的图标是 16x16，但是现在高 dip 设备也越来越多，为了让图标更加清晰，也会提供 16 倍数的像素图标，也就是 32x32 和 48x48。除此之外，根据设备、应用、场景的不同，会使用不同尺寸大小的图片，比如不同的浏览器、不同的场景（标签、Google 搜索结果、书签、移动端图标）、不同的系统如 IOS 和 Windows 使用的图片大小都可能不同。常用的浏览器图标也就是 16x16、32x32、48x48。

选择 SVG 也是一个不错的方案。SVG 文件非常轻量，并且可以无限扩展（毕竟是矢量图），没有大图片加载消耗时间长的问题。但是支持度不好，目前只有 Chrome、Firefox 和 Opera 支持 SVG 格式的网站图标。

所以我们可能会这样写：

```html
<link rel="icon" href="/favicon.ico">
<link rel="icon" href="/favicon.svg" type="image/svg+xml">
```

`/favicon.svg`用于较新的浏览器，`/favicon.ico`用于老旧浏览器，或者搜索引擎等。在火狐是可以正常工作的，但在 Chrome 中有一个 [bug](https://x.com/subzey/status/1417099601090424833)：在 DPR 为 1 的时候，Chrome 会把 `svg`和 `ico`都下载下来，然后使用 `ico`……

这是因为 ico 并未设置 `sizes`，为了匹配最适合的尺寸，会找到 `ico`中的 16x16 图标。为了避免这个 bug，代码需要写为：

```html
<link rel="icon" href="favicon.ico" sizes="any">
<link rel="icon" href="/favicon.svg" type="image/svg+xml">
```

后来又建议改成了：

```html
<link rel="icon" href="/favicon.ico" sizes="32x32">
<link rel="icon" href="/icon.svg" type="image/svg+xml">
```

然后是 `apple-touch-icon`，顾名思义，针对苹果设备使用的图标，用于将网页添加到 iPhone 或 iPad 屏幕快捷方式时使用的图标。从 iOS 8+ 开始，iPad 需要具有 180×180 分辨率的图像。其他设备会缩小它。所以一般就是做 180x180。

最后是 `manifest`，这个格式来自于 PWA，它本身是一个 JSON 文件，包含浏览器将网站安装为系统应用程序的所需信息。根据文档信息：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/36b509261321470db278415ec40096c7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2274\&h=630\&s=260643\&e=png\&b=ffffff)

所以它至少需要一个 192x192 像素的图标和一个 512x512 像素的图标。

所以目前理论上最佳的代码应该是：

```html
<link rel="icon" href="/favicon.ico" sizes="32x32">
<link rel="icon" href="/icon.svg" type="image/svg+xml">
<link rel="apple-touch-icon" href="/apple-touch-icon.png"><!-- 180×180 -->
```

如果添加 PWA，HTML 加上：

```html
<link rel="manifest" href="/manifest.webmanifest">
```

`manifest.webmanifest`的代码为：

```json
{
  "icons": [
    { "src": "/icon-192.png", "type": "image/png", "sizes": "192x192" },
    { "src": "/icon-mask.png", "type": "image/png", "sizes": "512x512", "purpose": "maskable" },
    { "src": "/icon-512.png", "type": "image/png", "sizes": "512x512" }
  ]
}
```

注：关于 maskable，也就是可遮盖式图标，可以使用 [maskable.app](https://maskable.app/?demo=demo/spec.svg) 这个网站查看图标效果。简单来说，就是图标设计的时候，周边多留一些空白，这个才能在不同设备以不同形状显示图标时比较完美：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b19261ae856545c1ac66c4a4497923d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1301\&h=582\&s=195409\&e=gif\&f=44\&b=323232)

目前我还没有看到使用这样代码的图标制作平台（所以如果看到了，告诉我！）

使用那两个网站的好处在于方便快捷，上传图片就可以直接使用，而且基本浏览器都能正常显示图标，只是可能在少数平台上会有一些差距。

此外，我们在前言中看到掘金的代码中还有一句：

```html
<link rel="mask-icon" href="https://.../favicons/safari-pinned-tab.svg" color="#1E80FF">
```

这是因为 Safari 以前要求固定选项卡使用 SVG 单色图标，但是，从 Safari 12 开始，常规的图标也可以使用了，所以就无所谓了。
