## 1. 前言

使用 Tailwind CSS 的时候，经常会遇到一个问题，那就是当项目变得复杂，尤其是要处理组件的多种样式的时候，繁琐的判断会让代码显得混乱。

我们以 Ant-Design 的 Button 组件为例，一个 Button 有大、中、小三种尺寸，有五种类型：主按钮、次按钮、虚线按钮、文本按钮和链接按钮：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dea51bf1da834844972d36a44a75f6f2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1686\&h=170\&s=34277\&e=png\&b=fefefe)


如果我们的项目中写这种组件，代码很可能会变成这样（Next.js 代码）：

```jsx
"use client";

import { twMerge } from "tailwind-merge";
import { clsx } from "clsx";

function cn(...inputs) {
  return twMerge(clsx(inputs));
}

function Button({ type = "default", size = "middle" }) {
  return (
    <button
      type="submit"
      className={cn("rounded p-2", {
        "bg-blue-600 text-white": type === "default",
        "border border-black bg-white text-black": type === "primary",
        "border border-dashed border-black bg-white": type === "dashed",
        "text-blue-600": type === "link",
        "text-black": type === "text",
        "px-2 py-2": size === "small",
        "px-4 py-2": size === "middle",
        "px-6 py-2": size === "large",
      })}
    >
      Default Button
    </button>
  );
}

export default function Home() {
  return (
    <div className="p-4">
      <Button />
    </div>
  );
}
```

可以看到，代码并不直观，且随着样式越来越多，className 的代码会变得臃肿难以维护。这个时候就需要 [cva（Class Variance Authority）](https://github.com/joe-bell/cva)了。

一位国外网友对 CVA 的评价是：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c4b7076fc673443a8a727752ddd97385~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1818\&h=312\&s=76569\&e=png\&b=0e1113)

本篇我们就来详细介绍下 cva 这个库。

## 2. 安装与智能提示

安装并不复杂，它就是一个工具库：

```bash
npm i class-variance-authority
```

当我们使用 cva 的时候，因为 Tailwind CSS 的类名改为在 `cva()`函数中书写，所以 VSCode 插件 [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) （这是 Tailwind CSS 官方提供的插件，可用于自动补全、Lint、悬浮预览等）将不会再有智能提示。

为了解决这个问题，需要修改一下配置。Command + Shift + P，打开命令面板，搜索 `User Settings`：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cc6d164e967a461bb294f172fc34cd40~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2074\&h=280\&s=59605\&e=png\&b=1f3a58)

打开后，在 `settings.json`中添加如下代码：

```json
{
  // ... 原本的配置项
  "tailwindCSS.experimental.classRegex": [
    ["cva\\(([^)]*)\\)", "[\"'`]([^\"'`]*).*?[\"'`]"],
    ["cx\\(([^)]*)\\)", "(?:'|\"|`)([^']*)(?:'|\"|`)"]
  ]
}
```

然后重启 VSCode 或者 Reload Window：

## 3. 基本用法

`cva` 的用法如下：

```jsx
const component = cva("base", options);
```

1.  `base`：基础类名，即所有变体公用的类名
2.  `options`
    1.  `variants`：声明变体
    2.  `compoundVariants`：基于之前声明的变体的混合变体
    3.  `defaultVariants`：默认变体

### 3.1. variants、compoundVariants

这样写有些抽象，让我们直接看个例子。新建 `src/components/Button.astro`：

```jsx
---
import type { HTMLAttributes } from "astro/types";
import { cva, type VariantProps } from "class-variance-authority";

const button = cva("button rounded-md", {
  variants: {
    intent: {
      primary: [
        "bg-blue-500",
        "text-white",
        "border-transparent",
        "hover:bg-blue-600",
      ],
      secondary: [
        "bg-white",
        "text-gray-800",
        "border-gray-400",
        "hover:bg-gray-100",
      ],
    },
    size: {
      small: ["text-sm", "py-1", "px-2"],
      medium: ["text-base", "py-2", "px-4"],
    },
  },
  compoundVariants: [{ intent: "primary", size: "medium", class: "uppercase" }],
});

export interface Props
  extends HTMLAttributes<"button">,
    VariantProps<typeof button> {}

const { intent = "primary", size = "medium" } = Astro.props;
---

<button class={button({ intent, size })}>
  <slot />
</button>
```

这段代码展示了 `cva`的基本用法，那就是先通过 `variants`声明不同的变体，其中一个是 `intent`（intent 中文翻译为“意图”、“目的”，你可以把它理解为我们常用的 type），一个是 `size`。

`intent`又分为两种，`primary` 和 `secondary`，`size`分为两种，`small` 和 `medium`，这样我们就建立了不同的变体。

最后我们调用 `button({ intent, size })`传入具体的 `intent` 和 `size` 的值，cva 就能算出该变体对应的类名。使用起来就是这么简单。

比如当我们输入：

```javascript
// "button rounded-md bg-blue-500 text-white border-transparent hover:bg-blue-600 text-sm py-1 px-2"
button({ intent: "primary", size: "small" });
```

其中， `button rounded-md`来自 `base`，`bg-blue-500 text-white border-transparent`来自变体 `primary`，`text-sm py-1 px-2`来自变体 `small`。

那你可能问了，`compoundVariants`是啥意思呢？以这个例子作为说明：

```javascript
compoundVariants: [{ intent: "primary", size: "medium", class: "uppercase" }],
```

这句话的意思是当 `intent` 是 `primary`，`size`是 `medium`时，再添加 `uppercase`类名。

所以当我们输入：

```javascript
// "button rounded-md bg-blue-500 text-white border-transparent hover:bg-blue-600 text-base py-2 px-4 uppercase"
button({ intent: "primary", size: "medium" });
```

其中，`button rounded-md`来自 `base`，`bg-blue-500 text-white border-transparent hover:bg-blue-600`来自变体 `primary`，`text-base py-2 px-4`来自变体 `medium`，最后的 `uppercase`来自于 `compoundVariants`。

我们展示下不同变体的效果。新建 `src/pages/cva.astro`，代码如下：

```jsx
---

---

<html class="h-full">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <meta name="generator" content={Astro.generator} />
    <title>cva: Astro with Tailwind CSS</title>
  </head>
  <body class="p-[60px]">
    <div class="grid gap-4">
      <Button intent="primary" size="small"> primary small button </Button>
      <Button intent="primary" size="medium"> primary medium button </Button>
      <Button intent="secondary" size="small"> secondary small button </Button>

</html>
```

### 3.2. defaultVariants

这个例子中没有展示 `defaultVariants`，是**因为对于 Astro 组件，Astro 建议像这个例子一样，通过 Astro.props 设置默认变体。**

不过我们还是说明一下 `defaultVariants` 的用法.还是这个例子，在此基础上加上 `defaultVariants`：

```javascript
const button = cva("button rounded-md", {
  variants: {
    intent: {
      primary: ["bg-blue-500", "text-white", "border-transparent", "hover:bg-blue-600"],
      secondary: ["bg-white", "text-gray-800", "border-gray-400", "hover:bg-gray-100"],
    },
    size: {
      small: ["text-sm", "py-1", "px-2"],
      medium: ["text-base", "py-2", "px-4"],
    },
  },
  compoundVariants: [{ intent: "primary", size: "medium", class: "uppercase" }],
  defaultVariants: {
    intent: "primary",
    size: "medium",
  },
});
```

效果描述：

```javascript
// 按照 primary 和 medium 返回（因为符合 compoundVariants，还会返回 `uppercase`）
button();
// 按照 primary 和 medium 返回（因为符合 compoundVariants，还会返回 `uppercase`）
button({ intent: undefined, medium: undefined });
// 按照 secondary 和 medium 返回
button({ intent: "secondary" });
```

## 4. 进阶用法

### 4.1. 不需要用数组

声明变体的时候也可以不使用数组，而是直接使用字符串：

```javascript
const button = cva("button rounded-md", {
  variants: {
    intent: {
      primary: "bg-blue-500 text-white border-transparent hover:bg-blue-600",
      secondary: "bg-white text-gray-800 border-gray-400 hover:bg-gray-100",
    },
    size: {
      small: "text-sm py-1 px-2",
      medium: "text-base py-2 px-4",
    },
  },
  compoundVariants: [{ intent: "primary", size: "medium", class: "uppercase" }],
});
```

### 4.2. 多个变体条件

使用混合变体的时候，也可以通过数组指定多个变体条件，比如：

```javascript
// components/button.ts
import { cva } from "class-variance-authority";

const button = cva("…", {
  variants: {
    intent: { primary: "…", secondary: "…" },
    size: { small: "…", medium: "…" },
  },
  compoundVariants: [
    {
      intent: ["primary", "secondary"],
      size: "medium",
      class: "…",
    },
  ],
});
```

这样 `button({ intent: "primary", size: "medium" })`和` button({ intent: "secondary", size: "medium" })`都会触发混合变体，添加对应的 class 类名。

### 4.3. 添加额外类名

所有的 `cva`组件都会添加一个可选的 `class`或 `className` prop，可用于将一些类名添加到最后（搭配本篇后面介绍的 twmerge，可以实现覆盖类名效果）：

```javascript
// components/button.ts
import { cva } from "class-variance-authority";

const button = cva(/* … */);

button({ class: "m-4" });
// => "…buttonClasses m-4"

button({ className: "m-4" });
// => "…buttonClasses m-4"
```

### 4.4. 也可用于动态文案

其实 cva 本质就是根据不同的变体返回不同的字符串，所以其实也可以用于动态文案：

```javascript
const greeter = cva("Good morning!", {
  variants: {
    isLoggedIn: {
      true: "Here's a secret only logged in users can see",
      false: "Log in to find out more…",
    },
  },
  defaultVariants: {
    isLoggedIn: "false",
  },
});

greeter();
// => "Good morning! Log in to find out more…"

greeter({ isLoggedIn: "true" });
// => "Good morning! Here's a secret only logged in users can see"
```

### 4.5. 样式冲突问题

使用 `cva`虽然让样式声明更加清晰，但并不解决样式冲突问题，举个例子，假设变体声明为：

```javascript
const button = cva("button rounded-md text-white py-1 px-2 text-sm", {
  variants: {
    intent: {
      primary: "bg-blue-500",
    },
    size: {
      small: "bg-red-500",
    },
  },
});
```

那么 `button({ intent: 'primary', size: 'small'})`会返回什么呢？最终的效果是什么样的呢？

结果会返回：

```javascript
"button rounded-md text-white py-1 px-2 text-sm bg-red-500 bg-blue-500";
```

浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8879d7d2b8b2471d89fb6370055cdd7e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2862\&h=656\&s=255889\&e=png\&b=2c2c2c)

`bg-blue-500` 虽然位置上在类名 `bg-red-500`后面，但不会覆盖 `bg-red-500`，这是为什么呢？

**这是因为 HTML 元素的类名书写顺序并不影响类的优先级，类的优先级取决于样式文件中出现的先后顺序，越晚出现，优先级越高。**

所以按钮最后是什么颜色，取决于 Tailwind CSS 生成的样式表中的文件的类名先后顺序，但这是不可控的，这就可能会造成错误。

所以为了解决可能出现的样式问题，需要借助 `tailwind-merge`这个库。

安装依赖项：

```bash
npm i tailwind-merge
```

新建 `src/pages/cvawithmerge.astro`，代码如下：

```jsx
---
import { cva, type VariantProps } from "class-variance-authority";
import { twMerge } from "tailwind-merge";

const buttonVariants = cva("button rounded-md text-white py-1 px-2 text-sm", {
  variants: {
    intent: {
      primary: "bg-red-500",
    },
    size: {
      small: "bg-blue-500",
    },
  },
});

interface ButtonVariants extends VariantProps<typeof buttonVariants> {}

const button = (variants: ButtonVariants) => twMerge(buttonVariants(variants));
---

<button class={button({ size: "small", intent: "primary" })}>
  Primary Button
</button>

<style>
  body {
    padding: 60px;
  }
</style>
```

`twMerge()` 函数支持传入多个参数，如果发生冲突，后传入的类名优先级更高，会覆盖之前的类名。

原本 `buttonVariants(variants)`会返回 `button rounded-md text-white py-1 px-2 text-sm bg-red-500 bg-blue-500`，经过 `twMerge`处理后，因为后传入的相同类名优先级更高，会覆盖之前的类名，所以 `twMerge`最终会处理为 `button rounded-md text-white py-1 px-2 text-sm bg-blue-500`

浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a4573a3055440668d71afe552fbf6f3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2276\&h=676\&s=167471\&e=png\&b=2c2c2c)

如果不希望那么多声明，这段代码也可以写为：

```jsx
---
import { cva } from "class-variance-authority";
import { twMerge } from "tailwind-merge";

const button = cva("button rounded-md text-white py-1 px-2 text-sm", {
  variants: {
    intent: {
      primary: "bg-red-500",
    },
    size: {
      small: "bg-blue-500",
    },
  },
});
---

<button class={twMerge(button({ size: "small", intent: "primary" }))}>
  Primary Button
</button>

<style>
  body {
    padding: 60px;
  }
</style>
```
