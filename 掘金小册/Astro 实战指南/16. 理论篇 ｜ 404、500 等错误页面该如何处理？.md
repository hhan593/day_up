### 2.1. 自定义 404 错误页面

如果想要自定义 404 错误页面，你可以在 `src/pages` 中创建 `404.astro` 或 `404.md` 文件。它将生成 404.html 页面。

404 页面很重要。

对于爬虫而言：

> 搜索引擎蜘蛛在请求某个 URL 时得到“404”状态回应时，即知道该 URL 已经失效，便不再索引该网页，并向数据中心反馈将该 URL 表示的网页从索引数据库中删除，于是就产生了 404 错误页面。404 页面的存在不仅对于是用户访问网站的一种反馈，而且在搜索引擎中占有重要的地位。

对于用户而言：

> 当用户不小心访问了某一个不存在的页面并且没有设置错误提示时，用户会怎么做？如果是初次到访的访客，那么回头率是多少呢？毫无疑问，会抱怨为何出现错误，之后当然是直接关闭窗口离开。
>
> 所以设置了 404 页面也相当于是做了一个针对用户的提示页面，当用户访问了某一个不存在的页面后就会转到 404 页面，然后因为你在这个 404 页面上有友好的提示，并且有首页和主要栏目页的连接，用户就很有可能会再次点击进入你的首页。
>
> 但是，Web 服务器默认的 404 错误页面，无论是 Apache 还是 IIS，均十分简陋、呆板且对用户不友好，无法给用户提供必要的信息以获取更多线索，无疑这会造成用户的流失。因此，很多网站均使用自定义 404 错误的方式提供用户体验避免用户流失。

那么如何设计一个 404 页面呢？这里有一些建议：

### 2.2. 建议 1：保持风格一致

以 [PIXAR 的 404 页面](https://www.pixar.com/404)为例：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/65d365548285465f8d20edfae9cd56e7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2802\&h=2374\&s=2143102\&e=png\&b=fefdfd)

这里的图片选择了 PIXAR 的《头脑特工队 2》中的焦焦（对应情绪中的焦虑），然后告诉你虽然来到了 404 页面，但不要烦躁，Everything is gonna be OK。算是一个设计上的小巧思。

### 2.3. 建议 2：帮助用户解决问题

既然来到了 404 页面，就说明用户遇到了问题，此时应该尝试帮助用户解决问题。比如 [Airbnb 的 404 错误页面](https://zh.airbnb.com/404)：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/13ec564c66a34f0a8f12a60e26c6c241~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1100\&h=735\&s=304158\&e=gif\&f=32\&b=fefefe)

右侧是一个冰淇淋掉在地上的动画，以此表达遇到 404 的失落。而左侧列出了可以跳转的导航，让用户不至于感觉来到了死胡同，指引用户继续浏览站点。

又或者[淘宝的 404 错误页面](https://error.taobao.com/app/tbhome/common/error.html)：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fa975e5db7b54ef288ed2ad2d0821e79~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2560\&h=1228\&s=1046183\&e=png\&b=fdfcfc)

也是相同的道理，解释出现问题的原因，指引用户去新的地址。我记得以前还有搜索框，可以搜索内容。

## 3. 500

### 3.1. 自定义 500 错误页面

astro\@4.10.3 后可以自定义 500 错误页面，状态码 500 一般用于服务端错误，所以 500 错误页面用于开启按需渲染的时候，也就是 `astro.config.mjs` 中的 `output` 选项为 `server` 或者 `hybrid` 的时候：

```javascript
import { defineConfig } from 'astro/config';
import node from "@astrojs/node";

export default defineConfig({
  output: 'server',
  adapter: node({
    mode: "standalone"
  })
});
```

500 错误页面不可用于预渲染页面，也无法预渲染。

对于渲染期间抛出的任何错误，Astro 会自动传递 error 属性。可以用于展示错误的详细信息（例如来自页面、中间件等）：

```html
// src/pages/500.astro

---
interface Props {
    error: unknown
}

const { error } = Astro.props
---

<div>{error instanceof Error ? error.message : 'Unknown error'}</div>
```
