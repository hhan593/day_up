## 1. 前言

关于 remark 和 rehype 的背景知识，参考[《理论篇 | remark 插件如何写？》](https://juejin.cn/book/7452253924608147467/section/7453022391260708883)。

本篇我们讲解如何写 rehype 插件。

## 2. 语法介绍

Rehype 插件的基本写法如下：

```javascript
export default function rehypeSlug() {
  return function (tree) {
    console.dir(tree, { depth: null });
  };
}
```

以这样一段 Markdown 文件为例：

```markdown
## Title 1

[冴羽的博客](https://yayujs.com)

## Title 2

[冴羽的翻译](http://ts.yayujs.com)
```

打印的 `tree` 值为（做了一点简化）：

## 3. 实战 1：添加外部链接标识

需求：为内容中的外部链接添加一个特殊标识，比如 🔗，期望的效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/606093afa67f48bc9e3113c200a83d03~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=884\&h=298\&s=36033\&e=png\&b=fefefe)

为了快捷实现这个效果，我们需要安装 2 个依赖项：

1.  [unist-util-visit](https://unifiedjs.com/explore/package/unist-util-visit/)：用于遍历 tree
2.  [hastscript](https://github.com/syntax-tree/hastscript)：用于生成 tree 节点（注意同样是构建节点，在 remark 中对应生成节点的库叫 unist-builder）

新建 `src/plugins/rehype-external-links.js`，代码如下：

```javascript
import { visit } from "unist-util-visit";
import { h } from "hastscript";

export default function rehypeSlug(options) {
  return function (tree) {
    visit(tree, "element", function (node, index, parent) {
      if (node.tagName === "a" && typeof node.properties.href === "string") {
        const url = node.properties.href;
        if (url.startsWith("https://") || url.startsWith("http://")) {
          if (options.content) {
            node.children.push(h("span", options.content));
          }
        }
      }
    });
  };
}
```

代码逻辑并不复杂，主要是判断是否符合条件，然后使用 `h("span", options.content)`构建了一个 tree 节点，将其添加到 node.children 中。这段代码其实就是 [rehype-external-links](https://github.com/rehypejs/rehype-external-links/tree/main) 这个插件的极简版实现。

修改 `astro.config.mjs`，添加代码如下：

```diff
+import rehypeExternalLinks from "./src/plugins/rehype-external-links.js";

export default defineConfig({
  markdown: {
    rehypePlugins: [
      [
+        rehypeExternalLinks,
+        {
+          content: { type: "text", value: " 🔗" },
+        },
      ],
    ],
  },
});
```

### 4.1. remark-directive

首先这种 `::github{repo="saicaca/fuwari"}` 写法出自于这样一个[通用指令提案](https://talk.commonmark.org/t/generic-directives-plugins-syntax/444)，在这个通用指令提案下有三种语法形式：

```markdown
// 第一种
:name[content]{key=val}

// 第二种
:: name [content] {key=val}

// 第三种
::: name [inline-content] {key=val}
contents, which are sometimes further block elements
:::
```

举些例子：`:wikipedia[Foobar]`用于渲染维基百科链接、`:tel[+375255318270]`用于渲染电话号码，`::youtube[title]{vid=09jf3ow9jfw}`用于渲染 Youtube 视频

下面这种：

```markdown
:::note{.warning}
if you chose xxx, you should also use yyy somewhere…
:::
```

可用于渲染类似于《理论篇 | remark 插件如何写？》讲到的 GitHub Alert 的功能。

当然这只是规范，是一种拓展 Markdown 语法的方式，具体的实现还需要依赖各种插件。

有一个库 [remark-directive](https://github.com/remarkjs/remark-directive) 可以帮助你识别这种语法。安装此依赖项：

```bash
npm install remark-directive
```

新建 `src/plugins/remark-directive-node.js`，代码如下：

```javascript
import { h } from "hastscript";
import { visit } from "unist-util-visit";

export default function remarkDirectiveNode() {
  return function (tree) {
    visit(tree, function (node) {
      if (node.type === "containerDirective" || node.type === "leafDirective" || node.type === "textDirective") {
        const data = node.data || (node.data = {});
        const hast = h(node.name, node.attributes || {});

        data.hName = hast.tagName;
        data.hProperties = hast.properties;
      }
    });
  };
}
```

修改 `astro.config.mjs`，添加代码如下：

```diff
+import remarkDirective from "remark-directive";
+import remarkDirectiveNode from "./src/plugins/remark-directive-node.js";

export default defineConfig({
  markdown: {
+    remarkPlugins: [
+      remarkDirective,
+      remarkDirectiveNode
+    ]
  },
});
```

此时如果 Markdown 中这样写：

```
:::main{#readme}

Lorem:br
ipsum.

::hr{.red}

A :i[lovely] language know as :abbr[HTML]{title="HyperText Markup Language"}.

:::
```

会被渲染成：

```html
<main id="readme">
  <p>Lorem<br />ipsum.</p>
  <hr class="red" />
  <p>A <i>lovely</i> language know as <abbr title="HyperText Markup Language">HTML</abbr>.</p>
</main>
```

当然这种简要的语法对于熟悉前端的同学并不算什么，但对于非技术同学却很有帮助。作者只需要阅读简单的使用指南，而不需要去了解 HTML 或 JavaScript。

### 4.2. remark-directive-youtube

不过到现在我们只是简单的渲染了指令，对于特殊的指令比如 `::youtube[title]{vid=09jf3ow9jfw}`却没有支持，我们来实现一个 youtube 指令，当我们使用的时候，它会被渲染成一个嵌入视频。

新建 `src/plugins/remark-directive-youtube.js`，代码如下：

```javascript
import { visit } from "unist-util-visit";

export default function remarkDirectiveYoutube() {
  return (tree, file) => {
    visit(tree, function (node) {
      if (node.type === "containerDirective" || node.type === "leafDirective" || node.type === "textDirective") {
        if (node.name !== "youtube") return;

        const data = node.data || (node.data = {});
        const attributes = node.attributes || {};
        const id = attributes.id;

        if (node.type === "textDirective") {
          file.fail("Unexpected `:youtube` text directive, use two colons for a leaf directive", node);
        }

        if (!id) {
          file.fail("Unexpected missing `id` on `youtube` directive", node);
        }

        data.hName = "iframe";
        data.hProperties = {
          src: "https://www.youtube.com/embed/" + id,
          width: 400,
          height: 200,
          frameBorder: 0,
          allow: "picture-in-picture",
          allowFullScreen: true,
        };
      }
    });
  };
}
```

修改 `astro.config.mjs`，添加代码如下：

```diff
import remarkDirective from "remark-directive";
import remarkDirectiveNode from "./src/plugins/remark-directive-node.js";
+import remarkDirectiveYoutube from "./src/plugins/remark-directive-youtube.js";

export default defineConfig({
  markdown: {
    remarkPlugins: [
      remarkDirective,
      remarkDirectiveNode,
+     remarkDirectiveYoutube
    ]
  },
});
```

### 4.3. rehype-component

现在如果我们在 Markdown 中这样写：

```
::github{repo="mqyqingfeng/blog"}
```

它会被渲染成自定义 HTML 元素：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/83caa2d4713e44da8097d9c83f5bc2a2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1158\&h=46\&s=18963\&e=png\&b=373737)

这是因为我们没有实现该语法对应的 `directive` 插件。理论上我们可以像上节的 remark-directive-youtube 一样写一个 remark 插件，但在这里，我们换一种实现方式，改为在 rehype 中实现。

正好有一个插件 [rehype-components](https://github.com/marekweb/rehype-components)，它可以将自定义 HTML 元素使用指定的元素进行替换。

```javascript
unified()
  // ...
  .use(rehypeComponents, {
    components: {
      github: GitHub,
    },
  });
// ...
```

在这段代码中，我们将自定义 HTML 元素 github 替换为 GitHub 组件，假设我们的 GitHub 组件代码为：

```javascript
const h = require("hastscript");

const GitHub = (properties, children) => h("a.github-card", [h("h1", properties.repo), ...children]);
```

现在我们的 `::github{repo="mqyqingfeng/blog"}`会在 remark 插件中被替换为 `<github repo="mqyqingfeng/blog"></github>`，在 rehype 插件中被替换为：

```html
<a class="github-card"><h1 id="mqyqingfengblog">mqyqingfeng/blog</h1></a>
```

### 4.4. rehype-component-github-card

当然上节只是举个例子，现在我们来完整的实现一下 `rehype-component-github-card`。

安装依赖项：

```bash
npm i rehype-components
```

新建 `src/plugins/rehype-component-github-card.js`，代码如下：

```javascript
import { h } from "hastscript";

export default function GithubCardComponent(properties, children) {
  if (Array.isArray(children) && children.length !== 0) return h("div", { class: "hidden" }, ['Invalid directive. ("github" directive must be leaf type "::github{repo="owner/repo"}")']);

  if (!properties.repo || !properties.repo.includes("/")) return h("div", { class: "hidden" }, 'Invalid repository. ("repo" attributte must be in the format "owner/repo")');

  const repo = properties.repo;
  const cardUuid = `GC${Math.random().toString(36).slice(-6)}`; // Collisions are not important

  const nAvatar = h(`div#${cardUuid}-avatar`, { class: "gc-avatar" });
  const nLanguage = h(`span#${cardUuid}-language`, { class: "gc-language" }, "Waiting...");

  const nTitle = h(`div`, { class: "gc-titlebar" }, [
    h("div", { class: "gc-titlebar-left" }, [
      h("div", { class: "gc-owner" }, [nAvatar, h("div", { class: "gc-user" }, repo.split("/")[0])]),
      h("div", { class: "gc-divider" }, "/"),
      h("div", { class: "gc-repo" }, repo.split("/")[1]),
    ]),
    h("div", { class: "github-logo" }),
  ]);

  const nDescription = h(`div#${cardUuid}-description`, { class: "gc-description" }, "Waiting for api.github.com...");

  const nStars = h(`div#${cardUuid}-stars`, { class: "gc-stars" }, "00K");
  const nForks = h(`div#${cardUuid}-forks`, { class: "gc-forks" }, "0K");
  const nLicense = h(`div#${cardUuid}-license`, { class: "gc-license" }, "0K");

  const nScript = h(
    `script#${cardUuid}-script`,
    { type: "text/javascript", defer: true },
    `
      fetch('https://api.github.com/repos/${repo}', { referrerPolicy: "no-referrer" }).then(response => response.json()).then(data => {
        if (data.description) {
          document.getElementById('${cardUuid}-description').innerText = data.description.replace(/:[a-zA-Z0-9_]+:/g, '');
        } else {
          document.getElementById('${cardUuid}-description').innerText = "Description not set"
        }
        document.getElementById('${cardUuid}-language').innerText = data.language;
        document.getElementById('${cardUuid}-forks').innerText = Intl.NumberFormat('en-us', { notation: "compact", maximumFractionDigits: 1 }).format(data.forks).replaceAll("\u202f", '');
        document.getElementById('${cardUuid}-stars').innerText = Intl.NumberFormat('en-us', { notation: "compact", maximumFractionDigits: 1 }).format(data.stargazers_count).replaceAll("\u202f", '');
        const avatarEl = document.getElementById('${cardUuid}-avatar');
        avatarEl.style.backgroundImage = 'url(' + data.owner.avatar_url + ')';
        avatarEl.style.backgroundColor = 'transparent';
        if (data.license?.spdx_id) {
          document.getElementById('${cardUuid}-license').innerText = data.license?.spdx_id
        } else {
          document.getElementById('${cardUuid}-license').classList.add = "no-license"
        };
          document.getElementById('${cardUuid}-card').classList.remove("fetch-waiting");
          console.log("[GITHUB-CARD] Loaded card for ${repo} | ${cardUuid}.")
      }).catch(err => {
        const c = document.getElementById('${cardUuid}-card');
        c.classList.add("fetch-error");
         console.warn("[GITHUB-CARD] (Error) Loading card for ${repo} | ${cardUuid}.")
      })
    `
  );

  return h(
    `a#${cardUuid}-card`,
    {
      class: "card-github fetch-waiting no-styling",
      href: `https://github.com/${repo}`,
      target: "_blank",
      repo,
    },
    [nTitle, nDescription, h("div", { class: "gc-infobar" }, [nStars, nForks, nLicense, nLanguage]), nScript]
  );
}
```

这段代码看起来有些复杂，其实主要就是通过 `hastscript`的 `h`函数不断构建节点。稍微复杂的地方就是写了一个 `script`节点，通过 GitHub 的 API 获取了仓库对应的信息。

修改 `astro.config.mjs`，添加代码如下：

