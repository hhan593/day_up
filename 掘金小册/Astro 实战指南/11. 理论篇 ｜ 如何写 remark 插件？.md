## 1. 前言

根据 Astro 官网的介绍：

> Astro 中的 Markdown 支持由 [remark](https://remark.js.org/) 提供，remark 是一个强大的解析和处理工具，拥有一个活跃的生态系统。目前不支持其他 Markdown 解析器，如 Pandoc 和 markdown-it。
>
> Astro 支持添加第三方 [remark](https://github.com/remarkjs/remark) 和 [rehype](https://github.com/rehypejs/rehype) 插件来解析 Markdown。这些插件允许你扩展你的 Markdown，以获得新的功能。



总结一下：Remark、Markdown-it、Pandoc 都是知名的 Markdown 解析器，Astro 基于的是 Remark。



Remark 的创建者是 Titus Wormer（[wooorm](https://github.com/wooorm)），同时也是 [unified](https://unifiedjs.com/) 生态的创建者。[unified](https://github.com/unifiedjs/unified) 是一个通过语法树来解析、检查、转化，以及序列化内容的接口，本身并不起任何作用，需要配置插件才能工作，比如：



```javascript
const processor = unified()
    .use(remarkParse)
    .use(remarkRehype)
    .use(rehypeDocument, { title: "👋🌍" })
    .use(rehypeFormat)
    .use(rehypeStringify)
    .process("# Hello world!");
```

此时输出为：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>👋🌍</title>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
  </head>
  <body>
    <h1>Hello world!</h1>
  </body>
</html>
```

在 unified 体系下，多个生态系统围绕不同类型的内容统一构建，比如 remark 生态用来处理 Markdown 类型的内容，rehype 生态用来处理 HTML 类型的内容，retext 生态用来处理自然语言类型的内容。这些生态因为统一构建，所以可以连接在一起使用。


对于 Markdown 而言，因为我们要编译成 HTML，所以编译大致分为两步，一步处理 Markdown，一步处理 HTML。处理的伪代码如下：


```javascript
import { unified } from "unified";
import remarkParse from "remark-parse";
import remarkRehype from "remark-rehype";
import rehypeSanitize from "rehype-sanitize";
import rehypeStringify from "rehype-stringify";

main();

async function main() {
  const file = await unified()
    .use(remarkParse) // 将 markdown 转换为 markdown AST
    .use(remarkRehype) // 转换为 HTML AST
    .use(rehypeSanitize) // HTML 消毒，处理不安全的内容，防止 XSS 攻击
    .use(rehypeStringify) // 将 AST 转换为 HTML
    .process("Hello, Astro!");

  console.log(String(file)); // <p>Hello, Astro!</p>
}
```

所以如果我们要自定义处理 Markdown，就需要写 Remark 插件，如果要自定义处理 HTML，就需要写 Rehype 插件。

本篇我们来聊聊如何写 Remark 插件。

## 3. 实战 1：添加文章阅读时长

“文章阅读时长”这个功能想必大家都见过，在文章页面有个预估阅读所需时间，小册目录页面也有学习时长：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6ea3a9ff0d3941a587d679b77d6e0e93~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1300\&h=176\&s=29648\&e=png\&b=ffffff)

如果我们要实现这个功能该如何实现呢？

首先我们需要安装 2 个库：

1.  [reading-time](https://www.npmjs.com/package/reading-time)：用于计算阅读分钟数
2.  [mdast-util-to-string](https://www.npmjs.com/package/mdast-util-to-string)：用于从 Markdown 中提取所有文本

```bash
npm install reading-time mdast-util-to-string
```

新建 `src/plugins/remark-reading-time.js`：

```javascript
import getReadingTime from "reading-time";
import { toString } from "mdast-util-to-string";

export function remarkReadingTime() {
  return function (tree, { data }) {
    const textOnPage = toString(tree);
    const readingTime = getReadingTime(textOnPage);
    data.astro.frontmatter.minutesRead = Math.ceil(readingTime.minutes);
  };
}
```

在这段代码中，我们计算了文章阅读所需时长，然后将其写入了 frontmatter 中。

修改 `astro.config.mjs`，添加代码如下：

```diff
import { defineConfig } from 'astro/config';
+import { remarkReadingTime } from './remark-reading-time.js';

export default defineConfig({
  markdown: {
+    remarkPlugins: [remarkReadingTime],
  },
});
```

此时我们就完成了插件的配置。

如果我们要使用文章阅读时长属性，也就是 `minutesRead`，要看我们渲染的方式。

如果我们用的是 `entry.render()`渲染的 Markdown 文件，就从 `await entry/render()`中获取修改后的 `remarkPluginFrontmatter`对象，从中获取 `minutesRead`：

```jsx
---
import { CollectionEntry, getCollection } from 'astro:content';

export async function getStaticPaths() {
  const blog = await getCollection('blog');
  return blog.map(entry => ({
    params: { slug: entry.slug },
    props: { entry },
  }));
}

const { entry } = Astro.props;
const { Content, remarkPluginFrontmatter } = await entry.render();
---

<html>
  <head>...</head>
  <body>
    ...
    <p>{remarkPluginFrontmatter.minutesRead}</p>
    ...
  </body>
</html>
```

如果我们直接用的就是 Markdown 文件，可以添加到布局中：

```jsx
---
const { minutesRead } = Astro.props.frontmatter;
---

<html>
  <head>...</head>
  <body>
    <p>{minutesRead}</p>
    <slot />
  </body>

```

## 4. 实战 2：添加文章最后修改时间

添加文章最后修改时间也是同样的原理。

1.  [dayjs](https://day.js.org/zh-CN/)：时间处理库

新建 `src/plugins/remark-modified-time.js`，代码如下：

```javascript
import { execSync } from "child_process";

export function remarkModifiedTime() {
  return function (tree, file) {
    const filepath = file.history[0];
    const result = execSync(`git log -1 --pretty="format:%cI" "${filepath}"`);
    file.data.astro.frontmatter.lastModified = result.toString();
  };
}
```

```diff
import { defineConfig } from 'astro/config';
+import { remarkModifiedTime } from './remark-modified-time.js';

export default defineConfig({
  markdown: {
+    remarkPlugins: [remarkModifiedTime],
  },
});
```

此时我们就完成了插件的配置。

## 5. 实战 3：自动生成摘要

比如我们生成文章列表的时候，就需要获取文章的摘要信息：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5fc8643e8e5e4780af8066612f1aab79~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1772\&h=250\&s=88537\&e=png\&b=ffffff)

新建 `src/plugins/remark-excerpt.js`，代码如下：

```javascript
import { toString } from "mdast-util-to-string";

/* 使用文章的第一个段落作为摘要 */
export function remarkExcerpt() {
  return (tree, { data }) => {
    let excerpt = "";
    for (let node of tree.children) {
      if (node.type !== "paragraph") {
        continue;
      }
      excerpt = toString(node);
      break;
    }
    data.astro.frontmatter.excerpt = excerpt;
  };
}
```

在这段代码中，我们遍历 `tree.children`，获取第一个类型为 `paragraph`的节点内容，也就是第一个段落作为摘要。

至于配置和显示都和前 2 个实战一致，但与前两个不同的是，我们开始处理了 tree 节点。

## 6. 实战 4：替换字符串

现在我想实现这样一个功能，考虑到一些同学不知道怎么念我的名字，当文本中出现“冴羽”的时候，修改为“冴（ya）羽（yu）”，并将其改成一个链接，链接到我的博客主页 <https://yayujs.com>。

我们需要 2 个依赖项：

1.  [mdast-util-find-and-replace](https://unifiedjs.com/explore/package/mdast-util-find-and-replace/)：查找和替换节点
2.  [unist-builder](https://github.com/syntax-tree/unist-builder#readme)：快速构建节点

```bash
npm i mdast-util-find-and-replace unist-builder
```

新建 `src/plugins/remark-link-yayu.js`，代码如下：

```javascript
import { findAndReplace } from "mdast-util-find-and-replace";

export function remarkLinkYaYu() {
  return (tree, { data }) => {
    findAndReplace(tree, [[/冴羽/gi, "冴（ya）羽（yu）"]]);
  };
}
```

在这段代码中，我们使用了 [mdast-util-find-and-replace](https://unifiedjs.com/explore/package/mdast-util-find-and-replace/)，它的作用就是查找和替换节点，我们使用在正则表达式，将“冴羽”替换为了“冴（ya）羽（yu）”。

浏览器效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/23e72af82f724122a490562fec081b9f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1440\&h=552\&s=175339\&e=png\&b=2c2c2c)

那如何修改成链接呢？我们可以使用 [unist-builder](https://github.com/syntax-tree/unist-builder#readme) 构建一个链接节点，然后将其替换为我们构建的节点：

```javascript
import { findAndReplace } from "mdast-util-find-and-replace";
import { u } from "unist-builder";

export function remarkLinkYaYu() {
  return (tree, { data }) => {
    findAndReplace(tree, [
      [
        /冴羽/g,
        function ($0) {
          return u("link", { url: "https://yayujs.com" }, [u("text", `冴（ya）羽（yu）`)]);
        },
      ],
    ]);
  };
}
```

## 7. 实战 5：GitHub Alert 新语法

在使用 GitHub 写 Markdown 的时候，有一个 GitHub Alert 语法，比如如果你这样写：

```
> [!NOTE]
> Highlights information that users should take into account, even when skimming.
```

浏览器效果为：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/22cf22123ae84f36876cc497e7961584~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1722\&h=218\&s=31000\&e=png\&b=ffffff)

我们来手写下这个效果。

安装依赖项：

1.  [unist-util-visit](https://unifiedjs.com/explore/package/unist-util-visit/)：用于快速访问特定类型的节点

```bash
npm install unist-util-visit
```

新建 `src/plugins/remark-note.js`，代码如下：

```javascript
import { visit } from "unist-util-visit";

const icon =
  '<svg class="octicon" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path d="M0 8a8 8 0 1 1 16 0A8 8 0 0 1 0 8Zm8-6.5a6.5 6.5 0 1 0 0 13 6.5 6.5 0 0 0 0-13ZM6.5 7.75A.75.75 0 0 1 7.25 7h1a.75.75 0 0 1 .75.75v2.75h.25a.75.75 0 0 1 0 1.5h-2a.75.75 0 0 1 0-1.5h.25v-2h-.25a.75.75 0 0 1-.75-.75ZM8 6a1 1 0 1 1 0-2 1 1 0 0 1 0 2Z"></path></svg>';

export function remarkNote(options = {}) {
  const { classPrefix = "remark-note" } = options;

  const RE = new RegExp(`^\\[\\!(NOTE)\\]([^\\n\\r]*)`, "i");

  return (tree, { data }) => {
    visit(tree, "blockquote", (node) => {
      const childNode = node.children.at(0);
      if (childNode?.type !== "paragraph") return;

      const content = childNode.children.at(0);
      if (content?.type !== "text") return;

      const match = content.value.match(RE);

      if (!match) return;

      const type = match[1].toLowerCase();
      const title = match[2].trim() || type;

      const matchText = match[0];
      content.value = content.value.slice(matchText.length).trimStart();

      node.data = {
        hName: "div",
        hProperties: {
          class: `${classPrefix} ${classPrefix}-${type}`,
        },
      };

      node.children = [
        {
          type: "paragraph",
          data: {
            hName: "p",
            hProperties: {
              class: `${classPrefix}-title`,
            },
          },
          children: [
            { type: "html", value: icon },
            { type: "text", value: title },
          ],
        },
        ...node.children,
      ];
    });

    return tree;
  };
}
```

在这段代码中，我们使用 unist-util-visit 这个库访问了引用类型的节点，然后进行正则匹配。节点的数据类型如下：

```typescript
interface Content {
  type: string;
  children: BlockContent[];
  value?: string;
  data?: {
    hName?: string;
    hProperties?: Record<string, string | number | boolean>;
  };
}
```

其中 `data.hName`用于设置对应的 HTML 标签名，`data.hProperties`用于设置对应的 HTML 属性。我们正是为节点添加了 `hName`和 `hProperties`属性，才为 GitHub Alert 这个新语法设置了元素和属性。

修改 `astro.config.mjs`，添加代码如下：

```diff
+import { remarkNote } from "./src/plugins/remark-note.js";

export default defineConfig({
  markdown: {
    remarkPlugins: [
+      remarkNote,
    ],
  },
});
```
