描边和填充在 SVG 图形中扮演着至关重要的角色。描边用于绘制图形的边缘轮廓，使得图形在视觉上更加清晰和突出，而填充则决定了图形内部的颜色或图案，为图形赋予了实际的视觉表现。它们是 SVG 图形中最重要的属性，直接影响着图形的外观和样式。

  


SVG 的描边和填充涵盖了多个属性：

  

![Frame 9195.png](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0d606ba89dad41a384a9924e5d6a3dbb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=481&s=87782&e=png&b=ffffff)



在 SVG 中，用于控制图形填充的属性主要有 `fill` 、`fill-rule` 和 `fill-opacity` ：

  


-   **`fill`** **（填充）** ：`fill` 属性用于设置图形的填充颜色（可以是纯色，也可以是渐变色）或图案。它可以接受各种颜色值，包括预定义颜色名称、十六进制 RGB 值、RGB 函数、HSL 函数等。除了颜色外，还可以填充图案、渐变等，从而实现丰富的视觉效果。
-   **`fill-rule`** **（填充规则）** ：`fill-rule` 属性用于确定在填充封闭图形时应如何确定填充区域。默认情况下，`fill-rule` 属性值为 `nonzero`，表示使用非零绕线规则进行填充。这意味着如果路径交叉，则按照交叉路径的数量来确定填充区域。另一个可能的值是 `evenodd`，表示使用奇偶规则进行填充，即通过计算路径与水平射线的交点数来确定填充区域。
-   **`fill-opacity`** **（填充不透明度）** ：`fill-opacity` 属性用于设置图形填充的不透明度，即填充的透明度程度。它的取值范围为 `0` 到 `1`，其中 `0` 表示完全透明，`1` 表示完全不透明。通过调整 `fill-opacity` 属性，可以实现图形填充的透明效果，从而使底层元素透过填充显示出来。

  


这些属性结合起来，可以对 SVG 图形进行丰富多彩的填充效果的控制，从而实现各种视觉上的效果和样式。

  


SVG 中还提供了一系列用于控制图形的描边效果，其包括：

  


-   **`stroke`** **（描边）** ：`stroke` 属性用于设置图形的描边颜色或图案。它可以接受各种颜色值，包括预定义颜色名称、十六进制 RGB 值、RGB 函数、HSL 函数等。类似于 `fill` 属性，`stroke` 属性也可以用来填充图案、渐变等。
-   **`stroke-opacity`** **（描边不透明度）** ：`stroke-opacity` 属性用于设置图形描边的不透明度，即描边的透明度程度。取值范围为 `0` 到 `1`，其中 `0` 表示完全透明，`1` 表示完全不透明。
-   **`stroke-width`** **（描边宽度）** ：`stroke-width` 属性用于设置图形描边的宽度。它可以接受各种长度单位，如像素（`px`）、百分比（`%`）等。通过调整 `stroke-width` 属性，可以改变描边的粗细程度。
-   **`stroke-linecap`** **（描边端点形状）** ：`stroke-linecap` 属性用于设置图形描边的端点形状。它有三个可能的取值：`butt`（默认值，平直端点）、`round`（圆形端点）和 `square`（方形端点）。
-   **`stroke-linejoin`** **（描边连接方式）** ：`stroke-linejoin` 属性用于设置图形描边的连接方式，即两条线段相交处的形状。它的取值可以是 `miter`（默认值）、`miter-clip` 、`round` 、`bevel` 和 `arcs` 等。
-   **`stroke-miterlimit`** **（尖角连接限制）** ：`stroke-miterlimit` 属性用于设置尖角连接的限制，即控制尖角连接处尖角的最大长度。当角度超过指定长度时，尖角将被截断为平直线段。
-   **`stroke-dasharray`** **（描边虚线样式）** ：`stroke-dasharray` 属性用于创建虚线效果的描边样式。它接受一组空格分隔的数值，用来指定虚线和间隔的长度。可以通过调整这些数值，创建不同样式的虚线效果。
-   **`stroke-dashoffset`** **（虚线偏移量）** ：`stroke-dashoffset` 属性用于设置虚线的起始偏移量。它指定了虚线模式的起始位置，从而影响虚线的显示效果。

  


这些属性结合起来，可以实现各种不同样式的描边效果，从简单的实线到复杂的虚线，满足各种设计需求。

## 如何使用 SVG 的描边属性

  


在创建 Web 应用程序、页面或动画中，使用 SVG 描边属性可以实现各种视觉效果，从简单的线条到复杂的装饰效果，都能够轻松实现。通过调整描边的颜色、宽度、线帽和连接方式，你可以打造出各种独特的描边效果，使图形更加生动和引人注目。

  


SVG 为基本图形元素提供了很多个与描边有关的属性，它们可以分为基本属性、线条样式和虚线样式：

  


-   基本属性：`stroke` （设置描边的颜色）、`stroke-width` （设置描边的宽度）、`stroke-opacity` （设置描边的透明度）
-   线条样式：`stroke-linecap` （定义线帽样式）、`stroke-linejoin` （定义线条的连接样式）、`stroke-miterlimit` （定义尖角连接的限制）
-   虚线样式：`stroke-dasharray` （定义虚线的样式）、`stroke-dashoffset` （定义虚线的偏移量）

  


其中 `stroke` 和 `stroke-opacity` 可以控制描边的颜色、`stroke-width` 、`stroke-linecap` 、`stroke-linejoin` 、`stroke-miterlimit` 、`stroke-dasharray` 和 `stroke-dashoffset` 主要用于控制描边线条的样式。

  


这些属性可以单独或结合使用，以实现各种复杂的描边效果。通过调整这些属性的数值和组合方式，可以创造出各种独特的线条样式，从简单的实线到复杂的装饰线条，都能轻松实现。

  


接下来，我们借助真实案例来探讨 SVG 的描边属性应该如何使用，以及它们会带来哪些效果？首先，我们使用 `<circle>` 元素在 SVG 视口中绘制了一个圆形：

  


-   通过 `cx` 和 `cy` 属性将圆心定位在 `x` 轴和 `y` 轴上的 `400` 个单位处，即圆心的位置是 `(400,400)` ，
-   半径（`r`）为 `200` 个单位，控制圆的大小
-   描边宽度（`stroke-width`）为 `40` 个单位
-   描边颜色（`stroke`）为 `#e387ff` ，并且将 `fill` 属性设置为 `none`

  


如此，我们就绘制了一个空心圆：

  


```XML
<svg viewBox="0 0 800 800" class="loader">
    <circle cx="400" cy="400" fill="none" r="225" stroke-width="50" stroke="#E387FF" />
</svg>
```

### 设置描边颜色：stroke

  


在 SVG 中，我们可以通过 `stroke` 属性来设置图形元素边框的颜色，它看上去有点类似于 CSS 中的 `border-color` 。它的值可以是 `<color>` 值类型中的任何一种，例如

  


-   颜色关键词 `red`
-   十六进制颜色 `#E387FF`
-   颜色函数值：`rgb() 、hsl() 、lch() 、lab()` ，甚至是最新的颜色函数，例如 `oklch() 、oklab()` 以及 `color() 和 color-mix()` 等

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3059acff24a4835a56cde56caf19340~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=980&h=1176&s=306420&e=png&b=fefefe)

  


可以说，你熟悉的和不熟悉的 CSS 颜色值都可以用于 SVG 的 `stroke` 属性。

  


使用 `stroke` 属性除了可以给图形元素设置一个纯色边框效果之外，还可以结合 SVG 的 `<linearGradient>` （线性渐变）和 `<radialGradient>` （径向渐变）给图形元素设置带有渐变的描边效果：

  


```XML
<svg viewBox="-400 -400 800 800" class="loader">
    <defs>
        <!-- 定义一个线性渐变 -->
        <linearGradient id="linearGradient" x2="1" y2="1">
          <stop offset="0%" stop-color="#a770ef" />
          <stop offset="50%" stop-color="#cf8bf3" />
          <stop offset="100%" stop-color="#fdb99b" />
        </linearGradient>
        
        <!-- 定义一个径向渐变 -->
        <radialGradient id="radialGradient" cy="60%" fx="45%" fy="45%" r="2">
          <stop offset="0%" stop-color="#48afc1" />
          <stop offset="10%" stop-color="#b4c63b" />
          <stop offset="20%" stop-color="#ef5b2b" />
          <stop offset="30%" stop-color="#503969" />
          <stop offset="40%" stop-color="#ab6294" />
          <stop offset="50%" stop-color="#1cb98f" />
          <stop offset="60%" stop-color="#48afc1" />
          <stop offset="70%" stop-color="#b4c63b" />
          <stop offset="80%" stop-color="#ef5b2b" />
          <stop offset="90%" stop-color="#503969" />
          <stop offset="100%" stop-color="#ab6294" />
        </radialGradient>
    </defs>
    <circle cx="-150" cy="0" fill="none" r="225" stroke-width="50" stroke="url(#linearGradient)" />
    <circle cx="150" cy="0" fill="none" r="225" stroke-width="50" stroke="url(#radialGradient)" />
</svg>
```

  


正如上面代码所示，我们需要给 `<linearGradient>` 和 `<radialGradient>` 绘制的渐变指定一个 `ID` 标识符，然后在 `stroke` 属性上使用 `url(#ID)` 的方式来引用已定义的渐变：

### 设置描边宽度：stroke-width

  


SVG 元素的 `stroke-width` 属性有点类似于 CSS 的 `border-width` ，可以用于设置图形描边的宽度。它指定了描边的粗细。你可以尝试着拖动下面示例中 `stroke-width` 的滑块，查看它设置不同值时，圆形描边的宽度变化：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/330fb3baa11d436d85340c7c9f6eaa31~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1150&h=584&s=1183338&e=gif&f=170&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/KKYmENE

  


以下是关于 `stroke-width` 属性的一些重要点：

  


-   语法：`stroke-width` 属性的值可以是一个整数值，例如 `stroke-width="20"` ，也可以是一个小数值，例如 `stroke-width="0.2"` 。注意，负值是一个无效值。其值可以显式设置长度单位，也可以不设置单位。
-   默认值：`stroke-width` 属性的默认值是 `1` ，表示一个用户单位
-   继承性：`stroke-width` 属性不会被父元素的 `stroke-width` 影响，它是不可继承的
-   效果：增加 `stroke-width` 值为使描边变得更粗，而减小值则会使描边变得更细。如果将值设置为 `0` ，则描边将不可见

  


注意，`stroke-width` 通常与 `stroke` 属性一起使用，以控制图形描边的颜色和宽度。下面任何一种情形，图形描边效果都不可见：

  


-   `stroke` 属性显式设置为 `none` 时，图形描边不可见
-   `stroke-width` 属性显式设置为 `0` 时，图形描边同样不可见

  


### 设置描边的不透明度：stroke-opacity

  


通常情况之下，`stroke-opacity` 与 `stroke` 一起使用，用于控制描边的外观和透明度。它定义了描边的透明度级别，允许 Web 开发者控制描边的可见度，其值范围从 `0` （完全透明）到 `1` （完全不透明）。具体而言：

  


-   `stroke-opacity` 的值为 `0` 时，表示描边完全透明，即描边不可见
-   `stroke-opacity` 的值为 `1` 时，表示描边完全不透明，即描边完全可见
-   `0 ~ 1` 之间的值表示介于完全透明和完全不透明之间的透明度级别

  


`stroke-opacity` 属性有点类似于 CSS 的 `opacity` 属性，它的值类型可以是一个数值（ `<number>` ），即 `0 ~ 1` ，也可以是一个百分比值（`<percentage>`），即 `0% ~ 100%` ，其中 `0%` 相当于 `0` ，`100%` 相当于 `1` 。

  


我们在上面的示例的 `<circle>` 元素上添加一个 `stroke-opacity` ：

  


```XML
<svg viewBox="0 0 800 800" class="loader">
    <circle stroke-opacity=".5" stroke-width="50" stroke="#E387FF"cx="400" cy="400" fill="none" r="225"  />
</svg>
```

### 设置描边样式：stroke-dasharray 和 stroke-dashoffset

  


非常不幸的是，在 SVG 中并没有一个类似于 `border-style` 的属性可以给图形元素设置描边的样式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a53e2036426447efb30722a741970275~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3582&h=2188&s=304029&e=jpg&b=fdfdfd)

  


如果你使用过诸如 Figma 这样的图形设计软件，那么你可能也发现了，我们在使用 Figma 图形设计软件给图形添加描边样式（Stroke Style）时，只有两种选择，一种是实线（Solid），另一种是虚线（Dash），如果你选择将图形描边设置为虚线（Dash）模式时，那么你可以通过 Dash 和 Gap 两个参数来定义符合自己需求的虚线效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8bdcdbd5c7b64fd3bca6e916a74b5f5c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1172&h=662&s=1253304&e=gif&f=331&b=f5f5f5)

  


庆幸的是，SVG 中为图形元素描边样式的定义有点类似于 Figma 中的 Stroke Style（描边样式），提供了 `stroke-dasharray` 和 `stroke-dashoffset` 两个属性，允许 Web 开发者使用他们创建出各种复杂的虚线效果，为 SVG 图形增添视觉吸引力和美感。

  


正如前面的示例所呈现的那样，默认情况下，SVG 图形元素的描边都是以实边的形式绘制。只有当你显式的在图形元素上设置了 `stroke-dasharray` ，图形元素的描边才会以虚线的形式绘制。SVG 的 `stroke-dasharray` 属性主要用于指定虚线的长度（Dash）和间隔（Gap）。它定义了一系列用于空格分隔的数值，每个数值表示一个短划线的长度，后跟一个空格表示间隔长度。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df98442dcd65466c9b958fc33b7a6901~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=917&s=407207&e=jpg&b=fbfafa)

  


通过在 `stroke-dasharray` 中设置不同的数值，可以创建各种类型的虚线效果，例如间断虚线、点状线和复杂的虚线模式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0baa5c512e0b44179f0bb6da27ae7711~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1339&h=968&s=657672&e=gif&f=548&b=3e423c)

  


如上图所示：

  


-   当 `stroke-dasharray` 只设置了一个值，如 `40` ，那么虚线长度（Dash）和间隔是相等的
-   当 `stroke-dasharray` 设置了两个值，如 `40 20` （两者之间一般使用空格进行分隔），那么第一个值（`40`）应用于虚线长度（Dash） ；第二个值（`20`）应用于间隔（Gap）
-   当 `stroke-dasharray` 设置了三个值，如 `40 20 80` ，那么第一段虚线的长度是 `40` ，第一个间隔是 `20` ，第二段虚线的长度是 `80` ，第二个间隔是 `40` ，第三段虚线的长度是 `20` ，第三个间隔是 `40` ，依此类推
-   当 `stroke-dasharray` 设置了四个值，如 `40 20 80 120` ，那么第一个值（`40`）应用于第一段虚线的长度，第二个值（`20` ）应用于第一个间隔，第三个值（`80`）应用于第二段虚线长度，第四个值（`120`）应用于第二个间隔，依此类推

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df60d14cc7c6435f93bb46846a3e80e5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1034&h=568&s=4064444&e=gif&f=429&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/dyLWLJQ

  


虚线和间隔的模式（`stroke-dasharray`）重复填充整个形状的描边长度。如上图所示，这个形状是一个圆，那么整个描边长度是圆的周长（`2πr`），如果这个形状是一个 `<path>` 元素，那么整个描边长度就是 `pathLength` 。

  


继续回到 `<circle>` 的示例中来，我们把 `stroke-dasharray` 的间隔设置为一个较大的值，确保整个圆只有一段虚线会显示出来。当虚线长度为 `0` 时，我们看不到任何描边效果（间隔占据了整个可见的描边），而当虚线的长度为 `1414` 时（整个形状的描边长度，在我们这个示例中，圆半径为 `r = 225`，整个长度为圆的周长，即 `2πr` ，因此得到约 `1414` 的值），我们将再次得到一个完整的圆：

  


```XML
<svg viewBox="0 0 800 800" class="loader">
    <circle cx="400" cy="400" fill="none"
    r="225" stroke-width="50" stroke="#E387FF" stroke-dasharray="10 1414"/>
</svg>
```

你可能已经发现了，上面所展示的案例，虚线的起始位置都是从路径的起点开始。如果你想调整虚线的起始位置，就得使用 `stroke-dashoffset` 属性。该属性主要用于指定路径描边的起始位置偏移量。它控制了路径描边中虚线模式的起始位置，从而实现了路径描边的动画效果或者特殊样式效果。

```XML
<svg viewBox="0 0 800 800" class="loader">
```

### 设置线帽样式：stroke-linecap

  


你可能已经注意到了，同样是以 `<circle>` 元素描边来制作圆形进度条，为什么进度条（圆形描边）两端是圆的，有的时候却又是平的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/54780237074d4bd59b83206ba056deee~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=716&s=194901&e=jpg&b=050505)

  


就圆形进度条的效果而言，进度条两端是圆的效果更令人感到舒适，对吧！

  


在 SVG 中，要做到这一点并不难。因为 SVG 的 `stroke-linecap` 就是用来做这件事情的。`stroke-linecap` 属性定义了开放子路径（即没有闭合的路径段）的端点形状，以及长度为零的子路径（无论是开放的还是闭合的）的绘制方式。它有三个值：`butt`、`round` 和 `square`，它们之间的区别如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b2c060bb518e41c1b74af58355aab644~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=716&s=100884&e=jpg&b=ffffff)

  


-   `butt`：对于每个子路径的端点，描边不会延伸到端点之外。如果子路径的长度为零，那么不会有任何描边效果
-   `round`：对于每个子路径的端点，描边的形状会延伸一个半圆，直径等于描边宽度的一半。如果子路径的长度为零，描边将完全由一个以子路径点为中心的完整圆组成。
-   `square`：对于每个子路径的端点，描边的形状会延伸一个矩形，宽度等于描边宽度，长度为描边宽度的一半。如果子路径的长度为零，描边将完全由一个以子路径点为中心的正方形组成，边长等于描边宽度，并且方向使得其中两条边与该子路径点处的有效切线平行。

  


这些不同的值决定了描边的端点形状，从而影响了路径的视觉效果。

  


```XML
<svg viewBox="0 0 800 800" class="loader">
    <circle cx="400" cy="400" fill="none"
    r="225" stroke-width="50" stroke="#E387FF"  stroke-dasharray="500 1414" stroke-dashoffset="0" stroke-linecap="var(--stroke-linecap, butt)" />
</svg>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/76e0eca6a49f4dbb9cb0adb35b92fa2e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1150&h=548&s=990527&e=gif&f=234&b=040404)

### 定义线条连接方式：stroke-linejoin

  


在 SVG 中，我们时常会[使用 `<polyline>` 和 `<polygon>` 元素来绘制折线和多边形](https://juejin.cn/book/7341630791099383835/section/7345813971552698406)，它们绘制的图形总会有线条连接处。以 `<polygon>` 为例，下面的代码将会绘制一个类似星星的形状：

  


```XML
<svg viewBox='-450 -450 900 900' class="star">
    <polygon points='250,0 20,20 0,250 -20,20 -250,0 -20,-20 0,-250 20,-20' fill="none" stroke="rgb(195 230 15)" stroke-width="20" />
</svg>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a0a383c704774afaa2ba1e7a41c816ec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=839&s=150866&e=jpg&b=050505)

  


通过前面的学习，我们知道 `stroke-linecap` 属性可以为路径两端设置样式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/97efd37e5b9b438780afd3483df011f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1510&s=369856&e=jpg&b=fefefe)

  


与之相似，如果你要为线段连接处设置样式，则需要使用 `stroke-linejoin` 属性。它主要用于指定在路径或基本形状的角点处（两线连接处）使用的连接方式（形状）。它影响了路径或基本形状中角点的外观（两条线在一个点相交时角的外观），尤其是在路径或基本形状由直线段组成并且这些直线相交形成锐角时。

  


`stroke-linejoin` 可接受的值有五个：

  


-   `miter` ：表示路径段的连接处是尖角。这意味着路径段的外边缘会延伸，直到它们相交形成锐角。如果连接处的尖角长度超过了指定的 `stroke-miterlimit` ，则连接会被转换为 `bevel` （斜角）
-   `miter-clip` ：表示路径段的连接处是尖锐的角。这个角是通过延伸路径路的切线上的外边缘直到它们相交而形成的。如果超过了 `stroke-miterlimit` ，则斜切将被裁剪为等于 `stroke-miterlimit` 值乘以路径段相交处的描边宽度（`stroke-width`）的一半的距离。在非常尖锐的连接或动画的情况下，这提供了比 `miter` 更好的渲染效果。
-   `round` ：表示路径段的连接处是圆角。这意味着在角点处会有一个圆形区域
-   `bevel` ：表示路径段的连接处是斜角，类似于 `miter` ，但顶点被截去
-   `arcs` ：表示路径段的连接处是弧形。弧形形状是通过延伸在连接点的外边缘，并具有与连接点处外边缘相同曲率的弧形而形成的

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2cc9549af7694ad2b492eba4bb6f0aa7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1510&s=394365&e=jpg&b=fefefe)

  


```XML
<svg viewBox='-450 -450 900 900' class="star">
    <polygon points='250,0 20,20 0,250 -20,20 -250,0 -20,-20 0,-250 20,-20' fill="none" stroke="rgb(195 230 15)" stroke-width="20" stroke-linejoin="var(--stroke-linejoin, miter)" />
</svg>
```

### 设置尖角连接的限制：stroke-miterlimit

  


`stroke-miterlimit` 属性顾名思义，它是对 `miter` 的延伸程度施加了限制。当超出此限制时，连接角简单地从 `miter` 转换为 `bevel` 。这意味着，`stroke-miterlimit` 取决于 `stroke-linejoin` 。当 `stroke-linejoin` 属性的值是 `round` 或 `bevel` 时，则不需要声明 `stroke-miterlimit` 。

  


简单地说，当两个线段以尖角相交时，系统会计算尖角的长度（`miterLength`）。如果尖角长度超过 `stroke-miterlimit` 属性指定的阈值，则会触发尖角截断机制。这时，尖角的长度将被限制在一个合理的范围内，以避免过度突出或不良显示的情况。当尖角长度超过 `stroke-miterlimit` 指定的阈值时，系统会将尖角截断并转换为斜角。这样可以确保尖角不会无限延伸，同时保持图形的整体外观。截断后的形状将呈现为一个截断的尖角或斜角，而不是一个过度突出的尖角。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cbfc78579f2646bd9458d0ada18b96c7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1510&s=481358&e=jpg&b=ffffff)

  


注意：当 `stroke-linejoin` 的值是 `miter-clip` ，它将使连接点在限制值处被剪裁，而不是成为斜角。

  


下面的演示显示，两个线段之间的角度越小，如果设置的 `stroke-miterlimit` 足够高，使其永远不会转换为斜角，那么斜接将延伸得越多。它还显示了随着 `stroke-miterlimit` 值的增加，斜接转换为斜角的角度减小的情况：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/194a9bdd148e46748a2f3dcb22ad30d2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1118&h=610&s=3458211&e=gif&f=481&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/PogjZvq

  


上面的演示中，`stroke-miterlimit` 的值范围从 `2 ~ 15` ，而实际尖角长度（`miterLength`）超过 `50` ，并且可以达到几百。尖角长度还取决于 `stroke-width` ，因此很明显，施加的限制不是在实际尖角长度本身上。相反，这个限制在尖角长度与 `stroke-width` 宽度之间的比率上。如下图所示，这个比率等于连接线之间夹角（*`θ`*）的一半的正弦的倒数。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/02f905183c7b498e985df62071f471d9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1510&s=560280&e=jpg&b=fefafa)

  


对于一个锐角（小于 `90°`），正弦随着角度的增加而增加，因此正弦的倒数（我们通过 `stroke-miterlimit` 设定的比率）会随着角度的减小而增加。例如，当尖角长度为 `1.414` 时，对于小于 `90` 度的角度，尖角（`miter`）将转换为斜角（`bevel`）；当尖角长度为 `4.0` 时，对于小于约 `29` 度的角度，尖角将转换为斜角；当尖角长度为 `10.0` 时，对于小于约 `11.5` 度的角度，尖角将转换为斜角。

  


因此，从数学上来说，我们可以这样表述：

  


```
[miterLength] / [stroke-width] > [stroke-miterlimit] = miter
[miterLength] / [stroke-width] < [stroke-miterlimit] = bevel
```

  


这有道理，对吧？如果斜接不能超过描边的宽度，那么它应该是一个平坦的边缘。否则，斜接可以增长并形成一个尖角。

  


到这里，有关于 SVG 描边属性我们就聊完了，接下来我们开始探讨 SVG 的填充属性。

### 定义填充规则：fill-rule

  


`fill-rule` 属性主要用于定义填充规则，它决定了在填充一个复杂路径时如何确定填充区域。常见的填充规则有两种：`nonzero` 和 `evenodd`。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d1b4752ac1f44cc7854541ff32019d4b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1050&h=642&s=689452&e=gif&f=115&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/QWPgEyV

  


你可能对会对些感到好奇，它是如何确定形状的“内部性“？这里我无法将背后所有原理都向大家阐述清楚，但我会尽可能的告诉大家，`fill-rule` 的 `nonzero` 和 `evenodd` 发生了什么。

  


#### nonzero

  


`nonzero` 是 `fill-rule` 的默认选项。它通过从特定点向任意方向绘制一条线穿过形状来确定该点的内部位置。然后，检查形状的各个部分是否穿过这条线。根据从零开始的计数，我们确定了形状的“内部”区域。每当路径段从左到右穿过线时，计数加一（顺时针方向），每当路径段从右到左穿过线时，计数减一（逆时针方向）。路径的绘制方向（顺时针或逆时针）将对计数结果产生重大影响。一旦计算完所有交叉点，如果最终计数为零，则该点在路径外部；如果计数结果不为零，则该路径被认为在形状内部。

## 如何控制元素描边和填充的绘制顺序

  


在处理 SVG 时，有时候总是会有一些麻烦。你本来只想调整图形的描边，结果发现它以意想不到的方式改变了视觉外观。例如，你只是将图形的描边宽度（`stroke-width`）增加了几个像素，你会发现描边宽度会同时向内和向外两侧改变：

  


```XML
<svg class="love" viewBox="-256 -256 1536 1536">
    <path d="M917.333333 166.4c-106.666667-106.666667-285.866667-106.666667-392.533333 0l-12.8 17.066667-17.066667-12.8C388.266667 64 209.066667 64 102.4 170.666667s-106.666667 285.866667 0 392.533333l375.466667 375.466667c17.066667 17.066667 42.666667 17.066667 59.733333 0l375.466667-375.466667c110.933333-110.933333 110.933333-290.133333 4.266666-396.8z" />
</svg>
```

  


```CSS
.love {
      --fill: #09f;
      --stroke: #E387FF;
      --stroke-width: 10;

      fill: var(--fill);
      stroke-width: var(--stroke-width);
      stroke: var(--stroke);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/46733d370e9c400fa35ce10104923caf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1138&h=570&s=687703&e=gif&f=131&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/WNWOGaw

  


造成这种现象的直接原因是 SVG 图形元素的描边通常是居中于形状的边界的。也就是说，描边的一半位于形状内部，另一半位于形状的外部。这意味着形状的可见宽度实际上是形状的宽度加上描边的宽度。也就是说，形状的两侧都会额外增加一半描边宽度，因此总描边宽度会增加一倍。这一现象在像 Figma 这样的图形设计软件中对应着“居中描边”的描边方式。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b6c6aecf819a42feb740644db57a5383~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=622&s=215184&e=jpg&b=ffffff)

  


尽管 SVG 规范提出 `stroke-alignment` 属性，用于控制描边位置，但到目前为止还没有浏览器支持。因此，你现在看到的模式都类似上图中的“居中描边”。

  


> 注意，我们可以通过一些 Hack 手段，例如裁剪和遮罩来改变 SVG 这种默认的描边方式，但这部分内容超出这节课的范畴，因此不在这里做过多阐述。但我会在介绍 SVG 的裁剪和遮罩的课程中介绍这种 Hack 技术。

  


针对上面示例这种情况，大部分开发者会选择更改路径或给定元素的宽高，以避免描边过多的覆盖填充区域。但事实证明，我们可以使用 `paint-order` 属性来改变填充和描边的绘制顺序。

  


`paint-order` 属性的默认值是 `normal` ，它定义了填充（`fill`）先绘制，然后是描边（`stroke`），最后是标记（`markers`）。但你可以重新定义绘制顺序。

  


```CSS
/* 单一属性 */
path {
    paint-order: normal; /* 默认顺序，fill 👉 stroke 👉 markers */
}

path {
    paint-order: stroke; /* stroke 👉 fill 👉 markers */
}

path {
    paint-order: markers; /* markers 👉 fill 👉 stroke */
}
    
/* 多属性 */
path {
    paint-order: stroke fill; /* stroke 👉 fill 👉 markers */
}

path {
    paint-order: markers stroke fill; /* markers 👉 stroke 👉 fill */
}
```

  


当只指定一个值的时候，这个值将会被首先渲染，然后剩下的两个值将会以默认顺序渲染，当只指定两个值的时候，这两个值会以指定的顺序渲染，接着渲染剩下的未指定的那个。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5c99a7a0a92a4cab941433ac4fd5de36~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1108&h=604&s=1784575&e=gif&f=355&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/bGJRBVB

  


备注： 在这个属性的值当中，`markers` 只有当在绘制 SVG 图形时引用了 `marker-*`属性（例如 `marker-start`）和 `<marker>` 元素才能进行控制。他们不适用于 HTML 文本，所以，这样的话，你只能决定 `stroke` 和 `fill` 的顺序。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9c3d0cc4f0b44238b3681a095b22886a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1053&s=400195&e=jpg&b=fdfdfd)

## 小结

  


在这节课中，我们深入探讨了如何给通过描边和填充相关的属性来定义矢量图形的外观。通过了解描边和填充的基本原理以及它们的属性设置，我们可以轻松地创建丰富多彩的图形和图像。

  


我们了解到描边（Stroke）是指沿着图形路径绘制的线条，它可以定义线条的颜色、宽度和样式。同时，我们学会了通过添加 `stroke` 属性来设置描边的颜色，以及如何使用 `stroke-width` 来调整描边的宽度。此外，我们还了解到可以通过 `stroke-opacity`属 性来调整描边的透明度，从而实现更灵活的效果。

  


我们还了解到了如何使用 `stroke-dasharray` 和 `stroke-dashoffset` 设置描边的虚线样式，以及如何利用它们来制作动画效果。与此同时，还掌握了如何使用 `stroke-linecap` 给开放路径的两端设置样式，如何使用 `stroke-linejoin` 和 `stroke-miterlimit` 给线条的连接角设置样式等。

  


另一方面，填充（Fill）是指图形内部的颜色或图案。我们学会了通过添加 `fill` 属性来定义填充，以及如何使用不同的颜色或图案来填充图形的内部。此外，我们还了解到可以使用 `fill-opacity` 属性来调整填充的透明度，以实现更多样化的视觉效果。

  


通过对SVG描边和填充进行深入研究，我们不仅掌握了如何使用这些样式设置来创建各种图形，还了解到了如何调整其外观和样式以满足我们的需求。总的来说，掌握 SVG 描边和填充的技巧，将为我们的图形设计和开发工作提供更大的灵活性和创造性。
