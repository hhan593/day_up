除此之外，还有几点需要注意：

  


-   `<svg>` 元素的 `viewBox` 定义 [SVG 坐标系统](https://juejin.cn/book/7341630791099383835/section/7345677438053810214)。在这种情况下，我们声明所有图形都相对于一个 `1000` 像素的正方形来放置

现在这已经接近，但还不够。所有的行星旋转起始角度和速度都相同。理想情况下，每颗行星应该有自己的速度和旋转。由于我们使用 CSS 自定义属性定义了旋转角度和速度，所以我们可以通过 JavaScript 为这些自定义属性设置唯一值。我们可以将距离整合到随机旋转速度中，使较远的轨道需要更长时间。

  


```JavaScript
// 获取 SVG 元素和按钮元素
const svgElement = document.querySelector(".solar");
const buttonHandler = document.querySelector(".button");

// 获取 SVG 的 viewBox 属性，并将其分割成数组以获取宽度和高度
const viewBox = svgElement.getAttribute("viewBox");
const viewBoxValues = viewBox.split(" ");
const width = viewBoxValues[2];
const height = viewBoxValues[3];

// 返回两个值之间的随机数
const random = (min, max) => {
    const difference = max - min;
    return min + difference * Math.random();
};

// 返回两个值之间的随机整数
const randomInt = (min, max) => {
    return Math.round(random(min, max));
};

// 返回 true 或 false。默认情况下概率是 50/50，可以传入一个概率值在 0 到 1 之间（越高的值越可能返回 true）
const randomBool = (probability = 0.5) => {
    return Math.random() > probability;
};

// 返回一个随机行星大小在 10 到 50 之间的函数
const randomPlanetSize = () => randomInt(10, 50);

// 返回一个随机轨道距离在 100 到 120 之间的函数
const randomOrbitDistance = () => randomInt(100, 120);

// 更新行星在 SVG 中的位置和属性
const updatePlanets = (starSize) => {
    // 定义行星的数组，每个行星包含其类名和轨道距离
    const planets = [
        { class: "mercury", orbit: 194 },
        { class: "venus", orbit: 305 },
        { class: "earth", orbit: 423 }
    ];

    // 遍历行星数组，为每个行星设置随机的大小和位置
    planets.forEach((planet) => {
        let planetSize = randomPlanetSize(); // 获取随机行星大小
        let distance = randomOrbitDistance(); // 获取随机轨道距离
        let cx = 500 + planet.orbit; // 计算行星的 x 坐标
        let cy = 500; // 设置行星的 y 坐标
        let planetElement = document.querySelector(`.${planet.class}__planet`); // 获取行星元素
        // 设置行星的半径、中心坐标及旋转属性
        planetElement.setAttribute("r", planetSize);
        planetElement.setAttribute("cx", cx);
        planetElement.setAttribute("cy", cy);
        planetElement.style.setProperty(`--start-rotation`, `${randomInt(0, 360)}deg`);
        planetElement.style.setProperty(`--rotation-speed`, `${distance * randomInt(40, 70)}ms`);
    });
};

// 更新太阳在 SVG 中的大小
const updateStar = (size) => {
    const sun = document.querySelector(".sun"); // 获取太阳元素
    sun.setAttribute("r", size); // 设置太阳的半径
};

// 更新整个太阳系的函数
const updateSolar = () => {
    // 随机生成恒星的大小
    let starSize = randomInt(70, 120);
    // 调用更新太阳和行星的函数
    updateStar(starSize);
    updatePlanets(starSize);
};

// 初始化绘制太阳系
updateSolar();

// 当按钮被点击时重新绘制太阳系
buttonHandler.addEventListener("click", updateSolar);
```

你会发现，我们[使用 CSS 和 JavaScript 操控着 SVG 元素的样式和行为](https://juejin.cn/book/7341630791099383835/section/7351339840161447945)，比如使用 CSS 给行星添加动画效果，使用 JavaScript 随机调整行星的大小和圆心位置，以及它的旋转速度和角度。这也是 SVG 的强大优势之一，简单地说，SVG 内联在 HTML 中，[我们可以使用 CSS 和 JavaScript 来操控 SVG 的元素](https://juejin.cn/book/7341630791099383835/section/7351339840161447945)。

  


添加随机性之后，整个太阳系的效果要更生动一些，但它们还是有点单调，所有星体都是白色。为了避免这个现象，我们可以给星体添加颜色。在这里，我们使用 `hsl()` 函数来调整星体的颜色，因为 `hsl()` 允许我们分别设置颜色的色调（H）、饱和度（S）和亮度（L），这样就可以在特定的参数范围内随机生成颜色。让我们更新上面代码中的 `updatePlanets()` 函数，使用随机生成的颜色填充行星：

  


```JavaScript
// 更新行星在 SVG 中的位置和属性
const updatePlanets = (starSize) => {
    // 定义行星的数组，每个行星包含其类名和轨道距离
    const planets = [
        { class: "mercury", orbit: 194 },
        { class: "venus", orbit: 305 },
        { class: "earth", orbit: 423 }
    ];

    // 遍历行星数组，为每个行星设置随机的大小和位置
    planets.forEach((planet) => {
        let planetSize = randomPlanetSize(); // 获取随机行星大小
        let distance = randomOrbitDistance(); // 获取随机轨道距离
        let cx = 500 + planet.orbit; // 计算行星的 x 坐标
        let cy = 500; // 设置行星的 y 坐标
        let planetElement = document.querySelector(`.${planet.class}__planet`); // 获取行星元素
        
        // 随机生成行星的填充颜色
        const hue = randomInt(0, 360);
        const saturation = randomInt(70, 100);
        const lightness = randomInt(50, 70);
        const color = `hsl(${hue} ${saturation}% ${lightness}%)`;
        
        // 设置行星的半径、中心坐标、填充颜色及旋转属性
        planetElement.setAttribute("r", planetSize); 
        planetElement.setAttribute("cx", cx);
        planetElement.setAttribute("cy", cy);
        planetElement.setAttribute("fill", color);
        planetElement.style.setProperty(`--start-rotation`, `${randomInt(0, 360)}deg`);
        planetElement.style.setProperty(`--rotation-speed`, `${distance * randomInt(40, 70)}ms`);
    });
};
```

  


请注意，色调是从 `0 ~ 360` 的值，而饱和度和亮度都是百分比（需要百分号）。我们从 `0 ~ 360` 之间随机选择色调，同时将饱和度和亮度限制在预定义的范围内。看起来就像这样：

如果你对 [HSL 颜色](https://juejin.cn/book/7223230325122400288/section/7231515598088306720#heading-5)有所了解的话，你应该知道 HSL 中的色相值是如何映射到颜色：

对于我们的用例，我们允许以下色调范围，因为我们要排除绿色和紫色色调：

  


-   红色：`0 ~ 30`，或 `330 ~ 360`
-   黄色：`40 ~ 60`
-   蓝色：`190 ~ 240`

  


通过一点数学计算，我们可以在这些范围内选择一个随机的色调：

  


```JavaScript
// 返回数组中随机的一个元素
const randomItemInArray = (array) => {
    return array[randomInt(0, array.length - 1)];
}

// 更新太阳在 SVG 中的大小
const updateStar = (size) => {
    // 注意红色的上限范围超过了360
    const hueRange = randomItemInArray([
        [330, 390],
        [40, 60],
        [190, 240]
    ]);

    // 将选择的数组作为参数传递
    let hue = randomInt(...hueRange);

    // 如果色调大于360，则取余数
    if (hue > 360) {
        hue = hue - 360;
    }

    // 对于我们的恒星，我们将使用比行星更高的饱和度和亮度值。
    const saturation = randomInt(90, 100);
    const lightness = randomInt(60, 80);
    const color = `hsl(${hue} ${saturation}% ${lightness}%)`;
  
    const sun = document.querySelector(".sun"); // 获取太阳元素
    sun.setAttribute("r", size); // 设置太阳的半径
    sun.setAttribute("fill", color); // 设置太阳的填充色
};
```

首先，我们将使用 `<feTurbulence> 滤镜基元`给行星添加纹理效果。有一点需要知道，如果直接将 `<feTurbulence>` 滤镜基元创建的纹理应用到行星上，它会溢出行星。我们需要添加一个 `<feComposite> 滤镜基元`将行星与纹理进行合成，即只显示它们重叠的部分：

  


```XML
<svg viewBox="0 0 1000 1000" class="planet">  
    <defs>
        <filter id="filter">
            <feTurbulence type="turbulence" baseFrequency="0.1" seed="0" numOctaves="1" result="TURBULENCE__10" />
            <feComposite operator="in" in="SourceGraphic" in2="TURBULENCE__10" result="TURBULENCE__20" />
        </filter>
    </defs>
    <!-- 行星 -->
    <circle cx="500" cy="500" r="500" filter="url('#filter')" />
</svg>
```

通过调整 `<feTurbulence>` 、`<feDiffuseLighting>` 和 `<feDistantLight>` 滤镜基元的属性值，我们能够为行星创建一些有趣的纹理。

  


接下来，我们把类似上面的纹理添加到太阳系的行星中。为了确保我们的行星是独特且不同的，我们使用 JavaScript 来动态创建滤镜：

  


```JavaScript
// 创建滤镜的函数
const createFilter = (id, color, size) => {
    // 随机选择涡流类型，可能是 "fractalNoise" 或 "turbulence"
    const turbulenceType = randomBool() ? "fractalNoise" : "turbulence";
  
    // 计算涡流的基本频率，分别在 x 和 y 方向上，频率根据大小随机确定
    const baseFrequencyX = random(0.5, 2) / size;
    const baseFrequencyY = random(2, 4) / size;
  
    // 生成随机的 octave 数量，范围在 3 到 10 之间
    const numOctaves = randomInt(3, 10);
  
    // 生成一个随机的种子
    const seed = Math.random();

    // 生成随机的光源高度，范围在 30 到 100 之间
    const elevation = randomInt(30, 100);
  
    // 生成随机的表面比例，范围在 5 到 10 之间
    const surfaceScale = randomInt(5, 10);

    // 创建滤镜的 SVG 元素字符串
    const filter = `
    <filter id="${id}">
        <feTurbulence
            type="${turbulenceType}"
            baseFrequency="${baseFrequencyX} ${baseFrequencyY}"
            seed="${seed}"
            numOctaves="${numOctaves}"
            in="SourceGraphic"
            result="${id.toUpperCase()}__TURBULENCE__10"
        />
        <feDiffuseLighting lighting-color="${color}" surfaceScale="${surfaceScale}" in="${id.toUpperCase()}__TURBULENCE__10" result="${id.toUpperCase()}__TURBULENCE__20">
            <feDistantLight elevation="${elevation}" />
        </feDiffuseLighting>

const createFilter = (id, color,size) => {
    // 随机选择涡流类型，可能是 "fractalNoise" 或 "turbulence"
    const turbulenceType = randomBool() ? "fractalNoise" : "turbulence";
  
    // 计算涡流的基本频率，分别在 x 和 y 方向上，频率根据大小随机确定
    const baseFrequencyX = random(0.5, 2) / size;
    const baseFrequencyY = random(2, 4) / size;
  
    // 生成随机的 octave 数量，范围在 3 到 10 之间

    // 生成一个随机的种子

    // 生成随机的光源高度，范围在 30 到 100 之间

    // 生成随机的表面比例，范围在 5 到 10 之间

## Web 视觉表现：CSS 、SVG 和 Canvas

  


如今，在 Web 浏览器中呈现视觉表现和动画的基本技术是 CSS 、SVG 和 Canvas（WebGL）。这三种技术大致可以描述为：

  


-   CSS：与 HTML 高度集成，使用起来很方便
-   SVG：擅长绘制路径和图形
-   Canvas（WebGL）：利用 GPU 加速，进行高速像素运算

  


在实际开发过程中，有时候这三种技术中的任何一种都能实现某些效果，而有时候又难以选择该使用哪种技术。事实上，我们选择某种技术时，其关键是：

  


-   能用任何一种技术实现的，应该尽量简单化
-   必须使用特定技术才能实现的，要判断准确

  


接下来，我们以常见的 Web 效果为例（比如，变形动画、移动动画和特效效果等），与大家一起探讨在实现 Web 视觉效果时，应该使用哪种技术更妥当一些。

`<marker>` 是一个略显特殊的功能，但它可以根据路径端点的方向自动旋转元素，因此在与路径变形结合使用时非常方便。这也是你看到的效果中，卡通猫和叶子虽没设置动画，但它们会随路径变换一起动起来。你可以想象一下，一根丝带随风顠动时，它的头和尾也会一起运动。

  


不知道大家是否还有印象，我在介绍 SVG SMIL 动画时，曾说过，SMIL 动画很有可能会被废弃。庆幸的是，它的很多功能都在 CSS 中有相应的替代方案。例如上面示例中的 `<animate>` 给 `<path>` 设置的动画效果，我们可以在 CSS 中像下面这样来实现：

        d: path("M0,400 C30,350 -30,200 0,150");
        animation-timing-function:cubic-bezier(0.5, 0.0, 1.0, 0.5);
    }
    100% {
        d: path("M0,400 C60,350 0,250 -30,200");
    }
 }
 
 .animated {
     animation: transformAnimation 1.5s cubic-bezier(0.0, 0.5, 0.5, 1.0) infinite;
 }
```

### 移动动画

  


如果只是简单的移动和旋转动画，那么可以直接使用 CSS 来实现。例如：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b8142d64ce0f48dbbc73d1d4078e15ce~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1188&h=624&s=2345551&e=gif&f=101&b=4b8213)

  


> Demo 地址：https://codepen.io/airen/full/ExBYjPz

  


星星（红点和绿点）绕着太阳（黄点）旋转，卡通猫碰到墙壁（虚线边框）后轻轻弹回。就这个动画效果，我们可以完全可以仅使用 CSS 来实现。虽然在视觉上，圆点看上去是绕着虚线的圆在做旋转运动，实际上它就是一个简单的旋转动画。再看卡通猫，它是一个复合运动，沿水平方向和垂直方向做平移。

  


```HTML
<div class="stage">
    <div class="system">
        <div class="sun"></div>
        <div class="planet">
            <div class="planet"></div>
        </div>
    </div>

    <div class="cat">
        <svg viewBox="0 0 106.39 121.31">
          <!-- 绘制卡通猫 -->
        </svg>
    </div>
</div>
```

  


与动画有关的 CSS 代码如下：

  


```CSS
@keyframes rotate {
    to {
        rotate:360deg;
    }
}
.planet {
    animation: 8s linear infinite rotate;
}

@keyframes mAaxisY {
    to {
        translate: 0 310px;
    }
}
@keyframes mAxisX {
    to {
        translate:520px 0;
    }
}

.cat {
    animation: 
        2s ease-in-out infinite alternate-reverse mAaxisY,
        3.3s ease-in-out infinite alternate-reverse mAxisX;
    animation-composition: add; 
}
```

  


就这个示例而言，稍微复杂的就是卡通猫碰墙有一个组合动画，即 `mAaxisY` 和 `mAxisX` 两个方向的平移组合在一起。有关于这方面更详细的介绍，可以移步阅读《[多个 CSS 动画与动画合成：创造更复杂的动画效果](https://juejin.cn/book/7288940354408022074/section/7308623246604599307)》！

  


需要知道的是，CSS 移动和旋转功能也只提供了简单的动画功能，允许你做水平、垂直的直线移动以及旋转移动。虽然圆形移动严格来说不是真正的移动，但通过调整元素的中心点（`transform-origin`）并旋转元素，可以使其看起来像是沿着圆周在移动。但要是希望元素沿着不规则的路径移动，比如曲线移动。那么使用 SVG 会要简单很多。

### 视觉特效

  


在现代 Web 开发中，对于 Web 上的视觉特效，我们可以使用 CSS 、SVG 和 Canvas 来实现。这三种技术都具有各自的滤镜功能，但它们的能力和使用方式略有不同。

  


我们先来看 CSS 制作的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f0b2dc13b75b4471a99fc862ce0fc212~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1312&h=798&s=7351574&e=gif&f=85&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/NWZKGKe

  


正如你所看到的，如果仅仅是为了改变颜色（卡通猫发出彩虹光的效果）。那么 CSS 滤镜和混合模式就足够了：

  


```HTML
<div class="cats">
    <div class="cat">
        <svg>
            <use href="#cat" />
        </svg>
    </div>
    <div class="cat">
        <svg>
            <use href="#cat" />
        </svg>
    </div>
</div>
```

  


```CSS
.cats {
     animation: 2s ease-in-out infinite party;
 }
    
.cat:nth-child(2) {
    mix-blend-mode: multiply;
    animation: 2s ease-in-out alternate-reverse infinite beat;
}
    
@keyframes beat {
    from {
        filter: blur(5px) brightness(1.0);
    }
    to {
        scale:1.2;
        filter: blur(30px) brightness(1.6);
    }
}

@keyframes party {
    to {
        filter: hue-rotate(360deg);
    }  
}
```

  


通过应用多个滤镜或者使用 `mix-blend-mode` 进行混合，即使是简单的滤镜也可以创建复杂的动画效果。

  


但是，请记住，[CSS 滤镜](https://juejin.cn/book/7223230325122400288/section/7259669043622690853)无法实现的表现也存在。虽然 CSS 滤镜提供了十多种类型，但除了 `blur()` 和 `drop-shadow()` 之外的滤镜只是简单地根据统一的计算公式改变像素的颜色。因此，仅凭 CSS 滤镜的组合来实现根据周围像素颜色或整体图像状态变化的表现是比较困难的。

  


另外一点就是，CSS 滤镜是 SVG 滤镜的简捷版，如果 CSS 滤镜无法实现你所需要的效果，那么请考虑使用 SVG 滤镜。SVG 滤镜的功能要比 CSS 滤镜强大得多，而且将多个滤镜基元组合在一起，可以帮助你创建出丰富的视觉效果。例如下面这个示例，使用 SVG 滤镜为卡通猫应用了手绘动画风格的效果：

请记住，[SVG 滤镜](https://juejin.cn/book/7341630791099383835/section/7366549423746187273)不同于 [CSS 滤镜](https://juejin.cn/book/7223230325122400288/section/7259669043622690853#heading-0)，它支持复杂的效果，例如生成噪声的 `<feTurbulence>` 和用于光照效果的 `<feDiffuseLighting>` ，这些在 CSS 中无法实现。此外，SVG 允许你使用滤镜基元创建自己定义滤镜效果，例如在这个示例中使用 `<feTurbulence>` 生成噪声，然的通过 `<feDisplacementMap>` 扭曲像素位置，实现手绘风格的效果。

  


SVG 滤镜的规范复杂，并且浏览器兼容性有限，但它能够补充 CSS 无法实现的效果，因此在需要时可以考虑使用。当然，如果你需要更高的自由度，那请考虑使用 Canvas（WebGL）来实现。例如下面这个效果：

然而，Canvas（WebGL）和 PixiJS 的优势不仅在于直接使用这些滤镜，而是在于 Web 开发者可以自己创建多样的滤镜。虽然需要了解 GLSL 知识，但是当需要独特的效果时，这是一个可以考虑的选择。

  


我们不去评价每种技术的好坏，但掌握每种技术的能力和局限性，可以有效的地扩展 Web UI的表现力。换句话说，你不需要完全掌握每种技术的所有功能，但了解每种技术能做什么、不能做什么将有助于你扩展 Web 视觉的表现力。
