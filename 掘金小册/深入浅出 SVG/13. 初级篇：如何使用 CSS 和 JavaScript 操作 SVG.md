CSS 是一种用于控制 Web 页面样式和布局的语言，而 JavaScript 是一种用于添加交互和动态功能的脚本语言。它们作为 Web 开发的两个核心技术，通过结合这两种技术，我们可以实现对 SVG 的各种操作，包括样式控制、动画效果和事件处理等。这节课，我们将深入探讨如何利用 CSS 和 JavaScript 操作 SVG，为 Web 开发者提供一些实用的技巧和指导。

  


首先，让我们来看看如何利用 CSS 对 SVG 进行样式控制。CSS 提供了丰富的选择器和属性，可以轻松地为 SVG 元素添加样式。通过选择器，我们可以选择 SVG 中的特定元素，并为其应用颜色、大小、边框等样式。此外，CSS 还支持动画和过渡效果，可以为 SVG 创建各种华丽的动画效果，使页面更加生动。

  


其次，我们将介绍如何使用 JavaScript 操作 SVG。JavaScript 可以通过 DOM（文档对象模型）来访问和修改 SVG 元素。借助 JavaScript，我们可以实现诸如创建新元素、修改属性、响应事件等功能。例如，我们可以通过 JavaScript 实现鼠标悬停时改变 SVG 图形颜色的效果，或者在点击按钮时动态添加新的 SVG 元素。

  


综上所述，CSS 和 JavaScript 为操作 SVG 提供了强大的工具和灵活性。通过熟练掌握这两种技术，开发者可以创建出各种丰富多彩、交互性强的 SVG 图形，为用户提供更加优秀的 Web 体验。在接下来的内容中，我们将深入探讨 CSS 和 JavaScript 在 SVG 操作中的具体应用，帮助大家更好地掌握这些技术，并将其运用到实际项目中。

  


## 简介

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/255d4355d2834bedb722a25de1311991~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=45120&e=avif&b=fdfafa)

  


众所周知，HTML 、CSS 和 JavaScript 是构建 Web 应用或网站的三大核心技术：

  


-   HTML 用于定义 Web 页面的结构和内容，包括文本、图像、链接等
-   CSS 用于定义网页的样式和布局，包括字体、颜色、大小、间距等
-   JavaScript 用于添加交互性和动态功能，可以处理用户输入、操作 DOM 元素、发送网络请求等

  


这三种技术共同作用，使得 Web 页面能够呈现出丰富的内容、漂亮的样式和交互性的功能，是构建现代 Web 应用的基础。

  


SVG 是一种基于 XML 的图形格式，它提供了一种描述矢量图形的方式。SVG 可以与 HTML、CSS 和 JavaScript 紧密结合，为 Web 带来丰富的图形效果和交互功能：

  


-   **SVG 与 HTML**： SVG 图像可以直接嵌入到 HTML 文档中，可以使用 `<svg>` 元素直接在HTML中编写 SVG 代码，也可以通过 `<img>`、`<object>`、`<embed>` 等元素引入外部SVG文件。这使得开发者可以在 Web 中直接使用 SVG 图像，而无需额外的插件或工具。
-   **SVG 与 CSS**： CSS 可以用来样式化 SVG 元素，类似于对 HTML 元素的样式化。可以通过 CSS 属性控制 SVG 图形的颜色、填充、边框、阴影、动画等效果。这样可以实现更加灵活的图形设计，并且使得 SVG 图形与网页的整体样式保持一致。
-   **SVG与JavaScript**： JavaScript 可以与 SVG 一起使用，实现动态效果和交互功能。通过 JavaScript，可以操作 SVG 元素的属性、创建动画、响应事件等。开发者可以利用 JavaScript 来实现复杂的图形动画、交互式图形界面等功能，为用户提供更加丰富的用户体验。

  


SVG 的融入为 Web 带来了更加灵活和丰富的图形设计和交互体验，使得开发者能够创建更加吸引人和功能丰富的网页应用。

  


另外，HTML、CSS 和 JavaScript 之间的界限（分工）很明确：HTML 关注内容和结构，CSS 关注外观，而 JavaScript 关注交互。至少可以说，SVG 模糊了这条界线。这么说是有原因的：

  


-   **内联性质**：SVG 可以直接嵌入到 HTML 文档中，与其他 HTML 元素一样。这意味着 SVG 图形可以与 HTML 内容混合在一起，使得 HTML 和 SVG 之间的分界变得模糊
-   **样式化**：CSS 可以应用于 SVG 元素，从而改变它们的外观和布局。这包括填充、边框、颜色、形状等方面的样式化。由于 SVG 元素可以使用 HTML 和 CSS 中的许多相同的属性和样式规则，因此 HTML、CSS 和 SVG 之间的界线变得模糊
-   **交互性**：SVG 元素可以与 JavaScript 交互，这意味着可以使用 JavaScript 来操作SVG 图形响应用户的行为。通过 JavaScript ，可以实现动态效果、事件处理等功能，这使得 SVG 图形的交互性与 HTML 元素的交互性非常接近
-   **矢量图形**：SVG 是一种矢量图形，可以无限缩放而不会失真。这使得 SVG 图形在设计和开发过程中可以与 HTML 、CSS 和 JavaScript 相互交织，而不必考虑固定尺寸和分辨率的限制，进一步模糊了它们之间的界限

  


SVG 的内联性质、样式化能力、交互性以及矢量图形特性等特点使得它与 HTML 、 CSS 和 JavaScript 之间的界线变得模糊，为 Web 开发者提供了更大的创造空间和灵活性。

  


接下来，我们就一起来深入的探讨 CSS 与 SVG ，JavaScript 与 SVG 之间故事！首先从 CSS 与 SVG 之间聊起。

  


## 使用 CSS 对 SVG 进行样式化

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9489653a928b4db3889f19047f19a584~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2554&h=1184&s=209412&e=jpg&b=500e47)

  


> CSS 与 SVG 的结合为 Web 打开了新的大门。

  


SVG 是一种基于 XML 的图形格式，可以用来描述二维矢量图形。如果仅从 SVG 的代码层面来说，那么它和我们熟悉的 HTML 几乎没有太大的差异。如果你是一名老程序员，那么你应该知道，在 CSS 还没有出现之前，给 Web 添加样式是非常有限的，而且都在 HTML 元素元素上完成，例如：

  


```HTML
<p>
    <font color="red">This paragraph is red.</font>
</p>
```

  


上面代码将会给你呈现一段红色的文本。就这一点而言，SVG 与 HTML 是一样的。

  


或许你也早就发现了，前面课程中，我们在使用 SVG 绘制图形元素时，会有很多涉及样式的属性出现在图形元素上，例如填充颜色（`fill`）、描边颜色（`stroke`）和描边宽度（`stroke-width`）等。

  


```XML
<svg viewBox="0 0 300 300">
    <rect x="10" y="10" width="280" height="280" fill="red" stroke="lime" stroke-width="4" />
</svg>
```

  


这是因为 SVG 1.1 不要求使用 CSS 来为 SVG 元素设置样式。SVG 规范对此是这么描述的：

  


> 可以使用 SVG 的“演示属性”（Presentation Attributes）来指定元素的样式。对于规范中定义的每个样式属性，在所有相关的 SVG 元素上都有相应的 XML “[演示属性](https://www.w3.org/TR/SVG11/propidx.html)”可用。

  


换句话说，XML 的“演示属性”是在 SVG 元素上设置 CSS 属性的一种简写方式。可以将它们视为特殊的样式属性。它们甚至参与样式级联（稍后会介绍）。下面是一个使用“演示属性”为一个五角星图形设置了边框颜色（`stroke`）、边框粗细（`stroke-width`）和填充颜色（`fill`）样式的 SVG 案例：

  


```XML
<svg class="star" viewBox="0 0 1024 1024">
    <path 
        fill="red" 
        stroke="orange" 
        stroke-width="6" 
        d="M908.1 353.1l-253.9-36.9L540.7 86.1c-3.1-6.3-8.2-11.4-14.5-14.5-15.8-7.8-35-1.3-42.9 14.5L369.8 316.2l-253.9 36.9c-7 1-13.4 4.3-18.3 9.3-12.3 12.7-12.1 32.9 0.6 45.3l183.7 179.1-43.4 252.9c-1.2 6.9-0.1 14.1 3.2 20.3 8.2 15.6 27.6 21.7 43.2 13.4L512 754l227.1 119.4c6.2 3.3 13.4 4.4 20.3 3.2 17.4-3 29.1-19.5 26.1-36.9l-43.4-252.9 183.7-179.1c5-4.9 8.3-11.3 9.3-18.3 2.7-17.5-9.5-33.7-27-36.3z" />
</svg>
```

  


`<path>` 元素中的 `fill` 、`stroke` 和 `stroke-width` 属性都是演示属性。注意，这些“演示属性”是可以直接对 SVG 元素进行样式化处理的。因此，上面代码给你呈现的五角星效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fa333c5bd68f4bd2a95eb1471a4e1d4e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=831&s=147011&e=jpg&b=060606)

  


> Demo 地址：https://codepen.io/airen/full/poBwxBK

  


在 SVG 中，可以通过 SVG 属性设置 CSS 属性的一部分，反之亦然。[SVG 规范列举了一些可以将 SVG 属性](https://www.w3.org/TR/SVG/propidx.html)设置为CSS 属性的 SVG 属性。其中，一些属性与 CSS 共享，例如 `opacity` 和 `transform`，而另一些则不同，例如 `fill`、`stroke` 和 `stroke-width` 等。

  


这方面在 [SVG 2 规范中也明确的指出](https://www.w3.org/TR/SVG2/styling.html#PresentationAttributes)：

  


> 一些样式属性不仅可以在样式表和 `style` 属性指定，还可以在“演示属性”中指定。这些属性的名称与给定的 CSS 属性匹配（或类似），其值被解析为该属性的值。

  


这不仅意味着 SVG 属性可以像演示属性或样式表中一样通过 CSS 进行样式化，而且还可以应用于 CSS 伪类，例如 `:hover` 或 `:active` 。另外，在 SVG 2 中，还引入了[更多可以用作样式属性的演示属性](https://www.w3.org/TR/SVG2/styling.html#PresentationAttributes)，例如 `width` 、 `height` 、 `x` 、`y` 、`cx` 和 `cy` 等，这些属性在 SVG 1.1 中无法通过 CSS 设置。这意味着，除了可以使用 CSS 样式化 SVG 外观，还可以使用 CSS 控制 SVG 元素的坐标和尺寸。

  


换句话说，除了使用 SVG 的演示属性进行样式处理之外，我们还可以使用 CSS 属性来样式化 SVG 元素。在这方面，与 HTML 中一样，可以通过元素的 `<style>` 属性（内联样式）来给 SVG 元素设置样式。例如：

  


```XML
<svg class="star" viewBox="0 0 1024 1024">
    <path 
        style="fill: red; stroke: orange; stroke-width: 6;"
        d="M908.1 353.1l-253.9-36.9L540.7 86.1c-3.1-6.3-8.2-11.4-14.5-14.5-15.8-7.8-35-1.3-42.9 14.5L369.8 316.2l-253.9 36.9c-7 1-13.4 4.3-18.3 9.3-12.3 12.7-12.1 32.9 0.6 45.3l183.7 179.1-43.4 252.9c-1.2 6.9-0.1 14.1 3.2 20.3 8.2 15.6 27.6 21.7 43.2 13.4L512 754l227.1 119.4c6.2 3.3 13.4 4.4 20.3 3.2 17.4-3 29.1-19.5 26.1-36.9l-43.4-252.9 183.7-179.1c5-4.9 8.3-11.3 9.3-18.3 2.7-17.5-9.5-33.7-27-36.3z" />
</svg>
```

  


如果你并不喜欢内联样式的方式，那么你可以将样式集中放置在 `<svg>` 标签中的 `<style></style>` 中。不过，采用这种方式给 SVG 元素设置样式的话，最好将所有样式规则放在 `<![CDATA[ ... ]]>` 段中：

  


```XML
<svg class="star" viewBox="0 0 1024 1024">
    <style type="text/css">
        <![CDATA[
            path {
                fill: red;
                stroke: orange;
                stroke-width: 6;
            }
        ]]>
    </style>
    <path  d="M908.1 353.1l-253.9-36.9L540.7 86.1c-3.1-6.3-8.2-11.4-14.5-14.5-15.8-7.8-35-1.3-42.9 14.5L369.8 316.2l-253.9 36.9c-7 1-13.4 4.3-18.3 9.3-12.3 12.7-12.1 32.9 0.6 45.3l183.7 179.1-43.4 252.9c-1.2 6.9-0.1 14.1 3.2 20.3 8.2 15.6 27.6 21.7 43.2 13.4L512 754l227.1 119.4c6.2 3.3 13.4 4.4 20.3 3.2 17.4-3 29.1-19.5 26.1-36.9l-43.4-252.9 183.7-179.1c5-4.9 8.3-11.3 9.3-18.3 2.7-17.5-9.5-33.7-27-36.3z" />
</svg>
```

  


如果你将 SVG 内联嵌套到 HTML 文档中，你还可以将设置 SVG 元素的样式放到 HTML 的 `<style>` 元素中。这相当于将样式放在 `<svg>` 元素之外：

另外，如《[如何使用 SVG](https://juejin.cn/book/7341630791099383835/section/7344089098363076620)》中所述，在进行 Web 开发的时候，难免需要将 SVG 文档保存为独立的 `.svg` 文件，提供给 HTML 元素或 CSS 其他属性使用，例如 `<img>` 元素和 `background-image` 属性。但你在维护 `.svg` 文件时，又希望将样式与标记完全分离，那么你可以使用 `<?xml-stylesheet>` 标签，将外部的 `.css` 文件引入到 SVG 文档中。这与 HTML 的 `<link>` 标签非常的相似：

  


```CSS
/* star.css （外部样式文件 ）*/

    fill: red;
    stroke: orange;
    stroke-width: 6;
}
```

  


```XML
<!-- star.svg 文档 -->

</svg>
```

正如你所看到的，与样式化 HTML 元素基本上是一样的。

  


对于这些选择，我更倾向于将 CSS 样式与 SVG 元素分离，这也是我强烈推荐的做法。如今，在 Web 开发中，我们通常将 SVG 嵌入到 HTML 文档中，这也是目前最流行的 SVG 嵌入方式之一。这种方式使得 CSS 和 SVG 的工作更加简单，因为可以在文档的任何位置放置样式规则来对 SVG 进行样式化处理。换句话说，样式不需要放置在 `<svg>` 元素的开始和结束标签之间才能生效；而对于其他技术，这一点是必需的。

  


### 避不开的级联

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/89d17888a4e3403e945d7ecf98d422cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=719&s=1582042&e=png&b=3e3b37)

  


不管在哪，只要是聊 CSS，那就绕不开 CSS 级联，使用 CSS 样式化 SVG 元素亦是如此。例如，在定义 CSS 样式时，经常会遇到一种情况，即单个元素可能应用多个样式属性值。例如，你可能会一个 `<path>` 元素上使用“演示属性” `fill` 给该元素设置一个填充颜色，例如 `fill="red"` ，同时可能在一个 `<style>` 代码块中设置：

  


```CSS
path {
    fill: green;
}

#star {
    fill: yellow;
}
```

  


又在外部样式表中为 `fill` 设置了另一个值：

  


```CSS
/* star.css */
svg path {
    fill: orange;
}
```

  


还有可能在元素的 `style` 属性中为 `fill` 设置了一个 `lime` 值。

  


```HTML
<html>
    <head>
        <title>Star</title>
        <link rel="stylesheet" href="./path/to/star.css" type="text/css" />
        <style>
            path {
                fill: green;
            }
            
            #star {
                fill: yellow;
            }
        </style>
    </head>
    <body>
        <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1024 1024">
            <path 
                id="star"
```

这就是 CSS 级联的作用所在：它规定了不同样式声明的瀑布流，一个取代另一个，直到最终确定使用的值。

  


先简单回顾一下，CSS 级联在 HTML 中的工作方式。在不考虑级联层规则（[CSS 的 @layer 规则](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)）的情况下，其标准如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8c3e551098644d97a693d21637a650d4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3186&h=995&s=1316309&e=jpg&b=131416)

  


随着 CSS 的 `@layer` 规则的诞生，它提供了一种结构化的方式来组织和平衡单一来源中的 CSS 规则，最终决定谁获胜！使得 Web 开发者更易于控制 CSS 的级联：

> 注意，[CSS 的 @layer 规则](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)是[现代 CSS 的一个重要特性](https://s.juejin.cn/ds/iF7sUGw7/)，它解决了一直以来令 Web 开发者感到头疼的问题，即控制 CSS 级联。不过，这部分内容超出了这节课的范畴，如果你对这方面知识感兴趣的话，建议你移步阅读《[CSS 分层：@layer](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)》

  


在很大程度上，CSS 级联在 SVG 中的工作方式与其在 HTML 中的工作方式相同。然而，SVG 中的“演示属性”为其增添了一个额外的变化。即“演示属性”位于客户端样式（例如浏览器默认样式）与外部样式之间：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bf2c7820e5314b018ebc303f84791c79~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1397&h=1198&s=55894&e=png&b=fefefe)

  


正如你所见，SVG 元素的“演示属性”被除了用户代理特定样式之外的所有其他样式所覆盖。

  


在 SVG 中，作用于 SVG 元素上的 CSS 规则，其优先级顺序由以下四个因素的相互作用决定：

  


-   样式声明来源：浏览器默认样式、Web 开发者写的样式或用户的偏好设置
-   规则的重要性（是否使用了 `!important`）
-   CSS 选择器的权重
-   声明的顺序：当其他条件相同时，后面的声明将替换之前的声明

  


每个元素上每个样式属性的最终使用值是 CSS 级联、继承和基于 Web 布局的额外计算的结果。

  


#### 样式声明的来源和重要性

  


从级联的角度来看，有五种可能的来源。它们按照以下顺序排名（除 !important 声明之外），排名较后的来源会覆盖排列较前的来源：

  


-   ①：Web 浏览器（或其他用户代理）对特定元素或伪类的默认样式。例如，在 SVG 元素上（如 `<svg>` 、`<image>` 、`<symbol>` 、 `<pattern>` 等），`overflow` 属性的默认值为 `hidden` （在 CSS 中，`overflow` 的默认值为 `visible`）；`<defs>` 元素的 `display` 始终被强制为 `none` （在 CSS 中， `display` 的默认值为 `inline` ）
-   ②：用户在系统中或 Web 浏览器中设置的偏好设置。例如，大多数浏览器允许用户指定默认字体
-   ③：Web 开发者定义的样式：在 SVG 元素标签中、在 <style> 块中，或者来自外部样式表
-   ④：在 `@keyframes` 动画规则中定义的样式
-   ⑤：CSS 的过渡（`transition`）。过渡属性遵循正常的级联规则，但是当一个过渡应用于另一个属性时，它会应用由任何其他来源创建的更改，包括关键帧动画

  


特别声明，需要注意带有 `!important` 规则，它的排列顺序通常会与上面的排名反过来。这一点与 CSS 在 HTML 中的应用是相似的。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b38663c1edf74ea0950aa2d8b6e5cc63~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=871&h=731&s=490333&e=jpg&b=131416)

  


#### 选择器权重和源顺序

  


当同一来源声明的多个样式规则可能适用于给定元素时，它们将根据所使用的 CSS 选择器来权重进行排名。选择器权重通常按照下面的规则（从最不具体到最具体）进行排序（由低到高）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/69abfd4da1bb4a57892a929205f9ba17~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=740&s=101362&e=png&b=ecf6f3)

  


-   ①：元素的标签名和伪元素选择器
-   ②：类、伪类和基于属性的选择器
-   ③：ID 选择器
-   ④：元素内联样式，即 `style` 属性定义的样式

  


需要注意的是，SVG 的“演示属性”是最“没存在感”的一种，它随时都能被被人替代。例如，通用选择器 `*` ，在比较样式规则时，该选择器的权重为 `0` ，但在该选择器的样式规则同样优于 SVG 元素的“演示属性”、继续值和浏览器默认值。这意味着，SVG 元素的“演示属性”被视为具有略低于 `0` 权重的选择器，任何应用于相同元素的 CSS 规则都将覆盖它。

  


另外，[现代 CSS](https://s.juejin.cn/ds/iFvc9H53/) 新增的选择器 `:is() 和 :where() 可以用于改变选择器权重计算`， **`:where()`** **的权重始终为** **`0`** **，而** **`:is()`** **则采用其参数中最具体选择器的权重**。如果你无法确认使用选择器权重，可以通过选择器权重计算器来辅助自己：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de2cce6253454ed0acdff426537f9a06~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=51641&e=png&b=f6f9fe)

  


> URL：https://polypane.app/css-specificity-calculator/

  


如果两个 CSS 声明具有相同的来源、重要性（`!important`）和权重，则使用最后的决定者：源顺序。最后的声明胜出。额外要提醒大家的是，[CSS 的 @layer 规则](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)出现将会改变源顺序。关于这一点，前面也简单的提到过。

  


注意，SVG 的“演示属性”没有特定的顺序，但是你不应该在同一元素上应用多个具有相同名称的属性。如果确实这样做了，结果是未定义的，浏览器可能选择第一个或最后一个版本。例如：

  


```XML
<svg viewBox="0 0 200 200">
    <rect fill="red" x="0" y="" width="200" height="200" fill="green" />
</svg>
```

  


上面代码中，`<rect>` 元素上同时设置了两个 `fill` （“演示属性”），最终呈现给你的是一个红色矩形。正常情况下，你可能不会这样使用，但需要知道，如果因手误这样做了，它很有可能会给你一个不确定性的结果。

  


#### 级联值和指定值

  


来源、权重和源顺序被用来对元素上所有样式声明进行排序。这就是级联过程：为每个属性和元素选择单个、排名最高的声明。

  


级联后，每个元素上的所有样式声明都被减少到每个属性的单个级联值，但也可能没有级联值。因为级联中任何地方都没有提及元素的属性的 CSS 声明。在这种情况下，元素被分配为初始值或继承值，具体取决于样式属性是否可被继承。此分配的结果称为指定值。通常情况下，每个元素（和伪元素）上的每个属性都有一个指定值。

  


当然，你也可以通过在 CSS 声明中使用这些关键字（如 ` inherit 、 initial 、unset 和 revert  `）强制级联值为初始值或继承值。它们可用作任何 CSS 属性中值的替代，包括 SVG 元素的“演示属性”。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b8c3f1724b414093baebc6834f9ea531~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1398&h=939&s=196829&e=jpg&b=e8e8eb)

  


注意，`initial` （初始关键词）不受特定元素的浏览器默认值的影响。因此，`overflow: initial` 将始终显示为可见（`visible`），而不管是 SVG 还是 HTML 。这是因为，在 CSS 规范中定义 `overfow` 属性的初始值时，它就是 `visible` 。

  


#### 继承和计算值

  


一旦你在每个元素的每个属性上有了指定值（可能是继承或初始值），你就完成了级联，可以继续进行继承。

  


在继承之前，许多属性值都会进行部分计算，创建计算值。例如，`em` 单位通常会转换为 `px` ；CSS 变量引用（`var()` 函数）将被其值替换；颜色会转换为 `rgb()` 表示法。

  


计算过程从继承树的顶部向下进行，从根元素开始。此时也会替换初始值和继承值：**初始值为规范中定义的值，继承值为父元素的计算值**。通常需要继承值来进行其他部分的计算，例如，你可能需要计算继承的字体大小（`font-size`），以将 `em` 转换为 `px` 。

  


在 SVG 中，`<use>` 元素引用创建的隐藏克隆元素会继承来自 `<use>` 元素本身的样式。换句话说，对于样式继承，克隆元素会被视为 `<use>` 元素的子元素，尽管它们对于 CSS 选择器匹配来说不是子元素。有关于这部分，我们曾在《[初级篇：组织 SVG](https://juejin.cn/book/7341630791099383835/section/7348735226207535138)》有过深度的探讨，这里不再重复阐述。

  


需要知道的是，CSS 选择器的权重和规则来源对继承值没有影响。低权重的“演示属性”不会被继承值替换，即使该继承值由 `!important` 规则或动画创建的。另外，继承的计算值不包括 `getComputedStyle()` 方法使用的所有最终计算值，例如将百分比转换为 `px` ，因为这些计算需要计算最终的 CSS 布局。

  


继承完成后，浏览器可以对 Web 页面进行布局，将计算的 CSS 值转换为其最终使用的值。百分比（`%`）和自动值（`auto`）会被解析， `calc()` 函数会被计算，`currentColor` 会被替换（如果之前没有），以及各种特定属性的夹紧和其他调整都会进行。

  


最后，浏览器可以实际绘制网页。换句话说，你样式化的 SVG 也最终会在浏览器中按你期望的方式呈现。

  


### 与 HTML 样式化不同之处

  


在大多数情况下，使用 CSS 对 SVG 元素进行样式化处理与 HTML 元素进行样式化处理是相同的，但也有一些差异存在。这里我们以 CSS 选择器和变换属性为例。首先来看 CSS 选择器之间的差异。

  


大多数 CSS 选择器都可以用于选择 SVG 元素。除了通用的类型（如元素选择器）、类和 ID 选择器之外，SVG 还可以使用动态伪类（如 `:hover` 、`:active` 和 `:focus` 等）和结构性伪类（如 `:first-child` 、`:nth-child()` 、`:nth-of-type()` 等）来进行样式化。与 CSS 在 HTML 中不同的是，CSS 的伪元素（如 `::before` 和 `::after`）不属于 SVG 语言定义的一部分，因此不会对 SVG 的样式产生影响。

不能发现，`.flash` 应用的 `pluse` 动画并不是我们想要的一个效果。`pluse` 动画只是在关键帧的 `50%` 处设置了 `scale: 1.25` ，放大 `1.25` 倍。你会发现，作用于 SVG 元素的 `scale` 与 HTML 元素存在一些差异。

  


造成这种现象的主要原因是，SVG 元素不像 HTML 元素，受盒模型的约束（SVG 元素没有盒模型的概念），因此没有边距（`margin`）、边框（`border`）、内距（`padding`）或内容框。

  


默认情况下，HTML 元素的变换原点（`transform-origin`）位元素的中心，即 `(50%,50%)` 。相比之下，SVG 元素的变换原点（`transform-origin`）位于用户当前坐标系的原点，即画布的左上角 `(0,0)` 点。

  


如果我们希望上面示例中的闪电（`.flash`）能与 HTML 元素一样，以图形自身中心来回缩放，而不是以 SVG 画布的左上角来回缩放，我们就需要重置 `transform-origin` 属性的值为 `50% 50%` （或 `center`）：

  


```CSS
.flash {
    transform-origin: center; /* 或 50% 50% */
    animation: pluse 2s ease-in-out infinite alternate,
      flash 0.5s ease-in-out infinite alternate;
}
```

> Demo 地址：https://codepen.io/airen/full/zYXdKbE

  


相比而言，在 HMTL 元素上设置变换原点（`transform-origin`）很简单：你指定的任何值都相对于元素的边框盒设置。但在 SVG 中，可以使用百分比值或绝对值（例如 `px`）设置变换原点（`transform-origin`）。如果你将 `transform-origin` 属性设置为百分比值，那么该值将相对于元素的边界框设置，其中包括用于绘制其边框的描边。如果你将 `transform-origin` 属性设置为绝对值，那么该值将相对于用户当前坐标系设置。

  


这也意味着，应用于 SVG 元素的变换要比应用于 HTML 的变换要复杂得多，它除了涉及到变换属性的基本使用之外，还需考虑 SVG 坐标系。换句话说，它将需要对坐标系进行变换处理。有关于这方面更详细的内容，将放到后面的课程中介绍。

  


## 使用 JavaScript 对 SVG 进行操作

  


与 HTML 类似，SVG 使用文档对象模型（DOM）表示。因此，你可以像操作 HTML 那样来操作 SVG。

  


通过《[初级篇：如何使用 SVG](https://juejin.cn/book/7341630791099383835/section/7344089098363076620)》一节课的学习，我们知道 SVG 可以通过多种不同的方式嵌入 HTML 文档：

  


```HTML
<!-- 作为图片使用 <img> 标签嵌入 -->
<img src="./path/to/my-svg.svg" alt="My SVG" />

<!-- 使用 <object> 标签作为对象嵌入 -->
<object type="image/svg+xml" data="./path/to/my-svg.svg">
    <!-- Fallback -->
</object>

<!-- 使用 <iframe> 标签作为 iframe 嵌入 -->
<iframe src="./path/to/my-svg.svg">
    <!-- Fallback -->
</iframe>

<!-- 使用 <embed> 标签作为 embed 嵌入  -->
<embed type="image/svg+xml" src="./path/to/my-svg.svg" />

<!-- 使用 <svg> 标签将 SVG 嵌套在 HTML -->
<svg viewBox="0 0 300 300">
    <rect x="10" y="10" fill="red" width="280" height="280" />
</svg>    
```

  


SVG 还可以作为 CSS 中的图片引入 HTML 中：

  


```CSS
/* 作为背景图片引入 */
.element {
    background-image: url("./path/to/my-svg.svg");
}

/* 作为遮罩图片引入 */
.mask {
    mask-image: url("./path/to/my-svg.svg");
}
```

  


如果 SVG 使用 `<img>` 标签加载，或者通过 CSS 作为背景加载，与来源无关：

  


-   CSS 和 JavaScript 不能与其交互
-   SVG 中包含的 JavaScript 被禁用
-   无法加载外部资源，如图像、样式表、脚本、字体

  


具体细节：
    

| **功能** | **内联 SVG** | **object / embed / iframe** | **img / background-image** |
| ---------------------- | ---------- | ----------------------------- | --------------------------- |
| **可与用户交互**             | ✅          | ✅                             | ✅                           |
| **支持动画**               | ✅          | ✅                             | ✅                           |
| **可运行自身的 JavaScript**  | ✅          | ✅                             | 👎🏼                        |
| **可从外部 JavaScript 操作** | ✅          | 👎🏼                          | 👎🏼                        |

  


也就是说，我们可以在 SVG 文档内部和外部嵌入 JavaScript 脚本，允许你使用 [DOM API](https://dom.spec.whatwg.org/) 和 [SVG API](https://svgwg.org/svg2-draft/types.html) 来操作 SVG，例如，选中元素、创建元素、删除元素、设置属性、改变样式和添加交互效果等。这里所说的 SVG 文档内部指的是在独立的 SVG 文档中嵌入 JavaScript 脚本，即在 `<svg>` 元素内嵌入 JavaScript：

  


```XML
<svg viewBox="0 0 1024 1024">
    <script>
        <![CDATA[ 
            console.log(document.getElementsByTagName('rect')); 
            console.log(document.getElementById('my-rect'));
            console.log(document.querySelector('.a-rect'));
            console.log(document.querySelectorAll('.a-rect'));
        ]]>
    </script>
    <rect x="0" y="0" width="100" height="100" fill="blue" id="my-rect" class="a-rect"  />  
</svg>

<!-- 或者 -->
<svg viewBox="0 0 1024 1024">
    <rect x="0" y="0" width="100" height="100" fill="blue" id="my-rect" class="a-rect"  />  
    <script>
        <![CDATA[ 
            console.log(document.getElementsByTagName('rect')); 
            console.log(document.getElementById('my-rect'));
            console.log(document.querySelector('.a-rect'));
            console.log(document.querySelectorAll('.a-rect'));
        ]]>
    </script>
</svg>
```

  


SVG 文档外部嵌入 JavaScript 脚本指的是 `<script>` 在 `<svg>` 标签之外，这常见于 SVG 嵌套在 HTML 文档中的场景：

换句话说，通过使用 `pointer-events` 属性可以塑造 SVG 图像的交互性，即控制文档的哪些部分可以接收点击、触摸和轻触事件。

正如你所看到的，上面示例中，如果你的鼠标没有放在正确的位置（上图是渐变色的地方），那么点击链接无效（`click` 事件也无效）。你可能会感到非常困惑！

  


是的，你可能已经猜到了，我们可以使用 `pointer-events` 来解决这个问题。接下来，我将告诉你如何以及为什么 `pointer-events` 可以解决这个问题。

  


SVG 与 HTML 结合使用 CSS 会在 HTML 上强加一个盒模型。在盒模型中，每个元素都会生成一个围绕其内容的矩形框。该矩形框（也是盒子）可以是内联的（`inline`）、内联块的（`inline-block`）、块（`block`）或弹性盒子（`flex` 或 `inline-flex`）等，它们可以由 `display` 属性的值来决定。我们也知道，每一个 HTML 元素都可以被视为一个盒子，它始终是一个矩形框。当我们为元素添加链接或监听事件时，交互区域与该矩形的尺寸相匹配。

  


注意，如果[交互元素框运用了 CSS Clipping 时](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-22)，那么超出剪切图形（`clip-path`）之外的东西都不可见，包括鼠标的悬浮、点击等。简单地说，剪切路路之外的地方，将失去悬浮和用户点击等行为。

  


SVG 在这方面则不同，它没有盒模型这个概念。当 SVG 嵌套在 HTML 文档中时，只有根元素 `<svg>` 会遵循 CSS 的盒模型的约定，但其所有后代元素不会受 CSS 盒模型约束。因此，与盒模型相关的属性（`margin` 、`padding` 、`border` 等）都不适用于 SVG （除内嵌在 HTML 的根元素 `<svg>` 之外）。

  


虽然 SVG 元素没有盒模型，但它有一个“**[边界框](https://svgwg.org/svg2-draft/coords.html#BoundingBoxes)**”（Bounding Box）。W3C 规范是这样定义的：“边界框（或 BBOX）是与元素的用户坐标系统的轴对齐的最紧密的矩形，完全包围了该元素及其后代元素”。一个 SVG 元素可以计算三种类型的边界框：

  


-   对象边框界（**Object Bounding Box**）是仅包含元素几何形状的边界框。对于基本形状来说，这是填充的区域。除非另有规则，否则这是指未经限定的“边界框”
-   描边边界框（***Stroke Bounding Box***）是包含元素几何形状及其描边形状的边界框
-   装饰边界框（***Decorated Bounding Box***）是包含元素几何形状、其描边形状和其标记的边界框

  


请注意，元素的 `opacity` 、 `fill` 、`fill-rule` 、`fll-opacity` 、`stoke-dasharray` 和 `stroke-dashoffset` 等属性的值对元素的边界框没有影响。

  


因此，当我们向 SVG 文档或元素添加链接或监听事件时，交互区域不一定是一个矩形，它会是一个边界框。换句话说，SVG 文档的哪些部分是可交互的取决于哪些部分是可见的和（或）被绘制的。这意味着，要搞清楚 SVG 元素什么时候可具交互性，就得搞清楚元素什么时候可见和（或）被绘制。

  


通过前面课程的学习，我们知道 SVG 元素可以被“填充”（`fill`），也可以被“描边”（`stroke` 加 `stroke-width`），其中填充指的是形状内部，描边则是指其轮廓。“填充”和（或）“描边”一起就完成了 SVG 图形的绘制操作，你在屏幕（SVG 画布）上就能看到被绘制的图形。换句话说，当我们谈论被绘制的元素时，指的就是该元素具有填充和（或）描边。通常情况之下，这也意味着该元素也是可见的。

  


然而，一个 SVG 元素可以被绘制，但不可见。例如，SVG 图形元素的 `visibility` 属性值为 `hidden` 或者 `display` 属性的值为 `none` ：

  


```XML
<svg viewBox="0 0 1024 1024">

```

  


上面代码中，虽然 `<path>` 元素设置了 `fill="red"` （表示被“填充”）以及 `stroke="lime"` 和 `stroke-width="6"` （表示被“描边”），但两个 `<path>` 元素分别设置了 `visibility="hidden"` 和 `display="none"` ，它们变得不可见（在 SVG 画布上看不到任何图形）。即使如此，但 `<path>` 元素存在并占据理论空间。我们只是看不到它。

  


> 注意，有一点与 HTML 不一样，如果显式给 HTML 元素设置了 `display: none` 样式，该元素不会占据任何物理空间。

  
更加令人困惑的是，一个元素可能是可见的，`visibility` 的计算值是 `visible` ，但却没有被绘制出来。这种情况发生在元素既没有描边又没有填充的情况下。

  


注意，透明通道（Alpha）值为 `0` 的颜色值（例如 ` rgb(0 0 0 / 0)` 或 ` hsl(120 50% 50% / 0)`）不会影响元素是否被绘制或可见。换句话说，如果一个元素的填充和描边颜色的透明度道的值为 `0` ，即使你在 SVG 画布上看不到任何图形，但它实际上也是被绘制的。只不过图形的填充和描边颜色是完全透明的。

  


了解 SVG 元素何时被绘制、可见或两者都不是对于理解 `pointer-events` 属性的每个值的影响至关重要。

  


前面也提到了，`pointer-events` 属性既是 CSS 属性又是 SVG 元素属性，但它们可取的值则有所不同。

  


-   如果 `pointer-events` 作为 [CSS 属性](https://drafts.csswg.org/css-ui/#pointer-events-control)，那么该属性只接受 `auto` 和 `none` 两上值，其中 `auto` 为默认值
-   如果 `pointer-events` 作为 [SVG 元素属性](https://svgwg.org/svg2-draft/interact.html#PointerEventsProperty)，那么该属性可接受的值有 `auto` 、`bounding-box` 、`visiblePainted` 、 `visibleFill` 、`visibleStroke` 、`visible` 、`painted` 、 `fill` 、`stroke` 、 `all` 和 `none` ，其默认值也是 `auto`

  


除 `auto` 值之外，大多数其他值可以分两组：

  


-   需要元素可见的值，即元素的 `visibility`属性值为`visible` ，主要包括 `visiblePainted` 、`visibleFill` 、`visibleStroke` 、`visible`
-   不需要元素可见的值，主要包括 `painted` 、 `fill` 、 `stroke` 和 `all`

  


这里简单介绍一下 `pointer-events` 每个值的含义，使大家对其有更全面认识：

  


-   `auto` ：与`pointer-events` 属性未指定时的表现效果相同，对于 SVG 内容，该值与 `visiblePainted`效果相同
-   `none` ：元素永远不会成为鼠标事件的目标。但是，当其后代元素的 `pointer-events` 属性指定其他值时，鼠标事件可以指向后代元素，在这种情况下，鼠标事件将在捕获或冒泡阶段触发父元素的事件侦听器。
-   `visiblePainted` ：只适用于 SVG。元素只有在以下情况才会成为鼠标事件的目标，即 `visibility` 属性值为 `visible`，且鼠标指针在元素内部（或在元素轮廓上），且`fill` （或 `stroke`）属性指定了 `none` 之外的值
-   `visibleFill` ：只适用于 SVG。只有在元素 `visibility` 属性值为 `visible`，且鼠标指针在元素内部时，元素才会成为鼠标事件的目标，`fill` 属性的值不影响事件处理
-   `visibleStroke` ：只适用于 SVG。只有在元素 `visibility` 属性值为`visible`，且鼠标指针在元素边界（元素轮廓上）时，元素才会成为鼠标事件的目标，`stroke` 属性的值不影响事件处理
-   `visible` ：只适用于 SVG。只有在元素 `visibility` 属性值为 `visible`，且鼠标指针在元素内部或边界（轮廓）时，元素才会成为鼠标事件的目标，`fill` 和 `stroke` 属性的值不影响事件处理
-   `painted` ：只适用于 SVG。元素只有在以下情况才会成为鼠标事件的目标，鼠标指针在元素内部（或元素的轮廓上），且 `fill` （或 `stroke`）属性指定了 `none` 之外的值。`visibility` 属性的值不影响事件处理
-   `fill` ：只适用于 SVG。只有鼠标指针在元素内部时，元素才会成为鼠标事件的目标，`fill` 和`visibility` 属性的值不影响事件处理
-   `stroke` ：只适用于 SVG。只有鼠标指针在元素边界（轮廓）上时，元素才会成为鼠标事件的目标，`stroke` 和 `visibility` 属性的值不影响事件处理
-   `all` ：只适用于 SVG。只有鼠标指针在元素内部或边界（轮廓）时，元素才会成为鼠标事件的目标，`fill`、`stroke` 和 `visibility` 属性的值不影响事件处理

有了这个基础之后，我们可以开始绘制表盘上的分钟和时钟线。在 SVG 中，我们有很多方法可以绘制这些线（分钟线和时钟线）。简单粗暴的方法是使用 `<line>` 元素，一个接一个地绘制每一条线，这种方式在定位上很费时，也易于出错。

  


另一种更有效的方式是，在一个圆（`<circle>`）上设置 `stroke-dasharray` 、`stroke-dashoffset` 和 `pathLength` 属性，将一个实线的圆变成一个虚线的圆。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/741d0af6bcec45038ef525342679beb9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1130&h=688&s=3208670&e=gif&f=290&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/OJGjdvM

  


通过《[初级篇：SVG描边和填充](https://juejin.cn/book/7341630791099383835/section/7349188496181887017)》一课中的学习，知道 `stroke-dasharray` 属性可以设置虚线的方式，例如：

  


```CSS
.ele {
    stroke-dasharray: 10 20;
}
```

  


示例中的 `stroke-dasharray` 属性设置了两个数字，第一个数字将是线段的长度，第二个数字将是间隙的长度。我们在绘制表盘中的分钟线就可以使用这种方式来绘制，即第一个数字是分钟线的长度，第二个是两个分钟线之间的间隔。这两者的总和应该恰好是圆上的一个分钟的长度。

  


这意味着，我们使用 `<circle>` 来绘制钟表盘上的分钟线（标记）。我们知道一小时是 `60` 分钟。所以我们可以计算圆的周长，然后除以 `60` 得到一个分钟的长度。但有一个更好的方法。与其计算圆的周长，我们可以换个方式，我们可以给圆（`<circle>`）设置一个 `pathLength` 属性，它相当于圆的周长。

  


换句话说，我们将 `pathLength` 设置为 `60` ，表示 `60` 分钟。现在，线段和间隙段的总和必须总共为 `1` 。如果将线段设置为 `.2` ，那么间隙就是 `.8` （`1 - .2 = .8`）。为了使整个分钟标记线能更完美一点，需要修复一下虚线的起点，即 使用 `stroke-dashoffset` 属性将其移动半个线段的长度。也就是说，如果 `stroke-shadarray` 的第一个值是 `.2` ，那么 `stroke-dashoffset` 就是 `.1` 。我们可以使用 CSS 自定义属性来描述：

  


```CSS
.clock {
    --dash: .2;
    --gap: calc(1 - var(--dash));
    --stroke-dasharray: var(--dash) var(--gap);
    --offset: calc(var(--dash) / 2);
    --stroke-dashoffset: var(--offset);
}
  
.marker__minute {
    stroke: lime;
    stroke-width: 20;
    stroke-dasharray: var(--stroke-dasharray);
    stroke-dashoffset: var(--stroke-dashoffset);
}
```

  


我们可以使用同样的方式来绘制小时的标记线。为了在视觉上使小时标记线有与分钟标记线有较明显区别，可以考虑将其线段宽度变得更粗一些，即 `stroke-dasharray` 的第一个值更大一点，例如 `.4` 。但要注意，第二个值可不是 `0.6` 。这是因为，小时标记只有 `12` 段，那么每段就是 `5` （即 ` pathLength  `**`÷`**`  12 = 5 ` ），减去第一个值 `0.4` ，就是 `stroke-dasharray` 的第二个值 `4.6` ，即 `stroke-dasharray: .4 4.6` 。

我们期望的是，它是一个真正的时钟。因此，我们需要通过 JavaScript 将时针、分针和秒针旋转到正确的位置。在 JavaScript 中，通过 `document.getElementById` API 选中相应的图形元素。然后创建一个 `Date` 对象，获取当前时间的小时、分钟和秒。最后，根据这些值设置元素的 `transform` 属性：

  


```JavaScript
const hoursElement = document.getElementById("hand__hour");
const minutesElement = document.getElementById("hand__minute");
const secondsElement = document.getElementById("hand__second");

const date = new Date();
const hour = date.getHours();
const minute = date.getMinutes();
const second = date.getSeconds();

hoursElement.setAttribute("transform", `rotate(${(360 / 12) * hour})`);
minutesElement.setAttribute("transform", `rotate(${(360 / 60) * minute})`);
secondsElement.setAttribute("transform", `rotate(${(360 / 60) * second})`);
```

  


我们设置了旋转变换，它需要一个数字。这个数字是 `0 ~ 36` 度之间的旋转角度。对于时针，我们将 `360` 除以 `12` ，得到每小时需要旋转的角度，并乘以当前小时数。这样应该能将时针转向当前小时。

  


对于分针和秒针，我们做同样的事情，只是我们将 `360` 除以 `60` ，因为一小时等于 `60` 分钟，`1` 分钟是 `60` 秒。

  


幸运的是，变换中心默认是原点，即 `0,0`坐标。如果不是这样的话，我们可以设置另一个变换原点，但由于我们的 `viewBox` 设置，我们不需要这样做。

```JavaScript
    secondsElement.setAttribute("transform", `rotate(${(360 / 60) * second})`);

    requestAnimationFrame(animate);
}

requestAnimationFrame(animate);

textElement.addEventListener("mousedown", () => {
    showDate = !showDate;
});
```

## 小结

  


在操作 SVG时，CSS 和 JavaScript 提供了丰富的功能和灵活性。CSS 可用于样式化 SVG 元素，而 JavaScript 则可用于与 SVG 元素进行交互和动态操作。以下是关于如何使用 CSS 和 JavaScript 操作 SVG 的一些总结：

  


-   CSS 操作 SVG：使用 CSS 样式表可以轻松地样式化 SVG 元素，包括填充颜色、边框样式、字体等；可以使用CSS 动画和过渡效果来创建 SVG 动画，从而实现各种视觉效果
-   JavaScript 操作 SVG：使用 JavaScript，可以在 SVG 中添加、删除或修改元素，以及修改其属性和样式；通过 JavaScript 事件处理程序，可以实现与 SVG 元素的交互，例如单击、双击、鼠标悬停等；可以使用 JavaScript 库（如SVG.js、GSAP 和 D3.js 等）来创建复杂的数据可视化图表和图形以及动画效果
-   CSS 和 JavaScript 的结合应用：可以结合 CSS 和 JavaScript 来实现更复杂的 SVG 效果，例如在 JavaScript 中计算数值并将其应用到 SVG 元素的样式中；通过 JavaScript 监听 SVG 元素的事件，并根据用户的操作动态修改 SVG 元素的样式和属性

  


总的来说，CSS 和 JavaScript 提供了强大的工具和技术，可以帮助开发者创建各种各样的交互式和动态的 SVG 图形和动画。熟练掌握这些技术，能够为网页和应用程序增添丰富的视觉效果和用户体验。
