嵌套 SVG 不仅使元素更有条理，还实现了更精准的布局和定位。通过巧妙组织和嵌套 SVG 元素，我们能够打造出更加生动、有趣的图像效果，为设计注入新的灵感和活力。

## 嵌套 SVG 元素

  


在 HTML 中，直接将 `<html>` 嵌套在另一个 HTML 文档中是不合法的，例如：

  


```HTML
<html>
    <head></head>
    <body>
        <div></div>
        <html>
            <head></head>
            <body>
                <div></div>
            </body>
        </html>
    </body>
</html>
```

  


如果你需要将一个 HTML 文档作为另一个 HTML 文档的子文档，你不能只是使用另一个`html`标签来创建一个复合文档。通常的做法是将子文档以 `<iframe>` 的形式嵌套在父文档中：

  


```HTML
<html>
    <head></head>
    <body>
        <iframe></iframe>
    </body>
</html>
```

  


与 HTML 不同，SVG 中的根元素 `<svg>` 可以相互嵌套。例如：

  


```XML
<svg viewBox="0 0 800 600" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
    <rect width="100" height="100" fill="lime" />
    
    <svg>
        <circle cx="50%" cy="50%" r="10%" fill="orange" />
    </svg>
</svg>
```

  


在使用嵌套 SVG 时，有几个事项需要注意：

  


-   内部的 `<svg>` 元素不需要显式指定命名空间 `xmlns` 或 `xmlns:xlink` ，因为它被假定为与外部 `<svg>` 的命名空间相同。当 SVG 内联嵌套在 HTML 文档中时，外部的 `<svg>` 元素也不需要命名空间。
-   在 SVG 中会使用 `<g>` 元素对 `<svg>` 内部的元素进行分组，并使用 `transform` 属性在画布上定位它们。然而，你也可以使用嵌套的 `<svg>` 来分组元素，并在父 `<svg>` 内部定位它们。这种方式比使用 `<g>` 元素更有一些优势。大多数情况下，通过元素的 `x` 和 `y` 坐标来定位比使用 `transform` 更方便。此外，`<svg>` 元素接受 `width` 和 `height` 属性，而 `<g>` 元素不具备这些属性。
-   通过为 `<svg>` 指定宽度（`width`）和高度（`height`），就相当于将其内容限制在由 `x`、`y`、`width` 和 `height` 属性定义的视口边界内。超出这个边界的任何内容都将被裁剪。如果不指定 `x` 和 `y` 属性，它们会默认为 `0`。如果不指定 `width` 和 `height`，内部的 `<svg>` 元素将会自动匹配其父级 `<svg>` 元素的宽度和高度。
-   在内部 `<svg>` 上指定的百分比值，其内部元素将相对于该 `<svg>` 进行计算，而不是相对于根 `<svg>` 。在内部 `<svg>` 上指定的百分比值，它自身将相对于根 `<svg>` 进行计算。

  


特别声明，接下来我们所说的嵌套 SVG 元素，指的是 `<svg>` 元素的相互嵌套！

  


## 嵌套 SVG 元素的作用

  


嵌套 SVG 元素最主要的作用之一是为元素创建新的视口。在 SVG 中，默认情况下只使用一个坐标系来控制视口内元素的位置和大小，即由 SVG 的 `viewBox` 属性建立的坐标系，也就是当前使用的坐标系。

  


然而，技术上来说，SVG 中存在两个默认的坐标系。但是，除非你明确更改了它们的值，否则在处理 SVG 时只有一个坐标系是相关的。如果你对 SVG 坐标系及其建立和使用方法不太熟悉，我强烈建议在继续学习之前，重新阅读小册的《[初级篇：SVG 坐标系统](https://juejin.cn/book/7341630791099383835/section/7345677438053810214)》和《[中级篇：SVG 坐标系变换](https://juejin.cn/book/7341630791099383835/section/7357982293367422991)》！

  


换句话说，SVG 中的操作都是在这个视口中完成的，比如设置图形元素的尺寸大小和位置等。在这方面，与 HTML 相似：

  


-   尺寸大小：HTML 元素由盒模型来决定，SVG 元素则由对象边界框来决定
-   位置（定位）：HTML 元素由 `position` 属性（非 `static` 值情况下）和偏移属性（`top`、`right` 、`bottom` 、`left` 或 `inset`）来决定，SVG 元素则由当前使用的坐标系来决定

  


相比之下，定位方面存在一些差异。

  


对于 HTML 元素，当元素的 `position` 值从默认的 `static` 更改时，它要么为其后代创建定位上下文，要么为自身创建定位上下文。每次默认值更改时，都需要一个定位上下文来指定元素将如何在默认页面内容流之外定位。

  


如果要将元素从页面的内容流中移除，可以通过绝对定位来实现。绝对定位意味着它将相对于其祖先之一定位，使用该祖先的框作为定位上下文。然而，每个定位上下文都需要坐标系。坐标系是由元素的盒模型的尺寸（宽度和高度）建立的。然后，元素的任何后代将使用该坐标系在元素内部和相对元素定位。

  


对于 SVG 元素，SVG 中的每个元素都是相对于 SVG 视口“绝对”定位的，视口内的位置由当前使用的坐标系控制。然而，SVG 元素与 HTML 元素不同，单个元素没有盒模型这个概念，因此也没有自己的坐标系，可用作其他元素的定位上下文。因此，如果想要将 SVG 元素相对于另一个 SVG 元素或一组元素进行定位，就需要使用嵌套 SVG。

  


嵌套 SVG 除了更好地使 SVG 元素进行相对定位外，还在 SVG 画布上添加了灵活性和流动性，这是非常有用的。

  


我们知道，使用 `viewBox` 和 `preserveAspectRatio` 可以创建响应式 SVG。最外层的 `<svg>` 的宽度可以设置为 `100%`，以确保它随着其容器（或页面）的增长或缩小而扩展或收缩。然后，通过使用 `viewBox` 和 `preserveAspectRatio` ，我们可以确保 SVG 画布也响应视口（最外层 `<svg>`）的变化。

  


然而，当我们像这样使用 SVG 响应时，绘制在整个画布上的元素都会同时响应和更改。有时，你可能只希望图形内部的一个元素是灵活的，而保持其他元素的位置和大小“固定”。这就是嵌套 SVG 可以发挥作用的地方。

  


一个 `<svg>` 元素可以拥有独立于其父级的坐标系统，并且可以拥有自己的 `viewBox` 和 `preserveAspectRatio` 属性，允许你以任何你想要的方式调整内容的大小和位置。

  


因此，为了使一个元素灵活，我们可以将其包装在一个 `<svg>` 元素中，并给该 `<svg>` 一个灵活的宽度，使其根据外部最外层 SVG 的宽度进行调整，然后指定 `preserveAspectRatio="none"`，使其内部的图形也随容器宽度的拉伸和收缩而伸展和收缩。

```HTML
</div>
```

  


这样做是为了确保容器变小时，`#eggshell--upper` 会向上移动并粘附到视口的顶部中心，因为它的 `preserveAspectRatio` 属性值为 `xMidYMin meet`；`#eggshell--lower` 会向下移动并粘附到视口的底部中心，因为它的 `preserveAspectRatio` 属性值为 `xMidYMax meet` 。

  


简言之，通过在每个内部的 `<svg>` 元素上使用不同的 `preserveAspectRatio` 值，我们可以确保每个 SVG 内容（即每个“图层”的内容）都与根 SVG 的任一边对齐，从而实现了容器缩小时小鸡破壳而出的效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/66eb4387f2d940809bd501e9827a5e16~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1062&h=652&s=4745472&e=gif&f=259&b=030303)

通过嵌套 SVG 元素，我们可以在根 `<svg>` 内创建单独的“图层”，并控制这些图层的内容。随着视口大小的变化，这些图层的内容在根 SVG 内部发生位置变化。通过这种方式，我们可以根据需要在 SVG 内部显示和隐藏不同部分的内容。

  


这个案例展示了嵌套 SVG 元素的多种应用：你可以从一个在小屏幕上的图形开始，然后随着屏幕的变大，逐渐揭示出新的内容。

  


另外，需要注意的是，嵌套的 SVG 不需要与其包含的 SVG 具有相同的高度和宽度。你可以指定一个高度和宽度，并使 SVG 的内容受到该高度和宽度所规定的边界的限制和裁剪。这完全取决于你想要实现的效果。

### 用于 SVG 中的相对定位

  


在 SVG 中，内部 SVG 的内容是相对于该 `<svg>` 元素本身定位，而不是相对于根 SVG 元素。这意味着，嵌套的 SVG 元素可以让我们将一个元素相对于另一个非 SVG 元素进行定位。

  


然而，在探讨这个问题之前，我们需要先了解 SVG 元素的对象边界框是什么。

  


SVG 元素与 HTML 元素并不完全相同。SVG 之所以强大，是因为其基本形状使我们能够创建各种非矩形形状，从任意路径到开放或封闭的折线和多边形，再到圆和椭圆。然而，CSS 盒模型仅适用于 HTML 元素，对 SVG 元素没有任何作用。为了弥补这一缺失，SVG 引入了对象边界框的概念。

  


[SVG 规范对元素的对象边界框进行了描述](https://www.w3.org/TR/SVGTiny12/coords.html#BoundingBox)：“SVG 元素的对象边界框（或简称为“bbox”）是相对于该元素的用户坐标系轴对齐的最紧密的矩形，完全包围它及其后代元素的区域。”

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/72006867b39c46d689675bbf212e5ff2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1361&s=259211&e=jpg&b=0b0b27)

  


上图中，白色虚线框表示了围绕着蓝色形状的最小矩形，因此是蓝色形状边界框的可视表示。

  


在诸如 Figma 这类图形编辑软件中，选中图形元素，你可以检索元素对象边界框的相关属性，例如 `x` ，`y` ，`width` 和 `height` 等。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c4f2a3c659d6472eacad76f443fcda63~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2876&h=1788&s=316000&e=jpg&b=f7f7f7)

  


SVG 图形对象边界框的计算与 CSS 盒模型计算有所不同。在计算边界框时，需要考虑填充（`fill`）、描边（`stroke`）、不透明度（`opacity`）或可见性（`visibility`）等属性的影响。对于曲线形状，边界框必须包括沿着边缘的所有部分，而不仅仅是端点。需要注意的是，曲线的控制点（例如，三次贝塞尔命令的第二个坐标对）不应影响边界框的尺寸，尽管这些点可能位于边界框的区域内，如果它们位于形状本身内部或沿着曲线附近。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b42663925dc445b6aaf6df6847c53e4a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=979&s=412023&e=jpg&b=050505)

  


边界框适用于任何具有正 `width` 或 `height` 属性以及 `display` 属性不为 `none` 的渲染元素，以及可能包含这些元素的容器元素。不参与渲染树的元素（例如位于 `<defs>` 元素中的元素，其 `display` 为 `none` 的元素等），以及没有参与渲染树的子元素（例如没有子元素的 `<g>` 元素）不应对父元素的边界框产生影响。尽管这些元素不对父元素边界框做出贡献，但在需要时仍应返回自己的边界框值。

  


SVG 元素对象具有三种不同类型的边界框：

  


-   **对象边界框（Object Bounding Box）** ：仅包含元素的几何形状的边界框。
-   **描边边界框（Stroke Bounding Box）** ：包含元素的几何形状以及其描边的形状的边界框。
-   **装饰边界框（Decorated Bounding Box）** ：包含元素的几何形状、其描边的形状以及其标记（`<marker>`）的形状的边界框。

通过使用元素的边界框，我们可以想象在该元素周围存在一个坐标系，然后利用这个坐标系来定位其他元素。具体来说，我们会创建并利用一个内部的 `<svg>` 元素来建立一个新的坐标系，这个 `<svg>` 元素的属性将由元素的边界框的 `x`、`y`、`width` 和 `height` 等属性来定义。

  


接下来，我们来看一个小鸟归巢的示例。假设你有一个像下面这样的，带有鸟和巢图形的 SVG：

通常情况下，我们可以通过使用整个 SVG 画布的坐标系在鸟巢上方指定鸟的位置。比如说，使用上一节的坐标变换的相关的知识，给小鸟应用一个平移的 `transform` ，就可以将小鸟归巢：

在理想情况下，我们希望能够通过百分比值来指定相对于鸟巢“盒子”（边界框）的位置。我们可以通过创建新的 `<svg>` 元素围绕鸟巢来模拟这一点。这个 `<svg>` 元素具有其宽度和高度确定的坐标系，我们将利用这个坐标系来弥补鸟巢缺失的坐标系。

  


然后，我们将鸟移动到 `<svg>` 标签中。由于被包含在 `<svg>` 中，鸟的位置将相对于在该 `<svg>` 上建立的坐标系来计算。

  


为了在鸟和鸟巢元素之间创建相对关系，我们需要给鸟创建一个定位上下文，这就是我们的嵌套 `<svg>` ，它类似于鸟巢周围的坐标系。

  


为了实现这一点，我们在视觉上将 `<svg>` 定位在鸟巢的顶部。需要注意的是，内部 SVG 实际上并不包裹鸟巢，而是鸟巢的图形元素被包裹在另一个独立的 `<svg>` 元素中，或者依旧是一个 `<g>` 元素。我们只是将鸟（`<svg>`）在视觉上定位在鸟巢的顶部，使得它看起来像是鸟巢的坐标系的视觉表示。

  


为了确定 `<svg>` 的确切位置（其在根 `<svg>` 内的 `x` 和 `y` 位置）以及其尺寸，我们将使用鸟巢的边界框属性。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df1f5c3eeabf4f9dae734d8a739e5198~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1063&s=381427&e=jpg&b=050505)

  


也就是说，使用单独的 `<svg>` 元素来包裹小鸟，并且使用前面提到的 `getBBox()` 方法来获得鸟巢边界框属性的值，将小鸟的 `<svg>` 元素的 `x` 、`y` 、`width` 和 `height` 属性的值设置为鸟巢边界框的 `x` 、`y` 、`width` 和 `height` 属性值：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container">
        <g id="bird">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


```JavaScript
const birdSVG = document.querySelector('.bird--container');
const bbox = document.querySelector('#bbox');

const x = Math.round(nestBbox.x);
const y = Math.round(nestBbox.y);
const width = Math.round(nestBbox.width);
const height = Math.round(nestBbox.height);

birdSVG.setAttribute('x', x);
birdSVG.setAttribute('y', y);
birdSVG.setAttribute('width', width);
birdSVG.setAttribute('height', height);
```

  


执行上面的代码之后，小鸟 `<svg>` （类名为 `.bird--container`）元素将会建立一个围绕着鸟巢的新坐标系统：

  


```XML

        <!-- 绘制鸟巢的图形内容 -->
    </g>

    <svg class="bird--container" x="699" y="1220" width="1055" height="641">

            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

```CSS
}
```

现在，小鸟所在的坐标系与鸟巢边界框是相匹配的。但要让小鸟站在鸟巢上，我们还需要在新的坐标系内部对小鸟进行定位。即，我们将在 `svg.bird-container` 内部定位小鸟（`g#bird`）。但问题是，SVG 的 `<g>` 元素是没有 `x` 和 `y` 属性。因此我们并不能通过 `x` 和 `y` 将小鸟（`g#bird`）移动到特定的位置。

  


当然，此时你可能依然会想到 SVG 的 `transform` 或 CSS 的 `transform` 以及 `translate` 等变换，将小鸟平移到鸟巢上（指定的位置）。是的，使用它们，例如 指定SVG 变换中的 `translate()` 函数的 `x` 和 `y` 值，将小鸟平移到指定位置。

  


然而，我们不得不面临新的问题。通常情况下，我们可以指定相对于元素的边界框来移动元素。例如，如果我们将一个元素向右移动 `50` 个用户单位，我们可以将 `transform` 属性设置为 `translate(50,0)` 。这将元素相对于其边界框在水平方向上向右平移 `50` 个用户单位。

  


但是，由于我们的边界框在新坐标系中，而不是根坐标系中，因此这种方法不适用于我们。我们需要一种方法来指定移动的实际值，而不是相对于边界框。

  


为了解决这个问题，我们可以使用边界框的位置属性：`x` 和 `y` 值。这将允许我们将元素放置在 `<svg>` 的左上角或右下角，然后移动到我们想要的位置。

  


因此，我们首先将小鸟 `g#bird` 放置在 `svg.bird--container` 的左上角。同样的，我们可以使用 `getBBox()` 方法获取小鸟（`g#bird`）边界框相关的信息，然后使用 `translate()` 将其放置在 `svg.bird--container` 的左上角。示例代码如下：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container">
        <g id="bird">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


```CSS
.bird--container {
    overflow: visible; /* 这个很重要 */
}
```

当然，要是你不太喜欢通过 SVG 的变换属性来移动元素，你可以将包裹小鸟的 `<g>` 元素替换成 `<svg>` 元素：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container" x="698" y="1220" width="1055" height="640" viewBox="0 0 1055 640">
        <svg id="bird" viewBox="0 0 1055 641" width="1055" height="640">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


因为 `<svg>` 元素支持 `x` 和 `y` 属性。注意，在这里新创建的 `svg#bird` 元素，它只作为一个容器，即使它创建了一个新的坐标系，我们也不会使用该坐标系。另外，我们将 `svg#bird` （包裹小鸟图形的 `<svg>` 元素）与其父容器 `svg.bird--container` （根据鸟巢边界框信息创建的一个新坐标系）元素具有相同的尺寸（`width` 和 `height`）和 `viewBox`。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/22a77e369a4d43fdba7de48578cda626~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1133&s=314138&e=jpg&b=060606)

  


不难发现，小鸟图形左上角与坐标系原点有一定的偏移距离，这个偏移距离等于小鸟图形边界框的 `x` 和 `y` 属性的值。你可以通过 `getBBox()` 函数来获得：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b75e96aea8f34bfb9ebab06077d5e9c1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1133&s=327453&e=jpg&b=060606)

  


这意味着，我们要移动小鸟到鸟巢之上，就需要考虑到这个偏移距离。因此，如果我们想将小鸟图形移动到坐标系的左上角，我们不能简单地将 `x` 和 `y` 设置为 `0` ：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container" x="698" y="1220" width="1055" height="640" viewBox="0 0 1055 640">
        <svg id="bird" x="0" y="0" viewBox="0 0 1055 641" width="1055" height="640">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


这样做并不会带来任何实质性的变化，因为 `<svg>` 元素的 `x` 和 `y` 属性的默认值就是 `0` 。这个时候你可能会想到将 `x` 和 `y` 属性设置负的偏移量：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container" x="698" y="1220" width="1055" height="640" viewBox="0 0 1055 640">
        <svg id="bird" x="-157" y="-235" viewBox="0 0 1055 641" width="1055" height="640">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0051af856ba3493ba4df31d6881003f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1133&s=241705&e=jpg&b=060606)

  


设置负偏移量，的确消除了小鸟图形与坐标系原点的空白距离。不过，这样做并不是较好的解决方案，因为我们接下来在定位小鸟的位置时，不得不考虑这个偏移量。

  


其实，在 SVG 中，我们有一种更合理，更好的方式来消除小鸟图形与坐标系原点之间的偏移距离，那就是更改 `svg#bird` 的 `viewBox` 属性值。我们可以将小鸟图形边界框的 `x` 和 `y` 值分别赋予给 `viewBox` 属性的 `<min-x>` 和 `<min-y>` ：

  


```XML
<svg  xmlns="http://www.w3.org/2000/svg" width="100%" viewBox="0 0 3945.8 2400" class="bird">
    <!-- 鸟巢 -->
    <g id="bbox" class="nest">
        <!-- 绘制鸟巢的图形内容 -->
    </g>
    
    <!-- 鸟 -->
    <svg class="bird--container" x="698" y="1220" width="1055" height="640" viewBox="0 0 1055 640">
        <svg id="bird" x="0" y="0" viewBox="157 235 1055 641" width="1055" height="640">
            <!-- 绘制鸟的图形内容 -->
        </g>
    </svg>
</svg>
```

  


这将使小鸟被定位在新建立的坐标系的左上角。所以，现在小鸟位于容器的左上角，此时只需要调整 `svg#bird` 元素的 `x` 和 `y` 属性的值，你就可以将小鸟移动到任意你想要的位置上：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/929b7d9d2222429da39e8c566aa53f6c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1226&h=756&s=1836746&e=gif&f=296&b=040404)

在 SVG 中，要定位元素需要一些特殊的思考方式，因为 SVG 元素没有像 HTML 元素那样的盒模型。在 SVG 中，元素的位置通常是相对于 SVG 视口而不是相对于父元素的。为了在 SVG 内部定位元素并创建相对定位的上下文，我们可以利用嵌套 SVG，并利用相对元素的边界框属性来建立新的坐标系。

  


通过嵌套 SVG 元素，我们可以在 SVG 内部创建新的坐标系，这些坐标系可以作为元素的定位上下文。这意味着我们可以模拟将元素放置在其他元素之上或之下的效果，并且可以使用内部 SVG 元素的边界框属性来确定元素的位置。

  


## 创建新视口的其他姿势

  


在 SVG 的世界中，嵌套 SVG 并不是创建新视口的唯一方式。我们还可以通过以下方式之一来创建新的视口：

  


-   使用 `<use>` 元素引用 `<symbol>` 实例来创建新视口
-   使用 `<image>` 元素引用 SVG 图像来创建新视口
-   使用 `<iframe>` 元素创建新视口
-   使用 `<foreignObject>` 元素创建新视口

  


### 使用 use 元素引用 symbol 实例来创建新视口

  


在 SVG 中，我们通常使用 `<symbol>` 元素来创建一个图形的实例，比如一个 Icon 图标。例如：

  


```XML
<svg  class="sr-only">
    <defs>
        <symbol id="walk" viewBox="0 0 1024 1024">
            <path d="M602.453 426.667h208.214v-76.8H656.213l-85.333-142.08c-12.8-21.334-35.84-35.84-62.293-35.84-7.68 0-14.507 1.28-21.334 3.413L256 247.467v221.866h76.8V312.747l90.027-28.16L256 938.667h76.8L455.253 592.64l99.414 132.693v213.334h76.8V665.173l-106.24-193.706 31.146-122.454m40.96-186.88c42.667 0 76.8-34.133 76.8-76.8s-34.133-76.8-76.8-76.8-76.8 34.134-76.8 76.8 34.134 76.8 76.8 76.8z"/>
        </symbol>
    </defs>
</svg>    
```

  


上面代码中，我们使用 `<symbol>` 元素创建了一个名为 `walk` 的实例。事实上， `<symbol>` 元素在创建 `walk` 这个实例时，就定义了一个新的视口。

  


实例化之后，我们可以通过 `<use>` 元素的 `href` 属性来引用它：

  


```XML
<svg class="walk">
    <use href="#walk" x="❓" y="❓" width="❓" height="❓" />
</svg>
```

  


上面代码中的问号（❓）表示可用于 `<use>` 元素的这些属性的值可能被指定或者不被指定。如果 `x` 和 `y` 未被指定，它们默认为 `0` ，而且 `width` 和 `height` 也可以不显式指定：

  


```XML
<svg class="walk">
    <use href="#walk" />
</svg>
```

  


需要知道的是，`<symbol>` 元素创建的实例，不通过 `<use>` 元素引用的话，它是不会被渲染的。换句话说，只有被 `<use>` 引用的 `<symbol>` 实例才会在 SVG 画布中呈现。此时，要是你通过浏览器开发者调试工具对 `<use>` 元素引用的实例进行审查时，你看不到 `<use>` 标签内的 `<symbol>` 内容。这是因为 `<use>` 的内容被渲染到一个 Shadow DOM 中，只有你在开发者工具中启用了检查 Shadow DOM，你才可以查看到它。

  


也就是说，当 `<symbol>` 被使用时，它被深度克隆到生成的 Shadow DOM 中，唯一的例外是 `<symbol>` 被一个 `<svg>` 替换：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a5deac8e12e424e81714091bb16ec3e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2564&h=1278&s=318227&e=jpg&b=050505)

  


如果在 `<use>` 元素上显式设置了 `width` 和 `height` 属性的值，那么这些属性将被传递到生成的 Shadow DOM 中的 `<svg>` 元素上：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e15e7c3024db4aafb7bbce48f729e719~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2546&h=1048&s=326429&e=jpg&b=060606)

  


如果 `<use>` 元素上没有显式设置 `width` 和 `height` 属性的值，那么 Shadow DOM 中的 `<svg>` 元素的 `width` 和 `height` 属性的值则为 `100%` 。

  


可以看到，`<use>` 元素引用的 `<symbol>` 创建的实例最终都会在 DOM 中生成一个 `<svg>` 元素，只不过该元素存在于 Shadow DOM 中。这也意味着，它创建了一个新的视口，与嵌套 SVG 有所不同。这个 SVG 实际上是包含在使用 `<use>` 的外部 SVG 元素中的。

  


因此，现在我们有了一个新的视口，其尺寸和位置可以通过 `<use>` 元素的 `x`、`y`、`width` 和 `height` 属性指定，并且其 `viewBox` 值也可以在 `<symbol>` 元素上指定。然后 `<symbol>` 的内容将在这个视口和 `viewBox` 内部被渲染和定位。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fb3850eb0486439cbf5cab65f05c1c51~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2552&h=1100&s=340496&e=jpg&b=050505)

  


最后但同样重要的是，`<symbol>` 元素还接受一个 `preserveAspectRatio` 属性值，允许你在由 `<use>` 建立的新视口内部定位 `viewBox` 。例如下面这个示例，当 `<use>` 元素的宽高比与 `<symbol>` 的 `viewBox` 宽高比不相等时，那么 `<symbol>` 元素上的 `preserveAspectRatio` 属性值将会影响到图形元素的位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/686c76e40756431b8ec0b601a76469fd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1116&h=684&s=3110074&e=gif&f=412&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/mdgaoMb

  


### 使用 image 元素引用 SVG 图像来创建新视口

  


`<image>` 元素可以引用位图（比如 PNG 或 JPG），也可以引用 MIME 类型为 `image/svg+xml` 的文件。与 `<use>` 元素类似，我们也可以通过 `<image>` 元素引用 SVG 文件的图像元素来创建一个新视口，因为引用的 SVG 图像（`.svg`）中也会有一个 `<svg>` 元素。

  


```XML
<svg>
    <image href="walk.svg" x="❓" y="❓" width="❓" height="❓" preserveAspectRatio="❓" />
</svg>
```

  


SVG 的 `<image>` 元素接受许多属性，其中 `x` 、`y` 、`width` 、`height` 和 `preserveAspectRatio` 属性与新创建的视口相关联。

  


通常情况之下，一个独立的 `.svg` 文件至少会有一个 `<svg>` 根元素，这个根元素可能只显式设置了一个 `viewBox` 和 `preserveAspectRatio` 属性的值，其 `x` 、`y` 、`width` 和 `height` 属性都有可能未显式设置（也有可能会显式设置）。

  


需要知道的是，当 `<image>` 元素引用 `.svg` 文件时，那么该 SVG 文件中根元素 `<svg>` 的 `x` 、`y` 、`width` 和 `height` 属性将会被忽略。另外，如果 `<image>` 元素的 `preserveAspectRatio` 属性的值不是以 `defer` 开头，那么将会忽略引用的 `.svg` 文件中根元素 `<svg>` 上的 `preserveAspectRatio` 属性。相反，引用 `<image>` 元素上的 `preserveAspectRatio` 属性定义了如何将 SVG 图像内容适配到视口中。

  


对于 `viewBox` 属性，如果 `<image>` 元素引用的 `.svg` 文件的根元素 `<svg>` 显式设置了 `viewBox` 属性，那么将会使用该值。对于大多数位图内容，应该使用图像的边界（即，图像元素具有隐式的 `viewBox` ，它的值为 `0 0 位图宽度 位图高度`）。当没有值可用时（例如，引用的 `.svg` 文件的根元素 `<svg>` 未显式设置 `viewBox` 属性），那么将会忽略 `preserveAspectRatio` 属性，并且仅使用由视口的 `x` 和 `y` 属性导致的平移来显示内容。

  


### 使用 iframe 元素来创建新视口

  


使用 `<iframe>` 元素引用 `.svg` 文件是另一种将 SVG 应用于 Web 的方法。它类似于 `<image>` 元素引用 `.svg` 文件的情况，会创建新的视口。`<iframe>` 元素还可以有 `x` 、`y` 、`width` 和 `height` 等属性，但 `preserveAspectRatio` 除外。

  


### 使用 foreignObject 元素来创建新视口

  


除了前面所提到的方法之外，还可以使用 SVG 的 `<foreignObject>` 元素为其中的内容创建一个新的视口。该元素允许你将非 SVG 内容添加到 SVG 文件中。通常，`<foreignObject>` 元素的内容被假定来自不同的命名空间。例如，你可以在 SVG 元素中插入一些 HTML 元素：

  


```XML
<svg viewBox="0 0 200 200" xmlns="http://www.w3.org/2000/svg">
    <polygon points="5,5 195,10 185,185 10,195" />

    <foreignObject x="20" y="20" width="160" height="160">
        <div xmlns="http://www.w3.org/1999/xhtml">Lorem ipsum dolor sit amet, consectetur adipiscing elit...</div>
    </foreignObject>
</svg>
```

  


`<foreignObject>` 元素也接受 `x` 、`y` 、`width` 和 `height` 等属性，用于定位对象并调整大小，创建用于呈现内部引用内容的边界。

  


## 小结

  


嵌套 SVG 是一种在 SVG 文件内部创建新的坐标系和视口的方法。通过嵌套 SVG，可以在一个 SVG 文件中创建多个独立的图形或图像，并将它们定位在不同的位置和大小。以下是关于嵌套 SVG 的一些要点：

  


-   **创建新视口**： 嵌套 SVG 允许在一个 SVG 文件内部创建新的视口。每个嵌套的 SVG 元素都可以视作一个独立的视口，其大小和位置可以通过元素的属性来指定。
-   **建立新的坐标系**： 每个嵌套的 SVG 元素都建立了一个新的坐标系。在这个坐标系内部，元素的位置和大小是相对于该 SVG 元素的，而不是相对于父级 SVG 元素或视口的。
-   **相对定位上下文**： 嵌套 SVG 创建了一个相对定位上下文。在这个上下文中，可以使用坐标系来相对于嵌套的 SVG 元素定位其他元素，从而实现相对于该元素的定位效果。
-   **多层嵌套**： 嵌套 SVG 可以多层次嵌套，即在一个嵌套的 SVG 元素内部再次嵌套另一个 SVG 元素。这样可以创建复杂的图形结构和层次关系。
-   **灵活性和控制性**： 嵌套 SVG 提供了更多的灵活性和控制性，可以更精确地控制图形元素的位置、大小和行为。通过嵌套 SVG，可以实现更复杂和多样化的图形效果。

  


总的来说，嵌套 SVG 是在 SVG 文件中创建新的视口和坐标系的有效方式。它提供了更多的灵活性和控制性，使得可以创建复杂的图形结构和实现更丰富的图形效果。
