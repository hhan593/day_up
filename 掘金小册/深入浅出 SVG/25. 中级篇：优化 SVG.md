随着现代 Web 技术的不断进步，SVG 作为一种图像格式，以其优越的可缩放性和无损性，已成为 Web 设计的基石。

然而，尽管 SVG 在各方面表现出色，未优化的 SVG 文件却可能导致加载缓慢、渲染效率低下，甚至影响整体用户体验。因此，在确保视觉效果的同时，如何最大限度地优化 SVG，已成为每个开发者必须掌握的重要技能。

## 为什么要优化 SVG

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aab02a3c85b347ee8eff9f48b796a5d5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=25828&e=png&b=9971ea)


在当今快速发展的互联网时代，用户对网站的性能和体验有着越来越高的要求。SVG 作为一种常用的图像格式，以其卓越的可缩放性和无损性，得到了广泛应用。然而，即使是功能强大的 SVG，如果未进行优化，也可能带来一系列问题。为什么呢？

因为，应用在 Web 上的许多复杂的 SVG 图像并不是由 Web 开发者手动编码实现的，而是由设计师使用诸如 Adobe Illustrator、Sketch、Inkscape、Figma 等设计工具绘制的。尽管这些图像可以在各种分辨率下保持高清无损的效果，但从这些工具导出的 SVG 文件可能包含大量冗余数据，例如多余的属性、注释和未使用的定义。

正如你所看到的，导出的 SVG 代码中包含了许多与设计软件相关的信息和数据，以及其他不影响视觉呈现的元素。这些多余的数据增加了文件的大小，导致加载时间延长和带宽使用量增加。较大的文件需要更多的时间来下载和渲染，从而影响整体用户体验，并可能因为页面加载速度较慢而降低搜索引擎的排名。


需要注意的是，这种现象在不同的设计软件中会有所不同。例如，Sketch 设计软件导出的 SVG 代码通常会像下面这样：

```XML
        </g>
    </g>
</svg>
```


上述的 SVG 文件中包含了大量关于 Sketch 的信息，如图标的标题和描述等。


与 Inkscape 类似，Adobe Illustrator 设计软件保存的 SVG 文件或导出的 SVG 代码通常也会包含大量臃肿且无用的内容：



相较而言，Figma 设计软件导出的 SVG 代码要清晰许多。举例来说，与 Inkscape 生成的同一张 SVG 图形相比，使用 Figma 导出的 SVG 代码更为整洁：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1a9ba918c73f4160b19b889a006c429f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3348&h=1652&s=2580266&e=jpg&b=25292e)


如你所见，每个图形软件都有其独特的方式保存 SVG 文件，但不论你使用哪个程序，若不加以优化，它们都可能存在一些共同的问题：


-   冗余数据：许多 SVG 文件中存在开发过程中遗留的冗余信息，如注释、多余的属性等，这些数据显著增加了文件大小。
-   未合并的路径：一些复杂的 SVG 可能包含多个重复或相似的路径，未进行路径合并会增加渲染负担。
-   不必要的元数据：SVG 文件有时包含了不必要的元数据，例如创作软件的信息，这些数据对实际展示并无帮助。
-   未压缩的文件：SVG 文件本质上是 XML 格式，未经过压缩处理的文本数据会占用大量存储空间。


这些问题也直接导致了 SVG 文件大小的明显差异：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3f4fae36a78c4a63aca3966567577871~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1618&h=1056&s=254372&e=jpg&b=362b27)



这就是未经过优化的 SVG 文件（特别是通过设计软件导出的 SVG 文件）会带来一系列问题的原因所在。

另外，优化 SVG 不仅可以减少文件大小，还能提高网页的性能和用户体验，具体体现在以下几个方面：

-   **提高加载速度**：SVG 文件未经过优化可能包含大量冗余数据，如不必要的注释、空白符、多余的代码等，这些都会增加文件大小。过大的 SVG 文件会导致网页加载速度变慢，尤其是在网络条件较差的情况下。**优化 SVG 可以显著减少文件大小，从而提高网页的加载速度，提升用户体验**。

-   **提升渲染性能**：复杂的 SVG 图形可能包含大量的路径、节点和嵌套元素，增加浏览器渲染的负担。这会导致页面渲染变慢，特别是在处理大量 SVG 或高分辨率图像时。**通过优化，可以减少 SVG 的复杂度，降低渲染成本，提高页面的整体性能**。
-   **改善用户体验**：现代用户对网页的期望不仅是功能齐全，还要体验流畅。如果一个页面因为未优化的 SVG 文件导致加载缓慢或操作迟钝，用户很可能会感到沮丧并离开网站。**优化 SVG 可以确保图像在各种设备和浏览器上快速加载和高效渲染，从而提供更好的用户体验**。
-   **减少带宽消耗**：对于流量有限或网络状况不佳的用户而言，较小的文件尺寸可以显著减少带宽消耗。**优化后的 SVG 文件不仅能降低服务器的带宽使用，还能为用户节省数据流量，尤其是在移动端环境中，这一点尤为重要**。
-   **提高 SEO 和可访问性**：优化 SVG 文件不仅仅是减小文件大小，还包括提升其可读性和语义化。**通过清理和优化 SVG 代码，可以使搜索引擎更容易理解图像内容，从而提升 SEO 排名**。同时，**优化的 SVG 也能更好地支持屏幕阅读器等辅助工具，提高网页的可访问性**。
-   **更好的维护性和可扩展性**：未优化的 SVG 文件通常混乱且冗长，不利于后期的维护和更新。**优化后的 SVG 文件结构清晰、语义明确，便于开发者理解和修改**。此外，**优化后的文件也更易于集成到各种开发框架和工具中，提升项目的可扩展性**。


也就是说，优化 SVG 是一个多方面的需求，不仅能提升网页的加载速度和渲染性能，还能改善用户体验，减少带宽消耗，提高 SEO 和可访问性，并为后期维护和扩展提供便利。因此，掌握 SVG 优化技术，已成为每个 Web 开发者必须具备的重要技能。





## 优化 SVG 的工具和方法

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5b289bf600f64f2eb22446e8e53b128d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1100&s=455617&e=png&b=f1f1f9)


上一节，我们讨论了 SVG 需要进行优化的诸多原因。接下来，我们一起来探讨如何借助工具和一些方法有效地优化 SVG。

首先从常用的优化工具开始。

我们在进行 Web 开发的过程中，有一些工具可以帮助我们快速、简便对 SVG 进行优化。我个人常用的 SVG 优化工具有：

-   **[SVGO](https://svgo.dev/)**：一个强大的开源工具，基于 Node.js 构建，专门用于优化 SVG 文件。它通过删除冗余数据和不必要的代码来显著减少文件大小。它提供了多种不同的使用方式，可以通过命令行或 GUI （[SVGOMG](https://jakearchibald.github.io/svgomg/)）对 SVG 进行优化，也可以集成到 [Webpack](https://github.com/webpack-contrib/image-minimizer-webpack-plugin/#optimize-with-svgo)、[Gulp](https://github.com/cssnano/cssnano/tree/master/packages/postcss-svgo)、[Vite](https://www.npmjs.com/package/vite-plugin-image-optimizer) 等构建工具中对 SVG 进行优化，同时还有 [nuxt-svgo](https://nuxt.com/modules/nuxt-svgo) 等版本，甚至还为常见设计软件提供相应插件，如 [Sketch](https://github.com/BohemianCoding/svgo-compressor)、[Figma](https://www.figma.com/c/plugin/782713260363070260/Advanced-SVG-Export) 等。
-   **[ImageOptim](https://imageoptim.com/mac)**：一款用于 Mac 的图像优化工具，支持多种图像格式，包括 SVG。它通过压缩和删除不必要的数据来优化文件。
-   **[SVG Optimizer](https://svgoptimizer.com/)**：免费在线工具，适合希望简单优化 SVG 图像的 Web 开发者。
-   **[Optimize.svg](https://www.figma.com/community/plugin/1254080912048972428/optimize-svg)**：Figma 插件，可以在设计平台内生成优化的 SVG 代码。
-   **压缩工具**：例如 gzip，可以将优化后的 SVG 文件进行进一步压缩，以减少传输过程中的数据量。

  


最为常用的还是 `SVGO` 和 `Optimize.svg`。


接下来，我们以 SVGO 为例，看看它是如何优化 SVG 的。

继续以 Inkscape 导出的 SVG 文件为例（将其称之为 `Inkscape.svg`），优化前文件大小是 `197kb` ，优化后大约是 `99kb` ：

正如你所看到的，同一个 SVG 文件，使用 SVGOMG 优化前后有着天壤之别。如果你不想花太多心思在构建工具或不想使用命令工具来优化 SVG 的话，那么 [SVGOMG](https://svgomg.net/) 是一个很不错的选择。因为它简单易于，且优化效果也不错。

  


### 使用 SVGO 进行 SVG 优化

顾名思义，使用 SVGO 来优化 SVG，指的就是通过使用 CLI 或相关构建工具来对 SVG 进行优化。前面我们提到过，SVGO 是一个基于 Node 的开源工具，可以清理冗长的 SVG 文件并输出优化后的版本。

要使用 SVGO 命令行来优化 SVG，首要的条件是先安装 SVGO。我们可以通过 NPM 来安装 SVGO：


```
npm install -g svgo
```


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b206ed7f3b1e4250bc50b09f9880e9dc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1916&h=334&s=94777&e=jpg&b=fefefe)


这个命令将全局安装 SVGO，使你可以在任何目录中使用它。例如，在命令行中进入你的测试目录，并运行以下命令优化 SVG：


```
svgo Inkscape.svg    
```


优化后，文件大小减少了 `43.3%` ，新文件大小为 `109.373 KiB`，优化后的 SVG 代码如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b7f0994917f64b658bbae1dc219805bf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3346&h=1890&s=2374600&e=jpg&b=2d313c)


你可以添加一些标记来进一步优化或调整输出。例如，使用 `--pretty` 标志可以使 XML 格式更漂亮：


```
 svgo --pretty Inkscape.svg  
```


优化后，文件大小为 `94.343kb` ，代码更易读：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0f198899883842b3b64897fdc0625bc8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3330&h=1952&s=2809690&e=jpg&b=25292e)

```

```


转换后，SVG 会被编码为 Base64：

但需要知道的是，将 SVG 转换为 Base64 格式，不见得会使文件变得更小，甚至还有可能变得更大：

除了全局运行 SVGO 之外，我们也可以在单个项目中运行 SVGO。例如，我们有一个项目名为 `svgs`，通过下面命令来初始化项目，并安装 SVGO：


```
npm init
npm install --save-dev svgo  
```



![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9acb25e6dddf445ba5c11a153ab19917~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1902&h=1450&s=303521&e=jpg&b=ffffff)

```Json

    "optimize-svg": "svgo --pretty -f ./icons -o ./output --config ./svgo.config.mjs",
  },

```

#### 导出

导出也很关键，它可能直接影响你最终的 SVG 代码质量。前面也向大家演示了，不同的图形设计软件，导出的 SVG 代码质量有明显的差异。

当然，很多时候这个过程是我们无法决定的了，但我们可以通过 SVGO 这样的优化工具对导出的 SVG 进行优化。这样可以使 SVG 代码变得更加干净和精简！



#### 优化 SVG 动画的加载时间

当你的页面上有多个动画时，首次内容绘制（FCP）或页面加载速度可能会受到影响，因为浏览器需要在渲染之前评估所有 JavaScript。

动画 SVG 内部包含 JavaScript，如果动画复杂，意味着 JavaScript 代码也很复杂，使得浏览器在显示第一个页面渲染之前需要做更多的工作。

避免这种情况的一种方法是在页面完全加载后再加载 SVG 中的 JavaScript。这反过来会导致更快的加载时间和更响应的页面。

为了实现这一点，你必须搜索 SVG 中的脚本标签，提取该 JavaScript，并将其放置在外部文件中，然后你可以在触发事件时设置其加载。

  


#### 使用 Gzip 减小 SVG 大小

SVG 文件本质上是 XML 格式的文本文件，它们的可读性虽然好，但未压缩的文件可能会比较大，尤其是在包含复杂图形时。通过使用 Gzip 压缩可以进一步减小 SVG 文件的大小，提高 Web 项目的加载速度和性能。

通过在服务器配置、构建工具和开发过程中集成 Gzip 压缩，你可以确保所有 SVG 文件都得到优化，从而为用户提供更快速的加载体验。无论你是使用 Apache、Nginx、IIS 还是 Node.js 构建服务器，或者使用 Webpack 或 Gulp 进行构建，Gzip 压缩都可以轻松集成到你的工作流程中。



需要提出的是，上面提到的仅是常用的一些优化 SVG 的方法，但优化 SVG 的技术手段绝不仅限于此。大家更应该根据实际出发，使用更贴合自己项目的技术手段来优化 SVG。


不管使用哪种技术手段，我们优化 SVG 的主要目的是提高网站速度。

  


## 实战：如何在 React 项目中优化 SVG


众所周知，React 是现代 Web 开发的流行框架之一，而 SVG 这种图像格式又特别适用于 Web，因为它确保 SVG 图像在任何尺寸下保持清晰和锐利，而不失真。

接下来，我们来看看在 React 项目中，应该如何对 SVG 进行优化。


### 创建 React 项目

要开始在 React 项目中使用 SVG，首先需要有一个 React 项目。

在这里，我们使用 [CRA](https://create-react-app.dev/)（`create-react-app`）来创建一个 React 项目。你可以在终端中运行以下命令来创建一个 React 项目：


```
npx create-react-app mysvg
```



### 将 SVG 导入到 React 组件中

将 SVG 整合到你的 React 组件中可以增强应用程序的视觉吸引力和交互性。React 的模块化特性使得 SVG 可以像任何其他组件一样被导入和管理，提供了无缝的开发体验。

由于 SVG 文件具有 XML 格式，可以像 HTML 一样进行查询和操作，因此在将 SVG 导入到 React 组件时，你不仅仅是导入一张图像；你正在引入一种多功能的图形，可以进行样式化，动画化和交互。


一旦设置完成，你就可以将 SVG 导入到你的 React 项目中。传统上，SVG 可以直接用于 JSX 中 `<img>` 元素，其中 `src` 属性指向 SVG 文件。除此之外，在 React 中，还可以使用 ` react-svg-loader  `，将 SVG 导入为 React 组件。这允许你直接将 SVG 作为组件使用，传递属性并像任何其他 React 组件一样应用样式。

以下是使用 `react-svg-loader` 将 SVG 文件导入为 React 组件的示例：

  


```JavaScript
// App.js
import React from 'react';
import logo from './logo.svg';

const App = () => {
  return (
    <div>
      <img src={logo} alt="React Logo" />
    </div>
  );
};

export default App;
```

  


然而，这种方法仅允许你充分利用 SVG 的全部潜力，例如操作其属性或应用动态样式。


其实，在使用 CRA 创建的 React 项目中，还可以直接使用 SVGR，在无需进行任何配置的情况下，可以通过 SVGR 直接将 SVG 转换为 React 组件。这使你可以将导入的 SVG 像 JSX 一样使用，你可以传递属性、添加事件并动态修改 SVG 元素。例如： 


```JavaScript
// App.js
import { ReactComponent as Logo } from './logo.svg';
import './App.css';

function App() {
  return (
    <div className="App">
      <header className="App-header">
          <Logo className="App-logo" />
      </header>
    </div>
  );
}

export default App;
```


在这个示例中，`Logo` 现在是一个 React 组件，你可以像处理其他组件一样应用类或传递其他属性。


### 使用 CRA 中的文件加载器处理 SVG 文件

CRA（Create React App）预配置了一个文件加载器系统，用于处理诸如 SVG 等文件。当你导入一个 SVG 文件时，文件加载器会对其进行处理。它将其添加到输出目录，通常在 `static/media` 文件夹中，使用唯一的哈希来确保浏览器缓存失效。


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4c2122fc47c0411795ba9ad116d15c9e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3354&h=1394&s=262680&e=jpg&b=23262d)



如果你需要自定义 SVG 文件的处理方式，你可能需要修改 Webpack 配置。不过，在大多数情况下，CRA 默认的功能已经足够用了。

另外，使用文件加载器，你还可以将 SVG 导入为 URL，这在你希望在 CSS 中引用 SVG 文件或当你需要将 SVG 作为 `src` 属性传递给 `<img>` 时非常有用：

使用 Create React App 中的文件加载器处理 SVG 文件

Create React App 预配置了一个文件加载器系统，用于处理诸如 SVG 等文件的包含。当你导入一个 SVG 文件时，文件加载器会对其进行处理。它将其添加到输出目录，通常在 static/media 文件夹中，使用唯一的哈希来确保浏览器缓存失效。



如果你需要自定义 SVG 文件的处理方式，你可能需要从 Create React App 中退出以直接修改 Webpack 配置。但是，对于大多数用例，默认配置应该足够。


使用文件加载器，你还可以将 SVG 导入为 URL，这在你希望在 CSS 中引用 SVG 文件或当你需要将 SVG 作为 src 属性传递给 img 标签时非常有用：

  


```JavaScript
// App.js

import logoUrl from './logo.svg';
import './App.css';

function App() {
    return (
        <div className="App">
            <header className="App-header">
                <img src={logoUrl} alt="React Logo" />
            </header>
        </div>
    );
}

export default App;
```

  


在这种情况下，`logoUrl` 是一个字符串，表示经文件加载器处理后的 SVG 文件的最终路径。



### React 中 SVG 加载器配置


Webpack 是 CRA 使用的模块打包工具，可以配置各种加载器来管理和构建过程中如何处理和优化 SVG 文件。对于 SVG 文件，你可能会使用特定的加载器，例如 `svg-url-loader` 或 `file-loader` 来处理。默认情况之下，CRA 的 Webpack 配置包含了处理 SVG 的必要设置，无需额外配置。但是，如果你不想使用 CRA 默认的功能，希望手动配置 Webpack，则必须自行设置加载器。


以下是 Webpack 使用 `file-loader` 处理 SVG 的示例：

  


```JavaScript
module.exports = {
  module: {
    rules: [
      {
        test: /.svg$/,
        use: ['file-loader']
      }
    ]
  }
};
```

  


该配置告诉 Webpack 对所有以 `.svg` 结尾的文件使用 `file-loader`。当你将 SVG 文件导入到 React 组件中时，Webpack 使用 `file-loader` 对其进行处理，将其发射到输出目录并返回 URL。


你也可以使用 `svg-url-loader` 加载器，它允许你直接将 SVG 作为编码的数据 URL 添加到你的 React 组件中。这对于小型 SVG 来说可能有益，因为它减少了获取资源所需的 HTTP 请求的数量，从而提高了性能。

  


```JavaScript
module.exports = {
  module: {
    rules: [
      {
        test: /.svg$/,
        use: [
          {
            loader: 'svg-url-loader',
            options: {
              limit: 8192, // 字节
            }
          }
        ]
      }
    ]
  }
};
```

  


通过这种设置，小于指定限制的 SVG 将作为数据 URL 内联。较大的 SVG 将回退到由 `file-loader` 处理。

  


### 为 SVG 优化自定义加载器配置


优化 SVG 是确保它们不会对应用程序性能产生负面影响的重要步骤。你可以自定义 Webpack 加载器配置，包括 SVG 优化工具，如 `svgo-loader`，它可以通过删除不必要的数据来最小化 SVG 文件，而不会影响其视觉外观。


以下是 Webpack 中配置 `svgo-loader` 的示例：

  


```JavaScript
module.exports = {
  module: {
    rules: [
      {
        test: /.svg$/,
        use: [
          'svg-url-loader',
          'svgo-loader'
        ]
      }
    ]
  }
};
```

  


在此配置中，`svgo-loader` 与 `svg-url-loader` 将一起处理 SVG。首先，`svgo-loader` 优化 SVG，然后 `svg-url-loader` 将其转换为数据 URL（如果其大小在指定的限制之下）。

  


### 在 React 应用中使用 SVG 的性能提示

在 React 应用中使用 SVG 时，应始终考虑性能。以下是一些性能优化建议：


-   **优化 SVG**：在将 SVG 导入项目之前，使用 SVGO 等工具优化它们，通过删除不必要的属性和元数据来减小文件大小。

-   **延迟加载**：对于大型 SVG 或包含许多 SVG 的页面，考虑延迟加载组件，以减少应用程序的初始加载时间。
-   **避免内联大型 SVG 文件**：尽管内联 SVG 对小文件有好处，但如果过度使用或文件过大，可能会使 JavaScript 包膨胀。对于较大的 SVG，使用外部引用。
-   **缓存 SVG**：如果使用外部文件的 SVG，确保它们被浏览器适当地缓存，以避免后续访问时重新获取。
-   **使用图标精灵**：在处理大量 SVG 图标时，使用 SVG 精灵以减少 HTTP 请求并利用浏览器缓存。



特别声明，虽然这里举的例子是 React 中使用 SVG 的优化方式，但对于其他现代框架中使用 SVG，优化手段亦是如此，只不过使用的文件加载器，优化配置略有不同。这里就不一一列举！
