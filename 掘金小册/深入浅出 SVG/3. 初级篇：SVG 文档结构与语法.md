SVG 是一种在 Web 开发中广泛应用的矢量图形格式，在现代 Web 开发中，它已经成为不可或缺的工具，为 Web 设计和交互提供了强大的图形能力。SVG 不仅能够在各种终端屏幕尺寸上无损缩放，而且具备丰富的图形绘制和动画交互功能。要充分利用 SVG 的威力和真正驾驭 SVG，深入了解其文档结构与语法显得尤为关键。这节课将带领大家一起探索 SVG 的内部机制，揭示 SVG 文档的构建方式以及其中蕴含的语法规则。

  


首先，我们将深入研究 SVG 文档的基本结构。SVG 文档实质上是一个 XML 文件，以 `<svg>` 元素为根容器，包裹着各种图形元素。我们将详细解析这些基本元素的嵌套关系，让大家对 SVG 文档的整体框架有清晰的认识。理解文档结构是理解 SVG 语法的第一步，也是掌握 SVG 图形制作的基础。

  


其次，我们将探讨 SVG 的语法规则。SVG 语法注重精确性和层次性，通过标签、属性和值的组合，定义图形的外观和行为。我们将着重介绍一些常见的图形元素，如矩形、圆形、路径等，以及它们的属性用法。通过深入了解这些语法规则，大家将能够更自如地操作 SVG 文档，实现自己所需要的图形效果。

  


最后，我们再来进行实例学习，将理论知识转化为实际操作。通过一步步构建简单到复杂的图形，以及添加基本到高级的动画效果，大家将亲身体验 SVG 的强大功能。这节课力求通过清晰的解释和实际案例的演示，使大家能够轻松地掌握 SVG 文档结构与语法，为创造出引人入胜的可缩放矢量图形打下坚实的基础。让我们一同开始这段有趣而富有挑战性的 SVG 之旅。

## 命名空间

  


不难发现，前面展示的示例中，在 `<svg>` 元素中有的时候显式应用了 `xmlns` 、`xmlns:xlink` 等属性，你或许会为此感到疑惑，它们表示的是什么意思？又起什么作用呢？

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
    <!-- 这里是 SVG 内容 -->
</svg>

<svg xmlns="http://www.w3.org/2000/svg">
    <!-- 这里是 SVG 内容 -->
</svg>

<svg class="heart" viewBox="0 0 800 800">
    <!-- 这里是 SVG 内容 -->
</svg>
```

  


我们来解释一下它们。

  


简单地说，它们是给 SVG 声明命名空间的一种方式。它和 HTML 是相似的，在 SVG 中，命名空间是一种机制。SVG 使用命名空间是为了解决在 XML 文档中的元素冲突问题，以及为了支持模块化、扩展性和跨平台的标准化。以下是 SVG 为什么要使用命名空间的一些原因：

  


-   **避免命名冲突**：SVG 元素和属性的命名可能与其他 XML 标准或用户定义的元素名称相冲突。通过使用命名空间，SVG 可以确保其元素和属性的名称在文档中不会与其他命名空间中的相同名称发生冲突，避免混淆和解析错误。
-   **模块化设计**：命名空间支持 SVG 的模块化设计，使得 SVG 规范可以被扩展和修改而不破坏现有标准。这使得开发者能够定义和使用自己的 SVG 元素或属性，而不必担心与 SVG 规范中的元素发生冲突。
-   **标准扩展**：命名空间允许 SVG 标准进行扩展，从而支持新的特性和功能。通过在不同的命名空间中引入新的元素和属性，SVG 规范可以在不同版本之间演进，并为未来的发展提供灵活性。
-   **互操作性**：命名空间确保 SVG 文档在不同的上下文和应用程序中能够被正确解析和处理。它促使标准的互操作性，允许不同的软件和系统之间共享和交换 SVG 文档而不损失信息。
-   **XML 规范兼容性**：SVG 作为 XML 的一种，使用命名空间是符合 XML 规范的。这有助于确保 SVG 文档可以与其他 XML 文档一起使用，并在 XML 处理工具中正确解析。
-   **语义清晰**：命名空间使得 SVG 文档中的元素拥有明确的语义，开发者可以清晰地了解元素的来源和用途。这有助于提高文档的可读性和维护性。

  


抛开上述这些原因不说。**在编写 SVG 代码时，使用命名空间是确保 SVG 文档正确解析和渲染的关键**。就这一点来说，个人建议你在编写 SVG 代码时，请务必给 `<svg>` 元素声明命名空间。

  


前面我们提到过，SVG 文档可以以 XML 或 HTML 形式存在，而命名空间的处理方式在这两种情况下略有不同。如果你的 SVG 文档是以 XML 形式存在，那么需要指定 SVG 命名空间，以确保所有 SVG 元素都被正确识别。常见的方法是在 `<svg>` 根元素上添加 `xmlns` 属性，例如：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100">
    <!-- SVG元素和内容 -->
</svg>
```

  


上面代码中，`xmlns="``http://www.w3.org/2000/svg``"` 指示所有子元素都属于 `http://www.w3.org/2000/svg` 这个 SVG 命名空间。

  


如果你的 SVG 文档嵌套在 HTML 中，那么 HTML 解析器会自动为 `<svg>` 元素提供命名空间，你无需显式声明。例如：

  


```HTML
<html>
    <body>
        <svg width="100" height="100">
            <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
        </svg>
    </body>
</html>
```

  


在上面这个示例中，`<svg>` 元素会被自动置于正确的命名空间，无需手动添加 `xmlns` 属性。

  


SVG 文档嵌套在 HTML 文档中还会存在另一种情形，那就是 `<html>` 根元素的所有子节点也被定义为默认命名空间，这个时候你需要重新为 `<svg>` 元素定义命名空间。例如：

  


```HTML
<html xmlns="http://www.w3.org/1999/xhtml">
    <body>
        <!-- 在这里放置一些 XHTML 标签 -->
        
        <svg xmlns="http://www.w3.org/2000/svg" width="300px" height="200px">
            <!-- 在这里放置一些 SVG 标签 -->
        </svg>
        
        <!-- 在这里放置一些 XHTML 标签 -->
    </body>
</html>
```

  


在这个例子中，`<html>` 根元素的 `xmlns` 属性定义了 XHTML 为默认命名空间。如此一来，`<html>` 元素和它的所有后代元素都会被用户代理解释为属于 XHTML 命名空间。为了避免 SVG 文档解析出错，你需要为 `<svg>` 元素重新定义属于它自己的命名空间，只有这样做，才能告诉用户代理 `<svg>` 元素以及它所包含的元素（除非它们也重新定义了命名空间）属于 SVG。

  


总体而言，SVG 命名空间确保了 XML 文档中的元素和属性的唯一性和一致性，使得 SVG 可以灵活地集成到各种 XML 文档和 Web 环境中。在处理 SVG 时，特别是在 XML 中，了解并正确使用命名空间是确保文档解析和渲染正确的关键。

  


我想你现在对 SVG 的命名空间有一定的认知了，但我还是想再啰嗦一下。我们在给 SVG 文档声明命名空间时，有一些细节还是需要注意的。

  


如果你的 SVG 文档是一个独立的 `.svg` 文件，那么建议你在 SVG 文档的开始处应该包含 XML 声明，以指示文档的 XML 版本和字符编码，例如：

  


```XML
<?xml version="1.0" encoding="UTF-8" ?>
```

  


不管 SVG 文档是以独立的 `.svg` 文件存在，还是嵌套到 XML 或 HTML 中，都应该为 SVG 文档的根元素 `<svg>`

显式添加 `xmlns` 属性（为 SVG 文档显式声明命名空间），确保 `<svg>` 元素及其所有后代元素都属于 SVG 命名空间：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100">
    <!-- SVG元素和内容 -->
</svg>
```

  


现在，你知道了，可以在 `<svg>` 元素指定 `xmlns` 属性的值给 SVG 文档声明命名空间。接下来，我们一起来简单地了解一下 `xmlns:xlink` 属性。

  


`xmlns:xlink` 是 SVG 中的一个命名空间声明，用于指定 `XLink` 命名空间（XML Linking Language）。`XLink` 是一种用于在 XML 文档中创建超链接的标准，它提供了在文档之间和文档内部创建链接的机制。

  


具体来说，`xmlns:xlink` 的作用是声明 `xlink` 命名空间，以便在 SVG 文档中使用 `XLink` 相关的属性，如 `href`，来定义超链接和链接行为。

  


以下是一个示例，演示了如何在 SVG 中使用 `xmlns:xlink`：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" width="100" height="100">
    <a xlink:href="https://www.example.com" target="_blank">
        <circle cx="50" cy="50" r="40" fill="red" />
    </a>
</svg>
```

  


在上述例子中：

  


-   `xmlns:xlink="``http://www.w3.org/1999/xlink``"` 声明了 `xlink` 命名空间。
-   `<a xlink:href="``https://www.example.com``" target="_blank">` 是一个超链接元素，使用了 `xlink:href` 属性来定义链接的目标地址。
-   `<circle>` 元素是 `<a>` 元素的子元素，使得整个圆形区域都成为了超链接，点击后打开指定的链接。

  


总体而言，`xmlns:xlink` 帮助定义 SVG 中与超链接相关的属性，使得开发者可以在 SVG 文档中使用 `XLink` 标准来创建链接、引用外部资源等。这提供了在矢量图形中嵌入超链接的能力，使得 SVG 文档能够更灵活和交互性。

  


最后再来说一下命名空间的前缀。虽然在 `<svg>` 根元素上的 `xmlns` 属性通常不需要前缀，但在 XML 文档的其他位置，尤其是当文档中同时包含多个命名空间时，可能需要使用前缀。这样可以明确指定某个元素属于哪个命名空间。

  


```SVG
<svg:svg xmlns:svg="http://www.w3.org/2000/svg" width="100" height="100">
    <svg:circle cx="50" cy="50" r="40" fill="red" />
</svg:svg>
```

  


在这个例子中，`xmlns:svg` 指定了 SVG 命名空间的前缀，而 `<svg:circle>` 中使用了该前缀，表示该元素属于 SVG 命名空间。

  


注意，这种形式在我们的案例中几乎不会出现，这里特意拎出来，是希望大家以后看到类似的 SVG 代码，知道它是什么意思。

  


## SVG 元素

  


SVG 元素是构成 SVG 图形的基本单元。它们可以创建各种形状、文本、图像和动画，使你能够在 Web 页面上呈现出丰富而灵活的图形内容。从简单的矩形到复杂的路径，SVG 元素提供了丰富的选择，使你能够以无损失的方式调整图形的大小。

  


SVG 中包括多种不同类型的 SVG 元素，大致分为基本形状元素、路径元素、文本元素、容器元素（文本结构元素）、动画元素等，每个分类都有其独特的用途和特性。

  


> 注意，在这节课中，我们并不会对所有 SVG 元素展开介绍，但小册后续课程会逐步覆盖所有 SVG 元素，尤其是一些常用的 SVG 元素，将会在小册不同课程中多次出现。

```SVG
<svg class="smiley" width="256" height="256" viewBox="0 0 256 256" version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">

    <desc>Smiley face drawn using SVG</desc>
    <style>
        <![CDATA[
            .smiley {
                cursor: pointer;
            }
        
            .left-eyebrow {
                transform: rotate(80deg);
                transform-origin: 100px 82px;
                transition: all .5s linear;
            }
        
            .right-eyebrow {
                transform: rotate(100deg);
                transform-origin: 156px 82px;
                transition: all .5s linear;
            }
        
            .smiley:hover .left-eyebrow {
                transform: rotate(100deg);
            }
        
            .smiley:hover .right-eyebrow {
                transform: rotate(80deg);
            }
        ]]>
    </style>
    
    <circle class="face" cx="128" cy="128" r="120" fill="gold" />
    
    <g class="eyes">
        <circle class="left-eye" cx="100" cy="104" r="12" />
        <circle class="right-eye" cx="156" cy="104" r="12" />
    </g>
    
    <g class="eyebrows">
        <rect class="left-eyebrow" x="97" y="66" width="6" height="32" rx="4" ry="4" />
        <rect class="right-eyebrow" x="153" y="66" width="6" height="32" rx="4" ry="4" />
    </g>

    <path class="mouth" d="M100,160 Q128,190 156, 160" fill="none" stroke="#000" stroke-width="6" stroke-linecap="round" />
</svg>
```

  


这段代码是一个使用 SVG 绘制的笑脸图形，通过 XML 命名空间指定了 SVG 的命名空间，并使用了一些 CSS 样式和动画效果。

  


-   `<svg>` 元素定义一个 SVG 图形，其中包含了宽度（`width`）、高度（`height`）、视图框（`viewBox`）、版本信息（`version`）和 XML 命名空间（`xmlns` 和 `xmlns:xlink`）。
-   `<title>` 元素用于提供图形的标题。
-   `<desc>` 元素用于提供对图形的描述。
-   `<style>` 元素包含了内联的 CSS 样式，使用 `CDATA` 标记包裹，定义了一些样式规则，包括悬停时眉毛的动画效果。
-   `<circle>` 元素绘制了笑脸的脸部，其中包括圆心坐标（`cx` 和 `cy`）、半径（`r`）和填充颜色（`fill`）。
-   `<g>` 元素用于组合多个元素，这里分别用于包裹眼睛和眉毛。
-   `<rect>` 元素绘制了眉毛，包括位置（`x` 和 `y` ）、大小（`width` 和 `height`）和圆角（`rx` 和 `ry`）属性。
-   `<path>` 元素绘制了嘴巴，通过路径数据（`d`属性）描述了一个二次贝塞尔曲线，同时定义了边框颜色（`stroke`）、宽度（`stroke-width`）和线帽（`stroke-linecap`）属性。

  


整体效果是一个笑脸图形，当鼠标悬停在笑脸上时，眉毛会发生动画变化。

需要知道的是，SVG 中的每个元素都有着自己的作用和功能，有关于这方面的内容，放到后续课程单独阐述。这里有两点要向大家提出来。

  


首先，SVG 的元素在语法上的使用与 HTML 标签是类似的。通常情况下，每个标签的应用都是成对出现的，包括打开标签和闭合标签。例如：

  


```SVG
<svg><!-- 打开标签 -->
    <rect x="10" y="10" width="200" height="200" fill="lime"><!-- 打开标签 -->
    </rect><!-- 闭合标签 -->
</svg><!-- 闭合标签 -->
```

  


但这并不是绝对的。如果 SVG 元素标签不包含任何后代元素标签时，我们可以直接以 `/` 来代表元素的闭合标签，这个有点类似于 HTML 的 `<img />` 和 `<hr />` 标签的使用。比如，上面示例中的 `<rect>` 元素并没有包含任何后代元素，那么它可以直接省略闭合标签，但记得在`<rect>` 标签的 `>` 前面加上 `/` ：

  


```SVG
<svg><!-- 打开标签 -->
    <rect x="10" y="10" width="200" height="200" fill="lime"><!-- 打开标签 -->
    </rect><!-- 闭合标签 -->
</svg><!-- 闭合标签 -->

<!-- 等同于 -->

<svg><!-- 打开标签 -->
    <rect x="10" y="10" width="200" height="200" fill="lime" />
</svg><!-- 闭合标签 -->
```

  


其次就是 SVG 元素的层叠顺序，即它在 `Z` 上的层级。拿上面“笑脸”为例，把绘制“笑脸”的 SVG 代码复制粘贴到文本编辑器中，并将其保存为 `.svg` 文件，例如 `face.svg` （可以是任何你喜欢的一个名字），然后使用图形编辑软件打开你刚才保存好的 `face.svg` 文件。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1b2850220d554f2ba8025a2926cad702~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1122&h=316&s=425249&e=gif&f=219&b=f6f6f6)

  


我使用 Figma 打开了 `face.svg` 文件，你可能发现，图形编辑软件已我们内联在 `<svg>` 中的样式（`<style>`）丢失了（其实一些元数据也丢失了，比如示例中的 `<title>` 和 `<desc>` ），通常情况下，在图形编辑软件中都会把不渲染的元素丢去。但这并不是重点。我们这里的得点是看元素在 `Z` 的层叠顺序。

  


不难发现，SVG 代码中的图形元素，例如 `<g>` 、`<circle>` 、`<rect>` 和 `<path>` 在图形软件中对应着相应的图层。为了能更好的让图层名称与前面的 SVG 代码相匹配，可以手动对图层进行命名：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/631a2fb21aa74626b7f4f4369850c35c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=927&s=569365&e=jpg&b=f8f8f8)

  


SVG 元素在源码中出现的越早（排在前面），它在图层中的层级最低，例如示例中的 `.face` （`<circle>`）；出现的越后（排在后面），它在图层中的层级最高，例如示例中的 `.mouth` （`<path>`）。

    </g>

    <path class="mouth" d="M100,160 Q128,190 156, 160" fill="none" stroke="#000" stroke-width="6" stroke-linecap="round" />

注意，为了能看到底下的眉毛和眼睛，调整了 `.face` 的透明度。

  


正如上图所示，在 SVG 中，元素的堆叠顺序（显示顺序）是由它们在文档流中的位置和层次关系决定的。元素的渲染顺序取决于其在 DOM（文档对象模型）中的位置和父元素关系。通常，后面出现的元素会覆盖在前面出现的元素之上。

  


也就是说，在 SVG 中，图层是通过元素的顺序来定义的。位于后面（出现在源码后面）的元素在图层中（`Z` 轴层又级）会显示在较前面（出现在源码前面）的元素之上。另外，到目前为止，我们是无法通过 CSS 的 `z-index` 属性来调整 SVG 元素的堆叠顺序（[据说在 SVG 2.0 中将会得到支持](https://github.com/w3c/svgwg/wiki/SVG-2-new-features#new-style-properties)）。

  


需要注意的是，在实际使用中，具体的情况可能受到 SVG 元素类型、属性、CSS 样式的影响。因此，需要根据具体的 SVG 结构和样式来理解和控制元素的显示顺序。如果你想更进一步的了解这方面的知识，那么建议你阅读 [SVG 2.0 规范中的渲染模块中的渲染顺序一节](https://svgwg.org/svg-next/render.html#RenderingOrder)。

  


## SVG 属性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3c73f6c7ed1240ec8aa3c3dbad8f092a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=600&s=46660&e=png&b=ffffff)

  


SVG 和 HTML 还有一个共同的特征，那就是 SVG 元素也具有多种属性，这些属性用于定义元素的外观、行为和其他特征。以下是一些常见的 [SVG 元素属性](https://www.w3.org/TR/SVG/attindex.html)，具体属性的可用性和效果可能取决于元素类型和上下文。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6a9b5d6fda8c4dba8795018c68e0498e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3350&h=1700&s=829059&e=jpg&b=fefdfd)

  


> URL ：https://www.w3.org/TR/SVG/attindex.html

SVG 属性可分为通用属性和特定元素属性两类。通用属性适用于所有 SVG 元素，而特定元素属性则用于特定的 SVG 元素。

  


### 通用属性

  


与 HTML 元素的属性一样，有很多属性是可以通用的，可以应用于所有 SVG 元素。例如：

  


-   `id` ：为元素定义唯一标识符
-   `class` ：为元素定义一个或多个类名
-   `style` ：为元素定义内联 CSS 样式
-   `fill`： 定义元素的填充颜色
-   `stroke`： 定义元素的边框颜色
-   `stroke-width`： 定义元素的边框宽度
-   `transform`： 定义元素的变换，如平移、旋转、缩放等
-   等等

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <circle id="myCircle" cx="50" cy="50" r="30" />
    <rect class="highlight" x="10" y="10" width="50" height="50" />
    <line x1="10" y1="10" x2="50" y2="50" style="stroke: black; stroke-width: 2;" />
    <rect x="10" y="10" width="50" height="30" transform="rotate(45)" />
    <rect fill="green" x="10" y="10" width="50" height="30" />
    <circle stroke="black" cx="50" cy="50" r="30" />
    <line x1="10" y1="10" x2="50" y2="50" stroke="black" stroke-width="2" />
    <circle visibility="hidden" cx="50" cy="50" r="30" />
</svg>
```

  


### 特定元素属性

  


从字面上就不难理解，意思就是说，SVG 有些属性只能应用于特定的元素上。例如 `d` 属性只适用于 `<path>` 元素：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <path d="M10 80 Q 95 10 180 80" />
</svg>
```

  


`x1` 、`y1` 、`x2` 和 `y2` 只适用于 `<line>` 元素：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <line x1="10" y1="10" x2="50" y2="50"  />
</svg>
```

  


虽然 SVG 属性大多数分为通用属性和特定元素属性，但这并不是绝对的，有些属性也适用多个元素。例如用于 `<circle>` 元素上的 `cx` 和 `cy` 属性，也适用于 `<ellipse>` 元素：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <ellipse cx="10" cy="10" rx="50" ry="50"  />
</svg>
```

  


上面代码中的 `rx` 和 `ry` 也适用于 `<rect>` 元素：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <rect x="10" y="10" width="200" height="200" rx="50" ry="50"  />
</svg>
```

  


这几个简单的示例也再次向大家说明，具体的属性和使用方式可能因元素类型而异。在实际使用中，你可以根据需要[查阅 SVG 规范和相关文档来获取更详细和具体的信息](https://www.w3.org/TR/SVG/attindex.html)。

  


虽然 SVG 元素通常会应用自身属性来定义图形的视觉外观和交互行为等，但这也导致 SVG 代码可能会变得冗长和复杂。不过，庆幸的是，通过利用 CSS 和 [SVG 2 规范即将推出的一些新功能](https://github.com/w3c/svgwg/wiki/SVG-2-new-features)，我们可以减少代码，使 SVG 标记变得更干净，更清晰。例如，我们通过 CSS ，可以将一些样式属性从 SVG 元素中抽离：

  


```SVG
<svg xmlns="http://www.w3.org/2000/svg">
    <circle cx="50" cy="50" r="30" class="my-circle" />
</svg>
```

  


```CSS
.my-circle {
    fill: #3498db;
    stroke: #2980b9;
    stroke-width: 2;
}
```

  


上面这个示例，我们只是使用 CSS 给 `<circle>` 元素定义了填充颜色（`fill`）、边框颜色（`stroke`）和边框宽度（`stroke-width`）。实际上，我们还可以做一些更复杂的事情。例如在 SVG 的 `<pattern>` 元素中，通过 CSS 给图形添加动画效果：

## SVG 绘制流程

  


如果你跟随课程的内容到这里，我想你已经掌握了 SVG 文档的结构和语法，也知道了可以通过 SVG 的元素和属性来绘制图形。在此基础上，我想带着大家使用 SVG 来绘制一只小熊，进一步了解 SVG 绘制图形的整个流程。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4065971c271449eeacfcb0a6b34610f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1196&s=182286&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/PogPLaR

  


上图是使用 SVG 绘制了小熊的脸部形状，通过几个基础图形元素，例如 `<circle>` （圆）、`<ellipse>` （椭圆）和 `<path>` （路径）等，就可以绘制出小熊的脸。

  


-   使用 `<circle>` 绘制了小熊的耳朵、眼睛
-   使用 `<ellipse>` 绘制了小熊的鼻子
-   使用 `<path>` 绘制了小熊的嘴
-   使用 `<circle>` 和 `<path>` 绘制了小熊的脸

  


我们使用 SVG 绘制上图可能会经过以下几个步骤。

  


### 步骤一：创建 SVG 元素

  


首先，在 HTML 文档中使用 `<svg>` 元素来创建 SVG 画布（容器）。这是 SVG 的根元素，所有 SVG 图形元素都将嵌套在这个容器中。

  


```SVG
<!-- Step 01: 创建 SVG 元素 -->
<svg version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" width="400" height="400" class="bear">
    <!--  SVG 元素都在这里 -->
</svg>
```

  


我们在 `<svg>` 元素上定义了类名（`class="bear"`）、指定了它的命名空间（`xmlns` 和 `xmlins:xlink`）和版本号（`version`），同时使用 `width` 和 `height` 属性定义了 SVG 容器的尺寸。

  


### 步骤二：组织文档结构

  


使用 SVG 绘制图形，这一步不是必须的。但为了更好的组织 SVG 文档，或者你编写的 SVG 代码结构更清晰，更易于阅读和维护，那么可以根据需要，使用 `<g>` 元素或其他类似的结构合理的组织文档。就我们这个示例而言，整个“小熊”脸部图形，我们可以根据脸部的位置来进行划分，比如熊的耳朵、眼睛、脸等分别进行分组。如果用代码来描述的话，它可能看起来像下面这样：

  


```SVG
<!-- Step 01: 创建 SVG 元素 -->
<svg version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" width="400" height="400" class="bear">
    <!--  Step 02: 使用 g 元素来分组 -->
    
    <!-- 熊的耳朵 -->
    <g class="bear__ears">
        <!-- 左耳朵 -->
        <g class="bear__ear--left">
            <!-- 绘制左耳朵 -->
        </g>
  
        <!-- 右耳朵 -->
        <g class="bear__ear--right">
            <!-- 绘制右耳朵 -->
        </g>
    </g>
    
    <!-- 熊的脸 -->
    <g class="bear__face">
        <!-- 绘制脸 -->
    </g>
    
    <!-- 熊的眼睛 -->
    <g class="bear__eyes">
        <!-- 绘制眼睛 -->
    </g>    
</svg>
```

  


到目前为止，虽然在 `<svg>` 中应用了多个 `<g>` 元素，但并没有任何形状会在 SVG 画布中呈现。这是因为我们还没有使用任何图形元素来绘制图形。

  


注意，`<g>` 元素不是必须的，你可以根据自己的喜好来应用。我个人一般会根据实际需要来应用。

  


### 步骤三：添加图形元素

  


图形要想在 SVG 画布上呈现，你必须得在 `<svg>` 元素中添加各种图形元素或可渲染的元素，例如线条（`<line>`）、圆（`<circle>`）、椭圆（`<ellips>`）、矩形（`<rect>`）和路径（`<path>`）等。每个图形元素都有相应的属性用于定义其外观和位置。

  


在这个示例中，可以使用 `<circle>` 来绘制熊的耳朵：

  


```SVG
<svg version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" width="400" height="400" class="bear">
    <!-- 熊的耳朵 -->
    <g class="bear__ears">
        <!-- 左耳朵 -->
        <g class="bear__ear--left">
            <!-- 绘制左耳朵 -->
            <circle cx="126" cy="118" r="32" fill="#8C6239" />
            <circle cx="126" cy="118" r="16" fill="#42210B" />
        </g>
    
        <!-- 右耳朵 -->
        <g class="bear__ear--right">
            <!-- 绘制右耳朵 -->
            <circle cx="290" cy="118" r="32" fill="#8C6239" />
            <circle cx="290" cy="118" r="16" fill="#42210B" />
        </g>
    </g>

  <!-- 熊的脸 -->
  <g class="bear__face">
    <!-- 绘制脸 -->
  </g>

  <!-- 熊的眼睛 -->
  <g class="bear__eyes">
    <!-- 绘制眼睛 -->
  </g>
</svg>
```
