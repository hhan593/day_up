这节来分析下百度网盘项目的技术选型：

首先，前端部分要做桌面端，所以我们要用 electron。

前端框架我会用我熟悉的 react，大家也可以用 vue 写，这个都行。

全局状态管理用 zustand，请求用 axios，组件库用 antd。

websocket 通信用 socket.io

前端技术选型就是 react + electron + zustand + axios + antd + socket.io。

后端我们分为 3 个微服务：

- 用户微服务：处理登录、注册、用户信息修改等业务。

- 文件微服务：处理文件列表、文件重命名等文件相关业务。

- 消息微服务：处理 IM 即时通信的聊天业务。


其中，用户和文件微服务都是普通的 http 的微服务，我们用 graphql 来写接口：

消息微服务比较特殊，是需要处理 websocket 消息的，用 socket.io

我们用 TypeORM 做 ORM 框架，它是直接用 ts 来写 entity，和 Nest 结合更好。

Prisma 是在 schema 文件里写 entity 定义，然后编译成 ts 类型。

和 Nest 结合也可以用，但不如 TypeORM 好用，比如我们可以直接在 TypeORM 的 entity 上加一些其他装饰器，但是 Prisma 的就不行。

数据层用 redis 做缓存，mysql 做持久存储，rabbitmq 做消息队列，nacos 做注册中心，Elasticsearch 实现全文检索，文件存储用 minio。

## 总结

这节我们梳理下技术选型。

前端： vite + react + antd + electron + zustand + axios + socket.io

后端微服务：

- 用户微服务：nest + typeorm + graphql

- 文件微服务：nest + typeorm + graphql

- 消息微服务：nest + typeorm + socket.io

数据层： redis + mysql + rabbitmq + nacos + elasticsearch

部署： docker compose、pm2

文档： swagger
