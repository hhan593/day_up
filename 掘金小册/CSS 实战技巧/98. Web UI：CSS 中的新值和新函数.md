![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4ed97dcb1d24da6bc50ad6a952757d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=1024&s=1715569&e=png&b=0d1527)

  


在这节课之前，我们在一些课程中阐述了 CSS 的一些新函数，比如用于[交替样式查询的 `toggle()` 函数](https://juejin.cn/book/7395119173656903715/section/7445287411581452338)，[新增了值类型的 `attr()` 函数](https://juejin.cn/book/7395119173656903715/section/7487175782831161394)，[计算元素总数量的 `sibling-count()` 函数和计算元素自身索引值的 `sibling-index()` 函数](https://juejin.cn/book/7395119173656903715/section/7487008847370928167)，[混合颜色的 `color-mix()` 函数](https://juejin.cn/book/7395119173656903715/section/7445261940961738761)，以及[可用于长度值与内在尺寸过渡和动画的 `calc-size()` 函数](https://juejin.cn/book/7395119173656903715/section/7490004064970342410)。这些函数都是 **[CSS 值和单位模块 Level 5](https://www.w3.org/TR/css-values-5/)** 中新增的函数，除此之外，该模块还新增了一些其他的 CSS 函数，比如 `mix()` 、`progress()` 、`first-valid()` 等。这些功能都是曾经我们一直向往的 CSS 功能。

  


这些新的 CSS 特性有一个共同的特点：它们简化了 CSS 代码。以前需要多条规则或曲线解决方案才能实现的功能，现在只需要一两行 CSS 即可实现。例如：

  


-   对 `attr()` 函数新增了值类型，可以用于数学运算，不再局限于 `content`
-   新的 `toggle()` 函数，用于轻松循环样式化嵌套元素，即交替样式查询
-   新的 `sibling-count()` 和 `sibling-index()` 函数，根据顺序和数量提供整数值进行操作
-   新的 `calc-size()` 函数，可以对内在尺寸（比如 `auto` 、`min-content` 、`max-content` 和 `fit-content`）进行插值计算，使固有长度值到内在尺寸值之间有平滑过渡动画效果
-   新的 `first-valid()` 函数，可用于避免自定义属性中无效值的问题
-   新的 `*-mix()` 系列函数，采用新的比率表示法
-   新的 `*-progress()` 系列函数，可用于计算范围内或媒体容器中的进度比率
-   新的 `random()` 和 `random-item()` 函数实现随机化，从范围或列表中返回随机值
-   新的 URL 修饰符，提供对 `url()` 请求的更多控制
-   新的函数表示汉，允许使用分号（`;`）分隔的值列表作为参数，以避免与逗号（`,`）分隔参数时的歧义
-   新增了 `interpolate-size` 属性，允许你对内关尺寸关键字做插值计算
-   扩展了 `position` 值类型，新增了流动相关的值

  


由于这些都是 CSS 的新特性，并且目前仍处于[工作草案阶段](https://www.w3.org/TR/css-values-5/)，因此在这节课中所描述的功能可能会发生变化，并且并非所有浏览器都支持（不过有些浏览器是支持的！）。即便它们还没有完全得到浏览器的支持，但这并不能阻碍我们来了解这些新函数和值的使用。接下来，我将简单的与大家分享一下这些新函数和值应该如何在 CSS 中使用，它们又将能帮助我们解决哪些问题？

  


  


## `attr()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0c6f728c040e459b88472ec1cc6b1fe5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1840&h=720&s=121450&e=jpg&b=d0b9e4)

  


`attr()` 函数并不是最新出现的 CSS 函数，只不过 `attr()` 一直以来功能非常有限，仅能处理字符串，并且只能在伪元素 `::before` 、`::after` 或 `::marker` 的 `content` 属性中使用。

  


现在，W3C 规范对 `attr()` 函数进行了一些更新，可以与元素的任何属性一起使用，并且可以与任何 CSS 属性一起使用。它可以接受两个可选参数：一个单位或属性，以及一个备用值。例如：

  


```HTML
<div data-percent="50%"></div>
<div data-percent="20%"></div>
```

  


```CSS
div {
    height: attr(data-percent type(<percentage>), 10%);
}
```

## `toggle()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fef813caf0d7403f917cbdd5c6b69710~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=234028&e=png&b=ffffff)

  


`toggle()` 函数引入了一种方便的新方式来定义嵌套元素中的值。它可以用于设置元素及其后人无事将循环的值，从而大大简化了 CSS 代码。告别复杂的选择器或重新定义。以多层无序列表为例，我们可以通过 `toggle()` 函数给每一层级设置不同的列表标记。它的 HTML 结构如下：

在没有 `toggle()` 函数之前，我们需要像下面这样编写 CSS 代码：

  


```CSS
ul {

```
```
ul {

}
```

## `calc-size()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d51f47a7681245338c326ac4fe02bfa8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=1619880&e=webp&b=fed9b4)

  


与 `calc()` 函数相比，大家更熟悉 `calc()` 函数，因为它是 CSS 中最早可用于数学运算的函数，但它无法与内在尺寸（比如 `auto` 、`min-content` 、`max-content` 和 `fit-content` 等）一起操作。而新增的 `calc-size()` 则不同，它允许你与内在尺寸进行操作。这一特性在过渡和动画中尤其有用。比如，你要从一个固定尺寸（比如 `4rem` ）过渡到 `auto` ，那么使用 `calc-size(auto, size)` 就可以使它们进行插值计算，使得它们有一个平滑的过渡动画效果。这意味着，它解决了我们要实现 `width:auto` 或 `height: auto` 的过渡和动画效果。比如下面这个示例：

  


```CSS
a {
    width: 4rem;
    transition: width 0.35s ease;
    
    &:is(:hover, :focus) {
        width: calc-size(auto,size);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aa09466034da4267bf4b5bb989ec7652~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1344&h=768&s=881112&e=gif&f=158&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ogNONGe

```CSS
:root {
    @supports (interpolate-size: allow-keywords) {
        interpolate-size: allow-keywords;
    }
}

a {
    width: 4rem;
    transition: width 0.35s ease;
    
    &:is(:hover, :focus) {
        width: auto;
    }
}
```

## `*-mix()` 系列函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/390704d15e6648bb8357a7fd10897d2c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1471&h=981&s=100400&e=avif&b=e6a173)

  


我知道 CSS 有一个 **[`color-mix()` 函数](https://juejin.cn/book/7395119173656903715/section/7445261940961738761)**，它可以在指定的颜色空间中对两个颜色进行混合，比如：

  


```CSS
.element {
    color: color-mix(in oklch, red, blue); 
}
```

  


我所不知道的是，CSS 还引入了一个新函数 `mix()` ，可以用来简化不同的 `*-mix()` 函数。如果你想混合颜色，可以像这样使用：

  


```CSS
.element {
    color: mix(90%, red, blue);
}

/* 等同于 */
.element {
    color: color-mix(red 90%, blue);
}

/* 计算出的结果是 */
.element {
    color: rgb(10% 0 90%);
}
```

  


除了可以混合颜色之外，还可以混合长度、变换函数等：

  


```CSS
.element {
    transform: mix(90%, translate(calc(1em + 50%)), rotate(30deg));
    /* 因为函数不匹配，它将通过 matrix() 插值。但是 translate(%) 需要布局信息才能转换为 matrix()，所以插值后的值实际上不能被表示。计算结果为：*/
    transform: mix(90%, translate(calc(16px + 50%)), rotate(30deg));
}
```

  


更有意思的是，这种语法还扩展到了其他的 `*-mix()` 系列函数：

  


-   `calc-mix()`
-   `color-mix()`
-   `cross-fade()`
-   `transform``-mix()`

  


这些函数都允许你指定一个进度比例，以及起始和结束的限制，它将返回这两者之间的对应值。这些函数可以与 `*-progress()` 系列函数结合使用，以获得更好的效果！

  


## `*-progress()` 系列函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/45f557ec712b4b95abe312eddad4a2ec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=1000&s=56982&e=png&b=0087fd)

  


`*-progress()` 系列函数表示给定值从一个起始值到另一个结束值的比例进度。结果是一个介于 `0` 和 `1` 之间的数字，可以在操作中使用，但当与之前描述的 `*-mix()` 系列函数结合使用时，它会特别方便。

  


这个系列函数主要有：

  


-   `progress()` 是一个通用函数，适用于任何数学函数
-   `media-progress()` 函数主要用于媒体查询，指定媒体查询的当前值作为两个明确的媒体查询值（作为进度起始值和进度结束值）之间的进度值。
-   `container-progress()` 函数主要用于容器查询，与 `media-progress()` 函数类似，唯一的不同是它接受容器特性代替媒体特性

  


以 `progress()` 函数为例：

  


```CSS
.element {
    width: calc(100% * progress(50px from 0px to 100px));
}
```

  


我们使用 `progress()` 函数来动态计算 `.element` 元素的宽度。在这里，`progress(50px from 0px to 100px)` 的意思是：

  


-   进度值是 `50px`
-   起始值是 `0px`
-   结束值是 `100px`

  


`progress()` 函数会根据进度值在 `0px ~ 100px` 之间计算一个比例，并返回一个数字（`<number>`），表示 `50px` 在这个区间的位置。具体计算公式是：

  


```CSS
(50px - 0px) ÷ （100px - 0px） = 0.5
```

  


所以 `progress(50px from 0px to 100px)` 将返回 `0.5`。`progress()` 函数的返回值再乘以 `100%` ，最终元素的宽度是 `50%` 。

  


## `random()` 和 `random-item()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e378b1e48b6b4768b41614f24a3e9956~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1000&h=420&s=52338&e=webp&b=fcfcfc)

  


其实，很多 Web 开发者都希望 CSS 能有一个原生的随机函数—— `random()` 。现在，大家得偿所愿了，CSS 引入了 `random()` 和 `random-item()` 函数：

  


-   `random()` 函数会在两个限制值之间选择一个随机值，有点类似于 JavaScript 的 `Math.random`
-   `random-item()` 函数会在一个选项列表中选择一个随机值

  


例如：

  


```CSS
.random {
    width: random(100px, 300px);
    height: random(100px, 300px, by 50px);
    background: random-item(--c, red, green, blue);
}
```

  


简单的解释一下上面的代码：

  


-   `random(100px, 300px)` 会生成一个介于 `100px` 和 `300px` 之间的随机值，可能是 `100px`、`300px` 或者它们之间的任何值，如 `234.5px`
-   `random(100px, 300px, by 50px)` 附带了一个 `by 50px` 的限制，意味着生的随机值必须是 `50px` 的倍数。也就是说，它会生成 `100px`、`150px`、`200px`、`250px` 或 `300px` 这样的值，确保结果是 `50px` 的倍数
-   `random-item(--c, red, green,blue)` 的作用是从给定的选项中随机选择一个值，可能是 `red`、`green`、`blue` 或者根据 `--c` 变量来生成

  


`random()` 和 `random-item()` 两者的区别是：

  


-   `random()` 用于生成一个在指定范围内的随机值，可以设置步长来限制生成的值
-   `random-item()` 用于从一组值中随机选择一个值，并且只能用于属性值中，会用 `random-item()` 解析出的随机值替代属性值

  


这两个函数可以在 CSS 中实现随机化效果，尤其在动画、布局、样式等场景中非常有用。

  


## `first-valid()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf554ea7d86542b0bff10ee60eb02be7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1980&h=990&s=13752&e=jpg&b=a48cc0)

  


到目前为止，**[CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)** 已经是非常成熟的 CSS 特性，我想大家在平时开发的过程都用了该特性。不过，在使用 CSS 自定义属性时的一个问题是，在声明中，它们被视为有效值，但它被引用时，很有可能是一个无效值。例如：

  


```CSS
.element {
    --a: 20px;
    --b: blue;
    background: var(--a);
}
```

  


上面代码中的 `--a` 实际上是一个有效值，它的值为 `20px` ，但将其应用于 `background` 时，`background` 的值为 `20px` ，可 `20px` 并不是一个有效的背景属性值，它将会导致 `background` 无效（被视为一个无效值）。

  


为了避免这种现象，CSS 引入了 `first-valid()` 函数，其思路是将一系列值传递给该函数，这些值会被解析，第一个有效的值将被使用。这在处理 CSS 自定义属性时尤其有用。例如：

  


```CSS
.element {
    --a: 20px;
    --b: blue;
    background: first-valid(var(--a), var(--b)); /* blue */
}
```

  


在这个示例中，`--a` 对于 `background` 是无效的，而 `--b` 则是有效的，这两个变量都传给了 `first-valid()` 函数，它会返回第一个对 `background` 属性有效的值，即 `--b` ，因此元素的背景颜色是 `blue` 。

  


## 函数参数分隔符

  


CSS 中很多函数是可以传入一个值列表，通常情况下，我们使用逗号（`,`）来分隔列表值，比如：

  


```CSS
rgb(125, 125, 112);
min(100px, 200px);
random(100px, 300px);
```

  


那么在这种情况下，我们应该如何区分值列表呢？假设我们想为页面设置一个随机的字体系列，并指定不同的选项：

  


-   `Times, serif`
-   `Arial, sans-serif`
-   `Roboto, sans-serif`

  


尝试一下，如果我们继续使用逗号分隔的话，CSS 代码会像下面这样：

  


```CSS
.element {
    font-family: random-item(Times, serif, Arial, sans-serif, Roboto, sans-serif);
}
```

  


这相当于我们给 `random-item()` 传递了六个值，这并不是我们想要的，我们想要的是给它传入三个值，每个值对应一个字体系列。这意味着，我们继续使用逗号（`,`）分隔符是无法满足我们的需求。这也是为什么有提议为函数引入分号（`;`）分隔符。如果使用分号分隔符，就可以很容易识别出一个参数的结束和下一个参数的开始：

  


```CSS
.element {
    font-family: random-item(
        Times, serif;
        Arial, sans-serif;
        Roboto, sans-serif
    );
}
```

  


这也意味着，一些新的函数可能会接受以分号分隔的值列表作为参数。为了避免混淆和歧义，我们可以用分号（`；`）代替分隔值的逗号（`,`）。

  


## `position` 值类型扩展

  


首先要明确一点，这里所说的 `position` 值类型指的位置属性，比如 `backgrounnd-position` 、`object-position` 、`mask-position` 等，请不要与 CSS 的 `position` 属性混淆。

  


如果你阅读过多语言排版相关的课程，你就应该知道，CSS 已经为外边距（`margin`）、内边距（`padding`）、边框（`border`）和定位（`position`）等属性提供了逻辑属性和（或）逻辑值，这些值是相对于文本书写方向的，并且可能会随着语言的不同而变化。如果你对这方面的知识感兴趣，可以移步阅读下面这些课程：

  


-   [CSS 排版：美化多语言网站技巧](https://juejin.cn/book/7395119173656903715/section/7445251774107090995)
-   [CSS 逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)
-   [Web 中的向左向右：Flexbox 和 Grid 布局中的 LTR 与 RTL](https://juejin.cn/book/7161370789680250917/section/7161625525763440647)
-   [Web 中的向左向右：Web 布局中 LTR 切换到 RTL 常见错误](https://juejin.cn/book/7161370789680250917/section/7161625415935590436)
-   [多语言 Web 应用需要注意的细节](https://juejin.cn/book/7199571709102391328/section/7199846338140307495)

  


现在这些概念被引入到了位置类型（比如 `background-position` 、`object-position` 和 `mask-position` 等），可以指定相对于文本流和方向的值。这些值包括：

  


-   `x-start`
-   `x-end`
-   `y-start`
-   `y-end`
-   `block-start`
-   `block-end`
-   `inline-start`
-   `inline-end`

  


比如，以后我们可以像下面这样来指定背景图片的位置：

  


```CSS
.element {
    background-image: conic-gradient(blue, red);
    background-size: 2rem 2rem;
    background-repeat: no-repeat;
    background-position: inline-start 15px block-end 30px;
}
```

  


## 小结

  


这节课中所提到的功能都是 CSS 的一些新特性，其中有些特性已经得到主流浏览器的支持，比如 `attr()` 、`calc-size()` 等，但有些还未得到浏览器的支持，但可以通过曲线方式实现相似的效果，比如 `toggle()` 、`sibling-count()` 和 `sibling-index()` 函数。

  


可以说，这些函数功能和特性非常有前景，其中一些功能也是 Web 开发者期待已久的功能，比如 `calc-size()` 、`sibling-count()` 、`sibling-index()` 、`random()` 和 `attr()` 等。

  


最后，需要提醒大家的是，这些功能虽然非常强大，但要应用于实际生产中时，可能还需要一段时间，或者说需要根据自己的实际用户群体进行评估，或者做好降级处理。不过，这不影响大家对新知识的探索与学习！如果你感兴趣或者对这些新特性有任何问题或意见，[可以在他们的 GitHub 仓库中提交问题](https://github.com/w3c/csswg-drafts/issues)！这样我们可以共同推进这些新的 CSS 特性得到主流浏览器的支持。
