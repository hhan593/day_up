稍微熟悉 CSS 的 Web 开发者应该对 `attr()` 函数不会感到陌生，但可以肯定的是，大多数 Web 开发者直到现在，只在伪元素（`::before` 、`::after` 或 `:marker`）的 `content` 属性上引用 `attr()` 函数，用于在 Web 页面上显示 HTML 属性的值（比如 `data-*` 定义的属性值）。其中最为典型的案例，就是通过它来制作简单的信息提示框（`Tooltips`）组件。

  


虽然 `attr()` 函数已经存在一段时间，并且在 CSS 中广泛使用。然而，它仅限于接受一个参数：你想要使用的属性名称。除此之外，`attr()` 函数返回的值总是被当作字符串处理，因此你只能在接受字符串值的属性中使用它。例如，前面提到的 `content` 属性。

  


在这里，我想告诉大家的是，自从 Chrome 133 版本开始，CSS 的 `attr()` 函数解锁了更多功能。现在，你可以将 `attr()` 用于任何 CSS 属性——包括自定义属性，并且它可以将值解析为除了字符串（`<string>`）之外的其他数据类型。

  


接下来，我们一起来探索 CSS `attr()` 函数解锁的新功能，看看这些新功能可以帮助我们做哪些事情？

  


## `attr()` 函数的基本用法

  


直到现在，我们只在伪元素（`::before` 、 `:after` 或 `::marker`）的 `content` 属性上使用 `attr()` 函数。通过 `attr()` 函数引用 HTML 的属性（通常是 `data-*` 的数据属性，但不仅限于数据属性）。例如， `attr(data-tooltip`) 会引用 HTML 元素上自定义的数据属性 `data-tooltip` 的值。到目前为止，它的最大局限性是，`attr(data-tooltip)` 始终解析为字符串。这意味着，直到现在，**`attr()`** **只能作为字符串值在伪元素的** **`content`** **属性中引用并使用**。其中最为典型的案例，就是通过这种方式来创建简单的信息提示框（`Tooltips`）组件。

庆幸的是，新的规范使 `attr()` 函数支持了除 `<string>` 之外的其他值类型，并且得到 Chrome 133 （或更高的版本）的支持。这意味着，现在使用 `attr()` 函数时，只需要声明它将是什么类型。例如下面这个简单的示例：

```HTML
    <div class="chart__item" data-num="3" data-color="green"></div>
    <div class="chart__item" data-num="4" data-color="slategray"></div>
    <div class="chart__item" data-num="5" data-color="mediumvioletred"></div>
    <div class="chart__item" data-num="6" data-color="tomato"></div>
</div>
```

  


```CSS
.chart__item {
    --s: 10vw;
    --_s: calc(var(--s) * attr(data-num type(<number>)));
    inline-size: var(--_s);
    background-color: attr(data-color type(<color>));
    
    &::before {
        content: attr(data-color);
    }
}
```

## `attr()` 函数的新特性

  


现在，我们可以将 `attr()` 函数与许多属性一起使用，这使得 `attr()` 函数比之前更强大。这个变化为 `attr()` 函数开启了一个全新的可能性世界。新的 `attr()` 函数的语法如下：

  


```CSS
attr(<attribute-name> type(<specific-type>))
```

  


有了这个新功能，你可以添加另一个参数来给这个数据属性指定类型。比如，如果我们希望根据 `data-color` 属性的值来指定元素的背景色，可以这样做。

  


首先，在你想要自定义的元素上指定 `data-*` 属性：

  


```HTML
<div data-color="red"></div>
```

  


然后，在 CSS 中，我们可以使用 `attr()` 函数与 `type()` 函数来指定 `data-color` 属性的值应该被视为 `<color>` 类型：

  


```CSS
[data-color="red"] {
    background-color: attr(data-color type(<color>));
}
```

  


`<color>` 类型表示一个颜色值，它与 `background-color` 属性完美配合。这会将 `div` 元素的背景颜色设置为 `red` 。

  


你甚至还可以更进一步，添加一个回退值，如下所示：

  


```CSS
[data-color="red"] {
    background-color: attr(data-color type(<color>), gray);
}
```

  


这意味着，现在的 `attr()` 函数除了支持 `<string>` 类型之外，你还可以在 `attr()` 函数中通过 `type()` 函数来指定其他值类型，这些类型包括：

  


-   `<angle>`: 角度（例如：`deg` 或 `turn`）
-   `<color>`: 颜色（例如：命名颜色、`rgb`、`hsl`、`oklch` 等）
-   `<custom-ident>`: 自定义标识符，用于命名诸如视图过渡名称等
-   `<integer>`: 离散的整数值
-   `<length>`: 数字加单位（例如： `1rem` 或 `10px` 等）
-   `<length-percentage>`: 可以是长度值或百分比（例如： `10px` 或 `10%` ）
-   `<number>`: 扩展整数，带有小数点以表示分数值
-   `<percentage>`: 百分比（`%`），通常相对于父对象
-   `<resolution>`: 用于媒体查询（例如：`dpi`）
-   `<time>`: 用于动画（例如：`s` 或 `ms`）
-   `<transform-function>`: 改变元素外观的函数（例如：`rotate()`）

  


注意：不能在图像或使用 `url` 语法类型时使用高级 `attr()`。根据规范，以下内容在计算值时都是无效的：

  


```CSS
background-image: src(attr(foo));   /* 无法直接使用 */
background-image: image(attr(foo));  /* 不能在其他接受 URL 的函数中使用 */
background-image: src(string("http://ex.com/?token=" attr(foo))); /* 不能通过另一个函数“清洗” */
--foo: attr(foo); background-image(src(var(--foo))); /* 不能通过另一个属性“清洗” */
```

  


所以不幸的是，你无法使用这种技术来处理图像或图标。

  


## `attr()` 应用场景

  


`attr()` 扩展后的功能非常强大，并且与我们目前构建的前端系统的方式（即通过在构建组件时传递的属性）更加契合。接下来，我们通过一些实际的案例来展示 `attr()` 函数的使用场景以及它扩展后的功能。

  


### 产品卡片的展示

  


在一些电商类的 Web 应用中，在某些类别的产品卡片中，会有颜色选择项。比如淘宝服饰类的产品卡片：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ee3b4159c84445e28e5fec6d9a00373e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2860&h=1452&s=701222&e=jpg&b=f4efee)

  


假设我们把上图中“颜色分类”用简单的颜色来描述：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/989819fbbaa14ae1a74143b20c8643b2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2688&h=1204&s=2037026&e=png&b=f5eeec)

  


这些颜色可能来自某个数据库，你可能会通过一个无序列表来展示这些颜色：

  


```HTML
<ul>
    <li data-color="#FBCEB1"></li>
    <li data-color="#B4B7B4"></li>
    <li data-color="#000000"></li>
    <li data-color="#FFFFFF"></li>
    <li data-color="#FFC0CB"></li>
    <li data-color="#FF00FF"></li>
    <li data-color="#6F4F28"></li>
</ul>
```

  


在此之前，我们可能会在 `li` 元素以内联的方式将颜色值存储到一个 CSS 变量中：

  


```HTML
<ul>
    <li data-color="#FBCEB1" style="--c: #FBCEB1;"></li>
    <li data-color="#B4B7B4" style="--c: #B4B7B4;"></li>
    <li data-color="#000000" style="--c: #000000;"></li>
    <li data-color="#FFFFFF" style="--c: #FFFFFF;"></li>
    <li data-color="#FFC0CB" style="--c: #FFC0CB;"></li>
    <li data-color="#FF00FF" style="--c: #FF00FF;"></li>
    <li data-color="#6F4F28" style="--c: #6F4F28;"></li>
</ul>
```

  


然后在 CSS 中将 `li` 的 `background-color` 设置为 `var(--c)`:

  


```CSS
li {
    background-color: var(--c);
}
```

  


这样做是可以的，但它有一个前提，每个 `li` 中的 `style` 需要人肉去内置或通过 JavaScript 来设置 `--c` 变量的值。比如：

  


```JavaScript
const listItems = document.querySelectorAll('ul li');

listItems.forEach(item => {
    const color = item.getAttribute('data-color');  
    item.style.setProperty('--c', color);
});
```

  


但这样做并不是最理想的。我个的原则是，CSS 能完成的事情，绝不依赖 JavaScript 来完成。我更倾向于保持更好的关注点分离。

  


现在，我们完全可以借助 CSS 的 `attr()` 函数的新功能，直接访问 `data-color` 属性，将其转换为 `<color>` 类型，然后应用于 `li` 元素的 `background-color` ：

  


```CSS
.card li {
    background-color: attr(data-color type(<color>), #000);
}
```

### 星级评分组件

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ff38ee9756b84679a736134c2e2b0c10~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=24842&e=jpg&b=32b5a3)

  


使用 `attr()` 函数，我们可以在上面进度条示例的基础上快速改造出一个星级评分组件。

  


```HTML
<div class="star-rating" data-rating="4.5"></div>
<div class="star-rating" data-rating="3"></div>
<div class="star-rating" data-rating="4.7"></div>
<div class="star-rating" data-rating="2.3"></div>
```

  


我们知道，通常情况下，一个星级评分组件会包括五颗星，这五颗星会铺满整个容器（所占宽度为 `100%`），那么每颗星所占据空间就是 `20%` 。假设我们的评分是 `4.5` 分，那么它所占空间就是 `4.5` 乘以 `2` 再除于 `10` ，最终这个值转换为 `90%` ，或者直接是 `4.5` 乘以 `20%` 。这意味着，我们需要将 `data-rating` 的值转换为百分比值，在 CSS 中，我们可以通过 `calc()` 函数来计算，比如 `calc(attr(data-rating type(<number>) * 20%))` 。我们可以把这个值存储在一个 CSS 变量中，比如：

  


```CSS
.star-rating {
    --percent-fill: calc(attr(data-rating type(<number>)) * 20%);
}
```

  


然后，我们使用两个线性渐变作为背景，其中底色是灰色，它将填充整个空间，上面一层是亮色（星星所占空间），由 `--percent-fill` 来控制渐变颜色的起止点：

  


```CSS
.star-rating {
    --percent-fill: calc(attr(data-rating type(<number>)) * 20%);
    
    &::before {
        background: 
            linear-gradient(to right, gold var(--percent-fill), transparent var(--percent-fill)),
            linear-gradient(to right, grey, grey);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3adee8829ef4e869ba50dd77adc7167~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2278&h=488&s=32373&e=png&b=ffffff)

```CSS

    &::before {
        background:
```

### 控制网格布局

  


我们还可以通过 `attr()` 来控制网格布局：

  


```HTML
<div class="grid">
    <div data-columns="3"></div>
    <div data-column-start="2" data-column-end="7" data-row="2"></div>
    <div data-columns="1" data-row="3"></div>
    <div data-column-start="5" data-column-end="9" data-row="4"></div>
</div>
```

  


正如你所看到的，我们在网格项目中定义了很多个自定义数据属性，其中：

  


-   `data-row` 明确设置了网格项目所在的行
-   `data-columns` 明确设置了网格项目应该跨越多少列
-   `data-column-start` 和 `data-column-end` 设置了列应该从哪里开始或结束

  


```CSS
.grid {
    display: grid;
    grid-template-columns: repeat(8, 1fr);

    > div {
        &[data-columns] {
            grid-column: span attr(data-columns type(<integer>));
        }
    
        &[data-column-start] {
            grid-column-start: attr(data-column-start type(<integer>));
        }
        
        &[data-column-end] {
            grid-column-end: attr(data-column-end type(<integer>));
        }
    
        &[data-row] {
            grid-row: attr(data-row type(<integer>));
        }
    }
}
```

### 使用 `attr()` 控制颜色

  


使用 `attr()` 函数来指定你想使用的颜色是非常棒的。例如：

  


```HTML
<p>In the <span data-color="gold">golden</span> light of dawn, <span data-color="#eee">silver</span> mists rise from <span data-color="oklch(84.08% 0.3256 143.87 / 86.52%)">emerald</span> valleys, painting the world in a shimmering canvas that whispers of <span data-color="hsl(285 100% 77%)">purple</span> dreams yet to unfold and promises kept in the gentle awakening of a new day.</p>
```

  


```CSS
[data-color] {
    color: attr(data-color type(<color>));
    background-color: color-mix(
        in srgb,
        attr(data-color type(<color>)),
        black 40%
    );
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d62ed78ee7cf4cae9ccc781f1c4ec757~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2278&h=608&s=93653&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/jEOeagx

  


这个示例突出了使用 CSS 额外“层”来处理事情的优点，而不是直接使用内联样式。在这里，我不仅仅是设置一个颜色，而是使用给定的颜色作为实际的颜色，然后[将相同的颜色与 `color-mix()` 函数混合一些黑色](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)，作为背景色。

  


为了进一步强调你所拥有的控制，我们可以在[相对颜色语法](https://juejin.cn/book/7395119173656903715/section/7445265166482505778)中[使用 `min()` 和 `max()` 值](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)，确保例如颜色具有一个我们认为对可读性最合适的最小亮度级别。

  


```HTML
<ul>
    <li data-color="orange">Lorem ipsum dolor, sit amet <span data-color="orange">consectetur</span> adipisicing.</li>
    <li data-color="darkred">Et, in? <span data-color="darkred">Impedit</span> voluptatum quibusdam doloribus nostrum!</li>
    <li data-color="plum">Commodi, dolorum <span data-color="plum">voluptatem</span> officia maxime accusantium architecto!</li>
    <li data-color="blue">Qui earum <span data-color="blue">adipisci</span> nihil eum itaque asperiores?</li>
    <li data-color="rebeccapurple">Enim <span data-color="rebeccapurple">consequatur</span> sapiente vero esse minima earum!</li>
</ul>
```

  


```CSS
li[data-color]::before {
    background-color: attr(data-color type(<color>));
}

span[data-color] {
    color: oklch(from attr(data-color type(<color>)) max(l, 0.9) c h);
}
```

  


每个项目列表符的颜色是 `data-color` 设置的颜色，但当我们用它为行中的文字着色时，我们将颜色转换为 `oklch()` 并确保其中的 `l` 部分（亮度）具有最小值 `0.9`（通过 `max(l, 0.9)` 来确保这一点）。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/61e8eb7b9b784dd3a5df09ea0cfb141f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2278&h=648&s=107491&e=png&b=222222)

  


> Demo 地址：https://codepen.io/airen/full/MYWPrYZ

  


我们可以将 `attr(data-color type(<color>))` 存储在一个 CSS 变量中，比如 `--c` ，就可以对 HTML 和 CSS 代码做进一步的优化：

  


```HTML
<ul>
    <li data-color="orange">Lorem ipsum dolor, sit amet <span>consectetur</span> adipisicing.</li>
    <li data-color="darkred">Et, in? <span>Impedit</span> voluptatum quibusdam doloribus nostrum!</li>
```

### 使用 `attr()` 控制字体大小

  


在上一节课中，有一个示例演示了 [`pow()` 函数](https://juejin.cn/book/7395119173656903715/section/7446278438929104922#heading-0)与自定义属性结合创建字体大小的**模数比例**（Modular Scale），即通过固定比例（`ratio`）确定页面上的所有字体大小，使它们保持一致的视觉层级。

  


```HTML
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

  


```CSS
:is(h1, h2, h3, h4, h5, h6) {
    font-size: calc(1rem * pow(1.5, var(--exp)));
}


h1 {
    --exp: 4;
}
h2 {
   --exp:3;
}
h3 {
    --exp:2;
}
h4 {
    --exp:1;
}
h5 {
    --exp:0;
}
h6 {
    --exp:-1;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/26c94c0471b24815bf99879aed4512a1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2278&h=1348&s=115151&e=png&b=ffffff)

  


> Demo: https://codepen.io/airen/full/ZYEMPQx

  


现在，我们可以使用 `att()` 来引用自定义数据必性：

  


```HTML
<h1 data-exp="4">Heading 1</h1>
<h2 data-exp="3">Heading 2</h2>
<h3 data-exp="2">Heading 3</h3>
<h4 data-exp="1">Heading 4</h4>
<h5 data-exp="0">Heading 5</h5>
<h6 data-exp="-1">Heading 6</h6>
```

  


```CSS
:is(h1, h2, h3, h4, h5,h6) {
    font-size: calc(1rem * pow(1.5, attr(data-exp type(<number>))));
}
```

  


效果是一样的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/10cc69299a6e4ee8bb4740d23880139d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2278&h=1348&s=115151&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/YPzJYxZ

  


也许你的设计系统对字体大小有严格的规则，只允许每次增加 `5px`。[通过使用 CSS 的 `round()` 函数](https://juejin.cn/book/7395119173656903715/section/7446278438929104922#heading-0)，我们可以通过这种方法实现这一点。

### **锚定弹出框**

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8bb0b65b5313427cb67489457f5063ce~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1360&h=340&s=6734366&e=gif&f=141&b=fefcfc)

上面这个示例，展示了多个弹出菜单与不同按钮的关联。

  


以前，我们需要为每个菜单按钮编写样式，以锚定菜单项的弹出框。当前最合理的做法是将其内联，具体的写法如下：

  


```HTML
<div class="menu">
    <button popovertarget="menu-items-01" style="anchor-name: --menu-01">
        <!-- 省略其他内容 -->
    </button>
    <ul id="menu-items-01" popover style="position-anchor: --menu-01" role="menu">
        <!-- 省略其他内容 -->
    </ul>
</div>
```

  


现在，你可以省去这些重复的个别样式，只需要在父元素上添加一次数据属性，如下所示：

  


```HTML
<div class="menu" data-anchor="--menu-01">
    <button popovertarget="menu-items-01">
        <!-- 省略其他内容 -->  
    </button>
    <ul id="menu-items-01" popover role="menu">
        <!-- 省略其他内容 -->
    </ul>
</div>
```

  


然后编写一次 CSS，使每个按钮和弹出菜单能够利用它们独特的 `data-anchor` 来设置 `anchor-name` 和 `position-anchor`：

  


```CSS
.menu {
    --anchor-id: attr(data-anchor type(<custom-ident>));

    button {
        anchor-name: var(--anchor-id);
    }

    [popover] {
        position-anchor: var(--anchor-id);
    }
}
```

### 使用 `attr()` 和 `type(<custom-ident>)` 的自动视图过渡名称

  


最后我们来看一个有关于视图过渡的用例。在下面这个示例中，`view-transition-name` 属性由元素的 `id` 属性决定。该属性值被解析为 `<custom-ident>` ，这是 `view-transition-name` 所需的类型。如果没有 `attr()` ，你必须为每张卡片声明一个 `view-transition-name` ，而现在你可以一次性设置它。

  


```HTML
<div class="cards">
    <div class="card" id="card-1"></div>
    <div class="card" id="card-2"></div>
    <div class="card" id="card-3"></div>
    <div class="card" id="card-4"></div>
</div>
```

  


```CSS
.card[id] {
    view-transition-name: attr(
        id type(<custom-ident>),
        none
    ); /* card-1, card-2, card-3, etc. */
    view-transition-class: card;
}

:root {
    view-transition-name: none;
}

::view-transition {
    pointer-events: none;
}
```

## 为什么总是使用 `data-*` ？

  


`attr()` 函数适用于常规的自定义属性和 `data-*` 属性。使用 `data-*` 属性确实有一些独特的好处，它也是使用此功能的一种更有目的方式。`data-*` 的好处包括：

  


-   不会与现有属性发生命名空间的冲突
-   `data-*` 使你可以通过 `element.dataset` 在 JavaScript 中访问数据列表，这意味着你可以更新它

  


换句话说，`data-*` 属性用于存储可以通过 JavaScript 访问的附加信息，而普通属性用于定义元素的固有属性。所以对于 CSS 来说，这取决于你和你的需求。

  


## 小结

  


我觉得 CSS 的 `attr()` 新特性非常棒，因为你将能够用尽可能最少的代码打造 UI。你可以自定义 UI 的每个方面，无需触碰 CSS。这本质上是一种“写一次，到处使用”的方式，使得 CSS 更具可重用性和可维护性。

  


目前，带类型的 `attr()` 函数仅在最新的 Chromium 内核浏览器中得到支持（具体来说是 Chrome 133 及更高版本）。但我相信其他浏览器很快也会支持这一功能。所以，我们值得期待！
