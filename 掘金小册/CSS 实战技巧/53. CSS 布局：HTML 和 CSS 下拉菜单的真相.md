我想大家对下拉菜单（`Dropdown`）应该不会感到陌生，因为它是 Web 上常见的一种设计模式。实现这种设计模式有多种姿势，例如下拉菜单、弹出菜单或弹出框。不管你曾经倾向于哪种方式，都是想实现它的核心功能——通过用户交互展示内容，即内容在显示和隐藏之间来回切换。

  


我们往往会依赖已有的知识和习惯。当涉及到内容的切换时，可能会习惯性使用 `display: none` 或 `opacity: 0` 或 `translate` ，再加上一些 JavaScript。不过，如今天的 Web 更“现代化”了，现在是时候全面了解不同的内容切换方式了。这节课，我将以常见的下拉菜单为例，带领大家如何使用原生的 HTML API 和 CSS 新特性来创建下拉菜单（内容显示与隐藏）。

  


## 背景

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2ffbfbc4973c4d6c9b6ffa96b75a04c0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1082&h=628&s=1273050&e=gif&f=200&b=f3f3f3)

  


一直以来，Web 开发者想要用一个按钮来展示某段内容，从来都不是一件简单或轻松的事情，总会让人想到各种“万一”。万一用户点击了按钮外的区域呢？按了 `Escape` 键呢？动画效果呢？这样的顾虑似乎没完没了。伴随着这些“万一”，Web 开发者不得不使用 JavaScript 来解决这些问题。

  


然而，这样一个简单的交互模式难道不应该通过 HTML 和 CSS 直接来实现吗？是的，我可以肯定地告诉大家，现在我们可以仅使用 HTML 和 CSS 来实现。因为浏览器已经支持了很多新的、原生的 API 与特性。接下来，我们就以上面所展示的下拉菜单为例，看看哪些更现代的方法可以实现，它们各利的利弊是什么？

  


我们开始吧！

  


## 创建下拉菜单

  


我个人有一个习惯，在开发过程中，如果能不依赖 JavaScript 的前提下能实现需求，我绝对不使用 JavaScript。我们的目标是要实现一个拉下菜单。如果这个下拉菜单只需具备最简单的内容显示和隐藏之间的切换，我首先会想到的是通过复选框的 `:checked` 与 CSS 选择器 `~` 来实现。

  


这种方案对 HTML 的结构有着较强的要求：

  


```HTML
<div class="dropdown">
    <input type="checkbox" id="dropdown" class="sr-only">

    <label class="dropdown__face" for="dropdown">
        <figure>
            <img src="http://i.pravatar.cc/500?img=2" alt="">
        </figure>
        W3cplus
        <svg viewBox="0 0 320 512">
            <path d="M137.4 374.6c12.5 12.5 32.8 12.5 45.3 0l128-128c9.2-9.2 11.9-22.9 6.9-34.9s-16.6-19.8-29.6-19.8L32 192c-12.9 0-24.6 7.8-29.6 19.8s-2.2 25.7 6.9 34.9l128 128z" fill="currentColor" />
        </svg>
    </label>

    <ul class="dropdown__menu">
        <li>
            <a href="#">
                <svg viewBox="0 0 448 512">
                     <path d="M224 256A128 128 0 1 0 224 0a128 128 0 1 0 0 256zm-45.7 48C79.8 304 0 383.8 0 482.3C0 498.7 13.3 512 29.7 512l388.6 0c16.4 0 29.7-13.3 29.7-29.7C448 383.8 368.2 304 269.7 304l-91.4 0z" fill="currentColor" />
                </svg>
                我的主页
            </a>
        </li>
        <!-- 省略其他 li -->
    </ul>
</div>
```

  


默认状态下（即复选框未选中），下拉菜单 `.dropdown__menu` 的样式设置为不可见：

  


```CSS
.dropdown {
    position: relative;
}

.dropdown__menu {
    position: absolute;
    right: 0;
    top: 50%;
    visibility: hidden;
    z-index: -1;
    opacity: 0;
    transition: all 0.4s cubic-bezier(0.93, 0.88, 0.1, 0.8);
}    
```

  


当复选框选中时，下拉菜单 `.dropdown__menu` 才可见。我们可以通过 CSS 的 `:checked` 和 `~` 选择器来处理：

  


```CSS
input[type="checkbox"]:checked ~ .dropdown__menu {
    translate: 0 60px;
    visibility: visible;
    opacity: 1;
}
```

  


这个时候，用户点击 `label` 标签时，复选框会处于选中状态，下拉菜单就会显示；当用户再次点击 `label` 标签时，复选框会处于未选中状态，下拉菜单就会隐藏。如此一来，实现了一个具有简单切换下拉菜单显示和隐藏的功能：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73ada6a85fef42eab5b554d9c3aa42f4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1082&h=628&s=1273050&e=gif&f=200&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/OPLmoKw

  


随着 [CSS 的 `:has()` 选择器](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)的出现，构建这个下拉菜单时，HTML 结构就不需要那么严谨了，因为 `:has()` 选择器可以和 `input` 的状态结合起来作为新的判断条件。与此同时，我们可以不需要通过 HTML 文档结构来选中下拉菜单（`~` 和 `+` 选择器对 HTML 结构要求比较严谨）。现在，CSS 可以这样写：

  


```CSS
.dropdown {
    position: relative;
    
    &:has(:checked) {
        .dropdown__menu {
            translate: 0 60px;
            visibility: visible;
            opacity: 1;
        }
    }
}

.dropdown__menu {
    position: absolute;
    right: 0;
    top: 50%;
    visibility: hidden;
    z-index: -1;
    opacity: 0;
    transition: all 0.4s cubic-bezier(0.93, 0.88, 0.1, 0.8);
} 
```

> Demo 地址：https://codepen.io/airen/full/jENmXEz

## 使用现代特性来优化下拉菜单

  


你会发现，下拉菜单（`.drop__menu`）的定位还是相对于父容器（`.dropdown`）定位。事实上，我们可以利用 [CSS 锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)特性来对下拉菜单进行优化。比如，使下拉菜单 `.dropdown__menu` 相对于 `label` 进行定位。这样做的好处之一是，`lable` 标签位置有任何变化，下拉菜单 `.dropdown__menu` 都能跟着变化。这意味将元素相互定位得更加灵活和可靠。

  


我们要做的事情很简单，在 `label` （视为锚点元素）元素上使用 `anchor-name` 定义一个锚点名称，比如 `--dropdown` ，然后在目标元素 `.dropdown__menu` 是使用 `position-anchor` 属性引用已定义的锚点 `--dropdown` ，同时使用 `position-area` 来设置目标元素相对于锚点元素的偏移位置：

  


```CSS
.dropdown {
    filter: url(#goo);
    
    &:has(:checked) {
        .dropdown__menu {
            translate: 0 30px;
            visibility: visible;
            opacity: 1;
        }
        
        .dropdown__face svg {
            rotate: 180deg;
        }
    }
}

.dropdown__face {
    anchor-name: --dropdown;
}

.dropdown__menu {
    position: absolute;
    position-anchor:--dropdown;
    position-area:end span-all;
    visibility: hidden;
    z-index: -1;
    opacity: 0;
    translate: 0 -30px;
    transition: all 0.4s cubic-bezier(0.93, 0.88, 0.1, 0.8);
}
```

  


你在上面的代码中找不到 `position: relative` 相关的代码，但锚点定位使们能获得与相对定位一样的效果，甚至还要更灵活：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/756dda6464eb442685cc225f3af5e681~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1082&h=628&s=1273050&e=gif&f=200&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/vEBmvxy

  


你可能发没有发现锚点定位带来的便利性，我这里向大家演示一下。尝试着把 `label` 调大，你会发现，定位依旧是完美的，我们不再需要因为 `label` 变大了，要重新调整下拉菜单的偏移位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/441629e113ef43cb971965e34fdbb686~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1074&h=664&s=1579446&e=gif&f=307&b=f3f3f3)

  


CSS 锚点定位的功能不仅限于此，它还可以根据可视区域来调整定位元素的偏移位置。例如，一个具有多级下拉菜单，锚点元素可能会位于不同的区域，那么锚点定位可以使它有一个更为灵活的展示效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6d1e86b51ba94306a62b0e3b3aca9f16~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=982&h=632&s=3369683&e=gif&f=376&b=f3f5fa)

  


正如上图所示，在没有锚点定位之前，即使使用 JavaScript 来实现这个效果都不是件容易的事情。

  


到目前为止，我们在下拉菜单中引用了锚点定位，锚点元素（`lable`）和目标元素（下拉菜单 `.dropdown__menu`）相互定位变得更加灵活和可靠。但你可能已经发现了，用户只能点击 `label` 元素才能显示和隐藏下拉菜单，用户点击别的区域并不能隐藏下拉菜单。

  


以往要实现这种效果，我们不得不借助 JavaScript 来实现。庆幸的是，现在我们可以通过 HTML 的 Popover API 来实现，而且不需要任何 JavaScript。只需要使用 `popover` 和 `popovertarget` 属性（`invoketarget` 将会替代 `popovertarget`）就可以让下拉菜单显示和隐藏，而且用户点击下拉菜单区域之外就可以隐藏下拉菜单，并且完全不需要使用 JavaScript。

  


如果使用 Popover API 来制作下拉菜单，不再需要复选框，也不需要使用 `:has()` 来做判断。一切的交互行为都将交给浏览器来处理。不过，我们的 HTML 结构需要做一些调整：

  


```HTML
<div class="dropdown">
    <button popovertarget="dropdown" invoketarget="dropdown" class="dropdown__face">
        <figure>
            <img src="http://i.pravatar.cc/500?img=2" alt="">
        </figure>
        W3cplus
        <svg viewBox="0 0 320 512">
            <path d="M137.4 374.6c12.5 12.5 32.8 12.5 45.3 0l128-128c9.2-9.2 11.9-22.9 6.9-34.9s-16.6-19.8-29.6-19.8L32 192c-12.9 0-24.6 7.8-29.6 19.8s-2.2 25.7 6.9 34.9l128 128z" fill="currentColor" />
        </svg>
    </button>

    <ul popover="auto" id="dropdown" class="dropdown__menu">
        <li>
            <a href="#">
                <svg viewBox="0 0 448 512">
                    <path d="M224 256A128 128 0 1 0 224 0a128 128 0 1 0 0 256zm-45.7 48C79.8 304 0 383.8 0 482.3C0 498.7 13.3 512 29.7 512l388.6 0c16.4 0 29.7-13.3 29.7-29.7C448 383.8 368.2 304 269.7 304l-91.4 0z" fill="currentColor" />
                </svg>
                我的主页
            </a>
        </li>
        <!-- 省略其他 CSS -->
    </ul>
</div>    
```

  


相应的 CSS 调整为：

  


```CSS
.dropdown:has(:popover-open) {
    .dropdown__face svg {
        rotate: 180deg;
    }
}

[popovertarget] {
    anchor-name: --dropdown;
}

[popover] {
    position: absolute;
    inset: unset;
    top: anchor(--dropdown bottom);
    right: anchor(--dropdown end);
    position-anchor: --dropdown;
    width: inherit;
    z-index: -1;
    transform-origin: top right;

    &:popover-open {      
        translate: 0 30px;
        opacity: 1;
        scale: 1;
        overflow: unset;
        transition: 
            translate 0.5s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            scale 0.5s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            opacity 0.5s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            display 0.5s allow-discrete cubic-bezier(0.93, 0.88, 0.1, 0.8),
            overlay 0.5s allow-discrete cubic-bezier(0.93, 0.88, 0.1, 0.8);
    
        @starting-style {
            opacity: 0;
            scale: 0;
            translate: 0 0;
        }
    }
    
    &:not(:popover-open) {
        opacity: 0;
        scale: 0;
        translate: 0 -30px;
        transition: 
            translate 0.3s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            opacity 0.3s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            scale 0.3s cubic-bezier(0.93, 0.88, 0.1, 0.8),
            display 0.3s allow-discrete cubic-bezier(0.93, 0.88, 0.1, 0.8),
            overlay 0.3s allow-discrete cubic-bezier(0.93, 0.88, 0.1, 0.8);
    }
}
```
