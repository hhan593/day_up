小册之前的课程多次提到了 CSS 锚点定位（Anchor Position）特性。它为我们提供了一种全新的定位方式，使 Web 开发者能够精确地将元素相对于其他元素进行定位。这个特性简化了界面元素在浏览器中的定位，例如将错误消息显示在输入字段旁边、工具提示框、下拉菜单等，直接在浏览器中实现。

  


如果你关注过我的另一本小册《[现代 CSS](https://juejin.cn/book/7223230325122400288?suid=1908407916041614&source=pc)》，应该知道有一节课专门介绍了 **[CSS 锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)** 特性。今天，重新花一节课的篇幅来聊 CSS 锚点定位是有原因的。因为 CSS 锚点定位的语法有所调整：

  


-   将 `inset-area` 属性命名为 `position-area`
-   将 `position-try-options` 属性重新命名为 `position-try-fallbacks`

  


简单地说，锚点定位最初是一个实验性属性，使用了 `anchor()` 函数和 `@position-fallback` （不是 `position-try` ，我们稍后会谈到）。随后，Webkit 提出了一个新方案，采用更类似网格系统的替代方案。从目前来看，过程中的一些挫折导致进展较慢，但也有一些好处：

  


-   我们可以同时拥有两得的优点，两个想法都很棒
-   原始的 `anchor()` 方案可能更强大和灵活，而网格方案则提供了一个更简单，快速的日常解决方案

  


除此之外，也有一些缺点：**由于存在多种使用锚点定位的方式，这增加了大家的学习成本，容易混淆**。

  


因此，在这节课中，我将全面的向大家介绍 CSS 锚点定位，使大家以最低的成本和最有效的方式能掌握 CSS 锚点定位的所有特性。在这个过程中，我们会探讨 CSS 锚点定位的关键概念，如锚点元素和锚点定位元素，演示如何设置元素之间的关系、使用 `anchor()` 函数进行定位，以及利用新的 `anchor-center` 值轻松实现居中。还将涉及一些高级技术，如多个锚点、`position-area` 属性、`anchor-size()` 函数和可见性选项等。

  


通过利用 CSS 锚点定位，Web 开发者可以简化实现，减少对第三方库的依赖，并在浏览器中释放未开发的创造力。那么，我们开始吧！

  


## 基础概念和术语

  


CSS 锚点定位是一个新特性，用于控制元素的布局，它的特点包括：“**可以控制当布局元素不在可视区域内时的行为**”和“**即使是通过** **[Popover API](https://juejin.cn/book/7395119173656903715/section/7442515150541684746)** **等被放置在顶部层的元素，也可以相对于锚点元素来控制其位置**”。此前，通常是通过 JavaScript 来计算元素的坐标和大小，然后根据这些计算结果来布局。而 CSS 锚点定位可以仅用 CSS 来处理这些布局操作。

  


简单地说，CSS 锚点定位引入了一种全新的方式，用于在页面上相对定位元素。为了让我们能更容易理解，以下是一些关键术语，用于确定哪个元素与哪个元素连接：

  


```HTML
<div class="anchor">anchor</div>
<div class="target">target</div>
```

  


上面代码中的 `.anchor` 和 `.target` 是两个相邻元素（兄弟元素），其中 `.anchor` 是锚点元素，`.target` 是目标元素：

  


-   **锚点元素**：这是用作参考的元素，用于定位其他元素，因此称为“锚点”
-   **目标元素**：这是一个相对于一个或多个锚点定位的绝对定位元素（`position: absolute`）。从现在开始，我们将使用“目标元素”来指代它，但在规范中通常将其称为“绝对定位元素”。

  


CSS 锚点位的核心是“绝对定位的元素”（即目标元素，设置 `position` 值为 `absolute` 的元素），我们接下来使用 CSS 锚点定位特性，可以使目标元素 `.target` 相对于锚点元素 `.anchor` 进行绝对定位，在此之间是不可能。

  


在深入学习 CSS 锚点定位之前，我们需要复习一些相关的概念：

  


-   **包含块（Containing Block）** ：这是包含元素的盒子。对于一个绝对定位的元素，其包含块是视口（Viewport）、最近的非 `static` 定位的祖先元素，或某些具有特定属性值（如 `contain` 或 `filter`）的祖先元素
-   **内边距调整的包含块（Inset-Modified Containing Block, IMCB）** ：对于绝对定位的元素，`inset` 属性（如 `top`、`right`、`bottom`、`left` 等）会缩小包含块的尺寸，从而生成一个新的盒子，称为“**内边距调整的包含块**”（**IMCB**）。这是一个非常重要的概念，因为我们之后要讨论的属性（如 `position-area` 和 `position-try-order`）依赖于这一概念。

  


理解概念总是枯燥无味的。既然如此，我们就从最简单的示例开始。

  


```HTML
<div class="anchor">anchor</div>
<div class="target">target</div>
```

  


这是再普通不过的 HTML 结构了。现在，我们给他们添加一点 CSS ：

  


```CSS
.anchor {
    display: grid;
    place-content: center;
    background: #09aefe;
    color: #fff;
    justify-self: center;
    padding: .5em;
    border-radius: .25em;
    font-size: 2rem;
    
    svg {
        display: block;
        width: 2em;
        aspect-ratio: 1;
    }
}
  
.target {
    display: grid;
    place-content: center;
    padding: .5em;
    background: #f50;
    color: #fff;
    border-radius: .25em;
}

body {
    display: grid;
    place-content: center;
    padding: 1rem;
    gap: 2rem;
}
```

  


现在，我们给目标元素添加 `position: absolute` ，并且设置一定的偏移值：

  


```CSSS
.target {
    position: absolute;
    top: 1rem;
    left: 50%;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/50a244718bff427aa54dc6e9b903f3f8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1080&s=146050&e=jpg&b=f4f4f4)

  


这样的结果是符合大家预期的，因为我们所熟悉的绝对定位，绝对定位元素 `.target` 在没有找到相对定位点时，它的偏移计算是相对于视窗来计算的。

  


然而，我们期望的效果是绝对定位元素（目标元素 `.target`）相对于锚点元素 `.anchor` 定位。现在，我们使用 CSS 锚点定位可以做到，而且有多种不同的方法。

  


第一方法是使用 `anchor()` 。首先，我们需要使用 `anchor-name` 定义我们的锚点。我们在 `.anchor` 中添加以下代码：

  


```CSS
.anchor {
    anchor-name: --anchor-name;
}
```

  


接下来，我们将 `position-anchor` 添加到 `.target` 元素中，这将创建元素之前缺失的链接（锚点）：

  


```CSS
.target {
    position: absolute;
    position-anchor: --anchor-name;
}
```

  


到目前为止，呈现给我们的结果没有明显的变化，因为我们仍然需要定位我们的元素。为此，我们在 `.target` 元素中引入 `anchor()` 来设置它的偏移值：

  


```CSS
.target {
    position: absolute;
    position-anchor: --anchor-name;
    top: anchor(center);
    left: anchor(right);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7ad6b075c61e4ae7b7c6447e475a7dad~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1102&s=145591&e=jpg&b=f4f4f4)

  


你会发现 `.target` 元素的顶部位置放置在 `.anchor` 的中心（`Y` 轴），并且将 `.target` 元素的左边位置放置在 `.anchor` 的右侧。更有意思的是，你拖动 `.anchor` 元素，目标元素 `.target` 始终会跟随 `.anchor` 移动，并且位置保持一致。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d90ba385cd3b4a57a48616bad7bcc91a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1088&h=574&s=1167925&e=gif&f=120&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/OPLxqjd

  


这就是 CSS 锚点定位中的 `anchor-name` 、`position-anchor` 属性和 `anchor()` 函数给绝对定位元素带来的变化。你可能会对它们感到好奇，想一探其究竟，对吧！请不要着急，我们稍后会介绍。

  


我们再来看第二种方式，即 `position-area` 。同样的，在 `.anchor` 中使用 `anchor-name` 设置一个锚点名称，并在 `.target` 中使用 `position-anchor` 引用已注册的锚点名称：

  


```CSS
.anchor {
    anchor-name: --anchor-name;
}

.target {
    position: absolute;
    position-anchor: --anchor-name;
}
```

  


不同的是，我们使用 `position-area` 来替代之前的偏移设置：

  


```CSS
.anchor {
    anchor-name: --anchor-name;
}

.target {
    position: absolute;
    position-anchor: --anchor-name;
    position-area: center end;
}
```

  


这个时候，你会发现 `.target` 元素的 `Y` 轴中心与 `.anchor` 元素的 `Y` 轴中心保持一致，而且 `.target` 元素的左边位置放置在 `.anchor` 的右侧。与此同时，`.target` 同样会跟着 `.anchor` 移动，但绝对定位的偏移位置不会变化（始终相对于 `.anchor` 定位）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5459d26af21946b492b09e525002bb50~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1094&h=564&s=831760&e=gif&f=100&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/jENGJJW

  


是的，现在又多了上个 `position-area` ，这是一个全新的属性。我想，很多 Web 开发者应该是初次知道 CSS 还有一个 `position-area` 属性。

  


正如大家所看到的，这两种方式都是 CSS 锚点定位提供的全新的定位方式，它们都可以使目标元素 `.target` 相对于锚点元素 `.anchor` 进行定位。既然如此，我们就有必要深入的了解它们，因为它们是 CSS 锚点定位核心功能之一。

  


## 将目标元素锚定在锚点元素

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1c43f7ca27134b8f8819c9033c9355ec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1102&s=225780&e=jpg&b=f4f4f4)

  


默认情况之下，一个普通元素既不是锚点元素，也不是目标元素。但是，CSS 锚点定位中的 `anchor-name` 和 `position-anchor` 属性可以将一个普通元素设置为锚点元素和目标元素。

  


最常见的方式是通过 `anchor-name` 显式将一个元素设置为锚点：

  


```CSS
.ele {
    anchor-name: none | <dashed-ident>#
}
```

  


锚点名称必须是一个 `<dashed-ident>` ，即以两个连字符（`--`）开头的自定义名称，例如上面示例中的 `--anchor-name` 。看上去有点与我们熟悉的 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)（或变量）的命名相似。注意，这个名称可以是你任意喜欢的名称。例如：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}
```

  


这样，我们就创建了一个锚点元素。接下来，我们需要做是通过 `position-anchor` 属性将某个目标元素与锚点元素关联起来。

  


```CSS
.ele {
    position-anchor: auto | <anchor-element>
}
```

  


`position-anchor` 接受一个有效的 `<anchor-element>` 值。因此，如果我们已经定义了一个元素为“锚点元素”，就可以使用它将目标元素与其关联：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    position-anchor: --tooltip;
}
```

  


通常情况下，如果未找到有效的锚点元素，则其他锚点相关属性和功能会被忽略。还有一点需要知道的是，目标元素是一个具有绝对定位（`position: absolute`）的元素。

  


> 温馨提示，我亲自测试了一下，目标元素还可以是固定定位（`position: fixed`）的元素，但不能是相对定位（`position: relative`）和粘性定位（`position: sticky`）的元素。

  


现在，我们已经知道了如何建立锚点元素与目标元素的关系，接下来可以设置目标元素相对于锚点元素的定位。由两种方式可以实现：

  


-   `anchor()` 函数设置目标元素的偏移位置
-   `position-area` 属性设置目标元素的偏移位置

  


虽然这两种方式都可以，但其中 `position-area` 是 W3C 规范最新定义的方式，我们先来看 `position-area`！还有一点就是，新的模式中还新增了 `position-visibility` 属性，用于定义当目标元素空间不足时是否隐藏。

  


## 目标元素的定位

  


目标元素相对于锚点元素定位的最简单方法是使用 `position-area` 属性，它在锚点元素周围创建了一个虚拟的 `3 x 3` 网格，并允许我们将目标元素放置在网格的一个或多个区域中：

  


```CSS
.ele {
    position-area: auto | <position-area>
}
```

  


`position-area` 通过使用逻辑值（如 `start` 和 `end` ）或物理值（如 `top`、`right` 、 `bottom` 、`left`）

  


```
position-area: auto | <position-area>
```

该属性通过使用逻辑值（如 `start` 、 `end` 和 `center`）或物理值（如 `top`、`left`、`right`、`bottom` 和 `center`）来设置网格的行和列，然后将目标元素的 IMCB（内边距调整的包含块）缩小到所选网格区域。例如：

  


```CSS
.target {
    position-area: center end;
    
    /* 或 */
    position-area: center right;
}
```

  


逻辑值依赖于包含块的书写模式。如果我们檙 让目标元素相对于其自身的书写模式定位，可以在前面加上 `self` 值：

  


```CSS
.target {
    position-area: self-start self-end;
}
```

  


在任何轴上都可以使用 `center` 值：

  


```CSS
.target {
    position-area: center right;
    /* 或 */
    position-area: start center;
}
```

  


如果你想将目标元素跨越两个相邻的网格区域，可以在值前面加上前缀 `span-` （不能是 `centerr`），每次跨越一行或一列：

  


```CSS
.target {
    position-area: span-top left;
    /* 或 */
    position-area: span-start start;
}
```

  


如果想跨越三个相邻的网格区域，可以使用 `span-all` 值：

  


```CSS
.target {
    position-area: bottom span-all;
    /* 或 */
    position-area: end span-all;
}
```

需要知道的是，`position-area` 属性值对于物理值（比如 `top` 和 `left`）的顺序没有严格要求，比如：

  


```CSS
.target {
    position-area: top left;
    
    /* 等同于 */
    position-area: left top;
}
```

  


但对于逻辑值，顺序很重要。例如：

  


```CSS
.target {
    position-area: start end;
}

.target {
    position-area: end start;
}
```

  


这两者的结果完全相反。

  


不过，我们可以通过添加轴的前缀（如 `y-` 、`x-` 、`inline-x` 或 `block-x`）来使逻辑值互换：

  


```CSS
.target {
    position-area: inline-end block-start;
    
    /* 等同于 */
    position-area: y-start x-end;
}
```

我们回过头来看另一种方式，即 `anchor()` ，这种方式是 CSS 锚点定位最初提供的设置目标元素偏移位置的方式。相比于 `position-area` 方法，`anchor()` 函数可以被视为一个更简便的方法，它会将目标元素附加到锚点元素上——指定锚点、要附加的侧面以及目标元素的大小。然而，正如我们接下来会看到的，这个函数也可能实现将一个目标元素附加到多个锚点元素上的可能性。

  


`anchor()` 函数作为 `top` 、`right` 、`bottom` 和 `left` （以及它们对应的逻辑属性）的值，它最多可以接受三个参数：

  


```
anchor( <anchor-element>? && <anchor-side>, <length-percentage>? )
```

  


简而言之，我们需要：

  


-   确定锚点元素
-   指定目标元素的偏移位置
-   定义目标元素的大小

  


例如：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    top: anchor(--tooltip bottom);
    left: anchor(--tooltip end, 50%);
}
```

  


在这个例子中：

  


-   第一个 `anchor(--tooltip bottom)` 说明目标元素应该定位在 **`--tooltip`** 锚点的**底部**
-   第二个 `anchor(--tooltip end, 50%)` 说明目标元素应该定位在 **`--tooltip`**锚点的**结束**（右侧），并且目标元素的宽度为 `50%`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9b1c63c8f9b04929b12e2a79f4ff1c06~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1104&h=584&s=2329271&e=gif&f=224&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/EaYwzYM

  


接下来，我们来分解一下 `anchor()` 函数中的三个参数（`<anchor-element>` 、`<anchor-side>` 和 `<anchor-size>`），看看它们是如何工作的。

  


首先来看 `<anchor-element>` 参数。这个参数指定了我们要将目标元素附加到哪个锚点元素。我们可以通过锚点的名称来指定锚点元素。例如：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    top: anchor(--tooltip bottom);
    left: anchor(--tooltip end, 50%);
}
```

  


这种方式通常也被称为“**显式锚点**”。

  


我们也可以选择不提供锚点元素。在这种情况下，目标元素会使用在 `position-anchor` 中定义的**隐式锚点**元素。

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    position-anchor: --tooltip;
    top: anchor(bottom);
    left: anchor(end, 50%);
}
```

  


上面的代码等同于：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    top: anchor(--tooltip bottom);
    left: anchor(--tooltip end, 50%);
}
```

  


需要知道的是，如果没有隐式锚点，并且在 `anchor()` 没有显式指定 `<anchor-element>` 在数，那么 `anchor()` 函数会使用回退值。否则，它将被视为无效并被忽略。

  


第二个参数 `<anchor-side>` 设置了我们要将目标元素定位到锚点元素的哪一侧，例如锚点的顶部（`top`）、右侧（`right`）、底部（`bottom`）或左侧（`left`）。简单地说，它是用于设置目标元素相对于锚点元素的偏移位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4b8c7de04e947c598784b501bc04e42~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2434&h=1384&s=342849&e=png&b=ffffff)

  


`<anchor-side>` 参数的值非常灵活，它可以是关键词、长度值、百分比值，还可以使用其他更灵活的选择：

  


-   **关键词**：物理值（如 `top` 、`right` 、`bottom` 、 `left` 和 `center`），包括逻辑侧面关键词（`inside` 、`outside`）、相对于用户书写模式的逻辑方向参数（`start` 、`end` 、`self-start` 、`self-end`）以及 `center`
-   **长度值（** **`<length>`** **：** 解析为锚点元素对应的侧面的 `<length>` 。它可以包含物理参数（如 `top` 、`right` 、`bottom` 和 `left` ）、逻辑侧面参数（如 `inside` 和 `outside`）、相对于用户书写模式的逻辑方向参数（`start` 、`end` 、`self-start` 、`self-end`）以及 `center`
-   **百分比值（** **`<percentage>`** **）** ：指的是从起点（`0%`）到结束点（`100%`）之间的位置。允许的值可以小于 `0%` 或大于 `100%`
-   **`<length-percentage>`** ：它将目标元素定位到相对于其包含块的固定 `<length>` 或 `<percentage>`

  


`<anchor-side>` 设置不同的参数将会获得不同的结果：

  


-   **使用物理参数：** 物理参数（如 **`top`**、**`right`**、**`bottom`**、**`left`**）可以用来定位目标元素，而不考虑用户的书写模式。例如，我们可以将目标的右侧和底部内嵌属性定位到锚点的 `top` 和 `left` 侧，从而将目标有效地定位到锚点的左上角
-   **使用逻辑侧关键字：** 逻辑侧参数（例如 **`inside`**、**`outside`**）取决于它们所在的内嵌属性。**`inside`** 参数将选择与其内嵌属性相同的侧面，而 **`outside`** 参数将选择相反的侧面。例如：
-   **使用逻辑方向：** 逻辑方向参数取决于两个因素，其一是**用户的书写模式**，它们可以跟随包含块的书写模式（**`start`**、**`end`**）或目标元素本身的书写模式（**`self-start`**、**`self-end`**）；其次是**它们使用的内嵌属性**：它们会选择与其内嵌属性相同的轴
-   **使用百分比值：** 百分比可以用来将目标元素定位在起点（`0%`）和终点（`100%`）之间的任何位置。由于百分比是相对于用户书写模式的，因此最好与逻辑内嵌属性一起使用。允许使用小于 `0%` 或大于 `100%` 的值，因此 `-100%` 会将目标元素移动到起始位置，`200%` 会将其移动到结束位置

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b412796b21a34f86a414cc2c92ffbd3b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3861&h=3280&s=712271&e=png&a=1&b=985eb5)

  


注意，设置目标元素偏移位置的 `top` 、`right` 、`bottom` 和 `left` 属性，可以使用对应的逻辑属性替代：

  


-   `top` = `inset-block-start`
-   `left`= `inset-inline-start`
-   `bottom` = `inset-block-end`
-   `right`= `inset-inline-end`

  


有关于这方面更详细的介绍，请参阅 《[CSS 逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)》一课。但需要知道的是，这里所说的是 `top` 、`right` 、`bottom` 和 `left` 是 CSS 物理属性以及它们对应的逻辑属性，而 `anchor()` 函数中的 `top` 、`right` 、`bottom` 、`left` 以及 `center` 是参数，指的是锚点元素的侧边，与之对应的逻辑属性是 `start` 和 `end` 。这个对于部分初学者来说，容易混淆！

  


还有一点需要知道的是，设置目标元素偏移位置的属性（比如 `top` 、`right` 、`bottom` 和 `left` 以及它们对应的逻辑属性）的值可以是一个数学表达式，例如：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    --center: anchor(--tooltip 50%);
    left: var(--center);
    bottom: calc(var(--center) + 3lh);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fe02eba495ff4302a4e2fe27ff49b8d8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1276&h=766&s=942339&e=gif&f=118&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/bNboyvM

  


不过，`anchor()` 函数中的参数值 `<anchor-side>` 不支持数学表达式！

  


虽然 `anchor()` 函数的参数可以使目标元素相对于锚点元素精确定位，但有时简单性是关键——特别是当你需要目标元素相对于锚点元素居中时。庆幸的是，CSS 引入了一个名为 `anchor-center` 的值，这个值可以与 `justify-center` 、`align-self` 、`justify-items` 和 `align-items` 属性搭配使用。例如，要将目标元素定位在锚点元素的上方并水平居中，只需要使用 `anchor-center` 就可以：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    bottom: calc(anchor(--tooltip top) + 1rem);
    justify-self: anchor-center;
}
```

> Demo 地址：https://codepen.io/airen/full/VYZMOEp

  


`anchor()` 函数的第三个值 `<anchor-size>` 是一个备用值，它有点类似于 `var()` 函数中的备用值。简单地说，如果 `anchor()` 函数引用锚点不存在或无法正确解析，那么 `<anchor-size>` 就是作为目标元素的备用偏移值。例如：

  


```CSS
.anchor {
    anchor-name: --tooltip;
}

.target {
    position: absolute;
    position-anchor: --tolltip;

    left: anchor(--tooltip center, 10%);
    bottom: anchor(--y start, 200px);
}
```

  


上面代码中的 `--y` 锚点名称并没有显式定义，因此目标元素的 `bottom` 将会是 `200px` ，在我们这个示例中，它始终离浏览器视窗底部的间距是 `200px` ；而 `--tooltip` 这个锚点名称存在，因此目标元素的 `left` 是相对于锚点元素的中心位置定位，即目标元素的左侧边缘与锚点元素中心位置相吻合：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ccc14bbe43744cadbf68248fe5fa4ee1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1364&h=778&s=2298424&e=gif&f=217&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/VYZMORb

  


CSS 锚点定位除了 `anchor()` 函数之外，还提供了另一个有价值的函数——**`anchor-size()`** **函数**。注意，它是一个 CSS 函数，与 `anchor()` 函数中的 `<anchor-size>` 参数不是同一个东东。

  


`anchor-size()` 函数与 `anchor()` 函数类似，它也会返回一个长度值，但它在尺寸调整上起到了不同的作用，而不是定位。换句话说，`anchor()` 函数是关联到锚点元素的相对位置，而 `anchor-size()` 函数是关联到它的相对尺寸。

  


`anchor-size()` 函数的独特之处在于，它可以根据锚点元素的大小来调整目标元素的尺寸。这在响应式设计中非常有用，尤其是在元素可能会因容器溢出而发生移动、调整大小或被遮挡时，可以确保目标元素随着锚元素的大小同步缩放。

  


`anchor-size()` 函数会获取锚点元素的一侧，并解析为其长度值（`<length>`），本质上是返回锚点元素的宽度（`width`）、高度（`height`）、内联尺寸（`inline-size`）或块尺寸（`block-size`）。

  


```CSS
.ele {
    anchor-size( [ <anchor-element> || <anchor-size> ]? , <length-percentage>? )
}
```

  


注意，`<length-percentage>` 参数可用作回退值。当目标没有有效的锚点或尺寸时，返回一个固定的 `<length>` 或 `<percentage>`，其值相对于包含块。

  


简单地说，我们可以在目标元素的 `width` （或 `inline-size`）和 `height` （或 `block-size`）属性上使用 `anchor-size()` 函数，使其大小与锚点元素同步，甚至可以同时调整宽度和高度。例如：

  


```CSS
.anchor {
    anchor-name: --anchor-name;
}

.target {
    position: absolute;
    position-anchor: --anchor-name;
    width: anchor-size(width, 20%); /* 使用默认锚点 --anchor-name * /     height: anchor-size(--other-anchor inline-size, 100px); /* 使用另一个锚点 --other-anchor */
}
```

  


此外，由于 `anchor-size()` 返回一个有效的长度值，它可以与 `calc()` 函数结合使用来建立比例关系。例如，要将定位元素的高度缩放为其锚点元素高度的 `1.5` 倍，可以使用以下 CSS：

  


```CSS
.target {
    height: calc(anchor-size(height) * 1.5);
}
```

  


例如 [@Jhey 在 CodePen 提供的这个案例](https://codepen.io/jh3y/full/GRbbprj)，Popover 的尺寸（`width` 和 `height`）就是根据锚点元素 `nav` 的 `width` 和 `height` 来计算的：

  


```CSS
.nav {
    anchor-name: --nav;
}

[popover] {
    position-anchor: --nav;
    inset: unset;
    bottom: anchor(bottom);
    left: anchor(left);
    width: anchor-size(width);
    height: anchor-size(height);
    border-radius: 21px;

    @starting-style {
        width: min(var(--width), 100vw - 2rem);
        height: min(var(--height), 80vh);
    }
}

[popover]:popover-open {
    width: min(var(--width), 100vw - 2rem);
    height: min(var(--height), 80vh);
    border-radius: 21px;

    @starting-style {
        width: anchor-size(width);
        height: anchor-size(height);
    }
}
```

到目前为止，大家应该知道如何通过 `anchor()` 和 `position-area` 设置目标元素的精确位置。但是，在某些情况下，你可能需要隐藏目标元素。例如，你可能希望将某个元素锚定在页面的某个滚动容器中。在这种情况下，可以使用 `position-visibility` 属性帮助你设置隐藏这些元素的规则。

  


```CSS
.ele {
    position-visibility: always | anchors-visible | no-overflow;
}
```

  


-   **`always`** **：** 目标元素始终显示，无论其锚点的状态或是否溢出
-   **`no-overflow`** **：** 如果在应用位置回退之后，目标元素仍然溢出其包含块，则目标元素将被强制隐藏
-   **`anchors-visible`** **：** 如果锚点（不是目标元素）完全溢出其包含块，或者完全被其他元素覆盖，则目标元素将被强制隐藏

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9bcce9c9e824336be99bea8b244439b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=49214&e=webp&a=1&b=f59e2d)

  


我们通过下面这个示例来加强对它的理解。当 `position-visibility` 选择 `no-overflow` 时，只要工具提示的任何部分越过视口边缘，它就会被隐藏；选择 `anchors-visible` 时，定位元素保持可见，直到其锚点元素完全离开视口。

  


```CSS
.message {
    position: absolute;
    position-area: top center;
    position-anchor: --anchor;
    position-visibility: no-overflow;
}
```

## 给目标元素设置备用位置

  


一旦目标元素相对于锚点定位完成，我们可以为目标元素提供额外的指令，告知其在空间不足时该如何处理。我们已经讨论过 `position-visibility` 属性作为一种解决方案——我们可以简单地让目标元素隐藏。然而，CSS 锚点定位为我们提供了更多控制选项，允许目标元素通过尝试锚点的其他侧面进行重新定位（`position-try-fallbacks`），以及决定它重新定位时尝试的顺序（`position-try-order`）。这两个属性可以通过 `position-try` 属性来简写。

  


`position-try-fallbacks` 属性接受一个以逗号分隔的定位回退列表，当目标元素溢出其包含块的空间时，会尝试每个回退值，直到找到合适的位置或用尽所有选项：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/442a7b54bc524b70bcfabe09a6a34fa4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=87556&e=webp&a=1&b=995eb6)

  


```CSS
.ele {
    position-try-fallbacks: none | [ [<dashed-ident> || <try-tactic>] | <'inset-area'>  ]#
}
```

  


-   `none`: 保持目标的定位选项列表为空
-   `<dashed-ident>`: 将一个自定义的 `@position-try` 回退添加到选项列表中，使用给定的名称。如果没有匹配的 `@position-try`，则忽略该值
-   `<try-tactic>`: 通过翻转目标当前的位置，在三个轴之一上创建一个选项列表，每个轴由一个不同的关键字定义。它们也可以组合在一起，叠加它们的效果。`flip-block` 关键字在块轴上交换值；`flip-inline` 关键字在内联轴上交换值；`flip-start` 关键字在对角线方向交换值
-   `<dashed-ident> || <try-tactic>`: 结合一个自定义的 `@try-option` 和一个 `<try-tactic>`，创建一个单一的定位回退。`<try-tactic>` 关键字也可以组合，以叠加它们的效果
-   `<"position-area">`: 使用 `position-area` 语法将锚点移动到一个新位置

  


示例代码：

  


```CSS
.target {
    position-try-fallbacks:
        --my-custom-position,
        --my-custom-position flip-inline,
        bottom left;
}
```

  


`position-try-order` 此属性根据在 `position-try-fallbacks` 属性中定义的回退值，选择一个新的位置，基于哪个位置能给目标提供最多的空间。其余的选项会重新排序，最宽或最高的可用空间排在最前面。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ddbd23d77ac84b4e91b8cc35342b37ab~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=29962&e=webp&a=1&b=995eb6)

  


```CSS
.ele {
    position-try-order: normal | most-width | most-height | most-block-size | most-inline-size
}
```

  


对于每个定位回退，它会查找目标的 IMCB（包含块的尺寸）。然后，它会选择给 IMCB 提供最宽或最高尺寸的值，具体取决于选择的选项：

  


-   `most-width`
-   `most-height`
-   `most-block-size`
-   `most-inline-size`

  


```CSS
.target {
  position-try-fallbacks: --custom-position, flip-start;
  position-try-order: most-width;
}
```

  


`position-try` 这是一个简写属性，它将 `position-try-fallbacks` 和 `position-try-order` 属性合并为一个声明。它首先接受顺序，然后是可能的定位回退列表。因此，我们可以将这两个属性合并成一个样式规则：

  


```CSS
.target {
    position-try: most-width --my-custom-position, flip-inline, bottom left;
}
```

  


## 自定义位置和大小回退

`@position-try` 这个 `@` 规则 为 `position-try-fallbacks` 属性定义了一个自定义的定位回退。

  


```CSS
@position-try <dashed-ident> {
    <declaration-list>
}
```

  


它接受多种属性来改变目标元素的位置和大小，并将它们作为新的定位回退供元素尝试。

  


假设你已经建立了锚点与目标元素之间的关系。你希望将目标元素定位到锚点的右上角，这可以使用之前看到的 `position-area` 属性轻松实现：

  


```CSS
.target {
    position: absolute;
    position-area: top right;
    width: 100px;
}
```

  


可以看到 `.target` 的宽度被设定为 `100px`？也许在某些屏幕上空间不足，它无法再显示在锚点的右上角。我们可以为 `.target` 提供之前讨论的回退值，让它尝试在更有空间的边缘重新定位：

  


```CSS
.target {
    position: absolute;
    position-area: top right;
    position-try-fallbacks: top left;
    position-try-order: most-width;
    width: 100px;
}
```

  


由于我们是追求简洁代码的优秀 CSS 开发者，我们也可以将这两个属性合并为 `position-try` 简写属性：

  


```CSS
.target {
    position: absolute;
    position-area: top right;
    position-try: most-width, flip-inline, bottom left;
    width: 100px;
}
```

  


到目前为止，一切顺利。我们有一个锚点元素，它从锚点的右上角开始，宽度为 `100px`。如果在这里空间不足，它会查看 `position-try` 属性，决定是否将目标重新定位到锚点的左上角（由 `flip-inline` 声明）或锚点的左下角——以最宽的空间为准。

  


但如果我们想在重新定位时同时调整目标元素的大小呢？也许目标元素在任何回退位置下，`100px` 的大小都太大，必须调整为 `50px`。我们可以使用 `@position-try` 来实现这一点：

  


```CSS
@position-try --my-custom-position {
    position-area: top left;
    width: 50px;
}
```

  


完成后，我们现在有了一个名为 `--my-custom-position` 的自定义属性，可以在 `position-try` 简写属性中使用。在这种情况下，`@position-try` 可以替换掉 `flip-inline`，因为它相当于 `top left`：

  


```CSS
@position-try --my-custom-position {
    position-area: top left;
    width: 50px;
}

.target {
    position: absolute;
    position-area: top right;
    position-try: most-width, --my-custom-position, bottom left;
    width: 100px;
}
```

  


这样，当 `.target` 元素尝试重新定位到锚点的右上角时，它的宽度会从 `100px` 调整为 `50px`。这提供了一个不错的灵活性，让我们在任何布局中都有更好的机会让元素适配。

  


## 多个锚点

  


元素可以锚定到多个锚点。这意味着，你可能需要设置相对于多个锚点的位置值。为此，请使用 `anchor()` 函数，并在第一个参数中明确说明你引用的是哪个锚点。在以下示例中，定位元素的左上角锚定到一个锚点的右下角，而定位元素的右下角锚定到第二个锚点的左上角：

  


```CSS
.target {  
    position: absolute;  
    top: anchor(--one bottom);  
    left: anchor(--one right);  
    right: anchor(--two left);  
    bottom: anchor(--two top);
}
```
