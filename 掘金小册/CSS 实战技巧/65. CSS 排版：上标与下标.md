要是你正在开发一个教学类的 Web 网站或应用，应该会有这样的一个需求——将数学表达式和化学公式完美的 Web 页面上呈现。例如：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/37964591f0894677a85ec927373d11d8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3198&h=1524&s=512500&e=jpg&b=ffffff)

  


面对这类需求，我们需要使用到**上标**和**下标**，因为它们是学术和学科内容中的重要元素。你可能会使用带有语义化的 HTML 标签 `<sup>` （上标）和 `<sub>` （下标），你也可能会使用非语义化的 HTML 标签，例如 `<span>` 、`<div>` 等。不管使用哪种标签，浏览器对这些元素的处理方式过于静态，这可能导致数学表达式和化学公式在 Web 上不能完美的呈现，比如在移动端设备上，上标和下标显得过小，而在桌面端上则可能显得过大。

  


这节课，我们来尝试使用一种更现代的方式来设置上标和下标的样式。我将向你展示为什么传统的静态处理方法难以满足需求，以及如何为所有视口提供更优雅的排版体验。最棒的是，这个解决方案是纯 CSS 的，不需要依赖任何 JavaScript 脚本或第三方库。

  


## 传统方式存在的缺陷

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a0e5755317ce4478a2e17234106bb236~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=121060&e=jpg&b=f5f5f5)

  


上图同时展示了上标和下标，其中第一个 `2` 是上标，第二个 `2` 是一个下标。我们使用 HTML 的原生标签 `<sup>` 和 `<sub>` 来创建它们：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6aa09401bced4b268fdc07addce0f9fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=170206&e=jpg&b=f5f5f5)

  


```HTML
<h1>
    Hlop<sup>2</sup>x<sub>2</sub>
</h1>
```

  


默认情况之下，`<sub>` 和 `<sup>` 元素的 `font-size` 属性的值为 `smaller` ，与其父元素的 `font-size` 相比，大约会缩小 `0.83` 倍。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6ceb67f01c5a450d9be1451a31609d10~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3176&h=1402&s=419264&e=jpg&b=ecf7f8)

  


这是传统的静态处理方式，比较适用于简单的文档，但在现代响应式 Web 设计中可能引发问题，尤其是在使用流体排版时，字体大小可以在极端（最小值和最大值）之间平滑缩放：

  


```CSS
h1 {
    font-size: clamp(1.5rem, 4vw + 1rem, 5rem);
    font-family: "Latin Modern Math", "STIX Two Math", "Cambria Math", serif;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/24ae8ed19d8f4234ab4ac5c76fb587e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1642&h=678&s=7151311&e=gif&f=254&b=121204)

  


温馨提示，使用 [CSS 的 `clamp()` 函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)设置文本的 `font-size` ，可以获得一个流体排版的体验，它的大小会根据浏览器视窗的宽度进行调整，并且会在 `MIN ~ MAX` 之间取得一个最佳值。

  


## 流体缩放：更优的解决方案

  


现在，我们可以结合固定单位（比如 `px`）和相对单位（比如 `vw` 或 `cqi`）为上标和下标提供一种能够更自然地适应不同文本大小的方案。这种方案在小尺寸（比如移动端）时保证可读性，同时在大尺寸（比如桌面端）时保持适当比例，避免了针对特定上下文的调整需求。

  


先直接上代码：

  


```CSS
:is(sup, sub) {
    font-size: calc(.5em + 4px);
    vertical-align: baseline;
    position: relative;
    top: calc(-.5 * .83 * 2 * (1em - 4px)); /* 即： calc(-.83em + 3.32px) */
}

sub {
    top: calc(.25 * .83 * 2 * (1em - 4px)); /* 即： clac(.42em - 1.66px) */
}

h1 {
    font-size: clamp(1.5rem, 4vw + 1rem, 5rem);
    font-family: "Latin Modern Math", "STIX Two Math", "Cambria Math", serif;
}
```

  


你将看到的效果如下：

接下来，我们一起来看看它的工作原理。

  


首先，将 `sub` 和 `sup` 的 `vertical-align` 属性的值重置为 `baseline` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ad4bdf1240c4bd5a7afb53e7c0b1d1e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1226&s=509077&e=jpg&b=f3f3f3)

并使用相对定位（`position: relative`）来设置 `sup` 和 `sub` 的位置，这样做可以避免这些元素影响行高，同时使我们能更好控制它们的偏移量，以满足具体需求。

  


你可能会好奇这些“神奇”的数字是哪来的，对吧！接下来，我们就来解开这些数字中的秘密。

  


### 计算字体大小（像素）

  


首先将 `sub` 和 `sup` 的 `font-size` 设置为 `calc(.5em + 4px)` ：

  


```CSS
h1 {
    font-size: clamp(1.5rem, 4vw + 1rem, 5rem);
}

:is(sub, sup) {
    font-size: calc(.5em + 4px);
}    
```

  


代码中的 `.5em` 和 `4px` 结合使用，确保了 `sup` 和 `sub` 的大小能与字体保持比例协调，又能在不同大小的字体中具有良好的可读性。

  


首先，解释一下 `em` 单位，它是一种相对单位，它相对于父元素的字体大小（`font-size`）计算。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2e7a98d09593487896bebff5b4e744ef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3091&h=1441&s=1045179&e=png&b=edecec)

  


之前提到过，在浏览器的默认行为中，`sup` 和 `sub` 的字体大小会按 `.83` 倍率（即 `font-size: smaller` 的效果）相对于父元素的 `font-size` 自动缩小。而 `0.5em` 相当于我们把倍率调整到 `0.5` 。选择 `0.5` 是主要原因是，许多常见字体的上标、下标（如 Unicode 上标字符）的大小比例接近 `0.5` 。换句话说，`sub` 和 `sup` 的字体大小通常被设计为父元素字体大小的 `50%` 左右，这个比例符合大多数排版规则和用户习惯。

  


当然，你也可以根据具体的需求场景来调整这个比例，比如 `.6em` 更大比例，适合强调内容；`0.4em` 更小的比例，适合轻量标记。

  


公式中的 `4px` 是一个固定值，用来解决单纯使用相对单位（比如 `.5em`）时在极小字号下可能出现的可读性问题。因为，仅使用 `.5em` 在小屏幕或小字号中会显得过小，不易于阅读，比如父元素的 `font-size` 为 `12px` 时，`.5em` 对应的值就是 `6px` （即 `12px × 0.5 = 6px`）。加上 `4px` 后，它就会变成 `10px` ，变得更加清晰。反之，在大字号中，`4px` 对结果的影响逐渐变小，例如父元素的 `font-size` 为 `48px` 时，`.5em` 等于 `24px` ，加上固定值 `4px` 之后是 `28px` ，比例仍然自然。

  


在我们这个示例中，父元素的 `font-size` 是一个在 `1.5rem ~ 5rem` 之间的值，文本会有一个流体缩放的效果：

  


-   在小屏幕时，对应的最小值是 `1.5rem` （默认情况下是 `16px × 1.5 = 24px`） ，`sub` 和 `sup` 的 `font-size` 则为 `24px × 0.5 + 4px = 16px` ，其中 `4px` 的固定值占主要作用，避免字体过小
-   在大屏幕时，对应的最大值是 `5rem` （默认情况下是 `16px × 5 = 80px`），`sub` 和 `sup` 的 `font-size` 则为 `80px × 0.5 + 4px = 84px` ，其中 `.5em` 成为主导，`4px` 仅作为微调

  


这在现代 Web 排版中显得尤其有用，因为现代响应式 Web 设计中，文本的字体大小（`font-size`）经常动态调整（例如 `clamp(1.5rem, 4vw + 1rem, 5rem)`），单纯使用 `em` 会导致字体比例失调。

  


虽然 `0.5em` 和 `4px` 是一个通用的设置，但可以根据具体需求调整：

  


```CSS
:is(sup, sub) {
    font-size: calc(0.6em + 3px); /* 更大比例，适合强调内容 */
}

:is(sup,sub) {
    font-size: calc(0.4em + 5px); /* 更小比例，适合轻量标记 */
}
```

  


甚至还可以结合 `min()` 和 `vw` 根据视常动态调整这个值，例如：

  


```CSS
:is(sup, sub) {
    font-size: calc(0.5em + min(4px, 0.25vw)); /* 根据视口动态调整 */
}
```

  


### 计算垂直偏移

  


刚才提到过了，我们使用相对定位替代了 `sub` 和 `sup` 的默认偏移。默认情况下，`sub` 和 `sup` 是通过 `vertical-align` 来设置的：

  


-   `sub` 的 `vertical-align` 值是 `sub`
-   `sup` 的 `vertical-align` 值是 `super`

  


现在统一将它们的 `vertical-align` 设置为 `baseline` （纵轴根据字体基线对齐），并通过 `top` 来设置它们在纵轴的偏移位置，这样能更好的控制它们的位置：

  


```CSS
sup {
    top: calc(-.5 * .83 * 2 * (1em - 4px)); /* 即： calc(-.83em + 3.32px) */
}

sub {
    top: calc(.25 * .83 * 2 * (1em - 4px)); /* 即： clac(.42em - 1.66px) */
}
```

  


`top` 的计算要复杂得多。它用到“魔法”数字也更多。接下来，我们来看看这些数字又是如何得来的。

  


从代码中，我们可以知道 `top` 的值使用了 `em` 单位，它直接增加了计算的复杂性。因为，`sub` 和 `sup` 的 `font-size` 使用 `em` 单位时，它是相对于父元素的 `font-size` 计算；而 `sub` 和 `sup` 的 `top` 属性（除 `font-size` 之外的其他属性）的值使用 `em` 单位时，它是相对于元素自身 `font-size` 计算，如果元素未显式设置 `font-size` ，则会相对于父元素的 `font-size` 计算（因为 `font-size` 是一个可继承属性）。

  


换句话说，在我们这个示例中，`sub` 和 `sup` 的 `font-size` 是相对于父元素的 `font-size` 计算，其计算公式是 `calc(0.5em + 4px)` ；它们的 `top` 属性则是相对于元素自身的 `font-size` 计算。然而，父元素的字体大小是流式的，为了使 `top` 的偏移量能与父元素的字体大小保持一致，我们就不能直接相对 `sub` 和 `sup` 元素自身的 `font-size` 计算，则需要相对于父元素的 `font-size` 计算。

  


这意味着，我们首先要补偿 `.5em` 的缩放比例。我们可以使用下面这个公式来计算：

  


```
.5em × Y = 1em 
```

  


计算得出 `Y` 等于 `2` 。这个时候 `top` 的值为：

  


```CSS
:is(sup, sub) {
    font-size: calc(0.5em + 4px);
    top: calc(2 * (1em - 4px));
}
```

  


公式中的 `1em` 表示 `sub` 和 `sup` 元素自身的 `font-size` 。在进行乘法之前，我们需要从当前的 `em` 值中减去固定的 `4px` 成分（或加上，视情况而定）。

  


现在，我们知道 `2 * (1em -4px)` 是怎么一回事了。但公式中还有 `.83` 、 `-0.5` 和 `.25` 几个数字：

  


```CSS
:is(sup, sub) {
    font-size: calc(0.5em + 4px);
    top: calc(-0.5 * 0.83 * 2 * (1em - 4px));
}

sub {
    top: calc(0.25 * 0.83 * 2 * (1em - 4px));
}
```

  


其中 `.5` （上标）和`-.25` （下标）是默认的垂直偏移值，这是一个经验值，在诸如 Tailwind CSS 和 Bootstrap 等 CSS 框架中都是使用的这个值。`.83` 则是为了考虑浏览器默认的 `font-size: smaller` 缩放因子，这是上标和下标的默认行为。

  


这种方法可以让我们的上标和下标在保留浏览器默认垂直位置的同时，受益于更流畅的缩放效果。结果既符合用户对传统浏览器渲染的预期，又在不同字体大小下实现了更自然的缩放。

  


这就获得了设置上标和下标的所有 CSS 代码：

  


```CSS
:is(sup, sub) {
    font-size: calc(0.5em + 4px);
    vertical-align: baseline;
    position: relative;
    top: calc(-0.5 * 0.83 * 2 * (1em - 4px)); /* 即： calc(-.83em + 3.32px) */
}

sub {
    top: calc(0.25 * 0.83 * 2 * (1em - 4px)); /* 即： clac(.42em - 1.66px) */
}
```
