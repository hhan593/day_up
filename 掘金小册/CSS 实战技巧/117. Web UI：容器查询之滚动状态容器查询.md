换句话说，现在可以使用纯 CSS 来管理 UI 状态，比如粘附、吸附以及溢出等滚动状态，这些都可以通过 CSS 进行查询和适配。

## 滚动状态容器查询概述

  


在此之前，CSS 的容器查询已经有了尺寸容器查询、样式容器查询和状态容器查询，现在新增了滚动状态容器查询。它与之前的容器查询类似，但它主要用于检测元素是否**粘附**、**吸附**或**溢出**等滚动状态。

  


在引入滚动状态查询之前，若要判断一个元素是否处于“粘附”、“吸附”或“可滚动”的状态，通常需要借助 JavaScript 来实现。现在，我们可以使用纯 CSS 来实现——通过滚动状态查询，可以进行查询和适配。此外，这项功能还为 CSS 带来了新的动画触发机制，使基于滚动触发的动画得以实现。

  


> 温馨提示，CSS **[滚动驱动动画](https://juejin.cn/book/7223230325122400288/section/7259272255786450981)**与**滚动状态容器查询触发的动画**有所不同，它们的工作机制也有差异，但它们之间仍然有很多待探索的空间。但这方面的讨论已经超出这节课的范畴，我们在此不进行深入探讨。

  


到目前为止，滚动状态查询主要有以下三种：

  


-   **`scrollable`** **（可滚动状态）** ：当元素发生溢出并变得可滚动时触发样式变化。例如，当滚动条位于顶部时，你可以显示一个提示，鼓励用户向下滚动查看更多的内容，而当用户开始滚动时则隐藏该提示
-   **`snapped`** **（吸附状态）** ：当元素在某个轴线上被吸附时触发样式变化。例如，你可能希望以某种方式高亮显示被捕捉的元素，或者显示其之前隐藏的内容
-   **`stuck`** **（粘附状态）** ：当元素被粘附在某个边缘时触发样式变化。例如，你可以给粘性元素（`position:sticky`）设置不同的配色方案或布局

  


## 滚动状态查询的基本语法

  


好消息是：你从容器查询中学到的一切知识，同样适用于这些新的滚动状态查询。因为滚动状态查询的实现与尺寸、样式和状态容器查询非常相似，基本上复用了 `container-type` 属性。因此，要检查滚动状态，我们需要设置该容器类型以便查询容器内部的子元素。

  


```HTML
<div class="container">
    <div class="query"></div>
</div>
```

  


换句话说，为了将容器元素 `.container` 设置为滚动状态查询容器，需要在其上设置 `container-type` 属性的值为 `scroll-state` 。你还可以选择为它提供一个 `container-name` ，以便使用特定的容器查询进行定位：

  


```CSS
.container {
    container-type: scroll-state;
    container-name: my-container;
}
```

  


然后，你可以创建一个 `@container` 块，指定查询、通过测试通过后应用到容器的子元素的规则，和可选的容器名称。如果不指定 `container-name`，容器查询将应用到页面上所有的滚动状态查询容器。

  


```CSS
.container {
    container-type: scroll-state;
    container-name: my-container;
  
    .query {
        @container <container-name> scroll-state(<scroll-state-type> : <scroll-state>) {
            /* 根据 .container 的状态做相应的操作 */
        }
    }
}
```

  


你还可以使用 [`@supports` 规则](https://juejin.cn/book/7223230325122400288/section/7258822669154877480)和[嵌套语法](https://juejin.cn/book/7223230325122400288/section/7259668111974989882)，只需要添加几行代码，就能实现渐进增强或按条件使用滚动状态查询：

  


```CSS
.container {
    /* CSS... */
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;
        container-name: my-container;
        
        .query {
            @container <container-name> scroll-state(<scroll-state-type> : <scroll-state>) {
                /* 根据 .container 的状态做相应的操作 */
            }
        }
    }
}
```

  


此外，如果你使用 `sscroll-state` 查询来触发页面上的动画，请记得使用[媒体查询的 `prefers-reduced-motion` 来包裹你的动画逻辑](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)，以尊重用户的“减少动画”偏好：

  


```CSS
.container {
    /* CSS... */
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;
        container-name: my-container;
        
        .query {
            @container <container-name> scroll-state(<scroll-state-type> : <scroll-state>) {
                /* 根据 .container 的状态做相应的操作 */
                
                @media (prefers-reduced-motion: no-preference) {
                    /* CSS 动画 */    
                }
            }
        }
    }
}
```

  


好了，以上就是滚动状态查询的基本用法。接下来，我们来创建第一个滚动状态查询。

  


假设我们有一个像下面这样的 HTML 结构：

  


```HTML
<div class="topbanner">This is just a top banner for the fun of it.</div>
<div class="stuck-top"><!-- container -->
    <div class="heading">Sticky header stuffs</div><!-- query -->
</div>
<div class="content"><!-- Other Content --></div>
```

  


我们需要将 `.stuck-top` 粘附在滚动容器的顶部，并且调整其子元素 `.heading` 的样式。

  


第一步，我们首先要定义滚动状态查询容器，这与定义其他查询容器一样，你希望查询哪个元素，就给它设置 `container-type` ，并可选地指定 `container-name` 。而对于 `scroll-state` 查询，你需要给发生粘附（`stuck`）、吸附（`snapped`）或溢出（`scrollable`）的元素设置 `container-type` 属性的值为 `scroll-state` ：

  


```CSS
.stuck-top {
    width: 100%;
    
    /* 粘附在顶部 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;  /* 定义滚动状态查询容器 */
    }
}  
```

  


第二步是选择这个容器的子元素，这个子元素会响应滚动状态。与其他类型的容器查询一样，这个不能设置了 `container-type` 的那个元素本身，必须是它的子元素。在我们这个示例中，滚动状态查询容器（`.stuck-top`）的子元素是 `.heading` 。我们在这里调整它的背景颜色——创建一个带有磨砂玻璃的背景效果：

  


```CSS
.stuck-top {
    width: 100%;
    
    /* 粘附在顶部 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
        
        .heading {
            transition: all 0.5s ease-out;
            background: #0081a7; /* 初始背景 */

            /* 元素粘附在顶部时，调整背景效果 */
            @container scroll-state(stuck: top) {
                background: color-mix(in oklch, #0081a7, transparent 70%);
                backdrop-filter: blur(8px);
            }
        }
    }
}
```

  


同样的，我们通过 `@container` 代码块来调整 `.heading` 元素的样式。为了将滚动状态查询与其他容器查询分开，滚动状态描述符和值放在 `scroll-state()` 的括号内，比如 `scroll-state(stuck: top)` 。`scroll-state()` 有点类似于[样式容器查询的 `style()`](https://juejin.cn/book/7223230325122400288/section/7259316003635462201#heading-1) 和[状态容器查询的 `state()`](https://juejin.cn/book/7223230325122400288/section/7259316003635462201#heading-13) 。请注意，滚动状态容器查询与状态容器查询是完全不同的容器查询，请不要混淆！

## 滚动状态容器查询的使用场景

  


之前我们提到过，滚动状态容器查询主要有三种不同的类型：粘附（`stuck`）、吸附（`snapped`）和可滚动（`scrollable`），它们可用于不同的场景中。接下来，我们通过实际的案例来向大家展示它们可以使用在哪些场景中，以及应该如何使用？

  


### 粘附：`stuck`

  


粘附我们又可以把它称为**滚动定位的状态查询**。我们平常在进行 Web 开发的过程中，时常会碰到粘性元素（`position: sticky`）元素是否被固定。比如，我们有一个导航栏，它应该保持粘性（`position: sticky`），并且只有它在被固定时才会显示阴影或带有磨砂玻璃的背景效果。以往要实现类似的效果，我们需要借助 JavaScript，或者我们直接在默认情况下就加上阴影或磨砂玻璃的背景。但通过滚动状态容器查询，这一切都变得简单了，我们只需要使用 CSS 就可以实现。

  


我们可以使用以下查询语法来实现粘附效果:

  


```CSS
.container {
    /* CSS... */
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;
        container-name: my-container;
        
        .query {
            @container my-container scroll-state(stuck : <scroll-state>) {
                /* 根据 .container 的状态做相应的操作 */
                
                @media (prefers-reduced-motion: no-preference) {
                    /* CSS 动画 */    
                }
            }
        }
    }
}       
```

  


其中 `<scroll-state>` 的值包括：

  


-   **`none`** ：粘性容器在任何方向上都没有被移动
-   **`top`** ：粘性容器被移动以保持在上边缘内
-   **`right`** ：粘性容器被移动以保持在右边缘内
-   **`bottom`**：粘性容器被移动以保持在下边缘内
-   **`left`** ：粘性容器被移动以保持在左边缘内
-   **`block-start`**：粘性容器被移动以保持在块级起始边缘内
-   **`inline-start`**：粘性容器被移动以保持在内联起始边缘内
-   **`block-end`** ：粘性容器被移动以保持在块级结束边缘内
-   **`inline-end`**：粘性容器被移动以保持在内联结束边缘内

  


注意，其中 `block-start` 、`inline-start` 、`block-end` 和 `inline-end` 是 `top` 、`left` 、`bottom` 和 `right` 对应的 **[逻辑属性值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)**。逻辑边缘根据查询容器的方向和书写模式映射到物理边缘。

  


需要知道的是，粘附特性将会查询一个粘性定位的容器是否在给定的边缘范围内，视觉上被移动以保持在粘性视图内。这意味着，查询容器需要设置 `position: sticky` ，比如：

  


```CSS
.stuck-top {
    position: sticky;
    top: 0;
}
```

  


如果查询容器不是粘性定位的，则没有任何值匹配。另外，它不能在同一轴的两个方向进行查询：

  


```CSS
/* 可能匹配 */
@container scroll-state((stuck: top) and (stuck: left)) {
    /* CSS... */
}

/* 永远不会匹配 */
@container scroll-state((stuck: left) and (stuck: right)) {
    /* CSS... */ 
}
```

  


简单而言之，滚动状态的 `stuck` 查询用于测试一个 `position:sticky` 的容器是否被固定在其滚动容器祖先的某个边缘。如果没有，查询返回 `false` ；如果查询返回 `true` ，`@container` 块中的规则将应用到匹配的 `position: sticky` 容器的后代元素。

  


接下来，我们通过不同的案例来展示粘附特性的使用场景。

  


#### 粘附时添加阴影

  


最常见的粘附（`stuck`）查询用例之一是导航栏：当导航栏粘附（`stuck`）时添加 `box-shadow` ，让它看起来像悬浮在页面内容之上。这是一个非常实用的视觉增强手法，使粘附元素在滚动时更具层次感。

  


让我们先设置基础的 HTML 结构：

  


```HTML
<div class="topbanner">
    <!-- 顶部横幅，非粘性 -->
</div>
<div class="stuck-top"><!-- 查询容器 -->
    <!-- stuck-top 将是粘性的 -->
    <div class="heading"><!-- 样式响应元素 -->
        <h1>Logo</h1>
        <nav><!-- 导航项 --></nav>
    </div>
</div>
<section class="content">
    <!-- 包含 main 和 aside 的大量内容 -->
</section>
```

  


接下来设置 `container-type` 和粘性定位：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
    }
}
```

  


为了让效果更加平滑，我们还需要给 `.heading` 添加一个过渡效果，用来平滑显示阴影：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
        
        .heading {
            transition: box-shadow 0.5s ease-out;
        }
    }
}
```

  


现在我们的 `container-type` 已经设置好了，剩下的就是创建状态查询，并在 `.stuck-top` 被固定时给 `.heading` 添加阴影：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
        
        .heading {
            transition: box-shadow 0.5s ease-out;
            
            @container scroll-state(stuck: top) {
                box-shadow: 
                    rgb(0 0 0 / 0.6) 0px 12px 28px 0px,
                    rgb(0 0 0 / 0.1) 0px 2px 4px 0px,
                    rgb(255 255 255 / 0.05) 0px 0px 0px 1px inset;
            }
        }
    }
}
```

#### 激活当前粘附的标题

  


另一个常见的粘性 UI 反馈场景是**高亮当前被粘附的元素**。比如在一个按字母排序的乐队列表中，当某个字母分组标题被粘附到顶部时将其高亮，能极大提升用户的浏览体验和方向感。

  


我们需要一个像下面这样的 HTML 结构：

  


```HTML
<div class="sticky-slide">
    <dl>
        <dt><!-- container -->
            <header>A</header><!-- query -->
        </dt>
        <dd>Aceyalone</dd>
        <!-- 省略其他 dd -->
    </dl>
    <!-- 省略其他 dl -->
</div>
```

  


在这个示例中，我们需要对每个 `dl` 中的 `dt` 进行粘性定位，同时将它设置为滚动状态查询容器：

  


```CSS
.sticky-slide {
    dt {
        /* 粘性定位 */  
        position: sticky;
        inset-block-start: 0;
        inset-inline: 0;

        @supports (container-type: scroll-state) {
            container-type: scroll-state; /* 定义滚动状态查询容器 */
        }
}
```

  


接着使用 `@container` 对其子元素 `header` 设置样式。即 `dt` 粘附在顶部时，其子元素 `header` 会有一个高亮的样式，即调整它的背景颜色并添加阴影效果：

  


```CSS
.sticky-slide {
    dt {
        /* 粘性定位 */ 
        position: sticky;
        inset-block-start: 0;
        inset-inline: 0;

        @supports (container-type: scroll-state) {
            container-type: scroll-state; /* 定义滚动状态查询容器 */
    
            > header {
                /* 设置平滑过渡效果 */
                transition: background 0.3s ease, box-shadow 0.5s ease;
    
                /* dt 粘附在顶部时，高亮 header*/
                @container scroll-state(stuck: top) {
                    background: hsl(265 100% 27%);
                    box-shadow: 0 5px 5px #0003;
                }
          }
      }
    }
}
```

### 吸附：`snapped`

  


吸附（`snapped`）查询又常称为“**滚动捕捉的状态查询**”。当实现了滚动捕捉时，滚动状态的 `snapped` 查询用于测试容器是否已经或即将沿着给定轴与滚动捕捉容器祖先对齐。也就是说，它会匹配任何触发 `scrollsnapchanging` 事件的对齐目标。如果没有，则返回 `false` ，如果返回 `true` ，`@container` 块中的规则将应用到匹配的滚动捕捉目标容器的后代元素。

  


我们可以使用以下查询语法来实现吸附效果:

  


```CSS
.container {
    /* CSS... */
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;
        container-name: my-container;
        
        .query {
            @container my-container scroll-state(snapped : <scroll-state>) {
                /* 根据 .container 的状态做相应的操作 */
                
                @media (prefers-reduced-motion: no-preference) {
                    /* CSS 动画 */    
                }
            }
        }
    }
}       
```

  


其中 `<scroll-state>` 的值包括：

  


-   **`none`**：查询容器不是对齐目标
-   **`x`**：如果查询容器是其滚动容器的水平对齐目标，则对齐容器特性匹配 `x`
-   **`y`**：如果查询容器是其滚动容器的垂直对齐目标，则对齐容器特性匹配 `y`
-   **`block`**：如果查询容器是其滚动容器在对齐容器的块级方向上的对齐目标，则对齐容器特性匹配 `block`
-   **`inline`**：如果查询容器是其滚动容器在对齐容器的内联方向上的对齐目标，则对齐容器特性匹配 `inline`

  


借助 `snapped` 状态查询，我们可以减少对 JavaScript 和 `scrollsnapchanging` 事件的依赖，将处理逻辑转移到 CSS 中。要使用 `snapped` 查询，必须满足以下三层结构：

  


-   一个设置了 `scroll-snap-type` 的滚动容器
-   内部的捕捉目标元素必须同时拥有 `scroll-snap-align` 和 `container-type:scroll-state`
-   捕捉项目的子元素用于适配和响应 `snapped` 状态查询

  


注意，`scroll-snap-type` 和 `scroll-snap-align` 是滚动捕捉中的相关特性，如果你从未接触过这方面的知识，建议你先移步阅读《[CSS 的滚动捕捉](https://juejin.cn/book/7199571709102391328/section/7199846103007625227)》，然后再继续往下。

  


使用吸附状态查询（`snapped`），我们通常需要一个类似下面这样的 HTML 结构：

  


```HTML
<ul class="list"><!-- 滚动捕捉容器：设置了 scroll-snap-type -->
    <li><!-- 捕捉项目：设置了 scroll-snap-align 和 container-type --> 
        <span>1</span><!-- 捕捉项目的子元素，用于适配和响应 snapped 状态查询 -->
    </li>
    <!-- 省略其他 li -->
</ul>
```

  


对应的 CSS：

  


```CSS
.list {
    scroll-snap-type: x mandatory;
    overflow-x: auto;
    
    li {
        scroll-snap-align: end;
        
        @supports (container-type: scroll-state) {
            container-type: scroll-state;
            
            span {
                @container not scroll-state(snapped: x) {
                    /* CSS ... */
                }
            }
        }
    }
}
```

  


这意味着，吸附（`snapped`）主要用于滚动捕捉的场景中。接下来，我们来看几个这方面的示例。

  


在滚动捕捉的应用场景中，高亮当前被捕捉的项目（当前被吸附的元素）是非常常见的做法。比如在卡片轮播中，给未捕捉到的卡片添加透明，而吸附项保持自然或高亮外观，从而实现视觉聚集。

### 可滚动状态：`scrollable`

可滚动状态查询主要用于测试一个容器的滚动祖先是否可以通过用户发起的滚动在给定的方向上滚动。如果不能，则查询返回 `false` ，如果可以，则 `@container` 块中的规则将应用到匹配的滚动容器的后代元素。注意，`scrollable` 不适用于隐藏容器，也不适用于负值的可滚动溢出区域。

  


可滚动状态查询对于显示一个滚动区域是否可以滚动非常有用。在此之前，这些信息非常难以判断。我们可以像下面这样使用可滚动状态查询：

  


```CSS
.container {
    /* CSS... */
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state;
        container-name: my-container;
        
        .query {
            @container my-container scroll-state(scrollable : <scroll-state>) {
                /* 根据 .container 的状态做相应的操作 */
                
                @media (prefers-reduced-motion: no-preference) {
                    /* CSS 动画 */    
                }
            }
        }
    }
}  
```

  


其中 `<scroll-state>` 的值包括以下这些：

  


-   **`none`**：滚动容器在任何方向上都没有可滚动的溢出内容
-   **`top`**：滚动容器有超出顶部边缘的可滚动溢出内容
-   **`right`**：滚动容器有超出右边缘的可滚动溢出内容
-   **`bottom`**：滚动容器有超出底部边缘的可滚动溢出内容
-   **`left`**：滚动容器有超出左边缘的可滚动溢出内容
-   **`block-start`** ：滚动容器有超出块级起始边缘的可滚动溢出内容
-   **`inline-start`** ：滚动容器有超出内联起始边缘的可滚动溢出内容
-   **`block-end`** ：滚动容器有超出块级结束边缘的可滚动溢出内容
-   **`inline-end`** ：滚动容器有超出内联结束边缘的可滚动溢出内容
-   **`x`** ：滚动容器有水平可滚动的溢出内容
-   **`y`** ：滚动容器有垂直可滚动的溢出内容
-   **`block`** ：滚动容器在其块级方向上有可滚动的溢出内容
-   **`inline`** ：滚动容器在其内联方向上有可滚动的溢出内容。

  


逻辑值根据查询容器的方向和书写模式映射到物理方向。如果容器不是滚动容器，则没有任何值匹配。

## 小结

  


现在，我们有了尺寸容器查询、样式容器查询、状态容器查询和滚动状态容器查询。其中尺寸容器查询、样式容器查询和状态容器查询可以帮助我们更好的构建响应式 UI 组件，而滚动状态容器查询为我们提供了更多互动性和更好的用户体验，同时不会对系统性能造成负面影响。

  


简而言之，滚动状态容器查询可以使我们更容易实现滚动感知设计，而且无需依赖任何 JavaScript 脚本，就可以实现粘附、吸附和溢出等丰富的交互效果。
