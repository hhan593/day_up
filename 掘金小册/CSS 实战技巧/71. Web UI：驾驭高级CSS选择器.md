![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/420dd0fdf76146ed90f79446546d339d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=152876&e=webp&b=d2eff4)

  


CSS 选择器是 CSS 领域中最基础的部分，Web 开发者必须通过 CSS 选择器来选中目标元素，并为其进行样式化处理。简单地说，CSS 选择器帮助我们决定如何将样式应用于特定的 HTML 元素。通过它，我们可以在短短几分钟内彻底改变网站的外观——甚至无需修改任何 HTML 标记。然而，尽管我们都清楚它的用处，但 CSS 选择器仍未被充分利用，我们有时倾向于在 HTML 中堆积过多的、不必要的类（`class`）和 `id` ，以及 `<div>` 和 `<span>` 标签。

  


为了避免这些“灾难”在你的 HTML 标记中蔓延，并保持代码清晰的最佳方法，就是使用更复杂的 CSS 选择器。通过这些选择器，我们可以在不依赖类（`class` ）和 `id` 的情况下选中定特元素，从而使代码和样式表更加灵活。

  


在这节课中，我们将会一起探索一些高级 CSS 选择器的用法，从而帮助你更好的驾驭 CSS 选择器，更轻松地对页面进行样式处理。

  


## CSS 选择器权重

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6a29f9c8cf224785b9c22dc3a1a41d1c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=740&s=101362&e=png&b=ecf6f3)

  


在深入了解高级 CSS 选择器之前，理解 CSS 选择器权重的工作原理是非常重要的，这样我们才能正确地使用 CSS 选择器，避免在调试 CSS 问题时浪费几个小时，而这个问题本来只需要关注权重就能轻松解决。

  


在编写 CSS 时，我们必须记住，有些选择器在层叠中会比其他选择器优先级（权重）更高，最近编写的选择器并不总是会覆盖之前为相同元素编写的选择器。

  


那么，如何计算特定选择器的权重呢？我们通常以四个由逗号分隔的数字来表示 CSS 选择器的权重，比如 `1,1,1,1` 或 `0,2,0,1` 。相对来说，它的计算还是比较简单的：

  


-   第一个数字（`a`）始终为 `0` ，除非在标记本身中对元素应用了 `style` 属性
-   第二个数字（`b`）是选择器中 `ID` 数量的总和
-   第三个数字（`c`）是选择器中其他属性选择器和伪类的总和。类（`class`）和属性选择器也包含在这里
-   第四个数字（`d`）计算元素（如 `div` ）和伪元素（`:first-line`）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/986dd7079a2243e7876a303bce4361d6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=778&s=149876&e=jpg&b=f7f4f3)

  


注意，通用选择器（`*`）的权重为 `0` ，以及子选择器或兄弟选择器（`>`, `~`, `+`）不会增加选择器权重。另外，带有 `!important` 的属性即使选择器的权重较低也会被应用。如果两个选择器的权重相同，最后在样式表中出现的选择器将会获胜。

  


本质上说，CSS 选择器权重就是类似于海底世界的生物生存规则一样，大鱼吃小鱼，小鱼吃虾米：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d92912ceb8c443f39894df704063b7cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1275&h=1204&s=440563&e=png&b=fbf9f9)

  


通常情况下，上述这些规则用于计算 CSS 选择器权不会有任何问题，Web 开发者一直以来都是通过这些规则来计算 CSS 权重的。但随着现代 CSS 的一些特性出现，这些规则不再是绝对的。例如：

  


-   CSS 的 `:where()` 可以将任何选择器的权重变为 `0`
-   CSS 的 `:not()` 、`:has()` 和 `:is()` 将会以其中最具体的选择器来决定权重

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e660161134c04ff5840b417f1af21674~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1001&h=1753&s=139868&e=png&b=f4f6fb)

  


如上图所示，默认情况之下，一个 `id` 选择器（`#a`），它的权重是 `0,1,0,0` ；但是它被 `:where()` 包裹之后，它的权重就变为 `0` ；被 `:not()` 、`:has()` 或 `:is()` 包裹之后，将会根据其内部最高权重来计算，哪怕是这个选择器不存在（比如 `#a#specificity`）也将会增加选择器权重。有关于这方面更具体的介绍，请移步阅读下面两节课：

  


-   [CSS 选择器：`:has()` 与 `:not()` 的组合](https://juejin.cn/book/7223230325122400288/section/7226251495276609569)
-   [CSS 选择器：`:where()` vs. `:is()`](https://juejin.cn/book/7223230325122400288/section/7226251495069450278)

  


虽然 `:has()` 和 `:not()` 对 CSS 选择器权重计算有一定的影响，但一般不会通过它们来改变 CSS 选择器权重，更多的时候将它们用于条件判断。如果仅是调整 CSS 选择器的权重，`:where()` 和 `:is()` 要更适用一些。

  


除此之外，CSS 还引入了 `@layer` 规则来管理选择器的权重与层叠：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/25c30ae2ef3b4fabae402d38f91a381b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2943&h=1444&s=166546&e=png&a=1&b=e1e1e1)

  


它的出现，帮助 Web 开发者解决了 CSS 领域中最为头痛的两个问题——**CSS 的级联（层叠）和权重**！有关于这方面更详细的介绍，请移步阅读《[CSS 分层：`@layer`](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)》！因为，我们这节课的主题还是以 CSS 选择器为主，在这里就不做过多的阐述！

  


## CSS 选择器

  


CSS 选择器有很多种，除了传统的 CSS 选择器（例如元素选择器 `div` 、类选择器 `.class` 和 `ID` 选择器）之外，还有伪类（比如 `:hover`）和伪元素（比如 `::before` 、`::after` 和 `::marker`）。而且，现代 CSS 还新增了很多其他伪类选择器，例如 `:has()` 、`:not()` 、`:where()` 和 `:is()` 等。

  


CSS 选择器的使用看似简单，但它有很多高级用法是不为人知的。接下来，我将和大家一起来聊聊 CSS 选择器，通过介绍这些选择器的使用，使大家能更好的驾驭 CSS 选择器。

  


### 属性选择器

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/49ea2f16c5c54b299163c823a0b4aa9b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=57935&e=png&b=95cafe)

  


CSS 属性选择器允许你根据元素的属性来选择该元素。你可以只指定元素的属性，这样 HTML 中所有拥有该属性的元素（无论属性值如何）都会被选中；或者更具体地，定位具有特定属性值的元素——这就是属性选择器展现其强大功能的地方。

  


CSS 提供了七种不同类型的属性选择器：

  


-   **`[att]`** **（属性选择器）** ：用于选择具有指定属性的元素，无论该属性的值是什么。这意味着，只要元素中包含指定的属性，就会被匹配。例如 `[data-text]` ，只要 HTML 元素显式设置了 `data-text` 属性，就会被选中
-   **`[att=value]`** **（属性等于指定值）** ：选择具有指定属性，并且该属性的值完全等于给定值的元素。例如 `[data-text="hello"]` ，这将选择所有 `data-text` 属性值等于 `"hello"` 的元素
-   **`[att~=value]`** **（属性值包含指定的单词，使用空格分隔）** ：选择具有指定属性，并且该属性的值中包含一个完整的单词匹配给定值（空格分隔的多个值中的一个）。例如 `[title~="old"]` ，这将选择所有 `title` 属性值中包含单词 `"old"` 的元素，例如 `title="old item"` 或 `title="new old item"`
-   **`[att|=value]`** **（属性值等于指定值或以该值开头并后跟一个连字符）** ：选择具有指定属性，且属性值等于给定值或属性值以给定值开头，后跟一个连字符（`-`）的元素。这通常用于选择语言属性，比如 `[lang|="en"]` ，这将选择所有 `lang` 属性值为 `"en"` 或以 `"en-"` 开头的元素（如 `lang="en-US"`、`lang="en-GB"`）
-   **`[att^=value]`** **（属性值以指定的值开头）** ：选择具有指定属性，且属性值以给定值开头的元素。例如 `a[href^="https"]`，这将选择所有 `href` 属性值以 `"https"` 开头的链接（如 `href="``https://example.com``"` ）
-   **`[att$=value]`** **（属性值以指定的值结尾）** ：选择具有指定属性，且属性值以给定值结尾的元素。例如 `a[href$=".jpg"]` ，这将选择所有 `href` 属性值以 `.jpg` 结尾的链接（如 `href="image.jpg"` ）
-   **`[att*=value]`** **（属性值包含指定的值，部分匹配）** ：选择具有指定属性，且属性值中包含给定值的元素。例如 `[title*="saur"]` ，这将选择所有 `title` 属性值中包含 `"saur"` 字符串的元素（如 `title="dinosaur"` 或 `title="saurian"`）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9335fed9ab84c39871978533d1c8189~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=1395&s=969005&e=jpg&b=f3f0f9)

  


属性选择器的另一个有用的用途是选择不同类型的输入元素（`input`），比如你想 **[自定义范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)** 的样式时，可以像下面这样使用属性选择器：

  


```CSS
input[type="range"] {
    --w: 50vmin; /* 输入框宽度 */
    --color: coral; /* 滑块颜色 */
    --progress-color: color-mix(in srgb, var(--color), #000 var(--p,0%)); /* 进度条颜色 */
    --track-color: color-contrast(var(--color) vs #ffba76, #09faec, #bbb, #555,#fff to AA); /* 轨道颜色 */
    --track-size: round(.2em,1px);  /* 轨道高度 */
    --thumb-size: round(1.3em,1px); /* 滑块尺寸 */
    --radius: calc(infinity * 1px); /* 无穷大的圆角半径 */
    
    appearance: none;        /* 禁用浏览器默认样式 */
    background: transparent; /* 清除应用于 input 的默认背景 */
    cursor: pointer;         /* 设置光标为手型 */
    font-size: inherit;
    inline-size: var(--w);
}
```

  


[在多语言排版或布局中使用属性选择器也能为你带来很多的便利](https://juejin.cn/book/7395119173656903715/section/7445251774107090995)：

  


```CSS
[lang="zh"] {
    font-style: normal;
    text-emphasis: circle;
}
```

  


现在，我们来看一个更具体的案例，假设你想给网站链接到不同类型的文件旁边添加不同的图标，以便网站用户知道他们将下载图片、PDF 文件、Word 文档等。你可以通过使用属性选择器来实现：

  


```HTML
<ul>
    <li>
        <a href="image.jpg">Image</a>
    </li>
    <li>
        <a href="word.pdf">PDF</a>
    </li> 
    <li>
        <a href="document.doc">Document</a>
    </li>    
</ul>    
```

  


```CSS
a[href$=".jpg"] {
    background: var(--jpg) no-repeat right 50% / 1em 1em;
    padding-inline-end: 1.5em;
}

a[href$=".pdf"] {
    background: var(--pdf) no-repeat right 50% / 1em 1em;
    padding-inline-end: 1.5em;
}

a[href$=".doc"] {
    background: var(--doc) no-repeat right 50% /1em 1em;
    padding-inline-end: 1.5em;
}
```

  


在这个例子中，我们使用了一个属性选择器，选择所有 `href` 属性以 `.jpg`、`.pdf` 或 `.doc` 结尾的链接（`a`）。

### 子选择器：`E > F`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f95f81cd24aa4d3481ed5653d16086b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1786&s=590615&e=png&b=fdfcfc)

  


子选择器由符号 `>` 表示。它允许你选择特定元素的直接子元素：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/51884517694040929272ea11ad9d1a14~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=340667&e=jpg&b=f3f0f9)

  


如上图所示，`.wrapper > div` 只能选中 `.wrapper` 的直接子元素 `<div>` 。其他之外的 `<div>` 元素不会被选中：

  


```HTML
<div></div>
<div></div>
<div class="wrapper">
    <div></div>
    <div></div>
</div>
<div></div>    
```

  


你还可以使用更深层级的子选择器来选择它的后代元素。例如，如果你想选择位于 `body` 元素的孙子元素 `<div>` 中的 `<blockquote>` 元素，可以像下面这样使用：

  


```CSS
body > div > div blockquote {
    /* CSS ... */
}
```

  


虽然它是一个有效的选择器，但不建议这样使用。通常情况之下，**CSS 选择器不要超过三级**。

  


### 兄弟选择器：`E + F` 和 `E ~ F`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5cf8a4d7ac104f3c8a78f337f17d7f73~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=774&s=477800&e=jpg&b=f3f0f9)

  


兄弟选择器有两种类型：相邻兄弟选择器（`E + F`）和一般兄弟选择器（`E ~ F`），也称为通用兄弟选择器（`E ~ F`）。

  


相邻兄弟选择器使用加号 `+` 来组合两个简单的选择器（`E + F`）。选择器中的元素有相同的父元素，且第二个元素（`F`）必须紧跟在第一个元素（`E`）后面：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/56a5b4a8d3304f4b87e3aec15820c43c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3401&h=1738&s=181080&e=png&b=ffffff)

  


相邻兄弟选择器非常有用，例如，当处理文本时。假设你想为所有跟随段落（`p`）之后的 `h2` 元素添加上边距，可以像下面这样使用：

  


```CSS
p + h2 {
    margin-block-start: 2rem;
}
```

  


这个选择器与单选按钮或复选框的 `:checked` 组合起来，可以替代 JavaScript 做的一些事情，比如小册很多个案例都应用了这个技巧，比如[下拉菜单](https://juejin.cn/book/7395119173656903715/section/7445265601544585225)、[手风琴组件的制作](https://juejin.cn/book/7395119173656903715/section/7444808804371267584)和[内容切换](https://juejin.cn/book/7395119173656903715/section/7443384705539375138)等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b5db637f215248659fa36d088ee4293c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1174&h=510&s=609521&e=gif&f=100&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/gbYdpEQ

  


相邻兄弟选择器与通用选择器 `*` 结合，即 `* + *`，可以选择除第一个元素之外的所有兄弟元素：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e295d8d198e14e488c3dbdccf2c37d0e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=774&s=166055&e=jpg&b=f4f1fa)

  


它与 `:not(:first-child)` 选择器功能相同。

  


一般兄弟选择器（`E ~ F`）与相邻兄弟选择器（`E + F`）工作模式相似，不同之处在于第二个选择器 `F` 不必紧跟在第一个选择器 `E` 后面：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/70af3590b98b49339b8d33ec00c58fa2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3401&h=1747&s=154948&e=png&b=ffffff)

  


正如上图所示，如果你需要选择所有在特定 `div` 内并跟着 `h1` 的所有 `p` 标签，你可以使用这个选择器：

  


```CSS
.post h1 ~ p {
    /* CSS ...*/
}
```

  


同样的，它也经常与单选按钮或复选框的 `:checked` 组合起来使用，替代一些需要 JavaScript 做的事情。同样以内容切换为例：

### 伪类选择器

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/da8f8067bb2542bda27830f1ca1634ba~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=62254&e=jpg&b=ffffff)

  


CSS 伪类选择器是一个大集合，它包含以下这些选择器类型：

  


-   动态伪类，比如 `:link` 、`:visited` 、`:hover` 、`:active` 、`:focus` 、`:focus-visible` 、 `:focus-within` 和 `:target`

-   语言伪类，比如 `:lang()`
-   方向伪类，比如 `:dir()`
-   UI 状态伪类，比如 `:checked` 、`:enabled` 和 `:disabled` 等
-   关系性伪类，比如 `:has()` 、`:not()` 、`:where()` 和 `:is()` 等

#### CSS 数量查询

  


假设我们有一个列表，它的数量不定，可能是一个，也可能是十个。现在，我们需要通过一种特殊选择器使用 CSS 具备数量查询的能力。然后根据数量不同调整 Web 布局：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73290b61ba0549f29b80498ee4307996~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1208&h=654&s=1948589&e=gif&f=315&b=222328)

  


为了帮助我们说明这种数量的限定，我接下来使用“蚊子”的示意图来代表 HTML 元素。其中绿色的“蚊子”表示符合目标 CSS 选择器的元素；红色的“蚊子”表示未被选中的元素，而灰色的“蚊子”则表示不存在的元素：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7a023fbae47e4afb973587419d810249~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=774&s=197446&e=jpg&b=f4f1fa)

  


在我们的 HTML 中，它有十个 `<ul>` ，每个 `<ul>` 包含的列表（`li`）不同，从 `1 ~ 10` ，其中每个列表就是一个“蚊子”的图标。

  


接下来的目标是，我们要使 CSS 具有计数的能力。比如，我们需要使 CSS 知道每个 `ul` 具体包含了多少个 `li` 。换句话说，在特定上下文中确定元素数量的关键是对它们进行计数——我们通过巧妙组合 CSS 选择器来实现。

  


到目前为止，CSS 的 `:only-child` 和 `:only-of-type` 是唯一一个基于数量的选择器，两者唯一的区别是：

  


-   `:only-child` 选择父元素唯一的子元素，不考虑元素类型
-   `:only-of-type` 选择父元素中唯一的某一类型的子元素

  


它们都可以用于在元素独立存在时对其进行样式化。例如：

  


```CSS
li:only-of-type {
    color: var(--selected);
}
```

  


这个时候，`ul` 只有一个 `li` （即只有一只“蚊子”）才能匹配上，它变成“绿色”：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/074162d26c3a4fcb86550c746b67df7e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=1338&s=586446&e=jpg&b=f4f4f4)

  


你可以通过使用 `:not()` 选择器来颠倒 CSS 逻辑，使条件变为“多于一个”：

  


```CSS
li:not(:only-of-type) {
    color: var(--selected);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4bcafc1e77cb4e708d540a4340af5f1e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=1338&s=550554&e=jpg&b=f4f4f4)

  


如果数量变成 `n` 呢？例如，多于或等于 `n` 怎么查询呢？我们需要一个选择器，允许我们按数字遍历任何数量的元素集合。

  


庆幸的是，CSS 的 `:nth-last-child(n)` 选择器接受数字 `n` ，使我们能够从集合的末尾向前计数元素。假设 `n=6` ：

  


```CSS
li:nth-last-child(6) {
    color: var(--selected);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5287bc4060d447ba836f0a1ff257e483~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2062&h=1338&s=609630&e=jpg&b=f4f4f4)

  


从末尾向前数，第六只“蚊子”（第六个 `li`）被选中。

  


我们将 `:nth-last-child(6)` 与 `:first-child` 组合在一起，情况就变得很有趣了：

  


```CSS
li:nth-last-child(6):first-child{
    color: var(--selected);
}
```

  


因为引入了第二个条件（`:frist-child`）。在这种情况下，上面的代码会匹配的既是倒数第六只“蚊子”（倒数第六个 `li` 元素），又是第一只“蚊子”（第一个 `li` 元素）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c999eabedca74a2ca9171ef07ff7abdf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=576056&e=jpg&b=f4f4f4)

  


如果这个元素存在，那么元素集合的数量必须正好是六个。某种程度上，我们已经使用 CSS 查询到了元素的数量。不难发现，到目前为止，我们选中的包含六个 `li` 元素中的第一个 `li` （它既然是倒数第六个，又是顺数第一个）。如果要选中这个列表中的所有 `li` 元素（“蚊子”），我们还需要使用 CSS 的通用兄弟选择器：

  


```CSS
li:nth-last-child(6):first-child, 
li:nth-last-child(6):first-child ~ li {
    color: var(--selected);
}
```

  


`li:nth-last-child(6):first-child ~ li` 选择器的意思是：选中所有出现在 `li:nth-last-child(6):first-child` 之后的 `li` 元素。这样一来，如果 `li` 元素总数恰好是六个，每个元素的文本颜色都是绿色（“蚊子”被选中）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d84ab974a5d4599843cb59306124f20~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=571728&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/NPKLNWB

  


你可以把上面代码中的 `6` 替换成任意你想要的一个数值，该值表示列表数（“蚊子”）的总数量。例如：

  


```CSS
li:nth-last-child(4):first-child, 
li:nth-last-child(4):first-child ~ li {
    color: var(--selected);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b023d0b676694ee98836d2973f1784ee~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=573683&e=jpg&b=f4f4f4)

  


如此一来，获得一个通用的公式：

  


```CSS
li:nth-last-child(n):first-child, 
li:nth-last-child(n):first-child ~ li {
    /* CSS ... */
}
```

  


`n` 表示列表数的总数（或者说查询元素的总数）！

  


到目前为止，我们使用上面的选择器可以匹配到具体数量的“蚊子”。然而，有的时候我们希望匹配到更多的范围，比如匹配到大于 `n` 或小于 `n` 的范围。这个需求是很常见的，比如：

  


-   总数恰好是 `6` 的“蚊子”变成绿色
-   总数小于 `6` 的“蚊子”变成紫色
-   总数大于 `6` 的“蚊子”变成红色

  


总数恰好是 `6` 的选择器我们已经有了：

  


```CSS
/* n = 6 */
li:nth-last-child(6):first-child, 
li:nth-last-child(6):first-child ~ li {
    /* CSS ... */
}
```

  


接下来，我们来看总数大于 `6` 的选择器应该怎么写。

  


稍微了解 CSS 结构选择器的开发者应该知道，结构伪类选择器可以采用算术参数，比如 `n + A` ，其中 `A` 是一个整数。例如，`:nth-child(n + 6)` 会选中从第六个元素开始的所有元素：

  


```CSS
li:nth-child(n + 6) {
    color: red;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f70e58be44a541fb95c5126a9b146f21~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=384117&e=jpg&b=f4f4f4)

  


尽管这有其自身的应用场景，但它并不是一种“数量感知”的选择方法：我们并不是因为总共有六个或更多元素（“蚊子”）而对它进行样式化；我们只是样式化那些恰好排列在第五个以上的元素。

  


为了正确解决这个问题，我们真正需要的是创建一个排除了最后五个项的元素集合。通过使用 `:nth-child(n + 6)` 的反向选择器 `:nth-last-child(n + 6)` ，我们可以从第六只“蚊子”开始，向集合的开始方向计数：

  


```CSS
li:nth-last-child(n+6) {
    color: red;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bf00ffe062144b2481485d8ac435228e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=945&s=451478&e=jpg&b=f4f4f4)

  


这将排队一个集合中的最后五个元素，这意味着当你将集合的长度减少到六个以下时，你将看不到任何被选中的元素。如果集合的总数确实大于或等于六个，那么剩下的就是对最后五个元素进行样式化。这很简单：当项数超过六个时，必须存在一个或多个符合 `:nth-last-child(n + 6)` 条件的元素（返回真）。所有这些存在的元素可以与 `~` 组合，以影响它后面所有的元素：

  


```CSS
li:nth-last-child(n+6),
li:nth-last-child(n+6) ~ li {
    color: red;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/77e0ab4586ac4e80bdf94c44b7e75ff3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=491162&e=jpg&b=f4f4f4)

  


正如你所看到的，总数大于 `6` 的“蚊子”都变成了红色。

  


接下来，我们要将总数小于 `6` 的“蚊子”都变成紫色。我们可以将上面的逻辑进行反转，从大于或等于 `n` 切换到小于或等于 `n` ，即 `:nth-last-child(-n + 6)` ，并结合 `:first-child` 条件，就可以实现小于或等于 `n` 。

  


```CSS
li:nth-last-child(-n+6):first-child,
li:nth-last-child(-n+6):first-child ~ li {
    color: purple;
}
```

  


实际上，使用 `-` 符号切换了选择的方向：它不再从第六个元素开始指向起始位置，而是从第六个元素开始指向集合的末尾。在每种情况下，选择器都会包括第六个元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/29ad14026c1349739f48229ccf5ff8d5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=561620&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/raBZMer

  


就是这么简单，将结构伪类选择器与通用兄弟选择器组合，就使 CSS 具备了查询数量的能力：

  


```CSS
/* 总数小于或等于 A*/

li:nth-last-child(-n + A):first-child,
li:nth-last-child(-n + A):first-child ~ li {
    /* CSS .... */
}
  
/* 总数等于 A */  
li:nth-last-child(A):first-child,
li:nth-last-child(A):first-child ~ li {
    /* CSS ... */
}

/* 总数大于或等于 A*/  
li:nth-last-child(n + A),
li:nth-last-child(n + A) ~ li {
    /* CSS */
}
```

  


注意，`A` 是一个整数，比如我们之前示例中的 `6` 。

  


按照同样的原理，通过结合 `:nth-last-child(An)` 和 `:first-child` 两个选择器来模拟一个 CSS 取模查询（Mod Query）。比如下面的代码，选择一切能被 `3` （`A = 3`）整除的列表：

  


```CSS
/*选择列表中所有能被3整除的列表*/ 
li:nth-last-child(3n):first-child, 
li:nth-last-child(3n):first-child ~ li {
    color: green;
} 
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b3bb36a3e8a947afbe0585eb65cf3fd0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=363663&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/bNbxoOZ

  


简单解释一下该选择器：

  


-   ①：`li:nth-last-child(3n)` ，从列表项最后一项开始计，位于第 `3n` 的列表项都会被选中；
-   ②：`li:nth-last-child(3n):first-child` ，列表项总数被 `3` 整除，并且结合 `:first-child` 选中第一个列表项。即选中被 `3` 整除的列表中的第一列表项；
-   ③：`li:nth-last-child(3n):first-child ~ li` ，通用兄弟选择器，选中被 `3` 整除的列表中的非第一个列表项之外所有列表项；
-   ④：`li:nth-last-child(3n):first-child` 和 `li:nth-last-child(3n):first-child ~ li` 组合在一起，将选中能被 `3` 整除的列表中的所有列表项。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/92f9e0868d584ef09bf3f5fef2c7dc12~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2658&h=1610&s=304253&e=jpg&b=555675)

  


你只需要改变 `:nth-last-child(An)` 中 `A` （整数） ，就可以获得不同数值的整数倍，比如分别被 `2` 、`3` 、`4` 和 `5` 整除：

  


```CSS
/* 列表项总数被 1 整除 */
li:nth-last-child(n):first-child,
li:nth-last-child(n):first-child ~ li {
    color: red;
}

/* 列表项总数被 2 整除 */
li:nth-last-child(2n):first-child,
li:nth-last-child(2n):first-child ~ li {
    color: green;
}

/* 列表项总数被 3 整除 */
li:nth-last-child(3n):first-child,
li:nth-last-child(3n):first-child ~ li {
    color: blue;
}

/* 列表项总数被 4 整除 */
li:nth-last-child(4n):first-child,
li:nth-last-child(4n):first-child ~ li {
    color: orange;
}

/* 列表项总数被 5 整除 */
li:nth-last-child(5n):first-child,
li:nth-last-child(5n):first-child ~ li {
    color: purple;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e08a59a0eff345a49c7408668f8a071f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=550452&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/VYZGMRQ

  


通过取模查询（Mod Query），如果列表项总数能被 `A` 整除的列表项都能选中，但列表项总数不能被 `A` 整除时，也就是有余数的时候，需要给列表一个不同的样式。

  


如果余数为 `B` 的时候，只需要从倒数第二个开始计数，而不是最后一个。如此一来只需要在查询中简单的 `+B` 就可以。

  


```CSS
/* Mod查询，列表长度能被3整除 */
li:nth-last-child(3n):first-child,
li:nth-last-child(3n):first-child ~ li {
    color: green;
}
  
/* Mod查询，列表长度能被3整除余数1 */
li:nth-last-child(3n+1):first-child,
li:nth-last-child(3n+1):first-child ~ li {
    color: red;
    rotate: 45deg;
}

/* Mod查询，列表长度能被3整除余数2 */
li:nth-last-child(3n+2):first-child,
li:nth-last-child(3n+2):first-child ~ li {
    color: blue;
    rotate: -45deg
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/afdce3b4d83d4d468ce545ca64f9f691~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=549496&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/qEWMPzP

  


按同样的原理，我们可以结合 `:nth-child(n)`（一切大于 `n`）和 `:nth-child(-n)`（一切小于 `n`）两个选择器实现一个范围选择器。假设要选择列表中的第 `3` 至 `5` 这几个列表项，可以这样使用：

  


```CSS
li:nth-child(n+3):nth-child(-n+5){ 
    color: red; 
} 
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2bf79792f7834510acd58a72ea8cfdfe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=394056&e=jpg&b=f4f4f4)

  


有了这些功能之后，查询最终并不像我预期想的那么困难，只需要把取模和范围选择器组合在一起。

  


```CSS
li:nth-last-child(3n):first-child ~ li:nth-child(n + 3):nth-child(-n + 5){
    color: #fa1289;
    rotate:45deg;
}
```

#### `:nth-child(An+B [of S]?)` 和 `:nth-last-child(An+B [of S]?)` 选择器

  


`:nth-child(An+B [of S]?)` 和 `:nth-last-child(An+B [of S]?)` 选择器也是结构伪类选择器中的一部分。目前可以在 Chromium 111、Safari 9 和 Firefox 113 看到该选择器的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/be428d2533df4646bebdcc20f97bb1df~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1098&h=564&s=5573980&e=gif&f=242&b=4b486a)

  


> Demo 地址：https://codepen.io/airen/full/YzJVpLQ

  


幸运的是，我们可以使用 `:has()` 、`:not()` 和 `~` 组合在一起来模拟 `:nth-child(An+B [of S]?)` 和 `:nth-last-child(An+B [of S]?)`。

  


先从简单的着手，使用下面选择器可以模拟出 `:nth-child(2 of .🦵)` 选择器，即 **`:nth-child(B of S)`**：

  


```CSS
.🦵 ~ .🦵:not(.🦵 ~ .🦵 ~ .🦵) {
    background-color: #09f;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9640bf4f13894c8b8eec865e74e45c28~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=888&h=572&s=1326417&e=gif&f=168&b=616d9e)

  


> Demo 地址：https://codepen.io/airen/full/QWZvMMv

  


它的工作原理是：

  


-   使用 `.🦵 ~ .🦵` 选择所有前面有 `1` 个 `.🦵` 的 `.🦵` ，从而有效地选择第 `2` 个、第 `3` 个、第 `4` 个、第 `n` 个 `.🦵` 兄弟元素。简单地说，会选择除了第一个 `.🦵` 之外的所有 `.🦵`
-   将该选择限制为排除任何前面有 `2` 个 `.🦵` 的 `.🦵`，从而从初始选择中排除第 `3` 个、第 `4`个、第 `n` 个 `.🦵` 兄弟元素。

也就是说，你可以通过在选择器中添加更多的 `~ .🦵` 部分，来选择第 `3` 个、第 `4` 个、第 `5` 个、第 `n` 个 `.🦵` 兄弟元素，从而实现 `:nth-child(B of S)` 选择器。

  


模拟 **`:nth-last-child(B of S)`** 选择器是相似的。比如，使用下面的代码可以模拟出 `:nth-last-child(3 of .🦵)` 选择器：

  


```CSS
.🦵:not(:has(~ .🦵 ~ .🦵 ~ .🦵)):not(.🦵:not(:has(~ .🦵 ~ .🦵))) {
    background-color: #09f;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6629b2c230f343c085c06e1dbc96f279~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1106&h=570&s=1830862&e=gif&f=178&b=616d9e)

  


> Demo 地址： https://codepen.io/airen/full/QWZvMeQ

  


它的工作原理是：

  


-   使用 `.🦵:not(:has(~ .🦵 ~ .🦵 ~ .🦵))` 选择最后 `3` 个 `.🦵`
-   使用 `.🦵:not(.🦵:not(:has(~ .🦵 ~ .🦵)))` 选择除最后 `2` 个 `.🦵` 之外的所有 `.🦵`

  


从两个选择中取交集就得到了 `:nth-last-child(3 of .🦵)`。

有趣的是，我们还可以使用 `:not()` 选择器来实现：选择两个类名之间的所有元素：

  


```CSS
.start ~ :not(.end, .end ~ *) {
    /* CSS ... */
}
```

  


简单的解释一下：

  


-   `.start ~`：选择 `.start` 后面的所有兄弟元素（即与 `.start` 同级的后续元素）
-   `:not(.end, .end ~ *)`： `:not(.end)` 排除具有 `.end` 类的元素；`:not(.end ~ *)` 排除 `.end` 后面的所有兄弟元素
-   综合起来，选择的是从 `.start` 到 `.end`（不包含 `.end` 本身及其后续兄弟元素）之间的所有元素

  


比如，你有一个列表，其中 `.start` 和 `.end` 是标记类，想要对两者之间的元素应用样式：

  


```HTML
<div class="container">
    <div></div>
    <div></div>
    <div class="start">👇</div>
    <div></div>
    <div></div>
    <div></div>
    <div></div>
    <div class="end">☝️</div>
    <div></div>
    <div></div>
    <div></div>
</div>
```

  


```CSS
.start ~ :not(.end,.end ~ *) {
    color: red;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/86474de98172498496e8b45f7c350b0f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1338&s=175998&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/VYZGrbG

  


另一个高级选择器是，使用 `:not()` 和 `~` 选择器选择带有特定类名的第一个和最后一个元素：

  


```CSS
.cat:not(.cat ~ *) {
    /* CSS ... */
}
```

  


上面的代码会选中第一个带有 `.cat` 类名的元素，并排除后面还有其他 `.cat` 类名的元素（即，仅匹配第一个 `.cat` 元素）。另外一种方法是直接使用 `:nth-child()` 的 `of` 语法，定位第一个类名 `.cat` ：

  


```CSS
:nth-child(1 of .cat) {
    /* CSS ... */
}
```

  


我们可以使用 `:has()` 和 `~` 来选择特定类名的最后一个元素。例如：

  


```CSS
.bird:not(:has(~ .bird)) {
    /* CSS ... */
}
```

  


上面的代码会选择最后一个类名为 `.bird` 元素，并排除前面的 `.bird` 元素（即，仅匹配最后一个 `.bird` 元素）。同样的，可以使用 `:nth-last-child()` 的 `of` 语法，定位最后一个类名为 `.bird` 的元素:

  


```CSS
:nth-last-child(1 of .bird) {
    /* CSS ...*/
}
```

  


-   使用 `:nth-child()` 和 `:nth-last-child()` 的 `of` 语法是 CSS 的一种简洁高效的方法，但需要现代浏览器支持
-   对于更复杂的选择，可以结合 `:not()` 和 `:has()` 选择器实现兼容性更强的解决方案

  


温馨提示，使用 `:has()` 和 `:not()` 选择器还可以选择多个范围组的元素。如果你对这方面知识感兴趣的话，请移步阅读《[CSS 选择器：`:has()` 与 `:not()` 的组合](https://juejin.cn/book/7223230325122400288/section/7226251495276609569#heading-4)》！
