[现代 CSS 提供了一个 `aspect-ratio` 属性](https://juejin.cn/book/7223230325122400288/section/7259316040151236666)，这是一个非常有用的属性，因为它无需强制设定具体尺寸就能更好的满足你的设计思维或目标效果。简单地说，我们可以根据元素的宽高比率来设定元素的尺寸。比如，我们需要一个正方形，只需要将 `aspect-ratio` 设置为 `1` ，浏览器将会根据元素设定的 `width` 或 `height` 获得另个维度的尺寸。换句话说，如果元素尺寸会随情景变化时，它显得尤其重要。例如一个视频，你希望视频不管在什么环境下（用户可能会使用横屏或竖屏方式浏览器视频），它都能始终保持 `16:9` 的比例展示，那么 `aspect-ratio` 就非常适用。因为你只需要将视频的 `width` 或 `height` 设置为 `100%` ，`aspect-ratio` 会根据你指定的比例获得另下维度的尺寸。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eeed94b6f70940bb8d6d8fefc6243c7c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2560&h=1440&s=1691456&e=png&b=ffffff)

## `aspect-ratio` 基本用法

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4a016751972c4016a3b1db813c49e736~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1707&h=1139&s=1304262&e=jpg&b=051e1a)

  


CSS 的 `aspect-ratio` 的基本用法非常简单，你可以像下面这样使用 `aspect-ratio`:

  


```CSS
.ratio {
    aspect-ratio: 16 / 9;
    aspect-ratio: 1.77778;
}
```

  


它指定的是元素的 `width/height` ，即可元素的宽度与高度的比例。这意味着，当你给元素指定了 `aspect-ratio` 时，浏览器会根据你指定的另一个值（比如 `width`）计算出它的另一个维度的值（`height`），反之亦然！

  


```HTML
<body>
    <video src="https://assets.codepen.io/3/mov_bbb.mp4" muted autoplay loop playsinline controls></video>
</body>
```

  


```CSS
video {
    display: block;
    width: 100%;
    aspect-ratio: 9/16;
    
    @media screen and (orientation: landscape) { 
        aspect-ratio: 16/9;
    }
}    
```

  


上面代码给 `video` 设置了 `width: 100%` ，它的宽度将会与 `body` 宽度相等。在这里，这也将与浏览器视窗的宽度相等。然后给它设置了 `aspect-ratio: 9 / 16` ，并在横屏时，将它的比例调整为 `16 / 9` 。这样一来，你在移动端浏览视频时，横屏和竖屏都将有一个比较好的体验：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c5f4528c7249438fae563e7585531be7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1214&h=814&s=10786567&e=gif&f=93&b=d4e8eb)

  


> Demo 地址：https://codepen.io/airen/full/PwYNrPB

  


你会发现，`aspect-ratio` 属性的值可以是带斜杠（`/`）的值，斜杠前后的值表示元素宽高比例；它也可以是一个数值（非负数）。如果它只是一个一个数值时，斜杠后面的值默认为 `1` 。比如，下面两种写法，使元素的宽高比保持一致：

  


```CSS
.ratio {
    aspect-ratio: 1.7778;
}

/* 等同于 */
.ratio {
    aspect-ratio: 1.7778 / 1;
}
```

  


需要知道的是，HTML 中的可替换元素（比如 `<img>` 、`<iframe>` 、`<video>` 和 `<object>` 等）具固有的宽高比。如下图所示，它是一张宽为 `260px` ，高为 `195px` 的图片，其固有的宽高比为 `1.33` ，即 `260 ÷ 195` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/783e8a7e42be49b2a947fec7efecf45c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1700&h=894&s=60280&e=jpg&b=f9f8f8)

  


这意味着，如果没有显式给 `img` 设置 `aspect-ratio` ，那么它的宽高比大约是 `1.33` 。正因为如此，我们在调整 `img` 的宽度或高度时，它始终会根据固有的比例来进行调整。如下图所示，当你调整图片的宽度或高度时，它的另一个维度的值会根据固有比例获得：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/58433be9fed642ffbf7bdcfab194750f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1700&h=600&s=55806&e=jpg&b=fefdfd)

  


除了给 `aspect-ratio` 属性指定数字值（即宽高的比例值， `w ÷ h`）之外，还可以将其设置为 `auto` 值。只不过，当 `aspect-ratio` 的值为 `auto` 时，它与元素类型有着紧密的联系：

  


-   如果元素是可替换元素（比如 `<img>`），那么 `auto` 等于元素固有的宽高比
-   如果元素不是可替换元素（比如 `<div>`），那么 `auto` 会使元素框没有首选宽高比

  


这使得我们可以给 `aspect-ratio` 属性同时设置两个值，即 `auto <ratio>` ：

  


-   如果该元素具有固有的宽高比（比如 `<img>` 元素），那么 `aspect-ratio` 的值为 `auto`，最终的值将会是元素自身的固有宽高比
-   如果该元素没有固有的宽高比（比如 `<div>` 元素），那么 `aspect-ratio` 的值为 `<ratio>` ，即你指定的宽高比作为首选宽高比

  


例如下面这个示例：

  


```HTML
<div class="ratio"></div>
<img class="ratio" src="https://picsum.photos/id/163/260/195" alt="" />
```

  


注意，上面 `img` 将会加载一张 `260px x 195px` 的图片（其固有的宽高比为 `1.333`）。假设我们将它们的 `aspect-ratio` 设置为 `auto 16 / 9` ：

  


```CSS
.ratio {
    width: 200px;
    aspect-ratio: auto 16 / 9;
}
```

  


你将看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/60cf33d03f064679a22d031c2c3569cc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1089&s=267222&e=jpg&b=607d8b)

  


> Demo 地址：https://codepen.io/airen/full/MYgeWQK

  


结果很明显：

  


-   `<div>` 是一个不可替换的元素，它的 `aspect-ratio` 选择了 `16 / 9` 作为元素的首选宽高比（大约是 `1.778`）
-   `<img>` 是一个可替换的元素，它的 `aspect-ratio` 选择了 `auto` 作为元素的首选宽高比（`img` 固有的宽高比，即 `260/195` ，比约是 `1.333`）

  


因此，具有 相同的宽度 `200px` 的 `div` 和 `img` 元素，最终获得的尺寸是不一样的，`div` 的高度要小于 `img` 的高度。

  


通常情况之下，`aspect-ratio` 的使用就是这么简单。但是，有时会出现一些问题，导致某个元素看起来没有遵守 `aspect-ratio` 的设置。要了解 `aspect-ratio` 属性何时会失效，首先得了解其工作原理。

  


## `aspect-ratio` 工作原理

  


大部分 Web 开发者通常只了解到 `aspect-ratio` 的字面意思，即指定元素的宽高比。事实上，`aspect-ratio` 属性定义了一个盒子的首选宽高比。通过指定宽高比，开发者可以在保持布局灵活的同时让元素根据宽高比自动调整尺寸。请注意，这里有两个重要的概念**首选宽高比**和**指定宽高比** ：

  


-   **首选宽高比**：定义宽度和高度的比率，用于决定盒子的最终大小。如果宽高或高度通过其他规则定义了一个明确的值，浏览器会自动根据宽高比推导出另一个维度
-   **指定宽高比**：使用 `<ratio>` （比如 `16 / 9`）表示宽高比，也可以结合 `auto` 使用（如 `aspect-ratio: auto 16/9` ）

  


当然，你可能对这些概念并不感兴趣，只是想掌握 `aspect-ratio` 的使用，并且使其能按照自己的意愿来工作。既然如此，那就需要了解其工作流程。简单地说，就是要知道元素的宽高比来源。你可以根据下图来判断元素宽高比的来源。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99f35050cccc4bb49c7207ccb2a94a55~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2352&h=3452&s=930984&e=jpg&b=ffffff)

  


简单的解释一下。首先判断元素是不是可替换元素（如 `<img>` 、`<video>` 等），接着判断是否指定了宽高比：

  


-   如果替换元素没有指定宽高比，则自动使用固有的宽高比，否则会使用指定的宽高比，即 `aspect-ratio` 指定的宽高比例 `<ratio>`
-   如果是非替换元素，`aspect-ratio` 会直接作为计算宽度和高度的依据

  


这里还将涉及到一个优先级规则，如果 `auto` 和 `<ratio>` 同时定义（如 `aspect-ratio: auto 16/9`），那么可替换元素则会使用固有的宽高比，非替换元素则会使用指定的宽高比 `<ratio>`。

  


接下来，我们再来了解一下 `aspect-ratio` 背后的计算逻辑。其计算主要涉及三个情景：

  


-   **元素有明确宽度**：如果元素显式设置了 `width` ，浏览器将会根据 `aspect-ratio` 推算出 `height` ：`height = width ÷ aspect-ratio`
-   **元素有明确高度**：如果元素显式设置了 `height` ，浏览器将会根据 `aspect-ratio` 推算出 `width` ：`width = height × aspect-ratio`
-   **没有明确的宽度或高度**：`aspect-ratio` 会尝试与父容器的约束规则（如 `flex` 或 `grid` 布局）结合，决定最终尺寸

  


了解了 `aspect-ratio` 的工用原理之后，你就可以很好的使用它。即使在使用的过程中，碰到问题也可以快速定位到问题，并解决问题。

  


## `aspect-ratio` 潜在的问题

  


接下来，我们就来聊聊几个使用 `aspect-ratio` 常见的问题。

  


### 问题一：同时设置宽高

  


第一个使 `aspect-ratio` 失效的原因是：**同时给元素设置了宽度（** **`width`** **）和高度（** **`height`** **）** 。当然，这同样适用于逻辑属性 `inline-size` 和 `block-size` 。

  


```CSS
.ratio {
    width: 400px;
    height: 300px;
    aspect-ratio: 1; /* 被忽略 */
}
```

  


如果你了解 `aspect-ratio` 的工作原理，那么理解上面的代码就不会有任何难度。之前我们提到过，只有元素明确的设定宽度或高度或者两者都没有设定的情况之下，浏览器才会根据 `aspect-ratio` 推导出另一维度的尺寸。

  


这也意味着，如果元素显式设置了宽度和高度，那么 `aspect-ratio` 将是无效的。就这一情景而言，还是很好理解的。但有些情景之下，你可能会感到困惑。例如，在 CSS 代码中只显式设置了 `width` 或 `height` ，但 `aspect-ratio` 却没有生效。以下面示例为例：

  


```HTML
<img src="figure.png" alt="" height="100" class="ratio" />
```

  


```CSS
.ratio {
    width: 400px;
    aspect-ratio: 4 / 3;
}
```

  


就这个示例而言，`height` （这个维度）的值来自于 HTML 中，它可能是你意想不到的地方，但它的确使浏览器忽略了 CSS 中的 `aspect-ratio` 。

  


也就是说，当你发现设置的 `aspect-ratio` 没有生效时，首先要排查的是元素是否同时设置了宽高和高度，包括 HTML 元素的 `width` 或 `height` 设置。

  


这种情况不仅限于 HTML 中的属性，同样也适用于 CSS 的外部信息。例如，我们在构建 Web 布局时，可能会使用 CSS 的 `min-width/height` 和 `max-width/height` 以及它们对应的逻辑属性 `min-inline-size/block-size` 和 `max-inline-size/block-size` 来限制尺寸。这些属性的设置同样会造成 `aspect-ratio` 失效。

  


### 问题二：Flexbox 或 Grid 布局中的拉伸

  


当下，CSS Flexbox 和 Grid 已成为主流布局技术，在一些场景中，我们可能会给 Flex 项目或 Grid 项目设置 `aspect-ratio` 。比如下面这个示例，我们给三个 `<div>` （Flex 项目）设置了不同的宽高比，但呈现给我们的结果是，三个 Flex 项目的尺寸是一样的：

  


```HTML
<div class="flex">
    <div class="flex__item" style="--ratio: 1/1"></div>
    <div class="flex__item" style="--ratio: 4/3"></div>
    <div class="flex__item" style="--ratio: 16/9"></div>
</div>
```

  


```CSS
.flex {
    display: flex;
    gap: 1rem;
    
    .flex__item {
        flex: 1;
        aspect-ratio: var(--ratio);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eeeab4bd70fd4751b48189cdabc6d567~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1089&s=171741&e=jpg&b=607d8b)

  


> Demo 地址：https://codepen.io/airen/full/MYgevZr

  


这些 Flex 项目并没有显式设置 `width` 或 `height` ，浏览器将会根据其父容器宽度（Flexbox 容器的可用空间）来计算 Flex 项目的宽度。根据 `aspect-ratio` 的工作原理，浏览器应该会根据 Flex 项目宽度和 `aspect-ratio` 推算出 Flex 项目的高度。事实是，浏览器并没有这么做。为什么会这样呢？

  


这是由于 Flex 项目会受到默认的 `align-items: stretch` 的影响，导致 Flex 项目的高度被强制拉伸。因此，`aspect-ratio` 看上去失效了（没有按照预期进行工作）。我们可以将 Flexbox 容器的 `align-items` 重置为 `flex-start` （只要不是默认的 `stretch`），你就能看到 `aspect-ratio` 开始正常工作：

  


```CSS
.flex {
    align-items: flex-start;
    
    .flex__item {
        flex: 1;
        aspect-ratio: var(--ratio);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/db8af308c9ec4197ad550452f1567c17~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1244&h=580&s=277342&e=gif&f=111&b=547080)

  


> Demo 地址：https://codepen.io/airen/full/jENrGGx

  


注意，除了将 Flexbox 容器的 `align-items` 属性的值设置为非 `stretch` 值之外，还可以将 Flex 项目的 `align-self` 属性的值设置为非 `stretch` 值，同样能使 `aspect-ratio` 正常工作。

  


另外，这种情况同样会发生在 Grid 布局中。不同之处是，Grid 项目会根据拉伸之后的高度和设置的 `aspect-ratio` 重新计算 Grid 项目的宽度：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(3, 1fr);
    
    .grid__item {
        aspect-ratio: var(--ratio);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/699dd58641144d5c95effe6bb770e21d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1480&h=572&s=523734&e=gif&f=144&b=0aa3fb)

  


> Demo 地址：https://codepen.io/airen/full/MYgeEQz

### 问题三：内容的高度

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbbc80eb53a2458d9a1a85b5e0da6c32~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1514&h=544&s=43126&e=png&a=1&b=f3e9da)

  


在 `aspect-ratio` 属性还没有得到浏览器支持之前，通常会使用 CSS Hack 手段来创建一个固定宽高比例的盒子。这个方法通过在零高度的盒子上使用 `padding-bottom` 或 `padding-top` 来创建一个盒子。百分比的 `padding-bottom/top` 将会根据元素的 `width` 进行计算，从而建立了宽度和高度之间的关系，这种关系也就是我们所说的宽高比。但这种宽高比是固定的，如果你在盒子的内容的高度超过盒子本身的高度，就会造成内容溢出。比如下面这个示例，黄色区域具有固定的宽高比，比如 `4:3` ，它会根据元素的 `width` 推导出其高度。当其内容高度超出黄色盒子高度时，内容会溢出：

  


```HTML
<div class="ratio">
    <div class="content">
        <p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>
    </div>
</div>
```

  


```CSS
.ratio {
    position: relative;
    height: 0;
    padding-bottom: 75%; /* 4:3 */
    max-inline-size: 45ch;
    margin-inline: auto;
    background: orange;
    
    .content {
        position: absolute;
        inset: 0;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ac71ba194394fa8a13fc50fb4ac9d55~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1212&h=696&s=564741&e=gif&f=170&b=557080)

  


> Demo 地址：https://codepen.io/airen/full/MYgeOpW

  


这种情景下，元素具有正确的宽高比，但这种结果通常不是我们所希望的——内容溢出了。

  


如果我们改用 `aspect-ratio` 来定义元素的宽高比，你会发现，当内容高度比计算出来的盒子高度（根据 `aspect-ratio` 和 `width` 推算出来的高度）更大时，元素框的高度会自动调整，并且适应内容高度：

  


```CSS
.ratio {
    max-inline-size: 45ch;
    aspect-ratio: 4 / 3;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/10fc2e0cebc04af788565fbf5565e7f2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1288&h=676&s=713612&e=gif&f=115&b=557080)

  


> Demo 地址：https://codepen.io/airen/full/ZYzOaXG

  


正如你所看到的，当内容高度大于黄色盒子高度时，`aspect-ratio` 将会被忽略，黄色盒子的高度最终会等于所有内容的高度总和。

  


## 案例：创建受限制的宽高比盒子

  


我想你现在应该对 `aspect-ratio` 有了更多的了解，甚至可以很好的用好它。接下来，我们来通过下面这个更具体的场景，帮助大家更深入的掌握 `aspect-ratio` 。

  


在 CSS 中还有诸多问题是无法彻底解决的。比如说，CSS 实现受限制宽高比盒子。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/75246d4702694592b21ac42fc4e7174b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3336&h=1652&s=127906&e=jpg&b=22252a)

  


上图展示了一个被高度或被宽度限制的盒子。一个白色的容器中包含一个红色的盒子，其中容器的高度可能是固定的，也可能由其他布局元素决定。其内部的红色盒子需要一个固定的宽高比，并在保持该比例的情况下尽可能填满白色容器。

  


这种情况可能看似复杂，但如果你考虑到例如灯箱效果（Lightbox）的实现，这种需求就很常见了。点击缩略图后，显示一个全屏幕覆盖的图像，按照其原始宽高比填充屏幕的可用空间。图像要么垂直填充屏幕，要么水平填充屏幕。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/876f45e4b5a1434594c6fae998233ff9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1056&h=650&s=18649449&e=gif&f=81&b=f8f4f4)

  


就这种布局效果而言，如果元素是可替换元素（例如 `<img>` 或 `<iframe>`），使用 `object-fit: contain` 可以轻易实现：

  


```CSS
.box img {
    display: block;
    width: 100%;
    height: 100%;
    object-fit: contain;
}
```

  


但这种方法仅适用于可替换元素，对于不可替换元素（比如 `<div>` 元素）就不行了。

  


庆幸的是，我们可以借助 [CSS 的比较函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)（比如 `min()`）和 [CSS 的视窗单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)对不可替代元素实现类似的效果。例如：

  


```HTML
<body>
    <div class="box"></div>
</body>
```

  


我们使用 CSS Grid 来创建布局：

  


```CSS
:is(html,body) {
    display: grid;
}

html {
    height: 100%; 
}

body {
    margin: 0;
}
```

  


如果你阅读过小册之前的课程《[CSS 布局：创建全屏布局](https://juejin.cn/book/7395119173656903715/section/7444358182228459560#heading-0)》，那么你会知道，上面的代码会使 `.box` 填充整个屏幕（它的大小与浏览器视窗尺寸相等）。接下来是创建受限制的宽高比盒子的关键之处：

  


```CSS
.box {
    --ratio: 16/9;
    place-self: center; /* 在其父容器中水平垂直居中 */
    width: min(100vw, var(--ratio) * 100vh);
    height: min(100vh, 100vw / var(--ratio));
}
```

  


注意，上面的代码中并没有使用 `aspect-ratio` 。事实上，上面的代码等同于下面这段代码：

  


```CSS
.box {
    --ratio: 16/9;
    place-self: center; /* 在其父容器中水平垂直居中 */
    width: min(100vw, var(--ratio) * 100vh);
    aspect-ratio: var(--ratio);
}
```

  


简单的解释一下。元素 `.box` 的 `width` 应用了 `min()` 函数，这意味着它的宽度可能是 `100vw` ，也可能是通过视窗高度（`100vh`）与宽高比（`--ratio`）计算出来的。关键是这两个值谁更小，那么 `width` 就用谁。这意味着，`width` 的值要么是 `100vw` ，要么是从视口高度和宽高比计算出来的，以较小的为准。元素的高度（`height`）会根据 `width` 和 `aspect-ratio` 计算出来。这样，我们就获得了想要的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f122cbd294be42a6a5931950ece857fe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1196&h=898&s=662572&e=gif&f=147&b=23363b)

  


> Demo 地址：https://codepen.io/airen/full/NPKrYWL

  


就是这么简单，借助 [CSS Grid 布局技术](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)、[比较函数 `min()`](https://juejin.cn/book/7223230325122400288/section/7241401565653762108) 和[视窗单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)（`vh` ，也可以是 `dvh`），使一个不可替换的元素拥有 `object-fit:contain` 相似的功能。

  


按照同样的逻辑，我们可以结合[容器查询](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)和[容器查询单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-2)，可以对任何元素创建受限制的宽高比。例如下面这个示例：

  


```HTML
<body>
    <div class="container">
        <div class="box"></div>
    </div>
    
    <!-- 省略其他的 container -->
</body>
```

  


具体的 CSS 代码如下：

  


```CSS
:is(html, body, .container) {
    display: grid;
}

html {
    min-height: 100%;
}

body {
    gap: 1rem;
    grid-template-columns: repeat(auto-fit, minmax(min(100% - 2rem, 16em), 1fr));
    font-size: clamp(.625em, 1vw + 1em, 1.5em);
}

.container {
    container-type: size;
    outline: 1px dashed;
    
    .box {
        --ratio: 16 / 9;
        place-self: center;
        width: min(100cqw, var(--ratio) * 100cqh);
        aspect-ratio: var(--ratio);
    }
}
```

## 总结

  


CSS `aspect-ratio` 属性为开发者提供了精确控制宽高比的工具，尤其在响应式设计和复杂布局中显得尤为重要。通过结合替换元素、宽高计算规则以及现代布局方式（如 `flex` 和 `grid`），`aspect-ratio` 能实现许多过去需要额外技巧才能达成的效果，同时提升代码的简洁性和可维护性。
