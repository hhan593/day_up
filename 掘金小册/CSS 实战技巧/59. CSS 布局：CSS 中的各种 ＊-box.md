![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bfb633da89c547698b56c7ac8acff82e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=930&s=727634&e=png)

  


在 CSS 的世界中，它将任何一个元素都被视作为一个**盒子**。因此，在 CSS 中会有很多种关于盒子的概念或术语，比如内联盒子、块盒、盒模型、视觉盒模型等。有趣的是，盒子中又有一些框的概念，比如外边距盒子框（`margin-box`）、边框盒子框（`border-box`）、内距盒子框（`padding-box`）和内容盒子框（`content-box`）。在一些图形元素中，还有填充盒子框（`fill-box`）、描边盒子框（`stroke-box`）和视图盒子框（`view-box`）。看到这些不同命名的盒子框（`*-box`）是不是头都大了。这也是很多 Web 开发者不太喜欢 CSS 的原因之一。

  


事实上呢？你可能从未去了解这些盒子框，但又在平时的开发中不知不觉的应用了这些盒子框的特性。其中最为常见的是在 CSS 中将 `box-sizing` 属性设置为 `border-box` ，从而改变元素尺寸（盒模型）计算方式。除此之外，我们在使用 CSS 样式化 Web UI 时，很多情景下都会应用到这些盒子框中的部分。比如，小册之前很多案例就应用到了盒子框的特性，比如在 `clip-path` 属性中加 `margin-box` 、在 `background-clip` 属性会用到 `padding-box` 等等。

  


接下来，我们就来聊聊这些盒子框，并且通过实例向大家展示，这些盒子框特性给我们带来哪些变化，应该如何使用它们？

  


## CSS 中的盒子框：`*-box`

  


在 CSS 中有一个非常基础且又重要的概念：CSS 世界中的万物都是一个盒子。而每个盒子都会有一个模型，也就是我们平时所说的**盒模型**。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/636f26f55b78479a8b3159dce33e3226~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=1024&s=96844&e=webp&b=fae9cf)

  


每个盒模型都将包含四个限制区域，每个限制区域对应着不同的布局框的范围：

  


-   元素内容会被限制在 `content-box` （内容盒子框）
-   元素内距（`padding`）会被限制在 `padding-box` （内距盒子框）
-   元素边框（`border`）会被限制在 `border-box` （边框盒子框）
-   元素外距（`margin`）会被限制在 `margin-box` （外距盒子框）

  


我用浏览器开发者工具中的盒模型平面图来展示它们之间的关系，大家更熟悉：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/622a98dda58f485fba3f11e8d6835dde~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=321&s=25750&e=webp&b=f9eadc)

  


我想大家对这部分内容应该不会感到陌生，但接下来的内容可能就会有点陌生了。

  


众所周知，CSS 还提供了一些属性，它可以用来绘制图形形状，比如 `clip-path` 、`shape-outside` 属性提供了绘制图形形状的函数，比如 `circle()` （圆形）、`ellipse()` （椭圆形）和 `polygon()` （多边形）等。不为人知的是，这些函数创建的形状都被定位在一个虚拟盒子（Virtual Box）中，也就是形状参考框（Shapes Reference Box）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cdc63897ff8a4af2bb42c44a29db2811~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1326&s=547638&e=jpg&b=fff7f6)

  


同样具有 `margin-box` 、`border-box` 、`padding-box` 和 `content-box` ，但它与我们熟悉的盒模型中的盒子框略有不同，它是用于限制图形的绘制区域：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/09e852b161c446d4ad5959328b9d5fff~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1425&h=1667&s=1490208&e=jpg&b=fef3ef)

  


请注意 `circle()` （右侧图中虚线圆形）绘制的圆变化。有关于这方面更详细的介绍，请移步阅读《[创建不规则 Web 布局](https://juejin.cn/book/7161370789680250917/section/7161625566062313503#heading-17)》！

  


到这并没有结束，我们除了会使用 `clip-path` 绘制图形形状之外，还会直接[在 HTML 中内联 SVG](https://juejin.cn/book/7341630791099383835/section/7344089098363076620) ，通过 [SVG 的图形元素来创建形状](https://juejin.cn/book/7341630791099383835/section/7345813971552698406)，比如 `<circle>` 元素。而且，我们有的时候为了获得一些 UI 效果，会使用 [CSS 变换](https://juejin.cn/book/7288940354408022074/section/7295240572736897064)来对形状（或元素）做一些处理，比如旋转、倾斜、缩放等。在一些场景中，我们需要调整变换原点（`transform-origin`）位置，但[由于 HTML 元素与 SVG 元素采用的坐标系统有所差异](https://juejin.cn/book/7341630791099383835/section/7345677438053810214)，我们在调整 SVG 元素的原点位置时，将会用到 SVG 元素独有的盒子框，比如 `fill-box` （填充盒子框）、`stroke-box` （描边盒子框）和 `view-box` （视图盒子框）。

  


如此一来，CSS 中就包含了：

  


-   应用于盒模型的盒子框：`margin-box` 、`border-box` 、`padding-box` 和 `content-box`
-   应用于形状的参考框：`margin-box` 、`border-box` 、`padding-box` 、`content-box` 、`fill-box` 、`stroke-box` 和 `view-box`

  


这些盒子框可用于不同的 CSS 属性：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4452b9c8068b4fe1b70a2dc0612b704f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3698&h=1870&s=354623&e=png&b=ffffff)

  


  


需要知道的是，盒框应用在不同的 CSS 属性上，将会起到不同的作用和功能。接下来，我们通过一些实际案例来演示其中部分盒子框的功能。

  


## `box-sizing` 改变盒模型尺寸计算

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3080e160dbfb4f128e0371a1b64c873c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=4096&h=2060&s=262496&e=jpg&b=313944)

  


最常见的是重置 `box-sizing` 属性的值为 `border-box` （默认是 `content-box`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9150469716d4f158b03e96533d3a12c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3480&h=1296&s=145534&e=jpg)

  


它会直接影响元素尺寸的计算：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1e59d6edbc0745a9b7798c3258cf4448~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=693&s=26932&e=webp&b=ffffff)

如上图所示，虽然元素的 `width` 显式设置为 `200px` ，但 `box-sizing` 属性值不同时，元素的 `width` 所包含的盒子框会不一样：

  


-   `content-box` ：`width` 是指内容盒子框的宽度，不包括 `padding` 和 `border`
-   `border-box` ：`width` 是指边框盒子框的宽度，包括 `padding` 和 `border`

  


具体的可以参考下图的演变过程：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b2443bbacd684a7696d72839fe86e241~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=1466&s=55352&e=webp&b=fefbfb)

  


这在构建 Web 布局的时候非常有用，例如：

  


```HTML
<div class="container">
    <div class="box"></div>
</div>    
```

  


```CSS
.box {
    width: 100%;
    padding: 1rem;
}

/* 或者 */
.box {
    width: 100%;
    border: 1rem solid;
}
```

  


默认情况之下，`.box` 会溢出其容器 `.container` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/26bec0fae2654b8aadf4563c3bd4fd28~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=418&s=37762&e=webp&b=fdfdfd)

  


但是，将 `.box` 元素的 `box-sizing` 重置为 `border-box` 时，就可以避免这个现象。因此，你在构建 Web 布局时，碰到元素尺寸打破 Web 布局时，首先要排查的是 `box-sizing` 属性的值。

  


## 盒模型的背景（以及它为何有用）

  


我们来看一个更具体，更复杂的案例。你是否曾经想过，如何确保一个（伪）元素的任何部分都不会显示在其父元素的边框盒子（`border-box`）之外？例如下面这个效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fcd3b374e82b402496ba4da07aa3267f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=428686&e=jpg&b=ebebeb)

  


为了增加一点难度，我们加一个限制，每个选项是一个 HTML 元素，我在这里使用的是 `li` 元素：

  


```HTML
<ol class="lists">
    <li style="--slist: #ebac79,#d65b56">Pancake muffin ...</li>
    <!-- 省略其他 li -->
</ol>
```

  


你可能已经发现了，`li` 上有一个使用内联方式定义的 `--slist` ，暂时可以先不用管它，我们后面会聊到它。

  


我们开始进入 CSS 的世界，看看 CSS 是如何实现我们所需要的效果。首先，我们使用 CSS Grid 来构建布局，能更容易确保所有 `li` 宽度一样（在没有设置 `width` 的前提下），并给 `li` 设置一些基本样式：

  


```CSS
.lists {
    --h: 5em;
    display: grid;
    gap: 2rem;
    list-style: none outside none;
    
    
    li {
        display: grid;
        align-items: center;
        height: var(--h);
        font: 1.5em / 1.375 trebuchet ms, verdana, sans-serif;
        background: silver;
        border-radius: calc(1px * infinity);
    }
}
```

  


上面代码并无特殊之处，只是在 `border-radius` 上应用了一个 `infinity` 关键，它表示无穷大，[使得 `li` 保持一种“胶囊”形状](https://juejin.cn/book/7199571709102391328/section/7199845563389444099)，可避免因高度变化来调整 `border-radius` 的值（也有开发者使用一个超大的数值，比如 `99999rem`）。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c14cb9cd55e340aaa3de22120f80797c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=174743&e=jpg&b=eaeaea)

  


在继续编码之前，我们回过头来看目标效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eb837adcf27449f9a6d49bc63fe550c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=428686&e=jpg)

  


每个项（`li`）背景自上而下有三个渐变，每个渐变放置在不同的盒子框中：

  


-   顶部渐变（`--g-top`）：限制在内容盒子框（`content-box`）内
-   中间渐变（`--g-middle`）：限制在内距盒子框（`padding-box`）内
-   底部渐变（`--g-bottom`）：限制在边框盒子框（`border-box`）内

  


简单来说，你可以将这些盒子框想象成嵌套的矩形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ac951663f3894c2790210021e9f8d9fa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2769&h=1651&s=944377&e=jpg&b=ededed)

  


你可能会想，既然使用了多背景，为什么不通过 `background-size` 和 `background-position` 来分层呢？这种思路不是不可以，但这种方法只能生成没有圆角的矩形。即使你对 CSS 的渐变达到炉火纯青的造诣，要使渐变的衔接处平滑，绝不是件易事。

  


然而，使用 `background-clip` 方法时，如果我们设置了 `border-radius` ，背景层就会遵循这个圆角。同时，设置的 `border-radius` 会用于圆滑化边框盒子（`border-box`）的角；同样的半径减去 `border-width` 后，会圆化内距盒子（`padding-box`）的角；再减去内边距（`padding`）后，同样会圆化内容盒子（`content-box`）的角。[这有点类似于元素的嵌套圆角效果](https://juejin.cn/book/7395119173656903715/section/7401099842379841545#heading-1)。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e526473f12df46178ce45ac3fe8ddc40~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=696&s=4309420&e=gif&f=166&b=2c2c2c)

  


> 温馨提示，小册之前的《[CSS 图形：创建多边框效果](https://juejin.cn/book/7395119173656903715/section/7411340113839390783)》和《[CSS 图形：创建渐变边框](https://juejin.cn/book/7395119173656903715/section/7413371655377911835)》都应用到了这个 CSS 技巧！

  


了解完这个之后，我们就可以进一步编写 CSS 代码。为了更好的控制 CSS 样式，我在这里引入一些 CSS 自定义属性，用来存储一些会变化的值：

  


```CSS
.lists li {
    --h: round(5em, 1px); /* 列表项的高度 */
    --g-top: linear-gradient(#dbdbdb, #fff) content-box; /* 顶部渐变，限制在内容盒子框中 */
    --g-middle: linear-gradient(var(--slist)) padding-box; /* 中间渐变，限制在内距盒子框中 */
    --g-bottom: linear-gradient(#fff, #dcdcdc) border-box; /* 底部渐变，限制在边框盒子框中 */
    --g: var(--g-top), var(--g-middle), var(--g-bottom); /* 顺序非常重要 */
    --b: round(.5em, 1px); /* 边框的厚度 */
    --p: round(.25em, 1px);/* 内距大小 */
}
```

  


还记得我们在 HTML 中内联定义的 `--slist` 吗？因为每个列表项的“边框”颜色都不一样，所以使用 `--slist` 来管理每个列表项的中间渐变的颜色。

  


接下来，给 `li` 元素设置一个透明的边框和增加一点内距，并将 `box-sizing` 设置为 `border-box` ，确保透明边框和内距从我们设定的尺寸中被正确扣除。最后，使用 `--g` 替代之前的 `silver` 颜色。需要知道的是 `--g` 叠加了三个渐变：顶部的渐变（`--g-top`）被限制在 `content-box` ；中间的渐变（`--g-middle`）被限制在 `padding-box` ；底部渐变（`--g-bottom`）被限制在 `border-box` 。这意味着 `li` 元素应用了多层背景（渐变），[需要确保它的顺序正确](https://juejin.cn/book/7199571709102391328/section/7199845838103773195#heading-8)，否则会产生不一样的效果。

  


```CSS
.lists li {
    --h: round(5em, 1px); /* 列表项的高度 */
    --g-top: linear-gradient(#dbdbdb, #fff) content-box; /* 顶部渐变，限制在内容盒子框中 */
    --g-middle: linear-gradient(var(--slist)) padding-box; /* 中间渐变，限制在内距盒子框中 */
    --g-bottom: linear-gradient(#fff, #dcdcdc) border-box; /* 底部渐变，限制在边框盒子框中 */
    --g: var(--g-top), var(--g-middle), var(--g-bottom); /* 顺序非常重要 */
    --b: round(.5em, 1px); /* 边框的厚度 */
    --p: round(.25em, 1px);/* 内距大小 */

    box-sizing: border-box;
    height: var(--h);
    border: var(--b) solid #0000; /* 透明边框 */
    padding: var(--p); /* 内距 */
    background: var(--g);
    text-indent: 1em;
}
```

  


我们还设置了 `text-indent: 1em` ，将文本内容往里缩进了 `1em` ，使文本不会紧挨着渐变边框 。这个时候你看到效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a89b01ce18214a4c957fd4e09e630a4f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=333143&e=jpg&b=f3f3f3)

  


再次强调一下，有这个效果的关键之处是 `background-clip` 根据设计需要将渐变限制在对应的布局盒子框中，即 `border-box` 、`padding-box` 和 `content-box` 。

  


接下来，给每个列表项添加编号。按理来说，我们使用的是有序列表 `<ol>` ，它会自带编号。[由于可用于 `li` 的标记符 `::marker` 的 CSS 属性有限](https://juejin.cn/book/7223230325122400288/section/7259668400379527205#heading-24)，加上为了更好的控制布局，[我们把 `li` 的 `display` 重置为 `grid` （默认是 list-item）](https://juejin.cn/book/7199571709102391328/section/7199844993073152038)，使得默认的编号就失去了作用。因此，我在这里选择 [CSS 的计数器（`counter`）](https://juejin.cn/book/7223230325122400288/section/7259668400379527205)，并结合伪元素 `::after` 的 `content` 内容来给列表项进行编号。

  


```CSS
.lists li {
    /* CSS ... */
    counter-increment: c;
    
    &::after {
        content: counter(c, decimal-leading-zero);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8c51d4fca0d403082ff17b84595ae65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=339641&e=jpg&b=f3f3f3)

  


现在编号有了，我们还需要对它进行样式化。首先，需要在 `li` 使用 `grid-template-columns` 将文本内容与编号排成一行：

  


```CSS
.lists li {
    display: grid;
    grid-template-columns: minmax(0, 1fr) auto;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e24917746b04f7db36f2162a61c1333~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2484&h=1334&s=279166&e=jpg&b=f2f2f2)

  


然后，将 `::after` 伪元素设置为一个正方形，其边长等于列表项的高度（`--h`）减去上下边框厚度（各为 `--b`），即 `calc(var(--h) - var(--b) * 2)` ，并把它存储到一个新的 CSS 变量中，比如 `--d` 。与此同时，将 `border-radius` 设置为 `50%` ，使它成为一个圆形：

  


```CSS
.lists li {
    --d: calc(var(--h) - var(--b) * 2);
    
    &::after {
        width: var(--d);
        aspect-ratio: 1;
        border-radius: 50%;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/454c7addc5164043a6d707cb641ce5a4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=390469&e=jpg&b=f2f2f2)

  


我们使用 CSS Grid 布局，将“数字编号”在绿色的圆形中水平垂直居中：

  


```CSS
.lists li {
    &::after {
        display: grid;
        place-content: center;
        text-indent: 0; /* 或者 unset */
    }
}
```

  


请注意，不要忘记将 `text-indent` 重置为 `0` 或 `unset` ，因为我们在 `li` 中设置了 `text-indent: 1rem` 将文本往里缩进 `1rem` ，它是一个可继承属性，会使伪元素 `::after` 的文本也往里缩进 `1rem` ，所以要将 `text-indent` 重置为 `0` 或 `unset` 。

  


另外，我们让伪元素 `::after` 继承父元素（`li`）的 `box-sizing` 和 `border` 属性，并为其背景设置类似于父元素的样式：

  


```CSS
.lists li {
    &::after {
        box-sizing: inherit;
        border: inherit;
        background: var(--g-middle), var(--g-bottom);
        color: #fff;
        box-shadow: inset 0 0 1px 1px #efefef, inset 0 calc(-1 * var(--b)) rgb(0 0 0 / 0.1);
    }
}
```

  


你可能已经注意到了代码中的 `box-shadow` ，[它给伪元素 `::after` 设置了一个内阴影](https://juejin.cn/book/7199571709102391328/section/7199844993455325216)，使其看起来更有质感（原本效果就有一个看似立体的效果）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce3725f028c244e4aae3355c01d20900~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=388417&e=jpg&b=f2f2f2)

  


> 温馨提示，上面代码中提到的 `unset` 和 `inherit` 是 CSS 显式默认值，如果你对这方面知识不太了解的话，建议你花几分钟时间阅读《[CSS 显式默认值：`inherit`，`initial`，`unset` 和 `revert`](https://juejin.cn/book/7223230325122400288/section/7232094160071688227)》！这些属性值能在一些场景中能使你减少很多不必要的麻烦，并且使 CSS 代码更易于维护。

  


到目前为止，整个效果看起来有点模样了！但我们仍然需要对 `::after` 的样式进行一些调整。我们需要将伪元素向上和向右移动一点位置，使它与父元素的边框相互吻合。我们可以使用负的 `margin` 来实现它：

  


```CSS
.lists li {
    &::after {
        margin-top: calc(-1 * var(--p));
        margin-right: calc(-1 * var(--p));
    }
}
```

  


这就是列表项编号的全部样式了：

.lists {

我们离最终效果更近了一步：

接下来的这部分有一点难度了。我们新增伪元素 `::before` ，用它来创建“编号”底下的一层：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/609d68d1ac6341ff87402fc3e32911b7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=451107&e=jpg)

  


我们使用绝对定位，将它放置在 `::after` 底部，并将其设置为一个正方形，其边长等于父元素的高度（`--h`）：

  


```CSS
.lists li {
    position: relative;
    
    &::after {
        z-index: 3;
    }
    
    &::before {
        content: "";
        position: absolute;
        right: calc(-1 * var(--b));
        top: calc(-1 * var(--b));
        width: var(--h);
        aspect-ratio: 1;
        background: color-mix(in srgb, #90acef, #0000 50%);
        z-index: 2;
    }
}
```

  


注意，代码中的 `background` 是一个临时值，[它 `color-mix()` 函数创建的带有透明度的颜色](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)，便于我们检查它们的对齐情况：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd5c94eade754468887487847490d4bb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=398545&e=jpg&b=f2f2f2)

  


好了，现在我们给这个 `::before` 伪元素添加一个临时的渐变背景，旋转它，然后给它添加圆角和阴影：

  


```CSS
.lists li {
    &::before {
        border-radius: var(--b);
        rotate: 45deg;
        box-shadow: 0 0 7px rgb(0 0 0 / .2);
        background: linear-gradient(-45deg, orange, purple);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8f668884d804e54978e636f96685606~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=457553&e=jpg)

  


你可能发现了，伪元素 `::before` 旋转之后，它的渐变方向也随之改变了，而且还溢出了 `li` 容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/107968f7163f44b7a4389855e29e28c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=881&s=384153&e=jpg&b=f3f3f3)

  


这个时候，问题就来了：如何确保旋转之后，伪元素的渐变还是我们所需要的渐变效果呢？在解决这个问题，我们需要在 `::before` 伪元素的渐变中设置渐变颜色的停止位置，使其与父元素的顶部和底部边缘对齐。换句话说，我们需要对伪元素 `::before` 的渐变背景做一些限制。

  


由于我们将 `::before` 伪元素旋转了 `45°` ，它的左上角现在指向上方；相反，它的右下角指向下方。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd40ce19ae324e3e8b690e16cfbfcf64~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1499&h=541&s=270790&e=webp&f=18&b=fefdfd)

  


指向正方形（`::before`）左上角的渐变是在 `-45°` 方向上的渐变（因为 `0°` 角位于 `12` 点钟位置，正方向与旋转一样是顺时针方向）。指向某个角的渐变意味着 `100%` 的位置位于该角上。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8bcfd3b84e2a402e83af86278ae05256~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1499&h=633&s=573350&e=webp&f=20&b=fdfbfb)

  


注意，渐变线的 `50%` 点位置总是通过渐变框的中点（即对角线交点）。而渐变框是用于绘制渐变的区域，其大小由 `background-size` 指定。由于我们没有设置 `background-size` ，渐变的默认行为是由 `background-origin` 定义的整个框，而 `background-origin` 的默认值是 `padding-box` 。也就是说，渐变的 `background-size` 大小与 `padding-box` 框大小相等。由于我们的 `::before` 伪元素没有设置 `border` 和 `padding` ，因此三个框（`content-box` 、`padding-box` 和 `border-box`）之间没有间距，并且它们的比例与渐变框相等。

  


在我们的示例中，垂直于指向 `-45°` 的渐变线方向的线包括：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fa2ee3cbd02c451281d7ea535336e00b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=565&s=10700&e=webp&b=fefcfc)

  


-   渐变线的 `0%` 位于伪元素 `::before` 的右下角
-   伪元素的父元素 `li` 的底边对应渐变范围的下边界
-   渐变线的 `50%` 位于父元素 `li` 的中线。沿对角线将正方形（`::before`）分成两个对称的直角等腰三角形。由于伪元素 `::before` 和 `li` 元素相互对齐，这条渐变线也是 `li` 的中线，将 `li` 分成两半，每半的高度等 `li` 元素高的一半
-   伪元素的父元素 `li` 的顶边对应渐变范围的上边界
-   渐变线的 `100%` 位于伪元素 `::before` 的左上角

  


这意味着，我们需要将 `::before` 伪元素上指向 `-45°` 的渐变限制在 ` calc(50% - var(--h)  ``/ 2``)` （对应 `li` 元素的底边）和 ` calc(50% + var(--h)  ``/ 2``)` （对应 `li` 元素的顶边）之间：

  


```CSS
.lists li {
    &::before {
        /* 调整前 */
        background: linear-gradient(-45deg, orange, purple);
        
        /* 调整后 */
        background: linear-gradient(
            -45deg, 
            orange calc(50% - var(--h) / 2), 
            purple calc(50% + var(--h) / 2)
        );
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/255af4ecf4cc4f98a020384b824ff000~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=881&s=315609&e=jpg&b=f3f3f3)

  


如果上图不够明显展示渐变调整前后的差异，我们可以在这些停止点位置添加从透明到颜色的清晰过渡，可以更明显的看出调整后的渐变是我们所需要的：

  


```CSS
.lists li {
    &::before {
        background: linear-gradient(
            -45deg, 
            #0000 calc(50% - var(--h) / 2), 
            orange 0,
            purple calc(50% + var(--h) / 2),
            #0000 0
        );
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f0982248f90649648b331d54fabb4114~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=443582&e=jpg&b=f2f2f2)

  


> 温馨提示，如果你对于渐变的背后工作原理不是太熟悉的话，个人建议你花点时间阅读《[你不知道的 CSS 渐变](https://juejin.cn/book/7199571709102391328/section/7199845781149810727)》！

  


现在，我们解决了第一个问题，渐变颜色位置是我们所需要的位置。接下来要解决第二个问题——限制伪元素本身！

  


同样是因为伪元素 `::before` 旋转了 `45°` ，使其溢出了父元素 `li` 。你可能会说，这个简单，只需要在 `li` 元素上设置 `overflow: hidden` 即可！真的吗？我们不妨尝试一下：

  


```CSS
.lists li {
    overflow: hidden;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b88528bd083047a5857d57124eb1f266~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1210&h=680&s=1481349&e=gif&f=148&b=f0efef)

  


很明显，这不是我们所需要的效果。

  


造成这个现象是因为 `overflow: hidden` 会裁剪掉超出元素 `padding-box` 之外的所有内容，但我们只需要裁剪掉超出 `border-box` 的部分。在我们这个示例中，`border-box` 比 `padding-box` 大，因为我们有一个非零的边框，而我们无法去掉这个边框（通过使用 `border-box` 等于 `padding-box` 来解决问题），因为我们需要在 `li` 元素应用三层背景：

  


-   顶部渐变（`--g-top`）限制在 `content-box` 区域
-   中间渐变（`--g-middle`）限制在 `padding-box` 区域
-   底部渐变（`--g-bottom`）限制在 `border-box` 区域

  


这个时候，[我们就要将 `clip-path` 搬出来啦](https://juejin.cn/book/7223230325122400288/section/7259668885224456252)。我们可以通过 `clip-path` 属性的 `inset()` 函数来裁剪溢出部分。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8dc76b8cbac34500ac5dd4c2071c2c40~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2438&h=1254&s=211029&e=jpg)

  


`inset()` 函数可以由顶部、右侧、底部和左侧边缘的距离裁剪出一个矩形。在我们这个示例中，裁剪的边缘就是 `border-box` 的边缘：

  


```CSS
.lists li {
    clip-path: inset(0);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5dca33e6133047f1a7c278b1ea07642f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=430698&e=jpg&b=f2f2f2)

  


正如你所看到的，伪元素 `::before` 溢出父元素 `li` 的部分被裁剪掉了，但裁剪出来的不是一个圆形。庆幸的是，`inset()` 函数还接受另一个参数，它可以接受任何我们想要的 `border-radius` 的值：

  


```CSS
.element {
    clip-path: inset(20px round 10px); /* 四个角的圆角半径都是 10px */
}

.element {
    clip-path: inset(20px 50px round 10px 5px); /* 左上角=右下角=10px, 右上角=左下角=5px */
}

.element {
    clip-path: inset(20px 50px 80px round 10px 5px 20px); /* 左上角 = 10px, 右上角 = 左下角= 5px, 右下角 = 20px*/
}

.element {
    clip-path: inset(20px 50px 80px 100px round 10px 5px 20px 30px); /* 左上角 = 10px, 右上角 = 5px，右下角 = 20px, 左下角 = 30px */
}
```

  


但在我们这个示例，不需要那么复杂，只要设置一个与父元素 `border-radius` 相等的半径值即可：

  


```CSS
.lists li {
    clip-path: inset(0 round calc(1px * infinity));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/62e6ec2f006f48f4b09f9cf08170724f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=433067&e=jpg&b=f2f2f2)

  


现在已经很接近我们所要的效果了。我们只需要给 `li` 网格添加一点间距（`gap` ），避免伪元素 `::before` 遮盖住文本内容：

  


```CSS
.lists li {
    column-gap: calc((cos(45deg) * var(--h)) - var(--d) / 2);
}
```

  


还有一点需要知道，我们在 `li` 上设置 `clip-path` ，它也会裁剪掉应用在 `li` 元素上的 `box-shadow` 。我们可以在 `.lists` 上应用 `filter` 的 `drop-shadow()` 来给 `li` 添加阴影：

  


```CSS
.lists {
    filter: drop-shadow(0 calc(2 * var(--b, round(0.5em, 1px))) calc(2 * var(--b, round(0.5em, 1px))) rgb(0 0 0 / .2));
}
```

## 改变变换参考点：`transform-box`

  


之前提到过，每一个元素都有一个本地坐标系统，CSS 的 `transform-origin` 可以用来指定元素的坐标系统的原点。对于 HTML 元素，`transform-origin` 属性的默认值为元素参考框的 `50% 50%` （元素边框框 `border-box` 的正中心位置）。

  


SVG 元素则有所不同，它们的坐标系统的默认 `transform-origin` 是其参考框（SVG 的 `viewBox`）的 `(0,0)` 位置，通常是 SVG 画布的左上角。

  


SVG 元素在应用变换属性 `transform` 或 CSS 变换 `transform` 、`translate` 、`rotate` 和 `scale` 时，都会围绕着 `transform-origin` 指定的参考点进行。不过，在某些情况下，SVG 中使用相对值（例如 `50% 50%` ）设置 `transform-origin` 时，其行为可能与预期不符。例如下面这个示例：

  


```XML
<svg class="transform" viewBox="-512 -512 1024 1024">
    <g stroke="lime" stroke-width="5" stroke-dasharray="30 10">
        <line x1="-512" y1="-2" x2="512" y2="-2" />
        <line y1="-512" x1="-2" y2="512" x2="-2" />
    </g>
    
    <circle cx="0" cy="0" r="424" fill="none" stroke="lime" stroke-width="2" stroke-dasharray="30 10" />
  
    <g>
        <rect class="rotate" x="-400" y="-400" width="200" height="200" stroke="orange" stroke-width="8" fill="none" />
        <circle cx="-300" cy="-300" r="10" fill="orange" />
    </g>

    <line x1="0" y1="0" x2="-300" y2="-300" stroke="orange" stroke-width="4" stroke-dasharray="30 10" class="rotate fill-box" />

    <circle cx="0" cy="0" r="20" fill="lime" />
</svg>
```

  


上面代码，通过改变 `viewBox` 的 `<min-x>` 和 `<min-y>` 将用户坐标系统 `viewBox` 移动到 SVG 画布（视口坐标系统）的中心位置（下图中绿色圆点位置）。在画布的左上角绘制了一个 `200 x 200` 的矩形（橙色矩形），并在这个矩形中正中心绘制了一个小圆点（模拟橙色矩形框的中心位置）。

  


```CSS
.rotate {
    transform-origin: 50% 50%;
    transform: rotate(var(--_a));
}
```

  


你会发现，即使 `transform-origin` 属性的值被设置为 `50% 50%` ，但矩形框在旋转时，并未围绕着矩形框自身中心旋转（橙色点）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c14361b65e9a423d82790657e3e29fd8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1058&h=636&s=1745943&e=gif&f=259&b=030303)

这并不是我们想要的结果。

  


我们可以通过 CSS 的 `transform-box` 属性来修复它。HTML 元素的默认变换参考框值是 `border-box`，而 `fill-box` 、`stroke-box` 和 `view-box` 更适用于 SVG 元素。其中 `fill-box` 和 `stroke-box` 的行为分别类似于 `content-box` 和 `border-box`。

  


在对 SVG 元素进行动画时，将 `transform-box` 设置为 `fill-box` 或 `view-box` 可以提供更平滑、更可预测的动画效果，特别是与 `transform-origin` 结合使用时。例如，上面小熊的示例，在应用变换和动画的元素上，显式设置 `transform-box` 为 `fill-box` ，一切都将变得完美：

  


```CSS
:is(.pupils, .bowtie){
    transform-box: fill-box;
}
```

正如你所看到的，现在它只是一个汉堡菜单图标。我们期望的效果是，当用户点击该图标时，它将变成一个关闭图标，再次点击时，它又回到汉堡菜单图标。

  


要实现这个效果，需要借助一点 JavaScript 脚本。例如，用户点击按钮时，可以切换它的类名。不过，在这个示例中没有切换类名，而是切换 `aria-pressed` 属性。即 `aria-pressed` 属性的值在 `true` 和 `false` 之间切换：

  


```JavaScript
const TOGGLE = document.querySelector('.button');

const HANDLE_TOGGLE = () => {
    TOGGLE.setAttribute('aria-pressed', TOGGLE.matches('[aria-pressed=true]') ? false : true);
}

TOGGLE.addEventListener('click', HANDLE_TOGGLE);
```

  


当按钮（`button`）的 `aria-pressed` 属性值为 `true` 时，需要调整SVG 的 `<rect>` 元素位置。即使用 CSS 变换来调整它们的位置：

  


```CSS
[aria-pressed="true"] rect {
    transform-origin: 50% 50%;
    
    &:nth-of-type(1) {
        translate: 0 333%;
        rotate: -45deg;
    }
    
    &:nth-of-type(2) {
        rotate: 45deg;
    }
    
    &:nth-of-type(3) {
        translate: 0 -333%;
        rotate: 45deg;
    }
}
```

请别遗忘了 `transform-box` 属性。请记住，在 SVG 变换中，即使你将元素的 `transform-origin` 属性的值设置为 `50% 50%` （设置变换原点为元素中心位置），还是不够的。因为它的参考框默认是 `view-box` （即 SVG 的 `viewBox`）。我们需要将 `transform-box` 属性值设置为 `fill-box` ，将参考框指定为图形对象边框框：

正如你所看到的，用户点击按钮时，“汉堡菜单图标”和“关闭图标”可以正常来回切换了。只不过，效果看上去有点生硬。我们可以使用 CSS 的 `transition` 给它们添加过渡动画。其中的关键之处，要为两种状态定义过渡延迟：

  


-   当按钮的 `aria-pressed` 属性值为 `true` 时，先平移线条（`<rect>` ），然后旋转它们
-   当按钮的 `aria-pressed` 属性值为 `false` 时，先旋转线条（`<rect>`），然后将它们平移到原始位置

  


```CSS
rect {
    [aria-pressed="false"] & {
      transition: rotate 0.2s 0s, translate 0.2s 0.3s;
    }
    
    [aria-pressed="true"] & {
      transition: translate 0.2s 0s, rotate 0.2s 0.3s;
    }
}
```

  


与此同时，按钮 `aria-pressed` 为 `true` 时，也给 `<svg>` 元素添加一个旋转的过渡效果：

  


```CSS
[aria-pressed="true"] {
    svg {
        rotate: 90deg;
        transition: rotate 1s 0.4s;
    }
}
```

## 小结

  


在这节课中，我们一起探讨了 CSS 中的各种 `*-box` ，尤其是 `border-box` 、`padding-box` 和 `content-box` ，因为这几个属性值很常见。它们时常出现在 `background-clip` （或 `mask-clip`）和 `background-origin` （或 `mask-origin`）中。可以帮助我们制作很有趣的效果，比如渐变边框。

  


当然，其他的 `*-box` 也非常有用，比如适用于 SVG 元素变换的 `fill-box` 和 `view-box` ，可用于异形布局的 `margin-box` 等。简而言之，在一些特定的场景中，我们离不开这些 `*-box` 。
