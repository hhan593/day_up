## 循环切换

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a2264cd8da41414d91586a3ac14cba9f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=754796&e=png&b=0c1932)

  


我们还是从实际需求出发，就以 Web 上常见的 `Alert` 或 `Notification` UI 组件为例，根据状态它会有多种不同的皮肤（或主题）效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3ebe9f832bf248cd97f01ff1d48434fd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=4096&h=2180&s=1226946&e=jpg&b=fef8f6)

  


[在 CSS 中有很多种方式可以实现该效果](https://juejin.cn/book/7395119173656903715/section/7445265166482505778)，但在这里我将向大家展示一些更高级的 CSS 方案。在开始之前，我们先从“循环切换”的概念开始。

  


课程开头提到过，循环切换是使用 CSS 自定义属性（CSS 变量）实现的一种巧妙技巧，它允许我们在多个状态之间循环切换。简单地说，循环是指一个变量引用了它自己，无论是直接引用还是间接引用。最简单的循环是 `--foo: var(--foo)` ，但它们可以有任意多个步骤，例如：

  


```CSS
:root {
    --a1: var(--a2);
    --a2: var(--a3);
    --a3: var(--a1);
}
```

  


然而，熟悉 CSS 自定义属性的同学会说：“当自定义属性相互引用时，循环中的所有自定义属性在计算值时都是无效的”。而且[规范也是这样描述](https://www.w3.org/TR/css-variables-1/#cycles)的：

  


> 这可能会创建循环依赖，其中一个自定义属性使用一个引用自身的 `var()` ，或者两个或更多自定义属性彼此相互引用。如果依赖中存在循环，循环中的所有自定义属性在计算值时都是无效的。

  


但是，[规范中还有另一段描述](https://www.w3.org/TR/css-variables-1/#using-variables)：

  


> 在 `var()` 中，逗号分隔符（`,`）后面没有任何内容时，必须将其视为有效，表示备用值为空。

  


也就是说，如果传递给 `var()` 函数的初始值丢失或无效，但后面紧跟着一个逗号（`,`），则计算值是一个有效的备用值，只是这个备用值为空。例如：

  


```CSS
div {
    --my-var: var(--missing-var,);
}
```

  


利用该特性，我们可以在所有变量中产生循环（除了一个变量）：

  


```CSS
.alert {
    --variant: var(--variant--default);
    --variant--default: var(--variant,);
    --variant--success: var(--variant,);
    --variant--warning: var(--variant,);
    --variant--danger: var(--variant,);
    
    background: 
        var(--variant--default, lavender)
        var(--variant--success, palegreen)
        var(--variant--warning, khaki)
        var(--variant--danger, lightpink);
}
```

  


上面的代码会给 `.alert` 元素设置一个默认的背景，即 `lavender` 。如果我们将 `--variant: var(--variant--success)` 设置在这个元素上，不管是通过 HTML 内联样式还是通过 CSS 中的状态或查询覆盖，我们都会看到 `.alert` 元素的背景变成浅绿色（`palegreen`）。

  


然后我们像这样使用：

  


```CSS
.alert--success {
    --variant: var(--variant--success);
}

.alert--warning {
    --variant: var(--variant--warning);
}

.alert--danger {
    --variant: var(--variant--danger);
}
```

  


这种方法的一个缺点是，由于 `--variant--success`、`--variant--warning` 和 `--variant--danger` 等变量背后的值是特定于 `--variant` 变量的，它们需要命名空间，以避免冲突。

  


另外，这种方式不仅限于单个属性，可以设置任意多个属性，并且同样可以在子元素或伪元素应用类似的逻辑。在这里，我们以同样的方式调整 `border-color` 和 `color` 属性的值：

  


```CSS
.alert {
    --variant: var(--variant--default);
    --variant--default: var(--variant,);
    --variant--success: var(--variant,);
    --variant--warning: var(--variant,);
    --variant--danger: var(--variant,);

    background: 
        var(--variant--default, lavender)
        var(--variant--success, palegreen) 
        var(--variant--warning, khaki)
        var(--variant--danger, lightpink);
    border-color: 
        var(--variant--default, #cccdd0)
        var(--variant--success, #a4dc9a) 
        var(--variant--warning, #e7cc6a)
        var(--variant--danger, #d97a77);
    color: 
        var(--variant--default, #434653)
        var(--variant--success, #215112) 
        var(--variant--warning, #6d360f)
        var(--variant--danger, #7b2631);
}

.alert--success {
    --variant: var(--variant--success);
}

.alert--warning {
    --variant: var(--variant--warning);
}

.alert--danger {
    --variant: var(--variant--danger);
}
```

  


样就获得了不同变体的 `Alert` ：

你是不是再一次感受到了 CSS 自定义属性的魔力，甚至说从未想过这样使用 CSS 自定义属性。但如果你尝试跟随逻辑，你可能会注意到一个关键点——**循环依赖**。

  


来看我们第一段代码示例中的前两条规则：

  


```CSS
.alert {
    --variant: var(--variant--default);
    --variant--default: var(--variant,);
}
```

  


我们把 `--variant` 赋值给 `--variant--default` ，然后又把 `--variant--default` 赋值给 `--variant` 。这不是无效的吗？的确如此！当我们像这样交叉引用自定义属性时，这两个变量都会变成“**保证无效的值**”。

  


正如我们之前所提到的，这也是符合自定义属性规范的定义：

  


> 这可能会创建循环依赖，例如一个自定义属性通过 `var()` 引用自身，或者两个及以上的自定义属性相互引用。
>
> 如果依赖图中存在循环，则所有涉及的自定义属性在计算值阶段都会变为无效。

  


然而，这正是我们想要的效果。来看看我们剩下的自定义属性定义：

  


```CSS
.alert {
    --variant--success: var(--variant,);
    --variant--warning: var(--variant,);
    --variant--danger: var(--variant,);
}
```

  


此时，`--variant` 是无效的，因此这三个变量应该跳转到它们的回退值。但你可能会注意到，这里的回退值只是一个空值，逗号（`,`）后面什么内容都没有。

  


在自定义属性的回退值中，我们可以**省略空格**，仍然获得相同的效果。这使得该技术更加健壮，因为它不会依赖空格，从而更安全地适应 CSS 压缩工具。正如规范所述：

  


> 逗号（即后面没有任何内容的 `,`），在 `var()` 中被视为有效，表示一个空回退值！

  


也就是说，`var(--variant,)` 是一个合法的函数，如果 `--variant` 是无效的或缺失的，则 `var()` 应该被替换为空。

  


至此，`--variant--default` 由于交叉引用 `--variant` 而变为无效，而其他变量则是有效的“空值”。

  


最终，我们来看 CSS 规则中的常规属性声明，以示例中的 `background` 为例：

  


```CSS
.alert {
    background: 
        var(--variant--default, lavender)
        var(--variant--success, palegreen) 
        var(--variant--warning, khaki)
        var(--variant--danger, lightpink);
}
```

  


这里有三个变量（即 `--variant--success` 、`--variant--warning` 和 `--variant--danger`）不会产生任何内容（因为它们的值是“空”），但它们仍然是有效折，所以不会跳转到回退值。而 `--variant--default` 是无效的，因此它会跳转到回退值 `lavender` 并应用。因此 `.alert` 元素的默认背景颜色（`background`）是 `lavender` 。

  


当我们在 `.alert--success` 元素中将 `--variant` 重新赋值，比如：

  


```CSS
.alert--success {
    --variant: var(--variant--success);
}
```

  


那么 `--variant--default` 就会变为空，而 `--variant--success` 则变为无效，这样，我们的 `background` 属性的值就会从一值（`lavender`）切换到另一个值（`palegreen`）。因此，`.alert--success` 的背景颜色变成了 `palegreen` 。依此类推，我们就获得了不同变体的 `Alert` 。

  


这就是“循环切换”的整个运作方式！简单地说，就是利用**循环依赖+无效变量+空回退值**，我们可以巧妙地通过切换一个变量来控制多个状态，从而实现高效、灵活的 CSS 变量切换方案。

  


除了可以切换单一变量之外，使用这个机制还可以进行多重切换。例如下面这个示例：

  


```CSS
.toggle {
    --is--round: var(--toggle,);
    --is--pink: var(--toggle,);
    --is--shadow: var(--toggle,);
    
    border-radius: var(--is--round, 50%);
    background: var(--is--pink, pink);
    box-shadow: var(--is--shadow, 0 0 .25em .125em rgb(0 0 0 / .5));
}
```

  


```HTML
<div class="toggle" style="--toggle: var(--is--pink)"></div>
<div class="toggle" style="--toggle: var(--is--round)"></div>
<div class="toggle" style="--toggle: var(--is--shadow)"></div>
<div class="toggle" style="--toggle: var(--is--pink) var(--is--round)"></div>
<div class="toggle" style="--toggle: var(--is--pink) var(--is--shadow)"></div>
<div class="toggle" style="--toggle: var(--is--round) var(--is--shadow)"></div>
<div class="toggle" style="--toggle: var(--is--pink) var(--is--round) var(--is--shadow)"></div>
```

通过为我们的循环切换指定多个值，我们使得每个值都变为无效，从而允许通过一个自定义属性一次切换多个空间切换。这是多么酷的功能！

  


## 空间切换

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2208c18fdca64d67af25b9f6c7384376~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=758874&e=png&b=0b162d)

  


循环切换的主要特点是，对于每个值，它们产生了一个常规则的“空间切换”：**在每个时刻，每个变量要么无效，要么为空**。

  


甚至切换变量本身也可以是一个空间切换：我们在上面的示例中有一个默认值，但如果我们将其设置为空呢？那么它将变为有效，并将其值传递给所有依赖于它的变量，这样就会使循环切换和所有值都为空，从而允许基于任何一个变量创建逻辑。

  


我们几乎可以为我们的空间切换创建逻辑门：我们可以使用逻辑来返回最终值，但主要问题在于，由于无法反转存储在空间切换中的“位”，我们无法创建一个自身输出空间切换的逻辑门。

  


我们来看一个简单的示例，使用我们的逻辑门来设置每个单元格的背景颜色：

在继续往下之前，这里简单介绍一下 `AND` 、`NAND` 、`OR` 、`NOR` 、`XOR` 和 `XNOR` 。这些是逻辑运算（逻辑门），用于处理二进制 （`0` 和 `1`）逻辑。它们是计算机硬件、布尔代码和编程中重要的基本概念。

  


-   **`AND`** **（与门）** ：只有当两个输入都为 `1` 时，输出才是 `1` ，否则输出则为 `0`
-   **`NAND`** **（与非门）** ：先执行 `AND` ，然后取反。它的特点是，只有两个输入都为 `1` 时，输出才是 `0`，否则都是 `1`
-   **`OR`** **（或门）** ：只要有一个输入是 `1` ，输出就是 `1` ；当两个输入都是 `0` 时，输出才是 `0`
-   **`NOR`** **（或非门）** ：先执行 `OR` ，然后取反。只有两个输入都为 `0` 时，输出才是 `1` ，否则输出都是 `0`
-   **`XOR`** **（异或门）** ：当两个输入不同时，输出是 `1` ；当两个输入相同时，输出则是 `0`
-   **`XNOR`** **（异或非门）** ：先执行 `XOR` ，然后取反。当两个输入相同时，输出是 `1` ；当两个输入不同时，输出是 `0`

  


需要知道的是，在 CSS 中我们通过自定义属性可以实现这些逻辑运算的功能，但在这节课中，不会深入的展开。不过，小册后面有一节课，我们会专门探讨这方面的知识。敬请期待！

  


我们继续回到上面的示例中来。

  


在“逻辑门”的概念中，我们可以创建一个“缓冲器”，但不能创建“反转器”。然而，对于“缓冲器”，我们可以为条件的两个分支定义结果，从而利用“非”分支。其工作方式如下：

  


```CSS
.cell--A {
    --TRUE: palegreen;
    --FALSE: lightpink;
    --NOT-A: var(--A) var(--FALSE);
    background: var(--A, var(--TRUE)) var(--NOT-A,);
}
```

  


我们可以看到，为了同时拥有两个值，我们需要声明一个中间变量，并在其中存放 `--FALSE` 的值。我们可以手动计算 `--A` 处于两种状态时的情况：

  


当 `--A` 无效时：

  


-   `--NOT-A` 变量由于其声明包含了一个必定无效的值，因此在计算值阶段变为“无效”
-   `var(--A, var(--TRUE))` 使用回退值，输出 `--TRUE` 的值
-   `var(--NOT-A,)` 使用空回退值，输出为空
-   结合这两者，我们得到 `var(--TRUE)` 和空值，最终结果为 `var(--TRUE)`
-   最终，我们得到 `palegreen` （淡绿色）

  


当 `--A` 为空时：

  


-   `--NOT-A` 变量变为空，并使用 `var(--FALSE)` ，返回 `--FALSE` 的值
-   `var(--A, var(--TRUE))` 不使用回退值，直接输出为空
-   `var(--NOT-A,)` 获取 `--NOT-A` 的值，即 `--FALSE`
-   最终，我们得到 `lightpink` （淡粉红）

  


我们无法在这里使用“反转器”，因为我们必须为 `--FALSE` 和 `--TRUE` 都使用有效值，作为一个缓冲区，以防止出现问题。当这些值是有效的，比如用于背景颜色时，一切都能正常运行。但是，如果 `--FALSE` 或 `--TTRUE` 代表的是两种不同的空间切换值，又会发生什么呢？

  


要实现一个“反转器”，我们需要将 `--TRUE` 设为空，并将 `--FALSE` 设为无效值。让我们看看在这种情况下，代码会发生什么变化：

  


```CSS
.cell-A {
    --TRUE: ;         /* 空 */
    --FALSE: initial; /* 无效 */

    --NOT-A:  var(--A) var(--FALSE);
    --OUTPUT: var(--A, var(--TRUE)) var(--NOT-A,);
}
```

  


当 `--A` 无效时：

  


-   `--NOT-A` 由于 `--A` 和 `--FALSE` 都是无效值，因此返回无效
-   `var(--A, var(--TRUE))` 使用回退值，输出 `--TRUE` 的值，而 `--TTRUE` 为空
-   `var(--NOT-A,)` 使用空回退，输出为空
-   结果是`--OUTPUT` 中什么都没有

  


当 `--A` 为空时：

  


-   `--NOT-A` 仍然是无效的，因为 `--FALSE` 是无效值，使得整个声明在计算值阶段变得无效
-   `var(--A,var(--TRUE))` 不使用回退值，直接输出 `--A` 的值，而 `--A` 为空
-   `var(--NOT-A,)` 试图获取 `--NOT-A` 的值，但正如我们所发现的，它是无效的，因此使用回退值，最终输出为空
-   结果是两个部分最终都是空值

  


无论我们在这个计算公式中放入什么，最终都会得到空值，这使得创建一个“反转器”变得不可能。

  


我们也可以为所有其他逻辑门使用有效的 CSS 值，但对于空间切换却行不通。正如上面示例中的 `XOR` ，我们可以将逻辑门串联起来，但这会导致在多个分支中重复相同的值。

  


```CSS
.cell--XOR {
    --A-AND-B: var(--A, var(--B));
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--FALSE);
    --A-NAND-B:
        var(--A-AND-B)
        var(--A-OR-B, var(--TRUE))
        var(--A-NOR-B,);
    background:
        var(--A-AND-B, var(--A-OR-B, var(--FALSE)))
        var(--A-NAND-B,);
}
```

  


另外，`NOR` 和 `XNOR` 都是“伪”的，因为我们只是根据不同情况翻转所有使用的值：

  


```CSS
.cell--NOR {
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--TRUE);
    background:
        var(--A-OR-B, var(--FALSE))
        var(--A-NOR-B,);
}

.cell--XNOR {
    --A-AND-B: var(--A, var(--B));
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--TRUE);
    --A-NAND-B:
        var(--A-AND-B)
        var(--A-OR-B, var(--FALSE))
        var(--A-NOR-B,);
    background:
        var(--A-AND-B, var(--A-OR-B, var(--TRUE)))
        var(--A-NAND-B,);
}
```

  


这种方法适用于常规值。

  


了解了这些之后，我们对上面示例的代码做一个简单的解释：

  


```CSS
.table {
    --_: var(--_);
    --ON: var(--_);
    --OFF: var(--_,);
    --TRUE: palegreen;
    --FALSE: lightpink;
}

.cell--A {
    --NOT-A: var(--A) var(--FALSE);
    background:
        var(--A, var(--TRUE))
        var(--NOT-A,);
}

.cell--B {
    --NOT-B: var(--B) var(--FALSE);
    background:
        var(--B, var(--TRUE))
        var(--NOT-B,);
}

.cell--AND {
    --A-AND-B: var(--A, var(--B));
    --A-NAND-B: var(--A-AND-B) var(--FALSE);
    background:
        var(--A-AND-B, var(--TRUE))
        var(--A-NAND-B,);
}

.cell--NAND {
    --A-AND-B: var(--A, var(--B));
    --A-NAND-B: var(--A-AND-B) var(--TRUE);
    background:
        var(--A-AND-B, var(--FALSE))
        var(--A-NAND-B,);
}

.cell--OR {
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--FALSE);
    background:
        var(--A-OR-B, var(--TRUE))
        var(--A-NOR-B,);
}

.cell--NOR {
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--TRUE);
    background:
        var(--A-OR-B, var(--FALSE))
        var(--A-NOR-B,);
}

.cell--XOR {
    --A-AND-B: var(--A, var(--B));
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--FALSE);
    --A-NAND-B:
        var(--A-AND-B)
        var(--A-OR-B, var(--TRUE))
        var(--A-NOR-B,);
    background:
        var(--A-AND-B, var(--A-OR-B, var(--FALSE)))
        var(--A-NAND-B,);
}

.cell--XNOR {
    --A-AND-B: var(--A, var(--B));
    --A-OR-B: var(--A) var(--B);
    --A-NOR-B: var(--A-OR-B) var(--TRUE);
    --A-NAND-B:
        var(--A-AND-B)
        var(--A-OR-B, var(--FALSE))
        var(--A-NOR-B,);
    background:
        var(--A-AND-B, var(--A-OR-B, var(--TRUE)))
        var(--A-NAND-B,);
}
```

  


上面的代码中使用了 `--_` 变量并将其设置为自身，以演示如何通过循环依赖变量获取 `--ON` 和 `--OFF` 。这样，我们可以实现相同的空间切换，但更可靠，因为它不会包含空格，避免了空格可能被压缩工具误删除的问题。

  


`AND` （与门）通过回退值将变量串联起来；`OR` （或门）直接列出空间切换变量，而不使用回退值。如果两个变量都是空，则输出为空；否则，它会变为无效值。

  


`XOR` （异或门）采用 `OR(AND, NOR)` 逻辑，由于我们无法从不完美的逻辑门中直接返回位值，这种方式可能较为复杂。我们不得不在两个地方定义 `--FALSE` ，这也让我们看到，串联逻辑门会增加代码的复杂性。

  


如果不需要应用“非”（`NOT`）分支，我们可以简化几乎所有示例——如果我们接受 CSS 属性可能无效或为空，那么可以省略某些部分。

  


虽然空间切换使 CSS 自定义属性变得更高级，但它也增加了代码的复杂度，尤其对于初学者，甚至有一些经验的 Web 开发者，都不能马上就可以理解清楚代码的意图或作用。这无疑增加了开发者的学习成本和降低了开发效率，甚至还易于出错。

  


事实上，你可能很少会用到这么复杂的逻辑计算，但在实际开发过程中，使用一些简单的逻辑计算，的确可以帮助我们减少代码量，提供更为便利的，灵活的代码。例如在一些只有 `0` 和 `1` 的状态下，还是非常有用的，其中典型的案例就是暗黑模式的切换：

  


```CSS
[data-type="error"] {
    --color: oklch(62.8% 0.25 29.23);
}

[data-type="ai"] {
    --color: oklch(58.11% 0.31 307.02);
}

[data-type="tip"] {
    --color: oklch(53.24% 0.23 256.22);
}

[data-type="warning"] {
    --color: oklch(61.47% 0.16 64.21);
}

[data-type="success"] {
    --color: oklch(47.06% 0.17 148.76);
}

[data-type="note"] {
    --color: oklch(41.84% 0 0);
}

html {
    color-scheme: light dark;

    @media (prefers-color-scheme: dark) {
        color-scheme: dark;
    }

    @media (prefers-color-scheme: light) {
        color-scheme: light;
    }

    &:has(:checked) {
        color-scheme: dark;
      
        :is(main, aside) {
            --i: 1;
        }
    }
    
    &:not:has(:checked) {
        color-scheme: light;
      
        :is(main, aside) {
            --i: 0;
        }
    }
}

aside {
    --lc-border: color-mix(in oklch, var(--color) 50%, oklch(100% 0 0 / 15%));
    --lc-background: color-mix(
        in oklch,
        var(--color) 25%,
        oklch(100% 0 0 / 50%)
    );
    --lc-color: color-mix(in oklch, var(--color) 100%, oklch(100% 0 0));
    --dc-border: color-mix(in oklch, var(--color) 5%, oklch(100% 0 0 / 15%));
    --dc-background: color-mix(in oklch, var(--color) 50%, oklch(0% 0 0 / 50%));
    --dc-color: color-mix(in oklch, var(--color) 50%, oklch(100% 0 0));
    --i: 0;
    --borders: var(--lc-border), var(--dc-border);
    --backgrounds: var(--lc-background), var(--dc-background);
    --colors: var(--lc-color), var(--dc-color);

    --_border: color-mix(in oklch, var(--borders) calc(var(--i) * 100%));
    --_background: color-mix(
        in oklch,
        var(--backgrounds) calc(var(--i) * 100%)
    );
    --_color: color-mix(in oklch, var(--colors) calc(var(--i) * 100%));

    border-color: var(--_border);
    background: var(--_background);
    color: var(--_color);

    @media (prefers-color-scheme: dark) {
        --i: 1;
    }
}

aside::selection {
    background: color-mix(in oklch, var(--color) 90%, oklch(100% 0 0));
    color: color-mix(in oklch, var(--color) 10%, oklch(100% 0 0));
}

main {
    --i: 0;
    --backgrounds: hsl(0 0% 98%), hsl(0 0% 2%);
    background: color-mix(in oklch, var(--backgrounds) calc(var(--i) * 100%));

    @media (prefers-color-scheme: dark) {
        --i: 1;
    }
}
```
