## CSS 自定义属性的基本特性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eba211f2c30d46e38215a478547ba63d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=675&s=72679&e=jpg&b=262626)

  


在正式进入 CSS 的真假切换之前，我们有必要回顾一下 CSS 自定义属性的基本特性。

  


### CSS 自定义属性作为变量

  


在今天使用 CSS 自定义属性时，它们主要作为 CSS 变量使用。如果你们使用过它们，你一定很熟悉下面这样的代码：

  


```CSS
:root {
    --s: 2vw;
    --p: 0.25vw;
}

.square {
    inlinne-size: var(--s);
    padding: var(--p);
    aspect-ratio: 1;
}

.square--big {
    --s: 16vw;
    --p: 1vw;
}
```

  


使用 `var()` 函数，我们创建了一个 CSS 变量，该变量会被替换为它所引用的自定义属性的值。例如，变量 `var(--s)` 将保存 `--s` 自定义属性的值（即 `2vw`），然后将其设置为 `inline-size` 属性的值。

  


有一点需要知道的是，CSS 自定义属性一旦被 `var()` 引用，它才是 CSS 变量，否则它只是 CSS 自定义属性。

  


### 回退值

  


CSS 变量的回退值是 `var()` 函数的第二个可选参数。例如，我们给 `.square` 元素设置背景为 `--c` 变量：

  


```CSS
.square {
    background-color: var(--c, lime);
}
```

  


如果我们没有在其他地方明确指定 `--c` 变量的值，那么就会使用回退值 `lime` 。

  


现在，假设我们在 `.square--big` 中指定了 `--c` 的值为 `hotpink` ：

  


```CSS
.square {
    background-color: var(--c, lime);
}

.square--big {
    --c: hotpink;
}
```

  


这样一来，默认的 `.square` （默认正方形）的背景颜色为 `lime` ，而带有 `.square--big` 类（大正方形）的背景颜色就会变成 `hotpink` 。

  


这里有几点需要注意。

  


首先，回退值可以是另一个 CSS 变量，而这个变量本身也可以有回退值，依此类推。这样我们可以无限的往里嵌套：

  


```CSS
.fallback {
    background: var(--c, var(--c0, var(--c1, var(--c2, var(--c3, var(--c4, #ccc)))))));
}
```

  


其次，以逗号分隔的列表是一个完全有效的回退值。事实上，`var()` 函数中第一个逗号后面指定的所有内容都构成了回退值，例如下面代码中的 `#ccc,#f90` 是`--stop-list` 的回退值：

  


```CSS
.fallback {
    background: linear-gradient(90deg, var(--stop-list, #ccc, #f90));
}
```

  


最后，但同样重要的是，我们可以为在不同位置使用的同一个变量设置不同的回退值，例如：

  


```CSS
a {
    border: 2px solid var(--c, color-mix(in oklch, var(--c), transparent 50%));
    color: var(--c, #ccc);
    
    &:is(:hover, :focus) {
        --c: #f90;
    }
}
```

  


### 使用 CSS 自定义属性影响多个 CSS 声明

  


在上面的示例中，CSS 设置了两类型的方块：常规大小（`.square`）的和大的方块（`.square--big`）。为了区分它们，我们需要切换 `.square--big` 类。切换这个类会影响两个自定义属性： `--s` 和 `--p` 。

  


但是，如果我们不想切换 HTML 类，而是切换一个 CSS 自定义属性，能实现吗？例如，我们想切换一个 CSS 自定义属性，并且让它自动影响 `--s` 和 `--p` 。

  


目前，要让一个 CSS 自定义属性影响多个其他 CSS 属性并不是非常容易，除非你使用一些 Hack 方案。

  


就以上面示例为例，我们将要改变的 `--s` 和 `--p` 仅仅是数值。我们可以在计算中使用二进制 CSS 自定义属性。给这些二进制自定义属性赋值为 `0` 或 `1` ，并在计算中使用它们。可以将这些二进制自定义属性想象成开关：**它们要么是关闭（假，即** **`0`** **），要么是开（真，即** **`1`** **）** ：

  


```CSS
:root {
    --is-big: 0;
}

.square {
    inline-size:calc(
        (1 - var(--is-big)) * 2vw /* 当 --is-big 为 0 （假）时应用的值 */
        + var(--is-big) * 16vw    /* 当 --is-big 为 1 （真）时应用的值 */
    );
    padding: calc(
        (1 - var(--is-big)) * .25vw /* 当 --is-big 为 0 （假）时应用的值 */
        + var(--is-big) * 1vw       /* 当 --is-big 为 1 （真）时应用的值 */
    );
    aspect-ratio: 1;
}

.square--big {
    --is-big: 1;
}
```

  


在上面的代码中，`--is-big` 自定义属性充当了一个二进制开关，控制 `calc()` 函数的结果。如果 `--is-big` 的值为 `0`，这些函数将返回一个特定的值，而当 `--is-big` 设置为 `1` 时，它将返回另一个值。

  


### 保证无效值

  


当你需要设置数值以外的内容（比如给大小方块设置不同的背景颜色）时，仅依赖 `0` 或 `1` 的开关，因为对颜色进行计算是无效的。

  


```CSS
.square {
    background-color: calc(
        hotpink * (1 - var(--is-big))
        + lime * var(--is-big)
    ); /* 这是无效值 */
}
```

  


因为 `calc()` 只可以对 `<length>`、`<frequency>`、`<angle>`、`<time>`、`<percentage>`、`<number>` 或 `<integer>` 类型的值进行计算。

  


然而，我们可以使用 CSS 自定义的“保证无效值”——你可以将自定义属性设置为“保证无效值”（即 `innitial`），从而强制 `var()` 函数使用它的回退值：

  


```CSS
:root {
    --is-lime: initial; /* var() 将使用回退值 */
}
.square {
    
    background-color: var(--is-lime, lime); /* lime */
}

.square--big {
    --is-lime: hotpink; /* 设置为初始值以外的任何值，即便只是一个空格，var() 不再使用回退值 */
}
```

  


这意味着你可以通过将自定义属性设置为 `initial` 的值来开启开关。这个技巧可以进一步优化，**将值（空格）视为开启位置，将** **`initial`** **的值视为关闭位置**。使用以下语法为新的自定义属性分配属性值： `--value-to-use-if-custom-toggle-is-on: var(--my-custom-toggle) value;`，其中将值放在 CSS 变量后面。

  


```CSS
:root {
    --toggler: initial;
    --hotpink-if-toggler: var(--toggler) hotpink;
}
```

  


要使用该值，请像之前一样使用 `var()` 语法（例如，添加回退值）：

  


```CSS
.square {
    background: var(--red-if-toggler, lime); /* 将是绿色！ */
}
```

  


所以，最终结果是自定义属性可以是空格（ ），也可以是 `initial`。开启或关闭。根据这个，你可以得到两个值：一个是开启时的值，另一个是关闭时的值。它依赖于 CSS 会忽略空格（例如， `lime` 变成简单的 `lime`），当引用的自定义属性包含 `initial` 时，CSS 会回退到 `var()` 的回退值。

  


```CSS
:root {
    --is-big: initial;    /* OFF */
    --is-medium: initial; /* OFF */
}

.square--medium {
    --is-medium:var(--_,) ; /* ON */
}

.square--big {
    --is-big: var(--_,); /* ON */
}

.square {
    --s-small: 2vw;
    --s-medium: var(--is-medium) 8vw;
    --s-big: var(--is-big) 16vw;
  
    --p-small: 0.25vw;
    --p-medium: var(--is-medium) 0.5vw;
    --p-big: var(--is-big) 1vw;
  
    --bgcolor-small: lime;
    --bgcolor-medium: var(--is-medium) red;
    --bgcolor-big: var(--is-big) hotpink;
  
    inline-size: var(--s-big, var(--s-medium, var(--s-small)));
    padding: var(--p-big, var(--p-medium, var(--p-small)));
    background-color: var(--bgcolor-big, var(--bgcolor-medium, var(--bgcolor-small)));
    aspect-ratio: 1/1;
    place-self: center;
}
```

## CSS 中的切换机制

  


CSS 中的切换机制有很多类型，比如之前所介绍的 **[交替切换](https://juejin.cn/book/7395119173656903715/section/7445287411581452338)**、**[循环切换](https://juejin.cn/book/7395119173656903715/section/7446278394624344074#heading-0)**、**[空间切换](https://juejin.cn/book/7395119173656903715/section/7446278394624344074#heading-1)**、**[分层切换](https://juejin.cn/book/7395119173656903715/section/7446278344482422822)**，以及我们今天要说的**真假切换**。

  


真假切换又主要分为：**在零值（** **`0`** **）与非零值（** **`1`** **）之间的切换**和**两个非零值之间的切换**。我们先来看第一种。

  


### 在零值与非零值之间的切换

  


先来看一个简单的示例：

  


```CSS
.figure {
    --i: 0;
    rotate: calc(var(--i) * 45deg);
}

html:has(:checked) {
    .figure {
        --i: 1;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/db72b86b23d74585ab94d18faa6ce5e3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=536&s=5009337&e=gif&f=148&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ByaPNvp

  


上面这个示例中，图片从 `0deg` （初始状态，即零值状态）旋转到 `45deg` 状态（非零值状态）。我们在 CSS 中，给 `.figure` 设置了一个旋转样式：

  


```CSS
.figure {
    rotate: calc(var(--i) * 45deg);
}
```

  


我们可以把自定义属性 `--i` 想象成一个切换开关，当开关关闭（`--i:0`）时，`rotate` 的值为 `0` ；当开关开启（`--i:1`）时，`rotate` 的值为非零（即 `45deg`），我们可以将该值（`45deg`）与开关变量 `--i` 相乘（即 `calc(var(--i) * 45deg)`）。其计算过程如下：

  


-   当开关关闭（`--i:0`）时，`calc(var(--i) * 45deg)` 计算的值为 `0 * 45deg = 0deg`，图片不会旋转
-   当开关打开（`--i:1`）时，`calc(var(--i) * 45deg)` 计算的值为 `1 * 45deg = 45deg` ，图片旋转 `45deg`

  


但是，如果我们希望开关关闭（`--i:0`）时，`rotate` 的值为非零（即 `45deg`）；开关开启（`--i:0`）时，`rotate` 的值为零，我们可以将该值（`45deg`）乘以 `(1 - var(--i))` （即开关变量的补值）。其计算过程如下：

  


-   当开关关闭（`--i:0`）时，`calc((1 - var(--i)) * 45deg)` 计算为 `(1 - 0) * 45deg = 45deg` ，图片旋转 `45deg`
-   当开关开启（`--i:1`）时，`calc((1- var(--i)) * 45deg)` 计算为 `(1 - 1) * 45deg = 0deg` ，图片不旋转

  


我们可以像下面这样编写 CSS：

  


```CSS
.figure {
    --i: 0;
    rotate: calc((1 - var(--i)) * 45deg);
}

html:has(:checked) {
    .figure {
        --i: 1;
    }
}
```

```CSS
aside {
    --borders: var(--lc-border), var(--dc-border);
    --backgrounds: var(--lc-background), var(--dc-background);
    --colors: var(--lc-color), var(--dc-color);
    
    --_border: color-mix(in oklch, var(--borders) calc(var(--i) * 100%));
    --_background: color-mix(
        in oklch,
        var(--backgrounds) calc(var(--i) * 100%)
    );
    --_color: color-mix(in oklch, var(--colors) calc(var(--i) * 100%));

    border-color: var(--_border);
    background: var(--_background);
    color: var(--_color);
}
```

  


你可能已经注意到了，`--borders` 、`--colors` 和 `--backgrounds` 分别包含了亮色系和暗色系两种模式下的颜色。但具体的控制方式是 `color-mix()` ，以 `--_border` 为例：

  


```CSS
aside {
    --_border: color-mix(in oklch, var(--borders) calc(var(--i) * 100%));
}
```

  


它相当于对 `--lc-border` （亮色模式下边框颜色）和 `--dc-border` （暗色模式下边框颜色）进行混合，并且混合比例由 `calc(--i * 100%)` 来决定：

  


-   当开关关闭（`--i:0`）时（亮色模式下），`color-mix(in oklch, var(--borders) calc(var(--i) * 100%))` 等同于 `color-mix(in oklch, var(--borders) 0%)` ，即 `color-mix(in oklch, var(--lc-border), var(--dc-border) 0%)` ，这意味着 `--dc-border` 混合比例为 `0` ，最终边框的颜色为 `var(--lc-border)`
-   当开关开启（`--i:1`）时（暗色模式下），`color-mix(in oklch, var(--borders) calc(var(--i) * 100%))` 等同于 `color-mix(in oklch, var(--borders) 100%)` ，即 `color-mix(in oklch, var(--lc-border), var(--dc-border) 100%)` ，这意味着 `--dc-border` 混合比例为 `100%` ，最终边框的颜色为 `var(--dc-border)`

正如你所看到的，开关变量（`--i` 的值为 `0` 或 `1`）类似于条件语句 `@if ... @else` 。其实，我们还可以使用 CSS 变量实现一些简单的逻辑运算。

  


#### 逻辑非：`NOT`

  


在逻辑运算中，**`NOT`** 是一种基本的运算符，用来对布尔值（`true` 或 `false`）进行反转。它将输入值反转成相反的布尔值。我们可以在 `calc()` 中，通过 `1` 减去开关变量（`--i`）就可以得到逻辑非：

  


```CSS
:root {
    --NOT-i: calc(1 - var(--i));
}
```

  


-   如果 `--i` 为 `0` ，那么 `--NOT-i` 为 `1` （即 `1 - 0 = 1`）
-   如果 `--i` 为 `1` ，那么 `--NOT-i` 为 `0` （即 `1 - 1 = 0`）

  


#### 逻辑与：`AND`

  


在逻辑运算中，`AND` 通常用来判断多个条件是否都为真。如果所有条件都为真，则结果为真；否则结果为假。换句话说，我们需要两个开关变量，比如 `--i` 和 `--j` ，每个变量可以是 `0` 或 `1` ，彼此独立。这意味着，我们可能会遇到以下四种情况

  


-   `--i: 0; --j: 0`
-   `--i: 1; --j: 0`
-   `--i: 0; --j: 1`
-   `--i: 1; --j: 1`

当两个开关变量（`--i` 和 `--j`）都为 `1` 时，`AND` 运算的结果才是 `1` ，否则为 `0` 。换句话说，如果两个开关变量中至少有一个是 `0` ，那么结果就是 `0` 。它对应着数学计算中的乘法计算——如果两个数相乘，其中一个数为 `0` ，那么它们的积就是 `0` 。因此，我们可以使用下面的公式来定义 `--AND` ：

  


```CSS
:root {
    --AND: calc(var(--i) * var(--j));
}
```

  


-   对于 `--i: 0, --j: 0`，`--AND` 为 `0`（ 即 `0 × 0 = 0`）
-   对于 `--i: 1, --j: 0`，`--AND` 为 `0`（即 `1 × 0 = 0`）
-   对于 `--i: 0, --j: 1`，`--AND` 为 `0` （即 `0 × 1 = 0`）
-   对于 `--i: 1, --j: 1`，`--AND` 为 `1`（即 `1 × 1 = 1`）

  


#### 逻辑与非：`NAND`

  


`NAND` （与非）实际上是逻辑与 `AND`（逻辑与）的否定。它首先执行 `AND` ，然后再取反。它的特点是，除非所有输入都为真，否则输出总为真。这意味着，`--i` 和 `--j` 开关变量折值都为 `1` 时，输出才是 `0` ，否则都是 `1` 。

  


在 CSS 中，我们可以结合之前的 `--NOT-i` 和 `--AND` 来表达 `--NAND` ：

  


```CSS
:root {
    --AND: calc(var(--i) * var(--j));
    --NOT-AND: calc(1 - var(--AND));
    --NAND: var(--NOT-AND);
}
```

  


简单化一下：

  


```CSS
:root {
    --NAND: calc(1 - var(--i) * var(--j));
}
```

  


与 `--AND` 类似，`--NAND` 也会出现四种情况：

  


-   当 `--i: 0, --j: 0` 时，`--NAND` 为 `1`（即 `1 - 0 × 0 = 1`）
-   当 `--i: 0, --j: 1` 时，`--NAND` 为 `1`（即 `1 - 0 × 1 = 1`）
-   当 `--i: 1, --j: 0` 时，`--NAND` 为 `1`（即 `1 - 1 × 0 = 1`）
-   当 `--i: 1, --j: 1` 时，`--NAND` 为 `0`（即 `1 - 1 × 1 = 0`）

  


#### 逻辑或：`OR`

  


逻辑或 `OR` 在两个或更多布尔值之间执行操作，只有在至少一个输入为真时，输出才为真。如果所有输入都为假，则输出为假。也就是说，`OR` 运算的结果是 `1` ，至少有一个开关变量是 `1` ，否则为 `0` 。事实上，`OR` 运算的结果是 `AND` 运算的否定。把它转化为 CSS 的话，会是下面这样：

  


```CSS
:root {
    --OR: calc(1 - (1 - var(--i)) * (1 - var(--j)));
}
```

  


将开关变量 `--i` 和 `--j` 的值（`0` 或 `1`）套到公式中，`--OR` 的值可能会是：

  


-   当 `--i: 0, --j: 0` 时，`--OR` 为 `0`（即 `1 - (1 - 0) × (1 - 0) = 1 - 1 × 1 = 0`）
-   当 `--i: 0, --j: 1` 时，`--OR` 为 `1`（即 `1 - (1 - 0) × (1 - 1) = 1 - 1 × 0 = 1`）
-   当 `--i: 1, --j: 0` 时，`--OR` 为 `1`（即 `1 - (1 - 1) × (1 - 0) = 1 - 0 × 1 = 1`）
-   当 `--i: 1, --j: 1` 时，`--OR` 为 `1`（即 `1 - (1 - 1) × (1 - 1) = 1 - 0 × 0 = 1`）

  


#### 逻辑或非：`NOR`

  


逻辑或非（`NOR`）是逻辑非（`NOT`）与逻辑或（`OR`）的组合操作。它先执行一个逻辑或操作，然后再对结果应用逻辑非操作。也就是说，逻辑或非（`NOR`）是逻辑或（`OR`）的反操作，只有在所有输入为假（`0`）时，输出才为真（`1`），否则输出为假（`0`）。

  


我们可以像下面这样来描述逻辑或非 `--NOR` ：

  


```CSS
:root {
    --OR: calc(1 - (1 - var(--i)) * (1 - var(--j)));
    --NOT-OR: calc(1 - var(--OR));
}
```

  


简化一下：

  


```CSS
:root {
    --NOR: calc(1 - (1 - (1 - var(--i)) * (1 - var(--j))));
    
    /* 等同于 */
    --NOR: calc(1 - 1 + (1 - var(--i)) * (1 - var(--j)));
    
    /* 等同于 */
    --NOR: calc((1 - var(--i)) * (1 - var(--j)));
}
```

  


将开关变量 `--i` 和 `--j` 的值（`0` 或 `1`）套到公式中，`--NOR` 的值可能会是：

  


-   当 `--i: 0, --j: 0` 时，`--NOR` 为 `1`（即 `(1 - 0) × (1 - 0) = 1`）
-   当 `--i: 0, --j: 1` 时，`--NOR` 为 `0`（即 `(1 - 0) × (1 - 1) = 0`）
-   当 `--i: 1, --j: 0` 时，`--NOR` 为 `0`（即 `(1 - 1) × (1 - 0) = 0`）
-   当 `--i: 1, --j: 1` 时，`--NOR` 为 `0`（即 `(1 - 1) × (1 - 1) = 0`）

  


#### 逻辑异或：`XOR`

  


当两个输入不同时（一个真，一个假），输出为真（`1`）；当两个输入相同时（都是真或都是假），输出为假（`0`）。一开始看起来比较复，但仔细思考（两个操作数需要不同才能得到 `1` ，否则为 `0`），我们就能在 `calc()` 中使用正确的算术运算——减法运算！

  


如果开关变量 `--i` 和 `--j` 相等，那么 `--i` 减去 `--j` 的结果是 `0` ；如果开关变量 `--i` 和 `--j` 不相等，那么 `--i` 减去 `--j` 的结果可能是 `-1` ，也可能是 `1` ：

  


-   当 `--i:0, --j:1` 时，`--i` 减去 `--j` 的结果为 `-1`
-   当 `--i:1, --j:0` 时，`--i` 减去 `--j` 的结果为 `1`

  


这已经非常接近我们想要的结果了，但还不完全正确。我们在四种情况中的三个得到了我们想要的结果，但在 `--i: 0, --j: 1` 的情况下，我们需要的结果是 `1` ，而不是 `-1` 。然而， `-1` 、`0` 和 `1` 有一个共同点，那就是它们与自身相乘得到的结果是它们的绝对值（`-1` 和 `1` 的绝对值都是 `1`）。因此，实际的解决方案是将这个差值与自身相乘：

  


```CSS
:root {
    --XOR: calc((var(--i) - var(--j)) * (var(--i) - var(--j)));
}
```

  


我们也可以使用 `pow()` 来替代 `calc()` 函数：

  


```CSS
:root {
    --XOR: pow(var(--i) - var(--j), 2);
}
```

  


测试我们的四种可能情况，我们得到：

  


-   当 `--i: 0, --j: 0` 时，`--XOR` 为 `0`（即 `(0 - 0) × (0 - 0) = 0`）
-   当 `--i: 0, --j: 1` 时，`--XOR` 为 `1`（即 `(0 - 1) × (0 - 1) = 1`）
-   当 `--i: 1, --j: 0` 时，`--XOR` 为 `1`（即 `(1 - 0) × (1 - 0) = 1`）
-   当 `--i: 1, --j: 1` 时，`--XOR` 为 `0`（即 `(1 - 1) × (1 - 1) = 0`）

  


#### 逻辑异或非：`XNOR`

  


逻辑异或非（`XNOR`）会先执行逻辑异或（`XOR`），然后再取反（`NOT`）。它的特点是，当两个输入相同时（都是真或都是假），输出是 `1` ；当两个输入不同时（一个真和一个假），输出是 `0` 。

  


在 CSS 中，我们可以结合 `--XOR` 和 `--NOT-XOR` 来描述 `--XNOR` ：

  


```CSS
:root {
    --XOR: calc((var(--i) - var(--j)) * (var(--i) - var(--j)));
    --NOT-XOR: calc(1 - var(--XOR));
    --XNOR: var(--NOT-XOR);
}
```

  


简化之后的 `--XNOR` 像下面这样：

  


```CSS
:root {
    --XNOR: calc(1 - ((var(--i) - var(--j)) * (var(--i) - var(--j))));
    
    /* 或者 */
    --XNOR: calc(1 - pow(var(--i) - var(--j), 2));
}
```

  


测试我们的四种可能情况，我们得到：

  


-   当 `--i: 0, --j: 0` 时，`--XNOR` 为 `1`（即 `(1 - (0 - 0) × (0 - 0)) = 1`）
-   当 `--i: 0, --j: 1` 时，`--XOR` 为 `0`（即 `(1 - (0 - 1) × (0 - 1)) = 0`）
-   当 `--i: 1, --j: 0` 时，`--XOR` 为 `0`（即 `(1 - (1 - 0) × (1 - 0)) = 0`）
-   当 `--i: 1, --j: 1` 时，`--XOR` 为 `1`（即 `(1 - (1 - 1) × (1 - 1)) = 1`）

  


### 在两个非常零值之间切换

  


但是，如果我们想要在开关关闭（`--i:0`）时使用一个非零值，而在开关开启（`--i:1`）时使用另一个不同的非零值，该怎么办呢？例如，我们希望一些文本的字体大小（`font-size`）在开关关闭（`--i:0`）时为 `2rem` ，在开关开启（`--i: 1`）时为 `10vw` 。在这个场景中，我们可以使用逻辑非来处理：

  


```CSS
.element {
    --i: 0;
    --j: calc(1 - var(--i));
    
    font-size: calc(var(--j) * 2rem + var(--i) * 10vw);
}
```

  


-   当 `--i:0` 时（开关关闭），`--j` 的值为 `1` ，对应的 `font-size` 的值就是 `2rem` （即 `calc(1 * 2rem + 0 * 10vw)`）
-   当 `--i:1` 时（开关打开），`--j` 的值为 `0` ，对应的 `font-size` 的值就是 `10vw` （即 `calc(0 * 2rem + 1 * 10vw)`）

  


上面是两个非零值之间切换的简单用例，稍后我们在实战中还会应用到这方面的技巧。

  


## 触发切换的因素

  


接下来，我们继续探讨另一方面：究竟是什么导致开关从开启状态切换到关闭状态，或者反过来呢？我们有几种选择。

  


### 基于元素的切换

  


这意味某些元素的开关处于关闭状态，而其他元素的开关处于开启状态。例如，我们可以通过奇偶性来确定。假设，我们想让所有偶数元素进行旋转，并且背景色改为橙色，而不是初始的灰色。

  


```CSS
.box {
    --i: 0; /* 开关关闭 OFF */
    --j: calc(1 - var(--i)); 
    rotate: calc(var(--i) * 45deg);
    background-color: hsl(
        var(--h, 36deg)
        calc(var(--i) * var(--s, 100%))
        calc(var(--j) * var(--l1, 80%) + var(--i) * var(--l2, 50%))
    );
    
    &:nth-child(2n) {
        --i: 1; /* 开关打开 ON */
    }
}
```

  


在奇偶性情况下，我们为每个第二个元素（`:nth-child(2n)`）切换开关：

### 基于状态的切换

  


这意味着当元素本身（或其父元素或某个前一个兄弟元素）处于某个状态时，开关是关闭的，而当它处于另一个状态时，开关是打开的。比如之前旋转图片的用例，当 `Switch` 被选中或取消选中时，开关就会被切换：

  


```CSS
.figure {
    --i: 0; /* 开关关闭 OFF */
    --j: calc(1 - var(--i));
    rotate: calc(var(--j) * 45deg);
}

html:has(:checked) {
    .figure {
        --i: 1; /* 开关打开 ON */
    }
}
```

### 基于媒体查询（或容器查询）的切换

  


另一种可能性是由媒体查询或容器查询触发切换，比如之前所展示的暗黑模式的示例，它会根据用户系统偏好来触发开关的切换：

```CSS
aside {

        --i: 1; /* 开关打开 ON */

main {
    --i: 0;/* 开关关闭 OFF */

    @media (prefers-color-scheme: dark) {
        --i: 1;/* 开关打开 ON */
    }
}
```

## 实战案例

### `Switch` 按钮

  


我们先从简单的 `Switch` 按钮开始：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd1de98d66c243a38100348c3ffa3009~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1080&h=432&s=221657&e=gif&f=127&b=1452fd)

  


这是一个非常简单的 `Switch` 按钮，我曾在《[CSS 图形：自定义选择控件](https://juejin.cn/book/7395119173656903715/section/7412570263411720219)》中深度的剖析了该控件如何制作。但在这里，我向大家展示另一种制作方式——通过真假切换来调整按钮选中和未选中状态时白色滑块的位置。

  


构建这个 `Switch` 按钮需要一个类似下面这样的 HTML 结构：

  


```HTML
<div class="switch">
    <input class="sr-only" type="checkbox" id="switch" />
    <label for="switch">
        <span>Photo</span>

    </label>
```

```CSS
.switch {

    label {
        display: grid;
        grid-auto-flow: column;
        grid-auto-columns: minmax(0, 1fr);
        position: relative;
        cursor: pointer;
      
        &::after {
            content: "";
            position: absolute;
            background: #fff;
            height: 100%;
            width: 50%;
            left: 0;
            top: 0;
            border-radius: calc(infinity * 1px);
        }
    }
    
    span {
        display: grid;
        padding: 15px 0;
        user-select: none;
        pointer-events: none;
        place-content: center;
        z-index: 2;
    }
}
```

接下来，我们需要添加一个开关变量 `--i` 来控制 `Switch` 按钮中白色滑块的位置。当开关处于关闭状态（`--i: 0`）时，白色滑块位于左侧（“Photo” 选项底部）；当开关处于打开状态（`--i: 1`）时，白色滑块位于右侧（“Video”选项底部）：

我们可以通过 `translate` 来控制它的位置：

  


-   复选框未选中，开关关闭 `--i:0` ，白色滑块（`label::after`）的 `translate` 值为 `0%`
-   复选框选中，开关打开 `--i: 1` ，白色滑块（`label::after`）的 `translate` 值为 `100%`

  


我们可以像下面这样编写 CSS：

  


```CSS
.switch {
    --i: 0; /* 开关关闭，复选框未选中 */
    
    label::after {
        translate: calc(var(--i) * 100%) 0; /* 白色滑块位于左侧*/
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ced680246b94471a80dab4597806e5db~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=374&s=26019&e=png&b=1563ff)

  


我们可以基于复选框的状态来切换开关变量 `--i` 的值。复选框要是选中了，`--i` 的值为 `1` ：

  


```CSS
.switch {
    --i: 0; /* 开关关闭，复选框未选中 */
    
    label::after {
        translate: calc(var(--i) * 100%) 0; /* 白色滑块位于左侧*/
    }
    
    &:has(:checked) {
        --i: 1; /* 开关打开，复选框选中 */
    }
}
```

  


注意，在这个示例中选择 `translate` 来控制白色滑块的位移，主要原因是我们可以动画化 `transition` ，它比动画化 `left` 性能要更好。

  


```CSS
.switch {
    --i: 0; /* 开关关闭，复选框未选中 */
    
    label::after {
        translate: calc(var(--i) * 100%) 0; /* 白色滑块位于左侧*/
        transition: translate 300ms;
    }
    
    &:has(:checked) {
        --i: 1; /* 开关打开，复选框选中 */
        
        label::after {
            transition: translate 300ms linear;
        }
    }
}
```

### 可折叠的搜索栏

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/60a8b251f05741a7a7e03691811eb422~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1144&h=432&s=897443&e=gif&f=192&b=1f1f1f)

  


这是我们将要完成的第二个实战案例。

  


从可用性的角度来看，在 Web 应用上使用这样的搜索框可能不是最佳的，因为人们期望按钮在搜索框后触发搜索，而不是关闭（折叠）搜索栏。但它仍然是一个有趣的编码练习，这也是我选择在这里分析它的原因。

  


构建这个搜索栏的 HTML 结构与之前 `Switch` 按钮类似：

  


```HTML
<div class="search--bar">
    <input class="sr-only" type="checkbox" id="search" name="search" />
    <input type="search" placeholder="Search ..." />
    <label for="search">Search</label>
</div>
```

  


首先，我们将 `label` 标签变成一个在的圆形按钮，并使用一个大的负值 `text-indent` 和 `overflow: hidden` 把 `label` 标签中文本内容移出视口：

  


```CSS
.search--bar {
    --btn-s: 5em;
    --btn-c: crimson;
    
    display: flex;
    align-items: center;
    justify-content: center;
    
    label {
        display: inline-grid;
        overflow: hidden;
        text-indent: calc(infinity * -1px);
        background: var(--btn-c);
        width: var(--btn-s);
        aspect-ratio: 1;
        border-radius: 50%;
        cursor: pointer;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e2094b1d32dc4ca195fd84301ce20b28~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=806&s=43031&e=png&b=252525)

  


我们继续添加 CSS，使整个搜索框看起来更完善一些：

  


```CSS
.search--bar {
    --btn-s: 5em;
    --bar-w: calc(var(--btn-s) * 4);
    --bar-h: calc(var(--btn-s) * 0.65);
    --bar-r: calc(var(--bar-h) / 2);
    
    
    [type="search"] {
        border: none;
        padding: 0 1em;
        width: var(--bar-w);
        height: var(--bar-h);
        border-radius: var(--bar-r) 0 0 var(--bar-r);
        background: #3f324d;
        color: #fff;
        font-size: 1em;
        
        &::placeholder {
            opacity: .5;
            color: inherit;
            font-size: .875em;
            letter-spacing: 1px;
            text-shadow: 0 0 1px, 0 0 2px;
        }
        &:focus {
            outline: none;
            box-shadow: 0 0 1.5em #ffeacc, 0 1.25em 1.5em rgb(0 0 0 / .2);
            background: #ffeacc;
            color: #000; 
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/023d4cd2cefa497db8d5e01327f5821f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=806&s=48040&e=png&b=252525)

  


正如你所看到的，搜索框的右边缘与按钮的左边缘重合。然而，我们希望它们有一点重叠——比如搜索框的右边缘与按钮的垂直中线对齐。考虑到我们在 Flexbox 容器中设置了 `align-items: center` ，即使我们在其中一个或两个元素之间设置了外边距（`margin` 的负值），它们仍然在水平方向保持居中对齐。


![fig-19.jpg](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de7412fd735e4d8d90787ed683e01914~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2730&h=1540&s=223428&e=png&b=ffffff)
  


如上图所示，搜索框与按钮的叠加量等按钮的半径（比如 `--btn-r`）。我们将搜索框的 `margin-right` 设置为一个负的 `--btn-r` ，与此同时，设置搜索框的右侧内距为 `--btn-r` ，以弥补重叠部分：

  


```CSS
.search--bar {
    --btn-s: 5em;
    --btn-r: calc(var(--btn-s) / 2);
    
    [type="search"] {
        margin-right: calc(-1 * var(--btn-r));
        padding: 0 1em;
        padding-right: calc(var(--btn-r) + 1em);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fdac6a4612084ce6815a8dab6b161de1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=806&s=47750&e=png&b=252525)

  


现在，我们得到了在扩展状态下按钮和搜索框的正确位置。

  


在这个状态下（扩展状态），搜索框和按钮组合的总宽度是搜索框的宽度（`--bar-w`）加上按钮的半径（`--btn-r`），因为我们有半个按钮的重叠部分。在折叠状态下，搜索框和按钮组合的总空度就是按钮的宽度 `--btn-s` 。

  


![fig-21.jpg](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d6a03fe376ef40b1aae7c812ce07618c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=778&s=177162&e=png&b=fefcfc)


  


搜索栏从展开状态切换到折叠状态时，我们希望它们始终在相同的中轴线（`.search--bar` 容器的垂直中线）上。如此一来，我们需要将按钮向左移动，移动的距离是展开状态下组合宽度的一半（即 ` calc((var(--bar-w) + bar(--btn-r))  / 2)`）减去按钮的半径（`--btn-r`）。我们可以把这个偏移量存储在 `--x` 变量中，并将按钮的 `translate` 设置为负的 `--x` （因为按钮沿着水平方向向左移动）：

  


```CSS
.search--bar {
    --btn-s: 5em;
    --btn-r: calc(var(--btn-s) / 2);
    --bar-w: calc(var(--btn-s) * 4);
    --x: calc((var(--bar-w) + var(--btn-r)) / 2 - var(--btn-r));
    
    label{
        translate: calc(-1 * var(--x));
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a7db491b14545e190024da0f721d43b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=806&s=48121&e=png&b=252525)

  


由于我们希望搜索框折叠成按钮的样子，所以我们将相同的偏移量（`--x`）应用于搜索框，但方向是向右称动：

  


```CSS
.search--bar {
    --btn-s: 5em;
    --btn-r: calc(var(--btn-s) / 2);
    --bar-w: calc(var(--btn-s) * 4);
    --x: calc((var(--bar-w) + var(--btn-r)) / 2 - var(--btn-r));
    
    label{
        translate: calc(-1 * var(--x));
    }
    
    [type="search"] {
        translate: var(--x);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d45fa3c37faf438f9814191571b8e9d8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=806&s=40593&e=png&b=252525)

  


到目前为止，我们所看到的都是静态效果。我们最终的效果是：**当复选框未选中时，搜索栏处于折叠状态；当复选框选中时，搜索栏处于展开状态**。这意味着，当复选框未选中时，搜索框和按钮会通过 `translate` 进行平移；而当复选框选中时，它们保持当前的位置（不进行平移）。

  


为了实现这一些，我们可以引入开关变量 `--i` ：

  


-   复选框未选中时，开关处于关闭状态，`--i` 变量的值为 `0` 。搜索栏处于折叠状态——搜索框和按钮需要进行平移
-   复选框选中时，开关处于打开状态，`--i` 变量的值为 `1` 。搜索栏处于展开状态——搜索框和按钮不需要进行平移

  


需要知道的是，复选框未选中时，`--i` 值为 `0` ，但我们要求的是搜索框和按钮需要进行平移。这意味着我们需要新增一个变量 `--j` ，并且它的值是 `--i` 变量的反值。我们可以使用之前所介绍的逻辑非来实现。引入开关变量之后的 CSS 代码如下：

  


```CSS
.search--bar {
    --i: 0; /* 开关关闭，复选框未选中 */
    --j: calc(1 - var(--i)); /* 取 --i 的相反值 */
    
    &:has(:checked) {
        --i: 1; /* 开关打开，复选框选中 */
    }
    
    label {
        translate: calc(var(--j) * var(--x) * -1);
    }
    
    [type="search"] {
        translate: calc(var(--j) * var(--x));
    }
}
```

  


现在，我们已经有了一个交互效果！点击按钮可以切换复选框的状态：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ee16525690e641e6b543cfa3424b35f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1376&h=432&s=291734&e=gif&f=118&b=1f1f1f)

  


我们可以通过 `transition` 对 `translate` 属性添加过渡动画，使搜索栏在切换的过程中，不会显得那么突兀：

  


```CSS
.search--bar {
    --i: 0; /* 开关关闭，复选框未选中 */
    --j: calc(1 - var(--i)); /* 取 --i 的相反值 */
    
    &:has(:checked) {
        --i: 1; /* 开关打开，复选框选中 */
    }
    
    label {
        translate: calc(var(--j) * var(--x) * -1);
    }
    
    [type="search"] {
        translate: calc(var(--j) * var(--x));
    }
    
    :is(label, [type="search"]) {
        transition: all 1s;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9870187f16ff458ab93c4d5ec0483fed~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1358&h=418&s=705515&e=gif&f=190&b=1f1f1f)

  


正如你所看到的，现在点击切换按钮时，搜索栏有一个平滑的过渡动画效果！

  


接下来，我们需要解决一个更大的问题——搜索栏在折叠状态下，需要隐藏溢出的搜索框。我们可以通过 CSS 的 `clip-path` 的 `inset()` 函数来解决。通过指定距离元素的边框盒子（`border-box`）顶部、右侧、底部和左侧的距离，来定义一个裁剪矩形。这个裁剪矩形外部的部分会被裁剪掉，只有矩形内部的部分会显示出来。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9105d2bc5e0643e785ff2c084f668f37~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3262&h=1690&s=320343&e=jpg&b=f4f3f3)

  


  


如上图所示，每个距离都是从边框盒子的边缘向内计算的。在这种情况下，它们是正值。但它们也可以向外扩展，在这种情况下，距离值为负，裁剪矩形的相应边缘会位于元素的边框盒子外部。你可能会认为没有理由这么做，但在我们这个示例中，确实需要这么做。

  


我们希望从顶部（`dt`）、底部（`db`）和左侧（`dl`）的距离为负值，并且足够大，以容纳在 `:focus` 状态下超出元素边框盒子的 `box-shadow` ，因为我们不希望它被裁剪掉。因此，解决方案是创建一个裁剪矩形，使这些方向的边缘位于元素的边框盒子之外。

  


右侧的距离（`dr`）在折叠状态（未选中复选框， `--i:0`）是整个搜索框宽度 `--bar-w` 减去按钮半径 `--btn-r` ，而在展开状态下（复选框选中，`--i: 1`）为 `0` ：

  


```CSS
.search--bar {
    --out-d: -3em;
    
    [type="search"] {
        clip-path: inset(
            var(--out-d) 
            calc(var(--j) * (var(--bar-w) - var(--btn-r)))
            var(--out-d)
            var(--out-d)
        );
    }
}
```

  


现在，我们有了一个可以在点击按钮时展开和收起的搜索框和按钮组合。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6cdddb511e35439ea30cdf1d00f9f257~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=420&s=662271&e=gif&f=201&b=1f1f1f)

```CSS
} 
```

我们在 `li` 元素中以内联的方式定义了 `--c0` 和 `--c1` 两个颜色变量，后面我们会通过它们来定义每个列表项的渐变色和文本颜色等。

  


我们先从布局开始，我们采用[ CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 和[容器查询的尺寸查询](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)来构建布局。当容器有足够的空间时，每个列表项是行方向（`row`）布局；反之，则是列方向（`column`）布局。与此同时，`font-size` 也会随之变小。

  


首先在 `.crads` 设置 `container-type` 的值为 `inline-size` ，告诉浏览器，`.cards` 是一个查询容器：

  


```CSS
.cards {
    container-type: inline-size;
}
```

  


这样一来，每个列表项（`li`）就可以根据查询容器 `.cards` 的 `inline-size` 来调整样式。比如：

  


```CSS
.cards {
    container-type: inline-size;
    
    li {
        display: flex;
        flex-direction: column;
        align-items: center;
        justify-content: center;
        background: red;
        
        &::before {
            content: attr(data-icon);
        }
        
        @container (width >= 400px) {
            flex-direction:row;
            background: lime;
        }
    }
}
```

  


调整容器 `.cards` 的大小，你将看到下面这样的效果：

现在的效果离我们所期望的效果还很远，接下来我们逐步完善，使我们获得最终期望的布局效果。

  


毫无疑问，我们需要引入开关变量，因为我们需要通过这些开头变量来控制 UI 样式：

  


```CSS
.cards {
    li {
        --k: var(--narr, 1);
        
        flex-direction: var(--narr, column);  
        
        @container (width >= 400px) {
            --narr: 0;
        }  
    }
}
```

  


默认情况下，`--narr` 变量尚未在别的地方显式声明，因此 `flex-direction` 会取 `--narr` 变量的备用值 `column` 。当容器宽度大于或等于 `400px` 时，`--narr` 变量的值为 `0`，它是一个无效的 `flex-direction` 值，因此它会回退到默认值 `row` 。

  


同时，我们还可以在该模式下调整列表项的 `font-size` ：

  


```CSS
.cards {
    li {
        --k: var(--narr, 1);
        
        flex-direction: var(--narr, column); 
        font: 900 calc((1.25 - .375 * var(--k)) * 1em) / 1.5 trebuchet ms, verdana,sans-serif; 
        
        @container (width >= 400px) {
            --narr: 0
        }  
    }
}
```

  


仔细观察目标效果，奇偶数的列表项，样式是不一样的，比如文本的对齐方式（`text-align`）、哪一侧的边框（`border`）需要设定非零值、边框渐变的位置和方向等。这些我们将由新的开关变量 `--parity` 来决定。此外，奇偶性和是否处理宽屏模式共同决定了列表项的外边距（`margin`）和内边距（`padding`）：

  


```CSS
.cards {
    li {
        --p: var(--parity, 0);
        --i: var(--wide, 1);
        --j: calc(1 - var(--i));
        
        flex-direction: var(--narr, column); 
        font: 900 calc((1.25 - .375 * var(--k)) * 1em) / 1.5 trebuchet ms, verdana,sans-serif; 
        
        text-align: var(--parity, right);
        margin: 
            0 
            calc(var(--i) * var(--p) * (var(--card-h) / 2)) 
            0
            calc(var(--i) * var(--q) * (var(--card-h) / 2));
        border: solid 0 transparent;
        border-width: 
            0 
            calc(var(--q) * var(--card-b)) 
            0
            calc(var(--p) * var(--card-b));
        padding: 
            var(--card-p) 
            calc(var(--q) * var(--card-p)) 
            calc((var(--j) + var(--i)*var(--q))* var(--card-p))
            var(--card-p)
            calc((var(--j) + var(--i)*var(--p))* var(--card-p));
        background: 
            linear-gradient(var(--c1), var(--c0)) content-box,
            linear-gradient(var(--c0), var(--c1));
        
        &:nth-child(2n) {
            --parity: 1;
        }
        
        @container (width <= 30.75em) {
            --wide: 0;
        }
      
        @container (width >= 400px) {
            --narr: 0
        }  
    }  
}
```

  


我们将偶数列表项 `li:nth-child(2n)` 的 `--parity` 变量设置为 `1` ，这个时候我们可以看到偶数列表项与奇数列表项在样式上有明显的差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a549be339b8460d992078e2e1d3e908~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2184&h=1720&s=598838&e=png&b=f9f9f9)

  


在这个示例中，图标是使用 `::before` 伪元素创建的，其排列顺序（`order`）也取决于 `--parity` 开关变量，但仅在宽容器式下生效，在窄容器模式下，图标始终位于文本的前面。图标的左右边距受奇偶性（`--parity`）和是否处于宽容器模式的影响。当处于宽容器模式时，图标部分溢出 `li` 容器，而窄容器模式下不会出现这种情况。此外，图标的字体大小（`font-size`）也取决于是否处于窄容器模式，这进一步影响了它的 `em` 计算尺寸和内边距 `padding` ：

  


```CSS

```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/43ae3a8350da4ec5a04c64a5daf9a403~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1360&h=872&s=16129885&e=gif&f=272&b=faf9f9)

```CSS
.cards {
    li::after {
        content: var(--wide, '');
    }
    
    @container (width <= 30.75em) {
        --wide: 0;
    }
}
```
