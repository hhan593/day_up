众所周知，在 CSS 中，我们受限于绝对单位和相对单位，因此在构建 Web 布局时（比如设置元素尺寸大小、间距等），要么使用固定尺寸（比如 `10px`），要么基于另一个元素的尺寸进行计算（比如 `%` 、`vw` 、`cqw` 等）。这两种方式各有利弊，因此并不存在“谁更好”一说，这一切都应该取决于元素的上下文。过于依赖某种方式并不能真正解决问题。

main {

## 快速回顾一下 `min()` 函数

  


CSS 的 `min()` 函数可以接受一个以逗号（`,`）分隔的值列表，并从值列表中返回较小的那个值，具体取决于元素的上下文。例如：

  


```CSS
.element {
    width: min(50vw, 500px);
}
```

  


`.element` 元素的宽度将取决于浏览器视窗宽度。如果 `50vw` 计算的值大于 `500px` ，那么 `min(50vw, 500px)` 将会忽略 `50vw` ，返回 `500px` ，此时元素 `.element` 宽度（`width`）值为 `500px` ；反之，如果 `50vw` 计算的值小于 `500px` ，那么 `min(50vw, 500px)` 将会忽略 `500px` ，返回 `50vw` ，此时元素 `.element` 宽度（`width`）值为 `50vw` 。

## `min()` 函数与响应式布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dd8bb850c5f04c3e976d8a8e1dfe52f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1260&s=75309&e=jpg&b=5d79d4)

  


`min()` 函数在某种程度上说，它有点类似于 **[CSS 的容器查询单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-2)**，因为它可以根据容器内的可用空间调整值。但不同之处在于，`min()` 函数并不会主动查询容器的尺寸来计算最终值，而是接受多个长度值（通常情况是两个长度值），并在上下文中选择较小的那个。因此，`min()` 函数（以及 `max()` 和 `clamp()` 函数）是适应视口（或容器）大小的响应式布局的有用工具。它使用条件逻辑来选择“最佳”值，从而可以在不依赖 **[CSS 媒体查询](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)**（`@media`）或 **[CSS 容器查询](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)**（`@container`）的情况下调整布局。例如：

  


```CSS
.element {
    width: min(50%, 200px);
}

/* 类似于 */
.element {
    width: 200px;
    
    @media (width >= 768px) {
        width: 50%;
    }
}
```

  


在这个例子中，CSS 的 `@media` 规则是在浏览器视窗宽度达到（或大于）`768px` 时，将 `.element` 元素的宽度设置为 `50%` 。而 `min()` 函数则是根据可用空间动态调整，无论浏览器视窗宽度如何变化，都会自动选择合适的值。

  


或者说，我们可以把 `min()` 函数看作是一个根据上下文做出“更智能”决定的工具，我们不需要手动计算该使用哪个值。然而，仅使用 `min()` 函数并搭配任意 CSS 单位并不足够。例如，相对单位通常比绝对单位更适用于响应式布局。从某种意义上讲，`min()` 函数也可以被视为设置了一个最大值，因为它不会低于第一个值，同时也会被第二个值限制住。

  


我之前提到过，`min()` 函数可以接受任何类型的长度单位。现在，让我们尝试专注于某种特定单位，并观察 `min()` 函数在完全依赖该单位进行响应式设计时的表现。在这里，我将使用视窗单位（比如，`vw` 和 `vh`），因为它们直接相对于浏览器视窗宽度计算。

  


[视窗单位有多种形式](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bb5b416c7eae4c2daa75ea93f4347d27~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=1037&s=57156&e=jpg&b=ffffff)

  


  


-   **`vw`** ：表示根元素（`html`）宽度的百分比，`1vw` 等于浏览器视窗宽度的 `1%`
-   **`vh`** ：表示根元素（`html`）高度的百分比，`1vh` 等于浏览器视窗高度的 `1%`
-   **`vmin`** ：表示浏览器视窗宽度和高度的最小值，如果浏览器视窗宽度大于其高度，则该值将根据浏览器视窗的高度计算，反之则根据浏览器视窗宽度计算。
-   **`vmax`** ：表示浏览器视窗宽度和高度的最大值，如果浏览器视窗宽度大于其高度，则该值将根据浏览器视窗的宽度计算，反之则根据浏览器视窗高度计算

  


其中 `vmin` 和 `vmax` 要更智能一些：

  


-   `vmin` 等于 `vw` 或 `vh` 中较小一个
-   `vmax` 等于 `vw` 或 `vh` 中较大一个

  


假设你给一个元素设置了宽度为 `10vmin` ：

  


```CSS
.element {
    width: 10vmin;
}
```

  


当浏览器视窗宽度小于高度时（例如移动手机竖屏模式时），则 `1vmin` 等于视窗宽度的 `1%` ；当浏览器视窗宽度大于高度时（例如移动手机横屏模式时），则 `1vmin` 等于视窗高度的 `1%` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/986be96fe97d4d88b704e00895b1e34c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=1143&s=98527&e=jpg&b=fefefe)

  


`vmax` 和 `vmin` 刚好相反。

  


```CSS
.element {
    width: 10vmax;
}
```

  


当浏览器视窗宽度大于高度时（例如移动手机横屏模式时），则 `1vmax` 等于视窗宽度的 `1%` ；当浏览器视窗宽度小于高度时（例如移动手机竖屏模式时），则 `1vmax` 等于视窗高度的 `1%` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f2121f0b64b44613b7fae0fdc6fd1a63~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=1143&s=99832&e=jpg&b=fefefe)

  


注意，如果你想创建一些响应式 UI 组件，那么你可以将 `min()` 函数与[容器查询单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-2)（比如 `cqw` 、`cqh` 、`cqi` 、`cqb` 、`cqmin` 和 `cqmax` ）相结合。

  


了解了这些之后，我们将逐步来做一些实验：**如果完全放弃媒体查询（或容器查询），仅依靠** **`min()`** **函数和视窗单位（或容器查询单位）来创建响应式 Web 布局（或响应式 Web 组件），会发生什么**？

  


## 字体调整

  


CSS 有多种方法可以实现响应式字体大小。其中最老的方式是通过媒体查询来调整元素的字体大小。例如：

  


```CSS
p {
    font-size: 1.1rem;
    
    @media (width >= 1200px) {
        font-size: 1.2rem;
    }
    
    @media (width <= 350px) {
        font-size: 0.9rem;
    }
}
```

  


这种方式虽然有效，但如果用户使用的是 4K 显示器或可折叠手机，将会发生什么呢？

  


不难想象，示例中的媒体查询（`@media`）只考虑到了两个断点，但现实中，这两个断点无法满足所有情景，而且我们也不可能为每一个断点来设置不同的 `font-size` 值。

  


当然，你可能会说，那可以直接通过视窗单位来设置字体大小，比如：

  


```CSS
p {
    font-size: 4vw;
}
```

  


仅使用视窗单位来控制 `font-size` ，你很容易陷入另一个尴尬的境地：

  


-   文本可能过大，导致排版失衡（比如 4K 显示器）
-   文本可能过小，甚至影响可访问性（比如手表显示屏）

  


当然，我们有更好的选择，比如使用 `clamp()` 函数进行限制，它可以确保字体大小不会过小或过大。同时，我们可以混入相对单位（比如 `rem`），以便尊重用户的字体大小偏好。例如：

  


```CSS
p {
    font-size: clamp(1rem, 4vw + 0.5rem, 1.5rem);
}
```

  


目前，使用 `clamp()` 函数来限制字体大小是最受欢迎的选择，而且在社区中有各种在线工具辅助大家使用 `clamp()` 函数来调整字体大小。比如 [@Adrian Bece 提供的现代流体排版编辑器](https://modern-fluid-typography.vercel.app/)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d283230233f24ded880df3176a967263~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1437&s=22838&e=avif&b=1f1e31)

  


> URL：https://modern-fluid-typography.vercel.app/

  


但在这里，我们要全程使用 `min()` 函数，并且只需一行代码，就能完全取代 CSS 媒体查询（或容器查询），从而大幅减少 CSS 代码量：

  


```CSS
p {
    font-size: min(6vmin, 1rem + 0.23vmax);
}
```

  


上面代码中的 `min()` 函数传入两个参数：

  


-   第一个参数是个长度单位值，即 `6vmin` ，它代表浏览器视窗宽度和高度中较小的 `6%` 这样字体可以随着较小的视窗缩小，适应更紧凑的布局
-   第二个参数是一个表达式，即 `1rem + 0.23vmax` 。其中 `1rem` 作为基础字体大小，`0.23vmax` 是浏览器视窗宽度或高度中较大的 `0.23%` ，它可以确保字体大小随着浏览器视窗尺寸增大而适度放大

  


`1rem + 0.23vmax` 表达式的和，使得字体大小可以在 `6vmin` 和 `1rem + 0.23vmax` 之间平滑过渡，而不会在浏览器视窗变大时突然变得过大。

  


`min()` 函数将会选取这两个值（`6vmin` 和 `1rem + 0.23vmax`）中较小的那个，确保字体大小适配当前屏幕尺寸：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e3fd080df79541daa48a01967086eedd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1350&h=532&s=2276363&e=gif&f=231&b=495958)

  


> Demo 地址：https://codepen.io/airen/full/gbOEbrO

  


`min()` 函数还允许我们限制文本大小的最大增长，比如将字体最大值设置为 `2rem` ，我们可以像下面这样使用 `min()` 函数：

  


```CSS
p {
    font-size: min(6vmin, 1rem + 0.23vmax, 2rem);
}
```

  


不过，这种方式并不是万能的方法。在实际使用的过程中，我们需要根据元素来调整 `min()` 函数中的值。比如说，上面的代码可能更适合正文文本（如 `<p>` 元素）的字体大小，但对于标题元素（比如 `<h1>`），可能要将参数调大一些，比如：

  


```CSS
h1 {
    font-size: min(7.5vmin, 2rem + 1.2vmax);
}
```

  


不管怎么说，这种字体大小调整方式是完全流动的，并且完全基于 `min()` 函数。在创建响应式 Web 布局时，它完全可以用于调整文本的字体大小，也算得上是一种完美的方案！

  


## 间距调整

  


间距在 Web 布局中至关重要，无论是响应式 Web 布局还是普通布局。在 CSS 中，我们通常会使用 `margin` 来设置元素与元素之间的间距，在 **[Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)** 或 **[Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)** 布局中，还可以[通过 `gap` 属性设置元素之间间距](https://juejin.cn/book/7395119173656903715/section/7438081482993696778)。对于内容与元素边框之间的间距，通常会使用 `padding` 。简单地说，我们需要外边距（`margin` 或 `gap`）和内边距（`padding`）来确保元素之间有足够的空间，使它们不会显得过于拥挤。

  


同样地，为了使间距能更好的适应不同尺寸的浏览器视窗（或容器尺寸），我们可以使用 `min()` 函数来设置间距。因为，`min()` 函数允许我们结合相对单位和绝对单位，从而实现更智能的布局。例如，结合 `px` （绝对单位）和 `vmin` （相对单位）：

  


```CSS
div {
    margin: min(10vmin, 30px);
}
```

  


这样设置的优势是：

  


-   在小屏幕上，`10vmin` 可能小于 `30px` ，所以 `min()` 函数会选择 `10vmin` ，让外边距随浏览器视窗尺寸变小而缩小
-   在大屏幕上，当 `10vmin` 大于 `30px` 时，`min()` 函数会限制最大值为 `30px` ，避免外边距过大影响布局

  


注意，这种方式同样适用于 `gap` 属性。例如：

  


```CSS
.grid {
    display: grid;
    gap: min(10vmin, 30px);
}
```

  


当然，你也可以在 `min()` 函数中使用表达式，但通常情况下不会这样使用，因为外边距不需要无限增长。在大屏幕上，元素之间的间距如果过大，视觉上会显得很奇怪！

  


对于内边距 `padding` ，我们可以直接使用 `min()` 函数，使其大小适配浏览器视窗尺寸的大小。但是，我们有一种更好的方式来设置 `padding` —— 我们直接给 `padding` 属性使用 `em` 单位值，这样它就能相对于元素的 `font-size` 计算，从而能够自动适配：

  


```CSS
.card {
    font-size: min(6vmin, 1rem + .12vmin);
    padding: 1.2em;
}
```

  


在这个示例中，`.card` 的 `font-size` 由 `min()` 函数控制，它将会确保字体大小随浏览器视窗尺寸调整，而 `padding: 1.2em` 将会自动随字体大小（`font-size`）变化，因此它继承了 `min()` 函数的效果，无需额外调整 `padding` ，也有实现响应式间距！

  


这意味着，内边距（`padding`）和字体大小（`font-size`）可以同步缩放，避免手动调整间距的麻烦，同时让排版更自然，更适应不同设备。

  


> 温馨提示，如果你想给元素设置更灵活的间距，可以移步阅读《[如何灵活设置元素之间的间距？](https://juejin.cn/book/7199571709102391328/section/7199845459563642913)》和《[CSS 布局：CSS 的 `gap`](https://juejin.cn/book/7395119173656903715/section/7438081482993696778)》！

  


## 宽度调整

  


需要知道的是，我们在构建 Web 布局时，通常情况下不会给元素显式指定高度（`height` 或 `block-size`），因为它的大小一般会由内容来决定。因此，我们在这里只聊一下元素宽度（`width` 或 `inline-size`）。

  


在创建响应式 Web 布局中，设置宽度并不一定要复杂。我们可以直接使用百分比（`%`）或视窗单位（`vw`）来控制元素占据的水平空间，这样它就能自动适配不同尺寸的浏览器视窗。这也是构建流式布局常用的方式。

  


不过，我们使用 `min()` 函数来设置元素的宽度，可以起到另一个作用——**设置元素的最大宽度，有点类似于** **`max-width`** **或** **`max-inline-size`** **的功能**。这在一些情景中特别有用，它可以确保元素不会变得太大或太小，保持良好的布局。

  


例如，我们希望一个容器（比如 `.container`）默认填充整个浏览器视窗的宽度，但最大宽度不超过 `65ch` 。在不使用 `min()` 函数的时，我们一般会像下面这样写 CSS 代码：

  


```CSS
.container {
    width: 100%;
    max-width: 65ch; /*或者 max-inline-size: 65ch */
    margin-inline: auto;
}
```

  


现在，要是使用 `min()` 函数的话，可以像下面这样写 CSS ：

  


```CSS
.container {
    width: min(100%, 65ch); /* 或者 inline-size: min(100%, 65ch) */
    margin-inline: auto;
}
```

  


上面代码中的 `min()` 函数会使容器（`.container`）：

  


-   在小屏幕（如手机）下，容器宽度为 `100%` ，让它占满整个屏幕
-   在大屏幕（如电脑）下，当 `100%` 超过 `65ch` 时，`min()` 函数会限制容器宽度最多为 `65ch` ，避免内容太宽导致阅读困难

  


我曾在《[CSS 排版：Web 排版技巧](https://juejin.cn/book/7395119173656903715/section/7445257653855223845#heading-8)》中提到过，文本区域太宽会导致阅读困难，因为用户的视线需要来回移动很长距离。研究显示，最佳的行宽约在 `45 ~ 100` 个字符之间。对于长篇内容的行宽，最好控制在 `65 ~ 75` 个字符之间。基于这个原因，我在 `min()` 函数中使用 `ch` 单位来控制文本的最大宽度（即容器的最大宽度）。这样一来，文本既能自适应屏幕，又不会变得过宽，影响阅读体验！

  


## 基于 `min()` 函数的尺寸方案

  


既然 `min()` 函数与CSS 的固定单位和相对单位相结合，可以根据屏幕或容器尺寸来调整字体大小、间距和宽度等，那么我们就可以根据团队的设计规范或经过大量的实验和优化，整理一套预定义的 `min()` 值，并将它们存储在 **[CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)**中。这样就可以用于不同的响应布局，让你的 CSS 代码更灵活，可复用。例如：

  


```CSS
:root {
    /* 适用于不同层级的字体大小 */
    --font-size-6x: min(7.5vmin, calc(2rem + 1.2vmax));
    --font-size-5x: min(6.5vmin, calc(1.1rem + 1.2vmax));
    --font-size-4x: min(4vmin, calc(0.8rem + 1.2vmax));
    --font-size-3x: min(6vmin, calc(1rem + 0.12vmax));
    --font-size-2x: min(4vmin, calc(0.85rem + 0.12vmax));
    --font-size-1x: min(2vmin, calc(0.65rem + 0.12vmax));
  
    /* 适用于不同的最大宽度 */
    --width-2x: min(100vw, 1300px);
    --width-1x: min(100%, 1200px);
  
    /* 适用于网格、间距 */
    --gap-3x: min(5vmin, 1.5rem);
    --gap-2x: min(4.5vmin, 1rem);
  
    /* 适用于大小（可用于 padding、margin、元素尺寸等） */
    --size-10x: min(15vmin, 5.5rem);
    --size-9x: min(10vmin, 5rem);
    --size-8x: min(10vmin, 4rem);
    --size-7x: min(10vmin, 3rem);
    --size-6x: min(8.5vmin, 2.5rem);
    --size-5x: min(8vmin, 2rem);
    --size-4x: min(8vmin, 1.5rem);
    --size-3x: min(7vmin, 1rem);
    --size-2x: min(5vmin, 1rem);
    --size-1x: min(2.5vmin, 0.5rem);
}
```

  


有了这些自定义属性之后，我们可以在需要的地方像下面这样引用 CSS 自定义属性：

  


```CSS
h1 {
    font-size: var(--font-size-6x);
}

.container {
    width: var(--width-2x);
    margin-inline: auto;
}

.cards {
    display: grid;
    gap: var(--gap-3x);
    
    .card {
        padding: var(--size-2x);
    }
}
```

  


这样，我们就实现了：

  


-   标题（`h1`）：根据浏览器视窗大小动态调整字体大小，无需依赖媒体查询
-   容器（`.container`）：自动适应不同设置，而且不会变得过宽
-   卡片容器（`.cards`）：使用 `min()` 动态调整 `gap` ，让布局更灵活
-   卡片（`.card`）：使用 `min()` 动态调整内距，同时让排版更自然，更适应不同设备

  


现在我们再回过头来看课程开头示例中的 CSS 代码：

  


```CSS
html {
    font-size: min(120%,5vw);
}

:is(h1, .product-price) {
    font-size: max(1.25rem, 12cqi - 1rem);
}

main {
    width: min(100% - (var(--margin, 2rem) * 2), var(--max-width, 80rem));
}
```

  


是不是很好理解了！需要提出的是，虽然这节课我们所探讨的是 `min()` 函数，但这些规则同样适用于 CSS 的 `max()` 函数！

  


## `min()` 函数是响应式设计的终极解决方案吗？

  


我想你已经猜到答案了！不管是使用 `min()` 、`max()` 还是 `clamp()` ，都不能说这就是创建响应式 Web 布局的终极解决方案。就像完全依赖媒体查询或容器查询或者完全依赖视窗单位或容器查询单位也不是最优解一样。但是，`min()` 函数提到了 Web 适配的能力，它只是更适用于响应式 Web 布局。

  


换句话说，任何一种方案都不是什么终极解决方案或者说最优方案，我们在开发的过程中，不能固守某一方法，而更多的去探索更多的解决方案，从中找到更适合当前场景的解决方案！比如说，我们之前可能热衷于[使用 `rem` 或 `vw` 来解决移动端的适配问题](https://juejin.cn/book/7161370789680250917/section/7165496907714789407#heading-0)，但随着 CSS 的 `clamp()` 函数的出现，我们可以根据设计图的尺寸、视窗的尺寸以及需要转换的值，通过一些简单的数学运算，[找到更适合解决移动端适配的方案](https://juejin.cn/book/7161370789680250917/section/7165496907714789407#heading-15)：
