![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6ce79603a0504169b9da67123728206f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=563169&e=png&b=3c76ce)

  


`overflow` 是 CSS 中一个基础且常见的属性，你可能知道 `overflow` 属性的值有 `hidden` 、`visible` 、`auto` 和 `scroll` ，但你知道 `clip` 吗？这是 `overflow` 属性的一个相对较新的值。那么它到底是什么的呢？

  


在深入了解 `clip` 之前，让我们快速回顾一下 `overflow` 及其值的作用。

  


## 回顾 `overflow`

  


`overflow` 有多种用途：启用滚动和裁剪内容。简单地说，它允许你控制当内容溢出容器时，如何显示这些内容：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3ff8b1ff99e648e6a680ea0585c56f47~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=754&s=433899&e=jpg&b=fdfdfd)

  


在还没有引入 `clip` 属性值之前，它有四个不同的值：

  


-   `visible` ：默认值，溢出容器的内容会显示出来
-   `hidden` ：溢出容器的内容会被剪切，溢出的部分被隐藏
-   `scroll` ：容器会变成一个可滚动的区域，并显示滚动条，无论内容是否溢出容器
-   `auto` ：根据内容是否适合，元素会自动显示滚动条。通常，你会选择 `auto` 而不是 `scroll` ，因为 `scroll` 一直会显示滚动条

  


新引入的 `clip` 与 `hidden` 有点类似，它也会裁剪溢出容器的内容，但它与 `hidden` 有着很多差异。比如一个设置了 `overflow: hidden` 的元素仍然可以通过 JavaScript 控制滚动，但 `overflow: clip` 是完全禁用了滚动，包括 JavaScript控制的滚动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ea8910f116904baeba47202a6a3474af~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=730&s=3778476&e=gif&f=267&b=fefefe)

  


> Demo 地址：https://codepen.io/airen/full/PwwJdpL

  


另外一方面，使用 `clip` 可以在一个方向上进行裁剪，同时在另一个方向上保持内容可见，而 `hidden` 是做不到的，这也是 CSS 给 `overflow` 新增 `clip` 值的主要原因之一，稍后我们会详细介绍这一点。

  


需要知道的是，`overflow` 是一个简写属性，它允许你为两个方向设置值，你可以分别为水平轴（`x`）和垂直轴（`y`）设置不同的值，其中第一个值为水平轴，第二个值为垂直轴：

  


```CSS
.container {
    overflow: hidden scroll;
}
```

  


你也可以使用它的子属性 `overflow-x` 和 `overflow-y` ，或者它们对应的 **[逻辑属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)** `overflow-inline` 和 `overflow-block` 单独为一个轴设置值：

  


```CSS
.container {
    overflow-y:  auto;
    overflow-x: hidden;
}

/* 等同于 */
.container {
    overflow: hidden auto;
}
```

  


设置两个不同方向的溢出值可以让你配置元素，使其（比如上面的代码）只在垂直方向上滚动，而在水平方向上隐藏溢出内容，这在常见需求中很有用。

  


这就是 `overflow` 最为基础的部分！

  


## 使用 `overflow` 的问题

  


我们平时在使用 `overflow` 控制溢出内容时，总是会碰到各种各样的问题。我曾在《[溢出常见问题与排查](https://juejin.cn/book/7199571709102391328/section/7213705145954074679)》课程中详细阐述了如何排查和解决 `overflow` 的问题。但在这里，我想和大家单独聊聊 `overflow: hidden` 的使用问题，也是大家容易忽视的问题。

  


假设我们给一个容器设置了下面这样的 CSS：

  


```CSS
.container {
    overflow-y: hidden;
}
```

  


在底层，浏览器会默认添加 `overflow-x: auto`，这意味着，上面的代码会等同于：

  


```CSS
.container {
    overflow-y: hidden;
    overflow-x: auto;
}
```

  


这将导致水平方向和垂直方向都被裁剪，并且在水平方向会出现滚动条。同样的，如果你仅是在 `x` 轴设置为 `hidden` ，`y` 轴也会变成 `auto` ；

  


```CSS
.container {
    overflow-x: hidden;
    
    /* 浏览器的计算值为 */
    overflow-x: hidden;
    overflow-y: auto;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ee3f8f7f066d4e7d8c2acffb7a630e3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1662&h=490&s=1535555&e=gif&f=383&b=fefefe)

  


你可能会问，为什么会这样呢？其实 W3C 规范已经对这个现象有如下说明：

  


> 当 `overflow-x` 或 `overflow-y` 中的一个不是 `visible` 或 `clip` 时，`overflow` 的 `visible/clip` 值将分别计算为 `auto/hidden` 。

  


也就是说，当某个方向（比如 `overflow-y`）被设置为 `hidden` 时，另一个方向（`overflow-x`）为默认变成 `auto` ，从而导致两个方向都被裁剪。这也证明，它们之间有一个隐藏的依赖关系：你不能在一个轴上设置为隐藏（`hidden` 、`scroll` 和 `auto` 都限制内容扩展溢出容器），而另一个轴上设置为 `visible` 。

  


这种行为并不总是我们想要的。在某些场景中，我们只希望裁剪一个方向。比如，我们有张描述个人信息的卡片，用户头像位于容器顶部中间。并且只希望在 `x` 轴方向进行裁剪，使卡片背景修饰图不在 `x` 轴方向溢出：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3012c0541034ffc857aa78cd3da2994~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1404&s=548822&e=png&b=717fc9)

  


在没有 `clip` 之前，我们通常会在 `.card` 容器上设置 `overflow-x: hidden` ，这个时候用户头像也会被裁剪：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a921c6cd915946c1be4a490cdbf68f85~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1404&s=475976&e=png&b=717fc9)

  


以往为了实现类似的效果，我们不得不添加额外的 HTML 来进行样式设置。

  


现在，我们使用 `clip` 就能轻易实现我们所需要的效果：

  


```CSS
.card {
    overflow-x: clip;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/71d8e01b271c41a0b1d29773caffe2af~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1404&s=548822&e=png&b=717fc9)

  


> Demo 地址：https://codepen.io/airen/full/dPPZVJE

  


这是因为，当我们在 `overflow-x` 设置为 `clip` 时，`overflow-y` 会保持原样（也就是 `visible`），仅裁剪 `x` 轴方向溢出的内容。

  


```CSS
.container {
    overflow-x: clip; 
    
    /* 浏览器计算出的样式为 */
    overflow-x: clip;
    overflow-y: visible;
}

/* 或者 */
.container {
    overflow-y: clip;
    
    /* 浏览器计算出的样式 */
    overflow-y: clip;
    overflow-x: visible;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fdb34d4a3d09408eb9e4bb67c64fed49~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1660&h=492&s=1372355&e=gif&f=365&b=fefefe)

  


有趣吧？这就是 `clip` 值在 `overflow` 中的工作方式。

### 滚动驱动的动画

  


从本质上讲，创建[滚动驱动的动画](https://juejin.cn/book/7288940354408022074/section/7307223031717724172)非常简单。只需将现有的 [CSS 动画](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)或 [WAAPI 动画](https://juejin.cn/book/7288940354408022074/section/7308623717105008652)与 `ScrollTimeline` 或 `ViewTimeline` 连接，就完成了。比例，在 CSS 中，你可以使用 `animation-timeline` 属性来实现：

  


```CSS
#target {
    animation: fade linear forward;
    animation-timeline: scroll();
}
```

  


大多数的时候，它能按预期工作，但有时它什么也不做。其中罪魁祸首是：在动画目标和你认为自己正在定位的滚动器之间存在一个 `overflow: hidden`。

  


在使用 CSS 滚动驱动动画时，你可能需要使用 `overflow: hidden` 来裁剪容器的内容。但这样做会与滚动驱动动画产生冲突。使用 `overflow: hidden` 会定义一个滚动容器（scroll container）。为了避免这种情况，我们可以改用 `overflow: clip`。**`overflow: clip`** **的工作方式与** **`overflow: hidden`** **完全相同，唯一不同的是，** **`overflow: clip`** **不会创建滚动容器**。

  


下面这个示例，有一个红色的框和绿色的框，左侧红色框的父元素应用了 `overflow: hidden` ，右侧绿色框的父元素应用了 `overflow: clip` 。这两个框同时都应用了一个 `rotate` 动画，并且都是由滚动来驱动它们的动画。你会发现，左侧的框无法找到根滚动器来驱动其动画，而右侧的框则可以：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/35bee02caec545daaa4d88ecffbdb1a5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1320&h=602&s=9534880&e=gif&f=135&b=475756)

`overflow-clip-margin` 属性用于指定元素的内容在被裁剪前可以超出元素框的最大距离。这个区域被称为溢出裁剪边缘。

  


```CSS
.element {
    height: 100px;
    overflow: clip; /* 必须设置 */
    overflow-clip-margin: 20px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4e1747f9394c44deb65197bd04873d67~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1179&s=410392&e=jpg&b=fefefe)

  


`overflow-clip-margin` 定义的区域会显示为元素框外的一块浅灰色区域。`overflow: clip` 会裁剪掉超出元素框的内容，而 `overflow-clip-margin` 决定这些内容可以在被裁剪前延伸多远。

  


`oveflow-clip-margin` 的语法规则比较简单：

  


```CSS
overflow-clip-margin: <visual-box> || <length [0,∞]>;
```

  


-   `<visual-box>`：当偏移值为 `0` 时，`visual-box` 指定哪个盒子边缘用作溢出裁剪边缘的起点。若未指定，则默认为元素的 `padding-box` （请参阅 《[CSS 布局：CSS 中的各种 *-box](https://juejin.cn/book/7395119173656903715/section/7445257036046991410)》）
-   `<length [0,∞]>`：偏移值，决定溢出裁剪边缘从指定盒子边缘向外延伸的距离。未设置时默认为 `0`。**不允许负值**

  


必须说明一点，要想 `overflow-clip-margin` 起作用，必须设置 `overflow: clip` 。

  


### 增加标签列表的边距

  


假设你有一个标签列表，并且在列表容器上设置了 `overflow-x: clip` 来避免当某个标签过长时，内容溢出：

  


```CSS
.badges {
    display: flex;
    flex-wrap: wrap;
    gap: .5rem;
    overflow-x: clip;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/96ebb890b39b43e7b962861eb571f7c4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=988&s=97592&e=png&b=ffffff)

  


与此同时，我们在徽标上设置了一个悬浮效果：

  


```CSS
li:hover {
    outline: solid 4px #203f87;
}
```

  


当用户鼠标悬浮在标签上时，靠近元素 `.badges` 边缘的徽标会出现被裁剪掉的现象：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/25ad03da8bc24451b47159725b0296fe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=490&s=702641&e=gif&f=225&b=ffffff)

  


在这之前，我们一般通过给 `.badges` 元素设置一个内距（`padding`）来避免这种现象出现：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/20ae0c6c90814ac4b10b440434aa709e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1140&h=488&s=344495&e=gif&f=92&b=ffffff)

  


但这样做，又会违背设计的初衷——设计师可能不希望徽标与容器框之间有额外的间距存在。

  


现在，我们可以通过在徽标的父元素 `.badges` 上使用 `overflow-clip-margin` 来修复这个问题：

  


```CSS
.badges {
    overflow-x:clip;
    overflow-clip-margin: 4px;
}
```

### 嵌套圆角

  


在《[CSS 图形：带圆角的形状](https://juejin.cn/book/7395119173656903715/section/7401099842379841545)》课程中介[绍嵌套的圆角](https://juejin.cn/book/7395119173656903715/section/7401099842379841545#heading-1)时，曾介绍了使用 `overflow-clip-margin` 来实现嵌套的圆角的技巧：

  


```CSS
.card {
    --p: 10px;
    --r: 20px;
    overflow: clip;
    border-radius: var(--r);
    
    &:nth-child(1) {
        padding: var(--p);
        overflow-clip-margin: content-box;
    }
    
    &:nth-child(2) {
        border: var(--p) solid transparent;
        overflow-clip-margin: padding-box;
    }
}
```

  


使用这个技巧，子元素（`<picture>` 和 `<footer>`）根本不需要设置自己的 `border-radius` 。它们会根据父容器进行裁剪，基本上就像 `overflow: hidden` 。注意，第一张卡片，我们通过内距 `padding` 来设置父子元素之间的间距，因此 `overflow-clip-margin` 的值为 `content-box` ，告诉浏览器裁剪现在发生在内边距内；而第二张卡片，我们通过边框 `border` 来设置父子元素之间的间距，因此 `overflow-clip-margin` 的值设置为 `padding-box` ，告诉浏览器裁剪现在发生在边框内。

  


你最终看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/657f13c5734a49eda36367e2a83a8255~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=498&s=11861505&e=gif&f=98&b=30595e)

  


> Demo 地址：https://codepen.io/airen/full/xxopNVK

  


注意，使用这种方法时，当 `--r` 小于或等于 `--p` 时，嵌套元素不会有圆角效果，相当于 `border-radius` 为 `0` 。

## 小结

  


正如课程中的示例所展示的那样，在隐藏溢出容器的内容时，使用 `overflow: clip` 要比 `overflow: hidden` 更有用。同时与 `overflow-clip-margin` 属性结合在一起使用的时候，你还可以灵活的控制溢出边缘。

  


简言之，`overflow: clip` 告诉浏览器：任何超出元素范围的内容都应该被裁剪掉——这和 `overflow: hidden` 类似。但不同的是，`clip` **完全禁止滚动**（无论是用户操作还是通过脚本），而且该元素**不会变成滚动容器**，也不会开启新的格式化上下文。也就是说，不会自动出现滚动条等行为。当你使用 `overflow: clip` 裁剪元素时，`overflow-clip-margin` 可以让元素内容超出边界一点点，但在设置的范围之外依然会被裁剪。
