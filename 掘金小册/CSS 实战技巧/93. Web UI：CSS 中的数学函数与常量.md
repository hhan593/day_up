在 CSS 中，除了一些常规的样式函数之外（比如颜色函数 `color-mix()` 、渐变函数 `linear-gradient()` 、属性函数 `attr()` 、滤镜函数 `filter()` 等）还有一些数学函数。比如我们熟悉的 `calc()` 函数，**[比较函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)** （`min()` 、`max()` 和 `clamp()` ）和 **[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)**（`sin()` 、`cos()` 和 `tan()`）。这些函数可以直接在 CSS 进行数学运算，它们可以用于计算尺寸、角度、透明度、颜色等属性，极大的增强了 CSS 的可编程性，为 Web 页面样式的精细化控制带来了前所未有的灵活性。

  


除此之外，CSS 还添加了许多新的数学函数，比如四舍五入（`round()`）、取余数（`rem()`）、取模数（`mod()`）、幂运算（`pow()`）、平方根（`sqrt()`）、指数计算（`exp()`）和对数计算（`log()`）等。这些函数进一步填补了 CSS 中数学函数的空缺。

  


这些数学函数为 Web 设计带来了极大的便利和创意空间。无论是处理复杂的布局、实现生动的动画效果，还是优化代码的可维护性，数学函数都扮演着不可或缺的角色。掌握这些数学函数，将使 Web 开发者能够创建出更高效、灵活且富有交互性的 Web 界面。

  


接下来，我们一起来探索这些新的数学函数的具体工作方式。

  


## 四舍五入：`round()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1e6402ff6649422fbe7ffdef5e559366~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=427756&e=webp&b=876c4e)

  


CSS 的 `round()` 函数与 JavaScript 中的 `Math.round()` 有点相似，它接受一个数值参数，并返回最接近的整数。例如：

  


```JavaScript
console.log(Math.round(2.2));    //  👉 2
console.log(Math.round(14.82));  //  👉 15
console.log(Math.round(5.5));    //  👉 6
```

  


类似地，CSS 中的 `round()` 函数也具有相同的行为，但它接受两个参数：

  


```CSS
.round {
    property: round(需要取整的值, 取整间隔)
}
```

  


-   **需要取整的值**：需要进行取整的数值或数学表达式
-   **取整间隔**：四舍五入时的间隔大小

  


在数学或 JavaScript 计算中，取整间隔（精度）通常是 `1` ，因为我们通常希望得到整数。如果我们希望在 CSS 中实现 JavaScript 的 `Math.round()` ，可以像下面这样写：

  


```CSS
line-height: round(2.2, 1);   /*  👉 2 */
line-height: round(14.82, 1); /*  👉 15 */
line-height: round(5.5, 1);   /*  👉 6 */
```

  


由于 CSS 需要处理各种单位和不同的度量方式，因此规范允许我们根据实际需求选择合适的方式来舍入。例如，在处理 `opacity` 这样不带单位的属性值时，它的合理范围是 `0 ~ 1` 。如果直接四舍五入为整数，那么结果只能是 `0` 或 `1` ，这可能不是我们想要的。但如果你希望使用不同的间隔（比如以 `0.1` 为间隔增加），我们可以将 `round()` 函数的间隔值设置为 `0.1` ：

  


```CSS
opacity: round(.56, 0.1); /* 👉 0.6 */
opacity: round(.54, 0.1); /* 👉 0.5 */
```

  


同样的逻辑也适用于带有单位的属性。例如，我们希望旋转的角度始终是 `45deg` 的倍数，可以这样做：

  


```CSS
rotate: round(20deg, 45deg); /* 👉 0deg */
rotate: round(30deg, 45deg); /* 👉 45deg */
rotate: round(80deg, 45deg); /* 👉 90deg */
```

  


CSS 甚至允许混合相同类型的单位，例如角度单位 `deg` 、`turn` 和 `rad` ，它们可以互换使用：

  


```CSS
rotate: round(20deg, .125turn); /* 👉 0deg */
rotate: round(80deg, .125turn); /* 👉 90deg */
```

  


CSS 的 `round()` 函数除了前两个参数（**需要取整的值和取整间隔**）之外，还支持一个可选的**舍入策略**参数，作为第一个参数：

  


```CSS
.round {
    property: round(<舍入策略>， 需要取整的值，取整间隔);
}
```

  


默认情况下，它会使用 `nearest` （四舍五入到最接近的值），所以以下两种写法是等效的：

  


```CSS
rotate: round(22.8deg, 1deg);          /* 👉 23deg */
rotate: round(nearest, 22.8deg, 1deg); /* 👉 23deg */
```

  


我们在使用 `round()` 函数时，可以选择不同的舍入策略：

  


-   `up` （向上取整）：向更大的数取整（负数会变得更“正”），类似于 JavaScript 的 `Math.ceil()`
-   `down` （向下取整）：向更小的数取整（负数会变得更“负”），类似于 JavaScript 的 `Math.floor()`
-   `nearest` （四舍五入）：这是默认值，四舍五入到最接近的整数倍。若刚好在两个目标值中间，则向上取整。类似于 JavaScript 的 `Math.round()`
-   `to-zero` （向零取整）：取最接近 `0` 的整数倍（正数减小，负数变得“更不负”），类似于 JavaScript 的 `Math.trunc()`

  


```CSS
/* 向上取整 */
font-size: round(up, 20.01rem, 1rem); /* 👉 21rem */
rotate: round(up, 83deg, 5deg);       /* 👉 85deg */
rotate: round(up, -83deg, 5deg);      /* 👉 -80deg */

/* 向下取整 */
font-size: round(down, 20.999rem, 1rem); /* 👉 20rem */
rotate: round(down, 83deg, 5deg);        /* 👉 80deg */
rotate: round(down, -83deg, 5deg);       /* 👉 -85deg */

/* 四舍五入 */
font-size: round(20.999rem, 1rem); /* 👉 21rem */
rotate: round(83deg, 5deg);        /* 👉 85deg */
rotate: round(-83deg, 5deg);       /* 👉 -85deg */

/* 向零取整 */
font-size: round(to-zero, 20.01rem, 1rem); /* 👉 20rem */
rotate: round(to-zero, 83deg, 5deg);       /* 👉 80deg */
rotate: round(to-zero, -83deg, 5deg);      /* 👉 -80deg */
```

  


需要知道的是 `to-zero` 的行为与 `down` 类似，但**正数向下取整，负数向上取整**（即趋向 `0`）:

  


```CSS
rotate: round(to-zero, 83deg, 5deg);   /* 👉 80deg */
rotate: round(to-zero, -83deg, 5deg);  /* 👉 -80deg */
```

  


对比其他策略的差异：

  


```CSS
rotate: round(up, 83deg, 5deg);       /* 👉 85deg */
rotate: round(up, -83deg, 5deg);      /* 👉 -80deg */

rotate: round(down, 83deg, 5deg);    /* 👉 80deg */
rotate: round(down, -83deg, 5deg);   /* 👉 -85deg */

rotate: round(83deg, 5deg);         /* 👉 85deg */
rotate: round(-83deg, 5deg);        /* 👉 -85deg */
```

  


简单的小结一下。CSS `round()` 函数提供了一种灵活的方式来进行数值舍入，它支持：

  


-   **两个参数**：需要取整的值和取整间隔（支持不同的 CSS 单位）
-   **可选的第三个参数**：舍入策略，可选择 `nearest`（默认）、`up`、`down` 和 `to-zero`

  


这意味着**最终的需要取整的值会根据指定的四舍五入策略，调整为最接近的取整间隔的整数倍**。

  


CSS `round()` 函数适用于需要按照特定间隔对数值进行舍入的场景，特别是在动态计算、响应式设计和视觉精度控制方面有很大用处。我们来看两个有关于这方面的具体案例。

  


当元素的大小、间距依赖于某些计算时，`round()` 可以保证其值避免出现小数像素值，提升对齐的精度。来看一个具体的示例：

  


```CSS
.box {
    --s: 1em;
    width: calc(round(50vmin, var(--s)) - 0.5 * var(--s));
    aspect-ratio: 1;
    border: solid calc(1.5 * var(--s)) transparent;
    background: repeating-conic-gradient(#000 0% 25%, #0000 0% 50%) 0 0 / var(--s) var(--s);
    mask: 
        linear-gradient(#000 0 0) exclude, 
        linear-gradient(#000 0 0) padding-box;
}
```

  


上面代码中的 `round(50vmin, var(--s))` 会将 `50vmin` 四舍五入到最接近于 `var(--s)` （即 `1em`）的整数倍。这意味着宽度（`width`）将被调整为 `1em` 的倍数。示例中最终宽度为 `round(50vmin, var(--s))` 减去 `0.5em`，确保宽度适合视口并且是一个与 `--s`（`1em`）对齐的值。

  


简单地说，元素的宽度根据视口大小动态计算，并四舍五入到最接近 `1em` 的值，确保无论在什么设备或屏幕尺寸下，宽度都保持一致。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8be07c5364c4e2eb514fdcaa9e4e6d9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1290&h=752&s=2137514&e=gif&f=173&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/EaxemLL

  


另一个示例是使用 `round()` 来控制 `font-size` 。在某些情况下，你的 `font-size` 是基于某些计算的（例如 `em` 、`vw` 等），浏览器计算出来的值可能会导致小数像素值，影响渲染和可读性。使用 `round()` 函数定义步长并在特定范围内获得精确的值。其中典型的案例就是流体排版，通常我们会像下面这样来设置 `font-size` ：

  


```CSS
p {
    font-size: clamp(1rem,1rem + 3vw,2.5rem);
}
```

  


现在，我们使用 `round()` 可以像下面这样设置 `font-size` ：

  


```CSS
p {
    font-size: clamp(1rem,round(down,1rem + 3vw,2px),2.5rem);
}
```

  


或者更友好的写法：

  


```CSS
p {
    --fluid: clamp(1rem,1rem + 3vw,2.5rem);
    font-size: round(down,var(--fluid),2px);
}
```

  


默认情况下 `1rem` 等于 `16px` ，那么 `2.5rem` 就等于 `40px` ，`round()` 函数使得 `font-size` 从 `16px` 到 `40px` 之间取值时，会按步长为 `2px` 取值，并且向下取值。

  


在同一视口宽度下（比如 `760px`），我们来对比两种写法最终计算出来的 `font-size` 值：

  


```CSS
font-size: clamp(1rem,1rem + 3vw,2.5rem);                 /* 👉 38.8px */
font-size: clamp(1rem,round(down,1rem + 3vw,2px),2.5rem); /* 👉 38px*/
```

  


正如你所看到的，`round()` 可以保证字体大小更符合设计规范，避免字体大小的小数误差，在使用动态计算的 `font-size` 在响应式布局中更稳定。

  


## 余数 `rem()` 和模数 `mod()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6f2945df153d42d7b3022b1f110c9b7b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=541162&e=webp&b=4d6487)

  


CSS 的 `rem()` 和 `mod()` 函数的式作方式与 JavaScript 余数运算符（`%`）类似。现在，我们可以在 CSS 中：

  


-   使用 `rem()` 函数来计算余数
-   使用 `mod()` 函数来返回模数

  


> **特别声明，** **`rem()`** **函数与** **`rem`** **单位没有关点关系！**

  


### CSS `rem()` 函数

  


CSS 的 `rem()` 函数主要用于返回两个数相除后的余数。余数是被除数除以除数后的剩余部分，它始终保持被除数的符号。

  


我们可以通过数学上的余数来理解 CSS 的 `rem()` 函数。数学上的余数概念来源于除法，表示当一个数不能被另一个数整除时剩余的部分。例如 `9 ÷ 4` ，其中被除数是 `9` ，除数是 `4` ，其中 `9` 不是 `4` 的倍数，因此 `4` 无法被 `9` 整除。这个除法运算的结果：商是 `2` ，余数是 `1` 。

  


在 JavaScript 中，我们可以使用余数运算符（`%`）来计算余数：

  


```JavaScript
console.log(9 % 4);     //  👉 1
console.log(5 % 4.1);   //  👉 0.9
console.log(1003 % 5);  //  👉 3
```

  


现在，我们在 CSS 中可以使用 `rem()` 函数来计算余数。它接受两个参数，与 JavaScript 的 `%` 运算符类似：

  


```CSS
.rem {
    property: rem(dividend, divisor)
}
```

  


其中第一个参数是被除数（`dividend`），第二个参数是除数（`divisor`），两个参数之间使用逗号分隔。以下 CSS 代码等效于前面的 JavaScript 示例：

  


```CSS
line-height: rem(9, 4);    /* 👉 1 */
line-height: rem(5, 4.1);  /* 👉 0.9 */
line-height: rem(1003, 5); /* 👉 3 */
```

  


由于 `rem()` 是 CSS 函数，我们还需要考虑单位。`rem()` 函数中两个参数必须具有**相同的类型、单位维度或百分比**，否则函数无效。例如：

  


```CSS
font-size: rem(10px, 5em);        /* ❌ 无效（px 和 em 不是相同维度）*/
rotate: rem(10deg, 3rad);         /* ✅ 有效（都是角度）*/
transition-delay: rem(100ms, 3s); /* ✅ 有效（都是时间）*/
```

  


这意味着 `rem()` 函数的**两个参数的单位可以不同，但必须属于相同的维度**（例如 `length`、`angle`、`time`、`frequency`）。

  


另外，`rem()` 函数中的两个参数，可以是数值（不带单位），也可以是带有相同维度的单位，还可以是表达式：

  


```CSS
/* 无单位 <number> */
line-height: rem(21, 2);  /* 👉 1 */
line-height: rem(14, 5);  /* 👉 4 */
line-height: rem(5.5, 2); /* 👉 1.5 */

/* 带单位的 <percentage> 和 <dimension> */
margin: rem(14%, 3%);       /* 👉 2% */
margin: rem(18px, 5px);     /* 👉 3px */
margin: rem(10rem, 6rem);   /* 👉 4rem */
margin: rem(26vmin, 7vmin); /* 👉 5vmin */
margin: rem(1000px, 29rem); /* 👉 72px - 假设根 font-size 为 16px */

/* 负数和正数取余 */
rotate: rem(200deg, 30deg);  /* 👉 20deg */
rotate: rem(140deg, -90deg); /* 👉 50deg */
rotate: rem(-90deg, 20deg);  /* 👉 -10deg */
rotate: rem(-55deg, -15deg); /* 👉 -10deg */

/* 计算表达式 */
scale: rem(10 * 2, 1.7);                        /* 👉 1.3 */
rotate: rem(10turn, 18turn / 3);                /* 👉 4turn */
transition-duration: rem(20s / 2, 3000ms * 2);  /* 👉 4s */
```

  


需要知道的是，`rem()` 函数的结果始终继承被除数（`dividend`）的符号。如果被除数是负数，结果也是负数，而除数（`divisor`）的符号不会影响结果。

  


```CSS
line-height: rem(9, 4);   /* 👉 1 */
line-height: rem(-9, 4);  /* 👉 -1 */
line-height: rem(9, -4);  /* 👉 1 */
line-height: rem(-9, -4); /* 👉 -1 */
```

  


`rem()` 函数可用于以下这些场景：

  


-   间距计算：避免小数值误差
-   旋转计算：确保角度保持在特定范围
-   动画时间和步进值控制，比如 `transition-duration`
-   自适应布局：确保宽度或（和）高度更符合网格规则

### CSS `mod()` 函数

  


CSS 的 `mod()` 函数用于返回模数，即被除数（`dividend`）除以除数（`divisor`）后的剩余部分，类似于 JavaScript 取余运算符 `%` 。不同的是，`mod()` 的结果始终具有与除数相同的符号。其计算公式是：

  


```CSS
mod(dividend, divisor) = dividend - Math.floor(dividend / divisor) * divisor
```

  


比如 `mod(21, -4)` ，被除数是 `21` ，除数是 `-4` ，首先计算 `21 ÷ -4 = -5.25` （商为 `-5.25`），然后 `Math.floor(-5.25) = -6`（向下取整为 `-6`），再乘以 `-4` 等于 `24`（即 `-6 × -4 = 24`），最后 `21` 减去 `24` 等于 `-3` （即 `21 - 24 = -3`）。所以 `mod(21, -4)` 返回的结果是 `-3`。

  


简单的说，`mod()` 函数与余数计算密切相关。当被除数和除数的符号相同时，`mod()` 函数与 `rem()` 函数计算结果相同：

  


```CSS
line-height: rem(9, 4);   /* 👉 1 */
line-height: mod(9, 4);   /* 👉 1 */
line-height: rem(-9, -4); /* 👉 -1 */
line-height: mod(-9, -4); /* 👉 -1 */
```

  


但是，`mod()` 函数的计算结果会继承除数的符号，而 `rem()` 函数的计算结果继承的是被除数的符号：

  


```CSS
line-height: mod(9, 4);   /* 👉 1 */
line-height: mod(-9, 4);  /* 👉 1 */
line-height: mod(9, -4);  /* 👉 -1 */
line-height: mod(-9, -4); /* 👉 -1 */
```

  


另外，在 `mod()` 函数中，如果被除数和除数的符号不同，我们需要用不同的思维方式来看待它。例如：

  


```CSS
line-height: rem(-9, 4);  /* 👉 -1 */
line-height: mod(-9, 4);  /* 👉 -3 */
line-height: rem(9, -4);  /* 👉 1 */
line-height: mod(9, -4);  /* 👉 3 */
```

  


在 `rem(9,4)` 计算中，我们通常认为 `9` 能包含多少个 `4` ？在这个例子中，`9` 可以包含两个 `4` （因为 `2 × 4 = 8`），那么余数就是 `9 - 8 = 1`。

  


但在 `mod()` 计算中，当符号不同时，我们会寻找大于被除数的最小倍数：

  


-   `mod(9, -4)`：寻找最接近 `9` 但大于 `9` 的 `-4` 的倍数，`-4 × 3 = -12`，然后计算 `12 - 9 = 3`，结果是 `3`
-   `mod(-9, 4)`：寻找最接近 `-9` 但小于 `-9` 的 `4` 的倍数，`4 × (-3) = -12`，然后计算 `-9 - (-12) = -3`，结果是 `-3`

  


如果你觉得上面的方式不易于理解或记忆，我们可以通过下面两个公式来计算 `rem()` 和 `mod()` 函数的返回值：

  


-   `rem(dividend, divisor)` 可以通过 `dividend - (Math.trunc(dividend / divisor) * divisor)` 计算
-   `mod(dividend, divisor)` 可以通过 ` dividend - (Math.floor(dividend  ``/ divisor``) * divisor)` 计算

  


唯一的差别是，`rem(dividend, divisor)` 函数计算出来的结果继承被除数（`dividend`）的符号，即与被除数（`dividend`）的符号相同；而 `mod(dividen, divisor)` 函数计算出来的结果会继承除数（`divisor`）的符号，即与除数（`divisor`）的符号相同。

  


例如：

  


```CSS
rem(21, -4); /* 👉 3 */
mod(21, -4); /* 👉 -3 */
```

  


`mod()` 函数有一点与 `rem()` 函数相似，被除数和除数可以是数值、带有单位的值或表达式：

  


```CSS
/* 无单位 <number> */
line-height: mod(7, 2);    /* 👉 1 */
line-height: mod(14, 5);   /* 👉 4 */
line-height: mod(3.5, 2);  /* 👉 1.5 */

/* 带单位的 <percentage> 和 <dimension> */
margin: mod(15%, 2%);       /* 👉 1% */
margin: mod(18px, 4px);     /* 👉 2px */
margin: mod(19rem, 5rem);   /* 👉 4rem */
margin: mod(29vmin, 6vmin); /* 👉 5vmin */
margin: mod(1000px, 29rem); /* 👉 72px - 假设根 font-size 为 16px */

/* 负数和正数取模 */
rotate: mod(100deg, 30deg);  /* 👉 10deg */
rotate: mod(135deg, -90deg); /* 👉 -45deg */
rotate: mod(-70deg, 20deg);  /* 👉 10deg */
rotate: mod(-70deg, -15deg); /* 👉 -10deg */

/* 计算表达式 */
scale: mod(10 * 2, 1.7);                       /* 👉 1.3 */
rotate: mod(10turn, 18turn / 3);               /* 👉 4turn */
transition-duration: mod(20s / 2, 3000ms * 2); /* 👉 4s */
```

  


另一个相似点是，`mod()` 函数的两个参数，必须具有相同的类型、单位维度或百分比，否则函数无效。两个参数的单位可以不同，但必须属于相同的维度（例如 `length`、`angle`、`time`、`frequency`）。

  


```CSS
mod(10px, 5em);   /* ❌ 无效（px 和 em 不是相同维度）*/
mod(10deg, 3rad); /* ✅ 有效（都是角度）*/
mod(100ms, 3s);   /* ✅ 有效（都是时间）*/
```

  


## 幂运算: `pow()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/37ef034401c349a4a48865aa9797d733~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=354642&e=webp&b=5a4a5b)

  


在数学中，幂运算允许我们指定一个数字自身相乘的次数。现在在 CSS 中，我们可以使用 `pow()` 函数，它有点类似于 JavaScript 中的 `Math.pow()` ：

  


```JavaScript
console.log(Math.pow(3, 2)); // 👉 3 × 3 = 9
console.log(Math.pow(3, 3)); // 👉 3 × 3 × 3 = 27
console.log(Math.pow(2, 4)); // 👉 2 × 2 × 2 × 2 = 16
```

  


上面的代码，在 CSS 中，我们可以像这样写：

  


```CSS
line-height: pow(3, 2); /* 👉 9 */
line-height: pow(3, 3); /* 👉 27 */
line-height: pow(2, 4); /* 👉 16 */
```

  


CSS 的 `pow()` 函数接受接受两个逗号分隔的参数：

  


```CSS
.pow {
    property: pow(base, number);
}
```

  


-   **`base`**：解析为 `<number>`，表示基数
-   **`number`**：解析为 `<number>`，表示指数

  


它返回一个 `<number>`，表示 `base` 的 `number` 次方，即 `base^number`。

  


`pow()` 函数与其他的 CSS 数学函数有所不同，**它只有处理纯数字，不能直接使用带单位的值**。因此，如果要在 CSS 中使用带单位的值，需要与 `calc()` 相结合：

  


```CSS
.element {
    font-size: calc(1rem * pow(2, 2)); /* 👉 1rem * 4 = 4rem */
    rotate: calc(5deg * pow(2, 5));    /* 👉 5deg * 32 = 160deg */
}
```

  


[在 W3C 规范中提供了一个示例](https://drafts.csswg.org/css-values/#example-bebe620e)，利用 `pow()` 函数创建字体大小的**模数比例**（Modular Scale），即通过固定比例（`ratio`）确定页面上的所有字体大小，使它们保持一致的视觉层级。

  


```HTML
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```

  


```CSS
:is(h1, h2, h3, h4, h5, h6) {
    font-size: calc(1rem * pow(1.5, var(--exp)));
}


h1 {
    --exp: 4;
}
h2 {
   --exp:3;
}
h3 {
    --exp:2;
}
h4 {
    --exp:1;
}
h5 {
    --exp:0;
}
h6 {
    --exp:-1;
}
```

CSS 的 `pow()` 函数在基于[宽高比](https://juejin.cn/book/7223230325122400288/section/7259316040151236666)（`aspect-ratio`）的布局中也非常有用，尤其是在需要动态高整元素的宽高比时。结合 [CSS 的 aspect-ratio](https://juejin.cn/book/7223230325122400288/section/7259316040151236666) 、`calc()` 和 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)，可以创建自适应响应式布局。

  


例如，我们可以使用 `pow()` 计算元素的宽度，使其根据高度自动调整，而且保持固定的宽高比：

  


```CSS
.container {
    --h: 100px;
    --ratio: 16 / 9;
    width: calc(var(--h) * pow(var(--ratio), 1));
    height: var(--h);
    background: #f90;
}
```

## 平方根：`sqrt()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/66cae5c1eeb04d8e9dae7a5207349a9d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=507878&e=webp&b=4b283e)

  


有时候我们在使用 CSS 绘制图形的时候，可能需要根据直角的边求出斜边的长度。除了使用三角函数之外，还可以使用平方根函数 `sqrt()` 。它类似于 JavaScript 的 `Math.sqrt()` ：

  


```JavaScript
console.log(Math.sqrt(4));  // 👉 2
console.log(Math.sqrt(9));  // 👉 3
console.log(Math.sqrt(12)); // 👉 约 3.4641
```

  


上面的代码在 CSS 中可以像下面这样写：

  


```CSS
line-height: sqrt(4);  /* 👉 2 */
line-height: sqrt(9);  /* 👉 3 */
line-height: sqrt(12); /* 👉  约 3.4641 */
```

  


`sqrt()` 函数接受一个参数，并且像 `pow()` 函数一样，仅支持纯数字（不支持带单位的值）。

  


```CSS
.sqrt {
    property: sqrt(x)
}
```

  


-   `x` 一个计算结果为大于或等于 `0` 的 `<number>`

  


`sqrt(x)` 会返回 `x` 的平方根，具体规则如下：

  


-   如果 `x` 是 `+∞` （无穷大），返回 `+∞`
-   如果 `x` 是 `0⁻`（负零），返回 `0⁻`
-   如果 `x` 小于 `0`，返回 `NaN`（不是一个数字）

  


`sqrt()` 与 `pow()` 函数类似，如果要在 CSS 中使用带单位的值，需要与 `calc()` 相结合：

  


```CSS
width: calc(100px * sqrt(9));  /* 👉 300px */
```

  


如果需要计算其他根值（如立方根等），可以使用 `pow()` 函数，通过指数的小数表示根值。例如，指数 `1/3` 表示立方根：

  


```CSS
line-height: pow(4, 0.5);   /* 👉 4 的平方根：2 */
line-height: pow(4, 1/2);   /* 👉 4 的平方根：2 */
line-height: pow(27, 1/3);  /* 👉 27 的立方根：3 */
line-height: pow(2, 3/2);   /* 👉 2 的立方后再求平方根，约 2.8284 */
```

  


在 CSS 中，我们通常会利用 `pow()` 和 `sqrt()` 函数来创建图形，比如下面这个示例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6a3bf046a1e34f51a07a308f73ca7067~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1360&h=648&s=1530166&e=gif&f=158&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/yyLxWaO

  


示例中有三个图形，即圆、三角形和正方形，它们各自沿着自己的形状围绕原点运动。

  


对于圆和正方形，它们的运动方式并不复杂（这里就不详细阐述，但你可以查看代码）。然而，三角形稍微复杂一些，因为动画需要按照三角形的三个顶点运动。

  


为了实现这一点，我们需要确定三角形三个顶点的 `x` 和 `y` 坐标，并且让三角形的中心位于 `(0,0)`。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6b0d6d6d763947dd86b4be7fb9ff1cdb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1051&s=135002&e=jpg&b=ffffff)

  


`x` 坐标的计算相对简单：

  


-   顶部的点在 `x=0` 处
-   底部的两个点各自距离中心 `n/2` ，其中 `n` 代表三角形的边形。要是 `n=16px` ，那么这两个点的 `x` 坐标分别是 `-8px` 和 `8px`

  


`y` 坐标的计算稍微复杂一些。我们首先需要计算三角形的高度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/45a147f74b0c46c5ad3a09cf463435c6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1051&s=148843&e=jpg&b=ffffff)

  


假设我们这是一个等边三角形，在这个三角形中：

  


-   `C` 是 `n` ，因为它是最长的边（等边三角形的边）
-   `A` 是 `n / 2` ，也就是底边的一半
-   `B` 代表三角形的高，我们需要计算它的值

  


我们可以根据勾股定理 `A² + B² = C²` 可以得到 `B` :

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9e73a32b9fdb4c04a49becc7c7f8ac1b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3138&h=310&s=42653&e=jpg&b=ffffff)

  


我们用 CSS 的变量 `--s` 来表示 `n` 。同时，我们还定义了 `--_s` ，它等于 `calc(var(--s) * 1rem)` ，因为在 CSS 里，某些数学计算需要纯数值，而 `--_s` 主要用于带单位的计算。这种方式可以避免在不同场景下手动添加或移除单位。

  


在 CSS 中，我们使用 `pow()` 函数来计算平方，并用 `sqrt()` 计算平方根。我们把计算的高度存储到 `--height` 变量中。比如：

  


```CSS
.triangle {
    --w: 1rem;
    --s: 3;
    --_s: calc(var(--w) * var(--s));
    --height: calc(
        var(--w) * (sqrt(pow(var(--s), 2) - pow(calc(var(--s) / 2), 2)))
    );
}
```

  


由于运动是相对于原点 `(0,0)` 进行的，因此三角形的顶点火会向上或向下移动高度的一半。我们把这个值存储到 `--midpoint` 变量里，计算方式如下：

  


```CSS
.triangle {
    --w: 1rem;
    --s: 3;
    --_s: calc(var(--w) * var(--s));
    --height: calc(
        var(--w) * (sqrt(pow(var(--s), 2) - pow(calc(var(--s) / 2), 2)))
    );
    --midpoint: calc(var(--height) / 2);
}
```

  


有了 `--s` 和 `--midpoint` ，我们就可以确定三角形的三个顶点坐标：

  


-   **左下角顶点**：`(-(s / 2), -midpoint)`
-   **顶部顶点**：`(0, midpoint)`
-   **右下角顶点**：`(s / 2, -midpoint)`

  


把这些都组合在一起后，三角形可以像三角形一样运动了！同时，圆形在旋转，正方形……按自己的方式移动。完整的 CSS 代码如下：

  


```CSS
.shapes {
    --w: 1rem;
    --s: 3;
    --_s: calc(var(--s) * var(--w));
    align-items: center;
    display: flex;
    gap: calc(var(--_s) * 2);
}

.shape {
    height: var(--_s);
    aspect-ratio: 1;
    background: var(--c);
}

.circle {
    animation: 2s circle infinite linear;
    border-radius: 50%;
}

.triangle {
    --height: calc(
        var(--w) * (sqrt(pow(var(--s), 2) - pow(calc(var(--s) / 2), 2)))
    );
    --midpoint: calc(var(--height) / 2);

    width: var(--_s);
    height: var(--height);
    clip-path: polygon(50% 0%, 100% 100%, 0% 100%);
    animation: 2s triangle infinite linear;
}

.square {
    animation: 2s square infinite linear;
}

@keyframes circle {
    0% {
        transform: rotate(0deg) translateX(var(--_s)) rotate(0deg);
    }
    100% {
        transform: rotate(360deg) translateX(var(--_s)) rotate(-360deg);
    }
}

@keyframes triangle {
    0% {
        translate: calc(var(--_s) / -2) var(--midpoint);
    }
    33% {
        translate: 0 calc(var(--midpoint) * -1);
    }
    67% {
        translate: calc(var(--_s) / 2) var(--midpoint);
    }
    100% {
        translate: calc(var(--_s) / -2) var(--midpoint);
    }
}

@keyframes square {
    0% {
        translate: calc(var(--_s) * -1) calc(var(--_s) * -1);
    }
    25% {
        translate: var(--_s) calc(var(--_s) * -1);
    }
    50% {
        translate: var(--_s) var(--_s);
    }
    75% {
        translate: calc(var(--_s) * -1) var(--_s);
    }
    100% {
        translate: calc(var(--_s) * -1) calc(var(--_s) * -1);
    }
}
```

如果你阅读过小册之前的《[CSS 图形：绘制三角形](https://juejin.cn/book/7395119173656903715/section/7397319555476881460)》，你会发现我们在[绘制等边三角形](https://juejin.cn/book/7395119173656903715/section/7397319555476881460#heading-2)时，就应用了 `sqrt()` 函数。

  


在 CSS 中，还新增了一些其他指数相关的函数，比如 `exp()` 函数，用于计算数学常数 `e` （约等于 `2.718`）的幂值，类似于 JavaScript 中的 `Math.exp()` ：

  


```CSS
line-height: exp(3); /* 👉 相当于 pow(e, 3) */
```

  


还有一个对数运算函数 `log()` ，它用于计算对数，接受两个参数（第二个参数可选）。如果只有一个参数，默认计算自然对数（以 `e` 为底）：

  


```CSS
opacity: log(2);     /* 👉 以 e 为底的 log(2) = 0.693147 */
opacity: log(2, 10); /* 👉 以 10 为底的 log(2) = 0.30103 */
```

  


在 JavaScript 中，如果想要指定底数，可以用 `Math.log(number) / Math.log(base)` 来实现类似的功能。

  


## 计算平方和的平方根：`hypot()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e505a988362547c8bea4bf268b9125eb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=422190&e=webp&b=4b414d)

  


在几何学中，直角三角形的斜边长度可以用勾股定理计算：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1000b520add44e4941aee9ffe8a6e9b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=675&s=47368&e=png&b=ffffff)

  


我们可以得到斜边长度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7e617b9b4cc649e2bfe397823651b1e3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2842&h=396&s=37099&e=jpg&b=ffffff)

  


现在，CSS 提供了 `hypot()` 函数（类似 JavaScript 的 `Math.hypot()`），它可以接受多个参数，计算它们的平方和再取平方根。假设 `a = 3` ，`b = 4` ，那么我们可以像下面这样写：

  


```CSS
c = hypot(3,4); /* 👉 5 */
```

  


它的计算过程如下所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3ff02077f5ee4494b6490cb91684bb34~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2776&h=336&s=65827&e=jpg&b=ffffff)

  


`hypot()` 函数的参数可以是数值也可以是带同一维度单位的值，比如：

animation-iteration-count: hypot(2);    /* 👉 2 */

## CSS 数学常量

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f0ff3d5e786243fc961212f76117a476~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=420590&e=webp&b=40363e)

  


CSS 除了新了许多数学函数之外，还新增了一些数学常量，比如 `pi` 、`e` 、`infinity` 和 `NaN` 等。这些常量可以用于 `calc()` 等数学函数中。这些数学常量为 CSS 提供了更强大的计算能力，使得复杂的数学计算变得更加直观和易读！

  


### CSS 中的 `pi`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f4b0e196eb3e4e4fac842a0be87df491~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=444496&e=webp&b=353338)

  


在 CSS 的数学函数（如 `calc()` 、`pow()` 和 `round()` 等）中，`pi` 代表数学常数 `π` （约等于 `3.142`）。就像 JavaScript 中的 `Math.PI` 一样，我们可以直接使用它，而不用手动输入近似值：

  


```CSS
width: calc(pi * 3px);       /* 👉 约 9.42px */
width: calc(pi * 1rad);      /* 👉 约 3.14rad = 180deg */
line-height: calc(pi * .5);  /* 👉 约 1.57 */
```

  


需要注意的是，所有这些数学常数都必须在数学函数内使用，否则它们会被解析为普通的字符串，比如 `animation-name: pi` 。切记，要是想把 `pi` 当作数学中的 `π` 使用，就必须使用数学函数包裹：

  


```CSS
line-height: pi;                         /* ❌ 无效 */
animation-name: pi;                      /* ✅ 作为字符串有效，可用于 @keyframes pi*/
animation-iteration-count: pi;           /* ❌ 无效 */
animation-iteration-count: calc(pi);     /* ✅ 有效，约 3.14 */
animation-iteration-count: round(pi, 1); /* ✅ 有效，结果为 3 */
line-height: pow(pi, 2);                 /* ✅ 有效，约 9.87 */
```

  


在 CSS 中使用 `pi` 的主要优势在于它能提高代码的可读性，而不是数学上的精确性。例如，在使用弧度（`rad`）定义角度时，`pi` 会使 CSS 代码更直观。假设要让元素旋转半圈，我们通常会像下面这样写：

  


```CSS
.rotate {
    rotate: 3.142rad; /* 👉 1 * pi */
}
```

  


但现在，我们可以这样写：

  


```CSS
.rotate {
    rotate: calc(pi * 1rad);
}
```

  


注意，以下这些方式都可以使元素旋转半圈：

  


```CSS
rotate: 180deg;
rotate: .5turn;
rotate: calc(pi * 1rad);
rotate: 3.142rad;
```

  


此外，我们还可以使用 `pi` 直接计算圆的周长。例如，圆的周长公式是 `2*π*r` ，在 CSS 中可以直接用 `calc()` 计算出来（即 `calc(2*pi*var(--r))`），在使用 `stroke-dasharray` 制作动画时，就显得非常有用！

  


### CSS 中的 `infinity`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9166ba8a70fd46709f365e1210975350~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=933&s=48816&e=jpg&b=010101)

  


在 CSS 中，有两个可以用来表示无穷大的常数：

  


-   `infinity`（无穷大）： 表示正无穷大的值（`+∞`），它表示最大可能的值
-   `-infinity`（无穷小）： 表示负无穷小的值（`−∞`），它表示最小可能的值，与 `calc(-1 * infinity)` 的结果等同

  


需要注意的是，所有这些关键词都属于 `<number>` 类型的值。

  


与一般 CSS 关键词规则相同，这些关键词不区分大小写。因此，`calc(InFiNiTy)` 是合法的。有一个重要的基本规则：**无穷大（** **`infinity`** **）和无穷小（** **`-infinity`** **）只能在 CSS 函数（比如** **`calc()`** **）中使用**。此外，它们是 `<number>` 值类型，因此要获得无穷大的长度值，需要使用类似 `calc(infinity * 1px)` 的表达式。

  


在 CSS 中，我们还可以通过以下方式来获得与 `infinity` 和 `-infinity` 等同的效果：

  


-   将一个值除以 `0` 将会产生 `+∞` 或 `−∞`，具体取决于标准符号规则。例如，` calc(1  / 0)` 将会产生 `+∞` ，` calc(-1  / 0)` 将会产生 `−∞`。这也适用于单位。因此，你可以使用 `calc(1px / 0)` 得到与 `calc(infinity * 1px)` 相同的值
-   将 `±∞` 添加或减去任何值都会产生相应的无穷大，例如 `calc(infinity + 1px)`
-   将任何值乘以 `±∞` 都会产生相应的无穷大，例如 `calc(infinity * 1px)`
-   将任何值除以 `±∞` 会产生零
-   数学函数中的某些参数组合被定义为产生无穷大，例如，`pow(0, -1)` 产生 `+∞`

  


虽然有这么多种方式可以在 CSS 中获得无穷大或无穷小，但我仍然建议直接使用 `infinity` 或 `-infinity` ，这样能更清晰地表达意图，而不是在样式表中放入一个巨大的数字（或数字值），以使 CSS 更加清晰。

  


CSS 无穷大（`infinity`）最典型的用例应该是 `z-index` 。如果将 `z-index` 的值设置为 `infinity` ，你将获取最大可能值，再也不用担心别人设置值胜过你。例如下面这个示例，无论你在蓝色的卡片上设置多大的 `z-index` 值，它都无法用于设置了 `z-index: calc(infinity)` 的紫色卡片：

  


```HTML
<div class="blue"></div>

<div class="purple"></div>
```

  


```CSS
.blue {
    --z-index: 999999999;
    z-index: var(--z-index);
}

.purple {
    z-index: calc(infinity); 
}
```

  


无论你在输入框中输入多少个 `9` ，最终无法胜过紫色卡片在 `z` 轴上的层级：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/06bf33f8fd9249878de94b551a6e580e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=650&s=722516&e=gif&f=128&b=fefefe)

  


> Demo 地址：https://codepen.io/airen/full/yyLRBWX

  


你可曾知道，`z-index` 的上限值是 `2147483647` 。那么当 `z-index` 属性值 `infinity` 遇到其上限值 `2147483647` 时，又谁将胜出呢？这是一个意思的问题。

  


通常情况之下，`z-index` 属性的 `infinity` 和 `2147483647` 值被视为相同的值，但无法确定 `calc(infinity)` 计算的值就等于 `2147483647` 。我们写个测试案例来验证它。

  


了解 [CSS 的 `z-index` 属性](https://juejin.cn/book/7199571709102391328/section/7204402059225858052)的开发者应该知道。 要比较 `z-index` ，那么它们必须位于相同的堆叠环境中。例如下面这个示例，应用 `z-index` 的两个元素不在同一个堆叠环境中，即使将 `z-index` 属性的值设置为 `infinity` 它也无法胜出：

  


```HTML
<div class="parent">
    z-index: 2;
    <div class="blue">z-index: infinity</div>
</div>

<div class="purple">z-index: 3</div>
```

  


```CSS
.parent {
    z-index: 2;

    .blue {
        z-index: calc(infinity);
    }
}

.purple {
    z-index: 3; 
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9f79b105761a412bb1cebaff9235c087~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2232&h=1444&s=329011&e=png&b=fdfdfd)

  


> Demo 地址：https://codepen.io/airen/full/wBvYwVG

  


上面示例这种情景，很多初学者都认为 `z-index` 失效了，其实这是它的基本规则。有关于这方面更详细的介绍，可以移步阅读《[防御式 CSS 精讲](https://s.juejin.cn/ds/iFr4KCEe/)》中的《[`z-index` 的失效与修复](https://juejin.cn/book/7199571709102391328/section/7204402059225858052)》一文。

  


继续回到 `z-index` 身上。`z-index` 还有另一个规则：“在相同的堆叠环境当中，如果两元素指定的 `z-index` 相同，那么将根据元素在 HTML 源码中出现的顺序来决定，即先出现的先胜。换句话说，如果我们的假设是成功的（`infinity` 比 `2147483647` 大），则无论 HTML 的顺序如何，它都应该位于顶部。

  


因此，我准备了两个堆栈上下文，如下所示，每个堆栈上下文都有不同的 HTML 顺序：


```HTML
<div class="wrapper">
    <div class="blue">z-index: 2147483647</div>
    <div class="purple">z-index: calc(infinity)</div>
</div>

<div class="wrapper">
    <div class="purple">z-index: calc(infinity)</div>
    <div class="blue">z-index: 2147483647</div>
</div>
```

  


```CSS
.wrapper {
    z-index: 0;
}

.blue {
    z-index: 2147483647;
}

.purple {
    z-index: calc(infinity);
}
```

  


将 `.blue` 和 `.purple` 放在相同的堆叠上下文环境中进行比较。如果 `z-index` 的 `infinity` 和 `2147483647` 相同，则按照 HTML 的顺序来决定谁胜出；如果是 `infinity` 大于 `2147483647` ，则设置了 `calc(infinity)` 元素（`.purple`）始终将位于 `z` 轴的顶部。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/13c4bb5c51634ac5878eaa7fa0436ed0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2232&h=1348&s=394309&e=png&b=fcfcfc)

  


> Demo 地址：https://codepen.io/airen/full/vEYVYRL

  


结果已经告诉我们答案了，`z-index` 的值 `infinity` 和 `2147483647` 相等。但这并不能说 `infinity` 就等于 `2147483647` 。听起来有点拗口，甚至晕乎。你可以尝试一下，`calc(infinity - 2147483647)` 结果和 `2147483647` 也一样。这就意味着它已经停止在 `z-index` 限值处。

  


这个案例从侧面又说明，`calc(infinity)` 不一定始终能让 `z-index` 胜出，只不过它的上限值 `2147483647` 并不广为人知，而且也很难记得住这一串数值。也就是说，当有一天，你将 `z-index` 属性的值设置为 `calc(infinity)` ，它并没有胜出的话，你需要知道其中的原委，并做出正确的选择。

  


另一个典型用例就是胶囊按钮或类似于胶囊外形的 UI：

  


以按钮为例，我习惯性会将 `border-radius` 设置为 `999rem` 或 `999vmax` ：

  


```CSS
.pill {
    border-radius: 999vmax;
}
```

  


这样做不管元素高度是多少，都可以实现胶囊 UI 的效果。你现在可以将 `999vmax` 替换成 `calc(1px * infinity)` ：

  


```CSS
.pill {
    border-radius: calc(1px * infinity);
}
```

  


这意味着我们不需要知道元素（矩形框）的尺寸，它也能正常的工作。不过，在某些边缘情况上，会遇到一些奇怪的行为。要是你想进一步了解 `border-radius` 边缘情况会发生什么，可以移步阅读《[你不知道的 `border-radius`](https://juejin.cn/book/7199571709102391328/section/7199845563389444099#heading-3)》。

  


另一个相似的用途可以用于隐藏文本或隐藏元素中。例如在视觉上隐藏元素（仅供屏幕阅读器可见）的代码片段：

  


```CSS
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
    left: -9999vmax;
}
```

  


代码中的 `left: -9999vmax` 可以使用 `calc(-1px * infinity)` 来替代：

  


```CSS
.sr-only {
    /* ... */
    left: calc(-1px * infinity);
}
```

  


从功能上说，没有啥区别。最终效果是相同的。但我认为它确实有助于使代码更具可读性，因为无穷大传达了真正的意图。

  


在《[Web 动画之旅](https://s.juejin.cn/ds/iFrsMxYy/)》的《[提升可访问性动画的关键技巧](https://juejin.cn/book/7288940354408022074/section/7308623518789763083#heading-7)》一文中，着重提到过，我们创建的一些 Web 动画效果对于部分群体是不好的。为了不给这些用户添加不必要的麻烦，我们需要添加下面这样的代码，减少页面上的运动：

  


```CSS
@media (prefers-reduced-motion: reduce) {
    *,
    ::before,
    ::after {
        animation-delay: -1ms !important;
        animation-duration: 1ms !important;
        animation-iteration-count: 1 !important;
        background-attachment: initial !important;
        scroll-behavior: auto !important;
        transition-duration: 0s !important;
        transition-delay: 0s !important;
    }
}
```

  


有了无穷大这个常量之后，你又可以为减少运动添加一个更简单的方法。你只需要将 `transition-delay` 和 `animation-delay` 的值设置无穷大，那么页面上的动画就永远不会开始播放：

  


```CSS
@media (prefers-reduced-motion: reduce) {
    *,
    ::before,
    ::after {
        animation-delay: calc(1s * infinity) !important;
        transition-delay: calc(1s * infinity) !important;
    }
}
```

  


### CSS 中的欧拉数 `e`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd41122b8d584e529a0fb99251fbd134~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=1024&s=2199965&e=png&b=3b7587)

  


CSS 现在提供了数学常数 `e` （欧拉数），它的值约等于 `2.71828` ，常用于对数计算和指数增长。同样地，`e` 也必须在数学函数内使用，例如：

  


```CSS
opacity: calc(e / 3);      /* 👉 约 0.906 */
line-height: pow(e, 2);    /* 👉 约 7.389 */
font-size: calc(e * 1rem); /* 👉 约 2.71828rem */
```

  


但直接使用 `e` 作为数值是无效的：

  


```CSS
opacity: e; /* ❌ 无效 */
```

  


### CSS 中的 `NaN`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/365ba05191c541a5b80dcf6cedb62542~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=1024&s=154721&e=jpg&b=c9d1d1)

  


CSS 还引入了 `NaN` （非数字）常数，这与 JavaScript 中的 `NaN` 类似。当 CSS 计算结果不可能产生数值时，就会返回 `NaN` 。例如，计算 `3` 除以 `0` 会返回 `NaN` ：

  


```CSS
opacity: rem(3, 0); /* 👉 结果为 NaN */
```

  


不过，在 CSS 计算时，最终的渲染值仍然会是有效的。例如：

  


```CSS
.element {
    opacity: 1;
    opacity: calc(NaN);
}
```

  


上面代码中的 `opcity: calc(NaN)` 仍然会被浏览器解析为 `opacity: 0` 。

  


如果 `NaN` 参与计算，例如：

  


```CSS
width: calc(NaN * 1px);
z-index: calc(NaN);
z-index: calc(NaN * infinity);
```

  


那么它的计算结果通常是 `0` 。

  


虽然 `NaN` 在 CSS 规范中有定义，但它的实际用途相对有限，在大多数情况下，你可能并不需要手动使用它。

  


简单的小结一下。CSS 现在支持多个数学常数，包括：

  


-   `pi`（`π ≈ 3.142`）：可用于角度计算、圆的周长等
-   `infinity` 和 `-infinity`（正无穷/负无穷）：适用于 `z-index`、定位计算等
-   `e`（欧拉数 `≈ 2.718`）：可用于对数和指数计算
-   `NaN`（非数字）：当计算无效时返回，最终渲染值通常是 `0`

  


这些常数让 CSS 的数学计算更加强大，同时也提升了代码的可读性和可维护性。

  


## 小结

  


CSS 提供了一系列强大的数学函数和常数，可以用来做各种复杂的计算，提升布局的灵活性，特别是在响应式设计和动画效果中。通过合理的组合使用这些函数，可以大大提高开发效率和代码的可维护性。
