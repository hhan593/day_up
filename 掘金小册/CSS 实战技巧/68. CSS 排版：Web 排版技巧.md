![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d5710cefcb5047a38ee60db13f150645~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2611&h=1366&s=135973&e=png&b=0f256e)

  


非设计师出身的 Web 开发者时常在如何让 Web 页面更好的看而犯难，尤其是在没有设计师的帮助的情况之下。比如如何 Web 页面的排版，其中典型的案例之一就是如何使长篇内容排版得当且易读。坦率地说，大部分 Web 开发者在这方面做出的的决策不太好，或者根没有注意这方面的决策。

  


幸运的是，CSS 提供了很多有利于 Web 排版的特性，使得我们设置排版样式变得更简单，甚至很多时候你只需要一行简单的 CSS 的代码就可以使 Web 内容排版变得更优雅，更便于用户阅读。在这节课中，我将通过一个简单的示例，带领大家如何使用 CSS 使 Web 内容排版看起来更加美观且易于阅读，特别是你需要使用哪些 CSS 规则来实现这一点。

  


最后，记住，在设计中，规则可以帮助你走得更远，但你不应该害怕打破规则。今天我会分享一些规则和建议，但这并不意味着你不能以不同的方式进行。如果你对自己的决策有信心——请打破规则！但如果你不确定，那么请坚持合理的默认设置。

  


我们开始吧！

## 可用于排版的 CSS 技巧

  


现在基础已经到位——语义化 HTML 和 CSS 重置。我们可以开始构建一些视觉样式了。

  


### 全局样式

  


我在编写 CSS 的时候有一个关键原则——尽可能地在全局范围内用更少的 CSS 代码做更多的事情。在这个上下文中，用于全局的样式并不多，但我仍然会坚持这个原则：

  


```CSS
@layer global {
    :root {
        --color-dark: #000000;
        --color-light: #ffffff;
        --color-primary: #1a8fe3;
    }

    body {
        background: var(--color-light);
        color: var(--color-dark);
        padding: 2em;
    }
    
    :is(h1, h2, h3) {
        font-weight: 800;
    }
    
    :is(ul, ol) {
        padding-inline-start: 1em;
    }
    
    img {
        display: block;
        max-width: 100%;
    }
    
    blockquote {
        padding-inline-start: 1em;
        border-inline-start: 0.3em solid;
    }
}
```

  


如你所见，主要在 `:root` 中设置了一些 CSS 变量，然后为 HTML 元素添加了样式。现在呈现在你面前的结果如下：

### 选择字体

  


你可以选择使用系统字体或自定义字体。对于排版要求不高的项目，可以考虑使用系统字体，比如 `system-ui` :

  


```CSS
@layer typography {
    :root {
        --font-system-ui: system-ui, sans-serif;
    }
  
    body {
        font-family:var(--font-system-ui);
    }
}
```

  


它的显示效果会根据你的系统略有不同，但无论如何看起来都不错：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fae85a2e99e94815af0a2409b92de0e2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1172&h=696&s=18826323&e=gif&f=60&b=efe4e0)

  


使用系统字体（本地字体）的优势是可以让网站加载速度更快，并完全消除未格式化文本闪烁的现象，但代价是可定制性降低。换句话说，如果要应用定制的字体，就需要将字体上传到服务器上，并通过 `@font-face` 规则定义自定义字体。比如，我们给标题应用 [Google Fonts 的 `Inter` 字体](https://fonts.google.com/specimen/Inter)：

  


```CSS
@import url('https://fonts.googleapis.com/css2?family=Inter:ital,opsz,wght@0,14..32,100..900;1,14..32,100..900&display=swap');

@layer typography {
    :root {
        --font-system-ui: system-ui, sans-serif;
    }
  
    body {
        font-family:var(--font-system-ui);
    }
    
    :is(h1, h2, h3) {
        font-family: "Inter", serif;
        font-optical-sizing: auto;
        font-style: normal;
    }
}
```

[打开 `Inter` 字体对应的链接](https://fonts.googleapis.com/css2?family=Inter:ital,opsz,wght@0,14..32,100..900;1,14..32,100..900&display=swap)，你会发现 `@import` 引用入的 CSS 文件中有很多个定义 `Inter` 字体的 `@font-face` 规则：

温馨提示，在使用自定义字体的时候有很多细节需要考虑，比如加载字体的策略、字体的子集、可变字体等等。在这里不会对这些 CSS 知识展该讨论，如果你感兴趣的话，请移步阅读下面这几节课的内容：

### 字体大小

  


设置完字体之后，我们来设置字号（`font-size`），字号应该足够大以便于阅读。所有浏览器默认的 `font-size` 为 `16px` ，在大多数情况下呈现的结果都还不错。但是，你选择的字体在这个大小下看起来太大或太小，可以通过百分比来调整根元素的 `font-size` ：

  


```CSS
@layer typography {
    html {
        font-size: 112.5%; /* 等于 18px */
    }
}
```

  


不同字体在相同大小下显示的效果可能有很大差异，因此如果更换字体，需要调整字体大小。

  


另外，我们在设置 `font-size` 时，更好的方式是使用字体比例，这样可以让设计从一开始就有节奏感和流动感。它还能简化决策过程，并保持设计的整体一致性。

  


需要知道的是，字体比例是基于比率的，这意味着每一步的大小是当前步长乘以所选的比例。例如：

  


```
Step 0: 1rem 
Step 1: 1.25rem (1 * 1.25) 
Step 2: 1.56rem (1.25 * 1.25) 
Step 3: 1.95rem (1.56 * 1.25) 
Step 4: 2.43rem (1.95 * 1.25)
```

  


温馨提示，上面这个比率来自于 [Major Third 工具](https://typescale.com/)。

  


在 CSS 中，我们可以使用自定义属性来描述字体比例：

  


```CSS
:root {
    --size-step-0: 1rem;
    --size-step-1: 1.25rem;
    --size-step-2: 1.56rem;
    --size-step-3: 1.95rem;
    --size-step-4: 2.43rem;
}
```

  


当然，你也可以选择使用 [open-props 定义的字体比例](https://open-props.style/#typography)：

  


```CSS
:root {
    --font-size-00: .5rem;
    --font-size-0: .75rem;
    --font-size-1: 1rem;
    --font-size-2: 1.1rem;
    --font-size-3: 1.25rem;
    --font-size-4: 1.5rem;
    --font-size-5: 2rem;
    --font-size-6: 2.5rem;
    --font-size-7: 3rem;
    --font-size-8: 3.5rem;
}
```

  


我在这里选择使用 [Major Third 工具](https://typescale.com/) 定义的字体比例。定义好字体比例之后，就可以将这些样式应用到相应的元素上：

  


```CSS
@layer typography {
    :root {
        --size-step-0: 1rem;
        --size-step-1: 1.25rem;
        --size-step-2: 1.56rem;
        --size-step-3: 1.95rem;
        --size-step-4: 2.43rem;
    }
    html {
        font-size: 112.5%; /* 等于 18px */
    }
    
    body {
        font-size: var(--size-step-0);
    }
    
    h1 {
        font-size: var(--size-step-4);
    }
    
    h2 {
        font-size: var(--size-step-3);
    }
    
    h3 {
        font-size: var(--size-step-2);
    }
    
    blockquote {
        font-size: var(--size-step-1);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1cdfae25e95e453aab6c74d4252d3fdb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=690&s=13166747&e=gif&f=61&b=f4edeb)

  


你会发现，这种字体比例已经对我们的设计产生了很大的影响，但我们可以做得更好。我们可以使用 [`clamp()` 函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)为文本设置一个流式的字号，它会使文本在较大的视口上使用更大的字号，同时在较小设备上使用更小的字号。对于我们的 CSS ，只需要更新变量即可：

  


```CSS
:root {
    --size-step-0: clamp(1rem, calc(0.96rem + 0.22vw), 1.13rem);
    --size-step-1: clamp(1.25rem, calc(1.16rem + 0.43vw), 1.5rem);
    --size-step-2: clamp(1.56rem, calc(1.41rem + 0.76vw), 2rem);
    --size-step-3: clamp(1.95rem, calc(1.71rem + 1.24vw), 2.66rem);
    --size-step-4: clamp(2.44rem, calc(2.05rem + 1.93vw), 3.55rem);
}
```

  


这个时候，你调整浏览器视窗的宽度时，文本的字号也会随之变化：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7bdf0262a6c8411aaf67aa4df00aba65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1350&h=742&s=8099664&e=gif&f=79&b=f8f4f3)

  


温馨提示，你可以通过 [Utopia 工具](https://utopia.fyi/type/calculator)来创建流式字体比例。

  


### 设置行高

  


设置完字体大小（`font-size`）之后，还需要考虑设置适当的行高。因为设置适当的行高是排版中最简单有效的改进方式之一。行间距太小会让文本难以阅读，行间距太大则会破坏视觉节奏，使换行变得困难。就我们这个示例所呈现的效果来看，不难发现，它的标题看起来有点奇怪。这是因为它们的行高太高了。

  


一个通用的经验法则是，随着字体的增大，行高应该减小。因为我们使用 `Inter` 作为标题的显示字体，它的升部和降部相对较浅，这意味着我们可以设置一个相对紧凑的行高：

  


```CSS
:is(h1, h2, h3) {
    line-height: 1.1;
}
```

  


在行高和标题之间可能会有所不同，但一般来说，目标是设置在 `1` 和 `1.3` 之间。

  


为了让正文内容的可读性更好，我们将全局设置行高为 `1.7` ：

  


```CSS
body {
    line-height: 1.7;
}
```

  


通过在 `body` 元素上设置，这个行高会被继承到所有元素，除了标题，因为它们有更高的权重。这意味着我们不需要在每个元素上重复设置，而是让浏览器为我们完成这项工作。CSS 真是太方便了，对吧？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f9e00e8be59e4e9e9ccec6a5c867d0a9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=692&s=18944399&e=gif&f=67&b=fcfbfb)

  


### 设置间距

  


虽然流式字体大小和行高的设置使整个效果有所改观，但到目前为止，元素之间的间距还是过于紧凑，使用户阅读起来有一种压迫感。我们仅用几行 CSS 代码就可以让整个效果得到很大的改善：

  


```CSS
@layer typography {
    article {
        > * + * {
            margin-block-start: var(--flow-space, 1em);
        }
    }
}
```

  


上面的代码会给 `article` 第一个元素之外的所有元素设置一个 `margin-block-start` （这是一个[逻辑属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)，如果书写模式为 `writing-mode: horizontal-tb` 时，它相当于 `margin-top`）。`margin-block-start` 属性的值是 `var(--flow-spce, 1em)` ，这个值非常有意思。如果定义了 `--flow-space` ，则使用它，否则默认为 `1em` ，它是相对于元素的 `font-size` 计算的。

  


这便是 CSS 的魔力所在，[因为我们使用了字体比例，`1em` 就具有很大的价值](https://juejin.cn/book/7223230325122400288/section/7249357892611440700)。它会使元素的 `margin-block-start` 具有不同的值，使整个排版获得不同的节奏感。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f03f53a3f05b44ae9716cd72365a021f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=686&s=13247542&e=gif&f=46&b=fdfcfc)

  


你还可以通过下面这种方式来改进它：

  


```CSS
@layer typography {
    article {
        > * + * {
            margin-block-start: var(--flow-space, 1em);
        }
    }
    
    :is(h1, h2, h3, blockquote) {
        --flow-space: 1.5em;
    }
    
    :is(h1, h2, h3) + * {
        --flow-space: 0.5em;
    }
}
```

  


这样会为标题（`h1` 、`h2` 、`h3`）和引言（`blockquote`）的顶部增加更多的空间。同时，它也减少了标题的相邻兄弟元素之间的间距，使其更像是一个内容部分。同查，我们使用了 `em` ，因为它是相对于元素 `font-size` 计算的。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dc7f9c7ef92d4c01a74ab83cfa922ace~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1276&h=692&s=15325240&e=gif&f=65&b=fdfdfd)

  


### 设置字符间距

  


调整字符间距有点复杂。如果你使用的字体设计精良，其字符间距通常已经优化了。如果不确定是否需要调整，那么很可能不需要。不过，在某些情况下，比如非常小的文字需要更易读，或非常大的文字需要更紧凑时，可能需要手动调整。

  


我们可以通过 CSS 的 `letter-spacing` 属性设置字符间距，通常使用 `em` 单位，因为它与当前元素的 `font-size` 相关：

  


```CSS
p {
    letter-spacing: 0.02em;
}
```

  


在这个示例中，并没有调整字符间距。

  


### 限制行宽

  


接下来我们需要对行宽做一些限制，因为长行文本非常难以阅读，尤其是对于视力和（或）认知有障碍的人士来说。除此之外，行宽过长在视觉上也不好看。

  


当然，行宽在移动端设备上通常不是问题，但对于宽屏用户，如果不限制行宽，阅读体验可能会很糟糕。研究显示，最佳的行宽约在 `45 ~ 100` 个字符之间。对于长篇内容的行宽，最好控制在 `65 ~ 75` 个字符之间。

  


在 CSS 中，没有直接设置行宽的方法，但我们可以通过限制文本容器的大小来实现。比如：

  


```CSS
@layer typography {
    article {
        width: clamp(320px, 90vw, 1000px);
        margin-inline: auto;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/384e0e93370845a9bdc8491e6036ee7a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1398&h=744&s=5637612&e=gif&f=107&b=fefefe)

  


当然，我们还可以使用小册之前课程（《[CSS 布局：水平居中](https://juejin.cn/book/7395119173656903715/section/7440736421910052864)》）提到的 CSS 技巧来限制行宽：

  


```CSS
@layer typography {
    article {
        margin-inline: max(1rem,50% - var(--w, 100%)/2);
        
        @media (width >= 992px) {
            --w: 960px;
        }
        
        @media (width >= 1200px) {
            --w: 1140px;
        }
        
        @media (width >= 1400px) {
            --w: 1280px;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cfc4d9d115b24075b10df5350112eb2d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1420&h=750&s=5268670&e=gif&f=105&b=fefefe)

  


除此之外，我们还可以结合 `ch` 单位（它等于你所选字体和大小中的数字 `0` 字符的宽度）为每个元素的行宽进行限制，例如：

  


```CSS
@layer typography {
    body {
        display: grid;
        justify-content: center;
    }
    
    article > * {
        max-inline-size: 65ch;
    }
    
    blockquote {
        max-inline-size: 50ch;
    }
    
    h1 {
        max-inline-size: 20ch;
    }
    
    :is(h2,h3) {
        max-inline-size: 28ch;
    }
}
```

  


需要注意的是，使用这种方式限制行宽，需要在布局上做一些微调，比如使用网格布局，将 `article` 水平居中。不然，呈现的效果会使页面的右侧有较多的空白空间，将会影响整体的视觉效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ebac14c9d269482881aa48bc4b500058~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=688&s=1649701&e=gif&f=117&b=fefefe)

  


### 选择颜色

  


对于正文来说，选择中性颜色可以减轻阅读者的视觉疲劳。通常，深色背景的页面会将文本设置为浅灰色，浅色背景的页面会将文本设置为深灰色。传统的纯黑色与纯白色（或反过来）在某些屏幕上对比度过于强烈。现在，背景是白的，但将它设置为略微偏白，使得阅读起来更轻松：

  


```CSS
:root {
    --color-dark: #252525;
    --color-light: #efefef;
    --color-primary: #1a8fe3;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a805818b244476aacc2856d2c7b47ca~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1196&h=624&s=12798835&e=gif&f=85&b=ececec)

  


界面元素（如按钮）通常会使用更丰富的颜色。确保界面元素上的文本具有足够的对比度非常重要。

  


庆幸的是，现代 CSS 提供了多种方式，比如使用 [CSS 的 `contrast-color()` 函数](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)或 [CSS 的相对颜色](https://juejin.cn/book/7223230325122400288/section/7231515598088306720#heading-14)（RCS）都可以使文本根据背景颜色动态调整颜色，并且达到 WCAG 的标准。

  


```CSS
body {
    background: var(--color-light);
    color: color-contrast(var(--color-light) vs #000, #fff to AA-Large);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2ad20b2c47594ef681a82497d7cd6d64~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1476&h=634&s=12956634&e=gif&f=182&b=fdfdfd)

  


温馨提示，有关于这方面更详细的介绍，请移步阅读小册之前的课程《[CSS 排版：颜色和对比度](https://juejin.cn/book/7395119173656903715/section/7445257290149527589)》！

  


现在，我们使用 `color-contrast()` 函数可以很容易处理颜色和对比度。但还有一些细节需要关注，比如调整链接的样式：

  


```CSS
a {
    color: currentColor;
    text-decoration-color: var(--color-primary);
    text-decoration-thickness: 0.3ex;
    text-underline-offset: 0.3ex;
    transition: all .2s;
    
    &:hover {
        text-decoration-color: color-mix(in srgb, var(--color-primary), #000 40%);
    }
}
```

  


我们将链接的颜色的颜色设置为为 `currentColor` ，将会与当前文本颜色相同，但它始终会一条下划线。并且在悬浮状态（`:hover`）下，链接的下划线颜色会变得更暗一些，这是通过 [CSS 的 `color-mix()` 函数](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)的来实现，它将 `--color-primary` 颜色与 `40%` 的黑色 `#000` 混合所得：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d55cfdc6fa314dfbab68d15bd534b652~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=624&s=1860899&e=gif&f=118&b=fefefe)

  


  


### 文本换行均衡

```CSS
:is(h1,h2,h3,blockquote) {

}
```

### 浏览器的特殊行为

  


在某些情况下，移动端浏览器（通常是 iOS 上的 Safari）可能会自动放大字体大小。通常，这并不是我们想要的行为，可以通过 `text-size-adjust` 属性禁用这种自动调整：

  


```CSS
html {
    text-size-adjust: 100%;
}
```

  


另外，CSS 提供了在 macOS 上选择字体渲染方式的能力。可以通过 `-webkit-font-smoothing` 和 `-moz-osx-font-smoothing` 属性进行设置。默认情况下，浏览器会自动选择最佳的渲染方式，通常是 `subpixel-antialiased`（子像素抗锯齿）。但你也可以手动将其设置为 `antialiased`（抗锯齿/灰度模式），这会让文本看起来更清晰。

  


从我的经验来看，启用该设置可以让渲染的文本看起来更锐利。

  


```CSS
html {
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
}
```

  


我建议根据具体情况决定是否使用字体平滑。如果你使用 Mac，可以检查是否启用抗锯齿后效果更好（或者找一位使用 Mac 的朋友帮忙）。
