![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/57935f25bbcb4c2bbee0ee079a148519~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=310170&e=webp&b=d6cdbb)

  


这节课将要聊的最小宽高指的是 CSS 的 `min-width` 和 `min-height` 属性。你可能会问，这么两个简单的 CSS 属性，有什么可聊的。你有这样的想法并没有错，但接下来，我将要和大家聊一些大家在使用 `min-width` 或 `min-height` 不太熟知的方面，而且这些小技巧可以帮助我们构建一些特殊的效果。例如，**我们希望根据元素** **`A`** **来定义元素** **`B`** **的宽度或高度**。是不是很好奇？如果是，请继续往下阅读。

  


## 最小宽高的基础应用

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a4e96c0dad34d78a79bcf06d41a887f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1460&h=730&s=29439&e=jpg&b=190e25)

  


不知道大家是否有留意过，有些 Web 开发者给元素设置尺寸的时候，习惯性的喜欢同时给元素指定 `width` 和 `height` 。孰不知，Web 内容是动态的，很容易使你的内容溢出容器，甚至是打破整个Web 布局。我曾在《[防御式 CSS 精讲](https://s.juejin.cn/ds/iD4W3aDE/)》的《[易碎的容器盒子：避免使用固定尺寸和长内容设置](https://juejin.cn/book/7199571709102391328/section/7199846286953512972)》一课多次强调，我们在构建 Web 布局时，应该尽可能的 **[使用内在尺寸来设置元素的尺寸](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)**。

  


即使需要使用外在尺寸来设置元素的大小，也应该尽可能的避免同时给元素设置固定的宽高。就我个人而言，我通常会结合 `min/max-width` 和 `min/max-height` 来设置元素尺寸。这些属性除了可以使元素根据内容来决定自身大小之外，还可以给元素的尺寸设置一些限制条件。而这些限制条件，很多时候是非常有用的。例如，[你在构建一个多语的 Web 网站或应用](https://juejin.cn/book/7161370789680250917/section/7161625525763440647)，你会发现“Done”按钮在拉丁语体系（`ltr`）显示非常完美，但在阿拉伯语体系（`rtl`）就不是那么理想了，甚至它不会影响用户点击按钮的操作。在此情景而言，如果我们给按钮添加一个最小宽度（`min-width`），那么它在不同语言体系下都会有一个较好的显示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e3002d2930744ea084f383a7eecc90d9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=604&s=283226&e=jpg&b=f0f2f4)

  


类似这样的情景在 Web 上层出不穷。但这节课，我想和大家探讨 `min-width` 和 `min-height` 另一个小技巧——**通过** **`A`** **元素的宽度（或高度）来设置** **`B`** **元素的宽度（或高度）** 。

  


## 从实际需求出发

  


不知道你在开发 Web 的过程中是否遇到这样的一个需求：需要将标题放置在图片的上方或下方，并且不超过图片的宽度。在大多数情况下，我们希望标题和图片的宽度相同。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6047ade98b5e446db4043065970959ac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2455&h=1358&s=1670297&e=jpg&b=ebdfd6)

  


你或许会觉得这个需求非常简单，但很多场景中，你得到的效果可能会像下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0420584be20749c1876df42673017bd7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3198&h=1358&s=1754334&e=jpg&b=edeff1)

  


当然，CSS 是很具魔性的，总是有很多方法可以实现所期望的效果。例如：

  


-   **将标题和图片设置相同的宽度（或设置卡片的宽度）** ：这种方案虽然能实现需求，但不是动态的，因为每次使用不同的图片尺寸时，都需要手动调整标题或卡片的宽度（❌）
-   **在标题中手动插入换行符**：这是非常糟糕的做法，但还是有不少开发者是这么做的（❌）
-   **将标题设置为绝对定位，然后将** **`left`** **和** **`right`** **设置为** **`0`** **（或使用** **`width: 100%`** **），这样宽度会参考一个与图片大小相同的容器**：这种方案过于复杂，而且会导致文本脱离文档流（❌）
-   **将图片的宽度设置为** **`max-width: 100%`** **，这样图片的宽度会参考一个与标题大小相同的容器**：这个方案会使图片变形，甚至变得模糊（❌）

  


你可能会认为，即使不使用上面提到的任何方案，我们使用 [CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 或 [CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)技术，就可以轻易实现我们的需求。这是真的吗？

  


为了印证大家这个想法，我们来尝试使用 CSS Flexbox 和 Grid 布局技术来实现我们的需求。

  


```HTML
<div class="card">
    <img src="https://picsum.photos/id/163/180/150" alt="">
    <h3>Ut gravida, nibh sit amet elementum</h3>
</div>

<div class="card">
    <img src="https://picsum.photos/id/163/250/150" alt="">
    <h3>Ut gravida, nibh sit amet elementum</h3>
</div>

<div class="card">
    <img src="https://picsum.photos/id/163/300/150" alt="">
    <h3>Ut gravida, nibh sit amet elementum</h3>
</div>

<div class="card">
    <img src="https://picsum.photos/id/163/500/150" alt="">
    <h3>Ut gravida, nibh sit amet elementum</h3>
</div>
```

  


我们使用了四种不同尺寸的 `<img>` ，但标题文本内容是相同的。假设 `<img>` 元素是我们所说的 `A` 元素，而标题 `<h3>` 是 `B` 元素。**现在希望通过** **`A`** **元素的宽度来设置** **`B`** **元素的宽度**。

  


首先尝试使用 CSS 的 Flexbox 布局技术来实现我们的需求：

  


```CSS
.card {
    display: inline-flex;
    flex-direction: wrap;
    
    img {
        display: block;
    }
    
    h3 {
        padding-inline: .5em;
        font-size: clamp(1.25rem, 3vw + .875rem, 1.5rem);
    }
}
```

  


很简单的一段 CSS 代码。注意，我们没有显式给 `.card` 指定任何尺寸，所有卡片都会根据其子元素的内容来决定自身的大小。在这里，使用的是 `inline-flex` 而不是 `flex` ，是有原因的。如果使用 `flex` ，Flexbox 容器将是一个块盒子，它的宽度会等同于它的父容器宽度。有关于这方面更详细的介绍，可以移步阅读《[如何根据 UI 形式选择视觉盒模型？](https://juejin.cn/book/7199571709102391328/section/7199844993073152038)》一课！

  


你将看到的实际效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7f0b19900ed14912a304b1f4eb0ce9a3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=647&s=471140&e=jpg&b=faf6f5)

  


仔细观察，不难发现。Flexobx 容器（也就是 `.card` 元素）的宽度将会以最宽的 Flex 项目为基准。

  


-   当标题宽度（`B` 元素）大于图片宽度（`A` 元素）时，图片会拉伸宽度，并且按照原始宽高比放大
-   当标题宽度（`B` 元素）小于图片宽度（`A` 元素）时，标题元素右侧会有留白

  


另外，当标题宽度大于图片宽度时，图片会拉伸变得更为模糊（如上图第一张卡片为例）。这是由于 Flexbox 容器的 `align-items` 值默认值为 `stretch` 引起的。但是，当我们调整 Flex 项目的对齐方式时，图片虽然不会被拉伸，但它的宽度有可能会小于其他 Flex 项目的宽度：

  


```CSS
.card {
    display: inline-flex;
    flex-direction: wrap;
    align-items: center;
 }
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/530daa733e864c22839075658ed22968~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=647&s=299738&e=jpg&b=fcfafa)

  


这效果同样不是我们所期望的效果。

  


但是，有一点需要大家知道，如果所有卡片的父容器调整了上下文格式，例如，它们的父容器自身就是一个 Flexbox 容器，并且其宽度与所有 Flex 项目最小宽度的总和相等，那么实际渲染的效果将会与我们所期望的效果非常接近：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2c289fa8a2b4463497d0bc15ec8b6019~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1348&h=302&s=12295920&e=gif&f=191&b=f5f0ef)

  


这是因为 Flex 项目的 `flex` 属性的默认值为 `auto` ，它会根据 Flexbox 容器的可用空间来调整 Flex 项目的尺寸。但不管怎么说，它始终没有实现我们所期望的效果。

  


接下来，我们来看 CSS Grid 布局技术，会不会与 Flexbox 类似。同样的，我们将 `.card` 元素的 `display` 设置为 `inling-grid` ，并且不显式指定列网格轨道的尺寸，由它的子元素自身的内容来决定：

  


```CSS
.card {
    display: inline-grid;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a35b4b82abd84ac2a57e4ee17c16db36~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=647&s=313775&e=jpg&b=fcfafa)

  


不难发现，卡片宽度同样有具有最大宽度的 Grid 项目决定。就我们这个示例而言，也只有当图片宽度大于标题宽度的时候，它才符合我们所期望的效果。

  


这意味着，并不像大家想象的那么简单，使用 Flexbox 或 Grid 布局，就能通过 `A` 元素的宽度来设置 `B` 元素的宽度。那么，到底要怎么处理，才能真正的实现我们所需要的效果呢？

  


## 解决方案

  


解决方案其实很简单，仅需要几行 CSS 代码即可：

  


```CSS
.card {
    display: inline-flex; /* 或 inline-grid 或 inline-block */
    
    h3 {
        width: 0;
        min-width: 100%;
    }
}
```

  


就这么简单的实现了我们的需求。实际效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8c1a6f09171413b8c10c5b23f7b79aa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=647&s=529413&e=jpg&b=607d8b)

  


> Demo 地址：https://codepen.io/airen/full/NPKGqxe

  


这个技巧的核心在于两点：

  


-   **将容器（例如** **`.card`** **元素）设置为内联框**：它的宽度由内容决定。可以使用 `inline-flex` 、`inline-grid` 或 `inline-block` 等，也可以是浮动元素、表格元素、Flex 项目或 Grid 项目等
-   **将** **`B`** **元素（例如** **`h3`** **元素）的宽度设置为** **`0`** **，同时将其** **`min-width`** **设置为** **`100%`** ：这样 `B` 元素不会影响容器的宽度，容器的宽度将由 `A` 元素决定。然后， `min-width:100%` 来调整 `B` 元素的宽度，使其与容器的宽度（即 `A` 元素的宽度）相同

  


需要知道的是，这个技巧不仅限于图片和标题的组合。这个技巧适用于任何类型的元素（即使是包含复杂内容的元素）都可以使用这个技巧。例如下面这个示例，由标题来决定段落的宽度：

  


```CSS
.card {
    display: inline-grid;
    
    p {
        width: 0;
        min-width: 100%;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/21f86ff3a9ac46f1a14be1bd9b842a85~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=946&s=359910&e=jpg&b=607d8b)

  


> Demo 地址：https://codepen.io/airen/full/mybeeeg

  


我们还可以将相同的逻辑扩展到列布局中，让 `A` 的高度定义 `B` 的高度。只是我们需将 `B` 元素的 `height` 设置为 `0` ，同时将其 `min-height` 设置为 `100%` 。

  


有一点需要知道的是，`min-height` 的百分比计算比较复杂，需要父元素明确指定高度。为了克服这个问题，可以考虑使用 CSS Grid 布局，这样就不需要明确指定父元素的高度。例如下面这个示例：

  


```HTML
<body>
    <div class="card">
        <h3>Donec consectetur elit massa</h3>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Suspendisse egestas, odio id molestie feugiat, justo erat tincidunt nisl, eu interdum metus dui ut libero.     </div>
    <div class="card">
        <h3>Nulla ligula lorem</h3>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Suspendisse egestas, odio id molestie feugiat, justo erat tincidunt nisl, eu interdum metus dui ut libero. Proin elit mi, dictum vitae sodales id, tincidunt in diam. Integer vehicula aliquet mi, quis tempus nisi tristique sit amet. </p>
    </div>
    
    <div class="card">
        <h3>Ut gravida, nibh sit amet elementum</h3>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Suspendisse egestas, odio id molestie feugiat, justo erat tincidunt nisl, eu interdum metus dui ut libero. Proin elit mi, dictum vitae sodales id, tincidunt in diam. Integer vehicula aliquet mi, quis tempus nisi tristique sit amet.Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer vehicula aliquet mi, quis tempus nisi tristique sit amet</p>
    </div>
</body>
```

  


上面代码中 `<p>` 元素包含的内容是不一样的。我们使用 CSS Grid 来构建布局，使三张卡片排成一行：

  


```CSS
.body {
    display: grid;
    grid-auto-flow: column;
    gap: 2rem;
    place-content: center;
}
```

  


默认情况，三张卡片的高度会相等，并且将会以内容最多的卡片高度为基准，内容少的卡片底部会留白：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/590ada60a20443a1ab4686d69ea53273~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=946&s=431872&e=jpg&b=f5f5f5)

  


这是因为，Grid 项目的 `align-items` 默认值为 `stretch`，它会将所有卡片高度保持一致。[这也是我们创建等高布局的常用方法之一](https://juejin.cn/book/7161370789680250917/section/7161624078397210638#heading-1)。如果我们在网格容器上调整 `align-items` 属性的值，比如 `start` ：

  


```CSS
.body {
    display: grid;
    grid-auto-flow: column;
    gap: 2rem;
    place-content: center;
    align-items: start;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ea9aa351f03648c9ad9db6f334aa6ceb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2828&h=1304&s=394639&e=jpg&b=677e8a)

  


你会发现，内容少的卡片，它的高度变得更低。

  


现在，我们把第一张内容少的卡片当作是 `A` 元素，另外两张卡片当作是 `B` 元素，我们希望通过 `A` 元素的高度来决定 `B` 元素的高度。或者说，使 `B` 元素的高度与 `A` 元素高度保持一致。我们就可以使用刚才提到的技巧来实现这个效果。即，将 `B` 元素的 `height` 设置为 `0` ，并且 `min-height` 设置为 `100%` ：

  


```CSS
body {
    display: grid;
    grid-auto-flow: column;
    gap: 2rem;
    place-content: center;
    align-items: start;
}

.card:not(:frist-child) {
    height: 0;
    min-height: 0;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5f1fdd7e2ab9479194a1a0af79e0b468~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2818&h=988&s=333449&e=jpg&b=667f8b)

  


现在，所有卡片的高度都相等了，并且是以第一张卡片（`A` 元素）的高度为基准。其他卡片即使内容比第一张卡片更多，也不会因此调整高度，只是会使内容溢出容器。我们可以给这些卡片添加 `overflow: auto` ：

  


```CSS
.card:not(:frist-child) {
    height: 0;
    min-height: 0;
    overflow: auto;
}
```

  


这样一来，当卡片内容超出定义的高度时，会显示滚动条：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4681ac7ee46c43abab8731647b318077~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1350&h=648&s=1073987&e=gif&f=110&b=557080)

  


> Demo 地址：https://codepen.io/airen/full/GgKpprV

  


你可能会好奇，为什么网格布局，不需要显式设置父容器的 `height` ，`min-height` 为百分比值时可以找到参照物进行计算呢？原理很简单：

  


> 在 CSS Grid 布局中，一旦每个网格区域的大小被确定，网格项目就会被放置到各自的包含块中。网格区域的宽度和高度在这种情况下被视为明确的。

  


这意味着，网格项目的 `height` 或 `min-height` 使用百分比值是安全的，因为该网格区域的高度将根据内容来定义。由于其中一个元素的高度为 `0` ，它不会对整体高度产生影响。

  


你也可以在 Flexbox 布局中使用这个技巧，但稍微复杂一些。如果直接在上面的示例基础上将 `body` 的 `display` 属性设置为 `flex` ，并无法获得我们所期望的效果：

  


```CSS
body {
    display: flex;
    gap: 2rem;
    justify-content: center;
    align-items: flex-start;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2fe37e8a7915460fbdda76300fdd3972~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2846&h=1776&s=698212&e=jpg&b=657d89)

  


即使我们将 Flexbox 容器的 `align-items` 设置为默认值 `stretch` ，也同样无法使 `min-height: 100%` 生效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/903973e4db9e4f9694ec679e50c25585~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2818&h=1778&s=680961&e=jpg&b=667e8a)

  


但这个问题并不是无解，我们只需要为使用 `height: 0; min-height: 100%` 的元素添加一个额外的容器：

  


```HTML
<body>
```

这样我们就可以依赖 Flex 项目的拉伸对齐（`align-items: stretch`）来定义尺寸，该尺寸将作为百分比的参考。因为，Flex 项目要是设置了 `align-self: stretch` ，则会重新布局其内容，将该尺寸视为明确的交叉轴尺寸（侧轴），以便能够解析百分比大小的子元素。对应的 CSS 代码如下：

  


```CSS
body {
    display: flex;
    gap: 2rem;
    justify-content: center;
}

section > .card {
    height: 0;
    min-height: 100%;
    overflow-y: auto;
}
```
