选择 CSS Grid 布局是有目的性的，因为我们后面需要为组件创建不同的变体（布局）。我们可以利用 CSS Grid 的 **[网格区域](https://juejin.cn/book/7395119173656903715/section/7436944613077680137)** 和 **[自动放置网格项目](https://juejin.cn/book/7395119173656903715/section/7434843979486593062)** 等特性实现所需要的布局效果。我将采用下图这种模式来定义网格和放置网格项目：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a0aff57c507b4a2da9fe8205fdeab23d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1027&s=338972&e=jpg&b=e5e5e5)

  


对应的 CSS 代码如下：

  


```CSS
.comment {
    --size: 3rem;
    --gap: 1rem;
    
    display: grid;
    gap: calc(var(--gap) / 4) var(--gap);
    grid-template-columns: var(--size) minmax(0, 1fr);
    grid-template-rows: auto max-content max-content;
    grid-template-areas:
        "avatar  header"
        "avatar  body"
        "...     footer";
    align-content: start;
    align-items: start;
    
    .comment__avatar {
        grid-area: avatar;
    }
    
    .comment__user {
        grid-area: header;
    }
    
    .comment__content {
        grid-area: body;
    }
    
    .comment__actions {
        grid-area: footer;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/49889ece444141db891c6eb112ef1cc6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3340&h=1644&s=437404&e=jpg&b=ffffff)

  


这里有一个小细节需要注意。当 `.comment` 容器被定义为 Grid 容器时，Grid 项目默认情况下会被拉伸（因为网格容器的 `align-items` 和网格项目的 `align-self` 的默认值都为 `stretch`），为了避免用户头像被拉伸，需要在网格容器上显式设置 `align-items` 为 `start` ，或者在用户头像（`.comment__avatar`）上显式设置 `align-self` 为 `start` 。还有一种做法是，给用户头像指定确定的宽高，这样也可以避免用户头像被拉伸：

  


```CSS
.comment__avatar {
    width: var(--size);
    aspect-ratio: 1;
}
```

## 创建评论组件

  


上面所介绍的媒体对象组件仅是我们要创建的评论组件中一个主体部分。接下来，我们的目标是要创建一个评论组件：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99397aa981ba4dbcb4fdf415a3ab7abf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1027&s=314580&e=jpg&b=f4f4f4)

  


在深入细节之前，我们需要先处理整体布局，并确保其功能良好。首先，我们需要一个 HTML 结构，在这里我选择了无序列表 `<ul>` ，因为它比较适合于我们要构建的评论组件：

  


```HTML
<ul class="comments">
    <li>
        <!-- 一级 -->
        <MediaObject />
    </li>
    <li>
        <!-- 一级 -->
        <MediaObject />
    </li>    
</ul>
```

  


这段 HTML 代码表示一个包含两条评论且没有回复评论的列表。我们给 `.comments` 一些简单的 CSS，使整个评论组件看起来美观一些：

  


```CSS
.comments {
    background: #fff;
    padding: 2rem;
    border-radius: .5em;
    margin-inline: max(1rem,50% - var(--w, 1024px)/2);
    display: grid;
    gap: 1rem;
    box-shadow: 0 0 .125em rgb(0 0 0 / .125);
}
```

正如你所看到的，虽然在 HTML 的结构上对每一级评论做了嵌套处理，但在视觉上并没有差异，如上图所示，一级（红色虚线框）、二级（紫色虚线框）和三级（蓝色虚线框）评论列表都是左侧对齐。

  


接下来，我们需要处理“回复”评论的缩进或间距问题。你可能想到了给除一级之外的 `<ul>` 添加一个 `padding-left` ，例如：

  


```CSS
.comments {
    --size: 3rem;
    --gap: 1rem;
    
    ul {
        padding-left: calc(var(--size) + var(--gap));
    }
}
```

  


这样做并没有错，但我更倾向于将所有的缩进处理放在 `<li>` 元素上，因为它们充当了评论组件（`<MediaObject />`）的容器。但需要注意的是，我们只能对二级以上 `<ul>` 的 `<li>` 元素设置 `margin-left` ，否则一级 `<ul>` 的子元素 `<li>` 也会向右偏移：

  


```CSS
.comments {
    --size: 3rem;
    --gap: 1rem;
    
    ul li {
        margin-left: calc(var(--size) + var(--gap));
    }
}
```

.comments {
