在现代 Web 开发中，标签（`Tags`）组件广泛应用于分类、筛选等场景。传统上，这类交互通常依赖 JavaScript 来实现。然而，借助 CSS 的 `:has()` 选择器，构建一个无需 JavaScript 的交互式标签管理 UI。接下来，我将通过一个简单的案例，向你展示如何通过结合 HTML 的复选框（` <input type="checkbox"  />`）、标签（`<label>`）和 CSS 的 `:has()` 选择器，实现标签的添加和移除功能，以及动态的提示信息显示。这种方法不仅结构灵活，样式自由，还能实现复杂的交互效果，适用于需要轻量级交互的场景。而且这种方法不仅简化了代码结构，还提升了页面的可维护性和性能。

  


## 核心思想

  


传统上，`<label>` （标签） 和 `<input>` （复选框或单选按钮）必须紧密排列（紧挨在一起），才能通过 CSS 实现交互效果——构建一个强大的 UI 组合。 例如，切换卡片的背景颜色：

  


```HTML
<input type="checkbox" id="toggle" class="sr-only" />
<label for="toggle" class="button">Toggle<label>
<div class="card">Toggle Background</div>
```

  


```CSS
input[type="checkbox"]:checked {
    + label::before {
        box-shadow: 0 0 25px 2px #0763f7;
    }

    ~ .card {
        --c: #acef90;
    }
}
```

基于这个概念，我们可以构建一个完全由 HTML 和 CSS 驱动的标签管理组件 —— 将使用复选框、标签和 `:has()` 选择器来设计一个可以添加和移除“标签”的UI。这个 UI 包含一组可供选择的标签和一组已经选择的标签。点击某个标签后，它会从一个区域消失，出现在另一个区域。这正是复选框（`checkbox`）与标签（`label`）组合大展拳脚的完美用例。而借助 `:has()` 选择器，我可以任意安排这两组标签在页面结构中的距离和深度，从而实现非常灵活的布局方式。

  


虽然 `:has()` 选择器的用途非常广泛，但在这节课中我们将专注于它的一种能力：**选择包含特定子元素的父元素**。`:has()` 前是父元素，括号内是它所包含的子元素。例如：

  


-   `p:has(> mark)` 会选择所有直接包含 `<mark>` 元素的 `<p>`
-   `div:has(:checked)` 会选择所有内部（不论层级）包含已选中复选框或单选框的 `<div>` 元素

  


现在我们已经掌握了基础，接下来我们将动手制作一个最终的案例，用菜肴的类型作为标签来展示这个标签管理组件。

## HTML 结构设计

  


这个组件分为两部分：

  


-   一部分包含了复选框（`<input type="checkbox" />`）
-   另一部分包含了复选框对应的标签（`<label>`）

  


为了使复选框与标签能相互交互，我们需要为每个复选框设定 `id` 名称，并且`label` 标签的 `for` 属性与对应复选框的 `id` 名相同。它看起来会像下面这样：

  


```HTML
<h2>What are your favorite cuisines?</h2>
<!-- 第一部分：未选中 -->
<ul>
    <li class="plus">
        <input type="checkbox" id="italian" />
        Italian
        <span>
            <svg viewBox="0 0 448 512">
                <path d="M256 80c0-17.7-14.3-32-32-32s-32 14.3-32 32l0 144L48 224c-17.7 0-32 14.3-32 32s14.3 32 32 32l144 0 0 144c0 17.7 14.3 32 32 32s32-14.3 32-32l0-144 144 0c17.7 0 32-14.3 32-32s-14.3-32-32-32l-144 0 0-144z" fill="currentColor" />
            </svg>
        </span>
    </li>
    <!-- 省略其他的 li -->
</ul>

<!-- 第二部分：已选中 -->
<ul>
    <li>
        <label for="italian" class="minus">
            Italian
            <span>
                <svg viewBox="0 0 448 512">
                    <path d="M432 256c0 17.7-14.3 32-32 32L48 288c-17.7 0-32-14.3-32-32s14.3-32 32-32l352 0c17.7 0 32 14.3 32 32z" fill="currentColor" />
                </svg>
            </span>
        </label>
        
    </li>
    <!-- 省略其他的 li -->
</ul>
```

  


每个复选框的父元素是 `.plus` ，表示“待选标签”，它们被放在一个 `ul` 中；每个标签（`<label>`）的类是 `.minus` ，表示“已选标签”，它们同样被放在一个 `ul` 中。

  


我们给它们设置一些基本样式，使 `.plus` 和 `.minus` 看起来好更像一个“标签”：

## 标签管理功能

  


我们先来看关键的 CSS 规则，然后再进行简化。这些规则是整个标签管理功能的核心。首先，我们对所有的 `li` 进行隐藏：

  


```CSS
ul li {
    display: none;
}
```

  


然后重置 `li.plus` 的 `display` 属性：

  


```CSS
ul li {
    display: none;
}

.plus {
    display: grid;
}
```

  


由于我们样式化“标签” UI 的风格采用的是 CSS Grid 布局，因此我们需要将 `display` 设置为 `grid` （或 `inline-grid`） 。如果你采用的是 CSS Flexbox 布局，则需要将 `display` 重置为 `flex` （或 `inline-flex`）。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e626d86f33fd46bd846f72f83192a090~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2552&h=1208&s=170570&e=png&b=fefefe)

  


这个时候，“待选标签”是可见的，已选标签隐藏不可见。

  


接下来，我们需要做的是，当用户勾选某个复选框时（`:has(input:checked)`），它的父元素 `.plus` 会被隐藏（`display` 重置为 `none`），这样已选择的标签就不再显示在“待选区域”。我们可以这样来写 CSS ：

  


```CSS
ul li {
    display: none;
}

.plus {
    display: grid;
    
    &:has(input:checked) {
        display: none;
    }
}
```

## CSS 动态提示信息

  


我们可以做得更好一些，比如通过 CSS 给用户显示一些提示信息：

  


-   当没有选择任何标签时，我们可以显示 `No tags included yet`
-   当至少选择了一个标签时，我们可以显示 `Following tags are included`
-   当所有标签都被选择时，我们可以显示 `All tags are included`

  


我们可以通过给第二个 `ul` 添加伪元素 `::before` ，并且根据不同的状态为 `::before` 的 `content` 设置不同的值：

  


```CSS
ul:nth-of-type(2) {
    &::before {
        display: block;
        text-align: center;
        width: 100%;
        flex: none;
    }

    :not(:has(input:checked)) &::before {
        content: "No tags included yet";
    }
    
    :has(input:checked):has(input:not(:checked)) &::before {
        content: "Following tags are included";
    }
    
    :not(:has(input:not(:checked))) &::before {
        content: "All tags are included";
    }
}
```

  


简单的解释一下上面的代码。你可能已经发现了，我们通过 `:has() 和 :not() 选择器相互结合来做一些判断处理`。其中：

  


-   `:has()` 和 `:not(:has())` 表示页面中是否存在特定选择器的元素（如已勾选或未勾选的复选框）
-   `input:checked` 比较好理解，表示复选框被勾选（已选中）
-   `input:not(:checked)` 表示复选框没有被勾选（未选中）

  


实现的逻辑是：

  


-   如果页面上没有任何勾选的复选框（即 `:not(:has(input:checked))`），那么伪元素 `::before` 的 `content` 属性的值为 `No tags included yet`
-   如果页面上至少有一个被勾选和一个未勾选的复选框（`:has(input:checked):has(input:not(:checked))`），那么伪元素 `::before` 的 `content` 属性的值为 `Following tags are included`
-   如果页面上所有复选框都被勾选（`:not(:has(input:not(:checked)))`），那么伪元素 `::before` 的 `content` 属性的值为 `All tags are included`

## 动态添加 CSS 规则

  


你可能已经注意到了，为了使“已选标签”区域中的标签能够正常显示，我们需要根据复选框的状态（例如 `:has(#filipino:checked)`）以及对应 `<label>` 标签的 `for` 属性（与复选框的 `id` 相匹配，例如 `:has([for="filipino"])`）来重置 `li` 容器的 `display` 属性。如果仅使用 CSS 处理的话，Web 开发者需要手动编写一大串 CSS 选择器：

```CSS
    display: revert;

```

  


这使得 CSS 选择变得较为复杂，而且 Web 内容是动态的，你可能要随时根据内容的变化来调整 CSS 选择器。这意味着这样编写 CSS 显得比较鸡肋，且易于出错，后期还不太好维护。在此情况之下，我们可以借助 JavaScript 来动态生成这些 CSS 规则。这样做的好处是，你可以根据任意数据动态生成自己的标签系统，无需手动硬编码：

  


```JavaScript
const checkboxes = document.querySelectorAll('input[type="checkbox"][id]');

let selectors = [];
checkboxes.forEach((checkbox) => {
    const id = checkbox.id;
    selectors.push(`:has(#${id}:checked) li:has([for="${id}"])`);
});

const styleContent = `${selectors.join(',\n')} {\n  display: revert;\n}`;

const styleTag = document.createElement('style');
styleTag.textContent = styleContent;
document.head.appendChild(styleTag);
```

  


这就是如何用现代 CSS 和 JavaScript 动态实现一个交互式的“添加/移除标签”功能组件。
