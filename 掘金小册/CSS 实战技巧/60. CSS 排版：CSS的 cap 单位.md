![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4690996386024586a2ebbfc6754064af~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=680&s=34250&e=webp&b=de873b)

  


`cap` 是 CSS 中新增的一种长度单位，用于精确测量字体中大写字母的高度（称为 cap-height）。它代表的是字体中大写字母（如 `A` 、`B` 、 `H` 等）的从顶部到基线的高度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/257d57c6338c4acb9a6e33bbae73eb87~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1596&h=899&s=112765&e=png&b=ffffff)

  


它与其他 **[字体相对单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-3)** 类似，与字体大小和字体类型动态关联，使页面在不同字体或语言环境下都能保持一致性，而且还可以帮助我们解决传统单位（如 `px` 或 `em`）难以做到精准对齐的问题，例如图标与文本垂直对齐。除此之外，使用 `cap` 单位还可以提升易读性，因为使用 `cap` 可以更符合视觉上的对齐逻辑。

  


简而言之，`cap` 单位的引入为 CSS 提供了一个精确控制排版的新选择。它特别适合需要基于字体特的高级排版需求，并为设计师和开发者带来了更多灵活性和精确性。接下来，我们通过几个简单的示例来展示 `cap` 单位的强大之处。

  


## 背景需求

  


假设，你需要在两个单词之间放置一个框，并且要使它的高度与大写字母的高度相等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9788049e51f74306acaf69e140fcfb7c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=220197&e=jpg&b=f4f4f4)

  


Web 上图标与文本在垂直方向对齐就是这种效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a3951a2e46f4001bbc9f4930380ced0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=237574&e=jpg&b=f5f5f5)

  


类似这种对齐效果，大家应该时常碰到，而且很难使图标与文本做到精确的对齐。

  


熟悉 [CSS 新特性](https://s.juejin.cn/ds/iyybrgS1/)的开发者可能会想到 [`text-box-trim` 和 `text-box-edge`](https://juejin.cn/book/7223230325122400288/section/7244118525081485351) 。是的，这两个特性的确能做到图标与文本对齐。因为 [`text-box-trim` 属性](https://juejin.cn/book/7223230325122400288/section/7244118525081485351#heading-3)允许剪切掉第一行和最后一行文本上方和下方的额外空间，从而允许更精确地控制字形周围的间距。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e5add2507f2e48c5ae6374f44f030a10~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1666&h=592&s=3266394&e=gif&f=72&b=ffffff)

  


  


然而，`cap` 单位自身就可以精确测量字体中大写字母的高度（称为 cap-height），这与 `text-box-trim` 不谋而合。换句话说，我们是不是使用 `cap` 就能达到与 `text-box-trim` 和 `text-box-edge` 相同的效果呢？我们接下来验证一下！

  


## 传统解决方案：固定大小

  


可能你会因为 `text-box-trim` 和 `text-box-edge` 两个属性太新，担心浏览器的兼容性（的确，目前仅 Safari TP 17支持），而不敢在项目中使用它们。从而选择给它设置固定尺寸。例如：

  


```HTML
<h1 class="title">
    <span>CSS</span>
    <span class="spacer"></span>
    <span>Is</span>
    <svg class="icon" viewBox="0 0 512 512"><path d="M47.6 300.4L228.3 469.1c7.5 7 17.4 10.9 27.7 10.9s20.2-3.9 27.7-10.9L464.4 300.4c30.4-28.3 47.6-68 47.6-109.5v-5.8c0-69.9-50.5-129.5-119.4-141C347 36.5 300.6 51.4 268 84L256 96 244 84c-32.6-32.6-79-47.5-124.6-39.9C50.5 55.6 0 115.2 0 185.1v5.8c0 41.5 17.2 81.2 47.6 109.5z" fill="currentColor"/></svg>
    <span>Awesome</span>
</h1>
```

  


通过浏览器开发者工具不断调试，我们找到了一个适合的高度：

  


```CSS
h1 {
    --size: 4rem;
    font-size: var(--size);
    display: flex;
    align-items: center;
    gap: 0.5rem;

    :is(.spacer, .icon) {
        height: 3.015rem;
        aspect-ratio: 1;
        position: relative;
        bottom: 0.0225em;
    }

    .spacer {
        background-color: #f50;
    }

    .icon {
        color: #f50;
        outline: 1px dashed;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4934e0bd605a47dd873495a708b071b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=122817&e=jpg&b=f4f4f4)

  


乍一看，效果似乎还不错。但是，我们尝试着调整一下 `font-size` ，看看会发生什么：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c3cfceb4020b4301884bae5dd6f027a7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=532&s=2350750&e=gif&f=204&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/KwPyEPb

  


很明显，红色正方形（`.spacer`）和 SVG 图标的高度并不会随大写字母（比如 `C` 、`I` 或 `A`）一起变化（不相等）。

  


## 使用 `lh` 单位

  


我尝试着使用 `lh` 单位来替代 `rem` 单位，`lh` 等于 `line-height` 的值（行高）。同时，我把 `h1` 的 `line-height` 设置为 `1` ，这个时候它等于 `font-size` 的值：

  


```CSS
h1 {
    --size: 4rem;
    font-size: var(--size);
    display: flex;
    align-items: center;
    gap: 0.5rem;
    line-height: 1;

    :is(.spacer, .icon) {
        height: .75lh;
        aspect-ratio: 1;
        position: relative;
        bottom: .035em;
    }
}    
```

  


这个时候改变 `h1` 的 `font-size` 时，看上去是有效的。`.spacer` 和 SVG 的图标高度与大写字母的高度近似相等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/13a38adec1ba45408a36d21e0aaa5956~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1176&h=598&s=2094342&e=gif&f=193&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/vEBWPwa

  


你可能发现了，`.spacer` 和 SVG 图标的 `height` 不是 `1lh` ，同样需要不断调试，找到一个适合的值，比如上面示例中的 `0.75lh` 。问题是，`lh` 会受到 `font-family` 的影响。这意味着，你需要为不同的字体提供不同的值，这几乎是办不到的，除非所有终端和平台都使用统一的字体：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/57a9e277ac364b38bd949d600219c3c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=464&s=799741&e=gif&f=223&b=f3f3f3)

  


## 使用 `ex` 单位

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/17ba9961c8ee446f9a814efacbc7cd26~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=954&h=522&s=17574&e=webp&b=2b2b2b)

  


既然 `lh` 会受 `font-family` 的影响，我又想到了另一个单位—— `ex` ，它等于小写字母的高度（一般指小写字母 `x` 的高度，即 `x-height`）。

  


```CSS
h1 {
    --size: 4rem;
    font-size: var(--size);
    display: flex;
    align-items: center;
    gap: 0.5rem;

    :is(.spacer, .icon) {
        height: 1.375ex;
        aspect-ratio: 1;
        position: relative;
        bottom: .035em;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ec031404b97431b999c85a30514f1d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=528&s=1627942&e=gif&f=172&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/mybqgda

  


初步看，也有效果。我继续尝试着调整 `font-family` ，很容易就能发现，它与 `lh` 存在相似的问题，需要为不同的字体设置不同的值：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fbd85152c1dc4569accf3d8a3eea93c9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=524&s=1716139&e=gif&f=232&b=f3f3f3)

  


## 使用 `cap` 单位

  


接着试验，我这次换上了 `cap` 单位：

  


```CSS
h1 {
    --size: 4rem;
    font-size: var(--size);
    display: flex;
    align-items: center;
    gap: 0.5rem;

    :is(.spacer, .icon) {
        height: 1.025cap;
        aspect-ratio: 1;
        position: relative;
        bottom: .035em;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e32eb89ac1b34031a12cf1f6e55b9ad8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=526&s=2455669&e=gif&f=210&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/LEPOvVd

  


初步效果与 `lh` 和 `ex` 相同。但进一步测试，`cap` 同样会受 `font-family` 的影响：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f2bb854f5fc34f249d1a91b7e71fc51f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1206&h=524&s=1633395&e=gif&f=186&b=f3f3f3)

  


从这三个实验结果来看，`lh` 、`ex` 和 `cap` 都会受到 `font-family` 的影响。主要原因，**[它们都是字体相对单位中的一部分](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-3)**。

  


这意味着，如果在确定字体的基础上，这几种单位都是完美的解决方案，尤其是 `cap` 单位，因为它直接等同于大写字母的高度。

  


需要知道的是，`cap` 更适合于拉丁语，比如英文。如果你需要 `.spacer` 和 SVG 图标的高度与汉字等高，那么可以考虑使用 `ic` 单位。`ic` 与 CJK（中文、日文、韩文）字符宽度或高度相等。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4444500a27ce44b694579e8e5a25f14b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3168&h=906&s=113310&e=png&b=eff0f1)

  


刚才提到过，在字体确认的基础上，使用 `lh` 、`ex` 或 `cap` 都可以使 `.spacer` 或 SVG 图标的高度与大写字母高度相等。相比而言，`cap` 更完美一些，因为它直接等同于大写字母的高度。不过，我们可以通过一些技巧，做得更好一些。

  


我们回到 `cap` 的示例中，尝试着在 `.title` 添加一个 `outline` ，很容易发现，顶部和底部会有空隙：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c0525bacd1ca494bbfd372123e20add4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1168&s=203621&e=jpg&b=f4f4f4)

  


如果我们将 `.title` 的 `height` 设置为 `1cap` 呢？

  


```CSS
h1 {
    --size: 4rem;
    font-size: var(--size);
    display: flex;
    align-items: center;
    gap: 0.5rem;
    height: 1cap;
    outline: 1px dashed;

    :is(.spacer, .icon) {
        height: 1.025cap;
        aspect-ratio: 1;
    }
}
```

  


你会发现，效果会更完美一些，即使是字体不一样，效果也要更佳：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f95a21b18eae47208913b4c4e48c59a9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=524&s=3915772&e=gif&f=317&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/mybqgMY

  


## 案例：对齐文格和图标

  


我们来看一个实际用例，文本和图标垂直对齐：

  


```HTML
<button>
    <svg class="icon" viewBox="0 0 512 512"><path d="M47.6 300.4L228.3 469.1c7.5 7 17.4 10.9 27.7 10.9s20.2-3.9 27.7-10.9L464.4 300.4c30.4-28.3 47.6-68 47.6-109.5v-5.8c0-69.9-50.5-129.5-119.4-141C347 36.5 300.6 51.4 268 84L256 96 244 84c-32.6-32.6-79-47.5-124.6-39.9C50.5 55.6 0 115.2 0 185.1v5.8c0 41.5 17.2 81.2 47.6 109.5z" fill="currentColor"/></svg>
    Like
</button>
```

  


```CSS
button {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    gap: .5cap;
    background: #f50;
    padding: .75em 1.25em;
    font-size: clamp(1.25rem, 3vw + .875rem, 1.5rem);
    color: #fff;
    border-radius: .25em;
    border: none;
    
    svg {
        display: block;
        width: 1.65cap;
        aspect-ratio: 1;
    }
}
```
