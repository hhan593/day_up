## 步骤指示器组件的结构

  


步骤指示器组件（`Stepper`）的结构包括了**步骤**、**状态指示器**、**进度条**、**标签**和**总结文本**（可选） 等关键元素，每个元素在组件中扮演着不同的角色。这些元素随着拥有着多种不同的状态，例如已完成状态（`stepper--completed`）、当前状态（`stepper--current`）、未完成状态（`stepper--pending`）等。我用下图向大家展示一个完整的步骤指示器结构：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/259a0f57debb425d806ba4db68ab556b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1232&s=364714&e=jpg&b=f5f6f8)

  


仔细观察，不难发现，每个步骤中它都包含了状态指示器、进度条、标签和总结文本。而且“当前状态”之前的是“已完成状态”，反之则是“未完成状态”，这些状态通常会使用不同的颜色或图标来区分。例如，绿色表示已完成、蓝色表示正在进行，灰色表示未完成。

  


另外，每个步骤之间通常是使用进度条来连接，它通常会在步骤之间自动调整，以展示步骤之间的连接关系和进度状态。

  


需要知道的是，步骤指示器组件中的“状态指示器”会用各种不同形式的 UI 呈现，比如色彩不同的圆，带有数字的圆或带有图标的圆等，但绝不仅限于圆形，这里就不一一陈述了。

  


在布局方面，通常以水平排列或垂直排列为主，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ba54ae240ed841879cb5e6dceebcc153~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1776&s=267536&e=jpg&b=f9f9f9)

  


到这里，你应该对步骤指示器组件有所了解了。接下来，我们就可以根据它的组成要素来编写 HTML 结构和定义一些 CSS 变量（即 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)）。

  


我们先从 HTML 结构开始。大多数 Web 开发者可能会根据它的结构来编写 HTML，例如：

  


```HTML
<ol class="stepper">
    <li class="stepper__item">
        <div class="stepper__indicator">1</div><!-- 状态指示器 -->
        <div class="stepper__track"></div><!-- 进度条轨道 -->
        <div class="stepper__label">Step 1</div><!-- 标签 -->
        <div class="stepper__summary">Summary Text(optional)</div><!-- 总结文本 -->
    </li>
    
    <li class="stepper__item stepper--current">
        <div class="stepper__indicator">2</div><!-- 状态指示器 -->
        <div class="stepper__track"></div><!-- 进度条轨道 -->
        <div class="stepper__label">Step 2</div><!-- 标签 -->
        <div class="stepper__summary">Summary Text(optional)</div><!-- 总结文本 -->
    </li>
    
    <!-- 其他步骤 -->
</ol>
```

  


上面这样编写 HTML 代码并没有错，但我们可以使它变得更简单一些：

  


```HTML
<ol class="stepper">
    <li class="stepper__item">
        <div class="stepper__label">Step 1</div><!-- 标签 -->
        <div class="stepper__summary">Summary Text(optional)</div><!-- 总结文本 -->
    </li>
    <li class="stepper__item stepper--current">
        <div class="stepper__label">Step 2</div><!-- 标签 -->
        <div class="stepper__summary">Summary Text(optional)</div><!-- 总结文本 -->
    </li>
    <!-- 其他步骤 -->
</ol>
```

  


要是你的步骤指示器不需要“总结文本”这一项，你甚至可以将 `li` 中的 `div` 都移除，一个简单的列表就可以完成任务：

  


```HTML
<ol class="stepper">
    <li class="stepper__item">Step 1</li>
    <li class="stepper__item stepper--current">Step 2</li>
    <!-- 其他步骤 -->
</ol>
```

  


当然，实际开发过程中还是要根据视步骤指示器组件的实际需求出发，适当调整构建它的 HTML 结构。

  


另外，在这里使用 `<ol>` 列表是从其语义上出发的，它与步骤指示器组件更匹配一些。此外，我们还在第二个 `<li>` 元素上使用了 `stepper--current` 类，它定义了当前或激活的步骤。这个类将控制颜色，如之前的指示器结构图所示，它自身是蓝色（当前状态或激活状态），其之前的 `li` 是绿色（已完成状态），之后的是浅蓝色（未完成状态）。

  


让我们定义一些变量，以便轻松控制我们的步骤指示器：

  


```CSS
.stepper {
    --pending: #a0a3a6; /* 未完成状态背景色 */
    --color-pending: color-contrast(
        var(--pending) vs #fff,
        #9fa0a2,
        #333
    ); /* 未完成状态文本色 */
    --current: #4344e7; /* 当前状态背景色 */
    --color-current: color-contrast(
        var(--current) vs #fff,
        #f4f1ff,
        #161c27
    ); /* 当前状态文本色 */
    --completed: #25be76; /* 已完成状态背景色 */
    --color-completed: color-contrast(
        var(--completed) vs #fff,
        #e8f9f1,
        #161c27
    ); /* 已完成状态文本色 */
    
    --circle: clamp(1.5rem, 5vw, 3rem); /* 圆圈大小 */
    --b: 4px;   /* 进度条高度 */
}
```

  


上面代码中，在定义文本颜色 `--color-*` 时，[使用了 CSS 的 `color-contrast()` 函数](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)，它能根据对应的背景颜色，在一个颜色列表中（例如 `#fff, #9fa0a2, #333`）选择最适合的颜色值作为最终值。这样有利于提高文本色（前景色）和背景色之间的对比度，提高文本的可阅读性！

  


## 水平步骤指示器

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/09c04e1fd5b14f72815e6da0b1676731~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1030&s=177300&e=jpg&b=f9f9f9)

  


上图中展示了两种不同的水平排列的步骤指示器，这两个步骤指示器主要差异就是“指示器”与“标签文本”排版不同，一个是垂直排版，另一个是水平排版。接下来，我们一起来给这两种步骤指示器设置样式。

  


首先，我们需要将所有步骤排成一行。在这里，我使用了 [CSS Flexbox ](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)来完成：

  


```CSS
.stepper {
    list-style: none outside none;
    display: flex;
    flex-wrap: wrap;
    justify-content: space-between;
    font-size: 22px;
    
    li {
        flex: 1 1 0%;
        min-width: 0;
    }
}
```

  


这种配置允许我们添加任意数量的步骤（`li.stepper__item`），而无需调整每个步骤。`space-between` 会自动完成这一工作。有关于这方面更详细的介绍，请移步阅读《[Flexbox 布局中的对齐方式](https://juejin.cn/book/7161370789680250917/section/7161623670622781471)》！

  


另外，我们在 `li` 上设置 `flex: 1 1 0%` ，使每个项目等宽。有一点需要知道的是，我还在 `li` 上设置了 `min-width` 为 `0` ，[这是为了确保 Flexbox 布局能实现一个真正等宽布局](https://juejin.cn/book/7199571709102391328/section/7199845170123112486)。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1eba9cef6c634744a32f7b3b7bb440fc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=505&s=252057&e=jpg&b=050505)

  


> 温馨提示，CSS 的 `flex` 属性是一个非常复杂的特性，但要用好 Flexbox 布局，你必须对它的工作原理和背后的计算要有所了解。个人强烈建议你花点时间阅读《[Flexbox 布局中的 `flex` 属性的基础运用](https://juejin.cn/book/7161370789680250917/section/7161623755972673550)》、《[Flexbox 中的计算：通过扩展因子比例来扩展 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7161623797794078750)》、《[Flexbox 中的计算：通过收缩因子比例收缩 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7164357320367931399)》和《[Flexbox 布局中的 `flex-basis`：谁能决定 Flex 项目的大小？](https://juejin.cn/book/7161370789680250917/section/7161623717074698247)》，这些内容将使你彻底掌握 `flex` 。

  


我们在构建任意一个 Web 布局时，在编写 CSS 过程中都需要具有以防万一想法，防止自己的样式打破整个 Web 布局。基于这个原因，[我还在 `.stepper` 中设置了 `flex-wrap: wrap`](https://juejin.cn/book/7199571709102391328/section/7199571708838150155) 。如此一来，当容器没有足够多的空间时，列表项会断行显示。这样做的好处是，它不会影响其他布局效果，坏处是，它有可能会使步骤指导示器会断行，不在同一水平线上排列。如果你并不希望它出现断行的现象，可以禁用该行样式。

  


注意，[CSS Grid 也能实现同样的布局效果](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)，但在这里，[我个人认为 CSS Flexbox 要比 Grid 更适合一点](https://juejin.cn/book/7161370789680250917/section/7161625643720507423)。当然，在后面我们同样也会应用到 CSS Grid 布局技巧！

  


不知道你是否发现了，我将 `ol` 的 `list-style` 设置为 `none outside none` ，去除了有序列有项的默认标记（阿里伯数字）。你可能会有所疑问，为什么要这样做呢？原因很简单，因为我们无法为默认的编号设置样式，所以我选择了禁用，并使用 [CSS 的计算数器 `counter`](https://juejin.cn/book/7223230325122400288/section/7259668400379527205)和伪元素 `::before` （或 `::after` ，也可以是 `::marker`）来替换自定义编号：

  


```CSS
.stepper {
    counter-reset: step;
    
    li::before {
        content: counter(step);
        counter-increment: step;
    }
}
```

  


这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ff0867fe00d34f9cbe4e3a88968de589~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=929&s=148331&e=jpg&b=050505)

  


接下来，我们就可以为步骤（`li.stepper__item`）设置样式。对于步骤，我将使用 CSS Grid 来完成：

  


```CSS
.stepper {
    /* 其他 CSS */
    --gap: clamp(0.25rem, 2vw, 0.5rem);
    
    li {
        display: grid;
        place-items: center;
        gap: var(--gap);
    }
}
```

  


上面代码很简单，就是一个非常普通的网格布局，到目前为止，它创建了一个一列两行的网格：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd9de54a0a7942a2b0e89dd354fa8965~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=627&s=280555&e=jpg&b=050505)

  


在这里使用了 `gap` 来设置行间距，并且 `--gap` 使用了 [CSS 的比较函数 `clamp()`](https://juejin.cn/book/7223230325122400288/section/7241401565653762108) ，它将大小限制在 `0.25rem ~ 0.5rem` 之间。

  


到这一步，我们仅仅是完成了步骤指示器组件最基本的布局样式。我们还需要添加一些 CSS 来美化它，比如文本颜色、背景色、文本大小等等：

  


```CSS
.stepper {
    
    li {
        color: var(--pending);
        font-weight: bold;
        font-size: clamp(1rem, 4vw, 1.25rem); 
        
        &::before {
            display: grid;
            place-content: center;
            color: var(--color-pending)
            background: var(--pending);
            width: var(--circle);
            aspect-ratio: 1;
            border-radius: 50%;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3e0f36feb12140658512224da5e2f202~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=929&s=126399&e=jpg&b=e3e9e9)

  


现在，整个步骤指示器已有了一个雏形。接下来，我们在 `li` 上添加另一个伪元素 `::after` 给步骤指示器添加进度条，整它变得更完善些：

  


```CSS
.stepper {
    /* 其他 CSS */
    
    li {
        &::after {
            content: "";
            display: block;
            width: 100px; /* 这个一个假设值，后面会被替代 */
            height: var(--b);
            background:linear-gradient(var(--pending) 0 0);
        }
    }
}
```

  


这个时候，你会在步骤标签底部看到一条长度为 `100px` ，高度为 `--b` 的直线，这就是步骤指示器的进度条：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aff7baa70b284c74a5808a86fd0b51de~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=131969&e=jpg&b=e3e9e9)

  


正如你所看到的，步骤指示器的进度条位置是不对的。我们需要通过定位将它移动到适合的位置。这也是大家在制作步骤指示器组件感到最麻烦之处，主要进度条的长度和位置不好计算。这也是我们接下来要着重解决的问题。

  


我们先来解决 `top` 定位的问题，因为它相对简单一些。通常情况之下，进度条（直线）位于状态指示器（圆圈）的中心位置。你或许会想到 `top: 50%` 和 `translate: 0 -50%` 来解决，但在这里不行，因为它是相对于整个 `li` 定位。如果这样设置，整个进度条会在 `li` 容器中垂直居中，所以我们需要通过 `calc()` 函数来计算它的 `top` 值。我们知道圆的大小是 `--circle` ，也就是说，将 `top` 设置为 ` calc(var(--circle) / 2)` 就能使进度条顶部位于圆心处。不过，我们还需要考虑进度条的厚度，所以我们需要在这个基础上减去进度条厚度的一半，才能使进度条 `Y` 轴的中心位于圆圈的圆心：

  


```CSS
.stepper{
    li {
        position: relative;
        
        &::after {
            position: absolute;
            top: calc((var(--circle) - var(--b)) / 2);
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7fbb341ad4aa4b5397af202d7cead9f6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=140311&e=jpg&b=e3e9e9)

  


解决完 `top` 定位之后，我们再来解决 `left` 的定位。试想一下，将 `left` 设置为 `50%` ，那么进度条（直线）最左侧边缘将位于圆圈（进度指示器）的中心位置。同时，将 `width` 的值设置为 `100%`。

  


```CSS
.stepper{
    li {
        position: relative;
        
        &::after {
            position: absolute;
            top: calc((var(--circle) - var(--b)) / 2);
            left: 50%;
            width: 100%;
        }
    }
}
```

  


你会发现，进度条现在将相邻两个圆连接在一起（两个圆圆心之间的距离）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/25514946918b4fc5b59b030f7f635fae~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=202328&e=jpg&b=e3e9e9)

  


实际上，进度条（上图中红色直线）应该是与相邻两个圆的边缘相连接。仔细观察上图，我们需要将 `left` 的值加大一点，在原来的 `50%` 基础上再加上圆圈半径（` var(--circle) / 2`）；它的宽度 `width` 需要在 `100%` 的基础上减去一个圆的尺寸（`100% - var(--circle)`）：

  


```CSS
.stepper{
    li {
        position: relative;
        
        &::after {
            position: absolute;
            top: calc((var(--circle) - var(--b)) / 2);
            left: calc(50% + var(--circle) / 2);
            width: calc(100% - var(--circle));
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d6a39bea682c4818bb910aabc26d6b91~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=269369&e=jpg&b=e3e9e9)

  


把隐藏的两个 `li` 的伪元素 `::after` 打开，你将看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2b4ef53a3ba24cca9ee350eaef87eabb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=111889&e=jpg&b=e3e9e9)

  


与我们实际要的步骤指示器组件效果相比，最后一个 `li` （步骤）后面多出了一个进度条，因此我们需要将这条“直线”删除。在 CSS 中，我们可以通过 [`:not()` 选择器](https://juejin.cn/book/7223230325122400288/section/7226251495276609569)和结构选择器 `:last-child` 结合来选择除最后一个 `li` 之外的所有 `li` ，并且只在选中的 `li` 中添加伪元素 `::after` ：

  


```CSS
li:not(:last-child)::after {
    content: "";
    height: var(--b);
    background:linear-gradient(var(--pending) 0 0);
    position: absolute;
    left: calc(50% + var(--circle) / 2);
    top: calc((var(--circle) - var(--b)) / 2);
    width: calc(100% - var(--circle));
    z-index: 2;
}
```

  


这个时候，你看到的步骤指示器组件，就像下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/26d2ce4ad20547b3bbf774c8d5d8d643~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=137558&e=jpg&b=e3e9e9)

  


额外提一下，有的时候，你可能希望进度条与指示器之间有一点距离，那么我们可以像下面这样来调整 `left` 和 `width` 的值：

  


```CSS
li:not(:last-child)::after {
    left: calc(50% + var(--circle) / 2 + var(--gap));
    width: calc(100% - var(--circle) - var(--gap) * 2);
}
```

  


与之前相比，`left` 在原来值的基础上增加了一个 `--gap` 的值，而 `width` 在原来值的基础上再减去两个 `--gap` 的值：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9133b0de963446296d634e4cc56cfe5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=396579&e=jpg&b=e3e9e9)

  


你可能已经注意到了，到目前我们所呈现的步骤指示器组件，状态指示器和标签文本都是垂直排列的。如果我们需要一种水平排列的怎么办呢？其实很简单，我们只需要对两个地方进行调整。首先调整 `li` 的网格布局方式，我们可以通过 `grid-template-columns` 创建一个一行三列的网格：

  


```CSS
li {
    grid-template-columns: var(--circle) max-content minmax(0, 1fr);
    gap: var(--gap);
}
```

  


上面的代码的意思是，第一列的列宽是状态指示器的大小（`--circle`），第二列的宽度将[根据元素容来决定（`max-content`）](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)，最后一列 `minmax(0, 1fr)` 会将剩余空间留给进度条。

  


其次，我们需要调整进度条的样式，它不再需要绝对定位了。同时将它的 `width` 设置为 `100%`：

  


```CSS
li:not(:last-child)::after {
    position: unset;
    width: 100%;
}
```

  


注意，[我在这里将 `position` 属性的值设置为 `unset`](https://juejin.cn/book/7223230325122400288/section/7232094160071688227#heading-7) ，这个关键词会执行与 `initial` 或 `inherit` 相同的操作，相当于 `position` 属性的值为 `static` 。另外，为了避免最后一个 `li` 与其他列具有相等的宽度，我还将它的 `flex` 重置为 `none` ：

  


```CSS
li {
    grid-template-columns: var(--circle) max-content minmax(0, 1fr);
    gap: var(--gap);
    
    &:last-child {
        flex: none;
    }
    
    :not(:last-child)::after {
        position: unset;
        width: 100%;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/982abc6e6cb141d3b83ca01d4a505955~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=942&s=107205&e=jpg&b=e3e9e9)

  


不难发现，进度条尾部与相邻的圆连在一起。我们可以在 `ol` 上设置一个 `--gap` 值，来避免这种现象：

  


```CSS
.stepper {
    gap: var(--gap);
}
```

  


正如你所看到的，我们只调整了一点点代码，就获得了三种不同 UI 风格的步骤指示器组件：

## 创建响应式步骤指示器

  


通过之前的学习，你现在已经能使用 CSS 构建出水平和垂直排列的步骤指示器组件了。有了这个基础，我们就可以很轻易地构建出一个响应式的步骤指示器组件。

  


提到[响应式 Web 设计](https://juejin.cn/book/7161370789680250917/section/7165845190614188062)，大多数 Web 开发者首先会想到的是 [CSS 媒体查询 `@media`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935) ，通过查询浏览器视窗的宽度来调整步骤指示器组件的布局。例如：

  


```CSS
.stepper {
    display: grid;
    justify-self: center;
    counter-reset: step;

    @media only screen and (width >= 640px) {
        display: flex;
        flex-wrap: wrap;
        justify-content: space-between;
        justify-self: unset;
        width: 100%;
    }

    li {
        display: grid;
        align-items: center;
        align-content: start;
        grid-template-columns: var(--circle) max-content;
        grid-template-rows: auto minmax(0, 1fr);
        gap: 0 var(--gap);
        position: relative;

        @media only screen and (width >= 640px) {
            flex: 1 1 0%;
            min-width: 0;
            place-items: center;
            grid-template-rows: auto;
            grid-template-columns: auto;
    
            &:not(:last-child)::after {
                position: absolute;
                left: calc(50% + var(--circle) / 2);
                top: calc((var(--circle) - var(--b)) / 2);
                width: calc(100% - var(--circle));
                height: var(--b);
                background: linear-gradient(to right, var(--pending) 0 0);
            }
    
            &.stepper--current::after {
                background: 
                    linear-gradient(var(--current) 0 0) 0 0 / 50% var(--b) no-repeat,
                    linear-gradient(var(--pending) 0 0) 100% 0 / 50% var(--b) no-repeat;
            }
        }
    }
}
```

  


在这里，默认情况之下，你看到的是一个垂直步骤指示器组件，只有浏览器视窗的宽度大于或等 `640px` 时，垂直步骤指示器组件才会替换成水平步骤指示器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0f3c23840fc4476a9bbd62bd923e272f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1182&h=628&s=4986205&e=gif&f=213&b=dee6e1)

  


> Demo 地址：https://codepen.io/airen/full/GRbdxJW

  


要是在多年之前，采用这种方案来制作响应式步骤指示器组件，也算得上是主流技术。但是，随着 [CSS 的容器查询 `@container`](https://juejin.cn/book/7223230325122400288/section/7259668032165773368) 出现，再采用这种方式来制作响应式步骤指示器组件就不是最佳的方案了。因为**媒体查询是一种宏观的布局（Macro Layout），可以用于整体页面布局；而容器查询可以调整组件的每个元素，创建了一种微观的布局（Micro Layout）** 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a86bc1e075b7462999940d4a348d33b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=1032&s=82348&e=jpg&b=faf9f9)

  


假设在页面中同时有两个步骤指示器，一个位于侧边栏，一个位于主内容栏。此时，通过媒体查询构建的响应式步骤指示器，就会存在诸多的问题。但是换成容器查询，这一切都不再是问题。

  


接下来，我们来看看容器查询构建的响应式组件有何不一样。

  


首先，需要一个额外的容器用来包裹步骤指示器：

  


```HTML
<div class="container">
    <ol class="stepper">
        <li class="stepper__item">Personal Info</li>
        <li class="stepper__item stepper--current">Education</li>
        <li class="stepper__item">Review</li>
    </ol>
</div>    
```

  


接着在 CSS 中使用 `container-type` 注册一个查询容器：

  


```CSS
.container {
    container-type: inline-size;
}
```
