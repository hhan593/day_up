CSS 的 `pointer-events` 属性通常用于控制元素如何响应指针事件（如鼠标点击、悬停和触摸交互等）。通过设置 `pointer-events` 属性，你可以决定一个元素是否应响应指针交互，或者忽略它们，让事件传递给底层元素。这在你想禁用某些交互但不影响网页整体功能时特别有用。例如，你可以禁用图片的指针事件，防止用户点击、拖拽或悬停。同样，也可以禁用链接的指针事件，使其不可点击。这种灵活性使开发者能够创建更直观、更友好的用户界面。

  


在 SVG 内容中，`pointer-events` 属性尤其有用，它可以根据元素的可见性（`visibility`）、填充（`fill`）或描边（`stroke`）属性来控制交互行为。这使其成为处理复杂图形和交互元素的重要工具。简而言之，通过添加 `pointer-events` 属性，可以改善 SVG 文档在用户交互时的行为方式。我们可以管理 SVG 文档或元素的哪些部分可以接收来自鼠标、触控板或手指等指针设备的事件。

  


在这节课中，我们将探索 `pointer-events` 属性的各种取值及其用途，并提供实际用例，帮助你更有效地将该属性应用于 Web 开发项目中。

  


## `pointer-events` 概述

  


CSS 中的 `pointer-events` 属性通过从一组预定义的关键字中选择一个值来设置。语法非常简单，使 Web 开发者可以轻松控制元素如何响应指针事件。比如：

  


```CSS
.overlay {
    pointer-events: none;
}
```

  


这会使 `.overlay` 元素忽略所有指针交互，用户可以与其下方的元素进行交互。

  


简单地说，通过 `pointer-events` 属性，你可以让元素忽略用户输入，或让点击“穿透”元素。这是构建多层界面、交互式设计和动态布局的强大工具。

  


`pointer-events` 属性的值主要分为三大类：全局值、HTML 元素专用值和 SVG 元素专用值：

  


```CSS
/* 全局值 */
pointer-events: inherit;
pointer-events: initial;
pointer-events: revert;
pointer-events: revert-layer;
pointer-events: unset;

/* HTML 专用值 */
pointer-events: auto;
pointer-events: none;

/* SVG 专用值 */
pointer-events: visiblePainted;
pointer-events: visibleFill;
pointer-events: visibleStroke;
pointer-events: visible;
pointer-events: painted;
pointer-events: fill;
pointer-events: stroke;
pointer-events: bounding-box;
pointer-events: all;
```

  


其中全局值主要用于继承或重置 **，** 它们可以用于 HTML 或 SVG 元素；HTML 专用值只能用于 HTML 元素，包括 `<svg>` 元素；SVG 专用值相对而言要更为复杂，这些值允许你细粒度控制图形的交互行为。`pointer-events` 属性的每个值都定义了元素在什么条件下可以成为指针事件的目标。这些值为 Web 开发者提供了灵活的交互控制手段。以下是对 `pointer-events` 属性不同取值的详细说明。

  


### HTML 元素专用值

  


常用于 HTML 元素的值主要有 `auto` 和 `none` ：

  


-   **`auto`** ：这是 `pointer-events` 的默认值，它表示元素会像没有设置该属性时一样正常响应指针事件。对于 SVG 元素，`auto` 的效果等同于 `visiblePainted`
-   **`none`** ：元素将不会响应任何指针事件。事件会“穿透”该元素，传递给下方面的元素。但其子元素仍然可以通过单独设置 `pointer-event` 来响应指针事件。在事件捕获和冒泡阶段中，相关事件仍会在父元素上触发监听器。需要知道的是，即使父元素设置了 `pointer-events` 为 `none` ，如果指针进入或离开其子元素，`pointerenter` 和 `pointerleave` 事件仍会在父元素上触发

  


### SVG 元素专用值

  


下面这些值主要用于 SVG 中的图形元素，也可以在 HTML 中尝试使用（实验性支持）：

  


-   **`visiblePainted`** ：元素在可见时（`visibility: visible`）且光标在填充区域（`fill`）或描边区域（`stroke`）上时，才响应指针事件。前提是 `fill` 和 `stroke` 不为 `none`
-   **`visibleFill`** ：元素在可见时，且光标在填充区域上时响应指针事件。`fill` 属性的值不影响指针事件处理
-   **`visibleStroke`** ：元素在可见时，且光标在描边区域上时响应指针事件。`stroke` 属性的值不影响指针事件处理
-   **`visible`** ：元素在可见时，且光标位于填充或描边区域上都能触发指针事件。`fill` 和 `stroke` 属性的值不影响指针事件处理
-   **`painted`** ：光标位于填充或描边区域时触发指针事件，前提是 `fill` 和 `stroke` 不为 `none` 。此时，元素是否可见不会影响指针事件处理
-   **`fill`** ：光标位于填充区域时响应指针事件，不受 `fill` 或 `visibility` 属性值影响
-   **`stroke`** ：光标位于描边区域时响应指针事件，不受 `stroke` 或 `visibility` 属性值影响
-   **`bounding-box`** ：仅当光标位于元素的边界框内时，元素才响应指针事件
-   **`all`** ：元素始终响应事件，无论是否可见或是否有填充/描边

  


除 `auto` 值之外，大多数其他值可以分两组：

  


-   需要元素可见的值，即元素的 `visibility`属性值为`visible` ，主要包括 `visiblePainted` 、`visibleFill` 、`visibleStroke` 、`visible`
-   不需要元素可见的值，主要包括 `painted` 、 `fill` 、 `stroke` 和 `all`

  


### 全局值

  


全局值主要包括 `inherit` 、`initial` 、`revert` 、`revert-layer` 和 `unset` ，主要用于继承或重置 `pointer-events` 属性的值：

  


-   **`inherit`** ：继承父元素的 `pointer-events` 属性的值
-   **`initial`** ：设置为初始值，等同于 `auto`
-   **`revert`** ：恢复为浏览器默认样式表中的 `pointer-events` 属性的值
-   **`revert-layer`** ：恢复当前层中的 `pointer-events` 属性的值
-   **`unset`** ：如果属性可继承则继承；否则设置为初始值

  


> 温馨提示，有关于 `inherit` 、`initial` 、`revert` 和 `unset` 更详细的介绍，请移步阅读《[CSS 显式默认值：`inherit`，`initial`，`unset` 和 `revert`](https://juejin.cn/book/7223230325122400288/section/7232094160071688227)》；有关于 `revert-layer` 更详细的介绍，请移步阅读《 [CSS 分层：`@layer`](https://juejin.cn/book/7223230325122400288/section/7259563116462080037#heading-14)》!

  


简单的小结一下：

  


| **常用值**        | **适用范围**   | **是否响应指针事件** | **说明**                  |
| -------------- | ---------- | ------------ | ----------------------- |
| `auto`         | 所有 HTML 元素 | ✅ 是          | 默认行为，元素正常响应事件           |

| `revert-layer` | 所有元素       | 根据当前图层样式来确定  | 恢复为当前图层的默认值             |
| `unset`        | 所有元素       |              | 如果可继承则继承，否则为初始值         |

| ---------------- | -------------- | ----------------- | -------------------------------------------- |
| `visiblePainted` | `visible` （可见） | `fill` 和 `stroke` | ✅ 需要有 `fill` 或 `stroke`                      |

| `visible`        | `visible` （可见） | `fill` 和 `stroke` | ❌ 忽略 `fill` 和 `stroke`                       |
| `painted`        | 无要求            | `fill` 和 `stroke` | ✅ 需要有 `fill` 或 `stroke`                      |

| `bounding-box`   | 无要求            | 元素边界框             | ❌ 忽略所有视觉属性                                   |
| `all`            | 无要求            | `fill` 和 `stroke` | ❌ 忽略 `fill` 、`stroke` 和 `visibility`         |

  


## 何时使用 `pointer-events`

  


`pointer-events` 在 UI 设计中非常实用，特别是处理覆盖层、动画或仅用于视觉展示的图层时。

  


### 让点击穿透透明图层

  


如果你有一个纯粹用于美观的覆盖层，但希望用户仍能与底层内容交互：

  


```CSS
.frosted-glass {
    pointer-events: none;
}
```

  


现在该图层不会阻挡任何链接或按钮的交互。

  


### 禁用特定元素的交互

  


当你希望某个元素不可点击（比如禁用按钮或只读层）：

  


```CSS
.disabled-button {
    pointer-events: none;
    opacity: 0.5;
}
```

  


这将禁用按钮的所有鼠标交互。或者禁用所有图片的指针事件：

  


```CSS
img {
    pointer-events: none;
}
```

  


### 实现条件点击控制

  


你可以通过 JavaScript 或类名动态切换交互状态，这在模态窗口、弹窗或加载状态中很常见：

  


```CSS
modal {
    pointer-events: none;
    
    &[open] {
        pointer-events: auto;
    }   
}
```

  


这会在模态窗口非激活状态时禁用交互，激活时重新启用。

  


### 仅在某些设备上禁用交互

  


你可以结合[媒体查询](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)，为特定设备设置交互规则：

  


```CSS
@media (hover: none) {
    .desktop-only-hover {
        pointer-events: none;
    }
}
```

  


这会在[不支持 `hover` 的设备](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-13)（如多数触摸设备）上禁用交互。

  


## 更深入了解 `pointer-events`

  


### 父元素禁用交互对子元素的影响

  


当你对父元素设置 `pointer-events: none`，其所有子元素也会一并无法交互。如果你希望子元素仍然可交互，可进行如下覆盖：

  


```CSS
.parent {
    pointer-events: none;
}
.child {
    pointer-events: auto;
}
```

  


这允许只有 `.child` 元素响应点击，尽管父级被禁用了。

  


需要知道的是，如果你对一个元素使用 `display: none` ，即使你对它的子元素使用 `display:block` ，也没有用——它仍然（子元素）仍然是隐藏的，因为它的父元素是隐藏的。

  


但对于 `visibility: hidden` 来说，情况就不一样了。子元素会被隐藏，因为 `visibility` 是可继承的，但如果你对子元素使用 `visibility:visible` ，它们会再次变得可见。

  


这就是 `pointer-events` 在这里发生的事情。如果你对父元素使用 `pointer-events:none`，然后对子元素使用 `pointer-events:auto` ，你就是在重新启用指针事件。利用这个特性，我们还可以实现一些有趣的效果。

### 管理 SVG 文档交互性

  


需要知道的是，当我们向 SVG 文档或元素添加链接或监听事件时，交互区域不一定是一个矩形，它会是一个边界框。一个 SVG 元素可以计算三种类型的边界框：

  


-   对象边框界（**Object Bounding Box**）是仅包含元素几何形状的边界框。对于基本形状来说，这是填充的区域。除非另有规则，否则这是指未经限定的“边界框”
-   描边边界框（**Stroke Bounding Box**）是包含元素几何形状及其描边形状的边界框
-   装饰边界框（**Decorated Bounding Box**）是包含元素几何形状、其描边形状和其标记的边界框

  


> 请注意，元素的 `opacity` 、 `fill` 、`fill-rule` 、`fll-opacity` 、`stoke-dasharray` 和 `stroke-dashoffset` 等属性的值对元素的边界框没有影响。

  


换句话说，SVG 文档的哪些部分是可交互的取决于哪些部分是可见的和（或）被绘制的。这意味着，要搞清楚 SVG 元素什么时候可具交互性，就得搞清楚元素什么时候可见和（或）被绘制。

  


我们知道 SVG 元素可以被“填充”（`fill`），也可以被“描边”（`stroke` 加 `stroke-width`），其中填充指的是形状内部，描边则是指其轮廓。“填充”和（或）“描边”一起就完成了 SVG 图形的绘制操作，你在屏幕（SVG 画布）上就能看到被绘制的图形。当我们谈论被绘制的元素时，指的就是该元素具有填充和（或描边）。此时，该元素也是可见的。

上面代码中，虽然 `<path>` 元素设置了 `fill="red"` （表示被“填充”）以及 `stroke="lime"` 和 `stroke-width="6"` （表示被“描边”），但两个 `<path>` 元素分别设置了 `visibility="hidden"` 和 `display="none"` ，它们变得不可见（在 SVG 画布上看不到任何图形）。但 `<path>` 元素存在并占据理论空间。我们只是看不到它。

  


> 注意，有一点与 HTML 不一样，如果显式给 HTML 元素设置了 `display: none` 样式，该元素不会占据任何物理空间。

  


更加令人困惑的是，一个元素可能是可见的，也就是说，`visibility` 的计算值是 `visible` ，但却没有被绘制出来。这种情况发生在元素既没有描边又没有填充的情况下。另外，如果一个元素的填充和描边颜色的透明度道的值为 `0` ，即使你在 SVG 画布上看不到任何图形，但它实际上也是被绘制的。只不过图形的填充和描边颜色是完全透明的。

### 在视图过渡运行时保持页面交互

  


在使用视图过渡时，你会注意到在视图过渡运行时，页面无法响应点击事件。例如，下面这个示例，允许你操作一个卡片列表。卡片的添加和移除通过视图过渡进行动画。你会发现，在视图过渡进行时，你无法快速点击加号按钮以添加卡片：按钮在视图过渡期间无法响应点击事件：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f2f96faa4f434ee69e72275daa444409~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=484&s=1336599&e=gif&f=124&b=fffefe)

  


其原因是 `::view-transition` 伪元素——它包含所有动画的快照——会覆盖在文档之上，并捕捉所有的点击事件。

  


```CSS
::view-transition /* 👈 捕捉所有点击！ */
└─ ::view-transition-group(root)
   └─ ::view-transition-image-pair(root)
      ├─ ::view-transition-old(root)
      └─ ::view-transition-new(root)
```

  


为了让点击事件穿透到下面的文档，你需要禁用伪元素上的指针事件。在 CSS 中，这只需包含以下规则：

  


```CSS
::view-transition {
    pointer-events: none;
}
```

  


现在，你可以快速连续点击加号按钮多次：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/77a0284975d141ce8691560c0021ab45~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1332&h=480&s=3203934&e=gif&f=184&b=fef7f7)

  


  


> Demo 地址：https://codepen.io/airen/full/PwwaqYJ

  


在这个示例中，我们还做了一个额外的操作：**阻止了** **`:root`** **元素作为视图过渡的一部分被捕获**。

  


```CSS
:root {
    view-transition-name: none;
}
```

  


这是因为，参与过渡的元素需要跳过它们在 DOM 中的位置的绘制，因为它们的图像会在相应的 `::view-transition-new()` 伪元素中绘制。同样，点击也会跳过，因为元素的 DOM 位置与它的内容呈现的位置不对应。通过不捕获 `:root`，我们为点击交互留出了空间，这样点击事件就可以穿透到文档中——只要不点击其他的快照。

  


> 温馨提示，有关于 CSS 视图过渡更详细的介绍，请移步阅读《[解锁 CSS View Transitions API 的魔力](https://juejin.cn/book/7223230325122400288/section/7259669097242329145)》！

  


## 小结

CSS 中的 `pointer-events` 属性是一个强大的工具，用于控制元素如何响应指针事件。通过设置此属性，你可以决定元素在何种条件下可以成为这些事件的目标，从而增强用户交互和体验。

  


无论你是在处理简单的 HTML 元素，还是复杂的 SVG 图形，`pointer-events` 属性都提供了所需的灵活性，让你能够精确控制用户如何与网页内容互动。只要理解它的不同取值和用途，你就可以创建出更具响应性和交互性的网页设计，带来更加流畅的用户体验。

  


试着在你的项目中使用 `pointer-events` 属性，看看它如何提升用户交互性，并改善整体的用户体验。
