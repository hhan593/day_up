![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73ee9e2bf8e84291a9df38a715c65247~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=626934&e=webp&b=bcc1c4)

  


在设计 Web 网站或应用时，总会有一些部分需要隐藏，其中最常见的就是滚动条。滚动条有时会破坏 Web 页面的整洁外观，尤其是当你希望页面看起来简洁而干净时。但是不用担心！通过 CSS 隐藏滚动条，你可以让 Web 页面看起来更加平滑和干净。

  


当内容超出容器时，滚动条就会出现，让用户可以滚动查看更多内容。但有时，滚动条只是多余的东西，反而妨碍了页面的美观。你可以通过隐藏滚动条来让用户体验更顺畅。通过 CSS，只需要一些简单的代码，就能让滚动条消失，而用户仍然可以方便地滚动页面。

  


在这节课中，我将介绍如何通过 CSS 隐藏滚动条，同时保持滚动功能不变。无论你是设计一个简洁、现代的界面，还是在提升用户体验，管理滚动条的可见性都是一个重要的 CSS 技巧。通过这节课的学习，你将学习如何控制滚动条的行为，并微条滚动条的可见性，以实现一个精致、流畅的布局。

  


## 为什么要隐藏滚动条？

  


通常出于 Web 可访问性（A11Y）的考虑，个人建议不要随意更改或覆盖浏览器的默认样式，但在某些情况下，隐藏滚动条是有充分理由的。

  


当 Web 内容超出浏览器或容器可视区域时，滚动条会自动出现。这种行为由浏览器的用户代理样式（即默认样式）控制。滚动条为鼠标或键盘操作提供了一个可视化的滚动提示，但在不同的系统和不同的浏览器中，滚动条的默认样式会不一样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c9f9e2a52f4a4c1cb5f54ec665d486b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2120&h=662&s=31063&e=png&b=ffffff)

[我们可以通过浏览器厂商内置的伪元素对滚动条进行美化](https://juejin.cn/book/7199571709102391328/section/7199846155080400899)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7ca6aa175efd496499ef991e48cc98d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3014&h=1272&s=216724&e=jpg&b=15181d)

  


虽然经过个性化定制的滚动条在外观上趋于统一和更美观，但在某些特性的布局模式中，滚动条其实并不是必需的，尤其是那些本身不需要用户滚动的操作场景，比如幻灯片播放，新闻滚动等。在这些情况下，隐藏滚动可以让交互体验更加流畅，避免滚动条对整体内容造成干扰。

  


通过隐藏滚动条，还可以回收它所占用的空间，增加屏幕可用区域。这不仅能让界面更加简洁，也能为设计留出更多空间。

  


另一个常见的隐藏滚动条的理由是提升移动端的浏览体验。在移动设备，尤其是智能手机上，用户通常期望页面是垂直滚动的，不需要横向滚动，因为屏幕本身是狭长的，内容自然从上到下排列。隐藏横向滚动条可以让浏览体验更加自然，同时减少浏览器技术细节的注意力分散。

  


简单的归纳一下，隐藏滚动条有以下几个关键原因：

  


-   **更简洁的 UI**：可见的滚动条可能会使简洁、极简的设计显得杂乱无章。隐藏它们可以确保界面更加精致、专业
-   **自定义滚动行为**：在使用自定义滚动或动画时，默认的滚动条可能显得不合适或不必要
-   **聚焦内容**：去除滚动条可以帮助用户专注于内容，而不会被 UI 元素分散注意力
-   **现代美学**：许多现代 Web 应用和组件优先考虑干净、无干扰的布局，其中滚动条通常是隐藏的

  


## 如何使用 CSS 隐藏滚动条

  


根据浏览器、容器类型以及是否希望保持滚动功能，CSS 有多种方法可以隐藏滚动条。接下来，我们将探讨最常见的方法：

  


-   使用 `overflow` 属性
-   使用 `::-webkit-scrollbar` 伪元素
-   使用 `scrollbar-width` 属性
-   将 CSS 与自定义滚动解决方案结合使用

  


每种方法都针对特定的使用场景和浏览器支持，接下来我们将逐步解析它们。

  


### 使用 `overflow` 属性

  


在 CSS 中，当内容溢出容器时，滚动条的行为是由 `overflow` 属性来控制的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/33e75a4415674947aefdd4cba9a682a9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=754&s=433899&e=jpg&b=fdfdfd)

  


换句话说，使用 CSS 隐藏滚动条最简单的方法是通过 `overflow` 属性：

  


```CSS
/* 隐藏水平滚动条 */
.no-horizontal-scrollbar {
    overflow-x: hidden;
}

/* 隐藏垂直滚动条 */
.no-vertical-scrollbar { 
    overflow-y: hidden;
}

/* 同时隐藏水平和垂直滚动条 */
.no-scrollbars {  
    overflow: hidden;
}
```

  


实质上它会将溢出容器的内容直接裁剪掉，同时不出现滚动条。虽然这种方法可以完全隐藏滚动条，但也会禁用滚动功能，用户无法访问被隐藏的内容。

  


另一个问题是，当你在某个方向（比如 `overflow-y`） 设置为 `hidden` （即隐藏垂直方向滚动条），另一个方向（比如 `overflow-x`）默认会变成 `auto` ，导致两个方向都被裁剪。这种行为在某些特定的场景下可能不是我们想要的，有时候我们希望只在一个方向裁剪，而另一个方向保持滚动或常显示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4532c8d34390459d95b3a650728e691b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1102&h=452&s=596340&e=gif&f=243&b=ffffff)

  


也就是说，一旦一个方向（比如 `y`）设置为 `hidden`，另一个方向（比如 `x`）就会自动变成 `auto` 。[W3C 规范对这个现象有过描述](https://www.w3.org/TR/css-overflow-3/#overflow-properties)：

  


> 如果 `overflow-x` 或 `overflow-y` 中有一个不是 `visible` 或 `clip`，那么 `visible` 会被计算成 `auto`。

  


而如果 `overflow-y` 设置为 `clip`，那么 `overflow-x` 就可以保持原样（比如 `visible`），这意味着我们可以只在一个方向上裁剪内容。换句话说，如果你只是希望隐藏一个方向的滚动条，使用 `clip` 会更好一些。比如：

  


```CSS
/* 隐藏水平滚动条 */
.no-horizontal-scrollbar {
    overflow-x: clip;
    overflow-y: auto;
}

/* 隐藏垂直滚动条 */
.no-vertical-scrollbar { 
    overflow-y: clip;
    overflow-x: auto;
}

/* 同时隐藏水平和垂直滚动条 */
.no-scrollbars {  
    overflow: hidden;
}
```

  


需要知道的是，将 `overflow` 设置为 `hidden` 或 `clip` ，我们都可以将滚动条隐藏，但需要知道的是，`clip` 同时会禁用 JavaScript 来控制内容的滚动，而 `overflow` 可以仅在视觉上隐藏滚动条，通过 JavaScript 来控制内容滚动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1e18fb2439764e65954abedd70f175e0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=726&s=1885880&e=gif&f=113&b=fefefe)

  


> Demo 地址：https://codepen.io/airen/full/PwwJdpL

  


我们在使用 `overflow` 隐藏滚动条时，需要注意：

  


-   **效果**：滚动条被隐藏，滚动功能完全禁用，尤其是 `clip` ，它将 JavaScript 的滚动功能也禁用了
-   **副作用**：如果内容溢出容器，用户无法访问这些内容。这可能会让用户感到沮丧，影响可用性，尤其是对于重要内容
-   **UI 问题**：避免在动态内容的容器中使用此方法，除非你提供了替代的导航方式（例如，按钮、分页视图）

  


### 使用 `::-webkit-scrollbar` 伪元素

  


在旧版 Chrome、Edge、Safari 以及其他基于 Webkit 内核的浏览器，我们可以使用 `::-webkit-scrollbar` 伪元素选择器来隐藏滚动条：

  


```CSS
.container {
    overflow-x: clip;
    overflow-y: auto;
    max-block-size: 30vmin;
    
    &::-webkit-scrollbar {
        display: none;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1785a83e738143dd840ae29893b4343d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1136&h=698&s=13437663&e=gif&f=135&b=f9f8f8)

  


> Demo 地址：https://codepen.io/airen/full/Qwwqomz

  


需要注意的是，这种写法不是标准规范，而是针对特定浏览器内核的做法，因此兼容范围有限。另外，通过 `::-webkit-scrollbar` ，你还可以对滚动条进行丰富的自定义，比如调整上下箭头按钮的样式，修改滚动条滑块和轨道的颜色、更换背景等等。

  


正如你所看到的，在上面这个示例中，使用 `::-webkit-scrollbar` 选中容器滚动条，然后将其 `display` 设置为 `none` ，滚动条就被隐藏了，但页面仍然可以通过鼠标和键盘进行滚动。但是，这种方法仅适用于 Chrome、Edge 、Safar 以及 Webkit 内核的浏览器。

  


另外，对于旧版本的 IE 浏览器，我们还需要使用 `-ms-overflow-style` 属性来控制滚动条的可见性：

  


```CSS
.container {
    overflow-x: clip;
    overflow-y: auto;
    max-block-size: 30vmin;
    -ms-overflow-style: none; /* IE 和 Edge */
    
    &::-webkit-scrollbar {
        display: none; /*Chrome、Safari 、Edge 和其他 Webkit 内核的浏览器 */
    }
}
```

  


这种方法不适用于 Firefox 浏览器！

  


简单地归纳一下，使用 `::-webkit-scrollbar` 伪元素选择器隐藏滚动条，需要注意以下几点：

  


-   **效果**：完全移除 Webkit 内核浏览器中的滚动条
-   **副作用**：此方法仅在 Webkit 浏览器中有效。对于 IE 浏览器，要使用 `-ms-overflow-style: none` 来隐藏滚动条，但 Firefox 浏览器或其他内核的浏览器，需要额外的属性
-   **UI问题**：类似于 `overflow` 方法，用户可能没有意识到内容是可滚动的，这可能对依赖可见滚动条的视障用户造成困扰

  


### 使用 `scrollbar-width`

  


`scrollbar-width` 属性允许你选择更窄的滚动条，甚至可以完全隐藏滚动条，同时保持滚动功能：

  


-   `auto`：平台提供的默认滚动条宽度
-   `thin`：平台提供的较细的滚动条变体，或比默认滚动条更细的自定义滚动条
-   `none`：有效地隐藏滚动条，但元素仍然可滚动

  


该属性已得到了主流的现代浏览器支持，这意味着，我们可以将滚动容器的 `scrollbar-width` 设置为 `none` 来隐藏滚动条：

  


```CSS
.container {
    overflow-x: clip;
    overflow-y: auto;
    max-block-size: 30vmin;
    scrollbar-width: none; /* 现代主流浏览器，比如 Chrome，Edge，Safari 和 Firefox */
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/58c70c95326c4ed4ad4615f76267275d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1330&h=658&s=3775418&e=gif&f=85&b=faf9f9)

  


> Demo 地址：https://codepen.io/airen/full/RNNLzRK

  


使用 `scrollbar-width` 隐藏滚动条，需要注意以下几点：

  


-   **效果**：用户仍然可以滚动，但滚动条保持隐藏
-   **副作用**：没有可见的滚动条，用户可能不知道内容是可以滚动的，从而降低可发现性。这对于移动用户或小屏幕设备尤其有问题
-   **最佳实践**：结合视觉提示（如箭头或阴影）来表示内容可以滚动

  


简单的小结一下：

  


| **方法**                     | **浏览器支持**                            | **滚动行为**                      | **可视效果** | **适用场景**     | **缺点**                   |
| -------------------------- | ------------------------------------ | ----------------------------- | -------- | ------------ | ------------------------ |
| `overflow: hidden`         | 所有浏览器                                | 完全禁用滚动，但可以通过 JavaScript 来滚动内容 | 隐藏溢出内容   | 内容永远不需要滚动的场景 | 内容变得不可访问                 |
| `overflow: clip`           | 所有浏览器                                | 完全禁用滚动，包括禁用 JavaScript 滚动内容   | 隐藏溢出内容   | 内容永远不需要滚动的场景 | 内容变得不可访问                 |
| `scrollbar-width: none`    | 现代浏览器                                | 保持滚动功能                        | 仅隐藏滚动条   | 内容需要滚动的场景    | 其他浏览器需要额外的 CSS           |
| `-ms-overflow-style: none` | IE 和 Edge                            | 保持滚动功能                        | 仅隐藏滚动条   | 内容需要滚动的场景    | 仅适用于 Microsoft 浏览器       |
| `::-webkit-scrollbar`      | 旧版 Chrome、Safar、Edge 以及 Webkit 内核浏览器 | 保持滚动功能                        | 仅隐藏滚动条   | 内容需要滚动的场景    | 仅适用于 Webkit 内核浏览器，需要厂端前缀 |

  


### 跨浏览器隐藏滚动条

  


不同浏览器的渲染引擎在管理滚动条可见性上各有差异，因此需要使用各种特定厂商前缀的 CSS 伪选择器和属性来实现兼容。

  


对于现代主流浏览器，我们可以将 `scrollbar-width` 设置为 `none` 来隐藏滚动条：

  


```CSS
.container {
    overflow-y: auto;
    overflow-x: clip;
    max-block-size: 50vmin;
    
    /* 现代主流浏览器: Chrome，Edge，Firefox 和 Safari */
    scrollbar-width: none;
}
```

  


对于 IE 或 Edge，我们可以将 `-ms-overflow-style` 设置为 `none` 来隐藏滚动条：

  


```CSS
.container {
    overflow-y: auto;
    overflow-x: clip;
    max-block-size: 50vmin;
    
    /* 现代主流浏览器: Chrome，Edge，Firefox 和 Safari */
    scrollbar-width: none;
    
    /* IE 和 Edge 浏览器 */
    -ms-overflow-style: none;
}
```

  


对于旧版本的 Chrome、Edge、Safari 以及 Webkit 内核的浏览器，我们需要使用 `::-webkit-scrollbar` 伪元素，并设置 `display: none` 来隐藏滚动条：

  


```CSS
.container {
    overflow-y: auto;
    overflow-x: clip;
    max-block-size: 50vmin;
    
    /* 现代主流浏览器: Chrome，Edge，Firefox 和 Safari */
    scrollbar-width: none;
    
    /* IE 和 Edge 浏览器 */
    -ms-overflow-style: none;
    
    /* 旧版本 Chrome，Edge，Safar 以及 Webkit内核浏览器 */
    &::-webkit-scrollbar {
        display: none;
    }
}
```

## 滚动条的隐藏影响 Web 可访问性

  


现在我们知道如何使用 CSS 来隐藏滚动容器的滚动条了。但是，我们需要知道，隐藏滚动条仅是为了 UI 的美观而做出的选择。然而，这种看似简单的设计决定可能会显著影响 Web 可访问性。简单地说，滚动条是至关重要的视觉指示器，它们为用户提供了内容长度和当前位置的空间意识。对于依赖屏幕阅读器的用户，滚动条提供了关于可导航内容的重要上下文，并有助于保持方向感 。

  


我们为了美观而隐藏滚动条，可能会造成以下障碍：

  


-   屏幕阅读器用户会失去关于内容长度的上下文
-   键盘用户可能会错过可滚动区域
-   依赖较大点击目标的行动障碍用户

  


这意味着，让你的网站有一个特定的可滚动内容区域时，保持滚动条可见对可用性和无障碍性有好处。然而，正如之前所说的，常年可见的滚动条在某些情况下可能会影响 Web 界面的美观性。那么问题就来了，美观和可用性如何选择呢？有没有一种两全其美的方案呢？

  


在这种情况下，你可以选择在隐藏了滚动条的区域提供一个其他指示器，比如一个向下的箭头，告诉用户可以滚动容器查看被隐藏的内容。或者，你可以让滚动条仅在悬停时可见。这意味着，如果目标区域没有被使用，滚动条将保持隐藏。

  


```CSS
.container {
    overflow-x: clip;
    overflow-y: hidden;
    max-block-size: 60vmin;

    &:is(:hover,:active,:focus) {
        overflow-y: scroll;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f019172d1ec54bf195446be7ed1e422f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=664&s=5563165&e=gif&f=77&b=fbfafa)

正如你所看到的，悬停触发了垂直滚动条的出现，但这也同时产生了布局偏移。因为浏览器调整了可滚动元素以适应垂直滚动条，这种调整可能会破坏整体的视觉流畅性。

  


为了削除这种偏移并实现平滑的滚动条出现效果，你可以使用 `scrollbar-gutter` 属性，该属性本质上为滚动条可能引起的布局偏移做好准备。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/90ec8ed605d24affb2c4deb0581d581b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2332&h=1179&s=1427112&e=jpg&b=dadff4)

  


通过将 `scrollbar-gutter` 设置为 `stable both-edges` ，它会从两个边缘进行预调整，以保持比例一致的外观：

  


```CSS
.container {
    overflow-x: clip;
    overflow-y: hidden;
    max-block-size: 60vmin;
    
    scrollbar-gutter: stable both-edges;

    &:is(:hover,:active,:focus) {
        overflow-y: scroll;
    }
}
```

  


为了进一步提升效果，你可以 **[自定义滚动条 UI 风格](https://juejin.cn/book/7199571709102391328/section/7199846155080400899)**：

  


```CSS
:root {
    --scrollbar-color-thumb: #888;
    --scrollbar-color-track: #f1f1f1;
    --scrollbar-width: thin;
    --scrollbar-width-legacy: 5px;
}
.container {
    overflow-x: clip;
    overflow-y: hidden;
    max-block-size: 60vmin;
    
    scrollbar-gutter: stable both-edges;
    
    /* 现代浏览器支持 `scrollbar-*` 属性 */
    @supports (scrollbar-width: auto) {
        scrollbar-color: var(--scrollbar-color-thumb) var(--scrollbar-color-track);
        scrollbar-width: var(--scrollbar-width);
    }
    
    /* 旧版浏览器支持 `::-webkit-scrollbar-*` 伪元素 */
    @supports selector(::-webkit-scrollbar) {
        &::-webkit-scrollbar-thumb {
            background: var(--scrollbar-color-thumb);
            border-radius: calc(var(--scrollbar-width-legacy) * 2);
        }
        
        &::-webkit-scrollbar-track {
            background: var(--scrollbar-color-track);
        }
        
        &::-webkit-scrollbar {
            max-width: var(--scrollbar-width-legacy);
            max-height: var(--scrollbar-width-legacy);
        }
    }

    &:is(:hover,:active,:focus) {
        overflow-y: scroll;
    }
}
```

需要知道的是，移动端是不支持 `:hover` 的，[因此你需要通过媒体查询 `@media` 单独为 `:hover` 进行处理](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-13)，有关于这方面更详细的介绍，请移步阅读《[CSS 媒体查询新特性：`@media`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)》！

  


虽然我们通过 CSS 可以直接隐藏容器滚动条，但我个人认为，与其完全隐藏滚动条，不如通过 CSS 对滚动条进行样式调整，使其不那么显眼。比如创建一个细而不显眼的滚动条，它既可以提示滚动区域来增强用户体验，又不会过于打扰 Web 界面 UI效果！

  


## 小结

  


CSS 提供了多种隐藏或自定义滚动条的方式，在美观和可用性之间取得平衡。虽然隐藏滚动条可以使 UI 更简洁，但可能会影响内容的可发现性和可访问性。可以选择极简的设计或使用视觉提示（如箭头）来指示可滚动性。并且要在不同浏览器和设备上进行测试，确保一致性。
