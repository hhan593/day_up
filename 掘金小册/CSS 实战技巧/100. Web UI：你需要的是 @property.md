`@property` 规则最初是 [CSS Houdini API ](https://developer.mozilla.org/en-US/docs/Web/API/CSS_Properties_and_Values_API/guide)的一部分，现在它已经得到了所有主流浏览器的支持。`@property` 的功能为 CSS 自定义属性（也称为 CSS 变量）解锁了新的控制和灵活性，让你的样式表变得更加智能和动态。虽然我在《[现代 CSS](https://s.juejin.cn/ds/ve7yRMIl7C0/)》中专门花了一节课的篇幅介绍了 `@property 规则`，不过那时它还是 CSS Houdini API 的一部分。

  


现如今，它已然成为 CSS 中非常重要的一个特性，并且大大地提升了 CSS 的能力！更为重要的一点是，随着它得到所有主流浏览器的支持后，我们在使用 CSS 自定义属性（或 CSS 变量）时，有了更多的选择。甚至是，在一些情景中（比如一个设计系统）使用 `@property` 要比使用 [CSS 的自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)要强得多。

  


你可能会感到好奇——不都是 CSS 自定义属性（或 CSS 变量）吗？为什么使用 `@property` 要更具优势？接下来，让我们一起来掘开其中的奥秘。在这个过程中我们还将会了解到它的内部工作原理。

  


## 回顾一下 CSS 的自定义属性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a914b222c8147608c709f529a4e7772~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1200&s=110275&e=jpg&b=f2f2f2)

  


时至今日，我想你对 CSS 自定义属性（或称为 CSS 变量）应该不会感到陌生了。如果你还从未接触过 CSS 自定义属性的话，那么我个人建议你先花点时间阅读下面这两节课的内容：

  


-   [CSS 自定义属性你知道多少](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)
-   [CSS 自定义属性可以用来做些什么](https://juejin.cn/book/7223230325122400288/section/7252964839705247755)

  


需要知道的是，在描述 CSS 中的占位符值时，“自定义属性”和“CSS 变量”这两个术语经常被互换使用，尽管它们是不同但又相关的两个概念。W3C 规划中的这个示例很好的阐述这两个概念的差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5ab01c9be1af418a83636fd573efc859~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=426&s=145351&e=png&b=f8f6e9)

  


简单地说，在 CSS 代码块中使用 `--` 注册的属性称为“自定义属性”，但只有被 `var()` 引用的“CSS自定义属性”才能被称为“CSS 变量”，而且其值由相关的自定义属性定义。

  


通常情况下，我们会将 CSS 变量用作某个值的占位符，以便在多个地方复用该值，这样可以避免重复书写相同的值，也便于日后统一修改。

  


```CSS
:root {
    --colorPrimary: magenta;
}

.element {
    box-shadow: 0 0 15px 25px var(--colorPrimary);
}
```

  


现在，我们还可以使用 `@property` 在 CSS 中注册自定义属性：

  


```CSS
@property --colorPrimary {
    syntax: '<color>';
    initial-value: magenta;
    inherits: false;
}
```

  


你可以像使用其他 CSS 自定义属性一样，通过 `var()` 函数来调用 `--colorPrimary` 自定义属性：

  


```CSS
.element {
    box-shadow: 0 0 15px 25px var(--colorPrimary);
}
```

  


然而，这里的区别是， `--colorPrimary` 不仅仅被当作字符串读取，它还包含数据！换句话说，不是 `@property` 注册的 CSS 自定义属性，它作为 CSS 变量使用（`var()` 函数引用）时会被识别为字符串，而在某些情景中（比如过渡动效）时，它是无法进行插值计算，最典型的案例就是为渐变颜色添加动画效果。这也正是 `@property` 的用武之地：**它允许我们不仅定义变量的值，还可以定义它的语法、初始值和是否继续，就像 CSS 规范中所描述的那样**。

  


我们来看一个简单的示例。我们使用 `@property` 注册了一个名为 `--circleSize` 的自定义属性，这个属性的值是一个百分比，初始值为 `10%` ，并且不会被子元素继承：

  


```CSS
@property --circleSize {
    syntax: "<percentage>";
    inherits: false;
    initial-value: 10%;
}
```

  


注册完之后，可以像下面这样引用 `--circleSize` ：

  


```CSS
.circle {
    clip-path: circle(var(--circleSize) at right bottom);
    transition: --circleSize 300ms linear;
    
    section:hover & {
        --circleSize: 200%;
    }
}
```

  


在这个例子中，我们使用 `circle()` 函数将 `.circle` 元素裁剪成一个圆形，这个圆形的半径大小是通过 `@property` 注册的自定义属性 `--circleSize` 来控制的。当鼠标悬浮在 `<section>` 元素上时，我们改变了 `--circleSize` 属性的值，并通过过渡动画展现变化。最终的效果类似于 Material Design 的水波纹动画：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1a48f1118854ddd9ebea22d26bfb3a7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1512&h=466&s=329552&e=gif&f=127&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/GgRbjrX

  


之所以 `transition` 对 `--circleSize` 做过渡动画处理有效果，是因为 `@property` 告诉浏览器，`--circleSize` 是一个百分比，而不是一个字符串。这意味着，我们现在可以自由定义和规范自己的 CSS 属性，这为我们带了新的动画超能力——比如以前只能用 JavaScript 实现的渐变颜色过渡，现在也可以用纯 CSS 实现了。稍后会介绍！

  


## `@property` 的好处

  


`@property` 规则允许 Web 开发者显式地定义自定义属性（也称为 CSS 变量）。这带来了几个好处：

  


-   **属性类型检查和约束**：可以指定自定义属性的数据类型，例如 `<number>` 、`<color>` 或 `<length>` 等。这告诉浏览器你的自定义属性持有什么样的数据，有助于防止错误并确保自定义属性的正确使用，对设计系统非常有利，毕竟大家都喜欢少犯错
-   **设置默认值**：可以通过 `initial-value` 属性为自定义属性设置默认值。这样可以确保在样式表的其他地方未设置该属性时，它依然有一个回退值。也就是说，如果后续赋予无效值，浏览器也会优雅地使用你定义的回退值（即默认值），这样就可以避免样式消失
-   **定义继承行为**：我们可以选择自定义属性是否可以继承父元素的值。这样，我们可以管理自定义属性在级联中的应用方式
-   **改进的错误处理**：增强的类型安全性和回退机制，打开了直接在 CSS 中进行测试和验证的新机会

  


## `@property` 基础

  


`@property` 规则为我们提供了一种更结构化和更强大的方式来创建和使用自定义属性。这为定制和主题化开辟了新的可能性。下面是如何定义一个名为 `--primary-color` 的自定义属性，初始值为 `hotpink` 的例子：

  


```CSS
@property --primary-color {
    syntax: "<color>";
    inherits: true;
    initial-value: hotpink;
}
```

  


这个定义指定了 `--primary-color` 属性的值类型是 `<color>` ，可以继承父元素的值，并且其默认值为 `hotpink` 。

  


定义好自定义属性后，就可以像其他任何属性一样在 CSS 声明中使用它：

  


```CSS
body {
    background-color: var(--primary-color);
}
```

  


上面的代码会将 `body` 元素的背景颜色设置为 `--primary-color` 自定义属性的值，默认下是 `hotpink` 颜色。这种方法的好处在于，要是有人将该属性（`--primary-color`）更新为一个数字值，`body` 元素的背景颜色仍然会是 `hotpink` ，因为在定义 `--primary-color` 时定义了回退值（即 `hotpink`）。

  


注意，你还可以通过 `CSS.registerProperty()` 在 JavaScript 中注册 CSS 自定义属性，比如：

  


```JavaScript
CSS.registerProperty({ 
    name: '--primary-color', 
    syntax: '<color>', 
    inherits: true, 
    initialValue: 'hotpink' 
}) 
```

  


你也可以使用 JavaScript 来操作自定义属性值，例如：`style.setProperty("--primary-color", "hotpink");`。

  


现在我们已经掌握了基础，接下来就是有趣的部分！让我们来谈谈 `@property` 给我们带来的新机会。

  


## 属性类型检查和约束

  


在学习 TypeScript 时，一个很好的起点是从类型原语开始。在 CSS 中，一些类型原语包括：

  


-   **`<length>`** ：任何有效的长度值，例如 `10px` ，`10vw` 等
-   **`<number>`** ：任何有效的数字值，例如 `1` 、`1000` 等
-   **`<percentage>`** ：任何有效的百分比值，例如 `10%`
-   **`<length-percentage>`** ：任何有效的长度值（`<length>`）或百分比值（`<percentage>`），例如 `10px` 或 `10%`
-   **`<color>`** ：任何有效的颜色值，例如 `red` 、`#fff` 、`rgb(0 0 0)` 等
-   **`<image>`** ：任何有效的图像值，例如 `url()` 函数引入的图像，`<gradient>` （渐变）绘制的图像等
-   **`<url>`** ：任何有效的 `url 值`，例如 `url(star.png)`
-   **`<integer>`** ：任何有效的整数值，例如 `1` 、`2` 等
-   **`<angle>`** ：任何有效的角度值，例如 `360deg` 、`400grad` 、`1rad` 和 `1turn` 等
-   **`<time>`** ：任何有效的时间值，例如 `.2s` 、`200ms` 等
-   **`<resolution>`** ：任何有效的分辨率值，例如 `dpi`、`dpcm` 和 `dppx`等
-   **`<transform-list>`** ：任何有效的变换函数，例如 `rotate()` 、`translate()` 等
-   **`<custom-ident>`** ：任何有效的 `ident 值`，例如 `easing` 、`linear` 等

  


除了上面所列的类型之外，还有诸如 `<string>` 、`<dimension>` 、`<ratio>` 、`<flex>` 、`<frequency>` 、`<position>` 、`<hue>` 和 `<url>` 等。更多的 CSS 值类型可以查看 [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Types) 和 csswg.org 上的[完整语法和类型列表](https://csswg.org/indexes/#types)。

  


通过使用 `@property` 来定义具有明确类型的自定义属性，我们可以更轻松地维护和更新代码。这使得在项目发展过程中修改自定义属性时更加节省时间，并减少错误。当我们构建一个会被第三方使用的设计系统时，这也会带来更大的优势，因为它能帮助 Web 开发者在开发工具中快速发现问题。

  


我们来举个例子。使用 `@property` 定义了一个 `<color>` 值类型的自定义属性 `--color` ：

  


```CSS
@property --color {
    syntax: "<color>";
    inherits: true;
    initial-value: hotpink;
}
```

  


假设我们有三张卡片，我们需要为每张卡片设置不同的背景颜色：

  


```HTML
<div class="card"></div>
<div class="card"></div>
<div class="card"></div>
```

  


```CSS
.card {
    background-color: var(--color);
    
    &:nth-child(1) {
        --color: #1a535c;
    }
    
    &:nth-child(2) {
        --color: rgb(100 200 0);
    }
    
    &:nth-child(3) {
        --color: oklch(0.5 0.22 272.56);
    }
}
```

```CSS
@property --hue {
    syntax: "<angle>";
    inherits: false;
    initial-value: 0deg;
}
```

  


  


再来看一个类型化的属性定义：


```CSS
@property --hue {
  syntax: '<angle>';
  initial-value: .5turn;
  inherits: false;
}

.card {
    --bg: oklch(50% .02 var(--hue));
    background-color: var(--bg);
    
    &:nth-child(1) {
        --hue: 45deg;
    }
    
    &:nth-child(2) {
        --hue: .5turn;
    }
    
    &:nth-child(3) {
        --hue: 100grad;
    }
}
```

  


上面代码中的 `--bg` 是一个 `<any>` 类型（因为目前它是一个未指定类型的自定义属性），并且包含了一个嵌套的自定义属性 `--hue` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ec4a245cf0de4ab6890cc374ae4989e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2518&h=988&s=69221&e=png&b=ffffff)

事实上，你可以嵌套多个级别，但建议不要嵌套得太深。而且，你可以为一些或所有变量设置类型。

  


有一点需要知道的是，`@property` 在定义自定义属性类型时，不会局限于单一类型，通过不同的语法规则，我们可以给自定义属性设置不同的类型或多个类型：

  


-   `+` （`U+002B`）：接受以空格分隔的列表，例如 `<length>+` ，表示接受以空格分隔的长度值（`<length>`）列表
-   `#` （`U+0023` ）：接受以逗号分隔的列表，例如 `<color>#` ，表示接受以逗号分隔的颜色值（`<color>`）列表
-   `|` （`U+007C` ）：接受以竖线分隔的列表，例如 `<length> | <lenthg>+` ，表示接受单个长度值（`<length>`）或者以空格分隔的长度值列表

  


例如：

  


```CSS
@property --c {
    syntax: "<color>"; /* 接收一个颜色值 */
    initial-value: hotpink;
    inherits: false;
}

@property --s {
    syntax: "<length> | <percentage>"; /* 接收长度或百分比参数，但是二者之间不进行计算合并 */
    initial-value: 0px;
    inherits: false;
}

@property --thickness {
    syntax: "small | medium | large";  /* 接收这些参数值之一作为自定义标识符 */
    initial-value: small;
    inherits: false;
}
```

  


## 定义继承行为

  


众所周知，CSS 中属性可以分为可继承和不可继承两种，比如 `font-size` 是一个可继承属性，`display` 和 `background-color` 是不可继承属性：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3f7218d5340347ac8806dbfd3f8b734c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1660&h=780&s=762072&e=jpg&b=deebfb)

  


`@property` 在定义一个自定义属性时，也可以通过 `inherits` 属性来指定该自定义属性是否可继承。如果 `inherits` 属性的值设置为 `true` ，表示该自定义属性的值可以被子元素继承。这意味着，如果我们在父元素上设置了自定义属性的值，那么该值将应用于所有子元素，除非它们为该属性设置了自己的值。这也是自定义属性的默认行为。

  


我们继承以 `--color` 自定义属性为例，并且在第二张卡片中嵌套一个子元素。现在更新后的 HTML 如下：

  


```HTML
<div class="card"></div>
<div class="card">
    <div class="inner"></div>
</div>    
<div class="card"></div>
```

  


我们给 `.inner` 添加一些样式。为了清楚地看到这个元素，我特意调整了它的位置和添加了一个黑色的边框：

  


```CSS
.inner {
    padding: 1rem 2rem;
    display: grid;
    place-content: center;
    place-self: end center;
    margin-bottom: -2rem;
    border: 4px solid #000;
    background-color: var(--color);
}
```

  


注意，我们在使用 `@property` 定义 `--color` 属性时，它的 `inherits` 属性的值为 `true` ：

  


```CSS
@property --color {
    syntax: '<color>';
    initial-value: hotpink;
    inherits: true;
}
```

  


也就是说，应用于 `.inner` 元素的 `background-color` 属性值 `--color` 会继承它的父元素的 `--color` 属性的值（即 `#f90`）：

  


```CSS
.card {
    background-color: var(--color);
    
    &:nth-child(1) {
        --color: #90acef;
    }
    
    &:nth-child(2) {
        --color: #f90;
    }
    
    &:nth-child(3) {
        --color: #f50;
    }
}
```

在这个时候，`.inner` 相当于将 `background-color` 属性的值设置为 `inherit` 。

  


> 温馨提示，如果你对 `inherit` 不太了解，那么我建议你花点时间阅读《[CSS 显式默认值：`inherit`，`initial`，`unset` 和 `revert`](https://juejin.cn/book/7223230325122400288/section/7232094160071688227)》！

  


也就是说，当 `.inner` 元素未显式指定 `--color` 属性的值时，它会继承父元素的 `--color` 属性的值。这也意味着，如果我们不希望 `.inner` 继承父元素的 `--color` 属性的值，只需要在该元素上显式设置 `--color` 属性的值即可，比如：

  


```CSS
.inner {

}
```

现在，让我更新自定义属性的声明，并将 `inherits` 设置为 `false` ：

  


```CSS
@property --color {
    syntax: "<color>";
    inherits: false;
    initial-value: hotpink;
}
```

  


而且我们在第一张卡片内的 `.inner` 显式设置 `--color` 的值为 `#cef` ，但在第二张卡片内的 `.inner` 不显式设置 `--color` 的值。这个时候呈现给你的结果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/22b4f7a17a8f4ce581e7c919c2447c75~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2518&h=988&s=67618&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ByagdOe

  


这个结果并不难理解，不管在定义 `--color` 属性时，`inherits` 是任何值（`false` 或 `true`），只要在应用的时候显式设置了 `--color` 的值（比如第一张卡片中的 `.inner` 元素），它都会以自身设置的 `--color` 为准。要是在定义 `--color` 时，`inherits` 的值为 `false` ，并且在 `.inner` 元素中未显式设置 `--color` 值，此时 `.inner` 元素不会继承它父元素或祖先元素的 `--color` 值，只会引用 `--color` 的初始值（比如第二张卡片中的 `.inner` 元素）。

  


这意味着，我们使用 `@property` 定义一个自定义属性时，应该根据属性的功能或作用来决定 `inherits` 属性的值为 `false` 或 `true` ：

  


-   如果我们需要更多控制个别元素样式并避免不必要的覆盖时（比如，在设计系统或 Web 组件中，或者会影响元素大小时），我们应该考虑将 `inherits` 设置为 `false` 。这样可以简化我们的 CSS 代码，使其更易于维护
-   如果我们希望一组元素的层级中保持一致的样式，比如定义的是颜色或字体相关的样式，那么将 `inherits` 设置为 `true` 是更适当的

  


因此，我们将 `inherits` 设置为 `false` 时应该谨慎使用，以避免在多人协作的大型项目中引起混淆。但这只是我个人的看法，写好文档可以大大减少这种困惑，好比 W3C 规范中的属性，它都有明显的指定该属性是否可继承！

### 设计系统的相关性

  


让我们构建一个类型安全的暗黑模式自适应配色方案。

  


首先，我们使用 `@property` 定义一个类型安全的品牌色调，比如 `--hue`。你可以在示例中的输入框输入内容，而且这个输入框的内容会赋值给 `--hue` 。由于我们给它指定了值类型，可以看到其他依赖于 `--hue` 的自定义避税性在值被设置为无效值时不会崩溃。

  


```CSS
@property --hue {
    syntax: "<angle>";
    initial-value: 5rad;
    inherits: true;
}
```

  


为了简洁起见，我们以 Web 上的卡片为例，给卡片设置自适应配色方案，这将为输入设计系统的过程提供相应的参考。

  


在这里，我新增了另外三个自定义属性，分别是 `--surface` 、`--surface-over` 和 `--surface-under` ，其中 `--surface` 将应用于页面的背景颜色，另外两个分别用卡片元素的背景颜色。同样使用 `@property` 来定义它们：

  


```CSS
@property --surface {
    syntax: '<color>';
    initial-value: #333;
    inherits: true;
}

@property --surface-over {
    syntax: '<color>';
    initial-value: #444;
    inherits: true;
}

@property --surface-under {
    syntax: '<color>';
    initial-value: #222;
    inherits: true;
}
```

  


这里重要的一点是，它们的类型是颜色类型（`<color>`）。

  


现在，我们可以将这些自定义属性应用于页面的元素上。在这里，我使用了[ CSS 的 `:has()` 选择器](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)和 [`light-dark()` 函数](https://juejin.cn/book/7395119173656903715/section/7445262578945064987)来实现暗黑模式颜色切换：

```CSS
}

    color-scheme: light;
}

html:has(#dark:checked) {
    color-scheme: dark;
}
  

main {
    background-color: light-dark(var(--l-surface), var(--d-surface));
}

.card {
    --shadow: light-dark(var(--l-surface-under), var(--d-surface-under));
    background-color: light-dark(var(--l-surface-over), var(--d-surface-over));

    &:nth-child(2) {

        --shadow-o: light-dark(var(--l-surface-over), var(--d-surface-over));
        background-color: light-dark(var(--l-surface-under),var(--d-surface-under));
        box-shadow: 
            inset 0 0 5px color-mix(in oklch, var(--shadow), oklch(0 0.1 var(--hue)) 10%),
            inset 0 1px 0 color-mix(in oklch, var(--shadow), oklch(0 0.1 var(--hue)) 10%),
            inset 0 -1px 0 var(--shadow-o);

        background-color: light-dark(var(--l-surface-under), var(--d-surface-under));
    }
}
```

### 色相旋转轮的动效

  


`@property` 另一个常见的应用是色相旋转轮的动效。我们通常会像下面这样来创建一个色相轮的旋转动画：

  


```HTML
<div class="wheel"></div>
```

  


```CSS
:root {
    --hue: 0deg;
}

@keyframes spin {    
    to {
        --hue: 360deg;
    }
}

.wheel::after {
    background: hsl(var(--hue) 90% 50%);
    animation: spin 5.5s linear infinite;
}
```

  


上面代码中的 `@keyframes` 定义的 `spin` 动画并没有生效，因为浏览器不知道如何动画化色相旋转（`--hue`）。然而，使用 `@property` ，我们可以明确指示浏览器将 `--hue` 变量解释为 `<angle>` 类型，从而实现平滑的动画效果：

  


```CSS
@property --hue {
    syntax: "<angle>";
    inherits: false;
    initial-value: 0deg;
}
```

### 动画化渐变背景

## 综合案例

  


最后我们通过下面这个效果，来进一步深入学习如何使用 `@property` 规则定义自定义属性：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1fa30f462dd846e09c440e2447cd0da0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1132&h=524&s=5846069&e=gif&f=231&b=ffffff)

  


  


HTML 代码如下：

  


```HTML
<div class="container">
    <div class="scrolling">
        <div class="content">
            <span>ONE</span> 
            <span>TWO</span> 
            <span>THREE</span>
        </div>
    </div>
</div>
```

  


上面代码包含了我们即将进行动画处理的文本字符。实际会滑动的文本放在 `.content` 中，实现文本滑入滑出视图的效果。

  


接下来在 CSS 中实现鼠标悬停时文本的垂直滑动。我们要做的是创建一个有固定高度的容器，当内容超出这个高度时就被裁剪（隐藏）掉。

  


```CSS
.scrolling {
    width: min-content;  /* 宽度为最小内容 */
    height: 1lh;        /* 设置容器高度为一行文字的高度 */
    overflow: hidden;   /* 溢出部分隐藏 */
}

.content {
    transition: translate 2.4s ease-in-out; /* 添加平滑过渡动画 */
    
    .container:hover & {
        translate: 0 -2lh; /* 悬停时让文本上移两行高度 */
    }
}
```

  


将 `.container` 容器的宽度设置为 `min-content` 可以让字符紧密排列，垂直堆叠成一列。容器的高度设置为 `1lh` ，即一行文字的高度。而由于我们对容器设置了 `overflow: hidden` ，所以在任意时刻，容器中只会显示一行字符。

  


当鼠标悬浮时，文本会向上移动 `2lh` ，也不是两行文本字符高度的距离。这意味着我们是将内容向上滑动两行字符的高度，从而在容器被悬浮时实现从第一行字符滑动到第三行字符的动画效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6f0797c9ec6640c094eae659f8e9acf5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1132&h=524&s=3929139&e=gif&f=207&b=fffefe)

  


> 温馨提示，`min-content` 是 CSS 的内在尺寸之一，如果你从未接触过 CSS 的内在尺寸，那么建议你花点时间阅读《[内在 Web 设计](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)》！

  


就这个平移动画效果而言，只要对 CSS 的 `transition` 和 `translate` （变换）有所了解，实现起来都不会有任何难度，关键是这里高度的设置，`lh` （相对行高计算）免去了很多不必要的计算！

  


接下来我们将应用《[Web UI：创建渐变文字与投影效果](https://juejin.cn/book/7395119173656903715/section/7446278062612463643)》课程中所提到的技巧，给文本添加渐变效果：

  


```CSS
.content {
    background: repeating-linear-gradient(
        var(--angle),
        var(--c1),
        var(--c1) 5px,
        var(--c2) var(--l),
        var(--c2) 6px
    );
    background-clip: text;
    color: transparent;
    background-size: 20% 20%;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/74ba46a435d8473682dd04d739626c5a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2570&h=1052&s=129923&e=png&b=ffffff)

  


挺酷的吧？现在看起来，我们的文字字符上像是被刷上了一层条纹图案。

  


你可能已经发现了，在 `repeating-linear-gradient()` 渐变函数中我们应用了几个 CSS 自定义属性： `--c1` 、`--c2` 、`--l` 和 `--angle` ，其中 `--c1` 和 `--c2` 是两个颜色，`--l` 是一个长度值，`--angle` 是一个角度值。更为重要的是，这些 CSS 自定义属性都是由 `@property` 定义的：

  


```CSS
@property --c1 {
    syntax: "<color>";
    inherits: false;
    initial-value: #1ce4ea;
}

@property --c2 {
    syntax: "<color>";
    inherits: false;
    initial-value: #f6de08;
}

@property --l {
    syntax: "<length> | <percentage>";
    inherits: false;
    initial-value: 5px;
}

@property --angle {
    syntax: "<angle>";
    inherits: false;
    initial-value: 180deg;
}
```

  


因为我们希望在容器悬浮时更新这些自定义属性的值，并且在更新的过程中它们有一个平滑过渡的动画效果：

  


```CSS
.content {
    background: repeating-linear-gradient(
        var(--angle),
        var(--c1),
        var(--c1) 5px,
        var(--c2) var(--l),
        var(--c2) 6px
    );
    transition: --l, --angle, --c1, --c2, background-size,translate 2.4s ease-in-out;
    transition-duration: 2s;
    
    .container:hover & {
        --c1: tomato;
        --c2: peachpuff;
        --l: 80%;
        --angle: 90deg;
        translate: 0 -2lh;
        background-size: 50% 100%;
    }
}
```

  


为了更清晰地了解发生了什么，下面是悬停时更新的自定义属性及其变化：

  


-   `--c1`：从 `#1ce4ea` 过渡到 `tomato`
-   `--c2`：从 `#f6de08` 过渡到 `peachpuff`
-   `--l`：从 `5px` 变为 `80%`
-   `--angle`：从 `180deg` 变为 `90deg`

  


因此，渐变中的两个颜色会渐变为新的颜色，同时整个渐变的尺寸也在增长，并发生了旋转。最终的效果如下所示：

我希望这个小案例不仅展示了我们可以用 CSS 自定义属性实现各种炫酷效果，同时也有助于你更清楚地区分“自定义属性”和“标准变量”之间的差异。

  


标准变量非常适合作为代码中可维护性的占位符（当然它们本身也能实现一些小技巧），但当你需要更新某个包含多个值的属性中的**某一个值**时——比如渐变中的颜色——这时候就轮到 `@property` 出场了。它允许我们为变量自定义规范，包括语法格式、初始值和继承行为。

  


当我们能够逐一、独立地修改这些值，并且还能实现平滑的动画，这不仅让代码结构更加清晰，还为我们以较少的代码实现复杂动画效果打开了新的可能性。

  


> 这就是为什么 `@property` 是一个值得牢记、随时可用的 CSS 标准工具，尤其当你在思考那些涉及**独立值变动**的动画设计时，它非常有用。

  


## 小结

  


CSS 注册的自定义属性（`@property`）是一个非常强大的功能，通过为 CSS 变量提供意义和上下文，扩展了 CSS 语言。现在，随着 `@property` 在基础版本中的引入，这一 CSS 超能力正在变得越来越强大。

  


可以说，`@property` 改变了 CSS 自定义属性（或称为 CSS 变量）的规则。对于需要创建设计系统或精美动画的人来说，这无疑是一个极好的补充。记住，课程中所展示的只是一些基础示例。我更期望你在这个基础上创建出更多有趣的用例，我在评论中期待着你们的分享！
