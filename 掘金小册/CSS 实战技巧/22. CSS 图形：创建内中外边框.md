作为一名 Web 前端开发者，我想你应该使用过诸如 Figma 和 Sketch 这样的图形设计软件。不知道你是否有留意过，在图形设计软件中（例如 Figma），给图形添加边框时（通常称为描边），它有内边框（Inside）、中间边框（Center）和外边框（Outside）三种类型。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/15b40a6388204c6c94b0230814b4f002~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1432&h=780&s=737174&e=gif&f=235&b=f1f1f1)

庆幸的是，我们可以通过一些技巧可以模拟出类似 Figma 设计软件中的边框类型。接下来，我们就分别来讨论在 CSS 和 SVG 中应该如何给元素设置内边框、中间边框和外边框。

-   [SVG 文本元素](https://juejin.cn/book/7341630791099383835/section/7346773005114507304)

-   [SVG 遮罩](https://juejin.cn/book/7341630791099383835/section/7366549423813296165)（`<mask>` ）和 [CSS 遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)（`mask`）

## 内边框、中间边框和外边框简介

  


首先要明确一点，内边框、中间边框和外边框等相关概念是源于诸如 Figma 和 Sketch 这样的图形设计软件中的图形形状描边（Stroke）类型。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6a9ef175aedb43609926f86fde3cd108~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=660&s=83682&e=jpg&b=fcfcfc)

  


如上图所示：

  


-   **内边框（Inside Border）** 是在图形内部绘制的边框，边框不会溢出元素框的边缘
-   **中心边框（Center Border）** 一半厚度位于图形或元素框外部，一半厚度位于图形或元素框内部
-   **外边框（Outside Border）** 是在图形外部绘制的边框，整个边框位于元素框的边缘外部

  


默认情况下，CSS 的 `border` 会给元素设置一个外边框，而 `text-stroke` 给文本描边时，它是一个中间边框。在 SVG 中使用 `stroke-width` 和 `stroke` 给图形元素或文本元素描边时，它默认也是中间边框。最为关键的是，CSS 或 SVG 的图形元素中并没有任何一个属性可以用于指定其位置。因此，我们需要通过其他的一些技巧使其在视觉上实现内边框、中间边框和外边框的效果。

  


## CSS 创建内中外边框

  


CSS 的 `border` 默认给元素添加边框时，就是一个外边框，只不过它并没有类似一个 `border-offset` 的属性来调整其位置。因此，如果你需要给元素添加一个内边框或中间边框，需要采用一些 Hack 技巧来实现。我们可以通过 CSS 的 `outline` 或 `box-shadow` 来实现内边框和中间边框。

  


### 创建内边框

  


**内边框的特点是在图形（或元素）内部绘制边框**。我们通过具体的实例来向大家展示，在 CSS 中如何给元素设置内边框。

  


假设你现在有一个 `.box` 元素，它具有一些基本样式：

  


```CSS
.box {
    width: 50vw;
    aspect-ratio: 21 / 9;
    background: rgb(120 90 120 / .5);
    border: 1px dashed;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/614b343bb371466f96416631e4db5e90~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=820&s=145673&e=jpg&b=060606)

  


注意，上图中的白线虚线实际上是元素 `.box` 真正的边框。在这里我使用它来模拟设计软件中的笔触，它相当于下图中的蓝色边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d1586cd045ac471db48672b5d22d37e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3346&h=1178&s=205573&e=png&b=f3f3f3)

  


我们的目标是要给 `.box` 元素设置了个内边框，假设这个内边框的厚度为 `--b` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d6b6ea0f8c1a461fb8b12cf5b44d1341~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=820&s=176506&e=jpg&b=050505)

  


最简单的方式是[使用 `box-shadow` 属性](https://juejin.cn/book/7199571709102391328/section/7199844993455325216#heading-4)，给元素 `.box` 添加一个内阴影，并且它的模糊扩展半径的值为 `--b` 。

  


```CSS
.box {
    --b: round(1em, 1px);
    --c: #09face;
    
    box-shadow: inset 0 0 0 var(--b) var(--c);
}
```

  


第二种方式是使用 `outline` 设置一个厚度为 `--b` 的轮廓，并将 `outline-offset` 设置为负值（`calc(-1 * var(--b))`），将轮廓拉向元素盒子框内：

  


```CSS
.box {
    --b: round(1em, 1px);
    --c: #09face;
    outline: var(--b) solid var(--c);
    outline-offset: calc(-1 * var(--b));
}
```

  


第三种方式则是使用伪元素 `::before` 或 `::after` ，并给伪元素设置一个厚度为 `--b` 的边框。在这里，我们需要将伪元素覆盖在元素之上。为了不影响元素自身的指针事件操作，需要给伪元素设置 `pointer-events: none`：

  


```CSS
.box {
    --b: round(1em, 1px);
    --c: #09face;
    
    position: relative;
      
    &::before {
        content: "";
        border: var(--b) solid var(--c);
        position: absolute;
        inset:0;
        pointer-events: none;
    }
}
```

  


注意，你也可以将伪元素上的 `border` 替换为 `outline` 和 `box-shadow` ，具体代码就展示了。

  


上述三种方式，不管采用哪种方式都允许你给元素创建一个厚度为 `--b` 的内边框。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/36deced3149c487a8bd192863aefc432~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=873&s=247039&e=jpg&b=060606)

  


> Demo 地址：https://codepen.io/airen/full/JjQgRXY

  


除了上述三种方式之外，我们还可以借助盒模型和布局限制框的特性来创建内边框。简单地说，将元素的 `padding` 设置为 `--b` ，并给元素设置一个背景色，然后使用 `background-origin` 和 `background-clip` 将颜色限制在 `padding-box` 区域：

  


```CSS
.box {
    --b: round(1em, 1px);
    --c: #09face;
    
    padding: var(--b);
    background: 
        linear-gradient(rgb(120 90 120) 0 100%) content-box,
        linear-gradient(var(--c) 0 100%) padding-box;
}
```

  


这种方式创建内边框的主要缺陷是，你无法创建一个镂空的边框效果。换句话说，当填充在 `content-box` 区域的背景带有一定透明度时，位于 `padding-box` 区域的背景就会露出，影响整个效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/20c3643bf3dd47e584c6ae2f348396ab~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=873&s=152745&e=jpg&b=060606)

  


> Demo 地址：https://codepen.io/airen/full/oNrKzJe

  


不管使用哪一种方式创建内边框，我们只需要掌握其设计原则：**在图形（或元素）内部绘制边框**，它不应该溢出元素元素所在框，包括视觉上的溢出！

  


### 创建中间边框

  


中间边框的特点是**一半厚度位于图形或元素框外部，一半厚度位于图形或元素框内部**。假设边框厚度为 `--b` ，那么二分之一的 `--b` 位于图形或元素框边缘的外部，另外二分之一的 `--b` 位于图形或元素框边缘的内部：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eb0404ed4b7b4dd0a4cd337ce5d614c9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=820&s=241543&e=jpg&b=050505)

  


在视觉上我们可以将其理解为双重边框，[这意味着我们可以使用之前创建双重边框的方式来创建中间边框](https://juejin.cn/book/7395119173656903715/section/7411340113839390783)：

  


-   `border` 和 `outline` 以及 `outline-offset` 相结合
-   `border` 和 `box-shadow` 相结合
-   `outline` 和 `box-shadow` 相结合
-   `box-shadow` 的外阴影和内阴影相结合
-   `border` 和 `padding` 相结合
-   与伪元素相结合

  


具体代码如下：

  


```CSS
.box {
    --bg: color-mix(in srgb, rgb(120 90 120), #0000 40%);
    --b: round(1em, 1px);
    --_b: calc(var(--b) / 2);
    --c: color-mix(in srgb, #09face, #0000 80%);
    
    position: relative;
    
    /* border + outline */
    &:nth-child(1) {
        border: var(--_b) solid var(--c);
        outline: var(--_b) solid var(--c);
        outline-offset: calc(-2 * var(--_b));
    }
    
    /* border +  box-shadow */
    &:nth-child(2) {
        border: var(--_b) solid var(--c);
        box-shadow: inset 0 0 0 var(--_b) var(--c);
    }
    
    /* outline + box-shadow */
    &:nth-child(3) {
        outline: var(--_b) solid var(--c);
        box-shadow: inset 0 0 0 var(--_b) var(--c);
    }
    
    /* box-shadow */
    &:nth-child(4) {
        box-shadow: 
            inset 0 0 0 var(--_b) var(--c),
            0 0 0 var(--_b) var(--c);
    }
    
    /* border + padding */
    &:nth-child(5) {
        border: var(--_b) solid var(--c);
        padding: var(--_b);
        background: 
            linear-gradient(var(--bg) 0 100%) content-box,
            linear-gradient(var(--c) 0 100%) padding-box;
    }
    
    /* outline + padding */
    &:nth-child(6) {
        outline: var(--_b) solid var(--c);
        padding: var(--_b);
        background: 
            linear-gradient(var(--bg) 0 100%) content-box,
            linear-gradient(var(--c) 0 100%) padding-box;
    }
    
    /* box-shadow + padding */
    &:nth-child(7) {
        box-shadow: 0 0 0 var(--_b) var(--c);
        padding: var(--_b);
        background: 
            linear-gradient(var(--bg) 0 100%) content-box,
            linear-gradient(var(--c) 0 100%) padding-box;
    }
    
    /* CSS pseudo-element */
    &:nth-child(8) {
        border: var(--_b) solid var(--c);
      
        &::before {
            content: "";
            position: absolute;
            inset: 0;
            border: inherit;
            pointer-events: none;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd220dfe10c34eb0a64d21e1f440d0aa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1170&s=440127&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/ExBqZbG

  


需要知道的是，上面阐述的方法只适合为规则的矩形框添加内中外边框，对于不规则形状，这些方法就不适合了。

  


## 文本描边

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/883b1b5e96ea43ba910adee6a690b66c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1170&s=209528&e=jpg&b=050505)

  


你可能会好奇，我们的主题是创建内中外边框，怎么就跟文本描边扯上关系了呢？原因很简单，文本的描边效果在视觉上就像是我们给文本添加了边框效果，只不过这个“边框”是根据文本内容来决定的。

  


我们在这里所讨论的文本描边主要涵盖了两个部分：

  


-   CSS 中使用 `text-stroke` 为 HTML 元素的文本内容添加描边效果
-   SVG 中使用 [`stroke` 和 `stroke-width`](https://juejin.cn/book/7341630791099383835/section/7349188496181887017) 为[文本元素](https://juejin.cn/book/7341630791099383835/section/7346773005114507304)（例如 `<text>` 和 `<tspan>`）的内容添加描边效果

  


```HTML
<h1 class="text">CSS Text Stroke</h1>

<svg viewBox="0 0 600 100">
    <text dx="50%" dy="50%" fill="currentColor" class="text" lengthAdjust="spacingAndGlyphs" dominant-baseline="middle" alignment-baseline="middle" text-anchor="middle">SVG Text Stroke</text>
</svg> 
```

  


使用 CSS 给它们设置一些基本样式：

  


```CSS
.text {
    font-family: inherit;
    font-size: clamp(1.25rem, 3vw  + .875rem, 1.5rem);
    text-align: center;
    line-height: 1;
    font-weight: bold;
}
```

  


到目前为止，文本没有任何描边效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/288c4a29d48646c6af7359c525554f94~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=757&s=181107&e=jpg&b=050505)

  


对于 HTML 元素的文本内容，CSS 有多种方式可以为其添加描边效果，但我们在这里只关注 `text-stroke` 。注意，到目前为止要使 `text-stroke` 生效，需要添加前缀 `-webkit` 。对于 SVG 的 `<text>` 元素，我们可以给元素设置 `stroke` 和 `stroke-width` 属性或在 CSS 设置 `stroke` 和 `stroke-width` 为其添加描边效果：

  


```CSS
.text {
    --b: round(.05em, 1px);
    --c: #f90;
    
    /* HTML 文本描边 */
    -webkit-text-stroke: var(--b)  var(--c);

    /* SVG 文本描边 */
    stroke-width: var(--b);
    stroke: var(--c);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/561e8606a464443c902f416e6a083fe7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=835&s=319617&e=jpg&b=060606)

  


不难发现，默认情况之下，不管是 `text-stroke` 还是 `stroke-width` 和 `stroke` 给文本添加的描边效果都是居中描边。比如，在我这个示例中，`--b` 一半的宽度在“形状”的外部，`--b` 另一半的宽度在“形状”的内部。

  


以往面对这种情况，大部分 Web 开发者会选择 Hack 技巧来调整描边的效果：

  


-   在 CSS 中，会选择伪元素叠加的方式给文本添加描边效果
-   在 SVG 中，会选择更改路径或给定元素的宽高，以避免描边过多的覆盖填充区域

  


在当下，我们可以使用 `paint-order` 属性来改变填充的描边的绘制顺序。

  


`paint-order` 属性的默认值是 `normal` ，它定义了填充（`fill`）先绘制，然后是描边（`stroke`），最后是标记（`markers`）。但你可以重新定义绘制顺序。

  


```CSS
/* 单一属性 */
path {
    paint-order: normal; /* 默认顺序，fill 👉 stroke 👉 markers */
}

path {
    paint-order: stroke; /* stroke 👉 fill 👉 markers */
}

path {
    paint-order: markers; /* markers 👉 fill 👉 stroke */
}
    
/* 多属性 */
path {
    paint-order: stroke fill; /* stroke 👉 fill 👉 markers */
}

path {
    paint-order: markers stroke fill; /* markers 👉 stroke 👉 fill */
}
```

  


当只指定一个值的时候，这个值将会被首先渲染，然后剩下的两个值将会以默认顺序渲染，当只指定两个值的时候，这两个值会以指定的顺序渲染，接着渲染剩下的未指定的那个。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ed602c7f54e64ed9a74cdda3664329fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=673&s=67727&e=jpg&b=fdfdfd)

  


> 注意： 在这个属性的值当中，`markers` 只有当在绘制 SVG 图形时引用了 `marker-*`属性（例如 `marker-start`）和 `<marker>` 元素才能进行控制。

  


需要知道的是，在此之前 `paint-order` 只服务于 SVG 元素。庆幸的是，它也开始支持 HTML 元素。因此，我们可以在 CSS 中将 `paint-order` 属性的值设置为 `stroke` ，告诉浏览器，填充在描边上面。如此一来，不管是 CSS 的 `text-stroke` 属性，还是 SVG 的 `stroke` 和 `stroke-width` 创建的描边效果，都将是向外延伸。

  


```CSS
.text {
    --b: round(.1em, 1px);
    --c: #f90;
    
    -webkit-text-stroke: var(--b)  var(--c);

    stroke-width: var(--b);
    stroke: var(--c);
    
    paint-order: stroke;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce5769d6aac74fb8b9b66008b5e0c502~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=875&s=693473&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/WNqVpeJ

  


需要知道的是，这种方式也适用于 [SVG 的其他元素](https://juejin.cn/book/7341630791099383835/section/7345813971552698406)，比如 `<rect>` 、`<circle>` 、`<ellipse>` 等。遗憾的是，它只能使我们实现外边框描边效果，但无法实现内边框描边效果。

  


## SVG 绘制内中外描边（边框）

  


我们在开发过程中难免会应用 SVG 绘制的矢量图形，这些矢量图形可以通过 SVG 元素的 `stroke` 和 `stroke-width` 等属性设置描边效果，也可以在 CSS 中通过 `stroke` 或 `stroke-width` 属性为 SVG 图形设置描边效果。

  


需要知道的是，SVG 创建的矢量图形或文本设置边框效果，一般被称为描边。而且在矢量图形中，描边是沿着路径绘制的可见线——可以将其看作刷子的笔触。在这种情况下，我指的是包围整个形状的路径，即它的轮廓。

  


通常，描边是居中绘制在形状的边界上：描边的一半在形状内部，另一半在外部。这意味着形状的可见宽度实际上是形状宽度加上描边的宽度。（你在形状的两侧各多出半个描边宽度，也就是额外的描边宽度）。

  


之前也提到过，在诸如 Figma 这样的图形设计软件中，你可以选择绘制内描边或外描边：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4162bc8f30bf470fb642aaeb119788a3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=398&s=30397&e=jpg&b=ffffff)

  


尽管 `<path>` 或 `<polygon>` 绘制的三角形的尺寸相同，但描边的对齐方式改变了最终形状的大小。

  


刚才也提到过了，默认情况之下，SVG 的描边是居中描边，但在实际开发的过程中，我们可能需要的是内描边或外描边。甚至有的时候需要将外描边和内描边结合在一起，创造出一种看起来像条纹描边的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99f4caeee9c0445aaa06e7371c319862~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3320&h=1028&s=529342&e=jpg)

  


另外，使用内外描边通常比结合不同大小的相同形状看起来更好。它提供了一条一致且均匀的线条，而结合不同大小的形状则会产生不整齐的效果。比如下图这样，红色的条纹比蓝色的条纹要粗得多，而且蓝色的条纹的宽度不一致：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/180e5631ac104f56a1cab0613d5a2210~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3320&h=1044&s=495405&e=jpg&b=0e0e0e)

  


对于更复杂的形状，结合不同大小的形状变得更加困难（有时甚至不可能）而不留空隙。

  


那么，问题就来了，如何给 SVG 元素设置外描边和内描边呢？

  


如果仅是给 SVG 元素设置外描边，很好解决，我们之前所介绍的 `paint-order` 属性即可解决，只需将它的值设置为 `stroke` 即可。例如：

```XML
<svg viewBox="0 0 100 100" class="heart">
```

这是使用 `<path>` 绘制的一个“心形”形状，如果未设置描边效果，它们的效果一模一样。现在，使用 `stroke` 和 `stroke-width` 给它们添加描边效果，不同之处是第二个使用 `paint-order: stroke` 调整了 `stroke` 和 `fill` 的绘制顺序（第一个形状的 `paint-order` 默认为 `normal`）：

```CSS
    }
}
```

效果非常明显，设置 `paint-order: stroke` 的“心形”的描边，就是一个外描边。除了使用 `paint-order` 属性来调整 `stroke` 和 `fill` 顺序之外，[我们还可以使用 SVG 的裁剪来绘制外描边](https://juejin.cn/book/7341630791099383835/section/7362748816769941540)。

  


首先给 SVG 形状绘制一个双倍宽度的中心描边，然后裁剪掉形状边界内的描边部分，或仅保留边界外的部分。不过，目前无法直接通过简单的裁剪来实现。因为你不能“反转”裁剪——即显示形状外部的部分，而不是内部。一种可能的方式是，在整个绘图边界周围绘制一个矩形，然后创建一个从矩形到原始裁剪路径的零宽度“桥”——这个新的裁剪路径将包含除了原始裁剪路径以外的所有内容。操作过程大致如下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6d609ccf889842fc9ca64d179ee3eb32~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=892&s=295406&e=jpg&b=050505)

  


这种裁剪方法有点类似于 [CSS 的 `clip-path` 的 `polygon()` 裁剪反转形状的方法](https://juejin.cn/book/7395119173656903715/section/7395172754053890063#heading-9)，但 [SVG 中的 `<clipPath>` 要这样使用](https://juejin.cn/book/7341630791099383835/section/7362748816769941540)，并不理想，尤其是根据形状来调整裁剪路径，更为复杂。

  


不过，[我们可以使用 SVG 的遮罩 `<mask>` 来实现](https://juejin.cn/book/7341630791099383835/section/7366549423813296165)。通过创建一个带有白色背景和黑色形状的遮罩，就可以裁剪掉双倍宽度描边中落在图像内部的部分，从而实现外描边效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0e917ea12558468890bc6d2b74c1f380~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3324&h=1106&s=473651&e=jpg&b=0c0c0c)

  


对应的 SVG 代码：

  


```XML

        <mask id="outerStroke">
            <rect x="0" y="0" width="100" height="100" fill="white" />
            <use href="#heart" fill="black" />
        </mask>
    </defs>

    <use href="#heart" stroke-width="10" fill="none" mask="url(#outerStroke)" />
</svg>
```

接下来，我们来看看如何创建内描边。首先明确一点，`paint-order` 属性只能控制两种描边效果，默认（`normal` ，相当于 `fill stroke`）是中间描边，`stroke` （相当于 `stroke fill`）是外描边，它没有相应的值可以将描边设置为内描边。尽管 SVG 规范提出 `stroke-alignment` 属性，用于控制描边位置，但到目前为止还没有浏览器支持。

  


也就是说，要给 SVG 形状设置内描边，我们只能通过些技巧来实现。你可以通过绘制一个双倍宽度的中心描边，然后裁剪掉形状边界之外的所有部分，或者仅显示形状内部的部分来实现内描边。[我们可以使用 SVG 的 `<clipPath>` 来实现这一点](https://juejin.cn/book/7341630791099383835/section/7362748816769941540)。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/19e525002a5e4a08855428b5435176d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3344&h=1056&s=506131&e=jpg&b=121212)

  


首先，我们定义一个 `<clipPath>` ，它包含一个形状——我们想要保留可见部分。接着，我们在图形元素的 `clip-path` 属性中引用这个裁剪路径，SVG 渲染器只会显示位于心形裁剪路径内的部分。这是对应的 SVG 代码：

  


```XML
<svg viewBox="0 0 100 100" class="heart">
    <defs>
        <path id="heart" d="m 31,11.375 c -14.986319,0 -25,12.30467 -25,26 0,12.8493 7.296975,23.9547 16.21875,32.7188 8.921775,8.764 19.568704,15.2612 26.875,19.0312 a 2.0002,2.0002 0 0 0 1.8125,0 c 7.306296,-3.77 17.953225,-10.2672 26.875,-19.0312 C 86.703025,61.3297 94,50.2243 94,37.375 c 0,-13.69533 -10.013684,-26 -25,-26 -8.834204,0 -14.702885,4.50444 -19,10.59375 C 45.702885,15.87944 39.834204,11.375 31,11.375 z"/>
        <clipPath id="insideStroke">
            <use href="#heart"/>
        </clipPath>
    </defs>

    <use href="#heart" stroke-width="10" stroke="red" fill="white" clip-path="url(#insideStroke)"/>
</svg>
```

## 小结

  


在这节课中，我们主要探讨了两种环境下创建内中外边框的技巧。

  


在 CSS 的环境中，使用 `border` 只能给元素框添加外边框，如果你需要创建内边框或中间边框，那么就需要通过一些 Hack 技巧来实现，比如创建内边框：

  


-   使用 `outline` 和 `outline-offset` 来创建内边框
-   使用 `box-shadow` 内阴影创建内边框
-   使用 `padding-box` 和 `content-box` 结合创建内边框
-   使用伪元素创建内边框

  


创建中间边框，需要其他一些组合：

  


-   `border` 和 `outline` 以及 `outline-offset` 组合
-   `box-shadow` 外阴影和内阴影组合
-   `border` 和 `box-shadow` 内阴影组合
-   `outline` 和 `box-shadow` 内阴影组合
-   `outline` 和 `outline-offset` 以及 `box-shadow` 外阴影组合
-   `border-box` 、`padding-box` 和 `content-box` 组合
-   伪元素叠加

  


对于 `text-stroke` 描边，默认是中间描边，但我们可以通过 `paint-order: stroke` 改变描边顺序，从而实现外边描。对于内描边，相对而言要复杂或者棘手一些，你可能需要使用伪元素叠加或利用裁剪或遮罩的方式来实现。

  


SVG 环境下则有所不同，我们通常使用 `stroke` 和 `stroke-width` 给 SVG 元素设置描边效果。默认情况下，它是中间描边。同样的，我们可以使用 `paint-order: stroke` 来实现外描边。另外，还可以通过 SVG 裁剪和遮罩来实现外描边。对于内描边，一般使用 SVG 裁剪来实现。

  


最后需要提醒大家一点是，这节课我们所探讨的都是创建内中外边框的常见方法，但绝不仅限于这些方法。因为，我个人认为这些方法相对而言要比较简单，并且实用。如果你有更好的方案，欢迎在下面的评论中与我们共享！
