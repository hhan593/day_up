![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5eec369f9fd844a6a1ad0979e24b4248~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1272&h=758&s=15277&e=avif&b=2b2e38)

  


你是否在开发的过程中碰到类似这样的需求。在某一时刻，需要知道一个元素在其兄弟元素中的位置，或者一个元素有多少个子元素，以便在 CSS 中计算某些东西，可能是为了实现某种错落的动画效果，其中每个元素延迟时间更长，或者可能是根据元素的兄弟元素的数量来改变元素的背景颜色。

  


类似这些需求，Web 开发者有许多常见的方法来为元素分配特定的索引或在设计中加入随机元素。Web 开发者通常通过 HTML （生成并作为内联样式应用）、CSS（使用预处理器，或者手动编写一大串 CSS 规则以有限的方式实现）或 JavaScript（通过 `MutationObserver` API 动态地为元素分配 CSS 属性）来完成这些操作。

  


今天，我想告诉大家的是，除了这些已知的方法之外，我们在不久的将来（或就是现在），可以在 CSS 中使用树计数函数 `sibling-count()` 和 `sibling-index()` 来获取父元素中的子元素总数和元素在兄弟元素中的位置。然后在 CSS 中直接做一些数学计算，实现想要的效果，比如之前提到的错落的动画效果、改变元素的位置和背景颜色等。

  


接下来，我们来看看它们能帮助我们做些什么事情？

  


## 树计数函数简介

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/279c183deb8643a7a6d355c37e18e26a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=46234&e=jpg&b=0e2434)

  


以上图圆形导航菜单为例，我们需要知道圆形导航菜单总共有多少个菜单项（元素总数）和单个菜单项在索引位置，才能通过 **[CSS 三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)** 计算出每个导航菜单项（单个元素）在圆形（或圆弧）上的位置。

  


这意味着，我们需要一组类似的函数，它们能返回元素的索引值和兄弟元素的数量，并且结果是整数。这对于 CSS 而言，并不是难事。我们现在可以通过 CSS 的结构伪类选择器（比如 `:nth-child()` 或 `:last-nth-child()`）来查询元素在 DOM 树中的位置，更不用说通过[ CSS 的 `:has()` 选择器查询元素的数量](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)。

  


庆幸的是，CSS 推出了 `sibling-count()` 和 `sibling-index()` 函数：

  


-   `sibling-count()` 函数返回元素的父元素中所有子元素的总数，返回值类型为 `<integer>`
-   `sibling-index()` 函数返回元素在其父元素的子元素中的索引。像 `:nth-child()` 一样，`sibling-index()` 索引从 `1` 开始计数。返回值类型为 `<integer>`

  


不幸的是，目前仅在 Chrome 浏览器能看到效果（写这节课的时候，使用的是 Chrome 140 版本）。即便是如此，也阻止不了我们一探其究竟！

然后根据 **[CSS 三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)** 计算每个菜单项在圆上的位置：

  


```CSS
nav:has(:checked) {
    li {
        translate: var(--x) var(--y);
    }
}

li {
    --position: calc((360 / var(--sibling-count)) * var(--sibling-index));
    --rotation: calc(var(--position) * 1deg);
    --x: calc(var(--distance) * cos(var(--rotation)));
    --y: calc(var(--distance) * sin(var(--rotation)));

    transition: all 0.2s;
}
```

虽然说，改进的方式精简了不少 CSS 代码，但试想一下，要是我们有 `100` 个 `li` 元素呢？仍然需要硬编码 `--sibling-index` 和 `--sibling-count` 属性一百次。庆幸的是，接下来的这种技巧将会以对数方式增加规则，具体来说是以 `2` 为底。所以，如果有一百个 `li` 元素的情况下，我们只需要编写大约十条 CSS 规则。

  


**特别声明，** **[这个技巧最早由 @Roman Komarov 提出，他在阐述原型化树计数函数时，详细地介绍了这个技巧的工作原理](https://kizu.dev/tree-counting-and-random/)**。这是一篇深度介绍树计数函数原型的教程，个人强烈建议你去阅读！如果你不想花太多时间了解其工作原理的话，可以继续往下。

  


这种技巧同样依赖于 CSS 自定义属性，但我们不再硬编码每个属性，而是使用两个自定义属性（比如 `--si1` 和 `--si2`），这两个自定义属性为每个元素构建 `--sibling-index` 属性。而且它们都从 `0` 开始：

  


```CSS
li {
    --si1: 0;
    --si2: 0;
}
```

  


真实的 `--sibling-index` 将通过这两个属性和一个因子（`F`）来构造，其中因子 `F` 代表一个大于或等于 `2` 的整数，我们可以根据公式 `sqrt(F) - 1` 选择多少个元素。例如：

  


-   当 `F = 2` 时，我们可以选择 `3` 个元素
-   当 `F = 3` 时，我们可以选择 `8` 个元素
-   当 `F = 5` 时，我们可以选择 `24` 个元素
-   当 `F = 10` 时，我们可以选择 `99` 个元素
-   当 `F = 25` 时，我们可以选择 `624` 个元素

  


正如你所看到的，增加因子值会指数性地提高我们可以选择的元素数量。那么这一切如何转换为 CSS 呢？

  


首先需要知道的是，计算 `--sibling-index` 属性值的公式是：

  


```CSS
calc(F * var(--si2) + var(--si1))
```

  


如果 `F = 3`，那么在 CSS 中它看起来像下面这样：

  


```CSS
li {
    --si1: 0;
    --si2: 0; 
    
    /* F = 3*/
    --sibling-index: calc(3 * var(---si2) + var(--si1));
}
```

  


接下来的选择器可能看起来是随机的，但请继续往下看。

  


对于 `--si` 属性，我们将写规则来选择因子倍数的元素，并将它们偏移 `1` ，直到达到 `F - 1` ，然后将 `--si1` 设置为偏移量。这将转化为以下 CSS：

  


```CSS
li {
    &:nth-child(Fn + 1) {
        --si1: 1;
    }
    
    &:nth-child(Fn + 2) {
        --si1: 2;
    }
    
    /* 依此类推 */
    &:nth-child(Fn + (F-1)) {
        --si1: (F - 1);
    }
} 
```

  


假设因子 `F` 等于 `3` ，我们将写出以下规则，直到达到 `F - 1` ，即 `2` 条规则：

  


```CSS
li {
    &:nth-child(3n + 1) {
        --si1: 1;
    }
    
    &:nth-child(3n + 2) {
        --si1: 2;
    }
}
```

  


对于 `--si2` 属性，我们将写规则来选择因子批量的元素（所以如果我们的因子 `F` 是 `3` ，我们每条规则选择 `3` 个元素），从最后一个可能的索引（在这种情况下为 `8` ）向后写，直到我们不能再选择更多元素为止。对应的 CSS 看起来要复杂一些：

  


```CSS
li:nth-child(n + F * 1):nth-child(-n + F * 1 - 1) {
    --si2: 1;
}

li:nth-child(n + F * 2):nth-child(-n + F * 2 - 1) {
    --si2: 1;
}

/* 依此类推 */
li:nth-child(n + (F * (F - 1))):nth-child(-n + (F * F - 1)) {
    --si2: (F - 1);
}
```

  


同样的，如果我们的因子 `F = 3` ，对应的 CSS 就会像下面这样：

  


```CSS
li {
    &:nth-child(n + 3):nth-child(-n + 5) {
        --si2: 1;
    }
    
    &:nth-child(n + 6):nth-child(-n + 8) {
        --si2: 2;
    }
}
```

  


就这样，通过设置 `--si1` 和 `--si2` 就可以计数最多 `8` 个元素。

  


最初看到这些计算时，你可能会感困惑，但一旦理解了其中的原理，所有一切就会变得非常清晰。[@Juan Diego 在 Codepen 提供了一个有关于这个技巧的演示案例](https://codepen.io/monknow/full/QwLMVom)，你可以看到如何通过这个公共式访问所有元素。你只需要将鼠标悬浮到代码片段上，就可以查看可以选择哪些元素，点击每个代码片段可以将它们组合成可能的索引：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f80b127506714d4ca44ba54fcf0b88f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1654&h=832&s=2550134&e=gif&f=267&b=151515)

接下来，我们新增另外两个变量 `--sc1` 和 `--sc2` ，并相同的公式来计算 `sibling-count` 的值：

  


```CSS
ul {
    --sc1: 0;
```

我们也可以通过 JavaScript 的 `MutationObserver` API 来实现：

  


```JavaScript
// 获取页面上的 .lists 元素
const elements = document.querySelector(".lists");

// 更新自定义 CSS 属性的函数
const updateCustomProperties = () => {
    let index = 1;

    // 遍历所有子元素，更新每个元素的 "--sibling-index" 属性
    Array.from(elements.children).forEach(element => {
        element.style.setProperty("--sibling-index", index);
        index++;
    });

    // 更新总元素数量
    elements.style.setProperty("--sibling-count", elements.children.length);
};

// 初始调用更新函数
updateCustomProperties();

// 使用 MutationObserver 监听子元素变化，并调用更新函数
const observer = new MutationObserver(updateCustomProperties);
const config = { attributes: false, childList: true, subtree: false };
observer.observe(elements, config);

// 获取用于控制元素数量的输入框和显示值的标签
const rangeInput = document.getElementById("elements");
const output = rangeInput.nextElementSibling;

// 创建新的 li 元素的函数
const createLiElement = () => {
    return document.createElement("li"); // 仅创建 li，不添加内容
};

// 监听滑动条输入事件，动态调整元素的数量
rangeInput.addEventListener("input", (event) => {
    const elementsNumber = elements.children.length;
    const targetElementsNumber = parseInt(event.target.value, 10); // 获取目标数量
    const difference = targetElementsNumber - elementsNumber; // 计算差值

    // 根据差值添加或删除元素
    if (difference !== 0) {
        // 使用文档碎片来减少多次的 DOM 操作
        const fragment = document.createDocumentFragment();
    
        if (difference > 0) {
            // 如果目标元素数量大于当前数量，添加新元素
            for (let i = 0; i < difference; i++) {
                fragment.appendChild(createLiElement());
            }
        } else {
            // 如果目标元素数量小于当前数量，移除多余元素
            for (let i = 0; i < Math.abs(difference); i++) {
                elements.removeChild(elements.lastElementChild); // 删除最后一个元素
            }
        }
    
        // 将新元素批量添加到 DOM 中
        elements.appendChild(fragment);
    }

    // 更新标签显示值
    output.textContent = targetElementsNumber;
});
```

## 案例一：错落动画

  


首先，我们来看一个简单的用例，使用 `sibling-index()` 和 `sibling-count()` 创建一个简单的错落动效。HTML 结构变得很简单：

  


```HTML
<div class="grid">
    <div>☰</div>
    <div>👍</div>
    <div>👎</div>
    <div>🎸</div>
    <div>🍽️</div>
    <div>🍈</div>
    <div>🥓</div>
    <div>🪜</div>
    <div>🎥</div>
    <div>🐟</div>
</div>
```

  


我们通过下面几行 CSS 代码，就可以给导航菜单切换时添加错落的动画效果：

  


```CSS
.grid {
    --stagger: calc((sibling-index() - 1) * 0.1s);
    --duation: calc((sibling-count() - 1) * 0.1s);
    --ease-spring: linear(
      0,
      0.009,
      0.035 2.1%,
      0.141 4.4%,
      0.723 12.9%,
      0.938 16.7%,
      1.017,
      1.077,
      1.121,
      1.149 24.3%,
      1.159,
      1.163,
      1.161,
      1.154 29.9%,
      1.129 32.8%,
      1.051 39.6%,
      1.017 43.1%,
      0.991,
      0.977 51%,
      0.974 53.8%,
      0.975 57.1%,
      0.997 69.8%,
      1.003 76.9%,
      1
    );
    --ease: cubic-bezier(0.25, 0, 0.3, 1);
    --hue: calc(100 + sibling-index() * 50);

    > div:not(:first-child) {
        background: oklch(70% 70% var(--hue));
        opacity: 0;
        translate: -150% 0;
        transition: 
            opacity var(--duration) var(--ease) var(--stagger),
            translate var(--duration) var(--ease-spring) var(--stagger), 
            background 0.15s;

        &:hover {
            background: white;
        }

        @starting-style {
            opacity: 0;
            translate: -150% 0;
        }
    }
    
    &.open > div {
        opacity: 1;
        translate: 0 0;
    }
}
```

  


除了第一个 `div` 之外的其他 `div` ，它的初始状态是向左平移了 `150%` （即 `translate: -150% 0`），并且透明度 `opacity` 为 `0` 。当我们点击第一个 `div` 时，它的父元素（`.grid`）会添加一个`.open` 类名，此时将其他 `div` 的 `translate` 设置为 `0` ，并且 `opacity` 设置为 `1` 。

  


在这个示例中，我们是通过 CSS 的 `transition` 给导航项创建过渡动效，为了使整个过渡动效有一个错落的感觉，每个导航项将会根据它自身在 DOM 中的顺序来调整 `transition-delay` 的时间。而这个时间是由 `sibling-index()` 来计算的：

  


```CSS
.grid > div {
    --stagger: calc((sibling-index() - 1) * 0.1s);
}
```

  


整个过渡动画所持续的时间（`transition-duation`）是通过 `sibling-count()` 来计算：

  


```CSS
.grid > div {
    --duation: calc((sibling-count() - 1) * 0.1s);
}
```

  


而且我们还为每个导航项设置了不同的背景颜色，而背景颜色也是由 `sibling-index()` 来控制：

  


```CSS
.grid > div {
    --hue: calc(100 + sibling-index() * 50);
}
```

## 小结

  


`sibling-count()` 和 `sibling-index()` 允许我们根据元素所在 DOM 的树型结构来计算元素的索引值和总数量。这在一些需要计算位置、颜色、角度和时间等场景中特别有用，尤其是在创建错落动效的情景中。但它们离真正能使用还需要一段时间，这个时间可能很快，也可能很慢。即便是如此，在 CSS 中我们还有多种其他的方式可以实现类似于 `sibling-count()` 和 `sibling-index()` 函数的功能，只是使 HTML 和 CSS 的代码变得更为冗长。

  


最后，让我们一起期待它们能早点得到主流浏览器的支持。
