## CSS 内在尺寸

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5c573ad7158841199447dca046a873d2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=743&h=305&s=24966&e=webp&b=23212d)

  


既然 `calc-size()` 函数与 **[内在尺寸](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)** 有关联，那么就先来简单地了解一下内在尺寸相关的知识。

  


在 CSS 中，给一个元素设置大小时，主要有两种方式：

  


-   根据元素框内在的内容来决定，比如 `auto` 、`min-content` 、`max-content` 和 `fit-content` 等
-   根据元素框上下文来决定，比如 `<length-percentage>`

  


这两种方式分别就是 CSS 中的“内在尺寸”和“外在尺寸”：

  


-   **内在尺寸** ：元素根据自身的内容（包括其后代元素）决定大小，而不需要考虑其上下文，其中 `min-content`、`max-content` 和 `fit-content` 能根据元素内容来决定元素大小，因此它们统称为**内在尺寸**
-   **外在尺寸** ：元素不会考虑自身的内容，而是根据上下文来决定大小，最为典型的案例，例如 `width: 50%`

  


下面这个示例展示了内在尺寸与外在尺寸之间的差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4471802c6e0546c1843bd68ea2c8f54e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1132&h=892&s=9225882&e=gif&f=154&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/qBKGrEK

  


到目前为止，我们可以通过尺寸关键词 `auto` 、`min-content` 、`max-content` 和 `fit-content` 使元素根据内在的内容来决定元素框的大小，例如：

  


```CSS
h1[data-width="auto"] {
    width: auto;
}

h1[data-width="min-content"] {
    width: min-content;
}

h1[data-width="max-content"] {
    width: max-content;
}

h1[data-width="fit-content"] {
    width: fit-content;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/76d168bbacb648ef95a88c5a627cd295~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1470&h=650&s=1330447&e=gif&f=182&b=f5f4f4)

  


> Demo 地址：https://codepen.io/airen/full/mdKYWjy

  


上面这些关键词可以应用于元素尺寸相关的属性，比如 `width` （或 `inline-size`）、`height` （或 `block-size`）以及 `min-width/height/inline-size/block-size` 和 `max-width/height/inline-size/block-size` 等。[对于 `flex-basis` 属性，还可以设置为 `content` ](https://juejin.cn/book/7161370789680250917/section/7161623717074698247#heading-2)。未来可能还支持其他的尺寸关键词，比如 `stretch` 和 `contain` 。

  


> 温馨提示，有关于这方面更详细的介绍，请移步阅读《[内在 Web 设计](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)》！

  


虽然内在尺寸能够灵活地将传统的，久经考验的 Web 布局技术和现代布局方法和工具结合起来，以便根据 Web 的内在内容创造独特的布局。换句话说，内在 Web 设计不是内容以设计为导向，而是只专注于让设计受内容驱动！

  


但是，内在尺寸也有一定的缺陷性，比如，它们不能直拉参与 CSS 的 `calc()` 函数（CSS 中最早可用于运算的数学函数）的运算。而这正是为什么引入新的 `calc-size()` 函数——它允许对内在尺寸进行计算！

  


另一个显著的缺点就是，内在尺寸被应用于过渡动效时，不会有平滑过渡的效果。这是因为元素具体的值（比如 `0px` ，它是 `<length-percentage>`）动画化到内在尺寸的值（比如 `auto` ），不能直接参与插值计算，因此不具有平滑的过渡效果。这也是为什么 CSS 要引入 `calc-size()` 函数的另一个原因。与此同时，CSS 还引入了 `interpolate-size` 属性，它同样允许元素的具体值平滑过渡到内在尺寸。

  


我们来看一个简单的示例：

  


```CSS
a {
    width: 4rem;
    transition: width 0.35s ease;
      
    &:is(:hover, :focus) {
        width: auto;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d58f202983e4b9d99cd2c7724db9496~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1342&h=770&s=361957&e=gif&f=135&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/NPWJrNJ

  


示例中的 `width: 4rem` 动画化到 `width: auto` 时，过渡效果非常突兀！也就是说，`transition` 动画没有发生，因为浏览器不支持过渡到像 `auto` 或 `min-content` 这样的内在尺寸关键字。庆幸的是，CSS 新引入的 `calc-size()` 函数和 `interpolate-size` 属性将允许你绕过这个限制，并以安全、明确定义的方式对内在尺寸进行数学运算。

  


也就是说，**使用** **`interpolate-size`** **属性或** **`calc-size()`** **函数，使过渡和动画能够在长度值和内在尺寸关键字之间平滑过渡**。

  


## `calc-size()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a4d3711bca90434f93a6bfad2e914da2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=1024&s=2131996&e=png&b=0f1d39)

  


在 CSS 中，我们通常会使用 `calc()` 函数对属性值进行计算，其中一个最大的特点是可以混合不同的数据类型，例如像素和百分比，比如 `width: calc(100% - 10px)` 会给出你想要的值，即容器宽度的 `100%` ，再减去 `10px` 。

  


`calc()` 函数的缺点是它不支持对内在尺寸关键字进行计算，包括 `auto` ，而这正是为什么引入了新的 `calc-size()` 函数——**它允许对内在尺寸关键字进行计算，从而实现到内在尺寸的过渡和动画**。

  


CSS 的 `calc-size()` 函数类似于 `calc()` 函数，但它还支持对以下关键字之一进行操作：`auto` 、`min-content` 、`max-content` 、`fit-content` 、`stretch` 或 `contain` ，这些都是内在尺寸关键字。这使得这些值（或这些值的数学函数）进行过渡和动画成为可能，只要在过渡或动画的任意端点使用 `calc-size()` 函数。

  


`calc-size()` 的语法如下：

  


```CSS
.element {
    width: calc-sie(<calc-size-basis>, <calc-sum>)
}
```

  


该函数接受两个参数，顺序如下：

  


-   `<calc-size-basis>` ：需要在计算前进行插值的固有尺寸值，可以是一个 `<intrinnsic-size-keyword>` （即 `auto`、`min-content`、`max-content` 或 `fit-content` ），也可以是一个嵌套的 `calc-size()` 。该值最终会解析为一个 `<length>` ，它是一个 `calc-size` 的基础
-   `<calc-sum>` ：包含 `<calc-size-basis>` 的表达式，即一个 `calc-size` 的计算，允许你使用 `calc-size` 基础进行计算。要引用 `calc-size` 基础，得使用关键词 `size`

  


以下是一些示例：

  


```CSS
/* 仅插值 */
width: calc-size(auto, size);        /* 👉 原始的 auto 宽度，未修改 */
width: calc-size(min-content, size); /* 👉 原始的 min-content 宽度，未修改 */
width: calc-size(max-content, size); /* 👉 原始的 max-content 宽度，未修改 */
width: calc-size(fit-content, size); /* 👉 原始的 fit-content 宽度，未修改 */
```

  


上面这种用法，`calc-size()` 函数只是仅仅是使内在尺寸可以做插值计算，并未改变元素的尺寸。从视觉效果上来看，结果与使用 `interpolate-size` 时完全相同。所以在这种特定情况下，你应该使用 `interpolate-size` （稍后会介绍）。

  


现在，我们可以像下面这样来改造之前的示例：

  


```CSS
a {
    width: 4rem;
    transition: width 0.35s ease;
    
    &:is(:hover, :focus) {
        width: calc-size(auto,size);
    }
}
```

  


现在导航菜单项有一个平滑的过渡动效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a14adb86b6c244af83592f960c7c50bb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1342&h=770&s=926045&e=gif&f=224&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ogNONGe

  


`calc-size()` 发挥作用的地方在于它能够进行计算，而这是 `interpolate-size` 无法做到的：

  


```CSS
/* 插值和计算 */
width: calc-size(auto, size * 2);            /* 👉 原始的 auto 宽度的两倍 */
width: calc-size(min-content, size / 2);     /* 👉 原始的 min-content 宽度的一半 */
width: calc-size(max-content, size + 100px); /* 👉 原始的 max-content 宽度的基础上加上 100px */
width: calc-size(fit-content, size - 100px); /* 👉 原始的 fit-content 宽度的基础上减去 100px */
```

  


需要注意的是，在 `calc-size()` 函数的计算中使用百分比（`%`）时，可能会产生意外的结果，因为它们的计算值取决于元素的上下文。例如：

  


```CSS
.element {
    width: calc-size(auto, size + 50%);
}
```

  


元素的宽度并不是 `150%` 。相反，它将是 `size` 加上元素包含块宽度（即元素父容器）的 `50%` ，因为 `width` 的百分比值是相对于其父容器宽度计算。如果你想得到 `150%` 的大小，你应该乘以或除以一个无单位的值，例如：

  


```CSS
.element {
    width: calc-size(auto, size * 1.5);
}
```

  


`calc-size()` 函数的第二个参数（表达式）甚至可以包含其他 CSS 函数，如 `min()`、`calc()`，以及 CSS 变量等：

  


```CSS
/* 计算与附加函数 */
width: calc-size(auto, min(200px, size * 2));
/* 计算与 CSS 变量 */
width: calc-size(auto, size * var(--multiplier));
/* 计算与附加函数及 CSS 变量 */
width: calc-size(auto, min(300px, size * var(--multiplier)));
```

  


例如，如果你希望页面上的所有段落的宽度调整为最接近 `50px` 的倍数，你可以使用以下代码：

  


```CSS
p {
    width: calc-size(fit-content, round(up, size, 50px));
    height: calc-size(auto, round(up, size, 50px));
}
```

  


注意，`round()` 函数是一个四舍五入的函数，上面代码中的 `round()` 函数会向上取整，有关于这方面更详细的介绍，请移步阅读《[Web UI：CSS 中的数学函数与常量](https://juejin.cn/book/7395119173656903715/section/7446278438929104922#heading-0)》！

  


`calc-size()` 还允许你在两个 `calc-size()` 之间进行插值，当它们的 `calc-size` 基础相同时。这也是 `interpolate-size` 无法实现的功能。

  


```CSS
.element {
    width: min-content; 
    transition: width 0.35s ease;
    &:hover {
        width: calc-size(min-content, size + 10px); 
    }
}
```

  


你可能会问？为什么不在 `calc()` 函数中直接支持 `<intrinsic-size-keyword>` 呢？原因之一是，进行计算时不允许混合和匹配内在尺寸关键字。例如，你可能会试图使用 `calc(max-content - min-content)` ，看起来有效，但实际并不行。`calc-size()` 强制保证正确性，因为它不同于 `calc()` ，它的第一个参数只能接受一个单一的 `<intrinsic-size-keyword>` 。

  


另一个原因是上下文感知。一些布局算法对特定的内在尺寸关键字有特殊行为。`calc-size()` 被明确地定义为表示内在尺寸，而不是 `<length>` 。因此，这些算法能够将 `calc-size(<intrinsic-size-keyword>, ...)` 视为 `<intrinsic-size-keyword>` ，并保持该关键字的特殊行为。

  


## 使用 `interpolate-size` 在内在尺寸关键字之间进行动画处理

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/48ed4de340d442a08dde3006a9ce3819~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=1024&s=1742231&e=png&b=236abc)

  


`calc-size()` 函数解决了我们在内在尺寸（比如 `auto`、`min-content` 、`max-content` 和 `fit-content` 等）上的过渡问题，但除非你正在进行实际计算，否则它仍然感觉像是一个临时解决方案。

  


正因如此，CSS 引入了 `interpolate-size` 属性，它让你选择插值行为，并决定是否让浏览器对内在尺寸进行插值计算。

  


`interpolate-size` 属性的默认值是 `numeric-only` ，这就是我们熟悉的行为，只对数值（比如 `100px`）的过渡和动画有效；另一个值是 `allow-keywords` ，它允许你在关键词之间进行插值计算。也就是说，为了启用这种新的动画行为，你可以在自己的 CSS Reset 中添加下面这段 CSS：

  


```CSS
:root {
    @supports (interpolate-size: allow-keywords) {
        interpolate-size: allow-keywords;
    }
}
```

  


[使用 `@supports` 对 `interpolate-size` 进行检测](https://juejin.cn/book/7223230325122400288/section/7258822669154877480)，支持 `interpolate-size` 属性的浏览器会在内在尺寸关键字和和度或百分比之间进行过渡和动画。

  


当然，你也可以将其限制在你想要的元素上，但 W3C 规范也是建议你为整个页面启用这种新行为：

  


> 在根元素上指定 `interpolate-size: allow-keywords` 会为整个页面选择新的行为。

  


一旦我们在页面中设置了 `interpolate-size` 为 `allow-keywords` ，我们就可以去掉 `calc-size()` 函数，仅使用 `auto` 值（也可以是其他内在尺寸，比如 `min-content` 、`max-content` 、`fit-content` 等）。现在，我们通过 `interpolate-size` 属性来改造之前的案例：

  


```CSS
:root {
    @supports (interpolate-size: allow-keywords) {
        interpolate-size: allow-keywords;
    }
}

a {
    width: 4rem;
    transition: width 0.35s ease;
    
    &:is(:hover, :focus) {
        width: auto;
    }
}
```

  


这个时候，链接 `a` 元素宽度（`width`）从 `4rem` 过渡到 `auto` 关键字时会有一个平滑的过渡的动画效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e313421a29aa40f7a6579fc4a465ba60~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1342&h=770&s=926045&e=gif&f=224&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ByaEwrd

  


## 使用哪种方法？

  


正如你所看到的，我们有两种方式可以使内在尺寸关键字在过渡和动画中进行插值计算：

  


-   在根元素或指定元素上设置 `interpolate-size` 属性值为 `allow-keywords`
-   使用 `calc-size()`

  


那么我们应该如何选择呢？

  


在大多数情况下，建议在 `:root` 上声明 `interpolate-size: allow-keywords` 。这是启用从内在尺寸关键字到长度的过渡和动画的最简单方法：

  


```CSS
:root {
    @supports (interpolate-size: allow-keywords) {
        interpolate-size: allow-keywords;
    }
}
```

  


这段代码是一个很好的渐进增强，因为不支持它的浏览器将回退到不使用过渡或动画。

  


`interpolte-size` 是一个非常快速的解决方案，但你可能会遇到一些更精细化的计算，比如它要比内在尺寸稍大或稍小的情况。那么使用 `calc-size()` 就更适合一些，比如：

  


```CSS
a {
    width: 4rem;
    transition: width 0.35s ease;
    
    &:is(:hover, :focus) {
        width: calc-size(auto, size + 1rem);
    }
}
```

  


在这个示例中，我传递了我要进行计算的内在尺寸关键字（`auto`），然后在逗号后，我指定将计算出来的大小加上 `1rem` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d82f61c4cb974b18a47b68a93a17b847~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1344&h=770&s=1731020&e=gif&f=217&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/RNwOLvp

  


仔细观察上面示例的效果，第一个和第二个都是从 `4rem` 过渡到 `auto` 关键字，第三个是从 `4rem` 过渡到 `auto + 1rem` 。

  


需要知道是，当你使用 `calc-size()` 函数是，不论是否声明了 `interpolate-size: allow-keywords` ，从固有尺寸到内在尺寸之间过渡或动画都可以进行插值计算，即它们之间的过渡或动画有平滑的效果。

  


不过，如果不需要对元素进行计算，个人建议还是使用 `interpolate-size: allow-keywords` 来实现这个功能。使用 `interpolate-size` 可以更简单地“启用”全站范围内的内在尺寸关键字动画和过渡，而当你需要根据关键字值进行实际计算时，再依赖 `calc-size()`。

  


另外，值得注意的是，正如 `interpolate-size: allow-keywords` 不支持关键字值之间的过渡或动画，`calc-size()` 也是如此。

  


## 案例：手风琴折叠和展开添加过渡动效

  


有了这两个 CSS 特性之后，我们在 CSS 中创建 `width: auto` 或 `height: auto` 过渡或动画效果将不再是难事。或者说，我们可以将它们应用诸如手风琴、下拉菜单等 Web 组件。比如，我们之前《[CSS 布局：创建手风琴组件](https://juejin.cn/book/7395119173656903715/section/7444808804371267584#heading-4)》中所演示的案例，只需要下面几行 CSS 代码，就使手风琴每个项目在折叠和展开之间具有平滑的动画效果：

  


```CSS
:root {
    interpolate-size: allow-keywords;
}

::details-content {
    transition: height 0.5s ease, content-visibility 0.5s ease allow-discrete;
    height: 0;
    overflow: clip;
}

[open]::details-content {
    height: calc-size(auto, size); /* 或 auto */
}
```

## 小结

  


现在，我们可以**使用** **`interpolate-size`** **属性或** **`calc-size()`** **函数，使过渡和动画能够在长度值和内在尺寸关键字之间平滑过渡**。但是，使元素从固定的长度值到内在尺寸要具有更平滑的过渡或动画效果，还有其他的一些细节，这些我们将放到后面的课程中来阐述。
