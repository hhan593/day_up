## 从最简单的提示框开始

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b66a81f01aeb4e4293898fe8a2571b04~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1353&s=313842&e=jpg&b=8d4bef)

  


类似上图这种提示框（Tooltip）算是 Web 上最简单、最常见的一种了。我们就从这类提示框开始，然后逐步过渡到其他更复杂的提示框。

  


首先要强调的是，我们在这节课重点是关注提示框形状的制作，尤其是提示框的尾巴。接下来我们尽可能的使用一个元素来完成提示框的制作。例如：

  


```HTML
<div class="tooltip">CSS Tooltip</div>
```

  


### 设置提示框的基础样式

  


首先，我们使用 CSS 给提示框添加一些基础样式，比如设置它的大小，背景颜色、文本颜色和圆角等。设置提示框的尺寸应该是最简单的了，在这里我将采用 [CSS 的内在尺寸](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)和[宽高比 `aspect-ratio`](https://juejin.cn/book/7223230325122400288/section/7259316040151236666)来设置元素的尺寸。例如：

  


```CSS
.tooltip {
    width: fit-contnet;
    aspect-ratio: 21 / 9; 
    padding: 1em;
    
    display: grid;
    place-content: center;
    text-align: center;
    font-size: clamp(1rem, 0.6rem + 2vw, 3rem);
}
```

  


你也可以根据自己的需要设置 `.tooltip` 的 `width` （或 `height`）和宽高比 `aspect-ratio` 。在这个过程中尽可能的使用 [CSS 的相对单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700)，例如字体相对单位 `em` 、视窗相对单位 `vw` 、容器相对单位 `cqi` 等，这将使你的提示框更具弹性，且不易于造成内容溢出等现象。

  


接着给提示框添加背景颜色。我将使用 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733) `--color` 和 `--bg` 来定义提示框的前景色（例如文本颜色）和背景颜色，而且 `--color` 的值是通过 [`color-contrast()` 函数](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)，从一个颜色列表中找出一个最适合的颜色值，这个颜色值与 `--bg` 对比，至少能达到 `AA` 级别的颜色对比度。这样做可以确保，文本颜色在任何背景颜色之下都易于阅读：

  


```CSS
.tooltip {
    --primary: #ff5722;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196F3, #9C27B0, #FF9800, #9E9E9E, #607D8B, #fff, #bbb;
    --color: color-contrast(var(--bg) vs var(--color-set) to var(--contrast-target));
    
    background: var(--bg);
    color: var(--color);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d165983094664f738dd6ec440f4d2b2c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1076&h=634&s=10210695&e=gif&f=369&b=000000)

  


### 创建提示框的小尾巴

  


提示框的小尾巴（Tooltip Tail）指的是提示框（Tooltip）组件中突出的部分，通常指向触发提示框的元素，以视觉上表明提示框与该元素的关联。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4b7b8c0c081d4420954bf1f5bb158a67~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1353&s=374821&e=jpg&b=8d4bef)

  


它有可能位于提示框组件任何一侧。它的形状也是多样式的，通常是一个三角形或其他复杂的形状。在这里，我们从最简单的三角形开始。

  


在 CSS 中，给提示框添加三角形尾巴有很多方法，你可能最熟悉，最常用的方式之一就是通过 CSS 的伪元素 `::before` 或 `::after` 来创建三角形，并将其定位到切确的位置。但在这里，我将向大家演示一些不太熟悉的，而且更现代的方法。

  


首先，我定义了两个 CSS 自定义属性 `--tail-w` 和 `--tail-h` 来设置提示框尾巴的大小：

  


```CSS
.tooltip {
    /* 基础样式 */
    
    /* 提示框尾巴尺寸 */
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
}
```

  


然后使用现代 CSS 中的 `border-image` 属性创建一个“溢出的颜色”：

  


```CSS
.tooltip {
    /* 基础样式 */
    
    /* 提示框尾巴尺寸 */
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    border-image: conic-gradient(lime 0 0) 0 fill // var(--tail-h); 
}
```

  


你可能已经注意到了 `border-image` 属性值中包含了一个双斜杠（`//`），甚至对此感到困惑。我把 `border-image` 属性拆分一下，你的困惑就能立即烟消云散：

  


```CSS
.tooltip {
    border-image: conic-gradient(lime 0 0) 0 fill // var(--tail-h); 
    
    /* 等同于 */
    border-image-source: conic-gradient(lime 0deg, lime 0deg);
    border-image-slice: 0 fill;
    border-image-width: 1;
    border-image-outset: var(--tail-h);
    border-image-repeat: stretch;
}
```

  


`border-image` 是一个简写属性，简写时的语法规则如下：

  


```CSS
border-image: 
    <border-imagesource>   /* 定义用于边框的图像,可以是任意<image> 值类型 ，初始值是 none */
    <border-image-slice>   /* 定义边框图像的哪一部分将用于边框，初始值是 100% */
    /
    <border-image-width>   /* 定义边框的宽度，初始值是 1*/
    /
    <border-image-outset>  /* 定义边框图像超出元素边框盒的距离，初始值是 0 */
    <border-image-repeat>  /* 定义如何重复图像以填充边框, 初始值是 stretch */
```

  


注意，在 `<border-image-slice>` 和 `<border-image-width>` 后面分别跟了一个反斜杠 `/` 。到目前为止，`border-image` 可能是唯一一个可以使用双斜杠的属性。

  


在我们这个示例中，只用到了 `<border-image-source>` 、`<border-image-slice>` 和 `<border-image-outset>`。其中 `<border-image-source>` 由 `conic-gradient` 构成，而 `<border-image-width>` 和 `<border-image-outset>` 的工作方式与设置 `border-width` 、`padding` 和 `margin` 相同。它们可以接受 `1 ~ 4` 个值，具体取决于你是否希望单独为元素的边缘设置特定的 `<border-image-width>` 和 `<border-image-outset>` 值。有时，我们只需要指定 `<border-image-outset>` ，此时 `<border-image-width>` 就不需要。因此，你可能会看到我这样的简写：

  


```CSS
.element {
    border-image: <border-image-source> <border-imageslice>//<border-image-outset>;
}
```

  


即：

  


```CSS
.tooltip {
    border-image: conic-gradient(lime 0 0) 0 fill // var(--tail-h); 
}
```

  


另外，`<border-image-slice>` 也可以接受 `1 ~ 4` 个值（可以是无单位值或百分比），还可以包含一个可选的 `fill` 关键词。这使得 `border-image` 属性难以理解，因为切片的概念不容易直观地呈现。

  


> 有关于 `border-image` 属性在这里就解释到这里，不加以展开讨论。不过请放心，小册后面会专门花一节课的篇幅来介绍 `border-image` 属性，进一步被充《[现代 CSS](https://s.juejin.cn/ds/ih69x2nG/)》中所涵盖的知识点。

  


那么，请把注意力回到提示框尾巴制作上来。刚才提到过，`border-image` 属性创建了一个“溢出的颜色”，我用下图来解释，这样更易于理解一些：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f07601d5c6b84937b34fb1936fa82769~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1194&h=692&s=3464847&e=gif&f=233&b=000000)

当然，你也可以用更直观的方式来实现完全相同的效果，例如定义背景色，然后[使用 `border`](https://juejin.cn/book/7199571709102391328/section/7199845356845629472) （或 `padding`）来为提示框的尾巴创建空间：

  


```CSS
.tooltip {
    background: var(--bg);
    border-bottom: var(--tail-h) solid #0000; /* 根据尾巴的位置调整 border */
}
```

  


你甚至[还可以使用 `box-shadow`](https://juejin.cn/book/7199571709102391328/section/7199844993455325216) 或 `outline` 来设置外部颜色：

  


```CSS
.tooltip {
    background: var(--bg);
    box-shadow: 0 0 0 var(--tail-h) var(--bg);
}
```

  


虽然这些方法简单实用，但相比单一的 `border-image` ，它们需要额外的声明。而且，我们稍后会看到，`border-image` 对于实现更复杂的形状非常有用。

  


接下来，是关键所在，我们将使用 `clip-path` 的 `polygon()` 函数裁剪出一个提示框的形状：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a879e383a1a4ac59121e688fa24bc65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1968&s=683080&e=jpg&b=ffffff)

### 调整提示框尾巴位置

  


不难发现，上面示例展示的提示框，不管它的尾巴位于元素的哪一侧，它们都有一个共同的特点，位置在该侧居中。但是，实际开发过程中，尾巴的位置不一定要居中。不过，要调整提示框尾巴的位置，并不难。我们只需要在上面的基础上新增一个 CSS 变量，例如 `--tail-pos` ，使用它来控制提示框尾巴的位置。

  


以尾巴在提示框底部为例，我们在 `clip-path` 的 `polygon()` 中使用了 `50%` ，这会将尾巴定位在提示框底部的水平中心。如果我们将这个值赋值给 `--tail-pos` ，就可以通过调整 `--tail-pos` 的值来调整尾巴在提示框底部的水平方向的位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd12ce6a992d4090899e5fe4121c0fec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1968&s=710829&e=jpg&b=ffffff)

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    border-image: conic-gradient(var(--bg) 0 0) 0 fill // var(--tail-h); 
    
    --shape: 0 0, 100% 0, 100% 100%, calc(var(--tail-pos) + var(--tail-w) / 2) 100%, var(--tail-pos) calc(100% + var(--tail-h)), calc(var(--tail-pos) - var(--tail-w) / 2) 100%, 0 100%;
    clip-path: polygon(var(--shape));
}
```

  


`--tail-pos` 的值范围可以从 `0%` 到 `100%` ，其中 `0%` 位于最左侧，`100%` 位于最右侧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a0ae5dcedbbd4002a6626a53de1b37a3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1210&h=524&s=826604&e=gif&f=128&b=010101)

  


看上去似乎不错，但它存在一个小问题，当 `--tail-pos` 的值是一个极端值（例如 `0%` 或 `100%` ）时，提示框的尾巴会滑出提示框矩形左侧或右侧边缘。

  


仔细观察，当 `--tail-pos` 的值为 `0%` 时，尾巴向左侧多移出了一半的宽度（即 `--tail-w / 2`）；当 `--tail-pos` 的值为 `100%` 时，尾巴向右侧同样移出了一半的宽度。这意味着，我们需要调整 `polygon()` 中的 `P4` 和 `P6` 两个点的坐标：

  


```
P4 = calc(var(--tail-pos) + var(--tail-w) / 2) 100%
P6 = calc(var(--tail-pos) - var(--tail-w) / 2) 100%
```

  


其中 `P4` 点的 `x` 轴坐标最大不能超过 `100%` ，这意味着它要被限制为 `100%` ，避免尾巴在提示框的右侧溢出；而 `P6` 点的 `x` 轴坐标最小不能小于 `0%` ，它要被限制为 `0%` ，避免尾巴在提示框的左侧溢出。在 CSS 中，我们可以[使用 CSS 的比较函数 `min()` 和 `max()` 来设定范围限制](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)：

  


```
P4 = min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%
P6 = max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%
```

  


我们在 `polygon()` 中更新 `P4` 和 `P6` 两个点的坐标：

  


```CSS
.tooltip {
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    border-image: conic-gradient(var(--bg) 0 0) 0 fill // var(--tail-h); 
    
    --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, var(--tail-pos) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
    clip-path: polygon(var(--shape));
}
```

  


这样，我们就修复了边缘情况，现在尾巴不会溢出提示框的两侧，而是呈现出不同的形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4717e72a590848e6a1aadb488483b482~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1178&h=666&s=2799656&e=gif&f=330&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/oNrMdrp

  


很完美吧！我们可以按照同样的方式来调整其他方向的尾巴位置：

  


```CSS
.tooltip {

    --tail-h: round(1em, 1px); /* 高度*/

    }
    
    &.bottom {
        --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, var(--tail-pos) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd47fac298b44e299ea2544dd80e92ab~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2228&h=1056&s=558709&e=jpg&b=f1f1f1)

  


我们以三角形尾巴位于提示框的左下角顶点处为例，仔细观察，使用 `clip-path` 的 `polygon()` 只需要六个点的坐标就可以实现。与之前思路一样，在定义 `polygon()` 中的坐标点时，我引入了两个 CSS 自定义属性，它们分别是 `--s` 和 `--b` ，其各自代表的意思如下图所示：

```CSS
.tooltip {

```

### 调整提示框尾巴形状

  


首先，需要明确一点，这里所说的“调整提示框尾巴形状”指的是调整三角形的形状。如果你仔细观察，之前应用于提示框的三角形尾巴，都是一个等腰三角形。其实，这个三角形尾巴可以是任意你喜欢的三角形。

  


简单地说，你只需要调整 `polygon()` 函数中的 `P4` 、`P5` 和 `P6` 三个点的坐标，就可以获得任意形状的三角形。因为，在 CSS 中，`clip-path` 的 `polygon()` 函数，只需要知道三个点，就可以[创建出三角形](https://juejin.cn/book/7395119173656903715/section/7397319555476881460)。当然，要是你熟悉一些几何知识，[你还可以利用三角函数创建三角形](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)。不过，我们这里不会涉及到这么复杂的三角形，在我们这个示例中，只需要调整 `P5` 点的 `x` 轴坐标，就可以创建出各种不同形状的三角形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/53b4ae82d78043fca2838f9df2ce862d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1264&h=824&s=2132542&e=gif&f=184&b=f8fbf5)

  


> URL：https://bennettfeely.com/clippy/

  


我们只需要在原来的基础上新增一个 CSS 变量，比如 `--posX` ，然后通过它来调整 `P5` 坐标点：

  


```
/* 调整前 */
P5 = var(--tail-pos) calc(100% + var(--tail-h))

/* 调整后 */
P5 = calc(var(--tail-pos) + var(--posX))
```

  


这样，你就可以使用它来调整尾巴的形状：

  


```CSS
.tooltip {
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    --posX: -1em;    /* 调整 P5 点 x 轴坐标 */
    
    border-image: conic-gradient(var(--bg) 0 0) 0 fill // var(--tail-h); 
    
    --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, calc(var(--tail-pos) + var(--posX)) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
    clip-path: polygon(var(--shape));
}
```

  


注意，`--posX` 的值可以是正数或负数，可以使用任何你想要的单位。你可以尝试着调整 `--posX` 的值来查看提示框的三角形尾巴的变化：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c1abfe81735d45e583d9cbd321ed11a8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=704&s=3493464&e=gif&f=492&b=010101)

  


你会发现，当 `--posX` 为负值时，提示框的尾巴会指向左侧，反之则指向右侧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/075c114b13274294b06b9491901a4e97~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=608&s=440327&e=jpg&b=24fe91)

  


不知道你有没有注意到，当尾巴越靠近提示框左侧边缘或右侧边缘时，三角形尾巴会溢出，造成三角形被裁：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd4e829897634b57b3bbff15cf3427d4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=608&s=393636&e=jpg&b=24fe91)

  


我们可以将 `border-image-outset` 属性设置一个极大值，例如 `100vw` 来避免这个现象。这是因为，尾巴的形状可以是任何类型的三角形，并且可以占据更大的面积。由于我们无法知道准确的 `border-image-outset` 的值，所以我们使用这个极大值 `100vw` 来确保无论尾巴的形状如何，都有足够的空间填充颜色。

  


调整之后的 CSS 代码如上：

  


```CSS
.tooltip {
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    --posX: -1em;    /* 调整 P5 点 x 轴坐标 */
    border-image: conic-gradient(var(--bg) 0 0) 0 fill // 100vw; 
    
    --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, calc(var(--tail-pos) + var(--posX)) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
    clip-path: polygon(var(--shape));
}
```

  


现在你将能看到一个完美的提示框效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad8d419e14834693924b38005d9d91c0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1328&h=672&s=5244485&e=gif&f=504&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/MWMBXNr

  


需要知道的是，不管你怎么调整变量，你始终获得还是一个三角形尾巴。但我们实际开发过程中还会碰到各种奇形怪状的尾巴形状。例如：

## 创建带圆角的提示框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f949e8e35b5046cfbe0dc32b7f705947~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2228&h=1056&s=132046&e=jpg&b=f3ce96)

  


很多时候，你可能需要的是一个类似上图这种带有圆角效果的提示框。面对这种类型提示框，你可能首先会想到给 `.tooltip` 元素设置一个 `border-radius` 。不幸的是，添加了 `border-radius` 并不会给提示框带来任何的变化。这是因为 `border-radius` 和 `border-image` 并不完全兼容。这意味着，如果想继续享受 `border-image` 带来的便利，我们就得想办法调整 `border-image` ，来规避 `border-image` 和 `border-radius` 的不兼容性。

  


接下来，我将逐步向大家演示，应该如何调整我们的 CSS 代码，才能实现带有圆角效果的提示框。

  


我们首先要做的第一件事情，那就是增加一个新的 CSS 变量 `--radius` ，用它来设置圆角的半径。与此同时，我们给 `.tooltip` 设置 `border-radius` 和 `background` ，并暂时禁用 `border-image` 。

  


```CSS
.tooltip {
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    --posX: -1em;    /* 调整 P5 点 x 轴坐标 */
    --radius: round(1.25em, 1px); /* 圆角半径 */
    /*  border-image: conic-gradient(var(--bg) 0 0) 0 fill // 100vw;  */
    border-radius: var(--radius);
    background: var(--bg);
    
    --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, calc(var(--tail-pos) + var(--posX)) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
    clip-path: polygon(var(--shape));
}
```

  


此时，你在浏览器中能看到一个类似下图的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1708c05cdacb46b883da2f8df65efd86~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1366&h=786&s=3120454&e=gif&f=299&b=010101)

  


看到的只是一个带有圆角的矩形框是吧！不用着急！

  


现在，我们打开禁用的 `border-image` ，同时将 `conic-gradient` 的渐变颜色调整一下。这只是暂时的，目的就是为了让大家更好的理解调整 `border-image` 的过程。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/af44f9c5377c4b0f8679e90a03f06a5c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1284&h=584&s=1142403&e=gif&f=206&b=010101)

  


请注意，上图中的灰色部分，它就是 `border-image` 创建的“溢出的颜色”。很显然，这不是我们所需要的，它完完全全将元素都遮盖住了，而我们期望的是，只在元素的底部溢出。或者说，我们只希望要一个类似下图这样的一个 `border-image` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a5c50b55d3c444008462f09d71518139~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1662&s=348218&e=jpg&b=000000)

  


请注意图中灰色区域的变化，它代表着 `border-image` 的变化。如果你不熟悉 `border-image` 的话，要获得下面这个形状就会非常棘手。我尝试着通过 `border-image` 的在线生成器来向大家演示，要如何调整 `border-image` 才能获得上图中展示的灰色条形状（底部三角形是 `clip-path` 裁剪出来的）：

虽然提示框的圆角效果有了，但还有一个瑕疵存在。当你改变 `--tail-pos` 值调整尾巴位置时，当尾巴接近容器左侧或右侧边缘时，提示框自身和它的尾巴之间会产生一个锯齿状边缘：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/16e293d1afe74c7f9ef8e1974611c673~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=736&s=298930&e=jpg&b=000000)

  


这个现象是由 `border-image` 和 `border-radius` 重叠时所产的。为了解决这个问题，我们需要根据尾巴的位置（`--tail-pos`）来调整 `border-radius` 的值。

  


[熟悉 CSS `border-radius` 属性的开发者应该知道](https://juejin.cn/book/7199571709102391328/section/7199845563389444099)，它允许使用最多八个值，每个角落都可以分别设置水平垂直方向的半径：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3a84e9999a8417e95ae78d8e91f203e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=586&s=51700&e=jpg&b=101010)

  


在这个情况下，并不需要更新提示框的所有半径，只需要更新它的左下角和右下角的水平方向的半径：

  


```CSS
.tooltip {
    border-radius: 
        var(--radius) 
        var(--radius) 
        min(var(--radius), 100% - var(--tail-pos) - var(--tail-w) / 2)
        min(var(--radius), var(--tail-pos) - var(--tail-w) / 2)
        /
        var(--radius)
}
```

  


简单的解释一下。对于左下角顶点，当尾巴的位置在右侧时，`--tail-pos` 的值会很大，以确保 `border-radius` 的值等于 `--radius` ，这是最小值。当 `--tail-pos` 越接近提示框左侧边缘时，`--tail-pos` 的值会减小，并在某些时候变得小于 `--radius` 。结果是 `border-radius` 的值逐渐减小，直到为 `0` 。它会随着位置（`--tail-pos`）的更新而调整。你可以尝试着慢慢地拖动下面示例中的 `--tail-pos` 对应的滑块来更新它的位置，并仔细观察尾巴与边缘接触的变化，我想你能从这个视觉变化中了解 `border-radius` 变化：

  


```CSS

    --radius: round(1.25em, 1px); /* 圆角半径 */

        var(--radius) var(--radius) 
        min(var(--radius), 100% - var(--tail-pos) - var(--tail-w) / 2)
        min(var(--radius), var(--tail-pos) - var(--tail-w) / 2)
```

## 创建渐变提示框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d24eea36095a440e810ce90f3e16f2ec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1104&s=137687&e=jpg&b=f5f5f5)

  


当你看到上面示例代码的时候，你可能会有另一种想法，这个“现代方法”似乎将 CSS 复杂化了，难道真的有必要这么做？

  


敦不知，制作一个纯色的提示框（Tooltip）或语音气泡（Speech Bubble）真正的麻烦始于创建带有渐变的背景或边框。这是因为，当涉及到渐变时，让尾巴的颜色平滑地过渡到容器颜色并不容易。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6c33542049664e5a842be1121cd3351d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=513&s=126993&e=jpg&b=ffffff)

  


但对于我们来说，不会有这个问题存在，因为我们已经在 `border-image` 中使用了 `conic-gradient` 渐变来创建边框图片，只不过我们之前的案例只应用了一个颜色，因此大家看到的是一个纯色的提示框。换句话说，我们只需要将 `border-image` 的 `conic-gradient` 调整为两个颜色以上，就可以轻易实现一个带有渐变效果的提示框：

  


```CSS
.tooltip {
    --g: conic-gradient(from 0deg at top left,var(--bg), hsl(from var(--bg) var(--h, 45) s l));
    border-image: var(--g) var(--border-image);
    background: var(--g);
```

在创建渐变的时候，我在这里使用了 [CSS 的相对颜色语法](https://juejin.cn/book/7223230325122400288/section/7231515598088306720#heading-14)，基于 `--bg` 颜色（实际上是 `--primary`）在 `hsl` 颜色空间中改变了 `h` （色相）的值，因此你看到的是每个提示框的渐变颜色是有差异的。

  


另外需要注意的就是 `border-image-outset` 的值。当使用单一颜色时，我们不需要关注这个属性的值，只要给它设置一个足够大的值即可，比如 `100vw` ，使图片边框溢出覆盖整个 `clip-path` 区域。然而，当使用渐变时，我们不应该使用过大的值，以避免不小心裁剪到渐变，因此你可以将其设置为一个具体的值，比如 `border-image-outset: 0 0 var(--tail-h) 0` 。考虑到极限边缘的情况，我在示例中使用了两倍的 `--tail-h` 作为 `border-image-outset` 属性的值。

  


也就是说，当你使用这种方案创建渐变提示框时遇到了问题，通常需要先检查 `border-image-outset` 属性的值是否符合要求！

  


之前所展示的案例都是基于 `border-image` 来实现的。它虽然有很多的优势，但在处理圆角方面还是令不少人止步，加上 `border-image` 的使用是真的复杂，这也是部分开发者不太喜欢使用 `border-image` 的另一个原因。甚至很多开发者更热衷于“元素加伪元素”的方式来创建提示框。为了满足这部分开发者的需求，我们接下来就来探讨使用伪元素创建提示框的方法。

  


我们继续以渐变提示框为例，但接下来的代码中你将看不到 `border-image` 。

  


与之前制作提示框相似，首先定义一些 CSS 自定义属性，便于控制提示框的 UI 样式：

  


```CSS
.tooltip {
    /* 控制提示框颜色 */
    --primary: #ff5722;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196f3, #9c27b0, #ff9800, #9e9e9e, #607d8b, #fff,#bbb;
    --color: color-contrast(var(--bg) vs var(--color-set) to var(--contrast-target));
    
    --fs: 1.5em; /* font-size */
    
    /* 控制提示框尾巴的尺寸 */
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
    --posX: -1em; /* 调整 P5 点 x 轴坐标 */
    --radius: round(1.25em, 1px); /* 圆角半径 */
    
    /* 创建渐变 */
    --g: conic-gradient(from 0deg at top left,var(--bg), hsl(from var(--bg) var(--h, 60) s l));
}
```

  


并且给其添加一些基础样式：

  


```CSS
.tooltip {
    width: fit-content;
    aspect-ratio: 21 / 9;
    display: grid;
    place-content: center;

    font-size: var(--fs);
    text-align: center;
    padding: 1em;
    place-self: center;

    color: var(--color);
    position: relative;
    z-index: 0;
    border-radius: var(--radius);
    background: var(--g);
    
    &::before {
        content: "";
        position: absolute;
        z-index: -1;
        inset: 0 0 calc(-1 * var(--tail-h));
        background-image: inherit;
        opacity: .5; /* Test Code */
    }
}
```

  


注意，上面代码中的 `::before` 伪元素主要是用于创建提示框尾巴的形状，我们依旧以创建三角形尾巴为例。它继承了 `.tooltip` 的 `background` ，并且将 `bottom` 的值设置负的 `--tail-h` （尾巴的高度），使其向元素 `.tooltip` 底部延伸了一个 `--tail-h` 高度，用于放置三角形尾巴。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/379066072cff48e393a7a9b09803dd53~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=948&s=415254&e=jpg&b=ff8223)

  


接下来的关键是在伪元素 `::before` 使用 `clip-path` 的 `polygon()` 裁剪出所需要的三角形尾巴：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    /* 裁剪尾巴形状 */
    --shape: 
        50% 50%,
        min(100%, var(--tail-pos) + var(--tail-w) / 2) calc(100% - var(--tail-h)),
        var(--tail-pos) 100%,
        max(0%, var(--tail-pos) - var(--tail-w) / 2) calc(100% - var(--tail-h));
      
   &::before {
       /* 省略其他 CSS */
       clip-path: polygon(var(--shape));
   }   
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ac8985636d464501b63bcbb22410a402~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=948&s=380962&e=jpg&b=000000)

  


正如你所看到的，使用 `clip-path` 裁剪出类似上图这样的一个多边形形状，这个形状就是提示框的尾巴。你可能会好奇，我们要的是一个三角形，为什么这里裁剪出一个多边形呢？这样裁剪是便于尾巴和提示框衔接，避免两者之间产生间隙：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ae0611e7d6df4f87a6f99cc99a8f94f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1376&h=742&s=5385131&e=gif&f=421&b=010101)

  


现在，尾巴和提示框元素两者之间的渐变效果能完美无缝的衔接。但有一个问题是，尾巴接近极限边缘时，它与提示框元素衔接处会存在一定的缺陷。我们可以按照之前案例所介绍的方法，调整 `--radius` 值：

  


```CSS
.tooltip {
    --_radius: var(--radius) var(--radius) min(var(--radius), 100% - var(--tail-pos) - var(--tail-w) / 2) min(var(--radius), var(--tail-pos) - var(--tail-w) / 2) / var(--radius);
    border-radius: var(--_radius);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b28620ffef89435ebd3c7b5f6e8cdf3e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1352&h=736&s=8116813&e=gif&f=392&b=000000)

  


你可能已经发现了，到目前为止，我们看到的三角形是一个等腰三角形。因为，上面的代码还没有将改变三角形形状的 `--posX` 引用到裁剪尾巴的 `polygon()` 坐标中。我们把 `--posX` 用起来：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    --shape: 
        50% 50%,
        clamp(var(--tail-w),var(--tail-pos) + var(--tail-w)/2,100%) calc(100% - var(--tail-h)),
        calc(var(--tail-pos) + var(--posX)) 100%,
        clamp(0%,var(--tail-pos) - var(--tail-w)/2,100% - var(--tail-w)) calc(100% - var(--tail-h));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4b21eb92fff94076ac68d30e79320df6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1230&h=700&s=2041339&e=gif&f=213&b=010101)

  


除此之外，这个方案的还有另一个重要点，**需要使用** **`background-size`** **属性设置背景图尺寸**。因为伪元素 `::before` 覆盖了一个更大的区域（向底部延伸），所以必须增加渐变的高度，以便它覆盖相同的区域。

  


另一个重要的点是 `.tooltip` 中声明的 `background-size`。由于伪元素覆盖了一个更大的区域，因为负底部值，我们必须增加渐变的高度，以便它覆盖相同的区域。

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    --e: max(0%,-1 * var (--posX) - var (--tail-pos), var (--posX) + var (--tail-pos) - 100% );
    --g: 
        conic-gradient(from 0deg at top left,var(--bg), hsl(from var(--bg) var(--h, 60) s l)) 
        50% 0 / calc(100% + 2 * var(--e)) calc(100% + var(--tail-h));
        
   &::before {
       inset: 0 calc(-1 * var(--e)) calc(-1 * var(--tail-h));
       padding-inline: var(--e);
       clip-path: polygon(var(--shape)) content-box;
   }     
}
```

  


我们新引入了 `--e` 变量，它定义了渐变的宽度，用于覆盖 `.tooltip` 元素以及伪元素的内距 `padding-inline` 以及 `left` 和 `right` 的偏移量：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e6d417559e7f413d9b514a1f5fd92268~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1242&h=662&s=2546958&e=gif&f=183&b=010101)

  


看起来可能像是一个复杂的配置，但实际上，`--e` 在大多数情况下会等于 `0`，这样我们就得到了与我们之前示例相同的代码。然而，当尾部溢出 `.tooltip` 容器时，`--e` 的值会增加，从而扩大渐变的区域，以覆盖溢出部分。你看到最终的效果如下：

Web 开发者常常将 [CSS 渐变属性](https://juejin.cn/book/7199571709102391328/section/7199845781149810727)（例如 `linear-gradient` 、`radial-gradient` 和 `conic-gradient` 等）和[多背景](https://juejin.cn/book/7199571709102391328/section/7199845838103773195)结合在一起来创建各种形状。比如用它们来[创建各式各样的纹理](https://css-pattern.com/)：

同样的，我们可以利用这些特性与 [CSS 的遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)（`mask`）结合起来创建提示框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/58285b2b49404fedaee7fd27c524c6ac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1488&s=720645&e=jpg&b=000000)

  


如上图所示，我们使用径向渐变 `radial-gradient` 和锥形渐变 `conic-gradient` 来创建带有圆角的矩形形状，然后使用另一个锥形渐变来绘制底部的三角形。前两个渐变只覆盖元素的内距区域（`padding-box`），这样我们可以使用 `border-bottom` 属性来控制最后一个渐变的大小。

  


试想一下，我们把边框变成透明颜色，然后将其他的渐变颜色统一起来，是不是就得到了一个纯色的提示框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf022c488d9646ccaff0e05b4ed09b58~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1124&h=654&s=1776074&e=gif&f=408&b=000000)

  


当然，我们最终不会将这些渐变用于 `background` ，因为在某些场景之下，这样做是有一定的局限性的，比如创建一个渐变的提示框。在这里，我只是想通过 `background` 来向大家演示使用渐变创建一个提示框的过程。我们最终还是会将这些渐变当作遮罩图，并且用于 `mask` 属性。这样做，我们能得到与 `background` 类似的效果，但它的灵活性更强，除了创建纯色的提示框之外，还可以创建渐变提示框。

  


同样的，在这个过程中，我仍然考虑引入一些 CSS 自定义属性，这样易于我们更好的控制提示框的样式，还可以快速获得不同的变体。详细的 CSS 代码如下：

  


```CSS
.tooltip {
    --r: round(1.25em, 1px); /* 控制圆角半径 */
    --t: round(1.25em, 1px); /*  控制尾巴尺寸 */
    --a: 60deg; /* 控制尾巴角度*/
    --p: 50%; /* 控制尾巴位置 */

    --primary: #ff5722;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196f3, #9c27b0, #ff9800, #9e9e9e, #607d8b, #fff, #bbb;
    --color: color-contrast(var(--bg) vs var(--color-set) to var(--contrast-target));
    --fs: 1.5em; /* font-size */

    width: fit-content;
    aspect-ratio: 21 / 9;
    display: grid;
    place-content: center;

    font-size: var(--fs);
    text-align: center;

    padding: 1em;
    place-self: center;

    color: var(--color);
    --g: linear-gradient(45deg, var(--bg), hsl(from var(--bg) 60 s 50%));
    --m: 
        conic-gradient(at var(--r) var(--r), var(--bg) 75%, #0000 0) calc(var(--r) / -2) calc(var(--r) / -2) padding-box,
        radial-gradient(50% 50%, var(--bg) 98%, #0000) 0 0 / var(--r) var(--r) space padding-box;
    --_g: #0000, var(--bg) 1deg var(--a), #0000 calc(var(--a) + 1deg);
    --_m: conic-gradient(from calc(var(--a) / -2) at var(--p) 100%, var(--_g))  bottom/100% 50% no-repeat border-box,
      var(--m);
    border-bottom: var(--t) solid #0000;
    padding: calc(2 * var(--r) / 3);
    background: var(--g) border-box;
    mask: var(--_m);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2225a520e6234541a07ce38920500878~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1364&h=792&s=9523671&e=gif&f=377&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/WNqgwdv

  


需要知道的是，在这个方案中，提示框的三角形尾巴是通过 `conic-gradient` 创建的，这意味着你只需要调整 `conic-gradient` 相关参数就可以获得不同的三角形变体。如果你对 `conic-gradient` 感兴趣的话，[可以点击这里获取更多相关的信息](https://juejin.cn/book/7223230325122400288/section/7259668771856941111)。另外一点就是，如果三角形位于提示框元素的其他侧，那么 `border-bottom` 就需要做出相应的调整。

  


在这节内容中，向大家展示了三种不同的方式来创建带有渐变效果的提示框，它们都各有利弊，大家在实际应用中可以根据具体的需求，选择自己最熟悉，最能掌握的方案来创建渐变提示框！

  


## 创建带边框的提示框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/751dc29a6e6a4bcd8323edc643ec07aa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=948&s=158284&e=jpg&b=f9fafc)

  


如果说创建渐变提示框很棘手，那么创建带有边框的提示框就是恶梦。因为我们需要确保边框围绕的主元素和带有边框的尾巴无缝衔接。就这一点，已经使很多 Web 开发者感到很痛苦了。接下来，我们来看看现代 CSS 又是如何构建带有边框的提示框。

  


通过前面的内容的学习，我想你肯定可以使用 `border-image` 和 `clip-path` 构建不带边框的提示框。这意味着，你独立创建一个带边框的提示框已经不远了。

  


在开始之前，我首先要告诉大家的是，制作带边框的提示框，并不一定要使用 `border` 属性，甚至可以说，使用 `border` 并不是一个合适的选项。它不但起不了作用，还会带来更多令你感到棘手的问题。

  


其实，在 CSS 中，除了 `border` 属性之外，还有很多方法可以使元素在视觉上看上去带有边框效果，比如多个背景就是一种不错的方法，比如使用伪元素 `::after` 或 `::before` 给元素添加边框效果。这意味着，我们可以使用相似的方法来创建带边框的提示框。例如，我们分别在主元素 `.tooltip` 和它的伪元素 `::before` 或 `::after` 使用之前的方法（`border-image` 加 `clip-path`）创建不带边框的提示框，然后再将两者组合在一起，就可以使我们获得一个带有边框的提示框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/be8f088d6cb8452080084cd9e7f8bbc8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1125&s=235820&e=jpg&b=f9fafc)

  


正如上图所示，在 `.tooltip` 和 `.tooltip::before` 创建的两个提示框，它们的大小是有差异的，只有这样才能拼结出我们所需要的带有边框的提示框。

  


现在，是不是思路很清晰了。如果是的话，我们就可以开始编写 CSS 代码了。与之前一样，引入一些 CSS 自定义属性来控制提示框的 UI 样式：

  


```CSS
.tooltip {
    --primary: #ff5722;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196F3, #9C27B0, #FF9800, #9E9E9E, #607D8B, #fff, #bbb;
    --color: color-contrast(var(--bg) vs var(--color-set) to var(--contrast-target));
    
    /* 控制边框颜色和厚度 */
    --border-color: color-mix(in srgb, var(--primary), #000 60%); /* 边框颜色 */
    --b: round(.5em, 1px); /* 边框厚度 */
    
    --fs: 1.5em; /* font-size */
    
    /* 控制尾巴 */
    --tail-w: round(2em, 1px); /* 宽度 */
    --tail-h: round(1em, 1px); /* 高度*/
    --tail-pos: 50%; /* 位置 */
}
```

  


与之前相比，新增了 `--b` 和 `--border-color` 变量，分别用于控制边框的厚度和颜色。

  


接着，就可以使用之前介绍的 `border-image` 和 `clip-path` 方法分别在主元素 `.tooltip` 和它的伪元素 `::before` 上创建提示框：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    --g: conic-gradient(var(--border-color) 0 0);
    --shape: 0 0, 100% 0, 100% 100%, min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%, var(--tail-pos) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
    
    border-image: var(--g) 0 fill // var(--tail-h); 
    clip-path: polygon(var(--shape));
    position: relative;
    z-index: 0;
    
    &::before {
        content:"";
        position: absolute;
        inset: var(--b);
        z-index: -1;
        
        --g: conic-gradient(var(--bg) 0 0); 
        border-image: var(--g) 0 fill // var(--tail-h);
        clip-path: inherit;
    }
}
```

  


不难发现，`.tooltip` 和它的伪元素 `::before` 使用了相同的 `border-image` ，仅渐变颜色有差异。并且伪元素继承了 `.tooltip` 的 `clip-path` 。另外，伪元素使用了 `inset` 属性，使伪元素向内收缩 `--b` 距离，伪元素尺寸会变小。

  


这个时候，你在浏览器中看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1be98c1e2ce241899242dbd80a12d702~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=738&s=7048592&e=gif&f=449&b=123d4c)

  


乍一看，效果还不错。但是，一旦调整 `--tail-pos` 值来改变尾巴的位置，向左或向右移动时，越接近主元素的两侧，问题就出来了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d66b8ba4e5a4c3b881551bf4e74d590~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=630&s=302663&e=jpg&b=164559)

  


这是因为两个 `clip-path` 裁剪出来的形状没有对齐，因为伪元素覆盖的区域比主元素 `.tooltip` 要小（小两个 `--b`）。我们需要将 `inset` 属性左右值设置为 `0` （即 `left` 和 `right` 的值）：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    &::before {
        /* 省略其他 CSS */
        
        inset: var(--b) 0;
    }
}
```

  


这个时候，你看到的效果就变成下图这样了，左右边框被丢失了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fe00fbef26f24a19bf3c9b08b7bd6400~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=662&s=1029813&e=gif&f=138&b=123d4c)

  


这是因为伪元素的 `border-image` 覆盖了父元素。我们需要调伪元素的 `border-image` ：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    &::before {
        /* 省略其他 CSS */

        border-image: var(--g) 0 fill /0 var(--b)/ var(--tail-h) 0;
    }
}
```

  


现在一切看起来都很棒：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d498efb3a52d4bbda65d8f417e11bbec~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1370&h=702&s=8880905&e=gif&f=405&b=123d4c)

  


正如你所看到的，两个提示框现在可以完美的对齐了。整个效果看上去几乎是完美的。为什么说是“几乎”完美呢？那是因为尾巴的边框厚度还有一点瑕疵。当我们调整 `--tail-w` 和 （或）`--tail-h` 来调整尾巴的尺寸时，它的边框厚度与主元素的边框厚度不一致：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/74f85ec5011442b484c779a3582a137a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1608&h=682&s=4771618&e=gif&f=438&b=123d4c)

  


下图显示了尾巴边框明显要比主元素边框细得多，尤其是随着 `--tail-w` 和 `--tail-h` 比例越大时，差异越明显：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8e209046c73541a987a7f615c305ac07~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1409&s=648710&e=jpg&b=164559)

  


当然，在大多数情况下，这个现象不会这么明显，要是你不具备像素眼，几乎看不出它们之间的差异。反过来说，如果你是位完美主义者，容不得任何细微的差异，那就无法接受了。不过，并不是无解，[我们可以使用三角函数可以完美解解决这个问题](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)。

  


在此之前，三角形尾巴的大小都是由 `--tail-w` （宽度）和 `--tail-h` （高度）两个变量来控制大小，但这种做法已经不适合用于带有边框的提示框中，因为在当前设置下无法得到我们想要的效果。这意味着我们需要调整控制三角形尾巴尺寸的方法。在这里我将考虑新增一个角度变量 `--a` 和原有的高度 `--tail-h` 来控制三角形尾巴的宽度 `--tail-w` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ec67f3d2fb484bf587d2978de01c6b0c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=824&s=239635&e=jpg&b=f9fafc)

  


```CSS
--tail-w = 2 × tan(α ÷ 2) = calc(2 * tan(var(--a) / 2) * var(--tail-h))
```

  


现在按照这个方式重新更新 CSS 的代码：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    
    --a: 60deg; /* 控制三角形顶角角度 */
    --tail-h: 1em; /* 高度*/
    --tail-w: calc(2 * tan(var(--a) / 2) * var(--tail-h)); /* 宽度 */
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9c693cf9c03843fe9ccb4d4adf3b2385~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1594&h=686&s=2662579&e=gif&f=242&b=123d4c)

  


看上去没啥变化，对吧！我们需要重新更新伪元素的 `clip-path` 的 `polygon()` 的坐标，不能直接继承主元素 `.tooltip` 的 `clip-path` 。因为我们需要一个新的值来获得正确的尾巴边框厚度。这是一个烧脑的过程，需要应用一些数学原理计算出绘制三角形的坐标：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0cdcdb3e5f4b47dbaeeeeb01fbb9b262~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=709&s=132930&e=jpg&b=f9fafc)

  


正如上图所示，根据已知的变量 `--b` 、`--tail-h` 和 `--a` 以及三角函数计算出所需的坐标：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    --a: 60deg; /* 三角形顶角角度 */
    --tail-w: calc(2 * tan(var(--a) / 2) * var(--tail-h)); /* 宽度 */
    --tail-h: 1em; /* 高度*/
    --shape: 
        0 0, 
        100% 0, 
        100% 100%,
        min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%,
        var(--tail-pos) calc(100% + var(--tail-h)),
        max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;
        
    &::before {
        /* 省略其他CSS */
        --shape: 
            0 100%, 
            0 0, 
            100% 0, 
            100% 100%,
            min(
                100% - var(--b),
                var(--tail-pos) + var(--tail-w) / 2 - var(--b) * tan(45deg - var(--a) / 4)
            ) 100%,
            var(--tail-pos) calc(100% + var(--tail-h) + var(--b) * (1 - 1 / sin(var(--a) / 2))),
            max(
                var(--b),
                var(--tail-pos) - var(--tail-w) / 2 + var(--b) * tan(45deg - var(--a) / 4)
            ) 100%;
        clip-path: polygon(var(--shape));
    }        
}
```

  


看起来很复杂对吧！事实上，你只要了解一些几何知识，这一切都就变得顺理成章了。如果你不想深究其背后的逻辑，那么你只需要调整几个变量就可以控制这一切：

  


```CSS
.tooltip {
    --primary: #ff5722;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196f3, #9c27b0, #ff9800, #9e9e9e, #607d8b, #fff, #bbb;
    --color: color-contrast(var(--bg) vs var(--color-set) to var(--contrast-target));
    --border-color: color-mix(in srgb, var(--primary), #000 60%);
    --b: 0.5em; /* 边框厚度 */
    --fs: 1.5em; /* font-size */
    
    width: fit-content;
    aspect-ratio: 21 / 9;
    display: grid;
    place-content: center;

    font-size: var(--fs);
    text-align: center;
    padding: calc(1em + var(--b));
    place-self: center;

    color: var(--color);

    --a: 60deg; /* 三角形顶角角度 */
    --tail-w: calc(2 * tan(var(--a) / 2) * var(--tail-h)); /* 宽度 */
    --tail-h: 1em; /* 高度*/
    --tail-pos: 50%; /* 位置 */
    --g: conic-gradient(var(--border-color) 0 0);
    
    border-image: var(--g) 0 fill // var(--tail-h);;

    --shape: 0 0, 100% 0, 100% 100%,min(100%, calc(var(--tail-pos) + var(--tail-w) / 2)) 100%,var(--tail-pos) calc(100% + var(--tail-h)), max(0%, calc(var(--tail-pos) - var(--tail-w) / 2)) 100%, 0 100%;

    clip-path: polygon(var(--shape));
    position: relative;
    z-index: 0;

    &::before {
        content: "";
        position: absolute;
        z-index: -1;
        inset: var(--b) 0;
        --g: conic-gradient(var(--bg) 0 0);
      
        border-image: var(--g) 0 fill / 0 var(--b) / var(--tail-h) 0;

        --shape: 0 100%, 0 0, 100% 0, 100% 100%, min( 100% - var(--b), var(--tail-pos) + var(--tail-w) / 2 - var(--b) * tan(45deg - var(--a) / 4)) 100%, var(--tail-pos) calc(100% + var(--tail-h) + var(--b) * (1 - 1 / sin(var(--a) / 2))),max(var(--b),var(--tail-pos) - var(--tail-w) / 2 + var(--b) * tan(45deg - var(--a) / 4)) 100%;
        clip-path: polygon(var(--shape));
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aa50c4360511494fbcc0bb158b628f2c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1588&h=680&s=4551105&e=gif&f=402&b=123d4c)

  


> Demo 地址：https://codepen.io/airen/full/WNqgGXY

  


经过严谨的数学计算获得的 `polygon()` 坐标，使你创建的带边框提示框才是完美的。你可以尝试着调整控制尾巴的相关变量，获得不同变体。你也可以按照相似的方式，调整尾巴的方向：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d67448c2f1514dc787a9f51185190c6e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1646&h=690&s=7535823&e=gif&f=432&b=123d4c)

详细代码请参阅 Demo 源码！需要知道的是，这种方案仅适用于尾巴是等腰三角形，要是其他类型的三角形，会增加相应的计算难度。在这里就不挑战这种极限了，如果你感兴趣的话，不妨尝试一下！

## 创建有趣的尾巴形状

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf1b88e8dc56497fb2eb32590302e735~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=965&s=242021&e=jpg&b=f0f0f0)

  


你可能已经注意到了，之前所展示的案例，提示框的尾巴都是三角形形状。通过前面内容的学习，我相信你可能创建出符合你需求的三角形尾巴。但往往有的时候，提示框的尾巴不总是三角形形状，它有可能是各种奇形怪状的尾巴形状（如上图所示）。

  


接下来，我们来简单的探讨一下，如何使用现代 CSS 实现非常三角形形状的尾巴。以下图这个提示框为例：

我们主要通过 CSS 渐变和[遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252)来制作，具体步骤如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ff88f46987824abca71bb278504795a4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2021&s=504788&e=jpg&b=ffffff)

  


整个遮罩图由两个 `radial-gradient` 和一个 `linear-gradient` 绘制而成，其中提示框底部不规则的尾巴是由两个 `radial-gradient` 组成（两个四分之一的圆拼接而成）。[其制作原理与之前所介绍的切割元素角落是相似的](https://juejin.cn/book/7395119173656903715/section/7400975207551139875)。

  


同样的，为了便于控制提示框的 UI 形状，我们同样会引入几个 CSS 变量：

  


```CSS
.tooltip {
    --s: 1.5em; /* 尾巴尺寸 */
    --p: 48%; /* 尾巴位置 */
    --r: 1em; /* 圆角半径 */
}
```

  


设置提示框的基础样式很简单，但这里需要给元素设置一个透明边框，它的大小和提示框尾巴大小相等，即 `--s` ：

  


```CSS
.tooltip {
    --s: 1.5em; /* 尾巴尺寸 */
    --p: 48%; /* 尾巴位置 */
    --r: 1em; /* 圆角半径 */
    border: var(--s) solid #0000;
    border-radius: calc(var(--r) + var(--s));
    background: linear-gradient(60deg,var(--bg),hsl(from var(--bg) var(--h, 60) s l)) border-box;
}
```

  


请注意，`background` 是从 `border-box` 开始填充。现在，看到效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd73a62277964a5cbc3cf7d873d49d07~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1374&h=726&s=10434145&e=gif&f=271&b=123d4c)

  


接下来的关键就是使用 `linear-gradient` 和 `radial-gradient` 创建遮罩图，并将它们应用于 `mask` ：

  


```CSS
.tooltip {
    /* 省略其他 CSS */
    --_m: 100% / var(--s) var(--s) no-repeat;
    --_p: clamp(
        var(--r) + 3 * var(--s) / 2,
        var(--p),
        100% - var(--r) - 3 * var(--s) / 2
    );
    mask: 
        radial-gradient(100% 100% at 100% 100%, #0000 99%, orange 102%) calc(var(--_p) + var(--s) / 2) var(--_m),
        radial-gradient(100% 100% at 0 100%, #0000 99%, orange 102%) calc(var(--_p) - var(--s) / 2) var(--_m),
        linear-gradient(red 0 0) padding-box;
}
```

## 小结

  


简单的小结一下，前面我们通过多个案例向大家展示了如何使用现代 CSS 来创建各种提示框（Tooltip）和语音气泡(Speech Bubble)的 UI 效果。乍一看，CSS 的代码都很复杂，但只要你对现代 CSS 特性有所了解，那理解这些代码都将不是问题。
