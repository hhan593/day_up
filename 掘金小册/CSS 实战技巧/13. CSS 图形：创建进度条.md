进度条组件（`ProgressBar`）是用户界面设计中的一个重要元素，它在 Web 上的应用非常广泛，它能显著改善用户体验，通过觉觉反馈告诉用户当前操作的进展情况。例如文件上传和下载、应用程序的安装和更新、数据加载、表单提交、多步骤流程和音乐或视频播放等场景。根据不同的应用场景，进度条可以是简单的横向条形，也可以是复杂的圆形或自定义形状。通过合理使用进度条，可以有效地管理用户期望，提升操作的透明度和用户满意度。

  


在 Web 中，有时候也会将[面包屑组件](https://juejin.cn/book/7395119173656903715/section/7405199145461350438)（`Breadcrumb`）和[步骤指示器组件](https://juejin.cn/book/7395119173656903715/section/7406461672572846080)（`Stepper`）视为进度条中的一种，因为它们和进度条组件（`ProgressBar`）有着相似的功能，即通过视觉反馈告诉用户当前操作的进展情况。虽然在某些场景下，它们的 UI 风格或功能可以是相似的，但在制作进度条组件时，与之前所介绍的面包屑组件和步骤指示器还是有着明显的差异。其中最大的差异就是，HTML 为进度条组件提供了一个原生的标签元素，即 `<progress>` 。

## 我们就从 `<progress>` 元素开始

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/be1e0ec12b0c4de684b00539610ffccc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2232&h=886&s=270659&e=jpg&b=feede7)

  


[HTML 提供了一个用于显示进度的原生元素](https://html.spec.whatwg.org/multipage/form-elements.html#the-progress-element)，即 `<progress>` ，它向用户提供有关完成情况的视觉或听觉反馈。这种视觉反馈在诸如提交表单、显示数据加载或文件下载信息等场景中非常有用。

  


我们使用现有的 HTML `<progress>` 元素来制作进度条组件（`ProgressBar`），可以节省很多在 Web 可访问性方面的工作。通过调整颜色和布局，推动阵内置元素的自定义极限，以使组件现代化，并更好地适应设计系统。

  


然而，`<progress>` 美中不足的是，默认的 `<progress>` 元素在不同的浏览中呈现的 UI 效果可能不一致。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d930a8d310b14e5e8d050d594e56767b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=503&s=135403&e=jpg&b=e3e9e9)

  


庆幸的是，我们可以通过一些 CSS 技巧，使进度条的 UI 风格在各浏览器中保持一致！稍后我们聊到进度条样式设置的时候会详细介绍！

  


要在 Web 上呈现一个进度条，最简单的方式就是像下面这样，使用一个 `<progress>` 元素：

  


```HTML
<progress></progress>
```

  


代码中的 `<progress>` 元素没有设置值，则表示该元素的进度为不确定状态，它将在浏览器中呈现一种加载动画。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5c6df06271f4162bbb8b944eecac732~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1138&h=510&s=157239&e=gif&f=106&b=1f5b6e)

  


> Demo 地址：https://codepen.io/airen/full/dyBeKMr

  


**注意，如果无特殊说明，我们接下来展示的效果都是在 Chrome 浏览器的效果！**

  


我们也可以给 `<progress>` 显式定义 `max` 和 `value` 属性。

  


-   `max` 用来设置进度条范围，它的默认值为 `1` ，因此进度条范围在 `0 ~ 1` 之间。也可以将 `max` 的值设置为 `100` ，表示 `100%`，进度条范围则设置 `0 ~ 100` （即 `0% ~ 100%`）
-   `value` 用来设置当前进度，它的值在 `[0, max]` 的范围内

  


例如：

  


```HTML
<progress max="100" value="70"></progress>
<progress max="1" value=".7"></progress>
```

  


它们都表示当前进度是 `70%` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d193c44894d240e78979704a529e1b31~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=893&s=164242&e=jpg&b=246677)

  


但我们实际使用的时候，我更喜欢使用 `<label>` 标签元素来包裹 `<progress>` 元素，以便跳过显式的关系属性，改用隐式关系：

  


```HTML
<label>
    <span class="sr-only">加载进度</span>
    <progress max="100" value="70">70%</progress>
</label>
```

  


在我的演示中，我选择只为屏幕阅读器提供提示文本。通过将提示文本包裹在 `<span>` 中并应用一些样式，使其有效地隐藏在屏幕外（只有屏幕阅读器这样的辅助技术可以识别）：

  


```CSS
.sr-only {
    clip: rect(1px, 1px, 1px, 1px);
    clip-path: inset(50%);
    height: 1px;
    width: 1px;
    margin: -1px;
    overflow: hidden;
    padding: 0;
    position: absolute;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fe511f6d97f34cd3a3a0ba41956b352a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3202&h=1708&s=509448&e=jpg&b=366677)

  


另外，我还在 `<progress>` 元素内部添加了文本 `70%` （对应 `value` 的值）。这并不是必须的，但我个人建议你还是加上它，以防用户使用的浏览器较旧且不支持 `<progress>` 元素。尽管现在这种情况已经非常少见，但我们这样做能使自己的应用覆盖更多的浏览器，即使是非常旧的浏览器。

  


此时，你在浏览器中看到的效果与之前没用 `<label>` 标签包裹的进度条效果是一致的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/79822f2c89554c238bc45739b75aa6fe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=893&s=160110&e=jpg&b=246677)

  


> Demo 地址：https://codepen.io/airen/full/MWMGXpr

  


如果你的视力正常，可能很容易将进度指示器与相关元素和页面区域关联起来，但对于视觉障碍用户来说，这并不那么清晰。我们可以通过将 `aria-busy` 属性添加到最顶部的元素中，该元素会在加载完成时发生变化（在 JavaScript 中，从任务开始时将 `aria-busy` 切换为 `true`，完成后切换为 `false`）。此外，通过 `aria-describedby` 指明进度和加载区域之间的关系：

  


```HTML
<main id="loading-zone" aria-busy="true">
    <label>
        <span class="sr-only">加载进度</span>
        <progress aria-describedby="loadding-zone" max="100" value="70">70%</progress>
    </label>
</main>
```

  


虽然 `<progress>` 元素的隐式角色是 `progressbar`，我已为缺乏这种隐式角色的浏览器明确指定它。我还添加了 `indeterminate` 属性，以明确将元素置于未知状态，这比观察元素没有设置值要清晰得多。

  


```HTML
<main id="loading-zone" aria-busy="true">
    <label>
        <span class="sr-only">加载进度</span>
        <progress 
            indeterminate 
            role="progressbar" 
            aria-describedby="loadding-zone" 
            tabindex="-1"
            >未知</progress>
    </label>
</main>
```

  


使用 `tabindex="-1"` 使进度元素可以通过 JavaScript 聚焦。这对屏幕阅读器技术很重要，因为随着进度变化，将焦点设置到进度条上，会向用户宣布更新后的进度已达到多少。

  


注意，上面代码中的 `role` 、`aria-busy` 、`aria-describedby` 等属性都是 [ARIA 方面的知识](https://www.w3.org/TR/html-aria/)，这些内容已经超出这节课的范畴，在这里就不做过多的阐述。

  


为了使接下来的演示简单化，我们统一使用下面这个 HTML 结构来构建进度条组件：

  


```HTML
<label>
    <span class="sr-only">加载进度</span>
    <progress value="70" max="100"  tabindex="-1" role="progressbar" >70%</progress>
</label>    
```

  


在我们处理 CSS 之前，应该注意到 `<progress>` 元素的实现因浏览器而异，因为其内部的 HTML 结构是不同的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ea873c042f8f4055878129f9d365f44e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1657&s=989202&e=jpg&b=eaf1f1)

  


主要有两种结构：

  


```HTML
<!-- Firefox 浏览器使用的结构 -->
<progress>
    <div pseudo="-moz-progress-bar"></div>
</progress>

<!-- Webkit 和 Blink 内核浏览器（如 Chrome、Safari 和 Edge ）使用的结构 -->
<progress>
    <div pseudo="-webkit-progress-inner-element">
        <div pseudo="-webkit-progress-bar">
            <div pseudo="-webkit-progress-value"></div>
        </div>
    </div>
</progress>
```

  


这种差异使得样式化 `<progress>` 元素变得非常麻烦，因为我们必须编写适用于这两种结构的样式。

  


## 美化 `<progress>`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd3ce391a1f749c5ba047d341267d3f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=800&s=1792660&e=png&b=211428)

  


刚才提到过，`<progress>` 元素内置的 HTML 元素（Shadow DOM）会因浏览器而异，这些特殊的隐藏部分的样式也可能会因浏览器而异，因此你看到的同一个 `<progress>` 元素构建的进度条，会在不同的浏览器中呈现不一样的 UI 效果。

/* Chrome 和 Edge */
progress {

}
progress::-webkit-progress-bar {
    background-color: gray;
    block-size: 100%;
    inline-size: 100%;
    box-sizing: border-box;
    -webkit-user-modify: read-only !important;
}
progress::-webkit-progress-value {
    background-color: green;
    block-size: 100%;
    inline-size: 50%;
    box-sizing: border-box;
    -webkit-user-modify: read-only !important;
}

/* Safari */
progress {
    appearance: auto;
    box-sizing: border-box;
    display: inline-block;
    block-size: 1em;
    inline-size: 10em;
    vertical-align: -0.2em;
}

    box-sizing: inherit;
    height: 100%;
    width: 100%;
}

    height: 100%;

不难发现，`<progress>` 元素在 Chrome 和 Edge 浏览内置的默认样式是相同的，Safari 与 Chrome 几乎相同，差异较大的是 Firefox 浏览器中的默认样式。

  


换句话说，CSS 样式化 `<progress>` 元素还是有点棘手的，最为麻烦的是，隐藏的部分可能难以选择，并且通常只提供有限的样式属性！

  


这个时候，你的疑惑可能就来了，既然如此，那要如何才能使 `<progress>` 样式在各主流浏览器中保持一致呢？这就是我们接下来要探讨的主要内容。

  


### 禁用浏览器的默认样式

  


首先，我们要做的是通过以下代码来禁用浏览器的默认样式：

  


```CSS
progress[value] {
    appearance: none;
    border: none; /* 去掉 Firefox 中的边框 */
}
```

  


你可能会好奇，为什么在选择器中要添加 `[value]` 。这是因为，没有设置 `value` 属性的 `<progress>` 元素不会指示进度，而这并不是我们在这节课中感兴趣的行为。之前就提到过，如果没有 `value` 属性，进度条将是“未确定的”，这表示某个活动正在进行，但没有指示它预计需要多长时间的提示。换句话说，`progress[value]` 选择器中的样式就不会影响到“未确定的进度条”（不带 `value` 属性的 `<progress>` ）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a6b53d53d50a4283818647e08a34fa30~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=406&s=114741&e=gif&f=48&b=1e5b6d)

  


### 设置进度条尺寸

  


重置完浏览器默认样式之后，我们来设置进度条尺寸。通常情况之下， 一个进度条在 UI 呈现上主要包括两个部分：进度条轨道（Track）和当前进度（Progress）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/655541940fb640b980800070dc9c6bde~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1657&s=260432&e=jpg&b=e3e9e9)

  


可以将 `<progress>` 元素自身视作为进度条的轨道。以简单的水平进度条为例，它的大小将由其 `width` 和 `height` （或者 `inline-size` 和 `block-size` ）来决定。

  


```CSS
progress[value] {
    --track-size: min(10px, 1ex);   /* 轨道高度*/
    --radius: calc(infinity * 1px); /* 无穷大 */
    
    appearance: none;
    border: none;
    
    inline-size: 100%;
    block-size: var(--track-size);
    border-radius: var(--radius);
    position: relative;
    overflow: hidden;
}
```

  


在这里，我定义了两个 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)，分别设置进度条轨道的高度 `--track-size` 和圆角半径 `--radius` 。`--track-size` 属性的值使用 [CSS 的比较函数 `min()`](https://juejin.cn/book/7223230325122400288/section/7241401565653762108) ，它会在 `10px` 和 `1ex` 中取更小的值，其中 `ex` 是 [CSS 字体相对单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-3)中一个，它相以于 `x` 字母的高度计算：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3ebd05f31f6d42f9b98776d7c90b0323~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=954&h=522&s=46059&e=png&b=2b2b2b)

  


`--radius` 的值为 `calc(infinity * 1px)` ，这是一个无穷大的值，以便进度条始终有一个类似胶囊形状，这是 [CSS `border-radius` 的一个小技巧](https://juejin.cn/book/7199571709102391328/section/7199845563389444099)。

  


[`inline-size` 和 `block-size` 是 CSS 的逻辑属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)，当书写模式为 `ltr` 或 `rtl` 时，`inline-size` 等同于 `width` ，`block-size` 等同于 `height` 。这里选择 CSS 逻辑属性，除了易于排版之外，那就是多个主流浏览器在设置进度条的尺寸时，采用的也是逻辑属性 `inline-size` 和 `block-size` 。

  


[`overflow: hidden` 是一个有争议的样式](https://juejin.cn/book/7199571709102391328/section/7213705145954074679)。它使一些事情变得容易，例如不需要将 `border-radius` 值传递到进度条的进度（Progress）元素，但也意味着进度条的子元素不能溢出该元素。

  


现在，你看到的效果大致如下，它能自动匹配 `label` 元素的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3be99afb5e3f49aa908c3473134fa11a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1260&h=492&s=819530&e=gif&f=211&b=1e5b6d)

  


### 给进度条着色

  


使用 CSS 给 `<progress>` 元素着色有多种方式。最简单的方式就是使用 `accent-color` ，它允许你仅写一行 CSS 代码，就可以将你的品牌色运用于 `<progress>` 元素，使你免于自定义工作。[`accent-color` 是 CSS 的一个新特性](https://juejin.cn/book/7223230325122400288/section/7238477651873103927#heading-0)，可以用来设置有单控件的高亮色（即着重色），并且可以让支持的终端达到一致的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/be8ae1994e1d4269a1744a21a7d5f046~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=882&h=486&s=831778&e=gif&f=177&b=fdeef3)

  


> Demo 地址：https://codepen.io/airen/full/JjmVQZv

  


```CSS
:root {
    --brand-color: deeppink;
    accent-color: var(--brand-color);
    
    @media (prefers-color-scheme: dark) {
         --brand-color: hsl(328 100% 65%);
    }
}

progress {
    color-scheme: light dark;
}
```

  


上面代码中，还使用了 [`color-scheme` 属性](https://juejin.cn/book/7223230325122400288/section/7238477651873103927#heading-4)使你的网站支持适应暗黑模式（DarkMode）的 `<progress>` 元素。同时还使用[媒体查询 `@media` 的 `prefers-color-scheme` 查询条件](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-18)，使你的 `<progress>` 元素还能根据用户的喜好来设置进度条的颜色。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/caf91de1d52f4e47b71f0d75eab4ebb8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=536&s=267574&e=gif&f=24&b=1e5b6d)

  


注意，`<progress>` 元素的背景颜色会根据 `accent-color` 从浅色变为深色。浏览器会确保适当的对比度：非常巧妙。

  


上图所展示的效果，你可能会有疑问，你不是说 `accent-color` 就可以调整 `<progress>` 进度条的颜色？那么，为什么第一个进度条没有任何变化呢？这个很好解释，因为我们开始给它写样式的时候，就使用了 `appearance: none` 禁用了系统默认的样式。简单地说，这是因为我们在自定义进度条样式，因此进度条的轨道颜色和进度颜色，我们都需要重新定义。

  


我们可以在 `<progress>` 元素上设置另外两个自定义属性，一个用于轨道颜色（比如 `--track-color`），另一个用于进度颜色（比如 `--progress-color`）。并在 `prefers-color-scheme` 媒体查询中，为 `--track-color` 和 `--progress` 提供新的颜色值。

  


```CSS
progress[value] {
    /* 亮色模式下 */
    --track-color: hsl(228 100% 90%);    /* 轨道颜色 */
    --progress-color: hsl(228 100% 50%); /* 进度颜色 */

    /* 暗色模式下 */
    @media (prefers-color-scheme: dark) {
        --track-color: hsl(228 20% 30%);     /* 轨道颜色 */
        --progress-color: hsl(228 100% 75%); /* 进度颜色 */
    }
}
```

  


定义好颜色之后，就可以通过选择每个浏览器暴露的 `<progress>` 元素部分来自这义样式。使用 `<progress>` 元素是一个单一标签，但它由几个通过 CSS 伪选择器暴露的子元素组成。对于 Firefox 浏览器，它仅暴露了 `<progress>` 元素上的 `::-moz-progress-bar` 伪选择器。这意味着，我们需要在 `progress` 上设置轨道颜色，然后在 `progress::-moz-progress-bar` 伪选择器设置进度颜色。

  


```CSS
progress[value] {
    /* 亮色模式下 */
    --track-color: hsl(228 100% 90%);    /* 轨道颜色 */
    --progress-color: hsl(228 100% 50%); /* 进度颜色 */

    /* 暗色模式下 */
    @media (prefers-color-scheme: dark) {
        --track-color: hsl(228 20% 30%);     /* 轨道颜色 */
        --progress-color: hsl(228 100% 75%); /* 进度颜色 */
    }
    
    background: var(--track-color); /* Firefox 进度条轨道着色 */
    
    
    &::-moz-progress-bar {
        border-radius: var(--radius);
        background: var(--progress-color); /* Firefox 进度条着色 */
    }
}
```

  


注意，`progress[value]::-moz-progress-bar` 上显式设置 `border-radius` 属性的值为 `var(--radius)` ，这是使进度条的进度条也具有圆角。但要注意，不能直接将其值设置 `inherit` ，在 Shadow DOM 中的元素无法直接继承 `<progress>` 的 `border-radius` 的值。

  


这个时候，你在 Firefox 浏览器看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d280b16aaad149e0bf390b9b2eb28854~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=696&s=1325590&e=gif&f=133&b=1e5b6d)

```CSS

    &::-webkit-progress-bar {
        border-radius: var(--radius);
        background: var(--track-color);
    }
    
    /* Safari、Chrome 和 Edge 浏览器进度着色 */
    &::-webkit-progress-value {
        border-radius: var(--radius);
        background: var(--progress-color);
    }
}
```

基于一个 `<progress>` 元素，使用 CSS 来创建圆形进度条，其核心是 [CSS 的锥形渐变 `conic-gradient`](https://juejin.cn/book/7223230325122400288/section/7259668771856941111) 和[遮罩 `mask`](https://juejin.cn/book/7223230325122400288/section/7259668885224456252) 或者使用一个锥形渐变 `conic-gradient` 加一个径向渐变 `radial-gradient`。但它们的原理都是一样的，首先使用 `conic-gradient` 绘制一个饼图，然后使用 `mask` 或 `radial-gradient` 遮盖饼图的中间部分，就完成了一个圆形进度条的绘制。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/98840de5c874458b9c9c1c237d98642c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=924&s=229079&e=jpg&b=fbf7f7)

  


我在这里选择 `conic-gradient` 和 `mask` 结合的方式，其中遮罩图同样也是使用 `radial-gradient` 来绘制的。使用这种方式制作圆形进度条，首先我们需要引入一个新的 CSS 自定义属性 `--p` ，它的值等于 `<progress>` 元素的 `value` 值，用于表示圆形进度条的进度，相当 `conic-gradient` 绘制的饼图的占比：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad2475f56cc946f09ff70abe620ac9db~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=924&s=225231&e=jpg&b=faf5f5)

  


另外，与水平进度条一样，我们分别使用 `--progress-color` 和 `--track-color` 来定义进度条和轨道的颜色； `--track-size` 定义进度条的大小。不同的是，进度条的 `inline-size` 和 `block-size` 相等，因为我们要绘制一个圆形：

  


```CSS
progress[value] {
    --p: 85;
    --track-size: 50vmin;
    --color-set: #001219,#005f73,#0a9396,#94d2bd,#e9d8a6,#ee9b00,#ca6702,#bb3e03,#ae2012,#9b2226;
    --contrast-target: AA;
    --progress-color: #fd5108; 
    --track-color: color-contrast(var(--progress-color) vs var(--color-set) to var(--contrast-target));
    
    /* 重置浏览器默认样式 */
    appearance: none;
    border: none;
    
    /* 设置进度条尺寸 */
    inline-size: var(--track-size);
    block-size: var(--track-size);
    border-radius: 50%;
    
    overflow: hidden;
    position: relative;
}
```

  


你可能注意到了，我在这里并没有使用 `aspect-ratio: 1` 来设置 `<progress>` 元素的宽高相等，这是因为浏览器在定义`<progress>` 元素尺寸时，显式的使用了 `inline-size` 和 `block-size` （或 `width` 和 `height`），这个时候再使用 `aspect-ratio:1` 并不会生效。具体原因就不在这里阐述了，如果你想一探其中奥秘，[请点击这里获取更多相关的信息](https://juejin.cn/book/7223230325122400288/section/7259316040151236666)。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eb0698e27a4b49b69e2d66e422a163b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1028&s=193761&e=jpg&b=216f78)

  


到目前为止，我们获得了一个圆形的进度条。上图中绿色和灰色分别是浏览器给轨道和进度条设置的默认颜色。接下来，我们要做的是，使用浏览器暴露出来的伪元素，将默认的进度条颜色设置为透明色：

  


```CSS
progress[value] {
    /* 其他 CSS */
    
    
    /* Webkit 内核浏览器 （Safari、Chrome 和 Edge）*/
    &::-webkit-progress-value {
        background: #0000; /* 设置进度条的颜色为透明色 */
    }

    /* Firefox */
    &::-moz-progress-bar {
        background: #0000; /* 设置进度条的颜色为透明色 */
    }
}
```

  


现在你只能看到进度条轨道的颜色：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/196431cbb56d48f8b99af6ad2bfecb8a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1028&s=201671&e=jpg&b=246677)

  


需要知道的是，我们不能直接将进度条轨道的颜色也设置为透明色，因为我们直接在 `<progress>` 元素上设置 `background` 时，在 Webkit 内核的浏览器中并不会生效。因此，我们还是要分别在 `progress` 和 `progress::-webkit-progress-bar` 进行设置 `background` 。而且 `background` 并不是一个纯色，它是 `conic-gradient` 绘制的一个渐变色，这个渐变色是由 `--progress-color` （进度）和 `--track-color` （轨道）组成：

  


```CSS
progress[value] {
    --p: 85;
    --track-size: 50vmin;
    --color-set: #001219,#005f73,#0a9396,#94d2bd,#e9d8a6,#ee9b00,#ca6702,#bb3e03,#ae2012,#9b2226;
    --contrast-target: AA;
    --progress-color: #fd5108; 
    --track-color: color-contrast(var(--progress-color) vs var(--color-set) to var(--contrast-target));
    
    --g: conic-gradient(var(--progress-color) calc(var(--p) * 1%),var(--track-color) 0);
    
    background: var(--g); /* Firefox 浏览器 */
    
    &::-webkit-progress-bar {
        background: var(--g);
    } 
}
```

  


这个时候，你看到的效果如上：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9c9e9fcce1a84ea590636401df502393~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1206&h=714&s=11649659&e=gif&f=471&b=1e5b6d)

  


正如你所看到的，[使用 `conic-gradient` 绘制了一个饼图](https://juejin.cn/book/7223230325122400288/section/7259668771856941111#heading-5)。这离我们所需要的圆形进度条只有一步之遥了，只需要使用 `mask` 将 `conic-gradient` 绘制的饼图镂空，就得到了我们所需要的圆形进度条：

    --track-size: 50vmin;

    background: var(--g); /* Firefox 浏览器 */
    
    mask: radial-gradient(#0000 59%,#000 60% 70%,#0000 0);
    
    &::-webkit-progress-bar {
        background: var(--g);
    } 
}
```

progress[value] {

    --progress-color: #fd5108;
    --track-color: color-contrast(var(--progress-color) vs var(--color-set) to var(--contrast-target));
    --g: conic-gradient(var(--progress-color) calc(var(--p) * 1%),var(--track-color) 0);
    appearance: none;
    border: none;
    display: block;
    margin: 0 auto;
    inline-size: var(--track-size);
    block-size: var(--track-size);
    position: relative;
    overflow: hidden;

    border-radius: 50%;
    background: var(--g);
    mask: radial-gradient(#0000 59%, #000 60% 70%, #0000 0);

    &:focus-visible {

    }

    &::-webkit-progress-bar {
        background: var(--g);
    }

    &::-webkit-progress-value {
        background: #0000;
    }

    }

    }

```

  


注意，我在这里[使用 `@supports` 条件规则](https://juejin.cn/book/7223230325122400288/section/7258822669154877480)，为不支持 `color-contrast()` 的浏览器做了一个降级处理，将 `--track-color` 设置为 `#000` 。

  


除此之外，调整一下 `conic-gradient` 的值，就可以获得一个速有渐变效果的圆形进度条：

  


```CSS
progress[value] {
    /* 其他代码相同 */
    --g:conic-gradient(
        from 0deg at 50% 50%,
        var(--progress-color) , color-mix(in hsl, var(--progress-color), var(--track-color) 20%) calc(var(--p) * 1%),
        var(--track-color) calc(var(--p) * 1%)
    );
}
```

> Demo 地址：https://codepen.io/airen/full/GRbdbLx

  


在这个示例中，进度条的颜色只是从 `--progress-color` 过渡到 `--progress-color` 与 `20%` 的 `--track-color` 的混合色（`color-mix(in hsl, var(--progress-color), var(--track-color) 20%)`）。注意，[这个混合色是通过 `color-mix()` 函数获得的](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)。你可以根据实际需求来调整这个渐变颜色。

  


或许你更喜欢的是类似下图这种进度条效果，进度条两端是圆形的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d2e34659f91a432eaa322f3a942b52e8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1707&h=960&s=1276145&e=png&b=f7ecdd)

  


实现类似上图这样的带有圆角的圆形进度条要比之前的稍微复杂一点，你需要在 `conic-gradient` 的基础上新增两个 `radial-gradient` ，这两个径向渐变绘制的圆要与进度条的边框厚度相等，并且它们分别位于 `conic-gradient` 绘制的进度条的起点和终点。我使用下图来向大家展示它们，这样会更清晰一些：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d095b6d089194d54a95ede3e6ca2be61~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1541&s=350646&e=jpg&b=f9f3f3)

  


在这里，我将新增一个自定义属性 `--b` ，用它来定义进度条的厚度，同时也是用它来定义径向渐变绘制圆的大小。使用径向渐变 `radial-gradient` 绘制一个圆形，它并不复杂：

  


```CSS
.circle {
    --b: 10%;
    background: radial-gradient(farthest-side,var(--c,lime) 98%,#0000) top/var(--b) var(--b) no-repeat;
}
```

  


上面的代码绘制了一个大小为 `10%` 的绿色（`lime`）圆。我们可以按类似的方式绘制两个圆，并且与 `conic-gradient` 绘制的饼图组合在一起：

  


```CSS
progress[value] {
    /* 其他代码相同 */
    --b: 10%;
    --g:
      radial-gradient(farthest-side,var(--c,lime) 98%,#0000) top/var(--b) var(--b) no-repeat,
      radial-gradient(farthest-side,var(--c,lime) 98%,#0000) calc(50% + 50% * cos(360deg * var(--p) / 100 - 90deg)) calc(50% + 50% * sin(360deg * var(--p) / 100 - 90deg))/var(--b) var(--b) no-repeat,
      conic-gradient(var(--progress-color) calc(var(--p) * 1%),var(--track-color) calc(var(--p) * 1%));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f588c456c78d4738a496c67c41946568~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=738&s=1912223&e=gif&f=259&b=1e5b6d)

  


正如你所看到的，第一个圆它始终固定在 `conic-gradient` 绘制的饼图的初始位置（即 `conic-gradient` 渐变 `from` 关键词后面指定的起始角度，在这里是 `0deg`）；第二个圆的位置会随着 `<progress>` 的 `value` 值变化，但它始终在 `conic-gradient` 绘制的饼图的终点位置。但要知道的是，`conic-gradient` 绘制的饼图的终点位置也是由 `<progress>` 元素的 `value` 值决定，也就是 `--p` 的值决定。

  


也就是说，现在的关键是，我们要知道第二个圆在大圆圆周上的位置（这个大圆可以理解为 `conic-gradient` 绘制的饼图所在圆，也可以理解为 `<progress>` 元素，因为它经过 CSS 处理之后也是一个圆）。我们可以通过一些几何知识，例如[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)来计算出它在圆周上的位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8747c5eca78b414391136d42ac6f163e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1541&s=377116&e=jpg&b=f7f0f0)

  


```
posX = calc(50% + 50% * cos(360deg * var(--p) / 100 - 90deg)) 
posY = calc(50% + 50% * sin(360deg * var(--p) / 100 - 90deg)
```

  


注意，`cos(α)` 和 `sin(α)` 函数中的角度值 `α` 可以根据 `<progress>` 元素的 `value` 值来计算，它的最大值是 `100` ，对应的就是 `360deg` 。我们可以根据 `360deg / 100 * var(--p) - 90deg` 计算出 `α` 的值。另外需要知道的是，公式中的 `50%` 相当于圆的半径，即元素宽度的一半，再加上 `50%` 可以从 **[极坐标系](https://link.juejin.cn/?target=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FPolar_coordinate_system)** 转换到了 **[笛卡尔坐标系](https://link.juejin.cn/?target=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FCartesian_coordinate_system)**，也就是我们 Web 坐标系中描述点的位置。

  


另外，`radial-gradient` 绘制的圆，我们是通过 `background-size: var(--b) var(--b)` 来控制其大小的。与此同时，`mask` 的遮罩图也需要随着 `--b` 进行调整：

  


```CSS
progress[value] {
    --b: 10%;
    --g:
      radial-gradient(farthest-side,var(--c,lime) 98%,#0000) top/var(--b) var(--b) no-repeat,
      radial-gradient(farthest-side,var(--c,lime) 98%,#0000) calc(50% + 50% * cos(360deg * var(--p) / 100 - 90deg)) calc(50% + 50% * sin(360deg * var(--p) / 100 - 90deg))/var(--b) var(--b) no-repeat,
      conic-gradient(var(--progress-color) calc(var(--p) * 1%),var(--track-color) calc(var(--p) * 1%));
    mask: radial-gradient(50% 50%,#0000 calc(100% - var(--b) * 2 - 1%),#000 calc(100% - var(--b) * 2) 100%,#0000 102%);   
}
```

  


这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fb03be298a4944629518b37b6d819b6d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=722&s=3559319&e=gif&f=382&b=1e5b6d)

## 动画化进度条

  


之前我们所展示的进度条，不管是水平进度条还是圆形进度条，都没有显示进度文本。我想大家可能发现了，即使我们在 `<progress>` 元素内包裹了进度文本，例如 `85%` ，但它并不会在浏览器中渲染出来。因此，我们需要另外处理。

  


接下来，我们以圆形进度条为例，我将使用伪元素 `::before` 给进度条添加进度文本，并给它们添加一些动画效果。关键 CSS 代码如下：

  


```HTML
<label class="progress--container">
    <span class="sr-only">加载进度</span>
    <progress max="100" value="85" tabindex="-1" role="progressbar">85%</progress>
</label>
```

  


```CSS
@property --p {
    syntax: "<integer>";
    initial-value: 0;
    inherits: true;
}

@keyframes progress-text {
    to {
        --p: 85;
    }
}

.progress--container {
      display: block;
      position: relative;
      
      &::before {
          counter-reset: num var(--p);
          content: counter(num) "%";
          display: grid;
          place-content: center;
          position: absolute;
          inset: 0;
          font-size: clamp(2rem, 3vmin, 3rem);
          animation: progress-text 1s ease-in forwards;
    }
    
    progress[value] {
        /* 省略其他代码 */
        animation: progress-text 1s ease-in forwards;
    }
}
```

  


上面的代码中，[使用了 CSS 的 `@property` 注册了一个名为 `--p` 的变量](https://juejin.cn/book/7223230325122400288/section/7258870477462962236)，并指定了它的数据类型以及初始值等，并且[在 `@keyframes` 的 `to` 选择器中指定 `--p` 的值为 `<progress>` 元素的 `value` 值](https://juejin.cn/book/7288940354408022074/section/7295617447058407474#heading-15)。这样做的主要原因有两点：

  


-   **[动态计数](https://juejin.cn/book/7223230325122400288/section/7258870477462962236#heading-6)**：使用 `counter() 以及伪元素的 content 将数字转换成字符串`，并使用这些字符串作为伪元素 `content` 的值，例如 `content: counter(num) "%"` ，其中 `num` 是使用 `counter-reset: num var(--p)` 定义的
-   **[`@keyframes 中的插值计算`](https://juejin.cn/book/7288940354408022074/section/7295617447058407474#heading-15)**：浏览器在 [`@keyframes`](https://juejin.cn/book/7288940354408022074/section/7295617447058407474#heading-15) 中对常规的 CSS 自定义属性的值做插值计算。因此需要使用 `@property` 告诉浏览器注册的 CSS 自定义属性的值类型，例如示例中的 `--p` 。它会告诉浏览器 `--p` 只支持整数（`<integer>`）

我们还可以与 [CSS 的滚动驱动动画](https://juejin.cn/book/7288940354408022074/section/7307223031717724172)结合起来，将上面这个圆形进度条当作为一个阅读指示器。我们只需在上面示例中设置了 `animation` 的地方引用一个 `animation-timeline: scroll(nearest block)` ，它就能和离它最近的滚动容器绑定在一起，通过滚动来驱动动画，而不是像上一个示例那样，自动播放。关键代码如下：

  


```CSS
@property --p {
    syntax: "<integer>";
    initial-value: 0;
    inherits: true;
}

@keyframes progress-text {
    to {
        --p: 85;
    }
}

.progress--container {
      display: block;
      position: relative;
      
      &::before {
          /* 省略其他代码 */
          animation: progress-text 1s ease-in forwards;
          animation-timeline: scroll(nearest block);
    }
    
    progress[value] {
        /* 省略其他代码 */
        animation: progress-text 1s ease-in forwards;
        animation-timeline: scroll(nearest block);
    }
}
```

上面示例的方法，同样适用于水平进度条，感兴趣的同学可以尝试一下！但需要知道的是，在 `::-webkit-progress-value` 和 `::-moz-progress-bar` 伪元素中无法直接使用 `animation` 引用`@keyframes` 定义的动画。换句话说，如果需要给水平进度条添加动画效果，它会比较棘手一些，你需要对 Shadow DOM 相关的有所了解。但我们可以另辟蹊径，采用别的方法来实现。例如：

  


```HTML
<label class="progress--container" style="--p: 25; --c1:#d30000; --c2:#a9009f;">
    <span class="sr-only">加载进度</span>
    <progress max="100" value="25" tabindex="-1" role="progressbar" class="sr-only">25%</progress>
</label>
```

  


我们在 `label` 添加了三个自定义属性，其中 `--p` 用来表示进度条的进度，它的值与 `<progress>` 元素的 `value` 值相等，而 `--c1` 和 `--c2` 主要用来制作斑马线渐变效果。另外，在 `<progress>` 添加了 `sr-only` 类名，并通过下面的 CSS 代码，将其在视觉上隐藏：

  


```CSS
.sr-only {
    clip: rect(1px, 1px, 1px, 1px);
    clip-path: inset(50%);
    height: 1px;
    width: 1px;
    margin: -1px;
    overflow: hidden;
    padding: 0;
    position: absolute;
}
```

  


接着，我们在 `.progress--container` 元素上定义进度条轨道的样式，并在它的伪元素 `::before` 定义进度条的样式：

  


```CSS
.progress--container {
    --track-size: min(1em, 2ex);
    --radius: calc(infinity * 1px);
    --w: 50vw;
    --track-color: linear-gradient(to bottom, #141414, #343434);
    --c1: #d30000;
    --c2: #a9009f;
    --progress-color: linear-gradient(
          45deg,
          rgb(255 255 255 / 0.15) 25%,
          #0000 25%,
          #0000 50%,
          rgb(255 255 255 / 0.15) 50%,
          rgb(255 255 255 / 0.15) 75%,
          #0000 75%,
          #0000
        )
        0 0 / 1rem 1rem,
      linear-gradient(to right, var(--c1), var(--c2));

    display: block;
    inline-size: var(--w);
    block-size: var(--track-size);
    border-radius: var(--radius);
    position: relative;
    overflow: hidden;
    background: var(--track-color);

    &:focus-visible {
        outline-offset: 5px;
        outline-color: var(--progress-color);
    }

    &::before {
        content: "";
        position: absolute;
        inset: 0;
        border-radius: inherit;
        background: var(--progress-color);
        inline-size: calc(var(--p) * 1%);
    }
}
```

  


这跟之前直接在 `<progress>` 元素设置进度条样式几乎无差别。

  


设置好进度条样式之后，我们就可以使用 `@keyframes` 和 `animation` 给它添加动画效果：

  


```CSS
@keyframes stripes {
    from {
        background-position: 15px 0, 0 0;
    }
    to {
        background-position: 0 0, 0 0;
    }
}
  
@keyframes scale {
    from {
        scale: 0 1;
    }
    to {
        scale: 1 1;
    }
}

@property --p {
    syntax: "<integer>";
    initial-value: 0;
    inherits: true;
}

.progress--container {
    /* 其他 CSS */
    
    &::before {
        /* 其他 CSS */
        animation: scale 1s linear forwards,stripes 1s linear infinite;
        transform-origin: left center;
    }
}
```
