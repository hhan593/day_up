## 我们的目标

  


我们将使用 CSS 实现一个在电商网站中非常常见的效果：图片放大（放大查看）效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ef5f1b30371142d8b90cd17512d6f62b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=642&s=17321347&e=gif&f=200&b=fbf3f2)

  


> Demo 地址：https://codepen.io/airen/full/wBBOroO

  


这个缩放效果有两个关键点：**一是图片变大，二是显示它的一部分**。核心思想是：在放大图片的同时，只显示它原始区域的一部分。为此，我们通常会使用 CSS 的 `scale` 来放大图片，而隐藏部分内容则有几种实现方式，比如 CSS 的 `overflow:hidden` 或 `clip-path` 。

  


## CSS 如何缩放图片

  


刚才提到过了，在 CSS 中，使用 `scale` 可以对图片进行缩放，比如：

  


```HTML
<img src="https://picsum.photos/600/800?radom=154" alt="" />
```

  


```CSS
img {
    --x: 50%;
    --y: 50%;
    --s: 50vmin;
    scale: var(--zoom);
    transform-origin: var(--x) var(--y);
    width: var(--s);
    aspect-ratio:1;
    cursor: pointer;
    transition: .4s;
    
    &:hover {
        --zoom: 1.2;
    }
 }
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1b842d93010c4b1d9fa411beb3894c7a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1338&h=644&s=6587966&e=gif&f=75&b=fffaf9)

  


> Demo 地址：https://codepen.io/airen/full/wBBOPJG

  


不难发现，虽然图片放大了，但依旧能看到图片所有区域。这与我们所期望的效果中第二部分不符——**只显示它的一部分**！

  


## 使用 `overflow: hidden` 隐藏部分内容

  


最为常见的办法是使用 `overflow` 的 `hidden` 或 `clip` 来隐藏部分内容。不过，使用这种方式，我们需要将 `img` 放置在一个容器中，比如：

  


```HTML
<figure>
    <img src="https://picsum.photos/600/800?radom=154" alt="" />
</figure>
```

  


```CSS
figure {
    display: inline-grid;
    overflow: hidden; /* 或 clip */
    
    img {
        --x: 50%;
        --y: 50%;
        --s: 50vmin;
        scale: var(--zoom);
        transform-origin: var(--x) var(--y);
        width: var(--s);
        aspect-ratio:1;
        cursor: pointer;
        transition: .4s;
        
        &:hover {
            --zoom: 1.2;
        }
     }
}
```

  


这段代码非常简单，我们在 `figure` 容器上设置了 `overflow: hidden` ，并使用 `scale` 对图片进行缩放。有一点你可能会感到疑惑，容器未设置具体的尺寸，怎么就能对溢出的内容进行裁剪呢？这是因为，我们在 `figure` 容器上设置了 `display: inline-grid` 。它会让容器的宽度自动适应图片的宽度，并且可以避免图片底部出现空白间距。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8ee529f62ff45a9a13b9a502d28732a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=644&s=6349617&e=gif&f=96&b=fbf7f6)

  


> Demo 地址：https://codepen.io/airen/full/JoozOye

  


需要知道的是，将 `overflow` 的值设置为 `clip` ，能实现同样的效果，不过 `overflow` 的 `hidden` 和 `clip` 还是有所不同的，如果你想进一步了解它们之间的差异，请移步阅读《[溢出常见问题与排查](https://juejin.cn/book/7199571709102391328/section/7213705145954074679)》和《[Web UI：你知道 `overflow: clip` 吗？](https://juejin.cn/book/7395119173656903715/section/7496427019152932864)》！

  


不知道你留意到没有，示例中的 `transform-origin` 的值是两个 CSS 变量：`--x` 和 `--y` ，可以通过它们控制缩放的中心点。换句话说，图片会围绕着这个点进行缩放。这样做的好处是，我们通过一段简单的 JavaScript ，可以使这个点跟随鼠标移动，动态改变缩放的中心点：

  


```JavaScript
let img = document.querySelector("img");

img.onmousemove = function(e) {
    e.target.style.setProperty('--x',(100*e.offsetX/e.target.offsetWidth)+'%');
    e.target.style.setProperty('--y',(100*e.offsetY/e.target.offsetHeight)+'%'); 
}
```

  


这样就实现了我们所期望的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eed33142c6664b4e94527c2b9b8858df~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=640&s=15896119&e=gif&f=179&b=fbf5f4)

  


> Demo 地址：https://codepen.io/airen/full/RNNdjyN

  


## 使用 `clip-path` 隐藏部分内容

  


如果你想使用额外的容器，也是可以的。因为我们的目标是：**在图片被缩放后隐藏一部分内容**。我们可以使用 `clip-path` 或 `mask` 来实现。不过，在我们这个案例中，使用 `clip-path` 会更合适——通过 `inset()` 定义一个“内缩矩形”，它的作用是从四边向内裁剪元素。

  


我们先来看一个简单的示例，仅是图片缩放的效果：

  


```CSS
img {
    --s: 50vmin;
    width: var(--s);
    aspect-ratio: 1;
    --d: 1.5em;
    box-sizing: border-box;
    cursor: pointer;
    padding: calc((1 - var(--hov, 0)) * var(--d));
    object-fit: cover;

    clip-path: inset(var(--d));
    transition: padding 0.4s;

    &:hover {
        --hov: 1;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dc599bbe9b7f4ae09719f12caa1ad1dc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1328&h=642&s=7062056&e=gif&f=121&b=fffafa)

> Demo 地址：https://codepen.io/airen/full/QwwoOPO

  


在这个示例中，我们并没有使用 `scale` ，图片怎么就有缩放效果呢？这是因为，我们利用 CSS 变量和 `clip-path` 的结合，实现了图片初始状态有内距裁剪（看起来缩小了），鼠标悬停时，内距逐渐消失，图片边缘被显示出来的视觉“放大”效果：

  


-   **初始状态：** 图片外围有 `padding: 1.5em`，而 `clip-path` 会裁剪图片四边 `1.5em`，这意味着图片看起来像“缩小”了一点，四周有空白的效果（`padding` 和 `clip-path`结合起来创造这种视觉）
-   **悬停状态：** 当鼠标悬停时，`padding` 动态变为 `0`，图片的边缘“扩展”到容器边界，`padding`消失。由于裁剪区域固定，图片边缘从裁剪区域外露出来，视觉上像是图片放大或突出出来

  


核心思想就是：**`clip-path: inset()`** **固定裁剪区域；** **`padding`** **动态变化控制可视范围**：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cb0e460a8ce942df828982c6412f1bdc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3669&h=2592&s=1051964&e=jpg&b=ffffff)

  


注意，我们还使用了 `object-fit: cover;` ，它能保证图片内容按照比例裁剪填充容器，不变形，也不会有空白。

  


需要提出来的是，如果你只是希望基于一个 `img` 元素实现图片缩放效果，那么这个技巧是一个不错的选择。但要实现我们的目标效果，这种方式还是无法完成，目标效果需要动态调整缩放中心点。要实现这个效果，不得不依赖 CSS 的变换来实现—— `scale` 和 `transform-origin` 。

  


由于 `scale` 和 `transform-origin` 的加入，`clip-path` 的 `inset()` 会变得稍微复杂一点：

  


```CSS
img {
    --x: 50%;
    --y: 50%;
    --s: 50vmin;
    scale: var(--zoom);
    transform-origin: var(--x) var(--y);
    clip-path: inset(/* ❓❓❓ */);
    width: var(--s);
    aspect-ratio: 1;
    cursor: grab;
    transition: all .5s;
    
    &:hover {
        --zoom: 2;
    }
}
```

  


在没有应用 `clip-path` 的时候，图片会围绕着缩放中心进行缩放：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/42da2b4f15a7452b9b234979534115a3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1330&h=642&s=7869623&e=gif&f=65&b=fef7f6)

  


接下来的重点就是如何计算 `clip-path` 属性的 `inset()` 函数的参数。它的计算依赖着其他变量，比如 `--x` ，`--y` 和 `--zoom` 。大家可以通过下图来理解 `inset()` 参数的值的计算：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9bfa0422276f4c6bb86f7035318c9302~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3669&h=3120&s=1228563&e=jpg&b=ffffff)

  


上图中黄色的点表示 `transform-origin` 的位置（即缩放中心）。注意，应用 `scale` 之后，这个点位置于红色矩形的中心。我们的目标是使用 `inset()` 隐藏红色矩形之外的部分，因此 `inset()` 的值必须定义一个与红色矩形位置和尺寸相同的裁剪区域。

  


我们在图中展示了 `inset()` 中 `left` 值的计算方式：

  


-   `x` 是我们通过 `--x` 定义的缩放中心的 `x` 轴坐标
-   `x'` 是该坐标在缩放后的位置
-   它们的差值就是我们要找到的裁剪偏移量

  


需要注意的一点是，我们需要用 `x'` 与 `x` 的差值（`x' - x` ）除以 `zoom` （缩放倍数），这是因为图片被放大了，要通过除法来得到实际坐标，而不是放大后的值。其它几个方向（`top` 、`right` 和 `bottom`）的值采用同样的逻辑计算，这样就计算出了 `inset()` 函数的值：

  


```CSS
img {
    clip-path: inset(
        calc((1 - 1/var(--zoom)) * var(--y))          /* top */
        calc((1 - 1/var(--zoom)) * (100% - var(--x))) /* right */
        calc((1 - 1/var(--zoom)) * (100% - var(--y))) /* bottom */
        calc((1 - 1/var(--zoom)) * var(--x))          /* left */
    );
}
```

  


完整的 CSS 代码如下：

  


```CSS
img {
    --x: 50%;
    --y: 50%;
    --s: 50vmin;
    scale: var(--zoom);
    transform-origin: var(--x) var(--y);
    clip-path: inset(
        calc((1 - 1 / var(--zoom)) * var(--y)) /* top */
        calc((1 - 1 / var(--zoom)) * (100% - var(--x))) /* right */
        calc((1 - 1 / var(--zoom)) * (100% - var(--y))) /* bottom */
        calc((1 - 1 / var(--zoom)) * var(--x)) /* left */
    );
    width: var(--s);
    aspect-ratio: 1;
    cursor: grab;
    transition: .5s;
    
    &:hover {
      --zoom: 2;
    }
}
```

  


现在，我们只需要修改 `--x` 、`--y` 和 `--zoom` 几个变量的值，就可以灵活控制缩放效果，而不需要改变 `inset()` 的具体数值。比如，我们通过 JavaScript 动态调整 `--x` 和 `--y` 的值，就获得了我们最终所期望的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/657b797dd9e148eaa14313a0e75cfb4e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=644&s=7644349&e=gif&f=87&b=fdf8f7)
