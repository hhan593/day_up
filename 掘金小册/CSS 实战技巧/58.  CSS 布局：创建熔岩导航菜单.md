## 熔岩导航菜单简介

  


熔岩导航菜单与普通的导航菜单相比，它有着明显的特征，它具有一个“熔岩块”，这个熔岩块通常是一个高亮块（高亮条）。熔岩块（或条）会随着用户的鼠标悬停或点击，平滑地移动并调整大小，以突出当前选中的菜单项：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ea84614f4c5b4c8199c210b35f1e4aa9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=680&s=89041&e=gif&f=104&b=2c3d50)

  


熔岩块（或熔岩条）的滑动动画效果通常使用 CSS 动画（`animation` 和 `@keyframes`）或过渡动画（`transition`）来实现，使熔岩块（或熔岩条）在菜单项之间移动时非常平滑、自然。

  


正如你所看到的，熔岩导航菜单的交互性非常强。用户的鼠标悬浮时，熔岩块（或熔岩条）会跟随鼠标移动，突出当前悬浮的菜单项；用户点击某个菜单项后，熔岩块（或熔岩条）会停留在该菜单项上，表示该菜单项为当前选中项。而且，熔岩块（或熔岩条）通常使用渐变色、模糊边缘、阴影等视觉效果，模拟熔岩或液体流动的感觉。这种效果增强了页面的现代感和互动感。

  


由于熔岩导航菜单有一个熔岩块（或熔岩条），因此在构建熔岩导航菜单时，通常会使用一个额外的 HTML 标签元素当作熔岩块（或熔岩条）：

  


```HTML
<div class="menu">
    <ul>
        <li class="active">Home</li>
        <li>About</li>
        <li>Services</li>
        <li>Contact</li>
    </ul>
    <div class="lava"></div>
</div>
```

  


上面代码中的 `.lava` 就是用来制作熔岩块（或熔岩条）的。当然，有时候也可以使用伪元素 `::before` 或 `::after` 来替代。

  


## 创建熔岩导航菜单

  


熔岩导航菜单一般是通过组合 HTML、CSS 和 JavaScript 来实现。HTML 用于定义菜单项，CSS 用于创建熔岩块（或熔岩条）的视觉效果和动画，JavaScript 用于处理用户交互（如鼠标悬浮或点击）。

  


```HTML
<div class="menu">
    <ul>
        <li class="active"><a href="#">Home</a></li>
        <!-- 省略其他 li -->
    </ul>
    <div class="lava"></div>
</div>
```

  


```CSS
.menu {
    position: relative;

    ul {
        display: flex;
        list-style: none;
        padding: 0;
        margin: 0;
    
        li {
            padding: 10px 20px;
            cursor: pointer;
            position: relative;
            
            &.active {
                color: #ff6a00;
            }
        }
    }
}

.lava {
    position: absolute;
    bottom: 0;
    left: 0;
    height: 5px;
    background: linear-gradient(90deg, #ff6a00, #ffbb00);
    border-radius: 5px;
    transform-origin: left center;
    will-change: transform;
    transition: transform 0.2s ease-out, width 0.2s ease-out;
}
```

  


```JavaScript
const menu = document.querySelector('.menu');
const lava = document.querySelector('.lava');
const items = document.querySelectorAll('.menu li');

// 初始化熔岩块位置
const activeItem = document.querySelector('.menu li.active');
lava.style.width = `${activeItem.offsetWidth}px`;
lava.style.transform = `translateX(${activeItem.offsetLeft}px) scaleX(1)`;

// 更新熔岩块位置
function updateLava(target) {
    const { offsetLeft, offsetWidth } = target;
    lava.style.transform = `translateX(${offsetLeft}px) scaleX(${offsetWidth / lava.offsetWidth})`;
}

// 点击事件：更新 active 类和熔岩块位置
items.forEach((item) => {
    item.addEventListener('click', () => {
        // 移除其他项的 active 类
        items.forEach((el) => el.classList.remove('active'));
        // 为点击的项添加 active 类
        item.classList.add('active');
        // 更新熔岩块位置
        updateLava(item);
    });

    // 鼠标悬停：熔岩块跟随悬停项
    item.addEventListener('mouseenter', () => updateLava(item));
});

// 鼠标移出：熔岩块回到选中项
menu.addEventListener('mouseleave', () => {
    const activeItem = document.querySelector('.menu li.active');
    if (activeItem) {
        updateLava(activeItem);
    }
});
```

上面示例所展示的是创建熔岩导航菜单的通常做法。事实上，我们完全可以不依赖任何 JavaScript 来创建熔岩导航菜单。接下来，我将向大家展示几种使用 CSS 创建熔岩导航菜单的方案，这些方案中有比较传统的，也有现代的。通过这些实际你将能掌握更多纯 CSS 创建熔岩导航菜单的技巧，也能进一步了解这些技巧的利弊。

  


首先来看第一种，算得上是非常传统的一种方式，既没有额外添加 HTML 标签，也无需任何 JavaScript。使用这种方式创建熔岩导航菜单，它的 HTML 结构非常简洁：

  


```HTML
<div class="menu">
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Services</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
    <div class="lava"></div>
</div>
```

  


我们使用 CSS 给熔岩块 `.lava` 设置一个样式，将它放置在导航菜单的底部：

  


```CSS
.menu {
    --size: 120px;
    --gap: 1rem;
    position: relative;
    margin-left: auto;
    
    ul {
        gap: var(--gap);
    }
    
    li {
        cursor: pointer;
        z-index: 2;
        width: var(--size);
    }
}

.lava {
    position: absolute;
    height: 100%;
    top: 0;
    left: 0;
    z-index: 0;
    transition: all 0.5s ease 0s;
    border-radius: 8px;
    height: 100%;
    width: var(--size);
    border-radius: 5px;
    transform-origin: left center;
    will-change: transform;
    transition: all 0.2s ease-out;
    translate: 0%;
    background-color: #1abc9c;
}
```

  


现在呈现在你面前导航菜单效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/16acae3b549c49e482325ee7491f8d3a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1102&s=126031&e=jpg&b=f4f4f4)

  


它是静态的，不具有任何交互效果。接下来，我们需要使熔岩块跟随鼠标移动——用户鼠标悬浮在菜单项上时，熔岩块要平滑到对应的菜单项。在这里，我将借助 CSS 的 `:has()` 选择器结合 `li:hover` 状态以及 `~` 选择器来调整熔岩块的位置：

  


```CSS
ul:has(li:nth-of-type(2):hover) ~ .lava {    
    translate: calc(100% + var(--gap));
    background-color: #e74c3c;
}

ul:has(li:nth-of-type(3):hover) ~ & {
    translate: calc(200% + var(--gap) * 2);
    background-color: #3498db;
}

ul:has(li:nth-of-type(4):hover) ~ & {
    translate: calc(300% + var(--gap) * 3);
    background-color: #9b59b6;
}
```

  


简单的解释一下，由于 `li` 与 `.lava` 并不是兄弟元素，无法直接使用 `li:nth-of-type(2):hover ~ .lava` 选择器来选中 `.lava` 元素，因此引入了 `:has()` 选择器。其中 `ul:has(li:nth-of-type(2):hover)` 选择器的意思是，当第二个 `li` 处于悬浮状态（`:hover`）时，条件为真，反之为假。在此基础上叠加 `~ .lava` （即 `ul:has(li:nth-of-type(2):hover) ~ .lava`）时，所起作用就是，当第二个 `li` 处于悬浮状态时选中 `.lava` 元素。

  


另一点就是，`.lava` 是个绝对定位元素，而且是相对于 `.menu` 容器定位，它的初始位置是 `left:0` ，正好在第一个菜单项底部。如果要将它放置在第二个菜单项底部，我们需要调整 `left` 值，或者使用 `translate` 来移动它的位置。我在这里选择了 `translate` ，它的值是 `100%` 加上菜单项之间的间距 `--gap` 。

  


这个时候，用户鼠标悬停某个菜单项上时，熔岩块会从初始位置平滑过去，在视觉上有一个跟随鼠标移动的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1c79eba6a2084fdd9ee824c9e05280d1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1102&h=580&s=178865&e=gif&f=125&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/OPLOXBG

  


这种方式有一个明显的缺陷，鼠标移出菜单项时，熔岩块将会回到初始位置，而且也无法停留在当前菜单菜（即点击无效果）。如果需要当前选项的效果，还得依赖 JavaScript。

  


第二个缺点是，熔岩块的大小是固定的，如果菜单项的内容长度不相等，需要手动调整熔岩块的尺寸。就这一点而言，整个熔岩导航菜单就缺失了灵活性和适配性。

  


第三个缺点是，如果菜单项增加，对应的 CSS 选择器也要随之增加，这使得 CSS 代码会变得非常冗余，而且增加维护的成本。

  


有一点需要知道的是，我们可以使用伪元素 `::after` 来替代 `.lava` 元素：

  


```CSS
.menu {
    --size: 120px;
    --gap: 1rem;
    position: relative;
    margin-left: auto;

    ul {
        display: flex;
        list-style: none;
        gap: var(--gap);
        font-size: 1.5rem;

        li {
            cursor: pointer;
            z-index: 2;
            width: var(--size);
        }

        &::after {
            --c: #1abc9c;
            content: "";
            position: absolute;
            height: 100%;
            top: 0;
            left: 0;
            z-index: 0;
            transition: all 0.5s ease 0s;
            border-radius: 8px;
            height: 100%;
            width: var(--size);
            border-radius: 5px;
            transform-origin: left center;
            will-change: transform;
            transition: all 0.2s ease-out;
            translate: 0%;
            background-color: var(--c);
        }

        &:has(li:nth-of-type(2):hover)::after {
            translate: calc(100% + var(--gap));
            --c: #e74c3c;
        }
      
        &:has(li:nth-of-type(2):hover)::after {
            translate: calc(100% + var(--gap));
            --c: #e74c3c;
        }
      
        &:has(li:nth-of-type(3):hover)::after {
            translate: calc(200% + var(--gap) * 2);
            background-color: #3498db;
        }

        &:has(li:nth-of-type(4):hover)::after {
            translate: calc(300% + var(--gap) * 3);
            --c: #9b59b6;
        }
    }
}
```

```CSS
        cursor: pointer;
        text-align: center;
        height: 100%;
        display: grid;
        border-radius: calc(var(--radius) - var(--border));
        place-items: center;
        color: hsl(0 0% 100% / calc(0.5 + var(--highlight, 0)));
        transition: background, color;
        transition-duration: 0.25s;
        transition-timing-function: var(--ease, ease);
    }

    &::after {
        pointer-events: none;
        content: "";
        width: calc(100% / var(--count));
        height: 100%;
        background: hsl(0 0% 100%);
        position: absolute;
        border-radius: calc(var(--radius) - var(--border));
        mix-blend-mode: difference;
        translate: calc(var(--active, 0) * 100%) 0;
        transition: translate, outline-color;
        transition-duration: var(--speed);
        transition-timing-function: var(--ease, ease);
        outline: 2px solid transparent;
    }
}
```

  


你可能已经注意到了，上面代码中的 `--count` 变量并没有显式定义。`--count` 是用来定义 Tab 选项卡数量的，通常情况之下，会选择在 `.tabs` 元素上以内联的方式来定义 `--count` ，它的值对应 `.tabs` 中包含的 `input` 数量（即选项卡的数量）：

  


```HTML
<div class="tabs" style="--count: 2;">
    <input type="radio" id="html" name="fav_language" value="HTML" checked class="sr-only" />
    <label for="html">HTML</label>
    <input type="radio" id="css" name="fav_language" value="CSS" class="sr-only" />
    <label for="css">CSS</label>
</div>
```

  


要是你不喜欢在行内定义 `--count` ，我们可以使用 CSS 的 `:has()` 和 `input` 的数量来定义 `--count` ，例如：

  


```CSS
.tabs {
    /* CSS ... */
    &:has(input:nth-of-type(2)) {
        --count: 2;
    }
    
    &:has(input:nth-of-type(3)) {
        --count: 3;
    }
    
    &:has(input:nth-of-type(4)) {
        --count: 4;
    }
    
    &:has(input:nth-of-type(5)) {
        --count: 5;
    }
    
    :checked + label {
        --highlight: 1;
    }
}
```

  


这个时候就能计算出“熔岩块”（`.tabs::after`）的大小，避免了给熔岩块指定一个固定的值。另外，`.tabs` 的背景颜色是黑色，伪元素（`.tabs::after`）的背景颜色是白色，标签文本（`label`）是白色。[当你在伪元素（`.tabs::after`）上使用 `mix-blend-mode: difference` 时](https://juejin.cn/book/7223230325122400288/section/7259669043622690853)，会产生一个效果，使文本从白色变成黑色。这个时候，你看到的 Tab 选项卡效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e8f3fcecbebb4603b64678cc1157d721~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1207&s=128826&e=jpg&b=f5f5f5)

  


与此同时，熔岩块的位置是由 `translate` 控制的：

  


```CSS
.tabs {
    /* CSS ... */
    translate: calc(var(--active, 0) * 100%) 0;
}
```

  


请留意代码中的 `--active` 变量，它默认为 `0` ，它表示的选项卡的选中或当前状态。我们可以通过 CSS 的 `:has()` 选择器和单选按钮的 `:checked` 来改变 `--active` 的值：

  


```CSS
.tabs {
    /* CSS ... */
    &:has(:checked:nth-of-type(1)) {
        --active: 0;
    }
    
    &:has(:checked:nth-of-type(2)) {
        --active: 1;
    }
    
    &:has(:checked:nth-of-type(3)) {
        --active: 2;
    }
    
    &:has(:checked:nth-of-type(4)) {
        --active: 3;
    }
    
    &:has(:checked:nth-of-type(5)) {
        --active: 4;
    }
}
```

  


这个时候，当 `input` 被选中时，`--active` 的值会变化，应用在 `.tabs::after` 上的 `translate` 也会改变。再加上 `transition` 属性的设置，你将看到 Tab 选项卡的熔岩块会有一个平滑的移动动效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c8ad3f20b3bb463db87671e88bf016e7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1096&h=640&s=202958&e=gif&f=167&b=f3f3f3)

  


最后，我们再调整一下选项卡每个选项在 `:hover` 状态的样式：

  


```CSS
input:not(:checked) + label:hover {
    --highlight: 0.35;
    background: hsl(0 0% 20%);
}
```

仔细观察，不难发现，之前这几种创建熔岩导航菜单的方案有一个共同的特性：需要给熔岩块指定尺寸，并且是相对于祖先容器（比如 `.menu` 或 `.tabs`）定位。然而，随着 **[CSS 锚点定位](https://juejin.cn/book/7395119173656903715/section/7445249534449434674)** 的出现，我们可以改为这一现象——**熔岩块不需要指定具体的尺寸，并且可以相对于菜单项定位**。

  


我曾在《[CSS 锚点定位：探索下一代 Web 布局](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)》中向大家展示了 **[如何通过 CSS 锚点定位来创建熔岩导航菜单](https://juejin.cn/book/7223230325122400288/section/7259669151743279159#heading-9)**：

  


```CSS
.nav {
    anchor-name: --nav-menu;

    &::before {
        content: "";
        position: absolute;
```

效果虽然不错，但 CSS 过于复杂。

  


事实上，我们使用 CSS 锚点定位来创建熔岩导航，不需要这么复杂。我们只需要做到两点：声明锚点名称和引用锚点。

  


```CSS
.menu {
    margin-left: auto;

    ul {
        list-style: none outside none;
        display: flex;
        gap: 1rem;
        isolation: isolate;
        color: #fff;
        font-size: 1.25rem;

        &::before {
            content: "";
            position: absolute;
            z-index: -1;
            position-anchor: --li;
            background: #f50;
            height: 0.4em;
            border-radius: 5em;
            inset: auto calc(anchor(center) - 0.2em) anchor(bottom) calc(anchor(center) - 0.2em);
            transition: 0.3s 0.2s, 0.3s 0.2s cubic-bezier(0, -900, 1, -900) bottom;
        }
    }

    li {
        position: relative;

        &::before {
            content: "";
            position: absolute;
            z-index: -1;
            inset: 100% 50% 0;
            border-radius: 50%;
            background: #f50;
            transition: 0.2s;
        }
    }

    a {
        color: #fff;
        text-decoration: none;
        font-weight: 900;
        line-height: 2.25;
        padding-inline: 1em;
        display: block;
    }
}

ul li:is(:hover, .active)::before {
    anchor-name: --li;
    inset: 0;
    border-radius: 3%;
    transition: 0.2s 0.5s;
}

ul:has(li:hover) li.active:not(:hover)::before {
    anchor-name: none;
    inset: 100% 50% 0;
    border-radius: 50%;
    transition: 0.2s;
}

ul:not(#_) li:nth-child(1)::before {
    bottom: -0.1px;
}

ul:not(#_) li:nth-child(2)::before {
    bottom: -0.2px;
}

ul:not(#_) li:nth-child(3)::before {
    bottom: -0.3px;
}

ul:not(#_) li:nth-child(4)::before {
    bottom: -0.4px;
}

ul:not(#_) li:nth-child(5):before {
    bottom: -0.5px;
}
```

  


在我们这个示例中，熔岩块是由 `ul::before` 创建，并且是相对于 `li:hover` 和（或 `li.active`）的`::before` 定位。因此我们的锚点在 `li:hover::before` 和（或）`li.active::before` 上声明：

  


```CSS
ul li:is(:hover, .active)::before {
    anchor-name: --li;
}
```

  


然后在 `ul::before` 使用 `position-anchor` 引用已声明的锚点名称 `--li` ：

  


```CSS
ul::before {
    position: absolute;
    z-index: -1;
    position-anchor: --li;
    inset: auto calc(anchor(center) - 0.2em) anchor(bottom) calc(anchor(center) - 0.2em);
}
```

  


需要注意的是，我们需要在 `li.active` 非 `:hover` 状态下，使用 `anchor-name: none` 覆盖已声明的锚点名称：

  


```CSS
ul:has(li:hover) li.active:not(:hover)::before {
    anchor-name: none;
}
```
