手风琴（`Accordion`）是一种常见的 Web UI 组件，用于在有限的页面空间内展示内容分组。它允许用户通过展开中折叠不同的部分来查看或隐藏内容，从而实现信息的层级化和交互性。手风琴通常用于 FAQ 页面、侧边栏菜单、数据展示等场景。

  


手风琴组件与我们之前探讨的 [Popover](https://juejin.cn/book/7395119173656903715/section/7442515150541684746) 和[下拉菜单](https://juejin.cn/book/7395119173656903715/section/7445265601544585225)有一点相似之处，是切换内容显示与隐藏的典型案例。默认情况之下，手风琴只有一个或多个部分可见，其他部分被折叠。用户点击某一部分时，其内容会展开，而其他部分可以保持折叠状态或自动折叠。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4b69603bb50b41aab51c9468728b4548~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1214&h=600&s=16612946&e=gif&f=182&b=fefefe)

  


以往我们实现这种 UI 交互效果或多或少都需要依赖 JavaScript 。但是，在这节课中，我将使用 CSS 创建一个手风琴 UI 组件。这是另一个可以用极少代码实现的常见组件。大多数技巧依赖于 `input` （单选按钮 `radio` 或复选框 `checkbox`）的“黑魔法”，但这次我们会尝试一种不同的方法—— 使用 CSS 与 HTML 的 `<details>` 和 `<summary>` 元素来创建手风琴组件。在这个过程中，你还将获取一些新的 CSS 特性。比如：

  


-   使用 `::details-content` 伪元素样式化手风琴展开和收起的部分
-   使用 `interpolate-size` 或 `calc-size()` 将高度动画化为 `height: auto`

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bc6ca466b4fd4b57acd1286f699d9515~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1255&s=699227&e=jpg&b=ffffff)

  


上图是一个手风琴组件（`Accordion`）的结构示意图，标注了手风琴的主要组成部分。以下是各部分的解释：

  


-   **单个手风琴选项**：它包括了标题、内容区域和图标。多个这样的部分组合在一起构成整个手风琴组件
-   **标题**：每个手风琴项都有一个标题，用户可以点击它来展开或折叠对应的内容区域
-   **折叠/展开图标**：标题旁边的图标通常是一个箭头，指示当前的状态。例如“减号”（`-`）图标表示内容已展开（显示）状态；“加号”（`+`）图标表示内容目前是折叠（隐藏）状态。注意，图标不是必须的，可以根据具体的情景来决定
-   **内容区域**：初始状态下，内容区域是隐藏的，当用户点击标题时，内容区域会展开，显示具体的内容

  


手风琴组件根据其布局方式，又常分为水平手风琴和垂直手风琴。其中，水平手风琴的内容区域会沿着水平方向展开或折叠，如下图所示：

## 传统方案：使用 `input` 魔法创建手风琴组件

  


熟悉手风琴组件的 Web 开发者应该知道，手风琴有两种常见的交互模式。第一种是，用户点击每个标题，对应的内容区域会展开，而且只有再次点击标题时，该内容区域才会折叠。这种模式可以同时展开多个内容区域：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1dd846f407194f3eb2d7666529391bd1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1146&h=766&s=1075740&e=gif&f=197&b=f3f3f3)

  


第二种模式是，手风琴始终只有一个面板内容区域会展开。当用户点击标题时，对应的内容区域会展形，与其相邻的其他面板的内容区域则会折叠：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/427ff6fe4c034ce49e5b04e928fbddad~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1228&h=762&s=1410575&e=gif&f=159&b=f3f3f3)

  


这两种模式非常适合使用 `input` 的 `checkbox` （复选框）和 `radio` （单选按钮）来制作：

  


-   复选框可以多选，对应第一种模式，手风琴可以同时有多个面板的内容区域展开
-   单选按钮只能单选，对应第二种模式，手风琴仅有一个面板的内容区域展开

  


不管使用哪种方式，都将使用到 CSS 条件判断相关的特性，比如将 `input:checked` 和 `~` 组合在一起，展开相应的内容区域。换句话说，`input` 被选中，与其相邻的内容区域就会展开，否则就会折叠。

  


需要知道的是，使用这种方式，对于 HTML 结构有一定的要求，比如：

  


-   `<input>` 的 `id` 与 `label` 的 `for` 关联在一起，其中 `label` 对应的就是手风琴的标题
-   `<input>` 需要出现在内容区域（`.accordion__content`）元素之前，因为 `~` （普通兄弟选择器） 或 `+` （相邻兄弟选择器）选择器只能选择与 `input` 相邻的兄弟元素

  


虽然 CSS 的 `:has()` 选择器的出现，可以允许我们的 HTML 结构不需要那么严谨，但我个人建议，在编写 HTML 时应该有一个更良好的习惯，将相互之关联的元素放置在一起，并且应该考虑得更全面，比如浏览器不支持 `:has()` 选择器，我们可以通过 `~` 或 `+` 选择器实现相似的功能。

接下来是编写 CSS 代码。最为关键的是如何控制内容区域的折叠（隐藏）与展开（显示），比较简单的是：

  


-   折叠状态将内容区域（`.accordion__content`）的高度（`height`）设置为 `0` ，并将 `overflow` 设置为 `hidden` 。当然，你也可以同时将 `opacity` 设置为 `0` 和 `visibility` 设置为 `hidden` ，但这不是必须的，只是以防万一
-   展开状态将内容区域 (`.accordion__content` ）的高度（`height`）重置为 `auto` ，让浏览器根据元素内容来设计其高度。如果你设置了 `opacity` 和 `visibility` 属性的话，你需要分别将它们重置为 `1` 和 `visible`

  


由于 CSS 无法直接使用 `transiiton` 对 `height: auto` （即从 `0` 过渡到 `auto`）动画化，但又为了让内容区域的折叠与展开有一个平滑的过渡动效，往往会采用 `max-height` 替代 `height` ，并且在展开状态设置一个较大的 `max-height` 值。具体的 CSS 代码如下：

  


```CSS
/* 折叠状态 */
.accordion__content {
    overflow: hidden;
    transition: max-height 0.7s;
    max-height: 0;
}

/* 展开状态 */
input:checked ~ .accordion__content {
    max-height: 50em; /* 你可以根据需要设置一个更大的值 */
}
```

  


或者使用 `:has()` 选择器来替代 `~` 选择器：

  


```CSS
/* 折叠状态 */
.accordion__content {
    overflow: hidden;
    transition: max-height 0.7s;
    max-height: 0;
}

/* 展开状态 */
.accordion__panel:has(:checked) .accordion__content {
    max-height: 50em;
}
```

## 传统方案：使用 `:target` 创建手风琴组件

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/27113e4713f94f558a973b81b84040df~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=49846&e=png&b=f4ebeb)

  


不知道大家是否还有印象，我曾在《[CSS 布局：防止锚点链接滚动到固定或粘性头部后面](https://juejin.cn/book/7395119173656903715/section/7445248285679304713)》课程末尾提到一个锚点链接的小技巧——使用 `:target` 选择器在跳转到页面锚点时添加样式，比如：

  


```CSS
h2:target {
    color: #9c27b0;
}

section:has(h2:target) {
    outline-color: #9c27b0;
    color: #9c27b0;
}
```

  


这个技巧同样可以用来创建手风琴组件。

  


继续以上面示例为基础，我们使用 `<a href="id">` 与 `.accordion__panel` 的 `id` 绑定在一起，来替代之前示例中 `input` 与 `label` 。具体的 HTML 结构如下：

  


```HTML
<div class="accordion">
    <div class="accordion__panel" id="account">
        <a href="#account" class="accordion__heading">Heading</a>
        <div class="accordion__content">Content</div>
    </div>  
    <!-- 省略其他 Panel -->
    <div class="accordion__panel">
        <a href="#" class="accordion__heading">Log Out</a>
        <div class="accordion__content"></div>
    </div>
</div>    
```

  


注意，上面代码中最后一个 `.accordion__panel` 并没有设置 `id` ，并且其子元素 `.accordion__heading` 的 `href` 属性值为 `#` ，用它来折叠手风琴已展开的内容区域。

  


完成手风琴折叠和展开的 CSS 代码如下：

  


```CSS
.accordion__content {
    overflow: hidden;
    transition: max-height 0.7s;
    max-height: 0;
}

.accordion__panel:target .accordion__content{
    max-height: 50em;
}     
```

## 现代方案：使用 `<details>` 和 `<summary>` 创建手风琴组件

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/64d995c7cb09487ea85f78f2498e1a51~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=128562&e=webp&b=e2e8f0)

  


之前我们所展示的手风琴案例的内容区域都是沿着垂直方向进行折叠和展开，接下来我们来看一个内容区域沿着水平方向折叠和展开的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/00cf2cb6a11e4cdc8b2fcee3eee5acbd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1214&h=600&s=16612946&e=gif&f=182&b=fefefe)

  


从 Chrome 131 开始，我们可以使用 `<details>` 和 `<summary>` 元素的结构来创建手风琴组件，因为你可以更多地自定义 `<details>` 和 `<summary>` 元素的结构。

  


```HTML
<div class="accordion">
    <details name="accordion" id="boating" open class="accordion__panel">
        <summary class="accordion__heading">
            <span>🚤</span>
            <img draggable=false src="https://raw.githubusercontent.com/kevin-powell/accordion/8551c559e3e8d9a9cca7a983c9e8903ef533f189/public/assets/boat.webp" alt="">
        </summary>
        <div class="accordion__content">
            <h3>Boating</h3>
            <p>Port mutiny draught handsomely ye furl flogging line shrouds hulk. Spirits Plate Fleet code of conduct.</p>
        </div>
    </details>
    <!-- 省略其他 details -->
</div>
```

  


简单的解释一下上面的代码：

  


-   `<details>` 元素对应着手风琴组件的每一个面板（`Accordion Panel`）
-   `<summary>` 元素对应着手风琴组件每个面板的标题（`Accordion Heading`）
-   `<div class="accordion__content">` 元素对应着手风琴组件每个面板的内容区域（`Accordion Content`）

  


如果你运行上面的代码，并点击标题（`<summary>`），就可以展开或折叠内容（`.accordion__content`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6fe279b41815427d96706fc001bb493b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1190&h=882&s=1972989&e=gif&f=134&b=f3f3f3)

  


我们已经差不多完成了！通过使用原生的 HTML 元素，我们已经实现了手风琴的功能。

  


> `<details>` 元素创建了一个信息隐藏或显示的组件，只有当该组件被切换到“打开”状态时（`open`），信息才会显示。必须使用 `<summary>` 元素提供一个摘要或标签。

  


现在，我们只需要对其进行样式化即可。

  


请仔细观察我们即将要实现的手风琴组件效果，当用户点击标题时，当前面板宽度会变宽。类似的效果我们在小册的《[CSS 布局：构建不规则网格布局（Part4）](https://juejin.cn/book/7395119173656903715/section/7441391705011519488)》课程中有介绍过。简单地说，我们使用 **[CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)** 或 **[CSS Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)** 都可以实现类似的布局效果，我在这里选择了 CSS Flexbox 布局技术来对整个手风琴进行布局处理。

  


首先将 `.accordion` 容器定义为一个 Flexbox 容器，将手风琴面板排成一行：

  


```CSS
.accordion {
    display: flex;
    gap: 1rem;
    width: min-content;
    margin-inline: auto;
}
```

  


我们把 Flexbox 容器的宽度（`width`）设置为 `min-content` ，它将会根据元素内容来决定容器的大小。这是 **[内在 Web 布局](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)** 常见的一种方式。另外，[`margin-inline` 是一个逻辑属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)，书写模式为 `ltr` 或 `rtl` 时，它等同于 `margin-left` 和 `margin-right` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b5f329527d3a4a388fdc8d35c26d662c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1098&h=560&s=2248578&e=gif&f=137&b=f3f3f3)

  


正如你所看到的，现在手风琴所有面板排成一行了，但面板宽度并没有随着内容展开而变宽。这是我们接下来需要处理的。

  


`<details>` 元素自带了两个有趣的功能，可以用来为手风琴组件添加样式：

  


-   **`::marker`** **伪元素**：这是标题旁边的小三角形。每个浏览器都会为 `::marker` 定义默认样式，当内容展开或折叠时，`::marker` 的样式会随之变化，例如折叠时，三形形朝右；展开时，三角形朝向
-   **`open`** **属性**：它有点类似于 `<dialog>` 元素的 `open` 属性，如果你希望手风琴面板内容区域展开，只需要将对应面板（`<details>` ）添加 `open` 属性。需要知道的是，用户点击标题（`<summary>`）时，浏览器会自动为其父元素 `<details>` 添加 `open` 属性，而其他相邻的 `<details>` 元素的 `open` 属性（如果有）会自动移除

  


此外，`open` 属性除了可以用来定位已展开的 `<details>` 元素之外，还可以为其应用特定样式。

  


```CSS
details {
    flex-shrink: 0;
    display: flex;
    background: transparent;
    color: white;

    height: 30rem;
    border-radius: 2rem;
    overflow: hidden;

    position: relative;
    z-index: 1;

    ::marker {
        content: "";
    }

    &:open {
        summary img {
            filter: brightness(0.5);
        }
        .accordion__content p {
            translate: 0 0;
            opacity: 1;
            transition-delay: 0.5s;
        }
    }

    &:hover,
    &:has(summary:focus) {
        outline: 3px solid cadetblue;
        outline-offset: 3px;
        cursor: pointer;
    }
}

summary {
    padding: 1rem 1em;
    width: 5rem;
    flex-shrink: 0;
    text-align: center;
    cursor: pointer;

    span {
        display: grid;
        place-content: center;
        width: 100%;
        aspect-ratio: 1;
        border-radius: 50%;
        background: rgb(0 0 0 / 0.25);
    }

    img {
        position: absolute;
        inset: 0;
        width: 100%;
        height: 100%;
        object-fit: cover;
        object-position: center;
        z-index: -1;
        transition: filter 0.5s ease;
    }

    &:focus {
        outline: none;
    }
}

.accordion__content {
    padding: 1.5rem 1em;
    max-width: 300px;
    display: grid;
    gap: 1.5rem;

    :first-child {
        margin-top: 0;
    }
    
    :last-child {
        margin-bottom: 0;
    }

    p {
        translate: 0 2rem;
        opacity: 0;
        transition: all 0.5s ease;
        transition-delay: 0.5s;
        line-height: 1.8;
    }
}
```

  


上面的代码中，重置了 `::marker` 的 `content` 属性的值为空字符串，这样就会去除默认的三角形。与此同时，我们在 `open` 状态调整了 `<summary>` 子元素 `img` 的样式和 `.accordion__content` 子元素 `p` 的样式。

上面的代码中，我们重置了 `::marker` 的 `content` 属性的值为空，从而移除了默认的三角形指示符。需要知道的是 `::marker` 的使用与我们熟悉的 `::before` 和 `::after` 是相似的。另外，我们也可以像下面这样使用 `::marker` 覆盖默认的内容：

  


```CSS
summary::marker {
    content: "+";
}

summary[open]::marker {
    content: "-";
}
```

  


其优点是，使用方法简单，允许使用任意字符（包括表情符）。不过，`::marker` 有一定的限制性，只有特定的 CSS 属性可以使用，不能完全自定义样式。到目前为止，我们只能在 `::marker`上使用下面这几个 CSS 属性：

  


-   `animation-*`
-   `transition-*`
-   `color`
-   `direction`
-   `font-*`
-   `content`
-   `unicode-bidi`
-   `white-space`

  


像 `background`、`width`、`height` 等盒模型相关的属性用在 `::marker`上也是无效的。有关于这方面更详细的介绍，请移步阅读《[CSS 计数器：`@counter-style`](https://juejin.cn/book/7223230325122400288/section/7259668400379527205#heading-24)》！

  


换句话说，如果你希望创建更具个性化的指示器，那么可以先将 `::marker` 的 `content` 重置为空，然后再使用 `summary` 的 `::before` 或 `::after` 来替代 `::marker` 伪元素，因为 CSS 样式化 `::before` 或 `::after` 更为灵活：

  


```CSS
summary::marker {
    content: "";
}

details summary::before {
    content: "";
    /* 元素折叠时的样式 */
}

details[open] summary::before {
    /* 元素展开时的样式 */
}
```

  


另外一点就是，在更早的时候，我们无法更改 `<details>` 元素的 `display` 属性（即显示类型）。现在，这个限制已经放宽了，允许你在 `<details>` 元素上使用网格（`display: grid`）或 Flexbox（`display: flex`）布局等。

  


如果你使用浏览器开发者工具来审查 `<details>` 元素，你会发现 `<details>` 元素是通过 Shadow DOM 实现的。它包含一个用于 `<summary>` 的 `<slot>` （默认包含一个 `<summary>` 子元素）和一个用于所有剩余内容的 `<slot>` ，意味着 `<details>` 元素的所有子元素（除了 `<summary>` 元素）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/23cac419635943f3ad92c95f8f329e07~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3338&h=1456&s=703585&e=jpg&b=252c2c)

  


与此同时，内容插槽（`slot#details-content`）还增加了一个 `::details-content` 伪元素：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c70f348d5f0340a59cd49f89f8829677~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3338&h=1140&s=667767&e=jpg&b=283031)

  


我们可以使用这个伪元素（`::details-content`）来样式化包裹 `<details>` 元素内容的容器。例如：

  


```CSS
details::details-content {
    border: 5px dashed hotpink;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4d0ca0da55bb4ee1b064c89dad6b651d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1255&s=830924&e=jpg&b=f4f4f4)

  


我们也可以在 `::details-content` 前面加上 `[open]` 属性选择器，这样就仅在 `<details>` 元素处于展开状态时应用样式：

  


```CSS
[open]::details-content {
    border: 5px dashed hotpink;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad285d4565a04dbcb1bfb2b854630399~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1584&s=909582&e=jpg&b=f4f4f4)

  


我个人建议仅在 `<details>` 元素处于 `[open]` 状态时应用对 `::details-content` 伪元素的样式。

  


我们回到上一个示例中来，在上一个示例中，我们是在 `.accordion__content` 元素中设置了一个 `max-width: 300px` 。现在，我们可以通过 `::details-content` 伪元素来调整手风琴面板内容区域的大小。例如，在折叠状态下设置它的 `width` 为 `0` ，在展开状态再重置其 `width` ：

  


```CSS
::details-content {
    width: 0;
}

[open]::details-content {
    width: min(30vw, 300px);
}
```

  


这与在 `.accordion__content` 元素上显式设置 `width` 或 `max-width` 达到的效果是相似的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/12ef9bd11ee0478c9d32281a60148b49~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1326&h=532&s=14532314&e=gif&f=121&b=f3f3f3)

  


还有一点需要知道是，浏览器将 `::details-content` 的 `display` 设置为 `block` ，而之前它的 `display` 设置的是 `contents` 。如果这个改变影响到了你的布局，你可以像下面这样将 `display` 属性的值重置为 `contents` ：

  


```CSS
details[open]::details-content {
    display: contents;
}
```

  


温馨提示，`display:contents` 可以在不调整 HTML 结构的情况下，**允许你以某种方式从盒子树（Box Tree）中移除一个元素，但仍保留其内容** 。[W3C 文档这样描述的](https://www.w3.org/TR/css-display-3/#box-generation)：

  


> **显式设置** **`display`** **属性值为** **`contents`** **的元素自身不会产生任何盒子，但它的子元素和伪元素仍会产生盒子，文本运行也正常。对于盒子的生成和布局，该元素必须被视为在元素树中被其内容所取代，包括其源文件的子元素和伪元素，比如** **`::before`** **和** **`::after`** **伪元素，它们在该元素的子元素之前或之后正常生成** 。

  


如果你想更深入的了解 `display: contents` 相关的知识，建议你花点时间阅读《[`display：contents` 改变 Flexbox 和 Grid 布局模式](https://juejin.cn/book/7161370789680250917/section/7161624465313366046)》！

  


到目前为止，我们通过 `::details-content` 伪元素调整了手风琴面板的内容区域的大小。接下来，我们使用 CSS 的 `transition` 来动画化 `::details-content` 伪元素的宽度（从 `0` 过渡到 `min(30vw, 300px)`），使 `<details>` 元素展开时对其内容进行动画处理：

  


```CSS
::details-content {
    transition: width 0.5s ease, content-visibility 0.5s ease allow-discrete;
    width: 0;
}

[open]::details-content {
    width: min(30vw, 300px);
}
```

我们可以像下面这样使用 `interpolate-size` 或 `calc-size()` 来动画化 `height`。另外，为了防止内容溢出 `::details-content` 伪元素，可以给它应用 `overflow: clip` ：

  


```CSS
:root {
    interpolate-size: allow-keywords;
}

::details-content {
    transition: height 0.5s ease, content-visibility 0.5s ease allow-discrete;
    height: 0;
    overflow: clip;
}

[open]::details-content {
    height: calc-size(auto, size); /* 或 auto */
}
```

  


我们也可以使用 [CSS 的 `@supports` 规则](https://juejin.cn/book/7223230325122400288/section/7258822669154877480)来检测浏览器是否支持 `::details-content` 伪元素：

  


```CSS
::details-content {
    transition: height 0.5s ease, content-visibility 0.5s ease allow-discrete;
    height: 0;
    overflow: clip;
}

/* 浏览器支持 interpolate-size */
@supports (interpolate-size: allow-keywords) {
    :root {
        interpolate-size: allow-keywords;
    }
    
    [open]::details-content {
        height: calc-size(auto, size); /* 或 auto */
    }
}

/* 不支持 interpolate-size 的浏览器的回退 */
@supports not (interpolate-size: allow-keywords) {
    [open]::details-content {
        height: 300px;
        overflow-y: scroll; /* 如果内容高度超过 300px 时 */
    }
}
```

另外，如果你希望 `<details>` 的内容有部分可见，需要将 ，`content-visibility` 始终设置为 `visible`。高度使用 `calc-size()` 进行动画处理，因为其中涉及到计算。

  


```CSS
::details-content {
    content-visibility: visible; /* 始终可见 */
    transition: height 0.5s ease;
    height: 150px;
    overflow: clip;
}
[open]::details-content {
    height: calc-size(auto, size + 0.5rem); /* 使用 calc-size 因为我们需要添加长度 */
}
```

  


为了方便样式的设置，内容被包装在一个外部 `div` 中。这个外部 `div` 应用了布局样式，例如 `padding`，而 `::details-content` 伪元素则进行了动画处理。
