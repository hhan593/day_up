## 理解 `border-image` 的强大特性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd03c9e215964bd799eb9defb05ad595~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=473115&e=png&b=fefefe)

  


  


首先，我要特别强调的一点是，**`border-image`** **属性的语法比较复杂**。这也是导致大多数 Web 开发者感到困惑和不喜欢使用它的原因所在。以下是 `border-image` 属性的语法：

  


```CSS
.element {
    border-image: <border-image-source> <border-image-slice> / <border-image-width> / <border-image-outset> <border-image-repeat>
}
```

  


`border-image` 也我们熟悉的 `background` 属性类似，它也是一个简写属性。它拆分出来包含以下几个属性：

  


```CSS
.element {
    border-image-source: none | <image>;
    border-image-slice: [<number [0,∞]> | <percentage [0,∞]>]{1,4} && fill?;
    border-image-width: [ <length-percentage [0,∞]> | <number [0,∞]> | auto ]{1,4};
    border-image-outset: [ <length [0,∞]> | <number [0,∞]> ]{1,4};
    border-image-repeat: [ stretch | repeat | round | space ]{1,2};
}
```

  


其中 `border-image-source` 类似于 `background-image` ，`border-image-repeat` 类似于 `background-repeat` ；而 `border-image-width` 和 `border-image-outset` 的工作方式类似于 `border-width` 和 `outline-offset` 。

  


在使用 `border-image` 简写属性时，其中 `border-image-width` 和 `border-image-outset` 不是必须的。比如，有时我们只需要指定 `border-image-outset` ，而无需设置 `border-image-width` ，这时 `border-image` 的简写方式可能会像下面这样：

  


```CSS
.element {
    border-image: <border-image-source> <border-image-slice> // <border-image-outset> <border-image-repeat>;
}
```

  


到目前为止，这可能是 CSS 中唯 一一个支持**双斜杠** （`//`）的 属性！

  


刚才提到过，`border-image` 属性的使用相对而言要复杂一些，尤其是它的切片逻辑（`border-image-slice`）和延伸（`border-image-outset`）概念让人不容易上手。为了使大家更易于理解 `border-image` 属性的特性和使用，我们可以使用 MDN 提供的 `border-image` 生成器来辅助大家理解和掌握 `border-image` 的使用：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6b865e8db539490f85d2aace95a29037~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1650&h=816&s=3921180&e=gif&f=596&b=fdfcfc)

  


> Demo 地址：https://mdn.github.io/css-examples/tools/border-image-generator/

  


在实际使用过程中，我们可以按照下面的步骤来使用 `border-image` ，这样不易于出错：

  


-   `border-image-source` ：指定你喜欢的图像源，
-   `border-image-slice` ：切片图像，以指定在边框中使用的部分。它会将图像源切割成一个九宫格
-   `border-image-width` ：调整图像的宽度（厚度），控制图像在边框区域的缩放方式
-   `border-image-outset` ：如需图像超出元素边框，选择定义一个外扩值
-   `border-image-repeat` ：决定图像如何适应或重复在边框周围（如 `repeat` 、`stretch` 、`round` 或 `space`）

  


通过这些步骤，可以有效地在 Web 中使用自定义图像作为元素边框。

  


假设我们现在有一张图片（如下图所示）用于装饰元素 `.box` 的边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad0b15a3c8374d889f2876971ebbc0dc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2160&h=1440&s=423019&e=png&a=1&b=a6200b)

  


```HTML
<div class="box"></div>
```

  


```CSS
.box {
    width: 50vw;
    aspect-ratio: 21 / 9;
    background-color: hsl(90deg 50% 15%);
    border: 30px solid orange;
}
```

  


接下来，我们一起来看看 CSS 的 `border-image` 是如何使用上图来装饰元素 `.box` 的边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de78f5e8fe434c16923deb50893204ce~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=827&s=107960&e=jpg&b=ffffff)

  


注意，上图中橙色是元素 `.box` 的初始边框样式（`border`）。另外，作为参考，上图中橙色边框区域是我们将用图像替换的部分，橙色框内的白色区域表示元素内容区域和内边距区域。

  


### 指定边框图片

  


首先可以使用 `border-image-source` 属性来指定边框图像的来源，它的使用与 `background-image` 类似，可以接受 `<image>` 值或渐变，并将其应用于元素的边框：

  


```CSS
.box {
    border: 30px solid orange;
    border-image-source: var(--image);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c829e7b6f70f40b8bbc031917c178353~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=840&s=123067&e=jpg&b=000000)

  


你会发现边框图片 `--image` 只出现在元素的四个角落上。这和我们预期的效果不同，对吧！这只是第一步，要让边框图片在元素的边框区域完整显示还面要进一步设置。此时，我们还没有指定如何切分或分布边框图片，这正是接下来需要处理的步骤。

  


你可能已经注意到了，在代码中我们使用了 `border` 给元素 `.box` 设置了一个厚度为 `30px` 的橙色实线边框。这是因为 `border-image` 只有在元素定义了边框的情况下才能显示。`border-width` 设置了边框图像可占用的空间，而 `border-style` 确保了边框图像能够正常呈现。如果没有这些设置，即使定义了 `border-image-source` 属性，边框图像也不会显示。

  


不过，如果你使用的是 `border-image` 简写属性，元素的 `border` 属性不是必须的。我们可以使用 `border-image-width` 属性，使边框图像有相应的展示区域。稍后我们会介绍到 `border-image-width` 属性。

  


还有一点需要知道的是，元素同时设置 `border` 和 `border-image` 时，**容易不小心覆盖掉图像边框**。[根据 CSS 背景与边框模块（Backgrounds and Border Module Level 3）规范所述](https://www.w3.org/TR/css-backgrounds-3/#border-images)，`border-image` 应该会替换掉你定义的常规边框（`border`），但实际情况不总是如此。试试下面的代码，你会发现最终只显示了一个绿色边框：

  


```CSS
.box {
    border: 30px solid orange;
    border-image-source: var(--image);
    border: 30px solid green;
}
```

  


只有当我们注释掉 `border-image-source` 属性之后的 `border` ，你才能看到边框图像在元素的边框区域呈现：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2b003623ca7a4c928dc5bf986d19a299~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1110&h=494&s=233185&e=gif&f=112&b=010101)

  


这说明，在处理 `border-image` 时，声明的顺序非常重要。正确的顺序是，**`border-image`** **要在** **`border`** **之后声明**。对于初次接触 `border-image` 的开发者来说，这种行为可能会令你感到困惑。

  


为了避免 `border-image` 与 `border` 产生冲突，我个人建议在实际开发过程中，尽可能的避免在同一个元素上同时设置 `border` 和 `border-image` ，这样可以避免覆盖掉我们想创建的图像边框，并且仅用一个属性来控制元素的边框样式。换句话说，当你在使用 `border-image` 属性，碰到了奇怪的结果，首要检查的是否在某处声明了 `border` ，并与 `border-image` 发生了冲突。

  


### 切分边框图片

  


**切分边框图片**是 `border-image` 中一个非常重要的概念，也是令很多 Web 开发者感到头痛之处。简单地说，切分图片决定了边框图片的哪些部分会显示在元素边框区域的哪些位置，是角落还是边缘。例如，你把我们示例中所用的边框图片想象成一张纸，然后你使用剪切在距纸的四边剪了四刀。我使用下图来向大家展示，毕竟一图胜过千言万语：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/87b50838cd444376b39e09baf502ad3f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3069&h=2100&s=1418289&e=jpg&b=ffffff)

  


我们可以使用 `border-image-slice` 属性来切分边框图片。正如上图所示，它使用四条虚拟线来切分边框图片，这些线分别位于边框图片边缘指定的距离（上图位于 `400` 个单位处进行切分）。四条切分线会将边框图片分为九个区域：四个角、四条边和一个中间区域，这就是常说的九宫格。这些区域将决定了边框中交显示哪些部分的边框图片。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b98bc28f2194a03b6367b69544fa56d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3069&h=2100&s=1562377&e=jpg&b=ffffff)

  


```CSS
.box {
    border: 30px solid orange;
    border-image-source: var(--image);
    border-image-slice: 400;
}
```

  


`border-image-slice` 属性会将切分出来的边框图片按下图的顺序放置在边框对应的区域：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5d315b6f38a140ba8ea6059f821f5245~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3069&h=3414&s=2056030&e=jpg&b=ffffff)

  


最终，元素 `.box` 拥有类似下图这样的图片边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0c4ad6c80bd14d36835eda9620bd051a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=174893&e=jpg&b=fffefe)

  


现在效果好多了！需要注意的是，切分后的效果会因图片的不同而有所差异，因此你可以尝试不同的切分值，以达到最佳效果。

  


默认情况下，`border-image-slice` 会丢弃边框图片的中间区域（即图中第 `9` 个单元格对应的图片）。当然，要是你想保留它，可以在切分值后加上关键词 `fill` ：

  


```CSS
.box {
    border: 30px solid orange;
    border-image-source: var(--image);
    border-image-slice: 400 fill;
}
```

  


这个示例中的选择的边框图片可能看不出 `fill` 关键词的效果。我们更换另一张边框图片来演示带有 `fill` 关键词的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/40d365168ba14138ae86035a83401045~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=446739&e=jpg&b=fefcfc)

  


我们可以通过 [MDN 提供的 `border-image` 生成器](https://mdn.github.io/css-examples/tools/border-image-generator/)来展示 `fill` 设置与否的差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce6721ef3ddf407799db4864f16e2f70~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1344&h=820&s=1294050&e=gif&f=191&b=fcfafa)

  


利用这个 `fill` 关键词，我们可以将 `border-image-slice` 的切分值设置为 `0` ，并显式设置关键词 `fill` 。此时，整个边框图片都将位于九宫格的第九格中，那么它将完整的填充在元素中。从视觉上看，它所呈现的效果与 `background` 等同：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3b942d1ee5414552baaf71cff57b3a06~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1350&h=820&s=2843532&e=gif&f=519&b=fdfcfc)

### 调整宽度

  


之前提到过，只有元素显式设置了 `border` （至少是 `border-style` 和 `border-width`），`border-image` 才能生效。就这一点而言，很多 Web 开发者一直也是这么认为的。事实上，并非如此，我们可以使用 `border-image-width` 来决定边框图像的厚度，它将会影响边框图像在边框区域内的缩放效果。换句话说，使用 `border-image-width` 可以设置边框图像的宽度（有点类似于 `border-width`）：

  


```CSS
.box {
    border-image-source: var(--image);
    border-image-slice: 420;
    border-image-width: 30px;
}
```

  


需要知道的是，`border-image-width` 属性将会影响边框图像在边框区域内的缩放效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a4f26b6461314d1a9948de032757a364~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1152&h=654&s=677841&e=gif&f=158&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/abeYPWx

  


特别声明，如果元素未显式设置 `border` ，那么元素必须显式设置 `border-image-width` ，否则 `border-image` 不会有任何效果。

  


你会发现，下面两段代码所获得的效果是相同的：

  


```CSS
.box {
    border: 30px solid #0000;
    border-image-source: var(--image);
    border-image-slice: 420;
}

.box {
    border-image-source: var(--image);
    border-image-slice: 420;
    border-image-width: 30px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0c9d12825fb9404888c8d7b9a8f069c1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=174893&e=jpg&b=fffefe)

  


此时，你可能会有另一个困惑：给元素添加 `border-image-width` 有什么用呢？这里简单的解释一下。在第一段代码中，我们给元素显式设置了 `border` 样式，使元素边框的厚度为 `30px` ，但并没有显式设置 `border-image-width` 。此时，浏览器会认为 `border-image-width` 的值为 `1` （它的初始值，注意是不带任何单位的 `1`）。这意味着边框图像的宽度（`border-image-width`）将等于相应 `border-image-slice` 的固有宽度或高度（取决于适用情况）。如果图像没有所需的固有尺寸，就会使用对应的 `border-width` 值。

  


这个时候，你可能又会问：`border-image-width` 和 `border-width` 又有什么区别呢？简单地说，`border-image-width` 控制边框图像的宽度，即图像在边框区域内的缩放效果；而 `border-width` 定义了元素边框区域（即 `border-box`）的大小，也就是我们常说的元素边框的厚度。

  


虽然我在之前强调过，为了避免 `border-image` 被 `border` 覆盖，我们在使用过程中尽可能的只使用 `border-image` 。但话又说回来，它们是可以共存的，只是需要注意书写顺序，需要确保 `border-image` 始终出现在 `border` 属性之后。例如：

  


```CSS
.box {
    border: 30px solid #0000;
    border-image: var(--image) 420 / var(--w, 30px);
}
```

  


上面的代码等同于：

  


```CSS
.box {
    border: 30px solid #0000;
    border-image-source: var(--image);
    border-image-slice: 420;
    border-image-width: var(--w, 30px);
}
```

  


这意味着元素 `.box` 同时设置了 `border-width` 和 `border-image-width` 值，并且初始状态下，它们的值相等，都是 `30px` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/772cc17274a9415f9c26698f73bd5f2c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=262877&e=jpg&b=ffffff)

  


上图中浅蓝色区域对应的是元素的边框区域（`border-box`），即 `border-width` 属性设置的大小。注意，这是 `border-image-width` 和 `border-width` 相等情况下的效果。那么它们不相等的情况下，又会有什么效果呢？我们拖动滑块来改变 `border-image-width` 属性的值：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8c0efad860ce48e3803a8bab528f6e7b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1382&h=640&s=1863593&e=gif&f=272&b=ffffff)

  


不难发现，`border-image-width` 决定了边框图像如何在分配的边框区域内缩放：

  


-   如果 `border-image-width` 大于 `border-width` ，边框图像会向内超出边框区域，甚至会覆盖到内边距区域（`padding-box`）或内容区域（`content-box`）
-   如果 `border-image-width` 小于 `border-width` ，边框图像可能无法填满整个边框区域，并且可能因为被挤压进较小的空间而出现失真或缩小的效果

  


### 边框图片超出边框

  


`border-image` 还提供了一个 `border-image-outset` 属性，它允许你将边框图片延伸到边框盒子（`border-box`）之外。它有点类似于 `outline-offset` ，但 `border-image-outset` 只接受正值。例如下面这个示例，你可以尝试着拖动滑块来调整 `border-image-outset` 的值，查看边框图片在边框区域的变化：

  


```CSS
.box {
    border: 30px solid #0000;
    border-image-source: var(--image);
    border-image-slice: 420;
    border-image-width: 30px;
    border-image-outset: var(--w, 30px);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3fea9da818bc44ba89340aafd5e0057f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1388&h=642&s=1255617&e=gif&f=270&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/RwXyzoB

  


温馨提示，上图中的浅绿色就是 `border-image-outset` 对应的大小（通过伪元素模拟出来的一个可视区域）。

  


在上面代码中，图像宽度（`border-image-width`）和 边框宽度（`border-width`）都是 `30px` ，整个边框图片会填满元素的边框区域。同时，`border-image-outset` 的值为 `30px` ，它会使边框图片向边框盒子的外边缘延伸 `30px` 。最终，边框及其延伸部分的总宽度为 `60px` （即 `border-image-outset` 和 `border-image-width` 的和或者 `border-image-outset` 和 `border-width` 的和）。

  


注意，在使用 `border-image` 简写属性时，虽然规范对于 `border-image-width` 和 `border-image-outset` 的书写顺序没有严格的要求，但我个人还是建议你将 `border-image-outset` 书写在 `border-image-width` 属性之后。哪怕是不使用简写属性，也尽可能保持这个习惯，因为良好的一个书写习惯，有的时候可以避免带来一些不必要的麻烦。

  


### 控制布局

  


`border-image` 还有一子属性 `border-image-repeat` ，它与 `background-repeat` 非常类似，可以用来定义切分后的边框图像部分如何沿着边框区域排列。为了便于大家更好理解 `border-image-repeat` 的工作方式，我在这里换过一张边框图像，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0b64512beb5a4cc991c4836186c3e37d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=640&h=640&s=70686&e=png&b=fffdfd)

  


然后使用 `border-image-slice` 分别距离边框图像的边缘的 `70` 个单位处进行切分，确保每个角落的橙色花瓣是完整的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a5e118e2913a483b8826c18ff46d9f4c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2100&s=936528&e=jpg&b=ffffff)

  


接下来，按照之前的方式，我们使用 `border-image-width` 设置边框图片宽度为 `30px` ：

  


```CSS
.box {
    width: 90vw;
    aspect-ratio: 21 / 9;
    
    border-image-source: var(--image);
    border-image-slice: 70;
    border-image-width: 30px;
}
```

  


注意，示例中采用的边框图像尺寸是 `640px x 640px` ，元素 `.box` 的宽度是 `90vw` ，而且宽高比是 `21 / 9` 。通过上面代码处理之后，元素具有一个类似下图这样的图像边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3c9a6609ddf4446699444b60ea2ddb0d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=290148&e=jpg&b=ffffff)

  


非常明显，切分后的图像在相应的边框区域被拉伸了。这是因为切分后的边框图像所在部分的尺寸小于边框区域，其中 `5 ~ 9` 区域更明显，为了使切分后的边框图像能填满所在区域，浏览器默认会对它们做拉伸处理。

  


为了使切分后的边框图像部分能在所对应区域有一个更好的展示，我们可以通过 `border-image-repeat` 属性来对它们进行调整。它与我们熟悉的 `background-repeat` 工作原理相似，可以通过以下几个值来调整它们在边框区域的布局（排列）：

  


-   **`round`**：使图像在边框周围重复，但不会裁剪，必要时图像部分会适当拉伸以完美填充边框区域
-   **`space`**：通过添加额外的空白而不是拉伸来确保图像适配边框
-   **`stretch`**：不重复图像，而是拉伸以填满整个边框区域
-   **`repeat`**：将图像重复填充边框，但如果图像不完全适配边框区域，可能会被裁剪

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/03d3282e53b740d6a94c9e42f4a51ac6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1270&h=570&s=1409212&e=gif&f=227&b=fffefe)

  


> Demo 地址：https://codepen.io/airen/full/PoMapMW

  


如果给 `border-image-slice` 属性添加 `fill` 关键词之后，调整 `border-image-repeat` 将会看到类似下图的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a216da4572d4105815acbc425e9c66a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1270&h=572&s=6439381&e=gif&f=263&b=fefbfa)

  


注意，通常在设置 `border-image` 属性时，`border-image-repeat` 会放在最后，因为它决定了边框图像在元素的边框区域中的最终布局。

  


### 注意事项

  


现在，我们知道了 `border-image` 以及它的子属性 `border-image-source` 、`border-image-slice` 、`border-image-width` 、`border-image-outset` 和 `border-image-repeat` 的特性与应用。如果你觉得它们不太好理解，那么你可以与我们熟悉的一些属性结合起来理解：

  


-   `border-image-source` 类似于 `background-image` ，它的值可以是 `<image>` 或 CSS 渐变
-   `border-image-slice` 相当于对边框图像裁剪了四次，会将边框图像切分成一个九宫格。它的值可以是 `1 ~ 4` 个值，类似于 `padding` 和 `margin` 。除此之外，可以在数值之前或之后设置关键词 `fill`
-   `border-image-width` 类似于 `border-width` ，它的值也可以是 `1 ~ 4` 个值
-   `border-image-outset` 类似于 `outline-offset` ，它的值也可以是 `1 ~ 4` 个值，类似于 `padding` 和 `margin` ，但它的值不可以是负值
-   `border-image-repeat` 类似于 `background-repeat` ，它的值可以是一个，也可以是两个。当它的值是两个时，其中第一个值将用于指定边框图像水平方向的排列布局（类似于 `background-repeat-x`），第二个值将用于指定边框图像垂直方向的排列布局（类似于 `background-repeat-y`）

  


如果你不想分开写，你也可以使用简写属性 `border-image` 。在使用简写属性时，不是所有子属性是必须的。不过需要注意的是，`border-image-slice` 与 `border-image-width` 和 `border-image-outset` 之间会使用斜杠（`/`）分隔，这个类似于 `background-position` 与 `background-size` 一样。正因此，当使用简写的 `border-image` 属性时，如果缺省 `border-image-width` ，你会发现 `border-image` 会有两个斜杠（`//`），并且它们之间没有任何值。

  


```CSS
.box {
    border: 30px solid #0000;
    border-image: var(--image) 70 fill //30px round;
} 
```

  


上面的代码等同于：

  


```CSS
.box {
    border: 30px solid #0000;
    border-image-source: var(--image);
    border-image-slice: 70 fill;
    border-image-width: 1; /* 初始值 */
    border-image-outset: 30px;
    border-image-repeat: round;
}
```

  


需要注意的是，当元素同时未显式设置 `border` （至少是 `border-width` 和 `border-style`）和 `border-image-width` ，并且 `border-image-slice` 值没有设置关键词 `fill` ，那么边框图像不会在元素上有任何显示：

  


```CSS
/* 不会显式边框图像 */
.box {
    border-image-source: var(--image);
    border-image-slice: 70;
    border-image-outset: 30px;
    border-image-repeat: round;
}

/* 边框图像第9个部分会填满元素的 border-box 内边缘的所有区域，视觉上类似于 background */
.box {
    border-image-source: var(--image);
    border-image-slice: 70 fill;
    border-image-outset: 30px;
    border-image-repeat: round;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/537b7b6599f34180a01cef28f9db4f71~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=390345&e=jpg&b=020202)

  


> Demo 地址：https://codepen.io/airen/full/poMKPbO

  


因此，当你发现边框图片未生效时，可以检测元素是否显式设置了 `border` （建议是一个透明边框）或 `border-image-width` 。我个人建议设置 `border-image-width` ，这样可以避免在元素上同时使用 `border` 属性。因为，在元素上同时使用 `border` 和 `border-image` 属性时，在视觉上，出现在后面的属性会覆盖掉出现在前面的属性：

  


```CSS
/* 你看到的是一个 30px 宽度的红色边框 */
.box {
    border-image: var(--image) 70 / 30px round;
    border: 30px solid red;
}

/* 你看到的是一个 30px 宽度的图像边框 */
.box {
    border: 30px solid red;
    border-image: var(--image) 70 / 30px round;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d27a4146570a424ca27a44f3624e9d8e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=293369&e=jpg&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/MWNXmjz

  


还有一点需要知道的是，`border-image-slice` 的值可以是 `0` 。当 `border-image-slice` 的值设置为 `0` 时，切分后的边框图像中的 `1 ~ 8` 个部分的尺寸都是 `0` （可视为沿着边框图像四个边缘进行切分）。此时，整个边框图像都位于第 `9` 个区域：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eb5b816c56be4d63a0ea694c81c2c908~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2100&s=935413&e=jpg&b=ffffff)

  


这个时候，如果 `border-image-slice` 不带 `fill` 关键词，边框图像在元素上不会有任何显示；反之元素的边框区域会露出元素的背景， 边框区域之内（`border-box` 区域内边缘所在区域）会显示边框图像：

  


```CSS
/* 边框图像不会有任何显示 */
.box {
    border-image: var(--image) 0 / 30px round;
}

/* 边框图像看上去像元素的背景图片(background-clip:padding-box) */
.box {
    border-image: var(--image) 0 fill / 30px round;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7bda50fcde5643d9b94dcbb9eaff0499~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=320484&e=jpg&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/yLmEbgQ

  


与此同时，元素不显式设置 `border-image-width` ，边框图像应用在元素上时，就类似于元素的背景图片：

  


```CSS
.box {
    border-image: var(--image) 0 fill  round;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1b71398c23a64399a9b0ec62afca683b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=978&s=393380&e=jpg&b=010101)

  


最后一个注意事项是 **`border-image`** **会绘制在元素的背景（** **`background`** **）和** **`box-shadow`** **之上，但在元素内容的下方**。

  


```CSS
.box {
    background: linear-gradient(45deg, #90acef, #90efac);
    box-shadow: inset 20px 20px 1em rgb(90 90 90 / .5);
    border-image: var(--image) 70  / 30px round;
}
```

## `border-image` 的应用

  


在课程最开始的时候，就提到过，小册之前很多案例都有 `border-image` 属性的身影，例如[创建心形图形](https://juejin.cn/book/7395119173656903715/section/7398183482347749428)、[丝带形状](https://juejin.cn/book/7395119173656903715/section/7399654560878788658)、[自定义滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)、[提示框或语音气泡](https://juejin.cn/book/7395119173656903715/section/7408966599622787083)、[区域或标题分隔符](https://juejin.cn/book/7395119173656903715/section/7406965718895296553)、[内弧圆角选项卡](https://juejin.cn/book/7395119173656903715/section/7411342285018136587)、[多重边框](https://juejin.cn/book/7395119173656903715/section/7411340113839390783)、[渐变边框](https://juejin.cn/book/7395119173656903715/section/7413371655377911835) 和 [突破元素](https://juejin.cn/book/7395119173656903715/section/7418491025368612901)等等。

  


接下来，我们来通过实际案例向大家展示 `border-image` 在实际开发中的应用以及一些技巧。

  


### 渐变叠加

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8948c04c9968471cb1e3a18fa7f4b47e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1080&h=683&s=67600&e=webp&b=f5f0ee)

  


在 Web 开发中，很多时候文本会叠加在图片或视频等媒体元素上面。为了提高文本的可读性，往往会在图片或视频与文本之间叠加一个渐变层。这是 Web 中常见的一种设计模式。

  


在 CSS 中有很多种方法可以实现文本和内容之间的叠加效果，其中最常用的方法是在它们之间插入一个带有渐变的效果的伪元素。现在，我们可以使用 `border-image` 来实现这个效果，而且只需一行代码即可：

  


```CSS
h3 {
    border-image: linear-gradient(#0000, #000) 0 fill;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd6958e0bf024241bed9e8d92e23ac94~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=347846&e=jpg&b=8cb5c7)

  


> Demo 地址：https://codepen.io/airen/full/jOgKJgW

  


在这个示例中，我们可以看到 `border-image-source` 是一个线性渐变 `linear-gradient()` ，并且 `border-image-slice` 为 `0 fill` 。这相当于 `1 ~ 8` 切片都是空的，而中间的切片（第 `9` 个）则填充整个渐变。就好像只有一个切片一样。同时，我们也没有显式定义 `border-image-width` 和 `border-image-outset` ，并且也未显式设置 `border` 。此时，`border-image-width` 会等于 `0` （因为 `border-width` 未显式设置，它被视为 `0`），因此 `1 ~ 8` 区域的大小为 `0` ，只留下中间区域（整个 `border-image` 的区域）。由于我们没有指定 `border-image-outset` ，因此 `border-image` 的区域等于元素的区域。

  


最终，我们在中间区域放置了一个切片，而另外八个切片则完全看不见。而且 `border-image` 会绘制在元素背景之上，同时位于文本内容下方，因此渐变会位于文本之下。从而实现一个渐变叠加的效果，提高了文本的可读性。

  


现在，我们来尝试一种不同的语法，但输出结果是相同的：

  


```CSS
h3 {
    border-image: linear-gradient(#0000, #000) 50% / 50%;
}
```

  


上面的代码设置了 `border-image-slice` 和 `border-image-width` 的值为 `50%` ，而没有显式设置 `border-image-outset` 。此时，`border-image-slice` 切分之后的边框图像（渐变），其中 `1 ~ 4` 切片各占边框图像的四分之一。与此同时，切片 `5 ~ 9` 是空的（它们的大小等于 `0`）。根据 `border-image-slice` 的切分逻辑，切片 `1 ~ 4` 各占元素区域的四分之一，它们刚好填满了整个元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6835d2534dbd46c7b04a2d7e2d0d5bb3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=507301&e=jpg&b=8cb6c6)

  


> Demo 地址：https://codepen.io/airen/full/mdNKgmN

  


正如你所看到的，虽然输出的结果是相同的，但 `border-image` 使用了不同的方法。

  


这个示例的诀窍是首先理解渐变是如何被分割成不同的切片，然后理解这些切片是如何被放置到不同的区域，而这些区域的大小由 **`border-image-width`** **和** **`border-image-outset`** 决定。掌握这个机制可能需要一些时间，但一旦掌握，就会觉得非常有趣！

  


温馨提示，在这个示例中，[整个卡片使用了 CSS Grid 进行布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)，使用 `img` 和 `h3` 位于同一网格区域，[从而实现类似绝对定位的效果](https://juejin.cn/book/7161370789680250917/section/7161623932439625758)。另外，就我们这个示例而言，你也可以使用 `background` 来替代 `border-image` ，它们达到的效果是相似的。

  


要是你仔细阅读了示例源码，你可能会发现，我在 `h3` （应用了 `border-image` 属性的元素上）使用了 `clip-path` 的 `inset()` 函数裁剪出一个带有圆角的效果。这是因为，元素的 `border-radius` 并无法直接使边框图片具有圆角效果。

  


```CSS
h3 {
    border-image: linear-gradient(#0000, #000) fill 0; 
    clip-path: inset(0 round 18px);
}
```

  


### 全宽背景

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3ffcfee688444f74b7b111048bfae437~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=768&h=966&s=165616&e=png&a=1&b=fbfafa)

  


继续以 Web 中常见的设计模式为例，我们希望元素的背景延伸到屏幕的两侧。正如小册的《[CSS 图形：创建突破元素](https://juejin.cn/book/7395119173656903715/section/7418491025368612901)》一课所述，CSS 有很多方法可以实现这个效果。常见的一种方式是通过调整 HTML 的结构来实现，但很多时候，Web 开发者可能无法自由控制 HTML 的结构，那么要实现上图这种效果就有点棘手。

  


例如，我们现在有一个类似下面这样的 HTML 结构：

  


```HTML
<div class="wrapper">
    <header></header>
    <section class="hero"></section>
    <main></main>
    <footer></footer>
</div>
```

  


并且 CSS 给容器元素 `.wrapper` 设置了一个宽度：

  


```CSS
.wrapper {
    width: 100%;
    max-width: 1360px;
    margin-inline: auto;
}
```

  


如此一来，`.wrapper` 所有子元素的宽度都限制在其父元素的宽度内。假设你现在需要将 `.hero` 的背景突破其父容器 `.wrapper` 的宽度限制，以便延伸到浏览器视窗的两侧。

  


刚才也提到过了，方法有很多。但我想你已猜到了，在这里我们来使用 `border-image` 实现它。我们既可以不调整任何 HTML 的结构，并且仅一行 CSS 代码就可以实现该效果：

  


```CSS
.hero {
    border-image: linear-gradient(#69cde3,#69cde3) fill 0 // 0 100vw;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f2d8e26ee5f744baa27a59b0f0d8bb57~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=412058&e=jpg&b=8cb5c7)

  


> Demo 地址：https://codepen.io/airen/full/OJKEYJW

  


你可能会想，全宽背景是一个纯色，这里为什么要使用一个渐变呢？这是因为，`border-image` 中并不像 `background` 属性，有一个名为 `background-color` 的属性。到目前为止，它仅支持 `<image>` 值类型，所以我们在这里使用线性渐变来创建一个纯色边框图片。注意，你也可以使用其他 CSS 渐变来创建纯色边框图片，例如锥形渐变 `conic-gradient(#69cde3 0 0)` 。

  


与上一个示例不同之处是，我们设置了 `border-image-outset` 的值为 `0 100vw` ，它将 `border-image` 的区域从左右两侧扩展到视口宽度的 `100%` （即 `100vw`）:

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/02e63a4d9358409ba4d12ee8ad38dc27~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=459877&e=jpg&b=8cb5c7)

  


从技术上讲，我们可以使用小于 `100vw` 的值，因为我们需要覆盖的空间很可能更小，但使用较大的值总是更好的，它可以确保我们考虑所有可能性。最为神奇的是，将 `border-image-outset` 设置很大的值不会触发滚动条。

  


需要注意的是，你可能想到了使用 CSS 的无穷值 `infinity` 来替代 `100vw` ，但我实测下来，它应用于 `border-image-outset` 中并无法达到我们预期的效果。

  


```CSS
/* 请不要这样使用 */
.hero {
    border-image: linear-gradient(#69cde3,#69cde3) fill 0 // 0 calc(infinity * 1px);
}
```

  


另外，上面示例演示的是一个渐变创建的一个纯色边框图像，但实际使用的时候，你可以将纯色渐变替换成任何你喜欢的边框图像，比如下面这个示例，使用 `repeating-linear-gradient()` 创建的斑马斜纹条：

  


```CSS
.hero {
    border-image: repeating-linear-gradient(45deg,#cde 0 15px,#0000 0 30px) fill 0 // 0 100vw;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/060792bf5d8f46a3b7cde5f451014934~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=642436&e=jpg&b=98bdcf)

  


> Demo 地址：https://codepen.io/airen/full/qBeyWEz

  


正如你所看到的，[这个技巧也可以用来制作区域分隔符](https://juejin.cn/book/7395119173656903715/section/7406965718895296553#heading-2)。我们可以将 `border-image` 与 `clip-path` 结合起来：

  


```CSS
main {
    --a: 3deg; /* 控制角度（应保持较小） */
  
    border-image: conic-gradient(#FFC107 0 0) fill 0 // 9999px;
    clip-path: 
        polygon(
            -9999px calc(tan(var(--a)) * 9999px),
            9999px calc(tan(var(--a)) * -9999px),
            calc(100% + 9999px) calc(100% - tan(var(--a)) * 9999px),
            calc(100% - 9999px) calc(100% + tan(var(--a)) * 9999px)
        );
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/599b0c3a453c46928c91a4563db0438f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=910627&e=jpg&b=a0c1d4)

  


> Demo 地址：https://codepen.io/airen/full/eYqjOzQ

  


在这个示例中，`border-image-outset` 在所有方向设置了一个相当大的值 `9999px` 。这迫使颜色溢出元素的边界，使我们可以使用 CSS 的 `clip-path` 来裁剪出所需要的形状。在使用 `clip-path` 的 `polygon()` 函数裁剪倾斜四边形形状时，还用到了 [CSS 的三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)。如果你对这方面感兴趣，[可以点击这里获取更详细的信息](https://juejin.cn/book/7395119173656903715/section/7406965718895296553#heading-3)。

  


虽然这部分演示的是使用 `border-image` 制作全宽背景，但我们还可以利用这个技巧来[创建一些“突破元素”的效果](https://juejin.cn/book/7395119173656903715/section/7418491025368612901)。例如，丝带标题的一侧扩展到浏览器视窗的一侧：

  


```CSS
.ribbon {
    --c: #c7f464;

    border-image: conic-gradient(from 45deg at calc(100% - 1lh), #0000 25%, var(--c) 0) fill 0 //0 0 0 100vw;;
    padding-right: 1.3lh;
    line-height: 2;
    width: fit-content;
    color: #333;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a3fb07e9286a4972a5c621c9e9f0b5b8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=668484&e=jpg&b=99bed0)

  


> Demo 地址：https://codepen.io/airen/full/rNXrByP

  


这次我们使用 `border-image-outset` 扩展了左侧的边框区域，即 `0 0 0 100vw` 。按照相似的原理，你可以在任意方向扩展，实现你所需要的效果。注意，这个示例中使用了[锥形渐变 `conic-gradient()`](https://juejin.cn/book/7223230325122400288/section/7259668771856941111) 创建了一个类似丝带的图像作为边框图像。需要知道的是，在 CSS 中有很多种不同的方式可以创建丝带形状，如果你感兴趣的话，请返回去阅读小册的《[CSS 图形：创建丝带效果](https://juejin.cn/book/7395119173656903715/section/7399654560878788658)》一课。

  


除此之外，我们还可以利用这个技巧来处理图像效果，例如给用户头像添加无限扩展的阴影效果：

  


```CSS
img {
    --b: 8px;
    --w: 200px;

    --_w: calc((sqrt(2) - 1) / (2 * sqrt(2)) * var(--w));
    width: var(--w);
    aspect-ratio: 1;
    border-radius: 50%;
    padding: var(--b);
    box-sizing: border-box;
    box-shadow: 0 0 0 var(--b) inset var(--c);
    --_c: conic-gradient(var(--c) 0 0) fill 0;

    &.top-left {
        border-image: var(--_c) / 0 var(--_w) var(--_w) 0/999px 0 0 999px;
        clip-path: polygon(120.71% 50%,calc(120.71% - 999px) calc(50% - 999px),calc(50% - 999px) calc(120.71% - 999px), 50% 120.71% );
    }
    
    &.top-right {
        border-image: var(--_c) / 0 0 var(--_w) var(--_w) / 999px 999px 0 0;
        clip-path: polygon(-20.71% 50%,calc(999px - 20.71%) calc(50% - 999px),calc(50% + 999px) calc(120.71% - 999px),50% 120.71%);
    }
    
    &.bottom-left {
        border-image: var(--_c) / var(--_w) var(--_w) 0 0/0 0 999px 999px;
        clip-path: polygon(50% -20.71%,calc(50% - 999px) calc(999px - 20.71%),calc(120.71% - 999px) calc(50% + 999px),120.71% 50%);
    }
    
    &.bottom-right {
        border-image: var(--_c) / var(--_w) 0 0 var(--_w) / 0 999px 999px 0;
        clip-path: polygon(50% -20.71%, calc(50% + 999px) calc(999px - 20.71%),calc(999px - 20.71%) calc(50% + 999px),-20.71% 50%);
    }
}
```

### 为图像添加内边框圆角

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0a22fae92f1e4607817c92102a6f880d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=456098&e=jpg&b=8eb7c8)

  


以往我们要实现上图所示的圆角效果，往往是通过添加伪元素来实现。现在，我们可以结合 `border-radius` 和 `border-image` 来实现。

  


之前我提到过，如果元素同时应用了 `border-image` 和 `border-radius` ，元素盒子在浏览器中渲染出来的结果并不会有任何圆角效果。这是因为 `border-image` 会直接穿透 `border-radius` 。这种现象在某些情况下可能是一个缺陷，但在某些场景中它却是优点。例如，我们创建具类似上图所示，带有内圆角的图像效果。

  


```CSS
img {
    --c: #ff90ce;
    --s: 2rem; 
  
    width: 250px;
    border-radius: 50%; 
    border-image: conic-gradient(var(--c) 0 0) fill 0//var(--s);
    
    &:nth-child(2) {
        border-radius: .5em;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/498519fa75184a57820e407dac13be73~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1116&s=456098&e=jpg&b=8eb7c8)

  


> Demo 地址：https://codepen.io/airen/full/ZEgjEMO

  


现在，你只需要调整 `border-radius` 属性的值和 `--s` 变量，可以使图像具有不同半径的内边框圆角效果。

  


### 渐变边框

```CSS
.gradient--border {
    --b: round(0.5em, 1px);
    --r: var(--b);
    --s: 40vw;
    --c: red;
    --a: 180deg;
    --g: conic-gradient(from var(--a) in oklch longer hue, var(--c) 0 0);
    --bg: color-mix(in srgb, var(--c), #000 60%);
    --fs: 1rem;
    border-image: var(--g) 1 / var(--b);
    clip-path: inset(0 round var(--r));
}
```

## 结论

  


通过这节课的学习，我们你对 `border-image` 会有一个更全面的认识。尽管理解和使用 `border-image` 具有一定的挑战性，但它的确可以帮助我们做很多事情，而且还可以保持代码的简洁和清晰。简而言之，它是一个复杂而强大的 CSS 特性。

  


如果你在理解 `border-image` 的切片逻辑和延伸扩展有一定的困难，请不用担心。这是很正常的。我也花了很多时间才完全理解 `border-image` 的工作原理以及如何以不同的语法来使用它。你也要给自己足够的时间。多次重复阅读这节课的内容，将有助于你更好地理解 `border-image` 。

  


抛开 `border-image` 的复杂性不说，我更希望你能在实际的开发过程使用 `border-image` ，因为它真的非常强大，除了能创建丰富多样的 UI 效果之外，还能真正的帮助你解决实际问题。
