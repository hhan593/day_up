在许多 Web 网站或应用程序中，我们需要为了美观原因对元素进行切割。现在，我们可以使用 CSS 渐变、剪切和遮罩对元素的角落进行切割。有很多种技巧可以从任何元素的角落切出不同的形状，比如角度切割、圆形切割和曲线切割。在这节课中，我们将考虑现代技术来创建独特的角落形状，同时尝试使用可重用的代码，通过调整变量来产生不同的结果。

## 角度切割

  


角度切割是最简单的一种形状。我们从最常见，最简单的一种形状开始，即元素角度直接直线切除：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/03dc4fe1d3ba49baa52fdd0f53ad0d06~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1514&s=242509&e=jpg&b=ed6137)

  


为了获得这种形状，我们可以直接使用 [CSS 的 `clip-path` 属性](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-1)像下图这样对元素进行裁剪，其中切割的大小使用 CSS 自定义属性来定义，例如 `--s` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/17b569bdf59444849fadacf77dd0eed8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1514&s=357189&e=jpg&b=ffffff)

  


```CSS
.notch {
    --s: 25px; /* 控制切口大小 */
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf,#333); /* 控制文本颜色 */
    
    clip-path: polygon(
        var(--s) 0, /* P1 */
        calc(100% - var(--s)) 0, /* P2 */
        100% var(--s), /* P3 */
        100% calc(100% - var(--s)), /* P4 */
        calc(100% - var(--s)) 100%, /* P5 */
        var(--s) 100%, /* P6 */
        0 calc(100% - var(--s)), /* P7 */
        0  var(--s) /* P8 */
    );
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b216a372cca545b3aa1376bda6e7dc8d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1150&h=678&s=3198103&e=gif&f=279&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/eYwEgWr

  


这是一个特殊的角度切割，`.notch` 元素的每个角落切割了一个等腰直角三角形，它的腰长为 `--s` 。

  


除了使用 CSS 的 `clip-path` 之外，我们还可以使用 [CSS 的 `mask` 属性](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)实现：

  


```CSS
.notch {
    --s: 25px; /* 控制切口大小 */
    
    --mask: 
        conic-gradient(from 45deg at calc(var(--s) * 2) calc(var(--s) * 2),#000 75%,#0000 0) calc(var(--s) * -2) 0/100% 51% repeat-x,
        conic-gradient(from 135deg at calc(var(--s) * 2) calc(100% - calc(var(--s) * 2)),#0000 25%,#000 0) calc(var(--s) * -2) 100%/100% 51% repeat-x;        
    mask: var(--mask);
}
```

  


从代码的角度来说，它要比 `clip-path` 复杂一些，但最终所呈现的结果是一致的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/509c2ed9e46b409ea1894f1637e08f3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1256&h=754&s=3373687&e=gif&f=255&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/qBzXRwB

刚才提到过，上面这两个示例所展示的都是一个特殊的角度切割。那么，我们就从普通的角度切割着手，一步一步来看看 CSS 的 `mask` 是如何帮助我们对元素进行角度切割，并且实现我们想要的效果。

  


通常情况之下，角度切割会包含切割的大小（`--s`）和角度（`--a`）这两个参数。为了获得这种形状，我们需要一个[锥形渐变](https://juejin.cn/book/7223230325122400288/section/7259668771856941111)（`conic-gradient`）来处理每个角落。我们把这个过程分解开来看。首先将 `conic-gradient` 应用到元素上，如下面代码所示：

  


```CSS
.notch {
    --s: 25px; /* 控制切割大小 */
    --a: 135deg; /* 控制切割角度 */

    background: 
        conic-gradient(from calc(var(--a) / -2 - 45deg) at top var(--s) left var(--s),#0000 var(--a),red 0) 0 0,
        conic-gradient(from calc(var(--a) / -2 + 45deg) at top var(--s) right var(--s),#0000 var(--a),blue 0) 100% 0,
        conic-gradient(from calc(var(--a) / -2 - 135deg) at bottom var(--s) left var(--s),#0000 var(--a),green 0) 0 100%,
        conic-gradient(from calc(var(--a) / -2 + 135deg) at bottom var(--s) right var(--s),#0000 var(--a),orange 0) 100% 100%;
    background-size: 51% 51%;
    background-repeat: no-repeat;
}
```

  


上面的代码中，我们使用了四个 `conic-gradient` 创建了四个渐变，它们分别占据元素的四分之一区域。尝试着调整 `--s` （切割的大小）和 `--a`（切割的角度），查看 `conic-gradient` 的渐变：

注意，上面的示例为了让大家能更好的理解每个 `conic-gradient` 的作用，特意设置了不同的渐变色。换句话说，如果将它们的颜色统一为 `--bg` 时，整个效果如下图所示：

注意，在这里我们新增了四个不同的 CSS 自定义属性来描述相应的 `conic-gradient` ，这样做的好处是，Web 开发者看到自定义名称，就知道 `conic-gradient` 所创建的角度切割是位于元素的哪个角落，例如 `--bg-top-left` ，它对应的是元素的左上角的角度切割！

  


虽然使用 `conic-gradient` 可以任意你想要的角度切割形状，但它也有一定的局限性。例如，元素的背景是一个渐变色，或者说被裁剪的元素是一张图片时，这种方式就不适合了。庆幸的是，我们可以使用 `mask` 来替代 `background` ：

```CSS
.notch {

    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf, #333); /* 控制文本颜色 */
    --g: #0000 var(--a), var(--bg) 0; /* 渐变颜色 */

    --m-top-left:conic-gradient(from calc(var(--a) / -2 - 45deg) at top var(--s) left var(--s),var(--g));
    /* 右上角遮罩图 */
    --m-top-right:conic-gradient(from calc(var(--a) / -2 + 45deg) at top var(--s) right var(--s),var(--g));
    /* 左下角遮罩图 */
    --m-bottom-left:   conic-gradient(from calc(var(--a) / -2 - 135deg) at bottom var(--s) left var(--s),var(--g));
    /* 右下角遮罩图 */
    --m-bottom-right:conic-gradient(from calc(var(--a) / -2 + 135deg) at bottom var(--s) right var(--s),var(--g));
    mask-repeat: no-repeat;
    background: linear-gradient(to right, var(--bg) 0%, hsl(from var(--bg) h s 60%) 100%);

    /* 左上角角度切割 */
    &:nth-child(1) {
        mask-image: var(--m-top-left);
        mask-position: top left;
        mask-size: 100% 100%;
    }

    /* 右上角角度切割 */
    &:nth-child(2) {
        mask-image: var(--m-top-right);
        mask-position: top right;
        mask-size: 100% 100%;
    }
    
    /* 左下角角度切割 */
    &:nth-child(3) {
        mask-image: var(--m-bottom-left);
        mask-position: bottom left;
        mask-size: 100% 100%;
    }

    /* 右下角角度切割 */
    &:nth-child(4) {
        mask-image: var(--m-bottom-right);
        mask-position: bottom right;
        mask-size: 100% 100%;
    }

    /* 左上角 + 右上角角度切割 */
    &:nth-child(5) {
        mask-image: var(--m-top-left), var(--m-top-right);
        mask-size: 51% 100%;
        mask-position:  top left, top right;
    }
    
    /* 左下角 + 右下角角度切割 */
    &:nth-child(6) {
      mask-image: var(--m-bottom-left), var(--m-bottom-right);
      mask-size: 51% 100%;
      mask-position:  bottom left, bottom right;
    }
    
    /* 左上角 + 右下角度切割 */
    &:nth-child(7) {
        mask-image: var(--m-top-left), var(--m-bottom-right);
        mask-size: 51% 100%;
```

除了前面这两种方案之外，我们还可以使用 `clip-path` 属性来切割元素的角落。只不过，使用 `clip-path` 要复杂的多，每个角落需要三个点来定义，而且[点的坐标还需要通过三角函数进行计算](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/db379fd93dcc4f9c93dfabb7534d168d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1734&s=963226&e=jpg&b=376366)

  


虽然使用 `clip-path` 进行角度切割，我们需要 `12` 个坐标点，但仔细观察，你会发现，这些坐标点具有一定的对称性，例如 `P1` 和 `P2` 围绕着元素中心的 `Y` 轴 对称，因此它们的 `x` 坐标的总和为 `100%` ，`y` 坐标相等。与此同时，`P1` 又和 `P8` 是围绕着元素中心的 `X` 轴对称，因此它们的 `y` 坐标的总和为 `100%` ，`x` 坐标相等。也就是说，我们只需要关注三个点的坐标，再利用图形的对称性，可以快速获得所有点的坐标。

  


正如上图所示，我们通过[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)，计算出 `T` 的值：

  


```
T = (1 - tan((a - 90) / 2)) * s

P1  = (T, 0)
P12 = (s, s)
P11 = (0, T)
```

  


根据图形对称性，获得其他点的坐标：

  


```
T = (1 - tan((a - 90) / 2)) * s

P1  = (T, 0)
P2  = (100% - T, 0)         /* P2 与 P1 绕着 Y 轴对称 */
P3  = (100% - s, s)         /* P3 与 P12 绕着 Y 轴对称 */
P4  = (100%, T)             /* P3 与 P11 绕着 Y 轴对称 */
P5  = (100%, 100% - T)      /* P5 与 P4 绕着 X 轴对称 */
P6  = (100% - s, 100% - s)  /* P6 与 P3 绕着 X 轴对称 */
P7  = (100% - T, 100%)      /* P7 与 P2 绕着 X 轴对称 */
P8  = (T, 100%)             /* P8 与 P1 绕着 X 轴对称 */
P9  = (s, 100% - s)         /* P9 与 P12 绕着 X 轴对称 */
P10 = (0, 100% - T)         /* P10 与 P11 绕着 X 轴对称 */
P11 = (0, T)
P12 = (s, s)
```

  


为了使 `polygon()` 中的坐标点与元素角落相对应，我们调整一下它的起始点，即：

  


```CSS
.notch {
    clip-path: polygon(
        /* 左上角 */
        P11, P12, P1,
        
        /* 右上角 */
        P2, P3, P4,
        
        /* 右下角 */
        P5, P6, P7,
        
        /* 左下角 */
        P8, P9, P10
    )
}
```

  


我们把代码中的 `T` 、`a` 和 `s` 分别使用 [CSS 的自定义属性](https://juejin.cn/book/7223230325122400288/section/7252964839705247755)来描述：

  


```CSS
.notch {
    --s: 25px;   /* 控制角度切割的大小 */
    --a: 45deg;  /* 控制角度切割的角度 */
    --t: calc((1 - tan((var(--a) - 90) / 2)) * var(--s)); /* T = (1 - tan((a - 90) / 2)) * s*/
}
```

到目前为止，我们有三种创建角度切割的方式：

  


-   多个 `conic-gradient` 应用于 `background` ，通过 `background-size` 和 `background-position` 来调整背景图（`conic-gradient` 创建的渐变图片）的位置和尺寸，从而实现单个或多个角度切割的效果
-   多个 `conic-gradient` 应用于 `mask` ，通过 `mask-size` 和 `mask-position` 调整遮罩图（`conic-gradient` 创建的遮罩图）的位置和尺寸，从而实现单个或多个角度切割的效果
-   使用 `clip-path` 的 `polygon()` 创建单个或多个角度切割的效果

  


不管哪一种方案，都可以帮助你创建所需要的角度切割，在实际使用的过程中，应该选择较为更为合适的一种。我个人更推荐第二种（`mask` 加 `conic-gradient`）和第三种（`clip-path`），相对于第一种方案而言，它们要更灵活些。

  


## 圆形切割

  


与角度切割类似，对于圆形或圆角切割，我们也可以使用多个渐变来实现，只不过，我们在这里使用四个径向渐变 `radial-gradient` 来实现。要切割四个角，逻辑上我们要使用四个径向渐变，每个角落一个：

  


```CSS
.circular {
    --s: 25px; /* 控制切口大小 */

    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf,#333); /* 控制文本颜色 */

    /* 左上角 */
    --bg-top-left: radial-gradient(circle var(--s) at top left, #0000 98%, hsl(from var(--bg) h s 20%)) top left;
    /* 右上角 */
    --bg-top-right: radial-gradient(circle var(--s) at top right, #0000 98%, hsl(from var(--bg) h s 40%)) top right;
    /* 左下角 */
    --bg-bottom-left: radial-gradient(circle var(--s) at bottom left, #0000 98%, hsl(from var(--bg) h s 60%)) bottom left;
    /* 右下角 */
    --bg-bottom-right: radial-gradient(circle var(--s) at bottom right, #0000 98%, hsl(from var(--bg) h s 80%)) bottom right;
    background: 
        var(--bg-top-left),
        var(--bg-top-right),
        var(--bg-bottom-left),
        var(--bg-bottom-right);
    background-size: 51% 51%;
    background-repeat: no-repeat;
    color: var(--color);
}
```

简单解释一下。上面代码中每个渐变占据元素尺寸的四分之一，以左上角的渐变为例：

  


```CSS
.circular {
    /* 左上角 */
    --bg-top-left: radial-gradient(
        circle var(--s) at top left, 
        #0000 98%, hsl(from var(--bg) h s 20%)
    ) top left;
}
```

  


它的意思是，将在左上角绘制一个半径为 `--s` 的圆，主要颜色是透明的（`#0000`），其余颜色是 `hsl(from var(--bg) h s 20%)` （可以是任意你想要的颜色）。整个渐变被放置在元素上左上角（`background-position: top left`）。其他三个渐变也遵循相同的逻辑。由于我们明确指定了一个半径值，所以可以省略关键词 `circle` ：

  


```CSS
.circular {
    /* 左上角 */
    --bg-top-left: radial-gradient(var(--s) at top left, #0000 98%, hsl(from var(--bg) h s 20%)) top left;
    /* 右上角 */
    --bg-top-right: radial-gradient(var(--s) at top right, #0000 98%, hsl(from var(--bg) h s 40%)) top right;
    /* 左下角 */
    --bg-bottom-left: radial-gradient(var(--s) at bottom left, #0000 98%, hsl(from var(--bg) h s 60%)) bottom left;
    /* 右下角 */
    --bg-bottom-right: radial-gradient(var(--s) at bottom right, #0000 98%, hsl(from var(--bg) h s 80%)) bottom right;
}
```

  


另外，在这里，第一个渐变颜色（`#0000`）的停止位置设置的是 `98%` 而不是 `100%` ，它可以用来避免锯齿边缘。与此同时，`background-size` 的值设置为 `51%` 而不是 `50%` ，可以使渐变重叠在一起，避免出现露出底色。

  


上面四个渐变采用不同的渐变颜色，主要是为了便于大家更好理解每个渐变所起的作用。在实际使用的时候，我们可以将渐变颜色提取出来，存到一个名为 `--g` 的变量中：

注意，这里没有显式指定 `radial-gradient` 的大小（默认情况下，它的宽度和高度都是 `100%` ，即 `mask-size` 或 `background-size` 的值为 `100% 100%`）。这会在元素的中心会创建一个镂空的圆。我们通过将渐变平移（遮罩图或背景图）元素宽度和高度的一半（`mask-position` 或 `background-position` 从 `0 0` 平移到 `calc(var(--w) / -2) calc((var(--w) / var(--ratio)) / -2)`），将这个圆移动画一个角落。由于默认情况下，`mask-repeat` （或 `background-repeat`）为 `repeat` ，我们在每个角落都会得到相同的效果。使用一个径向渐变可以切换四个角！

  


这种方法唯一的缺点是我们事先需要知道元素的宽度和高度，否则无法计算 `mask-position` 或 `background-position` 的位置。你可能会问，难道不能使用 `-50%` 来替代元素宽度和高度的一半吗？

  


不幸的是，在这里我们不能这么做，因为 `mask-position` 或 `background-position` 的值为百分比时，它的计算非常复杂。我们以 `background-position` 为例（`mask-position` 取值百分比的计算与 `background-position` 相似），`background-position` 取百分比值时，会同时考虑容器的尺寸和背景图片的尺寸，即相对于两者尺寸做计算。

  


[W3C规范中是这样描述的](https://www.w3.org/TR/css3-background/#the-background-origin)：

  


> A percentage for the horizontal offset is relative to (width of background positioning area - width of background image). A percentage for the vertical offset is relative to (height of background positioning area - height of background image), where the size of the image is the size given by ‘background-size’.

  


简单地说，当背景图片尺寸（`background-size`）不做任何的重置（也就是`100% 100%`）时，水平百分比的值等于容器宽度百分比值减去背景图片宽度百分比值。垂直百分比的值等于容器高度百分比值减去背景图片高度百分比值。

  


即 `background-position` 取值为百分比值（`%`）时，它会先从相应的容器尺寸中减去背景图片尺寸，然后将结果值的百分比用作距左边界（或顶部）的直接偏移量。相应的计算公式如下：

  


```
(容器宽度 - 背景图片宽度) x (background-position-x设置的百分比值) = background-position-x 的计算值 
(容器高度 - 背景图片高度) x (background-position-y设置的百分比值) = background-position-y 的计算值 
```

  


即：

  


-   水平方向（`x` 轴），即 `background-position-x` 的 `%` 值是相对于元素的 `width` 与背景图片的 `width` 差计算；
-   垂直方向（`y` 轴），即 `background-position-y` 的 `%` 值是相对于元素的 `height` 与背景图片的 `height` 差计算。

  


特别声明，背景图片的大小将会受 `background-size` 值的影响，上面提到的图片的尺寸是指 `background-size` 的值为 `auto` 的情景下的尺寸，即背景图片的初始尺寸（原始尺寸）。 比如下面这个示例：

  


```CSS
.element { 
    width: 500px; 
    height: 300px; 
    background-position: 70% 30% 
} 
```

```CSS

background-position-y: 30% ⇒ (300px - 180px) x 30% = 36px 
```

简单地说，当一个渐变（或背景层）的宽度和高度等于元素的宽度和高度时，使用百分比值移动它会使事情变得复杂化，我们并不知道这个百分比值设置多少才适合。换句话说，我们可以改变它的大小。比如，我们定义一个等于 `99.5% 99.5%` 的值（即 `mask-size` 或 `background-size` 的值设置为 `99.5% 99.5%`），它比元素宽度和高度小于 `0.5%` ，同时保持相同的视觉效果，因为 `99.5%` 与 `100%` 之间的差异非常小，小到肉眼无法发现。现在，我们的渐变大小不等于 `100%` ，我们就可以使用百分比值来移动它。我们可以使用下面这个公式来计算移动的位置：

  


```
100% * (50/(100 - 99.5)) = 100% * 100 = 10000%
```

  


这是一个奇怪的值，但它确实有效：

  


```CSS
.circular {
    --w: 50vw;
    --ratio: 1.5; 
    --s: 25px; /* 控制切口大小 */

    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf, #333); /* 控制文本颜色 */
    --g: #0000 98%, var(--bg);
    --m: radial-gradient(var(--s), var(--g));
    
    width: var(--w);
    aspect-ratio: var(--ratio);
    color: var(--color);
    background: var(--bg);
    mask: var(--m) 10000% 10000% / 99.5% 99.5%;
}
```

正如你所看到的，这个技巧非常有效。无论元素的大小是多少，我们都可以使用一个径向渐变来切割四个角。然而，当元素的宽度或高度为小数值时，这种方法有一个小缺点。例如，上面示例中，元素的宽度是使用的是一个相对值 `50vw` ，当你调整视窗宽度时，元素的宽度或高度可能会出现小数值，你可能会看到像下图这样的一个效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/053236f68770496cb6a4c2e1299dd2cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=606&s=1437621&e=gif&f=153&b=31565c)

  


这并不是我们想要的效果。庆幸的是，我们可以像下面这样使用，可以完美避免这种现象：

  


```CSS
.circular {
    --w: 50vw;
    --ratio: 1.5; 
    --s: 25px; /* 控制切口大小 */

    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf,#333); /* 控制文本颜色 */
    --g: #0000 98%, var(--bg);
    --m: radial-gradient(var(--s) at var(--s) var(--s), var(--g));
    
    width: var(--w);
    aspect-ratio: var(--ratio);
    color: var(--color);
    background: var(--bg);
    mask: var(--m) calc(-1 * var(--s)) calc(-1 * var(--s));
}
```

这种方法非常完美，因为它只使用一个渐变并且没有四舍五入的问题，但它有一个缺点：半径的值（`--s`）被使用了五次。这不是什么大问题，因为我们可以使用自定义属性来处理它：

  


```CSS
.circular {
    --s: 25px;
    --g: #0000 98%, var(--bg);
    --m: radial-gradient(var(--s) at var(--s) var(--s), var(--g));
    
    mask: var(--m) calc(-1 * var(--s)) calc(-1 * var(--s));
}
```

  


让我们快速回顾一下我们刚刚介绍的三种方法：

  


-   第一种方法使用四个渐变，使用上没有任何缺点。当然，它比较冗长，但适用于任何类型和大小的元素
-   第二种方法使用一个渐变，但在某些特定情况下可能会出问题。它适用于固定大小的元素，可以使用，但可能不那么频繁
-   第三种方法使用一个渐变，并且没有舍入问题。在所有方法中，这是最完美的方法，但它需要在渐变值中多次使用半径

  


现在我们已经看到了元素四个角落的情况（圆形切割），但我们有的时候可能只需要一两个角落需要有圆形切割的效果。那么，我们又应该如何来实现呢？

  


如果你使用的是第一种方法来创建圆形切割（四个渐变），那么很简单，只需要移除其渐变，并调整其他渐变的大小（`mask-size` 或 `background-size`）即可。

  


```CSS
.circular {
    --ratio: 1.5;
    --s: 25px; /* 控制切口大小 */
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf, #333); /* 控制文本颜色 */
    --g: #0000 98%, var(--bg);

    /* 左上角遮罩图 */
    --m-top-left: radial-gradient(circle var(--s) at top left, var(--g)) top left;
    /* 右上角遮罩图 */
    --m-top-right: radial-gradient(circle var(--s) at top right, var(--g)) top right;
    /* 左下角遮罩图 */
    --m-bottom-left: radial-gradient(circle var(--s) at bottom left, var(--g)) bottom left;
    /* 右下角遮罩图 */
    --m-bottom-right: radial-gradient(circle var(--s) at bottom right, var(--g)) bottom right;

    /* 四个角落都被切割 */
    mask: 
        var(--m-top-left), 
        var(--m-top-right), 
        var(--m-bottom-left),
        var(--m-bottom-right);
    mask-size: 51% 51%;
    mask-repeat: no-repeat !important;
    color: var(--color);
    background: var(--bg);

    /* 仅左上角被切割 */
    &:nth-child(2) {
        mask: var(--m-top-left);
        mask-size: 100% 100%;
    }

    /* 仅右上角被切割 */
    &:nth-child(3) {
        mask: var(--m-top-right);
        mask-size: 100% 100%;
    }

    /* 仅左下角被切割 */
    &:nth-child(4) {
        mask: var(--m-bottom-left);
        mask-size: 100% 100%;
    }

    /* 仅右下角被切割 */
    &:nth-child(5) {
        mask: var(--m-bottom-right);
        mask-size: 100% 100%;
    }

    /* 左上角 + 右下角被切割 */
    &:nth-child(6) {
        mask: var(--m-top-left), var(--m-bottom-right);
        mask-size: 51% 100%;
    }

    /* 右上角 + 左下角被切割 */
    &:nth-child(7) {
        mask: var(--m-top-right), var(--m-bottom-left);
        mask-size: 51% 100%;
    }

    /* 左上角 + 右上角被切割 */
    &:nth-child(8) {
        mask: var(--m-top-left), var(--m-top-right);
        mask-size: 51% 100%;
    }

    /* 左下角 + 右上角被切割 */
    &:nth-child(9) {
        mask: var(--m-bottom-left), var(--m-bottom-right);
        mask-size: 51% 100%;
    }
}
```

```CSS
}
```

## 曲线切割

  


最后要与大家一起探讨的是曲线切割类型，它比之前介绍的所有角落切割都要更复杂。例如下图这样的曲线切割：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ed3e14edb8a7463e972a6121725ebd59~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=804&s=134223&e=jpg&b=febfc8)

  


正如你所看到的，切割区域上的曲线将是最具挑战性的部分，因为这些曲线以不同的方式连接在一起，使得用一组简单的 CSS 规则创建这些曲线几乎不可能的。

  


就曲线问题而言，我们可以通过制作一个 SVG 背景图像来解决，因为 [SVG 的路径](https://juejin.cn/book/7341630791099383835/section/7368317966821425190)（`<path>`）可以实现所需的曲线形状。然而，问题在于 [SVG 通常具有固定的宽高比](https://juejin.cn/book/7341630791099383835/section/7345677438053810214)，这意味着，如果我们强行给它设定自定义的宽高比，它可能会变形。虽然[使用 SVG 是一种可行的方法](https://s.juejin.cn/ds/irkEuphT/)，但在这节课中我们不会讨论这种方法。如果你对这方面感兴趣的话，可以移步阅读《[SVG 剪切](https://juejin.cn/book/7341630791099383835/section/7362748816769941540)》和《[SVG 遮罩](https://juejin.cn/book/7341630791099383835/section/7366549423813296165)》！

  


在这里，我们依旧来探讨使用 CSS 创建曲线的方法，即使用 CSS 渐变来创建曲线。

  


首先，我们可以将渐变的停止点堆叠在一起以创建一条分明的线条，这意味着我们可以利用这些渐变来绘制图形。例如，如果我们想要一个圆形，可以使用径向渐变配合这个技巧，它就会为我们绘制出一个圆形。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6fe5048055074a5a9653f11c35ca95ab~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1136&h=858&s=9277739&e=gif&f=204&b=0f0325)

  


```CSS
.circle {
    background: radial-gradient(circle, rgb(245 72 12) 50%, rgb(2 0 36 ) 50%);
}
```

  


其次，我们可以使用任何颜色和任何不透明度。这非常重要，因为我们不仅可以在两个不透明的颜色之间创建渐变，还可以在所需颜色和透明颜色之间创建渐变。这将使我们能够绘制任何想要的形状，并使其具有透明背景：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6404677224a84192881d1b729554ad90~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1130&h=828&s=13405977&e=gif&f=220&b=e6e6e6)

  


```CSS
.circle {
    background: radial-gradient(circle, rgb(245 72 12) 50%, #0000 50%);
}
```

  


你会发现，我们之前介绍的圆形角落切割也应用到了这两个技巧。

  


有了这个认知，我们要创建曲线就容易多了。首先，我们将这些曲线拆解成一些简单的形状：

第一步很简单，除右上角之外的三个角[使用 CSS 的 border-radius 属性](https://juejin.cn/book/7199571709102391328/section/7199845563389444099)就可以轻松实现：

  


```CSS
.curved {
    border-radius: 50px 0 50px 50px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bdb6caea5d05423eb8b86c2d761d964f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1069&s=173620&e=jpg&b=376366)

  


就这个示例而言，右上角的曲线切割才是最关键的，也是最棘手的。仔细观察，你会发现右上角的曲线切割有两个小圆分别代表切口的左上角和右下角的曲线：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c89a09fc2404408abfddce529323a663~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=804&s=91884&e=jpg&b=ffc0c8)

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/822d7ee0d9fb4ba58d29bb184503ea56~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=804&s=96020&e=jpg&b=fdbecb)

  


除了考虑形状之外，还需考虑顶部和右侧。但要知道的是，我们不能仅用一个盒子框来完成，因为一个盒子框，它会隐藏圆形的曲线，所以我们必须制作两个盒子框，延伸到边缘并在中心停止，以保留圆形的曲线。基于这个原因，我们将大形状（右侧的曲线切割）拆分成更小，更简单的形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4e1fd07ad4454a14bcc5df376e3b0379~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=804&s=138847&e=jpg&b=fdbecd)

  


正如你所看到的，这样一拆分，这个复杂的曲线形状变得简单明了。换句话说，使用 CSS 绘制这样的曲线切割，我们需要四个线性渐变（`linear-gradient`）用于绘制上图中四个矩形框（两个浅绿色和两个浅紫色矩形框）和三个径向渐变（`radial-gradient`）用于绘制两个小圆点和一个大圆切口。

  


现在，我们可以开始编写 CSS 了，让我们逐一绘制这些形状。首先，让我们使用第一个径向渐变，绘制大圆切口：

  


```CSS
.curved {
    border-radius: 50px 0 50px 50px;
    /* 绘制圆 */
    background: radial-gradient(circle, #fff 30px, var(--bg) 31px);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5005edcaf57b4acd8591c8b37abb3304~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1069&s=206104&e=jpg&b=376366)

  


代码中的 `radial-gradient` 绘制了一个半径为 `30px` 的白色圆。这个 `30px` 相当于大圆切口的半径，你可以将其存储到一个 CSS 自定义属性中，例如：

  


```CSS
.curved {
    --big-circle-r: 30px; /* 大圆切口半径 */
    /* 绘制圆 */
    background: radial-gradient(
        circle,
        #fff var(--big-circle-r),
        var(--bg) calc(var(--big-circle-r) + 1px)
    );
}
```

  


你会看到，第一个渐变颜色（`--bg`）的停止点是在前一个渐变颜色（`#fff`）停止点的基础上加了 `1px` ，即 `calc(var(--big-circle-r) + 1px)` 。这是因为，如果两个停止点完全相同，浏览器会绘制一个锯齿状的、不平滑的线条，因此我们使用这个技巧在每个停止点上添加 `1px` ，以增加一点模糊度，使渐变颜色之间的线条更平滑。

  


> 我曾在《[你所不知道的 CSS 渐变](https://juejin.cn/book/7199571709102391328/section/7199845781149810727#heading-16)》中也提到了些避免渐变产生锯齿的小技巧，如果你感兴趣的话，[可以阅读这节课](https://juejin.cn/book/7199571709102391328/section/7199845781149810727#heading-16)，了解相关的小技巧！

  


注意，径向渐变应该绘制一个半径为 `--big-circle-r` 的透明色圆，这里我使用 `#fff` 是为了便于大家能更好的理解这个制作过程，最终我们还是需要将 `#fff` 替换为透明色 `#0000` 。

  


需要知道的是，到目前为止，大圆切口位于元素 `.curved` 中心，它并没有位于我们所需要位置（右上角）。为了修正这个问题，只需在径向渐变中，指定圆心的位置（渐变开始的位置）：

  


```CSS
.curved {
    --big-circle-r: 30px; /* 大圆切口半径 */
    /* 绘制圆 */
    background: radial-gradient(
        circle at 100% 0,      /* 位于元素上右上角顶点，也可以是 top right*/
        #fff var(--big-circle-r),
        var(--bg) calc(var(--big-circle-r) + 1px)
    );
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/908df6d39a47463bae6e50e92b7902d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1194&h=716&s=608431&e=gif&f=109&b=31565c)

  


现在，这看起来就像我们之前拆分的大圆形切口！

  


接下来，我们使用相似的方法来绘制小圆圈，但为了保证其位置的正确性，我们需要做一些简单的数学计算。简单地说，我们可以按照下面这种方式计算小圆圆心之间的距离：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/584223008a144b01a0264a3387b1fde7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=804&s=128320&e=jpg&b=fdbecb)

  


小圆圈之间的距离（`--circles-space`）等于柱子大小（`--pillar-size`）和大圆半径（`--big-circle-r`）的和：

  


```CSS
.curved {
    --big-circle-r: 30px; /* 大圆切口半径 */
    --small-circle-r: 13px; /* 小圆圈半径 */
    --pillar-size: 34px;  /* 柱子大小 */
    --circles-space: calc(var(--bg-circle-r) + var(--pillar-size)); /* 小圆圈之间的距离 */
    
    
    background: 
        /* 大圆切口 */
        radial-gradient(
            circle at 100% 0,      /* 位于元素上右上角顶点，也可以是 top right*/
            #fff var(--big-circle-r),
            var(--bg) calc(var(--big-circle-r) + 1px)
        );
}
```

  


注意，柱子的大小（`--pillar-size`）是大圆切口的上边和右边的内边距，这是我们用来将盒子移动到底部和左侧的值，**在绘制大圆切口时，柱子大小（** **`--pillar-size`** **）不能小于小圆的半径（** **`--small-circle-r`** **）** 。

  


到这里，我们已经知道了圆圈之间的距离，并且知道它们需要绘制的边缘，我们就可以按照下面这个方式来绘制圆圈：

  


```CSS
.curved {
    --big-circle-r: 30px; /* 大圆切口半径 */
    --small-circle-r: 13px; /* 小圆圈半径 */
    --pillar-size: 34px; /* 柱子大小 */
    --circles-space: calc(var(--big-circle-r) + var(--pillar-size)); /* 小圆圈之间的距离 */
    --small-circle-d: calc(var(--small-circle-r) * 2); /* 小圆直径 */

    --bg: #c73226; /* 主背景色 */
    /* 大圆切口 */
    --big-circle: radial-gradient(circle at 100% 0%,#fff var(--big-circle-r), var(--bg) calc(var(--big-circle-r) + 1px)) 0px var(--pillar-size) / calc(100% - var(--pillar-size)) 100% no-repeat;
    /* 顶部小圆圈 */
    --small-circle-top: radial-gradient(circle closest-side,var(--bg) 100%,#fff calc(100% + 1px))  calc(100% - var(--circles-space)) 0 / var(--small-circle-d) var(--small-circle-d) no-repeat;
    /* 右侧小圆圈 */
    --small-circle-right: radial-gradient(circle closest-side,var(--bg) 100%,#fff calc(100% + 1px)) 100% var(--circles-space) / var(--small-circle-d) var(--small-circle-d) no-repeat;
    
    background: var(--big-circle), var(--small-circle-top), var(--small-circle-right);
    border-radius: 50px 0 50px 50px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4faa137650b04433b993b8cc5b2c6c8b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1069&s=207067&e=jpg&b=376366)

  


`radial-gradient` 在绘制小圆的时候，我们使用了关键词 `closest-side` ，以便从最近的边计算百分比。[你可以在这里了解更多关于这个关键词的信息](https://juejin.cn/book/7199571709102391328/section/7199845781149810727#heading-8)。

  


另外，我们分别使用 CSS 自定义属性来存储每个径向渐变绘制的图形：

  


```CSS
.curved {
    /* 大圆切口 */
    --big-circle: radial-gradient(circle at 100% 0%,#fff var(--big-circle-r), var(--bg) calc(var(--big-circle-r) + 1px)) 0px var(--pillar-size) / calc(100% - var(--pillar-size)) 100% no-repeat;
    /* 顶部小圆圈 */
    --small-circle-top: radial-gradient(circle closest-side,var(--bg) 100%,#fff calc(100% + 1px))  calc(100% - var(--circles-space)) 0 / var(--small-circle-d) var(--small-circle-d) no-repeat;
    /* 右侧小圆圈 */
    --small-circle-right: radial-gradient(circle closest-side,var(--bg) 100%,#fff calc(100% + 1px)) 100% var(--circles-space) / var(--small-circle-d) var(--small-circle-d) no-repeat;
}  
```

  


看到这样的代码，你可能会有一种恐惧感，这其实就是 `background` 的简写属性，以 `--big-circle` 为例，它可以拆分为：

  


```CSS
.curved {
    background: radial-gradient(circle at 100% 0%,#fff var(--big-circle-r), var(--bg) calc(var(--big-circle-r) + 1px)) 0px var(--pillar-size) / calc(100% - var(--pillar-size)) 100% no-repeat;

    background-image: radial-gradient(circle at 100% 0%,#fff var(--big-circle-r), var(--bg) calc(var(--big-circle-r) + 1px));
    background-position: 0px var(--pillar-size);
    background-size: calc(100% - var(--pillar-size)) 100%;
    background-repeat: no-repeat;
}
```
