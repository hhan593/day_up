## 创建花朵形状

  


在开始编写 CSS 之前，我们首先来了解花朵形状的背后几何知识。假设我们即将要创建一个类似下图的花朵形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/456b45fecf1e41fd8a5a44e34a35ffa8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=113149&e=jpg&b=ffffff)

  


仔细观察上图，不难发现，这个类似“花朵”的形状，可以看作是由十二个小圆围绕着一个大圆排列的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3128e6e73b624b3a89e0069116db7925~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=278547&e=jpg&b=ffffff)

  


所有小圆围绕着大圆排列，而且它们小圆彼此接触，但又相互不重叠。现在，我们需要知道是这些圆的关键信息，比如大圆的半径、小圆的数量、小圆的半径以及小圆圆心的坐标点。事实证明，这正是 CSS 中 **[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)** 的完美应用场景。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/073b4070e596413697a07c6f51036fb2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1634&s=701927&e=jpg&b=ffffff)

  


这个形状可以通过两个变量来控制：小圆圈的数量（`n`）和它们的半径（`r`）。从这些变量中，我们可以确定整个形状的大小以及大圆圈的半径。正如上图所示，根据三角函数，我们可以计算出元素的大小：

  


```
2 * (x + r) = 2 * (r/sin(180deg/n) + r) = 2 * r * (1 + 1/sin(180deg/n))
```

  


同时也可以计算出大圆的半径 `c` ：

  


```
c  = r/tan(180deg/n)
```

  


这样一来，我们就可以使用 CSS 的自定义属性来存储这些变量：

  


```CSS
.flower {
    --n: 12;   /* 小圆圈的总数 */
    --r: 30px; /* 小圆的半径 */
    --c: calc(var(--r) / tan(180deg / var(--n))); /* 大圆的半径 */
    --w: calc(2 * var(--r) * (1 + 1 / sin(180deg / var(--n)))); /* 元素的宽度 */
}
```

  


到目前为止，我们知道元素的大小、大圆的半径和小圆的半径。但还缺另一个关键信息，那就是每个小圆的圆心位置（坐标）。稍后我们会介绍如何获取这个关键信息。

  


有一点需要知道的是，在这里我们并没有打算每个圆使用独立的 HTML 元素。换句话说，我们只使用一个元素（也不使用任何伪元素）来创建我们最终所需要的花朵形状。在 CSS 中，我们可以使用 CSS 的径向渐变来创建圆，不管是大圆还是小圆。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d07f9c6b109c4379bbec066e2e5dbdbe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1324&h=698&s=2176753&e=gif&f=233&b=d8dde3)

  


> URL:https://gradient.style/

  


```CSS
.circle {
    width: 50vmin;
    aspect-ratio: 1;
    background-image: radial-gradient(25vmin circle at 50% 50% in oklab,red 99% , #0000 101%);
}
```

  


有了这些基础之后，就可以开始编写 CSS 代码。

  


首先，我们使用径向渐变 `radial-gradient` 绘制大圆：

  


```CSS
.flower {
    --n: 12; /* 小圆圈的总数 */
    --r: 30px; /* 小圆的半径 */
    --c: calc(var(--r) / tan(180deg / var(--n))); /* 大圆的半径 */
    --w: calc(2 * var(--r) * (1 + 1 / sin(180deg / var(--n)))); /* 元素的宽度 */
    --bg: #c73226;
    width: var(--w);
    aspect-ratio: 1;
    background: radial-gradient(
        var(--c) circle at 50% 50%,
        var(--bg) 99%,
        #0000 101%
    );
}
```

  


到目前为止，没有什么特别之处，我们只是将三角函数计算出来的一些参数以 CSS 自定义属性存储起来，并引入 CSS 中，然后使用 `radial-gradient` 绘制了一个大圆：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4a6d56beb0a44669ad818ab191a4fb18~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=196317&e=jpg&b=376366)

  


注意，为了让大家更易于理解 CSS 是如何创建花朵形状，暂时将渐变绘制的图形作为元素的背景使用，后续我们可以使用 `mask` 来替代 `background` ，这样你可以将 CSS 创建的花朵形状应用于任何元素。另外，[我们在这里将元素的 `aspect-ratio` 设置为 `1`](https://juejin.cn/book/7223230325122400288/section/7259316040151236666) ，使元素的高度始终等于它的宽度（`--w`）。注意，元素的宽度 `--w` 是根据小圆的半径 `--r` 和数量 `--n` 计算得到的。

  


接下来，我们继续使用 `radial-gradient` 来绘制小圆：

  


```CSS
.flower {
    --n: 12; /* 小圆圈的总数 */
    --r: 30px; /* 小圆的半径 */
    --c: calc(var(--r) / tan(180deg / var(--n))); /* 大圆的半径 */
    --w: calc(2 * var(--r) * (1 + 1 / sin(180deg / var(--n)))); /* 元素的宽度 */
    --bg: #c73226;
    width: var(--w);
    aspect-ratio: 1;
    /* 大圆 */
    --big-circle: radial-gradient(
        var(--c) circle at 50% 50%,
        var(--bg) 99%,
        #0000 101%
    ) no-repeat;
    /* 小圆 */
    --small-circle: radial-gradient(
        var(--r) circle at 50% 50%,
        hsl(from var(--bg) 80 s l / .85) 99%,
        #0000 101%
    )  center / calc(2 * var(--r)) calc(2 * var(--r)) no-repeat;
    background:var(--small-circle), var(--big-circle);
}
```

  


小圆的半径是 `--r` ，所以径向渐变的大小等于 `calc(2 * var(--r))` （即 `background-size`）。到目前为止，小圆和大圆的位置都位于元素的正中间，因此你看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/23bc74070ebf4a43b0b1348f25d9e641~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=202315&e=jpg&b=376366)

  


也就是说，我们现在要确定每个小圆的圆心位置。其实，小圆的圆心位置同样可以通过三角函数来获得：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b0905892ddf437599b7fffe22334bcb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1634&s=682484&e=jpg&b=ffffff)

  


根据上图所示，我们使用[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)来计算每个小圆的圆心坐标：

  


```
x = calc(50% + 50% * cos(360deg * (var(--i) / var(--n))))
y = calc(50% + 50% * sin(360deg * (var(--i) / var(--n))))
```

  


注意，引入的变量 `--i` 对应的是小圆的索引号，它可以从 `0` 开始，也可以从 `1` 开始，我们这里从 `1` 开始。另外，上面公式还对坐标进行了一个转换，从极坐标系转换到了迪卡尔坐标系。

  


接下来，我们把上面的公式引入到 CSS 中：

  


```CSS
.flower {
    --n: 12; /* 小圆圈的总数 */
    --r: 30px; /* 小圆的半径 */
    --c: calc(var(--r) / tan(180deg / var(--n))); /* 大圆的半径 */
    --w: calc(2 * var(--r) * (1 + 1 / sin(180deg / var(--n)))); /* 元素的宽度 */
    --bg: #c73226;
    width: var(--w);
    aspect-ratio: 1;
    /* 大圆 */
    --big-circle: radial-gradient(
        var(--c) circle at 50% 50%,
        var(--bg) 99%,
        #0000 101%
    ) no-repeat;
    
    /*
      x = calc(50% + 50% * cos(360deg * (var(--i) / var(--n))))
      y = calc(50% + 50% * sin(360deg * (var(--i) / var(--n))))
    */
    
    /* 第一个小圆 */
    --x1: calc(50% + 50% * cos(360deg * (1 / var(--n))));
    --y1: calc(50% + 50% * sin(360deg * (1 / var(--n))));
    --sc-1: radial-gradient(
        var(--r) circle at 50% 50%,
        hsl(from var(--bg) 80 s l / .85) 99%,
        #0000 101%
    )  var(--x1) var(--y1) / calc(2 * var(--r)) calc(2 * var(--r)) no-repeat;
    background:var(--sc-1), var(--big-circle);
 }
```

  


下图演示了小圆从元素的中心移动到相应位置的变化：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/56ffc83ad85c42069f17ffe278593eb2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1210&h=612&s=338962&e=gif&f=102&b=31565c)

之前我们所展示的是通过小圆的半径 `--r` 来获得元素的宽度，你也可以反过来操作，通过元素的宽度 `--w` 来获取小圆的半径。这可能更方便一些，因为我们通常想要控制元素的大小。

  


```CSS
.flower {

    --c: calc(var(--r) / tan(180deg / var(--n))); /* 大圆的半径 */
    
    width: var(--w);
    aspect-ratio: 1;
    
    /* 其他的 CSS 和之前的一样 */

## 花朵形状反转

  


上一部分创建的花朵形状，其中小圆部分都是向外延伸。我们也可以将之面的花朵形状反转过来，小圆部分向内而不是向外延伸。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4371c204b683437fb58e0dbb6a7ba0dd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=180538&e=jpg&b=ffffff)

  


首先要明确一点的是，制作小圆和大圆与之前方法是一样的，代码几乎是相同的。不过，整个制作过程会略为复杂一些。

  


在继续编码之前，我们先简单的回忆一下。要是使用图形编辑软件（例如 Figma），是如何实现该效果呢？其过程可能大致如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a38e768a093496bafd95a856d4be13b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1156&h=652&s=1780855&e=gif&f=389&b=f3f3f3)

  


简单地说，**我们要从大的圆形中剪切出小圆形**。

  


为了实现该效果，我们需要引入额外的渐变，并且还要使用 `mask-composite` 特性，对遮罩层进行合成处理。我用下图来简单的描述遮罩图合成的过程：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3c86b005b5ac4080a7d42232dbeb4b4c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=748&s=70204&e=jpg&b=fefefe)

  


因为 `mask` 中的合成计算和我们常用的设计软件合成是相似的。换句话说，`mask-composite` 对应的值，比如 `add`、`subtract`、`intersect` 和 `exclude` 可以指定遮罩层的合成计算，它们分别对应设计软件中的联集（`add`）、减去顶层（`subtract`）、交集（`intersect`）和差集（`exclude`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ca92670e92114d70a78bed7d4d64cd14~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=924&s=85596&e=jpg&b=fefefe)

  


> 如果你想更深入的了解 `mask` 的 `mask-composite` 特性，建议你移步阅读《[CSS 的 Clipping 和 Masking](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-16)》一文！

  


通过这两张图，并结合你熟悉的图形编辑软件，我想你对 `mask-composite` 特性有了基本的了解，而且在这里我们并不会使用很复杂的功能。既然如此，我们就回到反转花朵形状的制作中来。下图足以说明整个制作过程：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/475cdace8a8948c1b4cad32114c7f38e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=4933&h=2766&s=1227471&e=jpg&b=ffffff)

  


我们将执行两次合成操作。

  


首先，将一个填充整个元素区域的渐变（上图中的红色矩形）和所有小圆进行 `exclude` （差集）合成，`exclude` 合成操作将它们从彼此中“排除”，以获得一个新形状，其中小圆莆变成了孔（镂空）。换句话说，小圆形现在是我们形状的透明部分。

  


接着，我们将创建的大圆渐变与第一次合成的结果（带有小孔的新形状）进行 `intersect` （交集）合成操作，仅显示中间区域，从而获得最终形状（反转的花朵形状）。

  


需要注意遮罩层的先后顺序，所有小圆位于最底层（`--small-circle-group`），其次是填充整个区域的渐变（也是新增的渐变，即 `--g-full`），最顶上的是大圆（`--big-circle`）：

    /* 第 4 个小圆 (i = 4) */

    --sc-4: var(--pos4) var(--gsc);

    /* 第 5 个小圆 (i = 5) */

    --sc-5: var(--pos5) var(--gsc);

    /* 第 6 个小圆 (i = 6) */
    --x6: calc(50% + 50% * cos(360deg * (6 / var(--n))));

    /* 第 7 个小圆 (i = 7) */
    --x7: calc(50% + 50% * cos(360deg * (7 / var(--n))));

    --pos7: var(--x7) var(--y7);
    --sc-7: var(--pos7) var(--gsc);

    /* 第 8 个小圆 (i = 8) */
    --x8: calc(50% + 50% * cos(360deg * (8 / var(--n))));
    --y8: calc(50% + 50% * sin(360deg * (8 / var(--n))));
    --pos8: var(--x8) var(--y8);
    --sc-8: var(--pos8) var(--gsc);

    /* 第 9 个小圆 (i = 9) */
    --x9: calc(50% + 50% * cos(360deg * (9 / var(--n))));

.flower {

.flower {

```

## 花朵形状的变体

  


接下来，我们一起来制作一个像下图这样的“花朵形状”：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3483c94a41214679885e4b66fabfac3d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=956&s=107530&e=jpg&b=ffffff)

  


在外形上，它看起来也像花朵形状，我在这里把它称为“花朵形状的变体”。其实，它的形状更像是一个 Blob 形状。

  


既然是花朵形状的变体，那么制作这个形状的原理与之前所介绍的花朵形状制作原理是相似的。只不过，我们会使用更多个渐变和 `mask-composite` 来完成这个效果。

  


与之前一样，我使用下图来说明这个图形的制作过程：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73ac9d66da3145ffa4937dd9407772f2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=7249&h=2766&s=1720083&e=jpg&b=ffffff)

  


一个由小圆圈组成的环形结构使用“排除”（`exclude`）合成将圆圈挤出，然后使用“相交”（`intersect`）合成来形成类似齿轮的形状，最后使用“添加”（`add`）合成来得到最终的blob形状。 如你所见，一组小圆圈在“排除”合成中用于创建内曲线，而另一组小圆圈在“添加”合成中用于创建外曲线。其思想是将这些圆圈向相反的方向移动，以创建和控制我们的波浪圆圈。

  


另一个图示说明了这个技巧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/143d26a1a59b411ba241424d5b674408~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=896&s=371574&e=jpg&b=ffffff)

  


上图中的（1️⃣）显示了初始形状，其中所有小圆排列成一个更大的圆圈，并且相互接触。红色圆是被排除（`exclude`）的圆圈，蓝色圆是添加（`add`）的圆圈。当蓝色圆向中心移动，同时红色圆向相反的方向移动（上图中的 2️⃣ 所示）。结果看起来有些奇怪，因为圆圈不再相互接触，但如果我们增加它们的半径，就能得到一个完美的结果。

  


这个想法是移动圆圈的同时调整它们的半径，以便它们始终相互接触：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8fe016d4739347468cfb8d803a977c3d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1114&h=362&s=4261172&e=gif&f=295&b=dce1c4)

  


现在，我们知道创建这个形状的基本原理了，接下来要做的就是找到一个变量，使其能同时控制小圆的位置和半径。简单地说，我们将通过一个角度 `--a` 来控制它们：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6edfd30b66e24d63b2e82406c79a526b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3310&h=1634&s=1055181&e=jpg&b=ffffff)

  


上图中白色线是两个相邻小圆圆心相连的直线，图中左侧是初始状态，红色圆和蓝色圆都未移动，因此可以认为 `--a` 的值等于 `0` 。当红色圆向外移，蓝色圆向内移时，白色线条会有一个较少的放置值，这个值就是 `--a` 变量的值。

首先，我们在前面示例的基础上新增一个变量 `--a` ：

## 简单的用例

  


现在，我想你已经掌握了创建[锯齿形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-0)、[凹槽形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-1)、[凸起形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-2)、[波浪形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-3)和花朵形状背后的原理以及制作这些形状的公式。将它们应用于 `background` 时，你可以为元素添加各种花式的形状的背景，要是将它们用于 `mask` 的话，你可以使整个元素具有不同的形状。注意，这些形状用于 `background` 和 `mask` 最大的区别之一，`mask` 可以用于任何元素之上。

除此之外，你还可以为其添加动画效果。例如，我们动态化更新 `--a` 的值：

  


```CSS
img {
    --a: 10deg;
    transition: --a .28s;
    
    &:hover {
        --a: 28deg;
    }
}
```

  


上面的代码只是改变了形状，但整个过程并没有平滑的动画效果，因为默认情况下，我们不能对 CSS 自定义属性进行动画化处理。庆幸的是，[我们可以通过 CSS 的 @property 来改变这一现状](https://juejin.cn/book/7223230325122400288/section/7258870477462962236)：

  


```CSS
@property --a {
    syntax: "<angle>";
    inherits: true;
    initial-value: 0deg;
}

img {
    --a: 10deg;
    transition: --a .28s;
    
    &:hover {
        --a: 28deg;
    }
}
```

  


这个时候，你将鼠标悬停在用户头像时，花朵变化会有一个平滑的动画效果：

使用同样的原理，我们就可以实现课程开头展示的动画效果。为了节约时间，我使用 SCSS 来编写这个效果：

  


```SCSS
$n: 12; 

@property --a {
    syntax: "<angle>";
    initial-value: 0deg;
    inherits: true;
}

@property --i {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

@keyframes rotate {
  to {
    --a: 360deg;
  }
}

img {
    --r: 50px; 
    --f: 1.7; 
    --c: #e4844a;

    width: calc(var(--r) * (1 + 1 / sin(180deg /#{$n})));
    aspect-ratio: 1;
    border-radius: 50%;
  
    $m: ();
    @for $i from 1 through ($n) {
        $m: append(
            $m,
            radial-gradient(var(--c) 70%, #0000 72%) no-repeat
            calc(50% + (50% - var(--i, 0px)) * cos(360deg *#{$i/$n} + var(--a, 0deg)))
            calc(50% + (50% - var(--i, 0px)) * sin(360deg *#{$i/$n} + var(--a, 0deg)) ) / var(--r) var(--r),
            comma
        );
    }
    
    --_m: radial-gradient(
        var(--c) calc(72% - var(--r) / 2 - var(--i, 0px)),
        #0000 0
    ),
    #{$m};
    
    mask: linear-gradient(#000 0 0) top/100% 50% no-repeat, var(--_m);
    background: var(--_m);
    --_a: rotate linear infinite;
    animation: var(--_a) 10s, var(--_a) 16s reverse;
    animation-composition: add;
    cursor: pointer;
    transition: --i 0.4s, scale 0.4s;
    
    &:hover {
        --i: calc(var(--r) / var(--f));
        scale: calc((1 + 1 / sin(180deg /#{$n})) / (1 - 2 / var(--f) + 1 / sin(180deg /#{$n})));
        animation-play-state: running, paused;
    }
}
```

## 小结

  


在这节课中，我们使用了[现代 CSS 特性](https://s.juejin.cn/ds/iruQqDVX/)，例如[遮罩 mask](https://juejin.cn/book/7223230325122400288/section/7259668885224456252) 、[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)、[自定义属性 @propery](https://juejin.cn/book/7223230325122400288/section/7258870477462962236) 等，创建了一个看似很复杂的花朵效果。那么，结合上一节课，我们总共介绍了创建[锯齿形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-0)、[凹槽形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-1)、[凸起形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-2)、[波浪形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-3)和花朵形状的背后逻辑以及相关公式。你可以将这些形状应用于任何元素上。

  


如果仅从编写 CSS 代码的角度来看，你可能会觉得这些过于复杂，甚至会认为这些复杂的形状在实际开发中有可能永远用不到。从而觉得花这么多时间和精力学习这些不太切合实际的东西，有点得不尝失。从我个人的角度来看，这些技巧看上去似乎非常复杂，制作出来的东西也略显花哨，但这并不能代表它在实际应用中没有空间与场景。只不过大家很多时候忽略了用代码来实现这些效果，更习惯性的使用图片来替代这些效果。即使如此，通过这两节课的学习，你将掌握一堆 CSS 技巧，例如 [CSS 渐变](https://juejin.cn/book/7199571709102391328/section/7199845781149810727)、[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)、[自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)、[遮罩和剪切](https://juejin.cn/book/7223230325122400288/section/7259668885224456252)等，这些都技巧都可以在其他地方使用。

  


换句话说，即使你现在或未来都不可能会采用这个技巧来绘制这些形状，但在这里我们学习了如何使用 `mask-composite` ，学会了如何使用 `@property` 动画化 CSS 自定义属性，玩转了 CSS 渐变。等等，有一天，你肯定会需要用到这些 CSS 技巧！
