## 从最简单的全屏布局开始

  


我猜想大家或多或少有接过全屏布局的需求。比如启动页面、背景图片、视频封面和全屏滚动网站等。这些页面都有一个共同的特征，Web 元素或容器的大小和屏幕的大小相同，即元素的宽度和高度都填满整个浏览器视窗。这种布局常用于需要确保内容覆盖整个屏幕的情况，例如下图所示这个效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/26f4a931ca5b470f961dbaac082463fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2048&h=1152&s=242687&e=jpg&b=dcd7ce)

  


通常情况之下，大部分 Web 开发者会使用[视窗单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)来设置容器的尺寸，比如：

  


```CSS
.container {
    width: 100vw;
    min-height: 100vh;
}
```

  


根据视窗单位 `vw` 和 `vh` 的定义来说，这并没有错，它会使容器填满整个浏览器视窗。但是，这样使用会使你的页面在不同的系统中渲染上存在一定的差异。因为 `vw` 和 `vh` 单位并没有考虑到设备上的导航条收缩等因素。以 `vw` 单位为例，它在 Mac OS 系统的浏览器中会使容器宽度与浏览器视窗宽度相等，因为垂直滚动条默认是隐藏的（滚动条的宽度被添加到总宽度中）。但是，在 Windows 系统的浏览中，你会发现页面会出现水平滚动条，因为滚动条的宽度被添加到宽度中，此时容器的宽度会大于浏览器视窗的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2205082efbb04575a0b4af784e661e60~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2560&h=910&s=48985&e=png&b=fafafa)

  


当然，这种现象我们是可以避免的，比如使用 `dvw` 和 `dvh` 分别替代 `vw` 和 `vh` 。或者使用 `min()` 函数，让浏览器自己来做出正确的选择：

  


```CSS
.container {
    width: min(100%, 100vw);
    min-height: 100dvh;
}
```

  


除此之外，你可能会借助 CSS 的级联特性来创建全屏布局：

  


```CSS
:where(html, body) {
    height: 100%;
}

.container {
    min-height: 100%;
}
```

  


这种方式虽然有效，但必须将 `body` 的 `margin` 和 `padding` 重置为 `0` ，否则页面也会出现滚动条：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ea7793f4272344eb90624f566ad86d33~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1162&h=612&s=319229&e=gif&f=119&b=059fec)

  


事实上，我们使用 CSS Grid 布局技术创建全屏容器会更简单，还可以避免一些不必要的麻烦。例如：

  


```CSS
html {
    display: grid;
    min-height: 100%;
}

.container {
    height: 100%;
}
```

  


你会发现，我们并没有在 `html` 和 `body` 元素上显式设置 `height` 属性的值，也没有使用 `100vh` 或 `100dvh` 。但 `.container` 容器的宽高与浏览器视窗的宽高完全相等，甚至它还完全兼容浏览器给 `body` 设置的默认 `margin` 值，也不会像之前的案例那样出现滚动条：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7e0bdf1bc8ba4e4e897bf8e09f46e059~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1089&s=116220&e=jpg&b=05a9f0)

  


不仅如此，该容器还能在内容超过屏幕高度时自动扩展：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/230043a6b5d949bf9dfc84e76ef29c94~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1160&h=608&s=14710976&e=gif&f=100&b=545f44)

  


> Demo 地址：https://codepen.io/airen/full/VYZvXbq

  


这意味着，上述代码会将最小高度（`min-height`）设置为等于屏幕高度。

  


稍微了解 CSS 百分比（`%`）工作原理的开发者应该都知道，`min-height` 和 `height` 属性的值为百分比时，它的计算还是比较复杂的，因为要有相应的参照物，它才能获得切确的值。那么，上面这代 CSS 代码是如何工作的？其背后的逻辑又是什么？

  


首先，我们通过 `min-height: 100%` （而不是 `height: 100%`）将 `html` 元素的高度设置为和浏览器视窗高度一样大，同时允许其内容超出时自动扩展。然后，我们利用 CSS Grid 的默认拉伸对齐特性（`html` 元素设置了 `display: grid` ，它是一个网格容器），使得 `body` 元素可以填满 `html` 元素的宽度和高度，同时保留其默认的外边距（`margin`）。这非常实用，因为我们可以轻松给 `body` 添加 `paddint` 或 `margin` ，而且不会影响全屏的效果。

  


由于 `body` 元素已经被拉伸到填满整个空间，我们在 `.container` 元素上使用 `height: 100%` ，让它填满相同的空间，从而创建一个全屏的容器（它的宽高和 `body` 元素的宽高将保持一致）。

  


你可能会感到困惑：**为什么** **`body`** **元素没有明确的高度的情况下，** **`.container`** **可以使用** **`height: 100%`** ？这就是使用 CSS Grid 和默认拉伸对齐的神奇之处。不知道你是否还记得，[上节课在创建等高列布局（由 `A` 元素的高度来决定 `B` 元素的高度）时，也使用了这个小技巧](https://juejin.cn/book/7395119173656903715/section/7444138525539434530)。其原理非常简单：

  


> 在 CSS Grid 布局中，一旦每个网格区域的大小被确定，网格项目就会被放置到各自的包含块中。网格区域的宽度和高度在这种情况下被视为明确的。

  


这意味着，被拉伸的元素（网格项目，即 `body` 元素）的大小被视为“确定的”，因此其子元素（`.container`）可以安全地使用百分比高度。

  


除了给 `.container` 元素显式设置 `height: 100%` 之外，我们还可以直接将 `body` 元素也定义为一个网格容器，这样 `.container` 元素就会自动拉伸，我们可以不给 `.container` 指定 `height: 100%` 也能实现相同的布局效果：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    min-height: 100%;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/af9e45014dae4a4488bd3d30b77a884d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1152&h=604&s=10474101&e=gif&f=134&b=545f44)

  


> Demo 地址：https://codepen.io/airen/full/MYgaGyX

  


值得注意的是，在使用这种方案创建全屏容器或全屏布局时，它有一个必要条件，那就是网格项目（需要被全屏的元素）必须处于被拉伸的状态。如果你在网格容器上重置了 `align-content` 或（和）`align-items` 的默认值 `stretch` ，或者在网格项目上重置了 `align-self` 的默认值 `stretch` ，那么这个方案将会失效。

  


## 变体一：添加页**眉和（或）页脚**

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d1d1235e708e4d0d9155f93a032d5ecd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2351&h=1660&s=630887&e=jpg&b=607d8b)

  


接下来，我们在之前的布局中添加一个页眉（`<header>`）和（或）页脚（`<footer>`），并确保主要内容（`<main>`）占据剩余的高度。这也是 Web 中经典布局之一，尤其是在内容较少时，希望页脚始终固定在页面底部——我们常说的 **Sticky Footer 布局**。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e929b64525014d03a7fef56872a7f928~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1090&h=459&s=32395&e=png&b=fcfcfc)

  


温馨提示，你使用 [Flexbox 布局技术](https://juejin.cn/book/7161370789680250917/section/7161623855054716935#heading-4)也能很容易实现这类布局效果，但我们在这里主要以 CSS Grid 为主。

  


创建这个布局，你可以需要使用像下面这样的 HTML ：

  


```HTML
<html>
    <body>
        <header></header>
        <main></main>
        <footer></footer>
    </body>
</html>
```

  


首先，我们使用之前创建全屏容器的 CSS 代码，将 `html` 和 `body` 元素定义为一个网格，并且将 `html` 的 `min-height` 设置为 `100%` ：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    min-height: 100%;
}

body {
    gap: 1rem;
}
```

  


这个时候，浏览器会给 `body` 创建一个三行一列的网格，因为它包含了三个子元素，在没有明确指定网格列和行时，它会自动按照行的方式来放置网格项目（因为 `grid-auto-flow` 的默认值为 `row`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b08c0790998747e598e3f8cbacb32977~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2852&h=1806&s=238223&e=jpg&b=717b54)

  


接下来，我们在 `body` 上使用 `grid-template-rows` 来定义行轨道的尺寸：

  


```CSS
body {
    grid-template-rows: auto minmax(0, 1fr) auto;
}
```

  


上面的代码会使 `header` 和 `footer` 根据内容来定义高度，而 `main` 会占据网格容器所有剩余的高度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9584dfb9d2f448d18e59c99de33a938e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1089&s=145611&e=jpg&b=5e6b4f)

  


你会发现，当 `main` 中的内容较少时，页脚会固定在页面的底部。一旦 `main` 中内容较多时，会将页脚推到浏览器视窗之外，并在整个页面上显示滚动条：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d0301819f2024c7ca72ccff61ca9a43e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1160&h=608&s=13486744&e=gif&f=127&b=545f44)

  


> Demo 地址：https://codepen.io/airen/full/GgKpxzJ

  


这个就是我们平常所说的 **Sticky Footer 布局**。

  


除此之外，我们还可以像下面这样来定义网格行轨道的尺寸，也能实现同样的效果：

  


```CSS
body {
    grid-template-rows: min-content auto min-content;
}
```

  


在这里，我们使用了内在尺寸 `min-content` 来定义 `header` 和 `footer` 所在行的高度，它将由其子元素所有高度来决定。与此同时，剩余的空间同样都会留给 `main` 。简单的说，网格行轨道尺寸将由网格项目 **[内在尺寸](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)** 来决定。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b27baa908f9d4d99805962a09b1e5d68~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1160&h=608&s=13486744&e=gif&f=127&b=545f44)

  


> Demo 地址：https://codepen.io/airen/full/NPKGMpa

  


完整的 CSS 代码如下：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    min-height: 100%;
}

body {
    gap: 1rem;
    grid-template-rows: min-content auto min-content; /* 或者 auto minmax(0, 1fr) auto */
}
```

  


有的时候，当 `main` 内容比较多的时候，并不希望滚动条出现在页面上，而是希望将滚动条移到 `main` 元素上，从而使页脚始终保持可见。我们也把这种布局效果称为**百分百无滚动布局** ，它也是 Web 十大经典布局之一。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/794532c050894ad299994eaf1b9e44d5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1301&h=2591&s=991528&e=png&a=1&b=00ecfd)

  


  


为了实现这种布局效果，我们可以在上例的基础上，将 `html` 元素的 `min-height` 替换为 `height` 。接着，给 `body` 元素添加 `min-height:0` 。注意，这一点非常重要。因为，CSS Grid 或 CSS Flexbox 为子项目定义了一个默认的最小高度（和最小宽度）。以 CSS 网格项目为例，为了给网格项目提供更合理的默认最小尺寸，`min-width` 或 `min-height` 的值为 `auto` 时，其计算如下：

  


-   该元素不是滚动容器
-   它至少跨越一个尺寸定义为 `auto` 的轨道
-   如果它跨越多个轨道，则这些轨道中没有一个是弹性的

  


只有同时满足以上三个条件，`auto` 的计算值才是基于内容的最小尺寸，否则 `auto` 的计算值为 `0` 。

  


简单地说，Flex 项目和 Grid 项目的 `min-width` 和 `min-height` 的默认值为 `auto` 时，浏览器计算出来的值不再是 `0` ，而是关键词 `auto` 。

  


如果你觉得上面的内容不易于理解，那么只需要记住：在某些情况下，当遇到奇怪的溢出问题或希望某个元素即使内容较多也能折叠（或收缩）时，可能需要添加 `min-height` 或 `min-width` 为 `0` 。这是一个非常有趣的技巧，为什么这么说呢？以 Flexbox 布局为例，很多时候，Web 开发者都会认为，只要将 Flex 项目的 `flex` 属性设置为 `1` ，就可以获得一个均分列的布局效果，即每一列等宽。实际上，这样做不一定正确，当某个 Flex 项目内容过多时，它会占用其他列的宽度，最终造成列不等宽的现象。只有同时满足 `flex:1` 和 `min-width:0` 时，所有 Flex 项目的宽度才是等宽。有关于这一点，我在《[防御式 CSS 精讲](https://s.juejin.cn/ds/iDgBLPC8/)》中，专门花了一节课的篇幅进行阐述，如果你感兴趣的话，请移步阅读《[Flexbox 和 Grid 中的最小内容尺寸](https://juejin.cn/book/7199571709102391328/section/7199845170123112486)》！

  


这意味着，要使页脚始终可见，滚动条就需要移动到 `main` 元素上。我们的 CSS 就需要调整成像下面这样：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    gap: 1rem;
    grid-template-rows: min-content auto min-contnet; /* 或 auto minmax(0, 1fr) auto*/
    min-height: 0; /* 这个很重要 */
    
    main {
        overflow-y: auto;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2cbc92c0d2af46339025c80af84b9810~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1158&h=608&s=10753678&e=gif&f=133&b=545f44)

  


> Demo 地址：https://codepen.io/airen/full/GgKpBKz

  


注意，如果你希望使用 CSS Flexbox 实现类似的布局效果，[那么你可点击这里获取相关的介绍](https://juejin.cn/book/7161370789680250917/section/7161623855054716935#heading-5)！

  


上面展示的示例是同时包含了页眉和页脚，如果你的布局只需要包含页眉或页脚的话，那么只需要调整 `body` 元素的 `grid-template-rows` 属性的值即可：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

/* 只包含页眉 */
body {
    gap: 1rem;
    min-height: 0;
    grid-template-rows: min-content auto; /* 或者 auto minmax(0, 1fr) */
}

/* 只包含页脚 */
body {
    gap: 1rem;
    min-height: 0;
    grid-template-rows: auto min-content; /* 或得 minmax(0, 1fr) auto */
}

/* 同时包含页眉和页脚 */
body {
    gap: 1rem;
    min-height: 0;
    grid-template-rows: min-content auto min-content; /* 或得 auto minmax(0, 1fr) auto */
}
```

  


## 变体二：添加侧边栏

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f77b4eeea876469f91babb59f16d67b4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2351&h=1660&s=770951&e=jpg&b=607d8b)

  


现在，我们在上一个布局的基础上添加一个侧边栏：

  


```HTML
<html>
    <body>
        <header></header>
        <aside></aside>
        <main></main>
        <footer></footer>
    </body>
</html>
```

  


在上面的示例的基础上，我们需要做的是，在已有的三行布局基础上，再新增两例。

  


```CSS
:is(html, body) {
    display: grid;
}

body {
    gap: 1rem;
    grid-template-rows: min-content auto min-content;
    grid-template-columns: 220px minmax(0, 1fr);
    min-height: 0%;
    
    main {
        overflow-y: auto;
    }
    
    :is(header,footer) {
        grid-column: 1  / -1;
    }
}
```

  


上面代码中，将侧边栏（`aside`）的宽度设置为 `220px` ，你也可以使用百分比值或 `auto` 来替代，甚至可以使用内在尺寸来替代，比如 `fit-content()` 函数。

  


由于新增了两列，我们需要将 `header` 和 `footer` 跨越两列。在 CSS Grid 布局中，有多种方式可以做到这一点，比如将 `grid-column` 的值设置为 `1 / -1` ，或者 `span 2` 都可以。

  


这样我们就获得了一个带有侧边栏的全屏布局，而且侧边栏与主内容列的高度相等。与上一个案例相似，当 `main` 元素内容比较多的时候，它会出现滚动条。具体的效果如下：

正如你所看到的，我们仅调整一两行代码，就可以让你获得拥有不同侧边栏的多列布局。但是，我想告诉你的是，我们还可以将 [CSS 的 `:has()` 选择器](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)与 [CSS 网格区域](https://juejin.cn/book/7395119173656903715/section/7436944613077680137)结合起来动态创建侧边栏。简单地说，我们可以根据 HTML 结构编写条件代码，来动态调整网格。比如：

  


-   没有侧边栏
-   如果侧边栏存在，并且位于 `main` 元素之后（右侧边栏）
-   如果侧边栏存在，并且位于 `main` 元素之前（左侧边栏）
-   如果有两个侧边栏，它们分别位于 `main` 元素前后

  


以上四种情况分别对应的 HTML 结构会像下面这样：

  


```HTML
<!-- 没有侧边栏 -->
<body>
    <header></header>
    <main></main>
    <footer></footer>
</body>

<!-- 拥有右侧边栏 -->
<body>
    <header></header>
    <main></main>
    <aside></aside>
    <footer></footer>
</body>

<!-- 拥有左侧边栏 -->
<body>
    <header></header>
    <aside></aside>
    <main></main>
    <footer></footer>
</body>

<!-- 同时拥有左右两个侧边栏 -->
<body>
    <header></header>
    <aside></aside>
    <main></main>
    <aside></aside>
    <footer></footer>
</body>
```

  


需要知道的是，上面代码所示的 HTML 结构不是绝对的，你可以根据自己需要来调整它们。你需要记住的是，要是 HTML 结构被调整了，那么接下来的 `:has()` 选择器也需要做出相应的调整，否则无法匹配到相应的元素。

  


现在，我们可以使用 `:has()` 选择器，来调整 `body` 网格的列数以及相应的网格区域命名。具体的代码如下：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    gap: 1rem;
    min-height: 0;
    grid-template-rows: min-content auto min-content;
    
    /* 无侧边栏 */
    grid-template-areas:
      "header"
      "main"
      "footer";
    
    header {
        grid-area: header;
    }
    
    main {
        grid-area: main;
        overflow-y: auto;
    }
    
    footer {
        grid-area: footer;
    }
    
    aside {
        grid-area: sidebar;
    }
    
    /* 右侧边栏 */
    &:has(main + aside) {
        grid-template-columns: minmax(0, 1fr) 220px;
        grid-template-areas: 
            "header   header"
            "main     sidebar"
            "footer   footer";
    }
    
    /* 左侧边栏 */
    &:has(aside + main) {
        grid-template-columns: 220px minmax(0, 1fr);
        grid-template-areas: 
            "header   header"
            "sidebar  main"
            "footer   footer";
    }
    
    /* 同时有左右两侧边栏  */
    &:has(aside + main + aside) {
        grid-template-columns: 220px minmax(0, 1fr) 220px;
        grid-template-areas:
            "header    header header"
            "sidebar-l main   sidebar-r"
            "footer    footer footer";
      
        aside:nth-of-type(1) {
            grid-area: sidebar-l;
        }
      
        aside:nth-of-type(2) {
            grid-area: sidebar-r;
        }
    }
}
```

> Demo 地址：https://codepen.io/airen/full/MYgaqXZ

  


使用这种方案，你只需要把注意力集中在 HTML 结构上即可。你可以尝试调整 HTML 结构，浏览器会根据 CSS 代码自动调整布局。注意，这里核心是 `:has()` 选择器根据不同的 HTML 结构预设了一些布局 CSS 代码，只有条件符合才会生效，否则 CSS 代码不会产生相应的结果。这就是条件 CSS 的魅力之处。如果你对条件 CSS 感兴趣的话，建议你花点时间阅读下面两节课的内容：

-   [条件 CSS 之 CSS 属性/值和 CSS 函数](https://juejin.cn/book/7199571709102391328/section/7217644982898720779)

  


同样的操作也可以应用于页眉和页脚，这样可以使 Web 页面的页眉和页脚是可选的（即动态的）。如果你感兴趣的话，不妨挑战一下自己，权当是留给大家的一个小作业，复习我们之前所讲解的内容。

  


温馨提示，CSS 的 `:has()` 选择器是一个非常强大的 CSS 特性，它可以帮助我们做很多之前需要依赖 JavaScript 才能做的事情。如果你感兴趣的话，可以阅读下面这几节课：

更有意思的是，如果你想实现侧边栏的固定效果（Sticky），只需要给侧边栏 `aside` 的 `align-self` 属性设置为 `start` （禁用默认的拉伸对齐），并且将其 `position` 设置为 `sticky` 即可：

## 变体三：滑动全屏

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6d82adb083ca4d089543e69e0599a824~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1152&h=598&s=18905486&e=gif&f=59&b=2b172c)

  


大家对于类似上图这种全屏滚动的布局效果应该不会感到陌生。简单地说，页面包含了多个区块，并且每个区域的宽高与浏览器视窗的宽高相等，用户每滑动一次就是会滑出或滑入一个区块。它对应的 HTML 结构大致如下：

  


```HTML
<body>
    <section></section>
    <!-- 省略其他的 section -->
</body>
```

  


我们只需要将第一个布局的代码稍微调整一下，就可以实现它：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
    overflow: hidden; /* 禁用页面滚动 */
}

body {
    grid-auto-rows: 100%; /* 所有行轨道尺寸相等，并且与浏览器视窗高度相等 */
    overflow-y: auto;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1fe797b94b94fe9a4614fe29c038013~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1108&h=610&s=19803827&e=gif&f=77&b=c8dbe5)

  


现在，每个 `section` 的宽高都与浏览器视窗宽高相等，但整个滚动效果不太好。我们可以借助 [CSS 的滚动捕捉特性](https://juejin.cn/book/7199571709102391328/section/7199846103007625227)来增强滚动体验：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
    overflow: hidden; /* 禁用页面滚动 */
}

body {
    grid-auto-rows: 100%; /* 所有行轨道尺寸相等，并且与浏览器视窗高度相等 */
    overflow-y: auto;
    scroll-snap-type: y mandatory;
    scroll-behavior: smooth;
    
    section {
        scroll-snap-align: start;
        scroll-snap-stop: always;
    }
}
```

  


注意，代码中的 `scroll-behavior: smooth` 也是用来增强用户滚动体验的一个特性，[它能使整个滚动变得更丝滑](https://juejin.cn/book/7199571709102391328/section/7199845609447096358#heading-7)。经过上面代码处理之后，用户每次只滚动一屏（即一个 `section`），而且整个滚动效果非常丝滑，你可以打开 Demo 亲自体验一下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/32a7e6d559d643dfa6a0c1a68d2d3015~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1108&h=600&s=19489132&e=gif&f=63&b=201633)

  


> Demo 地址：https://codepen.io/airen/full/XJrmGMe

  


在此基础上，我们可以引入 [CSS 变量](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)，使布局相关的代码变得更通用一些。比如，引入 `--n` 来定义每屏显示的区域数量，`--g` 定义轨道之间的间距。我们就可以通过修改 `grid-auto-rows` 的值来控制每屏显示区块的数量：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
    overflow: hidden; /* 禁用页面滚动 */
}

body {
    --n: 2; /* 每屏可见区块数量 */
    --g: 1rem; /* 区块之间间距 */
    gap: var(--g);
    grid-auto-rows: calc((100% - (var(--n) - 1) * var(--g)) / var(--n));
    overflow-y: auto;
    scroll-snap-type: y mandatory;
    scroll-behavior: smooth;
    
    section {
        scroll-snap-align: start;
        
        &:nth-child(2n+1) {
            scroll-snap-stop: always;
        }
    }
}
```

  


直观上，我们应该使用 ` 100%  ``/ var(--n)` 来定义 `section` 的高度（即网格行高），但我们必须考虑到行间距，因此在除以 `var(--n)` 之前需要从总高度（`100%`）中减去 `n - 1` 个间距（`--g`）。当两个区块可见时，间距也显示出来了。另外，我们将第 `3` 、`5` 、`7` ，即 `2n + 1` 个 `section` 的 `scroll-snap-stop` 设置为 `always` ，可以确保每次向上（或回退时）滚动两个 `section` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9b3acb26f7f54aaeba03bb7f6e310de1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1112&h=608&s=18138171&e=gif&f=63&b=c7d6df)

  


> Demo 地址：https://codepen.io/airen/full/QwLyyMO

  


我们也可以通过简单的修复上一个示例的布局代码，将布局转换为垂直布局。即每个 `section` 的高度与浏览器视窗高度相等，所有区域以列的方式排列，并且每屏显示的列数同样由 `--n` 来控制：

## 创建可复用的布局模板

  


现在，我们把之前介绍的布局技巧组合起来创建一个可复用的布局模板。这个布局模板能根据相应的 HTML 结构自动调整布局效果。

  


以下面这个 HTML 结构为模板：

  


```HTML
<body>
    <header></header>
    <aside></aside>
    <main>
        <section></section>
        <!-- 省略其他的 section -->
    </main>
    <footer></footer>
</body>
```

  


在这里为了不使问题复杂化，以最常见的布局为例，它可以拥有一个页眉（`<header>`）、一个页脚（`<footer>`）、一个可选的侧边栏（`<aside>`）和可滚动的主内容区块（`<main>`），并且所有元素在同一屏可见。

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    --g: 1rem;
    --s: 220px;
    --n: 2;
    --m: 3;
    
    min-height: auto;
    gap: var(--g);
    min-height: 0;
    grid-template-rows: auto minmax(0, 1fr) auto;

    &:has(main + aside) {
        grid-template-columns: minmax(0, 1fr) var(--s);
    }
    
    &:has(aside + main) {
        grid-template-columns: var(--s) minmax(0, 1fr);
    }

    :is(header, footer) {
        grid-column: 1 / -1;
    }

    main {
        display: grid;
        gap: var(--g);
        grid-auto-flow: column;
        grid-auto-columns: calc((100% - (var(--n) - 1) * var(--g)) / var(--n));
        grid-template-rows: repeat(var(--m), minmax(0, 1fr));
        scroll-snap-type: x mandatory;
        overflow-x: auto;
        padding: var(--g);
        scroll-padding: var(--g);

        section {
            scroll-snap-align: start;
        }
    }
}
```

:is(html, body) {

    min-height: auto;

    grid-template-rows: auto minmax(0, 1fr) auto;

    &:has(main + aside) {
        grid-template-columns: minmax(0, 1fr) var(--s);
    }
  
    &:has(aside + main) {
        grid-template-columns: var(--s) minmax(0, 1fr);
    }

    :is(header, footer) {
        grid-column: 1 / -1;
    }

    main {
        display: grid;
        gap: var(--g);
        grid-auto-flow: column;
        grid-auto-columns: calc((100% - (var(--n) - 1) * var(--g)) / var(--n));
        grid-template-rows: repeat(var(--m), minmax(0, 1fr));
        scroll-snap-type: x mandatory;
        overflow-x: auto;
        padding: var(--g);
        scroll-padding: var(--g);

        section {
            scroll-snap-align: start;
        }
    }

    @media (width <= 767px) {
        --s: 150px;
        --n: 1;
        --m: 2;
    }
    
    @media (width <= 480px) {
        grid-template-columns: 1fr !important;
        grid-template-rows: none;
        min-height: auto;

            grid-auto-flow: row;
            grid-auto-rows: 1fr;
        }
    }
}
