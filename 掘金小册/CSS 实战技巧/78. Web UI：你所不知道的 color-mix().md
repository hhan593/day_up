## 了解 `color-mix()` 函数

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f4388e5b0a8743f0ab71d715c49ecda8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=1024&s=275396&e=webp&b=385577)

  


`color-mix()` 函数允许我们指定两种要混合的颜色，并输出混合后的结果。我们可以指定颜色空间（即颜色插值算法）和控制每种颜色在混合中的比例，从而获得不同的混合颜色：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e731bd66360441a5be860c3ec8504c08~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1529&s=523742&e=jpg&b=ffffff)

  


上图展示了 `color-mix()` 函数的语法规则，其中颜色空间是一个必填参数。稍后我们会详细介绍。目前，我们先使用 `srgb` 进行示例讲解。

  


我们可以用百分比来指定每种颜色的混合比例。如果省略两种颜色的百分比，`color-mix()` 函数会默认使用 `50%` 进行对比：

  


```CSS
:root {
    --c: color-mix(in srgb, blue, red);    
}

/* 等同于 */
:root {
    --c: color-mix(in srgb, blue 50%, red 50%);
}
```

  


上面的代码将 `50%` 的 `blue` （蓝色）与 `50%` 的 `red` （红色）混合在一起，将会获得一个紫色（`#800080`）。

  


如果只指定一种颜色的百分比，那么另一种颜色的百分比会自动调整，以保证总和为 `100%` 。例如，无论我们指定 `blue 90%` 还是 `red 10%` ，最终的颜色结果都是相同的，即 `#1a0016` ：

  


```CSS
:root {
    --c: color-mix(in srgb, blue 90%, red);
    /* 等同于 */
    --c: color-mix(in srgb, blue 90%, red 10%);
    /* 等同于 */
    --c: color-mix(in srgb, blue, red 10%);
}
```

  


如果两种颜色的百分比总和小于 `100%` ，`color-mix()` 的行为会稍有不同：总各会作为一个透明度乘数（比率），并用它来缩放两种颜色，使它们的总和达到 `100%` 。例如：

  


```CSS
:root {
    --c1: color-mix(in srgb, blue, red); /* 结果是 rgb(128 0 128) */
    --c2: color-mix(in srgb, blue 20%, red 20%); /*结果是 rgb(128 0 128 / .4)*/
}
```

  


虽然上面两个 `color-mix()` 都是在 `srgb` 颜色空间中对 `blue` 和 `red` 进行混合，但 `--c1` 的两种颜色混合比例都是 `50%` （总和为 `100%`），它们的混合结果是 `rgb(128 0 128)` 。而 `--c2` 的两种颜色混合比例总和为 `40%` ，因此它的最终颜色结果与 `--c1` 相同，但透明度为 `0.4` ，实际结果为 ` rgb(128 0 128  / .4)` 。

  


这是一个很有意思的地方，我们可以利用这个来调整颜色的透明度，稍后我们会介绍。你可以尝试调整下面示例中的颜色和混合比例，查看混合后的结果：

## 在 `color-mix()` 中指定颜色空间

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f09ab3e807c04f9a9f38452656cdff0d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=523141&e=png&b=071d44)

  


在之前的示例中，我们在 `color-mix()` 函数中使用的是 `srgb` 颜色空间。然而，通过更改颜色空间，我们可以显著改变最终的颜色效果。

  


颜色空间是一个复杂的知识体系，这已超出这节课的范畴，但在使用 `color-mix()` 函数时，了解不同的颜色空间的优缺点仍然很重要。如果你对颜色空间不太了解的话，个人建议你花点时间阅读：

  


-   [现代 CSS 中的颜色格式： RGB，HSL，HWB，LAB 和 LCH](https://juejin.cn/book/7223230325122400288/section/7231515598088306720)
-   [新的 CSS 颜色空间：为 Web 设置高清颜色](https://juejin.cn/book/7223230325122400288/section/7233227753909125178)

  


需要知道的是，`color-mix()` 函数中第一个参数是必选参数，它决定了颜色插值的算法——决定了一个颜色如何过渡到另一个颜色。可视化这一过程的好方法是使用渐变，正如《[新的 CSS 颜色空间：为 Web 设置高清颜色](https://juejin.cn/book/7223230325122400288/section/7233227753909125178)》课程在“[颜色插值](https://juejin.cn/book/7223230325122400288/section/7233227753909125178#heading-28)”部分所示。

  


经典的 RGB 插值可能会导致渐变中心区域（中间过渡部分）的颜色变得浑浊（俗称“灰色死亡区”），而使用 `lch` 或 `oklch` 时，颜色更鲜艳。与 `srgb` 和 `hsl` 不同，`oklch` 和 `lch` 都是感知均匀的。这意味着，相同数值的坐标变化会产生相同的感知颜色差异。因此，在使用 `color-mix()` 进行颜色插值时，我更倾向于选择 **[高清颜色空间](https://juejin.cn/book/7223230325122400288/section/7233227753909125178#heading-12)**，比如 `lch` 、`oklch` 、`oklab` 等。例如下面这个示例，你可以尝试着调整颜色，对比 `color-mix()` 函数使用不同的颜色插值的结果：

  


```CSS
li {
    background: color-mix(
        in var(--s),
        var(--base-color),
        var(--blend-color) calc(var(--i) * 2.5%) /* --i 是 li 索引号，从 0 开始*/
    );
}
```

在极坐标（或环形）颜色空间（如 `oklch` 、`lch` 、`hwb` 和 `hsl`）中，我们可以选择颜色插值的方向。当两个颜色等量混合时，最终的色相角度将位于两个颜色角度的中间。但这个角度的计算方式会有所不同，具体取决于插值是沿着短路径（顺时针或逆时针的最短距离）还是长路径（绕更长的方向）：

  


```CSS
:root {
    --c1: color-mix(in hsl, rgb(255 88 88), rgb(86 86 255));
    --c2: color-mix(in hsl longer hue, rgb(255 88 88), rgb(86 86 255));
}
```

  


上面代码中的 `color-mix()` 在 `hsl` 颜色空间中比较不同插值路径，其中：

  


-   `--c1` 是短路径插值（`color-mix()` 默认为短路径 `shorter hue` ），会产生偏粉紫色（如下图左侧所示）
-   `--c2` 是长路径插值（`longer hue`），会产生亮绿色（如下图右侧所示）

## 你所不知道的 `color-mix()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/255985db1901441e81d15e6e22bb7140~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=526265&e=png&b=7ecbf2)

  


通过前面的学习，我想你对 CSS 的 `color-mix()` 函数有了基本的认识，并且可以通过它来创建想要的颜色变体。事实上，我们要是能用好它，可以帮助我们解决很多实际问题，尤其是颜色的管理和维护方面。接下来，通过一些实际用例来展示它的一些功能与使用技巧。

  


### 使用 `color-mix()` 创建浅色和深色变体

  


> 我们可以使用 `color-mix()` 来替代CSS 处理器（例如 Sass）中的 `lighten()` 和 `darken()` 函数。

  


在实际应用中，我们经常需要创建品牌颜色的浅色（变亮）或深色（变暗）变体。为了实现这一点，我们可以使用 `color-mix()` 将白色（`white`）或黑色（`black`）与基色按不同的经例混合，从而调整颜色的亮度：

  


-   与白色混合将获得更亮的颜色（浅色变体），类似于 Sass 的 `lighten()` 函数
-   与黑色混合将获得更暗的颜色（深体变体），类似于 Sass 的 `darken()` 函数

  


下面的示例演示了如何通过 `color-mix()` 将不同比例的白色和黑色与红色（基色）混合，以创建它的浅色和深色变体，从而调整基色（`red`）的亮度：

  


```CSS
:root {
    --base-color: red;
    --c-ratio: 5%; 
}

/* 变亮 */
.lighten {
    background-color: color-mix(in oklch, var(--base-color), white calc(var(--i) * var(--c-ratio)));
}

/* 变暗*/
.darken {
    background-color: color-mix(in oklch, var(--base-color), black calc(var(--i) * var(--c-ratio)));
}
```

> Demo 地址：https://codepen.io/airen/full/jEObmKw

  


需要知道的是，`color-mix()` 的百分比和 `darken()` 或 `lighten()` 的百分比不完全匹配，而且不同的颜色空间会产生不同的结果。另外，`color-mix()` 函数调整颜色的亮度与 [CSS 相对颜色语法调整颜色的亮度](https://juejin.cn/book/7395119173656903715/section/7445265166482505778#heading-4)也略有不同。感兴趣的同学可以自己测试一下！

  


### 使用 `color-mix()` 创建透明度变体

  


在 LESS 和 Sass 中，我们可以使用变量创建相同颜色的不同透明度版本：

  


```SCSS
$color-primary: rgb(255 125 0);
color: rgb($color-primary, 0.5);
```

  


即使透明度已经在变量中定义，也可以覆盖它：

  


```SCSS
$color-highlight: rgb(255 125 0 / 0.5); // 初始 alpha 0.5
background-color: rgb($color-highlight, 0.9); // 透明度被覆盖为 0.9
```

  


但在 CSS 中，还没有相对颜色语法和 `color-mix()` 之前，我们通常会使用 **[CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)** 来实现类似的功能：

  


```CSS
:root {
    --color-primary-rgb: 255 125 0;
    --color-primary: rgb(var(--color-primary-rgb));
}

.element {
    background-color: rgb(var(--color-primary-rgb) / 0.75);
    color: var(--color-primary);
}
```

  


然而，如果要为每个颜色都定义 `--color-rgb` 变量，管理起来会很麻烦。随着 [CSS 相对颜色语法的出现](https://juejin.cn/book/7395119173656903715/section/7445265166482505778#heading-9)，在 CSS 中创建颜色透明度变体要简单得多：

  


```CSS
.element {
    --brand-color: green;  
    background: oklch(from var(--brand-color) l c h / 50%); /* 指定透明度为 50% */
}

/* 或者 */
.element {
    --brand-color: green;
    background: oklch(from var(--brand-color) l c h / calc(alpha / 2)); /* 👉 alpha = 1 ÷ 2 = 50% */
}
```

  


现在，我们还可以用 `color-mix()` 来调整颜色透明度。关键在于混合透明色（`transparent`）：

  


```CSS
:root {
    --base-color: red;
    --opacity-ratio: 5%;
}

.element {
    background: color-mix(in oklch, transparent, var(--base-color) calc(var(--i) * var(--opacity-ratio)));
}

/* 或者 */
.element {
    background: color-mix(in oklch, transparent calc(var(--i) * var(--opacity-ratio)), var(--base-color));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7a8ffa7cb52846a5ae9ec9c3ce9ff483~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1370&h=316&s=2988274&e=gif&f=126&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/azbvRbV

  


需要知道的是，如果百分比作用于基本颜色（`--base-color`），该百分比对应颜色的透明度，比如：

  


```CSS
:root {
    --base-color: rgb(255 125 0);
    color: color-mix(in oklch, transparent, var(--base-color) 75%); /* 等同于 rgb(255 125 0 / 0.75) */
}
```

  


如果百分比作用于 `transparent` ，那么颜色的透明度为 `100%` 减去 `transparent` 的混合比例，比如：

  


```CSS
:root {
    --base-color: rgb(255 125 0);
    color: color-mix(in oklch, transparent 75%, var(--base-color)); /* 等同于 rgb(255 125 0 / 0.25) */
}
```

  


如果不显式指定混合比例，那么颜色的透明度为 `50%` ：

  


```CSS
:root {
    --base-color: rgb(255 125 0);
    color: color-mix(in oklch, transparent, var(--base-color)); /* 等同于 rgb(255 125 0 / 0.5) */
}
```

  


需要注意的是，在某些情况下，透明色（`transparent`）会导致颜色的偏移，为了避免这个现象出现，尽可能使用**相同颜色的透明版本**，而不是 `transparent`：

  


```CSS
:root {
    --base-color: #f06d06;
    --transparent: #f06d0600;
    --c1: color-mix(in srgb, var(--base-color), var(--transparent) 50%);
    --c2: color-mix(in srgb-linear, var(--base-color), var(--transparent) 50%);
    --c3: color-mix(in rec2020, var(--base-color), var(--transparent) 50%);
    --c4: color-mix(in display-p3, var(--base-color), var(--transparent) 50%);
    --c5: color-mix(in a98-rgb, var(--base-color), var(--transparent) 50%);
    --c6: color-mix(in prophoto-rgb, var(--base-color), var(--transparent) 50%);
    --c7: color-mix(in xyz, var(--base-color), var(--transparent) 50%);
    --c8: color-mix(in xyz-d50, var(--base-color), var(--transparent) 50%);
    --c9: color-mix(in xyz-d65, var(--base-color), var(--transparent) 50%);
    --c10: color-mix(in lab, var(--base-color), var(--transparent) 50%);
    --c11: color-mix(in oklab, var(--base-color), var(--transparent) 50%);
    --c12: color-mix(in lch, var(--base-color), var(--transparent) 50%);
    --c13: color-mix(in oklch, var(--base-color), var(--transparent) 50%);
    --c14: color-mix(in hsl, var(--base-color), var(--transparent) 50%);
    --c15: color-mix(in hwb, var(--base-color), var(--transparent) 50%);
    --c16: color-mix(in lch longer hue, var(--base-color), var(--transparent) 50%);
    --c17: color-mix(in oklch longer hue, var(--base-color), var(--transparent) 50%);
    --c18: color-mix(in hsl longer hue, var(--base-color), var(--transparent) 50%);
    --c19: color-mix(in hwb longer hue, var(--base-color), var(--transparent) 50%);
}
```

### 使用 `color-mix()` 创建暖色和冷色变体

  


虽然 `color-mix()` 最常用于创建某个颜色的浅色或深色变体，但我们还可以通过混合暖色或冷色来生成不同的色调，使原始调色板更具变化。

  


下面这个示例，我们首先定义一个基础调色板，并指定用于混合暖色和冷色的颜色，所有这些都使用 CSS 自定义属性管理：

  


```CSS
:root {
    --yellow: rgb(221 215 141);
    --peach: rgb(220 191 133);
    --chocolate: rgb(139 99 92);
    --khaki: rgb(96 89 77);
    --grey: rgb(147 162 155);
    --mix-warm: red;  /* 暖色混合基准 * /     --mix-cool: blue; /* 冷色混合基准 */
}

ul {
    --c: var(--yellow);
}
```

  


接下来，我们利用 `color-mix()` 在原始颜色的基础上混合第二种颜色，并且可以指定混合的比例：

  


```CSS
li {
    --amount: 10%;
    --c-warm: color-mix(in srgb, var(--c), var(--mix-warm) calc(var(--i) * var(--amount)));
    --c-cool: color-mix(in srgb, var(--c), var(--mix-cool) calc(var(--i) * var(--amount)));
    background: var(--c_bg);
}
```

  


我们通过 `--i` 和 `--amount` 来调整混合比例。这样，我们可以灵活地创建暖色和冷色的不同层次变化，并应用到整个 UI 设计中。

### 使用 `color-mix()` 创建颜色数组

  


不知道大家是否还记得，我曾在《[CSS 图形: CSS 渐变技巧](https://juejin.cn/book/7395119173656903715/section/7429225312224804899#heading-6)》课程中提到过 `color-mix()` 函数可用来创建**两个颜色的数组**。

  


我们通过一个 CSS 自定义属性来储存两个颜色：

  


```CSS
:root {
    --colors: black, white;
}
```

  


然后，使用 `color-mix()` 返回其参数列表中的两个颜色之一：

  


```CSS
:root {
    --colors: black, white;
    --i: 0;
    --c: color-mix(in oklch, var(--colors) calc(var(--i) * 100%));
}
```

  


上面的代码中新增了变量 `--i` ，通过更新 `--i` 的值来选择颜色。`--i` 的值要么是 `0` ，要么是 `1` ，类似于布尔检查。

  


-   如果 `--i` 的值为 `0` ，那么 `--c` 的值为 `color-mix(in oklch, black, white 0%)` ，结果是黑色，因为白色的混合比例是 `0%` 。我们将 `100%` 的黑色与 `0%` 白色混合得到了纯黑色
-   如果 `--i` 的值为 `1` ，那么 `--c` 的值为 `color-mix(in oklch, black, white 100%)`，结果是纯白色，因为白色的混合比例是 `100%` ，而黑色的混合比例是 `0%`

  


想象一下，如果你想在网站设计中添加暗黑模式，这种技术就非常有用了。你可以将两种颜色都定义在同一个变量中。然后在切换暗黑模式时，调整 `--i` 的值即可。

  


```CSS
[data-type="error"] {

}

[data-type="ai"] {
    --color: oklch(58.11% 0.31 307.02);
}

[data-type="tip"] {
    --color: oklch(53.24% 0.23 256.22);
}

[data-type="warning"] {
    --color: oklch(61.47% 0.16 64.21);
}

[data-type="success"] {
    --color: oklch(47.06% 0.17 148.76);
}

[data-type="note"] {
    --color: oklch(41.84% 0 0);
}

    color-scheme: light dark;

    @media (prefers-color-scheme: dark) {
        color-scheme: dark;
    }

    @media (prefers-color-scheme: light) {
        color-scheme: light;

            --i: 1;

    --lc-color: color-mix(in oklch, var(--color) 100%, oklch(100% 0 0));

}

}

    --backgrounds: hsl(0 0% 98%), hsl(0 0% 2%);
    background: color-mix(in oklch, var(--backgrounds) calc(var(--i) * 100%));

    @media (prefers-color-scheme: dark) {
        --i: 1;
    }
```

### `color-mix()` 与 `currentColor`

  


在 CSS 中给颜色属性设置颜色时，还可以使用 `currentColor` ，而且 `currentColor` 是一个很有意思的属性，它可以根据元素当前的 `color` 或其父元素（或祖先元素）来决定 `currentColor` 的值。换句话说，在一些场景中使用 `currentColor` 作为颜色属性的值时，可以根据 `color` 来动态匹配颜色值。

这个相对变量 `currentColor` 同样也能与 `color-mix()` 一起使用。例如：

  


```CSS
.box {
    color: red;
    border: 5px solid color-mix(in srgb, currentColor 30%, white);
    background-color: color-mix(in srgb, currentColor 30%, black);
    outline: 5px solid currentColor;
    outline-offset: 5px;
}
```

### 使用 `color-mix()` 和 `none` 模拟相对颜色语法

  


这是一个非常有趣的用例，我们可以通过 `color-mix()` 和关键词来模拟 [CSS 的相对颜色语法](https://juejin.cn/book/7395119173656903715/section/7445265166482505778)。例如：

  


```CSS
button {
    /* 相对颜色语法:  
        background: linear-gradient(
            oklch(from var(--color) .92 c h),
            oklch(from var(--color) .8 c h);
    )
    */
    background: linear-gradient(
        color-mix(in oklch, var(--color) 0%, oklch(.92 none none)),
        color-mix(in oklch, var(--color) 0%, oklch(.8 none none))
    );
        
    /* 相对颜色语法：
       border: 1px solid oklch(from var(--color) .5 c h);
    */
    border: 1px solid color-mix(in oklch, var(--color) 0%, oklch(.5 none none));
        
    /* 相对颜色语法: 
       color: oklch(from var(--color) .32 c h);
    */
    color: color-mix(in oklch, var(--color) 0%, oklch(.32 none none));
}
```
