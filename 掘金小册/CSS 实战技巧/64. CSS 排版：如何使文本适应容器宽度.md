## 我们的目标

  


这节课的目标非常明确，使用纯 CSS 实现文本适应容器宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6dd02778bce0401aa1377a395578a7b9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1668&h=670&s=13291052&e=gif&f=264&b=131304)

  


当我面对这个需求的时候，我首先想到的是 CSS 容器单位。但我碰到了一个问题：**CSS 如何计算出容器可用空间与文本字符宽度的比例**。这个比例非常关键，它将作为一个修改器应用到文本的 `font-size` 上。这意味着，我们要使用现有的 CSS 特性来获得这个比例值。

  


这就引出两个问题：**我们如何获得这个比例呢？我们如何测量内容的内联大小，尤其是当它未知时？** 接下来，我将带大家一起来解开这两个秘密！

  


我们直接从代码开始！首先，你需要类似下面这样的 HTML 结构：

  


```HTML
<h2 class="container-fill-text">
    <span class="container-fill-text__container">
        <span class="container-fill-text__display">CSS Is<br>Awesome.</span>
    </span>
    <span class="container-fill-text__reference" aria-hidden="true">CSS Is<br>Awesome.</span>
</h2>
```

  


你可能已经发现了，上面代码中有重复的内容（`.container-fill-text__reference`），为什么要多添加这样的一个 `<span>` ，稍后会解释。另外，我们在这个 `<span>` 元素设置了 `aria-hidden="true"` ，用于隐藏重复的内容，并且对于辅助技术（如屏幕阅读器）来说是无感的。此外，我更倾向于用额外的 HTML 替代 JavaScript，因为 HTML 的稳定性更高，不容易出错！

  


实现目标效果所需要的 CSS 代码如下：

  


```CSS
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    --container-fill-text-captured-length: initial; /* 捕获的长度 */
    display: flex;
    container-type: inline-size;
    max-width: unset;
    line-height: 0.85;
    font-weight: 900;

    > [aria-hidden] {
        visibility: hidden; /* 这个很重要 */
    }

    > :not([aria-hidden]) {
        --container-fill-text-captured-length: 100cqi; /* 捕获的长度 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length); /* 可用空间 */
        flex-grow: 1;
        container-type: inline-size;

        > * {
            --container-fill-text-captured-length: 100cqi; /*--captured-length 捕获的长度*/
            --container-fill-text-ratio: tan(
                atan2(
                    var(--container-fill-text-available-space),
                    var(--container-fill-text-available-space) - var(--container-fill-text-captured-length)
                )
            ); /* 容器可用空间与文本字符长度的比例 */
            --font-size: clamp(
                clamp(1.055rem, 0.95rem + 0.52vw, 1.35rem),
                1em * var(--container-fill-text-ratio),
                var(--container-fill-text-max-font-size, infinity * 1px)
            ); /* 字号*/
            display: block;
            inline-size: var(--container-fill-text-available-space);
            font-size: var(--font-size);
            
            @container (inline-size > 0) {
                white-space: nowrap;
            }
        }
    }
}
```

## 它是如何工作的？

  


刚才我们提到过，实现这个需求的关键之处是：如何计算出容器可用空间与文本字符长度的比例。这意味着，我们需要两个尺寸：容器可用空间的宽度和文本字符（而且是未知文本字符）长度。

  


相比而言，CSS 获取容器可用空间（或者说容器尺寸）要简单的多，我们使用 CSS 容器查询单位就可以获取到，比如 `100cqi` （容器的内联尺寸 `inline-size`），然而要获取未知文本字符的内联尺寸就要复杂得多。因为，当我们设置一个容器查询上下文时，容器会停止考虑其子元素的尺寸，我们就失去了这个值。

  


换句话说，如果我们在未知内联尺寸的文本上设置了容器查询，就无法直接测量它的尺寸。不过，我们可以反其道而行：不直接测量元素自身的内联尺寸，而是测试容器剩余空间的大小！如果我们能测量顶级容器的宽度，就可以从中减去剩余空间，最终得到那个原本未知的，现在已知的宽度。正因如此，我们在 HTML 结构中才会有一个重复内容的 `.container-fill-text__reference` 。

  


```HTML
<h2 class="container-fill-text">
    <span class="container-fill-text__container">
        <span class="container-fill-text__display">CSS Is<br>Awesome.</span>
    </span>
    <span class="container-fill-text__reference" aria-hidden="true">CSS Is<br>Awesome.</span>
</h2>
```

  


我们通过一个简化的示例来演示这种技术，大家更易于理解。就上面的 HTML 结构而言，其中：

  


-   `.container-fill-text__display` 对应的是适应宽度的文本，它是未知的（如下图中红色区域）
-   `.container-fill-text__reference` 对应的是适应宽度的文本的副本（如下图中绿色区域）
-   `.container-fill-text` 对应的就是顶级容器（如下图中橙色边框）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9e90a1c688de4990bc581e034f0b51e7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=453996&e=jpg&b=000000)

  


也就是说，当我们在容器（`.container-fill-text`）中放置一个未知宽度的元素（`.container-fill-text__reference`），然后在其旁边添加另一个设置了 `container-type: inline-size` 和 `flex-grow: 1` 的元素（`.container-fill-text__container`）时，它会扩展以占据剩余空间，同时作为一个查询容器存在。这种方式符合查询容器的规则：其子元素（`.container-fill-text__display`）不会影响容器，但兄弟元素（`.container-fill-text__reference`）会对其产生影响！

  


我们使用下面的 CSS 代码来描述上图：

  


```CSS
.container-fill-text {
    display: flex;
    container-type: inline-size;
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        flex-grow: 1;
        container-type: inline-size;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e44004a0e3a34a26ad4d97827dc502b8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1660&h=382&s=1269912&e=gif&f=196&b=121204)

  


> Demo 地址：https://codepen.io/airen/full/yyBvdZW

  


现在，`.container-fill-text` （顶部容器）和 `.container-fill-text__container` （内部容器）都是查询容器，它们都设置了 `container-type: inline-size` 。

  


-   顶部容器，占据所有可用空间
-   内部容器，通过代理测量未知文本的宽度

  


我们希望能够同时访问这两个容器的尺寸，但当前的 CSS 还不支持这个功能。目前，CSS 中只有容器查询单位（如 `cqi`），根据规范，它们的值取自最近的查询容器。那么，如何同时获取最近查询容器的尺寸，并跳过它以访问另一个查询容器的尺寸呢？

  


**[CSS 的自定义属性 `@property`](https://juejin.cn/book/7223230325122400288/section/7258870477462962236)** 可以帮助我们解决这个问题。当我们使用 `@property` 注册一个 CSS 自定义属性并为其赋值时，它会在计算值阶段捕获该值，从而可以通过继续访问它。具体的 CSS 代码如下：

  


```CSS
/* 注册一个 CSS 自定义属性 */
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    display: flex;
    container-type: inline-size; /* 1️⃣ */
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        container-type: inline-size; /* 2️⃣ */
        flex-grow: 1;
        /* 捕获未知文本字符长度 */
        --container-fill-text-captured-length: 100cqi; /* 3️⃣ */
        /* 可用空间 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length); /* 4️⃣ */
        
        > * {/* .container-fill-text__display */
             --container-fill-text-captured-length: 100cqi;/* 5️⃣ */
        }
    }
    
    > [aria-hidden] { /* .container-fill-text__reference */
        visibility: hidden; /* 6️⃣ */
    }
}
```

  


简单的解释一下上面的代码：

  


-   1️⃣：将最外层的容器（`.container-fill-text`）设置为可查询容器，这样就能捕获顶容器的内联尺寸（宽度）
-   2️⃣：将元素 `.container-fill-text__container` 也设置为可查询容器，它会占据最外层容器的剩余空间。此步骤确保可以通过子容器测量未知文本的宽度
-   3️⃣：将 `@property` 注册的自定义属性 `--container-fill-text-captured-length` 的值设置为 `100cqi` ，该值将在此元素上进行计算。由于容器查询单位仅适用于某些容器的子元素，即使此子元素本身是可查询容器，它仍会从其父级获取 `cqi` 值
-   4️⃣：将计算后的值保存到另一个自定义属性 `--container-fill-text-available-space` 中，因为我们后面还会重复使用 `--container-fill-text-captured-length` 自定义属性，因此将其值保存到另一个自定义属性中（`--container-fill-text-available-space`）。CSS 自定义属性会在使用时扩展，因此即使稍后覆盖了 `--container-fill-text-captured-length` ，子元素仍然可以通过 `--container-fill-text-available-space` 访问捕获的值
-   5️⃣：在更深层次的子元素（`.container-fill-text__display`）中访问查询容器宽度。顶级容器的宽度可以通过 `--container-fill-text-available-space` 访问，中间容器的宽度则通过 `100cqi` 访问
-   6️⃣：`visibility: hidden;` 很重要，用于隐藏元素 **但仍然保留其在页面上的空间占位**。具体来说，它的行为是让元素“不可见”，但该元素仍然参与页面的布局计算

  


这样，我们就能实现灵活的容器查询和宽度测量逻辑，即使目前 CSS 尚未支持命名的容器查询单位。

  


现在，我们有了两个长度：

  


-   最外层容器（顶级容器）的内联尺寸 `--container-fill-text-available-space`
-   最接近容器（内部容器）的内联尺寸 `100cqi`

  


按理说，我们只需要将一个除以另一个值就能获得文本长度和容器宽度的比例。只不过，我们现在还不能直接这样做，因为目前还没有浏览器实现这一点。庆幸的是，我们可以借助 **[CSS 的三角函数](https://juejin.cn/book/7223230325122400288/section/7258870477462962236)** `tan(atan2())` 将 CSS 类型转换为数值。也就是说，通过使用 `tan()` 和 `atan2()` 函数，可以将一个长度除以另一个长度，并获得无单位的比例作为结果！这正是我们所需要的。

  


```CSS
/* 注册一个 CSS 自定义属性 */
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    display: flex;
    container-type: inline-size;
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        container-type: inline-size;
        flex-grow: 1;
        /* 捕获未知文本字符长度 */
        --container-fill-text-captured-length: 100cqi;
        /* 可用空间 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length);
        
        > * {/* .container-fill-text__display */
             --container-fill-text-captured-length: 100cqi;
             
             /* 文本长度与容器宽度的比例 */
             --container-fill-text-ratio: tan(
                atan2(
                    var(--container-fill-text-available-space),
                    var(--container-fill-text-available-space) -  var(--container-fill-text-captured-length)
                )
            );
        }
    }
    
    > [aria-hidden] { /* .container-fill-text__reference */
        visibility: hidden; 
    }
}
```

  


我们把这个比例保存在另一个 CSS 变量中 `--container-fill-text-ratio` ，因为我们后面计算文本的 `font-size` 需要用到这个变量。由于 `tan(atan2())` 在相关的浏览器还存在一定的缺陷，不能直接在 `--container-fill-text-ratio` 中使用 `100cqi` 来计算，但我们可以先将 `100cqi` 存到另一个 CSS 变量（`--container-fill-text-captured-length`）中，然后计算出比例，再重用它。

  


在继续往下之前，我想在这里简单的介绍一下 `tan(atan2())` 的工作原理。

  


如果你在 CSS 中使用过 `calc()` 函数做一些简单的数学运算，那么你肯定碰到过无法对长度类型进行除法运算，比如 ` calc(100vw  ``/ 5px``)` 是不起作用的。不过，这个功能最终会得到浏览器的支持，因为它已经在规范中，并且随着支持力度的增加，可能很快就会实现。但目前来说，还没有办法基于尺寸生成标量值（标量是一个纯数值）。

  


庆幸的是，我们可以使用 CSS 三角函数 `tan()` 和 `atan2()` 来实现。`atan2()` 函数接受两个参数，本质上是斜率（`y` 除以 `x` ，即垂直高度和水平宽之比），例如 `atan2(height, width)` ，它返回从斜率底部到顶部的角度。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d12ffd42d409424b890f5ed2aa81e4cb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=1029&s=31219&e=png&a=1&b=000000)

  


简单地说，`atan2()` 输入的是两个值 `y` 和 `x` ，输出的是一个弧度，即根据比例计算角度。而 `tan()` 是一个三角函数，接受一个角度（以弧度为单位），并返回该角的正切值，即“对边长度 ÷ 邻边长度”：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3172ee41e5604a528ba9a0633a2667fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2160&h=1620&s=291265&e=jpg&b=ffffff)

  


`tan(atan2(y,x))` （或者 `tan(atan2(height, width))`）是 `atan2()` 和 `tan()` 两个函数的组合，其作用是“还原”输入的比例关系（即 `Y ÷ X`）。

  


简单的小结一下：

  


-   **`atan2(Height, Width)`** 计算出的角度
-   **`tan(angle)`** 等于 `height/width` 的值
-   **`tan(atan2(Height, Width))`** 等于`height/width`

  


我们来看几个这方面的简单应用。

  


第一个案例就是计算屏幕尺寸。很多人希望能够获取浏览器视窗的像素宽度或高度作为数值，用来计算宽高比（`aspect-ratio`）或进一步进行其他计算。理论上，像下面这样使用 `tan(atan2())` 函数就可以：

  


```CSS
:root {
    --px-w: tan(atan2(100vw, 1px));
}
```

  


但是，到目前为止，`atan2()` 在浏览器中表现非常不稳定。为了避免这种现象出现，我们需要借助 `@property` 的特性，注册一个 CSS 自定义属性，并将其 `syntax` 设置为 `<length>` 类型，然后赋值为 `100vw` ，浏览器就会自动完成这个转换：

  


```CSS
@property --w {
    syntax: "<lenth>";
    initial-value: 0px;
    inherits: false;
}

:root {
    --w: 100vw;
    --px-width: tan(atan2(var(--w), 1px));
}
```

  


下面这个示例，向大家展示了如何通过 `tan(atan2())` 来获取浏览器视窗的宽度和高度：

  


```CSS
@property --w {
    syntax: "<length>";
    initial-value: 0px;
    inherits: false;
}

@property --h {
    syntax: "<length>";
    initial-value: 0px;
    inherits: false;
}

:root {
    --w: 100vw;
    --h: 100vh;
    --px-width: calc(10000 * tan(atan2(var(--w), 10000px)));
    --px-height: calc(10000 * tan(atan2(var(--h), 10000px)));
}

.box {
    height: calc(var(--px-height) * 1px);
    width: calc(var(--px-width) * 1px);

    &::before {
        counter-reset: w var(--px-width) h var(--px-height);
        content: "w: " counter(w) " h: " counter(h);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2b80ed49ea244546a123c30bdf644f4b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1374&h=664&s=2436520&e=gif&f=149&b=131305)

  


> Demo 地址：https://codepen.io/airen/full/RNbQXEq

  


你可以用同样的方法来处理任何容器查询单位，包含混合单位的计算尺寸，获取 `rem` 的像素值，或者其他你想要的东西：

  


```CSS
@property --length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.box {
    font-size: clamp(1.5rem, 3vw + 2rem, 6rem);

    &::after {
        --length: 1em;
        counter-reset: val calc(10000 * tan(atan2(var(--length), 10000px)));
        content: counter(val);
    }
}
```

> Demo 地址：https://codepen.io/airen/full/pvzLvNz

  


通过这个技巧，可以在 CSS 中利用 `tan(atan2())` 实现当前 `calc()` 尚不支持的除法操作，同时扩展了在容器查询和单位计算中的应用潜力！

  


我想，你现在已经了解 `tan(atan2())` 的工作原理了。这也是为什么我们在计算 `--container-fill-text-ratio` （文本宽度与容器宽度比例）的时候为什么要使用 `tan(atan2())` 函数了。

  


到目前为止，我们使用 CSS 获得了我们最想要的关键数据——**文本长度与容器宽度的比例**。利用这个比例，就可以计算出文本的 `font-size` 了：

  


```CSS
/* 注册一个 CSS 自定义属性 */
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    display: flex;
    container-type: inline-size;
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        container-type: inline-size;
        flex-grow: 1;
        /* 捕获未知文本字符长度 */
        --container-fill-text-captured-length: 100cqi;
        /* 可用空间 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length);
        
        > * {/* .container-fill-text__display */
             --container-fill-text-captured-length: 100cqi;
             
             /* 文本长度与容器宽度的比例 */
             --container-fill-text-ratio: tan(
                atan2(
                    var(--container-fill-text-available-space),
                    var(--container-fill-text-available-space) -  var(--container-fill-text-captured-length)
                )
            );
            --font-size: clamp(
                clamp(1.055rem, 0.95rem + 0.52vw, 1.35rem),
                1em * var(--container-fill-text-ratio),
                var(--container-fill-text-max-font-size, infinity * 1px)
            );
            font-size: var(--font-size);
        }
    }
    
    > [aria-hidden] { /* .container-fill-text__reference */
        visibility: hidden; 
    }
}
```

  


在定义 `--font-size` 的值时，我们应用了两个 `clamp()` 函数。`clamp()` 函数有三个参数：

  


-   第一个参数 `clamp(1.055rem, 0.95rem + 0.52vw, 1.35rem)` 是最小值，它自身也是一个 `clamp()` 函数。这意味着 `font-size` 的最小值又会从 `clamp()` 中选择符合条件的值
-   第二个参数 `1em * var(--container-fill-text-ratio)` 是一个理想值（即 `--container-fill-text-ratio` 与 `1em` 的乘积），以确保不会影响用户整体字体大小的能力。如果仅使用视窗或容器查询单位，当用户缩放时，容器会变大，可能导致糟糕的效果
-   第三个参数 `var(--container-fill-text-max-font-size, infinity * 1px)` 是最大值。我们使用 `--container-fill-text-max-font-size` 自定义属性，如果未设置，则回退为 `infinity * 1px`，本质上没有大小限制

  


因为我们修改大小的文本位于一个正在增长但受限制的元素中，它并没有获得所有可用的空间。这不是大问题：我们已经保存了 `--container-fill-text-available-space` ，现在可以应用它，这样增大尺寸的元素就不会换行了。

  


```CSS
/* 注册一个 CSS 自定义属性 */
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    display: flex;
    container-type: inline-size;
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        container-type: inline-size;
        flex-grow: 1;
        /* 捕获未知文本字符长度 */
        --container-fill-text-captured-length: 100cqi;
        /* 可用空间 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length);
        
        > * {/* .container-fill-text__display */
             --container-fill-text-captured-length: 100cqi;
             
             /* 文本长度与容器宽度的比例 */
             --container-fill-text-ratio: tan(
                atan2(
                    var(--container-fill-text-available-space),
                    var(--container-fill-text-available-space) -  var(--container-fill-text-captured-length)
                )
            );
            --font-size: clamp(
                clamp(1.055rem, 0.95rem + 0.52vw, 1.35rem),
                1em * var(--container-fill-text-ratio),
                var(--container-fill-text-max-font-size, infinity * 1px)
            );
            font-size: var(--font-size);
            inline-size: var(--container-fill-text-available-space);
        }
    }
    
    > [aria-hidden] { /* .container-fill-text__reference */
        visibility: hidden; 
    }
}
```

  


这里有一个小细节需要注意，我们调整后的文本宽度有可能会超出容器的可用空间，毕竟我们这只是做了近似处理——文本字符宽度约等于容器宽度。这就有可能会造成文本换行。如果不希望它换行的话，我们可以通过容器查询来加以限制：

  


```CSS
/* 注册一个 CSS 自定义属性 */
@property --container-fill-text-captured-length {
    syntax: "<length>";
    initial-value: 0px;
    inherits: true;
}

.container-fill-text {
    display: flex;
    container-type: inline-size;
    
    > :not([aria-hidden]) { /* .container-fill-text__container */
        container-type: inline-size;
        flex-grow: 1;
        /* 捕获未知文本字符长度 */
        --container-fill-text-captured-length: 100cqi;
        /* 可用空间 */
        --container-fill-text-available-space: var(--container-fill-text-captured-length);
        
        > * {/* .container-fill-text__display */
             --container-fill-text-captured-length: 100cqi;
             
             /* 文本长度与容器宽度的比例 */
             --container-fill-text-ratio: tan(
                atan2(
                    var(--container-fill-text-available-space),
                    var(--container-fill-text-available-space) -  var(--container-fill-text-captured-length)
                )
            );
            --font-size: clamp(
                clamp(1.055rem, 0.95rem + 0.52vw, 1.35rem),
                1em * var(--container-fill-text-ratio),
                var(--container-fill-text-max-font-size, infinity * 1px)
            );
            font-size: var(--font-size);
            inline-size: var(--container-fill-text-available-space);
            
            @container (inline-size > 0) {
                white-space: nowrap;
            }
        }
    }
    
    > [aria-hidden] { /* .container-fill-text__reference */
        visibility: hidden; 
    }
}
```
