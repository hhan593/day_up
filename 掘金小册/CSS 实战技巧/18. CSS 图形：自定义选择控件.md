## 概述

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d747245145849f89993cf3a1d7be47a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2040&h=812&s=632838&e=jpg&b=edeae9)

  


切换（Toggle）这个词指的是一种带有短柄的开关，每次被激活时会在两种状态之间切换。你每次“打开”灯的时候就会遇到它。至于“单选按钮”（Radio Buttons），这个词源于汽车收音机。过去的汽车收音机通常在拨盘下方有一组按钮，这些按钮可以用来快速切换电台，每按下一个按钮，它就会保持按下状态，直到你按下另一个按钮。这让你可以更方便地在电台之间切换。

  


有意思的是，用于 Web 上的用户界面元素，其设计通常来源于生活。例如，我们所说的选择控件，比如复选框、单选按钮、切换开关、单选标签和多选标签。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5c6ddd23cbb342198b20567bed4ba9cc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1138&s=356789&e=jpg&b=f8f8f8)

  


-   复选框（Checkboxes）：用于存在一个或多个独立选项时，用户可以选择任意数量的选项，包括不选、选择一个或多个
-   单选按钮（Radio Buttons）：用于存在两个或多个互斥选项时，用户必须从中选择一个
-   切换开关（Toggle Switches）：用于只有两个互斥选项的情况，并且总有一个默认值。切换的选择会立即生效
-   单选标签（Choice Chips）：是单选按钮的紧凑替代方案。包含至少两个选项，单选标签表示用户可以做出的单一选择
-   多选标签（Multi-select Chips）：是复选框的紧凑替代方案。允许用户选择多个选项，主要用于移动设备上的筛选功能

  


到目前为止，上述提到的这些选择控件通常都是使用 `input` 的 `type` 为 `radio` 或 `checkbox` 来实现，包括切换开关。例如：

  


```HTML
<!-- 复选框 -->
<label class="checkbox" for="love">
    <input type="checkbox" name="love" id="love" /> I have a bike
</label>

<!-- 单选按钮 -->
<label class="radio" for="male">
    <input type="radio" id="male" name="gender" value="male" /> Male 
</label>
<label class="radio" for="female">
    <input type="radio" id="female" name="gender" value="female"/> Female 
</label>

<!-- 切换开关 -->
<label  class="switch" for="switch">
    Label text
    <input type="checkbox" role="switch" id="switch" switch />
</label>
```

  


它们与之前课程中介绍的[范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)（`<input type="range" />`）和[进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)（`<progress>`）类似，默认情况之下，它们的 UI 效果并不太好，几乎达不到 Web 设计师的需求。而且，它们在不同浏览器之间也有所不同：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/69a603ea15f048e0925615b4134f2510~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=476&s=225010&e=jpg&b=fbfbfb)

  


出于这个原因，我们总是倾向于采用自定义的方式为用户提供统一的 UI 和体验。

  


在这节课中，我们将研究一些 CSS 技术，这些技术允许我们在不增加额外 HTML 的情况之下更新这些元素的样式。是的，我们将仅使用 CSS 为我们的 `<input>` 元素赋予一个全新的 UI 视觉。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ef2b36f184f044548fcf6f3e61640772~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1931&h=1326&s=358659&e=jpg&b=faf6f2)

## 从 HTML 结构开始

  


我们从它们的 HTML 结构开始。在 HTML 中布局单选按钮、复选框和切换按钮有两种合适的方法。

  


第一种是将 `input` 包裹在 `label` 元素内。这种方法隐式地将 `label` 与 `input` 关联起来，同时也增加了它们的点击区域，让用户能与标签互动切换输入源：

  


```HTML
<!-- 单选按钮 -->
<label class="form__control">
    <input type="radio" id="CSS" name="language" /> CSS
</label>

<!-- 复选框 -->
<label class="form__control">
    <input type="checkbox" id="CSS" name="language" /> CSS
</label>

<!-- 切换按钮 -->
<label class="form__control">
    <input type="checkbox" id="CSS" name="language" switch /> CSS
</label>
```

  


你也可以根据实际需要，标签文本放置在 `input` 元素之前：

  


```HTML
<!-- 单选按钮 -->
<label class="form__control">
    CSS <input type="radio" id="CSS" name="language" /> 
</label>

<!-- 复选框 -->
<label class="form__control">
    CSS <input type="checkbox" id="CSS" name="language" /> 
</label>

<!-- 切换按钮 -->
<label class="form__control">
    CSS <input type="checkbox" id="CSS" name="language" switch /> 
</label>
```

  


很多时候，它们的标签内容是不可见的。在这种情况之下，可以考虑使用另一个标签来包裹标签文本内容：

  


```HTML
<!-- 单选按钮 -->
<label class="form__control">
    <span class="sr-only">CSS</span> 
    <input type="radio" id="CSS" name="language" /> 
</label>

<!-- 复选框 -->
<label class="form__control">
    <span class="sr-only">CSS</span> 
    <input type="checkbox" id="CSS" name="language" /> 
</label>

<!-- 切换按钮 -->
<label class="form__control">
    <span class="sr-only">CSS</span> 
    <input type="checkbox" id="CSS" name="language" switch /> 
</label>
```

  


我们可以给 `.sr-only` 应用下面 CSS ，从视觉上隐藏文本内容，这样可以确保辅助技术（例如屏幕阅读器）可以识别到标签内容：

  


```CSS
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}
```

  


第二种是将 `input` 和 `label` 设为兄弟元素，并使用 `for` 属性将其值设置为 `input` 的 `id` ，使 `label` 与 `input` 关联起来：

  


```HTML
<!-- 单选按钮 -->
<div class="form__control">
    <label for="css">CSS</label> 
    <input type="radio" id="css" name="language" /> 
</div>

<!-- 复选框 -->
<div class="form__control">
    <label for="css">CSS</label> 
    <input type="checkbox" id="css" name="language" /> 
</div>

<!-- 切换按钮 -->
<div class="form__control">
    <label for="css">CSS</div> 
    <input type="checkbox" id="css" name="language" switch /> 
</div>
```

  


同样的，如果不希望 `label` 的内容在视觉上呈现，也可以给 `label` 添加一个 `sr-only` 类名。注意，第一种方式也可以在 `label` 上设置 `for` 属性，并且与 `input` 的值关联起来。

  


需要注意的是，对于一组单选按钮，还需要给 `input` 提供相同的 `name` 属性。对于一组复选框，虽然没有明确要求给 `input` 提供相同的 `name` 属性，但我个人建议给它们提供相同的 `name` 。这样也可以告诉浏览器，这些复选框属于同一个组。

  


```HTML
<!-- 单选按钮 -->    
<div class="form__control">
    <label for="css">
        CSS
        <input type="radio" id="css" name="language" /> 
    </label> 
    <label for="html">
        HTML
        <input type="radio" id="html" name="language" /> 
    </label> 
    <label for="javascript">
        JavaScript
        <input type="radio" id="javascript" name="language" /> 
    </label> 
</div>

<!-- 复选框 -->
<div class="form__control">
    <label for="css">
        CSS
        <input type="checkbox" id="css" name="language" /> 
     </label> 
     <label for="html">
        HTML
        <input type="checkbox" id="html" name="language" /> 
     </label>
     <label for="javascript">
        JavaScript
        <input type="checkbox" id="javascript" name="language" /> 
     </label>
</div>
```

  


还有一点，需要知道的是选择控件的状态管理。

  


复选框和单选按钮可以被选中或未选中，切换开关可以处于关闭或开启状态。所有这些控件都有启用、禁用、悬停、聚焦和按下的状态。我知道看起来状态很多，但创建这些状态有助于确保交互的可靠性。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5db8f1a56fc84c8c91ac59e86f5762e3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1578&s=618734&e=jpg&b=f6f6f6)

  


除此之外，别忘了复选框还有另一种状态，既不是选中，也不是未选中。因为状态不完全确定，所以被称为“不确定状态”。这个状态常用于处理这样的一种情况：当一个父级复选框有多个子级复选框时，其中一些子选项被选中了，而另一个则没有被选中：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9411b47b2e14439fa723d2030cde9519~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=872&s=244422&e=jpg&b=f8f8f8)

  


注意，切换按钮通常使用的是 `<input type="checkebox" />` ，因此它具有不确定状态。

  


虽然它们有很多种不同的状态，但浏览器会自动管理它们。例如，我们可以给 `input` 元素设置 `checked` 、`disabled` 和 `indeterminate` 来管理它们的状态：

  


```HTML
<div class="form__control">
    <label for="css">
        CSS
        <input type="checkbox" id="css" name="language" /> <!-- 未选中状态 -->
     </label> 
     <label for="html">
        HTML
        <input type="checkbox" id="html" name="language" checked /><!-- 选中状态 --> 
     </label>
     <label for="javascript">
        JavaScript
        <input type="checkbox" id="javascript" name="language" disabled /><!-- 禁用状态 --> 
     </label>
     <label for="php">
        PHP
        <input type="checkbox" id="php" name="language" indeterminate /><!-- 不确定状态 --> 
     </label>
</div>
```

  


浏览器默认会为这些状态设置相应的样式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/88d69b779c8a40a598e6c3dd4f95023d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=476&s=225010&e=jpg&b=fbfbfb)

  


在 CSS 中，我们可以通过相关的伪类选择器 `:checked` （已选中）、`:disabled` （禁用）、`:indeterminate`（不确定状态）、 `:hover`（悬停）、`:active` （激活）和 `:focus` （或 `:focus-visible` 聚焦）为选择控件的不同状态设置样式。换句话说，如果不处理其可能处于的各种状态，那就不算完整。

  


不知道你是否已经注意到了，虽然切换按钮使用的也是复选框，但它多了一个 `switch` 属性。这是 Safari 在 TP 185 中推出的一个叫做“新开关控件”。也就是说，到目前为止仅 Safar TP 185 以上的浏览器支持 `switch` 属性。

  


换句话说，要使用新的切换开关，你需要定义一个 `type` 为 `checkbox` 的 `<input>` 元素，并为其添加 `switch` 属性。Safari 还提供了两个 CSS 伪元素来定制开关的样式：`::track` 和 `::thumb` 。这两个元素名称对经验丰富的 CSS 开发者来说可能听起来很熟悉，因为 `type` 为 `range` 的 `<input>` 元素或 `<progress>` 元素也有类似的伪元素。例如，`::-webkit-slider-thumb` 和 `::-moz-range-thumb` 表示用户可以在 `<input type="range">` 的“槽”中移动的“滑块”。还有 `::-webkit-scrollbar-thumb`，表示浏览器滚动条中可拖动的部分。

  


在这里我们可以看到两种变体。首先是类型为 `checkbox` 的 `<input>`，然后是带有 `switch` 属性的新的 `<input>`。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/735e621a40954b679fa1048af1fcd8ed~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1436&s=363893&e=jpg&b=3a3a3a)

  


它还有动画效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/075cfb927b33495d91a08796690ea196~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1078&h=518&s=375341&e=gif&f=65&b=343434)

  


任何遇到新开关属性但不支持它的浏览器，很可能会忽略它并简单地显示一个复选框。考虑到渐进增强，开发者可以开始使用它，支持的浏览器会显示它，而对旧版浏览器没有任何风险。

  


你可能还有另一个疑惑，切换开关为什么使用 `type` 为 `checkbox` 的 `<input>` 元素，它可以使用 `type` 为 `radio` 的 `<input>` 的元素吗？我的回答是，不建议这样使用，因为我们的切换按钮通常是独立的，而不是以组的方式存在。而单选按钮仅有一个的时候，你一旦选中它，就无法取消它。

## 自定义单选按钮和复选框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a316e486008d4dac9692be10d0c452be~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1051&s=397765&e=jpg&b=f8f8fa)

  


我们的目标是使用纯 CSS 创建自定义的、可访问的、跨浏览器的、可主题化的和可缩放的单选按钮和复选框：

  


-   根据提供给 `label` 的 `font-size` 进行缩放
-   使用单选按钮和复选框获得与 `label` 相同的颜色，以便于主题化
-   实现一致的跨浏览器设计风格，包括它们的各种状态
-   保持键盘和颜色对比的可访问性

  


### 定义变量

  


首先，我们定义一些 CSS 自定义属性，便于我们更好的控制单选按钮和复选框：

  


```CSS
.form__control {
    --s: round(1.15em, 1px);  /* 控制大小 */
    --b: round(.15em, 1px); /* 控制边框厚度 */
    
    /* 控制颜色 */
    --primary: hsl(80 0% 80%);
    --color-inactive: var(--primary);
    --color-active: hsl(from var(--primary) h 60% 45%);
    --color-disabled: hsl(from var(--primary) h 0% 70%);
    
    --fs: clamp(1rem, 0.6522rem + 1.7391vw, 2rem); /* 控制字号 */
}
```

  


`.form__control` 包含了整个组件子元素中使用的自定义属性，并为集中式定制提供了入口。在这里，[我们使用了 CSS 的字体相对单位 `em`](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-3) ，使单选按钮和复选框可以根据 `label` 的 `font-size` 进行放。同时，使用 [CSS 的相对颜色特性](https://juejin.cn/book/7223230325122400288/section/7231515598088306720#heading-14)，来控制单选按钮和复选框的不同状态（未选中状态、选中状态和禁用状态）的主题色。

  


另外，在这里还使用了 [CSS 的比较函数 `clamp()`](https://juejin.cn/book/7223230325122400288/section/7241401565653762108) 定义了 `--fs` （即 `font-size`）的大小，使其值限制在 `1rem ~ 2rem` 之间，并且会根据视窗宽度自动调整 `--fs` 的值。

  


### 布局

  


定义完所需的变量之后，我们可以使用[现代 Web 布局](https://s.juejin.cn/ds/ih76nyL6/)技术中的 [Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 或 [Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648) 对该组件进行布局。通常情况下，给组件布局都会首先 [CSS Flexbox 布局技术](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)，我也不例外。我选择将 `.form__control` （`label` 标签元素）设置为一个 Flexbox 容器，因为它包含了子元素用来计算布局的私有和公共的自定义属性：

  


```CSS
.form__control {
    display: flex;
    flex-wrap: wrap;
    align-items: center;
    gap: 1ch;
}
```

  


上面的代码，对于任何一位熟悉 [Flexbox 布局技术](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)的开发者来说不会任何问题。在这里，我需要特别提出的是，我在这里显式设置了 [`flex-wrap` 属性的值为 `wrap`](https://juejin.cn/book/7199571709102391328/section/7199571708838150155)，其主要功能是，当 Flexbox 容器没有足够空间的情况下，它会自动换行，避免打破整体的 Web 布局效果。[这是一种“防御式”的编程思想](https://s.juejin.cn/ds/ih76tmVx/)，作为一名优秀的 Web 开发者，我们在进行编码的时候，应该具备这种理念，尽可能的考虑一些极限边缘，以防万一。现代 CSS 中，有很多类似这样的小技巧，如果你感兴趣的话，可以移步阅读《[防御式 CSS 精讲](https://s.juejin.cn/ds/ih76tmVx/)》。

  


另外，我们还可以在 `input` 元素显式设置 `order` 属性的值来调整它的顺序。例如，要将文本内容放在 `input` 元素之前，可以像下面这样设置：

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    order: 1;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/abe0fec5c2954d0599b786737898538c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1058&h=418&s=149621&e=gif&f=87&b=040404)

  


使用 Flexbox 布局非常的灵活。例如，要将标签放在单选按钮或复选框上方或下方，只需将 `flex-direction` 属性的值设置为 `column` 即可。

  


### 标签样式

  


我们的标签（`<label>`）使用了 `.form__control` 类。我们在这里包含的基本样式是字体样式和鼠标指针样式：

  


```CSS
.form__control {
    /* 省略其他 CSS */
    
    font-family: system-ui, sans-serif;
    font-size: var(--fs);
    font-weight: bold;
    line-height: 1.1;
    cursor: pointer;
    
    &:has(input:disabled) {
        cursor: not-allowed;
    }
}
```

  


这里值得提出的是，通过 CSS 的 `:has()` 选择器来做条件判断，当 `.form__control` （即 `<label>` 元素）包含了状态为 `disabled` 的 `input` 时，它的 `cursor` 值更改变 `not-allowed` ，它会显式禁止操作的一个手型（指针类型）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c919afe8020c4a3fa257670543742c37~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1290&h=516&s=387165&e=gif&f=149)

  


在移动设备上，浏览器会为 `<label>` 和 `<input>` 添加点击高亮和文本选择功能。这些效果会对单选按钮和复选框所需的样式和视觉交互反馈产生负面影响。通过几行 CSS 代码，我们可以移除这些效果：

  


```CSS
.form__control {
    /* 省略其他 CSS */
    user-select: none;   
    -webkit-tap-highlight-color: transparent;
}
```

  


并不总是建议移除这些样式，因为它们可以提供有价值的视觉交互反馈。如果你移除了这些效果，确保提供自定义的替代方案。

  


到目前为止，设置 `label` 的全部 CSS 如下所示：

  


```CSS
.form__control {
    --s: round(1.15em, 1px); /* 控制大小 */
    --b: round(0.15em, 1px); /* 控制边框厚度 */

    --color-active: hsl(from var(--primary) h 60% 45%);
```

### 设置输入框的样式

  


接下来，进入自定义单选按钮和复选框的关键阶段。我们将按以下几个步骤来美化它们的样式。

  


#### 步骤1：隐藏原生输入框

  


在这里，我为了避免使额外的 HTML 元素，我选择了利用 `<input>` 元素的伪元素 `::before` 和（或）`::after` 来设置单选按钮和复选框的视觉指示器。因此，不能直接使用类似之前提到的 `.sr-only` 对应的样式来隐藏 `<input>` 的元素。

  


除此之外，为了保持其在技术上的可访问性，以便启用正确的键盘交互，同时保持对聚集状态（例如 `:focus`）的访问。我选择了 `appearance: none` 在视觉上隐藏 `<input>` ：

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    appearance: none;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d6bbdf29bc294b9eaf0c46700b8d65ef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1051&s=179847&e=jpg&b=060606)

  


正如你所看到的，它会移除几乎所有继承的浏览器样式，并允许我们样式 `input` 的伪元素 `::before` 或 `::after` 。请注意，我们还需要几个额外的属性来完成重置：

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    appearance: none;
    background: transparent;
    margin: 0;
    cursor: pointer;
    
    &:disabled {
        cursor: not-allowed;
    }
}
```

  


需要知道的是，并不是所有类型的 `<input>` 元素都支持伪元素 `::before` 和 `::after` 。

  


#### 步骤2：自定义未选中状态样式

  


对于我们的自定义单先按钮和复选框，我们将更新 `input` 的基础样式。这包括继承字体样式，以确保使用 `em` 能产生所需要的大小效果，并使用 `currentColor` 来继承 `label` 标签颜色的任何更新。

  


我们使用 `em` 来设置 `input` 的宽度、高度和边框厚度，以保持相对外观。

  


```CSS
.form__control {
    /* 省略其他的 CSS */
    color: var(--color-inactive);
}

input[type="radio"],
input[type="checkbox"] {
    appearance: none;
    font: inherit;
    width: var(--s);
    aspect-ratio: 1;
    color: currentColor;
    border: var(--b) solid currentColor; 
}
```

  


上面代码中的 `currentColor` 将会继承 `color` 属性的值，而且 `color` 是一个可继承属性，因此 `input` 的颜色和边框颜色会与 `label` 的 `color` 保持一致。刚才也提到过了，`currentColor` 会继承 `label` 的 `color` 的任何更新，例如选中状态（`:checked`）、悬浮状态（`:hover` ）和禁用状态（`:disabled` ）状态：

  


```CSS
.form__control {
    /* 省略状态 */
    
    color: var(--color-inactive);
    
    &:hover {
        color: var(--color-active);
    }
    
    &:has(input:disabled) {
        cursor: not-allowed;
        color: var(--color-disabled);
    }
    
    &:has(input:checked) {
        color: var(--color-active);
        
        &:hover {
            color: color-mix(in srgb, var(--color-active), #000 30%);
        }
    }
}
```

  


上面代码中，[通过 `:has()` 选择器和 `input` 的状态伪类选择器](https://juejin.cn/book/7223230325122400288/section/7224404685799555111#heading-3)为不同状态设置 `color` 值，并且[使用 `color-mix()` 给选中状态的 `lable` 设置了新的颜色](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)，该颜色的值是 `--color-active` 与 `30%` 的 `#000` 混合色。这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1f1d572364f2494ebfa75a83602c6660~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1214&h=580&s=646225&e=gif&f=126&b=040404)

  


到目前为止，单选按钮和复选框的样式是一样的。我熟悉的单选按钮，它是一个圆形，而复选框是一个带有圆角效果的边框。为此，我们引入一个新的变量 `--r` 来控制它们的圆角：

  


```CSS
.form__control {
    --r: round(0.25em, 1px); /* 控制圆角半径 */
    
    /* 省略其他 CSS */
}

input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    border-radius: var(--r);
}

input[type="radius"] {
    --r: 50%;
}
```

  


我们为单选按钮重置了 `--r` 属性的值为 `50%` ，单选按钮变成了一个圆形。因为，在这个示例中，单选按钮和复选框的宽和高是相等的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6c8caed67cd49ac9ca61f347066844d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1196&h=540&s=538834&e=gif&f=131)

  


#### 步骤3：自定义选中状态样式

  


到目前为止，单选按钮和复选框的选中状态（`:checked`）仅有颜色上的差异，并没有相应的视觉指示器。通常情况之下，单选按钮中间会有一个实心圆，而复选框中间会有一个“打勾”的形状。

  


我将使用 `input` 的伪元素 `::before` 来样式化 `:checked` 状态下的指示器：

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    
    &::before {
        content: "";
        display: block;
        width: calc(var(--s) * .6125);
        aspect-ratio: 1;
        border-radius: inherit;
        box-shadow: inset 1em 1em currentColor;
    }
}
```

  


选中状态指示器的大小为 `--s` 的 `61.25%` ，这个数值是根据我自己的喜好来设置的，你也可以根据你自己的喜好进行调整，关键是，它不能大于或等于 `input` 的大小，否则整个指示器会填满整个容器，除非你在设计上需要这样的一个 UI 效果。

  


将 `border-radius` 属性的值设置为 `inherit` ，是便于继承其父容器 `input` 的 `border-radius` 属性的值。这样一来，单选按钮的指示器也是一个圆，复选框的指示器则是一个带有圆角的正方形。你可能注意到了，在我们这个示例中，有多个地方使用到了 `inherit` 值。这是一个很有意思的属性值，它能强制元素继承其祖先元素相应属性的值。如果你对它感兴趣的话，[可以点击这里了解更多关于它的信息](https://juejin.cn/book/7223230325122400288/section/7232094160071688227#heading-6)。

  


需要知道的是，[我使用了 `box-shadow` 替代了 `background-color`](https://juejin.cn/book/7199571709102391328/section/7199844993455325216) ，这样做的好处是可以使单选按钮和复选框在打印时仍然可见。这算是 CSS 中的一个小技巧吧！

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/437c31016e44407995236bdd2e84e1e9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1206&h=542&s=535432&e=gif&f=165&b=030303)

  


很明显，你现在看到的单选按钮和复选框有明显的缺陷，其一，所有 `input` 都有选中状态的指示器，其二指示器的位置并没有在容器中垂直居中。

  


首先来解决第二个问题，[我将在 `input` 元素上使用 CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161624078397210638#heading-0)，通过两行简单的代码，将选中状态的指示器位于容器正中间。

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    display: grid;
    place-content: center;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b072f0fddb134010b61f96bea0d4fa44~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1051&s=216001&e=jpg&b=060606)

  


接着，[我们使用 CSS 变换中的 `scale` 来解决第一个问题](https://juejin.cn/book/7223230325122400288/section/7259668493158023205#heading-4)。默认状态（未选中状态）下，将 `scale` 的值设置为 `0` ，单选按钮和复选框在选中状态（`:checked`）时，将 `scale` 设置为 `1` ：

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    
    &::before {
        scale: 0;
    }
    
    &:checked::before {
        scale: 1;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a75bb2c86559456c96e7b7bc8e4ca83b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1208&h=540&s=1037029&e=gif&f=223&b=040404)

  


正如你所看到的，离我们所期望的效果越来越近了。可以说，单选按钮几乎是我们所需要的效果了。对于复选框，我们希望的是一个“打勾”的形状，所以我们需要单独再对它做一些额外的处理。

  


我们有多个选择，[比如使用 SVG 作为背景图像](https://s.juejin.cn/ds/ih7b4tVw/)。然而，这样的解决方案意味着我们将失去对 CSS 自定义属性的访问，而我们依赖这些属性来对 `input` 进行“主题化”。

  


相反，[我们将使用 `clip-path` 属性重新定义默认的盒子形状](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-1)。这个属性的 `polygon()` 函数或 `path()` 函数允许我们将伪元素的盒子裁剪出所需要的形状：

  


```CSS
input[type="checkbox"]::before {
    clip-path: polygon(14% 44%, 0 65%, 50% 100%, 100% 16%, 80% 0%, 43% 62%);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34f33a1dc4ee4e5cadb511586ccaa3a5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=504&s=1520566&e=gif&f=353&b=040404)

  


完美吧！但我们还可以继承使它们变得更完美一些，而且不需要太多的代码。[我们可以在伪元素 `::before` 上应用一个 `transition` ，并给 `scale` 设置一个过渡动画](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)，整个效果就会生动起来：

  


```CSS
.form__control {
    --ease: linear(0 0%, 0 2.27%, 0.02 4.53%, 0.04 6.8%, 0.06 9.07%, 0.1 11.33%, 0.14 13.6%, 0.25 18.15%, 0.39 22.7%, 0.56 27.25%, 0.77 31.8%, 1 36.35%, 0.89 40.9%, 0.85 43.18%, 0.81 45.45%, 0.79 47.72%, 0.77 50%, 0.75 52.27%, 0.75 54.55%, 0.75 56.82%, 0.77 59.1%, 0.79 61.38%, 0.81 63.65%, 0.85 65.93%, 0.89 68.2%, 1 72.7%, 0.97 74.98%, 0.95 77.25%, 0.94 79.53%, 0.94 81.8%, 0.94 84.08%, 0.95 86.35%, 0.97 88.63%, 1 90.9%, 0.99 93.18%, 0.98 95.45%, 0.99 97.73%, 1 100%);
}

input[type="radio"],
input[type="checkbox"] {
    &::before {
        transition: scale .28s var(--ease);
    }
}

input[type="checkbox"]::before {
    transform-origin: bottom left;
}
```

  


作为个人偏好，我还复选框的伪元素 `::before` 的`transform-origin` 更改为使用 `bottom left` 值，而不是默认的 `center`，以模拟一种“勾选”动画。而且，[我还使用了 `linear()` 函数，模拟了一个类似弹簧的过渡动效](https://juejin.cn/book/7288940354408022074/section/7301665456824254515)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8e8e93edf7d14a30a7965c6af81f6df2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1300&h=556&s=662611&e=gif&f=253&b=030303)

  


#### 步骤4：自定义禁用状态样式

  


接下来，我们来继续完善禁用状态（`:disabled`）的样式。禁用状态可能会存在两种情况：禁用未选中状态和禁用选中状态：

  


```HTML
<!-- 禁用未选中状态 -->
<label class="form__control" for="css">
    <input type="radio" id="css" name="language" disabled />
</label>

<!-- 禁用已选中状态 -->
<label class="form__control" for="css">
    <input type="radio" id="css" name="language" disabled checked />
</label>
```

  


前面我们使用 `:has(input:disabled)` 给禁用状态设置了不同的颜色样式。我们需要进一步调整一下 CSS 的样式设置：

  


```CSS
.form__label {
    /* 省略其他 CSS */
    
    /* 默认状态，未选中状态 */
    color: var(--color-inactive);

    /* 悬浮状态 */
    &:hover {
        color: var(--color-active);
    }

    /* 选中状态但未禁用状态 */
    &:has(input:checked:not(:disabled)) {
        color: var(--color-active);
      
        &:hover {
            color: color-mix(in srgb, var(--color-active), #000 30%);
        }
    }
    
    /* 禁用状态 */
    &:has(input:disabled) {
        cursor: not-allowed;
        color: var(--color-disabled);
    }
}
```

  


请注意它们在源码中出现的先后顺序，否则可能会影响不同状态的颜色渲染。这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/36d2c835f43d44f89a600a5b9b0a22ac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1278&h=514&s=758066&e=gif&f=196&b=040404)

  


我们还可以考虑给禁用未选中状态设置额外的样式：

  


```CSS
.form__control {
    /* 省略其他 */
    
    color: var(--color-inactive);

    &:hover {
        color: var(--color-active);
    }

    &:has(input:checked:not(:disabled)) {
```

#### 步骤5：自定义焦点环样式

  


根据你的浏览器，你可能已经看到了一种焦点环样式，通常表现为轮廓（`outline`）。我们将进行一点小的自定义，使其匹配我们的输入框颜色，并通过使用 `outline-offset` 提供一些与输入框的空间。在这里分别使用 `:focus` 和 `:focus-visible` 为两种行为触发的焦点环设置样式。

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    
    /* 非键盘聚焦时焦点环样式 */
    &:focus:not(:focus-visible) {
        outline: 2px solid var(--color-focus);
        outline-offset: 2px;
        box-shadow: 0px 1px 1px var(--color-focus);
    }
    
    /* 键盘聚焦时焦点环样式 */
    &:focus-visible {
        outline: 6px solid var(--color-focus);
        outline-offset: 0;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/48f75dc3c04f4d8eb10512900091cad9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1356&h=662&s=1220499&e=gif&f=331)

  


注意代码中选择器的使用，`:focus:not(:focus-visible)` 选中的是非 `:focus-visible` 状态，`:focus-visible` 通常是通过键盘获得焦点的状态，比如，当你按下键盘的 `Tab` 键，就会触发 `:focus-visible` 。有关于这方面更详细的介绍，请移步阅读《[CSS 焦点样式：`:focus-visible` 和 `:focus-within`](https://juejin.cn/book/7223230325122400288/section/7226969870444232742)》。

  


还需要知道的是，在我们这个示例中，多处使用了 `:has()` 和 `:not()` 选择器，而且很多时候都是同时结合使用 `:has()` 和 `:not()` 选择器，帮助我们选择目标元素。[`:has()` 和 `:not()` 选择器](https://juejin.cn/book/7223230325122400288/section/7226251495276609569)是现代 CSS 中提供的两个非常强大的选择器，它很多时候替代了以往需要依赖 JavaScript 才能做的事情。

#### 步骤6：自定义“不确定状态”样式

  


对于复选框而言，我们还有一个“不确定状态”的样式需要自定义。通常情况，它是一条在容器中水平垂直居中的直线。这个样式很简单，我们只需要在 `:indeterminate` 选择器中将复选框的“打勾”形状换成一条直线即可：

  


```CSS
.form__control {

    &:has(input:indeterminate) {
        color: color-mix(in srgb, var(--color-active), #fff 20%);
    }
}

input[type="checkbox"] {
    /* 省略其他 CSS */
    &:indeterminate::before {
        height: round(0.185em, 1px);
        clip-path: unset;
        scale: 1;
    }
}
```

  


你可能会感到疑惑，为什么只给复选框自定义“不确定状态”的样式呢？因为 `:indeterminate` 选择器针对的元素是：

  


-   `<input type="checkbox">` 元素，其 `indeterminate` 属性被 JavaScript 设置为 `true` 时
-   `<input type="radio">` 元素，当表单中具有相同 `name` 值的所有单选按钮均未被选中时
-   处于不确定状态的 `<progress>` 元素

  


另外，仅在 HTML 的 `<input type="checkbox" />` 元素中显式设置 `indeterminate` 属性，往往是看不到不确定状态的。我们需要额外使用 JavaScript 将其设置为 `true` ：

  


```JavaScript
const input = document.querySelector("input[indeterminate]");
input.indeterminate = true;
```

#### 步骤7：高对比度主题和强制颜色

  


通常情况之下，Web 开发者使用 CSS 自定义单选按钮和复选框时，到上一步就截止了。其实，我们还需要确保我们的单选按钮和复选框按钮能够响应一种称为“[Windows 高对比度模式](https://blogs.windows.com/msedgedev/2020/09/17/styling-for-windows-high-contrast-with-new-standards-for-forced-colors/)”（WHCM）的状态。在这种模式下，用户的操作系统会用减少的调色板替换掉颜色相关的属性，[这也是 CSS 规范中的一个新部分，称为“强制颜色”（`forced-colors`）](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-19)。

  


在这种模式下，我们的 `box-shadow` 会被完全移除，使得这些用户无法看到选中状态的指示器。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5505e6564bc44a23afbaf0be01263371~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1654&h=618&s=1564481&e=gif&f=154&b=040404)

  


幸运的是，解决这个问题只需要添加一个额外的属性。我们将提供一个 `background-color`，在强制颜色模式中通常会被移除，但如果使用了定义的强制颜色，则会保留。在这种情况下，我们选择 `CanvasText`，它会与常规的正文文本颜色匹配。

  


由于样式的堆叠顺序，我们为常规模式主题化的 `box-shadow` 实际上会视觉上覆盖在 `background-color` 上，这意味着我们可以在不做进一步修改的情况下同时使用两者。

  


```CSS
input[type="radio"],
input[type="checkbox"] {
    /* 省略其他 CSS */
    
    &::before {
        /* 省略其他 CSS */
        background: CanvasText;
        box-shadow: inset 1em 1em currentColor;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d9e54d817e345c0b1ac43462cb28167~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1564&h=628&s=2461494&e=gif&f=249&b=030303)

  


你也可以通过 [CSS 的媒体查询的 `forced-colors: active`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935) 来调整 `.form__control` 中有关于颜色相关的设置。但上面示例中所展示的方法是最简单的一种方式。

## 自定义切换按钮

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/61e3ed8ce76a412084c262d67f3ed859~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=938&s=295240&e=jpg&b=a085bc)

  


使用 CSS 自定义切换按钮的步骤与自定义单选按钮和复选框是相似的，只是在 UI 的呈现上有所不同。切换按钮更像是之前所介绍的[范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)和[进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)组件，因为切换按钮也有轨道和滑块的概念存在：

也就是说，自定义切换按钮的 UI 外观，主要工作重点是在轨道和滑块上。


## 未来创建切换按钮的姿势

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/942a3d3850b4400980454dd3c7a50f02~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1040&h=896&s=25298&e=webp&b=302840)

  


早在 2022 年 4 月底，[@Tab Atkins Jr.](https://twitter.com/tabatkins) 和 [@Miriam E. Suzanne](https://twitter.com/terriblemia) 提出了一个关于 [CSS Toggles 方面的草案](https://tabatkins.github.io/css-toggle/)，并且 [Chromium 将开始设计 CSS Toggles 的原型](https://groups.google.com/a/chromium.org/g/blink-dev/c/06dTetmFjEU/m/NdkJtoVYHAAJ)。

  


> CSS Toggles 是一种将可切换状态与 DOM 元素关联的机制。这个状态可能通过 CSS 属性进行定义并激活事件连接，还可以在 CSS 选择器中查询。这为描述行为提供了一种声明式机制，类似于现有的 HTML 复选框或单选按钮的行为。

  


简单的说，这个功能允许我们通过 CSS 根据 `ON` 或 `OFF` 值来修改样式。这也意味着，我们可以利用这个功能来制作切换按钮。

  


**特别声明，CSS Toggles 目前仅作为非官方草案存在的提案。因此，在本文中出现的语法规则很有可能会因时间推移会有所变化。而且，接下来的演示是使用了** **[JS-polyfill](https://github.com/oddbird/css-toggles)** **来实现的（仅在 Chrome 浏览器中可运行），请注意**。

  


[CSS Toggles 提案](https://tabatkins.github.io/css-toggle/)中提供了一个功能性伪类 `:toggle()` ，它允许我们可以根据切换状态选择元素。在 `:toggle()` 之前，我们只能利用内置的浏览器状态来为元素添加样式或交互。例如，复选框或单选按钮的 `:checked` ，你可以使用它在复选框或单选按钮选中时改变其外观。我们之前演示的切换按钮就是采用这种机制完成的。

  


有了 `:toggle()` ，你将可以在任何元素上创建自定义状态，并为不同状态定义样式变化。除此之外，CSS Toggles 规范还引入了一些属性，比如 `toggle-visibility` 和 `toggle-group`，这些属性在构建如标签页（Tab ）和手风琴（Accordion）以及我们要制作的切换按钮等控件时尤其有用。

  


接下来，我们来看看如何使用 `:toggle()` 来创建一个切换按钮。首先，我们从 HTML 结构开始：

  


```HTML
<div class="switch">
    <label for="switch" class="switch__label">Dark Mode</label>
    <button class="switch__button" id="switch"></button>
</div>
```

  


与之前一样，使用 CSS 渐变和遮罩 `mask` 创建切换按钮的 UI：

  


```CSS
.switch {
    --bg: #fff;
    --color: #333;
    --fs: 1.125rem;
  
    display: flex;
    align-items: center;
    justify-content: space-between;
    gap: 2ch;
    padding: 2rem;
    border-radius: 0.5em;
  
    background: var(--bg);
    color: var(--color);
    min-width: 320px;
  
    font-size: var(--fs);
    line-height: 1.1;
    cursor: pointer;
    user-select: none;
    -webkit-tap-highlight-color: transparent;
}

.switch__button {
    --s: round(4em, 1px);
    --color-active: #8fbe00;
    --color-inactive: #cc333f;
    --color-disabled: #948c75;

    height: var(--s);
    aspect-ratio: 2;
    border-radius: var(--s);
    border: calc(var(--s) / 10) solid #0000;
    padding: calc(var(--s) / 10);
    background: conic-gradient(var(--color-active) 50%, var(--color-inactive) 0) border-box var(--p, left) / 200%;
    mask: 
        radial-gradient(#000 70%, #0000 72%) no-repeat var(--p, left) / calc(3 * var(--s) / 5) content-box,
        conic-gradient(#000 0 0) padding-box exclude, conic-gradient(#000 0 0);
```

现在看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e29f486714bf4476ac83f9614f4f590b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1160&h=528&s=192584&e=gif&f=128&b=030303)

  


正如你所看到的，不管怎么点击，它都不具备切换效果。这也是我们接下来要使用 CSS Toggles 完成的功能。

  


为了使用 CSS 的 `:toggle()` ，我们需要声明一个值，声明的位置应该是在我们希望使用 `:toggle()` 的父元素或兄弟元素上。在这个例子中，由于我们希望设置 `.switch__button` 的 `mask-position` ，我们需要在 `.switch` 元素中进行声明。

  


你可以在 `.switch` 元素中使用 `toggle-root` 属性进行声明，它的值可以是任意你喜欢的字符串，例如：

  


```CSS
.switch {
    toggle-root: switch;
}
```

  


然后，你可以在 `.switch__button` 元素通过 `toggle-trigger` 属性描述该状态如何根据用户交互进行更改：

  


```CSS
.switch__button {
    toggle-trigger: switch;
}
```

  


这样你就可以使用 `:toggle()` 伪类让元素响应状态变化：

  


```CSS
.switch__button:toggle(switch) {
    --p: right;
}
```

  


仅仅这几行 CSS 代码，你就可以让切换按钮动起来。它会在选中和未选中两个状之间来回切换：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c1968bc415aa47ad99d7a4d32cf1c47d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1096&h=520&s=277826&e=gif&f=132)

  


更有意思的是，`:toggle()` 除了可以更换切换按钮`.switch__button` 的样式之外，还可以调整调团的样式，比如我们调整其父容器 `.switch` 的颜色，以呈现切换到暗黑模式下的效果：

  


```CSS
.switch:toggle(switch) {
    background-color: #1e2121;
    color: #fff;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f46b6474b6de4f6283b37ade163f67bd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1204&h=540&s=470832&e=gif&f=218&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/poXGWEZ

  


就是这么神奇，仅仅几行 CSS 代码，我们除了可以实现切换按钮的样式和交互效果之外，还可以快速切换主题颜色。

  


上面示例的完整代码如下所示：

  


```CSS
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

.switch {
    toggle-root: switch;
}

.switch__button {
    toggle-trigger: switch;
}

.switch__button:toggle(switch) {
    --p: right;
}

.switch:toggle(switch) {
    background-color: #1e2121;
    color: #fff;
}

.switch {
    --bg: #fff;
    --color: #333;
    --fs: 1.125rem;
    display: flex;
    align-items: center;
    justify-content: space-between;
    gap: 2ch;
    padding: 2rem;
    border-radius: 0.5em;
    background: var(--bg);
    color: var(--color);
    min-width: 320px;
    font-size: var(--fs);
    line-height: 1.1;
    cursor: pointer;
    user-select: none;
    -webkit-tap-highlight-color: transparent;
}

.switch__button {
    --s: round(4em, 1px);
    --color-active: #8fbe00;
    --color-inactive: #cc333f;
    --color-disabled: #948c75;

    height: var(--s);
    aspect-ratio: 2;
    border-radius: var(--s);
    border: calc(var(--s) / 10) solid #0000;
    padding: calc(var(--s) / 10);
    background: conic-gradient(var(--color-active) 50%, var(--color-inactive) 0) border-box var(--p, left) / 200%;
    mask: 
        radial-gradient(#000 70%, #0000 72%) no-repeat var(--p, left) / calc(3 * var(--s) / 5) content-box,
        conic-gradient(#000 0 0) padding-box exclude, conic-gradient(#000 0 0);
    transition: 0.5s;
    cursor: pointer;

    border-color: #0009;
    outline: none;
}
```

  


这个示例只是向你展示了 CSS Toggles 的最基本功能。除了上述提到的 `toggle-root` （切换根元素）、`toggle-trigger` （切换触发器）和 `:toggle()` 功能性伪类之外，它还引入了 `toggle-visibility` 属性，你可以通过该属性创建各种类型的可折叠组件，比如选项卡（Tab）、手风琴（Accordion）和模态框（Modal）等，而且不需要编写任何 JavaScript 代码。这个属性允许一个元素自动将其显示状态与特定的切换状态绑定。

  


如果你有多个切换开头，还可以使用 `toggle-group` 属性将它们进行分组。请记住，同一组中一次只能有一个切换开关处于激活状态。这类似于 HTML 中的单选按钮行为，两个按钮不能同时被选中。

  


这也意味着，一旦在浏览器中全面实现，CSS Toggles 将成为 CSS 开发者的一个非常有用的工具。这是一个强大的功能，毫无疑问将大大简化 HTML 元素的交互性添加过程。掌握其全部语法可能需要时间，但这是值得投资的。不过，在这里就不展开阐述了，因为它超出了这节课的范畴！

  


最后，再次提醒大家，请大家记住，CSS Toggles 目前仍然是一个非正式草案。这些行为可能会随着时间的推移进行更新或更改。

### 初始化 `input`

```CSS
.switch {
```

`box-shadow` 应用了五个阴影来创建框架，并使用 `border-radius` 属性让阴影在角落处变圆。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9c9523c854641ad8cee2e743c6feb09~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1111&s=108748&e=jpg&b=fbfcff)

  


### 制作 3D 按钮形状

  


3D 按钮它至少都有三个面，为了尽可能的不使用额外的 HTML 标签元素，我选择在 `input` 元素和它的伪元素 `::before` 和 `::after` 使用 [CSS 3D 变换](https://juejin.cn/book/7288940354408022074/section/7295240572736897064)将这三个面彼此垂直，使其在视觉上看起来像一个 3D 按钮：

  


```CSS
.switch {
    --color: #ff1818;
    --rotation: 25deg;
    --pivot-distance: 20px;
    --width: 150px;
    
    perspective: 700px;
    
    input {
        display: block;
        width: 100%;
        height: 100%;
        position: relative;
        cursor: pointer;
      
        background: linear-gradient(
            color-mix(in srgb, var(--color), #000 25%) 0%,
            color-mix(in srgb, var(--color), #000 33%) 30%,
            color-mix(in srgb, var(--color), #000 33%) 70%,
            color-mix(in srgb, var(--color), #000 25%) 100%
        ) no-repeat, #9b0621;
        
        transform-origin: center center calc(-1 * var(--pivot-distance));
        transform: translateZ(var(--pivot-distance))  rotateX(calc(-1 * var(--rotation)));
        transform-style: preserve-3d;
      
        transition: all 0.3s cubic-bezier(1, 0, 1, 1);
        
        &::before,
        &::after {
            content: "";
            width: 100%;
            height: 50px;
            position: absolute;
        }
        
        &::before {
            top: 0;
            transform-origin: top;
            transform: rotateX(-90deg);
            background: linear-gradient(
                color-mix(in srgb, white, transparent 20%) 10%,
                color-mix(in srgb, white, transparent 70%) 30%,
                color-mix(in srgb, var(--color), #000 35%) 75%,
                color-mix(in srgb, var(--color), #000 45%)
            ) 50% 50% / 97% 97% no-repeat, color-mix(in srgb, var(--color), #000 20%);
        }
        
        &::after {
            bottom: 0;
            background-image: linear-gradient(
                color-mix(in srgb, var(--color), #000 35%),
                color-mix(in srgb, var(--color), #000 45%)
            );
    
            transform-origin: top;
            transform: translateY(50px) rotateX(-90deg);
    
            box-shadow: 0 50px 8px 0px black, 0 80px 20px 0px color-mix(in srgb, black, transparent 50%);
        }
        
        &:checked {
            transform: translateZ(var(--pivot-distance)) rotateX(var(--rotation));
            box-shadow: 0 -10px 20px var(--color);
        }
    }
}
```

  


上面代码中涉及了 [CSS 3D 变换相关的知识](https://juejin.cn/book/7288940354408022074/section/7295240572736897064)，你需要掌握[透视 `perspective`](https://juejin.cn/book/7288940354408022074/section/7295240572736897064#heading-8) ，变换原点 `transform-origin` 、`transform-style` （它告诉浏览器，`input` 元素是否位于一个 3D 空间）和 [3D 变换函数](https://juejin.cn/book/7288940354408022074/section/7295240572736897064#heading-3)（例如 `translateX()` 、`translateY()`、`rotateX()` 等）。在这里，我不会花篇幅来阐述这些基础特性的使用，如果你感兴趣的话，[请点击这里获取更多有关于这方面的信息](https://juejin.cn/book/7288940354408022074/section/7295240572736897064)。

  


这些变换将整个按钮沿 `X` 轴旋转 `--rotation` （即 `25deg`），并使用 `transform-origin` 设置变换原点，确保按钮看起来像是围绕按钮内部的某个点旋转，而不是围绕 `input` 旋转。并在按钮的选中状态（`:checked`）改变它的旋转角度

  


另一点需要的知道的是，[使用了 CSS 的 `color-mix()` 函数实现了 Sass 处理器内置的颜色函数](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)，比如 `rgba()` 、`lighten()` 和 `darken()` 等：

  


```
color-mix(in srgb, white, transparent 20%) => rgba(white, .8)
color-mix(in srgb, var(--color), #000 35%) => darken(var(--color), 35%)
color-mix(in srgb, var(--color), #fff 35%) => lighten(var(--color), 35%)
```

  


现在，浏览器将会给你呈现一个类似 3D 开头的按钮，你点击这个开头时，它们在开和关之间不断切换：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/18310b595d23460784de5cbfbc8fe57b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1114&h=518&s=676183&e=gif&f=123)

  


你可能已经发现了，开关在切换时有一个过渡动效。这是因为使用了 CSS 过渡（`transition`）给开关的来回旋转添加了一个过渡动画。为了使这个过渡动画看起来有一种从慢到快的真实感，[在这里使用了自定义的 `cubic-bezier()` 缓动函数](https://juejin.cn/book/7288940354408022074/section/7298995488580337714)：

  


```CSS
transition: all 0.3s cubic-bezier(1, 0, 1, 1);
```

  


这个曲线意味着过渡开始时较慢，结束时较快，就像一个真实的开关一样，慢慢转动直到“点击”结束。

  


### 添加 LED 渐变

  


为了使整个开头更真实一些，我将使用 CSS 渐变和动画来给按钮添加 LED 灯光效果。如下图所示，可以叠加两个渐变来模拟隐藏在半透明红色塑料下的 LED 灯泡，上面有小圆形突起：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/18962574d6c0472c9df7d2945e6e6388~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1111&s=277380&e=jpg&b=d7dbe6)

  


这两个渐变分别作为 `.switch` 元素的 `::before` 和 `::after` 伪元素的背景图，这是因为第一个径向渐变模拟了灯光“闪烁”的动效，因此才分开来放置。换句话说，要是没有这个动画效果，我们可以在一个伪元素上完成所有事情：

  


```CSS
.switch {
    /* CSS ... */
    position: relative;
    
    &::before,
    &::after {
        content: "";
        position: absolute;
        inset: 20px;
      
        transform-origin: center center calc(-1 * var(--pivot-distance));
        transform: translateZ(var(--pivot-distance)) rotateX(calc(-1 * var(--rotation)));
        transform-style: preserve-3d;
        transition: all 0.3s cubic-bezier(1, 0, 1, 1);
    }

    &::before {
        opacity: 0;
        animation: light-off 1s;
        background: radial-gradient(
            hsl(from color-mix(in srgb, var(--color), #fff 20%) 35 s l),
            var(--color) 40%,
            transparent 70%
        );
        z-index: 1;
    }

    &::after {
        background-image: radial-gradient(
        transparent 30%,
            color-mix(
                in srgb,
                color-mix(in srgb, var(--color), #000 35%),
                transparent 30%
              )
              70%
        );
        background-size: 10px 10px;
        z-index: 2;
    }

    &:has(input:checked) {
        &::before,
        &::after {
            transform: translateZ(var(--pivot-distance)) rotateX(var(--rotation));
        }

        &::before {
            animation: flicker 0.2s infinite 0.3s;
        }
    }
}

@keyframes flicker {
    0% {
        opacity: 1;
    }
    80% {
        opacity: 0.8;
    }
    100% {
        opacity: 1;
    }
}

@keyframes light-off {
    0% {
        opacity: 1;
    }
    80% {
        opacity: 0;
    }
}
```

  


你会发现，伪元素 `::before` 和 `::after` 应用了与 `input` 一样的变换函数，这样做是为了确保它们与开关面板大小相等。另外一点，我们使用了 [CSS 相对颜色特性](https://juejin.cn/book/7223230325122400288/section/7231515598088306720#heading-14)实现了与 Sass 处理器中颜色函数 `adjust-hue()` 功能。

  


注意，伪元素 `::before` 有一个闪烁的动画效果，它是通过 `@keyframes` 来调整 `opacity` 实现的：

  


```CSS
@keyframes flicker {
    0% {
        opacity: 1;
    }
    80% {
        opacity: 0.8;
    }
    100% {
        opacity: 1;
    }
}

@keyframes light-off {
    0% {
        opacity: 1;
    }
    80% {
        opacity: 0;
    }
}
```

  


并且通过 `:has()` 与 `input:checked` 相互结合，在开关的关闭（`:unchecked`）和打开（`:checked`）两个状态应用子 `light-off` 和 `flicker` 动画：

  


```CSS
.switch {
    /* CSS ...*/
    
    &::before {
        opacity: 0;
        animation: light-off 1s;
      
        /* CSS ... */
    }
    
    &:has(input:checked)::before {
        animation: flicker 0.2s infinite 0.3s;
    }
}
```

  


这个时候，你看到的效果如下：

### 给开关添加 I/O 字符

  


在 CSS 中，可以使用多种技巧来制作 “I/O”字符。我们可以使用真实的字母并为其应用样式，或使用特殊字体。但由于这些字符相对简单，我决定在 `.switch` 的 `::after` 上增加两个渐变来制作 `I/O` 字符，其中使用一个线性渐变来制作 `I` 字符，然后使用一个径向渐变来制作 `O` 字符。这样使我们能更好的控制它们的位置和大小。

```CSS
.switch {

                transparent 30%,

             ) 0 0 / 10px 10px repeat;
}
```
