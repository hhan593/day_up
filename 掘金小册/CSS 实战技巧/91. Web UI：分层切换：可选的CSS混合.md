## 我们从 Sass 的 `@mixin` 开始

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7b6c72481f414ca3a67d64e5163bbad4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=140965&e=png&b=fd8c36)

  


我们先快速回顾一下 Sass 的 `@mixin`。通常情况下，我们会先使用 `@mixin` 定义一段需要重复使用的 CSS 规则：

  


```SCSS
@mixin ellipsis($lines: 1, $substract: 0) {
    @if $lines == 1 {
        white-space: nowrap;
        text-overflow: ellipsis;
        width: 100% - $substract;
        overflow: hidden;
    } @else {
        overflow:hidden;
        display:-webkit-box;
        display:box;
        -webkit-line-clamp: $lines;
        line-clamp: $lines;
        -webkit-box-orient: vertical;
        box-orient: vertical;
    }
}
```

  


然后通过 `@include` 来引用已定义好的 `ellipsis` 混合宏：

  


```SCSS
.text-overflow {
    @include ellipsis;
}

.line-clamp {
    @include ellipsis(3);
}
```

  


编译出的 CSS 代码如下：

  


```CSS
.text-overflow {
    white-space: nowrap;
    text-overflow: ellipsis;
    width: 100%;
    overflow: hidden;
}

.line-clamp {
    overflow:hidden;
    display:-webkit-box;
    display:box;
    -webkit-line-clamp: 3;
    line-clamp: 3;
    -webkit-box-orient: vertical;
    box-orient: vertical;
}
```

  


这就是在 Sass 中使用 `@mixin` 的全部过程。需要注意的是，没有被 `@include` 引用的 `@mixin` 不会被编译。另外，要是你在不同的选择器中使用了十次 `@mixin ellipsis` ，那么这些样式块会在 CSS 中重复输出十次。虽然说文本压缩已不是什么太大的问题，但这肯定没有直接在 CSS 中引用单一样式块那么高效。

  


## CSS 分层切换

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4c996146b53c414fa2e2e8edba4e7c57~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=776219&e=png&b=0a1c2a)

  


我们这里所说的分层切换是一种 CSS 技巧——结合使用 [CSS 的层](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)（`@layer`）与[空间切换或循环切换](https://juejin.cn/book/7395119173656903715/section/7446278394624344074)。它本质上允许我们在任意元素上应用任意数量的 CSS Mixins！

  


例如下面这个示例：

  


```HTML
<button>Default</button>
<button class="can-be-hovered">Hover Me!</button>
<button class="pseudo-hovered">As if hovered</button>
```

  


```CSS
@layer default, mixins, demo;

@layer default {
    button {
        display: inline-flex;
        place-content: center;
        padding: 1em 1.5em;
        border-radius: 0.5em;
        background: var(--PINK);
        color: CanvasText;
        border: none 0;
        font: inherit;
        cursor: pointer;
        transition: all .2s;
    }
}

@layer mixins {
    * {
        --hover: var(--hover--off);
        --hover--off: var(--hover,);
        --hover--on: var(--hover,);
    
        background-color: 
            var(--hover--off, revert-layer)
            var(--hover--on, var(--GREEN));
    }
}

@layer demo {
    :root {
        --PINK: oklch(0.84 0.12 2.82);
        --GREEN: oklch(0.89 0.2 166.25);
    }
    
    .can-be-hovered:is(:hover, :focus-visible),
    .pseudo-hovered{
        --hover: var(--hover--on);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5d098fc967704af2adf2fc8272653db0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1298&h=508&s=359157&e=gif&f=91&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/QwWxKMb

  


我们来看看它背后的工作原理。

  


首先，这里使用了 [CSS 的层（`@layer`）](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)。这是该技术的核心要求：

  


```CSS
@layer default, mixins, demo;

@layer default {
    /* CSS ... */
}

@layer mixins {
    /* CSS ... */
}

@layer demo {
    /* CSS ... */
}
```

  


我们定义了 `default` 、`mixins` 和 `demo` 三个级联层，它们的权重是：`default < mixmins < demo` 。它们可以有效的帮助我们控制样式何时覆盖其他样式。

  


-   `default` 层包含默认样式
-   `mixins` 层存放可选的 Mixins（混合宏），当它启用时，它们可以覆盖默认样式
-   `demo` 层是应用层，可以启用已定义的 Mixins

  


在 `mixins` 层中，我们使用循环依赖的空间切换来定义样式，其中关键点是 `revert-layer` 作为默认值（循环切换的值之一）。

  


```CSS
@layer mixins {
    * {
        --hover: var(--hover--off);
        --hover--off: var(--hover,);
        --hover--on: var(--hover,);
    
        background-color: 
            var(--hover--off, revert-layer)
            var(--hover--on, var(--GREEN));
    }
}
```

  


由于 `revert-layer` 的机制，它允许我们“将控制权归还”给低层（`default`）的默认样式，除非某个条件满足（比如在 `demo` 层启用 Mixins）。

  


> 需要注意的是，诸如 `revert-layer` 或 `inherit` 等关键词，只能作为 CSS 变量的即时回退值来使用！

  


最终，我们的设置完成了，现在可以通过在 CSS 中随处切换循环变量的值来调用 Mixins。当某个 Mixins 被激活，比如通过选择器、媒体查询、行内样式等，它的样式就会覆盖默认样式。在我们这个示例中，在 `demo` 层中启用了 `mixins` 层中定义的 Mixins，此时 `mixins` 中的样式将会覆盖 `default` 层中定义的默认样式。

  


```CSS
@layer demo {
    :root {
        --PINK: oklch(0.84 0.12 2.82);
        --GREEN: oklch(0.89 0.2 166.25);
    }
    
    .can-be-hovered:is(:hover, :focus-visible),
    .pseudo-hovered{
        --hover: var(--hover--on);
    }
}
```

  


这也意味着，我们现在可以基于单个 CSS 变量，在任何元素上有条件地应用样式，同时不会破坏默认样式！但是，要是没有 CSS 分层这个特性，仅使用通配符 `*` 创建伪 Mixins 是会问题的：

  


-   如果 Mixins 的样式只定义在 `*` 选择器规则中，任何后续的规则都可能覆盖它，因为 `*` 选择器的权重为 `0`
-   如果 Mixins 过于强大，它可能会覆盖整个页面的样式，变得难以控制

  


庆幸的是，分层切换（CSS 的 `@layer` 规则结合循环依赖的空间切换）解决了这些问题，使 Mixins 既能按需应用，又不会影响整体层叠结构。

  


当然，这种分层切换技术也有一些限制，比如：

  


-   所有 Mixins 共享同一个作用域，这意味着如果多个 Mixins 需要切换相同的 CSS 属性，我们必须定义如何处理冲突
-   Mixins 只能影响自身，而不能影响子元素。由于我们通过通配符选择器 `*` 应用样式，Mixins 只能作用于元素本身，而不会影响其子元素。不过，我们可以利用样式查询解决这个问题。但在某些情况下，这种“继承禁用”反而是一个优点，因为我们可能并不希望 Mixins 影响嵌套元素
-   无法切换其他循环切换
-   依赖 CSS 层（`@layer`），这意味着这一技术要正常工作，就得浏览器支持 CSS `@layer` ，并且所有主要样式都得由 `@layer` 进行管理。因此，除非你能确保 Web 应用目标浏览器支持 `@layer` ，否则可能需要谨慎使用该技术

  


这些局限性，有些是可以找到相应的解决方案的。我们继续往下。

  


假设，我们想要使用两个 Mixins，并且它们都需要设置 `background` ，那么谁优先？我们又该如何定义规则呢？毫无疑问，我们可以通过 CSS 的 `@layer` 规则来解决这个问题。我们可以为 `mixins` 创建一个独立的子层，这样每个 Mixins 都能独立处理自己的属性，并且可以通过 `revert-layer` 将某些属性恢复到另一个 Mixins 之前定义的值，或直接恢复为默认值。

  


这种方式让我们可以更加灵活地管理多个 Mixins，而且不会相互冲突！我们使用这种方式来改造之前的示例：

  


```CSS
@layer default, mixins, demo;

@layer default {
    button {
        display: inline-flex;
        place-content: center;
        padding: 1em 1.5em;
        border-radius: 0.5em;
        background: var(--PINK);
        color: CanvasText;
        border: none 0;
        font: inherit;
        cursor: pointer;
        transition: all .2s;
    }
}

@layer mixins.alpha {
    * {
        --mix-alpha: var(--mix-alpha--off);
        --mix-alpha--off: var(--mix-alpha,);
        --mix-alpha--on: var(--mix-alpha,);
        --mix-alpha-color: currentColor;
        --mix-alpha-value: 75%;

        color:
            var(--mix-alpha--off, revert-layer)
            var(--mix-alpha--on,
                color-mix(
                    in srgb,
                    var(--mix-alpha-color)
                    var(--mix-alpha-value),
                    transparent
                )
            );
    }
}

@layer mixins.red {
    * {
        --mix-red: var(--mix-red--off);
        --mix-red--off: var(--mix-red,);
        --mix-red--on: var(--mix-red,);

        background:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on, var(--RED));
        color:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on, #FFF);
        text-shadow:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on, none);
    }
}

@layer demo {
    :root {
        --PINK: oklch(0.84 0.12 2.82);
        --GREEN: oklch(0.89 0.2 166.25);
        --RED: oklch(0.65 0.28 29.99);
    }
    
    .with-alpha {
        --mix-alpha: var(--mix-alpha--on);
    
        &:is(:hover, :focus-visible) {
            --mix-alpha-value: 50%;
        }
    }
    
    .is-red {
        --mix-red: var(--mix-red--on);
    }
}
```

  


我们在 `mixins` 层的基础上创建了两个子层：`mixins.alpha` 和 `mixins.red` 。由于 `mixins.red` 层位于 `mixins.alpha` 之后，它的权重要更大（它的属性将会“获胜”）。而且这些 Mixins 都是完全独立的，都可以覆盖 `color` ，但每个 Mixins 只能根据其自定义属性生效。

  


在这个示例中，第二个按钮文本带有一定的透明度（`color-mix() 改变了文本的颜色的透明通道的值`），第三个按钮中的 `color` 和 `background` 将会覆盖 `default` 和 `mixins.alpha` 层定义的 `color` 和 `background` 。你将看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/be13485ce5e3487e8cda931c53d62411~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1526&h=476&s=228165&e=gif&f=91&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ZYERaRz

  


正如你所看到的，我们可以通过不同的层叠层，创建多个 Mixins，并控制它们的优先级，使它们独立工作，同时允许它们动态应用不同的样式！

  


到目前为止，每个 Mixins 彼此独立，互不知晓，这也导致了 `mixins.red` 完全覆盖了 `mixins.alpha` 。不过，我们还是有很多方法可以让它们协同工作的，其中最优的方式是让后定义的 `mixins.red` 回退到前一个 `mixins.alpha` ，即如果它检测到前一个 `mixins.red` 已启用，就条件性地使用 `revert-layer` 来获取它的值。

  


修改后的 CSS 代码如下：

  


```CSS

    button {
        display: inline-flex;
        place-content: center;
        padding: 1em 1.5em;
        border-radius: 0.5em;

        color: CanvasText;
        border: none 0;
        font: inherit;
        cursor: pointer;
        transition: all .2s;
    }
}

@layer mixins.alpha {
    * {
        --mix-alpha: var(--mix-alpha--off);

        --mix-alpha-color: currentColor;
        --mix-alpha-value: 75%;

        color:
            var(--mix-alpha--off, revert-layer)
            var(--mix-alpha--on,
                color-mix(
                    in srgb,
                    var(--mix-alpha-color)
                    var(--mix-alpha-value),
                    transparent
                )
            );
    }
}

@layer mixins.red {
    * {
        --mix-red: var(--mix-red--off);
        --mix-red--off: var(--mix-red,);
        --mix-red--on: var(--mix-red,);
        --mix-alpha-color:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on, #FFF);
        color:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on,
                var(--mix-alpha--off, #FFF)
                var(--mix-alpha--on, revert-layer)
            );
        background:
            var(--mix-red--off, revert-layer)
            var(--mix-red--on, var(--RED));
        text-shadow:
            var(--mix-red--off, revert-layer)

        --mix-alpha: var(--mix-alpha--on);

        &:is(:hover, :focus-visible) {

        }
    }

        --mix-red: var(--mix-red--on);

当我们的 `mixins.red` 启用时，还会检查 `mixins.alpha` 的状态。如果 `mixins.alpha` 没有开始，那么 `color` 设置为 `#fff` ；如果 `mixins.alpha` 已开启，那么 `color` 继续使用 `revert-layer` ，将继承 `mixins.alpha` 的值。

  


那么，如何将第三个按钮的颜色改为白色呢？很简单，`mixins.alpha` 提供了 `--mix-alpha-color` 变量，`mixins.red` 直接使用它。

  


另外，这里的每个 Mixins 互相知晓彼此的状态，每个 Mixins 都知道另一个 Mixins 在做什么。比如，`mixins.alpha` 继承了 `mixins.red` 定义的 `--mix-alpha-color` ，而 `mixins.red` 也知道 `mixins.alpha` 已启用，并决定是否回退（`revert-layer`），因此会调整对颜色的处理方式！

}

@layer mixins.child-hover {
    * {
        --mix-child-hover: var(--mix-child-hover--off);
        --mix-child-hover--off: var(--mix-child-hover,);
        --mix-child-hover--on: var(--mix-child-hover,);

        @container style(--mix-child-hover--off: ) {
            background: var(--GREEN);
            color: CanvasText;
        }
    }
}

@layer demo {
    :root {
        --PINK: oklch(0.84 0.12 2.82);
        --GREEN: oklch(0.89 0.2 166.25);
        --RED: oklch(0.65 0.28 29.99);
    }

    .is-red {
        --mix-red: var(--mix-red--on);
    }
  
    .child-hover:is(:hover, :focus-visible):not(:has(.child-hover:is(:hover, :focus-visible))) {
        --mix-child-hover: var(--mix-child-hover--on);
    }
}
```

  


我们可以利用 `--mix-child-hover--on` 或 `--mix-child-hover--off` 的值，并根据其父元素的循环切换值来为该元素的直接子元素设置样式，因为它的值要么是空的，要么是 `initial`。

## 未来的 CSS Mixins

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/656dcd64708f487892f7522372a73a02~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=576&s=811298&e=png&b=081a31)

  


虽然使用分层切换可以实现 CSS Mixins，但它仍然是一种 Hack 技术，而且循环切换调式也比较复杂。好消息是，[@Miriam Suzanne 提出的 Custom CSS Functions & Mixins 提案已经被 CSSWG 接受](https://github.com/w3c/csswg-drafts/issues/9350)，这意味着 CSS 原生的 Mixins 即将到来。

  


到目前为止，该技术非常新且未经完善，草案中还有很多待办事项，甚至该草案还没有给出 Mixins 的定义，但我们可以先初步的了解一下 CSS 的原生 Mixins 将会是什么样的。除此之外，[@Miriam Ssuzanne 还发布了一篇更详尽的解释](https://css.oddbird.net/sasslike/mixins-functions/)，我们可以从中获取一些 CSS Mixins 的关键点。

  


### 自定义函数是高级自定义属性

  


我曾在介绍[交替样式查询](https://juejin.cn/book/7395119173656903715/section/7445287411581452338)时提到过，CSS 内置了很多单一用途的函数，比如我们熟悉的 `calc()` 函数、[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)（`sin()` 、 `cos()`）和[比较函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)（`min()` 、 `max()` 和 `clamp()`）等。要知道的是，在不久的将来，CSS 会有原生的 `@function` （类似于 Sass 的 `@function`）可以定义自定义函数，这些函数包含返回预期值的逻辑。

  


这使得自定义函数与自定义属性非常相似。自定义属性只是一个占位符，用来表示一些预期的值，我们通常在前期就定义它：

  


```CSS
:root {
    --primary-color: hsl(125 100% 50%);
}
```

  


自定义函数看起来非常相似，只是它们通过 `@function` 定义，并且接受参数。当前的语法规则像下面这样：

  


```CSS
@function <function-name> [( <parameter-list> )]? {
    <function-rules>
    result: <result>;
}
```

  


`result` 是自定义函数最终计算出的值。这一点可能会令人感到困惑，但我们可以这样来理解，自定义函数返回的是一个自定义属性。比如下面这个计算圆面积的示例：

  

CSS
```
@function --circle--area(--r) {
    --r-square: pow(var(--r), 2);
    result: calc(pi * var(--r-square));
}
```

  


调用该函数有点类似于声明自定义属性，只是**不需要** **`var()`** **，并且可以为定义的参数传递参数**：

  


```CSS
.element {
    inline-size: --circle-area(--r, 1.5rem); /* = ~7.065rem */
}
```

  


看起来我们可以用当前的 CSS 特性实现与自定义属性相同的效果：

  


```CSS
:root {
    --r: 1rem;
    --r-square: pow(var(--r), 2);
    --circle-area: calc(pi * var(--r-square));
}

.element {
    inline-size: var(--circle-area, 1.5rem);
}
```

  


也就是说，我们选择自定义函数而不是自定义属性的原因是自定义函数可以一次返回多个值，并且它们支持条件规则，比如 `@supports` 和 `@media` ，还可以根据需要返回不同的值。比如下面这个示例，它根据视口的内联尺寸（`inline-size`）返回多个值中的一个。

  


```CSS
@function --sizes(
    --s type(length),
    --m type(length),
    --l type(length),
) returns type(length) {
    --min: 16px;
    
    @media (inline-size < 20em) {
        result: max(var(--min), var(--s, 1em));
    }
    
    @media (20em < inline-size < 50em) {
        result: max(var(--min), var(--m, 1em + 0.5vw));
    }
    
    @media (50em < inline-size) {
        result: max(var(--min), var(--l, 1.2em + 1vw));
    }
}
```

  


### Mixins 有助于保持 DRY 和可重用的样式块

  


将来的 Mixins 可能看起来会有点像 `@function` ，只不过会使用 `@mixin` 来替代 `@function` ，这也正式 Sass 中定义 Mixins 的方式。

  


```CSS
/* 自定义函数 */
@function <function-name> [( <parameter-list> )]? {
    <function-rules>
    result: <result>;
}

@mixin <mixin-name> [( <parameter-list> )]? {
    <mixin-rules>
}
```

  


因此，自定义 Mixins 和自定义函数非常相似，但它们确实有所不同：

  


-   自定义函数是通过 `@function` 定义的，Mixins 是通过 `@mixin` 定义，但它们都用带有连字符的标识符命名，比如 `--name`
-   函数返回一个值，Mixins 返回样式规则

  


这使得 Mixins 非常适合抽象出可能作为工具类使用的样式，例如用于屏幕阅读器的隐藏文本类：

  


```CSS
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}
```

  


我们可以将此类应用于 HTML 中的元素以隐藏文本：

  


```CSS
<a class="sr-only">Skip to main content</a>
```

  


不过回到 Mixins！关键在于，我们可以几乎像使用小的 CSS 代码片段一样使用工具类，构建其他样式规则，并保持标记和样式之间的更清晰分离。如果我们将之前的 `.sr-only` 样式变成 Mixins：

  


```CSS
@mixin --sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}
```

  


与其跳到 HTML 中应用样式，我们可以通过一个新的 `@apply` at-rule 将它们嵌入到其他 CSS 样式规则中：

  


```CSS
header a:first-child {
    @apply --sr-only;
}
```

  


这与 Sass 的 `@include` 用法不同，Sass 使用 `@include` 来调用 Mixins，而不是 `@apply`。我们甚至可以返回更大的样式块，例如元素的 `::before` 和 `::after` 伪元素样式：

  


```CSS
@mixin --center-me {
    display: grid;
    place-items: center;
    position: relative;
    
    &::after {
        background-color: hsl(25 100% 50% / .25);
        content: "";
        inset: 0;
        position: absolute;
    }
}
```

  


当然，我们看到 Mixins 接受参数，就像自定义函数一样。如果你想让样式更灵活，可以使用参数，例如用不同颜色定义一致的渐变：

  


```CSS
@mixin --gradient-linear(--color-1, --color-2, --angle) {
    /* 等等 */
}
```

  


我们能够为每个参数指定语法，作为类型检查的一种形式：

  


```CSS
@mixin --gradient-linear(
    --color-1 type(color),
    --color-2 type(color),
    --angle type(angle),
) {
    /* 等等 */
}
```

  


我们可以进一步抽象这些变量，并为它们设置默认值：

  


```CSS
@mixin --gradient-linear(
    --color-1 type(color),
    --color-2 type(color),
    --angle type(angle),
) {
    --from: var(--color-1, orangered);
    --to: var(--from-color, goldenrod);
    --angle: var(--at-angle, to bottom right);
    /* 等等 */
}
```

  


然后，我们将 Mixins 的样式规则编写为变量占位符：

  


```CSS
@mixin --gradient-linear(
    --color-1 type(color),
    --color-2 type(color),
    --angle type(angle),
) {
    --from: var(--color-1, orangered);
    --to: var(--from-color, goldenrod);
    --angle: var(--at-angle, to bottom right);
    background: linear-gradient(var(--angle), var(--from), var(--to));
}
```

  


如果需要的话，可以在其中加入条件逻辑：

  


```CSS
@mixin --gradient-linear(
    --color-1 type(color),
    --color-2 type(color),
    --angle type(angle),
) {
    --from: var(--color-1, orangered);
    --to: var(--from-color, goldenrod);
    --angle: var(--at-angle, to bottom right);
    
    background: linear-gradient(var(--angle), var(--from), var(--to));
    @media (prefers-contrast: more) {
        background: color-mix(var(--from), black);
        color: white;
    }
}
```

  


所有这些都可以通过 `@apply` 在我们想要的规则集中使用：


```CSS
header {
  @apply --gradient-linear;
}

.some-class {
  @apply --gradient-linear;
}
```

  


我们还可以将它们与其他 Mixins 结合使用：

  


```CSS
header {
    @apply --gradient-linear;
    @apply --center-me;
}
```

  


最后声明，到目前为止， [Custom CSS Functions & Mixins 草案中](https://github.com/w3c/csswg-drafts/issues/9350) 并没有关于 CSS Mixins 的相关定义，上面这些内容暂时是由 @Miriam Suzanne 提供的设计思路，最终有可能会是这样，但也很有可能是别的更好的方式。这里仅是提供大家参考，告诉大家，未来原生 CSS Mixins 可能会是这样使用！
