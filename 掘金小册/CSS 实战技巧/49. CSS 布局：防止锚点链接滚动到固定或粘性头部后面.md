![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bf882d836b4444ca8c0971477a8922de~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1218&s=432738&e=jpg&b=ffffff)

  


粘性或固定页眉是 Web 中一种非常流行的布局模式。例如，它在 Web 页面顶部显示导航栏，始终提供菜单项的访问，即使在滚动页面时也能保持可见。以往，我们通常使用固定定位（`position: fixed`）来创建这种布局效果。随着 CSS 技术不断往前发展，现如今，除了使用固定定位之外，还可以使用粘性定位（`position: sticky`）来创建。

  


然而，粘贴定位是 CSS 中一个相当微妙的功能，[受许多因素影响容易失效](https://juejin.cn/book/7199571709102391328/section/7199845514462887948)。例如，如果粘性元素（比如页眉 `header`）需要与其他元素配合来组合高度（比如 `100vh`），就很难正常工作。另外，通常情况之下，粘性页眉能为用户提供更好的用户体验，但页面中的菜单使用锚点让用户可以即时跳转到页面的特定部分，我们就会遇到一个问题。当点击锚点链接时，页面会滚动并将锚点定位到视口的最顶部，这意味着该部分的标题甚至可能被粘性页眉遮挡住。

  


接下来，我们将以这两个在 Web 开发中常见的情景为例，向大家阐述，有哪些 CSS 特性可以使我们构建出一个更优雅的粘性页眉布局效果，并且能给用户提供更好的体验。

  


> 温馨提示，在这节课我们并不会深入讨论粘性定位失效的原因以及如何修复。如果你平时在使用粘性定位碰到相关问题，请移步阅读《[`position: sticky` 失效与修复](https://juejin.cn/book/7199571709102391328/section/7199845514462887948)》！

  


  


## 粘性页眉 + 全屏高度：一个棘手的组合

  


从标题的字面上我们就能知道，我们即将要创建的布局效果中包含了一个粘性定位的页眉，并且与另一个元素的组合高度（两个元素高度加起来）与浏览器屏幕视窗高度相等。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/20fde0733f884bcd9864ae8631d8bac4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1510&h=632&s=3826337&e=gif&f=363&b=121214)

  


看到上图这个效果，你可能会想起之前小册《[CSS 布局：创建全屏幕布局](https://juejin.cn/book/7395119173656903715/section/7444358182228459560)》中所提到的布局技术。但仔细一对比，使用这种技术创建的布局效果离我们所期望的效果还是有一定差异的。比如，当紧邻粘性页眉（`header`）之后的元素（例如 `main` 元素）自身高度超出一定的高度时，`header` 和 `main` 的高度就超出了全屏高度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/986e00c049b74a68bb79d28372d745ee~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1146&h=828&s=11390004&e=gif&f=111&b=545f44)

  


而我们实际希望的效果是，`header` 和 `main` 元素的高度总和是 `100vh` ，甚至是 `header` 和它之后的另何一个元素的高度总和是 `100vh` 。

  


接下来，我们就来看看，如何使用 CSS 创建这样的布局效果。首先，从 HTML 结构开始：

  


```HTML
<body>
    <header>Sticky Header</header>
    <section>
        <img src="https://picsum.photos/id/163/1920/1024" alt="" />
    </section>
    <!-- 省略其他的 Section -->
</body>
```

  


在 CSS 中将 `header` 粘贴在页面顶部非常容易，我们使用 `position: sticky` 就可以做到：

  


```CSS
header {
    position: sticky;
    top: 0;
}
```

  


接下来，我们需要使 `header` 元素与之后的任意一个 `section` 元素的高度总和为 `100vh` 或 `100dvh`（等于浏览器视窗的高度）。你可能首先会想到的是，给 `header` 指定一个 `height` 值，然后使用 `100vh - height` 来计算出每个 `section` 元素的高度:

  


```CSS
html {
    --h: 88px; /* Sticky Header Height */
}

header {
    height: var(--h);
    position: sticky;
    top: 0;
}

section {
    height: calc(100dvh - var(--h));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3f7411539f0f44538223b15926418e52~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=628&s=20360379&e=gif&f=80&b=d0cca6)

  


> Demo 地址：https://codepen.io/airen/full/bNbBQPm

  


乍一看，这效果是我们所期望的。但它有一个明显的缺陷，我们需要给 `header` 指定一个明确的 `height` 值。这意味着，如果 `header` 元素的 `height` 是一个确切值，且不会改变，那么这个方案可行。但要构建一个更为灵活的布局效果，这种方法就略显不足。稍微想想：

  


-   如果 `header` 的子元素需要在不同的屏幕下换行或重新排列，或者在移动设备上需要增加高度以保持可读性怎么办？
-   如果 JavaScript 动态修改了 `header` 元素的内容怎么办？

  


所有这些情况都可能会改变 `header` 元素的高度，而为每种情况为 `header` 元素指定一个 `height` 值可能会演变成一个无法维护的噩梦。我认为这种仅宜之计从感觉上来说也是不对的。固定高度违背了 CSS 布局的一个核心优势——**元素能够根据内容自动伸缩**。而放弃这种优势通常只会让我们的工作变得更难，而不是更简单。

  


既然如此，我考虑将用额外的一个元素来包裹所有的 `section` 元素，例如：

  


```HTML
<body>
    <header>Sticky Header</header>
    <main>
        <section>
            <img src="https://picsum.photos/id/163/1920/1024" alt="" />
        </section>
        <!-- 省略其他的 Section -->
    </main>
</body>
```

  


这样构建 HTML 结构，使得我们在 CSS 中使用 CSS Flexbox 或 CSS Grid 布局就能很容易做到 `header` 和 `main` 元素的高度总和是 `100vh` 。但这种做法，我们需要禁用页面默认的滚动条，然后给 `main` 添加滚动条。

  


是的，你可能已经猜到了，我想着使用之前小册所介绍的创建 **[全局布局的技术](https://juejin.cn/book/7395119173656903715/section/7444358182228459560)** 来构建所需要的布局效果：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    grid-template-rows: min-content auto;
    min-height: 0;
}

main {
    overflow-y: auto;
}

header {
    position: sticky;
    top: 0;
}    
```

  


很快你就会发现，虽然 `header` 和 `main` 的高度总和是 `100vh` ，但无法确保 `header` 与每个 `section` 元素的高度总和是 `100vh` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/77d0bfb4081a4b23bcb0060c54f5c73e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1198&h=766&s=15436618&e=gif&f=51&b=c5d8e1)

  


你可能想到了给每个 `section` 元素的 `height` 设置为 `100%` ，使它的高度等于 `main` 元素的高度。这种想法是可行的，但我在这里想告诉大家另一个更现代的方法——采用 **[CSS 容器查询特性](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)**。

  


我们只需要将 `main` 元素的 `container-type` 设置为 `size` ，然后就可以查询 `main` 容器的尺寸，从而调整其子元素 `section` 的样式。比如，我们把 `section` 的 `height` 设置为 `100cqh` ，就能确保每个 `section` 元素的 `height` 与 `main` 容器的高度保持一致（即 `100%` 的 `main` 容器高度 ）：

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    grid-template-rows: min-content auto;
    min-height: 0;
}

main {
    overflow-y: auto;
    container-type: size;
}

header {
    position: sticky;
    top: 0;
}

section {
    height: 100cqh;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ae468331e65d44f78f93cb1162d1d361~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1190&h=610&s=15865132&e=gif&f=62&b=d0caa6)

  


> Demo 地址：https://codepen.io/airen/full/raBWoWe

  


如果你足够仔细的话，不难发现，使用这种技术，`header` 元素是否采用粘性定位（`position: sticky`），最终效果都是一致的。换句话说，我们采用了现代布局技术来模拟了一个粘性定位布局效果。这种技术方案在一些场景中是非常适用的，比如后面我们要介绍的另一个案例——防止锚点链接滚动到粘性页眉底部。

  


我们再来看 Web 布局中的另一个常见场景。例如，我们只需要将粘性定位的 `header` 元素与其紧邻的横幅区域的高度总和是 `100vh` ，而页面中的其他的元素的高度根据自己的内容来决定。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dd642607c4554a709f3c869561b76679~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1510&h=632&s=3826337&e=gif&f=363&b=121214)

  


你的第一反应可能跟我一样，使用一个容器来包裹 `header` 和横幅区域，并将该容器设置为 `100vh` （或 `100dvh` ，它有动态考虑的 UA UI，[详细阐述请点击这里获取](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)），让它占据整个浏览器视窗的高度。然后可以使用 Flexbox 或 Grid 来分布子元素，让横幅扩展填充剩余的空间：

  


```HTML
<body>
    <div class="container">
        <header>Sticky Header</header>
        <section class="hero">
            <img src="https://picsum.photos/id/163/1920/1024" alt="" />
        </section>
    </div>
    <main>Main Content</main>
</body>
```

  


```CSS
.container {
    height: 100dvh;
    display: flex;
    flex-direction: column;
    
    .hero {
        flex-grow: 1;
    }
}

header {
    position: sticky;
    top: 0;
}
```

  


或者：

  


```CSS
.container {
    height: 100dvh;
    display: grid;
    grid-template-rows: auto minmax(0, 1fr);
}

header {
    position: sticky;
    top: 0;
}
```

  


乍一看，这似乎是正确的做法，但当页面滚动超过横幅区域时，看看会发生什么：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2e198afaaec1427ea439a53855fc1243~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=614&s=13086568&e=gif&f=44&b=cfcca4)

  


> Demo 地址：https://codepen.io/airen/full/WbeoLyN

  


不难发现，当页面滚动超过横幅区域时，`header` 的 `position: sticky` 失效了。如果你阅读过《[`postion: sticky` 失效和修复](https://juejin.cn/book/7199571709102391328/section/7199845514462887948)》，就应该知道，`sticky` 是 `relative` 和 `fixed` 定位的结合，这意味着它在文档的正常流中起作用，直到触及其滚动容器的边缘，此时它会变为固定定位。虽然将 `sticky` 视为春他值的组合是一种有用的记忆方式，但这种解释遗漏了一个重要的区别：**`sticky`** **和** **`fixed`** **元素的行为并不完全相同**。

  


一个 `position: fixed` 的元素不会在意它嵌套在哪个父元素或祖先元素中。它会脱离文档的正常流，并直接相对于浏览器视窗进行偏移，就好像被“粘”在屏幕边缘某个固定距离上一样。

  


相反，`position: sticky` 的元素会随着视口（或最近的滚动容器）的边缘一起移动，但它永远不会突破其直接父容器的边界（除非你通过视觉上的 `transform` 改变它的位置）。换句话说，**`position: sticky`** **在某种意义上是局部范围内的** **`position: fixed`** 。这是一个有意设计的特性，旨在支持特定部分的粘性标题，比如在移动端中广为人知的按字母排序列表中的粘性标题效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8c4a481740044cbebe877e28bb7db6f1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1196&h=650&s=2341058&e=gif&f=255&b=9ba2b0)

  


> Demo 地址：https://codepen.io/airen/full/ByBQvvX

  


好的，这种方法是无法实现我们所需要的布局效果。我们需要另寻其他的布局方案。你可能想到了之前案例展示的方法，通过 CSS Flexbox 或 CSS Grid 来模拟粘性定位的方法。

  


```HTML
<body>
    <header>Sticky Header</header>
    <main>
        <section class="hero">
            <img src="https://picsum.photos/id/163/1920/1024" alt="" />
        </section>
        <div class="container">
            <!-- 页面其他内容 -->
        </div>
    </main>
</body>
```

  


```CSS
:is(html, body) {
    display: grid;
}

html {
    height: 100%;
}

body {
    grid-template-rows: min-content auto;
    min-height: 0;
}

main {
    overflow-y: auto;
    container-type: size;
}

header {
    position: sticky;
    top: 0;
}

.hero {
    height: 100cqh;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/13f386ffd55045909f1b002139e5108f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1194&h=610&s=19133386&e=gif&f=62&b=1f1633)

  


> Demo 地址：https://codepen.io/airen/full/azoBPMV

  


正如你所想，这种方法实现了我们所期望的布局效果。即便如此，我还是想花一点时间来告诉大家另一种解决方案。

  


我们的需求非常明确——希望 `header` 和 `.hero` 两个元素的高度总和是 `100vh` ，但不通过显示设置高度或将它们包裹在一个容器中。换过一个角度来思考，我们是否可以找到一个已经是 `100vh` 的元素，并将它们与其对齐呢？这时，我想到了 [CSS Grid 创建全屏布局的技术](https://juejin.cn/book/7395119173656903715/section/7444358182228459560)。

  


我们可以新增一个全屏宽高的元素，比如 `.placeholder` ，这个元素的高度为 `100vh` ，占据网格的整个宽度。接着，告诉浏览器，这个占位元素 `.placeholder` 占用网格的两行，并将其锚定在页面的顶部。注意，`.placeholder` 元素仅是一个占位元素，它是不可见的，唯一的作用就是用它来引导网格的布局行为。

  


如此一来，我们可以把 HTML 结构调整为：

  


```HTML
<body>
    <div class="placeholder"></div>
    <header>Sticky Header</header>
    <section class="hero">
        <img src="https://picsum.photos/id/163/1920/1024" alt="" />
    </section>
    <main>Main Content</main>
</body>
```

  


```CSS
body {
    display: grid;
    grid-template-rows: min-content minmax(0, 1fr);
}

.placeholder {
    height: 100vh;
    grid-row: 1 / span 2;
    grid-column: 1 / -1;
    z-index: -1;
}

header {
    position: sticky;
    top: 0;
    grid-column-start: 1;
    grid-row-start: 1;
}

.hero {
    grid-column-start: 1;
    grid-row-start: 2;
}
```

  


首先我们将 `body` 定义为一个 `2 x 1` （两行一列）的网格，并且使用 `grid-template-rows` 定义了行网格轨道的尺寸，其中第一行高度为 `min-content` ，第二行高度为 `minmax(0, 1fr)` 。与此同时，占位元素 `.placeholder` 将会占据网格的两行，并且它的高度为 `100vh` ，这使得网格的第一行和第二行高度总和等于 `100vh` ，即等于浏览器视窗高度。

  


然后，我们分别将 `header` 放置在网格第一行，将 `.hero` 放置在网格第二行，它们正好与占位元素层叠在一起。这将使得网格第一行高度就是 `header` 元素的内容高度（因为第一行网格轨道高度设置为 `min-contnet`），第二行的高度则是 `1fr` ，在这里就是 `100vh` 减去 `header` 元素内容高度。这意味着，`header` 和 `.hero` 两元素的高度总和为 `100vh` ，正好符合我们的需求。

  


现在，我们要做的只是将 `header` 的 `position` 设置为 `sticky` ，使它始终粘贴在页面的顶部。就是这么简单，最终获得我们所需要的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bc300e416ddf47059a0c21acdc8786b4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=608&s=8509378&e=gif&f=29&b=1f1734)

  


> Demo 地址：https://codepen.io/airen/full/yyBVwax

  


值得注意的是，HTML 中元素的顺序在这里非常重要。如果我们将 `.placeholder` 放置在 `.hero` 元素后面，它将覆盖并阻碍对下方元素的访问。我们可以通过以下方式解决这个问题：

  


-   将 `.placeholder` 元素的 `order` 设置为 `-1`
-   将 `.placeholder` 元素的 `z-index` 设置为 `-1`
-   将 `.placeholder` 元素的 `visibility` 设置为 `hidden`

  


需要知道的是，上面演示的是一个非常简单的示例。如果你在页面左侧添加了一个侧边栏，那么需要根据具体的场景重新放置网格项目。有关于这方面更详细的介绍，可以阅读下面这两节课的内容：

  


-   [CSS 布局：隐式网格和自动放置](https://juejin.cn/book/7395119173656903715/section/7434843979486593062)
-   [网格项目的放置和层叠](https://juejin.cn/book/7161370789680250917/section/7161623932439625758)

  


这种方法具有较强的适应性。如果你希望屏幕高度被三个元素分配而不是两个，你只需要让这个不可见的占位元素（`.placeholder`）跨越三行，并将可见元素分配到适当的行中即可。即使 `.hero` 元素的内容导致其高度超过 `100vh` ，网格也能适应而不会破坏布局。而且，这种方法在兼容所有现代浏览器。

  


## 防止锚点链接在粘性或固定页眉后面

  


之前的案例我们主要讨论的是 CSS 如何使粘性页眉与其紧邻的元素高度总和等于浏览器视窗的高度。其实，粘性页眉（或固定页眉）还存在另一种交互模式。那就是在粘贴页眉放置一些锚点链接，当用户点击锚点链接时，用户可以即时跳到页面的特定部分。然而，我们时常会碰到一个问题，当用户点击锚点链接时，页面会滚动并将锚点定位到浏览器视窗的最顶部，这意味着该部分的标题可能被粘性页眉遮挡住：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c495097a450e4b79a1aebb5c5bc4c314~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1218&s=432738&e=jpg&b=ffffff)

  


我们希望要的是类似上图右侧部分展示的效果。

  


构建这样的布局效果，我们可能会使用类似下面这样的 HTML 结构：

  


```HTML
<body>
    <header>
        <ul class="menu">
            <li><a href="#kiwis">Kiwis</a></li>
```

上面的代码并没有什么特殊之处，有三点简单的提示一下：

  


-   通过 `position: sticky` 将 `header` 设置为一个粘性元素，它将会粘贴在页面的顶部
-   通过 `margin-inline` 与 `max()` 函数的结合使用，将所有元素水平居中。关于这方面的详细介绍，请阅读小册的《[CSS 布局：水平居中](https://juejin.cn/book/7395119173656903715/section/7440736421910052864)》

原本我想着，给 `body` 设置一个与页眉等高的 `padding-top` 可以解决。但事实告诉，这种方案不可行。于是，我把思路切回到之前所提到的全屏布局技术方案：

  


```CSS
html {
    display: grid;
    height: 100%;
}
  
   
body {

    display: grid;
    grid-template-rows: min-content auto;
    min-height: 0;

        }
    }
}
  
header {
    position: sticky;
    top: 0;
    border-image: conic-gradient(var(--c) 0 0) fill 0//0 100vw;

}
```

庆幸的是，我们有了一个新的、简单的、一行 CSS 解决方案：**`scroll-margin-top`** **和** **`scroll-padding-top`** 。

  


-   `scroll-margin-top` 属性定义了锚点部分（即容器的子元素）的顶部外边距，浏览器在将滚动元素对齐时会使用这个外边距
-   `scroll-padding-top` 属性应用于父容器，作用类似于 CSS 的顶部内边距，定义了滚动区域顶部的偏移量

  


在我们这个示例中，我们可以像下面这样使用：

  


```CSS
html {
    scroll-padding-top: 137px;
}
```

> Demo 地址：https://codepen.io/airen/full/GgKNLYz

  


或者将 `scroll-margin-top` 用于每个锚点部分：

  


```CSS
section h2{
    scroll-margin-top: 137px;
}
```

> Demo 地址：https://codepen.io/airen/full/wBwoZRV

  


你可能会感到好奇，示例中的 `scroll-padding-top` 和 `scroll-margin-top` 属性值 `137px` 是如何得来的？目前没有比较好的方法，你可能需要通过浏览器的开发者调试器对效果进行多次调试，获取一个最终最适合的值。

  


还需要知道的是，这些属性仅适用于滚动对齐，因此它们不会影响 HTML 元素的实际内边距或锚点部分之间距义的外边距。另外，你无需担心浏览器的兼容性，因为它们都得到了主流浏览器的支持。

  


既然聊到了锚点链接，那么就再多花一点点篇幅向大家介绍几个 CSS 小技巧，这几个小技巧可以使整个锚点链接的体验更好。

  


之前我们提到过，一般在长内容的 Web 页面中会使用到锚点链接，用户通过点击锚点链接能快速跳转到对应的内容区域。通常情况之下，这种跳转是比较生硬的，基本上是一闪而过。其实，我们只需要在滚动容器添加一行 CSS 代码，可以使用整个跳转变得很丝滑：

  


```CSS
html {
  scroll-behavior: smooth;
}
```

## 小结

  


最后，我想再次提醒大家，CSS 的 `position: sticky` （粘性定位）是一个非常微妙的特性，在使用它创建粘性页眉或侧边栏时，很容易使其失效，大家需要根据具体的场景来分析并找到解析问题的方案。在这节课，我们主要介绍了两个粘性定位实际用例：

  


-   粘性页眉+全屏高度，这是一个比较棘手的布局组合。我们通过增加占位符与 CSS Grid 布局特性，实现了它，并且具有较强的灵活性和可适配性
-   通过 `scroll-margin-top` 或 `scroll-padding-top` 避免了锚点链接在粘性或固定页眉后面

  


希望这两个案例能更好的帮助大家解决实际需求，使你能快速有效的创建灵活适配的 Web 布局！
