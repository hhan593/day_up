## 在暗黑模式的世界里发生了什么？

  


在实际应用中，你可能会结合 **[CSS 媒体查询](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)** 和 **[自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)** 为暗黑模式设置颜色。你会先为亮色模式设置颜色，然后当特殊的暗色模式匹配时，重新声明颜色变量的值，使用它们。例如：

  


```CSS
:root {
    color-scheme: light dark;
    --dark-color: #292524;
    --light-color: #f5f5f4;
    --text-color: var(--dark-color);
    --bg-color: var(--light-color);
    
    @media (prefers-color-scheme: dark) {
        --text-color: var(--light-color);
        --bg-color: var(--dark-color);
    }
}

body {
    color: var(--text-color);
    background-color: var(--bg-color);
}
```

  


然后，CSS 新增的 `light-dark()` 函数，使我们创建暗黑模式变得更容易。现在你可以这样做：

  


```CSS
:root {
    color-scheme: light dark;
    --light: #292524;
    --dark: #f5f5f4;
}

body {
    color: light-dark(var(--light), var(--dark));
    background-color: light-dark(var(--dark), var(--light));
}
```

  


本质上，这避免了你使用 `@media` 查询并重新声明变量。我很喜欢这个方法，它使得 CSS 代码更加易读和简洁。当然，这还依赖于变量命名和使用的方式。

  


当我第一次看到 `light-dark()` 函数时，我很喜欢它，同时也产生了一个困惑。一开始，我还认为，一旦你为用户提供了切换暗黑模式的按钮，使用 `light-dark()` 函数就会遇到麻烦。毕竟，你无法改变 `prefers-color-scheme` 媒体查询返回的内容，因此也无法改变 `light-dark()` 会选择两个值中的哪个。但测试之后的结果告诉，这一切都不是问题！其中的诀窍就在于 `color-scheme` 属性，它会强制 `light-dark()` 函数选择相关的值。

  


你可能会感到好奇——为什么呢？那么，我们就先从 `color-scheme` 属性开始吧！

  


## CSS `color-scheme` 及其相关特性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/45459487fa59422ab1a6f71c4ffe5eee~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=263806&e=webp&b=dbd4cc)

  


众所周知，当 Web 页面禁用 CSS 时，它会以“白底黑字”的形式在浏览器中呈现。孰不知的是，它还内置了一个“黑底白字”的版本。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5e00096749564115856f9c560ef04713~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1420&h=740&s=1904548&e=gif&f=166&b=111111)

  


如果你想创建一个暗黑模式的界面，这将是一个很好的基础，它可以帮助你省去处理一些烦人的细节，比如暗色模式下的输入框（`<input>`）、按钮（`<button>`）和其他交互元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d7648d0bb4e749588a86f062b96be126~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=840&s=15544&e=webp&b=141414)

  


这些原生主题（“白底黑字”或“黑底白字”）都是系统内置的颜色。在 CSS 中，你可以使用来自多个 **[颜色空间](https://juejin.cn/book/7223230325122400288/section/7231515598088306720)** 的颜色。比如，你可以使用颜色关键词（如 `red`）、十六进制（如 `#ff0000`）、特定的颜色空间的函数（如 `rgb(255 0 0)` ），或[高清颜色空间的 color() 函数](https://juejin.cn/book/7223230325122400288/section/7233227753909125178)（如 `color(display-p3 0.92 0.2 0.14)`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/06a6789ab91446b8967ff17661f736ce~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3058&h=1786&s=1136591&e=png&b=f4fafb)

  


除此之外，CSS 还包含**系统颜色**，这些颜色在 [CSS Color Module Level 4 规范](https://www.w3.org/TR/css-color-4/#css-system-colors)中定义。系统颜色由浏览定义，并以关键字的形式表示。

  


例如，系统颜色 `Canvas` （不要与 `<canvas>` 元素混淆），表示“应用内容或文档的背景颜色”。它通常与 `CanvasText` 搭配使用，后者表示“应用内容或文档中的文本颜色”。我们在 CSS 中，可以这样使用它们：

  


```CSS
body {
  color: CanvasText;
  background-color: Canvas;
}
```

  


默认情况下，`CanvasText` 的颜色接近黑色，而 `Canvas` 的颜色接近白色。具体的实现取决于浏览器。例如，在 Chrome 中，`CanvasText` 的默认颜色是 `#121212`，而在 Safari 中，它的颜色是略浅的 `#1e1e1e`。

  


这些系统颜色还有很多。它们是不区分大小写的，通常使用驼峰命名法或帕斯卡命名法以提高可读性。在 CSS 中大约有 `19` 个系统颜色命名：

  


-   `AccentColor`: 强调用户界面控件的背景颜色
-   `AccentColorText`: 强调用户界面控件的文本颜色
-   `ActiveText`: 活动链接的文本颜色
-   `ButtonBorder`: 控件的基础边框颜色
-   `ButtonFace`: 控件的背景颜色
-   `ButtonText`: 控件的文本颜色
-   `Canvas`: 应用程序内容或文档的背景颜色
-   `CanvasText`: 应用程序内容或文档中使用的文本颜色
-   `Field`: 输入字段的背景颜色
-   `FieldText`: 表单输入字段中的文本颜色
-   `GrayText`: 禁用项的文本颜色（例如禁用的控件）
-   `Highlight`: 选中项的背景颜色
-   `HighlightText`: 选中项的文本颜色
-   `LinkText`: 非活动、未访问链接的文本颜色
-   `Mark`: 在 `<mark>` 元素中标记的文本背景颜色
-   `MarkText`: 在 `<mark>` 元素中标记的文本颜色
-   `SelectedItem`: 选中项的背景颜色（例如选中的复选框）
-   `SelectedItemText`: 选中项的文本颜色
-   `VisitedText`: 已访问链接的文本颜色

  


注意，这些系统颜色有两种模式的，而且不同的浏览器对应的色值略有差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7b4a1859a6ee46ca97e5ee396406623e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1863&h=1564&s=952245&e=png&b=ffffff)

  


刚才我们提到过，系统内置了两套主题——白字黑字和黑底白字。简单地说，浏览器有一个内置的“颜色方案”机制，可以使用 `color-scheme` 属性或相应的 `<meta>` 标签来指定。

  


`color-scheme` 可以设置为 `light` 或 `dark` ，以指示页面支持哪种模式，也可以设置为空格分隔的 `light dark` 值，表示页面支持这两种模式。

  


通常，你会在 `html` 元素上设置这个属性，以便让页面上的所有内容继承这个值：

  


```CSS
html {
  color-scheme: light dark; 
}
```

  


甚至可以直接在 HTML 文档的 `<meta>` 标签中实现这一点，无需 CSS：

  


```HTML
<meta name="color-scheme" content="light dark">
```

  


无论使用 CSS 还是 HTML，它们的作用是一样的——告诉浏览器**同时支持亮色和暗色模式，并自动应用符合用户偏好的模式**。我们不需要在 CSS 中到处写 `prefers-color-scheme` 相关的逻辑，因为浏览器已经内置了这一机制！大多数操作系统都提供内置的可访问性设置，允许用户选择“亮色”、“暗色”或“自动”模式，而浏览器会尊重这些设置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2838825059f347a98e7fb27ffc4d7067~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=523&s=79884&e=webp&b=333334)

  


然后，浏览器使用解析后的颜色方案值来确定页面背景、文本颜色、滚动条、表单控件和系统颜色的默认颜色。

  


值得澄清的是，`color-scheme` 与 `prefers-color-scheme` 媒体查询是完全不同的概念，后者你可能曾用于实现暗色模式。媒体查询仅关注用户的偏好，即使你在使用媒体查询，它也应与 `color-scheme` 属性配合使用，以便滚动条和表单控件获得更合适的默认样式。

  


```CSS
html {
    color-scheme: light;

    @media (prefers-color-scheme: dark) {
        color-scheme: dark;
    }
}
```

  


此外，你可以选择不同的方式（纯 CSS 或 JavaScript）来调整 CSS 的 `color-scheme` 属性的值，但 `<meta>` 的值只能通过 JavaScript 来调整。 稍后我们会聊到这个部分。

  


现在我们了解了 `color-scheme`，接下来可以看看新的 `light-dark()` 特性！

## 创建暗黑模式的新方式：`light-dark()`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eac0d5bebaa649659053b0305f9d4104~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=455106&e=webp&b=402242)

  


到目前为止，`color-scheme` 是一种单向信号，对所用 `color-scheme` 值做出响应是系统颜色专有的功能。换句话说，浏览器可以使用 `color-scheme` 来调整默认颜色（和系统颜色），但开发者无法在自己的代码中使用它来调整自定义样式。

  


随着 `light-dark()` 函数的出现，将会改变这样的现状，正如 [W3C 规范](https://www.w3.org/TR/css-color-5/#light-dark)所述：

  


> 系统颜色可以响应当前使用的颜色方案值。`light-dark()` 函数为开发者提供了相同的功能！

  


`light-dark()` 是一个接受两个颜色值的 CSS 函数，系统会根据所使用的配色方案（`color-scheme`）选择其中一种：

  


-   当 `color-scheme` 为亮色模式（`light`）或未知时，返回第一个颜色
-   当 `color-scheme` 为暗色模式（`dark`）时，返回第二个颜色

  


`light-dark()` 函数的结果是一个 `<color>` 类型值，可以在任何接受颜色的 CSS 规则中使用。例如，可以用在 `color` 、`background-color` 、`border-color` ，甚至是渐变函数中（比如 `linear-gradient()`）。

  


需要知道的是，为了让 `light-dark()` 正常工作，你需要在 CSS 指定 `color-scheme` 属性的值，或者在页面的 HTML 中使用相应的 `<meta>` 标签，因为它允许该函数尊重用户的系统偏好，或者你在 `color-scheme` 中设置的任一单一亮色或暗色值。

  


另外，`light-dark()` 函数的值也可以是 CSS 自定义属性。例如：

  


```CSS
:root {
    color-scheme:  light dark;
    --accent-light: white;
    --accent-dark: black;
}

.element {
    color: light-dark(var(--accent-light), var(--accent-dark));
}
```

  


为了能更充分的利用 `light-dark()` 函数，我们可以在全局范围内使用它来定义主题相关的自定义属性。例如：

  


```CSS
:root {
    color-scheme: light dark;
    
    --accent: light-dark(deeppink, hotpink);
    --bg: light-dark(white, black);
    --c: light-dark(black, white);
}
```

  


一旦定义了这些自定义属性，我们可以在整个 CSS 代码中使用它们。注意，我们不需要重复写具体的颜色值。因为这些自定义属性是通过 `light-dark()` 函数设定的，所以整个页面能够自动适配亮色模式和暗色模式，而无需在组件级别做任何额外调整。

  


```CSS
:root {
    color-scheme: light dark;
    
    --accent: light-dark(deeppink, hotpink);
    --bg: light-dark(white, black);
    --c: light-dark(black, white);
}

*:focus-visible {
    outline: 2px solid var(--accent);
}

:any-link {
    color: var(--c);
}

button {
    color: var(--c);
    border: 1px solid var(--c);
}

.surface {
    background-color: var(--bg);
}
/* 其他样式... */
```

  


目前，我们看到的 `light-dark()` 函数，主要是用来替代 CSS 的 `@media (prefers-color-scheme)` ，但通常我们还需要更多功能：Web 网站或应用应该默认尊重用户的系统偏好，但用户也应该能够手动更改网站的颜色模式。

  


然而，`light-dark()` 函数绑定的是 `color-scheme` ，而不是直接绑定用户的系统偏好（**[系统的偏好设置由 CSS 的 `@media` 来查询](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-14)**）。尽管 `color-scheme: light dark` 或 HTML 的 `<meta>` 标签会默认匹配用户偏好设置，但我们仍然可以手动覆盖它。

  


假设我们网站有一个主题切换按钮（它可能是一组单选按钮或一个下拉选择框），通过 CSS 的 `:has()` 选择器，可以在 CSS 中手动调整 `color-scheme` 。例如：

  


```CSS
:root {
    color-scheme: light dark;

    &:has(input[name="color-scheme"][value="light dark"]:checked) {
        color-scheme: light dark;
    }
  
    &:has(input[name="color-scheme"][value="light"]:checked) {
        color-scheme: light;
    }
  
    &:has(input[name="color-scheme"][value="dark"]:checked) {
        color-scheme: dark;
    }
}
```

  


或者：

  


```CSS
:root {
    color-scheme: light dark;

    &:has(option[value="light dark"]:checked) {
        color-scheme: light dark;
    }
  
    &:has(option[value="light"]:checked) {
        color-scheme: light;
    }
  
    &:has(option[value="dark"]:checked) {
        color-scheme: dark;
    }
}
```

  


更改 `color-scheme` 后，所有使用 `light-dark()` 的地方都会自动更新为相应的颜色值。上面的代码中，我们是在根元素上设置 `color-scheme` ，但 `light-dark()` 也可以用于所有继承 `color-scheme` 的子元素。

当然，你还可以引入 CSS 自定义属性，它能将 CSS 代码变得更简洁且易于维护。感兴趣的同学，可以尝试一下！

  


注意，上面示例展示的是纯 CSS 调整 `color-scheme` 属性的值，你也可以使用 JavaScript 来调整。如果你在 HTML 的 `<head>` 中通过 `<meta>` 标签设置了 `color-scheme`：

  


```HTML
<meta name="color-scheme" content="light dark">
```

  


你可以先创建一个有用的常量，如下所示：

  


```JavaScript
const colorScheme = document.querySelector('meta[name="color-scheme"]');
```

  


然后你可以根据需要修改它，设置为亮色或暗色：

  


```JavaScript
colorScheme.setAttribute("content", "light"); // 切换到亮色模式
colorScheme.setAttribute("content", "dark"); // 切换到暗色模式
colorScheme.setAttribute("content", "light dark"); // 切换到自动模式
```

  


这种方法与使用 `<meta>` 标签非常相似，但如果你在 CSS 中设置了 `color-scheme` 属性，它会有所不同：

  


```CSS
html { 
    color-scheme: light dark; 
}
```

  


与我们在上一个示例中使用 `<meta>` 标签时设置常量 `colorScheme` 的做法不同，这里你可能选择 `html` 元素：

  


```JavaScript
const html = document.querySelector('html');
```

  


现在你的操作看起来像这样：

  


```JavaScript
html.style.setProperty("color-scheme", "light"); // 切换到亮色模式
html.style.setProperty("color-scheme", "dark"); // 切换到暗色模式
html.style.setProperty("color-scheme", "light dark"); // 切换到自动模式
```

  


你也可以将这些操作封装成函数，这样可以重复使用它们：

  


```JavaScript
function switchAuto() {
    html.style.setProperty("color-scheme", "light dark");
}

function switchLight() {
    html.style.setProperty("color-scheme", "light");
}

function switchDark() {
    html.style.setProperty("color-scheme", "dark");
}
```

  


另外，如果你喜欢尽量保持代码的 DRY（不要重复自己），你可以这样做：

  


```JavaScript
function switchMode(mode) {
    html.style.setProperty("color-scheme", mode === "auto" ? "light dark" : mode);
}
```

  


除此之外，还有一个小细节需要注意。我们还可以借助 JavaScript 来存储用户的选择，这样就能记住用户的选择，以便下次访问。具体的就不展开阐述了，感兴趣的同学可以阅读与面这段代码：

  


```JavaScript
const html = document.querySelector("html");
const mode = localStorage.getItem("mode");
const lightSwitch = document.querySelector('#lightSwitch');
const darkSwitch = document.querySelector('#darkSwitch');
const autoSwitch = document.querySelector('#autoSwitch');
if (mode === "light") switchLight();
if (mode === "dark") switchDark();
function switchAuto() {
    html.style.setProperty("color-scheme", "light dark");
    localStorage.removeItem("mode");
    lightSwitch.setAttribute("aria-pressed","false");
    darkSwitch.setAttribute("aria-pressed","false");
    autoSwitch.setAttribute("aria-pressed","true");
}

function switchLight() {
    html.style.setProperty("color-scheme", "light");
    localStorage.setItem("mode", "light");
    lightSwitch.setAttribute("aria-pressed","true");
    darkSwitch.setAttribute("aria-pressed","false");
    autoSwitch.setAttribute("aria-pressed","false");
}

function switchDark() {
    html.style.setProperty("color-scheme", "dark");
    localStorage.setItem("mode", "dark");
    lightSwitch.setAttribute("aria-pressed","false");
    darkSwitch.setAttribute("aria-pressed","true");
    autoSwitch.setAttribute("aria-pressed","false");
}
```

  


加载时，按钮会有一个默认设置，即“自动”模式按钮处于活动状态。如果在 `localStorage` 中有其他模式，我们会立即获取并运行 `switchLight()` 或 `switchDark()`，这两者都会包含与该模式相关的 `aria-pressed` 更改。

  


```HTML
<button id="autoSwitch" aria-pressed="true" type="button" onclick="switchAuto()">Auto</button>
<button id="lightSwitch" aria-pressed="false" type="button" onclick="switchLight()">Light</button>
<button id="darkSwitch" aria-pressed="false" type="button" onclick="switchDark()">Dark</button>
```

  


`aria-pressed` 的最后一个好处是，我们还可以用它来进行样式设置：

  


```CSS
button[aria-pressed="true"] {
    background-color: transparent;
    border-width: 2px;
}
```

  


最后，我们有了一个漂亮的小按钮切换器，它清晰地显示和宣布其状态，并在你返回时记住你的选择。完成！

## 使用 `color-scheme` 和 `light-dark()` 函数比使用 `@media` 查询更好吗？

  


首先， `@media` （媒体查询）是一个强大的 CSS 特性，我们可以通过它来 **[查询用户系统的偏好设置](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-14)**。例如，`prefers-color-scheme` 可以查询到用户系统的暗黑模式的设置。这是系统级别的暗黑模式切换功能。然而，我们构建 Web 网站或应用的时候，除了要尊重用户系统偏好设置之外，还应该给用户提供更多的选择，而这些选择与他们偏好的设备 UI 是否为暗色或亮色无关。

  


当然，CSS 可以很容易做到这一点，只不过，大多数 Web 开发者习惯了用额外的 CSS 类、复制样式或使用自定义属性来实现这一目标。

  


另外，使用 `color-scheme` 还有多个好处：

  


-   它为你免费提供了基本的单色暗黑模式
-   它可以根据操作系统的模式偏好自动切换模式
-   你可以使用 JavaScript 在亮色和暗色模式之间切换（也可以使用纯 CSS 来切换），`light-dark()` 函数中声明的颜色会跟随切换

  


本质上，我们所做的就是为 `color-scheme` 设置三种选项之一：`light` （亮色）、`dark` （暗色）或 `light dark` （自动）。在实际使用时，我个人建议这三种选择作为独立的选项，便于用户选择，这样可能为你消除一些复杂性！任何新用户都可能处于自动模式，因为接受用户的操作系统设置是最不突兀的默认状态。然后，你可以给用户提供选择，继续使用当前模式或换成其他模式。这样，就不需要去探测用户偏好哪种模式。也不需要在 `prefers-color-scheme` 上保持事件监听，以便应对变化——你在 CSS 中声明的 `color-scheme: light dark` 会为你处理这个问题。

  


另外，借助现代 CSS 的特性，我们可以无需任何 JavaScript 就可以在 CSS 中重置 `color-scheme` 属性的值。但有一些小细节需要留意：

  


-   你不能使用 `<button>` ，只能使用单选按钮或在 `<select>` 中的 `<option>` （提供三种状态选择时），还可以使用复选框（通常是两种状态之间的切换）
-   它只在每个页面上有效，而不是整个网站上有效，这意味着在重新加载或刷新页面时，所做的更改会丢失，因为 CSS 还无法做到状态存储
-   如果你使用 `:has()` 选择器的话，需要考虑它的兼容性；如果不使用 `:has()` 选择器，则需要更严谨的 HTML 结构，而且不能使用 `<select>` 来做判断

  


## 小结

  


最后，在结束这节课时，我要再次强调一下，`color-scheme` 的设置是 `light-dark()` 函数正常工作的首要条件。另外，千万不要让自己陷入老旧的暗黑模式的争论中。不管是使用媒体查询还是 `light-dark()` ，两者都很好，甚至都很棒！它们之间只有适合之分，没有好坏之分！你可以在你下一个项目中试试两者，放开选择的束缚，拥抱多样性。
