作为一名 Web 前端开发者，对于 Pop UI 组件（`<Popover>`）应该不会感到陌生。以往创建 Pop UI 组件可能需要借助 JavaScript 脚本才能实现。现在，浏览器引入了一个名为 Popover API 的功能，使我们可以不使用 JavaScript 的前提下，就可以实现 Pop UI 组件相关的功能。除此之外，它使得我们创建诸如下拉菜单、菜单、工具提示框等组件变得更加容易。

  


在这节课中，我将基于 Web 上一些常见的用例向大家阐述 CSS 与 Popover API 的结合可以做些什么？

  


## Popover API 是什么？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/03d66ad2d1a341da91873f0fcd0e6075~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=34446&e=jpg&b=010101)

  


Popover 是 Web 中常见的一种交互组件，是一种小型覆盖框，当用户点击或悬浮在 Web 页面或应用中的特定元素上时出现。它可以显示额外的内容或信息，而无需将用户重定向到新页面。Popover 可以包含文本、图片、链接以及其他交互元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5b4410abe6fe4f079f09c84ac0a9fe56~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1200&s=861038&e=png&b=f0f2f2)

  


与 Dialogs（模态框）不同，Popover 采用非模态（Non-modal）方式。这意味着当 Popover 出现时，用户仍然可以与 Popover 后面的页面进行交互。

  


Popover 可以用于多种用途，例如：

  


-   **工具提示（Tooltips）：** Popover 通常提供简短且有用的信息，类似于工具提示，但通常更复杂且具有交互性
-   **表单（Forms）：** 它可以包含用户输入的表单，例如登录表单、搜索栏或反馈表单
-   **通知（Notifications）：** Popover 可用于以更显眼的方式显示通知、提示、警报或更新，而不仅仅是简单的文本消息
-   **菜单（Menus）：** 下拉菜单或右键菜单等点击时出现的菜单也可以视为 Popover
-   **附加信息（Additional Details）：** 它们非常适合显示与特定项目相关的附加信息或选项，而不会让主界面显得杂乱无章

  


正如我之前提到的，以往我们要在 Web 上实现一个 Popover，通常需要依赖 JavaScript 来实现。现在，我们可以使用 Popover API 来实现，而且可以不依赖任何 JavScript 脚本。

  


需要知道的是，[Popover API 是由 Open UI ](https://open-ui.org/components/popover.research.explainer/)（隶属于 W3C 的社区组织）提出的浏览器的扩展功能，旨在定义并拓展内置 Web 组件的能力，例如创建重叠的用户界面元素，同时解决与 `z-index` 堆叠、关闭逻辑、不完善的无障碍设计等相关的常见问题。

  


从本质上来说，Popover API 是 **HTML 属性**（例如 `popover`）、**CSS 属性**（例如`:popover-open`）和 **JavaScript API** （例如 `togglePopover()` 方法）的结合体，为 Web 开发者提供了一种灵活且可定制的方式来创建 Popover。

  


## Popover API 的使用

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/253907e9b15d4f119ef851d2048316da~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=675&s=58743&e=jpg&b=e193a5)

  


首先我要告诉大家的是，在这里我并不会对 Popover API 的每个属性展开讨论，你可以在 [Open UI 官方文档](https://open-ui.org/components/popover.research.explainer/)中获取更多的信息。但我会以一个简单的示例让大家快速掌握 Popover API 的相关特性，然后我们再利用这些特性来制作一些 Popover 组件，例如工具提示框（Tooltip）、菜单（Menu）等。

  


假设我们的页面有一个 HTML 元素：

  


```HTML
<div class="target">
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


我们使用之前小册《[CSS 图形：创建工具提示和语音气泡](https://juejin.cn/book/7395119173656903715/section/7408966599622787083)》课程中所介绍的技巧，将 `.target` 美化成一个气泡形状：

  


```CSS
.target {
    --primary: #3f51b5;
    --bg: var(--primary);
    --contrast-target: AA;
    --color-set: #001219, #2196f3, #9c27b0, #ff9800, #9e9e9e, #607d8b, #fff,
      #bbb;
    --color: color-contrast(
      var(--bg) vs var(--color-set) to var(--contrast-target)
    );
    --fs: 1.5em; /* font-size */

    width: fit-content;
    /* display: grid; */
    place-content: center;

    font-size: var(--fs);
    text-align: center;
    padding: 1em;
    place-self: center;
    color: var(--color);

    --s: 1.5em; /* 尾巴尺寸 */
    --p: 48%; /* 尾巴位置 */
    --r: 1em; /* 圆角半径 */

    border: var(--s) solid #0000;
    border-radius: calc(var(--r) + var(--s));
    background: linear-gradient(60deg, var(--bg), var(--bg)) border-box;
    --_m: 100% / var(--s) var(--s) no-repeat;
    --_p: clamp(
      var(--r) + 3 * var(--s) / 2,
      var(--p),
      100% - var(--r) - 3 * var(--s) / 2
    );
    mask: radial-gradient(100% 100% at 100% 100%, #0000 99%, orange 102%)
        calc(var(--_p) + var(--s) / 2) var(--_m),
      radial-gradient(100% 100% at 0 100%, #0000 99%, orange 102%)
        calc(var(--_p) - var(--s) / 2) var(--_m),
      linear-gradient(red 0 0) padding-box;
}
```

  


注意，我把上面代码中的 `display: grid` 先注释掉了。随着你慢慢往下阅读，自然会知道为什么要注释这行 CSS 代码。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/556beed408ff47fa94859c3fa946f96d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=910&s=156948&e=jpg&b=f4f4f4)

  


接下来，我们给 `.target` 元素添加一个 `popover` 属性：

  


```HTML
<div class="target" popover>
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


神奇的事情就发生了，现在页面上什么也看不到了，`.target` 元素就像是消失了一样。你可能会感到困惑——为什么会这样？但打开浏览器开发者工具一看，你会发现浏览器给 `.target` 元素预置了一些 CSS 样式，其中就有 `display: none` 这一条。这就是 `.target` 元素加上 `popover` 属性消失的主要原因，也是要在 `.target` 中注释掉 `display: grid` 的原由：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/875789b941b54446ab7b8d3a42b690a8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3344&h=1308&s=390882&e=jpg&b=f4f4f4)

  


这也是符合 Popover 工作原理的。通常情况之下，Popover 是隐藏不可见的，只有当用户触发的时候才会显示。这意味着，要将任意 HTML 元素创建为 Popover，只需要给该元素设置一个名为 `popover` 属性即可。此时，该元素就会作为一个 Popover 工作。

  


那么，如何让它显示出来呢？你可能想到的是通过 JavaScript 来切换 `.target` 元素的 `display` 属性。嗯，这是最传统的方式。然而，使用 Popover API 是不需要使用 JavaScript 的，我们只需要在锚点元素（即触发 Popover 显示的元素，比如一个 `button` ）使用 `invoketarget` 属性，并将其值与 Popover 元素（`.target`）的 `id` 绑定在一起。

  


```HTML
<button class="anchor" invoketarget="cart" popovertarget="cart">
    <svg  viewBox="0 0 576 512">
        <path fill="currentColor" d="M0 24C0 10.7 10.7 0 24 0L69.5 0c22 0 41.5 12.8 50.6 32l411 0c26.3 0 45.5 25 38.6 50.4l-41 152.3c-8.5 31.4-37 53.3-69.5 53.3l-288.5 0 5.4 28.5c2.2 11.3 12.1 19.5 23.6 19.5L488 336c13.3 0 24 10.7 24 24s-10.7 24-24 24l-288.3 0c-34.6 0-64.3-24.6-70.7-58.5L77.4 54.5c-.7-3.8-4-6.5-7.9-6.5L24 48C10.7 48 0 37.3 0 24zM128 464a48 48 0 1 1 96 0 48 48 0 1 1 -96 0zm336-48a48 48 0 1 1 0 96 48 48 0 1 1 0-96z"/>
    </svg>
</button>

<div class="target" popover id="cart">
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


注意，上面代码中同时设置了 **`invoketarget`** 和 **`popovertarget`**。 这是因为，到目前为止，`popovertarget` 属性的浏览器支持优于 `invoketarget`。`popovertarget` 是 Popover 指定的，提供了一种声明式的方式来切换 Popover 的开与关。不过，一些聪明的开发者意识到，按钮也需要一种声明式方式来打开对话框（例如模态框）并执行其他任务，这就是为什么有两种方式做同一件事。但在未来，`invoketarget` 计划取代 `popovertarget`，最终将导致 `popovertarget` 被弃用。`invoketarget` 还未得到浏览器支持，如果要使用的话，需要依赖相应的 [Polyfill](https://www.npmjs.com/package/invokers-polyfill)。

  


现在，你在浏览器点击 `button` 后，`.target` 元素就会显示，然后点击 `.target` 元素之外的任意地方，它又将会消失：

> Demo 地址：https://codepen.io/airen/full/OPLpwom

  


这个时候，你现次使用浏览器开发者工具对 `.target` 元素进行审查时，你会发现 `[popover]:not(:popover-open):not(dialog[open])` 样式没有了，即 `display: none` 被删除。因此，Popover 就显示了。与此同时，HTML 中会新增一个 `#top-layer` 的 ShadowDOM，表示 Popover 是最顶层（稍后会介绍）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/759209ac284f474cb5b4312ac98173ef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3322&h=1234&s=424658&e=jpg&b=242b2b)

  


Popover 在没有 JavaScript 的情况下工作的确很不错，但使用 JavaScript 切换元素的 `display: none` 从来不是一个难题。然而，Popover 确实带来了更多的功能：

  


-   **轻量级关闭功能**：点击 Popover 外部区域或按下键盘的 `Escape` 键将关闭 Popover
-   **超越** **`z-index`**：Popover 利用了最顶层
-   **将元素动画化进出顶层**：默认情况下，Popover 的动画是简单的淡入和淡出效果，但我们可能使用 CSS 动画来自定义这种行为
-   **焦点管理**：当你打开 Popover 时，下一个 `Tab` 的停靠点将是 Popover 内部的第一个可聚焦元素。如果你在 Popover 内聚焦了一个元素，然后关闭 Popover，焦点将返回到正确的位置（这是使用 JavaScript 时很难做到的）

  


接下来，我们一起来探讨一下 `popover` 自带的一些功能与特性。

  


### 轻量级关闭功能

  


`popover` 属性可以设置为 `auto` 或 `manual` ：

  


-   当 `popover` 设置为 `auto` 时，Popover 具有轻量级关闭功能，即用户点击 Popover 外部区域或按下 `Escape` 键可以轻松关闭 Popover。同时，始终只有一个自动关闭的 Popover 会处于打开状态
-   当 `popover` 设置为 `manual` 时，Popover 不具有轻量级关闭功能，Popover 必须通过再次点击按钮显式关闭（或者通过在 JavaScript 中调用 `hidePopover()` 方法），而且按下 `Escape` 键也不会关闭 Popover。同时，多个手动关闭的 Popover 可以同时打开

  


例如，我们将上面示例的 `popover` 属性设置为 `manual` ：

  


```HTML
<button class="anchor" invoketarget="cart" popovertarget="cart">
    <svg  viewBox="0 0 576 512">
        <path fill="currentColor" d="M0 24C0 10.7 10.7 0 24 0L69.5 0c22 0 41.5 12.8 50.6 32l411 0c26.3 0 45.5 25 38.6 50.4l-41 152.3c-8.5 31.4-37 53.3-69.5 53.3l-288.5 0 5.4 28.5c2.2 11.3 12.1 19.5 23.6 19.5L488 336c13.3 0 24 10.7 24 24s-10.7 24-24 24l-288.3 0c-34.6 0-64.3-24.6-70.7-58.5L77.4 54.5c-.7-3.8-4-6.5-7.9-6.5L24 48C10.7 48 0 37.3 0 24zM128 464a48 48 0 1 1 96 0 48 48 0 1 1 -96 0zm336-48a48 48 0 1 1 0 96 48 48 0 1 1 0-96z"/>
    </svg>
</button>

<div class="target" popover="manual" id="cart">
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


这个时候你点击 `button` 能打开 Popover，但你点击 Popover 外部区域或按下 `Escape` 键将无法关闭 Popover ；你必须再次点击按钮本或通过在 JavaScript 中调用 `hidePopover()` 方法，才能关闭 Popover：

  


```JavaScript
document.querySelector(".close").addEventListener("click", event => {
    document.querySelector("[popover]").hidePopover();
});
```

### 声明式触发器

  


默认情况下，用户点按钮会切换 Popover 的显式和隐藏状态，这是一个常邮的设计模式。为了简化这一模式，并避免在这种常见场景中使用 JavaScript ，可以通过 `popovertarget` 和 `popovertargetaction` 两个属性声明性地切换、显示或隐藏 Popover:

  


```HTML
<button class="anchor" invoketarget="cart" popovertarget="cart">Toggle Popover</button>
<button class="anchor" invoketarget="cart" popovertarget="cart" popovertargetaction="show">Show Popover</button>
<button class="anchor" invoketarget="cart" popovertarget="cart" popovertargetaction="hide">Hide Popover</button>

<div class="target" popover id="cart">
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


-   当点击第一个 `button` 时，如果 `.target` 处于隐藏状态，浏览器会调用 `.showPopover()` 显示它；如果已显示，则调用 `.hidePopover()` 关闭它
-   当点击第二个 `button` 时，如果 `.target` 处于隐藏状态，浏览器会调用 `.showPopover()` 显示它；如果它已显示，则什么都不做
-   当点击第三个 `button` 时，如果 `.target` 处于显示状态，浏览器会调用 `.hidePopover()` 关闭它；如果它已隐藏，则什么都不做

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3c790afae5de454da4caff0c5f309ab9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=686&s=521356&e=gif&f=265&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/WbejNZO

  


注意：`popovertargetaction` 的默认值是 `"toggle"`。上面代码中第一个 `button` 元素并没有显式设置该属性，你也可以显式设置 `popovertargetaction="toggle"` 。

  


之前提到过，`popovertarget` 属性将来是会被 `invoketarget` 属性替代的。但与 `invoketarget` 属性对应的有一个名为 `invokeaction` 属性（它会不会替代 `popovertargetaction` 属性，我目前还没查阅到相关文档）。`invokeaction` 属性同样提供了一些不同的值，对应着不同的操作：

  


-   `showpopover` ：显示一个 Popover
-   `hidepopover` ：关闭一个 Popover
-   `showmodal` ：打开一个对话框元素（`<dialog>`），作为模态对话框（Modal）
-   `close` ：关闭一个对话框元素

  


如果省略 `invokeaction` 属性，默认行为取决于上下文：

  


-   如果 `invokeaction` 设置的目标是一个 Popover ，它将调用 `.togglePopover()`
-   如果 `invokeaction` 设置的目标是一个对话框，它将在对话框关闭时调用 `showModal()` ，如果对话框已经打开，则关闭该对话框

  


我们以对话框为例：

  


```HTML
<button invoketarget="dialog">打开对话框</button>
<dialog id="dialog">
    对话框内容放在这里。
    <button invoketarget="dialog" invokeaction="close">关闭对话框</button>
</dialog>
```

  


> 温馨提示，如果你对 `<dialog>` 相关的知识感兴趣，请移步阅读《[用于美化模态框的 `:modal` 和 `::backdrop`](https://juejin.cn/book/7223230325122400288/section/7229323379793199104)》！

  


除了内容的操作，我们还可以编写自定义操作。这超出了这节课的范畴，因为自定义操作可以做任何事情——它不一定与对话框或 Popover 相关。

  


虽然触发器可以使我们避免使用 JavaScript，但它们也提供了一个新的 JavaScript 调用事件，如果你需要比默认行为更多的功能。这一事件会在 Popover 或对话框上触发，而不是按钮上。

  


```JavaScript
const popover = document.querySelector("[popover]");
popover.addEventListener("invoke", etv => {
    console.log(etv.action);
    console.log(etv.invoker);
    // 在这里做一些有用的事情...
})
```

  


在事件处理器中，你可以通过 `event.invoker` 获取触发调用的按钮的引用，并通过 `event.action` 确定指定的操作。

  


### Popover 方法和事件

  


通常情况之下，Popover 并不需要 JavaScript。不过，在一些场景中（比如我们想显示一个 Toast 通知），用户不需要与按钮互动，那应该如何处理呢？

  


就类似情景而言，我们可以通过 Popover API 提供的一些方法和事件来实现，例如：

  


-   `showPopover()` ：显示 Popover
-   `hidePopover()` ：关闭 Popover
-   `togglePopover()`：Popover 在显示和关闭之间切换

  


我们以 Toast 通知为例，它并不需要用户做任何操作就会自动消失，这非常适用于 Popover 没有单独的打开或关闭事件的情景：

  


```HTML
<div class="toast" id="toast" popover="manual">
    Hello I am a toast notification
</div>
```

  


我们完全可以借助 Popover 相关事件处理。在调用 `hidePopover()` 之前，先检查 Popover 是否已经隐藏（关闭）。在 JavaScript 中，我们可以使用 `.matches(':popover-open')`、`.checkVisibility()` 或 `event.newState === 'open'`，这些方法都会在 Popover 打开时返回 `true`。具体代码如下：

  


```JavaScript
const toast = document.querySelector("#toast");  

// 显示弹窗
setTimeout(() => toast.showPopover(), 4000);

// 处理 toggle 事件
toast.addEventListener('toggle', (event) => {
    if (!toast.matches(':popover-open')) return;

    // 使用 requestAnimationFrame + setTimeout 确保流畅的动画
    const hideAfter = 5000;
    const start = performance.now();

    const hide = (timestamp) => {
        if (timestamp - start >= hideAfter) {
            toast.hidePopover();
        } else {
            requestAnimationFrame(hide);
        }
    };

    requestAnimationFrame(hide);
});
```

还有一个 `beforetoggle` 事件，类似于 `toggel` 事件，但允许你在事件处理器中调用 `event.preventDefault()` ，这在处理动画时可能会非常有用。相比之下，`toggle` 事件是不可取消的。

  


```JavaScript
const toast = document.querySelector("#toast");  

// 定义定时器变量

// 显示弹窗的定时器

// 监听 beforetoggle 事件

    const isOpening = !toast.matches(':popover-open');

    // 根据状态调用阻止逻辑

        console.log(isOpening ? 'Opening prevented!' : 'Closing prevented!');
        event.preventDefault();
    }

    // 清除隐藏定时器（避免资源浪费）
    if (!isOpening) {
        clearTimeout(hideTimer);
    }
});

// 监听 toggle 事件
toast.addEventListener('toggle', () => {
    if (toast.matches(':popover-open')) {
        // 设置 5 秒后隐藏的定时器
        hideTimer = setTimeout(() => toast.hidePopover(), 5000);
    }
});

// 判断是否阻止打开弹窗

    // 自定义打开条件（返回 true 阻止打开）
    return false; // 替换为实际条件

    // 自定义关闭条件（返回 true 阻止关闭）
    return false; // 替换为实际条件
}
```

### 超越 `z-index`：顶层

  


一些 JavaScript 框架（比如 React）提供了类似“Portal”的概念，用于渲染像工具提示（`Tooltip`）和对话框（`Modal`）这样的内容。但是，往往很多时候 “Portal”是很难使用的。正如 [React 文档](https://react.dev/reference/react-dom/createPortal#rendering-to-a-different-part-of-the-dom)所述：

  


> “Portal”允许组件将部分子元素渲染到 DOM 中的不同位置。这使得组件的一部分可以“脱离”它可能所在的容器。例如，一个组件可以显示一个模态对话框（`Modal`）或工具提示（`Tooltip`），它们会出现在页面的其他部分之上或外面。你可以使用“Portal”创建一个浮动在页面其他内容之上的模态对话框，即使调该对话框的组件位于 `overflow: hidden` 的容器内。

  


然而，当使用 `<dialog>` 元素（而不是通过 `<div>` 元素构建）或 `popover` 属性时，你完全可以避免这个问题——不需要“Portl”。它们在 DOM 中的位置并不重要。通常，将 `popover` 或 `<dialog>` 的标记与触发它的按钮放在一起是很方便，它们可以在 HTML 中的任何位置，不会被父元素的 `overflow: hidden` 裁剪。它们利用了“顶层”层级，这是一个原生的 Web 解决方案，用于渲染在文档其余部分之上的内容。顶层位于文档之上，总是优先于 `z-index` 。要是你使用浏览器开发者工具审查一个 Popover ，当它显示的时候，浏览器会自动创建一个名为 `top-layer` 的 Shadow DOM，并且位于最顶层，即便你使用 `z-index` 控制页面中的任一元素的层级，它始终无法超越 Popover。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/484f6d382db0477db39e7a5fa85b9865~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3326&h=1190&s=291884&e=jpg&b=f4f4f4)

  


正如之前的示例所示，位于顶层的元素可以通过 `::backdrop` 样式化处理：

  


```CSS
::backdrop {
    backdrop-filter: blur(5px);
}
```

  


如此一来，Popover 显示时，其之外的元素都会变得更模糊：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de4583cffed34641bc936833d39c9ab6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=938&s=182362&e=jpg&b=f4f4f4)

  


如果你足够仔细的话，你会发现我们之前的案例都没有给 Popover 设置 `z-index` 属性的值。想象一下，以往我们的开发过程是，是不是必须给 Popover 或 Dialog 设置一个较大值的 `z-index` 。这意味着，使用 `popover` 创建的 Popover 或模态框，我们无需它们的 `z-index` 会比 Web 页面上的其他元素更低，因为它们已是最顶层，不会被任何人超越。

  


### 给 Popover 添加动画

  


默认情况下，当 Popover 被打开时，它会立刻显示出来。你可能希望给 Popover 显示时添加一个自定义的动画效果，比如 `zoomIn` 和 `zoomOut` 。我们可以通过 `@starting-style` 和 `:popover-open` 伪类给 Popover 添加自定义动画效果。

  


```CSS
[popover]{
    /* CSS... */
    
    /* 打开状态   */
    &:popover-open {
        opacity: 1;
        scale: 1;
        transition: 
            scale 0.5s, 
            opacity 0.5s, 
            display 0.5s allow-discrete,
            overlay 0.5s allow-discrete;

         /*   淡入  */
        @starting-style {
            opacity: 0;
            scale: 0;
        }
    }

    /*  淡出  */
    &:not(:popover-open) {
        scale: 0;
        transition: 
            scale 0.3s, 
            display 0.3s allow-discrete,
            overlay 0.3s allow-discrete;
    }
}
```

其中 `@starting-style` 是 CSS 的一个新特性，它可以通过 [CSS 过渡](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)（`transition`）将元素动画化进入视图（在使用 `@keyframes` 时不需要 `@starting-style`）。`@starting-style` 既适用于向 DOM 中添加新元素，也适用于已经存在 DOM 中的元素，且其 `display` 属性从 `none` 过渡到其他值（比如 `grid`）。

  


另外，当元素进出顶层时，`overlay` 过渡是必要的代码。`overlay` 属性纯粹是为了这种用例添加到 CSS 中的，别无其他实际用途。它是一个不寻常的属性，因为在过渡之外，只有浏览器可以指定该属性——你不能通过自己的 CSS 设置它。默认情况下，当 `dialog` 或 `popover` 关闭时，它们会立即从顶层移除，这会导致元素被裁剪和遮挡。通过过渡 `overlay` ，元素会一直保留在顶层，直到过渡完成。

  


[`transition-behavior` 也是一个新的 CSS 属性](https://juejin.cn/book/7288940354408022074/section/7292735608995184678#heading-5)，它的值可以为 `normal` 或 `allow-discrete` 。在上面的示例代码中，我使用了简写形式。

  


```CSS
[popover]:open-popover {
    transition: display 0.5s allow-discrete;
}

/* 等同于 */
[popover]:open-popover {
    transition-name: display;
    transition-duration: .5s;
    transition-behavior: allow-discrete;
}
```

## HTML 的 `<dialog>` 元素与 `popover` 有什么区别？

  


现如今，我们可以通过以下三种方式，在不依赖任何 JavaScript 脚本或库的前提下对 Web 内容的显式和隐藏进行切换：

  


-   HTML 的 `<details>` 和 `<summary>` ，可用它们来制作手风琴组件（`Accordion`）
-   HTML 的 `<dialog>` ，常用于制作模态框组件（`Modal`）
-   HTML 的 `popover` 属性，常用于制作工具提示框组件（`Tooltip`）、弹窗组件（`Popover`）、通知（`Notification`、`Toast` 和 `Alert` ）和菜单（`Menu`）等

  


其中 `<dialog>` 和 `popover` 更为相似，都会弹出一个元素，前者采用的是模态对话框的方式（Modal），后者采用的是非模态方式（Non-modal）。除此之外，它们两者有很多区别。

  


首先，最基本的区别就是**具有不同的 HTML 结构**。对话框（Dialog）是一个独立的元素，例如：

  


```HTML
<button invoketarget="dialog">Open Dialog</button>

<dialog id="dialog">
    Dialog content goes here.
    <button invoketarget="dialog" invokeaction="close">Close dialog</button>
</dialog>
```

  


而 Popover 是一个附加在其他元素上的属性，例如：

  


```HTML
<button class="anchor" invoketarget="cart" popovertarget="cart">Toggle Popover</button>

<div class="target" popover id="cart">
    🛒 Your cart is full of soft tacos 🌮 🌮 🌮 🌮 🌮 
</div>
```

  


默认情况之下，`<dialog>` 和 `popover` 都是隐藏的，不会显示在页面上，也不会出现在辅助技术（如屏幕阅读器）的可访问性树中，但它们仍在 DOM 中。不过，我们可以通过给 `<dialog>` 元素添加一个 `open` 属性，使 `<dialog>` 默认设置为可见状态：

  


```HTML
<button invoketarget="dialog">Open Dialog</button>

<dialog id="dialog" open>
    Dialog content goes here.
    <button invoketarget="dialog" invokeaction="close">Close dialog</button>
</dialog>
```

  


但你无法仅通过 HTML ，让一个 Popover 默认为可见状态。

  


正如之前的 Popover 相关的案例所示，可以不依赖任何 JavaScript 脚本，仅通过 HTML 触发器让一个 Popover 正常工作——**实现打开和关闭**。然而，`<dialog>` 是无法仅通过 HTML 触发器来实现打开和关闭的功能，需要依赖 JavaScript 事件理器来实现它的打开和关闭功能。

  


`<dialog>` 和 `popover` 第二个不同之处是 JavaScript API。`<dialog>` 有两种截然不同的打开方式：

  


-   `show()` ：以非模态状态打开对话框，即没有背景遮罩，也不会限制焦点在对话框内。需要注意的是，通过 `<dialog>` 元素的 `open` 属性来打开对话框的效果也是非模态的
-   `showModal()` ：以模态状态打开对话框，这种情况下会显示背景遮罩，并将焦点限制在对话框内。需要注意的是，如果对话框已经以非模态状态打开（`show()` 方法打开或 `open` 属性），再次调用 `showModal()` 方法会抛出异常

  


与此同时，`<dialog>` 提供一个 `close()` 方法，用于关闭对话框（如果它当前是打开的）。

  


Popover 也有相应的 JavaScript API ，但其打开和关闭方法与模态对话框不同，且没有交集。这些方法相对直观易懂：

  


-   `.showPopover()` ：打开弹出框
-   `.hidePopover()` ：关闭弹出框
-   `.togglePopover()` ：打开和关闭弹出框，如果弹出框已打开，它会关闭弹出框；反之会打开弹出框

  


与 `<dialog>` 不同之处是，Popover 重复调用这些方法不会抛出错误。

  


另外一个比较明显的区别是 Popover 具有轻量级关闭功能，这是 Dialog 不具有的。默认情况之下，用户可以通过“点击 Popover 外部区域” 或按 `Escape` 键就可以关闭 Popover。需要注意的是，如果 `popover` 的值为 `manual` ，那么点击 Popover 外部区域或按 `Escape` 键就不能关闭 Popover。

这里应用了三个锚点（`<button class="anchor"`），它们使用 `invoketarget` （和 `popovertarget`）属性值与目标元素 `<div class="target">` 的 `id` 值相对应，并且在 `.target` 元素上设置了 `popover` （即 `popover="auto"`）。

  


这个时候，你点击每个 `button` 时，对应的 Popover （`.target`）就会打开，点击Popover 之外或按下 `Esapse` 键，它就会关闭：

不难发现，现在 Popover 的位置不是我们所想要的，我们可以使用 CSS 锚点定位来解决这个问题。由于我们有三个锚点（`button`），因此通过内联的方式来声明锚点名称：

  



这种方式非常适用于我们这个情景——当你有很多个锚点时。如此一来，我们就可以通过 `anchor-name` 和 `position-anchor` 引用已定义好的锚点名称，与此同时使用 `position-area` 调整 Popover 偏移位置：

  


```CSS
[popovertarget] {
    anchor-name: var(--anchor);
}

[popover] {
    position: absolute;
    inset: unset;
    position-anchor: var(--anchor);
    position-area:  end span-all;
    margin-top: 1rem;
    
    &:popover-open {
      display: grid;
      overflow: unset;
    }
}
```

  


这个时候，你会发现每个 Popover 打开的位置与锚点的位置相对应：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ee3a8fe4b7ed489fa2a0b55a834b3680~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1292&h=788&s=2063993&e=gif&f=127&b=f3f3f3)

在此基础上，我们还可以给 Popover 添加一些过渡动效：

  


```CSS
[popover] {

        }
    }

    /*  淡出  */
    &:not(:popover-open) {
        scale: 0;
        transition: 
            scale 0.3s,

    }
}

}

    [popover]:popover-open::backdrop {
        background-color: rgb(0 0 0 / 0%);
    }
}
```
