            <details open>

## 移除 `<details>` 元素的默认样式

  


你可能已经注意到了，手风琴的每个条目的标题（`<summary>`）前都有一个小三角形，而且它会随着条目展开和折叠的状态改变方向。这就是 `<details>` 元素默认的样式，这些默认的样式可以允许我们快速构建一个粗造的手风琴组件，但我们现在的目标是要做一个精致的，优美的手风琴。这意味着，我们首先要移除 `<details>` 元素的默认样式。

  


`<details>` 元素的默认样式（也称为“用户代理样式表”，即浏览器的默认样式）与表单控件（比如 `<input>`）的默认样式类似，不同浏览器厂商定义的默认样式和实现方式不一样:

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/66cb49c42b5346659c423c8ba4764477~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2381&h=835&s=513608&e=png&b=f6fcfd)

  


在 Chrome、Edge 和 Firefox 浏览器中，`<summary>` 默认是 `display: list-item` ，就像 `<li>` 元素那样，因此会带有一个可样式化的 `::marker` 伪元素，通常通过 `list-style` 控制，比如 `list-style-type: disclosure-open` （小三角形）。我们可以通过以下两种方式之一来移除三角形：

  


-   设置 `list-style: none` 来移除所有 `::marker`
-   或者将 `display` 改成不是 `list-item` 的值，比如 `flex` 或 `grid` 等，从而彻底跳过列表样式的渲染

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2dd5d37042e1421ab184dc5a580efaca~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3346&h=1606&s=968991&e=jpg&b=f2fbfb)

  


在 Safari 浏览器中，它之前的实现方式完全不同，它使用了内部伪元素 `::-webkit-details-marker` 来显示三角形。因此，你需要使用这个选择器来将三角形隐藏（考虑老版本的 Safari）。庆幸的是，从现在开始，浏览器厂商已经统一了 `<details>` 的默认样式，Safari 新版本也是使用 `::marker` 元素定义三角形。这意味着，我们可以像下面这样统一移除三角形：

  


```CSS
summary {
    display: flex; 
    
    /* 如果不需要考虑老版本的 Safari，这行样式可以删除 */
    &::-webkit-details-marker {
        display: none;
    }
}
```

  


现在，呈现给你的手风琴已经看不到三角形标记了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ee87b9d2604d470b830bda64b99ad195~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1238&s=1463949&e=png&b=121122)

  


移除三角形标记之后，我们需要为手风琴条目添加其他的标记，这样用户能更好的理解当前条目处于哪种状态。比如，我们可以使用 SVG 绘制加号（`+` 表示折叠状态）和减号（`-` 表示展开状态），当然你也可以使用 CSS 伪元素 `::before` 和 `::after` 来绘制。我在这里使用 `::after` 和 [CSS 的 `clip-path`](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-1) 绘制了加减号标记：

  


```CSS
summary {
    --b: .25rem;
    --s: 1rem;
    --c: #a371f7;
    display: flex;
    align-items: center;
    justify-content: space-between;
    
    &::after {
        content: "";
        width: var(--s);
        aspect-ratio: 1;
        background: var(--c);
        /* 折叠状态是 + 号 */
        clip-path: polygon(calc(50% - var(--b)/ 2) 0%, calc(50% + var(--b)/2) 0%, calc(50% + var(--b)/2) calc(50% - var(--b)/2), 100% calc(50% - var(--b)/2), 100% calc(50% + var(--b)/2), calc(50% + var(--b)/2) calc(50% + var(--b)/2), calc(50% + var(--b)/2) 100%, calc(50% - var(--b)/2) 100%, calc(50% - var(--b)/2) calc(50% + var(--b)/2), 0% calc(50% + var(--b)/2), 0% calc(50% - var(--b)/2), calc(50% - var(--b)/2) calc(50% - var(--b)/2));
    }
}

/* 展开状态是 - 号 */
[open] summary::after {
    clip-path: polygon(0% calc(50% - var(--b)/2), 100% calc(50% - var(--b)/2), 100% calc(50% + var(--b)/2), 0% calc(50% + var(--b)/2));
}
```

  


你可以通过调整 CSS 变量 `--b` 、`--s` 和 `--c` 获取不同尺寸和颜色的标记：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9834efe291964aae8efebe62f4c6d28f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1238&s=1446898&e=png&b=121122)

  


是不是看上去要更美观多了！

  


## 给手风琴添加“互斥”效果

  


要让多个 `<details>` 元素在同一时间只展开一个，其实非常简单——只需要给它们设置相同的 `name` 属性即可：

  


```HTML
<div class="accordion--container">
    <div class="accordion">
        <div class="accordion__details">
            <details open name="accordion-name">
                <summary>Automate any workflow</summary>
                <p>Optimize your process with simple and secured CI/CD.</p>
            </details>
            <details name="accordion-name">
                <summary>Get up and running in seconds</summary>
                <p>Start building instantly with a comprehensive dev environment in the cloud.</p>
            </details>
            <!-- 省略其他 details -->
        </div>
        <div class="accordion__thumbnail">
            <img src="https://picsum.photos/id/215/1000/1000" alt="" />
            <img src="https://picsum.photos/id/251/1000/1000" alt="" />
            <!-- 省略其他 img -->
        </div>
    </div>
</div>
```

  


没错，就是这么简单！

  


当多个 `<details>` 元素具有上同的 `name` 值时，它会自动形成一个“互斥组”——你打开一个，其他的就会自动关闭。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4800c340df984e30a739cb5c161ee60b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1332&h=610&s=4126062&e=gif&f=121&b=0f0f1c)

  


通过这些细节上的调整，我们就实现了手风琴组件的核心功能。再加上一些样式之后，它看起来就越来越像一个正式组件了！

  


## 给手风琴组件添加动画效果

  


到目前为止，手风琴条目的内容切换（折叠和展开）时有点生硬——切换内容没有任何动画或过渡效果。那么，如何让手风琴内容“平滑的”展开呢？

  


这个时候，就该祭出我们的 CSS 魔法工具箱了！

  


### 可切换的 `details` 区域：`::details-content`

  


要为手风琴内容添加动画，第一就就是：**选中需要动画的内容部分**。

  


很多人第一时间会想到：把 `<summary>` 和其他内容包裹进一个 `<div>` ，然后通过控制这个 `<div>` 容器的样式实现动画。但这种方式常常会带来各种 CSS 问题，比如不必要的嵌套、难以处理的间距等等。

  


庆幸的是，浏览器引入了一个新特性，用来避免这些额外的包装元素 —— `::details-content`。这个新的伪元素允许我们**选择** **`<details>`** **中除了** **`<summary>`** **元素之外的所有内容**，也就是我们想要添加动画的那一部分。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7f68f8c7f7b94b00aeef6d7327819c2c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3342&h=1386&s=814781&e=jpg&b=f3fcfc)

  


后续我们就可以用它来添加滑动画了—— 只需要结合 `transition` 、`interpolate-size` 和 `transition-behavior` 等现代 CSS 特性即可。在开始之前，我们来测试一下 `::details-content` 是否生效。比如，我们给 `<summary>` 之外的其他内容添加一个渐变背景：

  


```CSS
details {
    /* CSS */
    &::details-content {
        background: linear-gradient(135deg, #90a, #0fa);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9bd88e89b2314dccb626a6a491515424~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=604&s=4281942&e=gif&f=115&b=0f0f1c)

  


事实告诉我们，`::details-content` 是有效的，它会让展开的内容有一个渐变背景。不过，给 `::details-content` 设置背景不是我们的目的，我们真正感兴趣的是：**通过** **`::details-content`** **给内容展开时添加滑动动效**。

  


### 使用 `interpolate-size` 动画化高度

  


现在我们已经可以选中 `<summary>` 后面的隐藏内容了，接下来直觉上可能会尝试使用 `overflow: hidden` 和 `height` 做动画：

  


```CSS
details {
    /* CSS ... */
    
    &::details-content {
        overflow: hidden;
        height: 0;
        transition: height .3s;
    }
    &[open]::details-content {
        height: auto;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/32b526120c4a45fe94ea8df2df549d9f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=606&s=2919804&e=gif&f=86&b=0f0f1c)

  


事实上，上面这段 CSS 代码并没有生效。这是因为，`auto 是一个内在尺寸`，在 CSS 中，无法从一个明确的长度值（`<length-percentage>`），比如示例中的 `0` 过渡到一个内在尺寸值（比如 `auto` 、`min-content` 、`max-content` 或 `fit-content`）——[这是传统 CSS 动画的限制](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)！

  


庆幸的是，你现在可以通过[设置 `interpolate-size` 属性的值为 `allow-keywords`](https://juejin.cn/book/7395119173656903715/section/7490004064970342410#heading-2) ，让浏览器就会自动在关键词之间进行插值计算，比如从固值 `0px` 过渡到 `auto` 值：

  


```CSS
:root {
    interpolate-size: allow-keywords;
}
```

  


设置在 `:root` 上可以让整个页面默认继承这个新行为。试试看，现在 `<details>` 元素展开时，就能看到非常流畅的滑动动画了！

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4d9d996eabe74af6ba3601b2ad5c2bf2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1328&h=604&s=4593588&e=gif&f=140&b=0f0f1c)

  


除了使用 `interpolate-size` 动画化高度（`height`）之外，我们还可以使用 CSS 新增的 `calc-size() 函数`。只需要在展开状态将 `height` 的值设置为 `calc-size(auto,size)` 即，它同样能使 `height` 的固定值（比如 `0px`）平滑过渡到内在尺寸值（比如 `auto`）：

  


```CSS
details {
    /* CSS ... */

    &::details-content {
        overflow: hidden;
        height: 0;
        transition: height 0.3s;
    }
    
    &[open]::details-content {
        height: calc-size(auto,size);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/437d88ec85f9480b95fc5a8b92739325~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1328&h=604&s=4593588&e=gif&f=140&b=0f0f1c)

  


两种方式所达到的效果是一样的。需要知道的是，虽然它们都可以使固定值平滑过渡到内在尺寸值，但两者在使用上还是有一些差异的，有关于这方面更详细的介绍，请移步阅读《[Web UI：CSS `calc-size()` 函数与内在尺寸](https://juejin.cn/book/7395119173656903715/section/7490004064970342410)》！

  


你可能已经注意到了，手风琴条目展开时有非常流畅的滑动动画了，但折叠时动画还不太完美。接下来我们就该解决手风琴条目折叠时的动画问题了。

  


### 使用 `transition-behavior` 动画化离散属性

  


当你使用浏览器开发者工具审查 `<details>` 元素的切换行为时，你会发现折叠状态下的 `::details-content` 被自动添加了 `content-visibility: hidden` :

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/567b1116c553413bb9d63cd42604c72a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1662&h=870&s=10241976&e=gif&f=208&b=eaf5f8)

  


这就解释了为什么我们看不到折叠时的滑动动画。

  


每当我们折叠 `<details>` 元素时，`::details-content` 就会自动应用 `content-visibility: hidden` 。这个属性会立即让任何可见的视觉效果消失，因为当值从 `visible` 变成 `hidden` 时，已经没有内容可以进行过渡动画了，元素会立刻被隐藏。

  


如果你曾尝试制作“退出动画”，却发现因为 `display: none` 会立即隐藏元素而无法实现，那你遇到的正是同样的问题。那问题就来了——浏览器该如何过渡那些“离散属性值”，比如 `hidden` 、`visible` 呢？

  


默认情况下，直接切换这些值时，浏览器会忽略你定义的 `transition` 。这也是为什么 `display` 从 `block` 过渡到 `none` ，或者 `content-visibility` 从 `visible` 过渡到 `hidden` 不会有过渡动画效果，因为它们会立即生效。

  


幸运的是，CSS 提供了 `transition-behavior` ，它将改变这种行为。我们只需要将 `transition-behavior` 设置为 `allow-discrete` ，浏览器就会启用离散动画，对应的离散属性值不会立即变化，而是会在过渡过程的 `50%` 时点才切换。对于像 `display` 和 `content-visibility` 这种特殊情况，属性值会在整个过渡结束后才生效。这样就能让我们先过渡其他属性，再在动画结束时设置 `display:none` 或者 `content-visibility: hidden` 。如此一来，我们可以这样编写 CSS：

  


```CSS
details {
    /* CSS ... */

    &::details-content {
        overflow: hidden;
        height: 0;
        transition: height 0.3s, content-visibility 0.3s;
        transition-behavior: allow-discrete;
    }
    
    &[open]::details-content {
        height: calc-size(auto,size);
    }
}
```

  


在我们这个示例中，启用 `transition-behavior` 后，`::details-content` 会等到指定的渡（`0.3s`）结束后，才切换到 `content-visibility: hidden` 。现在你来看看效果吧，是不是很棒：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0acf5d28bb624aa7bd5374ae3212a362~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1330&h=602&s=5081799&e=gif&f=139&b=0f0f1c)

  


现在，手风琴组件除了功能齐全之外，还具有优雅丝滑般的动画效果。这离我们的目标效果越来越近了！

  


## 使用 `:has()` 选择器响应 DOM 状态变化

  


现在，我们来实现手风琴组件右侧图片的展示功能。之前提到过，在我们这个手风琴组件中，`<details>` 元素和 `<img>` 元素数量应该保持一致。其中主要问题就是，我们是否可以根据当前展开的 `<details>` 项目来显示对应的图片？

  


答案是肯定的！这也正是 `:has()` 选择器大显身手的时候。如果你对 CSS 的 `:has()` 选择不是太了解的话，那么我强烈建议你花点时间阅读下面这些课程，这样你能彻底的掌握 `:has()` 选择器，以及它能做什么：

  


-   [CSS 的父选择器：`:has()`](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)
-   [CSS 选择器 `:has()` 能解决什么问题？](https://juejin.cn/book/7223230325122400288/section/7224404685799555111)
-   [CSS 选择器：`:has()` 与 `:not()` 的组合](https://juejin.cn/book/7223230325122400288/section/7226251495276609569)
-   [Web UI：使用 `:has()` 增强页面交互效果（Part1）](https://juejin.cn/book/7395119173656903715/section/7445262185204285491)
-   [Web UI：使用 `:has()` 增强页面交互效果（Part2）](https://juejin.cn/book/7395119173656903715/section/7445258672210788403)
-   [Web UI：根据控件状态调整 UI 效果](https://juejin.cn/book/7395119173656903715/section/7445266944741097481)

  


有了这个基础之后，我们编写 CSS 代码就要容易得多。

  


首先通过 `opacity: 0` 将所有图片隐藏起来，同时使用 `scale` 将图片稍微放大一点，让它们在变得可见时有一个柔和的过渡效果：

  


```CSS
.accordion {
    /* CSS ... */
    img {
        opacity: 0;
        scale: 1.1;
    }
}
```

  


这个时候所有图片都不可见，并且被放大了 `1.1` 倍：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dc5db9dedcca465e94a202dcdc846923~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1238&s=130539&e=png&b=121122)

  


接下来，使用 `:has()` 选择器，并根据当前 `<details>` 元素的 `open` 属性来判断对应图片是否应该显示：

  


```CSS
.accordion {
    img {
        opacity: 0;
        scale: 1.1;
    }
    
    &:has(details:nth-child(1)[open]) img:nth-child(1) {
        opacity: 1;
        scale: 1;
    }
}
```

  


这个选择器有点绕，我们来逐步拆解：

  


-   `details:nth-child(1)[open]`：选中第一个并且是展开状态的 `<details>`
-   `:has(...)`：如果 `.accordion` 内部满足括号里的条件（即有第一个展开的 `<details>`），那么这个选择器匹配 `.accordion`
-   `img:nth-child(1)`：对 `.accordion` 内部的第一个 `<img>` 应用样式

  


也就是说：**如果第一个** **`<details>`** **被展开，就让第一个图片显现出来。**

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c76afef1b4734bf3904b7094f3a674dc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1238&s=801510&e=png&b=121122)

  


按照同样的方式，扩展到所有项：

  


```CSS
.accordion {
    img {
        opacity: 0;
        scale: 1.1;
    }
    
    :is(
        &:has(details:nth-child(1)[open]) img:nth-child(1),
        &:has(details:nth-child(2)[open]) img:nth-child(2),
        &:has(details:nth-child(3)[open]) img:nth-child(3),
        &:has(details:nth-child(4)[open]) img:nth-child(4),
        &:has(details:nth-child(5)[open]) img:nth-child(5) 
    ) {
        opacity: 1;
        scale: 1;
    }
}
```

  


现在每个 `<details>` 被展开就会显示对应位置的 `<img>` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7c3bc60366574f8181ff5099558663f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=606&s=4874618&e=gif&f=92&b=0f0f1c)

  


同样的，我们可以给图片的显示添加过渡动画效果：

  


```CSS
.accordion {
    img {
        opacity: 0;
        scale: 1.1;
        transition: scale 0.3s, opacity 0.3s;
    }
    
    :is(
        &:has(details:nth-child(1)[open]) img:nth-child(1),
        &:has(details:nth-child(2)[open]) img:nth-child(2),
        &:has(details:nth-child(3)[open]) img:nth-child(3),
        &:has(details:nth-child(4)[open]) img:nth-child(4),
        &:has(details:nth-child(5)[open]) img:nth-child(5) 
    ) {
        opacity: 1;
        scale: 1;
    }
}
```

  


打开不同的 `<details>` 项时，对应图片会以淡入淡出和缩放的动画方式显现出来：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2674b8be53494eaea7d7d2bceb296efb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=610&s=8695078&e=gif&f=127&b=0f0f1c)

  


现在我们差不多完成了，只有一个小功能缺失。你可能已经注意到了：当所有的 `<details>` 元素都关闭时，所有的图片都会被隐藏，遗憾的是，我们无法阻止 `<details>` 元素被关闭。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bff7be1fb37d4d979cff9b705b000112~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1328&h=606&s=7446007&e=gif&f=128&b=0f0f1c)

  


为了解决这个问题，我们可以在图片容器的最后添加一个 `.fallback` 图片：

  


```HTML
<div class="accordion--container">
    <div class="accordion">
        <div class="accordion__details">
            <details open>
                <summary>Automate any workflow</summary>
                <p>Optimize your process with simple and secured CI/CD.</p>
            </details>
            <details>
                <summary>Get up and running in seconds</summary>
                <p>Start building instantly with a comprehensive dev environment in the cloud.</p>
            </details>
            <!-- 省略其他 details -->
        </div>
        <div class="accordion__thumbnail">
            <img src="https://picsum.photos/id/215/1000/1000" alt="" />
            <img src="https://picsum.photos/id/251/1000/1000" alt="" />
            <!-- 省略其他 img -->
            
            <!-- 备用图片，放大最后，用于所有 `details 关闭状态 -->
            <img class="fallback" src="https://picsum.photos/id/21/1000/1000" alt="" />
        </div>
    </div>
</div>
```

  


并在没有打开 `<details>` 元素时，显示这张备用图片:

  


```CSS
.accordion {
    img {
        opacity: 0;
        scale: 1.1;
        transition: scale 0.3s, opacity 0.3s;
    }
    
    :is(
        &:has(details:nth-child(1)[open]) img:nth-child(1),
        &:has(details:nth-child(2)[open]) img:nth-child(2),
        &:has(details:nth-child(3)[open]) img:nth-child(3),
        &:has(details:nth-child(4)[open]) img:nth-child(4),
        &:has(details:nth-child(5)[open]) img:nth-child(5) 
    ) {
        opacity: 1;
        scale: 1;
    }
    
    &:not(:has(details[open])) .fallback {
        opacity: 1;
        scale: 1;
    }
}
```

  


现在就完美了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a2e2c68a0ffb4af9ac8e88368ebac58d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=602&s=15966780&e=gif&f=227&b=0f0f1c)
