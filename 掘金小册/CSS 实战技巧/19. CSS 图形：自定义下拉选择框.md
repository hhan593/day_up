## 了解 `<select>` 元素的工作原理

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1adffc23205d4f29b4474f758fa775d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=960&s=1794140&e=jpg&b=737373)

  


要理解为什么 CSS 样式化 `<select>` 下拉选择框非常棘手，我们需要了解它的内部机制。

  


使用以下 HTML 标记，我们可以创建一个包含七个可选择选项的下拉列表：

  


```HTML
<label for="job">Main job role:</label>
<select id="job">
    <option>Select role...</option>
    <option>UI/UX Designer</option>
    <option>Frontend Engineer</option>
    <option>Backend Engineer</option>
    <option>QA Engineer</option>
    <option>Unicorn</option>
</select>
```

  


与所有表单字段类型一样，`<select>` 在各个浏览器中的初始外观是不同的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99a314c1dd564083a5046d6837e594e5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=670&s=258020&e=jpg&b=d7dbe6)

  


这些差异包括框的大小、字体大小、行高，最显眼的差异是下拉指示器的样式（框右侧的箭头），其次，下拉列表（`<option>`）的样式也有明显的差异。

  


如果我们花点时间使用浏览器开发者工具检查这个控件，我们会发现它是作为一个 Web 组件实现的，位于 Shadow DOM 内部：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/09499e3883f54aa9acc52c857bddcfdd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1111&s=560428&e=jpg&b=d7dbe6)

  


正如你所看到的，下拉选择框的 UI 不是 DOM 的一部分，它以原生 UI 元素的形式呈现，无法通过 CSS 进行样式化（在某些浏览器中，有限的 CSS 属性可能有效），因此它的实现隐藏在 Shadow DOM 节点中。并且它们（`<select>` 和 `<option>`）并不像我们之前创建自定义 **[范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)**和**[进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)** ，浏览器并没有为它们提供相应的伪元素选择器，以便 Web 开发者来美化下拉选择框的 UI 外观。

  


然而，`<select>` 元素是一个 DOM 元素，在所有现代浏览器中使用用户代理样式表进行样式化，因此我们可以使用纯 CSS 对其进行自定义。为这些所有主流浏览器创建相同的初始外观，包括多选和禁用状态。

  


注意，下拉列表 `<option>` 一直以来在 Web 开发者的认知中，都是无法进行样式化，所以一旦打开 `<select>` ，它仍然会采用各个浏览器的默认样式。庆幸的是，[Open UI](https://open-ui.org/) 为我们提供了一种新的机制，使我们既可以使用纯 CSS 来美化 `<select>` 元素的 UI 外观，还可以美化 `<option>` 元素的外观。我们将在课程的后半部分会详细阐述。

  


还有一点需要知道的是，到目前为止，文本是 `<option>` 元素唯一有效的内容。`<option><span>text</span></option>` 在技术上是无效的，最终浏览器将 `<span>` 元素的文本内容 `text` 作为 `<option>` 元素的内容，与此同时会删除 `<span>` 标签。因此，你通过浏览器开发者工具审查该元素时，你看到的将是 `<option>text<option>` 。

  


了解完它的工作原理之后，我们开始着手使用 CSS 来自定义下拉选择框！

  


## 使用纯 CSS 自定义 `<select>` 下拉选择框

### 重置和移除继承的样式

  


与之前自定义[单选按钮](https://juejin.cn/book/7395119173656903715/section/7412570263411720219#heading-2)、[复选框](https://juejin.cn/book/7395119173656903715/section/7412570263411720219#heading-2)、[切换按钮](https://juejin.cn/book/7395119173656903715/section/7412570263411720219#heading-15)、[范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)和[进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)一样，我们首先重置和移除 `<select>` 的继承样式。

  


```CSS
select {
    appearance: none;
    color: inherit;
    background-color: transparent;
    border: none;
    padding: 0 1em 0 0;
    margin: 0;
    width: 100%;
    font-family: inherit;
    font-size: inherit;
    cursor: inherit;
    line-height: inherit;
}
```

  


这里有一个重要的代码是 `appearance: none` 声明。它指示浏览器移除原生下拉菜单的默认样式。这是在应用自定义样式之前必须采取的关键步骤。查看下面的结果，你会发现将 `appearance` 属性设置为 `none` 也移除了原生下拉箭头：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/594d7ea5d55247e2b554f826fa1638b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1092&h=462&s=815158&e=gif&f=245&b=040404)

  


最后一步是移除默认的 `outline` 。不用担心，稍后我们为会控制添加焦点环样式。

  


```CSS
select{
    /* CSS ... */
    outline: none;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/820e9e2f84c94b4ab842dffe9e7eb413~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1062&h=460&s=821656&e=gif&f=157&b=030303)

  


你可以看到，在点击之前，`<select>` 没有任何视觉提示显示这是一个下拉选择框。

  


### 自定义选择框样式

  


首先，我们定义一些 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)，便于我们更好地控制下拉选择框的外观：

  


```CSS
.select {
    --dropdown-background: #fff;
    --dropdown-color: #000;
    --dropdown-hover-color: #eee;
    --dropdown-border-color: #caced1;
    --dropdown-border-radius: round(.25em, 1px);
    --dropdown-icon-color: var(--dropdown-border-color);
    --dropdown-focus: #2196f3;
    
    --fs: 1.125rem;
    
    @media (prefers-color-scheme: dark) {
        --dropdown-background: #000;
        --dropdowncolor: #fff;
        --dropdown-hover-color: #222;
        --dropdown-border-color: #555;
    }
}
```

  


> 可访问性提示：作为一个用户界面元素，选择框的边框颜色与周围表面颜色的对比度必须达到 `3:1` 或更高。你可以尝试使用[现代 CSS ](https://s.juejin.cn/ds/ike1daK2/)中的[颜色对比度函数 `color-contrast()`](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)创建易于访问的用户界面!

  


现在，我们可以在 `.select` 元素（包裹 `<select>` 元素的容器）设置样式，即自定义下拉选择框样式：

  


```CSS
.select {
    width: clamp(15ch, 100%, 30ch);
    border: 1px solid var(--dropdown-border-color);
    border-radius: var(--dropdown-border-radius);
    padding: round(0.675em, 1px) round(1em, 1px);
    font-size: var(--fs);
    cursor: pointer;
    line-height: 1.1;
    color: var(--dropdown-color);
    background-color: var(--dropdown-background);
    background-image: liner-gradient(to top, color-mix(in srgb, var(--dropdown-background), transparent 50%), var(--dropdown-background));
}
```

  


首先，使用 [CSS 的比较函数 `clamp()`](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)给元素的 `width` 设置了一些限制，`width` 会被限制在 `15ch ~ 30ch` 之间，其中 `min=15ch` ，`max=30ch` ，你可以根据自己的使用场景修改这些值。

  


接着，我们应用了些盒模型属性，包括边框（`border`）、圆角（`border-radius`）和内距（`padding`）。注意，`em` 单位的使用，它将使这些属性保持与设置的字体大小（`font-size`）成比例。另外，`ch` 单位也会与设置的字体和字体大小成比例。因为 `1ch` 与 `0` 字符大小相等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a1a1c86d1874f4d82f58b9cd5ef4c65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3168&h=906&s=113310&e=png&b=eff0f1)

  


注意，上图中的 `cap` 、`ex` 、`ch` 、`ic` 和 `em` 都是 [CSS 相对单位中的字体相对单位](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-3)，它们的大小都与字体、字号、行高等因素有关。如果你对这方面的知识感兴趣，[请点击这里](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)，你将能全面了解 CSS 中相对单位的知识。

  


在重置 `<select>` 样式时，将多个属性（例如，`color` 、`font-family` 、`font-size` 、`line-height` 和 `cursor` 等）的值设置为[关键词 `inherit`](https://juejin.cn/book/7223230325122400288/section/7232094160071688227) ，因此 `<select>` 将会继承 `.select` 元素的 `color` 、`font-family` 、`font-size` 、`line-height` 和 `cursor` 属性的值。

  


最后给它设置背景样式，包括使用线性渐变 `linear-gradient` 来增加轻微的层次感。请注意，我在渐变中使用了 [`color-mix()` 函数](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)，它将 `var(--dropdown-background)` 和 `50%` 的透明度混合成一个新的颜色，相当于给 `var(--dropdown-background)` 颜色设置了 `50%` 的透明度。这是 CSS 中的一个小技巧，你可以将其理解为诸如 Sass 处理器中的 `rgba()` 颜色函数。另外，要是移除了背景属性，那么选择框的背景将变为透明，你将看到位于元素底部的颜色，它可能是其父容器的背景或页面的背景。

  


由于我们在定义 CSS 变量的时候使用了 [CSS 媒体查询的 `prefers-color-scheme` 查询条件](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-18)，调整了颜色相关变量的值。所以，我们现在自定义的下拉选择框是具备两种主题效果，即暗色和亮色。你现在看到的效果大致如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3cb8f64b4ff34eecb6e2d7af91075f3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1402&h=642&s=2092442&e=gif&f=318&b=ffffff)

  


你现在看到的下拉选择框是没有下拉箭头的，这是因为我们重置 `<select>` 继承样式时应用了 `appearance:none` ，它将原生的下拉箭头也移除了。所以，我们接下来需要下拉选择框自定义一个下拉箭头。

  


### 自定义选择框的下拉箭头

  


接下来，我们在 `.select` 的伪元素 `::after` 上为自定义选择框设置一个类似下图这样的下拉箭头：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bcd978ba0db047b583f7717b72e6727c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=871&s=264506&e=jpg&b=d7dbe6)

  


我将使用线性渐变 `linear-gradient` 和 `clip-path` 来绘制下拉框的下拉箭头。使用线性渐变绘制箭头所需要的上下两部分，然后通过 `clip-path` 裁剪也所需要的形状。

  


```CSS
.select {
    /* CSS ... */
    
    &::after {
        content: "";
        width: round(0.8em, 1px);
        aspect-ratio: 3 / 4;
        background: linear-gradient(to bottom, var(--dropdown-border-color) 0 40%, #0000 40% 60%, var(--dropdown-border-color) 60% 100%) center no-repeat;
        clip-path: polygon(50% 0%, 100% 35%, 100% 65%, 50% 100%, 0 65%, 0 35%);
        pointer-events: none;
    }
}
```

  


请注意上面代码中的 `pointer-events:none` ，它将禁用箭头元素上的指针事件，以确保功能正常。这样做的好处是，当用户点击箭头时，`<select>` 元素会接收到点击事件，而不是箭头本身。换句话说，如果没有给 `::after` 伪元素显式设置 `pointer-events: none` ，你点击选择框的下拉箭头时，下拉选择框没有任何反应：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/95a22d02623b49c193b77b1540f5879c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=464&s=181456&e=gif&f=114&b=ffffff)

  


反之，则会禁用箭头元素（`.select` 的伪元素 `::after`）上的指针事件可以确保与箭头指示器交互时下拉选择框会按预期打开，从而提供流畅且用户友好的体验：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a26685e0019242468cec31e21d854229~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1112&h=430&s=920611&e=gif&f=230&b=ffffff)

  


如果你在跟随我一起操作的话，你可能已经发现了，尽管我们显式给伪元素 `::after` 设置了 `width` 和 `aspect-ratio` 。检查后发现，`::after` 元素的实际宽高仍然是 `0` 。就这个现象而言，我们有多种方式可以解决，最直接的方式是设置 `::after` 元素的 `display` 属性的值为 `block` 。或者通过更新 `.select` 元素的布局来解决这个问题。例如：

  


```CSS
.select {
    display: grid;
    align-items: center;
    
    select, &::after {
        grid-area: 1 / 1 / -1 / -1;
    }
    
    &::after {
        /* CSS ... */
        place-self: end;
    }
    
    select {
        padding-right: round(0.8em + .5em, 1px);
    }
}
```

  


这样相当于给 `::after` 设置了一个类似 `display: block` 的效果。

  


另外，我选择在 `.select` 元素上使用 [CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)有多个好处，除了使 `::after` 可见之外，还可以更好的控制 `<select>` 和 `.select::after` 在垂直方向对齐。我们只需要在网格容器上使用 `align-items: center` 即。与此同时，将伪元素 `::after` 的 `place-self` 设置为 `end` ，可以使下拉箭头始终位于网格容器的内联轴的末端。[这些都是 CSS 网格布局的最基本特性](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)，我想你已经对这些特性非常的熟悉了。

  


现在，浏览器呈现给你的下拉选择框的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1a068064900d4f979f3f6a4461772a34~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1338&h=580&s=2247986&e=gif&f=362&b=ffffff)

  


正如你所看到的，[这里使用现代 Web 布局的 Grid 布局技术](https://juejin.cn/book/7161370789680250917/section/7161624114308841509)，除了易于控制元素对齐方式之外，还易于控制元素的层叠方式。如果你还在使用古老的绝对定位的话，那么可以在下一个项目中考虑使用这个比较现代的 CSS 技巧。

  


### 自定义选择框状态样式

  


`<select>` 元素同样有各种状态，比如聚焦状态（`:focus` 或 `:focus-visible`）、禁用状态（`:disabled`）和悬浮状态（`:hover`）等。接下来，我们来看看如何给自定义选择框定义这些常见状态的样式。我们先从最简单的悬浮状态开始。

  


我想大家已经猜到了，给下拉选择框设置悬浮状态的样式非常简单，我们只需要在 `.select:hover` 中设置所需要的样式，例如改变它的背景颜色：

  


```CSS
.select {
    /* CSS ... */
    
    transition: all .2s;
    
    &:hover {
        background-color: var(--dropdown-hover-color);
    }
}
```

你可能已经发现了，[下拉选择框的 `background-color` 有一个平滑的过渡效果](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)，这是因为在 `.select` 设置了 `transition: all .2s` ，当 `.select` 元素的样式从 `A` （`background-color` 的值是 `var(--dropdown-background-color)`）过渡到 `B` 状态（`background-color` 的值是 `var(--dropdown-hover-color)`）时会有一们平滑过渡的动画效果。注意，在这个示例中，只有 `background-color` 在初始状态和悬浮状态 有变化。你也可以根据自己需要进行调整，比如在 `:hover` 状态增加更多的样式变化，前提条件是要符合 UI 的外观。

  


接着我们给下拉选择框添加焦点环样式。它相对而言要更麻烦一点，因为我们使用了一个普通的 `div` 元素来定义下拉框的 UI 外观。`div` 并不是一个[可聚焦元素](https://html.spec.whatwg.org/multipage/interaction.html#focusable)，不能直接通过 `:focus` 或 `:focus-visible` 给下拉选择框设置焦点样式。不过，它的子元素 `<select>` 是一个可聚焦元素，[我们可以通过 `:focus-within` 来设置下拉选择框的焦点环样式](https://juejin.cn/book/7223230325122400288/section/7226969870444232742#heading-6)：

  


```CSS
.select {
    /* CSS ...*/
    
    &:focus-within {
        outline: 2px solid var(--dropdown-focus);
        outline-offset: 2px;
    }
}
```

  


代码中的 `.select:focus-within` 的意思是，只要它或它的后代元素获得焦点，都可以触发 `:focus-within` 选择器。该选择器有点类似于 JavaScript 的事件冒泡，从可获得焦点元素开始一直冒泡到 HTML 的根元素 `<html>` ，都可以接收触 `:focus-within` 。

  


这个时候，当 `select` 获得焦点时，下拉选择框就拥有了 `outline` 和 `outline-offset` 设置的焦点环样式。其效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/94c07d1f6999464e8848bb9d4af4ae84~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1358&h=630&s=1887959&e=gif&f=315&b=010101)

  


在现代 CSS 中，除了使用 `:focus-within` 伪类选择器之外，还可以将 [`:has()` 选择器与 `<select>` 元素的状态选择器 `:focus` 和 `:focus-visible` 组合起来使用](https://juejin.cn/book/7223230325122400288/section/7224404685799555111#heading-3)：

  


```CSS
.select {
    /* CSS ... */
    
    &:has(:focus, :focus-visible) {
        outline: 2px solid var(--dropdown-focus);
        outline-offset: 2px;
    }
}
```

  


这两种方案都可以给我们的下拉选择框设置焦点环样式，并且最终的结果是一致的。

  


有意思的是，我们也可以将 `:has()` 选择器与 `:disabled` 状态选择器结合起来给下拉选择框设置禁用状态的样式。例如：

  


```CSS

    /* CSS ... */

    @media (prefers-color-scheme: dark) {
        --dropdow-disabled: #666;
    }
    
    &:has(:disabled) {
        cursor: not-allowed;
        background-color: var(--dropdow-disabled);
        background-image: linear-gradient(to top, var(--dropdow-disabled), color-mix(in srgb, var(--dropdow-background) 50%, currentColor 15%) 33%);
    }
}
```

## 基于 Popover API 和 JavaScript 自定义下拉选择框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4e48b01a5dfd47618cfc4eed12bf6726~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1921&h=1080&s=817457&e=jpg&b=f9f9f9)

  


如果你是一名 Web 组件库的开发者或者对对 UI 的细节有着极致的追求者，那么你肯定无法忍受下拉选择框的默认下拉列表效果。为了使下拉列表的 UI 也能跨浏览器统一，你不得不选择基于非 `<select>` 元素和 JavaScript 来自定义下拉选择框。这也是自定义下拉选择框的另一种常见方式。

  


与基于 `<select>` 元素自定义下拉选择框相比，最大的优势是，你完全可以控制下拉框的 UI 样式，包括它的交互动效。然而，一个下拉选择框包含许多组成部分，并且具有许多需要考虑的状态，例如：

  


-   键盘绑定（用于进入/退出交互）
-   点击外部区域以关闭
-   弹出框的管理（打开一个时关闭其他弹出框）
-   Tab 键焦点管理
-   可视化显示已选中的选项值
-   箭头的交互样式
-   状态管理（打开/关闭）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8d5ed85a2608484ca54a9adad9df825f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1111&s=266932&e=jpg&b=fefefe)

  


目前，自己管理所有这些状态非常困难，而平台也没有提供简单的解决方案。这意味着，在自定义下拉选择框时，你需要使用 JavaScript 做很多事情。

  


庆幸的是，Web 平台为了解决这个问题，对下拉选择框的组成部分进行拆解，并正在推出一些基础功能，比如 HTML Popover API 和 [CSS 锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)等，它们不仅能帮助 Web 开发者更好的样式化下拉选择框，还能实现更多功能。接下来，我将通过一个简单的案例来向大家展示，如何基于这些新特性来自定义一个下拉选择框。

  


> 注意，接下来的重点将会放在 HTML Popover API 和 CSS 锚点定位上，不会对 JavaScript 要做的事情展开讨论！

  


在开始之前，我们简单的了解一下 HTML Popover API 和 CSS 锚点定位的基础知识。

  


### HTML Popover API 的基本用法

  


`popover` 是 HTML 的一个新增的全局属性。当你给一个 HTML 元素设置了 `popover` 属性之后，浏览器将会给这个元素默认使用 `display: none` 隐藏，直到通过按钮或 JavaScript 之类的触发器将其打开。例如，如果你要创建一个的弹出框，只需要在元素上设置 `popover` 属性，并使用 `popovertarget` 将其 `id` 与按钮关连起来。现在，按钮就是触发器。

  


```HTML
<button popovertarget="my-popover">打开 Popover</button>

<div id="my-popover" popover>
    <p>我是一个带有更多信息的弹出框。</p>
</div>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d6bcbcf280c403f9e6e3a12849e9dda~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1122&h=618&s=661883&e=gif&f=153&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/rNEbYxg

  


`popover` 属性提供了 `auto` 和 `manual` 两个值：

  


-   `popover="auto"` 是默认值，会创建一个可以轻松关闭的弹出框。可以通过点击外部或按下键盘上的 `ESC` 键关闭。
-   `popover="manual"` 将创建一个不能轻松关闭的弹出框。因此，如果需要，应该提供自定义关闭操作。如果你希望用户在弹出框中执行特定操作后再关闭，这是你需要的值。

  


这个新 API 的一个最大优势是我们可以选择自己的元素和 `aria-roles`。[与总是具有对话框角色的 `<dialog>` 元素不同](https://juejin.cn/book/7223230325122400288/section/7229323379793199104)，Popover API 可以灵活地用于多种情况，只需通过正确的元素或角色（如对话框、工具提示、警告等）来精心规划。

  


Popover API 还引入了`popovertarget` 和 `popovertargetaction` 新属性：

  


-   `popovertarget` 用于引用我们希望显示、隐藏或切换的弹出框的 `id`
-   `popovertargetaction` 提供应执行的操作，选项包括：`show`（显示）、`hide`（隐藏）或 `toggle`（切换）

  


如果不希望弹出框被轻松关闭，可以这样创建：

  


```HTML
<button popovertarget="some-popover" popovertargetaction="show">Open the popover</button>
<div id="some-popover" popover="manual" role="dialog">
    Hi, I'm a popover!
    <button popovertarget="some-popover" popovertargetaction="hide">❌</button>
</div>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/33a69aa8c1e74dd9b065d134e95d41d1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1070&h=624&s=940911&e=gif&f=232&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/XWLQzNv

  


正如你所看到的，启用 `popover` 属性后，浏览器无需额外脚本即可处理许多关键行为，包括：

  


-   **提升到顶层**：放置在页面其他内容之上的独立层，不需要处理 `z-index`
-   **轻触关闭功能**：点击弹出框区域外部即可关闭弹出框并恢复焦点
-   **默认 Tab 键焦点管理**：打开弹出框后，Tab 键的下一个焦点点会在弹出框内
-   **内置键盘绑定**：按下 Esc 键或双重切换会关闭弹出框并恢复焦点
-   **默认组件绑定**：浏览器会语义化地将弹出框与其触发器关联

  


**重要提示**：`popover` 属性本身不包含语义含义，因此你仍需使用适当的 HTML 或 ARIA 来为弹出框提供上下文意义。此外，`popover` 不会使页面的其余部分变为非交互状态或陷入焦点。如果你需要这种体验，`<dialog>` 元素可能更适合。

  


在使用 `popover` 时，你还可以使用下面这几个 CSS 特性，为设置 `popover` 属性的元素提供入场和退出（即淡入淡出）的交互动效：

  


-   在关键帧时间线上为 `display` 和 `content-visibility` 属性添加动画
-   使用 `transition-behavior` 属性的 `allow-discrete` 关键字为离散属性（如 `display`）启用过渡效果
-   通过 `@starting-style` 规则从 `display: none` 到顶层元素实现入场动画
-   使用 `overlay` 属性在动画期间控制顶层行为

  


```CSS
/*  打开前状态   */
@starting-style {
    [popover]:popover-open {
        translate: 0 -22rem;
    }
}

/*  打开状态  */
[popover]:popover-open {
    translate: 0 0;
}

/*   退出状态  */
[popover] {
    transition: translate 0.7s ease-out, display 0.7s ease-out allow-discrete;
    translate: 0 -22rem;
} 
```

  


> 温馨提示，如果你想更深入的了解 `transition-behavior` 和 `@starting-style` 特性，[请点击这里获取更多的相关信息](https://juejin.cn/book/7288940354408022074/section/7292735608995184678#heading-4)！

  


### CSS 锚点定位

  


`popover` 只是开始，一个非常令人兴奋的特性是 [CSS 的锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)。通过几行代码，浏览器可以处理将一个定位元素绑定到一个或多个锚点的逻辑。你可以使用 `anchor-name` 属性设置锚定元素，并通过 `position-anchor` 属性将定位元素绑定到锚点元素。

  


以上面示例的弹出框为例。由于弹出框处于文档流之外，跟踪相对定位比较困难。但在某些情况下，我们希望将弹出框“锚定”到另一个元素上（比如示例中的按钮上）。在这种情景之下，CSS 锚点定位就非常有意义了。

  


要使锚定生效，我们需要为元素指定一个锚点名称，例如：

  


```CSS
.anchor {
    anchor-name: --popover;
}
```

  


现在，如果我们想将顶部图层元素（`div[popover]`）“锚定”到 `.anchor` （触发打开弹框的按钮 `button`），我们可以使用锚点名称和它的位置：

  


```CSS
[popover] {
    position-anchor: --popover;
    bottom: calc(anchor(top) + 1.5rem);
    top: unset;
}
```

  


这个时候，弹框的定位是相对于 `button` 元素进行的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1046766578da4a7ea30efed5ab5c0be1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1180&h=624&s=1207269&e=gif&f=189&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/bGPJYKm

  


注意，这里所展示的仅是 CSS 锚点定位的最基础功能，如果你希望更深入了解它，[请点击这里获取更多的信息](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)。

  


### 自定义下拉选择框

  


我想你现在应该对 Popover API 和 CSS 锚点定位有了一个基本的认识。现在，我们可以通过 Popover API 和 CSS 锚点定位来自定义一个炫酷的下拉选择框。

  


从 HTML 结构开始：

  


```HTML
<button popovertarget="pop" class="select--button">
    <span>Minimalist</span>
    <svg  fill="none" viewBox="0 0 24 24" stroke-width="1.5" stroke="currentColor" class="icon--arrow">
        <path stroke-linecap="round" stroke-linejoin="round" d="M8.25 15 12 18.75 15.75 15m-7.5-6L12 5.25 15.75 9" />
    </svg>
</button>

<div id="pop" popover="auto">
    <div class="popover__content">
        <ul>
            <li>
                <button data-value="Minimalist" data-selected="true" popovertarget="pop" autofocus>
                    Minimalist
                    <svg class="icon--tick">
                        <use href="#marker" />
                    </svg>
                </button>
            </li>
            <!-- 省略其他 li -->
        </ul>
    </div>
</div>

<svg class="sr-only">
    <defs>
        <symbol fill="none" stroke-width="1.5" stroke="currentColor" viewBox="0 0 24 24" id="marker">
            <path stroke-linecap="round" stroke-linejoin="round" d="m4.5 12.75 6 6 9-13.5" />
        </symbol>
    </defs>
</svg>
```

  


注意，这里使用两个 SVG，它们分别绘制了下拉框右侧的“箭头”和选中列表的标记（“打勾”）。我们在 `#pop` 元素上显式设置了 `popover="auto"` ，默认情况下 `#pop` 元素（下拉列表项）会隐藏。与此同时，`<button>` 设置了 `popovertarget` 属性的值为 `pop` ，它与下拉列表项容器 `div` 的 `id` 绑定在一起。

  


这个时候你点击 `button` ，`#pop` 元素会在显式和隐藏之间切换，这也是下拉选择框的基本交互效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3fe600b0c2bf4142b9a437ac1d087059~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1258&h=790&s=872206&e=gif&f=161&b=030303)

## 可样式化`<select>` 的未来

  


目前，[存在一个名为 Open UI 的在线倡议](https://open-ui.org/)，旨在标准化 Web 上的 UI 组件，例如 `<select>` 。 [Open UI](https://open-ui.org/) 是一个 [W3C 社区小组](https://www.w3.org/community/open-ui/)，致力于让 Web 开发者能够样式化和可扩展内置的 Web UI 组件和控件，例如下拉选择框（`<select>`）、复选框（`type` 为 `checkebox` 的 `input`）、单选按钮（`type` 为 `radio` 的 `input`）、日期（`type` 为 `date` 的 `input`）和颜色（`type` 为 `color` 的 `input`）选择器。

  


在这里，我们继续以 `<select>` 为例。得益于 Open UI 工作小组，在未来，我们可以基于 Popover API 和 CSS 锚点定位来样式化 `<select>` 。为什么说是未来呢？可自定义的 `<select>` 目前仍处在实验阶段。不过，你在 Chromium 浏览器中可以体验到这个新功能。要使用它，请在 `about:flags` 中启用 "Experimental Web Platform features" 标志。建议使用 Chrome Canary 以获取最新的 API 状态。

  


目前你可以使用 CSS 中的 `appearance` 属性，但不是将其设置为 `none` ，而设置为 `base-select` 。根据标准化讨论，`appearance: base-select` 还必须应用于 `select::picker(select)` 伪元素，以选择加入选择器弹出窗口。这意味着，一旦设置好 `appearance` ，你就会选择新的、自定义的 `select` 体验：

  


```CSS
select,
::picker(select) {
    appearance: base-select;
}
```

`<select>` 元素的第一个子 `<button>` 将插入到 `<select>` 的 UA ShadowRoot 中，并用于打开弹出选项列表。

  


`<selectedoption>` 元素允许开发者声明式地将已选 `<option>` 元素的内容克隆到按钮中，并使其样式与 `<option>` 区分开来。这种用法在 “在按钮中不同渲染选项” 的示例中得到了展示。

  


当 `<selectedoption>` 元素放置在 `<select>` 的 `<button>` 内时，浏览器会通过调用当前选中的 `<option>` 元素的 `cloneNode()` 来替换其 DOM 内容。每当用户选择不同的 `<option>` 或已选 `<option>` 中发生 DOM 更改时，浏览器会再次内部调用 `cloneNode()` 来更新 `<selectedoption>` 的内容。

  


由于 `<selectedoption>` 元素的内容是由浏览器维护的，开发者不应在 `<selectedoption>` 内放置任何 HTML/DOM 内容。

  


`<select>` 将支持 `selectedoptionelement` 属性，该属性是一个指向单个 `<selectedoption>` 元素的 IDref 用于更新内容。这允许 `<selectedoption>` 存在于 `<select>` 外部，以支持分离按钮的用例。其中一个按钮用于打开选择器的弹出窗口，另一个用于显示选定内容并提交表单。

  


自定义 `<select>` 提供了多种样式化工具，包括针对不同状态的伪选择器和用于不同部分的伪元素。
