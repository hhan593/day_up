![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9fb33b7b26a744a88811f88b57ba7141~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=1024&s=1927237&e=png&b=396e97)

  


一直以来，Web 开发者可以通过 CSS 提供的相关特性来绘制图形，并且使用图形进行裁剪和动画。比如，结合 CSS 的 **[裁剪](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-1)**（`clip-path`）、**[遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)**（`mask`）、**[渐变](https://juejin.cn/book/7395119173656903715/section/7429225312224804899)**（比如 `conic-gradient()` 、`radial-gradient()` 和 `linear-gradient()` 等）和 **[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)** 等创建不同的形状以及复杂而丰富的 UI 效果。其中，最为常见的就是使[用 clip-path 绘制各种不同形状的图形](https://juejin.cn/book/7395119173656903715/section/7395172754053890063)。比如，它允许你在布局完成之后，把元素边缘修剪成一个波浪形。这种方式可以产生类似遮罩（`mask`）的效果，但相比遮罩渲染效率更高。另一个图形的应用场景是 `offset-path` 属性，[它可以让元素沿着一条路径进行动画运动](https://juejin.cn/book/7288940354408022074/section/7308623339038670860)。最后，`shape-outside` 或 `shape-inside` 属性可以让文本围绕着非矩形的形状排版，[实现不规则的布局效果](https://juejin.cn/book/7161370789680250917/section/7161625566062313503)。

  


CSS 提供了一些图形函数，允许你在 `clip-path` 、`offset-path` 、`shape-outside` 和 `shape-inside` 属性中使用基本图形。这些图形函数包括：

  


-   `inset()` ：绘制矩形
-   `circle()` ：绘制圆形
-   `ellipse()` ：绘制椭圆形
-   `polygon()` ：绘制多边形
-   `path()` ：使用 **[SVG 路径](https://juejin.cn/book/7341630791099383835/section/7368317966821425190)** 的语法，绘制更复杂的图形
-   `url()` ：通常引用 SVG 路径定义的图形

  


尤其是其中的 `path()` 函数，它允许你使用 SVG 路径来改变元素的形状，但它有一个致命的缺点，`path()` 函数中引用的 SVG 路径不具备响应能力。这意味着，你必须在响应式的多边形和更复杂但不具备响应能力的 SVG 路径之间做出选择。

  


而现在，借助新的 `shape()` 函数，这些属性（`clip-path` 、`offset-path` 、`shape-outside` 和 `shape-inside`）可以创建非常复杂的非多边形形状，同时仍然具备响应性。接下来，我们以 `clip-path` 的 `path()` 和 `shape()` 创建复杂的非多边形形状为例，看看 `shape()` 是如何创建出我们所需要的图形形状，它又具备哪些独特的功能！

  


我们开始吧！

## `shape()` 函数简介

  


`shape()` 函数也隶属于 `<basic-shape>` 数据类型的一部分，它与 `polygon()` 和 `path()` 等函数相似，用来为 `clip-path` 、`offset-path` 、`shape-outside` 和 `shape-inside` 等属性定义形状。它通过一个初始点和一系列描述路径的形状指令组合而成。比如：

  


```CSS
/* <fill-rule> */
clip-path: shape(nonzero from 0 0, line to 10px 10px);
/* <move-command>、<line-command> 和 close */
offset-path: shape(from 10px 10px, move by 10px 5px, line by 20px 40%, close);
/* <hvline-command> */
offset-path: shape(from 10px 10px, hline by 50px, vline to 5rem);
/* <curve-command> */
offset-path: shape(from 10px 10px, curve to 80px 80px via 160px 1px 20% 16px);
/* <smooth-command> */
offset-path: shape(from 10px 10px, smooth to 100px 50pt);
/* <arc-command> */
offset-path: shape(
    from 5% 0.5rem,
    arc to 80px 1pt of 10% ccw large rotate 25deg
);
/* 使用 CSS 数学函数 */
offset-path: shape(
    from 5px -5%,
    hline to 50px,
    vline by calc(0% + 160px),
    hline by 70.5px,
    close,
    vline by 60px
);
clip-path: shape(
    evenodd from 10px 10px,
    curve to 60px 20% via 40px 0,
    smooth to 90px 0,
    curve by -20px 60% via 10% 40px 20% 20px,
    smooth by -40% -10px via -10px 70px
);
```

  


它的正式语法规则是：

  


```CSS
<shape()> = shape( <'fill-rule'>? from <position>, <shape-command># )
<shape-command> =
  <move-command> |
  <line-command> |
  close |
  <horizontal-line-command> |
  <vertical-line-command> |
  <curve-command> |
  <smooth-command> |
  <arc-command>
```

  


简单解释一下它的参数：

  


-   **`<fill-rule>`** ：指定形状重叠区域的填充规则，这是一个可选值，主要值有 `nonzero` 和 `evenodd` 。其中 `nonzero` 表示从某点画一条射线，如果左到右穿过的线段数减去右到左穿过的线段数不为零，该点被认为在形状内容（它是 `<fill-rule>` 的默认值）；`evenodd` 表示如果射线穿过路径的次数是奇数，则认为该点在形状内容。我曾在介绍 `clip-path` 属性的 `polygon()` 函数时，[有详细阐述过这方面的规则](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-6)。注意，`<fill-rule> 不能用于 offset-path` ，如果使用会导致属性无效
-   **`from <coordinate-pair>`** ：定认第一个`<shape-command>` 的起始点，以 **[参考框](https://juejin.cn/book/7395119173656903715/section/7445257036046991410)** 左上角为原点的坐标（`x y`）。长度可以是百分比或固定单位
-   **`<shape-command>`** ：用一系列逗号分隔的命令描述形状，类似于 **[SVG 路径命令](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-1)**

  


`<shape-command>` 命令主要包括：

  


-   **`<move-command>`** ：它的格式为 `move [by | to] <coordinate-pair>` ，移动到某个点，它本身不会绘制任何东西，而是指定下一个命令的起始位置。如果它紧跟在 `close` 命令之后，则它定义了下一个形状或子路径的起点。[相当于 SVG 路径命令中的 `M`（或 `m`）](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-3)
-   **`<line-command>`** ：它的格式 `line [by | to] <coordinate-pair>` ，它从起点绘制一条直线到指定的终点。[相当于 SVG 路径命令中的 `L` （或 `l`）](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-4)
-   **`<hv-line-command>`** ：它的格式为 `[hline | vline] [by | to] <length-percentage>`，画水平线（`hline`）或垂直线（`vline`）。使用 `hline` 时，从起点绘制一条水平线，到或相对于 `<length-percentage>` 指定的 `x` 位置（[相当于 SVG 路径命令中的 `H` 或 `h` ](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-5)）；使用 `vline` ，从起点绘制一条垂直线，到或相对于 `<length-percentage>` 指定的 `y` 位置（[相当于 SVG 路径命令中的 `V` 或 `v`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-6)）
-   **`<curve-command>`** ：它的格式为 `curve [by | to] <coordinate-pair> via <coordinate-pair> [<coordinate-pair>]` ，绘制贝塞尔曲线，其中 `via` 后指定的是贝塞尔曲线的控制点。如果只提供一个 `<coordinate-pair>`，则绘制一个二次贝塞尔曲线（三个点：起点、控制点、终点），[相当于 SVG 路径命令中的 `Q` 或 `q`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-7)；如果提供两个 `<coordinate-pair>`，则绘制一个三次贝塞尔曲线（四个点：起点、两个控制点、终点），[相当于 SVG 路径命令中的 `C` 或 `c`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-9)
-   **`<smooth-command>`** ：它的格式为 `smooth [by | to] <coordinate-pair> [via <coordinate-pair>]` ，绘制平滑曲线，保证曲线过渡连续。如果省略了 `via <coordinate-pair>`，则绘制一个平滑的二次贝塞尔曲线，使用前一个控制点和当前终点确定曲线形状，[相当于 SVG 路径命令中的 `T` 或 `t`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-8)；如果包含了 `via`，则通过 `<coordinate-pair>` 指定控制点，绘制一个平滑的三次贝塞尔曲线，使用前一个控制点、当前控制点和当前终点，[相当于 SVG 路径命令中的 `S` 或 `s`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-10)
-   **`<arc-command>`** ：它的 格式为 `arc [by | to] <coordinate-pair> of <length-percentage> [<length-percentage>] [<arc-sweep> | <arc-size> | rotate <angle>]` ，绘制椭圆弧，可以指定方向、大小、旋转角度。[相当于 SVG 路径命令中的 `A` 或 `a`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-11) ，这个命令是最为复杂的一个命令
-   **`close`** ：从当前点回到起始点，闭合路径。从当前点（上一个命令的结束点）绘制一条直线到路径起点（由 `from <coordinate-pair>` 指定）。如果想要闭合路径但不绘制连接线，可以在 `close` 前插入一个指向起点的 `<move-command>` ；如果 `close` 后紧跟一个 `<move-command>`，则定义了下一个形状或子路径的起点。[相当于 SVG 路径命令中的 `Z` 或 `z`](https://juejin.cn/book/7341630791099383835/section/7368317966821425190#heading-12)

  


刚才提到过，`<arc-command>` 是当中最为复杂的一个命令。因此多花点时间介绍一下这个命令。

  


椭圆弧命令定义了两个可能的椭圆（都经过起点和终点），每个椭圆可以顺时针或逆时针绘制，因此根据弧线大小、方向和角度，可能产生四种不同的弧线。其中`of` 关键字指定椭圆的大小：

  


-   第一个 `<length-percentage>` 是椭圆的水平半径，
-   第二个（可选的）是垂直半径。

  


为了选择具体哪条弧线，可以指定以下参数：

  


-   `<arc-sweep>`：指定弧线绘制方向，顺时针（`cw`）或逆时针（`ccw`）。如果省略，默认是逆时针（`ccw`）
-   `<arc-size>`：指定使用较大的弧线（`large`）还是较小的弧线（`small`）。如果省略，默认使用较小弧线（`small`）
-   `rotate <angle>`：指定椭圆相对于 `x` 轴的旋转角度（单位是度）。正值顺时针旋转，负值逆时针旋转。默认是 `0deg`

  


特殊情况说明：

  


-   如果只提供一个 `<length-percentage>`，则水平半径和垂直半径相同，相当于绘制圆弧。这种情况下 `<arc-size>` 和 `<angle>` 不起作用
-   如果任一半径为零，则退化为一条直线，相当于 `<line-command>`
-   如果任一半径为负，则使用其绝对值
-   如果指定的半径不足以让椭圆覆盖起点和终点（即使旋转了角度），系统会按比例放大半径，直到刚好覆盖
-   如果起点和终点位于椭圆的正对面，仅存在一个椭圆和两条弧线可选，此时只通过 `<arc-sweep>` 指定方向，`<arc-size>` 无效

  


这些命令的使用完全可以参照 SVG 路径中的命令，如果你对 SVG 路径中的命令不太熟悉的话，个人强烈建议你花一些时间阅读《[SVG 路径的高级技术](https://juejin.cn/book/7341630791099383835/section/7368317966821425190)》，因为只有掌握了这些命令如何使用，你才能用好 `shape()` 函数。

  


你可能已经注意到了，SVG 路径中的每个命令都有大写和小写之分，比如 `M` 和 `m` ，它们用于区分绝对定位还是相对定位。在 `shape()` 函数中提供了 `by` 和 `to` 两个关键词来区分：

  


-   `by`：相对于当前点（**相对定位**）
-   `to`：相对于参考框左上角（**绝对定位**）

  


## `shape()` vs. `path()`

  


`shape()` 函数允许你定义复杂的形状，但它很多方面与 `path()` 函数相似：

  


-   `shape()` 函数中的 `<fill-rule>` 参数与 `path()` 函数中的同名参数的工作方式完全相同
-   `shape()` 函数需要指定一个或多个 `<shape-command>`，每个命令对应一个底层路径命令，比如 `MoveTo`、`LineTo` 和 `ClosePath`

  


不过，相比使用 `path()` 函数，`shape()` 函数提供了几个优势：

  


-   `shape()` 使用标准的 CSS 语法，这让你可以更轻松地在样式表中直接创建和修改形状。而 `path()` 使用的是 SVG 路径语法，对不熟悉 SVG 的人来说不够直观
-   `shape()` 支持多种 CSS 单位，包括百分比（%）、`rem` 和 `em`。相比之下，`path()` 将形状定义为单个字符串，并且单位只限于 `px`
-   `shape()` 还允许使用 CSS 数学函数，比如 `calc()`、`max()` 和 `abs()` 等，在定义形状时提供了更多灵活性
-   `shpae()` 函数还允许你使用 CSS 变量的全部能力

  


如果你熟悉 [SVG 的剪切](https://juejin.cn/book/7341630791099383835/section/7362748816769941540)的话，CSS 的 `shape()` 函数功能与它更接近。只不过 SVG 的剪切通常是使用 `<clipPath>` 来包括 `<path>` ，定义剪切的路径，然后在 CSS 中通过 `clip-path` 的 `url()` 函数引用 `clipPath` 的 `id` 值。而 `shape()` 函数则可以直接在 CSS 中使用，而且同样具有响应式的能力！

  


## `shape()` 函数的使用

  


我们回到之前的箭头形状的示例中，`path()` 函数中引用了一段绘制箭头形状的 SVG 路径：

  


```CSS
.arrow {
    clip-path: path("M0 0 L 100 0 L 150 50 L 100 100 L 0 100 Q 50 50 0 0 z");
}
```

  


我们可以将其拆分成一系列“命令”，每个命令描述了路径的一段：

  


-   `M 0 0` ：移动到 `(0,0)`
-   `L 100 0` ：画直线到 `(100,0)`
-   `L 150 50` ：画直线到 `(150,50)`
-   `L 100 100` ：画直线到 `(100,100)`
-   `L 0 100` ：画直线到 `(0, 100)`
-   `Q 50 50 0 0` ：绘制二次贝塞尔曲线，控制点为 `(50, 50)`，终点为 `(0, 0)`
-   `Z` ：闭合路径

  


我们可以使用 `shape()` 函数的命令来替换上面的命令：

  


```CSS
.arrow {
    clip-path:shape(
        from 0 0,                        /* M 0 0 */
        line to 100px 0px,               /* L 100 0 */
        line to 150px 50px,              /* L 150 50 */
        line to 100px 100px,             /* L 100 100 */
        line to 0px 100px,               /* L 0 100 */
        curve to 0px 0px with 50px 50px, /*Q 50 50 0 0*/
        close                            /*Z*/
    )
}
```

}
```

.arrow {
    --h: 100px;
    --_h: calc(var(--h) / 2);
    width: 100%;
    height: var(--h);
}
```

```CSS
.arrow {

    height: var(--h);
    clip-path:shape(
        from 0% 0%,                       
        line to calc(100% - var(--h)) 0%,               
        line to 100% var(--_h),             
        line to calc(100% - var(--h)) 100%,           
        line to 0% 100%,              
        curve to 0% 0% with var(--_h) var(--_h),

    height: 100%;

```

```

  


简单的解释一下这个路径命令：

  


-   `M 0 20`：移动到 `(0, 20)`
-   `C 25 0 75 40 100 20`：通过控制点 `(25, 0)` 和 `(75, 40)` 画曲线到 `(100, 20)`
-   `V 80`：画一条垂直线到 `y=80`
-   `C 75 100 25 60 0 80`：通过控制点 `(75, 100)` 和 `(25, 60)` 画曲线到 `(0, 80)`
-   `z`：闭合路径（返回到起点）

    aspect-ratio: 4 / 3;
    outline: 1px dashed #000;
    overflow: hidden;
    resize: both;
}
.flag {
    height: 100%;
    background: green;
    clip-path: shape(
        from 0% 20%,
        curve to 100% 20% with 25% 0% / 75% 40%,
        vline to 80%,
        curve to 0% 80% with 75% 100% / 25% 60%,
        close
    );
}
```

    syntax: "<number>";
    inherits: false;
    initial-value: 0;
}

    syntax: "<angle>";

    initial-value: 0deg;

        line to calc(50% + var(--_r) * cos(180deg + var(--a) - var(--_a)))  calc(50% + var(--_r) * sin(180deg + var(--a) - var(--_a))),

    &:hover {
        --r: 0.35;

    }

}

## 小结

  


在这节课中，我们通过一些简单的用例向大家展示了 `clip-path: shape()` 如何让你裁剪出任意的响应式形状，这在过去，我们只能借助 CSS 的渐变、遮罩或 SVG 剪切来实现。需要知道的是，`shape()` 函数不仅仅局限于 `clip-path` 属性，它还可以用于 `offset-path` 、`shape-outside` 和 `shape-inside` 属性，如果你感兴趣的话，可以尝试一下 `shape()` 函数用于其他属性上的效果！
