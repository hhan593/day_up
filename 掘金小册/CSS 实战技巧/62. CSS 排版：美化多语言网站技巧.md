![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b106b1278bde4f88a42e981b7ef14309~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=140048&e=webp&b=fcedbc)

  


如果你的业务需求是一个面向国际化的，那么你有时会遇到设计和开发上的文化差异。虽然 UI 的国际化使得目标受众的本地化变得更容易，但 HTML 和 CSS 也可以成为强大的工具，让我们在实现本地化和国际化的同时取得成功。

  


或许有很多开发者会认为 HTML 和 CSS 与国际化完全无关，但仔细一想一想，国际化不仅仅是将网站内容翻译成多种语言就结束了。内容的呈现方式中有许多细微差别，这些差别会影响本地语言使用者的体验。

  


换句话说，对于 Web 设计师来说，挑战在于设计时要考虑本地化；对于 Web 开发者来说，挑战在于通过最小的代码输入实现这些要求。这涉及的内容非常广泛，从 Unicode 和字符编码的使用，到翻译内容的技术实现，再到如何呈现这些内容。这节课，我只会与大家讨论与多语言相关的 CSS 技巧。CSS 提供了一些方法和技巧，可以在多语言页面上应用不同的样式。此外，CSS 为不同语言（书写系统）提供布局和排版功能。

  


简而言之，CSS 在样式化多语言 Web 网站或应用中有着意想不到的重要作用。在这节课中，让我们一起来看看如何通过 CSS 提供令人愉悦的本地化用户体验。你将会获得 CSS 样式化多语言网站的方方面面。我希望这节课的内容，能为 Web 设计师和开发者提供一些 CSS 的隐藏技巧，让你的 Web 网站或应用更具包容性，能够轻松面向全球用户。

  


## 快速了解一些术语

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/83abed27f0d44485af3088f3f0d18427~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1710&h=960&s=57761&e=jpg&b=e84930)

  


在谈到多语言 Web 网站或应用时，有几个术语你应该熟悉：

  


-   **页面语言**：页面内容所使用的语言
-   **浏览器区域设置**：定义用户的语言和地区
-   **书写模式**：文本的书写方向。例如英文是从左到右书写的，阿拉伯语是从右到左书写的，而一些中文和日文是从上到下书写的

  


除此之外，在构建一个多语言网站或应用时，还有“本地化”和“国际化”两个概念会常出现在耳旁：

  


-   **本地化（Localization，常缩写为l10n）** ：将产品或内容适配到特定地区或市场的过程。例如，使用正确的拼写、数字格式、货币等
-   **国际化（Internationalization，常缩写为i18n）** ：以一种可以轻松本地化的方式设计和开发产品或内容的过程。例如，使用正确的编码、支持多种文本方向的设计、具有弹性的布局等

  


**国际化 (i18n) 和本地化 (l10n)** 让我们的应用可以适应不同语言和地区。

  


## HTML 元素属性：`lang` 和 `dir`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/915965c2e5a84678a6720f07cb805e02~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2858&h=1502&s=39236&e=png&b=c2dfef)

  


你在使用 Chrome 浏览网站的时候，是否有尝试过使用 Google 翻译将整个网站翻译成另一种语言：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fb57a8d62f064eeb90530afdcb33eff3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1478&h=720&s=3072566&e=gif&f=104&b=f9f7f7)

  


是不是很好奇？Chrome 浏览器是如何知道询问你是否希望翻译网页内容的？嗯，我也很好奇，后来发现这是因为 `<html>` 元素上的 `lang` 属性！

  


`lang` 属性非常重要，因为它标识了网页上文本内容的语言，这些信息在很多地方都有用。比如 Chrome 浏览器内置的 Google 翻译功能、搜索引擎的语言特定内容，以及屏幕阅读器。

  


换句话说，在创建多语言网站或应用时，设置 `lang` 属性是最基本，也是最重要的一步。`lang` 是一个全局属性，这意味着它可以用在所有 HTML 标签上。通常情况下，一个页面中只会使用一种语言，因此 `lang` 属性最常见的用法是将其设置在 `<html>` 元素上：

  


```HTML
<html lang="en">
```

  


在 HTML 元素上设置 `lang` 属性时，我们通常使用 **[IETF BCP 47 语言标签](https://en.wikipedia.org/wiki/IETF_language_tag)**。BCP 47 标签以语言代码开头，例如 `en` 表示英语，`fr` 表示法语，`es` 表示西班牙语，并且可以包含可选的区域标识符。

  


设置 `lang` 属性可以让屏幕阅读器和其他辅助技术更容易正确发音。同时，这也会告诉搜索引擎页面的语言，有助于提升 SEO（搜索引擎优化）。

  


我们几乎可以在任何 HTML 元素上设置 `lang` 属性。例如，我们可以在一个主要是英文的页面中间插入一段韩文的段落。

  


```HTML
<html lang="en">
    <!-- ... -->
    <body>
        <p>An English paragraph.</p>
        <p>Another English paragraph.</p>
        <p lang="ko">한국어 문단입니다.</p>
    </body>
</html>
```

  


刚才提到过，`lang` 属性的值通常是 **[BCP 47 语言标签](https://en.wikipedia.org/wiki/IETF_language_tag)**，可以是类似 `en`（英语）或 `ko`（韩语）的语言标签。BCP 47 标签可以通过添加符合 **[ISO Alpha-2 标准](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2)** 的区域代码进一步指定（例如 `BH` 表示巴林，`CN` 表示中国，`US` 表示美国）。因此，一个完整的区域标识符可能像 `en-US`（美式英语）或 `zh-CN`（中国使用的中文）这样表示。

  


```HTML
<p lang="en">Some English text.</p>
<p lang="en-US">Some American English text.</p>
<p lang="en-GB">Some British English text.</p>
<p lang="en-CA">Some Canadian English text.</p>
```

  


构建多语言 Web 网站或应用的另一个重要属性是 `dir` 。由于语言不同，它们的书写方向和阅读查式也会有所不同，比如像英语、法语这样的语言是从左到右（`ltr`）阅读的，像阿拉伯语、希伯来语和乌尔都语这样的语言是从右到左（`rtl`）阅读的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/241a6845d47b4863ad9e8f8682c2498f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=1050&s=87226&e=jpg&b=091e36)

  


HTML 的 `dir` 属性可以很好地处理这些语言。`dir` 属性的值可以是 `ltr` （从左到右）、`rtl` （从右到左）或 `auto` （基于内容自动判断）。

  


通常，由于 Web 内容只使用一种语言，因此最常见的用法是将 `dir` 设置在 `<html>` 元素上：

  


```HTML
<html lang="ar" dir="rtl">
```

  


所有现代浏览器都会根据 `dir` 指定的方向来呈现页面内容。而且，[现代布局中的 CSS Flexbox 和 CSS Grid 布局技术可以根据 `<html>` 元素的 `dir` 属性重新排列页面方向](https://juejin.cn/book/7161370789680250917/section/7161625525763440647)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf9bd3537d464425aaea455c4483e7f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=4452&h=2720&s=247973&e=jpg&b=ffffff)

  


与 `lang` 属性类似，`dir` 也是一个全局属性，可以应用于任何 HTML 元素，而不仅限于文档根元素（`<html>` 元素）：

  


```HTML
<p dir="ltr">But I must explain ...</p>
<!-- 等效于 dir="ltr" -->
<p dir="auto">But I must explain...</p>

<p dir="rtl">لكن لا بد أن أوضح لك أن كل هذه الأفكار المغلوطة حول</p>
<!-- 等效于 dir="rtl" -->
<p dir="auto"> لكن لا بد أن أوضح لك أن كل هذه الأفكار المغلوطة حول</p>
```

  


当 `dir="auto"` 时，浏览器会根据元素中第一个具有强方向性的字符来设置元素的方向。

  


注意，如果需要在文本运行中覆盖文本方向，可以使用 `<bdo>` 元素；而在文本运行中隔离元素时，可以使用 `<bdi>` 元素。

  


## 与语言相关的样式

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c0c6db99529944c8aebdfaa18aded239~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2240&h=1260&s=41560&e=webp&b=ffffff)

  


CSS 提供了两种方式使得我们可以根据语言类型来设置样式。

  


首先，我们可以使用 `:lang()` 伪类选择器来根据语言类型设置样式。例如，`:lang(en)` 将选择任何具有 `lang="en"` 属性的元素：

  


```CSS
/* 全局选择器 👉 选择任何具有 lang="en" 的元素 */
:lang(en) {
    color: deepskyblue;
}

/* 元素选择器 👉 选择所有具有 lang="ko" 的段落 */
p:lang(ko) {
    color: deeppink;
    border: 1px solid deeppink;
}
```

  


使用 `:lang()` 伪类选择器时，一般的语言区域可以匹配更具体的语言区域。例如，`:lang(en)` 可以匹配带有 `lang="en"`、`lang="en-US"` 的元素：

  


```HTML
<p lang="en">Some English text.</p>
<p lang="en-US">Some American English text.</p>
<p lang="en-GB">Some British English text.</p>
<p lang="en-CA">Some Canadian English text.</p>
```

  


```CSS
:lang(en) {
    color: pink;
}
p:lang(en-CA) {
    color: #d80621;
}
```

  


`:lang(en)` 选择器可以匹配特定的英语区域，例如 `en-US`（美式英语）、`en-GB`（英式英语）和 `en-CA`（加拿大英语）。

  


在以上示例中，全局选择器 `:lang(en)` 设置了一般英语文本的颜色（`color`），但 `p:lang(en-CA)` 会为加拿大英语提供特定样式，覆盖全局选择器（`:lang(en)`）的效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8fc11dc858af459288b31278309dd1d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=992&s=179351&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/raBJzOe

  


更酷的是，即使 `lang` 声明在元素外部，它也能识别内容的语言。例如，一行包含两种语言的标记：

  


```HTML
<p>We use <em>italics</em> to emphasise words in English, <span lang="zh">但是中文则是用<em>着重号</em></span>.</p>
```

  


可以用以下样式来设置：

  


```CSS
em {
    font-style: italic;
}

em:lang(zh) {
    font-style: normal;
    text-emphasis: circle;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1c4915986e9b45669a625110dee633af~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=992&s=168094&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/qEWxXZX

  


正如你所看到的， `<em>` 元素包裹的英文文本带有倾斜样式，而包裹的中文文本添加了强调标记——圆形着重号。关键是，`lang` 属性并没有应用到 `<em>` 元素上，而是应用到它的父元素（`<span>`）上。`:lang()` 伪类选择器依然有效。这意味着，`lang` 属性不一定需要直接设置在元素上。如果某个元素从 DOM 的祖先元素继承了 `lang="zh"` ，那么 `:lang(zh)` 也会选择它。

  


第二种方式是使用 CSS 的属性选择器，这也是更常见的一种方式，因为大多数 Web 开发者并不了解或知道 `:lang()` 伪类选择器。

  


CSS 属性选择器允许我们选择具有特定属性或具有特定属性值的元素。例如，我们可以使用以下语法精确匹配 `lang` 属性的值：

  


```CSS
[lang="zh"] {
    /* 只会匹配 lang 属性等于 zh 的元素 */
}
```

  


需要知道的是，如果使用属性选择器的话，`lang` 属性必须应用在对应的元素上：

  


```HTML
<p>We use <em>italics</em> to emphasise words in English, <span lang="zh">但是中文则是用<em>着重号</em></span>.</p>

<p>We use <em>italics</em> to emphasise words in English, <span>但是中文则是用<em lang="zh">着重号</em></span>.</p>
```

  


```CSS
[lang="zh"] {
    font-style: normal;
    text-emphasis: circle;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7b6064a35cb640f29ee14b569d49ee75~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=992&s=277981&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/EaYQbjB

  


正如你所看到的，上面示例中的 `[lang="zh"]` 选中的是设置了 `lang="zh"` 的 `<span>` 和 `<em>` 元素。

  


之前提到过，`lang` 属性的值可以是一个带我区域标识符的值，例如 `lang="zh-Hans"` 、 `lang="zh-Hant"` 、`lang=zh-HK` 或 `lang="zh-TW"` 等。关键是，语言标签的长度不止两个字母。但最通用的类别总是排在最前面，因此，如果要匹配以特定字符串开头的属性值，我们可以使用带有正则符号的属性选择器：

  


-   **`[lang="zh"]`** ：完全匹配 `lang="zh"` 的元素；用于精确匹配语言代码为 `zh` 的元素
-   **`[lang^="zh"]`** ****：匹配以 `zh` 开头的 `lang` 属性值的元素；用于选择所有以 `zh` 开头的语言代码，例如 `zh`, `zh-cn`, `zh-tw` 等
-   **`[lang|="zh"]`** ：完全匹配 `lang="zh"` 或以 `zh-` 开头的元素；可以匹配特定语言及其子语言。例如，`zh` 是主语言代码，而 `zh-cn` 表示该主语言的子集
-   **`[lang*="zh"]`** ：只要 `lang` 属性值中包含 `zh` 的元素都能匹配上；用来选择任何包含 `zh` 的语言代码，位置不限，灵活性较大

  


例如：

  


```HTML
<!-- 1️⃣ [lang="zh"] -->
<p lang="zh">中文段落</p> <!-- 会被匹配 -->
<p lang="zh-cn">中文段落</p> <!-- 不会被匹配 -->

<!-- 2️⃣ [lang^="zh"] -->
<p lang="zh">中文段落</p> <!-- 会被匹配 -->
<p lang="zh-cn">中文段落</p> <!-- 会被匹配 -->
<p lang="en-zh">英文段落</p> <!-- 不会被匹配 -->

<!-- 3️⃣ [lang|="zh"] -->
<p lang="zh">中文段落</p> <!-- 会被匹配 -->
<p lang="zh-cn">中文段落</p> <!-- 会被匹配 -->
<p lang="en-zh">英文段落</p> <!-- 不会被匹配 -->

<!-- 4️⃣ [lang*="zh"] -->
<p lang="zh">中文段落</p> <!-- 会被匹配 -->
<p lang="zh-cn">中文段落</p> <!-- 会被匹配 -->
<p lang="en-zh">英文段落</p> <!-- 会被匹配 -->
```



你可以根据具体需求选择合适的选择器来匹配目标语言。相比而言，我个人更推荐使用 `:lang()` 选择器，因为它没有属性选择器那么复杂，也对目标元素是否设置 `lang` 属性没有那么严格。

  


## 与方向相关的样式

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f829930a20554bce9d8614b316e6be75~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=51013&e=jpg&b=903532)

  


前面提到过，我们可以通过 `dir` 属性来设置文本的阅读方式。比如：

  


```HTML
<p dir="ltr">But I must explain ...</p>
<!-- 等效于 dir="ltr" -->
<p dir="auto">But I must explain...</p>

<p dir="rtl">لكن لا بد أن أوضح لك أن كل هذه الأفكار المغلوطة حول</p>
<!-- 等效于 dir="rtl" -->
<p dir="auto"> لكن لا بد أن أوضح لك أن كل هذه الأفكار المغلوطة حول</p>
```

  


如果你在元素上忘记设置 `dir` 属性或不方便设置该值时，我们可以使用 CSS 的 `directiono` 属性来设置文本的阅读方式：

  


```HTML
<p lang="en">But I must explain ...</p>

<p lang="ar">لكن لا بد أن أوضح لك أن كل هذه الأفكار المغلوطة حول</p>
```

  


```CSS
:lang(en) {
    direction: ltr; /* Left-To-Right */
}

:lang(ar) {
    direction: rtl; /* Right-To-Left */
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b225708187044d798ad78049fcae831e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=766466&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/QwLQOZz

  


虽然 `direction` 可以根据语言类型来设置正确的阅读模式，但我还是推荐优先使用 HTML 的 `dir` 属性，它才是最佳实践。

  


在样式化多语言 Web 网站或应用时，不仅仅是根据语言设置正确的阅读方式，还有可能会根据语言的阅读方式设置不同的样式。这个时候，我们可以使用 CSS 的 `:dir()` 伪类选择器，它会根据浏览器解析的方向（而非 `dir` 属性的值）来样式化元素：

  


```CSS
p:dir(ltr) {
    padding-left: 2rem;
    border-left: 4px solid;
    color: #f50;
}

p:dir(rtl) {
    padding-right: 2rem;
    border-right: 4px solid;
    color: #00bcd4;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d8f1142ddc5415f9d5d17743ad63e46~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=721014&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/KwPQyLL

  


需要知道的是，`:dir()` 伪类选择器能够匹配到 `dir="auto"` ，因为它依赖浏览器解析的方向，而不是直接读取 `dir` 属性的值：

  


```HTML
<p lang="en" dir="auto">Author of "Modern CSS", "Modern Web Layouts", "Mastering Defensive CSS", "SVG Deep Dive", "Web Animation Journey", and "CSS in Action"! Well-versed in HTML, CSS, and A11Y with extensive hands-on experience. A CSS evangelist in China!</p>
<p lang="ar" dir="auto">مؤلف كتب "CSS الحديثة"، "تصميمات الويب الحديثة"، "إتقان CSS الدفاعي"، "الغوص العميق في SVG"، "رحلة في الرسوم المتحركة للويب"، و*"CSS في العمل"*! لديه معرفة واسعة وخبرة عملية في مجالات HTML وCSS وإمكانية الوصول (A11Y). مبشّر CSS في الصين!</p> 
```

  


```CSS
p:dir(ltr) {
    padding-left: 2rem;
    border-left: 4px solid;
    color: #f50;
}

p:dir(rtl) {
    padding-right: 2rem;
    border-right: 4px solid;
    color: #00bcd4;
}
```

  


最终所获得的结果与 `dir` 设置具体的方向值，比如 `ltr` 或 `rtl` 效果是一致的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/690838e2f21e4dc0b124c630af164527~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=721014&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/EaYQoYy

  


当然，要是你在 HTML 中显式指定了 `dir` 属性的值为 `ltr` 或 `rtl` ，也可以通过 CSS 的属性选择器来设置样式：

  


```HTML
<p lang="en" dir="ltr">Author of "Modern CSS", "Modern Web Layouts", "Mastering Defensive CSS", "SVG Deep Dive", "Web Animation Journey", and "CSS in Action"! Well-versed in HTML, CSS, and A11Y with extensive hands-on experience. A CSS evangelist in China!</p>
<p lang="ar" dir="rtl">مؤلف كتب "CSS الحديثة"، "تصميمات الويب الحديثة"، "إتقان CSS الدفاعي"، "الغوص العميق في SVG"، "رحلة في الرسوم المتحركة للويب"، و*"CSS في العمل"*! لديه معرفة واسعة وخبرة عملية في مجالات HTML وCSS وإمكانية الوصول (A11Y). مبشّر CSS في الصين!</p>
```

  


```CSS
p[dir="ltr"] {
    padding-left: 2rem;
    border-left: 4px solid;
    color: #f50;
}

p[dir="rtl"] {
    padding-right: 2rem;
    border-right: 4px solid;
    color: #00bcd4;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6fd842dd2b8b469daf16c348823f965f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=721014&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/vEBdpYd

  


不过，`[dir="ltr"]` 和 `[dir="rtl"]` 仅针对显式设置的方向，不会匹配 `dir="auto"`。

  


上面我们看到的书写模式要么是从左到右（`ltr`），要么是从右到左（`rtl`），然而，世界上还有一些语言并不都是水平方向书写的，也有垂直方向书写的，甚至两者都可以，比如许多东亚文字（例如中文、日文、韩文和越南文）既可以横向书写，也可以纵向书写。这些文字的传统排列方式是从上到下，然后从右到左：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/41c7b9429d2144e2999f8f41f6282d87~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1215&h=911&s=115474&e=webp&b=cccccc)

  


针对这种书写模式，我们可以通过 CSS 的 `writing-mode` 属性来实现，它支持三种书写模式：

  


-   `horizontal-tb`：横排，从左到右
-   `vertical-rl`：竖排，从右到左
-   `vertical-lr`：竖排，从左到右

  


例如：

  


```HTML
<div lang="zh-TW">
    <h1>水调歌头</h1>
    <p>明月几时有把酒问青天不知天上宫阙今夕是何年我欲乘风归去又恐琼楼玉宇高处不胜寒起舞弄清影何似在人间转朱阁低绮户照无眠不应有恨何事长向别时圆人有悲欢离合月有阴晴圆缺此事古难全但愿人长久千里共婵娟</p>
</div>
```

  


```CSS
div[lang="zh-TW"] {
    writing-mode: vertical-rl;
    font-size: clamp(1.25rem, 4vw + 0.875rem, 1.5rem);
    line-height: 3;
    max-inline-size: 18ic;
    letter-spacing: 1ic;
    font-weight: 900;
    background: linear-gradient(-135deg, #FF5722, #0bff07);
    -webkit-text-fill-color: transparent;
    background-clip: text;
}

h1 {
    text-align: center;
}
```

## 物理属性（值）vs. 逻辑属性（值）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9e5b038bd9f74c089de0c2723f27b1e7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1384&s=34231&e=png&b=fcfcfc)

  


传统上，CSS 中控制元素大小、位置和间距的布局属性是物理属性，比如 `border-left` 、`width` 等。它不会考虑 `dir` （文本方向）和 `writing-mode` （书写模式）。例如，`border-left` 只会给元素设置边框，无论文本方向如何。如果需要兼容从左到右（`ltr`）和从右到左（`rtl`）的布局，就必须分别为每种方向编写不同的 CSS 规则：

  


```CSS
p:dir(ltr) {
    padding-left: 2rem;
    border-left: 4px solid;
    color: #f50;
}

p:dir(rtl) {
    padding-right: 2rem;
    border-right: 4px solid;
    color: #00bcd4;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/050e0a0aba5e42f3beda7c7a8db037f1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=721014&e=jpg&b=000000)

  


随着 **[CSS 逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)** 的引入，我们可以将 CSS 代码变得更为简单，因为逻辑属性和逻辑值会根据元素的 `dir` 和 `writing-mode` 自动适配方向。下面是使用逻辑属性实现相同效果的例子：

  


```CSS
p {
    padding-inline-start: 2rem;
    border-inline-start: 4px solid;
    inline-size: clamp(300px, 50vw, 76ch);
}
  
:dir(ltr) {
    color: #f50;
}

:dir(rtl) {
    color: #00bcd4;
}
```

  


在 `ltr` 方向，元素拥有左边框和左内距；在 `rtl` 布局中，它被解释为右边框和右内距。我们只需要使用 `padding-inline-start` 和 `border-inline-start` ，浏览器会根据容器的文本方向自动解析：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e1e7af812cfe444097f4547991c6a0dd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=662714&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/vEBdpdz

## 布局：CSS Flexbox 和 CSS Grid

到目为止，**[CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)** 和 **[CSS Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)** 应该是最主流的布局技术。这两种布局技术能帮助我们快速构建多语言网站或应用的布局，因为它们完美的融合了文本方向（`dir`）和书写模式（`writing-mode`）。换句话说，我们不需要根据语言的类型提供额外的布局代码。以下面这个示例为例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0db8f4aea7b14914a4da891c35394553~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1270&s=451245&e=jpg&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/ByBYJOX

  


构建上面这样的布局效果，你可能会使用像下面这样的 HTML 结构：

  


```HTML
<div class="card" lang="en" dir="ltr">
    <img src="http://i.pravatar.cc/300?img=2" alt="">
    <p >Author of "Modern CSS", "Modern Web Layouts", "Mastering Defensive CSS", "SVG Deep Dive", "Web Animation Journey", and "CSS in Action"! Well-versed in HTML, CSS, and A11Y with extensive hands-on experience. A CSS evangelist in China!</p>
</div>
<div class="card" lang="ar" dir="rtl">
    <img src="http://i.pravatar.cc/300?img=3" alt="">
    <p>مؤلف كتب "CSS الحديثة"، "تصميمات الويب الحديثة"، "إتقان CSS الدفاعي"، "الغوص العميق في SVG"، "رحلة في الرسوم المتحركة للويب"، و*"CSS في العمل"*! لديه معرفة واسعة وخبرة عملية في مجالات HTML وCSS وإمكانية الوصول (A11Y). مبشّر CSS في الصين!</p>
</div>
```

  


使用 CSS Flexbox 或 CSS Grid 来构建布局，我们不需要依赖 `:dir()` 选择器为不同的语言编写不同的 CSS 代码，我们可以像下面这样编写 CSS 代码，就能同时满足 `ltr` 和 `rtl` 阅读模式的布局效果：

  


```CSS
.card--flexbox {
    display: flex;
    gap: 1.5rem;
    align-items: flex-start;
    
    img {
        flex-shrink: 0;
        width: 120px;
    }
    
    p {
        flex: 1;
        min-inline-size: 0;
    }
}
  
.card--grid {
    display: grid;
    gap: 1.5rem;
    grid-template-columns: 120px minmax(0, 1fr);
    align-items: start;
    
    
    img {
        width: 100%;
    }
}
```

## 字体

  


让我们换个角度来探讨多语言（本地化或国际化）和排版。在选择字体时，我们需要选择支持目标语言所有字符的字体，例如拉丁文、阿拉伯文、韩文等。一些常见的选择包括**系统字体**、**多语言字体**以及**Google Noto**字体。

  


某些字何可能仅支持少数几种语言，而系统字体往往具有广泛的语言覆盖范围，并且预装在操作系统中，因此在多语言网站中使用是比较安全的选择，以下是主流操作系统的默认系统字体：

  


-   **Windows** 默认字体：Segoe UI
-   **macOS 和 iOS** 默认字体：San Francisco
-   **Android** 默认字体：Roboto

  


下面是一个常见的系统字体栈：

  


```CSS
html {
    font-family: system-ui, "Segoe UI", Roboto, Helvetica, Arial, sans-serif, "Apple Color Emoji", "Segoe UI Emoji";
}
```

  


不同操作系统的系统字体能显示多种语言的字符，适合多语言场景。

  


虽然系统字体安全可靠，但不一定能完全符合品牌设计的需求，为了在不同的操作系统中提供一致的用户体验，可以选择多语言字体。这些字体根据需要支持多种语言。比如 `Helvetica World` 、`Neue Helvetica World` 、`Greta Sans` 和 `Google Noto` 等。

  


你可以使用 `:lang()` 伪类选择器为不同的语言设置字体：

  


```CSS
:lang(en) {
    font-family: "Noto Serif", serif;
}

:lang(ar) {
    font-family: "Noto Naskh Arabic", "Noto Serif", serif;
}

:lang(jp) {
    font-family: "Noto Serif JP", sans-serif;
}
```
