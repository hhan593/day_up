HTML 表格（`<table>`）几乎与 Web 本身一样古老，多年来一直被使用——甚至被滥用。在 Web 早期，在没有其他合适方法的情况，Web 开发者通过表格拼凑出复杂的布局，这种布局技术也被称为表格布局，而且还用了很多年。直到 CSS 提供了更多 **[Web 布局技术](https://s.juejin.cn/ds/iU7PgNTE/)** 之后，使用表格布局就已不再常见，但表格仍然是 Web 的重要组成部分，对于展示二维数据（按行和列组织的数据）至关重要，这在 toB 的业务场景（比如，后台数据）非常常见。

  


就 Web 布局而言，[现代 Web 布局技术](https://s.juejin.cn/ds/iU7PgNTE/)（比如， [CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 和 [CSS Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648) 等）要比表格布局灵活的多，正因为如此，网上有一个小小的悖论：在 CSS Grid 布局出现之前，我们使用 `<table>` 来模拟网格布局；而现在，当我们有了 CSS Grid 布局之后，却用它来模拟表格！尤其是在创建响应式表格的时候，这种现象更为普遍。我想，你曾经或现在正在使用 CSS Grid 来模拟表格布局。

  


我不能明确的说这种做法是否正确。如果从语义化或 Web 或访问性的角度来说，`<table>` 是为表格化数据（Tabular Data）设计的，它用来展示二维数据是最合适不过；而用一堆诸如 `<div>` 元素来呈现表格化数据是没有意义的。庆幸的是，[W3C ARIA 规范](https://www.w3.org/TR/wai-aria/)，提供了一些属性，比如 `role` 、`aria-label`、`aria-sort`、`aria-rowindex`、`aria-labelledby` 和 `aria-describedby` 等，可以用来增强数据表格的可访问性。这使得 CSS Grid 来模拟表格在可访问性方面得到极大的改善。只不过，要是不能正确使用 ARIA 属性的话，反而会给 Web 可访问性带来极大的灾难。也就是说，**如果不到万不得已，我是不建议使用 CSS Grid 来模拟表格的**。

  


回过头来说，造成这种不规范做法的原因可能是 CSS 在样式表格时有点棘手。接下来，我将介绍一些在 HTML 和 CSS 中构建简单和复杂表格的技巧和注意事项，以及现代 CSS 如何为我们提供帮助。

  


我们开始吧！

  


## 从表格结构说起

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1a9b14dad58b4c5d8498ffe51ca8bc7d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=475181&e=png&b=051140)

  


我们从 HTML 标记开始。即使没有额外的样式，我们的表格也能在视觉上清晰易懂，同时支持屏幕阅读器解析。

  


编写一个具有语义化的 HTML 是一名 Web 开发者的基本素养，它使得使用辅助技术的人更容易理解内容，同时也使搜索引擎更容易理解内容。以下是一些可用于数据表格可访问的语义化 HTML 元素：

  


-   `<table>`：用于定义表格；它可以通过提供表格的结构来使数据表格可访问
-   `<caption>`：用于为表格提供标题；它可以通过描述表格的目的来使数据表格可访问
-   `<thead>`：用于定义表格的头部；它可以通过提供表格头部的结构来使数据表格可访问
-   `<tbody>`：用于定义表格的主体；它可以通过提供表格主体的结构来使数据表格可访问
-   `<tfoot>`：用于定义表格的脚部；它可以通过提供表格脚部的结构来使数据表格可访问
-   `<tr>`：用于定义表格中的一行；它可以通过提供表格行的结构来使数据表格可访问
-   `<th>`：用于定义表格中的头部单元格；它可以通过提供表格头部单元格的结构来使数据表格可访问
-   `<td>`：用于定义表格中的数据单元格；它可以通过提供表格数据单元格的结构来使数据表格可访问

  


而一个典型的 HTML 数据表格由以下元素组成：

  


-   `<table>`：用于定义数据表格；包含 `<thead>`、`<tbody>` 和 `<tfoot>` 元素
-   `<thead>`：用于定义数据表格的头部；包含 `<tr>` 元素，`<tr>` 元素包含 `<th>` 元素
-   `<tbody>`：用于定义数据表格的主体；包含 `<tr>` 元素，`<tr>` 元素包含 `<td>` 元素
-   `<tfoot>`：用于定义数据表格的脚部；包含 `<tr>` 元素，`<tr>` 元素包含 `<td>` 元素

  


通常情况之下，我们构建一个基本表格只需要 `<table>` 、`<tr>` 、`<th>` 和 `<td>` 四个元素即可。例如：

```HTML
<table>

        <th>Amount</th>

        <td>Seoul</td>
        <td>17 Dec, 2024</td>
        <td>Delivered</td>
        <td>$128.90</td>
    </tr>    
</table>    
```

如果你不太关注 Web 可访问性，那么 HTML 创建一个表格到这就可以结束了！然而，可访问性是 Web 设计的重要方法，也是 Web 开发者应该具有的最基本素养之一。因为它可以确保有障碍的人士也能访问和使用 Web ，同时改善所有用户的体验，包括有障碍的用户。

  


展示大量信息的数据表格，对于有障碍的人来说，可能会很难导航并找到所需的信息。使用屏幕阅读器（如 AWS、NVDA 和 VoiceOver）测试你的数据表格可能会很有帮助。利用这类辅助技术来检测表格的内容和结构，可以帮助你识别潜在的问题，确保表格能被视力障碍者正确解读和导航。

  


同时，使用键盘导航测试数据表格也是一个好主意。具有运动障碍的用户可能会遇到使用鼠标或其他指针设备的困难，因此他们可能需要依赖键盘导航来浏览表格数据。

  


我们在创建表格的结构时，有几种方法可以使数据表格变得可访问，让我们来看一下。

  


### 使用 `scope` 属性

  


为了确保辅助技术（如屏幕阅读器）能够轻松解读表格内容，我们可以为表头单元格添加 `scope` 属性，并赋值为 `row` 或 `column` ，以指明表头所属的轴：

  


-   `row`：表示头部单元格适用于一行数据单元格；它将头部单元格与同一行中的数据单元格关联
-   `col`：表示头部单元格适用于一列数据单元格；它将头部单元格与同一列中的数据单元格关联

  


例如：

  


```HTML
<table>
    <caption>Details of customer orders!</caption>
    <colgroup><col><col><col><col><col><col></colgroup>
    <thead>
        <tr>
            <th scope="column">Id</th>
            <th scope="column">Customer</th>
            <th scope="column">Location</th>
            <th scope="column">Order Date</th>
            <th scope="column">Status</th>
            <th scope="column">Amount</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>Zinzu Chan Lee</td>
            <td>Seoul</td>
            <td>17 Dec, 2024</td>
            <td>Delivered</td>
            <td>$128.90</td>
        </tr>   
    </tbody> 
</table> 
```

  


有些表格可以行和列都会带有表头（`<th>` 元素），那么我们可以为所在行的 `<th>` 元素添加 `scope="row"` 。例如：

  


```HTML
<table>
    <caption>月度销售报告</caption>
    <colgroup><col><col><col><col></colgroup>
    <thead>
        <tr>
            <th scope="col">月份</th>
            <th scope="col">产品A</th>
            <th scope="col">产品B</th>
            <th scope="col">产品C</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">一月</th>
            <td>$1000</td>
            <td>$1500</td>
            <td>$1200</td>
        </tr>
    </tbody>
</table>
```

  


上面的表格展示了月度销售报告，包括三个产品（A、B 和 C）以及每个月的销售数据。`scope` 属性被用于 `<th>` 元素，以定义头部单元格的作用范围。

  


-   `scope="col"` 属性用于表格头部行 `<thead>` 中的头部单元格，表示每个头部单元格适用于一列。
-   `scope="row"` 属性用于表格主体行 `<tbody>` 中的头部单元格，表示每个头部单元格适用于一行。

  


通过适当使用 `scope` 属性，辅助技术可以将头部单元格与其对应的数据单元格关联，增强表格的可访问性，帮助有障碍的用户更好地理解表格结构。

  


`scope` 属性除了 `row` 和 `column` 两个值之外，还有 `rowgroup` 和 `colgroup` ：

  


-   `rowgroup`：表示头部单元格适用于一组行
-   `colgroup`：表示头部单元格适用于一组列

  


### 使用 `summary` 属性

  


`summary` 属性用于提供表格的目的和结构的总结。它还帮助辅助技术（如屏幕阅读器）理解表格的目的和结构。以下是如何使用 `summary` 属性的示例：

  


```HTML
<table summary="Details of customer orders">
    <caption>Details of customer orders!</caption>
    <colgroup><col><col><col><col><col><col></colgroup>
    <thead>
        <tr>
            <th scope="column">Id</th>
            <th scope="column">Customer</th>
            <th scope="column">Location</th>
            <th scope="column">Order Date</th>
            <th scope="column">Status</th>
            <th scope="column">Amount</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>Zinzu Chan Lee</td>
            <td>Seoul</td>
            <td>17 Dec, 2024</td>
            <td>Delivered</td>
            <td>$128.90</td>
        </tr>   
    </tbody> 
</table> 
```

  


上面代码中，`summary` 属性被添加到 `<table>` 元素，并将其值设置为 `Details of customer orders` 。当辅助技术遇到 `summary` 属性的表格时，它们可以读取该总结，帮助用户了解表格的内容和目的。这对于依赖屏幕阅读器来浏览 Web 内容的视觉障碍用户尤其有价值。

  


包括有意义且信息丰富的 `summary` 属性是增强数据表格可访问性的重要做法，因为它允许用户快速了解表格的目的和上下文，而无需分析整个内容。

  


### ARIA 属性

  


ARIA（可访问的富互联网应用）属性也用于增强 Web 内容的可访问性。它们提供有关 Web 内容目的和结构的附加信息，使辅助技术能够更好地理解和导航内容。

  


一个常用的 ARIA 属性是 `role` 属性。该属性指定元素的语义角色，帮助辅助技术理解其目的。对于数据表格，`role` 属性通常设置为 `table` ，表示该元素代表一个表格。以下是如何使用 `role` 属性与表格的示例：

  


```HTML
<table summary="Details of customer orders" role="table">
    <caption>Details of customer orders!</caption>
    <colgroup><col><col><col><col><col><col></colgroup>
    <thead role="rowgroup">
        <tr role="row">
            <th role="columnheader" scope="column">Id</th>
            <th role="columnheader" scope="column">Customer</th>
            <th role="columnheader" scope="column">Location</th>
            <th role="columnheader" scope="column">Order Date</th>
            <th role="columnheader" scope="column">Status</th>
            <th role="columnheader" scope="column">Amount</th>
        </tr>
    </thead>
    <tbody role="rowgroup">
        <tr role="row">
            <td role="cell">1</td>
            <td role="cell">Zinzu Chan Lee</td>
            <td role="cell">Seoul</td>
            <td role="cell">17 Dec, 2024</td>
            <td role="cell">Delivered</td>
            <td role="cell">$128.90</td>
        </tr>   
    </tbody> 
</table> 
```

  


如果你是使用 `<div>` 来模拟 `<table>` ，那么 `role` 属性更是必不可少的部分：

  


```HTML
<div role="table">
    <div role="rowgroup">
        <div role="row">
            <div role="columnheader">Id</div>
            <div role="columnheader">Customer</div>
            <div role="columnheader">Location</div>
            <div role="columnheader">Order Date</div>
            <div role="columnheader">Status</div>
            <div role="columnheader">Amount</div>
        </div>
    </div>
    <dov role="rowgroup">
        <div role="row">
            <div role="cell">1</div>
            <div role="cell">Zinzu Chan Lee</div>
            <div role="cell">Seoul</div>
            <div role="cell">17 Dec, 2024</div>
            <div role="cell">Delivered</div>
            <div role="cell">$128.90</div>
        </div>   
    </div> 
</div> 
```

  


还有其他几个 ARIA 属性可以用来增强数据表格的可访问性，例如：

  


-   `aria-label`：用于为元素提供标签。可以用来为表格、行、列或单元格提供标签
-   `aria-sort`：用于指示可排序列的默认排序顺序。它可以具有值，如 `ascending`（升序）、`descending`（降序）或 `none`（无）
-   `aria-rowindex`：用于指示表格中行的索引
-   `aria-labelledby`：用于通过另一个元素的 `ID` 为元素提供标签。可以用来为表格、行、列或单元格提供标签
-   `aria-describedby`：用于通过另一个元素的 `ID` 描述元素。可以用来描述表格、行、列或单元格

  


还有许多其他 ARIA 属性可以用来增强数据表格的可访问性。更多信息，请参考[ W3C ARIA 规范](https://www.w3.org/TR/wai-aria/)。

  


如此一来，我们就创建了一个具有可访问性的表格。另外，将数据表格组织到适当的 HTML 元素中，我们可以为应用 CSS 样式和实现响应式设计技术奠定坚实的基础。这种结构化的方法使得管理数据表格的内容、增强其功能和优化其在不同设备和屏幕大小上的显示变得更加容易。

  


## 美化表格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/acf9ee7012324336bf4a15b2c55d3d43~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3332&h=1760&s=227691&e=jpg&b=3e4357)

  


刚才提到过，一直以来表格不受 Web 开发青眯，是因为使用 CSS 美化表格是件棘手的事情。造成这个现象的主要原因是大家对于 CSS 样式化表格不太熟悉，以及表格自身在渲染的时候与我们熟悉的 `<div>` 等元素有诸多的不同。接下来，我们就来聊聊 CSS 样式化表格常碰到的问题，以及如何使用现代 CSS 来帮助我们将表格制作的更美观。

  


### 用户代理样式

  


默认情况之下，在浏览器中查看我们的表格时，它的外观还远远称不上好看。标题和表格标题（`<caption>`）都居中对齐，无法区分表头、表尾和主体的行：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd397c619da948c28e6ab7c070148d5d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=884&s=107746&e=jpg&b=f4f4f4)

  


如今，各大浏览器对表格的默认样式已经比较一致。如果我们使用 Chrome 浏览器的开发者开具来审查创建表格的元素，可以看到如下样式被应用：

  


```CSS
table {
    display: table;
    border-collapse: separate;
    box-sizing: border-box;
    text-indent: initial;
    unicode-bidi: isolate;
    border-spacing: 2px;
    border-color: gray;
}

caption {
    display: table-caption;
    text-align: -webkit-center;
    unicode-bidi: isolate;
}

colgroup {
    display: table-column-group;
    unicode-bidi: isolate;
}

col {
    display: table-column;
    unicode-bidi: isolate;
}

thead {
    display: table-header-group;
    vertical-align: middle;
    unicode-bidi: isolate;
    border-color: inherit;
}

tbody {
    display: table-row-group;
    vertical-align: middle;
    unicode-bidi: isolate;
    border-color: inherit;
}

tfoot {
    display: table-footer-group;
    vertical-align: middle;
    unicode-bidi: isolate;
    border-color: inherit;
}

tr {
    display: table-row;
    vertical-align: inherit;
    unicode-bidi: isolate;
    border-color: inherit;
}

th {
    display: table-cell;
    vertical-align: inherit;
    font-weight: bold;
    text-align: -internal-center;
    unicode-bidi: isolate;
}

td {
    display: table-cell;
    vertical-align: inherit;
    unicode-bidi: isolate;
}
```

  


首先使你感到陌生的是表格元素的 `display` 属性，基本上每个元素的 `display` 属性值都有所不同。不过，仔细观察，它与表格元素功能有着相互映射的关系，比如 `<table>` 元素的 `display` 是 `table`，`<tr>` 元素的 `display` 是 `table-row` ，相对而言还是很好理解的。

  


通常情况之下，我们使用 CSS 来美化表格时，是不需要重置它们的 `display` 属性的值。当然，有些 Web 开发者可能对这些[上下文视觉模型格式](https://juejin.cn/book/7199571709102391328/section/7199844993073152038)不太熟悉，并且为了使样式设计更简单，会考虑将它们的 `display` 重置为别的上下文视觉模型格式，比如 `grid` 。仅从视觉实现的角度来看，没有什么，但要知道的是，这么做会破坏浏览器内置的可访问性功能，使依赖辅助技术的用户无法访问表格内容。这一点切记！

  


另外，很多时候我们并需要太多的关注浏览器给表格元素设置的默认样式。因此，某些默认样对我们来说是有用的。例如，浏览器会为 `<th>` 元素应用 `font-weight: bold` ，在没有其他样式的情况下就能将表头与表格其他单元格（`<td>`）区分开。这种设计让我们能够清楚地看到行和列标题与对应数据的关系，为用户提供了最低限度的可用体验。

  


当然，你也可能会认为浏览器给表格设置的默认样式不太友好。那么，你可以应用一些基础的 CSS 来“重置”表格样式，从而更方便地进行自定义设计：

  


```CSS
table, caption, tbody, tfoot, thead, tr, th, td {
    margin: 0;
    padding: 0;
    border: 0;
    font-size: 100%;
    font: inherit;
    vertical-align: baseline;
}

table {
    border-collapse: collapse;
    border-spacing: 0;
}
```

  


我想提醒大家的是，“重置”表格默认样式好不好，没有绝对性的答案，大家应该根据实际场景来决定。比如，你想创建一个分离式表格或者想给表格添加圆角时，就不需要重置 `border-collapse` 和 `border-spacing` 属性。大家应该根据自己应用的场景来决定，选择适用大多数场景的方案。

  


### 给表格添加边框

  


假设我们想要给表格添加边框。首先想到的是像下面这样设置 CSS：

  


```CSS
:is(table,th, td) {
    border: 1px solid #333;
}
```

  


你会发现，单元格之间以及单元格与表格边框之间会有一个小间距存在：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9009d42ffb7148cc9dadd3bfe5b857d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1014&s=360316&e=jpg&b=f4f4f4)

  


产生这种现象是因为浏览器给 `border-spacing` 属性设置了一个 `2px` 值。如果你不希望这个间距存在，可以将其重置为 `0` ：

  


```CSS
table {
    border-spacing: 0px;
}

:is(table, th, td) {
    border: 1px solid #333;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/07acd2b70d8b43cfa577d165e31adbc6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1014&s=343410&e=jpg&b=f4f4f4)

  


现在单元格之间的空白间距不存在了。这也是为什么大家喜欢将 `table` 的 `border-spacing` 属性重置为 `0` 的主要原因。因为我们创建的表格，在大多数的时候是不需要这个微小的间距（`2px`）存在的。当然，我们在制作一些其他 UI 风格的表格，这个属性还是非常有用的，比如分离表格的行和（或）列，它就非常有用。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d295aab1780404ead0bc23ec13645cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=580&s=1421422&e=gif&f=149&b=f2f2f2)

  


不难发现，`border-spacing` 属性的值越大，它们之间的间距就越大，它也使表格的行和列达到分离的效果。这个我们稍后还会介绍，我们还是把注意力拉回表格的边框上来。

  


眼尖的同学应该发现了，虽然单元格之间的间距不存在了（`border-spacing:0`），但边框的厚度明显变粗了，并不是我们设置的 `1px` 。为了避免这个现象存在，我们可以将 `table` 的 `border-collapse` 属性设置为 `collapse`，替代默认值 `separate`。这样可以让表格单元格共享边框，从而实现更符合预期的表格外观。

  


```CSS
table {
    border-spacing:0;
    border-collapse: collapse;
}

:is(table, th, td) {
    border: 1px solid #333;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f9550e60b0214cc38e601f96baa5d9fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1180&h=586&s=1637581&e=gif&f=168&b=f3f3f3)

  


注意，当 `table` 设置 `border-collapse: collapse;` 时，它的 `border-spacing` 属性将失去作用，即使设置大于 `0` 的值，表格单元格之间也不会有任何的间距存在：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eaf10c8c847449b0832f707f8016e045~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1178&h=618&s=4645928&e=gif&f=246&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/wBweLdB

  


有的时候，可能只希望给表格的行与行之间有分隔线。你可能会直接给表格的行 `<tr>` 设置边框：

  


```CSS
:is(table, tr) {
    border: 1px solid #333;
}
```

  


但是你会发现，它并没有生效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e25beadeb5f04c83b47320072524b580~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=616&s=7728776&e=gif&f=126&b=f3f3f3)

  


我们可以选择给 `th` 和 `td` 设置 `border-bottom` 或 `border-top` 。例如：

  


```CSS
thead th {
    border-bottom: 1px solid #333;
}

tbody tr:not(:last-child) {
    :is(th, td) {
        border-bottom: 1px solid #333;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/030494b3d5f843e7bf14b59705464840~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1182&h=624&s=5519662&e=gif&f=99&b=f3f3f3)

  


这里还有一个小技巧，将 `table` 的 `border-collapse` 设置为 `collapse` 后，就可以直接在 `tr` 上设置边框，而不需要单独针对每个单元格设置边框：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: var(--border-collapse, collapse);
}

:is(table, tr) {
    border: 1px solid #333;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/38f7e48353b14d63ac7a5737e34c99bc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=628&s=5846394&e=gif&f=179&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/YPKxKje

  


反过来，如果你只希望表格列与列之间添加分隔线，那怎么办呢？粗暴的做法是给 `th` 或 `td` 添加 `border-left` 或 `border-right` ：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: collapse;
    border: 1px solid #333;
}

:is(th, td):not(:last-child) {
    border-right: 1px solid #333;
} 

/* 或者 */
:is(th, td):not(:first-child) {
    border-left: 1px solid #333;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0d6de4d7f4d14524a96ca818944b5c51~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1186&h=624&s=5696429&e=gif&f=105&b=f3f3f3)

  


另一个小技巧是，我们可以使用 `col` 为例添加分隔线：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: collapse;
    border: 1px solid #333;
}

col {
    border: 1px solid #333;
}
```

  


所获得的效果是一样的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3da790fb940403b88079630e2706f3e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1186&h=624&s=5696429&e=gif&f=105&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/zxOdObN

  


### 样式化表格标题

  


默认情况之下，表格标题（`<caption>`）会居中对齐，并且紧挨着表头。我们可以像下面这样来样式化表格标题：

  


```CSS
caption {
    text-align: start; /* left */
    margin-block-end: 1rem;
    font-size: clamp(1.25rem, 4vw + 1rem, 1.5rem);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e608bb3e0f694c628542097fae8ebf96~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1084&s=353246&e=jpg&b=f4f4f4)

  


如果我们希望表格标题（`<caption>`）显示在表格内容底部，可以使用 `caption-side` 属性。这种方式既能调整视觉顺序，也不会影响辅助技术对表格元素的感知顺序。

  


```CSS
table {
    caption-side: bottom;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5d73a56b31dd4bf0b419804d382b0919~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1188&h=622&s=7764367&e=gif&f=262&b=f2f2f2)

  


> Demo 地址：https://codepen.io/airen/full/RNbZbXY

  


到目前为止，我们使用几行简单的 CSS 使整个表格有了一个看上去还不错的效果：

```CSS
table {

caption {

}

    padding: 1rem;
}
```

### 给表格着色

  


有的时候，你创建的表格可能会层叠在一个丰富的背景之上，这个时候仅仅给表格设置边框效果可能无法给用户提供很好的浏览体验：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c22793b992854bdc908cb7e70798744c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=624&s=9570550&e=gif&f=53&b=050f1b)

  


或者说，你的需求就是要创建一个色彩丰富的表格，例如下图这种带有毛玻璃背景效果的表格：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd4391cda94542faa42f267a64ca4b3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3326&h=1742&s=558551&e=jpg&b=d3cbd9)

  


接下来，我们的目标就是使用 CSS 使我们的表格拥有相似和色彩效果。

  


默认情况下，表格以及它的行和单元格背景都是透明。在 CSS 中，我们可以像给其他元素设置背景颜色一样，直接给 `table` 添加一个背景色，它可以是图片、渐变和纯色：

  


```CSS
table {
    --bg:lch(84 6.75 298.51);
    background-color: var(--bg);
    color: color-contrast(var(--bg) vs #333, #fff to AAA);
}
```

  


上面代码通过 `color-contrast() 函数来设置 color 的颜色`，这样可以确保文本颜色与背景颜色对比度始终达到 `AAA` 级别，它会根据 `--bg` 变化，从 `#333, #fff` 找到最合适的一个颜色作为文本颜色。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ae9e373b6feb452fa2ddd0456b715492~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1194&h=626&s=13755883&e=gif&f=116&b=cec7d7)

  


正如你所看到的，表格背景是一个纯色。我们的目标是使表格带有毛玻璃的背景效果，这在 CSS 中很容易能实现，我们只需要将表格的背景颜色增加一定的透明度，并利用 **[CSS 处理图像](https://juejin.cn/book/7223230325122400288/section/7259669043622690853)** 相关的特性，比如 [CSS 的 `backdrop-filter` 属性](https://juejin.cn/book/7223230325122400288/section/7259669043622690853#heading-48)，使背景变得模糊起来：

  


```CSS
table {
    --bg:lch(84 6.75 298.51 / 35%);
    background-color: var(--bg);
    color: #fefefe;
    backdrop-filter: blur(5px);
}
```

  


就是这么简单，这个时候呈现给你的表格是带有毛玻璃效果的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/abfec9fc20f040dfba6c4d6015176a2f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=622&s=14248890&e=gif&f=109&b=53596b)

  


> Demo 地址：https://codepen.io/airen/full/RNbZNLa

  


事实上，给整个表格着色并不能提高表格的可读性。通常情况之下，为表格行交替设置背景色（即斑马条纹），是提高表格可读性的另一种方式，这种方式往往可以作为表格边框的替代方案。我们的目标效果就是这种设计。

  


在为表格添加斑马条纹效果之前，我们先来美化一下表格标题的背景。为了美化表格标题背景，我使用了一个额外的容器包裹了整个表格，并将毛玻璃的效果应用到这个新增的容器上：

  


```HTML
<div class="container">
    <table summary="Details of customer orders" role="table">
        <caption>Details of customer orders!</caption>
        <!-- 省略其他的 HTML -->
    </table>
</div>
```

  


```CSS
.container {
    background-color: #fff5;
    backdrop-filter: blur(7px);
    box-shadow: 0 .4rem .8rem #0005;
    padding: 0 1rem 1rem;
    border-radius: .8rem;
}

table {
    border-spacing: 0px;
    border-collapse: collapse;
    font-family: system-ui;
    color: #fefefe;
}

caption {
    margin-block-end: 1rem;
    margin-inline: -1rem;
    padding: 1rem;
    background: #fff4;;
    color: #333;
    border-radius: .8rem .8rem 0 0;
}
```

  


请注意应用在 `caption` 元素上的 `margin-inline` 和 `padding` 。由于表格标题（`<caption>`）的宽度与 `table` 是相等的，为了使其宽度能与容器 `.container` 的相等，采用负的 `margin-inline` 值将其往两端拉伸。在视觉上，它与 `.container` 左右两端的边缘相吻合。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/126ce13b5ab249e9b949b2f09600761b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1126&h=616&s=16862666&e=gif&f=118&b=595f6d)

  


接着，给表头和表尾（如果有的话）着色：

  


```CSS
table {
    --c:lch(84 6.75 298.51);
    
    :is(thead, tfoot) {
        background-color: var(--c);
        color: color-contrast(var(--c) vs #333, #fff to AAA);
    }
}
```

  


在我们这个示例中，只有表头，因此你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/429d0ae1d14842d491bcc8d158e253e1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1140&s=735947&e=jpg&b=606774)

  


现在，剩下的就是给表格主体的行着色了，我在这里使用 CSS 的 `color-mix()` 函数为交替行生成更透明的背景颜色，从而实现更柔和的着色效果：

  


```CSS
tbody tr {
    background-color: color-mix(in srgb, var(--c), #0000 60%);

    &:nth-child(odd) {
        background-color: color-mix(in srgb, var(--c), #0000 40%);
    }
}
```

  


同样的，可以使用 `transition` 在 `:hover` 状态动画化表格主体行的背景颜色：

  


```CSS
tbody tr {
    transition: .5s ease-in-out .2s, background-color 0s;
    background-color: color-mix(in srgb, var(--c), #0000 60%);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
    cursor: pointer;

    &:nth-child(odd) {
        background-color: color-mix(in srgb, var(--c), #0000 40%);
    }
    
    &:hover {
        background-color: color-mix(in srgb, var(--c), #0000 20%);
    }
}
```

  


这样你就拥有一个行斑马条效果的表格：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e5f7a2ad021f4f70b8a3f504c307d096~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1124&h=616&s=10458784&e=gif&f=125&b=908b99)

  


> Demo 地址：https://codepen.io/airen/full/raBzVBX

  


如果你想要创建一个列的斑马条纹表格效果，可以像下面这样设置 `<col>` 样式：

  


```CSS
col {
    transition:  background-color .2s ease-in-out;
    background-color: color-mix(in srgb, var(--c), #0000 60%);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
    cursor: pointer;

    &:nth-child(odd) {
        background-color: color-mix(in srgb, var(--c), #0000 40%);
    }
    
    &:hover {
        background-color: color-mix(in srgb, var(--c), #0000 20%);
    }
}
```

  


上面代码中，我们给 `col:hover` 设置了样式，但这行 CSS 并没有起作用。奇怪的是，如果在开发者工具中选择某个 `<col>` 元素，并启用 `:hover` ，它会生效，但在真实场景中却不行：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/342e51034c5547458d48918275537de9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1644&h=710&s=10382510&e=gif&f=159&b=dce7ec)

  


为了解决这个问题，我们可以使用 `:has()` 选择器捕获单元格的悬浮状态，并为对应的 `<col>` 元素设置样式：

  


```CSS
col {
    transition:  background-color .2s ease-in-out;
    background-color: color-mix(in srgb, var(--c), #0000 60%);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
    cursor: pointer;

    &:nth-child(odd) {
        background-color: color-mix(in srgb, var(--c), #0000 40%);
    }
}
  
table {
    &:has(:is(td, th):nth-child(1):hover) col:nth-child(1),
    &:has(:is(td, th):nth-child(2):hover) col:nth-child(2),
    &:has(:is(td, th):nth-child(3):hover) col:nth-child(3),
    &:has(:is(td, th):nth-child(4):hover) col:nth-child(4),
    &:has(:is(td, th):nth-child(5):hover) col:nth-child(5),
    &:has(:is(td, th):nth-child(6):hover) col:nth-child(6){
        background-color: color-mix(in srgb, var(--c), #0000 20%);
    }
}
```

  


简单的解释一下，`table:has(:is(td, th):nth-child(1):hover) col:nth-child(1)` 的意思是，如果表格中有一个 `<td>` 或 `<th>` 是第 `nth-child(1)` ，并且当前正处于悬浮状态（`:hover`），那么选择对应的 `<col>` 元素（即第一列）并设置它的背景。需要注意的是，你需要为每一列重复编写代码，这对于列数较多的表格来说，是比较烦人的。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4f9cb2ffe7ec4c69916f6d37738920e8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=708&s=7256395&e=gif&f=78&b=918c99)

  


> Demo 地址：https://codepen.io/airen/full/zxOdKpb

  


### 设置表格圆角

  


给表格（`table`）设置圆角效果，并不能直接像给其他 HTML 元素（比如 `div`）那样使用 `border-radius` ，因为这样做在视觉上不会有任何效果：

  


```CSS
table {
    border-radius: 1rem; /* 表格并不会有圆角效果 */
}
```

  


有一个小技巧可以使其生效，那就是给 `table` 添加一个 `overflow:hidden` ：

  


```CSS
table {
    border-radius: 1rem;
    overflow: hidden;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/375e26d7cdb642218e6e07b906c84329~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=704&s=13960751&e=gif&f=80&b=908b98)

  


> Demo 地址：https://codepen.io/airen/full/ByBdQrq

  


不过，这个小技巧比较适合那种不带表格标题（`<caption>`）的场景。更为妥当的方式是，给表格的第一行和最后一行的第一个或最后一个单元格设置圆角：

  


-   给第一行第一个单元格左上角设置圆角，同时给第一行最后一个单元格右上角设置圆角
-   给最后一行第一个单元格左下角设置圆角，同时给最后一行最后一个单元格右下角设置圆角

  


具体的 CSS 代码如下：

  


```CSS
tr:last-of-type{
    :is(th, td) {
        &:first-of-type {
            border-end-start-radius: 1rem;
        }

        &:last-of-type {
            border-end-end-radius: 1rem;
        }
    }
}
  
thead th {
    &:first-of-type {
        border-start-start-radius: 1rem;
        border-end-start-radius: 0 !important;
    }

    &:last-of-type {
        border-start-end-radius: 1rem;
        border-end-end-radius: 0 !important;
    }
}
  
table:has(tfoot) tbody tr:last-of-type {
    :is(th, td) {
        &:is(:first-of-type,:last-of-type) {
            border-radius: 0;
        }
    }
}
```

  


上面的代码考虑到了 `<table>` 是否包含了 `<tfoot>` 情景。换句话说，上面的代码在不同的情景下都能使表格有圆角效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6eefe68f384e4ffcb2016b16fad97ead~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=702&s=15330453&e=gif&f=84&b=908b98)

  


> Demo 地址：https://codepen.io/airen/full/QwLMKxm

  


你可以尝试着删除上面示例中已注释的 `<tfoot>` 标记，查看表格的圆角效果。你会发现，不需要调整任何 CSS 代码，表格底部的圆角自动会切换到 `<tfoot>` 中 `<tr>` 中第一个单元格和最后一个单元格中。当然，你也可以根据实际情况来编写 CSS ，**只要确保** **`border-radius`** **设置在表格的第一行和最后一行的第一个或最后一个单元格即可**。只是这样做，需要根据 HTML 结构来调整 CSS。

  


看上去似乎不错，但要知道的是，我们有时候需要给带有边框的表格设置圆角。你可能会想到使用之前设置表格圆角的代码：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: collapse;
}

:is( th, td) {
    border: 1px solid currentColor;
}
  
tr:last-of-type{
    :is(th, td) {
        &:first-of-type {
            border-end-start-radius: 1rem;
        }

        &:last-of-type {
            border-end-end-radius: 1rem;
        }
    }
}
  
thead th {
    &:first-of-type {
        border-start-start-radius: 1rem;
        border-end-start-radius: 0 !important;
    }

    &:last-of-type {
        border-start-end-radius: 1rem;
        border-end-end-radius: 0 !important;
    }
}
  
table:has(tfoot) tbody tr:last-of-type {
    :is(th, td) {
        &:is(:first-of-type,:last-of-type) {
            border-radius: 0;
        }
    }
}
```

  


很快就会发现，上面的代码并不能使带边框的表格有圆角效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f513cc01c4ac426098f020117b7be2c0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=706&s=18675703&e=gif&f=99&b=5b6276)

  


这是因为：**如果你的表格使用了折叠边框（** **`border-collapse: collapse`** **），则无法添加圆角**。所以我们需要使用分离边框（`border-collapse: separate`）:

  


```CSS
table {
    border-collapse: separate;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/228761743b0046cdbe0eb3d30f57bd83~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=704&s=17659264&e=gif&f=118&b=596073)

  


是的，这样做使带边框的表格有了圆角效果。仔细观察，不难发现，由于 `border-collapse` 设置为 `separate` ，边框不能折叠，因此行与行，列与列之间的分隔线变得更粗了。这意味着，我们给表格设置边框时，要模仿折叠边框的效果。我们可以像下面这样编写 CSS:

  


```CSS
:is(th, td) {
    border: 1px solid currentColor;
    border-block-width: 1px 0;
    border-inline-width: 1px 0;

    &:last-of-type {
        border-inline-end-width: 1px;
    }
}
  
:is(tbody, tfoot) {
    tr:last-of-type {
        :is(th, td) {
            border-block-end-width: 1px;
        }
    }
}
  
table:has(tfoot) tbody tr:last-of-type {
    :is(th, td) {
        border-block-end-width: 0;
    }
}
```

  


现在，我们使带有边框的表格有圆角效果了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a200071ab3f24855b3e5cd63b97f4d29~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=702&s=19541978&e=gif&f=90&b=585f73)

  


细心的你可能已经发现了，由于我们的 `table` 设置了 `background-color` 。在此情况下，我们给第一行和最后一行的第一个和最后一个单元格设置 `border-radius` 时，表格的四个顶点会露出表格的背景颜色。我们可以在 `table` 上设置同样大小的 `border-radius` 。最终的 CSS 代码如下：

  


```CSS

    --radius: 1rem;
    border-spacing: 0px;
    border-collapse: separate;
    background-color: var(--bg);

}


:is(th, td) {
    border: 1px solid currentColor;
    border-block-width: 1px 0;
    border-inline-width: 1px 0;

    &:last-of-type {
        border-inline-end-width: 1px;
    }
}
  
:is(tbody, tfoot) {
    tr:last-of-type {
        :is(th, td) {
            border-block-end-width: 1px;
        }
    }
}
  
table:has(tfoot) tbody tr:last-of-type {
    :is(th, td) {
        border-block-end-width: 0;
    }
}

tr:last-of-type {
    :is(th, td) {
        &:first-of-type {
            border-end-start-radius:var(--radius);
        }

        &:last-of-type {
            border-end-end-radius: var(--radius);
        }
    }
}

thead th {
    &:first-of-type {
        border-start-start-radius: var(--radius);
        border-end-start-radius: 0 !important;
    }

    &:last-of-type {
        border-start-end-radius: var(--radius);
        border-end-end-radius: 0 !important;
    }
}

```

### 分离表格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/13dee42e24d7424aa1d79ce27c581abf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=800&s=122598&e=jpg&b=f8f7f7)

  


可能很多人认为，使用 `table` 就只能创建一些循规蹈矩的表格，对于一些个性化的表格，不能使用 `<table>` 来创建，只能使用其他 HTML 元素（比如 `<div>` 来创建）。有这种认知的同学，应该是不太了解表格的分离功能。

  


之前提到过，我们可以使用 `table` 的 `border-spacing` 属性对表格的行与行，列与列进行分离。其实，`border-spacing` 属性与我们之前介绍的 **`gap` 属性**有点类似：

  


-   `gap` 主要用来设置网格行与行，列与列之间的距离，或者设置 Flex 项目之间的距离
-   `border-spacing` 主要用来设置表格行与行，列与列之间的距离

  


`border-spacing` 属性可以接受一个值，也可以接受两个值。当它设置两个值时，第一个值用于设置列之间的间距，第二个值用于设置行之间的间距；当它只设置一个值时，表示行与行，列与列之间的间距相等。如果你使用浏览器开发者工具审查 `table` 元素时，你会发现 `border-spcing` 是 `-webkit-border-horizontal-spacing` 和 `-webkit-border-vertical-spacing` 的简写属性：

  


```CSS
table {
    border-spacing: 20px 40px;
    
    /* 等同于 */
    -webkit-border-horizontal-spacing: 20px;
    -webkit-border-vertical-spacing: 40px;
}
```

  


需要知道的是，`border-spacing` 只有 `table` 的 `border-collapse` 属性的值为 `separate` 时有效。

  


接下来，我们利用表格的这个功能，来创建一个具有个性化 UI 效果的表格，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/216f678a09754a65861b2b7ad216d4bb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1140&s=337890&e=jpg&b=fffafa)

```HTML
        <tr role="row">

            </th>

                <strong>$100</strong> | Month
            </th>
            <th role="columnheader" scope="column">

            </th>
        </tr>
    </thead>
    <tbody role="rowgroup">
        <tr role="row">
            <td role="cell">
                <h3>Free Plan</h3>
            </td>
            <td role="cell">
                <h3>Premium Plan</h3>
            </td>
            <td role="cell">
```

### 悬停和聚焦

  


在大型表格中，明确了解当前所处位置非常重要。为此，我们需要使用 `:hover` 给表格设置悬停样式，以及[使用 `:focus-visible` 在支持键导航的表格中设置焦点样式](https://juejin.cn/book/7223230325122400288/section/7226969870444232742)。

  


在“给表样着色”一节中，我们介绍了如何给表格行和列设置悬停样式，这里简单的介绍一下如何给表格单元格设置悬停样式。使用 CSS 给表格的 `th` 和 `td` 设置悬停样式非常简单，直接在 `th:hover` 和 `td:hover` 设置所需要的样式即可。例如：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: separate;
}

tbody td {
    transition: all 0.2s ease-in;

    &:is(:hover, &:focus:not(:focus-visible)) {
        background-color: #fff842;
        color: #403e10;
        font-weight: bold;
        border-radius: .2em;
        box-shadow: #7f7c21 -1px 1px, #7f7c21 -2px 2px, #7f7c21 -3px 3px,#7f7c21 -4px 4px, #7f7c21 -5px 5px, #7f7c21 -6px 6px;
        translate:6px  -6px 0;
        transition: all 0.3s ease-in .1s;
    }
}
```

  


当表格主体中的 `td` 在 `:hover` 或 `:focus` （但不是 `:focus-visible` ）状态时，单元格会有一个 3D 的视觉效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4d582f75791744248adfedf2a6a6d08f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1174&h=668&s=12773889&e=gif&f=122&b=928d9c)

  


> Demo 地址：https://codepen.io/airen/full/MYgvoay

  


需要知道的是，示例中单元格的 3D 效果是使用 [CSS 的 `box-shadow`](https://juejin.cn/book/7199571709102391328/section/7199844993455325216) 实现的。如果要使单元格阴影（`box-shadow`）在视觉上有效果，必须确保 `table` 的 `border-collapse` 属性的值为 `separate` ，否则你看不到阴影效果。

  


### 给表头和表尾设置样式

  


默认情况下，行标题和列标题会通过加粗字体与其他单元格区分开，但通过更明显的样式标记表头和表尾行，可以进一步提高表格的可读性。在之前的示例中，我们给表头和表尾更改了背景色，与此同时还可以为这些行添加更粗的边框，或者同时使用这两种方式来实现。

  


我们可以[使用逻辑属性 `border-block-start` 和 `border-block-end` 只设置区块轴上的边框](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)。例如：

  


```CSS
table {
    border-collapse: collapse;
}

:is(thead, tfoot) {
    background-color: var(--c);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
}

thead {
    border-block-end: 2px solid;
}

tfoot {
    border-block-start: 2px solid;
}
```

  


需要知道的是，直接在 `thead` 和 `tfoot` 设置边框时，需要确保 `table` 的 `border-collapse` 属性值为 `collapse` ，否则在视觉上不会有任何边框效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/beac4bfde5f946f19758d9ca3a0e4708~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1160&h=592&s=7588757&e=gif&f=122&b=938e9d)

  


然而，有的时候又不得不将 `table` 的 `border-collapse` 属性设置为 `separate` 值，比如之前单元格的悬浮效果，为了使 `box-shadow` 有效果，就必须要这么做。因此，为了确保表头和表尾添加的边框在更多场景下都有效，我们可以将边框设置在 `th` 或（和）`td` 上：

  


```CSS
table {
    border-spacing: 0px;
    border-collapse: separate;
}

:is(thead, tfoot) {
    background-color: var(--c);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
}

thead th{
    border-block-end: 2px solid;
}

tfoot td{
    border-block-start: 2px solid;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/389f82012cf84164959d8068aa2e0c67~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=590&s=5364573&e=gif&f=81&b=928c9c)

  


> Demo 地址：https://codepen.io/airen/full/vEBJdWe

  


### 设置表格尺寸

  


默认情况之下，表格的宽和高是由其内容决定的。我们可以通过 CSS 的 `width` 或 `inline-size` （书写模式是 `ltr` 或 `rtl` ）设置表格宽度，比如：

  


```CSS
table {
    inline-size: max(80vw, 980px);
}
```

  


给表格设置宽度，表格看起来会好一些，不会显得那么拥挤。需要知道的是，当你设置的 `width` 或 `inline-size` 属性值小于表格所内容宽度总和时，它将不起任何作用，就像是表格设置了一个 `min-width` 或 `min-inline-size` 。

  


虽然我们可以给表格设置一个宽度（通常要比所有内容宽度更大），但是，我们使用 `height` 或 `block-size` 给表格指定一个确切的高度时，只有表格所有内容高度比比设置的高度值更小时，它才有效，而且单元格高度也会随之变高：

  


```CSS
table {
    inline-size: max(80vw, 980px);
    block-size: 80vh;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dc0df365dab4419a9e9237ca6d75fcc0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1124&h=618&s=16575476&e=gif&f=147&b=908a98)

  


这种现象同样会发生在 `max-height` 或 `max-block-size` 属性上。你可能会想到，将表格的 `overflow` 设置为 `hidden` ，将溢出的内容裁剪掉。事实告诉我们，这是无效的！

  


虽然我们可以使用 CSS 来设置表格的宽度，但很多时候我们可能正在创建一个大型的表格，如果将表格宽度设置较小，势必会挤压每列的宽度，造成表格难以阅读：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5857ebb697bf483f84a7d40e767c6aac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1159&s=697196&e=jpg&b=b5b1c0)

  


反过来，我们给表格设置较大的宽度值时，表格会溢出页面或包裹它的容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/33ee7cf302f64beba0e2561b6e125fb7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1132&h=630&s=14374874&e=gif&f=108&b=87808f)

  


为了避免用户需要滚动整个页面才可以查看表格的内容，可以将表格包裹在一个容器中，并将其应用 `overflow: auto` 或 `overflow: scroll` ：

  


```CSS
.table--wrapper {
    padding: 1rem;
    height: 50vh;
    overflow-y: auto;
}
```

  


这样用户只需要滚动表格本身即可查看所有数据：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e1cc0b8efe9d4697afe388a6910f7a63~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=682&s=15137584&e=gif&f=109&b=928b9b)

  


> Demo 地址：https://codepen.io/airen/full/wBwqyxQ

  


这种方式也较比适合在移动端浏览表格数据，这也是响应式表格的一种简单处理模式：

  


```CSS
.table--wrapper {
    padding: 1rem;
    height: 80vh;
    overflow: auto;
}

table {
    border-spacing: 0px;
    border-collapse: separate;
    width: max(80vw, 300px)
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/74e6e713a37a4ef8867fbdf72ac1bcf4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=410&h=730&s=3717676&e=gif&f=109&b=aca7b6)

  


> Demo 地址：https://codepen.io/airen/full/WbeEMgm

  


不过，这种方式仍然存在不便之处：用户需要滚动到表格的开头才能查看行或列标题，容易导致迷失位置。我们可以通过固定表头来避免这个现象，稍后再介绍。

  


你可能会注意到我们的表格列宽不一致。这里每列的宽度由所在列的单元格（`th` 或 `td` ）的内容的大小决定。如果我们为表格设置列宽，表格会更易读。接下来，我们来看看表格列宽如何设置。

  


首先，我们可以通过为每个表头单元格设置宽度来解决这一问题：

  


```CSS
thead th {
    width: 15rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ea842284904433fa245c08f68100aad~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1433&s=691619&e=jpg&b=f4f4f4)

  


不难发现，当浏览器视窗宽度变小时，表格列宽仍然不均匀，因此我们需要更多的列宽控制。

  


为了更好的控制列宽，我们可以使用 `table-layout` 属性来更改浏览器用于确定表格布局的算法。将其设置为 `fixed` （默认值为 `auto`），会使浏览器忽略单元格内容，而是使用第一行的列或单元格上定义的宽度来解决列宽：

  


```CSS
table {
    width: 100%;
    table-layout: fixed;
}
  
thead th {
    width: 15rem;
}
```

  


这个时候，表格每一列的宽度都是 `15rem` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/96862a31921f44d58ace69e0a8464c45~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1132&h=646&s=5619434&e=gif&f=81&b=908b9a)

  


> Demo 地址：https://codepen.io/airen/full/wBwqNEY

  


你还可以根据需要，单独来调整某一列的宽度，剩下的列将会保持一致的宽度：

  


```CSS
table {
    width: 100%;
    table-layout: fixed;
}
  
thead th {
    width: 9rem;
    
    &:first-child {
        width: 4rem;
    }
    
    &:nth-of-type(2) {
        width: 18rem;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f3b6ab07530b4610b6f42f8e3a4f56e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1137&s=578735&e=jpg&b=b2adbc)

  


> Demo 地址：https://codepen.io/airen/full/RNbZmgZ

  


要使固定表格（`table-layout: fixed`）布局在现代浏览器中工作，我们还需要设置受影响表格的 `width` 属性。我们可以使用 CSS 的比较函数来设置 `width` 属性的值，比如 `max(65rem, 100%)` ，它会告诉浏览器选择两个参数（`65rem` 或 `100%`）较大的值。这意味着我们的表格在小视口时至少为 `65rem` （因为它是可滚动的），而在大视口时，它将占用可用的整个宽度：

  


```CSS
table {
    width: max(65rem, 100%);
    table-layout: fixed;
}
  
thead th {
    width: 9rem;
    
    &:first-child {
        width: 4rem;
    }
    
    &:nth-of-type(2) {
        width: 18rem;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d66102d93f0642c896109e272c79f10c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1244&h=696&s=10161371&e=gif&f=127&b=8a8592)

  


> Demo 地址：https://codepen.io/airen/full/bNbryoj

  


需要知道的是，当表格设置了 `table-layout: fixed` 时，如果单元格内容遇到了长字符串时，那么长字符串会溢出单元格；如果表格未设置 `table-layout: fixed` 时，如果单元格内容遇到了长字符串时，那么所在列的宽度将会等于长字符串的宽度，即使是该例显式设置了列宽也将无效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a210c4dc25e4ba693da724a26a79920~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=658&s=8108276&e=gif&f=156&b=908a98)

  


### 文本对齐

  


对于从左到右书写的语言（例如拉丁文语系），文本通常左对齐更容易阅读；而对于从右到右书写的语言（例如阿拉伯语系），文本右对齐更容易阅读。另外，对于数字数据，有时右对齐可能更合理，因为这样可以更方便地比较数值。

  


为了适用不同书写方向的语言，可以使用逻辑属性值来设置 `text-align` 属性。`text-align: start` 会根据表格（或文档）设定的方向，确保文本对齐方式合理。

  


尽管直接将 `text-align: start` 应用于 `<table>` 元素在所有浏览器中不一定有效，但可以将其应用到表格标题和表格标题行（`caption`）中。而对于包含数字的单元格和对应的列标题，可以将它们设置为对齐到末尾：

  


```CSS
:is(th,caption) {
    text-align: start;
}
thead th:not(:first-child),
td {
    text-align: end;
}
```

  


当表格列比内容窄时，文本会换行。默认情况下，单元格内容是垂直居中的。这在某些情况下可能是理想的，但对于某些数据类型，这会使表格难以阅读。对于文本内容，可能更合适将其与基线对齐。另一方面，列标题可能更适合放置在单元格的下端，以避免标题漂浮在数据之上。

  


如何选择对齐方式取决于设计和用户需求，但我们可以通过在表格单元格上使用 `vertical-align` 属性来控制垂直位置。让我们将列标题对齐到底部，表格主体单元格对齐基线。

  


```CSS
:is(th,td) {
    vertical-align: baseline;
}

thead th {
    vertical-align: bottom;
}
```

  


### 截断单元格内容

  


在某些情况下，截断或切割单元格内容可能会影响可读性或导致重要信息丢失。在决定是否截断内容之前，必须评估数据呈现的具体要求和优先级。

  


以下是一个截断内容的示例：

  


```CSS
:is(th,td) {
    white-space: nowrap;
    overflow: hidden;
    text-overflow: ellipsis;
}

td:nth-of-type(2) {
    min-width: 200px;
    max-width: 200px;
}
```

  


在这里，我们将第二列的最小宽度和最大宽度设置为`200px`，确保该列始终为`200px`宽，无论内容如何。我们还将 `white-space` 属性设置为 `nowrap`，以防止内容换行，并将 `overflow` 属性设置为 `hidden`，以隐藏溢出单元格的内容。最后，我们设置 `text-overflow` 属性为 `ellipsis`，以便在内容溢出单元格时添加省略号。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ed962c46e0f44ce78f6a3b95cb265bbf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1080&s=645533&e=jpg&b=b2adbc)

  


这种方法最适用于需要保持一致的行和列尺寸的场景。

  


有时，我们可能会遇到包含长链接或长单词的表格，这会导致表格溢出并破坏布局，影响表格的可读性和理解。为了解决这个问题，我们可以使用 `word-break` CSS 属性来拆分长单词和链接，防止它们破坏表格布局。

  


```CSS
:is(th, td) {
    word-break: break-all;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ffa680e4f734d85b8eccfc7d9b9b8eb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1080&s=601047&e=jpg&b=b2adbc)

  


### 固定行和列

  


[CSS 还没有 `position: sticky` 之前](https://juejin.cn/book/7199571709102391328/section/7199845514462887948)，要使表格表头（行表头或（和）列表头）固定不动，是件非常棘手的事情，基本上是通过 JavaScript 辅助来完成。现如今，使用 `position: sticky` 来固定表格的表头就非常容易了。例如，我们想固定表格列表头（即 `<thead>`），我们只需要像下面这样编写 CSS 即可：

  


```CSS
thead {
    position: sticky;
    top: 0;
}
```

  


需要知道的是，滚动容器一般是表格（`<table>`）的父元素。换句话说，我们需要使用一个元素来包裹 `<table>` 元素，并且该容器是一个滚动容器，即 `overflow` 为 `auto` 或 `scroll` ：

  


```HTML
<div class="table--wrapper">
    <table></table>
</div>
```

  


```CSS
.table--wrapper {
    height: 80vh;
    overflow: auto;
}
```

  


这个时候，当你滚动表格时，表头会固定在顶部：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f571c2c0241f4f39b3997a1830760841~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=664&s=20312735&e=gif&f=180&b=918c99)

  


> Demo 地址：https://codepen.io/airen/full/OPLjeLa

  


我们也可以使用同样的方式来固定行的表头（通常是表格的第一列），但我们不能直接将第一个 `col` 的 `position` 设置为 `sticky` 。我们需要将每一行的第一个 `th` 或 `td` （表头一般是 `th`）的 `position` 设置为 `sticky` ：

  


```CSS
tr {
    :is(th,td):first-of-type {
        position: sticky;
        left: 0;
        background-color: var(--c);
        color: color-contrast(var(--c) vs #333, #fff to AAA);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/22665dcb94bd4779bd85161d0261e41c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=980&h=662&s=6188825&e=gif&f=74&b=8c8692)

  


> Demo 地址：https://codepen.io/airen/full/ogverYE

  


细心的你，可能已经发现了，使用 `psoition: sticky` 将表格的表头固定了，但整个效果并不完美，我们在滚动表格的时候，表格单元格的层叠顺序有点混乱。这意味着，在制作一个粘性表格时，需要处理好这些重叠元素。

  


首先，一个粘性表格的单元格需要有背景，否则我们会看到重叠的内容，比如上面示例中，每行的第一个单元格设置了 `background-color` ：

  


```CSS
tr {
    :is(th, td):first-of-type {
        position: sticky;
        left: 0;
        background-color: var(--c);
        color: color-contrast(var(--c) vs #333, #fff to AAA);
    }
}
```

  


你可能会好奇，表格每一列的第一个单元格怎么没有设置背景呢？这是因为我们在 `thead` 中设置了背景：

  


```CSS
:is(thead, tfoot) {
    background-color: var(--c);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
}
```

  


这个时候，你可能会想到，我们是不是可以在第一个 `col` 设置背景呢？非常遗憾，到目前是行不通的。

  


```CSS
tr {
    :is(th, td):first-of-type {
        position: sticky;
        left: 0;
    }
}

col:first-of-type {
    background-color: var(--c);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7e7c72f9bf2948a59edb14d286647bcc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=974&h=662&s=6314478&e=gif&f=90&b=928c9a)

  


正如你所看到的，初始状态，第一列有背景颜色，但滚动表格的时候，你会发现第一列背景颜色随着滚动移动了，表格主体每一行的第一个单元格就露底了。

  


另外一个需要处理的是，需要处理表格单元格的 `z-index` ，使表格的固定表头不会被其他单元格内容遮盖。

  


-   确保 `tbody` 每行第一个单元格（`th` 或 `td`）位于普通单元格的上方，这样它们在水平滚动时能保持在最上面
-   确保 `thead` 中的单元格（`th`）位于 `tbody` 每行第一个单元格上方，以便垂直滚动时正确显示
-   确保 `thead` 中第一个单元格位于最上方，因为它需要在表格主体单元格及其同级标题上方，以便在水平滚动时正常显示

  


这看上去似乎有点乱，我们用 CSS 来描述：

  


```CSS
thead th {
    position: sticky;
    top: 0;
    background-color: var(--c);
    color: color-contrast(var(--c) vs #333, #fff to AAA);
}

tr {
    :is(th, td):first-of-type {
        position: sticky;
        left: 0;
        background-color: var(--c);
        color: color-contrast(var(--c) vs #333, #fff to AAA);
    }
}

:is(th, td) {
    z-index: 1;
    position: relative;
}

thead th {
    z-index: 2;

    &:first-of-type {
        z-index: 3 !important;
    }
}

tr {
    :is(td, th):first-of-type {
        z-index: 2;
    }
}
```

  


这样就完美了！

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3363e58952db4b91810679d0427677bd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=984&h=660&s=10651628&e=gif&f=150&b=8c8793)

  


> Demo 地址：https://codepen.io/airen/full/MYgvMOj

  


温馨提示，需要注意的是，直接在 `<thead>` 、`<tfoot>` 或 `<tr>` 元素上使用 `position: sticky` ，在一些 Chrome 版本中不生效（Safari 和 Firefox 不存在这个问题）。我个人建议，为了避免这个现象，尽量在 `th` 或 `td` 元素上应用 `position: sticky` 。另外，`position: sticky` 是一个非常微妙的特性，有诸多因素会使它不生效，如果你想了解更多关于这方面的信息，请阅读下面两节课的内容：

  


-   [`position: sticky` 的失效与修复](https://juejin.cn/book/7199571709102391328/section/7199845514462887948)
-   [CSS 布局：防止锚点链接滚动到固定或粘性头部后面](https://juejin.cn/book/7395119173656903715/section/7445248285679304713)

  


## 创建响应式表格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0aad087f4caa49ff936dc91cd0c261fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3200&h=2400&s=801934&e=png&b=fdfdfd)

  


我想你的老板或业务方会要求你创建的表格在移动端上有一个较好的浏览体验，然而创建响应式表格不像创建响应式 Web 页面来得那么容易。到目前为止，创建响应式表格主要有以下几种方式：

  


-   **缩小列宽**：如果列中的内容较少，它们可以在移动端屏幕上水平压缩而不会出现问题，因此不改变布局也能提供一个较好的体验；如果列中的内容较多，在移动端上会使表格数据难于阅读
-   **滚动表格**：如果布局和内容非常精确且至关重要，用户可以水平或垂直滚动表格。与固定表头相结合，可以为用户提供一个更佳的体验
-   **按行折叠**：将每一行拆分为一个单列的小型表格，以适应小屏幕。通过正常的表格标记，改变 `display` 属性可以实现
-   **按列折叠**：这是最具挑战性的情况。由于表格的代码顺序是按行排列的，而 `<tr>` 元素会固定这一结构，仅通过纯 CSS 无法实现按列折叠。我们要么需要更改 HTML 结构，要么通过 JavaScript 进行操作和处理

  


  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6fde57d184c4bb8a11a27dd8e604cb1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=593&h=498&s=151398&e=webp&f=135&b=f0f2f4)

  


现如今， CSS 的一些新特性（比如[媒体查询](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)，[容器查询](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)和 [CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)等）使我们创建一个响应式表格变得稍微容易一些，尤其是 CSS Grid 布局技术，因为它也是当下唯一的一种二维布局技术，它与 `<table>` 的二维数据展示非常的匹配。例如下面这个示例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6f169a938ff64e09824b666cc0b875ca~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=438&s=3134091&e=gif&f=197&b=f2f9fd)

  


> Demo 地址：https://codepen.io/airen/pen/OPLxPYg

  


我们同样使用 `<table>` 来创建这个响应式表格，它的 HTML 结构大致如下：

  


```HTML
<div class="table__wrapper">
    <table class="table">
        <colgroup><col><col><col><col><col></colgroup>
```
