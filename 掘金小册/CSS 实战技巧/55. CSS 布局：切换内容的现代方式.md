![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/54f9044ce826453b89f1718267e69885~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=4000&h=2256&s=603711&e=avif&b=384ac5)

  


切换内容指的是内容在显示（或展开）和隐藏（或折叠）之间来回切换，这是 Web 上一种常见的交互模式。例如，[模态框](https://juejin.cn/book/7223230325122400288/section/7229323379793199104)（`Modal`）、[弹窗](https://juejin.cn/book/7395119173656903715/section/7442515150541684746)（`Popover`）、[手风琴](https://juejin.cn/book/7395119173656903715/section/7444808804371267584)（`Accordion`）、[下拉菜单](https://juejin.cn/book/7395119173656903715/section/7445265601544585225)（`Dropdown`）、[树视图](https://juejin.cn/book/7395119173656903715/section/7444808804371267584#heading-4)（`TreeView`）和底部抽屉（`BottomSheet` ）等 Web 组件，都是切换内容的典型案例。

  


然而，我们在开发这些组件时，往往会依赖已有的知识和习惯。比如，习惯性的通过 JavaScript 动态调整元素的样式：

  


-   将 `display` 设置为 `none` （隐藏）或 `block` （显示）
-   将 `opacity` 设置为 `0` （隐藏）或 `1` （显示）

  


或者通过 JavaScript 动态调整元素的类名，并配置相应的 CSS 规则，使内容在显示和隐藏之间切换。

  


不过，如今的 Web 更现代化了，我们完全可以不使用任何 JavaScript 就可以实现内容的切换。比如：

  


-   利用 `input` （`checkbox` 或 `radio`）的 `:checked` 状态来切换内容
-   使用 `<details>` 和 `<summary>` （Disclosures API）来切换内容
-   使用 `<dialog>` （Dialog API）来切换内容
-   使用 `popover` （Popover API）来切换内容

  


上面所提到的方式，有一些是 CSS 的 Hack 方式，有些是浏览器提供的原生 API。这些方式都可以帮助我们实现内容显示和隐藏的功能，但它们各有利弊以及一些你不可能知道的细节。这也意味着，我们是时候全面了解不同的内容切换方式了。接下来，我会根据需求来讨论每种方法的适用场景。让你找到最为适合的一种方式来切换内容。

  


## 利用 `input` 的魔法来切换内容

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e991cc52ce2343ed8b9920cb0d5886bf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1640&h=1081&s=48820&e=webp&b=cf5c5c)

  


内容切换有一个明显的特征，用户通过点击切换触发器（它可能是一个按钮、一个链接或一个标题），可以显示（或展开）隐藏（或折叠）的内容；再次点击触发器时，内容会再次隐藏（或折叠）。简单地说，就是内容元素在真（显示或展形）与假（折叠或隐藏）之间切换。

  


最初在 Web 上创建此类交互效果都是借助 JavaScript 来实现的。但随着条件 CSS 得到越来越多的浏览器支持的情况下，Web 开发者可以不再使用 JavaScript，就可以实现内容的显示与隐藏的切换功能。最为普遍和常用的就是，使用 `input` 的 `:checked` 来做为内容切换的条件依据：

  


-   当 `input` 被选中（`:checked`），内容显示（或展开）
-   当 `input` 未选中（`:not(:checked)`），内容隐藏（或折叠）

  


注意，`type` 为 `radio` 或 `checkbox` 的 `<input>` 元素以及 `<select>` 元素中的 `<option>` 元素都具有 `:checked` 状态。

  


正因此，你会发现有很多 Web 开发者利用这个功能来创建选项卡（`Tab`）、下拉菜单（`Dropdown`）、评级（`Rating`）、模态框（`Modal`）、弹窗（`Popover`）、工具提示框（`Tooltip`）、手风琴（`Accordion`）、树视图（`TreeView`）和 底部抽屉（`BottomSheet` ）等组件。如果你阅读过小册之前的课程，你应该发现了，我们在介绍 CSS 创建[下拉菜单](https://juejin.cn/book/7395119173656903715/section/7445265601544585225)和[手风琴](https://juejin.cn/book/7395119173656903715/section/7444808804371267584)组件时，都应用到了这个技巧。

  


在此之前，使用 `input` （`checkbox` 或 `radio`）创建内容的切换有两个比较明显的限制：

  


-   对 HTML 结构要求比较严谨，`input` 元素必须出现在被切换内容元素之前
-   触发器只能是 `<label>` 元素，通过 `<label>` 元素的 `for` 属性与 `<input>` 元素的 `id` 属性关联在一起

  


因此，你需要的 HTML 结构如下：

  


```HTML
<input type="checkbox" id="toggle" name="toggle" class="sr-only" />
<label class="button" for="toggle">Toggle Content</label>
<div class="toggle__content">
    <!-- 被切换内容的元素 -->
</div>
```

  


这样设计 HTML 结构的主要原因是，[在没有 `:has()` 选择器之前](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)，我们仅能通过 `~` 或 `+` 选择器与 `input:checked` 相结合，才能选中 `.toggle__content` 元素。

  


-   `A + B` （相邻兄弟选择器）：选择紧挨着目标元素后的第一个兄弟元素。如果元素 `B` 紧跟在元素 `A` 之后，并且两者拥有相同的父级元素，那么 `B` 就会被选中
-   `A ~ B` （通用兄弟选择器）：择目标元素后面的所有兄弟元素（不必是相邻的），但它们必须拥有相同的父级元素。如果元素 `B` 出现在元素 `A` 之后，并且两者共享相同的父级，那么所有满足条件的 `B` 都会被选中

  


通常情况之下，我们会先将 `.toggle__content` 隐藏，然后在 `input:checked` 状态下将 `.toggle__content` 显示：

  


```CSS
.toggle__content {
    display: none;
    opacity: 0;
    scale: 0;
    transition: 
        opacity 0.3s ease-in, 
        scale 0.3s ease-in,
        display 0.3s ease-in allow-discrete;

    input:checked ~ & {
        display: block;
        opacity: 1;
        scale: 1;
        transition: 
            opacity 0.3s ease-out, 
            scale 0.3s ease-out,
            display ease-out 0.3s allow-discrete;
        @starting-style {
            opacity: 0;
            scale: 0;
        }
    }
}
```

  


与此同时，我们会使用下面的 CSS 在视觉上隐藏 `input` 元素：

  


```CSS
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}
```

  


你点击 `<label>` 标签，内容就是在显示和隐藏之间来回切换：

> Demo 地址：https://codepen.io/airen/full/QwLggog

  


自从 [`:has()` 选择器](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)得到越来越多的主流浏览器支持之后，使用 `input` 来切换内容时，它的 HTML 结构不需要像最初那样严谨。只需要确保 `input` 的 `id` 和 `lable` 的 `for` 保持一致即可：

  


```HTML
<div class="container">
    <label class="button" for="toggle">Toggle Content</label>
    <div class="toggle__content">
        <!-- 被切换内容的元素 -->
    </div>
    <input type="checkbox" id="toggle" name="toggle" class="sr-only" />
</div>
```

```CSS
.toggle__content {

    scale: 0;

    .container:has(:checked)  & {
        display: block;
        opacity: 1;
        scale: 1;
        transition: opacity 0.3s ease-out, scale 0.3s ease-out, display ease-out 0.3s allow-discrete;

            opacity: 0;
            scale: 0;
        }
    }
}
```

刚才我们提到过，我们可以使用 `type` 为 `checkbox` 或 `radio` 的 `<input>` 元素来制作内容切换。但是，需要知道的是，两者还是有一些差异的：

  


-   如果你仅对一个元素做切换，那么使用 `checkbox` 更合适，因为只有单个`radio` 时，一旦选中就需要使用 JavaScript 删除其 `checked` 属性，才能恢复到未选中状态
-   如果你希望对多个元素切换，并且多个元素中选择一个元素显示，那么使用 `radio` 更为合适，因为 `checkbox` 可以多个选中，但 `radio` 在同一个组中（`name`）只能选中一个

  


比如下面这个示例：

  


```HTML
<div class="container">
    <input type="radio" name="toggle" id="card1" class="sr-only" checked />
    <input type="radio" name="toggle" id="card2" class="sr-only">
    <label for="card1" class="button">Option1</label>
    <label for="card2" class="button">Option2</label>
    <div class="toggle__content card1"><!-- Content 1 --></div>
    <div class="toggle__content card2"><!-- Content 2 --></div>
</div>    
```

```CSS
.toggle__content {
    display: none;
    opacity: 0;
    scale: 0;
    transition: opacity 0.3s ease-in, scale 0.3s ease-in,display 0.3s ease-in allow-discrete;
}

.container:has(#card1:checked) label[for="card1"],
.container:has(#card2:checked) label[for="card2"] {
    outline: 2px solid hsl(257.44deg 76.79% 56.08%);
    outline-offset: 4px;
}

.container:has(#card1:checked) .card1,
.container:has(#card2:checked) .card2 {
    display: block;
    opacity: 1;
    scale: 1;
    transition: opacity 0.3s ease-out, scale 0.3s ease-out,display ease-out 0.3s allow-discrete;
    
    @starting-style {
        opacity: 0;
        scale: 0;
    }
}
```

## 使用 `<dialog>` 来切换内容

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/871ff98634e2434b98bab892a7ef6704~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=685&s=14626&e=webp&b=38a756)

  


`<dialog>` 是 **[HTML5](https://html.spec.whatwg.org/multipage/interactive-elements.html#the-dialog-element)** 中引入的一个语义化元素，用于创建对话框（模态对话框或非模态对话框）。与传统的实现方式（比如使用 `<div>` 和 CSS、JavaScript 来模拟模态框）不同，`<dialog>` 提供了一种更为简洁、内置的解决方案。

  


`<dialog>` 元素同样可以用于内容的切换，只不过它的使用场景更有针对性，例如：

  


-   确认框：显示需要用户确认的内容（如删除、提交等操作）
-   模态框：用于登录、注册等阻碍用户操作的场景
-   通知或消息提示框：例如警告或重要信息的弹出
-   多步骤表单：在对话框中实现分步操作
-   简化页面：减少额外页面跳转，将次要内容嵌入对话框

  


使用 `<dialog>` 创建的对话框主要分为**模态框（Modal Dialog）** 和**非模态框（Non-Modal Dialog）** 两种：

  


-   **模态框 (Modal Dialog)：** 模态框是一种会阻塞用户与页面其他内容交互的对话框。它通常需要用户完成某个操作（如确认、输入信息）后才能返回到主页面
-   **非模态框 (Non-Modal Dialog)：** 非模态框是一种不会阻止用户与页面其他内容交互的对话框。用户可以在对话框打开的同时，仍然与页面其他部分交互

  


两种不同模式的对话框的主要区别如下：

  


| **特性**   | **模态框（Modal Dialog）**        | **非模态框（Non-Modal Dialog）** |
| -------- | ---------------------------- | -------------------------- |
| **用户交互** | 阻止用户与页面其他内容交互                | 不阻止用户与页面其他内容交互             |
| **背景遮罩** | 页面其他部分会被半透明遮罩覆盖              | 页面内容保持可见且可交互               |
| **使用场景** | 关键操作（如警告、确认、登录）              | 辅助信息（如工具提示、非强制的选项）         |
| **显示方法** | 使用 `<dialog>.showModal()` 方法 | 使用 `<dialog>.show()` 方法    |
| **关闭方式** | 需要用户显式操作（如点击按钮或按下 `Esc`）关闭   | 用户可随时忽略，不影响页面其他交互          |

  


接下来，我们来看看 `<dialog>` 又是如何来切换内容。我们从 `<dialog>` 元素开始：

  


```HTML
<dialog><!-- 对话框的内容放在这里 --></dialog>
```

  


默认情况下它是隐藏的，但我们可以给 `<dialog>` 元素显式设置一个 `open` 属性，让它在页面加载时打开（显示）：

  


```HTML
<dialog open><!-- 对话框的内容放在这里 --></dialog>
```

  


需要知道的是，这种情况下它不是模态的，因为它没有通过 `showModal()` 打开，不包含交互内容。这种功能是不是所需要的，我不敢说。相反，你可能更想要在某种交互之后显示对话框，例如点击一个按钮，弹出对话框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aff3cce8d2ad4ec0a906ddb88da78e09~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1104&h=602&s=1917096&e=gif&f=184&b=f5f7fb)

  


> Demo 地址：https://gui-challenges.web.app/dialog/dist/

  


所以，通常情况下，`<dialog>` 会和一个触发器（例如 `<button>`）组合在一起使用。例如：

  


```HTML
<button>Open Dialog</button>
<dialog><!-- 对话框的内容放在这里 --></dialog>
```

  


我们也可以给 `<button>` 传递一个标识符，比如 `data-dialog="dialogA"` ，告诉 JavaScript 打开哪个对话框，这样我们就可通过一段 JavaScript 代码将对话框功能添加到所有 `<dialog>` 中，像下面这样：

  


```JavaScript
// 选择并循环所有带有 data-dialog 属性的元素
document.querySelectorAll("[data-dialog]").forEach(button => {
    // 监听交互（点击）
    button.addEventListener("click", () => {
        // 选择对应的对话框
        const dialog = document.querySelector(`#${ button.dataset.dialog }`);
        
        // 打开对话框
        dialog.showModal();
    
        // 关闭对话框
        dialog.querySelector(".closeDialog").addEventListener("click", () => dialog.close());
    });
});
```

  


别忘了为 `<dialog>` 添加一个匹配的 `id`，这样它就和显示它的 `<button>` 关联了：

  


```HTML
<button data-dialog="dialogA" >Open Dialog</button>
<dialog id="dialogA"> <!-- id 和 data-dialog = dialogA --> ... </dialog>
```

  


最后，加入“关闭”按钮：

  


```HTML
<button data-dialog="dialogA" >Open Dialog</button>

<dialog id="dialogA">
    <!-- 对话框的内容 -->
    <button class="closeDialog">Close dialogA</button>
</dialog>
```

```HTML
<dialog id="modalDialog">
    <p>This is a modal dialog</p>
    <button id="closeModal">Close</button>
</dialog>
<button id="openModal">Open Modal Dialog</button>
```

你会发现，`<dialog>` 的显示是需要通过 JavaScript 来辅助的。`<dialog>` 提供了几个常用的方法：

  


-   `show` ：将对话框以 **非模态** 形式显示（不阻塞页面的交互）
-   `showModal()` ：将对话框以 **模态** 形式显示（阻塞页面的交互，显示遮罩背景）
-   `close()` ：关闭对话框，并可设置一个返回值（可用于传递关闭状态）

  


另外，还提供了两个常用的事件：

  


-   `close` ：当对话框关闭时触发（无论是通过 `close()` 方法还是用户手动关闭）
-   `cancel` ：当用户通过“取消操作”（如按下 `Esc` 键）关闭模态对话框时触发

  


需要注意的是，如果你要实现的是一个非模态对话框，那么请使用：

  


-   `show()` 替代 `showModal()`
-   `dialog[open]`（针对两者）替代 `dialog:modal`

  


不过，随着 Popover API 的出现，我认为 `<dialog>` 更适合用于创建模态对话框，对于非模态框应该交给 Popover API，它更为适合，而且它不需要 JavaScript 也可以打开和关闭对话框。

  


虽然原生的 `<dialog>` 元素具有许多内置功能，比如：

  


-   当模态对话框被打开时，浏览器会将焦点移动到对话框内的第一个可聚焦的元素上；关闭模态对话框时，焦点会返回到触发打开对话框的元素上
-   用户可以通过按下 `ESC` 键关闭模态对话框

  


但有一个重要的功能默认未支持：**点击背景（对话框之外的区域）时自动关闭对话框**：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/30502e2108f44d848175957a312c6ed5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1136&h=592&s=430915&e=gif&f=193&b=f3f3f3)

  


如果需要这个功能，就需要给对话框添加一个 `click` 事件监听器，然后检查事件目标的标签名称：

  


```JavaScript
const modalDialog = document.getElementById("modalDialog");
const openModal = document.getElementById("openModal");
const closeModal = document.getElementById("closeModal");

openModal.addEventListener("click", () => modalDialog.showModal());
closeModal.addEventListener("click", () => modalDialog.close());
modalDialog.addEventListener("click", (evt) => {
    event.stopPropagation();
    if (evt.target.tagName === "DIALOG") {
        modalDialog.close();
    }
});
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a1ef113493e74afc974a2afa2ac78326~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1180&h=656&s=815563&e=gif&f=272&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/xbKrJRW

  


正如你所看到的，`<dialog>` 元素是可以用来切换内容的显示和隐藏，但它的显示离不开 JavaScript 。

  


了解完 `<dialog>` 元素切换内容的基本功能之后，我们来聊聊样式化 `<dialog>` 。不过，在这里我并不打算从头到尾向大家阐述如何使用 CSS 来样式化一个 `<dialog>` 元素。有关于这方面更多的细节，请移步阅读《[用于美化模态框的 `:modal` 和 `::backdrop`](https://juejin.cn/book/7223230325122400288/section/7229323379793199104)》！在这里，我主要会向大家介绍美化 `<dialog>` 的几个重要事项。

  


其中第一个就是 `<dialog>` 元素的 `display` 属性。默认情况下，`<dialog>` 的 `display` 属性的值为 `none` ，这也是页面加载时，`<dialog>` 不可见（隐藏）的主要原因。当 `<dialog>` 元素具有 `open` 属性时，`display` 属性的值会重置为 `block` ，此时对话框就会在页面中显示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e5ddf5ec90ab482ab9ccac09d4cef89b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3342&h=1676&s=544595&e=jpg&b=dbdbdb)

  


然而，我们现在主要以 [CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 或 [CSS Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648) 布局技术来构建 Web 布局。这意味着，如果你希望通过 Flexbox 或 Grid 构建对话框布局时，可以像下面这样重置 `<dialog>` 元素的 `display` 属性：

  


```CSS
dialog {
    display: none;
    
    &:[open] {
        display: grid; /* 或 inline-grid、flex、inline-flex */
    }
}
```

  


第二点就是**滚动限制**。当对话框显示时，用户仍然可以滚动它后面的页面，这可能不是大家想要的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4e29014f0024b2cb11fda5632eeb9c8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=946&s=15071261&e=gif&f=169&b=d1d9da)

  


通常情况下，我会[使用 `overscroll-behavior` 作为解决方案](https://juejin.cn/book/7199571709102391328/section/7199845609447096358#heading-3)，但[规范告诉我](https://drafts.csswg.org/css-overscroll-1/#ref-for-scroll-container%E2%91%A0%E2%91%A1:%7E:text=An%20element%20that%20is%20not%20scroll,methods%20supported%20by%20the%20user%20agent)，它对 `<dialog>` 元素无效，因为 `<dialog>` 不是一个滚动容器，即它本身不是一个可滚动区域，因此无法阻止滚动。现在，[我们可以直接通过 `:has()` 选择器来解决这个问题](https://juejin.cn/book/7223230325122400288/section/7224404685615005728)：

  


```CSS
html:has(dialog[open]) {
    overflow: hidden;
}
```

  


当 `<dialog>` 处于打开状态时，`html` 元素的 `overflow` 属性值被设置为 `hidden` ，从而禁止页面滚动。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bb9485a8704a40848951e5bac69f7dba~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=946&s=1403068&e=gif&f=184&b=d0d8d9)

  


最后一点，就是可以为 `<dialog>` 及其背景（`::backdrop`）定义过渡动画。以下是用于 `<dialog>` 设置动画的 CSS 代码：

  


```CSS
dialog {
    opacity: 0;
    scale: 0;
    transition: opacity 0.3s, scale 150ms, overlay 0.3s allow-discrete,  display 0.3s allow-discrete;
    
    &::backdrop {
        background: hsl(0 0 0 / 25%);
        backdrop-filter: blur(0px); 
        opacity: 0;
        transition: opacity .3s,
        overlay .3s allow-discrete, backdrop-filter .3s allow-discrete,display .3s allow-discrete;
    }

    &[open] {
        opacity: 1;
        scale: 1;
        transition: opacity 0.3s, scale 0.3s, overlay 0.28s allow-discrete,display 0.3s allow-discrete;
      
        @starting-style {
            opacity: 0;
            scale: 0;
        }
      
        &::backdrop {
            backdrop-filter: blur(25px); 
            opacity: 1;
            transition: opacity .3s,overlay .3s allow-discrete,backdrop-filter .3s allow-discrete,display .3s allow-discrete;
            
            @starting-style {
                opacity: 0;
                backdrop-filter: blur(0px); 
            }
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eaa2d7e35de44da8aabef9a95bfdcd9e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1166&h=526&s=859086&e=gif&f=122&b=f3f3f3)

  


除了使用 [`transition`](https://juejin.cn/book/7288940354408022074/section/7292735608995184678#heading-0) 和 `@starting-style` 给 `<dialog>` 添加过渡动画之外，我们也可以使用 [`animation`](https://juejin.cn/book/7288940354408022074/section/7292735608995184678#heading-15) 和 [`@keyframes`](https://juejin.cn/book/7288940354408022074/section/7295617447058407474) 添加更复杂的帧动画。不过，如果使用 `@keyframes` 动画的话，可以不使用 `@starting-style` 规则：

  


```CSS
@keyframes scaleDown {
    to {
        scale: .75;
    }
}

@keyframes slideInUp {
    0% {
        translate: 0% 100%;
    }
}

dialog {
    transition: opacity 0.5s cubic-bezier(0.25, 0, 0.3, 1);
    animation: scaleDown .5s cubic-bezier(.25,0,.3,1) forwards;

    &::backdrop {
        background: hsl(0 0 0 / 25%);
        backdrop-filter: blur(0px);
        transition: backdrop-filter 0.5s ease;
    }

    &[open] {
        animation: slideInUp .5s cubic-bezier(.25,0,.3,1) forwards;
      
        &::backdrop {
            backdrop-filter: blur(25px);
        }
    }
}
```

## 使用 Popover API （`<element popover>`）来切换内容

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbfab86a56c9473da9da677e88f4578b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=747417&e=jpg&b=856e8b)

  


在小册的《[CSS 布局： CSS 与 Popover API 的结合](https://juejin.cn/book/7395119173656903715/section/7442515150541684746)》课程中，曾提到过，Popover API 是 **HTML 属性**（例如 `popover`）、**CSS 属性**（例如`:popover-open`）和 **JavaScript API** （例如 `togglePopover()` 方法）的结合体，为 Web 开发者提供了一种灵活且可定制的方式来创建 Popover。

  


简单地说，Popover 就是我们常说的弹出框，它是另一种常用于切换内容的方式。适用的场景包括工具提示（`Tooltip`）、新手引导、通知（`Toast`）、可切换的导航以及其他非模态的覆盖层等。在这些场景下，弹出框在页面上显示时，并不会影响你对主文档的访问。显然，这些使用场景与模态框（`<dialog>` 创建的模态框）不同，但弹出框依然非常棒。从功能上讲，它与对话框（`<dialog>`）类似，但并不是模态的，并且不需要 JavaScript 来控制它的显示和隐藏。正是基于这一点，我个人认为，`<dialog>` 用来创建模态框，Popover API 用来创建非模态框，这样分工更为清晰，使用者也不易于混淆。

  


使用 Popover API 来切换内容非常简单，而且不需要依赖任何 JavaScript 。我们只需要一个 `id` 、 `popover` 和`popovertarget` 属性即可：

  


```HTML
<button popovertarget="toggle" class="button">Toggle Content</button>
<div class="toggle__content" id="toggle" popover>
  <!-- 弹出框内容放在这里 -->
</div>
```

  


需要注意的是，触发器不能是任意 HTML 元素，它通常是一个 `<button>` 或 `<input type="button">` 或一个 `<a>` 元素，需要给它添加 `popovertarget` 属性（它将来会被 `invoketarget` 属性替代），并且其值要与弹出框 `id` 属性相匹配。

  


就这样简单，你已经使用 `popover` 实现了内容的显示和隐藏功能了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a3732f209f184710a09f7e66179fb9f6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1128&h=608&s=946322&e=gif&f=155&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/JoPJaqG

  


你也可以在同一个页面使用多个 `popover` ，但始终只会有一个显示：

  


```HTML
<button popovertarget="card1" class="button">Option1</button>
<button popovertarget="card2" class="button">Option2</button>
<div class="toggle__content" id="card1" popover>
  <!-- 弹出框内容放在这里 -->
</div>
<div class="toggle__content" id="card2" popover>
  <!-- 弹出框内容放在这里 -->
</div>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a980fe0461b14b11be8f9317d6f97517~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1262&h=764&s=1196656&e=gif&f=116&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/pvzwxoY

  


通常情况之下，Popover 并不需要 JavaScript。不过，在一些场景中（比如我们想显示一个 Toast 通知），用户不需要与按钮互动，那应该如何处理呢？

  


就类似情景而言，我们可以通过 Popover API 提供的一些方法和事件来实现，例如：

  


-   `showPopover()` ：显示 Popover
-   `hidePopover()` ：关闭 Popover
-   `togglePopover()`：Popover 在显示和关闭之间切换

  


我们以 `Toast` 通知为例，它并不需要用户做任何操作就会自动消失，这非常适用于 Popover 没有单独的打开或关闭事件的情景：

  


```HTML
<div class="toast" id="toast" popover="manual">
    Hello I am a toast notification
</div>
```

  


```JavaScript
const toast = document.querySelector("#toast");  

// 显示弹窗
setTimeout(() => toast.showPopover(), 4000);

// 处理 toggle 事件
toast.addEventListener('toggle', (event) => {
    if (!toast.matches(':popover-open')) return;

    // 使用 requestAnimationFrame + setTimeout 确保流畅的动画
    const hideAfter = 5000;
    const start = performance.now();

    const hide = (timestamp) => {
        if (timestamp - start >= hideAfter) {
            toast.hidePopover();
        } else {
            requestAnimationFrame(hide);
        }
    };

    requestAnimationFrame(hide);
});
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4d8a6828d1d2467ea2beb84c2bfa6218~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1000&h=622&s=2221873&e=gif&f=400&b=f2f2f2)

  


> Demo 地址：https://codepen.io/airen/full/Wbejvpe

  


与 `<dialog>` 类似，Popover 默认情况下，它的 `display` 也是 `none` :

  


```CSS
[popover]:not(:popover-open):not(dialog[open]) {
    display: none;
}
```

  


我们可以根据需要重置 `display` 属性。另外，Popover 提供了一个伪元素 `:popover-open` 允许我们为打开状态的 Popover 设置样式。例如给 Popover 添加动画效果：

  


```CSS
[popover] {
    display: none;
    opacity: 0;
    scale: 0;
    border: none;
    background: #0000;
    inset: unset;
    top: 50%;
    left: 50%;
    translate: -50% -50%;

    transition: opacity 0.3s ease-in, scale 0.3s ease-in,display 0.3s ease-in allow-discrete;
    
    &:popover-open {
        display: block;
        opacity: 1;
        scale: 1;
        transition: opacity 0.3s ease-out, scale 0.3s ease-out,display ease-out 0.3s allow-discrete;
      
        @starting-style {
            opacity: 0;
            scale: 0;
        }
    }
}
```

  


另外一点就是，默认情况下，Popover 会居中显示在顶层，但你可以使用固定定位或绝对定位轻松地将它们调整到所需要的位置，但对于工具提示的弹出框，你可能希望它相对于触发它的元素进行定位。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9983b828e44046b7a4ef8d46c39fa8b8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1256&h=758&s=3912061&e=gif&f=126&b=f3f3f3)

  


> Demo 地址：https://codepen.io/airen/full/xbKdPBN

  


类似上图这样的 Popover 效果，我们可以借助 [CSS 的锚点](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)定位来实现。实现该效果的代码和过程就不在这里阐述了，因为我曾在小册的《[CSS 布局：CSS 与 Popover API 的结合](https://juejin.cn/book/7395119173656903715/section/7442515150541684746#heading-5)》课程中详阐述过，这里就不再复述了。

  


正如你所看到的，使用锚点定位实现多个工具提示框弹出，没有任何问题。但它有一个缺陷，你必须为所有锚点命名，对于一个有很多个工具提示的网站来说，这会显得有些多余，也不易于维护，但对于一个标签式组件来说则没问题。庆幸的是，在未来，我们可以将按钮上的 `id` 属性与弹出框（`<dialog>`）的 `anchor` 属性关联起来。这样就可以避免不每个锚点定义锚点名称。例如 [@Daniel Schwarz 在 Codepen 提供的这个示例](https://codepen.io/mrdanielschwarz/full/qBepdxK)：

  


```HTML
<main>
    <!-- id 应该与 anchor 属性匹配 -->
    <button id="anchorA" popovertarget="tooltipA">Show tooltipA</button>
    <button id="anchorB" popovertarget="tooltipB">Show tooltipB</button>
</main>
<dialog anchor="anchorA" id="tooltipA" popover>
    <button popovertarget="tooltipA">Hide tooltipA</button>
</dialog>
<dialog anchor="anchorB" id="tooltipB" popover>
    <button popovertarget="tooltipB">Hide tooltipB</button>
</dialog>
```

  


```CSS
[popover] {
    margin: 0;
    position: fixed;
    top: calc(anchor(bottom) + 10px);
    justify-self: anchor-center;

    &:not(:has(a)) [popovertarget] {
        display: none;
    }
}
```

  


```JavaScript
document.querySelectorAll("main [popovertarget]").forEach((popovertarget) => {
    const popover = document.querySelector(`#${popovertarget.getAttribute("popovertarget")}`);
  
    popovertarget.addEventListener("mouseover", () => {
        popover.showPopover();
    });

    if (popover.matches(":not(:has(a))")) {
        popovertarget.addEventListener("mouseout", () => {
            popover.hidePopover();
        });
    }
});
```

> Demo 地址：https://codepen.io/mrdanielschwarz/full/qBepdxK （来源于 [@Daniel Schwarz](https://codepen.io/mrdanielschwarz)）

  


最后一点，如果有需要，同样可以使用 `::backdrop` 选择器为 Popover 的背景设置样式。例如，可以对 Popover 的背景进行模糊处理：

  


```CSS
::backdrop {
    backdrop-filter: blur(5px);
}
```

## 使用 `<details>` 和 `<summary>` 来切换内容

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad1dc2aa83fa4204b6fa92f7521a3a5e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=46644&e=webp&b=ffffff)

  


到目前为止，我们已经掌握了三种切换内容的现代方式了，接下来要聊的是最后一种——使用 `<details>` 和 `<summary>` 来切换内容的显示（展开）和隐藏（折叠）。这种方式非常适用于 Web 上的手风琴（`Accordion`）、树形导航（`TreeView`）和选项卡（`Tab`）等组件。

  


`<details>` 和 `<summary>` 与之前介绍的 `<dialog>` 类似，都是 HTML 原生元素。早在 2011 年的时候，Chrome 12 就支持它们了。而且，就在今年，浏览器还为 `<details>` 和 `<summary>` 提供了更多新特性，比如：

  


-   使用 `::details-content` 伪元素样式化手风琴展开和收起的部分
-   使用 `interpolate-size` 或 `calc-size()` 将高度动画化为 `height: auto`

  


我们以手风琴为例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f171865cdb7d452e8d91bd52a128e329~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1074&h=612&s=1077790&e=gif&f=124&b=f3f2f2)

  


> Demo 地址：https://codepen.io/airen/full/ZYzyWOY

  


创建一个手风琴组件，你需要一个像下面这样的 HTML 结构：

  


```HTML
<div class="accordion">
    <details name="accordion" open class="accordion__panel" id="account">
        <summary class="accordion__heading">
            <svg viewBox="0 0 448 512">
                <path d="M224 256A128 128 0 1 0 224 0a128 128 0 1 0 0 256zm-45.7 48C79.8 304 0 383.8 0 482.3C0 498.7 13.3 512 29.7 512l388.6 0c16.4 0 29.7-13.3 29.7-29.7C448 383.8 368.2 304 269.7 304l-91.4 0z" fill="currentColor" />
            </svg>
            My Account
        </summary>
        <div class="accordion__content">
            <ul>
                <li>
```

`<details>` 与 `<dialog>` 类似，具有一个 `open` 属性。如果你希望手风琴面板内容区域展开，只需要将对应面板（`<details>` ）添加 `open` 属性。需要知道的是，用户点击标题（`<summary>`）时，浏览器会自动为其父元素 `<details>` 添加 `open` 属性，而其他相邻的 `<details>` 元素的 `open` 属性（如果有）会自动移除。

  


这也意味着，我们可以通过 `[open]` 属性选择器为展开的状态的 `<details>` 设置样式。而且，从现在开始，我们还可以使用 `::details-content` 伪元素和 `interpolate-size` 或 `calc-size()` 来动画化 `height`。另外，为了防止内容溢出 `::details-content` 伪元素，可以给它应用 `overflow: clip` ：

  


```CSS
:root {
    interpolate-size: allow-keywords;
}

::details-content {
    transition: height 0.5s ease, content-visibility 0.5s ease allow-discrete;
    height: 0;
    overflow: clip;
}

[open]::details-content {
    height: calc-size(auto, size); /* 或 auto */
}
```

  


老实说，你可能最想做的事情就是去掉那个烦人的标记（默认它是一个三角形）。这可以通过将 `<summary>` 的 `display` 属性设置为非 `list-item` 的任何值来实现：

  


```CSS
summary {
    display: block; /* 或其他非 list-item 的值 */
}
```

  


或者，重置 `::marker` 的 `content` 属性的值为空：

  


```CSS
summary::marker {
    content: "";
}
```

  


需要知道的是 `::marker` 的使用与我们熟悉的 `::before` 和 `::after` 是相似的。另外，我们也可以像下面这样使用 `::marker` 覆盖默认的内容：

  


```CSS
summary::marker {
    content: "+";
}

summary[open]::marker {
    content: "-";
}
```

  


但是，可用于 `::marker` 伪元素的 CSS 属性有限，我更建议通过 `::before` 或 `::after` 伪元素来替代 `::marker` ，这样你可以应用更多的 CSS 属性，创建更丰富的标记：

  


```CSS
summary::marker {
    content: "";
}

details summary::before {
    content: "";
    /* 元素折叠时的样式 */
}

details[open] summary::before {
    /* 元素展开时的样式 */
}
```

  


相对而言，它的使用要比 `<dialog>` 和 `popover` 简单一点点，至少它没有涉及到 JavaScript API。你只需要通过 HTML 标记和 CSS 就可以实现内容的切换。甚至是创建各种带有折叠和展开的 UI 效果，如 `TreeView` ：

## 小结

  


这节课中，我们介绍了切换内容显示（展开）和隐藏（折叠）的四种不同方式：

  


-   利用`input` （`type` 为 `checkbox` 或 `radio`）的 `:checked` 状态和 CSS 的 `~` 、`+` 或 `:has()` 选择器控制内容的显示与隐藏
-   利用 HTML的 `<dialog>` 元素控制内容的显示与隐藏，这种方法需要依赖于 JavaScript
-   利用 Popover API （即元素的 `popover`）控制内容的显示与隐藏
-   利用 HTML 的 `<details>` 和 `<summary>` 控制内容显示与隐藏

  


其中后面这三种方式都依赖于元素是否处于 `open` 状态来控制，其中 `<dialog>` 和 `<details>` 通过自身的 `open` 属性来控制，而 Popover API 是通过 `popover` 和 `popovertarget` 来控制内容显示或隐藏。在 CSS 方面，`<dialog>` 和 `<details>` 是通过 `[open]` 属性选择器来控制打开状态样式，而 Popover API 则是通过 `:popover-open` 选择器来控制打开状态样式。

  


虽然这几种方式都可以用于 Web 内容的切换，但在实际使用的时候应该根据实际场景选择最为合适的方式。另外，小册还有几节课与这节课的内容相关联，如果感兴趣的话，可以花时间多阅读几遍：
