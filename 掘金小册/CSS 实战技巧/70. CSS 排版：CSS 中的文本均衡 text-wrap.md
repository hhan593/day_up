![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/309f8b9a2ef6416dacc65e08c4164533~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=800&h=420&s=82269&e=jpg&b=9bb3c7)

  


在响应式布局中平衡文本行曾经是一个难题。但现在不再是！通过 `text-wrap:blance` ，自动文本排版即将进入 Web 浏览器。在《[现代 CSS](https://s.juejin.cn/ds/ifLmQbvW/)》小册中有一节课《[经典排版技术：使用 `text-wrap: balance` 实现文本平衡换行](https://juejin.cn/book/7223230325122400288/section/7249196160626196515)》专门探讨了它的实用性和局限性等相关理论。现在，我想通过 Web 上的一些实例与大家一起再次深入的探索该特性，看看它是如何实现文本均衡，在哪些场景适用？并首次了解它的新兄弟 `text-wrap: pretty` 。

  


## 我们从现象开始聊起

  


Web 上的内容并不是一成不变的，它是动态的。默认情况下，内容会在每一行填满整个容器宽度，然后才换到下一行。这通常会导致最后一行文本比前几行短。我们把这种现象称之为“**换行不均衡**”：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/97f6eb4253184c60bdcd14c2f6e6002b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=567500&e=jpg&b=4c4c4c)

  


为了让文本看起来更加“平衡”，Web 设计师和开发者的工作模式是完全不一样的。以一个标题为例，Web 设计师在设计工具（比如 Figma 或 Sketch）中工作，他们会输入标题文本，结果可能像下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dfe8873749bc428dac3ea136b392542b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=162326&e=jpg&b=4c4c4c)

  


Web 设计师可能会通过添加新行或调整文本框大小使文本看起来更加平衡。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d5a0d2369b64f338b62afea56b1a47d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=438171&e=jpg&b=4c4c4c)

  


看起来问题似乎解决了，对吗？但在项目的后续阶段，内容发生了变化。Web 设计师只需要按照上面的方式再次调整文框的宽度或添加心行即可。

  


虽然这种方法在设计工具中非常有效，但在浏览中实现就更具挑战性。因为设计师提供给 Web 开发者的样稿始终是静态的模板，通常情况下，Web 开发者会根据设计稿模板来设置文本容器的尺寸。然而，Web 上的内容是动态可变的，它的变化是不可预测的，可能会因为内容调整、容器尺寸调整（比如响应式 Web布局）或用户未加载我们使用的特殊字体而导致文本看起来“不平衡”。

  


那么，Web 开发者通常是如何让文本看起来更加“平衡”呢？

  


## 传统方式：浏览器中的文本平衡

  


由于 Web 开发者与设计师的工作模式不同，Web 开发者解决问题的方法与设计工具中的方式有所不同。在还没有 `text-wrap:balance` 之前，我能想到的方式有以下几种：

  


-   使用 `<br />` 标签将文本分成多行；或者使用多个标签元素将文本分成多行
-   给文本容器设置一个最大宽度并微调其外观

  


### 使用换行元素

  


Web 开发者常用的方式之一是在文本行中插入 `<br />` 标签，这样一来，`<br />` 标签前后的文本就会换行：

  


```HTML
<h1>How to make text look more balanced <br /> in CSS.</h1>
```

  


有的 Web 开发者可能会使用多个标签元素将文本分成多行：

  


```HTML
<h1>
    <div>How to make text look more balanced</div>
    <div>in CSS.</div>
</h1>    
```

  


上面两种方式都能强行将文本换行。你可以通过拖动下面示例中右下角的手柄来调整容器的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e8cab117a2a34c77b1a8545c870ab824~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1440&h=734&s=4896776&e=gif&f=309&b=f2f2f2)

  


> Demo 地址：https://codepen.io/airen/full/MYgBpxd

  


注意，示例中的小圆点是模拟换行符的位置。

  


正如你看到的，这种方法有效，但也有诸多的弊端。比如，当容器有更多的可用空间时，强制的换行可能变得不再合理，反而会让文本看起来不平衡。相反，当空间变小时，文本的换行会显得怪异，此时我们可能不再需要强制换行：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/991bec9201274b5693df9019a2a1c204~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=273838&e=jpg&b=f6f6f6)

  


其次，Web 内容是动态的，很有可能我们预先放置的 `<br />` 标签不是最佳位置，同样会使文本看起来不平衡。

  


可以说，手动使用 `<br />` 或其他 HTML 标签来平衡文本行非常麻烦，尤其是在响应式布局中，它的表现更不佳。当然，你可能会想到使用 JavaScript 动态插入 `<br />` 或其他 HTML 元素使文本换行。这样做会需要更复杂的计算，我个人不建议这样做。

  


### 给容器设置最大宽度

  


虽然我们可以给文本所在的容器设置最大宽度（类似在 Figma 中调整文本框的尺寸），使文本换行看起来平衡：

  


```HTML
<h1>How to make text look more balanced in CSS.</h1>
```

  


```CSS
h1 {
    max-inline-size: 20ch;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b11d04fb113c4fe2a611a282785f09d2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1250&h=754&s=1115229&e=gif&f=318&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/EaYpvex

  


如果文本内容是固定不变的，通过微调 `max-inline-size` （或 `max-width`）属性的值，可以使文本换行看起来平衡。反之，这种方式也不是最优解。

  


## 现代方式：CSS 中的文本均衡 `text-wrap`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/62097f7a4e0648ccb105860548e82da3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1788&h=1008&s=352965&e=png&b=114e55)

  


`text-wrap` 是 CSS 新增的一个特性，主要用于控制元素内部文本的换行方式。不同的取值提供以下功能：

  


-   排版改进，例如在断行的标题中实现更均衡的行长
-   完全关闭换行的方式，有点类似于 `white-space` 的 `nowrap`

  


`text-wrap` 属性可以通过以下值来控制元素中文本换行方式：

  


-   **`wrap`** **：** 文本在合适的字符（例如空格，对于使用空格作为分隔符的语言如英语）处换行，以减少溢出。这是默认值
-   **`nowrap`** **：** 文本不会换行。如果内容超出容器宽度，文本将溢出，而不是换行。有点类似于 `white-space: nowrap`
-   **`balance`** **：** 文本以一种尽可能平衡每行字符数量的方式换行，提升布局质量和可读性。由于计算字符并在多行间平衡是高耗能操作，这个值仅适用于一定行数限制内的文本块（Chromium 限制为 `6` 行及以下，Firefox 限制为 `10` 行及以下）
-   **`pretty`** **:** 与 `wrap` 行为类似，但浏览器使用较慢的算法，更注重布局质量而非性能。这适用于正文文本，特别是在需要保持更好的排版效果（例如尽量减少孤行现象）时
-   **`stable`** **:** 与 `wrap` 行为类似，但当用户编辑内容时，编辑区域前的行保持静止，而不是整个文本块重新换行

  


我们应该根据具体的场景指定 `text-wrap` 属性的值：

  


-   **短文本（如标题、说明和引用）：** 对于行数有限的内容，可以使用 `text-wrap: balance` 来平衡每行的字符数量
-   **长文本（如正文）：** 对于较长的文本部分，可以使用 `text-wrap: pretty`。需要注意的是，`pretty` 会对性能产生负面影响，因此仅在布局质量比性能更重要时使用
-   **可编辑内容：** 如果文本是 `contenteditable` 的，可以使用 `text-wrap: stable` 来改善用户体验。此值确保用户编辑文本时，编辑区域前的行保持稳定，不会因为重新换行而发生变化

  


让我们先来认识一下 `text-wrap: balance` 。[W3C 规范](https://drafts.csswg.org/css-text-4/#text-wrap-style)是这样描述的：

  


> 如果可能，换行会被选择平衡每个行框中的剩余（空白）空间，以达到比 `auto` 更好的文本平衡效果

  


为了更容易理解，你可以尝试调整下面示例的容器的宽度，用于比较应用了 `text-wrap: balance` 前后的文本排版效果：

  


```CSS
h1 {
    text-wrap: balance;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6f94072b3b5045a8b2b99fe5d9ca3274~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1664&h=598&s=2314547&e=gif&f=208&b=f6f6f6)

  


> Demo 地址：https://codepen.io/airen/full/RNbBjYb

  


正如你所看到的，我们现在只需一行 CSS 代码（即 `text-wrap: balance`）就能使换行的文本看起来平衡。需要注意的是，有时候将 `text-wrap: balance` 应用于左对齐或右对齐的文本可能会导致文本过渡堆叠到一侧，从而使布局看起来不平衡。这意味着，平衡的文本可能会使布局不平衡。因此，大家在使用的时候需要根据自己的实际情况做出正确的判断：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4320c74490154171a1b23ec2d97bd43e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1670&h=640&s=2713092&e=gif&f=190&b=f5f5f5)

  


> Demo 地址：https://codepen.io/airen/full/NPKBwJp

  


虽然 `text-wrap: balance` 会强力地平衡文本，但使用 `text-wrap:pretty` 则更加宽容。它最常见的好处可能是防止孤立词（孤立词是指段落结尾的单个词，它单独换行，可能看起来有些尴尬）。请观察下面示例中 `wrap` 和 `pretty` 的差异：

> Demo 地址：https://codepen.io/airen/full/WbeKXBO

  


`text-wrap: pretty` 的另一个好处是减少过度的连字符。看看它如何影响启用了连字符的文本（通过 `hyphens: auto`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/95e3623ed6c349209e6d7aed2fb250de~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3292&h=1088&s=188436&e=jpg&b=c7dbed)

  


如果在段落的结尾处有多行连续的单词以连字符结尾，`text-wrap: pretty` 将会调整它们。这是一个非常不错的可读性提升。但是，我实测下来，并没有发现有时候并没有太大的差异：

有一点需要知道的是，`text-wrap:pretty` 的计算是很昂贵的，特别是应用于大量文本时。因此，在使用的过程中需要评估它对性能的影响，并在最需要的地方选择性地使用它。

  


至于其他值，大家可以尝试切换下面案例中的 `text-wrap` 值和容器的宽度，查看效果的差异：

> Demo 地址：https://codepen.io/airen/full/ZYzjvpq

  


`text-wrap` 功能虽然强大，但我们在使用它时，至少需要考虑以下几点：

  


-   **性能**：每增加一行，计算负载就会增加。对于 `text-wrap: balance` ，浏览器通过在文本行数不超过一定数量时才进行相关处理来避免性能问题。目前，Chrome 和 Edge 的限制是六行，Firefox 的限制是十行。如果文本超过了这个限制，`text-wrap: balance` 将被忽略。对于 `text-wrap: pretty` 没有这样的限制，因此在使用它时需要根据具体的情况来做出正确的选择
-   **空白干扰**：如果设置了特定的 `white-space` 值，则此功能将无法工作
-   **尊重手动换行符**：它会尊重插入的 `<br>` 标签，因此你插入的手动换行不会被破坏

  


我认为这些前提条件是优点。起初，我担心 `text-wrap: balance` 可能过于“神奇”，使得它难以理解和调试。但尤其是它会尊重手动换行这一点，缓解了我的担忧。

## 小结

  


在 CSS 中平衡文本有助于使设计看起来更整洁、更易于阅读。像 `text-wrap: balance` 和 `text-wrap: pretty` 这样的功能帮助我们解决了这个问题。
