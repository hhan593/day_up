## 使用背景属性打造奇妙的悬停效果

  


以往我们创建一些奇妙的悬停效果都会依赖于 CSS 伪元素（`::before` 或 `::after`）、变换（`transform`）、裁剪（`clip-path`）、遮罩（`mask`）和过渡（`transition`）的组合。其实，很多奇妙的悬停效果，我们只需要依赖 CSS 的背景属性（比如 `background-position` 、`background-size` 等）就可以实现。你可以在那个示例中看到背景属性的工作方式，以及我们如何结合自定义属性和 `calc()` 函数实现更多效果。

  


我们先从简单的效果开始。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9faeecf2495548cdbef1a9a3bf9c90b9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=454&s=263715&e=gif&f=195&b=fef9f8)

  


> Demo 地址：https://codepen.io/airen/full/myyvrVM

  


实现该效果所使用的代码如下：

  


```CSS
nav {
    --c: #f0a;
    
    a {
        color: var(--c);
        background: conic-gradient(var(--c) 0 0) var(--p, 0) / var(--p, 0) no-repeat;
        transition: 0.4s, background-position 0s;
      
        &:is(:hover, :focus, .active) {
            --p: 100%;
            color: color-mix(in oklch, var(--c), #fff 100%);
        }
    }
}
```

  


我们仅使用了几行 CSS 代码就能实现这个效果。你可能会对代码的精简感到惊讶，但接下来你会看到我们是如何一步步实现这一点的。

  


虽然代码中的 `background` 属性对于你而言，应该非常简单，但我还是想花一点时间来解释一下。首先，我们使用 [CSS 的锥形渐变 `conic-gradient()` 绘制了一个纯色的渐变](https://juejin.cn/book/7223230325122400288/section/7259668771856941111)：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
}
```

  


这个语法可能看起来有点奇怪，但实际上告诉浏览器，这个颜色（`--c` ，即渐变颜色）被应用于两个颜色停止点，这就足以在 CSS 中定义一个渐变。两个停止点都为 `0` ，浏览器会自动将最后一个设置为 `100%` ，于是整个渐变区域都会被同一种颜色填满。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/70cb14138b95498b840243c66cf0ec48~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=988&s=82641&e=png&b=fffaf9)

  


当然，你也可以使用线性渐变 `linear-gradient()` 来替代锥形渐变 `conic-gradient()` ：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
    
    /* 或者 */
    background-image: linear-gradient(var(--c) 0 0);
}
```

  


接着我们来看第二个属性 `background-size` ，它主要用于设置背景图片（`background-image`）的尺寸。我们在使用 `background-size` 的时候，可以省略高度，因为渐变默认是全高度的（相当于 `100%`）。在我们这个示例中，我们引入了 `--p` 变量来控制背景图片水平方向（`x`）的大小：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
    background-size: var(--p, 0);
    background-repeat: no-repeat;
}
```

  


注意，`--p` 的默认值为 `0` ，相当于 `background-size` 的值为 `0% 100%` ，你将看不到任何背景效果。另外，为了不让背景图片重复铺满整个容器，我们需要将 `background-repeat` 设置为 `no-repeat` 。

  


当我们调整 `--p` 的值，背景图片宽度会随之变大，当它达到 `100%` 时会填满整个容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0beb97981226434fa5bf86950cc7bdc6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1332&h=542&s=1037023&e=gif&f=193&b=fffafa)

  


这意味着，我们可以直接从 `background-size: 0%` 过渡到 `background-size: 100%` ：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
    background-size: var(--p, 0);
    background-repeat: no-repeat;
    transition: background-size .4s;
    
    &:hover {
        --p: 100%;
    }
}
```

  


注意，在我们这个示例中，我们只需要在悬停状态给 `--p` 定义一个新的值（`100%`）来替换初始值（`0%`）即可。

  


最后，我们来了解一下 `background-position` 属性，默认情况下，它的值是 `left top` （即 `0% 0%`）。要实现这个悬停效果，我们需要在悬停时更新 `background-position` 的值。我们可以通过两个步骤实现：

  


-   鼠标悬停时，从右侧展开背景
-   鼠标移开时，从左侧收回背景

  


为此，我们也要在悬停时更新 `background-position` ：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
    background-size: var(--p, 0);
    background-repeat: no-repeat;
    background-position: left;
    transition: background-size .4s, background-position 0s;
    
    &:hover {
        --p: 100%;
        background-position: right;
    }
}
```

  


我们在代码中做了两件事情：

  


-   悬停时设置 `background-position` 为 `right`
-   设置 `background-position` 的过渡时间（`transition-duration`）为 `0s`

  


这意味着，当鼠标悬停在元素上时，我们会立即将 `background-position` 从 `left` 变为 `right` ，这样背景的尺寸就会从右侧开始扩展。接着，当鼠标离开时，过渡会反向执行，从右向左收缩，看起来就像背景从左侧缩小一样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/850a3ef047b940fe8ff15be44fc92a8b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1338&h=538&s=895897&e=gif&f=287&b=fffaf9)

  


我们可以把 `left` 和 `right` 替换为 `0 0` 和 `100% 0` ，因为我们的渐变图片高度是 `100%` ，我们可以只调整 `background-position-x` 的值，即从 `0%` 过渡到 `100%` 。在此，我们引用与 `background-size` 相同的变量 `--p` 来控制 `background-position-x` 的值。我们可以像下面这样编写 CSS：

  


```CSS
a  {
    background-image: conic-gradient(var(--c) 0 0);
    background-size: var(--p, 0);
    background-repeat: no-repeat;
    background-position: var(--p, 0%);
    transition: background-size .4s, background-position 0s;
    
    &:hover {
        --p: 100%;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f698a62eadd0466e86044e1d78814702~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=538&s=888156&e=gif&f=179&b=fffafa)

  


> Demo 地址：https://codepen.io/airen/full/GggzjBd

  


正如你所看到的，自定义属性 `--p` 同时定义了背景的位置（`background-position`）和大小（`background-size`）。悬停时，它会同时更新两者。这就是一个完美的示例，展示自定义属性如何帮助我们减少冗余代码，避免重复编写属性。我们用自定义属性定义设置，只需要在悬停时更新这个属性即可。

  


将 `background-*` 相关的属性合并到 `background` 属性中，就获得了我们示例所需要的 CSS 代码：

  


```CSS
a  {
    background: conic-gradient(var(--c) 0 0) var(--p, 0%) / var(--p, 0%) no-repeat;
    transition: background-size .4s, background-position 0s;
    
    &:hover {
        --p: 100%;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ccaffa8b08084fdbbacf3d808d9f7392~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=454&s=263715&e=gif&f=195&b=fef9f8)

> Demo 地址：https://codepen.io/airen/full/myyvrVM

  


在这个示例的基础上，我们只需要稍微调整一下 CSS 代码，就可以获得一个相反的效果——从左开始，到右结束。我们同样基于一个 `--p` 变量来实现。需要知道的是，我们要的只是让动画从 `100%` 变为 `0%` ，而不是从 `0%` 到 `100%` 。在 CSS 中，我们可以通过 `calc()` 函数来表达：

  


```CSS
a  {
    background: conic-gradient(var(--c) 0 0) calc(100% - var(--p, 0%)) / var(--p, 0%) no-repeat;
    transition: background-size .4s, background-position 0s;
    
    &:hover {
        --p: 100%;
    }
}
```

  


`--p` 会从 `0%` 变为 `100%`，但通过 `calc()`，背景的位置（`background-position`）会从 `100%` 变为 `0%`，从而实现反向动画。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7598457ada974860a0f5e7ccbfe24da5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=454&s=319175&e=gif&f=170&b=fef9f8)

  


> Demo 地址：https://codepen.io/airen/full/qEEgqVY

  


上面案例，渐变是单一颜色的，我们可以使用两个颜色的渐变，悬停前后有不同的背景颜色：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e1cade3ce91d488591dc89895d17d91f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=452&s=270635&e=gif&f=164&b=fffbfa)

  


> Demo 地址：https://codepen.io/airen/full/wBBNoNM

  


```CSS
nav {
    --c: #f0a;
    --_c: color-mix(in oklch, var(--c), #000 60%);
    a {
        color: var(--c);
        background: linear-gradient(
            90deg,
            var(--_c) 0% 50%,
            var(--c) 50% 100%
        ) 0 0 / 200%;

          background-position 0.3s cubic-bezier(0.47, 0.1, 1, 0.63) 0s,
          color 0.2s linear 0.15s;

            color: color-mix(in oklch, var(--c), #fff 100%);

        }
    }
}
```

  


注意观察发生了什么。鼠标悬停在导航项上时，深紫色（`--_c`）背景滑出，为滑入的紫色（`--c`）背景腾出空间。当鼠标移开时，紫色背景滑走，并带回深紫色背景！

  


我们来解释一下它的工作原理。首先，我们通过 `linear-gradient()` 绘制了一个双色渐变，从深紫色（`--_c`）渐变到紫色（`--c`），第一个渐变色从 `0%` 开始，到 `50%` 结束，第二个渐变色从 `50%` 开始，到 `100%` 结束：

  


```CSS
a  {
    background-image: linear-gradient(
        90deg,
        var(--_c) 0% 50%,
        var(--c) 50% 100%
    );
}
```

  


在视觉上，每个渐变颜色各种容器宽度的一半：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3690bca6a8324353ba6998e60c2fad42~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=988&s=89543&e=png&b=fffaf9)

  


  


与前面示例不同的，`backround-repeat` 的值是 `repeat` （默认值），它将平铺整个容器。我们将 `background-size` 调小一下，查看背景平铺的效果：

```CSS
        90deg,
        var(--_c) 0% 50%,
        var(--c) 50% 100%

    background-size: 50%; /* 测试值 */

```

而我们期望的效果是，默认情况下（鼠标未悬停在元素上时），它的背景色是深紫色（`--_c`），当鼠标悬停在元素上时，它的背景色变成紫色（`--c`）。我们可以通过调整 `background-size` 的值来获得所要的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3b21d2eb47ed4915adda02088cbc39cc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3113&h=1799&s=761034&e=jpg&b=ffffff)

我们的目标是，鼠标悬停在元素上时，深紫色背景需要移出，与此同时紫色背景移入。你可能已经想到了，我们只需要在 `:hover` 状态调整 `background-position` 的值即可：

在解释上面代码之前，我想通过下图来解释这个效果的实现过程：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d4cbad96d0f94b8c843e12baaecc0702~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3669&h=2342&s=599125&e=jpg&b=ffffff)

  


-   1️⃣ **初始状态：** 一个固定高度（`--b`）、全宽度的渐变背景位于元素容器之外（隐藏不可见）
-   2️⃣ **悬停开始**： 将渐变背景移动到底部边缘
-   3️⃣ **悬停继续：** 渐变背景的高度从固定值（`--b`）增加到 `100%`，覆盖整个元素

  


与之前示例有所不同的是，这个悬浮效果有两个过程，先对 `background-position` 进行过渡，再对 `background-size` 过渡。将这个过程转化为代码如下：

  


```CSS
a {
    background-image: conic-gradient(var(--c) 0 0);
    background-size: 100% var(--b, .125em);
    background-position: /* ❓❓❓ */;
    background-repeat: no-repeat;
    
    /* 从 A → B 的过渡 */
    transition:
        background-size .3s,
        background-position .3s .3s;
        
    &:hover {
        /* 从 B → A 的过渡 */
        transition: 
            background-size .3s .3s,
            background-position .3s;
        background-size: 100% 100%;
        background-position: /* ❓❓❓ */;
    }    
}
```

  


你可能已经发现了，我们为 `background-size` 和 `background-position` 设置了不同的过渡时间。悬停时先改变位置（`background-position`），然后延迟改变大小（`background-size`）；鼠标移出时，顺序相反。

  


这其实是使用 `transition` 实现了一种“**延迟交错动画**”的效果，其核心是通过交错延迟控制两个动画的顺序：

  


-   **初始状态 → 悬停状态**：先改变 `background-position` （立即执行，持续 `0.3s`）→ 渐变位置改变了，但高度（`background-size`）还保持原来的细条样式；`0.3s` 后开始改变 `background-size` （从细条扩展为整个背景）→ 造成一种从边缘扩散到整个区域的动画感。对应的是 `a:hover` 规则中的 `transition`
-   **悬停状态 → 初始状态**：先改变 `background-size` （立即执行，变回细条），`0.3s` 后再改变 `background-position` （位置回收）。对应的是 `a` 规则规则中的 `transition`

  


有关于这方面的详细介绍，请移步阅读《[帧动画与过渡动画：谁更适合你的业务场景？](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)》和《[深入了解 CSS 动画的持续时间和延迟时间](https://juejin.cn/book/7288940354408022074/section/7304843997364060214)》！

  


你可能已经留意到了，我们在两种状态中的 `background-position` 都没有指定具体的值（`❓`），那么它的值应该是什么呢？这是因为，`background-position` 的值为 `%` 时，它的计算比较复杂。它会同时考虑容器的尺寸和背景图片的尺寸，即相对于两者尺寸做计算。[W3C 规范是这样描述的](https://www.w3.org/TR/css-backgrounds-3/#the-background-origin)：

  


> A percentage for the horizontal offset is relative to (width of background positioning area - width of background image). A percentage for the vertical offset is relative to (height of background positioning area - height of background image), where the size of the image is the size given by ‘background-size’.

  


简单地说，当背景图片尺寸（`background-size`）不做任何的重置（也就是`100% 100%`）时，水平百分比的值等于容器宽度百分比值减去背景图片宽度百分比值。垂直百分比的值等于容器高度百分比值减去背景图片高度百分比值。

  


即 `background-position` 取值为百分比值（`%`）时，它会先从相应的容器尺寸中减去背景图片尺寸，然后将结果值的百分比用作距左边界（或顶部）的直接偏移量。相应的计算公式如下：

  


```
(容器宽度 - 背景图片宽度) x (background-position-x设置的百分比值) = background-position-x 的计算值 
(容器高度 - 背景图片高度) x (background-position-y设置的百分比值) = background-position-y 的计算值 
```

  


即：

  


-   水平方向（`x` 轴），即 `background-position-x` 的 `%` 值是相对于元素的 `width` 与背景图片的 `width` 差计算；
-   垂直方向（`y` 轴），即 `background-position-y` 的 `%` 值是相对于元素的 `height` 与背景图片的 `height` 差计算。

  


特别声明，背景图片的大小将会受 `background-size` 值的影响，上面提到的图片的尺寸是指 `background-size` 的值为 `auto` 的情景下的尺寸，即背景图片的初始尺寸（原始尺寸）。比如下面这个示例：

  


```CSS
.element { 
    width: 500px; 
    height: 300px; 
    background-position: 70% 30% 
} 
```

  


假设我们在 `.element` 中使用一张 `280px x 180px` 的背景图，那么 `background-position` 的百分比的计算值如下：

  


```CSS
background-position-x: 70% ⇒ (500px - 280px) x 70% = 154px 
background-position-y: 30% ⇒ (300px - 180px) x 30% = 36px 
```

```CSS
        background-size: 50.1% var(--p, 0.125em);
        transition: 
            0.3s var(--t, 0s),
            background-position 0.3s calc(0.3s - var(--t, 0s));

        &:is(:hover, :focus, .active) {
            --p: 101%;
            --t: 0.3s;
            color: color-mix(in oklch, var(--c), #fff 100%);
        }
    }
}
```

  


注意，在这个示例中，额外添加了一个 `--g` 变量，用来存储渐变，因为我们在这里使用了两个相同的渐变。另外，在设置背景图尺寸的时候，使用了 `50.1%` 而不是 `50%` ，这是为了防止渐变之间出现间隙。同样的原理，也给背景图的位置增加了 `1%`（即 `101%`）。

  


同样的，我们还可以使用 `--i` 变量来控制渐变的位置和大小，从而进一步优化 CSS 代码，只不过这样编写 CSS ，需要对 CSS 自定义属性的高级技巧要有一定的理解，不然阅读和理解代码会令你感到困惑：

```CSS
    a {
        --g: no-repeat conic-gradient(var(--c) 0 0);
        color: var(--c);
        background: 
            var(--g) calc(-100% + var(--i, 0) * 100%) 100% / 50% calc(100% * var(--i, 0) + 0.125em),
            var(--g) calc(200% - var(--i, 0) * 100%) 0 / 50% calc(100% * var(--i, 0) + 0.125em);

            0.3s calc(var(--i, 0) * 0.3s),
            background-position 0.3s calc(0.3s - var(--i, 0) * 0.3s);

        }
    }
}
```

```CSS
        background-position: 0 100%, 100% 100%;
    }
}
```

  


注意，代码中的 `❓` 就是我们需要填被的 CSS 代码，也是需要特别关注的部分。

## 使用背景、裁剪和遮罩打造奇妙悬停效果

  


你可能已经感觉到了，虽然我们一直在围绕着 CSS 背景创建悬停效果，但所应用到的 CSS 技巧越来越复杂，创建的悬停效果也越来越复杂。接下来，我们会接触到一些更复杂，更奇妙的悬停效果，这些效果是使用背景裁剪（`background-clip`）、CSS 裁剪（`clip-path`）、遮罩（`mask`）和 `calc()` 函数，甚至还会涉及到 3D 透视效果。换句话说，我们将会探索一些高级技术，突破 CSS 在悬停效果方面的极限。

  


### 使用 `background-clip` 创建悬停效果

  


我们先从 `background-clip` 开始，它相比与 CSS 的 `clip-path` 和 `mask` 等特性要简单得多。CSS 的 `background-clip` 属性有一个 `text` 值，它允许我们将渐变填充到元素的文本上，而不是实际的背景。我们通常利用这种技巧来创建渐变文字等效果，有关于这方面的详细介绍可以移步阅读《[Web UI：创建渐变文字与投影效果](https://juejin.cn/book/7395119173656903715/section/7446278062612463643)》！

  


我们先来看一个简单的示例，使用 `background-clip: text` 在悬停时改变文本颜色（类似于 `color`），但通过这种方式，我们可以动画化颜色变化——颜色填充有一个滑动的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/080ca9029f9a46f2a696533e07743657~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=450&s=128178&e=gif&f=130&b=fffaf9)

  


> Demo 地址：https://codepen.io/airen/full/emmxoMO

```CSS
    --c: #f0a;
    --_c: color-mix(in oklch, var(--c), #000 60%);
    
    a {
        background: linear-gradient(
            90deg, 
            var(--c) 50%, 
            var(--_c) 0
        ) var(--p, 100%) / 200% no-repeat;
        background-clip: text;
        -webkit-text-fill-color: #0000; /* 这个很重要，或者 color: #0000 */

            --p: 0%;
        }

```

### 使用 `mask` 创建悬停效果

  


CSS 的 `mask` 使用方式与 `background` 属性相似，所以你会发现接下来做的东西其实非常直接。要是你对 `mask` 不太了解的话，你可以先花一点时间阅读《[CSS 的 Clipping 和 Masking](https://juejin.cn/book/7223230325122400288/section/7259668885224456252)》，它将告诉你有关于 `mask` 的所有知识！

  


我们先来看 `mask` 创建的第一个悬停效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2515a34c18264b229f5ed5f86b79f4b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1330&h=446&s=341218&e=gif&f=199&b=fff9f8)

  


> Demo 地址：https://codepen.io/airen/full/oggmrvJ

  


```CSS
nav {
    --c: #f0a;
    --_c: color-mix(in oklch, var(--c), #000 60%);
    
    a {
        --s: 0.15em;
        --g: linear-gradient(#000 0 0); 
        padding: 0 calc(2.1 * var(--s)) calc(2.1 * var(--s));
        color: var(--_c);
        background: 
            conic-gradient(
                from 135deg at top, 
                var(--c) 90deg, 
                #0000 0
            ) left 0 bottom var(--s) / calc(2 * var(--s)) var(--s) repeat-x,
            conic-gradient(
                from -45deg at bottom, 
                var(--c) 90deg, 
                #0000 0
            ) left var(--s) bottom 0 / calc(2 * var(--s)) var(--s) repeat-x;
        mask: 
            var(--g) content-box,
            var(--g) 0 / var(--p, 0%) padding-box no-repeat;
        transition: 0.5s;

        &:is(:hover, :focus, .active) {
            --p: 100%;
            color: var(--c);
        }
    }
}
```

  


我们分步骤来解释一下上面的代码。

  


首先我们使用多个渐变创建了一条锯齿线，它位于元素的底部：

  


```CSS
a  {
    --s: .125em;
    padding: 0 calc(2.1 * var(--s)) calc(2.1 * var(--s));
    background: 
        conic-gradient(from 135deg at top, var(--c) 90deg, #0000 0) left 0 bottom var(--s) / calc(2 * var(--s)) var(--s) repeat-x,
        conic-gradient(from -45deg at bottom, var(--c) 90deg, #0000 0) left var(--s) bottom 0 / calc(2 * var(--s)) var(--s) repeat-x;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8cd488bdb9d14cf68c7a8713112df4df~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=988&s=78204&e=png&b=fffbfa)

  


  


关于创建锯齿形相关的代码就不在这里解释了，如果你理解上面的代码有点困难的话，请移步阅读《 [CSS 图形：创建花式边框效果（Part1）](https://juejin.cn/book/7395119173656903715/section/7403197889905688585)》课程中关于“[创建锯齿状边框](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-0)”的内容，课程中详细介绍了如何使用 CSS 来创建锯齿边框！

  


我们将重点放在 `mask` 属性上。这个 `mask` 属性是由两个渐变组成的：

  


```CSS
a {
    /* CSS ... */
    mask: 
        --g: linear-gradient(#000 0 0);
        var(--g) content-box,
        var(--g) 0 / var(--p, 0%) padding-box no-repeat;
}
```

  


第一个渐变使用不透明的颜色（`#000`），覆盖内容区域（即 `mask-clip: content`），它会使文本可见，并隐藏了底部的锯齿形状。注意，`mask-clip` 的使用与 `background-clip` 类似。

  


第二个渐变会覆盖整个区域（即 `mask-clip: padding-box`）。它的宽度由 `--p` 变量定义，并放置在元素的左侧。

  


现在，我们要做的就是在悬停时更改 `--p` 的值。来为第二个渐变创建滑动效果，进而显示下划线：

  


```CSS
a {
    /* CSS ... */
    mask: 
        --g: linear-gradient(#000 0 0);
        var(--g) content-box,
        var(--g) 0 / var(--p, 0%) padding-box no-repeat;
    transition: .5s;
    
    &:hover {
        --p: 100%;
    }
}
```

  


为了能让你更好的理解 `mask` ，我们可以先将 `mask` 层作为背景显示，这样可以更清楚地看到这个技巧发生的过程。

  


```CSS

        conic-gradient(from -45deg at bottom, var(--c) 90deg, #0000 0) left var(--s) bottom 0 / calc(2 * var(--s)) var(--s) repeat-x,
        linear-gradient(red 0 0) content-box,
        linear-gradient(lime 0 0) 0 / var(--p, 0%) padding-box no-repeat;

}
```

### 使用 `clip-path` 创建 3D 悬停效果

  


通常情况我们会借助 `box-shadow` 、`text-shadow` 以及伪元素 `::before` 或 `::after` 来创建 3D 效果。不过，在这里我要向大家展示的是基于一个元素，使用 CSS 的背景、`clip-path` 和 `transform` 创造出完美的 3D 效果。

  


这个效果看起来是我们在与 3D 元素互动，但实际上我们只是使用 2D 的技巧来绘制一个 3D 盒子。先看效果吧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f6bf7c6b23b34decb55beff560dfe821~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1338&h=482&s=250062&e=gif&f=87&b=fffaf9)

  


> Demo 地址：https://codepen.io/airen/full/yyywXJX

  


```CSS
h1 {
    --c: #f0a;
    --b: 0.1em;
    --d: 20px;
    --s: calc(var(--d) + var(--b));

    border: solid #0000;
    border-width: var(--b) var(--b) var(--s) var(--s);
    padding: 0.25em;
    cursor: pointer;
    background: 
        conic-gradient(
            at left var(--d) bottom var(--d),
            #0000 90deg,
            rgb(255 255 255 /0.3) 0 225deg,
            rgb(255 255 255 /0.6) 0
        ) border-box,
        conic-gradient(
            at left var(--s) bottom var(--s),
            #0000 90deg,
            var(--c) 0
        ) 0 100% / calc(100% - var(--b)) calc(100% - var(--b)) border-box;
    translate: calc(var(--d) / -1), var(--d);
    clip-path: polygon(
        var(--d) 0%,
        var(--d) 0%,
        100% 0%,
        100% calc(100% - var(--d)),
        100% calc(100% - var(--d)),
        var(--d) calc(100% - var(--d))
    );
    transition: 0.5s;
      
    &:hover {
        translate: 0;
        clip-path: polygon(
            0% var(--d),
            var(--d) 0%,
            100% 0%,
            100% calc(100% - var(--d)),
            calc(100% - var(--d)) 100%,
            0% 100%
        );
        color: var(--c);
    }
}
```

  


我们来逐步解释一下上面的代码。首先，我们定义 CSS 变量：

  


```CSS
h1 {
    --c: #f0a; /* 颜色 */
    --b: 0.1em;/* 边框厚度 */
    --d: 20px; /* 立方体深度 */
    
}
```

  


然后，我们创建一个透明边框，边框厚度使用上面定义的变量，并且新增 `--s` 变量，它的值为 `--b` 和 `--d` 的和。元素的顶部和右侧边框厚度为 `--b` ，而底部和左侧则为 `--s` 。为了使大家能更易于理解，我暂且将边框颜色设置有透明度的颜色：

  


```CSS
h1 {
    --c: #f0a; /* 颜色 */
    --b: 0.1em;/* 边框厚度：top 和 right */
    --d: 20px; /* 立方体深度 */
    --s: calc(var(--d) + var(--b)); /* 边框厚度： bottom 和 left */

    border: solid #0000;
    border-width: var(--b) var(--b) var(--s) var(--s);
}
```

接下来，使用 `conic-gradient()` 定义两个渐变，覆盖之前定义的所有边框区域：

  


```CSS
h1 {
    --c: #f0a; /* 颜色 */
    --b: 0.1em;/* 边框厚度：top 和 right */
    --d: 20px; /* 立方体深度 */
    --s: calc(var(--d) + var(--b)); /* 边框厚度： bottom 和 left */
```
