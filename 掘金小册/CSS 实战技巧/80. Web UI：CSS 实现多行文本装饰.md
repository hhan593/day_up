任何一个 Web 应用，它都有一些重要内容需要以更突出的方式呈现给用户。CSS 提供了多种不同的方式来样式化这些重点内容或信息元素。比如，使用 `text-emphasis` 给文本添加重点符、`text-decoration` 给文本添加下划线等。

  


然而，仅仅使用 `text-emphasis` 和 `text-decoration` 可能无法满足更多的设计需求。庆幸的是，现代 CSS 中提供了更多的技巧，可以帮助 Web 开发者为重点内容定制更为美观和个性化的装饰效果，比如不规则的、渐变的下划线以及更丰富的背景等。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd5e0ba1bb1d4ce28c5754f70b515adf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1366&h=881&s=14696&e=avif&b=fdfcfc)

  


但是，Web 的布局和内容是动态化的，当内容发生换行时，可能这些样式会使 Web UI 效果变得不太美观，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/134029b6e38148f083225b69029f7014~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=652&s=335677&e=jpg&b=fefcfc)

  


接下来，我们将在这节课中与大家一起探讨，如何使用 CSS 给多行文本添加美观的装饰效果，并且要如何避免这种断裂的 UI 效果。

  


## 如何给文本添加装饰效果

  


假设 Web 上面有一段像下面这样的内容，其中有部分内容是需要着重突出的。我在 HTML 中使用 `<strong>` 来包裹这部分内容：

  


```HTML
<p>
    In positive news, It seems to have finally solved email(!!!). Been using it several weeks and <strong>no longer dealing with spam, lonng lists of "unread" messages, or sorting out annoying but important docs.</strong> The relief it so real"
```

很明显，`<strong>` 元素包裹的文本内容明显要更粗（通过 `font-weight` 设置它的粗细），告诉用户这段“信息”很重要。

  


当然，你或者你的设计师可能会认为，默认的加粗效果，并不能满足设计上的需求，或能需要更显眼的方式来突出它。在 CSS 中，我们有多种方式可以做到。比如，使用 `text-emphasis` 添加提示符：

如果你不知道 `box-decoration-break` 是做什么的，不用着急，接下来我们一起来堔索它，并且弄清楚什么情况下需要使用 `box-decoration-break` 。

  


## 什么是盒子？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7aba59acd35a4324a6c5e0aef308d623~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=1024&s=151102&e=jpg&b=d3c3ae)

  


首先，我们从“盒子”开始！

  


如果你阅读过小册之前的《[CSS 布局：CSS 中的各种 `*-box`](https://juejin.cn/book/7395119173656903715/section/7445257036046991410)》课程，那么应该还记得。我在课程中提到过，在 CSS 中，一切都是“盒子”。Web 页面中的每一个元素都是一个盒子，包括你添加的每一个伪元素（比如 `::after` 、`::before` 或 `::marker`），甚至每一个文本节点也是一个盒子。

  


换句话说，整个 Web 页面是由多个盒子（矩形）组成。有不同类型的盒子（矩形），但对于 `box-decoration-break` 而言，我们只需要知道两种类型：块级盒子和内联盒子。它们分别对应着“块级元素”和“内联元素”，比如 `<p>` 元素就是一个“块级元素”，而 `<strong>` 就是一个“内联元素”。

  


```HTML
<p>
    In positive news, It seems to have finally solved email(!!!). Been using it several weeks and <strong>no longer dealing with spam, lonng lists of "unread" messages, or sorting out annoying but important docs.</strong> The relief it so real"
</p>
<p>
    In positive news, It seems to have finally solved email(!!!). Been using it several weeks and <strong>no longer dealing with spam, lonng lists of "unread" messages, or sorting out annoying but important docs.</strong> The relief it so real"
</p>
```

  


```CSS
p {
    font-size: clamp(1rem, 3vw + 0.875rem, 1.25rem);
    line-height: 1.8;
    font-weight: 300;
    border: 2px dashed;
    padding: 1rem;

    strong {
        --color-highlight-primary: #ffef7e;
        --color-highlight-secondary: #b7f9e9;
        background-image: linear-gradient(
            90deg,
            var(--color-highlight-primary),
            var(--color-highlight-secondary)
        );
        background-size: 100% 42%;
        background-repeat: no-repeat;
        background-position: 0 85%;
        padding: 0 4px;
        margin: -4px;
        font-weight: 900;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b57fdc1c0f0948d1b9df89ac5bddc02a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2394&h=1228&s=412150&e=jpg&b=fefcfc)

  


正如你所看到的，当一个元素是块级元素时，它会占据父元素的整个宽度，如果在它之后添加一个新的块级元素，该元素会被放置在第一个块级元素的下方。比如示例中的两个 `<p>` 元素，它们会一个接一个地堆叠在一起。

  


内联元素则有所不同，它只占据它们所需要的空间，并与周围的文本“内联”显示，比如示例中的 `<strong>` 元素。

  


## 什么是装饰？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8132682513d44c45b95184b03bf21483~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=134400&e=jpg&b=2b3959)

  


默认情况下，CSS 中的元素盒子是一个透明的。我们可能通过 CSS 给盒子添加一些装饰性的效果，比如边框、背景、阴影（还包括外边距、内边距和 `clip-path` 或 `mask` ）等效果。它们能让你将一个透明的矩形盒子变成一个漂亮的发光按钮、一个悬浮的卡片或是一个内凹的输入框。

  


通常，这些装饰会应用到盒子的所有边：上、右、下和左。当然，你也可以只将这些装饰应用到盒子的某一边。比如下面这个示列，我们使用 CSS 渐变，使元素盒子看上去有填充一半高度的渐变背景效果：

  


```CSS
strong {
    --color-highlight-primary: #ffef7e;
    --color-highlight-secondary: #b7f9e9;
    background-image: linear-gradient(
        90deg,
        var(--color-highlight-primary),
        var(--color-highlight-secondary)
    );
    font-weight: 900;
    background-size: 100% 42%;
    background-repeat: no-repeat;
    background-position: 0 85%;
    padding: 0 4px;
    margin: -4px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/44b6f666d2514f6ebec111ae87b3d07f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2346&h=952&s=213671&e=jpg&b=556262)

  


或者带有不规则圆角渐变的背景效果：

  


```CSS
strong {
    --color-highlight-primary: #ffef7e;
    --color-highlight-secondary: #b7f9e9;
    background-image: linear-gradient(
        135deg, 
        var(--color-highlight-primary), 
        var(--color-highlight-secondary)
    );
    color: #000;
    padding: .025lh .5lh;
    border-radius: 50% 3em 50% 3em;
    font-weight: 900;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6f0208477e5c47e68bf27e5b101b6c68~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2554&h=956&s=264051&e=jpg&b=556262)

  


不难发现，它们都有一个共同的现象，元素在换行时，其装饰效果有一个断裂的现象。那么，什么是断裂呢？

  


## 什么是断裂？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2f49115238af4ecdb54d022b29fd5df9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2858&h=1502&s=78645&e=png&b=c1deee)

  


首先我们要明确一点是，Web 的内容是动态的，它有可能多，也有可能少。但是，当内容比较多的时候，需要以多行的方式来显示。这意味着，元素的矩形盒子会分成多行，此时它有一个明显的现象——矩形盒子断裂。比如，我们给内联元素 `strong` 设置一个边框，请仔细观察内容少（不换行，不会有断裂现象）和内容多（换行，有断裂现象）的渲染效果：

  


```CSS
strong {
    font-weight: 900;
    border: 2px solid;
    color: #000;
    padding: 0.025lh 0.25lh;
    border-radius: 50% 3em 50% 3em;
    line-height: 2;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2c6fa9df9dac4a09af0f99ff6e0bb8f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2442&h=1028&s=251464&e=jpg&b=566363)

  


这也意味着，断裂过程可能不会按照你预期的方式发生——它会因元素（块元素或内联元素）不同，断裂的形式也会略有不同。

  


简单的说，断裂会分为内联断裂和块级断裂！

  


### 内联断裂

  


内联断裂是指当一个内联元素，例如上面示例中的 `<strong>` 元素包裹的文本内容，跨越多行时发生的情况。通常情况下，内联元素的内容跨越多行时并不会有任何问题。但是，当你试图给它添加背景或边框时，就会发生断裂现象。比如：

  


```CSS
strong {
    font-weight: 900;
    border: 2px solid;
    color: #000;
    padding: 0.025lh 0.25lh;
    border-radius: 50% 3em 50% 3em;
    line-height: 2;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ba507a8b3198478f90faa600156b72f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2442&h=1028&s=251464&e=jpg&b=566363)

  


示例中的 `<strong>` 元素添加了不规则的圆角和边框。你会发现，仅在 `<strong>` 元素的开头和结尾呈现圆角效果，而且边框的连接也因内容换行造成了断裂。

  


### 块级断裂

  


块级断裂发生在你将内容分为多列（使用多列布局）时，或者在打印时，内容被分为多个页面。

  


在以下示例中，内容被分成了多列，每个段落都有一个边框：

  


```CSS
.columns {  
    column-count: 5;
    
    p {
        border: 5px solid #34c9eb;  
        padding: 0.5em;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2f5207dd5ed245d3becbe4f7c87b40e3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2466&h=1730&s=595297&e=jpg&b=fefefe)

  


列的边框被切割了。

  


不管是内联断裂还是块级断裂，它们都会影响到 UI 视觉的美观。这个时候我们就需要引用 CSS 的 `box-decoration-break` 属性，它会告诉浏览器，当元素发生断裂时，应该如何处理！

  


## 引入 `box-decoration-break`

  


CSS 的 `box-decoration-break` 会告诉浏览器如何处理这些断裂。它提供了两种方式：

  


-   `slice`：元素被按照盒子被切割前的原始样式渲染，之后，针对每个行/列/页面将此假设框渲染成片段。请注意，假设框对于每个片段可以是不同的，因为如果中断发生在行内方向，则它使用自己的高度，如果中断发生在块方向，则它使用自己的宽度
-   `clone`：每个框片段与指定的边框、填充和边距独立呈现。`border-radius`、`border-image`、`box-shadow`独立地应用于每个片段，每个片段的背景也是独立绘制的，这意味着使用 `background-repeat``: no-repeat` 的背景图片仍然可能重复多次

  


以上面示例中的内联元素（`<strong>`）为例。当 `box-decoration-break` 的值为 `slice` 时，浏览器会将内联元素渲染为一条长线，然后把这条线分割成多个部分，导致除了第一行和最后一行之外，所有行的左右两侧都不会有圆角边框。

  


如果我们将 `box-decoration-break` 设置为 `clone` 时，浏览器会将这些装饰复制到每个部分，这意味着每一行都变成了自己的小盒子。

  


```CSS
strong {
    font-weight: 900;
    border: 2px solid;
    color: #000;
    padding: 0.025lh 0.25lh;
    border-radius: 50% 3em 50% 3em;
    line-height: 2;
    box-decoration-break: clone;
}
```

  


这不仅会让你期望的圆角边框重新出现，还可以看到左右两侧的内边距现在也应到了每一行，而不仅仅是应用于开头和结尾：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a947d8b9d65d4f4f94f4235d1d3064f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=462&s=517565&e=gif&f=142&b=fefefe)
