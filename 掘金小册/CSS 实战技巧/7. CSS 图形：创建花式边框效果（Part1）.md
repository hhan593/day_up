## 创建锯齿状边框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bf8f2ec590cc4429a06fe76697df1c07~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1202&h=648&s=313408&e=gif&f=196&b=fd4a1d)

  


如上图所示，使用 CSS 可以在元素的任意一侧、两侧或所有侧面应用锯齿边框。让我们从底部开始！

  


如果你对 [CSS 渐变](https://juejin.cn/book/7199571709102391328/section/7199845781149810727)（例如，线性渐变、径向渐变和锥形渐变等）比较熟悉的话，我想你首先会想到 [CSS 的锥形渐变 conic-gradient](https://juejin.cn/book/7223230325122400288/section/7259668771856941111) ，因为它最易于绘制一个锯齿形状。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/74ed35e1161747d1a04d27ff56337d9c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1226&h=762&s=1714445&e=gif&f=220&b=2c2f36)

  


> URL: https://gradient.style/

  


```CSS
.element {
    background-image: 
        conic-gradient(
            from 315deg at 50% 100% in oklab,
            oklch(70% 0.5 340) 25% 25%, oklch(90% 0.5 200) 0% 0%
        );
}
```

  


上面的代码中的 `conic-gradient` 渐变相当于创建了一个最小单元的锯齿，这个单元的具体尺寸由 CSS 的 `background-size` 来决定，例如 `80px 100%` 。你也可以这样理解，`conic-gradient` 渐变创建了一张宽度为 `80px` （这个值是一个假设值）和高度为 `100%` （等于元素的高度）的带有锯齿形状的背景图。有了这张背景图之后，我们就可以像使用 PNG 图片一样，将其填充到整个元素当中，将 `background-repeat` 设置为 `repeat` （或 `repeat-x`）就可以做到：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4edbf950250a4f848f116e835c9693cc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2493&h=1104&s=569354&e=jpg&b=376366)

  


具体的 CSS 代码如下：

  


```CSS
.zig-zag {
    background-image: 
        conic-gradient(
            from 315deg at 50% 100% in oklab,
            oklch(70% 0.5 340) 25% 25%, oklch(90% 0.5 200) 0% 0%
        );
    background-size: 80px 100%;
    background-position: bottom center;
    background-repeat: no-repeat;
    
    &:hover {
        background-repeat: repeat-x;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/569851d60cfc4c1fa2b8400dd8ea6993~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1210&h=670&s=951941&e=gif&f=145&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/yLdzdjo

  


注意，示例中 `conic-gradient` 渐变颜色的停止点也可以使用具体的角度值来描述，例如第一个颜色的停止点 `25%` ，它等同于 `90deg` （`25% × 360deg = 90deg`）：

  


```CSS
.zig-zag {
    background-image: 
        conic-gradient(
            from 315deg at 50% 100% in oklab,
            oklch(70% 0.5 340) 90deg, oklch(90% 0.5 200) 0deg
        );
}
```

  


需要注意的是，[CSS 渐变通常会在两颜色边缘处产生锯齿](https://juejin.cn/book/7199571709102391328/section/7199845781149810727#heading-16)。为了避免这种情况，我们要在颜色之间添加一个轻微的过渡，例如将 `oklch(70% 0.5 340) 90deg, oklch(90% 0.5 200) 0deg` 调整为 `oklch(90% 0.5 200), oklch(70% 0.5 340) 1deg 89deg, oklch(90% 0.5 200) 90deg` ：

  


```CSS
.zig-zag {
    background-image: 
        conic-gradient(
            from 315deg at 50% 100% in oklab,
            oklch(90% 0.5 200), oklch(70% 0.5 340) 1deg 89deg, oklch(90% 0.5 200) 90deg
        );
}
```

  


到目前为止，我们使用 `conic-gradient` 已经将元素底部变成了一个锯齿形状。只不过为了能让大家更易于理解，示例中使用了两个都不透明度的颜色。也就是说，只需要将 `conic-gradient` 中的蓝色（`oklch(90% 0.5 200)`）替换成透明色 `#0000` ，把紫色（`oklch(70% 0.5 340)`）替换成你喜欢的任意颜色即可。

  


```CSS
.zig-zag {
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    
    background: conic-gradient(from 315deg at 50% 100% in oklab,#0000, var(--bg) 1deg 89deg, #0000 90deg) 50% 100%/80px 100% repeat-x;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/740e92609db546d2bba142de1170fcb5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1376&h=712&s=6085276&e=gif&f=196&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/eYwGwLV

  


注意，示例中的颜色[使用了高清颜色](https://juejin.cn/book/7223230325122400288/section/7233227753909125178)，并且通过 `color-contrast() 函数`根据背景颜色来动态调整文本颜色。它们都是[现代 CSS 中的一些新特性](https://s.juejin.cn/ds/irS35qGu/)，如果你对这方面的知识感兴趣的话，请移步阅读《[新的 CSS 颜色空间：为 Web 设置高清颜色](https://juejin.cn/book/7223230325122400288/section/7233227753909125178)》和《[颜色对比度：`color-contrast()`](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)》！

  


现在，我们知道如何使用 `conic-gradient` 渐变来创建锯齿形状了。不过之前所演示的案例中，所有锯齿的大小都是一个假设值，即 `80px 100%` 。那么，如何精确控制锯齿的尺寸呢，尤其是宽度值？

  


接下来，我们就一起来探讨，应该如何才能精确的控制锯齿的大小。简单地说，我们可以从之前的步骤中提取两个变量来定义锯齿形状：大小（`--s`）和角度（`--a`）。

  


```CSS
.zig-zag {
    --s: 40px;  /* 锯齿的大小 */
    --a: 90deg; /* 角度 */
}
```

  


然后，我们可以通过 [CSS 的三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)计算出所需要的“宽度”值（`--w`）。我用下图来描述它们之间的关系：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/86fa3b41d4514816ab74dec2772a0c70~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2552&s=1048359&e=jpg&b=376366)

  


根据上图，我们把创建锯齿形状所需要的参数以 CSS 的自定义属性存储起来，最终的 CSS 如下：

  


```CSS
.zig-zag{
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2);
    --g: conic-gradient(from calc(var(--a) / -2) at bottom, #0000, var(--bg) 1deg calc(var(--a) - 1deg), #0000 var(--a)) 50%/var(--w) 100% repeat-x;
    background: var(--g);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/28270fc504c94823b1c6858ed0d190eb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1126&h=700&s=6337435&e=gif&f=374&b=31565c)

  


> Demo 地址：https://codepen.io/airen/full/eYweJRQ

  


到目前为止，虽然我们使用 `conic-gradient` 和三角函数使元素在底部带有一个锯齿形状，但将它作为 `background` 使用，还是有诸多的限制性，比如元素只能是纯色，比如不适用于 `<img>` 元素上。我们可以使用 `mask` 来替代 `background` ，就能完美的避免刚才所提到的一些限制：

  


```CSS
.zig-zag{
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2);
    --g: conic-gradient(from calc(var(--a) / -2) at bottom, #0000, var(--bg) 1deg calc(var(--a) - 1deg), #0000 var(--a)) 50%/var(--w) 100% repeat-x;
    mask: var(--g);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fbe1b206f9894664b0d180bf598b021f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1262&h=572&s=10189644&e=gif&f=204&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/BagmjVw

  


类似于底部锯齿形状，我只需做一点点调整，就可以创建顶部、右侧和左侧的锯齿形状：

  


```CSS
.zig-zag {
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2); /* 锯齿背景图宽度或高度 */
    mask: var(--g);
    
    /* 底部锯齿形状 */
    &.zig-zag--bottom {
        --g: conic-gradient(
            from calc(var(--a) / -2) at bottom, 
            #0000, 
            var(--bg) 1deg calc(var(--a) - 1deg), 
            #0000 var(--a)
        ) 50%/var(--w) 100% repeat-x;
    }
    
    /* 顶部锯齿形状 */
    &.zig-zag--top {
        --g: conic-gradient(
            from calc(180deg - var(--a) / 2) at top,
            #0000,
            var(--bg) 1deg calc(var(--a) - 1deg),
            #0000 var(--a)
        ) 50%/var(--w) 100% repeat-x;
    }
    
    /* 左侧锯齿形状 */
    &.zig-zag--left {
        --g: conic-gradient(
            from calc(90deg - var(--a) / 2) at left,
            #0000,
            var(--bg) 1deg calc(var(--a) - 1deg),
            #0000 var(--a)
        ) 50%/100% var(--w);
    }
    
    /* 右侧锯齿形状 */
    &.zig-zag--right {
        --g: conic-gradient(
            from calc(-90deg - var(--a) / 2) at right,
            #0000,
            var(--bg) 1deg calc(var(--a) - 1deg),
            #0000 var(--a)
        ) 50%/100% var(--w);
    }
}
```

  


仔细观察上面的代码，它们有很多共同点，我们可以把相同的部分提取出来，并将其存储在另一个 CSS 自定义属性中，例如：

  


```CSS
.zig-zag {
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2); /* 锯齿背景图宽度或高度 */
    --g: #0000, var(--bg) 1deg calc(var(--a) - 1deg), #0000 var(--a); /* 渐变颜色过渡 */
    mask: var(--_g);
    
    /* 底部锯齿形状 */
    &.zig-zag--bottom {
        --_g: conic-gradient(
            from calc(var(--a) / -2) at bottom, 
            var(--g)
        ) 50%/var(--w) 100% repeat-x;
    }
    
    /* 顶部锯齿形状 */
    &.zig-zag--top {
        --_g: conic-gradient(
            from calc(180deg - var(--a) / 2) at top,
            var(--g)
        ) 50%/var(--w) 100% repeat-x;
    }
    
    /* 左侧锯齿形状 */
    &.zig-zag--left {
        --_g: conic-gradient(
            from calc(90deg - var(--a) / 2) at left,
            var(--g)
        ) 50%/100% var(--w) repeat-y;
    }
    
    /* 右侧锯齿形状 */
    &.zig-zag--right {
        --_g: conic-gradient(
            from calc(-90deg - var(--a) / 2) at right,
            var(--g)
        ) 50%/100% var(--w) repeat-y;
    }
}
```

  


你可能发现了，我们只调整了 `conic-gradient` 渐变的起始角度（`from` 关键词之后的角度）、圆心位置、图片尺寸（`background-size` 或 `mask-size`）和平铺方式（`background-repeat` 或 `mask-repeat`）：

  


-   **底部锯齿**，渐变的旋转角度为 ` calc(var(--a)  ``/ -2``)` ，圆心位于底部（`at bottom` ），尺寸为 `var(--w) 100%` ，并且水平方向平铺 （`repeat-x`）
-   **顶部锯齿**，渐变的旋转角度为 ` calc(180deg - var(--a)  ``/ 2``)` ，圆心位于顶部（`at top`），尺寸为 `var(--w) 100%` ，并且水平方向平铺（`repeat-x`）
-   **左侧锯齿**，渐变的旋转角度为 ` calc(90deg - var(--a)  ``/ 2``)` ，圆心位于左侧（`at left`），尺寸为 `100% var(--w)` ，并且垂直方向平铺（`repeat-y`）
-   右侧锯齿，渐变的旋转角度为 ` calc(-90deg - var(--a)  ``/ 2``)` ，圆心位于右侧（`at right`），尺寸为 `100% var(--w)`，并且垂直方向平铺（`repeat-y`）

  


现在，你只需要调整几个变量的值，例如 `--bg` 、`--s` 和 `--a` ，就能得到不同的锯齿形状：

有了这个基础之后，你要在元素的两侧添加锯齿形状就变得非常容易。我们实际上可以重用相同的代码，要同时获得顶部和底部锯齿形状，只需要将顶部和底部的代码给合在一起。简单地说，`background` 或 `mask` 会同时应用两个 `conic-gradient` ，并且 `background-size` 或 `mask-size` 的高度不再是 `100%` ，通常会使用 `51%` ，这样避免元素中间留白。与此同时不要忘记调整 `background-position` 或 `mask-position` 值。同样的逻辑也适用于左右两侧都有锯齿形状的情况。具体的代码如下：

  


```CSS
.zig-zag {
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2); /* 锯齿背景图宽度或高度 */
    --g: #0000, var(--bg) 1deg calc(var(--a) - 1deg), #0000 var(--a); /* 渐变颜色过渡 */
    mask: var(--g1), var(--g2);
    
    &.zig-zag--up-down {
        /* 顶部锯齿 */
        --g1: conic-gradient(
            from calc(180deg - var(--a) / 2) at top,
            var(--g)
        ) top/var(--w) 51% repeat-x;
        /* 底部锯齿 */
        --g2: conic-gradient(
            from calc(var(--a) / -2) at bottom, 
            var(--g)
        ) bottom/var(--w) 51% repeat-x;
    }
    &.zig-zag--left-right {
        /* 左侧锯齿 */
        --g1: conic-gradient(
            from calc(90deg - var(--a) / 2) at left,
            var(--g)
        ) left/51% var(--w) repeat-y;
        /* 右侧锯齿 */
        --g2: conic-gradient(
            from calc(-90deg - var(--a) / 2) at right,
            var(--g)
        ) right/51% var(--w) repeat-y;
    }
}
```

你可能已经猜到了，如果元素的所有边上都应用锯齿，只需要将所有渐变组合在一起：

  


```CSS
.zig-zag {
    --s: 40px;  /* 锯齿大小 */
    --a: 90deg; /* 锯齿角度*/
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff, #aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(var(--a) / 2) * 2); /* 锯齿背景图宽度或高度 */
    --g: #0000, var(--bg) 1deg calc(var(--a) - 1deg), #0000 var(--a); /* 渐变颜色过渡 */
    --gt: conic-gradient(
            from calc(180deg - var(--a) / 2) at top,
            var(--g)
        ) top/var(--w) 51% repeat-x;
    --gr: conic-gradient(
            from calc(-90deg - var(--a) / 2) at right,
            var(--g)
        ) right/51% var(--w) repeat-y;    
    --gb: conic-gradient(
            from calc(var(--a) / -2) at bottom, 
            var(--g)
        ) bottom/var(--w) 51% repeat-x;    
    --gl: conic-gradient(
            from calc(90deg - var(--a) / 2) at left,
            var(--g)
        ) left/51% var(--w) repeat-y;    
    mask: var(--gt), var(--gr), var(--gb), var(--gl);
}
```

  


事实上，最终的结果并不是你所期望的那样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f1ed7bb095784a8c98f928813221e5e7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1258&h=652&s=1434316&e=gif&f=141&b=2e575b)

  


结果告诉我们，我们不能简单粗暴的将之前的每一侧锯齿形状拼接在一起。这是因为，四个渐变拼接在一起时，相互发生了遮挡，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1ca009be04184223b60ba15b587c4f29~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1754&s=664990&e=jpg&b=376366)

  


具体原因就不用在这里阐述了，因为这是 `background` 或 `mask` 最基础的知识。

  


换句话说，我们需要另辟蹊径来为元素四边同时添加锯齿形状。我们可以在原来四个锥形渐变 `coinc-gradient` （元素的每一次）的基础上新增一个线性渐变 `linear-gradient` ，并且将线性渐变放在中间。这种方法有一个关键技巧，那就是四个锥形渐变的尺寸要刚好等于锯齿的大小（`background-size` 或 `mask-size`），即 `s x w` （左右两侧） 或 `w x s` （上下两侧） ；线性渐变的尺寸是 `100% - 2s` （或 `100% - w`），如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b501fb5787794cb58eb96cc7c3d0b576~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=3119&s=1219631&e=jpg&b=376366)

  


另外，我们在锯齿的角度固定为 `90deg` ，这是经过实验之后得出的最佳角度值，因为这是唯一能够在不出现奇怪重叠的情况下创建更好看的角落的角度。请注意，`backround-repeat` 或 `mask-repeat` 的值需要应用 `space` ，而不是简单的 `repeat-x` 或 `repeat-y` ，使用该值主要是为了避免角落处出现不理想的效果。元素四边都具有锯齿形状的最终 CSS 代码如下：

  


```CSS
.zig-zag {
    --s: 40px; /* 锯齿大小 */
    --bg: #c73226; /* 主背景色 */
    --color: color-contrast(var(--bg) vs #fff,#aefeaf,#333); /* 控制文本颜色 */
    --w: calc(var(--s) * tan(45deg) * 2); /* 等同于 calc(var(--s) * 2)*/
    --_g: #0000, var(--bg) 1deg 89deg, #0000 90deg;
    
    /* 线性渐变，填充中间区域 */
    --g: linear-gradient(#000 0 0) center/calc(100% - var(--w)) calc(100% - var(--w)) no-repeat;
    /* 顶部锯齿 */
    --gt: conic-gradient(from 135deg at top, var(--_g)) 0 0 / var(--w) var(--s) space no-repeat;
    /* 右侧锯齿*/
    --gr: conic-gradient(from -135deg at right, var(--_g)) 100% 0 / var(--s)var(--w) no-repeat space;
    /* 底部锯齿 */
    --gb: conic-gradient(from -45deg at bottom, var(--_g)) 0 100% / var(--w) var(--s) space no-repeat;
    /* 左侧锯齿 */
    --gl: conic-gradient(from 45deg at left, var(--_g)) 0 0 / var(--s) var(--w) no-repeat space;
    
    mask: var(--g), var(--gt), var(--gr), var(--gb), var(--gl);
}
```

> Demo 地址：https://codepen.io/airen/full/poXdyEm

## 创建凹槽状边框

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f8e7836a0dc24a1fad9d82f1c28e53ea~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=302302&e=png&b=0e0f11)

  


凹槽状边框其实就是[上一节课中介绍的圆形角落切割的一种延伸版本](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-1)，它不再局限于元素框的角落，而是在元素的一侧，两侧或多侧平铺，使其外观看上去具有凹槽状的边框：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8d57deccb444412fbe40b8700afa7cb0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=352&s=297126&e=gif&f=210&b=fd4a1d)

  


这种 UI 效果经常用于区域分割、邮票或优惠券的 UI 上。

  


接下来，我们就来探讨 CSS 如何创建凹槽状边框。在上一节课中，我们介绍了如何使用 CSS 的径向渐变 `radial-gradient` 实现圆形角落切割。同样的，我们也可以使用径向渐变来制作凹槽形状边框。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/861ec8a28e5c4cd6a5d4a3bebd9991a5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1368&h=804&s=2497961&e=gif&f=368&b=282b31)

  


> URL:https://gradient.style/

  


正如你所看到的，使用 `radial-gradient` 可以在元素的任意一侧创建带还圆形凹槽的形状。我们可以像 `conic-gradient` 制作锯齿形状一样，将径向渐变 `radial-gradient` 绘制的凹槽形状沿着水平轴或垂直轴进行平铺。我们以底部凹槽形状边框为例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/56d5b510812244b98489fb2f0576a1da~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2493&h=1104&s=543247&e=jpg&b=376366)

  


与之前的锯齿形状边框相比，使用 `radial-gradient` 要更简单一些，因为不需要处理角度，只需要处理凹槽形状的大小，即圆半径。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/43c5c0d789f04c6fb9703be05e2bd2c5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2157&s=635524&e=jpg&b=376366)

  


再次说明，我们所做的只是将 `conic-gradient` 替换为以下内容：

  


```CSS
.scooped {
    --s: 40px; /* 凹槽大小 */
    --bg: #c73226; /* 主背景色 */
    --w: calc(1.85 * var(--s)); 
    --g: radial-gradient(var(--s) at bottom, #0000 98%, var(--bg)) 50% / var(--w) 100%;
    mask: var(--g);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5b19c3bd71f4b209e3b25582c524182~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1258&h=728&s=14435984&e=gif&f=304&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/wvLPdvj

  


`radial-gradient` 中透明颜色 `#0000` 停止点设置为 `98%` ，是易于理解的。因为我们多次提到过，使用 `98%` 替代 `100%` （理论上的值）是为了避免圆形产生锯齿边缘。但 `w` 的值，按理来说是 `s` 的两倍（`w = 2 * s`），为何这里设置 `1.85` 呢？

  


就这一点而言，我也无法找到合理的理论支撑，我只能告诉大家，`1.85` 就是一个实验出来的最佳值。我最初使用的也是 `2` ，但最终呈现的效果没有 `1.85` 那么理想，使用较小的 `1.85` 的值能使圆形之间的重叠更加平滑。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4ded7a5fd1f449219b862cc75b0bf1b0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1017&s=436874&e=jpg&b=376366)

  


现在，你可以基于底部凹槽形状边框的示例，在元素其他侧添加凹槽形状边框，你所要做的只是调整渐变圆心的位置，遮罩图（或背景图）尺寸。具体的代码如下：

  


```CSS
.scooped {
    --s: 40px; /* 锯齿大小 */
    --bg: #c73226; /* 主背景色 */
    --w: calc(1.85 * var(--s)); /* 遮罩图宽度 */
    --g: #0000 98%, var(--bg); /* 渐变颜色 */
    
    mask: var(--_g);

    /* 顶部凹槽形状 */
    &.scooped--top {
        --_g: radial-gradient(var(--s) at top, var(--g)) 50% / var(--w) 100% repeat-x;
    }
    
    /* 右侧凹槽形状 */
    &.scooped--right {
        --_g: radial-gradient(var(--s) at right, var(--g)) 50% / 100% var(--w) repeat-y;
    }

    /* 底部凹槽形状 */
    &.scooped--bottom {
        --_g: radial-gradient(var(--s) at bottom, var(--g)) 50% / var(--w) 100% repeat-x;
    }

    /* 左侧凹槽形状 */
    &.scooped--left {
        --_g: radial-gradient(var(--s) at left, var(--g)) 50% / 100% var(--w) repeat-y;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4e9e1e3c0c24ab2a93626549b7a5b72~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1656&h=580&s=18542670&e=gif&f=291&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/oNroWqe

  


有意思的是，在元素两侧（同时在顶部和底部或者左侧和右侧）添加凹槽形状，只用一个径向渐变即可实现：

  


```CSS
.scooped {
    --s: 40px; /* 凹槽大小 */
    --w: calc(1.85 * var(--s));
    --bg: #c73226; /* 主背景色 */
    --g:#0000 98%,var(--bg);
    --_g: radial-gradient(var(--s) at 50% var(--s),var(--g)) 50% 0/ var(--w) 100%;
    mask: var(--_g);
    
    &:hover {
        --_g:radial-gradient(var(--s) at 50% var(--s),var(--g)) 50% calc(-1 * var(--s))/ var(--w) 100%
    }
}
```

  


注意 `:hover` 状态下 `mask-position` 的变化，从 `50% 0` 过渡到 `50% -s` ，整个变化过程如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0527f6e5a340428089722ef71ba1d31d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1360&h=694&s=10268533&e=gif&f=348&b=2e575b)

> Demo 地址：https://codepen.io/airen/full/MWMOogy

  


最后一种情形是，元素所有边都有凹槽形状。它与锯齿形状边框类似，需要使用四个径向渐变和一个线性渐变，这也是唯一一种需要多个渐变才能实现凹槽形状的情景。而且，它比锯齿形状还更复杂一点，必须引入 CSS 的 `clip-path` 属性来修正角落重叠的问题。我们先来看未引入 `clip-path` 的效果：

  


```CSS
.scooped {
    --s: 40px; /* 凹槽大小 */
    --w: calc(1.85 * var(--s));
    --bg: #c73226; /* 主背景色 */
    --_g: #0000 98%, var(--bg);
    
    /* 填充中间区域 */
    --g: linear-gradient(#000 0 0) center/calc(100% - var(--s) * 2) calc(100% - var(--s) * 2) no-repeat;
    /* 顶部凹槽形状 */
    --gt: radial-gradient(circle farthest-side, var(--_g)) 0 calc(-1 * var(--s)) / var(--w) calc(var(--s) * 2) round no-repeat;
    /* 右侧凹槽形状 */
    --gr: radial-gradient(circle farthest-side, var(--_g)) calc(100% + var(--s)) 0 / calc(var(--s) * 2) var(--w) no-repeat round;
    /* 底部凹槽形状 */
    --gb: radial-gradient(circle farthest-side, var(--_g)) 0 calc(100% + var(--s)) / var(--w) calc(var(--s) * 2) round no-repeat;
    /* 左侧凹槽形状 */
    --gl: radial-gradient(circle farthest-side, var(--_g)) calc(-1 * var(--s)) 0 / calc(var(--s) * 2) var(--w) no-repeat round;
    
    mask: var(--g), var(--gt), var(--gr), var(--gb), var(--gl);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/407b4c3107ae465f9eae86b9bdbf00cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1260&h=696&s=3431744&e=gif&f=147&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/YzoEQXO

  


这就是为什么要引入 `clip-path` 的原因所在。我们可以像下面这样，使用八个点来对元素的角落进行裁剪，同时不影响其他边的效果：

  


```CSS
.scooped {
    clip-path: polygon(
        calc(2 * var(--s)) 0, 
        calc(100% - 2 * var(--s)) 0,
        100% calc(2 * var(--s)), 
        100% calc(100% - 2 * var(--s)),
        calc(100% - 2 * var(--s)) 100%, 
        calc(2 * var(--s)) 100%,
        0 calc(100% - 2 * var(--s)), 
        0 calc(2 * var(--s))
    )
}
```

  


最终的 CSS 代码如下：

  


```CSS
.scooped {
    --s: 40px; /* 凹槽大小 */
    --w: calc(1.85 * var(--s));
    --bg: #c73226; /* 主背景色 */
    --_g: #0000 98%, var(--bg);

    --g: linear-gradient(#000 0 0) center/calc(100% - var(--s) * 2)
      calc(100% - var(--s) * 2) no-repeat;
    --gt: radial-gradient(circle farthest-side, var(--_g)) 0 calc(-1 * var(--s)) /
      var(--w) calc(var(--s) * 2) round no-repeat;
    --gr: radial-gradient(circle farthest-side, var(--_g)) calc(100% + var(--s))
      0 / calc(var(--s) * 2) var(--w) no-repeat round;
    --gb: radial-gradient(circle farthest-side, var(--_g)) 0
      calc(100% + var(--s)) / var(--w) calc(var(--s) * 2) round no-repeat;

      calc(var(--s) * 2) var(--w) no-repeat round;
```

## 创建凸起状边框

  


这里所谓的“凸起状边框”指的是反过来的凹槽状边框：

对于这种边框，不管边的配置如何，我们总是需要两个渐变，一个径向渐变和一个线性渐变。其中径向渐变用来创建一个重复的圆形图案，然后用一个线性渐变来覆盖这些圆，只显示我们想要的边。继续以底部凸起边框为例：

```CSS
.scalloped {
    --s: 40px; /* 凹槽大小 */
    --w: calc(1.85 * var(--s)); /* 遮罩图宽度 */
    --bg: #c73226; /* 主背景色 */
    /* 线性渐变 */
    --gl: linear-gradient(to top, #0000 var(--s), var(--bg) 0);
    /* 径向渐变*/

}
```

同样的，只需调整一点点代码，我们就可以实现元素两侧或所有边都有凸起形状的边框效果：

  


```CSS
.scalloped {
    --s: 40px; /* 凹槽大小 */
    --w: calc(1.85 * var(--s));
    --bg: #c73226; /* 主背景色 */
    --g: radial-gradient(var(--s), var(--bg) 98%, #0000) 50%;
    mask: var(--gl), var(--gr);

    &.scalloped--up-down {
        --gl: linear-gradient(0deg, #0000 calc(2 * var(--s)), var(--bg) 0) 0 var(--s);
        --gr: var(--g) / var(--w) calc(2 * var(--s)) repeat space;
    }
    
    /* 左侧和右侧设置凸起形状 */
    &.scalloped--left-right {
        --gl: linear-gradient(-90deg, #0000 calc(2 * var(--s)), var(--bg) 0) var(--s);
        --gr: var(--g) / calc(2 * var(--s)) var(--w) space repeat;
    }

    /* 所有边设置凸起形状 */
    &.scalloped--all {
        --gl: linear-gradient(var(--bg) 0 0) 50% / calc(100% - var(--w)) calc(100% - var(--w)) no-repeat;
        --gr: radial-gradient(farthest-side, var(--bg) 98%, #0000) 0 0 / calc(2 * var(--s)) calc(2 * var(--s)) round;
    }
 }
```

这里所展示的波浪状应该是最简单的一种。如果要制作复杂的波浪形状，还是要 [SVG 路径](https://juejin.cn/book/7341630791099383835/section/7368317966821425190)和 [SVG 遮罩](https://juejin.cn/book/7341630791099383835/section/7366549423813296165)。但这并不意味着，使用 CSS 就不能制作出漂亮的波浪形状。接下来，我将带领大家一起来揭示 CSS 创建复杂波浪形状的所有秘密以及其背后逻辑！

  


事实上，任何有上下曲线的形状都可以被称为“波浪”，所以我们利用一些几何知识（比如三角函数）可以创建出复杂的波浪形效果。

  


让我们从最简单的示例开始，使用两个半径相同的圆，并且将它们并排放置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/695167ca92514e22885d25a81542d41b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1480&s=336624&e=jpg&b=376366)

  


如上图所示，第一个圆的上半部分和第二个圆的下半部分的红线组成了一条最简单的波浪线。将它们重复几次，呈现的波浪线更为明显：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0ac3827b630b4c8ea78f709eb9433ee4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=961&s=215797&e=jpg&b=3c6b6b)

  


现在，让我们填充底部（或顶部）部分，得到如下效果：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1f36aed257794f67925375f2f2ea0f20~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=961&s=134993&e=jpg&b=f72121)

  


就这么简单，我们得到了一个波浪形状，而且可以通过一个变量来控制圆形的半径。我们之前制作的最简单的波浪形状，就是这么做的。

  


我们稍微加点料，在之前的两个圆形上增加一点复杂性。例如，两个圆不再并排排列（圆心不在同一水平线上）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/63de6410977d465282f8ea93b8e9d020~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1480&s=449354&e=jpg&b=376366)

  


在这种情况下，红线不再覆盖每个圆的一半面积，而是覆盖较小的区域。这个区域由上图中紫色虚线控制，它穿过了两个圆的相遇点。要是你将上图中红线线重复一下，会得到一个更平滑的波浪形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/671e0ce5e87c42abb9d3a392897425e3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3586&h=1480&s=562389&e=jpg&b=3f706e)

  


通过这几张示意图，我想你也明白了。通过控制圆形的位置和大小，我们可以创建任何我们想要的波浪形状。我们甚至还可以为它们创建变量，比如 `p` 和 `s` ：

  


-   `s` 用来控制波浪大小
-   `p` 用来控制波浪的曲率

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7158047733a54ebf9f9acb28de812b22~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1480&s=383507&e=jpg&b=376366)

  


在这里将 `p` 设置为 `p = m * s` ，其中 `m` 是在更新波浪曲率时调整的变量。这使得我们即使更新 `s` ，也能始终保持相同的曲率。`m` 可以是 `0 ~ 2` 之间的任何值。其中 `0` 会给我们第一个特例，即两个圆水平对齐；`2` 是一个最大值。我们可以设得更大，但经过测试，发现任何大于 `2` 的值都会使波浪形状产生不良的效果。

  


接下来，我们就可以通过[三角函数](https://juejin.cn/book/7223230325122400288/section/7242216512176521277)计算出圆的半径 `r` ：

  


```
(2r)² = p² + s²
r = sqrt(p² + s²) / 2
r = sqrt((m * s)² + s²) / 2
r = sqrt(m² * s² + s²) / 2
r = s * sqrt(m² + 1) / 2
```

  


当 `m` 等于 `0` 时，`p` 也会等于 `0` ，那么 `r = s / 2` ；当 `m` 等于 `2` 时，`p` 会等于 `2s` ，那么 `r = s * sqrt(5) / 2` 。

  


有了这个基础之后，我们就可以开始将这些转化为 CSS 中的渐变效果了！

  


我们的波浪使用圆形，而 CSS 径向渐变是绘制圆形的不二之选。正如之前所述，我们的波浪形状是由两个圆定义的，从这一点来说，我们将使用两个径向渐变来创建这两个圆。

  


我们从两个圆在同一水平线（即 `p=0`）这个特例开始：

  


```CSS
.wavy {
    --r: 40px; /* 凹槽大小 */
    --bg: #c73226; /* 主背景色 */
    background: radial-gradient(var(--r) at 50% 0px,#0000 99%,var(--bg) 101%) 50% var(--r)/calc(4 * var(--r)) 100% no-repeat;
    
    &:hover{
        background-repeat: repeat-x;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/17150e63ae7b4a84a2a60ade9b47af8f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1322&h=662&s=859003&e=gif&f=135&b=2e575b)

代码中的 `radial-gradient` 创建了第一个曲率，同时填充了整个底部区域（它可以说是波浪的“水”）。在这里，将径向渐变的半径定义为 `--r` 。如果我们将其与 `s` 变量相比，它等于 `s/2` ，因为 `p=0` （也就是 `m=0`）。

```CSS
.wavy {
```
```CSS
    --r: 40px; /* 凹槽大小 */
    --w: calc(4 * var(--r)); /*遮罩或背景图片宽度*/
    --h: calc(2 * var(--r)); /*遮罩或背景图片高度*/
    --bg: #c73226; /* 主背景色 */
```

```CSS
        var(--r) at 50% var(--r), 
        var(--bg) 99%, 
        #0000 101%)  
        calc(50% - var(--h)) 0 / var(--w) 100% repeat-x;
    --g2: radial-gradient(
        var(--r) at 50% 0px, 
        #0000 99%, 
        var(--bg) 101%) 
        50% var(--r) / var(--w) 100% repeat-x;
    mask: var(--g1), var(--g2);
}
```

  


接下来，我们在上面这个代码的基础上新增一个变量 `--p`，使得我们在创建任何波浪形状时可以重用这段代码。

  


之前在介绍其背后原理的时候，我们提到，这种方法的关键技巧是移动圆形，使它们不再对齐（不在同一水平线上，即 `p` 不等于 `0`），所以我们将更新每个圆形的位置（即径向渐变的圆心位置）。我们将第一个圆形向上移动，第二个圆形向下移动。具体的 CSS 代码如下：

  


```CSS
.wavy {
    --s: 40px; 
    --p: 25px;
    --w: calc(4 * var(--s)); /*遮罩或背景图片宽度*/
    --h: calc(2 * var(--s)); /*遮罩或背景图片高度*/
    --bg: #c73226; /* 主背景色 */

    --g1: radial-gradient(
        var(--s) at 50% calc(var(--s) + var(--p)), 
        var(--bg) 99%, 
        #0000 101%)  
        calc(50% - var(--h)) 0 / var(--w) 100% repeat-x;
    --g2: radial-gradient(
        var(--s) at 50% calc(-1 * var(--p)), 
        #0000 99%, 
        var(--bg) 101%) 
        50% var(--s) / var(--w) 100% repeat-x;
    mask: var(--g1), var(--g2);
}
```

  


> 注意，我们把之前的 `--r` 替换为 `--s` ，因为之后的示例将使用 `--r` 来表示圆形的半径。

  


正如你所看到的，引入的 `--p` 变量来定义每个圆形的中心位置。第一个渐变（`--g1`）使用 `50% calc(var(--s) + var(--p))` ，使其中心向上移动；第二个渐变（`--g2`）使用 `50% calc(-1 * var(--p))` ，使其中心向下移动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d99f50b418194a3a984f15be08ab7e97~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1324&h=648&s=1075017&e=gif&f=152&b=2e575b)

  


> Demo 地址：https://codepen.io/airen/full/VwJrgoX

  


不难发现，现在这些圆既不对齐也不相互接触：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6187f5e5eb54b4aa1c073c84ebd26a3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2334&s=618437&e=jpg&b=376366)

  


造成这种现象的主要原因是我们将它们分开得很远，并且没有改变它们的半径，因此呈现给你的并不是所期望的波浪形状。但我们可以通过使用之前的计算公式来修复这个问题。记住，`r = sqrt(p² + s²)/2`。在我们的例子中，`--s` 等于 `s/2`；同样，`--p` 也等于 `p/2`，因为我们移动了两个圆形。因此，它们的中心点之间的距离是 `--p` 的两倍：

  


```
r = sqrt(pow(var(--s), 2) + pow(var(--p), 2))
```

  


注意，这里我们使用了 CSS 的 `pow()` 函数，其中 `pow(var(--s), 2)` 等同于 `var(--s) * var(--s)` 。

  


现在，我们把这个公式应用到 CSS 中：

  


```CSS
.wavy {
    --_s: 40;
    --_p: 25;
    --s: calc(var(--_s) * 1px);
    --p: calc(var(--_p) * 1px);
    --r: calc(sqrt(pow(var(--_s), 2) + pow(var(--_p), 2)) * 1px);

    --w: calc(4 * var(--s)); /*遮罩或背景图片宽度*/
    --h: calc(2 * var(--s)); /*遮罩或背景图片高度*/
    --bg: #c73226; /* 主背景色 */

    --g1: radial-gradient(
        var(--r) at 50% calc(var(--s) + var(--p)),
        hsl(from var(--bg) 50 s l) 99%,
        #0000 101%
      )
      calc(50% - var(--h)) 0 / var(--w) 100% no-repeat;
    --g2: radial-gradient(
        var(--r) at 50% calc(-1 * var(--p)),
        #0000 99%,
        var(--bg) 101%
      )
      50% var(--s) / var(--w) 100% no-repeat;

    background: var(--g1), var(--g2);

    &:hover {
        background-repeat: repeat-x;
    }
}
```

  


注意，示例中为了使 `sqrt()` 和 `pow()` 函数在 CSS 中生效，特意引入了不带单位的 `--_s` （相当于 `--s`）和 `--_p` （相当于 `--p`）。

四条边都有波浪形边框呢？制作它会比之前所有边有锯齿和凹槽形状的边框要棘手得多，你可能需要九个渐变，其中八个径向渐变，每两个径向渐变用于制作元素的波浪形状边框；另外一个是线性渐变，用于填充中间的空白区域。感兴趣的同学可以尝试一下！

  


最后再向大家展示一下波浪线的制作，因为它也是一种波浪形状。我们在制作波浪线形状时，引入了一个变量 `--b` 来控制波浪线的粗细，具体代码如下：

  


```CSS
.wavy {
    --s: 44px;
    --b: 16px;
    --m: 0.4;
    --r: calc(var(--s) * sqrt(pow(var(--m), 2) + 1) + var(--b) / 2);
    --bg: #c73226; /* 主背景色 */
    background: var(--bg);

    /* 水平波浪线 */
    &.wavy--line-h {
        --_g: #0000 calc(99% - var(--b)), var(--bg) calc(101% - var(--b)) 99%, #0000 101%;
        height: calc(2 * var(--r));
        width: 40vw;
      
        mask: 
            radial-gradient(
                var(--r) at left 50% bottom calc(-1 * var(--m) * var(--s)),
                var(--_g)
            )
            calc(50% - 2 * var(--s)) calc(50% - var(--s) / 2 - var(--b) / 2) /
            calc(4 * var(--s)) calc(var(--s) + var(--b)) repeat-x,
            
            radial-gradient(
                var(--r) at left 50% top calc(-1 * var(--m) * var(--s)),
                var(--_g)
            )
            50% calc(50% + var(--s) / 2 + var(--b) / 2) / calc(4 * var(--s))
            calc(var(--s) + var(--b)) repeat-x;
    }

    /* 垂直波浪线 */
    &.wavy--line-v {
        --_g: #0000 calc(99% - var(--b)), var(--bg) calc(101% - var(--b)) 99%, #0000 101%;
        width: calc(2 * var(--r));
        min-height: 50vmin;
      
        mask: 
            radial-gradient(
                var(--r) at top 50% right calc(-1 * var(--m) * var(--s)),
                var(--_g)
            )
            calc(50% - var(--s) / 2 - var(--b) / 2) calc(50% - 2 * var(--s)) /
            calc(var(--s) + var(--b)) calc(4 * var(--s)) repeat-y,
            
            radial-gradient(
                var(--r) at top 50% left calc(-1 * var(--m) * var(--s)),
                var(--_g)
            )
            calc(50% + var(--s) / 2 + var(--b) / 2) 50% /
            calc(var(--s) + var(--b)) calc(4 * var(--s)) repeat-y;
    }
}
```
