![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/109789e391ec427a925edf2b7aa49faa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1260&s=45318&e=jpg&b=42b6be)

  


通常情况之下，Web 开发者在处理 Web 布局中的间距时，首先会想到使用 `margin` 和 （或）`padding` 属性。比如，处理元素与元素之间间距时会使用 `margin` ，处理元素内容与元素边缘之间的间距时会使用 `padding` 。然而，很多时候处理元素之间的间距并不是一件轻易的事情，甚至是很棘手。例如下图所示的布局场景：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4f08a1894b0841c1b337b3c0b6d61ea9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=758&s=9303&e=png&a=1&b=2c3043)

  


就拿上图中的第一个布局来说，直接使用 `margin` 来设置元素之间的间距，就非常棘手。因为，我们并不希望位于最右侧和最底部的元素有 `margin-right` 或 `margin-bottom` 的存在。尤其是内容是动态的情况之下，处理起来更为棘手。

  


庆幸的是，在现代 Web 布局中，除了使用 `margin` 和 `padding` 处理间距之外，CSS 在多列布局、Flexbox 布局和 CSS 布局中还新增了一个 `gap` 属性，它可以更好的帮助我们处理元素之间的间距。接下来，我们就一起来探讨，CSS 的 `gap` 属性是如何处理元素之间的间距，我们应该在什么场景下选择使用 `gap` 属性，要比使用 `margin` 或 `padding` 属性处理间距来得简单和实用。

  


## 使用 `margin` 时的痛点

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/02e0dce89fa44b36bd35a2729b300138~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1080&s=627940&e=png&a=1&b=704ff3)

  


之前提到过，Web 开发者在处理元素之间的间距时，首先想到的是使用 `margin` 。比如，一个卡片组件，其中包含缩略图、标题和描述。在这样的布局中，`margin` 通常是用来控制这些元素之间间距的首选方案：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ec63e508c79d432187894aa011400e5c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1445&s=693591&e=jpg&b=010101)

  


虽然这种方法有效，但如果某个元素未显示，例如 “Read More” 按钮不存在。在这种情况下，描述内容的 `margin` 仍然存在，导致我们正在构建的布局出现不一致的现象：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/212ff16e03164fe781114b6dc4542cf4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2471&h=1445&s=1217127&e=jpg&b=020202)

  


为了解决这个问题，CSS 中有很多种方法，例如，[通过 `:has()` 选择器来判断 `.card` 是否包含链接 `.link` ](https://juejin.cn/book/7223230325122400288/section/7224404685799555111)，只给包含链接 `.link` 中的描述文本设置 `margin` ：

  


```CSS
.card:has(.link) p {
    margin-bottom: 1rem;
}
```

  


这个时候，卡片含有 `.link` （即 “Read More”） 链接时，描述文本 `p` 才会有 `margin-bottom` ，否则 `p` 不会有 `margin-bottom` 的设置。

  


除了内容发生变化要调整 `margin` 之外，在组件布局发生变化时，也需要调整相关的 `margin` 。例如下面这个示例，当卡片由纵向布局变成水平布局时，不得不调整组件中个别元素的 `margin` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8faf330ab5034f65a8e26c996fd5a222~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3280&h=1445&s=1255875&e=jpg&b=020202)

  


我们需要调整图片底部的 `margin` ，才能使卡片底部不会有额外的间距存在。[你可能想到了使用容器查询来解决这个问题](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)：

  


```CSS
.card {
    figure {
        margin-bottom: 1rem;
        
        @container (width >= 320px) {
            margin-bottom: 0;
            margin-right: 1rem;
        }
    }
}
```

  


这种现象在多语言布局中尤其明显。例如下面这个示例，同样的卡片组件，但由于语言不同，其书写方式也不同，使卡片的布局也略有不同，至少方向上相反的。如果使用物理属性 `margin` 的话，那么卡片组件在其他语言环境中的布局就会不符合我们的预期：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c3a58d6dd183455f947eb3b9f791addc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1914&s=1208478&e=jpg&b=020202)

  


熟悉 CSS 的开发者可能会想到[使用逻辑属性来替代物理属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)来规避这个问题的发生：

  


```CSS
.card img {
    margin-inline-end: 1rem;
}
```

  


最后再来看一个多列表的案例。为了使列表项之间有一定的间距，往往我们会给列表项设置一个 `margin-top` 或 `margin-bottom` ，但这种方式也致使第一个或最后一个列表项存在 `margin` 。这种现象，大多数并不是我们所期望的，我们期望的是第一个或最后一个列表项并没有 `margin` 存在：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a224a434d7e489582e5172bd0790828~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1559&s=1136908&e=jpg&b=010101)

  


在这种情景之下，我们可以使用 `:last-child` 对最后一个列表项的 `margin` 进行重置，或者[结合 `:not()` 选择器](https://juejin.cn/book/7223230325122400288/section/7226251495276609569)，给非最后一个列表项设置 `margin` ：

  


```CSS
li:not(:last-child) {
    margin-bottom: 1rem;
}
```

这里所展示的仅仅是 Web 中使用 `margin` 设置元素间距的常见案例。正如你所看到的，用 CSS 的 `margin` 来处理元素之间的间距需要更多的工作。虽然不复杂，但确实需要花费更多的时间和精力。

  


现在，我们有了更好的解决方案，那就是使用 CSS 的 `gap` 来设置元素之间的间距。在正式开始介绍 `gap` 属性之前，我们来看 Web 中一个经典的布局案例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/537662e03df7440287b955cce1d2f26f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1559&s=2353753&e=jpg&b=040404)

  


在 `gap` 属性还没有得到浏览器支持之前，我们要构建类似上图这样的一个布局效果，往往是通过 `margin` 来实现。其中核心是为每个子元素的左侧和顶部添加 `margin` ，然后通过在父容器上添加负 `margin` 来重置这些间距。

  


因此，构建这个布局，我们需要有一个额外的容器来包裹整个列表：

  


```HTML
<div class="wrapper">
    <ul class="figures">
        <li><img src="figure.jpg" alt="" /></li>
        <!-- 省略其他列表项 -->
    </ul>
</div>
```

  


```CSS
.wrapper {
    outline: 2px dashed;
    border-radius: .5em;
    padding: 1rem;
}

.figures {
    --gutter: 2rem;
    display: flex;
    flex-wrap: wrap;
    margin-left: calc(var(--gutter) * -1);
    margin-top: calc(var(--gutter) * -1);
    
    > * {
        flex: 0 0 calc((100% / 4) - var(--gutter));
        margin-top: var(--gutter);
        margin-left: var(--gutter);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/294387309cee4c83abc0130c697e77b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1053&s=963251&e=jpg&b=070707)

  


> Demo 地址：https://codepen.io/airen/full/JjgVQgB

  


现如今，要是使用 CSS 的 `gap` ，整个代码要简单很多：

  


```CSS
.wrapper {
    outline: 2px dashed;
    border-radius: .5em;
    padding: 1rem;
}

.figures {
    --gutter: 2rem;
    display: flex;
    flex-wrap: wrap;
    gap: var(--gutter);
    
    > * {
       flex: 0 0 calc((100%  - var(--gutter) * 3) / 4);
    }
}
```

  


最终实现的效果是相同的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/711cc0d86e0247b9ba151e46f4afc66f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1053&s=963251&e=jpg&b=070707)

  


> Demo 地址：https://codepen.io/airen/full/bGXJXww

  


## CSS 的 `gap` 属性

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b0ef0065a4f4436ca4efb04839176e64~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1350&s=423744&e=png&b=c3d1eb)

  


简单来说，`gap` 与 `margin` 属性有点类似，用于给元素之间设置间距，但不同的是，它只能用于多列布局、Flexbox 布局和 Grid 布局中。而且只能设置列与列之间的间距（`column-gap`）和（或）行与行之间的间距（`row`）。它不像 `margin` 属性那样有具体的方向：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/530d409473894ee899ed5aeba3557076~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1208&h=975&s=16300&e=webp&b=ffc757)

  


另外，下图展示了 `gap` 与 `margin` 最大不同之处：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c16562960b3f4c15864de22123b67d53~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1689&h=1799&s=161151&e=jpg&b=fcfcfc)

  


它的使用非常简单，例如：

  


```CSS
.flexbox {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

.grid {
    display: grid;
    gap: 1rem;
}
```

  


当 `gap` 属性只有一个值时，表示 `column-gap` 和 `row-gap` 的值相等。如果列间距（`column-gap`）和行间距（`row-gap`）不相等时，可以给 `gap` 设置两个值：

  


```CSS
.grid {
    display: grid;
    gap: 1rem 2rem;
}
```

  


上面的代码等同于：

  


```CSS
.grid {
    display: grid;
    row-gap: 1rem;
    column-gap: 2rem;
}
```

  


## 为什么选择使用 CSS 的 `gap`

  


CSS 的 `gap` 属性的基本使用非常简单，这对于任何一位 Web 开发者来说，都不是难事。但是，我猜想，更多的 Web 开发者可能会有这样的一个疑问，为什么要选择使用 CSS 的 `gap` 呢？接下来，我将通过一些常见且实用的案例来告诉大家，选择使用 CSS `gap` 属性的主要原因。

  


### 子元素换行时自动适配

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/374905dad1334b37923615f579c4c507~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3240&h=2440&s=1829116&e=png&a=1&b=fcfbfb)

  


在 Flexbox 或 Grid 布局中，子元素可能会换行。此时，`gap` 可以根据情况动态地应用列间距（`column-gap`）或行间距（`row-gap`）。比如，Grid 布局中的 RAM 布局，就是其中一个典型案例：

  


```CSS
.container {
    --gap: 2rem;
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(min(100% - 2rem, 300px), 1fr));
    gap: var(--gap);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e4d5e647b7024bf68d08bf197cf51a70~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=724&s=12507683&e=gif&f=66&b=ddd6ce)

  


> Demo 地址：https://codepen.io/airen/full/XWvwbpv

  


### 动态内容

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f09f2a4b01374b5ba4b4c074d881781c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2471&h=1445&s=1124976&e=jpg&b=020202)

  


之前我们提到过，由于 Web 的内容是动态的，因此在使用 `margin` 设置元素之间的间距时，不得不根据内容变化来调整元素的 `margin` 。现在，使用 `gap` 设置元素之间的间距，就不需要再花精力去考虑内容的动态化了。

  


```CSS
.card {
    display: grid;
    row-gap: 1rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eb1577a2235e4132bf55b44383db9051~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=999&s=883510&e=jpg&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/eYqaWPO

  


### 组件布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8cad9292f85340b89b63fbb605c031f1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2777&h=1445&s=1014826&e=jpg&b=010101)

  


在 Web 中，即便是同一个组件，它的布局会根据在页面上的位置做一些调整。在这种情况之下，我们虽然根据容器查询可以调整元素的 `margin` ，但使用 `gap` 属性之后，我们可以将更多的精力用于布局的调整上，不需要额外去考虑 `margin` 的设置。

  


```CSS
.card {
    display: grid;
    gap: 1rem;
    
    grid-template-columns: 1rem minmax(0, 1fr) 1rem;
    grid-template-areas:
        "figure figure  figure"
        "...    title   ..."
        "...    des     ..."
        "...    link    ...";
        
    figure {
        grid-area: figure;
    }
    
    h3 {
        grid-area: title;
    }
    
    p {
        grid-area: des;
    }
    
    a {
        grid-area: link;
    }
    
    @container (width >= 600px) {
        grid-template-columns: 220px minmax(0, 1fr) 1rem;
        grid-template-areas: 
            "figure ... ..."
            "figure title ..."
            "figure des ..."
            "figure link ..."
            "figure ... ...";
    }
}

.card__wrapper {
    container-type: inline-size;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5b7f92d9bd842e68a60102fff46d106~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=999&s=939195&e=jpg&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/KKOLmYL

  


正如你所看到的，我们只需要把精力用于处理布局的细节上即可，CSS 的 `gap` 属性会自动适配布局的调整。注意，在这个示例中，我们使用了 CSS 网格区域的功能来调整布局，如果你对这方面不太熟悉的话，请移步阅读小册上一节课《[CSS 布局：网格区域](https://juejin.cn/book/7395119173656903715/section/7436944613077680137)》！

  


事实上，这样的案例在 Web 开发过程中数不胜数，最后再拿多语言布局为例。虽然我们可以使用逻辑属性来避免物理属性 `margin` 带来的麻烦，但要是你对逻辑属性不太熟悉的话，使用 `gap` 属性将是一个很不错的选择：

  


```CSS
.card {
    --cols: 180px minmax(0, 1fr);
    --areas: "avatar content";
    --gap: 2rem;
    
    display: grid;
    gap: var(--gap);
    grid-template-columns: var(--cols);
    grid-template-areas: var(--areas);
    align-items: center;
    
    figure {
        grid-area: avatar;
    }
    
    .card__content {
        grid-area: content;
    }
    
    :lang(ar) {
        --cols: minmax(0, 1fr) 180px;
        --areas: "content avatar";
    }
}
```
