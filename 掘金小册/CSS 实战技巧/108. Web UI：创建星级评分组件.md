## 星级评分组件简介

  


星级评分组件（`StarRating`）是一种常用的 UI 元素，用于让用户以视觉化的方式表达他们对内容的满意度和偏好。它通常以 `1 ~ 5` 颗星的形式呈现。用户可以通过点击或触摸相应的星星数量来进行评分。有时也会支持半颗星，甚至更精细的评分。

  


星级评分组件在各种应用场景中都非常有用，包括电子商务平台的产品评价、社交媒体帖子反馈、服务质量评估等。通过这种方式，用户可以轻松地提供反馈，同时也为其他用户提供参考。

## 自定义范围滑块

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/72be2e760a1243d2ae0576e0a42e8572~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1112&s=222389&e=jpg&b=f3e29c)

  


你可能已经知道，给原生的 `<input type="range" />` 添加样式有点棘手，这主要是因为浏览器自带的默认样式和内部结构的差异。如果你使用浏览器的开发者工具审查一个 `range` 类型的 `input` ，你会发现 Firefox 浏览器和 Chrome（或 Safari 、Edge）浏览器的内部 HTML 结构是不一样的：

  


```HTML
<!-- Chrome 和 Edge -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div>
        <div pseudo="-webkit-slider-runnable-track" id="track">
            <div id="thumb"></div>
        </div>
    </div>
</input>

<!-- Safari -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div pseudo="-webkit-slider-container">
        <div pseudo="-webkit-slider-runnable-track">
            <div pseudo="-webkit-slider-thumb"></div>
        </div>
    </div>
</input>

<!-- Firefox -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div></div> <!-- track -->
    <div></div> <!-- progress -->
    <div></div> <!-- thumb -->
</input>
```

  


庆幸的是，我们仍然可以依赖一些通用的部分来实现样式。在这节课中，我们会针对两个不同的元素添加样式：

  


-   主元素，即 `<input>` 元素自身
-   滑块，即你用鼠标拖动来修改数值的那个小圆点

  


自定义范围滑块的 CSS 大致如下：

  


```CSS
input[type="range"] {
    /* 给主元素添加样式 */
  
    &::-webkit-slider-thumb {
        /* 给 Chrome、Safari 和 Edge 的滑块添加样式 */
    }
    
    &::-moz-range-thumb {
        /* 给 Firefox 的滑块添加样式 */
    }
}
```

  


有一个细节需要知道，`::-webkit-slider-thumb` 和 `::-moz-range-thumb` 块中的 CSS 代码是相同的，但我们不能像下面这样写 CSS：

  


```CSS
input[type="range" i]::-webkit-slider-thumb,
input[type="range"]::-moz-range-thumb {
    /* 给滑块添加样式 */
}

/* 或者 */
input[type="range"] {
    /* 给主元素添加样式 */
    
    &:is(::-webkit-slider-thumb, ::-moz-range-thumb) {
        /* 给滑块添加样式 */
    }
}
```

  


这是无效的，因为这个选择器在语法上对所有浏览器来说都不合法，Chrome 系列不识别 `::-moz-*` ，Firefox 也不识别 `::-webkit-*` 。

  


小册之前的《[CSS 图形：创建自定义范围滑块](https://juejin.cn/book/7395119173656903715/section/7408069117724327976)》课程中详细阐述如何使用 CSS 来美化一个范围滑块，这里就不重复阐述了。我们将省下来的时间直接进入到这节课的主题中。

  


## 创建星级评分组件

  


接下来的目标是，基于下面这个单一的 `input` 元素来创建一个可交互的星级评分组件：

  


```HTML
<input type="range" min="1" value="2" max="5" name="rating" class="star-rating" />
```

  


### 给主元素添加样式

  


我们首先定义尺寸大小：

  


```CSS
.star-rating {
    --s: 100px; /* 控制星星大小 */
    
    appearance: none; /* 移除浏览器默认样式 */
    height: var(--s);
    aspect-ratio: 5;
}
```

  


如果每颗星星的宽高都相等的话，那么对于一个五星评分组件来说，总宽度应该是高度的 `5` 倍，因此我们[使用了 `aspect-ratio` 属性](https://juejin.cn/book/7223230325122400288/section/7259316040151236666)，通过元素的 `height` 和宽高比（`aspect-ratio`）来控制元素的宽度。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0db3953a3728449b9c32d04b2dd45e04~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1332&h=476&s=144702&e=gif&f=98&b=2b2b2b)

  


这个 `5` 正好是 `input` 元素的 `max` 属性的值：

  


```HTML
<input type="range" min="1" value="2" max="5" name="rating" class="star-rating" />
```

  


通常情况下，星级评分组件是一个五星评分组件，如果你的设计比较独特，只需调整 `input` 元素的 `max` 属性值即可。这意味着，`aspect-ratio` 的值可能是一个动态的值，我们可以利用 **[CSS 的 `attr()` 函数的新特性](https://juejin.cn/book/7395119173656903715/section/7487175782831161394)**，直接获取 `input` 元素的 `max` 属性的值，这样可以避免手动调整 `aspect-ratio` 属性的值：

  


```CSS
.star-rating {
    --s: 100px; /* 控制星星大小 */
    
    appearance: none; /* 移除浏览器默认样式 */
    height: var(--s);
    aspect-ratio: attr(max type(<number>), 5);
}
```

  


现在，你只需要修改 `max` 属性，就能控制星星的数量了！这非常方便，因为 `max` 属性本身也被浏览器用来定义 `range` 的最大值，所以我们只需要修改一个地方，就同时影响到了我们的组件样式和浏览器行为。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fe9d492bda4c47049b8ca91b741ce141~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1542&h=364&s=814316&e=gif&f=201&b=2b2b2b)

  


不过，需要注意的是，这种增强版的 `attr()` 特性目前只有 Chrome 浏览器支持，因此接下来的案例效果请 Chrome 浏览器查看。如果你想在实际生产中使用 `attr()` 函数的新特性，就需要考虑为不兼容的浏览器做降级处理。要是你对 `attr()` 函数的新特性相关知识感兴趣的话，请移步阅读《[Web UI：CSS `attr()` 的新特性](https://juejin.cn/book/7395119173656903715/section/7487175782831161394)》！

  


接下来，我们要使用 [CSS 遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)（`mask`）来生成星星图案。因为我们需要重复绘制五颗星星（或者更多，取决于 `input` 的 `max` 属性值），所以遮罩的尺寸（`mask-size`）应该设置为：

  


-   `var(--s) var(--s)`
-   或 `var(--s) 100%`
-   或 `var(--s)` ，因为默认高度是 `100%`

  


```CSS
.star-rating {
    --s: 100px; /* 控制星星大小 */
    
    appearance: none; /* 移除浏览器默认样式 */
    height: var(--s);
    aspect-ratio: attr(max type(<number>), 5);
    
    /* 绘制五角星 */
    mask-image: ?;
    mask-size: var(--s);
}
```

  


那么 `mask-image` 怎么写呢？你可能已经猜到了，我们可以通过 CSS 的渐变来绘制，还可以是 SVG（[将 SVG 转换为 Data URI，然后应用于 CSS 中](https://juejin.cn/book/7341630791099383835/section/7344089098363076620)）。虽然我们要做一个星级评分组件，但我希望星星的部分具有通用性，这样你就可以很方便地替换成任何其他图形。

  


下面代码展示了两种遮罩的实现方式，一种是 CSS 渐为，另一种是 SVG：

  


```CSS
/* SVG 绘制的五角星 */
.star-rating {
    mask-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 100 100'><path d='M50 5 L79 95 L2 40 L98 40 L21 95 Z'/></svg>");
}

/* CSS 渐变绘制的五角星 */
.star-rating {
    mask-image:
        conic-gradient(from 162deg at 50% 65%,#000 .1turn,#0000 0),
        conic-gradient(from 18deg  at 21% 55%,#000 .1turn,#0000 0),
        conic-gradient(from 306deg at 79% 55%,#000 .1turn,#0000 0);
    mask-position: 0 calc(.4 * var(--s));
}
```

  


这两种方式最终所获得的效果都是一致的，相比而言，使用 SVG 要更简单些，代码也更简洁；使用 CSS 渐变创建星星的话，需要对 CSS 渐为技巧有足够深的了解。在这里，我将采用 SVG 的方式来绘制星星：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/01f4c4d189ed47af83a6d9a15cd7f649~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2256&h=986&s=54828&e=png&b=323232)

  


> 温馨提示，你可以根据 《[如何获取 SVG](https://juejin.cn/book/7341630791099383835/section/7343645889896022050)》和《[如何使用 SVG](https://juejin.cn/book/7341630791099383835/section/7344089098363076620)》两节课所介绍的方法来获取 SVG 图形，然后应用于 `mask-image` ！

  


到目前为止，有五颗星星填充到范围滑块中，接下来我们需要给滑块添加样式！

  


### 给滑块添加样式

  


在给滑块添加样式之前，你点击星星，然后注意滑块的位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/41da0e305989434a8f8c27469adcaa57~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=356&s=144256&e=gif&f=140&b=2a2a2a)

  


不难发现，不管滑块当前值是多少（从 `min ~ max`），滑块始终位于对应星星的区域内。但问题是，它在每颗星星中的位置并不一致。在这个示例中，只有当 `value` 的值为 `3` 时，滑块才位于星星的正中间。然而，我们期望的效果是，滑块始终处于星星的中心，而不是靠左或靠右，这样会更美观。

  


下面这张示意图展示了滑块的位置对比，以及我们如何调整它的位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/20c605c934d04449bfba6b507a8c6026~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2364&h=1184&s=476664&e=jpg&b=333333)

  


上图中左侧是滑块的默认效果，滑块从整个组件的最左侧滑到最右侧；右侧是我们期望的效果，通过在组件左右两侧添加空白区域，把滑块的活动范围限制在一个更合理的区域内，这样就可以使滑块手柄与星星的中心对齐。

  


这个空白区域的大小应该是半颗星星的宽度，即 `var(--s) / 2` 。我们可以使用 `padding-inline` 给 `input` 元素添加内边距，并加上 `box-sizing:border-box` 确保盒模型计算正确：

  


```CSS
.star-rating {
    padding-inline: calc(var(--s) / 2);
    box-sizing: border-box; /* 如果你全局设置了元素的 box-sizing ，这里可以忽略 */
}
```

  


这样，滑块的每个位置都能精确对齐到星星的中心位置。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/efce2e8f01fe4c8cb49db9a929c5fba1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=346&s=152119&e=gif&f=126&b=2a2a2a)

  


从上图呈现的效果上来看，滑块位置并没有与每颗星星的中心对齐。是的，因为我们还没有考虑滑块自身宽度，这就导致它无法真正做到居中对齐。

  


不过这不是大问题，因为我们可以将滑块的宽度设置得非常小，比如 `1px` ，这样基本就不会影响视觉上的居中效果：

  


```CSS
.star-rating {
    padding-inline: calc(var(--s) / 2);
    
    &::-webkit-slider-thumb{
        appearance: none;
        width: 1px;
        height: var(--s);
        background: black;
    }
    
    &::-moz-range-thumb {
        appearance: none;
        width: 1px;
        height: var(--s);
        background: black;
    }
}
```

  


这样处理之后，滑块几乎是“隐形”的，只作为一个选中位置的指示器使用：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e3650ef78244677bb2103c913934324~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1334&h=364&s=98467&e=gif&f=113&b=2a2a2a)

  


现在，滑块已经变成了一条黑色的细线，正好位于星星的中心位置。在这里设置为黑色，只是为了让大家能看到效果，实际上我们不需要设置颜色，它最终会是透明的：

  


```CSS
.star-rating {
    padding-inline: calc(var(--s) / 2);
    
    &::-webkit-slider-thumb{
        appearance: none;
        width: 1px;
        height: var(--s);
    }
    
    &::-moz-range-thumb {
        appearance: none;
        width: 1px;
        height: var(--s);
    }
}
```

  


这个时候，你不管点击哪里，在视觉上看不到任何效果，因为滑块是透明的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/93b4f06c1c224f77b805fa4d907df085~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1332&h=362&s=96345&e=gif&f=75&b=2a2a2a)

  


> 特别强调，由于 `::-webkit-slider-thumb` 和 `::-moz-range-thumb` 块中的 CSS 样式是完全一样的，因此在接下来的内容中，我只会在 `::-webkit-slider-thumb` 中编写样式代码，但在Codpen 上的示例会复杂一份相同的 CSS 给到 `::-moz-range-thumb` ！

  


### 给星级评分组件上色

  


现在，我们就只差一步了，而且是非常关键的一步——**使用** **`border-image`** **给星级评分组件上色**！

  


`border-image 是一个非常强大的 CSS 特性`，它有一个非常独特的功能——**[外延（`border-image-outset`）功能](https://juejin.cn/book/7395119173656903715/section/7431465585622810634#heading-4)**！通过 `border-image-outset` ，允许你将边框图片延伸到元素的边框盒子（`border-box`）之外。也正是这个原因，我们才把滑块做得又小且透明。

  


我们可以将一个双色的线性渐变作为边框图像源（`border-image-source`），然后我们可以像下面这样使用 `border-image` ：

  


```CSS
.star-rating {
    /* CSS ... */
    
    &::-webkit-slider-thumb{
        appearance: none;
        width: 1px;
        height: var(--s);
        border-image: linear-gradient(
            90deg, 
            var(--c0, gold) 50%, 
            var(--c1, gray) 0
        ) fill 0 // 0 100px;
    }
    
    &::-moz-range-thumb{
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


简单的解释一下。

  


首先我们使用了 `linear-gradient()` 线性渐变创建了一个双色渐变，这个渐变前一半是 `--c0` 颜色（默认为 `gold`），后一半是 `--c1` 颜色（默认为 `gray`）。实际渲染出来的效果是从左到右，先 `--c0` 颜色，再 `--c1` 颜色，中间有明显的分界线。这个渐变创建的图形将用于 `border-image-source` ，作为图像边框源。

  


接着使用 `border-image-slice` 属性对边框图像进行切割，在这里我们应用的值是 `fill 0` ，其中关键词 `fill` 表示填充中间区域，`0` 表示在距离边框图像四边 `0` 位置进行切割，相当于不切割图像。

  


代码中的 `// 0 100px` 表示 `border-image-width` 使用默认值 `1` ，`//` 之后的 `0 100px` 则是 `border-image-outset` 的值，即在元素四周“向外扩展”的距离，在这里只会向元素的水平方向扩展 `100px`，垂直方向扩展距离为 `0` 。

  


这个时候，你看到的效果像下面这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1dbd127b0ea54c6fb1ca6029dfe166a4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1336&h=366&s=271554&e=gif&f=200&b=2a2a2a)

  


正如你所看到的，滑块左侧是 `--c0` 颜色，表示选中，右侧是`--c` 颜色，表示未选中，而这一切都不需要额外的 HTML 或 JavaScript！

  


由于我们只是使用 `border-image-outset` 向左右扩展了 `100px` （即一个星星的宽度），因此你看到的是，最多 `100px` 的金色（`--c0`）和最多 `100px` 的灰色（`--c1`）。有趣的是，这种颜色填充会跟随滑块移动，因此每次点击一颗星星时，颜色都会滑动到正确位置。

  


到目前为止，这已经非常接近我们所需要的星级评分组件了，最后我们只需要调整 `border-image-outset` 的值，我们设置一个非常大的值，比如 `100vw` ：

现在颜色已经填满了所有的星星，但我们不希望它位于中间，而是应该覆盖整个选中的星星，否则就会出现上图所示的效果，总有半颗星处于未选中的状态。为此，我们需要调整一下渐变，不再使用 `50%` ，而是用 `50% + var(--s / 2)` 。我们添加了一个偏移量（半颗星星的宽度），这意味着第一种颜色将占据更多的空间，最终我们的星级评分组件就完美了！

### 创建半颗星评分组件

首先，我们需要将 `<input type="range"/>` 元素的步进值（`step`）调整为 `0.5` ，而不是 `1` ：

## 使用滚动驱动动效来创建星级评分组件

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ca862a8adb134c75a2ff858dab558912~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=1000&s=206736&e=webp&b=3680c9)

  


为了扩展大家的视野，我们接下来一起探讨另一种创建星级评分组件的方法——通过滚动驱动动效来创建星级评分组件，它可以帮助我们根据选择了多少颗星星来更新星星的颜色，这是使用 `border-image` 无法做到的！

  


> 如果你从未接触过 CSS 滚动驱动动效，那么建议你先移步阅读《[CSS 滚动驱动动效](https://juejin.cn/book/7223230325122400288/section/7259272255786450981)》，帮助你先了解一下 CSS 滚动驱动动效的工作原理！

  


你可能会感到疑惑，我们创建的星级评分组件本身没有什么可以滚动的地方，怎么就与滚动驱动动效扯上关系了呢？CSS 的世界就是这么神奇——它有很多你想不到的事情。既然如此，我们就逐步来解释其中的奥秘！

  


我们先从 CSS 滚动驱动动效的本质开始：

  


> CSS 滚动驱动动效允许我们基于“滚动进度”（即滚动时间轴的进度）来动画化属性值，而不是基于默认的时间轴（即传统的基于时间的动画）。

  


说白了，就是你可以“通过滚动”来控制动画，而不是[依靠 `@keyframes` 中的时间](https://juejin.cn/book/7288940354408022074/section/7295617447058407474)。通过《[CSS 滚动驱动动效](https://juejin.cn/book/7223230325122400288/section/7259272255786450981)》课程的学习，我们已然知道，滚动时间轴其实有两种类型：

  


-   [滚动进度时间轴](https://juejin.cn/book/7223230325122400288/section/7259272255786450981#heading-6)（Scroll Progress Timelines）：你滚动某个容器的 `scrollTop` 或 `scrollLeft` 会驱动动画
-   [视图进度时间轴](https://juejin.cn/book/7223230325122400288/section/7259272255786450981#heading-7)（View Progress Timelines）：你用某个元素进入或离开“滚动视口”的可视范围来驱动动画

  


重点来了，在星级评分组件中，虽然没有页面滚动行为，但滑块在滑动过程中在其容器中是“可视化位置变化”的。于是我们做了一个非常巧妙的事情：把滑块当作主题（Subjet），把 `input` 元素当作滚动容器（Scroller），然后用 `view-timeline` 跟踪滑块在 `input` 区域内的位置变化（相当于“可见进度”， `0% ~ 100%`），从而间接获得滑块当前的值（就像 `input.value` 那样）。

  


这意味着，我们可以追踪滑块的移动位置，计算它的进度百分比，并将其转换一个值，从而用来为 `input` 元素设置样式。本质上，我们正用纯 CSS 实现了类似 JavaScript 中 `document.querySelector("input").value` 的功能！

  


这就是为什么可以使用滚动驱动动效来创建星级评分组件的原理。接下来，我们要做的就是将所有内容转换为 CSS 代码。

  


```CSS
@property --val {
    syntax: "<number>";
    inherits: true;
    initial-value: 0;
}

@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}

.star-rating {
    --s: 100px; 
    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    appearance: none;
    cursor: pointer;
    overflow: hidden;
    height: var(--s);
    aspect-ratio: var(--max);
    mask: var(
        --m,
        url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 100 100'><path d='M50 5 L79 95 L2 40 L98 40 L21 95 Z'/></svg>")
    ) 0 / var(--s);
    background: linear-gradient(
        90deg,
        hsl(calc(30 + 4 * var(--val)) 100% 56%)  calc(var(--val) * 100% / var(--max)),
        #7b7b7b 0
    );
    
    
    timeline-scope: --val;
    animation: --val linear both;
    animation-timeline: --val;
    animation-range: entry 100% exit 0%;
    

    &::-webkit-slider-thumb {
        opacity: 0;
        view-timeline: --val inline;
    }

    &::-moz-range-thumb {
        opacity: 0;
        view-timeline: --val inline;
    }
}
```

是不是第一感觉，代码中有很多不认识的东东，这是因为你对 CSS 滚动驱动动效不太了解。不要紧，接下来逐行分析，你会发现它并没有想象中的那么复杂。

  


### 主题元素（Subject）和滚动容器（Scroller）

  


之前我们提到过，在我们这个示例中，主题元素是滑块，滚动容器是 `input` 元素。我们首先要做的就是定义主题元素。我们使用 `view-timeline` 来定义：

  


```CSS
.star-rating {
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


`view-timeline` 是一个 CSS 简写属性，用于定义一个具名的视图进度时间轴。这个时间轴的进度是基于某个元素在一个可滚动元素中的可见书生 变化来推进的。在我们这个例子中，时间轴的名称是 `--val` ，轴向是 `inline` ，因为我们是在水平方向（`x` 轴）上处理。

  


紧接着，我们要定义滚动容器，也就是 `input` 元素。为此，我们需要给它设置 `overflow: hidden` （或 `overflow: auto`）。这一步虽然简单，但也是最容易被忽略的，而且也是重为重要的，因为它是定义滚动容器的必要条件。

  


```CSS
.star-rating {
    overflow: hidden; /* 非常重要 */
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


### 动画部分

  


接下来，我们来创建一个动画，用来让 `--val` 变量在 `input` 的 `min` 和 `max` 值之间变化。同样地，我们通过 `attr()` 函数来动态获取 `input` 元素的 `min` 和 `max` 属性的值：

  


```CSS
@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}

.star-rating {
    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    overflow: hidden; /* 非常重要 */
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


需要知道的是，默认情况下，`--val` 变量是字符串，为了能够对它进行动画处理，[我们需要使用 `@property` 注册 `--val`](https://juejin.cn/book/7395119173656903715/section/7446278656840138762)：

  


```CSS
@property --val {
    syntax: "<number>";
    inherits: true;
    initial-value: 0;
}

@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}


.star-rating {
    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    overflow: hidden; /* 非常重要 */
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


紧接着，通过 `animation-timeline` 将动画 `--val` 与之前定义的主题元素（滑块）上的视图时间轴（`view-tiemline`）连接起来：

  


```CSS
@property --val {
    syntax: "<number>";
    inherits: true;
    initial-value: 0;
}

@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}

    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    overflow: hidden; /* 非常重要 */
    animation: --val linear both;
    animation-timeline: --val;
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


这里还要注意另一个容易被忽略但非常关键的特性：`timeline-scope`。默认情况下，具名的视图时间轴作用域限定在它被定义的元素及其子孙元素内部。而在我们的例子中，`input` 元素是滑块的父元素，所以默认情况下，它是无法访问在滑块上定义的具名视图时间轴的。为了解决这个问题，我们要使用 `timeline-scope` 来扩大作用域：

  


```CSS
@property --val {
    syntax: "<number>";
    inherits: true;
    initial-value: 0;
}

@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}


.star-rating {
    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    overflow: hidden; /* 非常重要 */
    timeline-scope: --val;
    animation: --val linear both;
    animation-timeline: --val;
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


**一定要记住这一点！** 有时候你明明一切都设置对了，但动画就是不生效，往往是因为你忘记设置 `timeline-scope`。

  


你可能已经注意到了，使用 `@keyframes` 定义 `--val` 动画时，它里的值是反着的。你可能会问，为什么在 `100%` 位置设置最小值 `--min` ，而在 `0%` 位置设置最大值 `--max` 。我们通过一个简单的示例来解释这一点。假设你有一个水平滚动容器，容器里有一个红色的圆。一开始，红色圆是隐藏在容器的右边。当你开始向右滚动时，它会从右边出现，接着继续滚动，它会从左边离开。也就是说，你虽然是从左向右滚动，但实际上红色圆的移动方向是从右向左：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2496b347983e471ab54eb48f7fdea54b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=376&s=241257&e=gif&f=194&b=ffffff)

  


在我们这个例子中，虽然没有真正的滚动行为（滑块不会溢出 `input` 元素），但逻辑是一样的。起点是右侧，终点是左侧。也就是说，动画的开始时滑块处于右侧（也就是 `input` 的最大值），动画结束时滑块位于左侧（也就是 `input` 的最小值）。

  


### 动画范围

  


最后就是通过 `animation-range` 设置动画范围：

  


```CSS
@property --val {
    syntax: "<number>";
    inherits: true;
    initial-value: 0;
}

@keyframes --val {
    0% {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}


.star-rating {
    --min: attr(min type(<number>), 1);
    --max: attr(max type(<number>), 5);
    
    overflow: hidden; /* 非常重要 */
    timeline-scope: --val;
    animation: --val linear both;
    animation-timeline: --val;
    animation-range: entry 100% exit 0%;
    
    &::-webkit-slider-thumb {
        view-timeline: --val inline;
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
}
```

  


默认情况下，动画从主题开始进入滚动区域的右侧时开始，从主题完全退出滚动区域的左侧时结束。这不好，因为滑块不会溢出滚动区域，因此它永远不会到达动画的开始和结束位置：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d0d346c32ff402fb8ce21f3a1431138~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1184&s=161595&e=jpg&b=ffffff)

  


我们可以使用 `animation-range` 来修正它，使动画的开始在主题完全从右侧进入滚动区域时（`entry 100%`），动画的结束在主题开始从左侧退出滚动区域时（`exit 0%`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dacf75eb31f84cc4a808971a5c591265~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1184&s=157745&e=jpg&b=ffffff)

  


简单的小结一下，滑块元素会在 `input` 的区域内移动，而这个移动过程会被用来控制一个动画的进度，这个动画会让一个变量在 `input` 的 `min` 和 `max` 属性值之间变化。这意味着，使用纯 CSS 实现了类似 `document.querySelector("input").value` 的功能！

  


温馨提示一下，我们在示例的代码中多处用到了 `--val` ，但它们不是相同的东西。简单的解释一下：

  


-   **一个用** **`@property`** **注册的 CSS 变量**： 它是被动画驱动变化的值，代表当前选择的值，用来给 `input` 添加样式
-   **一个具名的视图时间轴（View Timeline）** ：通过 `view-timeline` 定义，之后用 `animation-timeline` 引用它
-   **一个关键帧动画的名字**：名叫 `--val` 的动画，通过 `animation` 属性调用

  


如果这样命名容易造成混淆的话，你也可以用不同的名字来改写 CSS 代码：

  


```CSS
/* 注册 CSS 变量 */
@property --val {
  syntax: "<number>";
  inherits: true;
  initial-value: 0; 
}

/* 定义一个动画名称 */
@keyframes update_val {
    0% { 
        --val: var(--max) 
    }
    100% { 
        --val: var(--min) 
    }
}

.star-rating {
    --min: attr(min type(<number>));
    --max: attr(max type(<number>));
    overflow: hidden;
    
    timeline-scope: --timeline;
    animation: update_val linear both;
    animation-timeline: --timeline;
    animation-range: entry 100% exit 0%;
    
    
    &::-webkit-slider-thumb {
        view-timeline: --timeline inline; /* 视图时间轴的名字 */
    }
    
    &::-moz-range-thumb {
        /* 与 ::-webkit-slider-thumb 代码相同 */
    }
  
}
```

  


上面代码中：

  


-   `--val` 是我们要控制的变量
-   `--timeline` 是视图时间轴的名字
-   `val_update` 是关键帧动画的名字

  


这样一来，每个名字的用途就一目了然了！
