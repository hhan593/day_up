![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/415f99f901ce40038728e0644cd6ba49~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=258554&e=webp&b=c9c5b7)

  


表单（`<form>`）是用户与 Web 网站或应用交流的主要方式之一，通常情况下，我们会使用 `<input>` 、`<textarea>` 和 `<select>` 等表单字段来收集用户输入的信息。然而，它们通常有一个共同的缺点：**固定大小与内容不匹配：**

  


-   当表单字段的内容超过其固定大小时，溢出的内容可能使得用户很难与字段交互
-   当表单字段的内容小于固定大小时，字段可能会使得界面显得不平衡

  


换句话说，适当的表单文本区域大小设置对提升用户体验至关重要。通过合理的大小设置，用户可以舒适地输入文本，界面也会显得更加简洁易读。

  


然而，在没有 `field-sizing` 属性之前，要创建一个尺寸合适的表单字段，你只能猜测表单字段的平均大小，或者使用 JavaScript 来计算字符数量，并根据用户输入的信息动态增加元素的宽度或高度。这并不容易，而且在尝试跟随用户输入内容调整大小时容易出错。

  


现在，CSS 提供了一种纯粹的解决方案来自动调整表单字段大小。新的 CSS `field-sizing` 属性，只需要一行 CSS 就能根据表单字段内容动态调整大小，从而消除了对 JavaScript 的需求。这节课我们就来探讨一下 `field-sizing` 属性，以及如何利用它来优化表单字段的大小，从而提升用户使用表单的体验。

  


## 表单字段的局限性：固定大小

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f8061f755c2b480d967245f7d03065be~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=104252&e=webp&b=ece4cc)

  


之前提到过，`<input>` 、`<textarea>` 和 `<select>` 有一个共同的缺陷：固定大小与内容不匹配。当表单字段的内容超过其固定大小时，溢出的内容可能使用户很难与字段交互；相反，当内容明显小于固定大小时，字段可能会使界面显得不平衡。

  


设置表单字段的大小通常需要找到一个平衡点，以避免它太小或太大。当处理一个具有静态选项列表的 `<select>` 字段时，这个解决方案通常能很好地工作。但对于内容更动态的 `<input>` 和 `<textarea>` 字段，这个方案可能会很快失效。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b05ae4a026a644f8adc0a4d72a1ad4b7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1138&h=658&s=1237770&e=gif&f=253&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/GgKyRyd

  


一些元素（如 `<textarea>`）提供通过拖动字段角落的手柄来手动调整大小的功能。要启用此功能，可经将 CSS 的 `resize` 属性设置为：

  


-   `vertical`：只允许用户在垂直方向调整大小，保持宽度不变，只调整高度
-   `horizontal`：只允许用户在水平方向调整大小，保持高度不变，只调整宽度
-   `both;`：允许用户在水平方向和垂直方向上调整大小，完全自由地调整大小

  


通过适当配置这些选项，可以提供更加符合用户需求的文本区域。然而，这个功能在大多数移动浏览器上不起作用。

  


为了应对这些问题，开发人员通常使用 JavaScript 解决方案，根据内容动态调整表单字段的大小，比如 [React 的 `<TextareaAutosize>` 组件](https://github.com/Andarist/react-textarea-autosize)，提供了一个 `textarea` 元素的替代品，能够根据用户输入自动调整大小：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9eb8cfa494584d03897a4b92cd3bd6b9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=720&s=1404244&e=gif&f=188&b=0d181b)

  


虽然这种方法有效，但它会增加代码库的复杂性。如果有一个纯 CSS 解决方案岂不是更好？

  


通过使用 `field-sizing` 属性，只需要一行 CSS 就能根据内容启用自动调整尺寸的功能。摒弃传统的固定大小设置，使得表单字段根据内容的多少自然调整大小。例如，在反馈表单中，如果预计用户输入较长内容，使用此方法可以提供一个既自由又简洁的输入环境，避免浪费空间，并保持紧凑的设计。

  


```CSS
input, textarea, select {
    field-sizing: content;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1a6b9a8979114a66a06655f00b769254~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1428&h=706&s=1625945&e=gif&f=368&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/VYZyvKb

  


## `field-sizing` 简介

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/345a2e0cde294e3cb4ab085cadff82cd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1020&h=743&s=15068&e=webp&b=021528)

  


`field-sizing` 是一个 CSS 属性，允许你控制具有默认首选大小的元素（比如 `<input>` 、`<textarea>` 和 `<select>` 等）的尺寸行为。通过该属性，可以覆盖默认的尺寸行为，使表单控件根据其内容动态调整大小。通常用于样式化文本输入框（比如 `<input>` 和 `<textarea>`），允许它们根据内容自动缩小或扩大，以适应用户输入的文本。

  


`field-sizing` 的语法很简单，可以取两个值：

  


```CSS
.controler {
    field-sizing: fixed | content;
}
```

  


-   **`fixed`**：默认值。表单字段保持固定大小，不会根据其内容自动调整尺寸。如果内容超出固定空间，字段会显示滚动条以提示存在更多内容
-   **`content`**：表单字段会根据输入的内容自动增大或缩小，它会随着文本的变化调整表单控件的固有尺寸，变得宽度正好适合内容。例如，即使 `<form>` 元素的最大宽度被设置为 `500px`，也无法阻止表单控件继续增大。控件可能会导致整个表单宽度变大，或者控件会溢出可用空间

  


我们以 `<textarea>` 为例，请尝试在下面的 `<textarea>` 输入更多的文本，查看它们之间的差异（`field-sizing` 属性值 `fixed` 与 `content` ）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de1b5c94483d4967bb9a51b6ad9f42bd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1158&h=698&s=1049822&e=gif&f=164&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/JoPMYpE

  


## 哪些表单控件会受到 `field-sizing` 的影响

  


受 `field-sizing` 影响的表单控件主要有：

  


-   多行输入框，即文本域 `<textarea>`
-   下拉选择框 `<select>` 和 `<select multiple>`
-   文本输入类型表单控件：适用于接受用户直接输入文本的表单 `<input>` 元素，包括 `email`、`number`、`password`、`search`、`tel`、`text` 和 `url`
-   文件上传控件，`type` 为 `file` 的 `<input>` 元素

  


接下来，我们通过简单的案例来展示上述表单控件的 `field-sizing: content` 会有哪些影响。

  


### `<textarea>` 控件

  


我们使用一个容器来包裹 `<textarea>` 元素：

  


```HTML
<div class="input-controls">
    <textarea spellcheck="false" name="textarea" class="textarea" id="textarea" placeholder="Type your message..."></textarea>
</div>
```

  


并使用 CSS Grid 对它进行布局：

  


```CSS
.input-controls {
    width: 40ch;
    padding: 1rem;
    min-height: 140px;
    display: grid;
    
    .textarea {
        field-sizing: var(--field-sizing);
        resize: none;
        width: 100%;
    }
}
```

  


当你将 `field-sizing` 属性的值从 `fixed` 切换到 `content` 时，它不会有明显的变化：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/58991472dde841d69208feeca7e9455c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=412&s=180472&e=gif&f=114&b=010101)

  


默认情况下，Grid 项目在纵轴会被拉伸（`align-content` 、`align-items` 和 `align-self` 的值为 `stretch`）。在我们这个示例中，`textarea` 与其父容器高度相等。这就是 `textarea` 的 `field-sizing` 切换到 `content` 不会有任何变化的主要原因。一旦你改变它的布局模式，比如：

  


```CSS
.textarea {
    width: auto;
    place-self: start;
}
```

  


设置 `field-sizing: content` 的 `textarea` 会缩小到 `min-inline-size` （或 `min-width`）或刚好适合占位符（`placeholder`）的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9c708b3f080e449fbfddc840eb6b5dc5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1140&h=402&s=163546&e=gif&f=119&b=000000)

  


当用户输入文本时，`textarea` 会沿内联方向（宽度方向）增长，直到达到最大内联尺寸（`max-inline-size` 或 `max-width`），此时文本会换行，并且 `textarea` 的块级尺寸 `block-size`（高度 `height`）会随新行增加而增长：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d6a20cac5b164cd6981556db8f4890cf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1162&h=642&s=746729&e=gif&f=203&b=000000)

  


正如你所看到的，如果宽度受限无法增长，则会根据内容高度增加显示行数。但当高度也受限时，则会显示滚动条以查看全部内容：

  


```CSS
.textarea {
    max-block-size: 200px; 
    
    /* 或者 */
    max-height: 200px;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/01c55aa062b8425ebf972508c73dd02e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=428&s=1347928&e=gif&f=168&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/WbedrRv

  


对于 `<textarea>` 元素而言，它的 `rows` 和 `cols` 属性会影响默认首选尺寸，但如果设置 `field-sizing: content` ，它们将失效：

  


```HTML
<div class="input-controls">
    <textarea cols="30" rows="10" spellcheck="false" name="textarea" class="textarea" id="textarea" placeholder="Type your message..."></textarea>
</div>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/41a1fe268be840eca4e54460c9c08a5c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1432&h=792&s=2065353&e=gif&f=307&b=020202)

  


> Demo 地址：https://codepen.io/airen/full/pvzpEvj

  


### `<select>` 控件

  


设置 `field-sizing: content` 的 `<select>` 控件会因不同的类型有不同的表现行为：

  


-   普通的 `<select>` 会根据当前选项值的长度调整宽度。默认情况下，控件宽度会足够容纳最长的选项值。简单地说，`<select>` 会缩小以适应所选选项的大小
-   带有 `multiple` 属性的 `<select>` 会根据最宽的选增长宽度，并根据选项数量调整高度以适应所有选项

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e6c56b8e004d4e869205e42ee867f068~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1222&h=596&s=1621643&e=gif&f=393&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/jENYMLq

  


### 文本输入类型表单控件

  


文本输入类型表单控件指的是 `<input>` 元素的 `type` 为 `text` 、`password` 、`number` 、`email` 、`search` 、`tel` 和 `url` 。它们受到 `field-sizing: content` 的影响，表现如下：

  


-   输入框会缩小到 `min-inline-size` （或 `min-width`）或刚好适合占位符（`placeholder`）；如果未设置最小宽度和 `placeholder` ，其宽度只与文本光标一样宽；带有 `placeholder` 属性的控件会根据占位符文本的长度调整大小以适应显示
-   当用户输入文本时，输入框会沿内联方向增长，直到达到最大内联尺寸（`max-inline-size` 或 `max-width`），此时输入的值被裁剪溢出部分

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/931249f6553b47dc85de733288ebf361~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1164&h=718&s=1531447&e=gif&f=564&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/GgKyOWy

  


有时候你在使用 `<input>` 元素时，可能会给 `input` 元素设置一个 `maxlength` 属性，用于限制控件（`<input>`）的最大字符数。

  


```HTML
<input type="text" maxlength="100" />
```

  


这个时候，即使 `input` 设置了 `field-sizing: content` ，元素的 `maxlength` 也会加以限制。换句话说，当你输入的字符数达到这个限值（`maxlength` 属性的值）时，控件会停止调整大小：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e303b00fedd4f6eb75f060ea5667ac0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1034&h=712&s=603048&e=gif&f=354&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/ZYzvamG

  


### 文件上传控件（`type="file"`）

  


虽然用户无法直接通过文件上传控件（`<input type="file" />`）输入文本，但当用户选择新文件时，会显示文件名称。设置 `field-sizing: content` 的文件上传控件，输入框会缩小到按钮和预填充文件名文本的尺寸。当用户上传文件后，输入框的宽度会变成按钮宽度加上文件名文本的宽度：

## **`field-sizing`** **与其他尺寸属性的交互**

  


表单控件（`<input>` 、`<select>` 和 `<textarea>` 等）自身也有一些属性可以用来控制大小的，比如：

  


-   `<input>` 可以使用 `size` 属性来定义特定的大小
-   `<select>` 的 `size` 属性会将其从一个普通的下拉框转换为列表下拉框（`<select multiple>`）
-   `<input>` 的 `maxlength` 属性可以限制输入的文本字符数
-   `<textarea>` 的 `cols` （列）和 `rows` （行）属性可以设置文本域的大小

  


除此之外，CSS 提供了多种设置尺寸的方法，例如：

  


-   通过 `width` （或 `inline-size`）和 `height` （或 `block-size`）来设置表单控件的宽高
-   通过 `min-width` （或 `min-inline-size`）和 `min-height` （或 `min-block-size`）来控制表单控件的最小尺寸
-   通过 `max-width` （或 `max-inline-size`）和 `max-height` （或 `max-block-size`）来限制表单控件的最大尺寸
-   通过 [CSS 的比较函数 `min()` 、`max()` 或 `clamp()` ](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)来限制表单控件的尺寸
-   通过[内在尺寸 `fit-content` 、`min-content` 或 `max-content` ](https://juejin.cn/book/7161370789680250917/section/7161625601713897508)使表单控件适配其内容大小
-   通过 CSS 的布局，比如 [CSS Flexbox](https://juejin.cn/book/7161370789680250917/section/7161621092560273439) 或 [CSS Grid](https://juejin.cn/book/7161370789680250917/section/7161372229123440648) 来调整表单控件的尺寸

  


接下来，我们来看看 `field-sizing` 是如何与这些属性或 CSS 样式进行交互的。

  


### `size` 属性和 `field-sizing`

  


首先要明确的一点是，`size` 属性令在某些类型的 `<input>` 元素上有效。这意味着像数字输入框（`type="number"`）就不支持 `size` 属性了。我们这里以 `type="text"` 为例：

  


```HTML
<input type="text" placeholder="这是一个文本输入框" size="50" />
```

  


当你将 `input` 的 `field-sizing` 设置为 `content` 时，`size` 属性将会失效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbf5f15f23684eb7ac3b5c02b22517bc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1198&h=390&s=501102&e=gif&f=168&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/gbYoeYx

  


然而，`input` 元素的 `maxlength` 属性则有所不同。当用户输入的字符数达到这个限值时，即便 `input` 显式设置了 `field-sizing: content` ，也无法调大输入框的大小：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2960b0536eca4bab9d9c917cc4716f38~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1192&h=440&s=828358&e=gif&f=280&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/GgKyxgE

  


`<select>` 表单控件也可以设置 `size` 属性，它会将其从一个普通的下拉框转换为列表下拉框。它看起来和 `<select multiple>` 一样，但仍然只允许一次选择一个 `<option>` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/82462a712d0649908c8cb55417b88cf7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1036&h=254&s=408574&e=gif&f=154&b=010101)

  


同样的，设置 `field-sizing: content` 会使 `<select>` 的 `size` 失效，但设置 `size` 属性的下拉框依旧会以表列下拉框（`<select multiple>`）的方式呈现，它的高度是所有列表项的高度总和，宽度则会与内容长度最长的选项（`<option>`）的宽度相等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9c04ff395e7b4fc0a58d60dbf563998f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1180&h=510&s=905345&e=gif&f=232&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/JoPMLYY

  


相反，当你给 `<select multiple>` （列表下拉框）设置 `size="1"` 时，它会以普通的下拉框显示，但可以同时选择多个 `<option>` 。同样的，`field-sizing:content` 会使列表下拉框显示所有列表项，并且宽度以内容最长的列表项（`<option>`）为基准。

  


对于 `<textarea>` 元素也是一样的，它的大小可以通过 `cols` 和 `rows` 来定义：

  


```HTML
<textarea cols="30" rows="10" spellcheck="false" name="textarea" class="textarea" placeholder="Type your message..."></textarea>
```

  


只不过，`field-sizing` 会使 `textarea` 的 `cols` 和 `rows` 失效：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e7a4c38cd3424076a457aaecac77450c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1228&h=514&s=228272&e=gif&f=158&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/pvzpEvj

  


### **宽度和** **`field-sizing`**

  


**首先要声明一点，虽然这里聊的是宽度和** **`field-sizing`** **的交互，但是它也同样适用于高度**。

  


众所周知，CSS 中有很多设置尺寸的方法，比如使用 `width` 属性：

  


```CSS
input[type="text"] {
    width: 250px;
}
```

  


还有 `min-width` 和 `max-width` 属性：

  


```CSS
input[type="text"] {
    min-width: 250px;
    max-width: 500px;
}
```

  


我们可以使用 `min()` 或 `max()` 函数使其更智能：

  


```CSS
input[type="text"] {
    width: min(250px, 500px);
}
```

  


或者，我们可以使用 `clamp()` 函数将宽度限制在两个尺寸之间，并结合理想大小：

  


```CSS
input[type="text"] {
    width: clamp(250px, 50%, 500px);
}
```

  


那么问题来了，当我们将 `width` 和 `field-sizing` 结合使用时，又会怎么样呢？我们以 `<input type="text">` 为例：

  


```CSS
input[type="text"] {
    width: 320px;
    field-sizing: content;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e20c0a20ea1f43338569e1bc8d8dce1e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=266&s=659888&e=gif&f=360&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/PwYEREe

  


不难发现，设置了固定宽度的输入框不会受到 `field-sizing: content` 的影响。

  


我们再来看第二种情景，输入框未显式设置 `width` ，但显式设置了 `min-width` 和 `max-width` 值：

  


```CSS
input[type="text"] {
    min-width: 100px;
    max-width: 300px;
    field-sizing: content;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c3760d10c3d541f3a130b5b7f8b122b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1034&h=308&s=690470&e=gif&f=340&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/raBpdrO

  


你可能已经发现了，输入框达到该限制时会停止调整尺寸，比如上面这个示例中，输入框的宽度最小不会小于 `100px` ，最大不会大于 `300px` 。这个同样适用于 `min()` 、`max()` 或 `clamp()` 函数。

  


有一个非常奇怪的现象，我们不给 `input` 设置固定宽度，而是尝试着使用百分比，比如 `50%` ：

  


```HTML
<div class="input-controls">
    <input type="text" placeholder="这是一个文本输入框，未显式设置 width = auto" />
</div>
```

  


```CSS
.input-controls {
    width: 40ch;
}

input[type="text"] {
    width: 50%;
    field-sizing: var(--field-sizing);
}
```

  


默认情况下，没有什么不一样。但当你往文本框输入更多的内容时，设置为 `width: 50%` 的样式似乎失效了，输入框的固有大小变得比 `.input-controls` 的固定大小更大，导致表单控件会溢出其父容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bae6b0836c3d4d0c9496e6a1df85f48a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1292&h=240&s=795368&e=gif&f=397&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/pvzpLQz

  


因此，当使用 `field-sizing: content` 来调整表单元素的大小时，这一点需要特别注意。

  


### 布局模式和 field-sizing

  


正如之前提到的，CSS 的布局模式也会影响到表单控件的尺寸大小。以 CSS Grid 布局为例：

  


```HTML
<div class="input-controls">
    <input type="text" placeholder="Without place-self: start" />
</div>
```

  


```CSS
.input-controls {
    width: 40ch;
    display: grid;
    
    input[type="text"] {
        field-sizing: content;
        
        justify-self: var(--align, stretch);
    }
}
```

  


默认情况下，当 `input` 的父容器是一个网格容器的时候，`input` 元素会拉伸填满整个容器的宽度，因为 `input` 的 `justify-self` 默认为 `stretch` 。如果将它重置为非 `stretch` 的值（比如 `start` 、`center` 或 `end`），`input` 的宽度会与用户输入的内容（或 `placeholder` 占位符文本长度，如果有的话）相等。

  


在这种情况之下，即使 `input` 显式设置了 `field-sizing: content` ，其默认宽度会与布局模式渲染的结果相似，比如 `justify-self` 为 `stretch` 时，它的宽度是网格容器的可用空间；如果 `justify-self` 为 `start` （或 `center` 或 `end`），它的宽度与 `placeholder` 占位符文本长度相等。但是，`input` 会随着用户输入的文本内容增加而变宽，甚至会溢出网格容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5107048e066a47d1bbc3656001caf16e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1206&h=334&s=663606&e=gif&f=220&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/KwPZexR

  


因此，在这一点上，尤其需要注意。

  


温馨提示，对方方式只是影响 Flex 项目和 Grid 项目尺寸的因素之一，如果你想更深入的了解其背后的工作原理，个人建议你深入了解一下 Flexbox 和 Grid 布局中的对齐方式，可以花点时间阅读下面两节课的内容：

  


-   [Flexbox 布局中的对齐方式](https://juejin.cn/book/7161370789680250917/section/7161623670622781471)
-   [Grid 布局中的对齐方式](https://juejin.cn/book/7161370789680250917/section/7161624114308841509)

  


有关于 **[CSS Flexbox 布局](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)** 和 **[CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)** 技术更全面，更系统的介绍，可以移步阅读《[现代 Web 布局](https://s.juejin.cn/ds/iy9nUcFV/)》！

  


简单的小结一下：

  


-   使用 `field-sizing: content` 提供的尺寸灵活性可能会被其他 CSS 尺寸属性覆盖
-   避免在设置 `field-sizing: content` 时同时使用固定宽度和高度的属性（如 `width` 和 `height`），因为这些属性会重新强制控件为固定大小
-   推荐使用 `min-width` 和 `max-width` 等属性与 `field-sizing: content` 结合使用，既允许控件随内容调整大小，也限制其不会变得过大或过小

  


## 以防万一：空状态与溢出样式处理

  


使用 `field-sizing` 属性意味着你需要额外做一些工作。因为在应用 `field-sizing: content` 之前，表单控件通常具有固定尺寸，而应用之后，输入框可能会变得过小或过大。以 `<input type="text">` 为例：

  


```HTML
<input type="text" />
<input type="text" value="我是一个单行文本输入框" />
<input type="text" placeholder="我也是一个单行文本输入框" />
```

  


上面三个 `input` 元素，其中第一个 `input` 元素没有设置 `value` 和 `placeholder` ，它的内容是空的；第二个 `input` 默认会在文本框中显示 `value` 的值；第三个 `input` 默认会显示 `placeholder` 的值。假设这三个 `input` 元素没有设置任何与尺寸相关的样式，并且它们的尺寸也未受到任何布局模式的影响。当你将 `field-sizing` 属性的值从 `fixed` 切换到 `content` 时，它们呈现的结果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8f528db1380a40f5a1f0baf8b1ede49d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=366&s=959987&e=gif&f=198&b=010101)

  


> Demo 地址：https://codepen.io/airen/full/EaYoBma

  


很明显，没有设置 `value` 和 `placeholder` 的 `input` 元素，当 `field-sizing` 属性的值为 `content` 时，它的宽度只有一个光标的宽度（注意，上面示例中设置了 `padding`），而且所有输入框会因用户输入的内容变得越来越大。这意味着，设置 `field-sizing:content` 的表单控件在视觉上会进入不理想的状态。

  


然而，我们在编写 CSS 代码的时候，需要具备以防万一思想，其目标并不是明确规定某个元素的具体外观或行为，而是保护它不进入不理想的视觉状态。我们把这种 CSS 称为“**[防御式 CSS](https://s.juejin.cn/ds/iy9vx3aH/)**”。例如，为了防止表单控件缩得过小（变成一个微小的盒子，比如没有 `value` 和 `placeholder` 的 `<input>` 元素），同时对于 `<textarea>` 来说，也能防止其增长得过大。那么我们可以像下面这样来编写具有一定防御性的 CSS ：

  


```CSS
:is(input, textarea, select) {
    field-sizing: content;
}

input {
    min-inline-size: 7ch;
    max-block-size: 40ch;
}

select {
    min-inline-size: 5ch;
    min-block-size: 1.5lh;
}

textarea {
    min-block-size: 3.5lh;
    min-inline-size: 20ch;
    max-inline-size: 50ch;
}
```

  


当然，你可以根据自己的需要来调整相关的值。

  


需要知道的是，在 CSS 中除了使用 `min-width` 、`min-height` 、`max-width` 和 `max-height` 以及它们对应的逻辑属性 `min-inline-size` 、`min-block-size` 、`max-inline-size` 和 `max-block-size` 之外，也可以使用 CSS 的比较函数 `min()` 、`max()` 和 `clamp()` 来限制元素的大小，这样可以使得设置 `field-sizing: content` 的表单控件有一个更理想的视觉效果。

  


我们来看一个简单的示例：

  


```CSS
.textarea {
    resize: vertical;
    field-sizing: content;
    min-height: 3lh;
    max-height: 10lh;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/729ea6625e6d458da01e1d9cccd81141~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=626&s=2605321&e=gif&f=426&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/ZYzvdJd

  


上面示例中，使用了 `lh` 单位，可以根据行高来设置文本区域的高度。通过这种方式，我们可以在不同设备和屏幕大小下，根据行数调整文本区域的高度，确保内容美观地布局。例如，使用 `min-height: 3lh` 可以确保文本区域至少有三行的高度，从而提供一个视觉上舒适的输入区域。这不仅使得输入内容自然显示，还提高了可读性和视觉效果。

  


另外，我们通过组合使用 `resize` 、`field-sizing` 和 `lh` 单位，可以实现以下效果：

  


-   **自动调整大小**：文本区域会根据用户输入的内容自动调整大小，从而避免不必要的滚动，确保输入内容始终适应屏幕
-   **视觉舒适性**：使用 `lh` 单位可以确保行间距合适，输入的文本清晰易读，即使是长文本也能呈现美观的布局
-   **限制设置**：通过同时使用 `max-height` 和 `min-height` 属性，可以设置文本区域的最大和最小高度，防止出现过大或过小的输入区域，从而避免布局失衡

  


通过优化文本区域的大小调整，用户可以更轻松地输入信息，无需频繁滚动，且能够一眼看到输入内容。这种设计可以让表单更加舒适，用户体验得到极大提升。此外，提供清晰易读的表单可以减少用户的干扰，创造出直观且易用的输入环境。

  


### 总结

  


通过利用 CSS 的 `field-sizing` 属性和 `lh` 单位，我们可以优化表单控件的大小，使其既易于使用，又美观大方。这样，我们能够设计出更加注重用户体验的表单，去除多余的空间和复杂性，提供高效的 UI 设计。通过应用这些属性，优化表单布局，提升用户满意度是完全可行的。
