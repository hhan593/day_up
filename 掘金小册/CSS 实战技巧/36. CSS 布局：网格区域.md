## 我们从一个三列布局开始

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a181e73b1124479f9434f2478f91b541~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=871&s=250641&e=jpg&b=e12f2f)

  


这是一个非常简单的三列布局。使用 CSS Grid 可以很容易构建这个布局：

  


```CSS
.grid {

    gap: 1rem;
    grid-template-columns: 1fr 3fr 1fr;
}
```

```CSS
.grid__item {
```

## 使用网格区域构建 Web 布局

  


在使用 CSS Grid 构建布局时，可以使用 `grid-template-areas` 为每个网格区域命名，并通过 `grid-area` 引用已命名的网格区域名称，就可以将网格项目放置到相应的网格区域中。

  


继续以之前的三列布局为例，依旧在网格容器上使用 `grid-template-columns` 创建显式网格列。不同的是，我们还使用 `grid-template-areas` 为每个项目定义了一个区域：

  


```CSS
.grid {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    grid-template-areas: "item1 item2 item3";
    gap: 1rem;
}
```

  


然后，在网格项目上使用 `grid-area` 将网格区域名称映射到每个网格项目上：

  


```CSS
.grid {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    grid-template-areas: "item1 item2 item3";
    gap: 1rem;
    
    :nth-child(1) {
        grid-area: item1;
    }
    
    :nth-child(2) {
        grid-area: item2;
    }
    
    :nth-child(3) {
        grid-area: item3;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6be55485945f49d0aff4ae64d8227806~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1596&s=765966&e=jpg&b=020202)

  


> Demo 地址：https://codepen.io/airen/full/vYovmyV

  


这个时候，就不再需要去想具体的网格线索引号，只需要将对应的网格区域名称应用到相应的网格项目上，就可以将网格项目放置到指定位置。

  


我们再来看另一个示例，同样使用 `grid-template-columns` 定义了一个三列网格，但 `grid-template-areas` 命名网格区域名称时与上一个示例略有不同：

  


```CSS
.grid {
    display: grid;
    grid-template-columns: repeat(3, minmax(0, 1fr)); /* 等同于 1fr 1fr 1fr*/
    grid-template-areas: "featured featured article";
    gap: 1rem;
}
```

  


现在，我将同一个卡片组件分别放置在 `featured` 和 `article` 网格区域中，你将看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aba5c39a44264dd7a2fef52c1d242d59~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1596&s=928115&e=jpg&b=020202)

  


> Demo 地址：https://codepen.io/airen/full/wvVRdXB

  


注意到卡片组件的位置和大小的变化了吗？我们只是将卡片放置在不同的网格区域，就能轻易调整它的位置和大小，并且不再需要关注网格线索引号。相比之下，使用网格区域来构建布局要简单和灵活地多。

  


## 定义网格区域的规则

  


通过之前的示例，我们知道了如何通过网格区域的特性来构建 Web 布局。不过，在定义网格区域的时候，有一些规则需要遵守，否则网格区域无法为你工作。接下来，我将详细解释网格区域相关的语法规则，并引导你了解不同的示例和使用场景。

  


### 定义的网格区域应该是一个矩形

  


在代码中，定义的网格区域必须是一个矩形形状。比如下面这个 CSS 代码，它向大家展示了有效和无效的两种定义网格区域的规则：

  


```CSS
/* 无效的规则 */
.grid {
    grid-template-areas:
        "header    header"
        "sidebar   main"
        "header    footer";
}

/* 有效规则 */
.grid {
    grid-template-areas:
        "header   header"
        "sidebar  main"
        "sidebar  footer";
}
```

  


第一个规则，所有 `header` 连在一起并不是一个矩形形状，因此它是一个无效的规则。但第二个规则，每个区域所在区域连起来都是一个独立的矩形。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b5d3a34d34ff44ed8950fccf345b224d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1407&s=780672&e=jpg&b=161719)

  


### **所有区域必须被定义**

  


使用 `grid-template-areas` 时，即使不是所有区域都需要使用，也必须定义所有区域。不需要使用的区域可以使用点 `.` 来替代，但不能是空字符或不定义：

  


```CSS
/* 无效的规则 */
.grid {
    grid-template-areas:
        "header   header"
        "sidebar  main"
        "footer";
}

/* 有效规则 */
.grid {
    grid-template-areas:
        "header   header"
        "sidebar  main"
        "sidebar  footer";
}

/* 有效规则 */
.grid {
    grid-template-areas:
        "header   header"
        "sidebar  main"
        "...      footer";
}
```

  


请参考下图：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9b0de7cb9b8c49aa8eab9383a1a700af~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1989&s=1205216&e=jpg&b=161719)

  


注意，如果某个区域不想命名，可以使用任意数量的点来描述，比如 `.` 、`..` 和 `...` 是等效的，我个人偏好使用三个点来描述不需要命名的网格区域。

  


## 定义网格区域的语法

  


通常情况之下，在网格容器上使用 `grid-template-areas` 属性来定义网格区域。定义网格区域的时候主要要了解以下几点：

  


-   它可以接受多个区域字符串
-   一个字符串表示一维布局（仅包含列）
-   多个区域字符串表示多维布局（包含行和列）

  


网格区域有一个显著特点是，它在 CSS 中为我们提供了网格的可视化效果。

  


例如，我们有一个包含两列的一维网格，第一列为固定宽度，另一列根据可用空间自适应：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
    grid-template-areas: "aside  main";
}
```

  


一旦定义了网格区域，我们就可以在网格项目上使用 `grid-area` 属性来引用已定义的网格区域名称，它会将网格项目放置到相应的网格区域中。

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
    grid-template-areas: "aside  main";
    
    aside {
        grid-area: aside;
    }
    
    main {
        grid-area: main;
    }
}
```

  


请参见下图：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/628e50616f59476194ba4b9a02b32e05~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1596&s=612729&e=jpg&b=000000)

  


在上一个示例的基础上，我们需要在网格布局中添加一个页眉和页脚。该怎么做呢？

  


我们可以向 `grid-template-areas` 中添加另两个区域字符串。在 CSS Grid 中，当我们为 `grid-template-areas` 定义多个区域字符串值时，会自动创建隐式的网格行：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
    grid-template-areas:
        "header  header"
        "aside   main"
        "footer  footer";
}
```

  


有一个小细节需要注意，在 `grid-template-areas` 中添加多个区域字符串时，最好将它们分行排列，这样可以帮助我们更好地可视化布局。因为每个区域字符串代表网格中的一行（即每个双引号 `""` 代表一行）。你可以把它想象成一个表格：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
    grid-template-areas:
        ---------------------
        | "header | header" |
        ---------------------
        | "aside  | main"   |
        ---------------------
        | "footer | footer" |;
}
```

  


其中相同名称的部分还可以想象成合并了表格单元格：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
    grid-template-areas:
        ---------------------
        | "header   header" |
        ---------------------
        | "aside  | main"   |
        ---------------------
        | "footer   footer" |;
}
```

  


注意，代码中的 `-` 和 `|` 是不能真正出现在 CSS 代码中，在这里只是为了给大家提供一个可视化的理解。

  


现在，你在编写 CSS 代码的时候，就已经能想象出布局是什么样了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/132f943079184c04aa112c8c636ac00a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1927&s=886361&e=jpg&b=000000)

  


在这个基础上，我们只需要通过更改网格容器的 `grid-template-areas` 就可以改变布局。以下 CSS 保持不变，我们只需要更改网格区域字符串：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px       minmax(0, 1fr);
    grid-template-areas: /* 改变它的值，获得不同的布局 */
}


header {
    grid-area: header;
}

aside {
    grid-area: aside;
}

main {
    grid-area: main;
}

footer {
    grid-area: footer;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/326a86208f664ea0ad8e96288e0f60f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=3862&s=2299726&e=jpg&b=161719)

  


通过更改 `grid-template-areas` 字符串，整个布局会自动更新，使用相同的唯一网格区域名称（`header` 、`aside`、`main`、`footer`）。这种方式集中控制布局，使调整变得简单高效。

  


在 CSS Grid 中，使用 `grid-template-areas` 定义网格区域是最常用的一种方式。除此之外，我们还可以通过命名网格线来定义网格区域。不知道大家是否还有印象，小册的《[CSS 图形：创建突破元素](https://juejin.cn/book/7395119173656903715/section/7418491025368612901#heading-3)》一课中，介绍了如何使用 CSS Grid 来创建突破元素——[创建全屏布局（Full Bleed Layout）](https://juejin.cn/book/7395119173656903715/section/7418491025368612901#heading-0)。在这个方法中，我们应用了类似下面的 CSS 代码：

  


```CSS
body {
    --gap: clamp(1rem, 6vw, 3rem);
    --full: minmax(var(--gap), 1fr);
    --content: min(70rem, 100% - var(--gap) * 2);
    
    display: grid;
    row-gap: var(--gap);
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];
}
```

  


在使用 `grid-template-columns` 定义网格列的时候，加入以 `[*-start]` 和 `[*-end]` 定义的网格线名称（注意，`*` 是任意你喜欢的名称 ）。通常情况下，这种语法容易让人感到困惑，但在某些情况下非常有用。比如，刚才提到的创建全屏布局，就非常有用。

  


接下来，我会尽力向大家解释清楚。

  


首先，我们把上面代码中的 `grid-template-columns` 简化一下：

  


```CSS
body {
    grid-template-columns:
        var(--full) var(--content) var(--full);
}
```

  


它会创建一个三列网格。如果你打开浏览器开发者工具查看这个网格，你会发现网格线的数量等于网格列数加一。如果网格有三列，那么网格将会有四条网格线。默认情况之下，每条网格线都会用一个数字进行命名。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/db9c6699f7e945d3bef8955895f408d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=916&s=274583&e=jpg&b=dec14d)

  


事实上，我们可以为每条网格线命名，以替代默认的数字索引号。例如：

  


```CSS
/* 修改前 */
body {
    grid-template-columns:
        var(--full) var(--content) var(--full);
}

/* 修改后 */
body {
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];
}
```

  


这个时候，你可以看到每个网格线除了默认的数字索引号之外，还有相应的名称：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/344c350a802840cb846526058c271961~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1767&s=461104&e=jpg&b=000000)

  


从上图中不能发现：

  


-   网格线 `1` 对应的名称为 `full-start`
-   网格线 `2` 对应的名称为 `content-start`
-   网格线 `3` 对应的名称为 `content-end`
-   网格线 `4` 对应的名称为 `full-end`

  


但需要知道的是，虽然每条网格线有了具体的名称，但并不代表数字索引号的网格线就不存在了。换句话说，下面两种方式所起的作用是等同的：

  


```CSS
body {
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];

    > *:not(.full-bleed) {
        grid-column: 2 / 3;
    }
    
    .full-bleed {
        grid-column: 1 / 4;
    }
}

/* 等同于 */

body {
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];

    > *:not(.full-bleed) {
        grid-column: content;
    }
    
    .full-bleed {
        grid-column: full;
    }
}
```

  


这样做的好处是，我们可以语义化网格线名称，同样不需要去想象网格线的数字索引号。这在一些情景之下，我们只需要把注意力放在已定义的网格线名称上即可。

  


另外，在命名网格线名称的时候，在同一个中括号 `[]` 中，可以同时为同一条网格线定义多个名称。例如：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 200px minmax(0, 1fr);
}
```

  


很明显，上面的代码能让我们轻易构建一个两列布局（它创建了一个两列网格）。假设第一列是侧边栏（`aside`），第二列是主内容栏（`main`）。我们可以像下面这样给网格线命名：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: 
        [aside-start] 200px [aside-end main-start] minmax(0, 1fr) [main-end];
}
```

  


你可能已经发现了，代码中有一个 `[]` 号，它定义了两个名称 `[aside-end main-start]` 。在这个示例中，它同时给网格线 `2` 定义了两个名称，即 `aside-end` 和 `main-start` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7d7dc4931bf34483a82fcdd86dab1f1f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=916&s=178267&e=jpg&b=7ce547)

  


这使得我们在放置网格项目时候，可以像下面这样应用已命名的网格线名称：

  


```CSS
aside {
    grid-column: aside-start /  aside-end;
}

main {
    grid-column: main-start / main-end;
}
```

  


分别代表以应列的起始网格线名称和结束网格线名称。其实，它与下面代码所起的作用是相同的：

  


```CSS
aside {
    grid-column: aside-start / main-start;
}

main {
    grid-column: aside-end / main-end;
}
```

  


只不过，这样使用网格线名称，不易于理解，而且很容易混淆。

  


当然，你也可以随着布局变化来调整网格线的名称，甚至可以为同一条网格线定义更多的名称。例如：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns:
        [header-start aside-start footer-start] 200px
        [aside-end main-start] minmax(0, 1fr)
        [header-end main-end footer-end];
   grid-template-rows:
       [header-start] 88px
       [header-end aside-start main-start] minmax(0, 1fr)
       [aside-end main-end footer-start] min-content
       [footer-end];
}

header {
    grid-column: header-start / header-end;
    grid-row: header-start / header-end;
}

aside {
    grid-column: aside-start / aside-end;
    grid-row: aside-start / aside-end;
}

main {
    grid-column: main-start / main-end;
    grid-row: main-start / main-end;
}

footer {
    grid-column: footer-start / footer-end;
    grid-row: footer-start / footer-end;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2abeef3febd84208946e99edbb1a7478~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=916&s=512810&e=jpg&b=7ff15d)

  


这种方式，还可以使用 `grid-area` 来优化 CSS 代码：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns:
        [header-start aside-start footer-start] 200px
        [aside-end main-start] minmax(0, 1fr)
        [header-end main-end footer-end];
   grid-template-rows:
       [header-start] 88px
       [header-end aside-start main-start] minmax(0, 1fr)
       [aside-end main-end footer-start] min-content
       [footer-end];
}

header {
    grid-area: header;
}

aside {
    grid-area: aside;
}

main {
    grid-area: main;
}

footer {
    grid-area: footer;
}
```

  


它相当于创建了一个隐式的网格区域。

  


注意，在使用 `grid-template-columns` 和（或）`grid-template-rows` 创建网格时，并不是所有的网格线都必须命名，我们可以只命名布局中最关心的网格线，例如之前所展示的全屏布局示例。另外，在放置网格项目时，可以混合使用网格线的数字索引号和已命名的网格线名称，但个人不建议这样混合使用。因为，CSS Grid 布局本来就比较复杂，概念较多，这样混合使用更易于混淆，易于出错，且代码不好维护。

  


## 网格区域的应用场景

  


通过上面内容的学习，我想大家对网格区域有了更清晰的了解。接下来，我将通过一些实际案例，进一步向大家展示网格区域是多么强大，是如何更好的帮助我们构建 Web 布局。

  


### 反转网格方向

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5e294cd64e384c78a3718b2c9c85507f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=916&s=245081&e=jpg&b=000000)

  


上图所展示的是一个简单的卡片组件，它们唯一不同的是，用户头像所在的位置不同，一个居左，另一个居右。如果使用 CSS Flexbox 来构建这个布局效果，只需要在 Flexbox 容器上设置 `flex-direction: row-reverse` 即可。要是使用 CSS Grid 来布局，则需要更改每个网格项目在网格中的位置和定义不同的网格列：

  


```HTML
<div class="card">
    <figure>
        <img src="avatar.png" alt="avatar" />
    </figure>
    <div class="card__content">
        <h3>Rooms</h3>
        <p>Front-end technologist</p>
    </div>
</div>
```

  


```CSS
.card {
    --cols: 180px minmax(0, 1fr);
    --gap: 1rem;
    
    display: grid;
    grid-template-columns: var(--cols);
    
    figure {
        grid-column: 1 / 2;
    }
    
    .card__content {
        grid-column: 2 / 3;
    }
}
```

  


这是通过网格线来放置网格项目。并且用户头像居左，内容居右（上图中左侧所示）。现在，我们需要反过来，用户头像居右，文案居左（上图中右侧所示）。我们需要做两件事：

  


-   重新定义网格列，即调整 `--cols` 的值为 `minmax(0, 1fr) 180px`
-   更改每个网格项目的 `grid-column` 的值

  


具体的代码如下：

  


```CSS
.card--flip {
    --cols: minmax(0, 1fr) 180px;
    
    figure {
        grid-column: 2 / 3;
    }
    
    .card__content {
        grid-column: 1 / 2;
        grid-row: 1;
    }
}
```

  


注意，熟悉 CSS Grid 布局的同学应该知道，上面的代码并不是最简代码，实际上简化后的 CSS 代码如下：

  


```CSS
/* 默认效果 */
.card {
    --cols: 180px minmax(0, 1fr);
    --gap: 1rem;
    
    display: grid;
    grid-template-columns: var(--cols);
}

/* 反转效果 */
.card--flip {
    --cols: minmax(0, 1fr) 180px;
    
    figure {
        grid-column: 2;
    }
    
    .card__content {
        grid-column: 1;
        grid-row: 1;
    }
}
```

  


示例中没有采用简化后的 CSS 代码是不想增加大家理解 CSS Grid 的难度，并且能更好的与接下来的网格区域案例相比较。现在你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3a6080df6d7f4cb88b4b6b40617ebb98~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=916&s=245081&e=jpg&b=000000)

  


> Demo 地址：https://codepen.io/airen/full/dyxrRWQ

  


你可能猜到了，要是使用网格区域的相关特性，可以更简单很多。因为我们只需要调整网格列和网格区域的名称，无需考虑网格项目所对应的网格线索引号。例如：

  


```CSS
.card {

    }

}
```

```HTML
<div class="cards">

                <img src="figure.png" alt="" />
```

```HTML
            </button>     
        </div>
    </div>
    <!-- 省略其他的 Card -->

```

  


接着在 CSS 中，将 `.card__wrapper` 声明为一个查询容器：

  


```CSS
.card__wrapper {
    container-type: inline-size;
}
```

  


如此一来，我们就可以根据容器查询 `.card__wrapper` 的 `inline-size` 来调整 `.card` 的布局。

  


```CSS
.card {
    display: grid;
    gap: 1rem;
    align-content: start;

    grid-template-areas:
        "thumb"
        "title"
        "action";

    :is(.card__thumb, .card__badge) {

    }

    .card__title {
        grid-area: title;
    }

        grid-area: action;
    }
    
    @container (width >= 600px) {
        grid-template-columns: repeat(2, minmax(0, 1fr));
        grid-template-areas:
            "thumb title"
            "thumb action";
    }
}
```

### 响应式 Web 布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/21e4a96eef09418ca0d2c4aea6a9f49a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1944&s=566345&e=jpg&b=000000)

  


使用 CSS 网格区域来构建响应式 Web 布局也是非常便利的。接下来，我们来使用网格区域构建类似上图所示的响应式 Web 布局。

  


我们可以基于下面这个简单的 HTML 结构来构建所需要的 Web 布局：

  


```HTML
<main>
    <div class="featured"></div>
    <div class="secondary"></div>
    <div class="most-read"></div>
</main>
```

  


我们先从移动端着手：

  


```CSS
main {
    display: grid;
    gap: 1rem;
    grid-template-columns:auto minmax(0, 1fr) auto;
    grid-template-areas:
        "... featured  ..."
        "... secondary ..."
        "... most-read ...";
        
    .featured {
        grid-area: featured;
    }
    
    .secondary {
        grid-area: secondary;
    }
    
    .most-read {
        grid-area: most-read;
    }        
}
```

  


我们使用 `grid-template-columns` 定义了一个三列网格，其中第一列和第三列的宽度为 `auto` ，第二列的列宽为 `minmax(0, 1fr)` 。并且使用 `grid-template-areas` 定义了网格区域。你可能发现了，在定义网格区域的时候我们使用空单元素格（`...`）。这样做是为了使元素的水平方向始终与浏览器两侧会有一个 `1rem` 的间距（由 `gap` 属性决定）。你将看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b6cca435e09490b967dd2221b10638a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1260&h=656&s=3910917&e=gif&f=232&b=17d937)

  


接下来，我们只需要使用 [CSS 的媒体查询 `@media`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935) 在不同的断点中重置 `grid-template-columns` 和 `grid-template-areas` 属性的值：

  


```CSS
main {
    display: grid;
    gap: 1rem;
    grid-template-columns:auto minmax(0, 1fr) auto;
    grid-template-areas:
        "... featured  ..."
        "... secondary ..."
        "... most-read ...";
    
    @media only screen and (width >= 760px) {
        grid-template-columns:minmax(1rem, 1fr) repeat(2, minmax(auto, 30rem)) minmax(1rem, 1fr);
        grid-template-areas:
            "... featured featured  ..."
            "... secondary most-read ...";
    }
    
    @media only screen and (width >= 1024px) {
        grid-template-columns:minmax(1rem, 1fr) minmax(auto, 57.313rem) minmax(90px, 130px) minmax(1rem, 1fr);
        grid-template-areas:
            "... featured  most-read ..."
            "... secondary most-read ...";
    }
    
    .featured {
        grid-area: featured;
    }
    
    .secondary {
        grid-area: secondary;
    }
    
    .most-read {
        grid-area: most-read;
    }
}
```
