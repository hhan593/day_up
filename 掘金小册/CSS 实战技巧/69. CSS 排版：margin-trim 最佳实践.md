![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/11b5c2a492cb4b108361a3ad94dc71e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1210&h=678&s=56652&e=webp&b=f0f0f1)

  


随着 CSS 技术不断的革新，我们已经拥有有很多现代布局技术，这些布局技术能够帮助我们创建出色的布局，但要将细节打磨到完美却需要耗费大量的精力。可以说，这种精力的投入有时未免过多，而 Web 开发者通常没有足够的时间去做到这一点。

  


比如说，布局中的外边距（`margin`）就是一个非常令人头痛的问题。它的行为有些怪异，比如边距折叠，经常会导致你在构建布局的时候有不理想的间距。庆幸的是，最近几年 CSS 推出了许多新特性，比如 `margin-trim` 特性，可以帮助我们很好的解决外边距折叠的问题。

  


在这节课中，我们就一起来探讨一下 `margin-trim` 特性，它是如何帮助我们解决外边距折叠的烦恼。当然，为了大家能从此不再因为边距（`margin`）折叠感到烦恼，我会先向大家介绍一些导致外边距折叠的场景，包括一些技巧和窍门，帮助我们更好的地处理外边距折叠。然后再向大家展示 `margin-trim` 特性，看看它是如何移除元素子项的额外边距的。

  


## 掌握外边距折叠

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f83670b8c1d441e5b091c7f597d6e2e4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=459066&e=webp&b=f4ecd4)

  


CSS 的 `margin` 属性应该是大家常用的属性之一，总是会使用它来设置元素之间的间距。然而，`margin` 有一个令众多 Web 开发者感到头痛的问题，那就是外边距折叠。如果你不了解它是什么以及它是如何工作的，可能会使 Web 布局变得非常混乱。

  


事实上，很多 Web 开发者并不真正理解它是什么、发生了什么、它是如何工作的。因此，在探讨 `margin-trim` 之前，很有必要讨论一下外边距折叠，了解导致外边距折叠的场景，包括一些技巧和窍门。让我们更深入地了解外边距折叠，摆脱困惑！

  


### 什么是外边距折叠？

  


在 CSS 中，相邻元素的外边距有时会发生重叠。我们把这种现象称为“**外边距折叠（Margin Collapse）** ”，这个现象时常令 Web 开发者感到困惑，尤其是刚接触 CSS 的 Web 开发者。

  


以下是外边距折叠的典型案例，我们有两个相邻的兄弟元素：

  


```HTML
<header>Header</header>
<main>Main</main>
```

  


我们在 `header` 元素上应用了 `margin-bottom: 1.5rem` ，同时在 `main` 元素上应用了 `margin-top: 1.5rem` ：

  


```CSS
header {
    margin-bottom: 1.5rem;
}

main {
    margin-top: 1.5rem;
}
```

  


按理来说，`header` 和 `main` 在纵轴之间的间距应该是 `3rem` ，但事实上呢？它们之间的间距是 `1.5rem` ，两者的外边距合并到一起，占据了相同的的空间：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/41b380002bc34b2b9dd63738f3245cd7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1568&h=878&s=7577654&e=gif&f=189&b=ffffff)

  


似乎不易于理解，两个元素的外边距怎么就重叠在一起了。更令人感到困惑的是，外边距有时候会合并，有时候又不会发生合并。请注意我使用的是“有时候”这个词，因为并非所有相邻元素的外边距都会发生合并。甚至是，相邻元素外边距发生了合并，但合并的方式很奇怪且出乎意料。在实际项目中，许多情况都会让这一现象变得更为复杂。

  


不用担心，这其实并不复杂。它遵循一些规则，一旦我们理解了这些规则，它就变得清晰多了，就能很容易掌握它！

  


### 外边距折叠的规则

接下来，我们将通过各种场景和规则进行探讨，通过示例来讲解，这样你就不会再对外边距折叠感到困惑了。

  


#### 只有垂直外边距会折叠

  


你可能不知道，当外边距折叠被添加到 CSS 规范中时，语言设计者做出了一个有趣的选择：**水平外边距不应该折叠**。这是因为，早期的 CSS 并不是为了布局而设计的，编写规范的人当时设想的只是标题和段落，并没有考虑到列。

  


所以这是我们的第一个规则：**只有垂直方向的外边距才会折叠**。

  


```HTML
<main>Main Content</main>
<aside>Sidebar Content</aside>
```

  


```CSS
main {
    margin-right: 1.5rem;
}

aside {
    margin-left: 1.5rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d139929fc8ea4c4889db17ddf101c91a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1632&h=762&s=6284567&e=gif&f=167&b=ffffff)

如果你喜欢探索 CSS 的话，你可能已经知道了，CSS 提供了一个 `writing-mode` 属性，它可以帮助我们切换书写模式，让块级元素水平排列，而不是垂直排列。这时，外边距会在水平方向折叠，而不是垂直方向。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/92e877ba66c448848b92584d3cc000fd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=3991&s=1534829&e=jpg&b=9cceb5)

  


因此，更为准确的描述是：只有块级方向的外边距会折叠：

  


```CSS
main {
    margin-block-end: 1.5rem;
}

aside {
    margin-block-start: 1.5rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f0571c5889f14f2488278ffd28496456~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=3991&s=1620866&e=jpg&b=9cceb5)

  


注意，代码中的 `margin-block-end` 和 `margin-block-start` 是 CSS 的逻辑属性性。在多语言排版中，通常选择逻辑属性，这样更便于匹配语言的书写模式的阅读模式。有关于这方面更详细的介绍，请移步阅读下面这些课程的内容：

  


-   [CSS 逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)
-   [Web 中的向左向右：Flexbox 和 Grid 布局中的 LTR 与 RTL](https://juejin.cn/book/7161370789680250917/section/7161625525763440647)
-   [Web 中的向左向右：Web 布局中 LTR 切换到 RTL 常见错误](https://juejin.cn/book/7161370789680250917/section/7161625415935590436)
-   [多语言 Web 应用需要注意的细节](https://juejin.cn/book/7199571709102391328/section/7199846338140307495)
-   [CSS 排版：美化多语言网站技巧](https://juejin.cn/book/7395119173656903715/section/7445251774107090995)

  


#### 只有相邻元素的外边距会折叠

  


仅相邻元素的外边距才会折叠——元素必须在 DOM 中相邻，它们的外边距才会折叠。有的时候，你可能会在块元素之间使用 `<br />` 来增加元素之间的间距：

  


```HTML
<main>Main Content</main>
<br />
<aside>Sidebar Content</aside>
```

  


```CSS
main {
    margin-block-end: 1.5rem;
}

aside {
    margin-block-start: 1.5rem;
}
```

  


遗憾的是，`<br />` 标签会对我们的外边距产生不利影响——**阻止元素外边距的折叠**：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/693a6a0d99ee4b70a7b213b93b2a086a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=864&s=5798572&e=gif&f=159&b=ffffff)

  


`<br />` 标签虽然是不可见且为空的，但任何位于两个元素之间的元素都会阻止外边距折叠。但有一个细节需要注意，如果两个块元素之间有一个非 `<br />` 标签元素，比如 `<section>` ：

  


```HTML
<main>Main Content</main>
<section></section>
<aside>Sidebar Content</aside>
```

  


```CSS
main {
    margin-block-end: 1.5rem;
}

aside {
    margin-block-start: 1.5rem;
}
```

  


在这里，`<section>` 标签是一个空标签，什么内容都没有，包括什么样式也没设置。这个时候 `<section>` 元素并不会阻止 `main` 和 `aside` 两元素外边距的折叠。但是，`section` 一旦内容和样式，比如 `height` 为 `0.1px` ，那么它就会阻止 `main` 和 `aside` 两元素外边距折叠：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a5f12199192a4b69b3253d369a350d63~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1260&h=562&s=315930&e=gif&f=161&b=ffffff)

  


这意味着，位于两元素之间的空标签（无内容，无样式）不会阻止外边距的折叠！换句话说，**任何位于两个元素之间的元素（有内容或有样式的元素）都会阻止外边距折叠**！

  


#### 更大的外边距获胜

  


到目前为止，在所有的示例中，给元素设置的外边距是相同的值 `1.5rem` 。那么，当两个元素的外边距值不相等时怎么办呢？比如：

  


```CSS
main {
    margin-block-end: 3rem;
}

aside {
    margin-block-start: 1rem;
}
```

  


外边距折叠的基本理念是允许外边距重叠，因此更大的外边距决定了元素之间的间距——**较大的外边距获胜**：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ed00566ac81d4c8fa1c20a4e942c4371~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1422&h=830&s=5112361&e=gif&f=130&b=ffffff)

  


这个规则同样适用于负外边距情况——绝对值更大的负值会获胜：

  


```CSS
main {
    margin-block-end: -3rem;
}

aside {
    margin-block-start: -1rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/003770ae0b4a46fc91bf869f51d0b235~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=834&s=286004&e=jpg&b=9cceb5)

  


当同时使用负外边距和正外边距时，它们会按照基本的数学规则相互抵消——**负外边距和正外边距会相互抵消**：

  


```CSS
main {
    margin-block-end: 3rem;
}

aside {
    margin-block-start: -1rem;
}
```

  


两个元素之间折叠的外边距等于 `2rem` （即 `3rem - 1rem = 2rem`）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b12b4493bdbe40c7b8ed9e670bdf67c1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1135&s=421498&e=jpg&b=9cceb5)

  


#### 嵌套并不会阻止外边距折叠

  


接下来就开始变得有点奇怪了。请看下面的代码：

  


```HTML
<div>
    <p>段落一</p>
</div>
<p>段落二</p>    
```

  


```CSS
p {
    margin-block: 1.5rem;
}
```

  


我们将第一个段落 `<p>` 嵌套在一个 `<div>` 容器中，但两个 `<p>` 元素的外边距依然会折叠（重叠在一起）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5ee24e7d267648a58e5eb4baf1d50eda~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1300&h=844&s=4089838&e=gif&f=163&b=ffffff)

  


事实证明，我们中许多人对外边距（`margin`）的工作原理有所误解。

  


外边距的目的是增加兄弟元素之间的距离，而不是增加子元素与其父元素边框盒子（`border-box`）边界之间的间距。增加子元素与其父元素边框盒子（`border-box`）边界之间的间距应该是内距（`padding`）的事情。

  


外边距总是会尝试增加兄弟元素之间的距离，包括子元素将边距转移到其父元素上。在这种情况下，效果就像是我们将外边距应用于父元素 `<div>` ，而不是 `<p>` 元素。

  


这个时候，你可能会说，不可能。因为外边距的确使 `<p>` 元素（子元素）和 `<div>`（父元素）之间有一定的距离——看上去，就像是 `<div>` 元素设置了内距。

  


外边距只有在接触时才会被折叠。如果外边距之间有任何间隙或障碍，它们就不会被折叠。比如下面这些情况下，嵌套的外边距不会被折叠。

  


第一种情况就是父元素 `<div>` 有一个内距：

  


```HTML
<div>
    <p>段落一</p>
</div>
<p>段落二</p>    
```

  


```CSS
div {
    padding-block-end: 1.5rem;
}

p {
    margin-block: 1.5rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0f13e86617fe480d860e93617899247c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1426&h=820&s=5249800&e=gif&f=142&b=fffefe)

同样的，如果 `<div>` 元素设置了边框，即使是 `1px` 的边框，也会阻止外边距合并。

  


```CSS
div {
    border: 1px solid currentColor;
}

p {
    margin-block: 1.5rem;
}
```

  


需要知道的是，为了让内距（`padding`）或边框（`border`）阻止外边距折叠，它必须位于两个外边距中间。比如，`div` 元素有一个 `padding-block-start` 或 `boder-block-start` ，它们不会阻止其子元素 `<p>` 与兄弟元素 `<p>` 之间的外边距折叠。想象一下，这两个外边距是相邻的：它们能接触吗，还是有内距或边框在中间？如果它们能接触，它们就可以折叠，否则就不会被折叠。

  


另一种情况是，给父元素 `<div>` 设置固定高度可以防止两个 `<p>` 元素的外边距折叠：

  


```HTML
<div>
    <p>段落一</p>
<div>
<p>段落二</p>    
```

  


```CSS
div {
    height: 300px;
}

p {
    margin-block: 1.5rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5f9bc13700aa48019ff7ba7c41967b37~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1376&h=816&s=5006589&e=gif&f=163&b=ffffff)

你之前的规则一样，这个规则是按每个方向来操作的。在这个例子中，子元素 `<p>` 的外边距仍然可以被折叠。但因为子元素下面有空白空间（它的父元素 `<div>` 高度要更高），它的下外边距（`margin-block-end`）将永远不会合并。因为这个空间空间就像是一条壕沟，阻止了它们的折叠。

  


除此之外，它们还会被 `overflow` 属性创建的滚动容器阻止，因为外边距不能跨越滚动容器的边界折叠：

  


```CSS
div {
    overflow: hidden; /* 或 auto、clip 和 scroll */
}

p {
    margin-block: 1.5rem;
}
```

  


#### 外边距可以在相同方向上折叠

  


到目前为止，我们看到的所有示例都涉及相邻的相反方向的外边距：一个元素的底部外边距（`margin-block-end`）与相邻元素的顶部外边距（`margin-block-start`）折叠。然而，令人惊讶的是，**外边距甚至可以在相同方向上折叠**。

  


```HTML
<div class="parent">
    <p class="child">段落</p>
</div>    
```

  


注意，上面代码是两个相互嵌套的元素而不是相邻兄弟元素。

  


现在，我们给它们同一个方向上设置不相同的外边距：

  


```CSS
.parent {
    margin-block-start: 3rem;
}

.child {
    margin-block-start: 1rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/823708b3b67249f79e8945c8a0c662e5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1524&h=752&s=5808216&e=gif&f=212&b=ffffff)

正如你所看到的，子元素的外边距被“吸收”进父元素的外边距。两者结合在一起，并遵循我们迄今为止看到的外边距折叠的规则（例如，较大的外边距“获胜”）。

  


这个规则也为众多 Web 开发者带来了一些困惑。比如下面这个示例：

  


```HTML
<section class="blue">
    <p>Paragraph One</p>
</section>
<section class="pink">
    <p>Paragraph Two</p>
</section>
```

  


```CSS
.blue {
    background-color: lightblue;
}

.pink {
    background-color: lightpink;
}

p {
  margin-block-start: 3rem;
}
```

  


你可能期望两个部分（“蓝色”和“粉色”）紧挨着，但事实并没有所你所期望的那样，这是因为其子元素 `p` 应用了外边距 `margin-block-start: 3rem` 。因此，你所看到的效果是，两个部分之间有一个 `3rem` 的间距：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/55f100e47c5a4dfa9e00ee0c7178268c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=483&s=108258&e=jpg&b=ffffff)

  


虽然 `<section>` 元素未显式设置任何外边距，但可以将它的外边距视为 `0px` ，而子元素 `p` 的顶部外边距为 `3rem` ，它与父容器顶部外边距（`0px`）发生了折叠，而且子元素的外边距更大，它占据了上风。因此，在视觉上，两个相邻的 `section` 元素之间有 `3rem` 间距。

  


很少人会预料到外边距会与父容器折叠在一起。如果你想禁用这种行为怎么办呢？最简单的方式，就是创建一个新的块级格式化上下文（BFC）。比如说，将 `section` 的 `display` 设置为除 `block` 、`none` 和 `inline` 之外的值。比如你熟悉的 `flex` 或 `grid` ，还可以是 `flow-root` 。

  


```CSS
section {
    display: flow-root;
}
```

这相当于给 `section` 元素设置了一堵墙，没有外边距能够穿透它。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a547ad5c4e649289f601515bf5071d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=483&s=128533&e=jpg&b=ffffff)

  


#### 外边距折叠不仅限于两个

  


需要知道的是，外边距的折叠不仅仅限于两个外边距。在下面这个示例中，四个独立的外边距占据了相同的空间：

  


```HTML
<header>
    <h1>My Project</h1>
</header>
<section>
    <p>Hello World</p>
</section>
```

  


```CSS
header {
    margin-block-end: 1rem;
    
    h1 {
        margin-block-end: 2rem;
    }
}

section {
    margin-block-start: 3rem;
  
    p {
        margin-block-start: 4rem;  
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34e8798112f54e11b7a6cd03d9b6816e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1316&h=800&s=7002126&e=gif&f=149&b=ffffff)

  


  


虽然很难直接看到发生了什么，但这本质上是前面规则的组合：

  


-   兄弟元素（蓝色的 `header` 元素和浅蓝色的 `section` 元素）可以合并相邻的外边距（如果 `header` 元素有 `margin-block-end`，而 `section` 元素有 `margin-block-start`）
-   父元素和子元素可以合并相同方向的外边距
-   每个兄弟元素都有一个子元素，子元素贡献了相同方向的外边距

  


就我们这而言，`header` 和 `section` 之间的空间有四个独立的外边距在竞争占据这个空间：

  


-   `<header>` 想要它下面有空间
-   `<header>` 中的 `h1` 有下外边距，这个外边距与其父元素合并
-   `<section>` 下方的元素想要它上面有空间
-   `<section>` 中的 `p` 有上外边距，这个外边距与其父元素合并

  


最终，段落（`p`）有最大的外边距，因此它“获胜”，使得 `<header>` 和 `<section>` 之间的间隔为 `4rem`。

  


#### 外边距折叠仅限于流式布局

  


到目前为止，我们看到的所有示例都处在“流式布局”中。我们没有改变元素的 **[视觉盒模型格式](https://juejin.cn/book/7199571709102391328/section/7199844993073152038)**，即没有调整元素的 `display` 属性的值。换句话说，之前案例都是块元素（即 `display: block`）。那么问题来了，如果元素不是块级元素，它是行内元素（`display: inline`）或行内块元素（`display: inline-block` ）或者说是 Flexbox （`display` 为 `flex` 或 `inline-flex`）和 Grid （`display` 为 `grid` 或 `inline-grid`）上下文格式化元素，外边距还会被折叠吗？

  


首先要知道的是，行内元素（`display: inline`）的顶部和底部的外边距（`margin-block`）不起作用，因此它们不存在外边距折叠这一现象。

  


其次是内行块元素（`display: inline-block` ）允许我们添加顶部和底部外边距（`margin-block`），但即使它们与块级元素组合，外边距也不会被折叠。

  


最后就是 `display` 为 `grid` （或 `inline-grid`）和 `flex` （或 `inline-flex`）的情况。具体表现如下所述：

  


-   **`display: grid`**：对于`display`属性设置为`grid`的元素，边距会折叠。和块级元素一样，当它与行内块元素组合时，边距停止折叠
-   **`display: flex`**：对于`display`属性设置为`flex`的元素，边距也会被折叠。同样，当它与行内块元素组合时，边距不再被折叠
-   **`display: inline-flex`** **和** **`display: inline-grid`**：就像行内和行内块元素一样，`inline-flex`和`inline-grid`的边距不会折叠

  


但是，Flexbox 和 Grid 容器与它们的子元素（Flex 项目和 Grid 项目）的外边距不会折叠。

  


还有一点，当元素使用 Grid 或 Flexbox 对齐，或被移出流式布局（例如浮动、绝对定位）时，外边距将永远不会被折叠。

  


上面所列的就是我能想到的，外边距会发生折叠的所有场景，如果有遗漏，欢迎大家在评论中拍正！

  


## 修剪外边距：`margin-trim`

  


现在，让我们来看看 CSS 中另一个看似不起眼但实际上非常强大的特性—— `margin-trim` 。

  


在介绍 `margin-trim` 特性之前，我们来看两个实际用例。

  


假设在一个 `<article>` 元素中包含了多个段落（`<p>`），并使用下面的样式：

  


```HTML
<article>
    <p>Lorem ipsum dol...</p>
    <!-- 省略其他 p -->
</article>
```

  


```CSS
article {
    padding: 2lh;
    font-size: 1.2rem;
    line-height: 1.3;
    
    p {
        margin-block: 1lh;
    }
}
```

  


我们使用 `lh` 单位设置了段落之间的外边距和 `article` 容器的内边距。我们试图创建一个优雅的排版效果，但是，你会发现文本上下方（第一个 `p` 元素的上方和最后一个 `p` 元素的下方）有额外的空间：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/159a80d2c4bd490391acdbfe272d2beb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=1083&s=104406&e=avif&b=f8f0df)

  


上图中黄色区域是 `article` 容器的内距（`padding`），绿色区域是 `p` 元素的上下外边距 `margin-block` 。不难发现，第一个段落的上外边距 `1lh` 和最后一个段落的下外边距 `1lh` 都与 `article` 容器的内距 `2lh` 叠加在一起，总共形成了 `3lh` 的空间。这显得有些多了。

  


造成这个现象，就不用额外多说什么，因为我们之前花了很长的篇幅讨论这个现象——外边距折叠。在没有 `margin-trim` 之前，我们通常会避开给第一个和最后一个 `p` 元素设置外边距。比如：

  


```CSS
p:not(:first-of-type, :last-of-type) {
    margin-block: 1lh;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9a432c29d50847f4ad891e543d71a1ef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=745345&e=jpg&b=f9efe4)

  


> Demo 地址：https://codepen.io/airen/full/WbeKbWY

  


再来看一个卡片组件的用例：

  


```HTML
<div class="card">
    <img src="https://picsum.photos/id/163/1920/1024" alt="" />
    <h3>Kiwis</h3>
    <p>Kiwis (the fruit) are native to China and were originally called Chinese gooseberries. They grow on vines. 98% of kiwis grown in the US are grown in California.</p>
</div>
```

  


你可能出于某种考虑，[并没有使用诸如 gap 这样的属性来设置元素之间的间距](https://juejin.cn/book/7395119173656903715/section/7438081482993696778)，而是采用了 `margin` ：

  


```CSS
.card {
    display: grid;
    padding: 1lh;
    line-height: 1.3;
    
    * {
        margin-block-end: .5lh;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e3dcb2605fe04feda658447a8a0fc234~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=401503&e=jpg&b=898989)

  


可以看到，最后一个子元素的底部边距强行让卡片底部出现了额外空间，导致卡片的高度超出了预期。这种问题虽然可以通过 `:last-child` 选择器解决，例如：

  


```CSS
.card {
    display: grid;
    padding: 1lh;
    line-height: 1.3;
    
    *:not(:last-child) {
        margin-block-end: .5lh;
    }
}
```

  


或者使用 `* + *` 选择器解决：

  


```CSS
.card {
    display: grid;
    padding: 1lh;
    line-height: 1.3;
    
    * + * {
        margin-block-start: .5lh;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8f5b0ab031194756b9d6258e7241754f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1167&s=396220&e=jpg&b=898989)

  


> Demo 地址：https://codepen.io/airen/full/WbeKvpB

  


尽管解决了问题，但这种方法并不理想，因为它在组件化设计中不够灵活。子组件必须知道父组件需要移除它的边距，这种依赖关系让子组件与父组件耦合在一起。

  


新的 `margin-trim` 属性为我们提供了一种简单的方法来实现这一点。以第一个示例为例，使用 `margin-trim` 属性会告诉容器（比如 `article`），**去除与容器边缘相接触的任何外边距**。例如：

  


```CSS
article {
    margin-trim: block;
    font-size: 1.2rem;
    line-height: 1.3;
    padding: 2lh;
    
    p {
        margin-block: 1lh;
    }
}
```

  


现在，浏览器会自动裁剪在块方向（上下方向）上接触`<article>`容器边缘的所有外边距——在这个例子中，裁剪了容器顶部和底部的外边距。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/38562a5fdfb94e0d90592efe9a5428cd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=1083&s=112971&e=avif&b=f7eee0)

  


虽然外边距（`margin-block`）是定义在`<p>`段落上的，但`margin-trim`属性是定义在`<article>`容器上的。结果如下图所示，效果非常不错。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8572f849eec24b2a88d858fcd17a51a1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=1083&s=108656&e=avif&b=fdfdfd)

  


`margin-trim` 属性大约两年前在Safari 16.4中首次支持。目前，Safari是唯一支持该属性的浏览器。但你可以通过**渐进增强**技术从今天开始使用它。对于不支持的浏览器，你可以在特性查询中编写回退代码，例如：

  


```CSS
article {
    margin-trim: block;
    font-size: 1.2rem;
    line-height: 1.3;
    padding: 2lh;
    
    p {
        margin-block: 1lh;
    }
}

@supports not (margin-trim: block) {
    article {
        :first-child {
            margin-block-start: 0;
        }
        :last-child {
            margin-block-end: 0;
        }
    }
}
```

  


使用 `:first-child` 和 `:last-child` 选择器时，**仅修剪容器的第一个或最后一个直接子元素的外边距**。但对于未嵌套在任何元素中的内容或深层嵌套的内容，则不会被修剪。

  


例如，如果第一个子元素是一个带有上外边距的 `<figure>`，且该 `<figure>` 包含一张带有上外边距的图片，那么：

  


-   `margin-trim` 会修剪 `<figure>` 和图片的外边距
-   而 `:first-child` 仅修剪 `<figure>` 的外边距

  


```HTML
<article>
    <figure style="margin-top: 1em">
        <img style="margin-top: 1em" src="photo.jxl" alt="[alt]">
        <figcaption>[caption]</figcaption>
    </figure>
</article>
```

  


与旧的修剪外边距的方法相比，`margin-trim`使修剪更简单、更强大。你可以修剪接触容器边缘的块方向或行内方向的外边距，以下是可用选项：

  


-   `margin-trim: none`
-   `margin-trim: block`
-   `margin-trim: inline`
-   `margin-trim: block-start`
-   `margin-trim: block-end`
-   `margin-trim: inline-start`
-   `margin-trim: inline-end`

  


如果你想同时修剪两个方向的外边距，可以结合多个值，例如：

  


```CSS
margin-trim: block-start block-end inline-start inline-end;
```

  


最近，CSS工作组（CSSWG）决定允许使用更短的关键词组合，比如：

  


```CSS
margin-trim: block inline;
```

  


Safari 目前尚未更新以支持这一选项，但可以关注相关问题的更新动态。你可以通过交互式演示来尝试不同选项，观察 `margin-trim`的效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce1fb5a71ca0415aac38476b994f409a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1316&h=834&s=3534589&e=gif&f=249&b=1a1a1a)

  


> Demo 地址：https://codepen.io/airen/full/gbYjpVJ

  


## 小结

`margin-trim` 属性是一个非常简单但极其有用的属性，它可以轻松移除元素子元素的额外边距。在基于组件的设计系统中，这使得处理组件更加方便，因为你不再需要担心父组件或子组件的样式会溢出到它们自身之外。

  


现在正是 Web 排版的黄金时代。过去几年中发布的十几个小功能，让 Web 设计师和开发者能够以更高的质量完善排版细节，尤其是这些功能既简单又强大，使得实现这些目标变得更加实用。我希望你能好好利用它们！
