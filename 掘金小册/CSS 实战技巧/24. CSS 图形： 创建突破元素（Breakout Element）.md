![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/681a588d45384818a8c6cecfb5b79382~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2555&h=800&s=221652&e=jpg&b=d6b168)

  


在 Web 设计和开发中，“**突破元素**”（**Breakout Element**）通常指的是那些突破常规布局限制的元素。这些元素可能跨越多个网格区域、打破父容器的边界，或者以与周围内容不同方的方式呈现，从而实现更显眼或独特的视觉效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/470d579ae78d433881b5c38ceaf262f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=599&s=129529&e=jpg&b=0000fe)

  


Web 上最为典型的案例，那就是页面的英雄区域（Hero）通常会延伸到浏览器窗口的两侧，还有一些[区域的分隔符](https://juejin.cn/book/7395119173656903715/section/7406965718895296553)也会延伸到浏览器窗口的两侧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9b72d1c8533c4c448829c802869e56df~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=800&h=450&s=340616&e=gif&f=45&b=0c1c49)

  


[尤其是在构建一些创意性布局时](https://juejin.cn/book/7161370789680250917/section/7161625566062313503)，“突破元素”的场景就更多了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7d97c7d308f8482b89f26ccf5b486785~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2506&s=1151429&e=jpg&b=eeeae9)

  


有些效果对于 Web 开发者而言，调整一下 HTML 结构就能实现。但有很多“突破元素”的布局或 UI 效果会使 Web 开发者感到头痛，因为在创建“突破元素”时很容易使像父容器出现水平或垂直滚动条，或者打破整个 Web 布局，或者需要依赖“拼凑”的方式才能实现所要的效果。

  


那么这节课，我将和大家探讨一些创建突破元素的 CSS 技巧。我将从最传统的布局开始，然后再到一些[创意型布局](https://juejin.cn/book/7161370789680250917/section/7161625566062313503)，或“特殊” UI。从真正的实例中，让大家掌握创建突破元素的技巧。

  


为了能更好的阅读和理解接下来的内容，你需要具备以下基础知识：

-   [CSS 裁剪](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-1)（`clip-path`）和[遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)（`mask`）

## 创建全屏布局（Full Bleed Layout）：突破居中容器

  


在 Web 布局中，一个经典的案例就是视觉元素突破主内容列的边界。例如，你可能已经在 Medium 上看到过这种 Web 布局，主视觉通常是全屏（或称为 Full Bleed），而内容则保持居中：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/75586aace6f1480c9bb9e4204edf9c13~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=768&h=966&s=165616&e=png&a=1&b=fbfafa)

  


对于 Web 开发者而言，要实现它应该不是问题，问题是如何使整个布局更灵活，适配性更强！

  


假设你正在构建一个 Web 网站或应用，所有内容都限制在一个居中的容器中，例如：

  


```HTML
<div class="wrapper">
</div>
```

  


```CSS
.wrapper {
    width: clamp(20rem, 100% - 2rem, 70rem);
    padding-inline: 1rem;
    margin-inline: auto;
}
```

  


上面的代码会使容器的宽度限制在 `20rem ~ 70rem` 之间，并且水平居中。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/448417f8af6c4be1a4d04e39a7e26def~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1148&s=261402&e=jpg&b=f2f2f2)

  


问题是，并不是所有内容宽度都一样。比如，突然间你发现自己需要一个组件，它突破了 `.wrapper` 容器的宽度，需要跨越整个页面的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e557c4a514e4af1a4e975f6ff1c8bbd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1863&s=530473&e=jpg&b=f2f2f2)

  


这是一个常见的布局问题，那么我们如何在布局中处理“突破”的宽度呢？接下来，我会从最传统的方法开始，逐步过渡到更先进，更灵活的方法。这些现代方法在创建完全流体的布局时特别有用，并且非常适合与流体排版技术配合使用，从而实现真正随视口大小缩放的布局。

  


### 方案一：调整 HTML 结构

  


这应该是大多数 Web 开发者首先的一种解决方案，也是一种传统的解决方案。我们可以分别使用 `.full-bleed` 和 `.wrapper` 类名分别包裹：

  


-   将全屏内容包裹在一个名为 `.full-bleed` 类名中
-   非全屏内容的部分继续使用 `.wrapper` 类名

  


```HTML
<div class="wrapper"></div>
<div class="full-bleed"></div>
<div class="wrapper"></div>
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c724c4a04add416a80ea03a95329a938~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2311&s=739025&e=jpg&b=f4f4f4)

  


还有另外一种常见方式是，将所有 `.wrapper` 嵌套在一个容器中，比如 `<section>` ，并且给全屏宽度的 `<section>` 定义一个名为 `.full-bleed` 类名：

  


```HTML
<section>
    <div class="wrapper"></div>
</section>

<section class="full-bleed">
    <div class="wrapper"></div>
</section>

<section>
    <div class="wrapper"></div>
</section>    
```

  


然后重置 `.full-bleed` 中 `.wrapper` 的 `width` ：

  


```CSS
.wrapper {
    width: clamp(20rem, 100% - 2rem, 70rem);
    padding-inline: 1rem;
    margin-inline: auto;
    
    .full-bleed & {
        width: 100%;
        padding-inline: 0;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/477deb1de94d4fe9bf34fb132508a912~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2311&s=1118787&e=jpg&b=f4f4f4)

  


> Demo 地址：https://codepen.io/airen/full/YzmzpzV

  


你也可以引入 CSS 自定义属性，便于调整布局相关的参数：

  


```CSS
:root {
    --p: 1rem;
    --minW: 20rem;
    --maxW: 70rem;
}

.wrapper {
    width: clamp(var(--minW), 100% - 2 * var(--p), var(--maxW));
    padding-inline: var(--p);
    margin-inline: auto;
    
    .full-bleed {
        width: 100%;
        --p: 0;
    }
}
```

  


上面示例通过 `margin-inline: auto` （这是一个 [CSS 逻辑属性](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)）使 `.wrapper` 水平居中。其实，还可以通过 `padding-inline` 来使 `.wrapper` 的内容水平居中。

  


```CSS
:root {
    --p: 1rem;
    --wrapper-w: 70rem; /* wrapper 最大宽度 */
    --space: max(
        var(--p),
        (100% - var(--wrapper-w) - var(--p) * 2) / 2
    );
}

.wrapper {
    width: min(100%, 100vw); /* 避免出现水平滚动条，可以使用 100dvw 替代 */
    padding-inline: var(--space);

    padding-block: calc(2 * var(--p));
    background-color: var(--c);
    color: color-contrast(var(--c) vs #fff, #000);
}
```

  


它能根据浏览器视窗的宽度动态计算 `padding-inline` （即 `padding-left` 和 `padding-right`）的值：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f52ecd8b4daf43e88434d3ab092f5a4b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2543&s=1016754&e=jpg&b=fafafa)

  


注意，在我们这个示例中，假设 `.wrapper` 有一个 `--p` 的内距，所以在计算 `padding-inline` 时，使用了 `(100% - var(--wrapper-w) - var(--p) * 2) / 2` 。并且，使用了 CSS 比较函数 `max()` 来做判断，当 `(100% - var(--wrapper-w) - var(--p) * 2) / 2` 计算值小于 `--p` 的值时，`padding-inline` 的值为 `var(--p)` ，这样可以确保在小屏幕的时候，`.wrapper` 的内容始终与浏览器视窗两侧有一个内距。

  


另外，这种方式非常适合于页面上整个区域有一个底色的设计。当然，有的时候 `.wrapper` 内容是真实的全屏，并不仅限于其背景，这个时候我们可以在 `.wrapper` 上新增一个 `.full-bleed` 类名，并且在 `.full-bleed` 中将 `--space` 和 `--p` 的值设置为 `0` ：

  


```HTML
<div class="wrapper">
    <h1>Page Title</h1>
</div>

<div class="wrapper full-bleed">
    <img src="full-bleed.png" />
</div>

<div class="wrapper">
    <div calss="cards"><!-- .... --></div>
</div>    
```

  


```CSS
:root {
    --p: 1rem;
    --wrapper-w: 70rem; /* wrapper 最大宽度 */
    --space: max(
        var(--p),
        (100% - var(--wrapper-w) - var(--p) * 2) / 2
    );
}

.wrapper {
    width: min(100%, 100vw); /* 避免出现水平滚动条，可以使用 100dvw 替代 */
    padding-inline: var(--space);

    padding-block: calc(2 * var(--p));
    background-color: var(--c);
    color: color-contrast(var(--c) vs #fff, #000);

    &.full-bleed {
        --space: 0px;
        --p: 0px;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/373af66f3ce84909b2afccbcdd973310~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=748&s=9153807&e=gif&f=88&b=1e1e1e)

  


> Demo 地址：https://codepen.io/airen/full/VwowPzq

  


这几种方式得到的效果都还可以，但关键是你是否能够充分控制 HTML 结构。你可能会疑惑，自己构建的 Web 网站或应用，难道会没有自主掌握 HTML 结构的权利吗？这话看上去似乎没毛病，但总是有万一的情况，比如你的内容来自 CMS（内容管理系统）或你引用的是第三方组件库，你就有可能无法充分控制 HTML 的结构。这意味着，上面这几种方式就不能算是最灵活的解决方案！

  


### 方案二：视口宽度突破技巧

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/557d743438b24f7591ec186751bc8c14~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1100&h=1026&s=136620&e=png&b=15202b)

  


早在 2018 年的时候，[@Una Kravets 就在 Twitter 上分享过这个 CSS 技巧](https://x.com/una/status/951519740840873984)。使用视窗单位，我们可以强制一个元素突破 `.wrapper` 的限制，而无需更改原始的 HTML 结构。

  


假设你有一段像下面这样的 HTML 结构：

  


```HTML
<div class="wrapper">
    <header></header>
    <figure class="full-bleed">
        <img src="full-bleed.png" />
    </figure>
    <aside></aside>
    <main></main>
    <footer></footer>
</div>
```

  


默认情况之下，我们的内容都限制在 `.wrapper` 容器中：

  


```CSS
:root {
    --p: 1rem;
    --minW: 20rem;
    --maxW: 70rem;
}

.wrapper {
    width: clamp(var(--minW), 100%, var(--maxW));
    margin-inline: auto;
    
    .full-bleed {
        --p: 0;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/955fd8dc64724eec9c726eb8bba15b5d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2037&s=945057&e=jpg&b=050505)

  


到目前为止，包括全屏宽度 `.full-bleed` 都限制在 `.wrapper` 容器中。接下来，我们要使用视窗单位、定位等 Hack 技巧将 `.full--bleed` 元素突破 `.wrapper` 容器的限制，使其全屏展示：

  


```CSS
.full-bleed {
    --p: 0;
    position: relative;
    right: 50%;
    left: 50%;
    margin-inline: -50vw;
    max-width: 100vw;
    width: 100vw;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aeea8c2f4fdd4938abae143d310bde27~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1272&h=748&s=20186430&e=gif&f=140&b=85a0e9)

  


> Demo 地址：https://codepen.io/airen/full/MWNWVLE

  


[我们可以使用 CSS 的 `translate` 变换](https://juejin.cn/book/7223230325122400288/section/7259668493158023205)，使 CSS 代码变得更简洁一点：

  


```CSS
.full-bleed {
    --p: 0;
      
    margin-left: 50%;
    max-width: 100vw;
    width: 100vw;
    translate: -50% 0 0;
}
```

  


方便！但是你知道吗？我们可以通过 `transform` 做得更简洁：

  


```CSS
.full-width {
  width: 100vw;
  margin-left: 50%;
  transform: translate3d(-50%, 0, 0);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3eabafa988a648ebb26c948a9f0ec1e4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2037&s=1032115&e=jpg&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/Vwowxmw

  


不过有一个小问题：如何 `.wrapper` 元素上设置了 `overflow: hidden` ，这个方法就不起作用了，它会将溢出 `.wrapper` 容器的部分裁剪掉。不过，如果我们能避免这种情况，或者在无法控制 HTML 结构的情况下（甚至只需要一个快速修复方案时），这是一个不错的解决方案。

  


另一个缺点是，如果位于 `.wrapper` 中某个区块，例如 `header` 或 `footer` ，它们需要设置一个全屏的背景颜色或背景图，那么直接使用 `background` 是无法做到的，因为整个区域被限制在 `.wrapper` 容器中。不过，我们可能使用其他的 CSS 技巧来实现这一点。稍后我们会介绍这种技巧！

  


### 方案三：使用 CSS Grid 实现突破

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9039a213dd5460e8f73c881afadd6b8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1260&s=64157&e=jpg&b=4350ff)

  


相比而言，我个人认为使用 [CSS Grid 布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)技术创建全屏布局（Full Bleed）是最灵活的一种。我们可以创建一个三列网格，其中两列（第一列和第三列）的列宽可以根据容器的可用空间自动调整，[CSS Grid 中的 `fr` 单位](https://juejin.cn/book/7161370789680250917/section/7161624007702216735)很容易就可以做到；剩下的中间列就是 `.wrapper` 容器期望限制的宽度。

  


假设你有一个像下面这样的 HTML 结构：

  


```HTML
<body>
    <header class="wrappr"></header>
    <figure class="full-bleed"></figure>
    <section class="wrapper">
        <aside></aside>
        <main></main>
    </section>
    <footer class="wrapper"></footer>
</body>
```

  


注意，对于居中的内容你完全可以不显式设置 `class` 的值为 `wrapper` ，我这里是为了更好的区分全屏宽度的元素。

  


同样的，[我们可以定义一些 CSS 变量来控制布局相关的参数](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)：

  


```CSS
:root {
    --gap: clamp(1rem, 6vw, 3rem);
    --full: minmax(var(--gap), 1fr);
    --content: min(70rem, 100% - var(--gap) * 2);
}
```

  


-   `--gap` 代表页面左右两侧的间距大小。该值使用 [`clamp()` 函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108#heading-3)，实现更流畅、灵活的尺寸调整，它的值将会介于 `1rem ~ 3rem` 之间
-   `--full` 使元素横跨整个水平空间。通过将 `--gap` 设为最小值，它在较小屏幕上还充当页面可见边距的作用，有点类似 `padding-inline` 的功能
-   `--content` 作为主要的内容区域，[`min()` 函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108#heading-1)设定了此列的最大宽度。当可用空间小于该值时，宽度切换为 `100%`，同时减去左右的边距尺寸

  


代码中的 [`clamp()` 和 `min()` 是 CSS 中的比较函数](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)，它们可以动态调整值。另外一个“魔力”来自 [`minmax()` 函数](https://juejin.cn/book/7161370789680250917/section/7161624041885958151#heading-2)的使用，它定义了一个大小范围，取值大于或等于 `var(--gap)` 且小于或等于 `1fr` （其中 `1fr` 将根据网格容器的剩余大小来决定，它会随浏览器视窗大小进行调整）。注意，`minmax()` 和 `1fr` 只用于 CSS 网格布局中。

  


然后在 `body` 上[使用 `grid-template-columns` 来定义所需要的网格](https://juejin.cn/book/7161370789680250917/section/7161623971073359902)：

  


```CSS
body {
    display: grid;
    row-gap: var(--gap);
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/879748cacb124d30b47908291afb749e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2400&s=1455234&e=jpg&b=c27be6)

  


上面的代码对于熟悉 CSS Grid 布局技术的开发者不会有任何问题，但对于从未接触过或对 CSS Grid 布局技术不熟悉的 Web 开发者，我还是有必要简单的解释一下上面的代码。

  


在 `grid-template-columns` 中通过 `[关键词-start]` 和 `[关键词-end]` 的方式给列网格线命名，例如，在这个示例中第一条列网格线的名称是 `full-start` ，最后一条网格线名称是 `full-end` 。它们可以替代默认的数字索引号的网格线名称。如此一来，[我们就可以在网格项目上将 `grid-column` 属性的值设置为网格线的名称，就可以将网格项目放置在指定的位置](https://juejin.cn/book/7161370789680250917/section/7161623932439625758)。

  


同时，每条网格线都伴随着一个同名的 CSS 变量，该变量用于设置列网格轨道尺寸。在中间列（内容区域）之外，同一变量在 `-start` 之后和 `--end` 之前使用，以确保两侧的大小一致。继续使用 `full` 关键词为例，这些值是 `[full-start] var(--full)` 和 `var(--full) [full-end]`。

  


如此一来，[我们就可以在网格项目上使用 `grid-column` 属性将每个网格项目放置在指定的网格区域中](https://juejin.cn/book/7161370789680250917/section/7161623932439625758)：

  


```CSS
body {

    > *:not(.full-bleed) {
        grid-column: content;
    }
    
    .full-bleed {
        grid-column: full;
    }
}
```

  


简单的几行 CSS 代码就使 `.full-bleed` 能全屏显示：

  


```CSS
:root {
    --gap: clamp(1rem, 6vw, 3rem);
    --full: minmax(var(--gap), 1fr);
    --content: min(70rem, 100% - var(--gap) * 2);
}

body {
    display: grid;
    row-gap: var(--gap);
    grid-template-columns:
        [full-start] var(--full)
        [content-start] var(--content) [content-end]
        var(--full) [full-end];
    align-content: start;

    >*:not(.full-bleed) {
        grid-column: content;
    }

    .full-bleed {
        grid-column: full;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2c6ea44e0cb24b73ba6a5f1b29391998~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1324&h=748&s=17546636&e=gif&f=198&b=8e0e4a)

  


> Demo 地址：https://codepen.io/airen/full/VwowGqj

  


它的最大优势是不需要依赖任何[媒体查询（`@media`）](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)就可以构建一个具有[响应式的 Web 布局](https://juejin.cn/book/7161370789680250917/section/7165845190614188062)。而且，我们的 Web 页面在布局时，页面的内容不可能只是上面展示的这两种：`.wrapper` 具有最大宽度，水平居中；`.full-bleed` 全屏宽度。也有可能某个区域只需要左侧扩展到视窗左侧边缘，也有可能某个区域的内容要比 `.wrapper` 小一点，并且水平居中。等等：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/31e6750544a948f88b0357ba8c275388~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1503&s=437222&e=jpg&b=fefdfd)

  


上图中展示了同一个 Web 页面有四种不同的宽度。针对这种情景，我们采用 CSS Grid 布局，可以很容易做到，而且能真正做到随视窗大小缩放的布局。课程最后将会以这个为案例，向大家阐述如何利用 CSS Grid 布局来打破元素尺寸的限制！

  


## 使用容器查询来突破元素

  


之前所展示的案例几乎都是使用 `.full-bleed` 元素的宽度等于浏览器视窗的宽度，即全屏展示。它有可能真的要突破 `.wrapper` 容器的限制（当 `.full-bleed` 嵌套在 `.wrapper` 中）；它也有可能只是在视觉上突破 `.wrapper` 的限制（`.full-bleed` 和 `.wrapper` 是兄弟元素），其中 `.wrapper` 元素有一个最大宽度的限制，并水平居中，而 `.full-bleed` 则全屏显示。

  


然后，我们有很多时候，某个元素要突破其父容器宽度的限制，但又不是全屏显示。例如下图这个效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/af5f47576cc049ff90b4b8a6aede6ddd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=993&s=36824&e=avif&b=e6e5e1)

  


这是一个博客类网站的布局，一个简单的两栏布局。与平常所接到的两栏布局有所不同的是，位于主内容栏的某个元素或某几个元素它会突破主内栏宽度，并延伸到侧边栏，相当于跨越了整个宽度。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b3e7b9852dbb47fcbe2e2f2ba971ed12~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1335&s=477524&e=jpg&b=c7c7c4)

  


我原本想着使用之前所介绍的 CSS Grid 布局来实现，但这样会失去侧边栏布局的内在灵活性。我们需要一种能够保留灵活布局，同时又能提供扩展能力的方法。

  


我们从 HTML 结构开始：

  


```HTML
<section class="wrapper">
    <main>
        <!-- 博客内容栏 -->
        
        <div class="breakout">
            <iframe></iframe>
        </div>
    </main>
    <aside>
        <!-- 侧边栏内容在这里 -->
    </aside>
</section>
```

  


和之前的案例一样，我继续[使用 CSS Grid 对整个页面进行布局](https://juejin.cn/book/7161370789680250917/section/7161372229123440648)：

  


```CSS
.wrapper {
    --gutter: 1rem;
    --sidebar-width: 20rem;
    container-type: inline-size;
    display: grid;
    grid-template-columns: minmax(0, 1fr) var(--sidebar-width);
    padding: 0;

    @media only screen and (width <= 1024px) {
        grid-template-columns: auto;
    }

    main {
        padding-block: clamp(3rem, 2.4631rem + 2.6846vi, 5rem);
        padding-inline: var(--gutter);
        display: grid;
        gap: 2rem;
    }
    
    aside {
        padding-block: clamp(3rem, 2.4631rem + 2.6846vi, 5rem);
        padding-inline: var(--gutter);
        border-left: 1px dashed;

        @media only screen and (width <= 1024px) {
            border-left: none;
            border-top: 1px solid;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/48582ee3efb24921886c0a2992089b45~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1308&h=744&s=9836587&e=gif&f=114&b=090909)

  


现在的关键是，我们需要将 `.breakout` 元素突破 `main` 容器的尺寸，并且使它与 `.wrapper` 保持相等的宽度：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f9bcd7ba17ef4711b1a7a30bcbfbcd6f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1989&s=1095377&e=jpg&b=0b0b0b)

  


与全屏宽度的案例不一样，由于 `.breakout` 元素只是想与 `.wrapper` 等宽，我们无法直接使用 `100vw` 来设置该元素的 `width` 。而且又不能使用 `100%` 来计算它的宽度，因为它与 `.wrapper` 之间还隔了一个 `main` 元素。

  


庆幸的是，我们可以使用 [CSS 的容器查询单位 `cqi` 或 `cqw` ](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-2)来获取 `.wrapper` 容器宽度的百分比。[这些单位的使用与视窗单位类似](https://juejin.cn/book/7223230325122400288/section/7249357892611440700#heading-1)，只不过它表示容器的百分比。我们只需要将 `.wrapper` 声明为一个容器查询上下文就可以使用它们了：

  


```CSS
.wrapper {
    container-type: inline-size;
    
    main {
        padding-inline: var(--gutter);
    }
}

.breakout {
    width: calc(100cqi - 2 * var(--gutter));
    position: relative;
    z-index: 2;
} 
```

  


我们要重点关注的是上面代码中的 `100cqi` 。[由于我们在 `.wrapper` 元素上设置了 `container-type: inline-size` ](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)，`cqi` 单位是基于 `.wrapper` 元素内联尺寸（`inline-size` ，如果书写模式为 `ltr` 或 `rtl` ，它相当于元素的宽度 `width`）来计算，`100cqi` 等于 `.wrapper` 宽度的 `100%` 。但在这个示例中，由于 `main` 元素设置了内距，因此我们需要在 `100cqi` 的基础上减去 `main` 元素左右两侧的内距（`--gutter`）。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5089701138114bd8bc65d6e4d28535c8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2238&s=1547093&e=jpg&b=0d0d0d)

  


正如你看到的，`.breakout` 元素突破了 `main` 容器的限制，并且按照我们期望的效果扩展到侧边栏。但问题是，由于它的宽度变大了，将整个 `main` 容器也变大了。注意，`main` 的宽度是由 `.wrapper` 的 `grid-template-columns` 来决定的，在这里它对应的列宽是 `minmax(0, 1fr)` ，会自动扩展。

  


而我们期望的效果只是 `.breakout` 元素突破 `main` 容器的限制。[我们可以使用 CSS 的 `:not()` 选择器对非 `.breakout` 的元素设置一个 `max-width`](https://juejin.cn/book/7223230325122400288/section/7226251495276609569) 。需要知道的是，只有两栏布局时， `.breakout` 元素才会有突破 `main` 容器限制的需求。因此，我们加个 [CSS 媒体查询来对非 `.breakout` 元素做一个限制](https://juejin.cn/book/7223230325122400288/section/7257368158451793935)：

  


```CSS
main {
    @media only screen and (width >= 1024px) {
        > *:not(.breakout) {
            max-width: calc(
                100cqi - var(--sidebar-width) - 2 * var(--gutter) - 1rem
            );
        }
    }
}
```

  


现在你看到的效果，就是我们所期望的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7075852d92b141a287625194f4118e86~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1240&h=744&s=14073617&e=gif&f=155&b=0e0e0e)

  


> Demo 地址：https://codepen.io/airen/full/vYoYPLz

## 使用 `border-image` 突破背景

  


你可能已经留意到了，之前展示的案例中，页眉（`header`）都限制在 `.wrapper` 中：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6431e9d580242a48060f45ac88e6acd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=804861&e=jpg&b=0d0d0d)

  


很多时候，我们更希望只是 `header` 的内容限制在 `.wrapper` 内，而它的背景延伸到屏幕的两侧，使其在视觉上看上去是全屏宽度。

  


我们通常把这种效果称之为“突破”背景。以往的解决方案是调整 HTML 结构或改变布局方式来实现，比如之前所介绍的种种解决方案。

  


但是，我又不希望因为扩展元素背景来调整 HTML 结构，又不想调整网格布局。那怎么办呢？这就是我接下来要跟大家分享的另一个 CSS 技巧。我们使用 CSS 的 `border-image` 属性，一行代码就可以实现。

  


```CSS
header {
    border-image: conic-gradient(var(--c) 0 0) fill 0//0 100vw;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6a9018d3937c4ce8a88b1b8af2aa6ad0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=684145&e=jpg&b=0d0d0d)

  


> Demo 地址：https://codepen.io/airen/full/dyxyBYe

  


是不是很神奇！我想你一定想知道其中的奥秘。接下来，我来解释一下 `border-image` ，你就会豁然开朗！

  


首先来拆解代码中的 `border-image`:

  


```CSS
header {
    border-image: conic-gradient(var(--c) 0 0) fill 0//0 100vw;
}

/* 等同于  */
header {
    border-image-source:conic-gradient(var(--c) 0 0); /* 边框图片源 */
    border-image-slice: fill 0; 
    border-image-width: 1; /* 初始值为 1*/
    border-image-outset: 0 100vw; 
    border-image-repeat: stretch; /* 初始值为 stretch */
}
```

  


它是 CSS 中唯一支持双斜杠语法（`//`）的属性：

  


```CSS
.element {
    border-image: <border-image-source> <border-image-slice> / <border-image-width> / <border-image-outset> <border-image-repeat>;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8a0987d6389b4a5685d2951da1e3c741~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1642&h=816&s=7839108&e=gif&f=1378&b=fefdfd)

  


> URL: https://mdn.github.io/css-examples/tools/border-image-generator/

  


回到我们的示例当中来，我们是要给 `header` 添加一个纯色背景，并且“突破” 容器宽度的限制。`border-image` 与 `background` 和 `background-color` 属性不同，它不支持颜色值（`<color>`），所以我使用了一个可以生成纯色的渐变。为此，我在渐变中定义一个颜色（`--c`）和两个颜色停止点（`0 0`）。注意，`conic-gradient()` 函数的 `0` 和 `100%` 位于同一位置。当然，[你也可以使用任意你喜欢的渐变来绘制纯色图像](https://juejin.cn/book/7199571709102391328/section/7199845781149810727)。

  


有了图像源（`border-image-source`）之后，我们可以使用 `border-image-slice` 对图像边框进行切割，可以在元素的四个方向进行切割，使图像分成九宫格：四个角、四个边以及中心区域：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/80658f5536b4433bbdc424be07174aa8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=446739&e=jpg&b=fefcfc)

  


上图说明了每个区域的位置：

  


-   区域 `1 ~ 4` 为角区域，每一个都被用于组成最终边框图像的四个角
-   区域 `5 ~ 8` 边区域，在最终的边框图像中重复、缩放或修改它们以匹配元素的大小
-   区域 `9` 为中心区域，它在默认情况下会被丢弃，但如果设置了关键字 `fill`，则会将其用作元素的背景图像

  


注意，示例中的 `border-image-slice` 应用的是 `0 fill` ，同样会对图片源进行切割，只不过它距离四周的距离都是 `0` ，并且 `fill` 关键词会将中心区域填充整个 `header` 元素，用作背景图像。

  


要知道，我们并没有显式设置 `border-image-width` ，浏览器将会取其默认值 `1` ，它相当于是元素宽度的 `1` 倍。由于 `header` 元素并没有显式设置 `border` ，因此它的值为 `0` 。

  


不过，我们设置了 `border-image-outset` 属性的值为 `0 100vw` ，这使得 `border-image` 的区域从左侧和右侧扩展了 `100%` 的视窗宽度（`100vw`）。

  


从技术上讲，我们可以使用小于 `100vw` 的值，因为我们需要覆盖的空间可能很小，但使用更大的值总是好的，以确保我们考虑到所有可能性。而且 `border-image-outset` 有点类似于 `outline-offset` 不会触发滚动条，因此使用较大值也不用担心！

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df0f7d9025094a8d9cf575c56b9e6db7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1520&s=1048643&e=jpg&b=050505)

  


这就是 `border-image` 创建“突破”背景背后的原理，是不是没有你想象的那么复杂！

  


当然，我们并不仅限于纯色，因为我们正在使用渐变，如果需要，可以调整渐变，创建任何你需要的背景。例如：

  


```CSS
section {
    border-image: repeating-linear-gradient(45deg, var(--c) 0 20px, #0000 20px 40px) 0 fill // 0 100vw;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0592aeed526a445eb680e6295512cb1d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=1087806&e=jpg&b=ebe9e9)

  


> Demo 地址：https://codepen.io/airen/full/qBeENGJ

  


使用 `border-image` 属性除了可以使元素“突破”背景之外，还可以用来装饰标题，给它们添加华丽的边框。例如：

  


```CSS
.title {
    --b: 8px;
    --g: repeating-linear-gradient(45deg, #BD1550 0 10px, #E97F02 0 20px, #F8CA00 0 30px);
    border-image: var(--g) fill 0 / calc(100% - var(--b)) 0 0 / 0 100vw 0 0 repeat;
}
```

  


上面代码中的 `border-image-width` 定义了顶部区域的大小为 `calc(100% - var(--b))`，而其他区域的大小为 `0` 。这意味着我们将第五个区域的高度设置为 `calc(100% - var(--b))` ，因此中间区域的高高将只有 `--b` ，由于切片（`border-image-slice`）仍然为 `0` ，我们只有中间切片放置在那 `--b` 的空间内。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e2671a5f110c4472bc94b8ad18855c78~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=761&s=877517&e=jpg&b=fefefe)

  


我们也可以像下面这样使用 `border-image` :

  


```CSS
.title {
    --b: 8px;
    --g: repeating-linear-gradient(45deg, #BD1550 0 10px, #E97F02 0 20px, #F8CA00 0 30px);
    border-image: var(--g) 0 0 1 0 / 0 0 var(--b) 0 / 0 100vw 0 0;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ebe2635374df40948abfc39364cfba24~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=761&s=708198&e=jpg&b=fefefe)

  


这次，`border-image-slice` 定义了一个底切片等于 `1` ，这产生了两个切片，第七个（底部中心）和第九个（中心区域）。接着，`border-image-width` 将第七个区域的高度设置为 `--b` 。注意，这次没有使用 `fill` 关键词，因此中间区域没有像上次好样被填充。相反，我们只填充第七个区域，它占据了 `border-image` 的 `100%` 面积和 `--b` 的高度。

  


这是两种不同使用 `border-image` 的方式，但最终呈现的结果是一致的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3f8ca004a4444ee6a99e0befe3b232b5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1268&s=855415&e=jpg&b=050505)

## 案例：使用 CSS Grid 创建流体分离布局

  


最后，我们来看一个综合性的案例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e8932c0d1aab448483c7b6f78a5fdc49~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1503&s=437222&e=jpg&b=fefdfd)

  


上图是我们即将要构建的 Web 布局。在这个页面上，我们有四种可能的宽度，它们分别对应着不同的区块：

  


-   全屏宽度 （Full Bleed）
-   特征（Feature）
-   弹出（Popout）
-   内容（Content）

  


此外，还有一些特殊的“侧边栏固定”元素，它们“粘附”在屏幕的一侧。

  


我们从 HTML 结构开始：

  


```HTML
<body>
    <header>Full</header>
    <div class="full-bleed">Full Bleed</div>
    <div class="content">Content</div>
    <div class="feature">Feature</div>
    <div class="side-anchor">Side-Anch</div>
    <div class="popout">Popout</div>
    <div class="content">Content</div>
    <footer>Full</footer>
</body>
```

  


与之前的案例一样，我们可以为每种类型的宽度存储在 CSS 变量中：

  


```CSS
:root {
    --gap: clamp(1rem, 4vw, 2rem);
    --full: minmax(var(--gap), 1fr);
    --feature: minmax(0, 12vw);
    --popout: minmax(0, 2rem);
    --content: min(clamp(30rem, 52vw, 60rem), 100% - var(--gap) * 2);
}
```

  


简单的解释一下这些变量。

  


-   `--gap` ：用来设置侧边距，它的大小被设置在 `1rem ~ 2rem` 之间，首先宽度是 `4vw`
-   `--full`：它的值最小不会小于 `--gap` 值，而 `1fr` 会根据网格容器的空间进行扩展或收缩。它对应的是 `full` 网格区域，将会横跨整个浏览器视窗宽度，并且它将作为我们的边距，因此将 `--gap` 作为最小值
-   `--feature` 和 `--popout` 都设置了最小值为 `0` 。这就是让我们的列在没有空间时“消失”的原因！当网格的其他区域扩展时，这些列会在没有足够空间时收缩，基本上不占任何空间
-   `--content` 相对来说比较复杂一些，它会在 `min()` 函数中的两个值中作比较，选其中更小的值。这两个值分别是 `clamp(30rem, 52vw, 60rem)` 和 `100% - var(--gap) * 2` ，前者的值位于 `30rem ~ 60rem` 之间，但首选是 `52vw` （你需要知道 `clamp()` 函数的工作原理），后者很容易理解，它就是视窗 `100%` 宽度减去两侧的间距，即 `2 * var(--gap)`

  


其中 `--full` 、`--feature` 、`--popout` 和 `--content` 将会用来定义网格列轨道的尺寸。

  


```CSS
body {
    display: grid;
    grid-template-columns:
        [full-start] var(--full)
        [feature-start] var(--feature)
        [popout-start] var(--popout)
        [content-start] var(--content) [content-end]
        var(--popout) [popout-end]
        var(--feature) [feature-end]
        var(--full) [full-end];
}
```

  


初次接触 `grid-template-columns` 属性的开发者可能会认为它很复杂，其实很简单，其中`[*-start]` 和 `[*-end]` 用于给网格线命名，在 `[*-start]` 之后和位于 `[*-end]` 之前的变量是用来定义列网格轨道的尺寸。并且 `[*-start]` 和 `[*-end]` 组合在一起，相当于创建了一个网格区域，并且该网格区域的名称是 `*` 对应的关键词。例如 `[full-start]` 和 `[full-end]` 之间的网格区域，对应的名称是 `full` 。我想下图更易于帮助你理解 `grid-template-columns` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6460330d7786429194805e579c601c68~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=3129&s=887330&e=jpg&b=fcf7f7)

  


正如你所看到的。我们已经用自定义标识符命名了列并指定了列开始和结束的网格线名称，因此我们可以在网格项目中使用 `grid-column` 属性根据网格线名称来放置网格项目：

  


```CSS
.header {
    grid-column: full;
}

.full-bleed {
    grid-column: full;
}

.feature {
    grid-column: feature;
}

.popout {
    grid-column: popout;
}

.content {
    grid-column: content;
}

.side-anchor {
    grid-column: full-start / feature-end;
}

footer {
    grid-column: full;
}
```

  


如果我们想为网格项目创建一个默认分配，可以这样创建：

  


```CSS
body > * {
    grid-column: content;
}
```

  


这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d5d4bc1f9524fac82503d4cd73e983b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1208&h=772&s=17140664&e=gif&f=146&b=ebe9b0)

  


当浏览器视窗宽度变小时（网格容器变小），`--feature` 和 `--popout` 就会自动“消失”，因为网格容器没有足够空间时，`minmax()` 函数取更小的值 `0` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/db36470fb163478e80effef3b7aa81bc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=998&s=991596&e=jpg&b=0e0e0e)

  


现在，我们回过头来看页眉（`header`）和页脚（`footer`）两个部分。它们和 `.full-bleed` 元素一样，具有全屏宽度，但我们实际上希望其内容宽度是与 `--feature` 相等。如果我们直接将这两个元素的 `grid-column` 调整为 `feature` 时，它们内容的宽度是与 `--feature` 相同，但你会发现它们的背景没有扩展到浏览器视窗的两侧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9f61a04d5ab4410494b364db8f5e8f5a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1242&h=756&s=13312101&e=gif&f=82&b=161515)

  


这个时候，我们可以使用 `border-image` 使 `header` 和 `footer` 元素突破背景，即背景延伸到浏览器视窗的两侧：

  


```CSS
:is(header, footer) {
    grid-column: feature;
    border-image: conic-gradient(var(--c) 0 0) 0 fill//0 100vw;
}
```

  


此时，你看到的效果如下图所示，`header` 和 `footer` 内容宽度等于 `--feature` ，同时背景也会扩展到浏览器视窗的两侧：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/29770f81d3234793b4d4dd3b7b58f70a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1260&h=816&s=9849106&e=gif&f=75&b=342a34)

  


> 温馨提示，如果你熟悉 [CSS Grid 布局的 `subgrid` 特性](https://juejin.cn/book/7161370789680250917/section/7160657953932967967)（子网格），那么你也可以使用子网格来实现。只不过你需要调整 `header` 和 `footer` 的结构。要是你感兴趣，不妨尝试使用子网格来实现！

  


还剩下最后一个问题。目前， `.side-anchor` 元素左侧会一直延伸到浏览器左侧边缘。我们期望只是其背景向左侧扩展，而不是内容也向左移动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/91677f074f984ffb9679ce4beb436eb2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=998&s=292993&e=jpg&b=f2cdf6)

  


我们同样可以使用 `border-image` 属性，只让元素左侧向左扩展：

  


```CSS
.side-anchor {
    grid-column: feature;
    border-image: conic-gradient(var(--c) 0 0) 0 fill/ / 0 0 0 100vw;
    clip-path: inset(0 0 0 -9999px round 0 100px 0 0);
}
```

  


注意，`border-image` 会使元素的 `border-radius` 消失，[在这里使用 `clip-path` 的 `inset()` 来模拟一个 border-radius 的效果](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-3)。

  


现在，整个效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5627cd7cc4fa45b89ed1f0aa5c9bd658~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=584&s=19256420&e=gif&f=188&b=050133)

  


> Demo 地址：https://codepen.io/airen/full/XWvJNxX

  


这就是一个不依赖媒体查询的流动突破布局。整个示例的完整代码如下：

  


```CSS
@layer breakout {
    :root {
        --gap: clamp(1rem, 4vw, 2rem);
        --full: minmax(var(--gap), 1fr);
        --feature: minmax(0, 12vw);
        --popout: minmax(0, 2rem);
        --content: min(clamp(30rem, 52vw, 60rem), 100% - var(--gap) * 2);
    }
    
    body {
        display: grid;
        grid-template-columns:
            [full-start] var(--full)
            [feature-start] var(--feature)
            [popout-start] var(--popout)
            [content-start] var(--content) [content-end]
            var(--popout) [popout-end]
            var(--feature) [feature-end]
            var(--full) [full-end];
    
        > * {
            grid-column: content;
        }
    }
    
    .full-bleed {
        grid-column: full;
    }
    
    .content {
        grid-column: content;
    }
  
    .feature {
        grid-column: feature;
    }
  
    .popout {
        grid-column: popout;
    }
  
    :is(header, footer) {
        grid-column: feature;
        border-image: conic-gradient(var(--c) 0 0) 0 fill//0 100vw;
    }
  
    .side-anchor {
        grid-column: feature;
        border-image: conic-gradient(var(--c) 0 0) 0 fill/ / 0 0 0 100vw;
        clip-path: inset(0 0 0 -9999px round 0 100px 0 0);
    }
}
```
