![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5817b529036f4041bc683282b600a84b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=900&h=465&s=4176&e=png&b=222222)

  


Web 布局中的水平居中或水平垂直居中是一个亘古不变的话题，CSS 中有很多种方式可以实现，例如 [Flexbox](https://juejin.cn/book/7161370789680250917/section/7161623855054716935#heading-0) 和 [Grid](https://juejin.cn/book/7161370789680250917/section/7161624078397210638#heading-0) 布局技术就有多种不同的方式。虽然水平居中布局是一个经典布局模式，但有些情景中，要实现水平居中布局还是很棘手的。例如，一组元素以三列一排的方式排列，当剩余的元素数量不足三列时，它们会水平居中对齐：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/08f9d855d4b644458efa5a9cd937a2d2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1251&s=614688&e=jpg&b=fbfaf9)

  


实现这种布局的方法有很多，但要使这个布局更灵活并不是一个简单的过程，比如：

  


-   元素宽度可灵活调整：每个元素的宽度能够适应不同情况
-   最后一行居中对齐：即使元素数量不足一整行，最后一行的元素也要水平居中
-   统一宽度：所有元素的宽度需要保持一致
-   统一高度：同一行内的所有元素需要具有相同的高度（等高列）
-   列数自适应：列数应该根据屏幕宽度自动变化，无需手动调整
-   适配任意数量的元素：解决方案应该支持任意数量的元素，不会因为元素数量变化而失效
-   元素数之间有间距：每个元素之间需要有间隙
-   占满容器宽度：所有元素应该使用父容器的全部宽度，无多余留白
-   易于编辑：布局应该便于复制粘贴，以便快速应用到其他项目中

  


另外，要是布局反过来，元素数量不足一整行时，水平居中发生在第一行而不是最后一行：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0472ac271fb94a878b4bbe854c51c731~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1251&s=620940&e=jpg&b=fbfaf9)

## 容器水平居中

  


首先我们将容器 `.container` 元素水平居中。CSS 有很多方法可以将它水平居中，比如：

  


```CSS
body {
    padding-inline: 1rem;
}

.container {
    max-width: 1140px;
    margin-inline: auto;
}
```

  


我们给容器设置了一个最大宽度，当浏览器视窗宽度大于 `1140px` 时，`.container` 容器会在浏览器视窗中水平居中，反之它将填满其父容器 `body` （这里是 `100% - 2rem` ，使容器与浏览器视窗两侧有一个 `1rem` 的留白）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3a104e843db3462dba09d3d2e2c88909~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1398&h=670&s=5681932&e=gif&f=147&b=f7f7f4)

  


当然，如果你有需要，我们可以将 `1140px` 存储到一个 [CSS 变量](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)中，比如 `--w` ，然后使用[媒体查询 `@media`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935) 来调整 `--w` 的值，使不同断点下，`.container` 元素拥有不同的 `max-width` 值：

  


```CSS
.container {
    max-width: var(--w);
    margin-inline: auto;
    outline: 1px dashed;
    
    @meida (width >= 576px) {
        --w: 540px;
    }
    
    @media (width >= 768px) {
        --w: 720px;
    }
    
    @media (width >= 992px) {
        --w: 960px;
    }
    
    @media (width >= 1200px) {
        --w: 1140px;
    }
    
    @media (width >= 1400px) {
        --w: 1280px;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f031de928a154d46bd6b2c62e19448b7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1672&h=684&s=8907854&e=gif&f=219&b=f7f7f4)

  


上面我们用了两行代码 `max-width: var(--w)` 和 `margin-inline` 使 `.container` 容器在浏览器视窗中水平居中。其实，我们借助[现代 CSS 的比较函数 `max()` 特性](https://juejin.cn/book/7223230325122400288/section/7241401565653762108)，只需要一行 CSS 就可以使 `.container` 容器达到相同的效果：

  


```CSS
.container {
    margin-inline: max(0px,50% - var(--w)/2);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/436d65a483224eb3815da7883a0e9dcd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1670&h=682&s=7881331&e=gif&f=177&b=f7f7f4)

  


你还可以通过将 `max()` 函数中的 `0px` 替换为任意值，例如 `1rem`，来为小屏幕设置最小的边距。如此一来，就不需要在 `body` 中设置 `padding-inline` 为小屏幕设置最小的边距：

  


```CSS
.container {
    margin-inline: max(1rem,50% - var(--w, 100%)/2);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3092c73282b84a299aad5d314f745e9b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1674&h=684&s=7192667&e=gif&f=208&b=f7f7f4)

  


这是 CSS 中的一个小技巧，我用下图来解释其背后的工作原理：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/af74135b5e8946aabd0fc5f214fada74~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2528&s=1069330&e=jpg&b=607d8b)

  


简单地说，`max() 函数`会取列表参数中更大的值作为 `margin-inline` 属性的值：

  


-   当 `50% - var(--w) / 2` 的值大于 `1rem` 时，`margin-inline` 属性的值为 `50% - var(--w) / 2`
-   当 `50% - var(--w) / 2` 的值小于 `1rem` 时，`margin-inline` 属性的值为 `1rem`

  


> 温馨提示，`margin-inline` 是 CSS 的逻辑属性，当书写模式为 `ltr` 或 `rtl` 时，它等于同 `margin-left` 和 `margin-right` 。如果你对逻辑属性感兴趣话，可以阅读《[CSS 逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)》！

  


## 如何使用卡片换行

  


接下来，我们要让卡片换行排列。在 CSS 中，[Flexbox 和 Grid 都可以使卡片换行排列](https://juejin.cn/book/7199571709102391328/section/7199571708838150155)。不过，使用 Flexbox 需要涉及更多的计算。例如：

  


```CSS
.cards {
    --g: 1.5rem;
    display: flex;
    flex-wrap: wrap;
    gap: var(--g);
    justify-content: center;
    
    .card {
        flex: 0 1 calc((100% - 2 * var(--g)) / 3);
    }
}
```

  


将 `flex-wrap` 设置为 `wrap` 时，Flexbox 容器没有足够空间放置 Flex 项目时，会自动断行。而且，在这个示例中，我们根据列数（一行有三列）和间距 `--g` 计算出每个 Flex 项目的初始宽度（即 `flex-basis` 的值），并且将 `flex-grow` 设置为 `0` （禁止 Flex 项目扩展宽度），将 `flex-shrink` 设置为 `1` （允许 Flex 项目收缩宽度）。这个时候，你将看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3e64a5db181946e6b3a577dfddfa8a36~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1560&s=825982&e=jpg&b=f6f6f2)

  


看上去似乎不错，与我们期望的效果几乎相似。为什么说几乎相似呢？那是因为，目前这种方案不够灵活，当我们缩小浏览器视窗宽度时，你将会看到 Flex 项目无法根据 Flexbox 容器的剩余空间扩展 Flex 项目宽度，因为我们设置了 `flex-grow` 的值为 `0` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ea1591cd28546bc803368e28ecff4d0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=850&s=6169748&e=gif&f=77&b=f5f5f0)

  


你可能会说，那将 `flex-grow` 属性设置为 `1` 不就可以了吗？是的，这样做会扩展 Flex 项目的宽度，但又将引发另一个问题，最后一行的 Flex 项目也会扩展宽度，并且填满整个 Flexbox 容器：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/59b0e3b3e458408bac5f12ff147ebb1c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1408&h=848&s=16691875&e=gif&f=143&b=f6f6f2)

  


这离我们所期望的效果就越来越远了。我们希望的是卡片宽度能自适应容器宽度，但末尾一行需要居中对齐。

  


温馨提示，Flex 项目的尺寸计算是非常复杂的，如果你想深入了解和掌握这方面的知识，提高自己使用 Flexbox 构建 Web 布局的能力，那么个人强烈推荐你阅读下面这几节课的内容：

  


-   [Flexbox 布局中的 `flex` 属性的基础运用](https://juejin.cn/book/7161370789680250917/section/7161623755972673550)
-   [Flexbox 中的计算：通过扩展因子比例来扩展 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7161623797794078750)
-   [Flexbox 中的计算：通过收缩因子比例收缩 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7164357320367931399)
-   [Flexbox 布局中的 `flex-basis`：谁能决定 Flex 项目的大小？](https://juejin.cn/book/7161370789680250917/section/7161623717074698247)

  


通过上面示例的演示，你可能已经发现了 Flexbox 布局并不太适合我们这个场景。既然如此，我们就得考虑使用其他的布局方案。比如，CSS Grid 布局方案。

  


在 CSS Grid 中有一种叫作 RAM 布局技巧，即 [`repeat()` 、`auto-fit` 或 `auto-fill` 和 `minmax()` 相结合的布局技巧](https://juejin.cn/book/7161370789680250917/section/7161624041885958151)。它可以轻易使卡片换行，并且自适应网格容器的可用空间，并且不需要对 Grid 项目进行任何的计算。其代码大致如下：

  


```CSS
.cards {
    --g: 1.5rem;
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(min(100% - 2rem, 400px), 1fr));
    gap: var(--g);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9c4c2e97ede4b0aa52035bbf6de39cd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1378&h=850&s=7914883&e=gif&f=90&b=f5f5f1)

  


卡片是可以自适应容器的宽度，但带来了另一个问题，最后一行的卡片无法水平居中：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ee8c94ca423443ca6758a7c1cab06b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1050&s=619671&e=jpg&b=607d8b)

## 如何在 CSS Grid 中让最后一行元素居中

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0064b11bbd2c4ffd9cd4622c642fd288~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1251&s=614688&e=jpg&b=fbfaf9)

  


  


过去，我们常常会借助一些 CSS Hack 手段来解决问题，甚至是妥协于不完全符合所有标准的解决方案。然而，随着 CSS 的不断发展，现在完全可以实现这样的布局效果，并且不需要依赖任何 Hack 手段。

  


正如之前所述，CSS Grid 的 RAM 布局技巧可以满足我们所有要求，除了无法让最后一行的元素居中。换句话说，如果能解决这个问题，那么我们就实现了所期望的 Web 布局效果。接下来，我们接着使用 CSS Grid 来构建布局，但不是使用 RAM 布局，因为 RAM 的优点在于它们可以提供响应式布局，而无需使用媒体查询——你的网格项目会根据容器的大小自动响应，但它无法让最后一行的元素居中。

  


庆幸的是，我们可以使用 CSS Grid 为每个元素（卡片）分配两个单元格，然后将最后一行的元素移动到期望的位置。

  


我们先来简单的分析一下，我们即将要创建的布局效果，居中元素将会发生在以下两个场景：

  


-   两列布局时，最后一行会有一个居中元素
-   三列布局时，最后一行会有两个居中元素

  


先来看两列布局。我们可以在网格容器上使用 `grid-template-columns` 定义一个四列网格，因为每个卡片将会分配两个单元格：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 4;
    display: grid;
    grid-template-columns: repeat(var(--cols), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column: span 2;
    }
}
```

  


这个时候，你将看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f1021a62ecd34fe3b19b80a0af3d5c78~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2796&h=1798&s=459960&e=jpg&b=66808b)

  


我们要做的是，使用 CSS 的结构性选择器来选中最后一行的单个元素（该元素同时是奇数位置的元素）。注意，“奇数”可以表示为 `2` 的位置再加 `1` ，即 `2n + 1` 。然后，我们让这个元素结束于第 `4` 条列网格线：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 4;
    display: grid;
    grid-template-columns: repeat(var(--cols), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column: span 2;
        
        &:last-child:nth-child(2n + 1) {
            grid-column-end: 4;
        }
    }
}
```

  


请注意上面代码中的 `:last-child:nth-child(2n + 1)` 选择器，它可以使你根据数量来进行查询，然后匹配到你所需要选择的元素。这是 CSS 选择器的一个小技巧，被称为CSS 的**数量查询** （也称为**范围选择器** ）。我曾在介绍 CSS 的 `@` 规则和 CSS 的 `:has()` 选择器时，阐述过这个技巧，如果你想深入了解这方面的知识，你可以阅读下面两节课的内容：

  


-   [条件 CSS 之 `@` 规则和 CSS 选择器](https://juejin.cn/book/7199571709102391328/section/7199845944760729632)
-   [CSS 的父选择器：`:has()`](https://juejin.cn/book/7223230325122400288/section/7224404685615005728#heading-9)

  


上面代码会使最后一行的元素居中：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/92430b57eb1443b49174b0b24af0357d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2808&h=1802&s=443535&e=jpg&b=667e8a)

  


我们再来看三列布局，它的情况稍微复杂一些：

  


-   最后一行只有一个元素时：选择最后一个元素，同时该元素的位置符合序列 `4` 、`7` 、`10` 、`13` 等（即 `3` 的位置再加上 `1` ：`3n + 1`）。然后让它结束于第五条列网格线
-   最后一行有两个元素时：选择倒数第二个元素，同时该元素的位置符合序列 `4` 、`7` 、`10` 、`13` 等（同样是 `3n + 1`）。然后让它结束于第四条列网格线

  


具体的 CSS 代码如下：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 6;
    display: grid;
    grid-template-columns: repeat(var(--cols), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column: span 2;
        
        /* 最后一行只有一个元素 */
        &:last-child:nth-child(3n + 1) {
            grid-column-end: 5;
        }
        
        /* 最后一行有两个元素 */
        &:nth-last-child(2):nth-child(3n + 1) {
            grid-column-end: 4;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b8d874267c44c49989fa3ed2bc45990~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1346&h=888&s=6146387&e=gif&f=159&b=557080)

  


现在我们已经完成了 `2` 列和 `3` 列的布局，其原理已经非常清楚了。基于该工作原理，我们还可以实现 `4` 列，`5` 列和 `6` 列布局，如果有需要，你可以继续扩展下去。

  


工作原理很简单，但问题是，网格的列数如何确定呢？好比我们这个案例，浏览器视窗宽度不同时，网格容器（`.cards`）的列数也会随之变化。

  


其中一个传统的做法是，使用媒体查询 `@meida` 或容器查询 `@container` （如果有需要）定义不同的网格，并且在相应的断点下调整最后一行元素的位置：

  


```CSS
.cards {
    --g: 1.5rem;

    display: grid;
    grid-template-columns: repeat(var(--cols), minmax(0, 1fr));
    gap: var(--g);

    .card {
        grid-column: span 2;
    }
}

.container {
    margin-inline: max(1rem, 50% - var(--w, 100%) / 2);

    @meida (width >= 576px) {
        --w: 540px;
        --cols: 2;
    }

    @media (width >= 768px) {
        --w: 720px;
        --cols: 4;

        .card {
            &:last-child:nth-child(2n + 1) {
                grid-column-end: 4;
            }
        }
    }

    @media (width >= 992px) {
        --w: 960px;
        --cols: 6;
      
        .card {
            &:last-child:nth-child(2n + 1) {
                grid-column-end: unset;
            }
    
            &:nth-last-child(2):nth-child(3n + 1) {
                grid-column-end: 4;
            }
    
            &:last-child:nth-child(3n + 1) {
                grid-column-end: 5;
            }
        }
    }

    @media (width >= 1200px) {
        --w: 1140px;
    }

    @media (width >= 1400px) {
        --w: 1280px;
    }
}
```

事实上，我们可以反过来思考。

  


为了确定列数，我们需要定义卡片元素的宽度。请记住，我们的目标是实现一个允许元素宽度可变的布局。因此，我们从定义两个变量开始：

  


-   **元素宽度：** **`--s`**
-   **网格间隙：** **`--g`**

  


```CSS
.cards {
    --s: 300px;
    --g: 1.5rem;
}
```

  


与之前一样，使每个元素占用网格中的两列：

  


```CSS
.cards {
    --s: 300px;
    --g: 1.5rem;
    
    .card {
        grid-column: span 2;
    }
}
```

  


最为关键是定义网格：

  


```CSS
.cards {
    --s: 300px;
    --g: 1.5rem;
    
    display: grid;
    gap: var(--g);
    
    grid-template-columns: repeat(
        auto-fit,
        minmax(
            min(
                (var(--s) - var(--g)) / 2,
                (100% - var(--g)) / 2
            ),
            1fr
        )
    );
    
    container: grid-container / inline-size;
    
    max-width: calc(var(--s) * 6 + var(--g) * 5);
    
    .card {
        grid-column: span 2;
    }
}
```

  


你可能已经发现了，上面的代码使用了 RAM 布局。你可能会有疑惑了，刚才不是说，RAM 布局无法使最后一行的元素居中吗？怎么又使用 RAM 呢？随着你往下阅读，你会找到答案的。

  


在继续往下之前，我们来解释一下上面代码中的列宽度的计算。`minmax()` 函数中有两个值，其中第一个值是最小宽度，第二个值是最大宽度。最大宽度比较好理解，它是 `fr` ，它表示网格容器剩余空间的一个“分数”，它会根据网格容器的可用空间自动调整，有关于这方面的介绍，可以阅读《[Grid 布局中的计算](https://juejin.cn/book/7161370789680250917/section/7161624007702216735)》一课！

  


我们把重点放在网格列的最小宽度计算上。我们知道，卡片的最小宽度是 `--s` ，而且每张卡片占用两列加上一个列间距 `--g` ，即：

  


```
列宽 + 间距 + 列宽 = 卡片宽度
```

  


如此一来，我们就可以计算列宽度：

  


```
列宽 = (卡片宽度 - 列间距) ÷ 2 = (var(--s) - var(--g)) / 2 
```

  


为了在极窄屏幕（如手机）上，卡片元素的宽度需要是容器的 `100%`，以避免水平滚动。在这个环境之下，列的宽度是 `(100% - var(--g)) / 2` 。

  


我们把这两个公式放到 `min() 函数`中，它将会根据具体的环境自动选择较小的值作为列的最小宽度。将它们组合起来，就是 `grid-template-columns` 属性的值：

  


```CSS
.cards {
    grid-template-columns: repeat(
        auto-fit,
        minmax(min((var(--s) - var(--g)) / 2, (100% - var(--g)) / 2), 1fr);
    )
}
```

  


通过这种方法，我们不仅能够灵活地定义列数，还能确保布局适配不同的屏幕尺寸，实现更高的可用性和可移植性。

  


到目前为止，浏览器呈现的效果与之前所介绍的 RAM 布局结果是相同的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e545fe1f88564a73927eb50a674a67f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2846&h=1800&s=820540&e=jpg&b=eef0f7)

  


接下来，就是如何将使最后一行元素水平居中了。

  


你是否发现了，上面代码中有一个 `container` 属性，即 `container: grid-container / inline-size` 。这是 **[CSS 的容器查询特性](https://juejin.cn/book/7223230325122400288/section/7259668032165773368)**，`container` 将告诉浏览器，`.cards` 容器具备一个容器查询的上下文。简单地说，现在可以使用 `@container` 可以查询容器的 `inline-size` ，实现动态确定列数的效果。

  


同样的，我们从两列布局开始：

  


```CSS
.cards {
    /* CSS... */
    container: grid-container / inline-size;
    
    
    .card {
        grid-column: span 2;
        
        .card {
            grid-column: span 2;
    
            @container grid-container  (width >= calc(300px * 2 + 1.5rem)) {
                &:last-child:nth-child(2n + 1) {
                    grid-column-end: 4;
                }
            }
            
            @container grid-container  (width >= calc(300px * 3 + 1.5rem * 2)) {
                &:last-child:nth-child(2n + 1) {
                    grid-column-end: unset;
                }
                
                &:last-child:nth-child(3n + 1) {
                    grid-column-end: 5;
                }
                
                &:nth-last-child(2):nth-child(3n + 1) {
                    grid-column-end: 4;
                }
            }
        }
    }
}
```

  


遗憾的是，CSS 变量无法直接在媒体查询和容器查询中使用。因此我们使用具体的值来替代 CSS 变量。为了方便，我在容器查询中保留了列宽度计算的公式，比如 `calc(300px * 2 + 1.5rem))` 。最终的 CSS 代码如下：

  


```CSS
.cards {
    --s: 300px;
    --g: 1.5rem;

    display: grid;
    gap: var(--g);

    grid-template-columns: repeat(
        auto-fit,
        minmax(min((var(--s) - var(--g)) / 2, (100% - var(--g)) / 2), 1fr)
    );

    container: grid-container / inline-size;

    max-width: calc(var(--s) * 6 + var(--g) * 5);

    .card {
        grid-column: span 2;

        @container grid-container  (width >= calc(300px * 2 + 1.5rem)) {
            &:last-child:nth-child(2n + 1) {
                grid-column-end: 4;
            }
        }
        
        @container grid-container  (width >= calc(300px * 3 + 1.5rem * 2)) {
            &:last-child:nth-child(2n + 1) {
                grid-column-end: unset;
            }
            
            &:last-child:nth-child(3n + 1) {
                grid-column-end: 5;
            }
            
            &:nth-last-child(2):nth-child(3n + 1) {
                grid-column-end: 4;
            }
        }
    }
}

.container {
    margin-inline: max(1rem, 50% - var(--w, 100%) / 2);

    @meida (width >= 576px) {
        --w: 540px;
    }

    @media (width >= 768px) {
        --w: 720px;
    }

    @media (width >= 992px) {
        --w: 960px;
    }

    @media (width >= 1200px) {
        --w: 1140px;
    }

    @media (width >= 1400px) {
        --w: 1280px;
    }
}
```

## 如何在 CSS Grid 中让第一行元素水平居中

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d897803b1f14ca1b0fc64f45aca01fc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1251&s=620940&e=jpg&b=fbfaf9)

  


现在，我们要把布局效果反过来，当网格一行只有两列时，第一个元素在第一行中水平居中；当网格一行有三列时，第一个和第二个元素在第一行中水平居中。

  


原理是相通的，只不过方向反过来了。首先，我们以两例为例，使用 `grid-template-columns` 创建一个四列网格。注意，因为我们每张卡片占据两列列宽再加上一个列间距：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 4;
    display: grid;
    grid-template-columns: repeat(var(--cols), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column-end: span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f306d50cb3e94b34a067fff47234255d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2826&h=1800&s=427565&e=jpg&b=68828b)

  


注意，`grid-template-columns` 的重复次数是视觉上每行列数的两倍。每个网格项目会跨越两列显示，而不是只占一列。为了使网格的列数能与视觉上的列数相匹配，我们在定义网格的时候，可以像下面这样来设置：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 2;
    display: grid;
    grid-template-columns: repeat(calc(var(--cols) * 2), minmax(0, 1fr));
    gap: var(--g);
}   
```

  


最后，我们要移动第一个网格项目的列起始位置：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 2;
    display: grid;
    grid-template-columns: repeat(calc(var(--cols) * 2), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column-end: span 2;
        
        &:first-child {
            grid-column-start: 2;
        }
    }
} 
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/04de0db411824d0290167134275a8e7f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2826&h=1786&s=413080&e=jpg&b=687f8c)

  


接下来，我们来看第两种情景，就是每行有三列。我们将 `--cols` 调整为 `3` ：

  


```CSS
.cards {
    --g: 1.5rem;
    --cols: 3;
    display: grid;
    grid-template-columns: repeat(calc(var(--cols) * 2), minmax(0, 1fr));
    gap: var(--g);
    
    .card {
        grid-column-end: span 2;
        
        &:first-child {
            grid-column-start: 2;
        }
    }
} 
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7fa78676421a4eea979e996c2532ca59~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2838&h=1794&s=577120&e=jpg&b=66808b)

  


怎么样，是不是很完美！

  


最后，我们使用 CSS 的媒体查询在不同断点的时候，调整 `--cols` 的值：

  


```CSS
.cards {
    --g: 1.5rem;
    display: grid;
    grid-template-columns: repeat(calc(var(--cols, 1) * 2), minmax(0, 1fr));
    gap: var(--g);

    .card {
        grid-column-end: span 2;

        &:first-child {
            grid-column-start: 2;
        }
    }
}

@media (width <= 768px) {
    .cards .card:first-child {
        grid-column-start: 1;
    }
}

.container {
    margin-inline: max(1rem, 50% - var(--w, 100%) / 2);

    @meida (width >= 576px) {
        --w: 540px;
        --cols: 1;
    }

    @media (width >= 768px) {
        --w: 720px;
        --cols: 2;
    }

    @media (width >= 992px) {
        --w: 960px;
        --cols: 3;
    }

    @media (width >= 1200px) {
        --w: 1140px;
    }

    @media (width >= 1400px) {
        --w: 1280px;
    }
}
```

  


注意，当一行只有一列卡片排列的时候，需要将第一张卡片的 `grid-column-start` 重置回 `1` 。最终你将看到的效果如下：
