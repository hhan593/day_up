OTP （One Time Password，即一次性密码）输入框是许多需要用户验证身份的 Web 应用中的常见功能。它在网上银行、电子商务网站和视交媒体平台中尤为常见。OTP 是为每个交易生成的唯一代码，具有有限的有效期，通常在几分钟后过期。这确保了代码无法被重复使用或被黑客拦截。

  


随着在线安全问题日益受到关注，越来越多的 Web 开发人员将 OTP 输入框集成到他们的应用中，以保护用户的敏感信息。这节课我们来一起来聊聊如何创建 OTP 输入框！

## OTP 简介

  


[维基百科是这样描述的](https://en.wikipedia.org/wiki/One-time_password)：

  


> 一次性密码（OTP），也称为一次性 PIN，一次性授权码（OTAC）或动态密码，是仅在一次登录会话或交易中有效的密码，适用于计算机系统或其他数字设备。

  


简单地说，OTP 是一种增强用户身份验证的有效工具，广泛应用于互联网银行、电商网站、社交平台等场合，尤其是在处理敏感操作时。然而，它也有一些局限性，特别是在用户体验方面。因此，设计良好的 OTP 输入框和认证流程非常重要，以减少用户的困扰，同时提供高水平的安全保护。

  


你知道那种一次性密码（OTP）输入框吗？通常 UI 是 `4` 个或 `6` 个数字，每个数字都有独立的输入框。比如：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2cb55c99018d40ef9002221968ecc5b6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1920&h=1329&s=340993&e=png&b=e29343)

  


每个数字都有一个单独的输入框模式，本应让事情变得更清晰，但实际上并没有。比如，我们从邮件或短信中复制的码，可以粘贴吗？应该粘贴到哪里？粘贴的内容会自动填充到所有的小框里吗？有时候代码中间有一个破折号，那也要包含进去吗？

  


要做好这个设计真的非常复杂，考虑到用户在与这些小框（单个输入框）互动之前可能会产生的困惑。一旦你开始与这些框互动时，体验必须非常便捷。

  


虽然 HTML 为我们提供了很多不同类型的输入框，Web 开发者在使用这些输入框时，可以根据具体的需求指定 `<input>` 元素的 `type` 属性，可以为用户提供友好的体验方式

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fd9dd3019e074931a45d838515ca6f24~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1076&h=1280&s=624014&e=png&b=131517)

  


甚至统一使用 `type` 为 `text` 的 `<input>` ，只是设置不同的 `inputmode` 值，也可以用来改善用户体验，唤起不同方式软键盘，给用户提供最合适的输入方式。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7263e4381ada4f83a1c5c38aaa443b5b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2946&h=5964&s=3363975&e=png&b=e1e6e8)

  


但是，到目前为止，HTML 的 `<input>` 的 `type` 或 `inputmode` 并没有为 OTP 提供相应的类型或输入模式。那么，Web 开发者如何做，才能为用户提供一个具有友好交互体验的 OTP 呢？如果你感兴趣，请继续往下阅读！

  


## 传统方案：使用 JavaScript 创建 OTP 输入框

  


我们以下图这个效果为例：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/883107218a8f4edf8cc19958eeac2383~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=864&s=18101&e=png&b=f8fafc)

  


你可能会采用四个单独的 `<input>` 元素来创建上图所示的 OTP 输入框：

  


```HTML
<div class="card--verification">
    <h2>Mobile Phone Verification</h2>
    <p>Enter the 4-digit verification code that was sent to your phone number</p>
    <div class="otp">
        <input type="text" inputmode="numeric" pattern="^\d{1}$" class="otp__input" maxlength="1" />
        <input type="text" inputmode="numeric" pattern="^\d{1}$" class="otp__input" maxlength="1" />
        <input type="text" inputmode="numeric" pattern="^\d{1}$" class="otp__input" maxlength="1" />
        <input type="text" inputmode="numeric" pattern="^\d{1}$" class="otp__input" maxlength="1" />
    </div>
    <button>Verify Account</button>
    <p>Didn't receive code? <a href="">Resend</a></p>
</div>
```

  


需要注意的是，每个输入框仅允许输入一个数字。幸运的是，我们可以使用内置的 `maxlength` 属性来确保这个限制被强制执行。

  


通常情况下，OTP 输入框每个框输入的内容是 `0 ~ 9` 的数字，我们应该使用 `type` 为 `number` 的输入框（`<input>`）更符合用户的体验，它可以直接唤起数字键盘。但是，`maxlength` 对于 `type` 为 `number` 的输入框是无效的，如果你想强制输入框只输入一个数字，需要借助 JavaScript 脚本来实现。其次，当用户的鼠标悬浮在输入框上或输入框获得焦点时，输入框的右侧会呈现“上下箭头”，使得样式化 OTP 输入框时带来了不必要的麻烦：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/71cd5e9e1b4d4249b381e249bccd6c7d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1185&s=23738&e=png&b=ffcc00)

  


庆幸的是，我们可以通过 `input` 元素的 `inputmode` 和 `pattern` 属性，使 `type` 为 `text` 的输入框也能达到类似 `number` 的体验:

  


-   `inputmode="numeric` 可以唤起数字键盘
-   `pattern="^\d{1}$"` 输入必须是一个数字（`0` 到 `9`）且只能输入一个数字

  


这意味着，下面这行代码起到了多个作用：

  


```HTML
<input type="text" inputmode="numeric" pattern="^\d{1}$" class="otp__input" maxlength="1" />
```

  


-   `pattern="^\d{1}$"`：输入必须是一个数字（`0` 到 `9`）且只能输入一个数字
-   `maxlength="1"`：确保输入框最多只能输入一个字符（虽然 `pattern` 也能限制输入，但 `maxlength` 是直接控制输入长度的）
-   `inputmode="numeric"`：移动设备上唤起数字键盘

  


这样就可以限制用户只输入一个数字了。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ab1d95db747b4979a3fbc0c0039cd0e9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1512&h=494&s=331144&e=gif&f=258&b=ffffff)

接下来需要写一些 JavaScript 脚本。

  


我们希望确保每次只能输入一个数字到 OTP 输入框中。为了实现这一点，我们首先通过 `querySelectorAll('.otp__input')` 选择 `#otp` 容器中的所有输入框。然后，我们使用`Array.from()`将`querySelectorAll()`返回的`NodeList`转换为数组，以便我们可以使用`forEach()`等数组方法。

  


接下来，我们需要处理每个输入框的`keydown`事件。每当按下键时，该事件会被触发。在事件处理程序中，我们使用`/^[0-9]{1}$/`正则表达式模式检查按下的键是否为数字。如果不是数字，我们通过调用`e.preventDefault()`来阻止其默认行为。

  


值得注意的是，我们允许使用退格键和删除键来删除当前数字，就像你预期的那样。因此，我们还会将按下的键与这两个键进行比较。

  


以下是`keydown`事件处理程序的示例：

  


```JavaScript
const otpEle = document.getElementById('otp');
const inputs = [...otpEle.querySelectorAll('.otp__input')];

const handleKeyDown = (e) => {
    const { target } = e;
    const currentValue = target.value;
    const index = inputs.indexOf(target);
    switch (e.key) {
        case "ArrowDown":
            target.value =
                currentValue === "" || !/^[0-9]{1}$/.test(currentValue)
                  ? 0
                  : Math.max(parseInt(currentValue, 10) - 1, 0);
            break;
        case "ArrowUp":
            target.value =
                currentValue === "" || !/^[0-9]{1}$/.test(currentValue)
                  ? 0
                  : Math.min(parseInt(currentValue, 10) + 1, 9);
            break;
        default:
            if (
                !/^[0-9]{1}$/.test(e.key) &&
                e.key !== "Backspace" &&
                e.key !== "Delete"
            ) {
                e.preventDefault();
            }
          break;
    }
};

inputs.forEach((input) => {
    input.addEventListener("keydown", handleKeyDown);
});
```

  


当用户点击 `#otp` 容器中的一个输入框时，它会变为活动状态。为了让用户更方便地替换输入框中的当前数字，我们可以使用`focus`事件，在输入框获得焦点时自动选择输入框的内容。

  


我们通过在事件处理程序内使用目标元素的`select()`方法来做到这一点。这样可以选择输入框中的当前数字，使用户能够直接替换它，而无需先删除它。

  


```JavaScript
const handleFocus = (e) => {
    e.target.select();
};

inputs.forEach((input) => {
    input.addEventListener('focus', handleFocus);

});
```

  


用户通常会复制一个数字并粘贴到输入框中。为了处理这种情况，我们可以使用`paste`事件捕获用户粘贴文本的情况。然后，我们将第一个输入框的值拆分为一个数字数组，并将每个剩余输入框的值设置为数组中相应的数字。

  


以下是`handlePaste()`函数的示例：

  


```JavaScript
const handlePaste = (e) => {
    e.preventDefault();
    const pastedText = e.clipboardData.getData("text");
    if (!/^[0-9]{4}$/.test(pastedText)) {
        return;
    }
    const digits = pastedText.split("");
    inputs.forEach((input, index) => (input.value = digits[index]));
    inputs[inputs.length - 1].focus();
};

inputs.forEach((input) => {
    input.addEventListener("paste", handlePaste);
});
```

  


在这个函数中，我们首先通过`e.preventDefault()`阻止了默认的粘贴行为。然后，我们使用`e.clipboardData.getData('text')`获取复制的文本，并使用`split('')`将其拆分成一个数字数组。

  


需要记住的是，我们只允许粘贴四位数字。我们通过检查复制的文本是否匹配`/^[0-9]{4}$/`模式来确保这一点。

  


接下来，我们将循环遍历每个输入框，并用数组中的相应数字填充它。此外，处理程序还会自动将焦点转移到最后一个输入框，使过程更加顺畅。

  


目前，OTP输入框已经具备了一些有用的功能。但我们希望进一步改进，通过添加更多的快捷方式来使用户体验更流畅。

  


我们首先的改进是，当用户输入数字或按下右箭头键时，自动将焦点移动到下一个输入框。这样，用户可以快速输入整个OTP，而不必手动切换字段。为此，我们将在OTP容器的每个输入框上处理`keyup`事件。每次释放键时都会触发该事件。

  


在事件处理程序中，我们检查按下的键是否为数字或右箭头键。如果是，我们跳转到下一个输入框。我们只有在未到达最后一个输入框时才这么做，可以通过比较当前索引与输入框的总数来判断。

  


以下是该功能的示例：

  


```JavaScript
const handleKeyUp = (e) => {
    const index = inputs.indexOf(e.target);
    switch (e.key) {
        case "0":
        case "1":
        case "2":
        case "3":
        case "4":
        case "5":
        case "6":
        case "7":
        case "8":
        case "9":
        case "ArrowRight":
            if (index < inputs.length - 1) {
                inputs[index + 1].focus();
            }
          break;
        case "ArrowLeft":
        case "Backspace":
            if (index > 0) {
                inputs[index - 1].focus();
            }
            break;
        case "Home":
            inputs[0].focus();
            break;
        case "End":
            inputs[inputs.length - 1].focus();
            break;
        default:
            break;
    }
};

inputs.forEach((input) => {
    input.addEventListener('keyup', handleKeyUp);
});
```

  


同样，当用户按下左箭头或退格键时，我们希望将焦点移到上一个输入框。只需检查当前索引是否大于0即可。

  


```JavaScript
switch (e.key) {
    // ...
    case 'ArrowLeft':
    case 'Backspace':
        if (index > 0) {
            inputs[index - 1].focus();
        }
        break;
}
```

  


如何使得用户可以更轻松地导航到第一个和最后一个字段？我们可以通过添加对 `Home` 和 `End` 键的支持来实现这一点。在我们的`switch case`语句中增加几个额外的检查。

  


```JavaScript
switch (e.key) {
    // ...
    case 'Home':
        // 跳到第一个输入框
        inputs[0].focus();
        break;
    case 'End':
        // 跳到最后一个输入框
        inputs[inputs.length - 1].focus();
        break;
}
```

## 现代方案：使用现代 CSS 创建 OTP 输入框

  


正如你所看到的，上面这个 OTP 输入框的设计和用户体验都很有考虑，比如使用箭头键在输入框之间切换，以及处理粘贴功能。但实现这个效果用了大量的 JavaScript 代码？其主要因素是每个字符都在 HTML 中作为一个独立的 `<input />` 元素。这使得验证、输入、粘贴、可访问性、导航等事情变得复杂了。

  


我在想，要是通过一个 `<input />` 来实现呢，事情是不是要更简单化了很多？试想一下，那些数字背后的矩形只是视觉装饰，它只是一个样式问题，而不是语义性、可用性或其他方面的问题。换句话说，我们只需要使用 CSS 在视觉上将每个数字放在一个框中，就能达到我们所期望的 OTP 输入框效果。

  


### 将数字放入到框中

  


在开始之前，我们先来了解 CSS 如何将一串数字（比如 `9286` 或 `902356` ，OTP 通常是 `4` 位或 `6` 位数字组成）中每个数字放到单独的框中。

  


```HTML
<p>9286</p>
<p>902356</p>
```

  


你可能会好奇，CSS 如何分离这一串数字，并给将每个数字放到一个框中？这对于现代 CSS 而言，不是难事。我们可以结合 CSS 的自定义属性、渐变、裁剪和遮罩在视觉上将一串数字中的每个数字放到一个盒子框中。比如：

  


```CSS
p {
    --s: 0.4em;
    --d: 0.2em;
    --p: 0.3em;
    
    font-size: clamp(1.5rem, 4vw + 1rem, 2rem);
    color: #fff;
    font-weight: 900;
    justify-self: center;
    font-family: ui-monospace, monospace;
    line-height: 1.5;
    letter-spacing: calc(var(--s) + var(--d) + 2 * var(--p));
    padding-left: var(--p);
    padding-bottom: var(--d);
    clip-path: inset(0 calc(var(--p) + var(--s)) 0 0);
    --_g: calc(1ch + var(--s) + var(--d) + 2 * var(--p));
    background: 
        conic-gradient(
            at calc(100% - var(--d) - var(--s)) calc(100% - var(--d)),
            #0008 37.5%,
            #0004 75%,
            #0000 0
        ) 0 0 / var(--_g) 100% #8a9b0f; 
    mask: 
        conic-gradient(
            from -45deg at calc(100% - var(--s)) var(--d),
            #0000 62.5%,
            #000 0
        ) 0 0 / var(--_g) 50% repeat-x,
        conic-gradient(
            at var(--s) calc(100% - var(--d)), 
            #000 37.5%, 
            #0000 0
        ) calc(-1 * var(--s)) 100% / var(--_g) 50% repeat-x;
  }
```

让我详细解释各个部分。

  


首先定义 CSS 变量：

  


```CSS
p {
    --s: .4em; /* 定义了框之间的间距 */
    --d: .2em; /* 定义了深度（阴影或边距效果） */
    --p: .3em; /* 定义了内边距的大小 */
}
```

  


给 `p` 元素设置一些基本样式：

  


```CSS
p {
    --s: .4em; /* 定义了框之间的间距 */
    --d: .2em; /* 定义了深度（阴影或边距效果） */
    --p: .3em; /* 定义了内边距的大小 */
    
    font-size: clamp(1.5rem, 4vw + 1rem, 2rem);
    font-weight: 900;
    justify-self: center;
    font-family: ui-monospace, monospace;
    line-height: 1.5;
    letter-spacing: calc(var(--s) + var(--d) + 2 * var(--p));
    padding-left: var(--p);
    padding-bottom: var(--d);
    color: #fff;
    background: #000;
}
```

  


上面代码中的关键部分是：

  


-   字体使用等宽字体，即 `font-family: ui-monospace, monospace;` ，比较适用于数字
-   通过计算 `--s` 、`--d` 和 `--p` 的值来设置字母间距（`letter-spacing`），这个非常关键，它在视觉上使数字进行分离。其中 `calc()` 函数确保数字（或字母）间距适应不同的布局需求

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/26a331deb8fa4299b964e72d259941ba~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2510&h=988&s=64833&e=png&b=f9fafc)

  


正如你所看到的，数字之间已经有了足够宽的间距，在视觉上它们不是一串数字，看起来更像是单独的数字。

  


接下来，我们使用渐变来替代初始的黑色背景：

  


```CSS
p {
    --_g: calc(1ch + var(--s) + var(--d) + 2 * var(--p));
    background: 
        conic-gradient(
            at calc(100% - var(--d) - var(--s)) calc(100% - var(--d)),
            #0008 37.5%,
            #0004 75%,
            #0000 0
        ) 0 0 / var(--_g) 100% #8a9b0f; 
}
```

  


我们使用了一个**[锥形渐变](https://juejin.cn/book/7223230325122400288/section/7259668771856941111)**（`conic-gradient()`），开始于 `calc(100% - var(--d) - var(--s)) calc(100% - var(--d))`，也就是距离右下角一定的偏移位置。我们也可以写成 `at right calc(var(--d) + var(--s)) bottom var(--d)` ，这样更易于理解。

  


这个渐变是从颜色 `#0008` （半透明黑色）开始，经过 `#0004` （更透明的黑色）到完全透明（`#0000`）。这种渐变设计创建了一个深色的背景效果。与此同时，我们还指定了一个主色 `#8a9b0f` ，这通常是元素的底色。为了更好的控制这个主色，你可以将其存储到一个 CSS 变量中，比如 `--c: #8a9bof` 。

  


与此同时，`background-size` 控制渐变的大小，它的值为 `var(--_g) 100%` 。其中 `--_g` 变量是一个 `calc()` 表达式，它根据变量 `--s` 、`--d` 和 `--p` 来计算水平方的尺寸。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/879f5340b4c745a7bddbb4552ce9c6b2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2510&h=988&s=74101&e=png&b=f9fafc)

  


现在，每个数字有一个独方的背景（即位于拥有 `--c` 背景框中）。不难发现，元素框末尾多出了一部分。我们可以使用 `clip-path` 的 `inset()` 对元素的四个边进行裁剪，裁剪的数值定义了元素的可见区域。

  


```CSS
p {
    clip-path: inset(0 calc(var(--p) + var(--s)) 0 0);
}
```

  


上面代码中的 `calc(var(--p) + var(--s))` 表示右边的裁剪值，这个值结合了内边距和框之间的间距。这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99cba65820b444f4be73cd8b396f1f20~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2510&h=988&s=73775&e=png&b=f9fafc)

  


最后，我们需要将数字间的透明黑色裁剪掉。我们可以使用 CSS 的遮罩来实现：

  


```CSS
p {
    --_g: calc(1ch + var(--s) + var(--d) + 2 * var(--p));
    mask:
        conic-gradient(
            from -45deg at calc(100% - var(--s)) var(--d),
            #0000 62.5%,
            #000 0
        ) 0 0/var(--_g) 50% repeat-x,
        conic-gradient(
            at var(--s) calc(100% - var(--d)),
            #000 37.5%,
            #0000 0) calc(-1*var(--s)) 100%/var(--_g) 50% repeat-x;
}
```

  


我们使用了两个锥形渐为绘制的遮罩图，第一个渐变从透明开始，逐渐变成不透明，创建了一种渐变遮罩效果，部分区域为为透明；第二个渐变控制了另一个渐变方向和遮罩效果，确保元素的一部分保持可见，而其他部分被遮罩掉。

  


请注意渐变方向、位置和大小，其中开始位置和大小都是通过 `calc()` 动态计算获得。我们同样可以添加关键词来描述遮罩图的开始位置，这样更易于理解：

  


```CSS
p {
    --_g: calc(1ch + var(--s) + var(--d) + 2 * var(--p));
    mask: 
        conic-gradient(
            from -45deg at right var(--s) top var(--d),
            #0000 62.5%,
            #000 0
        ) 0 0 / var(--_g) 50% repeat-x,
        conic-gradient(
            at left var(--s) bottom var(--d), 
            #000 37.5%, 
            #0000 0
        ) calc(-1 * var(--s)) 100% / var(--_g) 50% repeat-x;
}
```

简单的小结一下，在这个示例中，我们使用了很多动态计算的值和渐变技术来创建复杂的效果，主要用于控制元素的外观，包括：

  


-   自定义的字母间距和内边距

    padding-inline-start: calc(((var(--otp-ls) - 1ch) / 2) * var(--otp-gap));

需要知道的是，这种纯 CSS 创建的 OTP 输入框仅是在视觉上实现了多个输入框的效果，但它的实质还是在一个输入框上输入验证码。虽然我们在 HTML 中设置了 `pattern` 属性，应用一些简单的正则规则来匹配用户输入的内容。但要获得更好的体验，还是需要通过 JavaScript 对 `input` 输入的内容进行验证，以及添加一些鼠标或键盘事件，这样能给用户提供更好的体验。但如果你不关注这些细节，只是要在视觉上实现多个输入框的假像，那么纯 CSS 就足够了。

  


虽然这种方式使 CSS 变得更为复杂一些，但原理还是比较简单的，你只要对 CSS 的变量、渐变、裁剪和遮罩等特性熟悉的话，你可以在这个基础上创建更多的 OTP 输入框变体，使你的 OTP 输入框有很强的个性化和定制化！
