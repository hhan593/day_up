![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3a2f13e736e54927829fd8ffbb776c1a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=56124&e=jpg&b=f7f6f6)

  


CSS 引入 `!important` 声明，旨在帮助 Web 开发者在修改样式表时能轻松覆盖常规的优先级规则。直到现在，`!important` 声明几乎没有什么变化。然而，在样式表中通过 `!important` 声明来提高优先级来覆盖其他样式规则，这对于样式的维护将是一个噩梦的开始。

  


庆幸的是，在现代 CSS 中，我们可以通过其他的一些机制来提高样式规则的优先级，达到类似于 `!important` 的功能，而且还有利于样式表的维护与管理。接下来，我们就来深入了的聊一聊 `!important` ，以及替代 `!important` 声明的一些技巧。在此过程中，你将会了解到这种声明到底是什么，以及你应该在什么情况下使用它，或者是否应该使用它。

  


## 从 CSS 的级联机制和权重开始

  


在深入探讨 `!important` 声明及其工作方式之前，我们先来了解一下 CSS 的级联机制和权重。稍微了解 CSS 的 Web 开发者都应该知道，在编写 CSS 的过程中，如果没有考虑到级联（也称层叠）和权重，它们肯定会引发问题，但如果你对这些概念有一个基础的理解，我保证你的 CSS 技能会飞速提升。

  


浏览器通常会根据级联算法来决定不同样式的权重，级联算法分为 4 个不同的阶段。

  


-   **①出现的位置和顺序**：CSS 规则出现的顺序
-   **②特异性**：一种确定哪个 CSS 选择器具有最强匹配的算法
-   **③来源**：CSS 出现的顺序及其来源，无论是浏览器样式、浏览器扩展中的 CSS 还是你创作的 CSS
-   **④重要性**：某些 CSS 规则的权重高于其他规则，尤其是具有 `!important` 规则类型的规则

  


级联算法在不同的规范中有不同的描述，在 Level 5 中提供了六个不同的级别。在不考虑级联层的情况下，其标准如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b884d9cafe3f4f04953da80c8b98e5ea~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3186&h=995&s=1316309&e=jpg&b=131416)

  


另一个基础且重要的概念是 **[CSS 选择器的权重](https://juejin.cn/book/7395119173656903715/section/7445260010408149002#heading-0)**。本质上说，CSS 选择器权重就是类似于海底世界的生物生存规则一样，大鱼吃小鱼，小鱼吃虾米：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b5a3b6b7717a4a59853931fac80c4187~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1275&h=1204&s=440563&e=png&b=fbf9f9)

  


从上图中不难看出，带有 `!important` 声明的规则，其权重是最高的。

  


需要知道的是，[CSS 引入的 `@layer` 规则对 CSS 级联机制做出了一定的调整](https://juejin.cn/book/7223230325122400288/section/7259563116462080037)，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/feae08672588495d9e8f026d75607504~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2943&h=1444&s=166546&e=png&a=1&b=e1e1e1)

  


  


我们把“来源和重要性”单独拎出来看：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3e434bdbc356405ab936d7cd7d3fc5de~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=871&h=731&s=490333&e=jpg&b=131416)

  


你可能已经注意到带有 `!important` 的来源与正常（即不带 `!important` 的来源）的对应物的顺序正好相反，这其实就是 CSS 的工作方式！ **当一个声明被显式标记为** **`!important`** **时，它在级联中的权重就会增加并颠倒优先顺序**。

  


这个反转规则也适用于级联层中的声明：**带有** **`!important`** **声明将被放在“重要开发者”来源（Important Author Origin），但与“正常作者”来源（Normal Author Origin）相比，级联层将有相反的顺序**。

  


在级联层中，当比较属于不同级联层的声明时，那么对于正常的规则，级联层在最后的声明获胜，而对于重要的规则（带有 `!important` 的规则），级联层在前的声明获胜。

  


通过这个基本框架，你应该已经能看出 `!important` 声明的权重有多高，以及它在级联机制中所起的作用。接下来，我们将更详细的了解 `!important` 。

  


## `!important` 是什么？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8e5f953fc6c44686a76fe1a27a450aaf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1000&h=396&s=21866&e=webp&b=fd5400)

  


`!important` 声明为 Web 开发者提供了一种方法，可以让某个 CSS 样式的权重高于它原本的优先级。当我们在 CSS 中添加这个关键字时，本质上是人为提升了它的**重要性**，让它在应用样式时比其他相匹配的样式具有更高的优先级。

  


需要知道的是，所谓的 “`!important` 声明”指的是整个 CSS 声明（包括属性和值），并且在末尾加上 `!important` 。下面这个简单示例，清晰展示了 `!important` 如何影响样式的正常应用方式：

  


```CSS
h2 {
    color: hsl(200 100% 40%); 
}

article h2 {
    color: hsl(35 100% 50%); 
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e7a3a85f23d348e2951e033c523ee9c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=116298&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/ZYYRXEP

  


正如你所看到的，卡片标题渲染成橙色是合理的——第二条样式规则不仅在代码中更靠后，而且选择器权重更高，因为 `article h2` 使用了子代选择器，比单独的元素选择器 `h2` 拥有更高的权重分数：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/72a7e24c2fef41448e5bb48d3514d275~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1052&h=357&s=86815&e=png&b=f5f7fc)

  


如果我们在第一条样式后面加上 `!important` ，它就会“强行”排在前面，覆盖第二条本应胜出的样式：

  


```CSS
h2 {
    color: hsl(200 100% 40%) !important;  
}

article h2 {
    color: hsl(35 100% 50%); 
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3fa4469818c43b68189da961da73961~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=116219&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/PwwaJZO

  


这相当于颠覆了 CSS 的级联机制：原本应该被覆盖的第一条样式声明，因为加上了 `!important` ，而变得比后面的样式更“重要”：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f77568309c1147a6bf775547c75da720~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3184&h=1570&s=635540&e=jpg&b=effafa)

  


在使用 `!important` 声明时，有一些注意事项需要知道：

  


-   在 CSS1 中首次引入 `!important` 时，Web 开发者（作者）样式中带有 `!important` 的声明权重大于用户样式中的 `!important` 声明；但为了提升 Web 的可访问性，这一规则在 CSS2 中被反转，用户的 `!important` 样式优先于 Web 开发者的样式
-   通常情况下，CSS 样式的级联顺序是：Web 开发者样式 > 用户样式 > 用户代理（浏览器）样式；但涉及`!important` 声明时，这个级联顺序会把转：用户代理的 `!important` 样式 > 用户的 `!important` 样式 > Web 开发者的 `!important` 样式。这意味着，你无法覆盖来自浏览器用户理样式表的 `!important` 声明
-   如果 `!important` 用在简写属性（比如 `margin` 、`font` 等）上，它会自动应用到该简写属性所代表的所有子属性上
-   `!important` 必须紧贴在语句末尾的分号之前，否则无效（不过，在 `!important` 和分号之间留空格是允许的）
-   如果你出于某种原因在同一个声明块中写了两个相同的属性，并且第一个加了 `!important` ，那么在除 IE6 以外的所有浏览器中，第一个会具有更高的权重

  


稍微了解 CSS 的 Web 开发者都知道，**[CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)** 具有灵活性，能够使代码遵循 DRY 原则（不要重复自己），并有利于提高代码的可维护性。另外，CSS 代码越多，CSS 的可操作性就越重要。当你在没有策略的情况下添加新代码时，CSS 代码会变得混乱，但如果你在其中添加 `!important` 声明，往往会不可避免地导致维护的噩梦。

  


但是，你尝试过在自定义属性中使用 `!important` 时，你会发现一些有趣的事情—— **`!important`** **声明在自定义属性声明中的行为与你想象的不同**！

  


CSS 自定义属性值的语法是宽松的，它允许几乎所有内容作为自定义属性值，只有少数几个特定的标记被禁止。这些被禁止的标记包括不匹配的 `]` 或 `}` ，分号（`;`）以及 `!` 字符。尽管 `!important` 声明包含了 `!` 字符，但它仍然被视为一个特殊情况，且是有效的！

  


例如：

  


```CSS
h2 {
    --color: red !important;
    color: var(--color);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9b8d6b6f48364e32bbd9313af83b40bf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=115789&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/MYYXEGg

  


正如你所看到的，`color` 属性被正确解析并应用。但需要注意的是，`!important` 在自定义属性值中的使用会带来一些意外的行为—— **CSS 解析器会移除** **`!important`** **关键字**！如果你[再次查看规范](https://www.w3.org/TR/css-variables-1/#syntax)，你会发现如下说明：

  


> 自定义属性可以包含一个结尾的 `!important` ，但是 CSS 解析器会自动从属性值中移除它，并使得自定义属性在 CSS 级联中变得“重要”。

  


这意味着你可以用另一个 CSS 属性覆盖已应用的“重要的 CSS 自定义属性”：

  


```CSS
h2 {
    --color: red !important;
    color: var(--color);
    color: yellow;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2263f297db8d4d65b4f515627ffafd07~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=116442&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/WbbyZgm

  


但是，规范定义了包含结尾 `!important` 的自定义属性在 CSS 层叠中会被标记为“重要”，那这又是怎么回事呢？

  


自定义属性的声明处理方式与“普通的 CSS 声明”类似。当有多个自定义属性声明时，CSS 层叠用于评估自定义属性的值。例如：

  


```CSS
h2 {
    --color: red !important;
    color: var(--color);
    
    article & {
        --color: yellow;
    }
}
```

  


在上面，具有 `!important` 声明会获取并赋值给 `--color` 。因此，你会看到，`h2` 中定义的 `--color` 将会覆盖 `article h2` 中定义的 `--color` ，标题依旧是红色：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e7d22dd27b348b2931316b5c542f010~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=115789&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/YPPvrRE

  


正常情况下，`--color` 值不带有 `!important` 声明，标题的颜色应该是 `yellow`:

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b167d4726eb54c0598e31cf16acfb283~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=116440&e=png&b=ffffff)

  


需要知道的是，当 `--color` 自定义属性被使用时，仍然没有将 `!important` 设置为颜色（`color`）的属性值。因为，`!important` 仅用于确定 CSS 声明的值，通过 CSS 级联来处理，并不会在后期阶段生效。这意味着，你仍然可以用其他颜色声明覆盖 `color` 属性（无论它是否通过 `!important` 声明给自定义属性赋值）：

  


```CSS
h2 {
    --color: red !important;
    color: var(--color);
}

article h2 {
    --color: yellow;
}

article:nth-child(2) h2 {
    color: lime;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4fbe41923f114171894a6424b6e227c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=119210&e=png&b=ffffff)

正如你所看到的，第二张卡片的标题颜色是 `lime` ，这是因为它的选择器（`article:nth-child(2) h2`）权重高于默认的 `h2` 选择器权重。对应的 CSS 规则 `color` 获胜，因此会覆盖最初的 `color` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/16b67c33e32f4a78bd87c8a2044de8f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3242&h=1416&s=582957&e=jpg&b=effafa)

  


是不是感觉非常绕！我个人建议，最好避免使用 `!important` ，如果真的需要使用，务必仔细考虑。而且，老实说，我不会将 `!important` 混入自定义属性值中，因为这种令人惊讶的“怪癖”非常难以发现！

  


## 何时应该使用 `!important`

  


是否使用 `!important` 取决于具体情境，有其利弊。那么，它究竟什么时候该用（如果真的需要用的话）？

  


使用 `!important` 的第一法则就是：**不要用它来强行覆盖样式**。只有在其他所有方法都失败之后，`!important` 才应作为最后的手段使用。如果你只是因为懒得调试、想节省时间，或者急于完成项目，就滥用 `!important` ，那么你就是在滥用——而你（或未来维护你项目的人）会因此付出代价。

  


即使你在样式表中只是偶尔使用 `!important` ，也会很快发现部分样式难以维护。正如前面所说，CSS 的样式优先级本该通过级联（层叠）或权重自然决定。而使用 `!important` 则会打破这种自然顺序，让某些本不该占优的规则获得过高的权重。

  


如果你从不使用 `!important` ，那反而是你真正理解了 CSS，并在编写前做了合理规划的表现。当然，也有例外，下面我们就来看看一些合理使用 `!important` 的场景。

  


### 用于调试布局和可访问性

  


`!important` 有一个合理且实用的用途：**帮助发现 CSS 问题**。比如，我们可以强制对没有设置 `alt` 文本的图片加上红色边框，以便于在页面中快速识别和修复。

  


```CSS
img:is([alt=""], :not([alt])) {
      outline: 5px dotted red !important;
}
```

  


同样的思路也可以用于排查 Web 布局问题。给元素添加轮廓可以可视化它们的尺寸，尤其当它们位置不正确时，能够准确地看出哪里出了问题，特别是当一个元素溢出容器时。

  


这种方法只适合本地开发的安全环境中使用，在项目上线时，应该删除这些调试代码（带有 `!important` 声明的调试代码），不然所有人都能看到！

  


### 用于打印样式表

  


虽然并非所有情况下都需要这么做，在某些情况下也可能不推荐使用，正如之前提到的原因，你可以在仅用于打印的样式表中添加 `!important` 声明，以帮助覆盖特定样式：

  


```CSS
body {  
    margin: 2em;  
    color: #fff;  
    background-color: #000;
    
    @media print {
        margin: 0 !important;    
        color: #000 !important;    
        background-color: #fff !important;  
    }
}
```

  


## 新的 `!important`

  


值得再次强调的是， `!important` 是一个自 CSS1 时代以来就存在的旧工具。近年来，CSS 提供了一些功能，可以帮助我们在不破坏级联机制和权重计算的情况下覆盖样式。你会发现，使用以下任何方法要比依赖 `!important` 更加高效。

  


### 级联层：`@layer`

  


我们可以将样式组织成不同的层，这些层按优先级进行排序。

  


```CSS
@layer base, layout, template;

@layer base {
    /* 基础样式：最不具体（权重最低）*/
}

@layer layout {
    /* 布局样式  */
}

@layer template {
    /* 模板样式：最具体（权重最高）*/
}
```

  


在上面的代码中，声明的层的顺序决定了权重的顺序，第一层（`base`）是权重最低的，而最后一层（`template`）是权重最高的。所以，即使我们将样式表的顺序颠倒过来，像这样：

  


```CSS
@layer base, layout, template;

@layer template {
    /* 模板样式：仍然是最高权重 */
}

@layer layout {
    /* 布局样式 */
}

@layer base {
    /* 基础样式：仍然是最低权重 */
}
```

  


模板层（`template`）仍然会获得最高优先级，即使它在样式表中排在最前。如果我们确实想要让基础层（`base`）成为最具体的层，只需要重新排序 `@layer` 声明的顺序即可。

  


我们来看一个简单的示例，展示了即使第二层的样式在第一层之前出现，第一层的样式仍然被第二层的样式覆盖了。这得益于级联层（`@layer`），我们可以将最重要的样式写在顶部，并保持其优先级高于其他样式：

  


```CSS
@layer layout, demo;

@layer demo {
    article {
        background: #111;
        color: #fff;
        border-radius: 8px;
        font-size: 1rem;
        padding: 2rem;
    }
  
    h2 {
        font-size: 3rem;
    }
}

@layer layout {
    article {
        background: #fff;
        line-height: 1.6;
    }

    h2 {
        color: hsl(340 70% 50%);
        font-size: 1.25rem;
    }
}
```

这里需要指出的是，未命名的级联层（即匿名级联层）比命名的级联层权重要更高。如果你需要快速提升权重，只需开始在一个新层中编写样式：

  


```CSS
@layer {
    /* 未命名的层样式：权重最高*/
}

@layer base {
    /* 基础样式：权重次之 */
}
body {
    /* 未分层样式：权重最低 */
}
```

  


你可能会想知道，如果在级联层中尝试使用 `!important` 会发生什么。其实很奇怪，因为它会 **[反转层的顺序和样式规则](https://juejin.cn/book/7223230325122400288/section/7259563116462080037#heading-13)**。

  


来看一个示例，使用 `@layer` 声明了四人级联层：

  


```CSS
@layer reset, base, theme, utilities; 
```

  


这些级联层中的正常声明都在“正常作者”来源（Normal Author Origin）中，并将按此排序：

  


-   ①：正常 `reset` 级联层
-   ②：正常 `base` 级联层
-   ③：正常 `theme` 级联层
-   ④：正常 `utilities` 级联层

  


然而，这些级联层中的重要声明（带有 `!important` 声明）都将进入“重要用户”来源（Important User Origin），并将被反向排序：

  


-   ①：重要 `utilities` 级联层
-   ②：重要 `theme` 级联层
-   ③：重要 `base` 级联层
-   ④：重要 `reset` 级联层

  


因为“普通未分层样式”（普通隐含级联层，Normal Unlayered Styles）隐含在最后，这也意味着“重要未分层样式”（重要隐含级联层，Important Unlayered Styles）将被放置在第一位。所以，在级联层中的“重要声明”会赢过在隐含级联层中的“重要声明”。这个重要声明指的是带有 `!important` 标记的声明。

  


来看一个具体的示例：

  


```HTML
<h1 class="title">With !important</h1>
<p class="title">Without !important</p> 
```

  


```CSS
@layer base, theme, component; 

@layer base { 
    h1 { 
        color: orange !important; 
    } 
    
    p { 
        color: orange; 
    } 
} 

@layer theme { 
    h1 { 
        color: lime !important; 
    } 
    
    p { 
        color: lime; 
    } 
} 

@layer component { 
    h1 { 
        color: yellow !important; 
    } 
    
    p { 
        color: yellow; 
    } 
}
```

  


上面代码使用 `@layer` 声明了 `base`、`theme` 和 `component` 三个级联层，在相应的级联层中都给 `h1` 和 `p` 元素设置了 `color` 样式规则，不同的是，`h1` 的 `color` 值带有 `!important` 标识符。

  


先来看不带 `!important` 的 `p` 元素，相应的级联层顺序：

  


-   ①：`base`
-   ②：`theme`
-   ③：`component`

  


最终级联层 `component` 获胜，因此 `p` 元素文本的颜色是 `yellow`。

  


再来看带有 `!important` 的 `h1` 元素，相应的级联层却刚好与不带 `!important` 的 `p` 元素相反：

  


-   ①：`component`
-   ②：`theme`
-   ③：`base`

  


最终级联层 `base` 获胜，因此 `h1` 元素文本的颜色是 `orange`。

### 功能性伪类：`:is()` 、`:where()` 、 `:not()` 和 `:has()`

  


`:is()`、`:where()`、`:not()` 和 `:has()` 是 CSS 中常用的**伪类函数**，它们不仅能简化复杂的选择器，还会影响选择器的**权重**，但它们各自的行为并不完全相同。

  


我们通常以四个由逗号分隔的数字来表示 CSS 选择器的权重，比如 `1,1,1,1` 或 `0,2,0,1` 。相对来说，它的计算还是比较简单的：

  


-   第一个数字（`a`）始终为 `0` ，除非在标记本身中对元素应用了 `style` 属性
-   第二个数字（`b`）是选择器中 `ID` 数量的总和
-   第三个数字（`c`）是选择器中其他属性选择器和伪类的总和。类（`class`）和属性选择器也包含在这里
-   第四个数字（`d`）计算元素（如 `div` ）和伪元素（`:first-line`）

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1dd096e11ded4b4e8ee5a7f49651503f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1184&h=778&s=149876&e=jpg&b=f7f4f3)

  


注意，通用选择器（`*`）的权重为 `0` ，以及子选择器或兄弟选择器（`>`, `~`, `+`）不会增加选择器权重。另外，带有 `!important` 的属性即使选择器的权重较低也会被应用。如果两个选择器的权重相同，最后在样式表中出现的选择器将会获胜。

  


`:is()`、`:where()`、`:not()` 和 `:has()` 计算选择器权重得分略有差异：

  


-   `:is()` ：选择器权重得分等于它的参数列表中最具体选择器的权重分数。所以，如果我们有 `:is(#id, .class, article)`，那么特异性就等于 `#id`，因为它在三个选择器中得分最高
-   `:where()` ：无论如何，选择器权重得分为 `0` 。换句话说，`:where()` 会完全消除选择器权重。所以，如果我们有 `:where(#id, .class, article)`，即使我们与上一个例子中使用的是相同的 `#id`，选择器权重也等于零
-   `:not()` ：像 `:is()` 一样，`:not()` 选择器权重得分等于它的参数列表中最具体选择器权重分数。所以，如果我们有 `:not(#id, .class, article)`，那么特异性就等于 `#id`，因为它在三个选择器中得分最高
-   `:has()` ：它与 `:is()` 和 `:not()` 选择器一样

  


通过简单的示例来展示它们权重的得分。

  


```CSS
:is(h1, h2, h3) {
    color: red;
}
```

  


`:is()` 中的 `h1`、`h2` 和 `h3` 都是元素选择器，它们的得分是 `(0,0,1)`，所以 `:is(h1, h2, h3)` 选择器的整体权重是 `(0,0,1)`：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8bea26a699fa4a848787e7b16d7b58ce~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3060&h=1160&s=214565&e=jpg&b=f7f8fd)

  


但如果你更复杂的选择器：

  


```CSS
:is(.foo, #bar) {
    color: blue;
}
```

  


`:is()` 中的 `#bar` 权重是 `(1,0,0)` ，它比 `.foo` （它的是 `(0,1,0)`）更高，所以最终 `:is()` 整体选择器的权重是 `(1,0,0)` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/732ee927aac74fcabe43c602a9f6eee1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2504&h=2600&s=359121&e=png&b=f5f8fd)

  


我们来看一个关于 `:is()` 、`:where()` 、`:has()` 和 `:not()` 选择器权重得分的示例：

  


```CSS
/* 权重： 1,0,0 */
:is(h1, #bar, .foo) {
    color: blue;
}

/* 权重： 0,0,0 */
:where(h1,#bar#foo, .foo) {
    color: yellow;
}

/* 权重： 2,1,0 */
:has(#bar#foo, h1, .foo, #bar#foo.baz) {
    color: green;
}

/* 权重： 3,0,0 */
:not(#bar,.foo, h1, #bar#baz#foo) {
    color: orange;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1f9c767150c4c43bdda997803eadce4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2504&h=3448&s=530319&e=png&b=f5f8fd)

  


也就是说，我们可以这样使用它们：

  


-   用 `:where()` 定义低优先级的默认样式（因为权重为 `0`）
-   用 `:is()` 合并选择器时要小心，因为**可能会意外提升权重**
-   `:not()` 和 `:has()` 会增加权重，要留意不要导致选择器过于“强壮”

  


这就好比，我们以往通过将选择器与其他选择器结合来提高选择器的权重。比如：

  


```CSS
.parent {
    /* 权重： 0,1,0 */
}

.child {
    /* 权重：0,1,0 */
}

.parent .child {
    /* 权重： 0, 2, 0 */
}
```

  


当然，也有很多 Web 开发者会使用 `ID` 选择器来增加选择器权重，但 `ID` 选择器很难被覆盖，因此更好的做法是逐步增加选择器权重，以便更容易管理。比如：

  


```CSS
.child {
    /* 权重： 0,1,0 */
}

.child:not(.specificity) {
    /* 权重： 0, 2, 0 */
}

.child:not(.specificity):not(.extra-specificity) {
    /* 权重：0, 3, 0 */
}
```

  


这样做有几个好处。首先， `:not()` 选择器确保了虚拟类的权重得以强制执行；其次，`:not()` 防止了一个意外的情况，例如你创建了一组样式使用了这些类，这样就可以避免这些样式被添加到选择器中。

### CSS 作用域：`@scope`

  


正如前面所介绍的那样，使用 `:not()` 选择器替代 `!important` 声明，来提高样式权重是一个不错的策略：

  


```CSS
.child {
    /* 权重： 0,1,0 */
}

.child:not(.specificity) {
    /* 权重： 0, 2, 0 */
}

.child:not(.specificity):not(.extra-specificity) {
    /* 权重：0, 3, 0 */
}
```

  


然而，我们实际上在讨论那些不包含任何样式的工具类，它们只是用来在需要时增加选择器权重。默认情况下，选择器权重很低，但通过添加一个或两个权重更大的类，可以逐渐增加选择器权重。不过，CSS 就像诗歌，它可以用许多不同的方式表达相同的东西。这只是另一种增加选择器在级联中的优先级的方法。

  


我敢打赌，至少有人会把这种方法看作是 `!important` 的一种复杂替代方案。也许你已经在写一个像这样的代码片段：

  


```CSS
.button.important {
    
}
/* 或者 */
.button.very.important {

}
```

  


或者，更好的是，在 `:is()` 参数中使用虚假的 `ID` 来人为权重分数：

  


```CSS
/* 权重: 1，0， 0  */
:is(.button, #important) {

}
```

  


然而，使用这种方式的人工类和 `ID` 可能并不是管理权重的最佳方式。比如，这是不是一个理想的[使用 `@scope` 的场景](https://juejin.cn/book/7223230325122400288/section/7259668111974989882)呢？作用域样式有一种非常酷的方式来设置样式的开始和结束位置，称为作用域限制。

  


```CSS
@scope (.card) to (:scope > .button) {
    /* 样式规则放这里 */
}
```

  


看看这个：在定义了作用域根 `.card` 后，我们用了 `to` 关键字来定义作用域限制 `.button`。如果我们的 HTML 结构是这样的：

  


```HTML
<article class="card">
    <img src="image.avif" alt="...">
    <h2>Card Heading</h2>
    <p>Card content.</p>
    <a class="button" href="#">Read Full Story</a>
</article>
```

  


那么 CSS 就知道 `.card` 类是作用域根样式的起点，并且当样式到达 `.button` 类时就会停止应用。如果我们没有确定作用域限制，就可能会遇到样式冲突的问题，比如：

  


```CSS
.card a {
    color: hsl(25, 100%, 50%);
}
```

  


这段代码不会影响 `.button`，因为我们明确指示了选择器将其排除在作用域之外。现在的 CSS 实在是太聪明了，可能你再也不会用到 `!important` 了。

  


## 案例：工具类的使用

  


在 CSS 中使用工具类时，其挑战就是如何提供比内联样式更多的价值，同时又不牺牲通用性。比如，下面这个设置边框厚度的工具类：

  


```CSS
.border-thick {
    border-width: var(--size-border-thick);
}
```

  


对于已有边框样式的元素，上面的样式是有效的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4f8e20b778b9472bae3eaefeb71bf900~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1024&h=282&s=24130&e=png&b=ffffff)

  


但如果没有边框，它就没有效果。如果你想给一个没有边框的元素添加边框，以便它能与背景区分开，你就没这么幸运了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b2d8738ef8348af8fe4907a062f9e86~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=504&s=54734&e=png&b=ffffff)

  


让我们再加上 `border-style`：

  


```CSS
.border-thick {
    border-style: solid;
    border-width: var(--size-border-thick);
}
```

  


这有效，但边框颜色默认继承了文本颜色，这显得有点过于突出：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7c534ccd66864db4852f0cfb8c448ad5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=624&h=504&s=36245&e=png&b=ffffff)

  


也许我们应该设置一个默认颜色：

  


```CSS
.border-thick {
    border-color: var(--color-border-subtle);
    border-style: solid;
    border-width: var(--size-border-thick);
}
```

  


但现在我们换了一个问题。我们的工具类在没有边框的元素上效果很好，但看看它给我们的按钮带来了什么：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fc632844ad0a408981e2f48fcc43d9ff~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=504&s=55321&e=png&b=ffffff)

  


前一个示例中的边框颜色变得更加微妙，但它错误地应用到了按钮上，导致颜色看起来有些混乱。

  


为了让我们的工具类在不超出范围的情况下独立工作，我们需要明确哪些 CSS 规则应该保持不变（在这种情况下是 `border-width`），哪些规则应该在遇到冲突时让步。

  


这个现象，在 CSS 中一直以来是一个难以解决的问题。传统的做法，可能会使用类链技术来增加某些样式的权重，比如：

  


```CSS
.button.button {
    border: var(--size-border-thin) solid var(--color-border-button);
}
.border-thick {
    border-color: var(--color-border-subtle);
}
.border-thick.border-thick {
    border-style: solid;
    border-width: var(--size-border-thick);
}
```

  


这样做是有效，在紧急情况下也很有用，但会有很多重复的代码。如果在整个项目中保持一致的选择器链长度，这将非常繁琐。

  


我们可以将工具类放在 CSS 的开头，并添加 `!important` 标志来指定我们希望作为覆盖的样式：

  


```CSS
/* 通用性：工具类  */
.border-thick {
    border-color: var(--color-border-subtle);
    border-style: solid;
    border-width: var(--size-border-thick) !important;
}

/* 组件 */
.button {
    border: var(--size-border-thin) solid var(--color-border-button);
}
```

  


这样更易读和写，但可能在关键 CSS 或其他带有 `!important` 的样式面前失效。

  


回顾这些技术，我明白为什么许多 CSS 框架选择不再使用它们。这确实令人失望，需要多个类来获得有用的结果，但那时这是更简单的选择。

  


强调一下，这只是“曾经”是。现在我们有了一些很棒的替代方法！

  


我们可以将默认值移到 `:where` 选择器中。这样应用相同的规则，但没有权重：

  


```CSS
:where(.border-thick) {
    border-color: var(--color-border-subtle);
    border-style: solid;
}

.border-thick {
    border-width: var(--size-border-thick);
}
```

  


现在，无论如何都会应用 `border-width`，但当出现冲突时，`border-color` 和 `border-style` 会让步：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4276b2175bb14422a958eb1d5a6923a0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=504&s=56176&e=png&b=ffffff)

  


或者，我们可以使用级联层。级联层总是比无层级的 CSS 优先级低：

  


```CSS
@layer {
    .border-thick {
        border-color: var(--color-border-subtle);
        border-style: solid;
    }
}

.border-thick {
    border-width: var(--size-border-thick);
}
```

  


但如果你打算在整个项目中使用这种方式，建议提前为你的层命名。在 CSS 中早早指定它们的优先级顺序：

  


```CSS
@layer base, component, utility;
```

  


现在，我们在基础层添加的任何样式都会推迟到组件层，再由组件层推迟到工具层，无论这些样式写在哪里：

  


```CSS
/* components/button.css */
@layer component {
    .button {
        border: var(--size-border-button) solid var(--color-border-button);
    }
}

/* utilities/border.css */
@layer base {
    .border-thick {
        border-color: var(--color-border-subtle);
        border-style: solid;
    }
}

@layer utility {
    .border-thick {
        border-width: var(--size-border-thick);
    }
}
```

  


级联层对于管理样式优先级非常有用，这个示例只是皮毛。

  


如果我们计划使用多个 `border-*` 工具类，可以扩展第一个选择器来设置更多默认值并匹配更多类：

  


```CSS
:where([class^='border-'], [class*=' border-']) {
    border-color: var(--color-border-base);
    border-style: solid;
    border-width: 0;
}

/* 或者 */
@layer base {
    [class^='border-'], [class*=' border-'] {
        border-color: var(--color-border-subtle);
        border-style: solid;
        border-width: 0;
    }
}
```

  


这个属性选择器匹配任何以 `border-` 开头的类（如 `border-thin`、`border-dots`、`border-purple` 等）。无需手动维护一个冗长的选择器列表！

  


通过这种方式，你可以更好的管理和维护你的 CSS 样式表，而且不需要依赖 `!important` 就可以轻易覆盖你想要覆盖的样式！

  


## 结论

  


`!important` 声明最好保留用于特定任务，比如调试布局和可访问性。最好的做法是利用层叠的默认行为，并尽可能保持其完整性，因为这是保持低权重且最易管理和维护的方式。因此，你应该尽量为你的 CSS 提前规划，避免在可能的情况下使用 `!important`。在使用 `!important` 之前，你会发现使用课程中介绍的其他方法会更好。

  


尽管如此，`!important` 仍然是有效的 CSS。你可能会继承一个项目，在这个项目中前任开发者使用了 `!important`，或者你可能需要快速修复某些问题——在这种情况下，它可能会派上用场。理解它并准备在需要时使用它，肯定是有益的。
