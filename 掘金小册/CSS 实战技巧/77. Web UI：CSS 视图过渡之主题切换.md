## CSS 视图过渡

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4304922258a44b79df86003a5e24ee4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1792&h=1024&s=370690&e=webp&b=382d39)

  


如今的 Web 应用变得越来越复杂，因此，作为一种功能性动画，在视图状态之间实现平滑过渡具有实际的好处，例如降低认知负荷、防止变化盲、并帮助用户更好地记忆空间的关系。

  


过去，给 DOM 不同的状态添加动画，需要大量的 JavaScript 和 CSS 代码，因为我们必须处理许多额外的因素。除了动画本身，我们还需要管理内容在不同状态下的加载和渲染，以及可访问性等问题。

  


现在，我们可以通过 CSS 的视图过渡（CSS View Transition API）来简化这一过程，它用于在不同的应用视图之间进行过渡动画。接下来，我们来看看它是如何工作的。

  


通常，要实现平滑动画，元素需要一个起始状态和一个结束状态，然后计算如何在这两个状态之间逐步过渡。CSS 过渡（`transition`）和动画（`animation`）之所以有效，是因为浏览器可以计算元素的中间状态值。但如果想让多个 DOM 节点参与同一个动画，情况就会变得非常复杂，因为计算需求会急剧增加。为了解决这个问题，视图过渡（View Transition API）采用了一种新的方法——**使用伪元素来处理复杂的中间状态，而不是直接在原始 DOM 结构上进行计算**。

  


视图过渡通过静态视觉捕捉（静态快照）保存旧的状态，并使用实时捕捉（动态更新）保存新状态，形成一棵伪元素树，用于实现过渡动画：

  


```CSS
::view-transition
├─ ::view-transition-group(name)
│  └─ ::view-transition-image-pair(name)
│     ├─ ::view-transition-old(name)
│     └─ ::view-transition-new(name)
└─ …其他组…
```

  


视图过渡的核心是 `startViewTransition()` 方法，它返回一个 `ViewTransition` 对象，并执行以下步骤：

  


-   1️⃣：调用 `startViewTransition()` 后，当前视图状态被捕获为“旧”状态，并且渲染暂停
-   2️⃣：`startViewTransition()` 方法接受一个回调函数，当旧状态捕获完成后，它会异步执行
-   3️⃣：在回调函数内，我们可以更新 DOM 以反映新状态
-   4️⃣：当 DOM 更新完成后，API 会捕获新的视图状态，将其作为“新”状态，并生成必要的过渡动画，使两个状态平滑衔接

  


有趣的是，即使由于配置错误或设备限制导致无法创建视图过渡，API 仍然不会阻止回调函数的执行。换句话说，`startViewTransition()` 无论动画是否成功，都会调用回调函数。这种设计方式十分巧妙，即使过渡动画失败，应用程序的状态更新仍然能继续执行。

  


> 温馨提示，如果你对 CSS View Transition API 感兴趣，建议你移步阅读《[解锁 CSS View Transition API 的魔力](https://juejin.cn/book/7223230325122400288/section/7259669097242329145)》！

-   左侧是一个大图的展示区域（主显示区）
-   右侧是一些缩略图

我们定义一个回调函数（这个函数会作为参数传递给 `startViewTransition()` 方法），用于更新主图像：

  


```JavaScript

```

  


在这个示例中，我们简单地将主图像的 `src` 设置为用户点击的缩略图 `src`。在实际应用中，应该提供不同尺寸的优化图片，以提高性能。

  


由于 CSS View Transitions API 目前并非所有浏览器都支持，我们需要进行兼容性检查：

  


```JavaScript
const clickTarget = event.target;
const displayNewImage = () => {
    mainImage.src = clickTarget.src;
};

if (clickTarget.classList.contains("thumbnail")) {
    if (!document.startViewTransition) {
        displayNewImage();
        return;
    }
 
    const transition = document.startViewTransition(() => displayNewImage());
}
```

  


我们把上面这些代码放置在一个新的函数中，比如 `thumbnailHandler()` ：

  


```JavaScript
const mainImage = document.querySelector("figure img");

const thumbnailHandler = (event) => {
    const clickTarget = event.target;
    const displayNewImage = () => {
        mainImage.src = clickTarget.src;
    };

    if (clickTarget.classList.contains("thumbnail")) {
        if (!document.startViewTransition) {
            displayNewImage();
            return;
        }
 
        const transition = document.startViewTransition(() => displayNewImage());
    }
};
```

  


当用户点击缩略图时会调整该函数：

  


```JavaScript
const thumbnails = document.querySelector(".thumbnails");
thumbnails.addEventListener("click", thumbnailHandler, false);
```

  


这个时候，你点击右侧缩略图时，主题会有一个淡入淡出的过渡动画效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dc092da21d904075ab83ddff64dcb5ed~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1084&h=756&s=14397440&e=gif&f=72&b=cfdde6)

  


> Demo 地址：https://codepen.io/airen/full/VYwLRWK

  


这是 CSS View Transition API 默认的动画效果：

  


-   旧视图的 `opacity` 由 `1` 过渡到 `0` （逐渐消失）
-   新视图的 `opacity` 由 `0` 过渡到 `1` （逐渐显现）

  


浏览器自动执行这一动画，无需额外的 CSS。如果默认效果足够用，我们不需要再编写动画代码。

  


整个页面都会参与这个最基本的过渡动画。但是，我们也可以针对特定元素进行动画，而不影响页面的其他部分。我们可以通过给元素指定 `view-transition-name` 来实现。比如，我们像下面这样创建一个新的动画，使主图看起来像是从缩略图放大展开而来：

  


```CSS
::view-transition-old(main-image),
::view-transition-new(main-image) {
    transform-origin: 100% var(--originY, 50%);
}
::view-transition-new(main-image) {
    animation: 400ms ease-out both grow;
}

figure {
    view-transition-name: main-image;
}

@keyframes grow {
    from {
        scale: 0;
    }
    to {
        scale: 1;
    }
}
```

  


简单的解释一下：

  


-   `view-transition-name: main-image` 让 `figure` 元素参与视图过渡动画
-   `::view-transition-old(main-imge),::view-transition-new(main-image)` 伪元素控制动画的起始点，即 `transform-origin`
-   `::view-transition-new(main-image)` 采用 `@keyframes` 定义的放大动画 `grow` ，该动画从 `scale:0` （缩放比率为 `0` ，图片完全不可见）变为 `scale:1` （缩放比率为 `1` ，保持与原图一样的大小）
-   `--originY` 变量动态控制 `transform-origin` ，确保图片从对应的缩略图位置放大展开

  


为了让主图动画从点击的缩略图位置展开，我们需要更新 HTML 和 JavaScript：

  


-   给每个缩略图添加 `data-index` 属性，用于计算 `transform-origin` 位置
-   在 JavaScript 中动态修改 `--originY` 变量

  


更新后的 HTML 结构如下：

  


```HTML
<!-- 大图显示区域 -->
<figure>
    <img src="https://picsum.photos/id/13/800/600" alt="Paul Jarvis" />
</figure>

<!-- 缩略图显示区域 -->
<ul>
    <li>
        <img data-index="0" src="https://picsum.photos/id/13/800/600" alt="Paul Jarvis" />
    </li>
    <!-- 省略其他的 li -->
</ul>
```

  


调整后的 JavaScript:

  


```JavaScript
const thumbnails = document.querySelector(".thumbnails");
const mainImage = document.querySelector("figure img");

const thumbnailHandler = (event) => {
    const clickTarget = event.target;
    const displayNewImage = () => {
        mainImage.src = clickTarget.src;
        document.documentElement.style.setProperty(
            "--originY",
            clickTarget.dataset.index * 25 + 12.5 + "%"
        );
    };

    if (clickTarget.classList.contains("thumbnail")) {
        if (!document.startViewTransition) {
            displayNewImage();
            return;
        }
 
        const transition = document.startViewTransition(() => displayNewImage());
    }
};

thumbnails.addEventListener("click", thumbnailHandler, false);
```

  


现在，当你点击某个缩略图时，主图的放大动画将从对应的缩略图位置展开，而不是直接交叉淡入淡出。这样，整个图片切换体验更加自然流畅：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2fdd9ab3760d4b1296b45f7348e5cf58~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1082&h=754&s=18209010&e=gif&f=95&b=cfdce6)

我们需要使用 JavaScript 给按钮 `button` 添加一个点击事件（`click`），使用户点击按钮时可以切换暗黑模式：

  


```JavaScript
const toggle = document.querySelector('[aria-pressed]')

document.documentElement.dataset.theme = 'dark'

const flip = () => {
    const isPressed = toggle.matches('[aria-pressed=true]')
    document.documentElement.dataset.theme = isPressed ? 'dark' : 'light'
    toggle.setAttribute('aria-pressed', isPressed ? false : true)
}

toggle.addEventListener('click', flip)
```

  


现在看到的效果与之前使用 `input` 创建的效果是相似的：

接下来，我们需要引入 CSS 视图过渡。为了在按钮点击时添加 View Transition，我们添加一个新的函数 `toggleTheme()` ，并且将之前的 `flip()` 函数传给 `startViewTransitions()` ：

  


```JavaScript
const toggleTheme = () => {
    if (!document.startViewTransition) return flip()
    document.startViewTransition(() => flip())
}

toggle.addEventListener('click', toggleTheme)
```

  


此时，它将会引用 CSS 视图过渡的默认动画效果（交叉淡入淡出）：
