## 显式网格 vs. 隐式网格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a7eeee91fde04263a3a1ed8a826dc349~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=720&s=72185&e=jpg&b=535353)

  


如果你曾有过使用 CSS Grid 布局技术的经验，那么就应该知道。在使用 CSS Grid 进行布局时，首先要做的是在想要变成网格容器的元素上显式设置 `display` 的值为 `grid` 或 `inline-grid` 。然后，通过组合使用 `grid-template-columns` 、`grid-template-rows` 和 `grid-template-areas` 来显式定义网格（它被称为**显式网格**）。接下来，再将网格项目放置到网格中。

  


这是使用 CSS Grid 的经典模式，也是大家习已为常的一种方式。然而，在创建网格的时候，还有一种无需显式定义就能创建的方法。我们称之为**隐式网格**。

  


对于大部分 Web 开发者而言，听到这些术语（显式网格和隐式网格）难免会有点困惑。不都是 CSS 创建的网格吗？为什么会有显式网格和隐式网格之分？这里简单的解释一下。

  


当你通过组合 `grid-template-rows` 、`grid-template-columns` 和 `grid-template-areas` 属性定义一个网格时，这个网格通常会有固定数量的网格轨道（网格行和网格列）。我们把这种方式定义的网格称之为显式网格。

  


当你在一个显式网格基础上，网格项目被放置在已确定的网格轨道之外时，网格容器会通过添加隐式网格线生成隐式网格轨道。这些网格线与显式网格一起形成了一个新的网格。此时，这个新网格被称之为隐式网格。

  


用通俗的话来说，浏览器会自动生成额外的网格轨道（网格的行和列），以防止网格项目被放置在已定义的网格之外。

  


我们来看一个简单的示例。假设我们有一段像下面这样的 HTML 结构：

  


```HTML
<div class="grid">
    <div class="grid__item">1</div>
    <div class="grid__item">2</div>
    <div class="grid__item">3</div>
    <div class="grid__item">4</div>
    <div class="grid__item">5</div>
    <div class="grid__item">6</div>
</div>
```

  


接着使用 CSS Grid 对它进行布局：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(3, minmax(0, 1fr));
    grid-template-rows: repaet(3, auto);
    grid-template-areas: 
        "item1 item2 item2"
        "item1 item3 item4"
        "item6 item3 item5";
    
    .grid__item {
        display: grid;
        place-content: center;
      
      
        &:nth-child(1) {
            grid-area: item1;
        }
      
        &:nth-child(2) {
            grid-area: item2;
        }
      
        &:nth-child(3) {
            grid-area: item3;
        }
      
        &:nth-child(4) {
            grid-area: item4;
        }
      
        &:nth-child(5) {
            grid-area: item5;
        }
      
        &:nth-child(6) {
            grid-area: item6;
        }
    }
}
```

  


上面的代码中，我们使用 `grid-template-columns` 和 `grid-template-rows` 创建了一个 `3 x 3` （三行三列）的网格，并且使用 `grid-template-areas` 定义了网格区域。这个网格就是一个显式网格：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fba2281b65e6418d83777c918d549a1c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=485418&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/abeagqv

  


现在，我们在上面这个示例基础稍作调整。我们通过网格线的名称来放置网格项目：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(3, minmax(0, 1fr));
    grid-template-rows: repaet(3, auto);
    grid-template-areas: 
        "item1 item2 item2"
        "item1 item3 item4"
        "item6 item3 item5";
    
    .grid__item {
        display: grid;
        place-content: center;
      
      
        &:nth-child(1) {
            grid-area: 1 / 1 / 3 / 2;
        }
      
        &:nth-child(2) {
            grid-area: 1 / 2 / 2 / 5;
        }
      
        &:nth-child(3) {
            grid-area: 2 / 2 / 4 / 3;
        }
      
        &:nth-child(4) {
            grid-area: 2 / 3 / 3 / 5;
        }
      
        &:nth-child(5) {
            grid-area: 3 / 3 / 3 / 5;
        }
      
        &:nth-child(6) {
            grid-area: 3 / 1 / 4 / 2;
        }
    }
}
```

  


同样的，我们使用 `grid-template-columns` 和 `grid-template-rows` 创建了一个 `3 x 3` 的网格。但是，我们使用 `grid-area` 放置网格项目时，并没有使用 `grid-template-areas` 定义的网格区域名称，而是使用网格线索引号。与此同时，还使用了不在显式网格范围内的网格线索引号，比如列网格线 `5` 。如此一来，初始化的 `3 x 3` （三行三列）的网格就变成了 `3 x 4` ，即三行四列的网格。这个新网格就是所谓的隐式网格：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5cf5b28aad284459b1a12a9e892d74e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=503745&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/YzmOoBr

  


## 自动放置

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/641c53550d724e49b6c9442c9eb674f1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1131&h=707&s=41597&e=jpg&b=f3f3f3)

  


自动放置是 CSS Grid 中的另一个重要概念。它指的是浏览器自动将网格项目放置在网格中的能力。我们需要知道的是，我们并不总是需要为每个网格项目指定位置。浏览器在处理网格项目时，自动放置网格项目会按下面五个步骤来放置网格项目：

  


-   ①：匿名网格项目的生成；
-   ②：使用显式网格线名称（或网格区域）明确放置网格项目；
-   ③：仅明确指定行位置；
-   ④：确定隐式网格中的列数；
-   ⑤：放置剩余的网格项目。

  


在这里不会详细介绍每一步的细节，要是你想深入了解它们，可以移步阅读《[网格项目的放置和层叠](https://juejin.cn/book/7161370789680250917/section/7161623932439625758)》一课。不过，我将通过简单的示例来向大家展示 CSS Grid 中自动放置网格项目的能力。

  


首先，我们来看一个简单的示例。

  


```HTML
<div class="grid">
    <div class="grid__item">1</div>
    <!-- 省略其他的网格项目 -->
    <div class="grid__item">32</div>
</div>
```

  


上面的 `.grid` 容器中包裹了 `32` 个 `.grid__item` 。现在，我们在 `.grid` 元素上仅使用 `grid-template-columns` 定义一个网格：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(8, minmax(0,1fr));
}
```

  


注意，我们只是通过 `grid-template-columns` 属性告诉浏览器，你给我创建一个八列的网格，至于多少行，那就根据网格项目（`.grid__item`）的数量来决定吧。在我们这个示例中，网格项目总数量是 `32` 个，浏览器将会创建一个四行八列的网格。而且网格项目会按照顺序依次排列，因为我们没有在网格项目中显式设置 `grid-column` 、`grid-row` 或 `grid-area` 属性，所以网格项目会自动放置。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d7064cb444b48fca23701f598055ca3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=807017&e=jpg&b=050505)

  


> Demo 地址：https://codepen.io/airen/full/LYwgQMK

  


这就是网格项目自动放置的默认行为。

  


当然，网格布局中的自动放置网格项目是非常强大的。例如，在上面这个示例的基础上，稍微对 `grid-template-columns` 属性的值调整一下，使得网格项目的自动放置能根据容器的尺寸自动换行：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(auto-fit, minmax(min(100% - 2rem, 200px),1fr));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/228593b550334efa8ad832ade7469934~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1662&h=896&s=9302344&e=gif&f=189&b=86a1ed)

  


> Demo 地址：https://codepen.io/airen/full/KKOGoze

  


该特性对于创建响应式 Web 布局非常有用。另外，该技巧还有一个更经典和专业的名称 —— RAM 布局技术。有关于这方面更详细的介绍，[可以点击这里获取](https://juejin.cn/book/7161370789680250917/section/7161624041885958151#heading-0)！

  


上面所展示的这两个示例，有着一个共同的特点，那就是我们在 CSS 中并没有明确地给任何一个网格项目指定位置。然而，我们在实际开发的过程中，有的时候是需要为网格项目指定位置。例如：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(8, minmax(0, 1fr));
    grid-template-rows: repeat(8, 60px);
    
    .grid__item:is(:nth-child(3), :nth-child(5), :nth-child(8), :nth-child(12), :nth-child(17)) {
        grid-column: span 3; /* 合并三列 */
        grid-row: span 2;    /* 合并两行 */
    }
}
```

  


上面代码中的 `grid-template-columns` 和 `grid-template-rows` 会告诉浏览器要创建一个八行八列的显式网格。并且在第三个、第五个、第八个、第十二个和第十七个网格项目上使用了 `grid-column` 和 `grid-row` 属性，它将会使这些网格项目跨区域显式，相当于合并了三列和两行。因此，你现在看到的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/44f2eb38e38a4881a60f740c03b984d9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=1061422&e=jpg&b=060606)

  


正如你所看到的，默认情况下，网格会按照 HTML 中的顺序排列网格项目。当遇到无法放置项目的位置时，它会跳过该位置。在布局中留下“空洞”。我们可以将 CSS 的 `grid-auto-flow` 属性的值设置为 `dense` 关键词改变这种行为，使网格可以忽略 HTML 的顺序。当网格找到适合空位的网格项目时，它会将该网格项目放入空位中——无论它是否在 HTML 中是下一个网格项目。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/04d8f1bce81e49ea8a75bb0c13686386~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1078&h=678&s=979997&e=gif&f=123&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/abeRYJo

  


看看第六个网格项目，它现在移动了，填补了它能适应的第一个空位：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0fad37fa35dc4ffd8445adaa51a620f2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=1260981&e=jpg&b=050505)

  


默认情况下，网格会向前推进，且不会回头检查之前的空位是否可以放入网格项目。但当我们在网格容器上声明 `grid-auto-flow` 的值为 `dense` 时，算法会尝试填满尽可能多的空位，而不考虑 HTML 文档的源顺序。

## 使用隐式网格和自动放置创建 Web 布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9dcfe86a17542b7bd44c04b87170fe9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=5184&h=3456&s=1350375&e=jpg&b=eaeae2)

  


我想，你现在对隐式网格和自动放置的相关概念和特性有所了解了。接下来，我们通过一些实际案例（例如动态侧边栏、九宫格、动态布局等），向大家展示这些特性是如何使我们仅用几行 CSS 代码就可以创建复杂且动态的 Web 布局。

  


### 动态侧边栏

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/25105926a89b4b6987cd8e5b1b739844~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1129&s=340498&e=jpg&b=f95320)

  


有的时候，我们给 Web 页面创建布局时，可能会碰到侧边栏是动态的，比如侧边栏可能是动态生成的（时有时无），也可能希望它能随着浏览器视窗的宽度调整布局。如上图所示，在移动端上，侧边栏与主内容区域位于同一列；而在桌面端，侧边栏和主内容区域位于不同的列。

  


这使得我们需要为 Web 页面考虑不同的布局。当然，我们可以使用不同的方式来实现该布局，但是要使该布局适用于所有不同的场景，对于部分 Web 开发者还是有一定难度的。

  


接下来，我们先从最简单的布局开始。假设 Web 页面没有侧边栏的情景。构建这个布局，你可能会考虑使用下面这个 HTML：

  


```HTML
<main>
    <section><!-- 主内容 --></section>
</main>
```

  


由于侧边栏可能是动态生成的，因此考虑使用 `<main>` 元素来包裹：

  


```HTML
<main>
    <section><!-- 主内容 --></section>
    <aside><!-- 侧边栏，它是动态的，有可能有，有可能无 --></aside>
<main>    
```

  


你可能会将 `<main>` 定义为一个网格容器，并显式使用 `grid-template-columns` 来定义一个网格：

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
}
```

  


使用上面的 CSS 创建了一个只有一列的网格。这个时候，当 `main` 元素中只有主内容 `section` 元素时，它就是一个一行一列的网格。能完美的匹配我们所要的布局。

  


但如果我们决定在 `main` 中添加侧边栏 `aside` 时，你会发现，网格变成了一个一列两行。这是由于当前创建的网格是一个显式网格，网格将不得不自动调整，为 `aside` 元素找到一个位置。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0b64a18213534147aa27f0ad26bdea87~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=967&s=190901&e=jpg&b=e59249)

  


其中 `section` 元素占用了 `main` 容器上显式设置的整例。同时，`aside` 会落到隐式行网格线 `2` 和 `3` 之间。这对于移动端而言，它非常的友好。但我们希望的是，在桌面端（浏览器视窗更大的场景）侧边栏（`aside`）和主内容（`section`）能并列显示，即 `aside` 移动到 `section` 旁边的列中。

  


你可能会想到，通过[媒体查询 `@media`](https://juejin.cn/book/7223230325122400288/section/7257368158451793935) 来调整 `main` 的 `grid-template-columns` 属性的设置，例如：

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
    gap: 1rem;
    
    @media only screen and (width >= 760px) {
        grid-template-columns: minmax(0, 1fr) 220px;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a39ddbbd5a9f4a8083000b9d33d74e2e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1378&h=684&s=3803634&e=gif&f=188&b=f02e35)

  


似乎看上去很完美。但我们需要考虑的是，侧边栏是动态的，它可能会不存在，或者可能会在主内容的左侧。那么，现在这种方法使得我们的布局在桌面端存在一定缺陷，不够灵活：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/74e79df22f5b4cd38cb3d394aa7e40a7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1386&h=682&s=2888810&e=gif&f=449&b=f02e35)

  


不难发现，当 `aside` 元素不存在时，主内容 `section` 并无法自动扩展到屏幕的右侧边缘，因为我们的定义的网格的列数始终是两例。另外，当我们在 HTML 中将 `<aside>` 元素放置在 `<section>` 元素前面时，侧边栏和主内容的宽度不相匹配。熟悉 CSS Grid 布局的开发者可能会说，我们可以通过 `order` 属性或 `grid-column` 来调整 `aside` 的位置。这样做的确可以，但我们需要使用更多的 CSS 代码，而且不得不面临宽度不匹配的现象。

  


你可能会问？那有没有更好的方法呢？答案是有的。我们依旧在 `main` 元素上使用 `grid-template-columns` 定义一个一列一行的网格。然后，仅在桌面端的时候将 `aside` 移动到 `section` 旁边的列中：

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
    gap: 1rem;
}

@media only screen and (width >= 760px) {
    aside {
        grid-column-start: 2;
    }
}    
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4c85d7dbe50d40329b5dd4b836cc6a10~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1316&h=684&s=3647985&e=gif&f=182&b=f73a2d)

  


这种方式使你的布局既能符合移动端的布局，又能符合桌面端的布局。

  


你是不是有点好奇，我们只是使用 `grid-template-columns` 创建了一个一行一列的网格，并没有在 `main` 网格容器上声明过第二列，怎么浏览器就自动为网格新增了一列呢？这个答案很简单，这里应用了之前所介绍的隐式网格的特性——**当网格项目被放置在这显式网格边界之外时，网格容器会通过向网格添加隐式网格线生成隐式网格轨道**。

  


这个强大的特性让我们能够实现动态布局。如果我们只有 `<section>` 元素，我们得到的只是一列；如果我们添加一个 `<aside>` 元素，就会自动创建一个额外的列来放置它。这就完美的满足 `<aside>` 是动态的需求，还能同时使布局在移动端和桌面端有一个完美的呈现：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/05e303fa870147f4ab39eede3664398f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=686&s=1231948&e=gif&f=178&b=f73a2d)

  


不过，这种方案有一个缺陷，那就是和 HTML 文档的源顺序有着紧密的关系。如果 `<aside>` 元素出现在 `<section>` 之前，我们就不得不调整应用在 `aside` 上的 CSS 代码：

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
    gap: 1rem;
}

@media only screen and (width >= 760px) {
    aside {
        grid-column-end: -2;
    }
}  
```

  


这样就会在网格的开头创建隐式列，而不是像之前的代码那样将隐式列放在末尾。这样做，虽然可以创建一个位于右侧或左侧的边栏，但它与 HTML 结构有着紧密的耦合——需要根据不同的 HTML 结构来调整 CSS 代码。然而，`<aside>` 是动态插入，它有可能是在 `<section>` 之前，也有可能是在它之后。因此，我们还需要一种更灵活的布局方案，可以使布局变得更灵活。

  


这种更简单，更灵活的方案就是将网格容器 `main` 的 `grid-auto-flow` 属性值设置为 `column` ，告诉浏览器将所有隐式轨道按列方向流动：

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
    gap: 1rem;
    
    @media only screen and (width >= 760px) {
        grid-auto-flow: column;
    }
}
```

  


现在不需要再给 `<aside>` 指定 `grid-column-start` 就可以将侧边栏放到主内容 `<section>` 的右侧了！

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9988385b8e94829bfe669255223ea66~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1358&h=684&s=3624482&e=gif&f=197&b=f6321c)

  


> Demo 地址：https://codepen.io/airen/full/XWvyZEZ

  


事实上，任何我们随时决定添加到网格中的其他网格项目，现在都会沿着列方向流动，每个网格项目都会位于自己的隐式网格轨道上。这对于事先不确定网格项目数量的情况来说非常有用：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7d14abdc96b647899ae3f135aa7f6cda~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1366&h=688&s=2496010&e=gif&f=186&b=f6321c)

  


话虽如此，如果我们希望将 `<aside>` 放置在 `<section>` 的左侧，仍然需要使用 `grid-column-end` 属性：

  


```CSS
@media only screen and (width >= 760px) {
    aside {
        grid-column-end: -2;
    }
}
```

  


即便是你将调整 HTML 结构也需要这么设置 CSS，否则 `<aside>` 会占据显式列，导致 `<section>` 被推到显式网格之外，进而强制它占用隐式列：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4042dc2009f641f1b32feba615b92b62~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1636&h=790&s=4074097&e=gif&f=279&b=f6321c)

  


> Demo 地址：https://codepen.io/airen/full/QWeJxEj

  


也就是说，我们将 `aside` 的 `grid-column-end` 设置为 `-2` 时，`<aside>` 在 HTML 中的顺序就不再重要了。通过将 `<aside>` 重新分配到其他位置，我们让 `<section>` 元素可以占据显式列。

  


这种方案还可以适用于创建多个侧边栏，比如你的 Web 页面有两个侧边栏，其他一个位于主内容左侧，另一个位于主内容右侧，你只需要将左侧边栏元素的 `grid-column-end` 设置为 `-2` 即可：

  


```HTML
<main>
    <section><!-- 主内容 --></section>
    <aside><!-- 左侧边栏 --></aside>
    <aside><!-- 右侧边栏 --></aside>
<main> 
```

  


```CSS
main {
    display: grid;
    grid-template-columns: minmax(0, 1fr);
    gap: 1rem;
    
    @media only screen and (width >= 768px) {
        grid-auto-flow: column;
        
        aside:nth-of-type(1) {
            grid-column-end: -2;
       }
    }
}       
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73f6e697570d4e19b4fae357b9c90ee8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1312&h=684&s=8895508&e=gif&f=639&b=f6321c)

  


> Demo 地址：https://codepen.io/airen/full/eYqQKvR

  


怎么样，完美吧！是不是很简单。

  


### 九宫格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b00c0f2b6a4a4d458e35da4b6b58ecaa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1006&h=1098&s=747509&e=jpg&b=f1eff7)

  


九宫格布局是 Web 中常见的一种布局模式。接下来，我们来创建一个简单的九宫格布局用于展示图片，其中包含一张大图和几张缩放图，并且缩放图放在大图的旁边或下方：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8ad7a71531834f6e894990423a270dd6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2408&h=967&s=1383009&e=jpg&b=a1cfcc)

  


我猜大部分 Web 开发者在构建诸如上图所示的九宫格布局时，首先会想到的是使用 `grid-template-columns` 和 `grid-template-rows` 创建一个 `3 x 3` 的显式网格：

  


```CSS
.figures {
    display: grid;
    grid-template-columns: repeat(3, minmax(0, 1fr));
    grid-template-rows: repeat(3, minmax(0, 1fr));
    gap: 10px;
}
```

  


然后将大图在网格中跨越多个单元格：

  


```CSS
img:first-of-type {
    grid-area: 1 / 1 / 3 / 4;
}

/* 或者 */
img:first-of-type {
    grid-area: 1 / 1 / 4 / 3;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/69501714cfff42ca84fd828a46f6fc89~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1056&s=1121651&e=jpg&b=a2d0cb)

  


这种方案是不错，而且只需要调整大图在网格中的位置，就可以获得不同变体的布局：

  


```CSS
img:first-of-type {
    grid-area: 2 / 1 / 4 / 4;
}

/* 或者 */
img:first-of-type {
    grid-area: 1 / 2 / 4 / 4;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/227d1b5e4b504bf18fd95ecbd91056cd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1056&s=1122069&e=jpg&b=a2d0cd)

  


> Demo 地址：https://codepen.io/airen/full/xxvQzvm

  


但我们还有更简单更灵活的方案来实现类似的效果。我们无需在网格容器上显式设置 `grid-template-columns` 和 `grid-template-rows` ，所要做的只是使用 `grid-area` 属性，将大图放置在多个列和行中：

  


```CSS
.figures {
    display: grid;
    gap: 10px;
    
    img:first-of-type {
        grid-area: span 2 / span 3;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2fc79c8134e6438d9bea1ae966286f4b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1197&s=738392&e=jpg&b=050505)

  


只需一行代码就能实现这个效果，是不是感到很惊讶！让我们仔细分析一下，这比你想象的要简单得多。

  


首先，`grid-area` 是一个简写属性：

  


```CSS
.grid-area {
    grid-area: <grid-row-start> / <grid-column-start> / <grid-row-end> / <grid-column-end>;
}
```

  


需要知道的是，`grid-area` 不仅仅可以应用 `grid-template-areas` 定义的网格区域名称，还可以使用网格线。也就是说，`grid-area` 还能用于定义网格项目在网格中的起止位置。在我们这个示例中：

  


```CSS
img:first-of-type {
    grid-area: span 2 / span 3;
}

/* 等同于 */
img:first-of-type {
    grid-row-start: span 2;
    grid-column-start: span 3;
    grid-row-end: auto;
    grid-column-end: auto;
}
```

  


这表示网格中的第一个网格项目（第一个 `img` 元素，对应的就是大图）将会跨越两行三列。但由于我们没有在 `.figures` 网格容器上定义任何行和列，浏览器会自动为我们创建这些行和列：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/889a9d0836e9447d9b706c45e6a380f9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1056&s=703973&e=jpg&b=060606)

  


这实际上是在 HTML 中将第一图片设置为占据一个 `2 x 3` 的网格（两行三列）。这意味着其他图片将自动放置在这三个列中，而无需额外指定任何内容。

  


总结一下，我们告诉浏览器第一张图片需要占据两行三列，但在网格容器上并没有显式定义这些行和列。浏览器会自动为我们设置这些行和列。结果是，HTML 中的其他图片会使用相同的列排列到位。

  


这一切都是通过一行 CSS 代码实现！这就是**隐式网格**和**网格项目自动放置**的强大之处。

  


上面这个示例，我们演示的是第一张图片占据两行三列。假设，你现在需要将第一张图片占据三行两例，其他的图片放置在大图的右侧或左侧，要如何处理呢？这也很简单，与之前 `<aside>` 放置在 `<section>` 左侧或右侧一样，只需要将网格容器的 `grid-auto-flow` 设置为 `column` 即可。

  


```CSS
.figures {
    display: grid;
    gap: 10px;
    grid-auto-flow: column;
    
    img:first-of-type {
        grid-area: span 3 / span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2c6359327da0424c8dedb0dda1e4628a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1056&s=642508&e=jpg&b=050505)

  


按照同样的方式，只需要调整第一张或最后一张图片的 `grid-area` 属性的值，我们将获得不同布局效果：

  


```CSS
.figures {
    display: grid;
    gap: 10px;
}
  
.figures:nth-child(1) {
    img:first-of-type {
        grid-area: span 2 / span 3;
    }
}
  
.figures:nth-child(2) {
    img:last-of-type {
        grid-area: span 2 / span 3;
    }
}
  
.figures:nth-child(3) {
    grid-auto-flow: column;
    
    img:first-of-type {
        grid-area: span 3 / span 2;
    }
}
  
.figures:nth-child(4) {
    grid-auto-flow: column;
    
    img:last-of-type {
        grid-area: span 3 / span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9853e1f529fa4ee8be070e55b7595ceb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2061&s=2249027&e=jpg&b=a2d0cd)

  


> Demo 地址：https://codepen.io/airen/full/abeQKKK

  


或许有的同学会问，这个方案是不是仅适用于第一张图片或最后一张图片跨越多行和多列呢？答案是否定的，我们可以将任何一张图片作为大图，使其跨越多行和多列。唯一不同的是，我们需要将 `grid-auto-flow` 的值设置为 `dense` 。如果创建的隐式轨道是行，则需要设置为 `row dense` ；如果创建的隐式轨道是列，则需要设置为 `column dense` ：

  


```CSS
.figures {
    display: grid;
    gap: 10px;
    grid-auto-flow: row dense;
}
  
.figures:nth-child(1) {
    img:nth-child(1) {
        grid-area: span 2 / span 3;
    }
}
  
.figures:nth-child(2) {
    img:nth-child(2){
        grid-area: span 2 / span 3;
    }
}
  
.figures:nth-child(3) {
    grid-auto-flow: column dense;
    
    img:nth-child(1) {
        grid-area: span 3 / span 2;
    }
}
  
.figures:nth-child(4) {
    grid-auto-flow: column dense;
    
    img:nth-child(2) {
        grid-area: span 3 / span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a9cc9422a14e4bceb42188ad1ad7dc71~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2061&s=2341723&e=jpg&b=090909)

  


> Demo 地址：https://codepen.io/airen/full/QWeJVZe

  


当然，我们还可以通过调整一个值来轻松更新网格，以适应更多图片。比如，我们将可以将大图设置为跨越三列三列，或者两行四列等：

  


```CSS
.figures {
    display: grid;
    gap: 10px;
    grid-auto-flow: row dense;
}
  
.figures:nth-child(1) {
    img:nth-child(1) {
        grid-area: span 3 / span 3;
    }
}
  
.figures:nth-child(2) {
    img:nth-child(2){
        grid-area: span 2 / span 4;
    }
}
  
.figures:nth-child(3) {
    grid-auto-flow: column dense;
    
    img:nth-child(1) {
        grid-area: span 3 / span 3;
    }
}
  
.figures:nth-child(4) {
    grid-auto-flow: column dense;
    
    img:nth-child(2) {
        grid-area: span 4 / span 4;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9f00a3a2d3a549adb00c8027df3ccc1c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=611&s=1060885&e=jpg&b=070707)

  


> Demo 地址：https://codepen.io/airen/full/abeQapN

  


### 动态布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f30f528558ba49388d446cf4cba1e9aa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1146&h=492&s=1010374&e=gif&f=307&b=f5f7fb)

  


接下来，我们将使用 CSS Grid 来构建一个更为复杂的布局——根据元素的数量来决定网格的配置。在这个示例中，我们可以有 `1 ~ 4` 个或者更多元素，网格会根据元素的数量进行调整，完美的适应元素的排列，而不会留下任何尴尬的空隙或缺失的空间。

  


我们先来看一种相对传统的方案。

  


```HTML
<main class="grid">
    <div class="box"></div>
</main>
```

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
}
```

  


当网格容器中只有一个元素时，我们不需要做任何事情。该元素会自动拉伸以填充网格中唯一创建的行和列。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/28f6009da9034d85b2bbf252a3b59efe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1197&s=245007&e=jpg&b=060606)

  


这是因为，`.grid` 元素是一个网格容器，在没有显式设置 `grid-template-columns` 和 `grid-template-rows` 时，浏览器将会根据其子元素个数来创建隐式的网格轨道。目前只有一个子元素，所以它会创建一个一行一列的网格。并且在没有调整[网格项目对齐方式](https://juejin.cn/book/7161370789680250917/section/7161624114308841509)的前提之下，网格项目会自动拉伸以填充网格中唯一创建的行和列。

  


但是，当我们添加第二个网格项目时，默认情况之下，浏览器会创建一个一列两行的隐式网格。我们需要使用 `grid-column-start: 2` 创建另一个隐式列：

  


```CSS
.box {
    &:nth-child(2) {
        grid-column-start: 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/848a869b07484721a30988906715099b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1250&h=720&s=1162480&e=gif&f=140&b=030303)

  


当我们添加第三个网格项目时，在没有显式改变网格流动方向时，浏览器将会创建第二个隐式行，如此一来，网格就变成了一个两行两列的隐式网格。因此，默认之下，第三个网格项目会位于第二行第一列：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e0be59f7c0848b3b53d10c1b142b3e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1248&h=720&s=817011&e=gif&f=103&b=030303)

  


我们更希望它（第三个网格项目）占列两列的宽度，但同时希望网格有第四个网格项目时，它又只占据一列。所以，我们不能直接在第三个网格项目上使用 `grid-column-start: span 2` ，但只有它是最后一个网格项目时才这样。为了实现满足这个需求，我们需要像下面这样来设置 CSS：

  


```CSS
.box {
    &:nth-child(2) {
        grid-column-start: 2;
    }
    &:nth-child(3):last-child {
        grid-column-start: span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/15a084c3f0b24e62834e9354f1acae87~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1246&h=720&s=2228255&e=gif&f=296&b=040404)

  


与此同时，网格有第四个网格项目时，第三个网格项目不会再占据两行，而且还不需要为第四个网格项目设置任何 CSS 代码，就能动态实现我们所需要布局效果。这是因为，当有第四个网格项目时，那么第三个网格项目就不是最后一个网格项目，其对应的 `:nth-child(3):last-child` 选择器设置的 `grid-column-start: span 2` 就无法再应用到第三个网格项目上。此时，浏览器会根据自动放置的算法来放置网格项目。

  


但是，当我们为网格新增更多的网格项目时，你会发现第五个、第七个、第九个等网格项目会出现类似于第三个网格项目的情景：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c0e09e9c33e143d3aec34785a1c04da2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1248&h=712&s=3233597&e=gif&f=310&b=040404)

  


换句话说，当网格项目有五个、七个或九个（甚至更多时），并且它们又是最后一个网格项目时，我们希望它能像第三个网格项目那样占据两列。可能有的同学会说，那还不简单，我们可以像设置第三个网格项目的方式来设置 CSS：

  


```CSS
.box {
    &:nth-child(2) {
        grid-column-start: 2;
    }
    
    &:is(:nth-child(3),:nth-child(5),:nth-child(7),:nth-child(9)):last-child {
        grid-column-start: span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b2d4c4620acb4c20a5beeb1c1a8aafbc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1236&h=712&s=2460435&e=gif&f=243&b=040404)

  


这个思路并没有错，但问题是，我们并不知道会有多少个网格项目添加到网格中。也就是说，我们还是需要一个更好的方案来解决这个问题。大家仔细观察，我们所说的第 `3` 、`5` 、`7` 、`9` 和 `11` 等，是可以使用一个表达式来描述的，即 `2n + 1` ：

  


-   当 `n=0` 时，那么 `2n + 1 = 1` ，可以匹配到第一个网格项目
-   当 `n=1` 时，那么 `2n + 1 = 3` ，可以匹配到第三个网格项目
-   当 `n=2` 时，那么 `2n + 1 = 5` ，可以匹配到第五个网格项目

  


以此类推，`:nth-child(2n+1)` 就可以让我们选中第 `1` 、`3` 、`5` 以及 `2n + 1` 个网格项目，这也正是我们所需要的。这样一来，将其与 `:last-child` 组合在一起，就能符合我们的需求。因此，只要将 CSS 调整为像下面这样，我们就可以拥有一个更灵活的动态布局：

  


```CSS
.box {
    &:nth-child(2) {
        grid-column-start: 2;
    }
    &:nth-child(2n + 1):last-child {
        grid-column-start: span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e7de24d021a348e28b07d7cbf872fc2e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1244&h=718&s=5204337&e=gif&f=443&b=030303)

  


> Demo 地址：https://codepen.io/airen/full/ExqOJLg

  


你开始明白这个技巧了吗？我们根据元素的数量（使用 `:nth-child`）给浏览器提供特定的指令，就可以完全改变布局。

  


```CSS
.box {
    &:nth-last-child(2n+1):nth-child(2){
        grid-column-start: 2;
    }
    
    &:nth-last-child(2n+1):first-child{
        grid-column-start: span 2;
    }
}
```

### 网格模式

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/006f336d6b304a2891a8038519aa3135~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2400&h=1260&s=261771&e=jpg&b=45bcac)

  


你可能在平时构建 Web 布局时见过那种两列布局，其中一列比另一列宽，并且每一行交替放置这些列：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6e5bc5d7d5fd4579bc6e3a0d3df31245~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1197&s=285198&e=jpg&b=050505)

  


这种布局在不知道具体内容的情况下可能很难实现，但 CSS Grid 的自动放置功能使其变得相对简单。

  


```HTML
<div class="grid">
    <div class="grid__item"></div>
    <!-- 省略其他 -->
    <div class="grid__item"></div>
</div>
```

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;

    :nth-child(4n + 1) {
        grid-column: span 2;
    }
    :nth-child(4n + 4) {
        grid-column: 2 / span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1582434f737043c5a1da42c959a3ef04~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=922&s=218294&e=jpg&b=060606)

  


> Demo 地址：https://codepen.io/airen/full/oNKQraj

  


示例代码并不复杂。不过，我们首先要做的是识别模式。问问自己：“在多少个元素后，模式应该重复？”在这个例子中，每四个元素后模式就会重复。所以，暂时我们只考虑四个元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/230fb68f86d640349495aea44d97bbfa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=922&s=173685&e=jpg&b=050505)

  


现在，让我们定义网格并使用 `:nth-child` 选择器设置一般模式，以交替放置网格项目：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;

    :nth-child(4n + 1) {
        /* CSS ... */
    }
    
    :nth-child(4n + 2) {
        /* CSS ... */
    }
    
    :nth-child(4n + 3) {
        /* CSS ... */
    }
    
    :nth-child(4n + 4) {
        /* CSS ... */
    }
}
```

  


我们说过，在这个示例中，我们的模式是每四个元素重复一次。因此，我们使用 `4n + x` ，其中 `x` 代表的是 `1 ~ 4` 。我们从 `n = 0` 开始：

  


```
n = 0
4n + 1 = 4 x 0 + 1 = 1 👉 第一个网格项目
4n + 2 = 4 x 0 + 2 = 2 👉 第二个网格项目
4n + 3 = 4 x 0 + 3 = 3 👉 第三个网格项目
4n + 4 = 4 x 0 + 4 = 4 👉 第四个网格项目

n = 1
4n + 1 = 4 x 1 + 1 = 5 👉 第五个网格项目
4n + 2 = 4 x 1 + 2 = 6 👉 第六个网格项目
4n + 3 = 4 x 1 + 3 = 7 👉 第七个网格项目
4n + 4 = 4 x 1 + 4 = 8 👉 第八个网格项目

n = 2
4n + 1 = 4 x 2 + 1 = 9  👉 第九个网格项目
4n + 2 = 4 x 2 + 2 = 10 👉 第十个网格项目
4n + 3 = 4 x 2 + 3 = 11 👉 第十一个网格项目
4n + 4 = 4 x 2 + 4 = 12 👉 第十二个网格项目
```

  


依此类推。我们就可以在第五个元素、第九个元素等处重复模式。现在，我们可以这样设置每个网格项目：

  


-   第一个网格项目需要占用两列，并从第一列开始（`grid-column: 1 / span 2`）
-   第二个网格项目需要放置在第三列（`grid-column-start: 3`）
-   第三个网格项目需要放置在第一列（`grid-column-start: 1`）
-   第四个网格项目需要占用两列，并从第二列开始（`grid-column: 2 / span 2`）

  


如此一来，我们可以像下面这样完善上面的 CSS 代码：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;

    :nth-child(4n + 1) {
        grid-column: 1 / span 2;
    }
    
    :nth-child(4n + 2) {
        grid-column-start: 3;
    }
    
    :nth-child(4n + 3) {
        grid-column-start: 1;
    }
    
    :nth-child(4n + 4) {
        grid-column: 2 / span 2;
    }
}
```

  


注意，上面代码中的 `grid-auto-rows` 和 `grid-auto-columns` 分别用于定义隐式行和列网格轨道的尺寸。

  


另外，我们还可以对上面的代码做一些优化。具体来说，就是依靠网格的自动放置功能来优化 CSS 代码。例如，我们可以将 `grid-column: 1 / span 2` 更新为 `grid-column: span 2` ，因为默认情况下，浏览器会将第一个网格项目放入第一列。我们还可以删除用于第三个网格项目的 CSS 代码。

  


通过放置第一个、第二个和第四个网格项目，网格会自动将第三个网格项目放置在正确的位置。优化之后的 CSS 代码如下：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;

    :nth-child(4n + 1) {
        grid-column: span 2;
    }
    
    :nth-child(4n + 2) {
        grid-column-start: 3;
    }
    
    :nth-child(4n + 4) {
        grid-column: 2 / span 2;
    }
}
```

  


事实上，我们还可以做得更好，可以把应用于第二个网格项目的 CSS 代码也删除掉。因为，我们将第四个网格项目放在第二列，并允许它占用两个完整的列，就相当于创建了第三个隐式列，从而使网格具有三列。同时，第四个网格项目不能放在第一行，因为第一个网格项目也占用了两列，所以它会流向下一行。如此一来，网格的第一行和第二行都留有一个空列。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9b97913025940cf872eb825dccb4090~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=853&s=251228&e=jpg&b=050505)

  


最终，浏览器会自动将第二个网格项目和第三个网格项目放置在那些空位上。所以，CSS 的代码就变得更简单了：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;

    :nth-child(4n + 1) {
        grid-column: span 2;
    }
    
    :nth-child(4n + 4) {
        grid-column: 2 / span 2;
    }
}
```

  


CSS Grid 布局就是这么强大，只需几行简单的代码就可以创建一个非常酷且非常灵活的网格模式。当然，对于不熟悉的 CSS Grid 的开发者，这些代码可能理解起来会有点困难，但当你了解了它们，并通过一些练习获得相关技巧之后，你就会更喜欢上 CSS Grid。

  


我们再反过来思考。如果我们知道网格的列数，那么使用 `grid-template-columns` 来定义一个显式网格，将会更直观一些：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(3, minmax(0, 1fr));
    grid-auto-rows: 100px;
    
    :is(:nth-child(4n + 1), :nth-child(4n + 4)) {
        grid-column: span 2;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2fab438822654f90bc15a9b2d7d9e054~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=922&s=218294&e=jpg&b=060606)

  


> Demo 地址：https://codepen.io/airen/full/ZEgVzGp

  


正如你所看到，最终结果是一致的。虽然使用这种方式对于 Web 开发者而言更为直观，但前提是你需要知道具体的列数。另外，在这节课我们主要目的是探讨 CSS Grid 的隐式网格和自动放置网格项目的功能，希望在这个探索过程中能获取更多关于 CSS Grid 布局思路和技巧。

  


为了能加强大家对这个技巧的理解和掌握，我们现来看另一种网格模式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/38fc2969e5fd4d5195eab306bde5df15~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1538&s=284913&e=jpg&b=050505)

  


这个网格模式每六个网格项目重复一次。第一和第六个网格项目各需占用两个完整的行。如果我们放置第一个和第六个网格项目，似乎不需要动其他网格项目，所以我们试试以下 CSS 代码：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;
   
    :nth-child(6n + 1) {
        grid-area: span 2 / 1;
    }
    
    :nth-child(6n + 6) {
        grid-area: span 2 / 2; 
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3867893a770c45589a2a43d3f456c996~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1836&s=340813&e=jpg&b=050505)

  


与我们所期望的效果不太相符。我们需要将第五个网格项目放在第一列。否则，网格会自动将它放在第二列：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;
   
    :nth-child(6n + 1) {
        grid-area: span 2 / 1;
    }
    
    :nth-child(6n + 5) {
        grid-column-start: 1;
    }
    
    :nth-child(6n + 6) {
        grid-area: span 2 / 2; 
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e1cd135243204e6aa7420665c219c9c0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1836&s=341786&e=jpg&b=050505)

  


即便如此操作，网格还是留有空洞。第六个网格项目需要往上移动一行，占用网格留下来的空洞。你可能会想到将第六个网格项目的起始行（`grid-row-start`）设置为 `3` ：

  


```CSS
.grid {
    :nth-child(6n + 6) {
          grid-area:3 / 2/ span 2; 
    }
}
```

  


但这样做不起作用，因为它会强迫所有 `6n + 6` 的网格项目放在同一个位置，导致布局混乱：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/73710e0cbe604ff1b9170f26d5f6e8fb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1836&s=320322&e=jpg&b=050505)

  


真正的解决方案是保持第六个网格项目的初始定义，并在网格容器 `.grid` 上设置 `grid-auto-flow` 的值为 `dense` ，使网格项目自动来填满网格的空洞：

  


```CSS
.grid {
    display: grid;
    gap: 1rem;
    grid-auto-rows: 100px;
    grid-auto-columns: 1fr;
    grid-auto-flow: dense;
   
    :nth-child(6n + 1) {
        grid-area: span 2 / 1;
    }
    
    :nth-child(6n + 5) {
        grid-column-start: 1;
    }
    
    :nth-child(6n + 6) {
        grid-area: span 2 / 2; 
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f302b134b91044f8aaa8872e7276699b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1836&s=345653&e=jpg&b=050505)

## 小结

  


CSS Grid 布局技术已经有很多年了，我也在《[现代 Web 布局](https://s.juejin.cn/ds/iAU9jxEK/)》中花了很多节课专门介绍 CSS Grid 布局相关的知识。但仍有许多鲜为人知且不常用的技巧没有被广泛讨论。隐式网格和自动放置功能就是其中两个！理解和掌握这两个特性，有助于我们更进一步掌握 CSS Grid 布局技术。
