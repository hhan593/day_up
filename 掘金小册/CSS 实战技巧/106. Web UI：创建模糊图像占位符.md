图像是 Web 应用中最为常见的媒介，为了让图像在 Web 上尽可能快地加载，Web 开发者会对图像加载做一些性能相关的优化，比如图像懒加载。在做图像懒加载时，思考图片加载前该渲染什么内容是非常重要的，因为这直接影响用户对性能的感知体验。

  


其中一个最有趣的探索方向就是占位符：在图像尚未加载完成时，该显示什么内容？常见的策略有留白、使用轮廓图或默认图像、使用主色块和模糊图像等。今天，我将与大家分享另一种纯 CSS 技术，可以生成模糊的图像占位符（LQIP），而无需在标记中增加杂乱内容——只需要一个自定义属性。这种技术与其他主流解决方案相比，能生成一个非常模糊的图像占位符。但关键是它简洁且不具侵入性！无需包装元素、带有长字符串数据的属性或任何时 JavaScript。

  


我们开始吧！

  


## 图像占位符的几种常见类型

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2454850caf41455083be7cfd5254b95c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2560&h=1440&s=18032&e=webp&b=feefd4)

  


之前提到过，在做图像懒加载时，思考加载前该渲染什么内容是非常重要的，因为这直接影响用户对性能的感知体验。常见的策略主要有以下几种：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a39364ac65a74978a0065012ab4ca2b4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1024&s=12541&e=avif&b=252525)

  


-   **留白**：直接为图像预留出空间。在响应式设计中，这样做可以避免布局跳动，而布局跳动不仅影响用户体验，还会带来性能损耗——浏览器需要反复重新计算布局（即产生生排和重绘）。提前为图像预留空间可以避免这种情况
-   **使用轮廓图或默认图像**：比如在展示用户头像时，如果图片还没加载出来，或才加载失败，或者用户根本没有上传头像，可以显示一个默认的轮廓图。这类图像通常是矢量图（SVG），文件小，非常适合内联到页面中
-   **使用主色块**：从目标图像中提取一个颜色（如主色、最鲜艳的颜色等），将其作为背景色，用于填充占位区域。这样能让从“无图”到“图像加载完成”的过渡显得更自然、顺滑
-   **模糊图像**：先显示一张体积非常小（像素和文件体积都很小）的图片，再切换成原图。为了减少失真（掩盖马赛克感），通常会放大并模糊这张小图像，从而获得比较自然的过渡效果

  


当然，还有很多其他变种，Web 开发者正在不断一发新的占位技巧。比如有一种变体，是用渐变来代替纯色背景。渐变相比纯色能更好地还原原图的色调，带来的文件体积增量也非常小。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/461a7a27d21f4851be0e8bf5b796814e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=726&h=480&s=14168&e=avif&b=486684)

  


  


除此之外，还有一种新的**低质量图像占位符（LQIP）** 技术，与 **SQIP** 技术非常相似，都是为了解决图片加载前的“白屏”或“跳动”问题，通过“先显示一个简易版本，再加载完整高清图”来提升用户体验。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6b8e2159dcf04845b0bc63ef6389b84f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3212&h=1384&s=283745&e=jpg&b=9fabaf)

  


## LQIP 概述

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6d59108150ef47d6a42c31f24a99b52f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=675&s=16757&e=avif&b=383052)

  


LQIP（Low Quality Image Placeholder）是一种低质量图像占位符技术，它是一捉在高质量图像加载前，先显示低分辨率、模糊版本的技术。

  


实现 LQIP（低质量图像占位符）有许多不同的技术，例如：

  


-   使用超低分辨率的 WebP 或 JPEG 图像（甚至是[“截头”的 JPEG 图像](https://engineering.fb.com/2015/08/06/android/the-technology-behind-preview-photos/)）
-   通过优化后的 SVG 形状拼接来近似原图内容（即 [SQIP 技术](https://github.com/axe312ger/sqip)）
-   直接应用离散余弦变换生成模糊占位（[BlurHash 方法](https://blurha.sh/)）
-   当然还有传统的渐进式 JPEG（progressive JPEG）和交错 PNG（interlaced PNG）

  


这些方法各有优劣，Web 开发者可根据项目需求选择合适的实现方式。

  


在技术端，通常会选择纯 CSS 或 JavaScript 来实现。比如，通过内联CSS的方式，简单地填充图像主色作为占位符：

  


```HTML
<img width="200" height="150" style="background-color: red" />
```

  


这种解决方案的优势是能立即渲染——即使是使用 `background-image: url(..)` 搭配一个 `dataURL` 也没问题！

  


纯 CSS 解决方方案的一个大缺点是，通常你需要在 HTML 标记中堆砌冗长的内联样式或令人厌烦的 `dataURL`。比如填充一个渐变的图像：

  


```HTML
<img width="200" height="150" style="background: 
    linear-gradient(45deg, #f4a261, transparent),
    linear-gradient(-45deg, #e76f51, transparent),
    linear-gradient(90deg, #8ab17d, transparent),
    linear-gradient(0deg, #d62828, #023047);
" />
```

  


当然，你也可以使用 JavaScript 来实现，比如 **[BlurHash](https://blurha.sh/)** ****技术，通过将图像数据压缩为一个简短的 `base-83` 字符串来最小化标记，然后使用 JavaScript 来解决和渲染这些数据：

  


```
<img width="200" height="150" src="…" data-blurhash="LEHV6nWB2yk8pyo0adR*.7kCMdnj" />
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/06e6f869f4f14c8abd685edbe754edb3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1282&h=538&s=2799340&e=gif&f=81&b=f8f3f2)

  


> BlurHash：https://blurha.sh/

  


如果我们有一种技术——仅通过 CSS 解码 BlurHash，那么是不是会更好一些呢？

  


## 纯 CSS 解码 BlurHash

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bcbe6b74fcd3408f95903d1cddd8cfe8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2464&h=816&s=238595&e=png&b=fdfcfc)

  


与 [BlurHash](https://blurha.sh/) 技术不同，我们不能使用字符串编码。因到，到目前为止，CSS 还没提供可以直接操作字符串的函数，因此字符串编码行不通。

  


庆幸的是，[@Kalabasa 提出了一种新的编码方式](https://leanrada.com/notes/css-only-lqip/)——**整数类型的哈希编码**方法！即**把多个数值编码进一个整数里（** **[位打包 bit packing](https://en.wikipedia.org/wiki/Bit_manipulation)** **），然后在纯 CSS 中“解码”这些数值（即提取出来）** 。简单的解释一下。比如，你有三个数字：

  


-   `a = 5` （用 `3` 位可以表示，范围 `0~7`）
-   `b = 3` （用 `2` 位可以表示，范围 `0~3`）
-   `c = 2` （用 `2` 位可以表示，范围 `0~3`）

  


你可以这样打包成一个整数：

  


```
packed = (a << 4) + (b << 2) + c
       = (5 << 4) + (3 << 2) + 2
       = (80) + (12) + 2 = 94
```

  


这个 `94` 就是 `a` 、`b` 和 `c` 三个值打包之后的结果，存储在一个整数中。

  


神奇的是，我们可以在纯 CSS 中解码——提取出 `a` 、`b` 和 `c` 的值！虽然 CSS 没有“位操作”这种语法，但可以用数学方法模拟。解码的关键是位移（Bit Shifting）和位掩码（Bit Masking）：

  


-   位移：想象你要提取某几位，就要先把它们“移”到最右侧，可以通过除法和向下取整操作来完成，正好与 CSS 中的 ` calc(x / y)` 和 `round(down, n)` 函数相匹配
-   位掩码：提取右侧的特定位，用的是“模运算”，正好与 CSS 中的取模函数 `mod(a, b)` 相匹配

  


例如：

  


```CSS
* {
    /* 二进制表示：0b11_00_001_101 */
    --packed-int: 781;

    --bits-9-10: mod(round(down, calc(var(--packed-int) / 256)), 4); /* 取出第9到10位，值为 3 */
    --bits-7-8: mod(round(down, calc(var(--packed-int) / 64)), 4);   /* 取出第7到8位，值为 0 */
    --bits-4-6: mod(round(down, calc(var(--packed-int) / 8)), 8);    /* 取出第4到6位，值为 1 */
    --bits-0-3: mod(var(--packed-int), 8);                           /* 取出第0到3位，值为 5 */
}
```

  


简单的解释一下。我们将要解码的数字 `781`（对应的二进制是 `0b11_00_001_101`） 存储在一个名为 `--packed-int` 的变量中，我们可以加上位数标记来理解它的结构：

  


| **位数范围** | **二进制值** | **对应值（十进制）** |
| -------- | -------- | ------------ |
| 位 `9-10` | `11`     | `3`          |
| 位 `7-8`  | `0`      | `0`          |
| 位 `4-6`  | `1`      | `1`          |
| 位 `0-3`  | `101`    | `5`          |

  


-   `--bits-9-10` 提取位 `9-10`（即：`11`，十进制是 `3`），`calc(var(--packed-int) / 256))` 右移 `8` 位，相当于只留下前两位（第`9`和`10`位）；`mod(..., 4)` 取模 `4`，得到这两位的值（`0~3`），返回的结果是 `3`
-   `--bits-7-8` 提取位 `7-8`（即：`00`，十进制是 `0`），`calc(var(--packed-int) / 64))` 右移 `6` 位，目标是位 `7-8` ；`mod(..., 4)`：取 `2` 位值（`0~3`），返回的结果是 `0`
-   `--bits-4-6` 提取位 `4-6`（即：`001`，十进制是 `1`），`calc(var(--packed-int) / 8))` 相当于右移 `3` 位，让我们关注位 `4~6` ；`mod(..., 8)`：取 `3` 位（`0~7`），返回的结果是 `1`
-   `--bits-0-3` 提取位 `0-3`（即：`101`，十进制是 `5`），`mod(var(--packed-int), 8)` 保留最后三位（二进制 `101 = 5`），返回的结果是 `5`

  


正如你所看到的，我们应用 CSS 的 `calc()`、`mod()`、`round()` 等函数可以对数值进行解码。当然，我们也可以使用 `pow(2, n)` 来代替这些写死的 `2` 的幂值。

  


> 温馨提示，如果你从未接触过 `mod()` 、`round()` 和 `pow()` 等函数，那么建议你花点时间阅读《[Web UI：CSS 中的数学函数和常量](https://juejin.cn/book/7395119173656903715/section/7446278438929104922)》！

  


所以，一个单独的 CSS 整数值就可以作为我这个 CSS的 “blobhash”（暂时这么称呼它）的编码方式。但问题是：**我们在一个 CSS 整数里，究竟能打包多少信息呢？**

  


## CSS 值的限制

  


[W3C 规范并没有明确说明整数值（`<integer>` ）可允许的范围](https://www.w3.org/TR/css-values-3/#integers)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1ca9fc47eaea4e8dbd99b9f554236f19~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2708&h=640&s=246924&e=jpg&b=ffffff)

  


CSS 理论上支持无限精度和无限范围的所有值类型，但在实际中，浏览器实现是有容量限制的。用户代理（UA）应当支持合理、实用的数值范围与精度。这意味道着自定义属性 `--packed-int` 的整数值所允许的范围只能交给各大浏览器厂商决定。

  


@Kalabasa 有做过这方面的测试，**自定义属性** **`--packed-int`** **的整数值只能在** **`-999999 ~ 999999`** **之间，一旦超出这个范围，就会开始丢失精度。** 比如 `1234567` 会被自动四舍五入为 `1234560` ，即精度降低到以十为单位 **。**

  


不过，在 `[-999999, 999999]` 这个范围内，我们有 **`1999999`** ****个有效的整数值。 ****这意味着只有一个整数作为哈希值，就可以表示接近两百万种 LQIP的配置组合。为了方便计算，把它向下取整到最按接 `2` 的幂值：

  


```
1048576 < 1999999 < 2097152
```

  


其中 `1048576` 是 `2` 的 `20` 次幂值（`pow(2, 20)`），`2097152` 是 `2` 的 `21` 次幂值（`pow(2, 21)`）。也就是说，我们有 `20` 位信息来编码 CSS LQIP 的哈希值。

  


> 为什么它被称为“哈希”？因为它是一种将任意大小的数据映射为固定大小值的方法。在这种情况下，虽然可能存在无限数量、任意尺寸的图像，但只能生成 `1999999` 种可能的哈希值。

  


## CSS LQIP 的解决方案

  


只用 `20` 位信息，LQIP 图像必然是原图像的极度简化版本。我们可以采用一个基础色加上六个亮度组件，并以`3×2` 网格叠加在基础色之上。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/096cf4b941964e039bc2e34412efa46c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=600&h=500&s=14244&e=png&b=333249)

  


这是一种非常极端的[色度子采样版本](https://en.wikipedia.org/wiki/Chroma_subsampling)。整个编码共包含 `9` 个数字，总共打包进 `20` 位整数中：

  


-   基础颜色：使用 OKLAB 颜色空间，编码在低 `8` 位中，其中 `2` 位表示亮度，`a` 和 `b` 坐标各 `3` 位。相比而言，OKLAB 效果比较平衡，但 RGB 同样可以使用
-   六个灰阶亮度值：编码在高 `12` 位，每个值占 `2` 位

  


另外，还面要一个[离线脚本](https://github.com/Kalabasa/leanrada.com/blob/7b6739c7c30c66c771fcbc9e1dc8942e628c5024/main/scripts/update/lqip.mjs#L118-L159)，它会把任意图像压缩成这个整数格式。该脚本首先会获取图像的平均色或主色（有很多 JavaScript 可以做这件事），然后将图像缩放到 `3×2` 像素，并提取灰阶值。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a678047e6ce04705a07e545099d650a0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2032&h=1286&s=2326193&e=png&b=302f2f)

  


> Demo 地址：https://codepen.io/airen/full/qEEdLKG

  


温馨提示，上面这个 Demo 是通过 AI 生成的 `--lqip` ，大家在实际使用的时候应该使用更好的算法来获得更精确的 `--lqip` 值。

  


一旦编码完成，这个整数就会通过 `style` 属性赋值给 `--lqip` ，然后在 CSS 中解码，具体的 CSS 代码如下：

  


```CSS
[style*="--lqip:"] {
    --lqip-ca: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 18))), 4);
    --lqip-cb: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 16))), 4);
    --lqip-cc: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 14))), 4);
    --lqip-cd: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 12))), 4);
    --lqip-ce: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 10))), 4);
    --lqip-cf: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 8))), 4);
    --lqip-ll: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 6))), 4);
    --lqip-aaa: mod(round(down, calc((var(--lqip) + pow(2, 19)) / pow(2, 3))), 8);
    --lqip-bbb: mod(calc(var(--lqip) + pow(2, 19)), 8);
  
    /* 渲染为 CSS 颜色前，需要把数值转换成颜色构造函数支持的格式 */
    --lqip-ca-clr: hsl(0 0% calc(var(--lqip-ca) / 3 * 60% + 20%));
    --lqip-cb-clr: hsl(0 0% calc(var(--lqip-cb) / 3 * 60% + 20%));
    --lqip-cc-clr: hsl(0 0% calc(var(--lqip-cc) / 3 * 60% + 20%));
    --lqip-cd-clr: hsl(0 0% calc(var(--lqip-cd) / 3 * 60% + 20%));
    --lqip-ce-clr: hsl(0 0% calc(var(--lqip-ce) / 3 * 60% + 20%));
    --lqip-cf-clr: hsl(0 0% calc(var(--lqip-cf) / 3 * 60% + 20%));
    --lqip-base-clr: oklab(
        calc(var(--lqip-ll) / 3 * 0.6 + 0.2)
        calc(var(--lqip-aaa) / 8 * 0.7 - 0.35)
        calc((var(--lqip-bbb) + 1) / 8 * 0.7 - 0.35)
    );
}
```

  


这个过程展示了如何在纯 CSS 中用一个整数值编码并解码一个高度简化的图像占位符，同时体现了现代 CSS 在数据处理上的潜力。

  


最后，渲染 LQIP。我使用了多个径向渐变来渲染灰度组件，并在底部添加了一个平坦的基础颜色。

  


```CSS
[style*="--lqip:"] {
    --lqip-stop10: 2%;
    --lqip-stop20: 8%;
    --lqip-stop30: 18%;
    --lqip-stop40: 32%;
    background-blend-mode: hard-light, hard-light, hard-light, hard-light,
      hard-light, hard-light, overlay, overlay, overlay, overlay, overlay,
      overlay, normal;
    background-image: radial-gradient(
        50% 75% at 16.67% 25%,
        rgb(from var(--lqip-ca-clr) r g b / 50%),
        rgb(from var(--lqip-ca-clr) r g b / calc(50% - var(--lqip-stop10) / 2))
          10%,
        rgb(from var(--lqip-ca-clr) r g b / calc(50% - var(--lqip-stop20) / 2))
          20%,
        rgb(from var(--lqip-ca-clr) r g b / calc(50% - var(--lqip-stop30) / 2))
          30%,
        rgb(from var(--lqip-ca-clr) r g b / calc(50% - var(--lqip-stop40) / 2))
          40%,
        rgb(from var(--lqip-ca-clr) r g b / calc(var(--lqip-stop40) / 2)) 60%,
        rgb(from var(--lqip-ca-clr) r g b / calc(var(--lqip-stop30) / 2)) 70%,
        rgb(from var(--lqip-ca-clr) r g b / calc(var(--lqip-stop20) / 2)) 80%,
        rgb(from var(--lqip-ca-clr) r g b / calc(var(--lqip-stop10) / 2)) 90%,
        transparent
      ),
      radial-gradient(
        50% 75% at 83.33% 25%,
        rgb(from var(--lqip-cc-clr) r g b / 50%),
        rgb(from var(--lqip-cc-clr) r g b / calc(50% - var(--lqip-stop10) / 2))
          10%,
        rgb(from var(--lqip-cc-clr) r g b / calc(50% - var(--lqip-stop20) / 2))
          20%,
        rgb(from var(--lqip-cc-clr) r g b / calc(50% - var(--lqip-stop30) / 2))
```

## 小结

  


这就是 CSS 创建 LQIP 的全部过程。在这个技术中，最为核心的是需要使用更好的算法获取图像的主色，并将它打包成一个整数值，然后将它存储在 `--lqip` 变量中。在实际使用的时候，通过 CSS 的数学函数对 `--lqip` 的值进行解码，然后应用于 `background-image` 。在实际使用的时候， CSS 可以使用固定使用上面的示例的代码。当然你也可以根据需要调整混合模式和渐变的方式，最终实现一个更接近你期望的效果！
