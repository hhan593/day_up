## 创建拼图网格

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/424b1e87105444a59acc3994642534c2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1600&h=900&s=1286947&e=png&b=703e12)

  


我想大家应该都玩过拼图游戏吧。要是我说，纯 CSS 也可以实现一个拼图游戏，我想很多 Web 开发者都不太会相信，甚至会觉得不可思议。但这是事实，CSS 真的很魔幻，也非常的强大！只有你想不到的，没有它做不到的。例如，[@Temani Afif 就使用纯 CSS 实现了一个拼图小游戏](https://codepen.io/t_afif/full/JjLQzqZ)：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9ed04d0aaf84b7ab5e42f0396865aa9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1252&h=678&s=6461670&e=gif&f=198&b=4b6d75)

  


> Demo 地址：https://codepen.io/t_afif/full/JjLQzqZ

  


在这里，我们不会探讨这个游戏如何制作，但我们将会探讨它的布局是如何实现的。例如：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8624d7350cfe4aa9a320cfb5d5c6e6c3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1071&s=425726&e=jpg&b=ebd077)

  


你可以将上图理解成拼图游戏中的一小部分。为了不使问题复杂化，我们将使用不同的图片来制作这个拼图效果。因为使用同一张图片，需要更多的、复杂的计算，这已超出我们这节课的范畴！

  


首先，我们使用 CSS Grid 布局技术来创建一个 `2 x 2` 的网格。这对于大家来说，应该非常简单了：

  


```CSS
.grid {
    --size: 200px;
    --gap: 10px;
    display: grid;
    grid-template-columns: repeat(2, var(--size));
    grid-auto-rows: var(--size);
    gap: var(--gap);
    
    img {
        width: 100%;
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f760f6cd80aa4776a08b66f400e1219a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1071&s=606202&e=jpg&b=010101)

  


仔细观察，每个拼图块（最小单元），有一边有一个凹槽形状，有一边有一个凸起形状，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/31aa8360ab384d2b85ce649351c2ff62~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1380&s=744761&e=jpg&b=020202)

  


注意，每一块的凹槽和凸起形状位置都是不同的，只有这样才能相互拼接。

  


接下来的重点是，我们将使用 [CSS 的渐变](https://juejin.cn/book/7395119173656903715/section/7429225312224804899)与[遮罩](https://juejin.cn/book/7223230325122400288/section/7259668885224456252#heading-9)（`mask`）组合起来，创建我们所需要的图形形状。我曾在小册的《[CSS 图形：创建花式边框效果](https://juejin.cn/book/7395119173656903715/section/7403197889905688585)》一课中详细的介绍制作[凹槽](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-1)和[凸起形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-2)的背后工作原理。

  


我们要做的是，将凹槽和凸起形状合并在一起作为遮罩图（`mask-image`），并使用 `mask` 裁剪出所需要的单个拼图形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5112b31a69f347dcbc477b843ff2f910~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=2524&s=927800&e=jpg&b=010101)

  


简单地说，我们需要两个径向渐变来生成遮罩图：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/17ee8d9dfb0e42a097ad7c42e58278dd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1380&s=225797&e=jpg&b=000000)

  


-   第一个径向渐变绘制蓝色的圆
-   第二个径向渐变绘制带有凹口的矩形形状（橙色区域）

  


为了更好的控制渐变绘制的形状，我们新增两个自定义属性来控制该形状：

  


-   `--g` 是网格的间距（它等同于之前的 `--gap` ，可以将它们统一起来，使用其中的一个）。
-   `--r` 控制拼图形状中圆形部分（凸起）的大小

  


为了便宜大家看到渐变所绘制的形状，我们先将 `radial-gradient()` 作为元素的 `background` ：

  


```CSS
.mask {
    --g: 10px;
    --r: 50px;
    background: 
        radial-gradient(var(--r) at left 50% bottom var(--r), #00FFFF 95%, #0000),
        radial-gradient(calc(var(--r) + var(--g)) at calc(100% + var(--g)) 50%, #0000 95%, orange) top/100% calc(100% - var(--r)) no-repeat;
}
```

  


上面的两个径向渐变，绘制了其中一个所需要的拼图形状：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4139be200acd4178993cb589132f4a18~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1380&s=306556&e=jpg&b=ffffff)

  


我们可以把渐变中的 `orange` 和 `#00FFFF` 替换成 `#000` ，这就生成了其中一张所需要的遮罩图：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/86924e91d3ea41d3a08b6f199d24369d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1380&s=136592&e=jpg&b=ffffff)

  


注意，将颜色替换为 `#000` 并不是必须的，这里只是便于使其看起来更像一个拼图块，并且能与 `mask` 的遮罩模式统一起来。

  


接下来，只需要修改一些值即可生成其他三种形状：

  


```CSS
.mask {
    --g: 10px;
    --r: 50px;
    
    background: var(--m);
    
    &:nth-child(1) {
        --m:
            radial-gradient(var(--r) at left 50% bottom var(--r), #000 95%, #0000),
            radial-gradient(calc(var(--r) + var(--g)) at calc(100% + var(--g)) 50%, #0000 95%, #000) top / 100% calc(100% - var(--r)) no-repeat;
    }
  
    &:nth-child(2) {
        --m: 
            radial-gradient(var(--r) at top 50% left var(--r),#000 95%,#0000),
            radial-gradient(calc(var(--r) + var(--g)) at 50% calc(100% + var(--g)),#0000 95%,#000) right / calc(100% - var(--r)) 100% no-repeat;
            
    }
  
    &:nth-child(3) {
        --m: 
            radial-gradient(var(--r) at top 50% right var(--r),#000 95%,#0000),
            radial-gradient(calc(var(--r) + var(--g)) at 50% calc(0% - var(--g)),#0000 95%,#000) left / calc(100% - var(--r)) 100% no-repeat;
            
    }
  
    &:nth-child(4) {
        --m: 
            radial-gradient(var(--r) at left 50% top var(--r),#000 95%,#0000),
            radial-gradient(calc(var(--r) + var(--g)) at calc(0% - var(--g)) 50%,#0000 95%,#000) bottom / 100% calc(100% - var(--r)) no-repeat, ;
    }    
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/482d1f076ba348faa5e0df7be1b6723f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1380&s=289104&e=jpg&b=ffffff)

  


我们把这几个形状存储在名为 `--m` 的变量中，并将它应用于图片元素的 `mask` 属性上。如此一来，我们就将图片裁剪成所需要的拼图形状：

  


```CSS
.grid {
    --size: 200px;
    --g: 10px;
    --r: 50px;
    display: grid;
    grid-template-columns: repeat(2, var(--size));
    grid-auto-rows: var(--size);
    gap: var(--g);
    
    img {
        width: 100%;
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;
        
        mask: var(--m);
    
        &:nth-child(1) {
            --m:
                radial-gradient(var(--r) at left 50% bottom var(--r), #000 95%, #0000),
                radial-gradient(calc(var(--r) + var(--g)) at calc(100% + var(--g)) 50%, #0000 95%, #000) top / 100% calc(100% - var(--r)) no-repeat;
        }
      
        &:nth-child(2) {
            --m: 
                radial-gradient(var(--r) at top 50% left var(--r),#000 95%,#0000),
                radial-gradient(calc(var(--r) + var(--g)) at 50% calc(100% + var(--g)),#0000 95%,#000) right / calc(100% - var(--r)) 100% no-repeat;
                
        }
      
        &:nth-child(3) {
            --m: 
                radial-gradient(var(--r) at top 50% right var(--r),#000 95%,#0000),
                radial-gradient(calc(var(--r) + var(--g)) at 50% calc(0% - var(--g)),#0000 95%,#000) left / calc(100% - var(--r)) 100% no-repeat;
                
        }
      
        &:nth-child(4) {
            --m: 
                radial-gradient(var(--r) at left 50% top var(--r),#000 95%,#0000),
                radial-gradient(calc(var(--r) + var(--g)) at calc(0% - var(--g)) 50%,#0000 95%,#000) bottom / 100% calc(100% - var(--r)) no-repeat, ;
        }  
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d16f259602234928b06d49b1afec19e0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=512045&e=jpg&b=010101)

  


现在，形状已经有了，但还需要将它们拼接在一起。由于每张图片受限于所在的网格单元格，它们看起来有点分散。并且仅调整图片之间的间距，无法直接将它们拼接在一起。因为每个形状的凸起和凹槽位置并没有完全对应。我们需要通过增加图片的高度或宽度来创建溢出效果。如上图所示：

  


-   第一张和第四张图片需要增加高度
-   第二张和第三张图片需要增加宽度

  


我们可以使用 `--r` 来控制图片的尺寸：

  


```CSS
.grid {
    /* CSS ... */
    
    img {
        /* CSS ... */
        
        &:is(:nth-child(1), :nth-child(4)) {
            width: 100%;
            height: calc(100% + var(--r));
        }
        
        &:is(:nth-child(2), :nth-child(3)) {
            width: calc(100% + var(--r));
            height: 100%;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34da2aee124e4447853109e4ef18cf72~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=964&s=442645&e=jpg&b=000000)

  


从上图的效果上来看，不难发现，第一张和第三张图片完美的拼接在一起，但第二张和第一张，第二张和第四张以及第三张和第四张并没有完美的拼接在一起。这是因为，我们增加了图片的宽度和高度，默认情况下图片会向右（增加宽度的图片）或向下（增加宽度的图片）重叠。所以，我们需要对第二张和第四张图片的位置进行调整，使它们能正确的拼接在一起。

  


解决的办法很简单，我们可以使用 `place-self` 来调整它们（第二张和第四张图片）的对齐方式：

  


```CSS
.grid {
    /* CSS ... */
    
    img {
        /* CSS ... */
        
        &:is(:nth-child(2), :nth-child(4)) {
            place-self: end;
        }
    }
}
```

  


这样我们就获得了一个类似拼图的网格布局效果：

.grid {
    --size: 200px;

## 创建嵌套网格布局

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/57c45178d1024d77a17f3a5751d91144~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2547&h=910&s=1474109&e=jpg&b=cd8c53)

  


你可能会觉得“拼图”网格布局实用性不强，那么类似上图这样嵌套网格布局，我想你会喜欢。

  


接下来，我们仍然使用 CSS Grid 、`clip-path` 和 `mask` 等技术来创建嵌套网格布局，你将学会通过不同的方式在网格中排列图片或其他元素，还会实现一些炫酷的悬停效果，带来真实而互动的图片浏览体验。

  


首先，我们来实现上图中第一个图片所示的布局效果。这个布局效果与之前的拼图网格布局相比较，网格容器中心有一张图片嵌套在不规则的图片当中。

  


与之前案例一样，首先创建一个网格。在这里，使用下面的代码创建一个 `2 x 2` 的网格：

  


```CSS
.grid {
    --size: 200px;
    --g: 10px;
    
    display: grid;
    grid-template-columns: repeat(2, auto);
    gap: var(--g);
    
    img {
        width: var(--size);
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;
    }
}
```

  


实际上，上面的代码将会创建一个 `2 x 3` 的网格（两列三行），这是因为网格容器中有五个 `<img>` 元素，浏览器会自动添加行数来匹配所需要的网格项目：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0899f90ae75f4ab6abab252b0b3b3b64~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1202&s=486923&e=jpg&b=000000)

  


你可能会问，那为什么说要创建一个 `2 x 2` 的网格呢？原因很简单，当中有一张图片（比如第五张）将会使用绝对定位，使其位于网格容器的正中心。

  


接下来，我们需要使用 `clip-path` 和 `mask` 技术对第 `1 ~ 4` 张图片进行角落切割。如果你阅读过小册之前的课程，我想你应该还有印象，我在《[CSS 图形：切割元素角落](https://juejin.cn/book/7395119173656903715/section/7400975207551139875)》中详细阐述了如何使用 `clip-path` 和 `mask` 对元素进行[角度切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-0)、[圆形切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-1)，甚至是[曲线切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-2)。在这个布局效果中，我们将会应用到当中的[角度切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-0)技巧。

  


很简单，我们需要做的是将角落裁剪掉一个小矩形:

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/207b3a71afb3476e8c2c0de11faa1f44~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1202&s=564463&e=jpg&b=c3915e)

  


在 CSS 中，使用 `clip-path` 或 `mask` 都可以轻易实现上图所示的效果，但相比而言，使用 `mask` 要更简单很多。因为我们只需要使用锥形渐变 `conic-gradient()` 调整起始角度值，就可以获得不同角落的裁剪效果，要是使用 `clip-path` 的话，需要调整多个不同的坐标点。

  


```CSS
.grid {
    /* CSS ... */
    
    img {
        /* CSS ... */
        
        mask: conic-gradient(from var(--a), #0000 90deg, #000 0);
        
        &:nth-child(1) {
            --a: 90deg;
        }
        
        &:nth-child(2) {
            --a: 180deg;
        }
        
        &:nth-child(3) {
            --a: 0deg;
        }
        
        &:nth-child(4) {
            --a: 270deg;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a835daf99c1412cb7b252a9453f7b4b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1254&s=570902&e=jpg&b=cd8b52)

  


接下来，只需要将第五张图片应用绝对定位将其放在网格容器的正中心：

  


```CSS
.grid {
    /* CSS ... */
    position: relative;
    
    img {
        /* CSS ... */
        
        &:nth-child(5) {
            position: absolute;
            left: 50%;
            top: 50%;
            translate: -50% -50%;
            clip-path: inset(calc(var(--g) / 4));
        }
    }
}
```

  


注意，上面的代码中还应用了 `clip-path` 的 `inset()` 对图片做了一个矩形裁剪。这是为了确保嵌套图片之间的间距一致。如果去掉它，你会发现图片之间的间距不一致。通过这种方式，我们稍微从第五张图片中裁剪一些内容，以获得精确的间距：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de02e144d25242d6b2455d321c011a71~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1254&s=562379&e=jpg&b=cd8b52)

  


> Demo 地址：https://codepen.io/airen/full/PoMMyMj

  


完整的代码如下：

  


```CSS
.grid {
    --size: 200px;
    --g: 10px;
    display: grid;
    grid-template-columns: repeat(2, auto);
    gap: var(--g);
    position: relative;

    img {
        width: var(--size);
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;

        mask: conic-gradient(from var(--a), #0000 90deg, #000 0);

        &:nth-child(1) {
            --a: 90deg;
        }

        &:nth-child(2) {

            --a: 0deg;
        }

            left: 50%;
            top: 50%;
            translate: -50% -50%;

    }
}
```

  


上面示例中，第五张图片定位到容器正中心是我们经常用的一种方法。其实，我们可以使用 `inset` 属性来简化上面的代码。使用 `inset` 只需要一行 CSS 代码就可以将图片放置到容器正中心。由于我们知道图片的大小（`--size`）和网格容器的大小（ `100%` ），因此我们通过一些简单的数学计算，就可以计算出 `inset` 的值应该为 `(100% - var(--size)) / 2` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f7fa8d81a0ab43b595f1ecf4c5147eaf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1254&s=797713&e=jpg&b=cd8b52)

  


```CSS
.grid {
    /* CSS ... */
    position: relative;
    
    img {
        /* CSS ... */
        
        &:nth-child(5) {
            position: absolute;
            inset: calc((100% - var(--size)) / 2);
            clip-path: inset(calc(var(--g) / 4));
        }
    }
} 
```

可能，有不少人会说：“既然我们可以将最后一张图片放在最上面，那么只需要给第五张图片添加一个边框，不需要使用任何遮罩就能实现，不是更简单吗？” 这个想法并没有错，这样做同样可以实现所需要的效果，代码也更容易理解，但它有一个明显的缺点：**边框颜色需要与主背景相同，才能使这种效果完美**！这个小缺点足以让我选择让代码更复杂，以换取真正的透明度，且不依赖于背景。尤其是在一些未知的情况之下，或者主背景是渐变或带有其他装饰性背景下，使用 `clip-path` 和 `mask` 技术将是最优的一种方案。这样一来，你就可以构建出不依赖于环境的组件。

  


请继续，接下来我们来实现另一种形状，中间图片是菱形形状，其他周边的图片进行角度切割：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/238f7dd50062471eb2d1931746c00f33~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=910&s=502242&e=jpg&b=cd8b52)

  


我们延续上一个示例，使用网格区域来构建布局：

  


```CSS
.grid {
    --size: 200px;
    --g: 10px;
    --col: calc((var(--size) - var(--g)) / 2);
    display: grid;
    grid-template-columns: repeat(4, var(--col));
    grid-template-rows: repeat(4, var(--col));
    gap: var(--g);
    font-size: 0;
    transition: .5s;
    
    &:hover {
        font-size: calc(var(--size)/5);
    }
    
    img {
        width: var(--size);
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;
        mask: var(--m);
        
        &:nth-child(1) {
            grid-area: 1 / 1 / 3 / 3;
        }

        &:nth-child(2) {
            grid-area: 1 / 3 / 3 / 5;
        }

        &:nth-child(3) {
            grid-area: 3 / 1 / 5 / 3;
        }

        &:nth-child(4) {
            grid-area: 3 / 3 / 5 / 5;
        }
        
        &:nth-child(5) {
            grid-area: 2 / 2 / 4 / 4;
        }
    }
}
```

  


注意，与之前有一点不同之处，设置了网格容器 `.grid` 的 `font-size` 为 `0` ，并且在 `:hover` 状态下调整了 `font-size` 的值为 ` calc(var(--size) / 5)` 。这是因为，给网格悬浮状态下添加过渡动画效果需要用到。

  


到目前为止，图片没有被切割，网格容器悬浮状态也无任何过渡动画效果，只能看到第五张图片叠放在其他四张图片上方：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/565325e0e6034250a8277e82a981a80d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1254&s=575293&e=jpg&b=cd8b52)

  


我们的目标效果是，第 `1 ~ 4` 图片有一个角度切割效果。理论上说，[使用 `clip-path` 对图片进行角度切割会更简单一些](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-0)，但在这里继续考虑使用 `mask` 来切割图片。出发点也是为了过渡动效，它要比动画化 `clip-path` 要简单。

  


与之前案例不同的是，我们使用线性渐变 `linear-gradient()` 来创建遮罩图。其 CSS 代码大致如下：

  


```CSS
.grid {
    /* CSS ... */
    
    img {
        /* CSS ... */
        --m: linear-gradient(var(--a),#0000 calc(25% + 0.2929 * var(--g) + 1em),#000 0);
        mask: var(--m);
        
        &:nth-child(1) {
            --a: -45deg;
        }

        &:nth-child(2) {
            --a: 45deg;
        }

        &:nth-child(3) {
            --a: -135deg;
        }

        &:nth-child(4) {
            --a: 135deg;
        }
    }
}
```

  


可能有的同学留意到 `linear-gradient()` 中的 `0.2929` 和 `1em` 了。对它们很好奇对吧！这里简单的解释一下这个 `0.2929` 如何获得的。

  


首先，线性渐变 `linear-gradient()` 的渐变线与元素图片元素框对角线位置相吻合，只不每个元素的渐变线的角度不一样（如下图中的红色箭头所示）：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2ce1ffaafe6e48cb88d9524cf34be004~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1254&s=650863&e=jpg&b=cd8b52)

  


要是先不考虑间距与交互动画，我们可以像下面这样绘制遮罩图形：

  


```CSS
.mask {
    background: linear-gradient(var(--a),#0000 25%,#000 0);
    
    &:nth-child(1) {
        --a: -45deg;
    }
    
    &:nth-child(2) {
        --a: 45deg;
    }
    
    &:nth-child(3) {
        --a: -135deg;
    }
    
    &:nth-child(4) {
        --a: 135deg;
    }
}
```

  


你可能已经发现了，在这个线性渐变中，`#0000` （透明颜色）在渐变线上的位置是 `25%` ，而不是 `calc(25% + 0.2929 * var(--g) + 1em)` 。创建的遮罩图会下像下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/05342aaaaed846168b1417863695a6bd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=772&s=372511&e=jpg&b=ffffff)

  


但为了使第五张图片与其他图片之间的间距保持一致，我们在绘制遮罩图时，需要考虑到间距 `--g` 。那具体是多少呢？我们需要一些几何计算，才能获得精确的值。

  


在这个示例中，行与行，列与列之间的间距，在容器正中间刚好是一个边长为 `--g` 的正方形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b9af4fd70e314431985ff24010eb1eea~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=772&s=437369&e=jpg&b=ffffff)

  


熟悉几何知识的同学应该知道，边长为 `--g` 的正方形，那么：

  


-   从中心到正方形边的距离为 `var(--g) / 2`
-   对角线的长度为 `var(--g) * √2`
-   半对角线的长度（即从正方形中心到角的距离）为 `(var(--g) * √2) / 2` ，大约等于 `0.707 * var(--g)`
-   中心到角的距离：` var(--g) / 2 - (((var(--g) * √2 ) - var(--g)) / 2) = var(--g) * (1 - 0.707)` ，大约等于 `0.2929 * var(--g)`

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a6d701ac499e4d47bf44242f1ea66110~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2500&h=1340&s=159933&e=jpg&b=ffffff)

  


简单地说，透明区域的偏移系数等于 `1 - √2 / 2 = 0.29292` 。这个值用于调整透明区域相对于图片间隙的动态变化，确保遮罩效果既统一又美观。这就是 `linear-gradient()` 中 `0.2929` 是怎么来的背后原理。

  


```CSS
.mask {
    background: linear-gradient(var(--a),#0000 calc(25% + 0.2929 * var(--g)),#000 0);
    
    &:nth-child(1) {
        --a: -45deg;
    }
    
    &:nth-child(2) {
        --a: 45deg;
    }
    
    &:nth-child(3) {
        --a: -135deg;
    }
    
    &:nth-child(4) {
        --a: 135deg;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/63f598190dfb4375911ed22a1a3ea284~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=772&s=270881&e=jpg&b=ffffff)

  


现在，渐变的透明区域要扩展到上图灰色边的外侧边缘，这里使用灰色是便于大家理解。

  


再来解释渐变中 `0.2929 * var(--g) + 1em` 中的 `1em` 。还记得我们在 `.grid` 容器中设置的 `font-size` 为 `0` 吗？这个时候 `1em` 等于 `0` ，渐变的透明区域并没有任何的变化。但在悬浮状态，`font-size` 变成了 ` var(--size) / 5` ，这个时候 `1em` 等于 ` var(--size) / 5` ，渐变的透明区域也就相应的变大了（透明颜色 `#0000` 在渐变线上的停止位置变成了 `25% + 0.2929 * var(--g) + var(--size) / 5）` ：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2e33e8e2aa3a4d158ca9ee6fb57c1595~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1144&h=674&s=3534233&e=gif&f=94&b=c67e47)

  


其实，我们可以直接把计算 `0.2929` 的公式在 CSS 中描述出来：

  


```CSS
img {
    --m: linear-gradient(var(--a),#0000 calc(25% + (1 - sqrt(2) / 2) * var(--g) + 1em),#000 0);
    mask: var(--m);
}
```

  


最后就是处理第五张图片，重新调整了它的宽度，并且使用 `clip-path` 将其裁剪成一个菱形形状：

  


```CSS
img:nth-child(5) {
    width: calc(var(--size) + 2.82em);
    place-self: center;
    clip-path: polygon(50% 0,100% 50%,50% 100%,0 50%);
}
```

  


注意，默认状态下，图片的宽度是 `var(--size)` ，因为 `.grid` 的 `font-size` 等于 `0` ，所以 `2.82em` 也等于 `0` 。但是，在悬浮状态下，`.grid` 的 `font-size` 等于 `var(--size) / 5` ，这个时候，图片的宽度等于 `var(--size) + 2.82 * (var(--size) / 5)` 。为什么是 `2.82em` ，这是一个调试值，通过多次调试得出来的一个最佳值。

  


需要知道的是，悬浮状态时，图片的尺寸会变大，为了使它始终位于网格区域的中间，我们需要设置 `place-self` 的值为 `center` 。这个时候，我们最终获得了我们所需要的一个网格效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/470e367e3c9a469b8342c57c4708e416~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=682&s=5423669&e=gif&f=99&b=c67e47)

  


> Demo 地址：https://codepen.io/airen/full/raBBWgo

  


最终的 CSS 代码如下：

  


```CSS
.grid {
    --size: 200px;
    --g: 10px;
    --col: calc((var(--size) - var(--g)) / 2);
    display: grid;
    grid-template-columns: repeat(4, var(--col));
    grid-template-rows: repeat(4, var(--col));
    gap: var(--g);
    font-size: 0;
    transition: 0.5s;
    cursor: pointer;

    &:hover {
        font-size: calc(var(--size) / 5);
    }

    img {
        width: var(--size);
        aspect-ratio: 1;
        object-fit: cover;
        object-position: center;
        --m: linear-gradient(
            var(--a),
            #0000 calc(25% + (1 - sqrt(2) / 2) * var(--g) + 1em),

        );
        mask: var(--m);

        &:nth-child(1) {
            --a: -45deg;
            grid-area: 1 / 1 / 3 / 3;

        }

        }

        &:nth-child(4) {

            grid-area: 3 / 3 / 5 / 5;
        }

        }
    }

```

  


最后，我们再来制作一个圆形的嵌套网格布局效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/98d131aa77804570b42b3fae3d7d2f3f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=514&s=6723919&e=gif&f=128&b=c67e47)

  


这个效果与上一个案例效果非常相似，只是图片裁剪的形状变了。中间的图片由菱形变成了圆形，周边的图片由[角度切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-0)变成了[圆角切割](https://juejin.cn/book/7395119173656903715/section/7400975207551139875#heading-1)。我们可以使用 `radial-gradient()` 来创建遮罩图片，从而获得圆角切割的效果：

  


```CSS
img {
    --pos: calc(var(--g) / -2);
    --m: radial-gradient(
        farthest-side at var(--_pos),
        #0000 calc(50% + var(--g) / 2), 
        #000 calc(51% + var(--g) / 2)
    );
    mask: var(--m);

    &:nth-child(1) {
        --_pos: right var(--pos) bottom  var(--pos);
    }

    &:nth-child(2) {
        --_pos: left var(--pos) bottom  var(--pos);
    }

    &:nth-child(3) {
        --_pos: right var(--pos) top  var(--pos);
    }

    &:nth-child(4) {
        --_pos: left var(--pos) top  var(--pos);
    }
}
```

  


上面代码只是根据图片位置，调整了径向渐变的圆心位置。同样地，为了确保中心图片与嵌套图片之间的间距，我们在绘制遮罩图时还考虑到了间距（`--g`），这样可以确保所有径向变中的圆心位于相同的位置（即网格容器中心）。如此一来，将它们组合起来就能形成一个连续的圆形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/07e027171bbd41e09541824c1423345b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1020&s=825368&e=jpg&b=c3915e)

  


现在我们有了布局和带有圆角切口的图片，接下来就可以处理它的过渡动效了。采用与上一例相同的原理，在创建遮罩图形时，我们在设置渐变颜色停止点时加上 `1em`，使渐变能够被动画化：

  


```CSS
.grid {
    font-size: 0;
    transition: .5s;
    
    &:hover {
        font-size: calc(var(--size) / 5);
    }
    
    img {
        --m: radial-gradient(
            farthest-side at var(--_pos),
            #0000 calc(50% + var(--g) / 2 + 1em), 
            #000 calc(51% + var(--g) / 2 + 1em)
        );
    }
}
```
