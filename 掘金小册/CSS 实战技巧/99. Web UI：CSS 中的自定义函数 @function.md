一直以来，CSS 被视作为声明式语言，不像其他程序语言那样，具有变量、函数和逻辑功能。然而，随着 CSS 不断的变革，它现在也具备了变量、逻辑运算和函数等特性。换句话说，CSS 现在正在拥抱函数式编程——那些曾经认为 CSS 不是“真正的编程语言”的人，可能需要重要重新考虑了。

## CSS 自定义函数：`@function`

  


> 首先声明，`@function` 是一个较新的 CSS 特性，仍处于实验阶段，目前仅限于 Chrome Canary 浏览器（需要手动启用实验性功能）。因此，接下来课程中的相关示例，请使用 Chrome Canary 浏览器查看，而且后续它的语法和使用规则也可能会有所变动。有关于它的最初原型设计，请移步阅读《[CSS Mixins & Functions Explainer](https://css.oddbird.net/sasslike/mixins-functions/)》！

  


在开始之前，我们暂时来回顾一下 Sass 处理器中的 `@function` 。比如：

  


```SCSS
@function strip-units($number){
    @return $number / ($number * 0 + 1);
}
```

  


上面代码定义了一个名为 `strip-units` 的函数，它的主要功能是帮助我们移除值的单位：

  


```SCSS
$font-size: 16px;
$base-size:  strip-units($font-size); // 16
$rem-size:   $base-size / 16 * 1rem;  // 1rem
```

  


现在，CSS 也引入了 `@function` 。最初我以为它就像 Sass 中的 `@mixin` ，是用来建立可重用模式的。甚至一度认为它只是帮助我们写更少 CSS 代码的“语法糖”。直到，我阅读完 `@function 的原型设计`之后，我才意识到它的强大之处。另外，W3C 规范对 `@function` 也描述的非常清楚，[你可以点击这里查阅](https://drafts.csswg.org/css-mixins-1/)！

  


接下来，我向大家分享一下我对 CSS 函数（`@function`）的几个关键点的理解！

  


### 基础：定义和调用函数

  


假设你要为页面上的多个元素添加相同的边框样式，传统的做法是：

  


```CSS
:is(div, p, section) {
    border: 2px dashed #000;
}
```

  


或者使用诸如 Tailwind CSS 、Bootstrap 的 UI 库，使用原子 CSS：

  


```HTML
<div class="border-2 border-dashed border-black"></div>
<p class="border-2 border-dashed border-black"></p>
<section class="border-2 border-dashed border-black"></section>
```

  


```CSS
.border-2 {
    border-width: 2px;
}

.border-dashed {
    border-style: dashed;
}

.border-black {
    border-color: #000;
}
```

  


更为现代一点的方式，可能会借助 CSS 自定义属性：

  


```CSS
:is(div, p, section) {
    border: var(--bw, 2px) var(--bs, dashed) var(--bc, #000);
}
```

  


虽然这样写 CSS 并不复杂，但如果能告诉 CSS —— “嘿，给我一个虚线边框”，这样岂不是更方便？有了 CSS 的 `@function` 特性之后，我们就可以轻松地做到这一点：

  


```CSS
@function --dashed-border() {
    result: 2px dashed #000;
}
```

  


这样我们就定义好了一个名为 `--dashed-border` 的函数，它将返回的结果是 `2px dashed #000` 。有了这个函数之后，我们只需要像下面这样调用就可以为元素添加边框：

  


```CSS
:is(div, p, section) {
    border: --dashed-border();
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/af11d737afd8478d89e346e203f091bf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=70133&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/JojqBPP

  


是不是很魔性！从现在开始，每当你需要一个虚线边框时，只需要调用 `--dashed-border()` 函数，CSS 会自动处理！

  


### 升级：带参数的函数

  


上面示例所展示的是 CSS 函数（`@function`）最基本的语法规则，在 `@function` 规则后面紧跟着函数的名称，它的形式是 `<dashed-ident> + ()` ，比如 `--dashed-border()` ：

  


```CSS
@function --dashed-border() {
    /* 函数要完成的功能 */
}
```

  


没有参数的函数就像 CSS 自定义属性一样，所以没什么特别之处。为了使它们更有功能性，我们可以在括号内（`()`）传递参数，参数也可以使用 `<dashed-ident>` ，比如：

  


```CSS
@function --dashed-border(--c) {
    result: 2px dashed var(--c);
}
```

  


此时，调用 `--dashed-border()` 函数时，就可以给它传递一个参数：

  


```CSS
div {
    border: --dashed-border(blue);
}

p {
    border: --dashed-border(red);
}

section {
    border: --dashed-border(#f90);
}
```

  


有了参数支持，CSS 函数变得更加强大:

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2dd0527bb67e4b2fa1b8f1f599c2a0dd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2572&h=988&s=73200&e=png&b=ffffff)

  


> Demo 地址：https://codepen.io/airen/full/raNgrLg

  


在定义函数时，我们甚至还可以使用默认值！只需写一个冒号（`:`），后跟该参数的默认值：

  


```CSS
@function --dashed-border(--c: blue) {
    result: 2px dashed var(--c);
}

div {
    border: --dashed-border(); /* 2px dashed blue*/
}

p {
    border: --dashed-border(red); /* 2px dashed red */
}

section {
    border: --dashed-border(#900); /* 2px dashed #900 */
}
```

正如你所看到的，**CSS 自定义函数赋予 Web 开发者与自定义属性相同的能力，但带有参数化。它们在你使用自定义属性的地方也可以使用，但函数根据我们传递的参数返回不同的内容**！

  


### 类型检查和默认值：更智能的错误预防

  


CSS 函数可以对参数和返回值进行类型检查，这与 [CSS 的 `@property`](https://juejin.cn/book/7223230325122400288/section/7258870477462962236) 创建的变量极其相似。而且只要内联条件语句（`if()`）得到支持，就可以根据参数类型进行不同的计算。

  


我们只需要在定义函数时传递一个语法组件，就可以给函数添加参数类型。比如：

  


-   `<angle>`: 角度（例如：`deg` 或 `turn`）
-   `<color>`: 颜色（例如：命名颜色、`rgb`、`hsl`、`oklch` 等）
-   `<custom-ident>`: 自定义标识符，用于命名诸如视图过渡名称等
-   `<integer>`: 离散的整数值
-   `<length>`: 数字加单位（例如： `1rem` 或 `10px` 等）
-   `<length-percentage>`: 可以是长度值或百分比（例如： `10px` 或 `10%` ）
-   `<number>`: 扩展整数，带有小数点以表示分数值
-   `<percentage>`: 百分比（`%`），通常相对于父对象
-   `<resolution>`: 用于媒体查询（例如：`dpi`）
-   `<time>`: 用于动画（例如：`s` 或 `ms`）
-   `<transform-function>`: 改变元素外观的函数（例如：`rotate()`）

  


还有语法乘法符号，比如加号（`+`），用于接受一个以空格分隔的该类型的列表。

  


```CSS
@function --custom-spacing(--a <length>) {
    resutl: var(--a);
}

@function --custom-color(--c <color>) {
    result: var(--c);
}

@function --custom-margin(--m <length>+) {
    result: var(--m);
}
```

  


如果我们想要定义返回值的类型，可以写 `returns` 关关键字，后面跟着语法组件：

  


```CSS
@function --progress(--current, --total) returns <percentage> {
    result: calc(var(--current) / var(--total) * 100%);
}
```

  


有一点需要注意，如果我们想要使用语法组合符号（`|`）接受多种类型，我们需要将类型放入 `type()` 函数中：

  


```CSS
@function --wideness(--d type(<number> | <percentage>)) {
    restul: var(--d);
}
```

  


另外，函数还可以有列表参数，我们可以通过将列表参数放入大括号中，作为参数传递给函数。比如：

  


```CSS
@function --max-plus-x(--list, --x) {
    result: calc(max(var(--list)) + var(--x));
}

div {
    width: --max-plus-x({1px, 7px, 2px}, 3px);
}
```

  


我们给 `--max-plus-x` 函数的 `--list` 传递了一个列表值，即 `{1px, 7px, 2px}` ，然后获取其中的最大值来进行加法计算。

  


> 温馨提示，到目前为止，Canary 似乎还看不到传入列表值的效果！

  


### 提前返回不可用

  


我们在定义 CSS 函数的时候，可能会与其他的一些 `@` 规则结合使用，比如 `@media` 、`@container` 或 `@supports` 等。在这些规则中可能会提前返回函数的值，比如：

  


```CSS
@function --font-size() {
    @media (width > 1000px) {
        result: 20px;
    }
    
    result: 16px;
}
```

  


`--font-size` 有两个 `return` （即 `result` 结果），一个是在 `@media` 规则内提前返回，另一个是在 `@media` 规则外部最后返回，它将造成最后的结果始终是最后一个返回的结果，即 `16px` 。

  


这意味着，在 CSS 函数中，提前返回的值是不可用的，这不是规范中尚未原型化的内容，而是根本不允许的事情。**我们必须改变返回的顺序，把条件结果放在最后**。这在其他编程语言中并不太有意义，因为函数在返回后就结束了，但 CSS 中的 `C` 代表层叠是有原因的：这种顺序允许条件结果覆盖最后的结果，这本质上非常符合 CSS 的特点：

  


```CSS
@function --font-size() {
    result: 16px;
    
    @media (width > 1000px) {
        result: 20px;
    }
}
```

  


如果它对你不起作用，我们可以使用一个临时的 CSS 自定义属性来存储需要的值，并在最后返回这个自定义属性：

  


```CSS
@function --font-size() {
    --size: 16px;
  
    @media (width > 600px) {
        --size: 20px;
    }
    
    result: var(--size);
}
```

  


## CSS 自定义函数能帮我们做些什么

  


接下来，我们来看看利用函数可以做些什么新东西。先来看下面这个小型的辅助函数：

  


```CSS
@function --fluid-type(--min, --max) {
    result: clamp(var(--min), 4vw + 1rem, var(--max));
}

h1 {
    font-size: --fluid-type(24px, 36px);
}
```

  


定义的 `--fluid-type` 函数主要是用来设置[流体字体大小](https://juejin.cn/book/7395119173656903715/section/7445257653855223845#heading-4)：

如果你感兴趣的话，你完全可以根据 @Adrian Bece 的《[Modern Fluid Typography Using CSS Clamp](https://www.smashingmagazine.com/2022/01/modern-fluid-typography-css-clamp/)》教程中的相关数学公式计算出 `clamp()` 函数中 `v` 和 `r` 的值 **：**

  


```CSS

```

  


然后使用 `@function` 来创建一个动态的流体字体大小的函数，从而实现类似 [Modern Fluid Typography 工具的功能](https://modern-fluid-typography.vercel.app/)：

```CSS
@function --intrinsic-container(--inline-margin: 1rem, --max-width: 60ch) {

}
```

  


我们也可以使用 CSS 函数做一些复杂的事情。比如下面这个示例，自定义一个 `--light-dark()` 函数，实现类似于[ CSS 的 `light-dark()` 函数](https://juejin.cn/book/7395119173656903715/section/7445262578945064987)的功能。

  


我们先来回顾一下 `light-dark()` 函数的功能：

乍一看，自定义的 `--light-dark()` 函数有点多此一举。其实它与 CSS 内置的 `light-dark()` 函数还是有所不同的，`--light-dark()` 函数不限于 `<color>` 值，可以与任何类型的值一起使用。比如：

`--light-dark()` 自定义函数与内置的 `light-dark()` 函数不同的是，它无法响应本地的 `color-scheme` 值，仅能响应设备用户偏好设置，即 `prefers-color-scheme` 媒体查询。

  


庆幸的是，我们可以使用容器查询的样式查询或 CSS 的内联条件语句（`if()`）来改造 `--light-dark()` 函数，使它也能响应本地的 `color-scheme` 值。

  


我们先来看使用容器查询的自定义 `--light-dark()` 函数。

  


首先，我们需要定义一个 CSS 自定义属性，比如 `--scheme` ，它的主要作用有两点：

  


-   使用名为 `--scheme` 的自定义属性在元素上设置偏好的色彩方案
-   重新设计 `--light-dark()` 函数，使用样式查询响应 `--scheme` 变量的值

  


`--scheme` 变量的值可能为 `light` 、`dark` 和 `system` 。当 `--scheme` 设置为前两个值之一时，色彩方案（`color-scheme`）会被强制设置为该值。具体的代码可能会像下面这样：

  


```CSS
@function --light-dark(--light, --dark) {
    /* 默认使用 --light 值 */
    result: var(--light);
    
    /* 如果容器设置为 dark, 则使用 --dark 值 */
    @container style(--scheme: dark) {
        result: var(--dark);
    }
}
```

  


在 `@function` 内部，`--light` 和 `--dark` 值作为参数传入函数。而 `--scheme` 自定义属性则是从调用该函数的元素中读取的。

  


为了确保 `--scheme` 有一个值，可以将其根据 `prefers-color-scheme` 值设置在 `:root` 上。`perfers-color-scheme` 的值也被复制到 `--root-scheme` 中，以支持 `--scheme` 的 `system` 值：

  


```CSS
:root {
    --root-scheme: light;
    --scheme: light;
    
    @media (prefers-color-scheme: dark) {
        --root-scheme: dark;
        --scheme: dark;
    }
}
```

  


为了允许按元素设置偏好的色彩方案，我们可以在元素上定义一个 `data-scheme` 属性，并通过 [CSS 的 `attr()` 函数](https://juejin.cn/book/7395119173656903715/section/7487175782831161394)将 `data-scheme` 解析为 CSS 中所需要的值。当值为 `light` 或 `dark` 时，直接使用该值。当值为 `system` 时，代码使用 `--root-scheme` 属性值。[为了与嵌套的暗黑模式上下文兼容，代码使用了 `@scope`](https://juejin.cn/book/7223230325122400288/section/7259668111974989882) ：

  


```CSS
/* 允许通过 data-scheme HTML 属性覆盖 --scheme */
@scope ([data-scheme]) {
    /* 从属性中获取值 */
    :scope {
        --scheme: attr(data-scheme type(<custom-ident>));
    }

    /* 当设置为 system 时，使用 --root-scheme 值（由 MQ 确定） */
    :scope[data-scheme="system"] {
        --scheme: var(--root-scheme);
    }

    /* 这也使原生的 light-dark() 函数能够正常工作 */
    :scope > * {
        color-scheme: var(--scheme);
    }

    /* 因为带有该属性的元素是额外的包装元素，我们可以直接显示其内容 */
    display: contents;
}
```

```CSS
@function --light-dark(--light, --dark) {
    result: var(--light);

    @container style(--scheme: dark) {
        result: var(--dark);
    }
}

:root {
    --root-scheme: light;
    --scheme: light;

    @media (prefers-color-scheme: dark) {
        --root-scheme: dark;
        --scheme: dark;
    }
}

@scope ([data-scheme]) {
    :scope {
        --scheme: attr(data-scheme type(<custom-ident>));
    }

    :scope[data-scheme="system"] {
        --scheme: var(--root-scheme);
    }

    :scope > * {
        color-scheme: var(--scheme);
    }

    display: contents;
}

.element {
    padding: 2rem;
    line-height: 2;
    color: --light-dark(#333, #e4e4e4);
    background-color: --light-dark(aliceblue, #333);
    border: 4px --light-dark(dashed, dotted) currentcolor;
    font-weight: --light-dark(500, 300);
    font-size: --light-dark(16px, 18px);

}
```

注意，由于我们使用了[容器样式查询](https://juejin.cn/book/7223230325122400288/section/7259316003635462201)，我们需要在 `.element` 元素外包裹一个容器！

  


另一种方式是使用内联 `if()` 来自定义 `--light-dark()` 函数。通过内联 `if()` 语句，可以省略容器查询方法所需的额外容器元素，因为你可以在声明中直接有条件地选择一个值。

  


由于 `if()` 语句也接受样式查询作为条件之一，因此整体方法保持不变：**使用自定义属性并响应其值**。然而，所需要的 CSS 的代码要简洁得多：

  


```CSS
@function --light-dark(--light, --dark) {
    result: if (
        style(--scheme: dark) : var(--dark);
        else: var(--light)
    );
}
```

  


而且内联 `if()` 语句还可以接受多个条件，比如：

  


```CSS
@function --light-dark(--light, --dark) {
    result: if(
        style(--scheme: dark) : var(--dark);
        style(--scheme: light): var(--light);
        else: var(--root-scheme))
    )
}
```

  


不过，这种用法超出了本文的讨论范围。

  


使用 `if()` 内联条件语句，使得 `--scheme` 为 `light` 或 `dark` 的代码也得简洁，因为它更容易回退到 `--root-scheme` 的值：

  


```CSS
:root {
    --root-scheme: light;
    --scheme: light;
    @media (prefers-color-scheme: dark) {
        --root-scheme: dark;
        --scheme: dark;
    }
}  

@scope ([data-scheme]) {
    :scope {
        --scheme-from-attr: attr(data-scheme type(<custom-ident>));
        --scheme: if(
          style(--scheme-from-attr: system): var(--root-scheme);
          else: var(--scheme-from-attr)
        );
        color-scheme: var(--scheme); 
    }
}
```

  


用法和之前一样，不同的是你可以直接在 `[data-scheme]` 元素上设置依赖于色彩方案的样式。

  


```HTML
<div data-scheme="light" class="element" >
    <!-- Light Mode -->
</div>

<div data-scheme="dark" class="element">
    <!-- Dark Mode -->
</div>

<div data-scheme="system" class="element">
    <!-- System Mode -->
</div>
```

  



## 小结

  


从本质上讲， CSS 自定义函数就像内建的混合宏，它们能够实现更加可复用的 CSS 代码，而无需额外的 CSS 处理器。这将使样式表变得更加可维护、简洁和易读，同时实现以往只能通过 CSS 处理器或 JavaScript 才能做到的高级动态效果。

  


尽管我们仍在等待全面的浏览器支持，但保持好奇，继续尝试。毕竟，任何能帮助我们避免重复代码的技术，都是值得庆祝的胜利！

  


通过拥抱原生函数，CSS 正在发展成为一个更强大、更灵活的样式语言。作为 Web 开发者，我们正处在 Web 设计新时代的前沿——一个我们的样式表不再只是静态规则，而是能够根据需求进行调整的动态、智能实体的时代。CSS 的未来光明，函数将成为其中的核心！
