![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/190b120046254cccbf8bb4812f44ca64~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=1000&s=182911&e=jpg&b=d7b9ee)

  


在 Web 设计中，“交互式用户界面轼能”能够显著提升用户体验。比如，滚动页面时导航栏变小，按钮显示或内容出现动画等。但这些交互功能往往依赖 JavaScript 来实现，而 JavaScript 是运行在主线程上的。这意味着它会与页面的渲染、布局计算等任务争夺资源，从而可能引起性能问题，尤其是在滚动频繁或设备性能较低时。

  


为了解决这一问题，纯 CSS 方案可以减少对主线程的负担。因为，CSS 是运行在浏览器的合成线程上的，适合处理样式和动画等渲染任务，因此相比 JavaScript ，在动画性能上可能更平滑。但纯 CSS 实现也有局限，比如无法进行复杂的逻辑判断或操作 DOM。

  


接下来，我将向你展示如何使用纯 CSS 实现滚动感知的 UI 状态。比如，你可以让一个元素在你滚动到某处时逐渐显示或变形，或者你可以设置一个区域在用户向下滚动时自动切换颜色或阴影效果等。换句话说，通过使用现代 CSS 技术，你可以让页面在滚动过程中实现状态切换、动画、内容变化等功能，从而创造出**动态、响应灵敏、充满生命力的 Web 网页**。除了能吸引用户的注意力，还能让每一次滚动或点击都有视觉反馈。

  


## 什么是滚动感知状态？

  


滚动感知状态，有时也叫做位置感知状态或视口感知状态，是一种 UI 设计模式，在这种模式下，组件的状态会根据用户在视口内的滚动位置动态变化。它使得 Web 开发者能够在 Web 界面上创建类似原生应用的交互性。这是提升 Web 应用用户体验的一种流行方法。

  


简单地说，滚动感知状态就是“滚动驱动的 UI”——它是一种用户体验模式，能够动态且自适应地响应用户在 Web 上的滚动操作。以下是一个滚动感知 UI 的例子，展示了它的外观和行为：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/61212b4bacaa42c8945c82053842cbef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1262&h=749&s=12863777&e=gif&f=97&b=313a49)

  


正如你所看到的，随着用户上下滚动页面，组件的内容或行为会相应的调整。以下是滚动感知状态的一些关键特征：

  


-   **响应性**：组件的状态会自动根据滚动位置的变化进行更新。这可以通过多种技术实现，例如事件监听器、Intersection Observer API 或自定义滚动检测机制
-   **条件渲染**：根据滚动位置，可能会显示不同的内容或 UI 元素。例如，当用户滚动到某个位置时，导航栏可能会固定在屏幕顶部，或者当用户接近页面底部时，可能会动态加载新的内容进行无限滚动
-   **平滑过渡**：理想情况下，状态变化应该平滑且无缝地发生，提供自然直观的用户体验。可以使用动画和其他视觉提示来增强过渡效果

  


在 UI 中使用滚动感知状态有几个好处：

  


-   **提升用户参与度**：通过动态响应用户的互动，滚动感知组件可以保持用户的参与度和兴趣
-   **增强导航和发现**：滚动感知状态可以随着用户的滚动，展示额外的信息或功能，促进 UI 内部的探索和发现
-   **高效的资源利用**：通过像无限滚动这样的技术，滚动感知状态可以通过仅加载对用户可见的内容来优化资源的使用

  


这种用户体验模式通常与 JavaScript API （比如 [Intersection Observer API](https://developer.mozilla.org/en-US/docs/Web/API/Intersection_Observer_API)）或第三方库（比如 [Framer Motion](https://motion.dev/)、[ScrollMagic](https://scrollmagic.io/)）一起使用，用于监控、计算和存储用户滚动进度和位置的状态。然后，这个位置可以触发 CSS 动画或样式，从而改变 Web 页面上元素或组件外观、行为或可见性。

  


滚动驱动的 UI 设计可以简单到使用 CSS 实现平滑的滚动、滚动捕捉、固定定位，甚至视差效果。它们也可以更加复杂。更有趣的是，这些效果我们可以仅通过 CSS 实现与 JavaScript 制作的复杂动画相同的效果。比如：

  


-   CSS 的 `scroll-behavior: smooth` 可以实现平滑的滚动效果
-   CSS 的 `overscroll-behavior: contain` 可以阻止滚动穿透下拉刷新
-   CSS 实现滚动捕捉效果

  


而且 CSS 的滚动驱动动画和滚动状态容器查询可以实现更为复杂的滚动感知交互效果：

  


-   CSS 滚动驱动动画可以实现与滚动进度联动的动画，而无需借助 JavaScript 。比如，你可以让一个元素在你滚动到某处时逐渐显示或变形
-   CSS 滚动状态容器查询允许你根据容器的滚动状态来应用不同的样式。比如，你可以设置一个区域在用户向下滚动时自动切换颜色或阴影效果

  


接下来，我将通过一些简单的示例，向大家展示 CSS 是如何创建滚动感知的 UI 状态。

  


## 创建丝滑般的滚动效果

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/29332c37f8ef435f8b229b499d87c671~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1664&h=698&s=17982785&e=gif&f=73&b=4a5a59)

  


这应该是最简单的一种滚动感知 UI 设计。

  


正如你所看到的，在一些长内容的 Web 应用中，通常会在页面的右下角添加一个返回顶部的按钮，我们期望用户在点击“返回顶部按钮”时有一个平滑的滚动效果。在 CSS 中，只需要将滚动容器的 `scroll-behavior` 设置为 `smooth` 即可：

  


```CSS
html {
    scroll-behavior: smooth;
}
```

  


CSS 的 `scroll-behavior` 属性会告诉浏览器：当触发锚点链接等滚动操作时，应该如何处理滚动效果。我们来看一下使用 `scroll-behavior: smooth` 前后的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c0c739efe96049e98330a47e5a08c68d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1166&h=830&s=17204404&e=gif&f=93&b=4e5d5c)

  


> Demo 地址：https://codepen.io/airen/full/RNNwPNM

  


很明显，左侧的滚动效果非常生硬，一闪而过；而右侧的滚动效果非常平滑。仅仅一行 CSS 代码可以给用户提供丝滑般的滚动效果。

  


但对于一些**容易晕动症的人来说**，这个滚动动画可能会引起不适。幸运的是，一些浏览器和操作系统允许用户通过“减少动画”偏好来尊重他们的喜好。[我们可以使用 CSS 的媒体查询（`prefers-reduced-motion: reduce`）来监听这个设置，并为他们关闭平滑滚动](https://juejin.cn/book/7223230325122400288/section/7257368158451793935#heading-15)：

  


```CSS
html {
    scroll-behavior: smooth;
    
    @media screen and (prefers-reduced-motion: reduce) {
        scroll-behavior: auto;
    }
}
```

  


尽管我自己会在更多的地方使用 `scroll-behavior: smooth` ，减少对 JavaScript 插件的依赖，但有些时候，JavaScript 方案仍然是更好的选择。比如：

  


-   **滚动距离很长时**，使用 `scroll-behavior` 的动画可能会出现卡顿。这让我很意外，本以为 CSS 会自动和浏览器的帧刷新率对齐
-   `scroll-behavior` **不支持** **[缓动函数](https://juejin.cn/book/7288940354408022074/section/7297493957557092404)** **或** **[时间控制](https://juejin.cn/book/7288940354408022074/section/7304843997364060214)**。如果你想控制动画速度或缓动曲线，就必须使用 JavaScript

  


尽管存在一些限制，**CSS 的** **`scroll-behavior`** **属性在许多项目中仍然是一个非常棒的解决方案**。

  


## 阻止滚动穿透下拉刷新

  


带有滚动的模态框是滚动穿透的典型案例之一，模态框内容溢出时会出现滚动条。此时，Web 页面就会有两个滚动条出现，一个是模态框（子滚动容器，也被称为隐式滚动器），另一个是 `body` （称为根滚动器，每个 Web 页面只有一个根滚动器）。浏览器针对此场景时，其默认的滚动行为如下所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aea389a53c6a40919e4e103828a16129~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1586&h=834&s=582439&e=gif&f=171&b=fbfafd)

  


当模态框内容滚动到模态框底部时，模态框内容所在的滚动器（隐式滚动器）没有溢出可消费的内容，根滚动器（`body` 的滚动器）会开始滚动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f8cd4ac480b74ed8a304e0adb054882e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=611&s=46406&e=jpg&b=f3f3f3)

  


浏览器这种默认行为通常是不需要的，并且会分散用户对模态框内容的注意力。以往为了改变这种行为，一般是借助 JavaScript 来完成。当模态框出现时，给 `html` 或 `body` 元素添加一个类名（比如 `.modal--open`），反之移除这个类名。然后在 `.modal--open` 类名上设置 `overflow: hidden` ：

  


```CSS
.modal--open {
    overflow: hidden;
}
```

  


现在，如果你使用原生的 `<dialog>` 创建模态框的话，可以将 CSS 的 `:has()` 选择器与 [Popover API ](https://juejin.cn/book/7395119173656903715/section/7442515150541684746)的结合相结合，来避免这种现象：

  


```CSS
body:has(dialog[open]){
    overflow: hidden;
}
```

  


除此之外，CSS 的 **`overscroll-behavior`** 的属性也可以阻止滚动穿透，让你可以控制浏览器过度滚动时的表现——也就是滚动到边界。

  


要是你给滚动容器指定 `overscroll-behavior` 属性的值为 `contain` ，默认的滚动边界行为不变（“滚动触底”或者刷新），但是可以阻止滚动穿透。比如，模态框滚动到底部时，位于其底下的 `body` 不会有滚动行为：

  


```CSS
.modal--content {
    overflow-y: auto;
    overscroll-behavior-y: contain;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1dec4ec2c4af4791ba2abf9b18ab9c9b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1586&h=834&s=333937&e=gif&f=142&b=fcfbfe)

  


我录制了一张 GIF 图来向大家呈现 `overscroll-behavior` 值为 `auto` 和 `contain` 的差异：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aee7b4d0393341739c8a5ed317415ded~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=820&h=606&s=2436633&e=gif&f=162&b=6fc4ce)

  


> Demo 地址：https://codepen.io/airen/full/RwYqejQ

  


除此之外，`overscroll-behavior` 还可以取 `none` 值，它除了可以帮助我们阻止滚动穿透之外，同时也会阻止滚动到边界时的反弹及刷新页面的效果，即移除滚动至顶部或底部的默认滚动特效（例如 Android 上的 Chrome 当滚动超过顶部边界时会刷新页面）。

  


```CSS
body {
    overscroll-behavior: none;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9e7227f9001c4d0b95ffe77d07978a65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1242&h=586&s=3135211&e=gif&f=337&b=70c5ce)

  


> Demo 地址：https://codepen.io/airen/full/qBMQJJX

  


注意，如果要移除滚动至顶部或底部的默认滚动特效，需要在 `html` 或 `body` 元素上设置 `overscroll-behavior` 的值为 `none` 。这也是禁用原生下拉刷新最有效的方案，而且它对于我们定制一个下拉刷新是非常有利的。否则就会出现两个下拉刷新的效果（一个是原生的，一个是定制的）。

  


## 滚动捕捉

  


滚动捕捉是滚动感知 UI 的另一个经典场景，你可以通过 CSS 滚动捕捉相关的属性（`scroll-snap-*` ，比如 `scroll-snap-type` ）实现简单的滚动对齐效果。例如：

  


  


```HTML
<ul class="list">
    <li>
        <div class="wrapper">
            <img src="https://picsum.photos/800/600?radom=154" alt="" />
            <h3>Introducing</h3>
            <p>Lorem ipsum, dolor sit amet consectetur adipisicing elit. Et consequuntur dolor similique.</p>
        </div>
    </li>
    <!-- 还有几个类似的列表项 -->
</ul>
```

  


```CSS
.list {
    scroll-snap-type: x mandatory;
    overflow-x: auto;
    scroll-padding-inline: 2ch;
    
    /* 其他 CSS */
    
    li {
        scroll-snap-align: center;
        /* 其他 CSS */
    }
}
```

  


使用 CSS 滚动捕捉相关的样式，我们就创建了一个滚动捕捉的场景，你将得到一个类似下图所示的效果：

## 使用 CSS 滚动驱动动画实现滚动感知

  


CSS 滚动驱动动画提供了一种新的、易于访问的方式来控制 CSS 动画的进度：滚动进度时间线和视图进度时间线。在滚动和视图进度线出现之前，我们只能使用默认的文档时间线。

  


文档时间线比较脆弱，缺乏自定义选项，并且会从页面初次加载时间连续进展，Web 页面动画会在页面加载时自动开始，并在指定的动画持续时间内继续播放。在滚动和视图进度时间线出现之前，没有灵活性来控制这种行为。现在，我们可以通过滚动进度时间线和视图进度时间线来改变这一切。

  


通过滚动驱动的 CSS 动画，你可以轻松地创建动态视觉效果，而无需依赖 JavaScript。这带来了更流畅的性能和更高的控制力，使用用户体验更自然、直观，吸引访问者持续参与并渴望发现更多的内容。它还为你的 Web 页面添加了精致感和互动性，使你的设计与众不同。

  


我们继续以“返回顶部按钮”为例。我们来给“返回顶部”按钮添加指示器效果（实现类似于 theneedledrop.com 网站上的“返回顶部”按钮效果）。

  


首先，我们在上面示例的基础上，给“返回顶部”按钮设置一个渐变背景：

  


```CSS
.back-to-top {
    --yellow: oklch(85.33% 0.185 92.12);
    --white: oklch(100% 0 0);
    background-image: linear-gradient(var(--yellow), var(--white));
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ef5b920eab964bf9acf02958552abc8c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=601&s=101979&e=jpg&b=566262)

  


为了避免一种颜色渐变到另一种颜色，我们添加颜色停止点：

  


```CSS
.back-to-top {
    background-image: linear-gradient(var(--yellow) 0% 50%, var(--white) 50% 100%);
}
```

  


我将第一个颜色从 `0%` 开始，到 `50%` 停止，第二个颜色从 `50%` 开始，到 `100%` 结束。效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6ae89b7af30e41c6b298f4d8e889f08a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=499&s=81412&e=jpg&b=566262)

  


实际上，我们希望它从 `0%` 开始，逐渐过渡到 `100%` 。

  


接下来，我们需要将颜色停止点放在自定义属性中，因为我们希望对它进行动画处理：

  


```CSS
.back-to-top {
    background-image: linear-gradient(var(--yellow) 0% var(--progress), var(--white) var(--progress) 100%);
}
```

  


传统上，动画会在文档时间线上运行。计时器从页面加载时的 `0` 开始，随着时间推移向前跳动。在此之前，这是我们唯一可以访问的动画时间线。随着 CSS 滚动驱动动画的出现，我们有了两个额外的时间线。其中一个与滚动容器在某个轴上的滚动位置相关联。为了使用它，我们像平常一样创建动画，但将 `animation-timeline` 属性设置为 `scroll()`。

  


```CSS
 @keyframes revealScroller {
    0%,
    10% {
        opacity: 0;
         translate: 0 100%;
    }
    18%,
    100% {
        opacity: 1;
        translate: 0 0;
    }
}

@keyframes progress {
    0% {
        --progress: 0%;
    }
    100% {
        --progress: 100%;
    }
}

.back-to-top {
    animation: revealScroller 0.2s ease-out, progress linear;
    animation-timeline: scroll();

    --yellow: oklch(85.33% 0.185 92.12);
    --white: oklch(100% 0 0);
    background-image: linear-gradient(
        var(--yellow) 0% var(--progress),
        var(--white) var(--progress) 100%
    );

    position: fixed;
    bottom: 1rem;
    right: 1rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b0fa9a67e504006aebf7cbe797f20ef~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1406&h=608&s=18628250&e=gif&f=76&b=4b5b5a)

  


到现在为止，我们仅看到了淡入淡出的动画效果（`revealScroller`），但并没有看到 `progress` 相关的动画效果。这是因为 `--progress` 是一个字符串，我们无法对字符串进行动画化。庆幸的是，[我们可以通过 `@property` 来注册 `--progress`](https://juejin.cn/book/7395119173656903715/section/7446278656840138762) ，并指定它的类型为 `<length-percentage>` 。比如：

  


```CSS
@property --progress {
    syntax: '<length-percentage>';
      inherits: false;
  initial-value: 0%;
}
```

## 滚动状态容器查询

  


CSS 引入了新的一种容器查询——**[滚动状态容器查询](https://juejin.cn/book/7395119173656903715/section/7497444799608258600)**，允许你根据元素的滚动状态（例如，当元素变为粘性、对齐到滚动点或溢出）直接应用样式变得非常容易，而不需要依赖 JavaScript 或滚动进度时间线以及视图进度时间线。使用滚动状态容器查询，你现在可以直接在 CSS 中查询和响应元素的滚动状态。

  


我们回到之前的滚动捕捉的示例中来：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2f1d39e1a8b84a148b1075bef24e957f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1338&h=666&s=20484893&e=gif&f=101&b=fcf6f6)

  


现在，我们希望，当列表项被捕捉时，位于中间的卡片能改变背景颜色，并且调整未被捕捉到的卡片透明度。

  


为此，我们将添加 `scroll-state` 容器查询，并将 `scroll-state` 类型设置为 `snapped` ，然后指定我们希望捕捉的方向，在这种情况上是 `inline` ，也可以是 `x` ：

  


```CSS
.list {
    scroll-snap-type: x mandatory;
    overflow-x: auto;
    scroll-padding-inline: 2ch;

    li {
        scroll-snap-align: center;

        .wrapper {
            background: slateblue;
            transition: all 0.4s ease-out;
        }

        @supports (container-type: scroll-state) {
            container-type: scroll-state;
    
            .wrapper {
                /* 吸附的卡片调整背景颜色 */
                @container scroll-state(snapped: x) {
                    background: lightcoral;
                }
                /* 未吸附的卡片调整透明度 */
                @container not scroll-state(snapped: x) {
                    opacity: 0.25;
                }
            }
        }
    }
}
```

上面这两个示例仅仅展示的是滚动状态查询中的吸附特性（`snapped`），它通常与 CSS 滚动捕捉结合使用。除此之外，它还有可滚动状态（`scrollable`）与粘附状态（`stuck`），其中粘附状态通常与粘性元素（`position:sticky`）结合使用。

  


其中，最常见的粘附（`stuck`）查询用例之一是导航栏：当导航栏粘附（`stuck`）时添加 `box-shadow` ，让它看起来像悬浮在页面内容之上。这是一个非常实用的视觉增强手法，使粘附元素在滚动时更具层次感。

  


让我们先设置基础的 HTML 结构：

  


```HTML
<div class="topbanner">
    <!-- 顶部横幅，非粘性 -->
</div>
<div class="stuck-top"><!-- 查询容器 -->
    <!-- stuck-top 将是粘性的 -->
    <div class="heading"><!-- 样式响应元素 -->
        <h1>Logo</h1>
        <nav><!-- 导航项 --></nav>
    </div>
</div>
<section class="content">
    <!-- 包含 main 和 aside 的大量内容 -->
</section>
```

  


接下来设置 `container-type` 和粘性定位：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
    }
}
```

  


为了让效果更加平滑，我们还需要给 `.heading` 添加一个过渡效果，用来平滑显示阴影：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
        
        .heading {
            transition: box-shadow 0.5s ease-out;
        }
    }
}
```

  


现在我们的 `container-type` 已经设置好了，剩下的就是创建状态查询，并在 `.stuck-top` 被固定时给 `.heading` 添加阴影：

  


```CSS
.stuck-top {
    /* 粘性定位，是必须要的 */
    position: sticky;
    top: 0;
    
    @supports (container-type: scroll-state) {
        container-type: scroll-state; /* 定义滚动状态查询容器 */
        
        .heading {
            transition: box-shadow 0.5s ease-out;
            
            @container scroll-state(stuck: top) {
                box-shadow: 
                    rgb(0 0 0 / 0.6) 0px 12px 28px 0px,
                    rgb(0 0 0 / 0.1) 0px 2px 4px 0px,
                    rgb(255 255 255 / 0.05) 0px 0px 0px 1px inset;
            }
        }
    }
}
```

## 使用 CSS 实现滚动感知 UI 的局限性

  


虽然 CSS 提供了强大的工具来创建滚动感知的 UI 元素，但它也存在一些固有的局限性，具体如下：

  


-   **有限的事件处理**：CSS 缺乏直接的滚动事件监听器，无法在用户滚动到特定元素或位置时精确触发动作
-   **间接的状态管理**：CSS 无法根据滚动位置直接改变元素的状态；它依赖于像 `:hover` 或 `:active` 这样的伪类，但这些对复杂的交互有一定的局限性
-   **性能考虑**：复杂的 CSS 滚动感知效果，特别是涉及多个过渡或动画的效果，可能会影响资源受限设备的性能
-   **浏览器兼容性**：并非所有 CSS 滚动感知属性在所有浏览器中都有一致的支持。因此，必须考虑浏览器兼容性，并在必要时提供填充或回退
-   **有限的交互范围**：仅使用 CSS 无法处理更复杂的基于滚动的交互，例如动态内容加载、无限滚动和基于滚动位置的实时更新

  


## 小结

  


CSS 提供了实现滚动感知 UI 状态的有前景的能力，为 Web 增加了一层视觉吸引力和互动性。通过理解滚动时间线、滚动关联动画、浏览器兼容性和潜在的 JavaScript 集成，开发人员可以创建动态且引人入胜的用户体验，能够无缝响应用户操作。

  


实现滚动感知设计的方式有很多种，几乎没有限制，只有想象力的限制。
