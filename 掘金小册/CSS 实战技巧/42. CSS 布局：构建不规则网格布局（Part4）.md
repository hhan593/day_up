## CSS 网格与形状相结合

  


我们之前花了好几节课专门介绍了 CSS 网格布局技术与 CSS 裁剪、遮罩、渐变等特性相结合来构建各种不规则的网格布局，例如[菱形网格](https://juejin.cn/book/7395119173656903715/section/7438887593427206184#heading-1)、[三角形网格](https://juejin.cn/book/7395119173656903715/section/7438887593427206184#heading-2)、[六边形网格](https://juejin.cn/book/7395119173656903715/section/7438887593427206184#heading-3)、[拼图网格](https://juejin.cn/book/7395119173656903715/section/7439748226019754036#heading-0)、[嵌套网格](https://juejin.cn/book/7395119173656903715/section/7439748226019754036#heading-1)和[交叉网格](https://juejin.cn/book/7395119173656903715/section/7441499053528973364)等。接下来，我们将会把网格布局与平行四边形、[锯齿形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-0)、[圆形凹槽形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-1)、[凸起形状](https://juejin.cn/book/7395119173656903715/section/7403197889905688585#heading-2)等相结合，构建具有不同形状的并且可交互的扩展面板。

上图曾经是 An Event Apart 网站的一个活动宣传页的一部分。我在该示例基础上做了一些调整，接下来将要完成的是像下图这样的效果：

构建这个效果，我们需要一个 HTML ：

  


```HTML
<header>
    <h1>An Event Apart:Set yourself Apart.</h1>
    <p>Build your skills in the company of the smartest developers, designers, and strategists in the industry.</p>
</header>
<main>
    <ul class="monoliths">
        <li>
            <a href="">
                <picture>
                  <img src="https://aneventapart.com/assets/img/landing/monoliths/rachel-andrew.jpg" alt="">
                </picture>
                <h4>Rachel Andrew</h4>
                <span>New CSS Features</span>
            </a>
        </li>
    </ul>
</main>
```

  


首先在 `body` 上定义一个网格，可以让图片展示区域 `main` 占用除 `header` 之外的剩余区域：

  


```CSS
body {
    display: grid;
    grid-template-rows: auto min-content;
}
```

  


使用 `grid-row` 和 `grid-column` 分别将 `header` 和 `main` 放置到指定的网格区域：

  


```CSS
header {
    grid-row: 2 / 3;
}
main {
    grid-row: 1 / 2;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3fc3e87151f5411b8526fc8aead6d053~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=143301&e=jpg&b=0f0f0f)

  


同时在 `header` 定义一个网格（它嵌套 `body` 网格中）：

  


```CSS
header {
    display: grid;
    grid-template-rows: 2rem min-content min-content;
    gap: 1rem;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2dfc7819cca54ecdbf5b211ff282c116~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=157800&e=jpg&b=f2e7e4)

  


在这个示例中，使用 `header` 的伪元素 `::after` 绘制了一个斑马线效果，并且使用 `grid-row` 指定其放置的位置：

  


```CSS
header::before {
    grid-row: 1 / 2;
}
```

  


`header` 的另外两个子元素，使用的是网格自动放置的方式来指定其位置。

  


现在剩下的是主区域 `main` 的布局。在分享者列表 `.monoliths` 中重新定义了一个网格，一个 `17 x 3` （`17` 列，`3` 行的网格），每一列的列宽都是 `minmax(0, 1fr)` （相当于 `1fr` ），行网格轨道高度是，图片区域占用剩下区域空间，列表中的标题 `h4` 和描述文本 `span` 占用的高度是根据其自身内容高度来决定，即 `min-content` ：

  


```CSS
.monoliths {
    display: grid;
    grid-template-columns: repeat(17, minmax(0, 1fr));
    grid-template-rows: auto min-content min-content;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/afbe5dcd873843acbc286143d0c576bb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=153846&e=jpg&b=0f0f0f)

  


这里有一点需要特别注意，每个列表项跨越了 `5` 列，并且相邻的列都有一个单元格是相互重叠的：

  


```CSS
.monoliths li {
    grid-row-start: 1;
    grid-row-end: span 3;
    grid-column-end: span 5;
}

.monoliths li:nth-child(1) {
    grid-column-start: 1;
}

.monoliths li:nth-child(2) {
    grid-column-start: 5;
}

.monoliths li:nth-child(3) {
    grid-column-start: 9;
}

.monoliths li:nth-child(4) {
    grid-column-start: 13;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1153b9c3da9444e89aa7ee1b8c41ac74~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=143836&e=jpg&b=0e0e0e)

  


这样做的主要目的是，为了避免在每个列表项中使用 `clip-path` 来裁剪列表项时，造成较大的空间：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2069445c0137463795c0c42f59d613d3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=157638&e=jpg&b=0d0d0d)

  


我们真正期望的是 `li` 运用 `clip-path` 达到下图这样的效果，所以需要列与列之间有一个列轨道是相互重叠的：

  


```CSS
.monoliths li {
    clip-path: polygon(25% 0, 100% 0, 75% 100%, 0 100%);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a236f963da2a4591a98febb5b3e7c1ab~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1142&h=780&s=153846&e=jpg&b=0f0f0f)

  


另外，为了更好控制每一列的参数，图片 `picture` 、标题 `h4` 和描述文本 `span` 能更好的对齐。这里还使用了 CSS Grid 布局中的子网格 `subgrid` 特性。即，在 `li` 上定义了一个网格，并且使用 `subgrid` 特性，继承其父网格行网格轨道的相关参数：

  


```CSS
.monoliths li {
    display: inherit;
    grid-template-rows: subgrid;
    row-gap: 0.2rem;
}
```

  


需要注意的是，在该示例中，`<picture>` 、`<h4>` 和 `<span>` 都被包裹在 `<a>` 标签元素内，它们不是 `<li>` 标签的子元素，也就无法成为 `li` 网格（它定义为一个网格）的网格项目。所以，需要显式将 `<a>` 元素的 `display` 的值设置为 `contents` ：

  


```CSS
.monoliths a {
    display: contents;
}
```

  


这样做的好处是，可以让 `<picture>` 、`<h4>` 和 `<span>` 变成 `<li>` 网格中的网格项目，它们也可以使用 `grid-row` 和 `grid-column` 来放置位置：

  


```CSS
.monoliths h4 {
    grid-row: 2;
    grid-column: 1 / -1;
}

.monoliths span {
    grid-row: 3;
    grid-column: 1 / -1;
}

.monoliths li::after {
    grid-row: 2 / 4;
    grid-column: 1 / -1;
    z-index: -1;
}
```

## 网格过渡动效

  


刚才提到过，我们要创建的扩展面板是一个经典的设计模式，它只有一行和一个全宽容器。接下来，我们主要来探讨一下网格过渡动效。因为我们后面的案例需要用到这个交互动效。

  


在开始进入网格之前，我们先以 [CSS Flexbox ](https://juejin.cn/book/7161370789680250917/section/7161621092560273439)创建的示例开始，然后逐步过渡到网格布局。这样做的出发点是考虑大部分 Web 开发者对 CSS Flexbox 布局更熟悉。

  


假设我们有一个类似下面这样的 HTML 结构：

  


```HTML
<div class="gallery">
    <img src="figure.png" alt="" />
    <!-- 省略其他 img -->
</div>    
```

  


我们可以通过调整 Flex 项目的缩放因子来改变其大小：

  


```CSS
.gallery {
    width: min(100%, 100vw);
    margin-inline: auto;
    display: flex;
    gap: 10px;

    img {
        flex: 1;
        min-width: 0;
        width: 100%;
        height: 100vh;
        object-fit: cover;
        object-position: center;
        filter: grayscale(80%);
        transition: 0.35s linear;
        cursor: pointer;

        &:hover {
            flex: 6;
        }
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0e5b7817c8fd4bcda4c8b44f8e50eabe~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1174&h=528&s=15534587&e=gif&f=65&b=d4d2cd)

  


> Demo 地址：https://codepen.io/airen/full/PwYomeq

  


在这个示例中，Flex 项目（即 `img` 元素）的 `flex` 属性的值为 `1` ，它会使每一个 Flex 项目均分 Flexbox 容器宽度，使得每个项目的宽度相等（即容器宽度的 `1/8`）。然后，在悬浮状态（`:hover`）调整当前项目的 `flex` 属性的值，这里设置的是 `6` 。此时，相当于将 Flexbox 容器宽度均分成 `13` 分，悬浮状态的 Flex 项目占其中的 `6` 份，其它的 Flex 项目各占 `1` 份。因此，当前的 Flex 项目就会变宽。从而得到一个简单的扩展面板的交互动效。

  


注意，Flexbox 的 `flex` 属性计算 Flex 项目尺寸是一个复杂的过程，如果你对这方面知识感兴趣的话，个人强烈建议你花一些时间阅读下面这几节课的内容，将能帮助你更好的掌握 Flexbox 布局：

  


-   [Flexbox 布局中的 `flex` 属性的基础运用](https://juejin.cn/book/7161370789680250917/section/7161623755972673550)
-   [Flexbox 中的计算：通过扩展因子比例来扩展 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7161623797794078750)
-   [Flexbox 中的计算：通过收缩因子比例收缩 Flex 项目](https://juejin.cn/book/7161370789680250917/section/7164357320367931399)
-   [Flexbox 布局中的 flex-basis：谁能决定 Flex 项目的大小？](https://juejin.cn/book/7161370789680250917/section/7161623717074698247)

  


接下来，我们从 Flexbox 布局过渡到 CSS Grid 布局。

  


接触过 CSS Grid 布局的开发者应该知道，[在 Grid 布局中有一个 `fr` 单位](https://juejin.cn/book/7161370789680250917/section/7161624007702216735)，它的计算与 Flexbox 的 `flex:1` 非常类似。可以通过下面的方式使每个网格项目的宽度相等（均分网格容器的宽度）：

  


```CSS
.gallery {
    width: min(100%, 100vw);
    margin-inline: auto;
    display: grid;
    gap: 10px;
    grid-template-columns: repeat(8, minmax(0,1fr))
}    
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a15d70be7cc4ffc95dc946c763a0878~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=779&s=1072024&e=jpg&b=d8cbc5)

  


虽然这种方法可以很好的让我们使每个网格项目的宽度相等，但我们不好使网格项目在悬浮状态时变宽。庆幸的是，我们可以利用 [CSS Grid 的隐式网格的特性](https://juejin.cn/book/7395119173656903715/section/7434843979486593062)，它将根据图片的数量自动创建网格的列数。我们要做的是，在网格容器中设置 `grid-auto-flow` 属性的值为 `column` ：

  


```CSS
.gallery {
    display: grid;
    gap: 10px;
    grid-auto-flow: column;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e54dec4908f946169d27626aeb921129~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=939&s=1021724&e=jpg&b=d8d5d0)

  


这是一个非常简单的网格布局，关键之处是网格项目（图片）的样式设置。我们为 `img` 添加下面这段 CSS：

  


```CSS
.gallery {
    display: grid;
    gap: 10px;
    grid-auto-flow: column;

    img {
        width: 0;
        height: 0;
        min-height: 100%;
        min-width: 100%;
        object-fit: cover;
        object-position: center;
        transition: all .5s;
        cursor: pointer;
        filter: grayscale(80%);
    }
}
```

  


我们即将实现的悬停效果依赖这段 CSS。你可能会觉得奇怪：为什么我们要让 `img` 的 `width` 和 `height` 为 `0` ，但又设置 `min-width` 和 `min-height` 为 `100%` ？这是一个非常巧妙的小技巧，能够实现我们想要的效果。

  


这里的逻辑是，它会告诉浏览器 `img` 的 `width` 和 `height` 都为 `0` ，但最小宽度和最小高度要等于 `100%` 。那么 `100%` 是相对于谁计算呢？在这个示例中，图片被放置在网格单元格中，而我们需要知道单元格的大小，才能理解 `100%` 是相对于什么。

  


浏览器在计算网格单元格大小时，会先忽略 `min-height: 100%` ，但会考虑 `height: 0` 。也就是说，图片不会对网格单元格的大小产生影响，因为它们在技术上并没有物理尺寸。这会使得网格的所例相等（在这个示例中只有列，如果有行的话，也会使网格行尺寸相等），网格单元格的大小是基于网格容器的大小来计算。

  


现在，我们已经得到了网格单元格的尺寸，浏览器会使 `min-width: 100%` 和 `min-height: 100%` ，会强制图片完全填满每个单元格的空间。整个过程可能看起来有点令人感到困惑，但核心思想是让网格的尺寸来定义图片的大小，而不是让图片来定义网格的大小。为什么这么做会更好？在我们添加悬浮效果之后，你就会明白了：

  


```CSS
.gallery {
    display: grid;
    gap: 10px;
    grid-auto-flow: column;

    img {
        width: 0;
        height: 0;
        min-height: 100%;
        min-width: 100%;
        object-fit: cover;
        object-position: center;
        transition: all .5s;
        cursor: pointer;
        filter: grayscale(80%);
        
        &:hover {
            filter: grayscale(0);
            width: 20vw;
        }
    }
}
```

  


注意，我们在图片的悬浮状态（`:hover`）下重置了其 `width` 的值为 `20vw` 。是不是更困惑了，刚才不是说图片的尺寸必须为 `0` 吗？怎么这又将图片的宽度设置为 `20vw` 呢？

  


这并不矛盾，我们只是对悬浮状态下的图片做了一个例外。告诉浏览器，只有一个被悬浮的图片会拥有非零的尺寸。这张图片会影响网格的大小，而其他图片的尺寸依然为 `0` 。简单地说，如果没有任何图片处于悬浮状态时，所有图片都没有物理尺寸，因此所有网格单元的大小相等。换句话说，只有悬浮状态的图片才有物理尺寸，从而影响了对应网格单元格的大小。浏览器会在悬浮状态时使这个特定的网格单元格变大，而其他网格单元格会因为网格容器的整体尺寸是固定的而变小，以保持网格整的尺寸不变。这样就实现了与 Flexbox 类似的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3fef64227404418d8e590ef56eb7e7ac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1176&h=524&s=20305544&e=gif&f=68&b=d4d1cd)

  


> Demo 地址：https://codepen.io/airen/full/ZYzEJxQ

  


既然聊到这个效果，就多花一点时间进一步扩展一下。我们可以在这个示例基础上引入一些 [CSS 自定义属性](https://juejin.cn/book/7223230325122400288/section/7249357815410589733)来控制网格过渡动画效果：

  


-   `--n` 控制网格行数
-   `--m` 控制网格列数
-   `--g` 控制网格轨道之间的间距
-   `--f` 控制网格项目（例如图片）的缩放比例

  


将这些变量放到上面示例中：

  


```CSS
.gallery {
    --n: 1;
    --m: 8;
    --g: 10px;
    --f: 0.8;
    width: 100vw;
    height: 100vh;
    display: grid;
    gap: var(--g);
    grid-template-columns: repeat(var(--m), auto);

    img {
        width: 0;
        height: 0;
        min-height: 100%;
        min-width: 100%;
        object-fit: cover;
        object-position: center;
        transition: all 0.5s;
        cursor: pointer;
        filter: grayscale(80%);

        &:hover {
            filter: grayscale(0);
            width: calc(100vh * var(--f) / var(--n));
            height: calc(100vw * var(--f) / var(--m));
        }
    }
}
```

  


上面代码也能使我们获得一个相同的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8028fd245ab342b8adafda54b066d386~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1176&h=524&s=20305544&e=gif&f=68&b=d4d1cd)

  


> Demo 地址：https://codepen.io/airen/full/xbKxXQQ

  


在这个示例的基础上，我们稍微调整一下，就可以创建一个 `m x n` 的全屏图片画廊的效果。假设 `--n` 等于 `4` ，`--m` 等于 `6` ，并且网格容器的宽度为 `100vw` ，高度为 `100vh` ，我们就可以通过下面的公式计算出图片的宽度（`--w`）和高度（`--h`）：

  


```
100vw = var(--m) * var(--w) + (var(--m) - 1) * var(--g)
100vh = var(--n) * var(--h) + (var(--n) - 1) * var(--g)

--w: (100vw - (var(--m) - 1) * var(--g)) / var(--m);
--h: (100vh - (var(--n) - 1) * var(--g)) / var(--n);
```

  


将它们放到 CSS 中：

  


```CSS
.gallery {
    --n: 4;
    --m: 6;
    --g: 10px;
    --f: 0.8;
    --h: calc((100vh - (var(--n) - 1) * var(--g)) / var(--n));
    --w: calc((100vw - (var(--m) - 1) * var(--g)) / var(--m)); 
    
    width: 100vw;
    height: 100vh;
    display: grid;
    gap: var(--g);
    grid-template-columns: repeat(var(--m), auto);

    img {
        width: 0;
        height: 0;
        min-height: 100%;
        min-width: 100%;
        object-fit: cover;
        object-position: center;
        transition: all 0.5s;
        cursor: pointer;
        filter: grayscale(80%);

        &:hover {
            filter: grayscale(0);
            width:calc(var(--w) * var(--f));
            height: calc(var(--h) * var(--f));
        }
    }
}
```

  


你将获得像下图所展示的效果：

## 创建不规则形状的扩展面板

  


我们回到网格过渡动效的最简单版本当中——网格是一个一行多列的网格，并且网格列数由网格项目数量来决定。例如，我们将创建一个具有六个面板（即六列）的网格。在这个示例中，我们不再需要行数 `--n` 和列数 `--m` 等 CSS 自定义属性，只需要使用 `grid-auto-flow: column` 使所有网格项目按列排列。

  


```CSS
.gallery {
    --g: 10px;
    --f: 0.8;

    width: 100vw;
    height: 100vh;
    display: grid;
    gap: var(--g);
    grid-auto-flow: column;
    
    img {
        width: 0;
        height: 0;
        min-height: 100%;
        min-width: 100%;
        object-fit: cover;
        object-position: center;
        transition: all 0.5s;
        cursor: pointer;
        filter: grayscale(80%);

        &:hover {
            filter: grayscale(0);
            width: calc(100vw / 6 * var(--f));
        }
    }
}
```

  


到目前为止，它就是一个简单的带有交互动效的扩展面板：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5a9078ab48c44a72a924225269959138~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1242&h=550&s=16537514&e=gif&f=50&b=847d78)

  


接下来，我们需要使用 `clip-path` 将图片裁剪成一个平行四边形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/08cad9d87d6e43168b7b70e349ddf470~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2524&h=1642&s=191481&e=jpg&b=fafbf6)

  


注意，在这里我将引入另一个 CSS 变量，比如 `--s` 用来控制平行四边形：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/99ae777ae8144a40a3bd88937cd4eae3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1266&s=645586&e=jpg&b=020202)

  


```CSS
.gallery {
    --s: 30px;
    /* 省略其他 CSS */
    
    img {
        clip-path: polygon(var(--s) 0%,100% 0%,calc(100% - var(--s)) 100%,0% 100%);

        /* 省略其他 CSS */
    }
}
```

正如你所看到的，我们使用 `clip-path` 将图片裁剪成平行四边形，但图片与图笥之间的间距变大了：

我们需要增加图片的宽度来创建重叠效果。需要将 `img` 的 `min-width` 属性的 `100%` 替换为 `calc(100% + var(--s))` :

  


```CSS

        min-width: calc(100% + var(--s));
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34763cc305d04edab10329a76e481a7a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1286&h=650&s=19570283&e=gif&f=41&b=817c79)

  


接下来，我们需要修复第一张和最后一张图片，使它们在没有间隙的情况下“溢出”网格容器。换句话说，我们需要去除第一张图片左侧的斜切和最后一张图片右侧的斜切。我们需要为这两张图片设置新的 `clip-path` 。

  


另外，我们还需要调整溢出效果。默认情况下，所有图片都会在两侧溢出，但对于第一张图片，我们只需要在右侧溢出；而对于最后一张图片，我们只需要在左侧溢出：

  


```CSS
.gallery {
    --s: 30px;
    /* 省略其他 CSS */
    
    img {
        min-width: calc(100% + var(--s));
        clip-path: polygon(var(--s) 0%,100% 0%,calc(100% - var(--s)) 100%,0% 100%);

        /* 省略其他 CSS */
        
        &:first-child {
            min-width: calc(100% + var(--s)/2);
            place-self: start;
            clip-path: polygon(0% 0%,100% 0%,calc(100% - var(--s)) 100%,0% 100%);
        }
        
        &:last-child{
            min-width: calc(100% + var(--s)/2);
            place-self: end;
            clip-path: polygon(var(--s) 0%,100% 0%,100% 100%,0% 100%);
        }
    }
}
```
