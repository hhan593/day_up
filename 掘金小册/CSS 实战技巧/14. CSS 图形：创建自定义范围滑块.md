## 概述

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8faa06f77651446c84ac03e0d1310f9c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1112&s=222389&e=jpg&b=f3e29c)

  


范围滑块（`<input type="range">`）是一种常见的表单控件，允许用户通过拖动滑块在给定的范围内选择一个值，为 `<input type="number">` 提供了一种替代输入类型。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/691f76c5e8304656a101414c1d32acee~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1406&h=704&s=584378&e=gif&f=153&b=eff6f7)

  


上图中，红色滑块是 `<input type="range" />` ，下面两个输入框是 `<input type="number" />` 。两者相比，我个人认为范围滑块通过提供直观的用户界面，使得复杂的数据输入和交互变得更加简单和直观：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/202fd1ad61ef4281b8945ca322ac84ba~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=671&s=286890&e=gif&f=79&b=f3f7f9)

  


范围滑块在 Web 中的使用场景非常广泛，例如表单输入、多媒体控制、数据可视化和交互和进度调节等。它在 Web 中的使用非常简单，显式给 `<input>` 元素的 `type` 属性指定为 `range` 即可：

  


```HTML
<input type="range" /> 
```

  


它除了具有我们熟悉的通用属性（例如 `class` 、`name` 、 `id` 等）之外，还包括 `value` 、`min` 、`max` 和 `step` 几个关键属性：

  


-   `value` 属性表示滑块的当前值，滑块的位置会随着这个值的改变而改变
-   `min` 属性定义了滑块所允许的最小值，默认为 `0` ，也可以是小于 `0` 的负值
-   `max` 属性定义了滑块所允许的最大值，默认为 `100` ，也可以是大于 `100` 的值
-   `step` 属性定义了滑块的移动步长。用户只能选择 `min` 和 `max` 之间的某些特定值。例如 `step="10"` 意味着用户只能选择 `10` 的倍数，比如 `0` 、`10` 、`20` 之类的

  


这些值不是必须值。不过，个人建议在使用范围滑块时，尽可能的都显式的设置这些属性：

  


```HTML
<input type="range" id="price" min="0" max="120" value="20" step="10" />
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/00d0e1edc3b74cdcb94c018f83281f6d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1262&h=542&s=443074&e=gif&f=149&b=1e5b6d)

  


但在实际使用时，应该尽可能的使 HTML 更具语义。我通常会使用下面这两种方式来编写 HTML 结构：

  


```HTML
<label>
    <span class="sr-only">Price Range:</span>
    <input type="range" id="price" min="0" max="120" value="20" step="10" />
    <output for="price">20</output>
</label>  

<!-- 或者 -->
<div class="control">
    <label for="price" class="sr-only">Price Range:</label>
    <input type="range" id="price" min="0" max="120" value="20" step="10" />
    <output for="price">20</output>
</div>
```

  


注意，一般只有在不需要显示文本时才应用 `sr-only` 类名，它所应用的 CSS 会在视觉上将文本信息“Price Range” 隐藏！

  


不过，我们这节课的目的是样式化范围滑块，并使它能在所有浏览器中保持一致的外观。因此，我们接下来将重点放在 CSS 上，但在开始编写 CSS 之前，先简单的介绍一下范围滑块的组成部分。

  


范围滑块主要由滑块（Thumb，有时也称手柄 Handle）和轨道（Track）部分组成：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2180b2e2f3c944c19cb4cf67981b2f37~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1383&s=347971&e=jpg&b=add9e6)

  


-   **滑块轨道（Track）：** 这是滑块（Thumb）所在的背景轨道，通常表示值的最小值（`min`）和最大值（`max`）之间的范围。换句话说，它是表示可选择的值范围的长条形元素
-   **滑块（Thumb）：** 滑块是用户交互的主要元素，它显示在轨道（Track）上，可以左右拖动，用户可以移动它来选择不同的范围值，它的位置表示当前选择的数值（`value`）

  


而我们要创建自定义的范围滑块，就需要在 CSS 中针对这两个部分进行样式设置。另一部分是进度条（Progress），它也可以通过 CSS 来设置样式，并且可以不依赖任何 JavaScript 脚本就可以更新。这个稍后会详细介绍。

  


## 为什么要自定义范围滑块？

  


范围滑块和[上一节课讨论的进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)（`<progress>`）类似，默认情况之下，它在四大主流浏览器中的渲染结果是有所差异的。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c5cd63c11a0a4be9b59384d1bb682781~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1084&s=295575&e=jpg&b=b7dce9)

  


注意，随着浏览器的更新，渲染效果可能会有所变化。

  


造成这种现象的主要原因是，`<input type="range">` 是一个原生 HTML 元素，每个浏览器厂商都有自己实现这种元素的方式。主要有两种不同的实现方式：

  


```HTML
<!-- Chrome 和 Edge -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div>
        <div pseudo="-webkit-slider-runnable-track" id="track">
            <div id="thumb"></div>
        </div>
    </div>
</input>

<!-- Safari -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div pseudo="-webkit-slider-container">
        <div pseudo="-webkit-slider-runnable-track">
            <div pseudo="-webkit-slider-thumb"></div>
        </div>
    </div>
</input>

<!-- Firefox -->
<input type="range" id="price" min="0" max="120" step="10" value="20">
    <div></div> <!-- track -->
    <div></div> <!-- progress -->
    <div></div> <!-- thumb -->
</input>
```

  


就 ` <input type="range"  />` 元素内部暴露的 HTML 结构（Shadow DOM）来看，Chrome 、Safari 和 Edge 是一样的。 Firefox 浏览器暴露的 HTML 结构则完全不同，它们不是相互嵌套，而且平级的三个 `div` 存在，这三个 `div` 分别是范围滑块的轨道（Track）、进度（Progress）和滑块（Thumb）。

## 自定义范围滑块

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/54e0aba8013a4c8ab91c7635c35766cc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1500&h=932&s=269739&e=jpg&b=ffffff)

  


既然我们已经看到每个浏览器渲染范围滑块（`<input type="range" />`）时存在多么严重的不一致性，接下来我们将看看如何使用 CSS 来统一它们的样式。

  


### 基础样式

  


刚才我们提到过，每个主流浏览器美化范围滑块的样式差异较大。因此，我们要自定义范围滑块，就需要从一个公平的起点开始。一旦各浏览器应用的默认样式被重置和禁用，我们就可以开始创建一个更统一的范围滑块的外观（UI）。

  


在 CSS 中，我们通常使用 `appearance: none` 和一些其他常见属性来重置和禁用所有浏览器给范围滑块提供的默认样式：

  


```CSS
input[type="range"] {
    appearance: none;        /* 禁用浏览器默认样式 */
    background: transparent; /* 清除应用于 input 的默认背景 */
    cursor: pointer;         /* 设置光标为手型 */
}
```

  


我在这里使用了 `input[type="range"]` 选择器来选择目标元素（即范围滑块）。注意，在更复杂的情况下，我们可能需要添加更多代码，以防其他默认样式应用到我们的元素上。不管如何设置基础样式，其目的就一个：**确保我们有一个没有任何视觉样式的“裸”元素**。

  


另外，我们还可以定义一些 CSS 自定义属性（即变量），以便轻松创建范围滑块的不同变体：

  


```CSS
input[type="range"] {
    --w: 50vmin; /* 输入框宽度 */
    --color: coral; /* 滑块颜色 */
    --progress-color: color-mix(in srgb, var(--color), #000 var(--p,0%)); /* 进度条颜色 */
    --track-color: color-contrast(var(--color) vs #ffba76, #09faec, #bbb, #555,#fff to AA); /* 轨道颜色 */
    --track-size: round(.2em,1px);  /* 轨道高度 */
    --thumb-size: round(1.3em,1px); /* 滑块尺寸 */
    --radius: calc(infinity * 1px); /* 无穷大的圆角半径 */
    
    appearance: none;        /* 禁用浏览器默认样式 */
    background: transparent; /* 清除应用于 input 的默认背景 */
    cursor: pointer;         /* 设置光标为手型 */
    font-size: inherit;
    inline-size: var(--w);
    
    &:is(:hover,:focus-visible) {
        --p: 25%;
    }
    
    @supports not (color-contrast(red vs #fff, #000)) {
        --track-color:#555;
    }
}
```

  


上面的代码中，我应用了几个现代 CSS 特性：

  


-   CSS 的 `round()` 函数的作用就是**根据选定的舍入策略返回舍入数**。例如 `round(.2em, 1px)` ，`.2em` 会根据 `font-size` 进行计算，使用 `round()` 函数可以确保 `.2em` 的值会被四舍五入到最接近的 `1px` 的倍数，从而确保元素在不同分辨率下都能呈现最佳效果，比如避免在某些分辨率下出现模糊的情况
-   [CSS 的 `color-mix()` 函数](https://juejin.cn/book/7223230325122400288/section/7237288025221234744)的作用是将两个颜色在指定的颜色空间按指定的比率进行混合，有点类似调色。在这里，`--progress-color` 的值将是 `--color` 和 `#000` 的混合物，其中 `#000` 颜色的混合比率由 `--p` 决定
-   [CSS 的 `color-contrast()` 函数](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)的作用是根据 `--color` 从颜色列表 `#ffba76, #09faec, #bbb, #555,#fff` 中选择一个能达到 `AA` 等级对比度的颜色值
-   [`:is()` 选择器](https://juejin.cn/book/7223230325122400288/section/7226251495069450278)在这里的主要作用就是用来简化 CSS 代码
-   [CSS 的 `@supports` 规则](https://juejin.cn/book/7223230325122400288/section/7258822669154877480)是对不支持 `color-contrast()` 函数做了一个降级处理，例如在 Firefox 浏览器中，`--track-color` 的值始终是 `#555`

  


经过上面代码处理之后，整个范围滑块只有滑块（Thumb）可见，并且使用的是默认样式，如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dd9d5a41fe9047bb8352e29a8373c71e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1208&h=412&s=640214&e=gif&f=163&b=1e5b6d)

  


### 样式化轨道（Track）

  


我们可以像[给进度条（`<progress>`）设置样式](https://juejin.cn/book/7395119173656903715/section/7405199788926304308)那样来给范围滑块的轨道（Track）设置样式。接下来，任何带有 `-webkit` 前缀的伪元素将应用于 Chrome、Safari 和 Edge 浏览器，带有 `-moz` 前缀的伪元素将应用于 Firefox 浏览器：

  


```CSS
input[type="range"] {
    
    /* Chrome，Safari 和 Edge*/
    &::-webkit-slider-runnable-track {
        background: var(--track-color);
        block-size: var(--track-size);
        border-radius: var(--radius);
    }
    
    /* Firefox */
    &::-moz-range-track {
        background: var(--track-color);
        block-size: var(--track-size);
        border-radius: var(--radius);
    }
}
```

  


样式化轨道必需的属性是 `block-size` 和 `background` 。这个 `block-size 是一个逻辑属性`，当书写模式是 `ltr` 或 `rtl` 时，它相当于物理属性 `height` ，有关于这方面更详细的介绍，可以参阅 [CSS 的逻辑属性和逻辑值](https://juejin.cn/book/7223230325122400288/section/7254820664442355773)。

  


然而，[通常会看到应用 `border-radius` 来使边缘圆滑](https://juejin.cn/book/7199571709102391328/section/7199845563389444099#heading-3)。我使用 `calc(infinity * 1px)` 表达式给 `border-radius` 设置了一个无穷大的值，使得轨道高度 `--track-size` 如何变化，它都能获得一个类似胶囊的 UI 形状。

  


你可能已经注意到了，上面代码中的两个伪元素（`::-webkit-slider-runnable-track` 和 `::-moz-range-track`）选择器对应的 CSS 样式规则是完全一样的。你可能会有所疑惑，为什么没有将两个选择器合并在一起，或者使用 `:is()` 选择器来优化代码呢？这种想法是符合大家平时编写 CSS 代码的逻辑，但现实是不允许的，当你将它们合并在一起时，浏览器会认为它们是一段无效的 CSS 规则。就这一点而言，我曾在介绍 `:is()` 选择器的时候做过相应的解释，这里就不重复累述了，感兴趣的话，[可以点击这里了解相关信息](https://juejin.cn/book/7223230325122400288/section/7226251495069450278#heading-3)。

  


现在，你在浏览器中能看到范围滑块的轨道了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a5e4f4aa8d724ca1b59e0518ea26311e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1124&h=632&s=4969119&e=gif&f=258&b=1e5b6d)

  


你会发现，调整 `--color` 的值之后，范围滑块的轨道颜色也会随着调整，[这就是 CSS 的 `color-contrast()` 函数的神奇之处](https://juejin.cn/book/7223230325122400288/section/7239644346935410720)。

  


从上面的效果中，不难发现，范围滑块的轨道没有显示出当前进度的状态。接下来，我们就来看看如何给轨道上添加进度的颜色。

  


如果你足够仔细的话，你会发现，在 Firefox 浏览器中，它有一个 `div` 是专门用于处理范围滑块中的进度（Progress），并且暴露了一个伪元素 `::-moz-range-progress` 。这意味着，对于 Firefox 浏览器，可以通过该元素来设置范围滑块进度的样式：

  


```CSS
input[type="range"] {

    &:::-moz-range-progress {
        background: var(--progress-color);
        border-radius: var(--radius);
    }
}
```

  


现在，Firefox 浏览器呈现的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8043fb7fd3ac404ea4b510aefa452188~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1084&h=536&s=1810993&e=gif&f=238&b=1e5b6d)

  


遗憾的是，其他几个主流浏览器（例如 Chrome、Safari 和 Edge）并没有像 Firefox 浏览那样暴露一个可用于设置进度的伪元素。这意味着，我们需要采用其他的方式来处理。在这里，我将向大家演示一个，使用 CSS 线性渐变来设置范围滑块进度条的样式。

  


熟悉 CSS 渐变的开发者应该知道，使用 `linear-gradient` 可以很容易得到两个颜色的渐变，而且两个颜色之间有明显的分隔线。例如：

  


```CSS
.linear-gradient {
    background: linear-gradient(to right, red 0 16.777%, orange 0);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/dedaa846199f4e8189f3119d906cfdc3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1077&s=206986&e=jpg&b=246677)

  


需要知道的是，`linear-gradient` 中的 `red` 颜色的停止点 `16.7%` 它需要与 `<input type="range" />` 元素中的 `value` 匹配。因此，我将引入一个新的 CSS 自定义属性 `--progress` 来表示范围滑块的当前进度。它的值是需要计算才能获得的：

  


```
P = (value - min) ÷ (max - min) × 100%
```

  


注意，上面公式中的 `value` 、`min` 和 `max` 分别对应的是 `<input type="range" />` 元素中的 `value` 、`min` 和 `max` 属性的值。

  


这样一来，在 CSS 中，我们可以获得 `--progress` 的初始值：

  


```CSS
input[type="range"] {
    /* 其他 CSS 省略 */
    --progress: 16.777%; /* calc((20 - 0) / (120 - 0) * 1%) */
}
```

  


有了这个变量之后，我们就可以使用 `linear-gradient` 绘制的渐变当作范围滑块轨道的背景，它将包括进度和轨道两个部分，从视觉上可以明显告诉用户范围滑块当前所处的位置：

  


```CSS
input[type="range"] {
    --color: coral; /* 滑块颜色 */
    --progress-color: color-mix(in srgb,var(--color),#000 var(--p, 0%)); /* 进度条颜色 */
    --track-color: color-contrast(var(--color) vs #ffba76,#09faec,#bbb,#555,#fff to AA); /* 轨道颜色 */
    --progress: 16.777%; /* (value - min) ÷ (max - min) × 100% */
    --g: linear-gradient(to right, var(--progress-color) 0 var(--progress), var(--track-color) 0);
    
    /* 轨道样式： Chrome 、Safari 和 Edge */
    &::-webkit-slider-runnable-track {
        background: var(--g);
    }

    /* 轨道样式： Firefox */
    &::-moz-range-track {
        background: var(--g);
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5bc51c19ab4948afbc02f8774ae12e23~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1202&h=558&s=343804&e=gif&f=121&b=1e5b6d)

  


虽然效果有了，但有一个很明显的缺陷，那就是用户在拖动滑块的时候，当前进度没有随着变化。在这个情况下，我们需要给 `--progress` 设置一个动态的值。当范围滑块的 `value` 值变化时（用户在拖动滑块时，它的值会随着变化），自定义属性 `--progress` 的值也要随着变化。因此，我们需要借助 JavaScript 脚本来处理：

  


```JavaScript
const range = document.querySelector("input[type='range']");

range.addEventListener('input', etv => {
    const min = range.min;
    const max = range.max;
    const currentVal = range.value;
    const progress = ((currentVal - min) / (max - min)) * 100;
    range.style.setProperty(`--progress`, `${progress}%`);
})
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5616a6f19b004fc0ae5486638b0f5fd3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1102&h=638&s=1771586&e=gif&f=246&b=1e5b6d)

  


现在你所看到的轨道效果才是我们期望的一个效果。

  


除了调整 `linear-gradient` 渐变颜色的停止点之外，还有一种更简单的方式。通过 `background-size` 来改变渐背景图的大小，这种方式下，你可以使用任何你喜欢的渐变，它可以是单一的纯色，也可以复杂的纹理（[可以参考上一节所介绍的进度条](https://juejin.cn/book/7395119173656903715/section/7405199788926304308#heading-6)）。在这里，为了节约大家的时间，下面展示的是纯色，即 `--progress-color` ：

  


```CSS
input[type="range"]{
    /* 省略其他 CSS */
    --progress: 16.777%; /* (value - min) ÷ (max - min) × 100% */
    --g: linear-gradient(var(--progress-color), var(--progress-color)) 0 0 / var(--progress) 100% no-repeat;
    
    /* 轨道样式： Chrome 、Safari 和 Edge */
    &::-webkit-slider-runnable-track {
        background:var(--track-color) var(--g) ;
    }

    /* 轨道样式： Firefox */
    &::-moz-range-track {
        background:var(--track-color) var(--g);
    }
}
```

  


这样做，可以得到相同的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5b4d46c7eb74473abf841f7e1207ea33~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1096&h=642&s=2355927&e=gif&f=262&b=1e5b6d)

  


你可能会问，有没有办法不依赖 JavaScript 呢？回答是肯定的。稍后我会向大家展示，不依赖任何 JavaScript 脚本，实现相似效果的解决方案。

  


### 样式化滑块（Thumb）

  


样式化滑块（Thumb）相对来说要考虑的细节更多，因为浏览器在这部分的渲染上存在更多不一致：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e366d45fe4d9475883b292ebe74b6c4e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1077&s=242228&e=jpg&b=366677)

  


观察上图，很容易就能发现滑块在不同浏览器渲染的差异：

  


-   Webkit 内核的浏览器渲染的滑块（Thumb）并没有居中在轨道（Track）上，而 Firefox 不存在这个现象
-   Firefox 浏览器渲染的滑块（Thumb）带有灰色边框，并且不是一个圆形，而 Webkit 内核的浏览器则没有灰色的边框，并且视觉上呈现的是一个圆形
-   Webkit 内核的浏览器和 Firefox 浏览器渲染的滑块（Thumb）的背景颜色不同

  


庆幸的是，我们可以通过暴露出来的伪元素 `::-webkit-slider-thumb` 和 `::-moz-range-thumb` 来对滑块进行样式处理。

  


首先，使用 `appearance: none` 禁用浏览器给滑块设置的默认样式：

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    --thumb-size: round(1.3em, 1px); /* 滑块尺寸 */
    
    /* 样式化滑块: Chrome, Safari 和 Edge */
    &::-webkit-slider-thumb {
        appearance: none;
    }
    
    /* 样式化滑块: Firefox */
    &::-moz-range-thumb {
        appearance: none;
    }
}
```

  


然后，根据自己的需要给滑块添加基础样式，比如大小、颜色等：

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    --thumb-size: round(1.3em, 1px); /* 滑块尺寸 */
    
    /* 样式化滑块: Chrome, Safari 和 Edge */
    &::-webkit-slider-thumb {
        appearance: none;
        inline-size: var(--thumb-size);
        block-size: var(--thumb-size);
        border-radius: 50%;
        background-color: var(--progress-color);
    }
    
    /* 样式化滑块: Firefox */
    &::-moz-range-thumb {
        appearance: none;
        border: none 0;
        inline-size: var(--thumb-size);
        block-size: var(--thumb-size);
        border-radius: 50%;
        background-color: var(--progress-color);
    }
}
```

  


注意，`::-moz-range-thumb` 选择块中多了 `border: none 0` ，这是为了去除 Firefox 浏览器给滑块添加的边框。如果你不怎么纠结多一行或少一行代码的话，也可以在 `::-webkit-slider-thumb` 加上这个样式。如此一来，即使有一天，你需要带有边框的滑块时，也可以统一设置。

  


经过上面代码处理之后，范围滑块在 Firefox 浏览器的效果几乎达到预期：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f471427f2fae497b87883c48c1e30d50~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1150&h=578&s=2886108&e=gif&f=278&b=1e5b6d)

  


但在 Webkit 内核的浏览器（Chrome、Safari 和 Edge）中还有一个小瑕疵，那就是滑块没有位于轨道的中间：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/da1073c4f8e64cc6beb340ba1084974e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1144&h=620&s=2715834&e=gif&f=250&b=1e5b6d)

  


为了将滑块正确地居中在轨道上，我们可以使用以下公式并将其应用于 `margin-top` （或 `margin-block-start`）属性上：

  


```
margin-top = (轨道高度 - 滑块高度) ÷ 2
```

  


即：

  


```CSS
margin-top: calc((var(--track-size) - var(--thumb-size)) / 2)
```

  


注意，只有 Webkit 内核浏览器的滑块才有这个问题，因此我们只将这个样式添加到 `::-webkit-slider-thumb` 选择器的代码块中：

  


```CSS
input[type="range"] {
    --track-size: round(0.2em, 1px); /* 轨道高度 */
    --thumb-size: round(1.3em, 1px); /* 滑块尺寸 */
    --progress-color: color-mix(in srgb,var(--color),#000 var(--p, 0%)); /* 进度条颜色 */
    
    /* 省略其他 CSS */
    
    /* 样式化滑块: Chrome, Safari 和 Edge */
    &::-webkit-slider-thumb {
        appearance: none;
        inline-size: var(--thumb-size);
        block-size: var(--thumb-size);
        border-radius: 50%;
        background-color: var(--progress-color);
        margin-block-start: calc((var(--track-size) - var(--thumb-size)) / 2);
    }
    
    /* 样式化滑块: Firefox */
    &::-moz-range-thumb {
        appearance: none;
        border: none 0;
        inline-size: var(--thumb-size);
        block-size: var(--thumb-size);
        border-radius: 50%;
        background-color: var(--progress-color);
    }
}
```

  


这个时候，你在四在主流浏览器看到的范围滑块的效果都将一致：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5f18c1c494cf4b9ba3391108cad1147e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1256&h=676&s=2224927&e=gif&f=289&b=1e5b6d)

  


### 设置焦点环样式

  


由于范围滑块（`<input type="range" />`）是一个表单控件，它也是一个交互元素，那么给它设置焦点环样式是必要的，这样益于提高 Web 的可访问性。当应用焦点环样式时，它为用户提供了一个视觉指示，特别是对于那些使用键盘导航页面的用户来说尤其重要。

  


然而，默认情况之下，焦点环样式在浏览器中渲染也是有所差异的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d84c4e3ae5e54bc5a9a24ac5faf21962~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1077&s=332673&e=jpg&b=366677)

  


接下来，得使用 CSS 将焦点环样式在所有浏览器中保持一致。

  


[根据 WAI-ARIA 滑块文档的建议](https://w3c.github.io/aria-practices/#slider)：**焦点环应该设置在滑块（Thumb）上！** 既然如此，我们首先要做的就是移除默认的焦点环样式，以便用自定义样式覆盖它们。

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    
    &:focus:not(:focus-visible) {
        outline: none;
    }
    
    &:focus-visible {
        --p: 25%;
        outline: none;
    }
}
```

  


然后使用 `::-webkit-slider-thumb` 和 `::-moz-range-thumb` 伪元素，并将它们与 `:focus-visible` 伪类结合使用。然后我们可以使用 CSS 的 `outline` 和 `outline-offset` 属性来按我们想要的方式进行样式化。

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    
    &:focus:not(:focus-visible) {
        outline: none;
    }
    
    &:focus-visible {
        --p: 25%;
        outline: none;
    }
    
    &::-webkit-slider-thumb {
        border: none;
        outline: 3px solid var(--track-color);
        outline-offset: 2px;
    }

    &::-moz-range-thumb {
        border: none;
        outline: 3px solid var(--track-color);
        outline-offset: 2px;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/84ecd9606b3348a68dab1304d0e77a3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1242&h=616&s=3544034&e=gif&f=420&b=1e5b6d)

我想你现在已经知道如何使用现代 CSS 技术创建一个在所有主流浏览器有着一致 UI 外观的范围滑块。不过，上面所展示的方案，还是需要依赖一段简单的 JavaScript 脚本来控制范围滑块的进度效果。接下来，我将向大家展示一种不需要依赖任何 JavaScript 脚本，而且更简单的 CSS 实现的范围滑块。

  


首先，我们回到范围滑块拥有基础样式上：

  


```CSS
input[type="range"] {
    --w: 40vw; /* 输入框宽度 */
    --color: coral; /* 滑块颜色 */
    --progress-color: color-mix(in srgb,var(--color),#000 var(--p, 0%)); /* 进度条颜色 */
    --track-color: color-contrast(var(--color) vs #ffba76,#09faec,#bbb,#555,#fff to AA); /* 轨道颜色 */
    --track-size: round(0.25em, 1px); /* 轨道高度 */
    --thumb-size: round(1.3em, 1px); /* 滑块尺寸 */
    --radius: calc(infinity * 1px); /* 无穷大的圆角半径 */
    
    appearance: none; /* 禁用浏览器默认样式 */
    background: transparent; /* 清除应用于 input 的默认背景 */
    cursor: pointer; /* 设置光标为手型 */
    font-size: inherit;
    inline-size: var(--w);

    /* 移除默认的焦点环样式 */

        outline: none;
    }

    &:is(:hover, :focus-visible) {
        --p: 25%;
    }

    @supports not (color-contrast(red vs #fff, #000)) {

    }

```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6cefbdb479484fd79ff19047cf092c16~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1140&h=498&s=1797830&e=gif&f=185&b=1e5b6d)

  


你看到的范围滑块是没有轨道的。你可能会问，是不是继续使用之前所介绍的伪元素 `::-webkit-slider-runnable-track` 和 `::-moz-range-track` 来设置轨道样式呢？

  


不是的，接下来我将使用魔性的 `border-image` 来设置范围滑块的样式，而且它依旧在 `::-webkit-slider-thumb` 和 `::-moz-range-thumb` 伪元素上完成。换句话说，这个滑块（Thumb）在 UI 外观上具备多个功能，它将同时拥有范围滑块的所有 UI 外观，即滑块（Thumb）、轨道（Track）和进度（Progress）。它将涉及到 `border-image` 的使用：

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    
    /* Chrome, Edge, Safari */
    &::-webkit-slider-thumb {
        /* 省略其他 CSS */
        border-image: linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0) 0 1/calc(50% - var(--track-size)/2) 100vw/0 100vw; 
    }

    /* Firefox */
    &::-moz-range-thumb {
        /* 省略其他 CSS */
        border-image: linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0) 0 1/calc(50% - var(--track-size)/2) 100vw/0 100vw; 
    }
}
```

  


这一串长长的代码看起来是不是很吓人，但将它拆解一下，你会发现其实并不那么复杂：

  


```CSS
input[type="range"] {
    /* 省略其他 CSS */
    
    /* Chrome, Edge, Safari */
    &::-webkit-slider-thumb {
        /* 省略其他 CSS */
        border-image: linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0) 0 1/calc(50% - var(--track-size)/2) 100vw/0 100vw; 
    
        /* 等同于 */
        border-image-source:linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0);
        border-image-slice: 0 1;
        border-image-width: calc(50% - var(--track-size)/2) 100vw;
        border-image-outset: 0 100vw;
    }

    /* Firefox */
    &::-moz-range-thumb {
        /* 省略其他 CSS */
        border-image: linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0) 0 1/calc(50% - var(--track-size)/2) 100vw/0 100vw;
        
        /* 等同于 */
        border-image-source:linear-gradient(90deg,var(--progress-color) 50%,var(--track-color) 0);
        border-image-slice: 0 1;
        border-image-width: calc(50% - var(--track-size)/2) 100vw;
        border-image-outset: 0 100vw; 
    }
}
```

  


其实，[在创建“心形”形状时，就曾提到过 `border-image` 的用法](https://juejin.cn/book/7395119173656903715/section/7398183482347749428#heading-5)。通过 `border-image` 属性，我们可以将应用于边框的图片切割成一个九宫格，使其具有强大的响应式：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/833040f076774373b445760ce9bd7e40~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1280&h=1172&s=91513&e=jpg&b=fef7f5)

  


这个示例中，图像是一个具有两种颜色的线性渐变，即 `--progress-color` 和 `--track-color` 。我们需要边框图像覆盖范围滑块的整个水平空间（即进度加轨道），因此我们为左右边距使用了一个大的值 `100vw` ，同时保持上下边距为 `calc(50% - var(--track-size)/2)` 。

  


但 `border-image-width` 的大小受限于元素的尺寸。为了克服这个问题，我们还需要为 `border-image-outset` 使用一个大的值，以增加边框图像的可用空间。使用 `border-image-outset` 渲染在元素的边框框之外的边框图像部分不会触发溢出滚动条，也不会捕获鼠标事件。

  


这个时候你看到的范围滑块会像下图这样：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d414e6ecf1d5437fad9449b9cca8fa03~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1240&h=700&s=4733092&e=gif&f=370&b=1e5b6d)

  


神奇吧！现在一个滑块（Thumb）拥有了范围滑块所有 UI 部分，并且拖动滑块（Thumb），进度和轨道会随着改变，还不需要任何 JavaScript 脚本。

  


正如你所看到的，由于 `border-image-outset` 设置了较大的值，范围滑块的进度条向左无限延伸，同时轨道向右无限延伸，它们都溢出了 `input` 容器。现在，你只需要在 `input[type="range"]` 上添加 `overflow: hidden` ，这个效果就完美了：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ed0970ff7340428d85e011d3aaacf0d7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1174&h=684&s=3495243&e=gif&f=301&b=1e5b6d)

  


就这样！我们使用几行代码创建了一个自定义的范围滑块，你可以通过调整一些变量来轻松控制它。不过，这种方案有一个缺点是，如果你想给轨道两端设置圆角，那么实现起来就非常棘手了。如果你感兴趣的话，可以挑战一下，看看有什么办法可以实现这个效果。

  


最后，我们还可以为范围滑块添加一些微妙的动画效果，可以提升用户与范围滑块互动时的体验。

  


首先，将滑块的边框圆形过渡到实心圆形。实现这个效果，我们只需要在 `:active` 和 `:focus-visible` 状态下，将 `box-shadow` 中的 `--track-size` 更新为 `--thumb-size` ：

```CSS
    }

    /* Firefox */

        /* 省略其他 CSS */
        box-shadow: 0 0 0 var(--track-size) inset var(--progress-color);
        transition: .3s;
    }
}

input:active::-webkit-slider-thumb,
input:focus-visible::-webkit-slider-thumb {
    box-shadow: 0 0 0 var(--thumb-size) inset var(--progress-color);
}

input:focus-visible::-moz-range-thumb {

}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3501e04a03114ca2a8e3aaa6dce15f2e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1198&h=636&s=3275516&e=gif&f=266&b=1e5b6d)

  


请注意，代码中的 `transition` 非常重要，它使 `box-shadow` 效果从 `A` 状态（边框圆）平滑地过渡到 `B` 状态（实现圆）。你要是对这方面知识感兴趣的话，[可以点击这里获取更多的相关信息](https://juejin.cn/book/7288940354408022074/section/7292735608995184678)！

  


这里再告诉大家一个小技巧，我们可以新增一个 CSS 自定义属性（比如 `--_b`）来优化 `box-shadow` 属性的代码，这样在给 `box-shadow` 设置过渡动画效果时，不需要编写那么长的代码。你可以看作是 CSS 代码的一个优化小技巧吧，具体的代码如下：

```CSS
        box-shadow: 0 0 0 var(var(--_b), var(--track-size)) inset var(--progress-color);
        transition: .3s;
    }
    
    &:is(:active,:focus-visible) {

    }
}
```

## 创建带有提示框的范围滑块

  


你可能和我有着同样的感受，之前所演示的范围滑块，虽然它的 UI 样式在主流浏览器中得到了统一，而且使用现代 CSS 可以创建出各种酷炫的效果，但它们都缺少一个信息元素，那就是范围滑块当前位置对应的数值没有在 UI 上呈现。或者说，我们期望的范围滑块除了 UI 美观之外，还需要有相应的数值信息呈现，例如下图这种带有提示框的范围滑块：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ac86d24447f449a9bbaa7397be488799~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1200&h=680&s=136172&e=gif&f=72&b=da2f29)

  


直观上，你可能会认为要实现类似上图这样的范围滑块，需要依赖一些 JavaScript 脚本来获取 `input` 的值并更新提示框的位置和内容。至于动画效果，可能是某种 JavaScript 动画库来实现数字翻滚。实际上，我们完全可以利用一些[现代 CSS 特性](https://s.juejin.cn/ds/ihNcxguu/)来实现，这些现代特性包括 **[CSS 的锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)**、 **[视图过渡](https://juejin.cn/book/7288940354408022074/section/7308623298618163212)** 和 **[滚动驱动动画](https://juejin.cn/book/7288940354408022074/section/7307223031717724172)** 等。这些[现代 CSS 特性](https://s.juejin.cn/ds/ihNcxguu/)是近两年来最受欢迎和令人兴奋的 CSS 特性之一。它们开启了许多可能性，并随着浏览器支持的改善和开发者对这些特性的了解，它们将在 Web 开发过程中发挥着更大的作用。

  


接下来，我将通过实际案例来向大家演示这些特性创建的范围滑块是多么的酷炫，又是多么的令人兴奋！而且，在这个过程中，你会发现，代码并没有想象中的那么复杂！

  


首先，我将 HTML 结构调整成下面这样：

  


```HTML
<label class="price-slider">
    <span class="sr-only">Price Range</span>
    <input type="range" id="price" min="0" max="200" value="20" step="1" />
    <output for="price" style="--min: 0; --max:200;"></output>
</label>    
```

  


上面代码新增了一个语义化的 `<output>` 元素显示当前值，也就是我们需要的提示框，后面我们会在这个元素上使用很多新的 CSS 特性，使它具有酷炫的效果。

  


需要注意的是，我在 `<output>` 元素中定义了两个 CSS 自定义属性，其中 `--min` 对应的是 `<input>` 元素的 `min` 属性的值，`--max` 对应的是 `<input>` 元素的 `max` 属性的值。

  


接着，我们使用上一节的方法，完成范围滑块的 UI 定制：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a965082795a54ef7a71831630ac73509~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1164&h=480&s=304545&e=gif&f=71&b=1e5b6d)

  


接下来，我们需要将 `<output>` 元素制作成类似于一个提示框（`Tooltips`）。比如：

  


```CSS
@property --val {
    syntax: "<integer>";
    inherits: true;
    initial-value: 0;
}

.price-slider {
    --color: coral; /* 滑块颜色 */
    --progress-color: color-mix(in srgb,var(--color),#000 var(--p, 0%)); /* 进度条颜色 */
    --track-color: color-contrast(var(--color) vs #ffba76,#09faec,#bbb,#555,#fff to AA); /* 轨道颜色 */
    --track-size: round(0.25em, 1px); /* 轨道高度 */
    --thumb-size: round(1.3em, 1px); /* 滑块尺寸 */
    --radius: calc(infinity * 1px); /* 无穷大的圆角半径 */
    --fs: 24px; /* font-size */
    
    position: relative;
    font-size: var(--fs);
    display: grid;
    place-content: center;

    output {
        background: var(--progress-color);
      
        &::before {
            content: counter(num);
            counter-reset: num var(--val);
            position: absolute;
            font-weight: bold;
            display: grid;
            place-content: center;
            padding-block: 0.5em;
            padding-inline: 1lh;
            width: fit-content;
            color: var(--track-color);
            background: inherit;
        }
    }
}
```

  


注意，我将定义在 `input[type="range"]` 中定义的 CSS 变量移到了它的父容器 `label` 中，这样调整并不会影响范围滑块的效果。

  


上面代码最为关键的是，使用了 [CSS 计数器 `counter()`](https://juejin.cn/book/7223230325122400288/section/7259668400379527205) 将 `@property` 注册的 `--val` 作为 `output::before` 的内容。由于 `--val` 的初始值为 `0` ，因此你看到的 `output` 输出的值是 `0` ，还是不是 `input` 的 `value` 值。但不用担心，随着 CSS 代码的完善，你将看到它的值将与 `input` 的 `value` 值同步。

  


到目前为止，仅在 `output` 和它的伪元素 `::before` 应用了一些基础样式，它看上去还不太像一个提示框的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/17b4f221555d4793ad7272e055c1bdf1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1064&h=638&s=2721437&e=gif&f=212&b=1e5b6d)

  


在这里为了节省时间，我直接使用了 [@Temani Afif 制作的提示框生成器](https://css-generators.com/tooltip-speech-bubble/)中的代码来样式化 `output::before` 的样式，使其在 UI 外观上看上去更像一个 `Tooltips` ：

  


```CSS
.price-slider {
    /* 省略其他 CSS */
    
    --tooltips-tail: round(0.8em, 1px); /* 提示框尖角大小 */
    --tooltips-radius: round(0.8em, 1px); /* 提示框圆角 */
    
    output {
      background: var(--progress-color);
      
        &::before {
            /* 省略之前的代码 */
        
            /* Tooltips UI */
            --_m: 100% / var(--tooltips-tail) calc(var(--tooltips-tail) + 1px) no-repeat;
            --_g: 100%, #0000 99%, #000 102%;
        
            border-bottom: var(--tooltips-tail) solid #0000;
            border-radius: var(--tooltips-radius) / var(--tooltips-radius) var(--tooltips-radius) calc(var(--tooltips-radius) + var(--tooltips-tail)) calc(var(--tooltips-radius) + var(--tooltips-tail));
            mask: 
                linear-gradient(#000 0 0) padding-box,
                radial-gradient(100% 100% at 100% var(--_g)) calc(50% + var(--tooltips-tail) / 2) var(--_m),
```

### 使用锚点定位调整提示框的位置

  


我们要做的第一件事是将信息提示框（Tooltips）放在正确的位置上，即滑块（Thumb）的上方或下方。以往都是通过绝对定位，相对于整个范围滑块容器（`<label>`）进行定位。这样做并没有错，只是在定位的时候我们需要通过 JavaScript 来计算提示框距离容器左侧边缘或右侧边缘的距离。但是，我们现在有了更多的选择，可以由 [CSS 的锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)来完成。具体的代码如下：

  


```CSS
input[type="range"] {

    &::-webkit-slider-thumb {
        anchor-name: --thumb;
    }
    
    &::-moz-range-thumb {
        anchor-name: --thumb;
    }
}

.price-slider {
    /* 省略其他 CSS */
    output {
        /* 省略其他 CSS */
        &::before {
            position-anchor: --thumb;
            position: absolute;
            inset-area: top; /* 或 bottom */
        }
    }
}
```

  


就这么简单！使用几行 CSS 规则，信息提示框就位于滑块的上方，并且会跟随滑块左右移动：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/180ca840f2554d1f9f387ac01b2fb2e5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1144&h=640&s=2405759&e=gif&f=251&b=1e5b6d)

  


CSS 的锚点定位是 `position: absolute` 的一种升级。在这里，我们可以将元素相对于页面上的任何元素进行定位，称为“锚点”。我们可以使用 `anchor-name` 来定义锚点，并可以使用任意值，它的值类型有点类似于 CSS 的自定义属性，比如我们示例中的 `--thumb` 。稍后，可以在绝对定位的元素中使用相同的值，通过 `position-anchor` 将其与锚点关联起来。

  


在这个示例中，我们将滑块元素定义为“锚点”（例如 `--thumb`），然后在 `output` 的伪元素 `::before` 上设置 `position-anchor: --thumb` 。这样我们就将 `output::before` （信息提示框）和滑块元素（Thumb）关联起来了。

  


但只是绑定他们之间的关系是不够的，我们还需要正确的定位元素。为此，我们使用 `inset-area` 属性来对其定位。该属性使锚点定位的元素可以相对于关联的锚点元素的边缘进行定位，将定位元素放置在隐式的 `3×3` 网格中的一个或多个网格单元格上，其中锚点元素位于中心单元格。如下图所示：

### 动态化提示框内容

  


现在，提示框可以跟随范围滑块左右移动，并且其位置始终保持一致。但提示框的内容并不是我们期望的，我们期望的是它的内容能与 `input` 元素的 `value` 相匹配。即用户拖动滑块时，提示框的内容也能做出相应的响应。

  


接下来，我们另用一个现代 CSS 特性来解决这个问题，这个新特性就是 **[CSS 的滚动驱动动效](https://juejin.cn/book/7288940354408022074/section/7307223031717724172)**。你可能会有所困惑，`input` 只是一个范围滑块，并没有滚动条，那么我们怎么使用滚动驱动动效呢？

  


如果你对 CSS 滚动驱动动效有所了解的话，应该知道，它有一个叫做“**[视图进度时间轴](https://juejin.cn/book/7288940354408022074/section/7307223031717724172#heading-8)**”的概念。我们可以根据元素（称为主题）在滚动区域中的可见性来推进这个时间轴。滚动区域中主题的可见性会作为进度的百分比来追踪。默认情况下，时间轴在主题首次出现在滚动区域的一边时为 `0%` ，当主题到达对面边缘时为 `100%` 。

  


试想一下，它是不是很契合范围滑块，因为我们有一个在输入框（滚动区域）内移动的滑块（主题），所以我们实际上不需要其他的滚动内容。了解了这个概念之后，我们就可以着手编写 CSS 代码。

  


首先，我们在 `input[type="range"]` 上显式设置 `overflow` 为 `hidden` 或 `auto` ，使该元素变成一个滚动容器。在这里，我个人建议将 `overflow` 设置为 `hidden` ：

  


```CSS
input[type="range"] {
    overflow: hidden;
}
```

  


注意，在我们这个示例中，为了隐藏 `border-image` 溢出的轨道，我们早就在 `input[type="range"]` 元素上设置了 `overflow: hidden` 。

  


接着，需要在滑块元素上使用 `view-timeline` 给时间轴定义一个名称和轴，例如：

  


```CSS
input[type="range"] {
    overflow: hidden; 
    
    /* Chrome, Edge, Safari */
    &::-webkit-slider-thumb {
        view-timeline: --thumb-view inline;
    }
    
    /* Firefox */
    &::-moz-range-thumb {
        view-timeline: --thumb-view inline;
    }
}
```

  


这样，`input` 就是我们的滚动区域，而滑块就是主题。

  


需要知道的是，如果你忘记了在 `input` 元素上显式设置 `overflow` ，那么另一个元素会被用作滚动区域，但你可能不知道是哪一个，结果就是无法按预期工作。请记住，你需要使用 `view-timeline` 定义主题，并用 `overflow` 定义滚动区域。在这里，**再次强调一下，不要忘记在滚动区域元素上定义`overflow`** ！

  


只定义滚动区域和主题还不够，我们还需要定义动画。

  


```CSS
@property --val {
    syntax: '<integer>';
    inherits: true;
    initial-value: 0; 
}

@keyframes range {
    from {
        --val: var(--max);
    }
    to {
        --val: var(--min);
    }
}

.price-slider {
    timeline-scope: --thumb-view;
    
    output {
        animation: range linear both;
        animation-timeline: --thumb-view;
    }
}
```

  


简单的解释一下上面的代码。

  


我们从 `timeline-scope` 开始。在使用滚动驱动动效时，它与锚点定位类似，我们需要一个限制范围，这个范围仅限于定义它的元素（主题）及其后代，因此我们需要扩大范围以使其对其他元素可用。这就是为什么要在 `label` 元素上设置 `timeline-scope` 的主要原因。

  


接下来使用 `@keyframes` 定义一个动画，使 `--val` 在 `--min` 和 `--max` 之间变化，然后使用 `animation-timeline` 将该动画与之前定义的时间轴关联起来。

  


为什么 `--max` 在 `from` （或 `0%`）处，`--min` 在 `to` （或 `100%`）处呢？这不是反了吗？直观上，用户在拖动滑块应该是“从左到右”，但这看起来更像是“从右到左”。要理解这一点，我们需要考虑该功能的“滚动”部分。刚才已经提到过，在我们这个情景中，它是没有滚动，但我们可以通过以下示例来理解这一点：

  


```HTML
<div class="box"></div>
```

  


```CSS
.box {
    width: 250px;
    display: flex;
    height: 80px;
    overflow: auto;
    border: 2px solid;

    &::before {
        content:"";
        min-width: 300%;
        background: radial-gradient(circle closest-side,red 98%,#0000)
    }
}
```

  


这是一个很普通的示例。它在浏览器中呈现的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d1d433a991384ad287c7be2dc66e6d91~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1016&h=316&s=254891&e=gif&f=250&b=ffffff)

  


正如你所看到的，当你从“左向右”滚动容器时，你会看到一个红色圆圈从“右向左”移动。我们仍然有“从左向右”的行为，它但与滚动相关。至于内容，它会逻辑上向相反方向移动，即“从右向左”。

  


当滚动条在左边时，元素在右边；当滚动条在右边时，元素在左边。即使没有实际的滚动，这种逻辑也适用于我们的滑块。当滑块在右边缘时，这就是我们的 `0%` 状态，我们需要在那设置最大值（即 `--max`）。左边缘将是 `100%` 状态，即最小值 `--min` 。

  


这里还有一个关键，那就是 `@keyframes` 中的 `--val` 。需要知道的是，浏览器并不会对字符串进行插值计算。换句话说，[我们需要通过 `@property` 告诉浏览器](https://juejin.cn/book/7223230325122400288/section/7258870477462962236)，注册的 `--val` 是一个整数类型，而不是一个字符串，你可以在 `@keyframes` 中对其进行插值计算。

  


此时，范围滑块的效果如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ead5910e5baf4c6e99c33a5e189659fd~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1140&h=602&s=846486&e=gif&f=186&b=1e5b6d)

  


再次说一下，提示框中的内容是通过 CSS 计数器 `counter()` 自动计数生成的，并最终作为 `output::before` 的 `content` 输出。这个在之前就提到过了！

  


你可能已经发现了，虽然 `counter()` 生成了内容，但它与 `input` 中的 `min` 、`max` 和 `value` 都不符呀。以第一个 `input` 为例：

  


```HTML
<input type="range" id="price" min="0" max="200" value="20" step="1" />
```

  


它的 `min` 、`max` 和 `value` 分别是 `0` 、`200` 和 `20` ，但实际生成出来的呢，却不是这样，生成出来的 `min` 等于 `9` ，而 `max` 等于 `191` 。造成这种现象是与滚动部分相关的另一个技巧有关，下图将告诉你发生了什么：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0e5ecb48a4a24b3fa37fe62c3ee58eac~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=773&s=135815&e=jpg&b=fefefe)

  


滑块的移动被限制在 `input` 容器（滚动区域）内，但 `0%` 和 `100%` 状态被定义为在滚动区域之外。在我们这个情景中，滑块无法达到 `0%` 和 `100%` ，因此它不能超出容器。庆幸的是，我们可以使用 `animation-range` 属性将 `0%` 和 `100%` 两个状态都在容器内（滚动区域内）：

  


```CSS
.price-slider {
    timeline-scope: --thumb-view;
    
    output {
        animation: range linear both;
        animation-timeline: --thumb-view;
        animation-range: entry 100% exit 0%;
    }
```

### 添加动画效果

  


接下来，我们给提示框添加一些有趣的动画效果，例如晃动动画，使整个范围滑块更生动一些。而且制作这个动画效果很简单。具体的代码如下：

  


```CSS
@property --e {
    syntax: '<number>';
    inherits: true;
    initial-value: 0; 
}

output::before {
    --e: var(--val);
    transition: --e .1s ease-out;
    rotate: calc((var(--e) - var(--val))*2deg);
}
```

  


我们添加了一个新的 CSS 变量 `--e`，类型为 `number`。这个变量将等于 `--val` 变量。到现在为止，还没有什么特别的。我们有两个值相同的变量，但其中一个有过渡效果。这就是魔法的开始。

  


当你移动滑块时，动画将更新 `--val` 变量在 `output` 元素中。伪元素将继承这个值来更新内容，并且也会更新 `--e`。但由于我们对 `--e` 应用了过渡，它不会立即更新，而是平滑地更新（你知道过渡是如何工作的！）。这意味着在短暂的时间内，`--e` 和 `--val` 不会相等，因此它们之间的差异不是 `0`。我们将这个差异用于旋转效果！

对于最后一个案例，这里简单的解释一下其制作过程。首先，可以通过在 `input[type="range"]` 的滑块元素上使用滚动动画来创建一个纯 CSS 的平衡滑块：

  


```CSS
::-webkit-slider-thumb {
    view-timeline: --thumb inline;
}
```

  


滚动动画由范围滑块的滑块（Thumb）驱动，在 `min` 和 `max` 之间动画化一个数字：

  


```CSS
@property --value {
    inherits: true;
    initial-value: 0;
    syntax: '<integer>';
}

@keyframes sync { 
    to { 
        --value: 100; 
    }
}
```

  


然后将其与 `contain` 动画范围绑定：

  


```CSS
.control {
    animation: sync both linear reverse;
    animation-timeline: --thumb;
    animation-range: contain;
}
```

  


提升视图时间轴，以便控件的所有部分都可以使用它！

  


```CSS
.control { 
    timeline-scope: --thumb; 
}
```

  


在计数器中使用该值，该计数器用于标签。为每侧创建一个低值和高值：

  


```CSS
.control__label {
    counter-reset: low var(--value)  high calc(100 - var(--value));
}
.control__label::before {
    content: "COFFEE " counter(low) "%";
}
.control__label::after {
    content: counter(high) "% MILK";
}
```

  


这就是更新标签值的魔力 。这个过程与动态化提示框内容是相似的。

  


对于大的轨道，这是一个虚假的轨道。你可以利用相同的 `--value` 属性，并使用 `calc()` 来计算每部分的宽度。

  


```HTML
<div class="control__track">
    <div class="control__indicator"></div>
</div>
```

  


```CSS
.control__track::before {
    width: calc(var(--value) * 1% - 0.5rem);
    background: var(--coffee);
    border-radius: 4px;
    transition: width 0.1s;
}
```

  


宽度留下了一个小间隙，用于指示器。对于 `--coffee` 的颜色计算不是太复杂，但你可以再次使用相同的 `--value`：

  


```CSS
.control__track {
    --coffee: hsl(24 74% calc(24% + (30% * ((100 - var(--value, 0)) / 100)) / 1) / 0.4);
}
```

  


现在是最后一部分。让轨道改变高度。你可以设置另一个自定义属性，并使用 `--thumb` 时间轴来动画化它的值：

  


```CSS
@property --shift {
    initial-value: 0;
    inherits: true;
    syntax: '<integer>';
}

@keyframes shift {
    0%, 31%, 61%, 100% { 
        --shift: 0; 
    }
    32%, 60% { 
        --shift: 1; 
    }
}
```

  


然后使用 `--shift` 更新标签的位移和轨道的高度：

  


```CSS
.label {
    transform: translateY(calc(var(--shift) * 50%));
    transition: transform var(--speed) var(--timing);
}
```

  


酷的是，你可以使用控件来计算 `@keyframes` 的百分比。对于小弹跳，[这里使用了 CSS 的 `linear()` 函数](https://juejin.cn/book/7288940354408022074/section/7301665456824254515)：

  


```CSS
:root {
    --timing: linear(0, 0.5007 7.21%, 0.7803 12.29%,0.8883 14.93%, 0.9724 17.63%,1.0343 20.44%, 1.0754 23.44%,1.0898 25.22%, 1.0984 27.11%,1.1014 29.15%, 1.0989 31.4%,1.0854 35.23%, 1.0196 48.86%,1.0043 54.06%, 0.9956 59.6%,0.9925 68.11%, 1);
}
```

## 创建带有刻度的范围滑块

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5ed69b73dd8400aa0f4ee1cea744786~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=924&s=226794&e=jpg&b=ffffff)

  


有的时候，我们可能需要一个类似上图这样的带有刻度的范围滑块。接上来，通过一个简单的示例向大家演示一下，如何基于原生的 `<input type="range">` 元素和 `<datalist>` 元素构建一个带有刻度的范围滑块。

  


我们从 HTML 结构开始，你可能需要一段类似下面这样的 HTML:

  


```HTML
<div class="range--container">
    <input type="range" list="ticks" min="0" max="100" />

    <datalist id="ticks">
        <option value="0" label="0%"></option>
        <option value="25" label="25%"></option>
        <option value="50" label="50%"></option>
        <option value="75" label="75%"></option>
        <option value="100" label="100%"></option>
    </datalist>
</div>
```
