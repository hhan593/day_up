## 将图像切成若干块

  


我们先从将图像切成若干块开始！在 CSS 中，我们有多种方式可以将图像切成若干块。首先来看第一种方式，这种方式将结合 CSS Grid 布局、遮罩和变换等特性，将一张图片分割成若干块。

  


```HTML
<figure>
    <div></div>
    <!-- 省略其他 24 个div -->
</figure>
```

  


在这里，我们将一张图像分割成 `25` 个小正方形。先通过 CSS Grid 布局技术，将所有 `div` 元素叠加在一起，并且将同一张图片作为它们的背景图：

  


```CSS
figure {
    --image: url("https://picsum.photos/id/1015/800/800");

    display: grid;
    width: 60vmin;
    aspect-ratio: 1;
    overflow: hidden;
    background-image: var(--image);
    background-size: 0 0;

    > div {
        grid-area: 1 / 1 / -1 / -1;
        background-image: inherit;
        background-size: cover;
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9a722605b28440edb164928560856f24~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2206&h=1336&s=1868505&e=png&b=313131)

  


注意，我们这里使用了 CSS Grid 替代了之前的绝对定位，将所有 `div` 叠加在同一个网格区域中。我曾在《[CSS 布局：减少使用绝对定位的技巧](https://juejin.cn/book/7395119173656903715/section/7438887593464627235)》课程中详细阐述过这方面的技巧！

  


接下来，我们引入两个 CSS 变量：

  


```CSS
figure {
    --n: 5;
    --s: 1.1;
}
```

  


其中 `--n` 表示图片被分割的行数和列数，在这里我们将它的值设置为 `5` ，这意味着图像将分割成 `25` 块。`--s` 表示图像的缩放比例，它将帮助我们控制图像块之间的间距。

  


```CSS
figure {
    --n: 5;
    --s: 1.1;
    scale: var(--s);
    
    > div {
        scale: calc(1 / var(--s));
    }    
}
```

  


上面的代码将容器放大了 `--s` 倍，而每个 `div` 缩小了 ` 1  ``/ var(--s)` 倍。这个时候你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8330456f0c884e06afa22933e7ed91f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2206&h=1336&s=1896004&e=png&b=313131)

  


你现在看到的是 `25` 个 `div` 拼接而成的图像缩小了，但它们之间并没有产生任何间距。

  


这个时候我们引入另一个变量 `--p` ，通过它来给每个小块（`div`）设置变换的原点（`transform-origin`），确保缩放和其他变找是基于每个小块的指定位置，即为每个 `div` 元素计算位置坐标。为了能更好的简化 CSS，我们新增了 `--x` 和 `--y` 变量，用于计算 `--p` 的值：

  


```CSS
figure {
    /* CSS */
    > div {
        --x: 0;
        --y: 0;
        --p: 
            calc(var(--x) * 100% / (var(--n) - 1))
            calc(var(--y) * 100% / (var(--n) - 1));
            
        transform-origin: var(--p);    

        /* 每列 x 坐标 */
        &:nth-child(5n + 1) {
            --x: 0;
        }
        
        &:nth-child(5n + 2) {
            --x: 1;
        }
      
        &:nth-child(5n + 3) {
            --x: 2;
        }
        
        &:nth-child(5n + 4) {
            --x: 3;
        }
        
        &:nth-child(5n + 5) {
            --x: 4;
        }

        /* 每行 y 坐标 */
        &:nth-child(n + 1):nth-child(-n + 5) {
            --y: 0;
        }
        
        &:nth-child(n + 6):nth-child(-n + 10) {
            --y: 1;
        }
        
        &:nth-child(n + 11):nth-child(-n + 15) {
            --y: 2;
        }
        
        &:nth-child(n + 16):nth-child(-n + 20) {
            --y: 3;
        }
        
        &:nth-child(n + 21):nth-child(-n + 25) {
            --y: 4;
        }
    }
}
```

  


简单的解释一下上面的代码。这里的关键是：

  


-   `--x` 和 `--y` 是每个格子（`div`）的行列索引（从 `0` 开始）
-   `--p` 是最终位置百分比，即变换原点位置（`transform-origin`）

  


其中 `--p` 的计算方式是：

  


```CSS
--p: calc(x 坐标 * 100% / (n-1))  calc(y 坐标 * 100% / (n-1))
```

  


例如对于 `--x: 2`，`--y: 1`，结果是：

  


```CSS
--p: 50% 25%
```

  


紧接着我们使用 `nth-child()` 选择器的模式来匹配每个 `div` 的列索引号，即 `--x` 的值：

  


-   `5n+1` 匹配的是第 `1` 、`6` 、`11` 、`16` 和 `21` 个 `div` 元素，它们都位于第一列，即 `--x:0`
-   `5n + 2` 匹配的是第 `2` 、`7` 、`12` 、`17` 、和 `22` 个 `div` 元素，它们都位于第二列，即 `--x: 1`
-   以此类推

  


以同样的方式来设置每行的 `y` 值，比如 `:nth-child(n+1):nth-child(-n+5)` 将匹配第 `1 ~ 5` 个子元素，即第一行，对应的 `--y` 的值为 `0` 。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8355c0b9e09f447396472739e927704c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2206&h=1464&s=2190579&e=png&b=313131)

  


调整之后，所有 `div` 拼接的图像填充了整个父容器 `figure` （与 `figure` 放大之后的效果相同）。但依旧看不到图像块。我们可以通过 CSS 的遮罩来实现。

  


```CSS
figure {
    > div {
        mask: linear-gradient(#fff, #fff) no-repeat;
        mask-size: calc(100% / var(--n) + 1px) calc(100% / var(--n) + 1px);
        mask-position: var(--p);
    }
}
```

  


上面的渐变遮罩会使每个小块的显示区域更为精确，其中：

  


-   `linear-gradient(#fff, #fff)` 表示遮罩图像，配合 `mask-size` 和 `mask-position` 达到裁剪效果
-   `mask-size: calc(100% / var(--n) + 1px)`：控制每个小块的大小，使得每个块的尺寸为容器的 `1/5` 加上 `1px` 的间距
-   `mask-position: var(--p)`：每个块的遮罩位置由 `--p` 控制，它的值与 `transform-origin` 的值相同

  


这个时候，你看到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d8df42e9a57e436c8816508dcc009795~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2206&h=1464&s=1976745&e=png&b=323232)

  


> Demo 地址：https://codepen.io/airen/full/raaOxYv

  


图像被分割成了 `25` 个小块。

  


在这个基础上，我们可以根据 `div` 的总数和每个 `div` 在 DOM 中的索引值来[设置 `transition-duration` 和 `transition-delay` 的时间](https://juejin.cn/book/7288940354408022074/section/7304843997364060214)，使得图像在视觉上有一个错落的过渡动画效果：

  


```CSS
figure {
    > div {
        /*       
        --stagger: calc(var(--i) * 0.1s);
        --duration: calc(pow(var(--n), 2) * 0.1s); */
        
        --stagger: calc((sibling-index() - 1) * 0.1s);
        --duration: calc((sibling-count() - 1) * 0.01s);
        --ease: cubic-bezier(0.25, 0, 0.3, 1);
        
        transition: opacity var(--duration) var(--ease) var(--stagger);
        
        @starting-style {
            opacity: 1;
        }
    }
    
    &:hover > div {
        opacity: 0;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b6b0ac0150694b01b740d740a5c48a81~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1100&h=726&s=4573521&e=gif&f=103&b=2b2b2b)

  


> Demo 地址：https://codepen.io/airen/full/myyeRbX

  


温馨提示，在这里我使用了 CSS 的两个新函数 `sibling-count()` 和 `sibling-index()` 函数来获取 `div` 元元素的总数量和它们在 DOM 中的索引值，从而计算出 `transition-duration` 和 `transition-delay` 。这两个函数替代了之前 CSS 自定义属性的计算。有关于这方面更详细的介绍，请移步阅读《[Web UI：CSS 树计数函数：`sibling-count()` 和 `sibling-index()`](https://juejin.cn/book/7395119173656903715/section/7487008847370928167)》！

  


在这个基础上还可以创建出更多的动画效果，稍后会介绍！

  


通过这种方法，我们只需要调整 `--n` 和 `div` 的数量，就将图像分割成 `pow(var(--n), 2)` 个方块：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b25056aeb1b44a46b5f53bfe5e7f1acc~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=2370&s=5223918&e=png&b=323232)

  


> Demo 地址：https://codepen.io/airen/full/NPPGvYd

  


虽然上面这种方法可以将图像切成若干块，但它需要使用额外的 `div` 元素。我们的目的是在一 `<img>` 的基础上仅使用 CSS 将图像切成若干块。

  


在 CSS 中，我们可以借助 **[CSS 的锥形渐变](https://juejin.cn/book/7223230325122400288/section/7259668771856941111)**（`conic-gradient()`）来创建遮罩图（`mask-image`），结合 CSS 变量来控制网格的行数、列数和间距，从而将图像切分成若干块。具体的 CSS 代码如下：

  


```CSS
.figure {
    --n: 9;      /* 控制行数 */
    --m: 9;      /* 控制列数 */
    --gap: 4px;  /* 控制间距 */
    mask: 
        conic-gradient(
            from 90deg at var(--gap) var(--gap),
            #000 90deg,
            #0000 0
        )
        calc(-1 * var(--gap)) calc(-1 * var(--gap)) /
        calc((100% + var(--gap)) / var(--m)) calc((100% + var(--gap)) / var(--n));
}
```

  


简单的解释一下上面的代码。

  


在这里，我们引入了三个 CSS 变量：

  


-   `--n`：控制网格的行数，这里设置了 `9`
-   `--m`：控制网格的列数，同样设置了 `9`
-   `--gap` ：控制单元格之间的间距，这里设置了 `4px`

  


这些 CSS 变量主要用于控制网格的大小和间距等。这意味着，这个示例会将图像切成 `9 x 9` 个小块（类似于一个 `9 x 9` 的网格）。

  


需要知道的是，这些变量后续会在 `mask` 属性中引用，真正起到切割作用的是它。我们利用 `conic-gradient()` 渐变，绘制出所需要的遮罩图形。为了便于大家理解，我们将 `conic-gradient()` 应用于 `background` ：

  


```CSS
.figure {
    background: conic-gradient(from 90deg at var(--gap) var(--gap),#000 90deg,#0000 0);
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c699bad9f0d14a8fbec204ec46b85826~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=1184&s=170727&e=jpg&b=ffffff)

  


紧接着，我们设置渐变的起始位置，即 `background-position` 的值，它向左和向上偏移 `--gap` 位置：

  


```CSS
.figure {
    background:conic-gradient(from 90deg at var(--gap) var(--gap),#000 90deg,#0000 0);
    background-repeat: no-repeat;
    background-position: calc(-1 * var(--gap)) calc(-1 * var(--gap));
}
```

  


得到的效果如下：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/94b8fa051cfd464a9a4437d4f76fb478~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2858&h=1808&s=111852&e=png&b=ffffff)

  


最后，通过 `background-size` 来调整渐变图形的大小，它由 `--m` 和 `--n` 两个变量来控制：

  


```CSS
.figure {
    background-size: 
        calc((100% + var(--gap))/var(--m))
        calc((100% + var(--gap))/var(--n));
}
```

  


-   `calc((100% + var(--gap))/var(--m))`: 水平方向上的大小。`100%` 表示元素（图像）的宽度，`+ var(--gap)` 是为了将间距考虑在内。然后除以列数 `--m` ，计算出每个单元格的宽度
-   `calc((100% + var(--gap))/var(--n))`: 垂直方向上的大小。类似地，`100%` 表示元素（图像）的高度，`+ var(--gap)` 是为了加入间距，再除以行数 `--n` ，计算出每个单元格的高度

  


然后将它沿着 `x` 和 `y` 轴平铺：

  


```CSS
.figure {
    --m: 9;
    --n: 9;
    --gap: 4px;
    background:conic-gradient(from 90deg at var(--gap) var(--gap),#000 90deg,#0000 0);
    background-position: calc(-1 * var(--gap)) calc(-1 * var(--gap));
    background-size: 
        calc((100% + var(--gap))/var(--m))
        calc((100% + var(--gap))/var(--n));
}
```

  


如此一来，我们就获得下面这样的一个遮罩图：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/316da93604c84efe8abaec816a31a062~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2858&h=1808&s=137640&e=png&b=ffffff)

  


我们将上面代码中的 `background` 替换成 `mask` ，并应用到 `img` 元素上，那么黑色区域遮罩的地方将可见，透明区域遮罩的地方将不可见。最后在视觉上就会将一张图像切分成 `9 x 9` 的小方块：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/85cb9c15f66f4d60b36fd7faf42bd8f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1296&s=827036&e=png&b=313131)

  


> Demo 地址：https://codepen.io/airen/full/qEEOPLN

  


正如你所看到的，通过使用 `conic-gradient` 作为遮罩的方式，结合 `calc()` 动态计算每个网格单元的大小，从而为图像应用一个带有间距的网格遮罩效果。通过调整 `--n`、`--m` 和 `--gap` 变量，你可以灵活地控制网格的行列数和单元格之间的间距，进而实现不同的视觉效果。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0824b68663d54d54b315926a8557971b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2286&h=1628&s=3508992&e=png&b=323131)

  


> Demo 地址：https://codepen.io/airen/full/EaaVbyg

## 使用 CSS 遮罩为图像添加动画效果

  


通过前面的学习，我们已经掌握了如何将图片切成若干块，类似于给图片添加了碎片效果，但如何给碎片（切分后的小块）添加动画效果？我们先暂时放一放，我们先来看看如何通过 CSS 遮罩给图像添加动画效果。

  


### 图像圆形消失

  


先从简单的示例开始，鼠标悬浮在图像上时，从图像中心以一个圆形的方式擦除图像，最终使图像不可见；当鼠标移出图像区域时，圆形逐渐消失在图像中心，从而使图像非得到可见状态。如下图所示：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/efc393c044b24f6bb269b7531b4ce92e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1054&h=556&s=958957&e=gif&f=80&b=2b2b2b)

  


第一步是给图像添加[渐变边框](https://juejin.cn/book/7395119173656903715/section/7413371655377911835)。只需在 `<img>` 元素上设置重复线性渐变（`repeating-linear-gradient()`），并给它添加一点内边距，就可以让渐变背景显现出来：

  


```CSS
.figure {
    --p: 1rem;
    padding: var(--p);
    background: repeating-linear-gradient(
        -45deg,
        #e02f1773 0 var(--p),
        #ff99005c 0 calc(var(--p) * 2)
    );
}
```

  


这样我们就获得了带有渐变边框的图像。我们的目标是鼠标悬浮在图像上时隐藏图像，但它的边框不需要隐藏。这时，CSS 遮罩（`mask`）就可以派上用场了。

  


CSS 遮罩（`mask`）并不复杂，但在这个上下文中稍微有点棘手。这里的技巧是将两个遮罩层链在一起，并明确地说明遮罩的应用位置：

  


```CSS
.figure {
    mask: 
        radial-gradient(#000 70%, #0000 71%) content-box 50% / var(--_s, 0% 0%) no-repeat,
        linear-gradient(#000 0 0);
}
```

  


现在我们有了两个遮罩图：

  


-   圆形遮罩图（`radial-gradient()`）仅限于图像的内容框区域（`content-box`）
-   矩形遮罩图（`linear-gradient()`）将会覆盖整个图像区域，包括内边距区域（即 `padding-box`）

  


> 温馨提示，任何一个元素都有多个不同的框模型，我们给元素应用背景图像或遮罩图像时，可以根据框模式来覆盖不同的区域。有关于这方面更详细的介绍，请移步阅读《[CSS 布局：CSS 中的各种 `*-box`](https://juejin.cn/book/7395119173656903715/section/7445257036046991410)》！

  


我们需要两个遮罩图层，因为这样我们可以使用 CSS 的 `mask-composite` 属性将它们结合起来。`mask-composite` 提供了多种不同的方式来组合遮罩层，其中一种是“排除”（`exclude`）两个遮罩层重叠的区域：

  


```CSS
.figure {
    mask: 
        radial-gradient(#000 70%, #0000 71%) content-box 50% / var(--_s, 0% 0%) no-repeat,
        linear-gradient(#000 0 0);
    mask-composite: exclude;    
}
```

  


为了便于理解，我们调整一下上面代码中的 `radial-gradient` 渐变：

  


```CSS
.figure {
    mask: 
        radial-gradient(#000 70%, #0000 71%) content-box no-repeat,
        linear-gradient(#000 0 0);
    mask-composite: exclude;    
}
```

  


我们移除了 `mask-position` 和 `mask-size` ，现在图像中心有一个圆心镂空：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0d8bf674dae04f038d84bd08570b103d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2188&h=1116&s=287386&e=png&b=323232)

  


让我们调整一下切口的大小和位置（图像中间的孔洞），看看会发生什么？比例，我们把它的位置定义在元素的中心（`mask-position: center`），并反大小减半（`mask-size: 50% 50%`）：

  


```CSS
.figure {
    mask: 
        radial-gradient(#000 70%, #0000 71%) content-box center / 50% 50% no-repeat,
        linear-gradient(#000 0 0);
    mask-composite: exclude;    
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c4281f71a2e44d2dbad1389d2a808439~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2188&h=1116&s=621924&e=png&b=313131)

  


我想通过这个过程，你已经看出这个效果的方向了。我们通过调整径向渐变的大小来隐藏图像（增加 `mask-size` 的值）或显式图像（减小 `mask-size` 的值）。为了完全隐藏图像，我们需要将 `mask-size` 的值调到足够大，使得圆形完全覆盖图像。这意味着我们需要一个大于 `100%` 的值，要是你想获得一个更精确的值，可以通过数学函数来计算；要是你不需要一个精确的值，你可以用一个大致的数值，比如 `150%` 或 `200%` 。

  


在这个示例中，我们期望的效果是：鼠标悬浮在图像上时，图像消失，鼠标移出图像时，图像显现。这意味着，初始情况下，`mask-size` 的值要足够小，小到圆形遮罩图不可见，我们将它设置为 `0% 0%` ，然后在悬浮状态（`:hover`）将其值设置为一个较大值，比如 `200% 200%` ：

  


```CSS
.figure {
    --p: 1rem;
    padding: var(--p);
    background: repeating-linear-gradient(
        -45deg,
        #e02f1773 0 var(--p),
        #ff99005c 0 calc(var(--p) * 2)
    );
    mask: 
        radial-gradient(#000 70%, #0000 71%) content-box center / var(--s, 0% 0%) no-repeat,
        linear-gradient(#000 0 0);
    mask-composite: exclude;
    transition: 0.5s;
    cursor: pointer;
    
    &:hover {
        --s: 200% 200%;
    }
}
```

  


为了图像显现和隐藏之间有一个平滑过渡，我们应用了一个 `.5s` 的过渡效果（`transition: .5s`）。这样就获得了我们所期望的效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7b4b6ec304a1424e875ca66a26e0a745~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1054&h=556&s=958957&e=gif&f=80&b=2b2b2b)

  


> Demo 地址：https://codepen.io/airen/full/xbbwpwj

  


我们只用了几行 CSS 就创建了图形消失的动画效果，而且没有额外的 HTML 标记，甚至是没有使用任何伪元素。更为神奇的是，我们只需要调整径向渐变的位置，就可以获得相同效果的变体：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbd83f40781443b89f31e2b480688003~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=486&s=3964043&e=gif&f=92&b=2a2a2a)

### 图像对角线消失

  


我们来提高一下难度，鼠标悬浮到图像上时，图像对角线消失：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/31a1f27194bb4148b65f74fa5f699b9a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=640&s=1796818&e=gif&f=109&b=2b2b2b)

  


看上去，图像从左上角和右下角同时向中间镂空，最终使图像消失。

  


让我们一步一步地来，首先从一个更简单的效果开始，我们使用之前示例相同的模式来创建遮罩效，不同之处在于，我们将径向渐变换成了线性渐变：

  


```CSS
.figure {
    mask: 
        linear-gradient(#000 0 0),
        linear-gradient(135deg, #000 50%, #0000 0) content-box 100% 100% / 200% 200% no-repeat;
    mask-composite: exclude;
  
    &:hover {
        mask-position: 0% 0%;
    }
}
```

  


你会注意到，这次悬浮调整的是 `mask-position` 的位置，而不是它的大小，从 `100% 100%` （右下角）到 `0% 0%` （左上角），从而创建了一个滑动效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e579158a83604e53bce13a8dee9dc2eb~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1228&h=604&s=870389&e=gif&f=104&b=2c2c2c)

  


我们可以通过将线性渐变的角度从 `135deg` 改为 `-45deg` ，然后遮罩图位置（`mask-position`）从 `0% 0%` 过渡到 `100% 100%` ：

  


```CSS
.figure {
    mask: 
        linear-gradient(#000 0 0),
        linear-gradient(-45deg, #000 50%, #0000 0) content-box 0% 0% / 200% 200% no-repeat;
    mask-composite: exclude;

    &:hover {
        mask-position: 100% 100%;
    }
}
```

  


这样就能改变滑动效果的方向。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bfc596ea6bc74cfe9ad65b5df4409595~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1276&h=560&s=3141312&e=gif&f=105&b=2c2c2c)

  


你可能已经注意到了，我们在悬浮状态（`:hover`）只给 `mask-position` 定义了一个值，但我们有两个渐变。你可能会问，它是如何工作的，`mask-position` 应用于第一个渐变，但由于渐变会占据整个应用区域，因此不能使用百分比值来移动它。这意味着，为了安全我们可以为两个渐变定义一个值，但它只会影响第二个渐变。这个工作原理与 `background-position` 是相似的。

  


接下来，我们尝试将两个效果结合起来：

  


```CSS
.figure {
    mask: 
        linear-gradient(#000 0 0),
        linear-gradient(135deg, #000 50%, #0000 0) content-box 100% 100% / 200% 200% no-repeat,
        linear-gradient(-45deg, #000 50%, #0000 0) content-box 0% 0% / 200% 200% no-repeat;
    mask-composite: exclude;
  
    &:hover {
        mask-position: 50% 50%;
    }
}
```

  


这次，两个渐变位置分别从右下角（`100% 100%`）和左上角（`0% 0%`）过渡到元素中心（`50% 50%`）。第一个渐变隐藏图像的左上部分，而第二个渐变隐藏右下部分。鼠标悬停时，两个渐变朝相同的方向滑动，隐藏完整的图像：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8b0c3196967c40389f8c08ce27193fd4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1234&h=602&s=1334596&e=gif&f=110&b=2c2c2c)

  


这种方法基本有效，但在某些情景下，由于渐变的情质和抗锯齿问题，出现了一条奇怪的对角线。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/01c7122e8a194ebfa5ef0ac3cd6ff6c7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1264&h=516&s=749724&e=gif&f=72&b=2c2c2c)

  


通常情况下，我们可以通过稍微增加百分比到 `50.5%` 来尝试修复这个问题，而不是使用 `50%`：

  


```CSS
.figure {
    mask: 
        linear-gradient(#000 0 0),
        linear-gradient(135deg, #000 50.5%, #0000 0) content-box 100% 100% / 200% 200% no-repeat,
        linear-gradient(-45deg, #000 50.5%, #0000 0) content-box 0% 0% / 200% 200%  no-repeat;
    mask-composite: exclude;

    &:hover {
        mask-position: 50% 50%;
    }
}
```

  


然而，这样反而更糟糕。你可能会想，是否应该减少百分比而不是增加它。试试看，结果是一样的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aafe03e8dc674265ae078a0cd82e0bd2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1218&h=496&s=798378&e=gif&f=82&b=2c2c2c)

  


解决方案是更新 `mask-composite` 属性。如果你还记得，我们已经使用了 `exclude` 值。我们需要做的是，除了单独声明 `exclude` 值之外，还要应用 `add` 值，以确保底动（滑动渐变）不会彼此被排除，而是加在一起：

  


```CSS
.figure {
    mask: 
        /* 第一层 */
        linear-gradient(#000 0 0),
        /* 第二层 */
        linear-gradient(135deg, #000 50%, #0000 0) content-box 100% 100% / 200% 200% no-repeat,
        /* 第三层 */
        linear-gradient(-45deg, #000 50%, #0000 0) content-box 0% 0% / 200% 200% no-repeat;
    mask-composite: exclude, add;
  
    &:hover {
        mask-position: 50% 50%;
    }
}
```

  


现在，第二层和第三层使用 `add` 组合，创建一个中间层，这个层将被从第一层中排除。换句话说，我们必须从第一层中排除所有其他层。

  


这修复了我们悬停效果中的那条线问题：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/85653eaf86c84f02b1d670ebf6e28201~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=640&s=1796818&e=gif&f=109&b=2b2b2b)

  


> Demo 地址：https://codepen.io/airen/full/gbbPMMe

  


我们调整一下渐变的角度和位置，就可以获得不同方向的过渡效果：

  


```CSS
.figure {
    --p: 1rem;
    padding: var(--p);
    background: repeating-linear-gradient(
        -45deg,
        #e02f1773 0 var(--p),
        #ff99005c 0 calc(var(--p) * 2)
    );
    mask: 
        linear-gradient(#000 0 0),
        linear-gradient(calc(135deg + var(--a, 0deg)), #000 50.1%, #0000 0) content-box 100% 100% /200% 200% no-repeat,
        linear-gradient(calc(-45deg + var(--a, 0deg)), #000 50.1%, #0000 0) content-box 0% 0% /200% 200% no-repeat;
    mask-composite: exclude, add;
    transition: 0.5s;
    cursor: pointer;
    
    &:nth-child(2) {
        --a: 90deg;
        mask-position: 100% 0%,0% 100%;/* 等同于 100% 0%,0% 100%, 100% 0% */
        
    }
    
    &:hover {
        mask-position: 50% 50%;
    }
}
```

  


请注意，第二个 `img` 中的 `mask-position` 设置了两个值（`mask-position: 100% 0%,0% 100%`），但我们有三个渐变。第一个值 `100% 0%` 应用于第一个渐变层，第二个值 `0% 100%` 应用于第二个渐变层。与此同时，第三个渐变层使用第一个值 `100% 0%` 。当 `mask-position` 的声明值少于 `mask` 层的数量时，逗号分隔的值会重复，直到所有的 `mask` 层都被覆盖。

## 创建图像碎片化效果

  


之前我们所展示的图像动画效果基本上是动画化 `mask-position` 或者 `mask-size` 等属性。其实，我们还可以给遮罩的渐变中的透明度值添加动画，从而实现一个透明度的渐变动画，甚至还可以让这些动画错落执行，打造出图像碎片化的效果。

  


需要知道的是，在 CSS 中，我们过去无法直接对渐变进行动画处理。不过，随着 `@property` 的引入，这才成为可能。我曾在《[Web UI：你需要的是 `@property`](https://juejin.cn/book/7395119173656903715/section/7446278656840138762)》中深入的阐述了 `@property` 所带来的强大动画能力，展示了它是如何让渐变值实现平滑过渡的！

  


简而言之，`@property` 允许我们创建自定义的 CSS 变量，并且可以给它们指定类型。比如：

  


```CSS
@property --c-0 {
    syntax: "<number>";
    initial-value: 1;
    inherits: false;
}

@property --c-1 {
    syntax: "<number>";
    initial-value: 1;
    inherits: false;
}
```

  


上面代码创建了两个变量： `--c-0` 和 `--c-1` ，它们都是数值类型（`<number>`），初始值为 `1` ，且子元素不可继承！

  


我们可以将这两个属性应用于 CSS 遮罩中的透明度（Alpha）值。例如：

  


```CSS
.figure {
    --g: / 50% 100% content-box  no-repeat;
    mask: 
        linear-gradient(rgb(0 0 0 / var(--c-0)) 0 0) left var(--g), 
        linear-gradient(rgb(0 0 0 / var(--c-1)) 0 0) right var(--g);
    transition: --c-0 .5s, --c-1 .3s .4s;
    
    &:hover {
        --c-0: 0;
        --c-1: 0;
    }
}
```

  


由于`--c-0` 和 `--c-1` 的默认值是 `1` （完全不透明），所以图像会完全显示出来。我们在悬浮状态将它们的值调整为 `0` （完全透明），使图像完全不可见。注意，在这里，我们为每个变量设置了不同的动画持续时间和延迟时间。因此，当你将鼠标悬浮在图像上时，第一个遮罩的渐变将首先淡出，接着第二个遮罩也会淡出。这样我们就实现了一个按顺序消失的遮罩动画效果，为图像创造了分层碎片感的视觉体验：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/43262e6d388d4007bef8f04432890b58~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1340&h=598&s=2846580&e=gif&f=93&b=2b2b2b)

  


> Demo 地址：https://codepen.io/airen/full/dPPGzvg

  


到目前为止，我们只用了两个线性渐变和两个自定义属性来实现遮罩。但如果我们想要做出瓷砖铺排式的碎片化效果，就需要更多的“瓷砖”，这意味着我们要用到更多的渐变层和更多的自定义属性。例如，之前的网格状图块排列方式，我们需要为每个图块定义一个 `@property`。

  


为了提高效率，我们在这里借助 SCSS 处理器来简化编码的过程。我们首先定义两个 SCSS 变量 `$x` 和 `$y` ，分别代表网格的列数和行数：

  


```SCSS
$x: 20; /*  行数 */
$y: 20; /*  列数 */
```

  


然后利用 SCSS 的 `@for` 循环为每个图块定义一个 `@property` ：

  


```SCSS
$x: 20; /*  行数 */
$y: 20; /*  列数 */

@for $i from 0 through ($x - 1) {
    @for $j from 0 through ($y - 1) {
        @property --c-#{$i}-#{$j} {
            syntax: "<number>";
            initial-value: 1;
            inherits: false;
        }
    }
}
```

  


紧接着在 `:hover` 状态让所有自定义属性的值变为 `0` ，从而使图片消失：

  


```SCSS
.figure:hover {
    @for $i from 0 through ($x - 1) {
        @for $j from 0 through ($y - 1) {
            --c-#{$i}-#{$j}: 0;
        }
    }
}
```

  


然后我们通过 SCSS 的 `@mixin` 创建一个 `image()` 的混合宏，用来生成所有的渐变遮罩和过渡属性：

  


```SCSS
@mixin image() {
    $all_t: (); 
    $all_m: (); 
    @for $i from 0 through ($x - 1) {
        @for $j from 0 through ($y - 1) {
            $all_t: append($all_t, --c-#{$i}-#{$j} transition($i,$j), comma);
            $all_m: append($all_m, linear-gradient(rgba(0,0,0,var(--c-#{$i}-#{$j})) 0 0) calc(#{$i}*100%/(#{$x} - 1)) calc(#{$j}*100%/(#{$y} - 1)), comma);
        }
    }
    transition: $all_t;
    mask: $all_m;
}
```

  


现在只需要在 `img` 上引用 `image()` 混合宏：

  


```SCSS
.figure {
    @include image();
}
```

  


由于我们的遮罩图尺寸都是一样的，所以可以用一个 `mask-size` 属性统一设置，每个图块的宽高由 `$x` 和 `$y` 决定：

  


```SCSS
.figure {
    @include image();
    mask-size: calc(100%/#{$x}) calc(100%/#{$y});
    mask-repeat: no-repeat;
}
```

  


你可能已经注意到了，我们在定义 `image()` 混合宏时，有下面这行代码：

  


```SCSS
$all_t: append($all_t, --c-#{$i}-#{$j} transition($i,$j), comma);
```

  


它的意思是，我们在同一个混合宏中同时生成了所有自定义属性的 `transition` 配置。

  


最后，我们通过 SCSS 的 `random()` 函数为每个图块生成一个不同的动画持续时间和延迟时间：

  


```SCSS
@function transition($i, $j) {
    @return  ($s*random())+s (($s*($i+$j))/($x+$y))+s;
}
```

  


完整的 SCSS 代码如下：

  


```SCSS
$x: 20; 
$y: 20; 
$s: 0.8;

@for $i from 0 through ($x - 1) {
    @for $j from 0 through ($y - 1) {
        @property --c-#{$i}-#{$j} {
            syntax: "<number>";
            initial-value: 1;
            inherits: false;
        }
    }
}

@function transition($i, $j) {
    @return  ($s*random())+s (($s*($i+$j))/($x+$y))+s;
}

@mixin image() {
    $all_t: ();
    $all_m: ();
    @for $i from 0 through ($x - 1) {
        @for $j from 0 through ($y - 1) {
            $all_t: append(
                $all_t, 
                --c-#{$i}-#{$j} transition($i, $j), 
                comma
            );
            $all_m: append(
                $all_m,
                linear-gradient(rgba(0, 0, 0, var(--c-#{$i}-#{$j})) 0 0) calc(#{$i}* 100% / (#{$x} - 1)) calc(#{$j}* 100% / (#{$y} - 1)),
                comma
            );
        }
    }
    transition: $all_t;
    mask: $all_m;
}

.figure {
    @include image();
    mask-size: calc(100% /#{$x}) calc(100% /#{$y});
    mask-repeat: no-repeat;

    &:hover {
        @for $i from 0 through ($x - 1) {
            @for $j from 0 through ($y - 1) {
                --c-#{$i}-#{$j}: 0;
            }
        }
    }
}
```

  


现在，我们只需调整 `$x` 和 `$y` 的值就可以控制图像碎片化的“颗粒度”。
