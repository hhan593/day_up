## 绝对定位简介

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/10e406a81ac344cfb3b15612a460660b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1400&h=600&s=138621&e=png&b=151d2a)

  


我想大家都应该知道绝对定位的最基础应用，但我还是在这里重复说一下。我们在使用绝对定位时，通常会在它的父容器或祖先元素中设置一个 `position:relative` 。例如：

  


```HTML
<div class="parent">
    <div class="child"></div>
</div>
```

  


```CSS
.parent {
    position: relative;
    
    .child {
        position: absolute;
    }
}
```

  


但这并不是绝对的，因为只要它的父元素或祖先元素的 `position` 属性的值不是 `static` ，绝对定位 `absolute` 都将生效。但需要注意的是，只有指定了 `inset` （或它的子属性 `top` 、`right` 、 `bottom` 或 `left` 中的任意一个）属性值，绝对定位的元素才会有偏移。否则，它只是脱离了文档流，不会有任何的位置偏移。

  


另外，绝对定位的元素位置偏移是相对于离它最近的 `position` 值不是 `static` 元素进行计算。如果它的父元素以及祖先元素没有任何一个设置了 `position` 值（指的是非 `static` 值），那么它会相对于浏览器边缘进行偏移计算。

  


在 Web 中，有很多场景都会用到绝对定位，例如下图所示，卡片标题、标签和按钮都堆叠在缩略图之上，这种情景下，绝大多数的 Web 开发者都会使用绝对定位来实现：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f8b308eb135d44fd93e4b3d3f281f7de~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2000&h=767&s=1103518&e=jpg&b=fcfafa)

这样做并没有错，但在一些场景中，例如多语言环境下，仅使用物理属性 `top` 、`right` 、`bottom` 或（和）`left` 来设置元素的偏移位置，会使我们的代码变得更为复杂，同时会增加一定的维护成本。当然。你可能会说，我们可以使用逻辑属性来替代物理属性，这个想法是正确的。但我们还有更好的方法，那就是使用 CSS Grid 来构建卡片的布局。它使得定位覆盖元素（比如上图中的标题、标签和按钮等元素）可以通过更具逻辑性和可读性的属性和值来实现。稍后我们会详细阐述！

  


再来看另一个场景，比如提示框组件：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7c0c7ef7db65418697a2380547b00f5a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1536&h=864&s=41053&e=png&b=f7f9fb)

  


在构建提示框组件时，通常情况下，我们会把内容与目标元素用同一个容器包裹起来，例如：

  


```HTML
<div class="tooltip">
    <div class="tooltip__anchor">Anchor</div>
    <div class="tooltip__content">Tooltip Content</div>
</div>    
```

  


在 CSS 中，会将 `.tooltip` 与 `.tooltip__anchor` 两者设置为相同的尺寸，并且在 `.tooltip` 元素上设置 `position` 为 `relative` ，然后再将 `.tooltip__content` 设置为绝对定位。CSS 代码大致如下：

  


```CSS
.tooltip {
    position: relative;
    
    .tooltip__content {
        position: absolute;
        top: 100%;
        left: 50%;
        translate: -50% 0;
    }
}
```

  


这是构建提示框很普遍的做法，但我想大家或许碰到过，有的时候可能为了实现某种 UI 效果会在 `.tooltip` 元素上设置 `overflow` 为 `hidden` ，这个时候就会造成绝对定位的内容无法完全显示。为此，你需要添加额外的代码，来规避这种现象。

  


与提示框组件相似的案例有很多，比如“熔岩菜单”的交互效果：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce2216a6c76a4b6ab9d7ae80940c595d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1330&h=450&s=451515&e=gif&f=166&b=4b496a)

  


创建这个效果，通常会用一个额外的元素或伪元素来创建上图中的粉红色的边框效果，并且借助 JavaScript 脚本根据鼠标位置来调整它的偏移位置。现在，随着 CSS 的锚点定位特性的出现，我们有了更好的方法来实现。因为它可以改变绝对定位的锚定元素。在此之前，绝对定位锚定的是它的父元素或祖先元素，现在可以锚定它的兄弟元素，甚至是页同上的任意元素。

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2a391f86edc34256bbbda79947d0dbaf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3046&h=1509&s=100157&e=png&b=ffffff)

  


这完全改变了我们对绝对定位的认知。

  


阅读到这里，你是不很好奇，CSS Grid 布局构建布局时，在什么样的情景下可以替代绝对定位，又应该如何使用？其次，锚点定位又是如何使绝对定位元素可以锚定兄弟元素或页面上任意元素？

  


接下来，我将通过具体的案例来告诉大家答案。

首先，使用 `display: grid` 将 `.card` 元素设置为一个网格容器。在我们这个示例中，并不需要使用 `grid-template-columns` 和 `grid-template-rows` 来设置网格的列与行：

  


```CSS
.card {
    display: grid;
}
```

  


默认情况下，CSS Grid 会根据 `.card` 中的子元素自动创建行。在我们这个示例中，它将会创建一个包含七行一列的网格：

```CSS
.card {
    display: grid;

    >*:not(figure) {
        z-index: 2;
    }
    
    &::before {
        z-index: 1;
    }
}
```

## 锚点定位改变了绝对定位的默认行为

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0f19ae074ba648b8bba69f84d51a3cb5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1664&h=936&s=22609&e=avif&b=111111)

  


众所周知，使用绝对定位可以精准地将目标元素放置到指定的位置，前提是它需要一个锚定元素。传统方式中，这个锚定元素是离目标元素（使用绝对定位的元素）最近的，并且 `position` 属性的值不是 `static` 。随着 [CSS 锚点定位](https://juejin.cn/book/7223230325122400288/section/7259669151743279159)的到来，它将打破这种传统的锚定关系。简单地说，绝对定位元素的锚定点可以是它的兄弟元素，甚至是页面上的任意元素。

  


我们以经典的提示框组件为例，一起看看锚点定位给绝对定位带来哪些变化？

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/61f64e24067d4f5b9a8607acee8f8855~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2490&h=1536&s=114581&e=png&b=ffffff)

  


上图所展示的是一个经典且普通的提示框组件。以往我们创建这个组件效果，通常会将锚点元素（比如上图中的链接）和提示框内容（目标元素）用一个容器包裹起来。例如：

  


```HTML
<p>
    Click <a class="tooltip">here <span class="tooltip__content">You'll be redirected to your dashboard</span></a> if page does not reload in 5 seconds.....
<p>
```

  


或者将提示框的内容以 `data-*` 形式存放在目标元素 `<a>` 中：

  


```HTML
<p>
    Click <a class="tooltip" data-tooltip="You'll be redirected to your dashboard">here</a> if page does not reload in 5 seconds.....
</p>
```

  


对于简单的提示框，我们可以采用后面这种方式，但对于复杂的提示框，就需要考虑使用第一种方式。在这里，我将采用第一种方式，以便与之后的锚点定位的案例统一化。

  


大家比较熟悉的做法是，首先会将 `.tooltip` 元素的 `position` 设置为 `relative` （相对定位），然后再将 `.tooltip__content` 设置为绝对定位，并且使用 `inset` （或 `top` 、`right` 、`bottom` 、 `left`）与 CSS 变换中的 `translate` 来确定它的位置。

  


```CSS
.tooltip {
    position: relative;

    .tooltip__content {
        position: absolute;
        bottom: calc(cos(45deg) * 1rem + 100%);
        left: 50%;
        translate: -50% 0;
        opacity: 0;
    }

    &:hover .tooltip__content {
        opacity: 1;
    }
}
```

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3888cffed1434783a5a43d201c706fc4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1112&h=616&s=429909&e=gif&f=134&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/poMXJdx

  


上面的代码非常简单，这里就不再花时间阐述了！如果你对提示框的形状制作要是感兴趣的话，可以移步阅读小册之前的课程《[CSS 图形：创建工具提示和语音气泡](https://juejin.cn/book/7395119173656903715/section/7408966599622787083)》！

  


接下来，我们使用锚点定位来重新实现上面示例的效果。首先，调整一下 HTML 结构，把嵌套在 `.tooltip` 元素中的 `.tooltip__content` 单独拿出来。你可以将它放置在 `</p>` 之前或之后：

  


```HTML
<p>
    Click <a class="tooltip">here</a> if page does not reload in 5 seconds.....
</p>

<span class="tooltip__content">You'll be redirected to your dashboard</span>
```

  


如果继续使用传统的绝对定位方法，我们是无法将 `.tooltip__content` 锚定到 `.tooltip` 的。但是，引入 CSS 锚点定位，情况就不一样了。

  


我们首先在 `.tooltip` 中设置 `anchor-name` 属性的值为 `--tooltip` ：

  


```CSS
.tooltip {
    anchor-name: --tooltip;
}
```

  


上面的代码将会定义一个名为 `--tooltip` 的锚点。有了这个基础之后，目标元素（绝对定位元素）就可以引用这个锚点，然后将其作为锚定点。换句话说，目标元素就会相对于这个锚定点进行定位：

  


```CSS
.tooltip__content {
    position: absolute;
    position-anchor: --tooltip;
    position-area: top span-all;
    margin-bottom: calc(cos(45deg) * 1rem);
}
```

  


最后处理一下它的显示与隐藏的切换。最终 CSS 代码如下：

  


```CSS
.tooltip {
    anchor-name: --tooltip;
}

.tooltip__content {
    position: absolute;
    position-anchor: --tooltip;
    position-area: top span-all;
    margin-bottom: calc(cos(45deg) * 1rem);
    opacity: 0;
}

p:has(a:hover) ~ .tooltip__content {
    opacity: 1;
}
```

  


最后的效果是一致的：

  


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/117c06736f2a4dcebe8a59f138d9a365~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1214&h=664&s=748524&e=gif&f=141&b=040404)

  


> Demo 地址：https://codepen.io/airen/full/ExqBjMj

  


在这个示例中，绝对定位元素与锚点元素并不是嵌套关系，也不是兄弟元素，但最终通过 CSS 锚点定位特性，将它们关联在一起。从而实现我们最终想要的效果。

  


当然，为了能更好的控制提示框内容的显示与隐藏的切换效果，你也可以用之前的方式将它们放在一起，例如下面这个示例：

  


```HTML
<div class="tooltips">
    <a href="#" class="anchor">Hove Me</a>
    <span class="anchored">Sample text for your tooltip!</span>
</div>
```
