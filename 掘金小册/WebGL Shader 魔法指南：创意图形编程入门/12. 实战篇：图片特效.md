## 扭曲画廊

在`Shaders`目录下新建`Slider`目录，并在该目录下新建 2 个`Shader`文件：顶点着色器`vert.glsl`和片元着色器`frag.glsl`。

```sh
mkdir src/Experience/Shaders/Slider
touch src/Experience/Shaders/Slider/vert.glsl
touch src/Experience/Shaders/Slider/frag.glsl
```

将下面的顶点着色器代码复制进`vert.glsl`，用来确定图片网格的位置。

```glsl
uniform float iTime;
uniform vec3 iResolution;
uniform vec4 iMouse;

varying vec2 vUv;

void main(){
    vec3 p=position;
    vec4 mvPosition=modelViewMatrix*vec4(p,1.);
    gl_Position=projectionMatrix*mvPosition;

    vUv=uv;
}
```

同样地将下面的片元着色器代码复制进`frag.glsl`，用来采样纹理，显示出图片。

```glsl
uniform float iTime;
uniform vec3 iResolution;
uniform vec4 iMouse;

uniform sampler2D uTexture;

varying vec2 vUv;

void main(){
    vec2 uv=vUv;
    vec4 tex=texture(uTexture,uv);
    vec3 color=tex.rgb;
    gl_FragColor=vec4(color,1.);
}
```

在`Slider.js`中，引入这两个`Shader`，作为`ig`的 2 个着色器参数。

```js
import sliderVertexShader from "../Shaders/Slider/vert.glsl";

export default class Slider extends kokomi.Component {

    super(base);

    this.ig = new kokomi.InfiniteGallery(this.base, {
      ...
      vertexShader: sliderVertexShader,
      fragmentShader: sliderFragmentShader,
    });

    ...
  }
  ...
}
```

在顶点着色器`vert.glsl`中，新增扭曲函数`distort`，并对`mvPosition`应用它。

```glsl
vec3 distort(vec3 p){
    return p;
}

void main(){
    vec3 p=position;
    vec4 mvPosition=modelViewMatrix*vec4(p,1.);
    mvPosition.xyz=distort(mvPosition.xyz);
    gl_Position=projectionMatrix*mvPosition;

    vUv=uv;
}
```

在着色器的顶部引入`PI`等数学常量。

```glsl
#include "/node_modules/lygia/math/const.glsl"
```

在扭曲函数`distort`中，用`sin`函数配合`UV`的`y`坐标来扭曲顶点坐标的`x`轴。

```glsl
vec3 distort(vec3 p){
    p.x+=sin(uv.y*PI)*100.;
    return p;
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ae42da71260c4acc891574026e483b1b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=298644&e=jpg&b=010101)

为了理解这一步骤，我们可以上公式可视化网站 [graphtoy](https://graphtoy.com/)，输入这个公式：`sin(x*PI)`。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/34956e0e6e96417993090cbaf8329eb9~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=726&s=161237&e=jpg&b=212121)

`uv.y`是一个分布在`[0,1]`之间的值，而且是`y`轴方向的，当它的值处于`[0,0.5]`区间时，`sin(uv.y*PI)`在图像上是递增的；当值正好位于`0.5`时，`sin(uv.y*PI)`达到了最大值`1`，`p.x`的偏移量就正好是`100`；当值位于`[0.5,1]`区间时，`sin(uv.y*PI)`在图像上是递减的；当值正好位于`1`时，`sin(uv.y*PI)`达到了最小值`0`，`p.x`的偏移量就变成了`0`，最后就产生了上面的那般扭曲效果。

目前`p.x`偏移量的基础值是写死的 100，我们可以用鼠标滚动的速度来代替它。

在初始化`InfiniteGallery`的那里，新增一个`uniform`变量`uVelocity`，它代表滚动速度。

```js
export default class Slider extends kokomi.Component {
  constructor(base) {
    super(base);

    this.ig = new kokomi.InfiniteGallery(this.base, {
      ...
      uniforms: {
        uVelocity: {
          value: 0,
        },
      },
    });

    ...
  }
}
```

在`update`方法里，从`scroll`那里解构出`delta`速度变量，调用`ig`的`iterate`方法来迭代每一个图片对象（一个图片对象`maku`包括它所对应的`DOM`元素`el`和`WebGL`网格`mesh`），将`delta`速度赋给网格材质的`uVelocity`变量。

```js
export default class Slider extends kokomi.Component {
  ...
  update() {
    this.ws.syncScroll();
    const { current, delta } = this.ws.scroll;
    this.ig.sync(current);

    this.ig.iterate((maku) => {
      maku.mesh.material.uniforms.uVelocity.value = delta;
    });
  }
}
```

在顶点着色器`vert.glsl`的顶部声明`uVelocity`变量。

```glsl
uniform float uVelocity;
```

在扭曲函数`distort`中，将 100 替换为`uVelocity`。

```glsl
vec3 distort(vec3 p){
    p.x+=sin(uv.y*PI)*uVelocity;
    return p;
}
```

现在，你可以滚动你鼠标的滚轮来体验下动态扭曲的效果了~

这就说明了一个问题：拉伸图片时，实际的宽高不一致会导致图片产生严重的变形。

其实在`CSS`中，已经有一个解决方案了，那就是`background-size: cover`。

可惜的是，在`Shader`中，并没有内置的方法可以做到这一点，我们依旧要靠计算来实现它。

通过查阅 [Stack Overflow](https://stackoverflow.com/)，发现已经有人给出了[一种解决的方案](https://stackoverflow.com/a/6565988)，并且也有人把它直接转化成了对应的 [Shader 代码](https://gist.github.com/statico/df64c5d167362ecf7b34fca0b1459a44)，再封装一下就成了下面的样子：

```glsl
vec2 cover(vec2 s,vec2 i,vec2 uv){
    float rs=s.x/s.y;
    float ri=i.x/i.y;
    vec2 new=rs<ri?vec2(i.x*s.y/i.y,s.y):vec2(s.x,i.y*s.x/i.x);
    vec2 offset=(rs<ri?vec2((new.x-s.x)/2.,0.):vec2(0.,(new.y-s.y)/2.))/new;
    uv=uv*s/new+offset;
    return uv;
}
```

这个函数需要 3 个参数，其实我们已经都有了：`UV`坐标`uv`、屏幕大小`iResolution`、还有个图片大小是`InfiniteGallery`里内置的`uniform`变量`uMediaSize`。

在片元着色器`frag.glsl`中，我们直接用这个函数吧。（别忘了在顶部补充`uMediaSize`变量的声明。）

```glsl
uniform vec2 uMediaSize;

void main(){
    vec2 uv=vUv;
    uv=cover(iResolution.xy,uMediaSize.xy,uv);
    vec4 tex=texture(uTexture,uv);
    vec3 color=tex.rgb;
    gl_FragColor=vec4(color,1.);
}
```

这样图片就能在裁剪后维持原来的比例，而不会发生变形。

> 不过选图时也要注意图片的周围有没有关键信息，因为裁剪会把周围裁掉，只保留中心。

## 拖拽交互

为了进一步地优化用户体验，除了滚轮交互外，我们还可以添加拖拽交互！

所幸的是`kokomi.js`内置了一个`DragDetecter`类，用它我们就能轻松地实现这个需求。

我们用`DragDetecter`来监听拖拽事件，并获取鼠标拖拽的距离`delta`，给滚动监听器`ws`的`scroll.target`减去当前画廊维度（横向对应的就是`x`）上的拖拽距离即可，如果效果不明显可以适当地乘以一定的倍数。

## 无缝过渡

我们的效果基本已经完成了。但是总感觉还缺了点什么？

回忆一下，平时我们做的网页一般都有列表页和详情页，用户点击列表上的一条数据，就会被导向它所对应的详情数据。

确实，目前我们仅仅算是完成了“列表”部分，“详情”部分依旧还没做。

然而，我们可以不拘泥于传统的网页跳转，利用`Shader`的力量，我们可以实现一种从列表无缝地过渡到详情的效果。

第一步，我们要确定用户点击了哪张图片，这个可以通过监听点击事件来获取。

在`World.js`中，将当前选中的网格设为空（点击前不会有网格被选中），调用`ig`的`iterate`方法来迭代每一个图片对象，将它们的网格`mesh`添加到可交互的对象里，监听网格的`click`事件，即可获取用户点击的图片对象。

```js
export default class World extends kokomi.Component {
  constructor(base) {
    super(base);

    this.base.am.on("ready", async () => {
      ...
      this.currentActiveMesh = null;

      this.slider.ig.iterate((maku) => {
        this.base.interactionManager.add(maku.mesh);
        maku.mesh.addEventListener("click", () => {
          console.log(maku);
        });
      });
      ...
    });
  }
}
```

随便点击几张图片，我们便可以在控制台里看见点到的图片对象，此外，我们还要获取除它以外的所有图片对象。

```js

  const otherMakus = this.slider.ig.makuGroup.makus.filter(
    (item) => item !== maku
  );

```

先来一个比较简单的效果吧：点击其中一张图片，让它旁边的其他图片直接消失。

这里我们要用到`JS`动画框架 [GSAP](https://greensock.com/) 来实现图片消失的动画。

`GSAP`的用法非常简单，举一个小例子吧：

```js
import gsap from "gsap";

const obj = {
  value: 0,
};

gsap.to(obj, {
  value: 1,
  duration: 0.5,
});
```

`gsap.to`函数可以实现缓动动画，第一个参数是动画的对象`obj`，第二个参数是动画的配置，包含我们需要缓动的属性值（这里是`value`）、时间`duration`、延迟`delay`、缓动函数`ease`等。上面语句的含义是将`obj`对象里的`value`属性值在 0.5 秒内缓缓地变动到 1。

回到正题，我们将材质的透明度变量缓动成 0，就能实现消失的动画效果。

```js
import gsap from "gsap";

...
maku.mesh.addEventListener("click", () => {
  console.log(maku);

  const otherMakus = this.slider.ig.makuGroup.makus.filter(
    (item) => item !== maku
  );

  if (!this.currentActiveMesh) {
    otherMakus.forEach((item) => {
      gsap.to(item.mesh.material.uniforms.uOpacity, {
        value: 0,
      });
    });
  }
});
```

我们新增了一个`uniform`变量`uOpacity`，代表图片的透明度，默认值是 1，需要在`ig`的材质参数内补充一下，并且我们还要开启材质的透明通道`transparent`。

```js
this.ig = new kokomi.InfiniteGallery(this.base, {
  ...
  uniforms: {
    ...
    uOpacity: {
      value: 1,
    },
    ...params,
  },
  materialParams: {
    transparent: true,
  },
});
```

在`Slider`的片元着色器`frag.glsl`中，顶部声明`uOpacity`，把它用作`gl_FragColor`的第四个参数透明度。

```glsl
uniform float uOpacity;

...

void main(){
    ...
    gl_FragColor=vec4(color,uOpacity);
}
```

`Slider`的顶点着色器`vert.glsl`中，顶部声明`uProgress`，创建过渡函数`transition`，并对`p`应用它。

```glsl
uniform float uProgress;

vec3 transition(vec3 p){
    return p;
}

void main(){
    vec3 p=position;
    p=transition(p);
    vec4 mvPosition=modelViewMatrix*vec4(p,1.);
    mvPosition.xyz=distort(mvPosition.xyz);
    gl_Position=projectionMatrix*mvPosition;

    vUv=uv;
}
```

接下来，我们就来实现这个过渡函数。

过渡的方法可以有很多种，但这里就选择一种最简单、也是最常见的方法——全屏过渡，简言之就是图片会缩放至整个画面的一个效果。

我们先在`Slider`的顶点着色器`vert.glsl`顶部声明两个`ig`内置的`uniform`变量，分别代表图片网格的大小和位置。

```glsl
uniform vec2 uMeshSize;
uniform vec2 uMeshPosition;
```

将过渡变量`uProgress`赋给`pr`，计算屏幕大小与图片大小的比例，它同时也是我们要缩放的比例，记为`targetScale`，利用`mix`函数实现从原比例`vec2(1.)`到目标比例`targetScale`的过渡，最后再给顶点的`xy`轴乘上过渡结果。

```glsl
vec3 transition(vec3 p){
    float pr=uProgress;
    vec2 targetScale=iResolution.xy/uMeshSize.xy;
    vec2 scale=mix(vec2(1.),targetScale,pr);
    p.xy*=scale;
    return p;
}
```

点击中间的图片，就可以看到缩放的动画效果了。

但当你点击旁边的图片时，就会出现下面的情况：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/da14d1fc97d8437aa172b8f6329c7802~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=480535&e=jpg&b=030303)

这是因为旁边的图并不在原点，只执行原地缩放操作的话，是不会占满整张屏幕的。

既然它不在原点，那我们就把它给挪到原点，给顶点的`xy`轴加上取负值的图片位置与过渡值的乘积即可。

```glsl
vec3 transition(vec3 p){
    ...
    p.xy+=-uMeshPosition*pr;
    return p;
}
```

这里还要注意一点的是：所有图片的`z`轴坐标是相同的，如果一张图片与另一张图片重叠，会产生深度冲突（`z-fighting`）的现象，为了避免这一点，我们可以给顶点的`z`轴加上过渡值，这样选中的图片和其他图片的`z`位置就不会相同，也就不会产生重叠了。

```glsl
vec3 transition(vec3 p){
    ...
    p.z+=pr;
    return p;
}
```

不错，效果是实现了，但总体还是有点平淡，可以给它加点料。

目前我们的过渡值在图片网格的每一个方向上都是同步进行的，那如果把它给沿着某个方向交错开来呢？

回忆一下，在`Shader`中，`UV`坐标的分量是一个渐变，四个方向的交错值，都可以用渐变来表示：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/730ee0c868524158a104ce2c48df8718~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=43222&e=jpg&b=f9f9f9)

例如我们要实现一个从左到右的交错，那么我们就用这张图里的`left`，对应的`Shader`代码就是`uv.x`。

如果是反方向（如从右到左），交错值就是`1.-该方向`。

如果是两个方向（如左上角），那么把这两个方向的值相乘就行。

我们可以把获取交错起始值这个过程直接封装成一个函数使用，在这个函数里我选取了右上角方向。

```glsl
float getStagger(vec2 uv){
    float left=uv.x;
    float bottom=uv.y;
    float right=1.-uv.x;
    float top=1.-uv.y;
    return top*right;
}
```

在过渡函数中，我们用这个函数获取交错起始值，然后用`smoothstep`函数来决定过渡的起始与终止。

```glsl
vec3 transition(vec3 p){
    float pr=uProgress;
    float stagger=getStagger(uv);
    pr=smoothstep(stagger*.8,1.,pr);
    ...
    return p;
}
```

Nice！

图片放大到全屏后，当用户再次点击图片时，应该把它给收回去，重新恢复成之前的列表页。

在`World.js`中，监听画面整个容器的点击事件，做和之前相反的操作。这里要注意的是：之前也监听过图片的点击事件，会和这个监听发生冲突。为了避免这个冲突，我们可以在缓动的`onUpdate`钩子里判断过渡进行到了一定的程度后，才进行选中图片的赋值操作。

另外，在图片放大到全屏后，我们要把之前的鼠标滚轮和拖拽交互给临时地禁用一下，图片回归原位后再给它恢复回来。

再有一点，我们要将拖拽操作和点击图片操作给区分开来，其实很简单，只要判断点击后的滚动速度比某个值大，就表明用户是在进行拖拽相关的操作，`return`掉后面的点击逻辑代码即可。

这里改动稍微有点多，就把整个`World`类的代码都贴出来了，顺便把缓动动画的缓动`ease`也调整了下。

```js
class World extends kokomi.Component {
  constructor(base) {
    super(base);

    this.base.am.on("ready", async () => {
      this.slider = new Slider(this.base);
      await this.slider.addExisting();

      this.currentActiveMesh = null;

      this.slider.ig.iterate((maku) => {
        this.base.interactionManager.add(maku.mesh);
        maku.mesh.addEventListener("click", () => {
          console.log(maku);

          if (Math.abs(this.slider.ws.scroll.delta) > 5) {
            return;
          }

          const otherMakus = this.slider.ig.makuGroup.makus.filter(
            (item) => item !== maku
          );

          if (!this.currentActiveMesh) {
            this.slider.ws.disable();
            this.slider.dd.disable();

            otherMakus.forEach((item) => {
              gsap.to(item.mesh.material.uniforms.uOpacity, {
                value: 0,
                ease: "power2.out",
              });
            });

            const that = this;
            gsap.to(maku.mesh.material.uniforms.uProgress, {
              value: 1,
              duration: 1,
              ease: "power2.out",
              delay: 0.5,
              onUpdate() {
                if (this.progress() >= 0.5) {
                  that.currentActiveMesh = maku.mesh;
                }
              },
            });
            // this.currentActiveMesh = maku.mesh;
          }
        });
      });

      this.base.container.addEventListener("click", () => {
        if (this.currentActiveMesh) {
          const that = this;
          gsap.to(this.currentActiveMesh.material.uniforms.uProgress, {
            value: 0,
            duration: 1,
            ease: "power2.inOut",
            onUpdate() {
              if (this.progress() >= 0.5) {
                that.slider.ws.enable();
                that.slider.dd.enable();

                that.currentActiveMesh = null;
              }
            },
          });
          this.slider.ig.iterate((item) => {
            gsap.to(item.mesh.material.uniforms.uOpacity, {
              value: 1,
              delay: 0.5,
              ease: "power2.out",
            });
          });
          // this.currentActiveMesh = null;
        }
      });

      document.querySelector(".loader-screen")?.classList.add("hollow");
    });
  }
}
```

## 优化建议

至此，我们已经基本完成了这个`3D`图形特效项目，下面是一些小小的优化建议：

1.  可以改变点击图片后详情部分的布局（不一定非得是一张图铺满全屏，也可以是合适大小的图片配上一定数量的文字，文字可以适当弄些动画）。
2.  `Shader`的扭曲部分可以尝试其他的扭曲方式（比如噪声扭曲等）。
3.  可以把图片内容替换成自己的东西（例如作品等），这样你就拥有了一个炫酷的个人作品集网站！
