`Shader`，之所以叫着色器，是因为它有`shading`着色这一个概念。

`shading`着色，指的是在三维模型或插画中通过不同的亮度表现深度的方法，这种方法可以使`3D`场景显得更加逼真、生动。

那么，我们如何计算物体表面的亮度呢？图形学给了我们答案——利用“**光照模型**”来计算。

## 冯氏光照模型

在当年，一位叫“裴祥风”的图形学学者提出了“冯氏光照模型”这一概念，后来得到了图形学界的普遍采用。

它主要由 3 种光照组成：`环境光`、`漫反射`、`镜面反射`。

让我们关注片元着色器，在主函数`main`中定义好几个变量：结果颜色`col`为黑色（因为还没有光），物体的基础颜色`objectColor`为白色，光照颜色`lightColor`为亮红色。

    void main(){
        vec2 uv=vUv;

        vec3 col=vec3(0.);
        vec3 objectColor=vec3(1.);
        vec3 lightColor=vec3(.875,.286,.333);

        gl_FragColor=vec4(col,1.);
    }

### 环境光

环境光是指物体表面受到来自周围环境光源的光照，这种光照均匀地分布在物体的表面上。

它的实现非常简单，我们只需给出光的颜色，乘上光的强度即可算出。

    float ambIntensity=.2;
    vec3 ambient=lightColor*ambIntensity;
    col+=ambient*objectColor;

设定环境光照强度`ambIntensity`为 0.2，环境光`ambient`是光照颜色与环境光照强度的乘积，算出后将它和物体颜色相乘，与输出颜色相加，就得到了光照影响后的物体。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ddf1484aabe846c6a63e3b658adfc539~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=540\&s=14137\&e=jpg\&b=000000)

我们可以看到物体整体偏暗，由于环境光的作用而显现出了淡淡的颜色。

### 漫反射

漫反射是指光源照射到物体表面后，光线被物体表面反射，反射光线会向所有方向散射的现象。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b0ae099160c24f91a5ec316b621d228d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=341\&s=15252\&e=png\&b=ffffff)

见上图，当光线照射至物体表面时，会和法向量`N`（法向量就是与物体表面垂直的向量）形成一个夹角`θ`，这个夹角叫入射角。入射角越大，光线强度越小；反之，入射角越小，光线强度越大；当入射角为 0 时，光线和法向量方向会重叠，此时光照直射物体，强度是最大的。

要获得漫反射光，大致需要这几个步骤：

1.  获取法向量`N`。
2.  获取光线的方向向量。
3.  计算漫反射因子。
4.  计算漫反射光。

#### 法向量

先说法向量`N`，这个变量其实在顶点着色器里已经有了，叫`normal`，把它传递给片元着色器就要用到`varying`变量。

在顶点着色器上方声明`vNormal`。

    varying vec3 vNormal;

在主函数`main`中，将`normal`赋给`vNormal`。

    void main(){
        ...
        vNormal=normal;
    }

在片元着色器上方也声明`vNormal`。

    varying vec3 vNormal;

这样我们就获得了法向量`N`。

#### 光线的方向向量

接下来我们要计算光线的方向向量，不过在计算它之前，我们先要求出光线照物体那一部分的片元的位置，这里的片元位置是世界空间坐标系的，将原位置进行模型变换就能得出。

在顶点着色器的顶部，声明`varying`变量`vWorldPosition`，表示片元的位置，然后在主函数`main`中计算出它。

    varying vec3 vWorldPosition;

    void main(){
        ...
        vWorldPosition=vec3(modelMatrix*vec4(p,1));
    }

在片元着色器上方也声明`vWorldPosition`。

    varying vec3 vWorldPosition;

这样我们就获得了片元的位置，可以计算光线的方向向量了。

    vec3 lightPos=vec3(1.,2.,1.);
    vec3 lightDir=normalize(lightPos-vWorldPosition);

光线的方向向量`lightDir`等于光线位置`lightPos`与片元位置`vWorldPosition`的差，而且结果需要用`normalize`函数来归一化。

然后我们要计算漫反射因子。

#### 漫反射因子

漫反射因子，也就是漫反射的强度，跟入射角`θ`有关，而且它们是成反比例关系的，因此我们用入射角的余弦值来作为漫反射因子。在数学中，我们可以通过求 2 个向量的点积，来计算它们夹角的余弦值，`GLSL`有一个内置函数`dot`，能计算 2 个数的点积，我们用它来计算法向量与光线的方向向量的余弦值。

    float diff=dot(vNormal,lightDir);

这里要注意的一点是：当角度大于 90 度时，余弦函数的值会变成负数，为了避免出现负数，可以用`max`函数来确保值不小于 0。

    diff=max(diff,0.);

#### 计算漫反射光

所有的量都齐全了，只要把光照颜色与漫反射因子相乘，就能获得漫反射光照了，将光照和物体颜色相乘，与输出颜色相加，就得到了光照影响后的物体。

    vec3 diffuse=lightColor*diff;
    col+=diffuse*objectColor;

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a4bfc452f53e4e4a89d87f55bd578511~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=540\&s=47874\&e=jpg\&b=000000)

球体上产生了很明显的明暗区域，这就是漫反射光所产生的效果。

### 镜面反射

镜面反射是指光源照射到物体表面后，光线被物体表面反射，反射光线呈现出明亮的高光点的现象。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f79682fc61c94957b3fd89ed3ea8bfd2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=341\&s=22351\&e=png\&b=ffffff)

见上图，跟漫反射类似，镜面反射也是跟法向量和光照方向有关系的，只不过多了个眼睛的方向。同时，镜面反射也取决于物体自身的属性，比如镜子，它的表面非常光滑，可以让光线同时往一个方向聚焦。

要获得镜面反射光，大致需要这几个步骤：

1.  获取反射光的方向向量。
2.  获取眼睛的方向向量。
3.  计算镜面高光因子。
4.  计算镜面反射光。

#### 反射光的方向向量

`GLSL`中，`reflect`函数能算出反射光的向量，它接受 2 个参数：入射光方向向量和法向量。

    vec3 reflectDir=reflect(-lightDir,vNormal);

`reflect`函数的第一个参数要求的方向是光照指向片元，但我们之前计算的光照方向`lightDir`是片元指向光照，因此我们要把它的方向给反过来，符号取反就行了。

#### 眼睛的方向向量

眼睛的方向向量`viewDir`等于眼睛位置`cameraPosition`减去片元的位置`vWorldPosition`，结果也要归一化。

    vec3 viewDir=normalize(cameraPosition-vWorldPosition);

#### 镜面高光因子

跟之前的漫反射因子计算类似，我们要计算反射光的方向向量与眼睛的方向向量的点积，同时要确保值不能是负数。

    float spec=dot(viewDir,reflectDir);
    spec=max(spec,0.);

然而，仅仅这样是不够的。因为物体本身的属性也会影响到镜面高光的强度，考虑到这一点，我们要引入一个额外的变量——“反光度”（`shininess`），反光度越大，光线就聚焦得更厉害，这里将反光度设为了 32。

    float shininess=32.;
    spec=pow(spec,shininess);

#### 计算镜面反射光

所有的量都齐全了，只要把光照颜色与镜面高光因子相乘，就能获得镜面反射光照了，将光照和物体颜色相乘，与输出颜色相加，就得到了光照影响后的物体。

    vec3 specular=lightColor*spec;
    col+=specular*objectColor;

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/820b21ce233c49bc942627203f8be219~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=540\&s=48092\&e=jpg\&b=000000)

我们能看见球体上有一个亮斑，这就是镜面反射的结果。

先要计算出一个半程向量`halfVec`，它是光照方向的向量`lightDir`和眼睛的方向向量`viewDir`相加并归一化后得到的值。

## IBL 镜面反射

`IBL`镜面反射是一种基于环境贴图的反射技术，其中`IBL`的全称是`Image-Based Lighting`，也就是基于图像的光照，这里的图像指的就是环境贴图。

我们用`THREE.CubeTextureLoader`来加载环境贴图。环境贴图通常可以在网站 [polyhaven.com](https://polyhaven.com/hdris) 上找到，也可以偷个懒，因为下面的代码直接提供了现成的环境贴图。（代码要写在`const material = new THREE.ShaderMaterial(...)`的上方。）

    const envmap = new THREE.CubeTextureLoader().load([
      "https://s2.loli.net/2022/09/29/X8TDZROlUo6uAyG.png",
      "https://s2.loli.net/2022/09/29/KYEJ9ylQNIe6h4R.png",
      "https://s2.loli.net/2022/09/29/GqseLg6tWoluDzV.png",
      "https://s2.loli.net/2022/09/29/LUk8P21MJG6AtNF.png",
      "https://s2.loli.net/2022/09/29/4BO1JHoM3phFCb7.png",
      "https://s2.loli.net/2022/09/29/5NvAxfCVqlKFRZU.png",
    ]);

在材质的`uniform`变量中，添加`iChannel0`变量，将环境贴图放进去。

    const material = new THREE.ShaderMaterial({
      vertexShader: /* glsl */ `...`,
      fragmentShader: /* glsl */ `...`,
      uniforms: {
        iChannel0: {
          value: envmap,
        },
      },
    });

在片元着色器的顶部，声明`iChannel0`变量，注意它的类型是`samplerCube`，也就是“立方体贴图”。

    uniform samplerCube iChannel0;

然后，我们在主函数`main`中计算`IBL`镜面反射。

    void main(){
        ...

        // IBL
        float iblIntensity=.2;
        vec3 iblCoord=normalize(reflect(-viewDir,vNormal));
        vec3 ibl=texture(iChannel0,iblCoord).xyz;
        vec3 iblLight=ibl*iblIntensity;
        col+=iblLight*objectColor;

        gl_FragColor=vec4(col,1.);
    }

先算出反射光的方向向量`iblCoord`，这里的入射光方向参数就用眼睛的方向向量`viewDir`，法向量用`vNormal`，结果要归一化。

用`texture`函数来对环境贴图`iChannel0`进行采样，坐标就选反射光的方向向量`iblCoord`，取前三个分量赋给`ibl`变量。

将`ibl`变量与定义好的光照强度`iblIntensity`相乘，就得到了`IBL`镜面反射光`iblLight`。

将`IBL`镜面反射光`iblLight`与输出颜色相加，就得到了光照影响后的物体。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3e6f15d77dd94187b4da93e5e398220e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960\&h=540\&s=52494\&e=jpg\&b=000000)

我们可以看见球体內部出现了环境的倒影，这就是`IBL`镜面反射光产生的结果。

## 菲涅尔反射

菲涅尔反射是指光源照射到物体表面后，光线一部分发生反射，一部分进入物体内部，发生折射或散射的现象。

计算现实世界中的菲涅尔反射是相当复杂的，因此我们将使用一个菲涅尔近似等式来计算它，它的代码实现如下：

    float fresnel(float bias,float scale,float power,vec3 I,vec3 N)
    {
        return bias+scale*pow(1.-dot(I,N),power);
    }

在`main`函数中计算菲涅尔光照。

    void main(){
        ...

        // fresnel
        vec3 fresColor=vec3(1.);
        float fresIntensity=.6;
        float fres=fresnel(0.,1.,5.,viewDir,vNormal);
        vec3 fresLight=fres*fresColor*fresIntensity;
        col+=fresLight*objectColor;

        gl_FragColor=vec4(col,1.);
    }

定义菲涅尔光照颜色`fresColor`、菲涅尔光照强度`fresIntensity`。

用`fresnel`函数计算出菲涅尔因子，`bias`一般固定是 0，`scale`和`power`可以根据需要自由调整，`I`传眼睛的方向向量`viewDir`，`N`传法向量`vNormal`。

将菲涅尔因子、菲涅尔光照颜色和菲涅尔光照强度三者相乘，就得到了菲涅尔光照。

将菲涅尔光照与输出颜色相加，就得到了光照影响后的物体。

## 法向量修复

其实到目前为止，我们都没意识到一个问题：我们所使用的法向量`normal`是否是正确的？答案是否。

尽管目前我们还看不出问题所在，但只要对代码稍作调整，就能让它浮现出来。

### 修复法向量朝向

我们把原先的球体`SphereGeometry`改成另一种几何体——环面扭结体`TorusKnotGeometry`。

```js
// const geometry = new THREE.SphereGeometry(2, 64, 64);
const geometry = new THREE.TorusKnotGeometry(1.2, 0.4, 128, 32);
```

![](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6803c780e92348129029d6c302ed8d82~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=76033&e=jpg&b=000000)

在`update`动画循环中，让它转起来。

```js
this.update(() => {
  ...

  const t = this.clock.elapsedTime;
  mesh.rotation.y = 0.2 * t;
});
```

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/59664f55858741f490c2a0eb3ad625f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=1905693&e=gif&f=163&b=060606)

可以发现它在旋转的过程中，边缘光变得越来越大，是因为光也在随着物体进行旋转，但事实并非如此，光应该是静止的才对。

那为什么会这样呢？是因为法向量`normal`并没有考虑到物体的变换。既然如此，我们对法向量也应用变换不就行了。

```glsl
void main(){
    ...
    // vNormal=normal;
    vNormal=(modelMatrix*vec4(normal,0.)).xyz;
    ...
}
```

这里你可能会注意到与变换矩阵相乘向量的第四维度`w`是 0，跟之前`position`的 1 不一样。画张图来说明下吧：

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6fd13f96f404470fab189e4e332340e6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=333&s=49181&e=jpg&b=ffffff)

假设我们就跟之前的`position`一样，把`w`设为 1，就会出现法向量也会有偏转的现象，而法向量是始终垂直于物体表面的，不应该发生偏转，把`w`设为 0，就能避免这个问题。

改完后让我们再次观察下旋转的物体吧。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bd6663d5eb7346d9866f315ef3016503~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=1744059&e=gif&f=133&b=070506)

边缘光的大小恢复了正常，问题解决\~

### 修复网格状图案

尽管可能不太明显，但只要你仔细观察下图形的表面，就能发现有一些网格状的图案。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a581102ea47f4db7b0d4f0439ad10c08~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=162929&e=jpg&b=000000)

这又是为什么呢？是因为法向量`normal`的长度并不为 1。

在每个顶点之间，`varying`变量是受插值影响的，`vNormal`自然也不例外，当对 2 个单位向量进行插值时，所得到的向量长度不一定是 1。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d9080a2e95414c0383c301098fcd86e0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=385&s=48592&e=jpg&b=ffffff)

那么如何修复呢？很简单，在片元着色器中对法向量再次进行归一化操作不就 OK 了。

先把之前在`main`函数里写的`vNormal`批量替换成`normal`，再在`uv`的下方声明归一化后的`normal`即可（懒的话可以直接复制下面的代码）。

```glsl
void main(){
    vec2 uv=vUv;

    vec3 normal=normalize(vNormal);

    vec3 col=vec3(0.);
    vec3 objectColor=vec3(1.);
    vec3 lightColor=vec3(.875,.286,.333);

    // ambient
    float ambIntensity=.2;
    vec3 ambient=lightColor*ambIntensity;
    col+=ambient*objectColor;

    // diffuse
    vec3 lightPos=vec3(10.,10.,10.);
    vec3 lightDir=normalize(lightPos-vWorldPosition);
    float diff=dot(normal,lightDir);
    diff=max(diff,0.);
    vec3 diffuse=lightColor*diff;
    col+=diffuse*objectColor;

    // specular
    vec3 reflectDir=reflect(-lightDir,normal);
    vec3 viewDir=normalize(cameraPosition-vWorldPosition);
    // float spec=dot(viewDir,reflectDir);
    vec3 halfVec=normalize(lightDir+viewDir);
    float spec=dot(normal,halfVec);
    spec=max(spec,0.);
    float shininess=32.;
    spec=pow(spec,shininess);
    vec3 specular=lightColor*spec;
    col+=specular*objectColor;

    // IBL
    float iblIntensity=.2;
    vec3 iblCoord=normalize(reflect(-viewDir,normal));
    vec3 ibl=texture(iChannel0,iblCoord).xyz;
    vec3 iblLight=ibl*iblIntensity;
    col+=iblLight*objectColor;

    // fresnel
    vec3 fresColor=vec3(1.);
    float fresIntensity=.6;
    float fres=fresnel(0.,1.,5.,viewDir,normal);
    vec3 fresLight=fres*fresColor*fresIntensity;
    col+=fresLight*objectColor;

    gl_FragColor=vec4(col,1.);
}
```

## 小结

来简要地总结下本文的要点吧：

1.  `Shader`本来的意义就来自于`shading`着色，着色的本质是计算物体表面的亮度，而计算的方式就是通过“光照模型”来计算。
2.  冯氏光照模型是最基础的光照模型，由环境光、漫反射、镜面反射这 3 种光照组成。
3.  `Blinn-Phong`光照模型是冯氏光照模型的改进版，可以优先使用它作为光照模型。
4.  `IBL`镜面反射是一种基于环境贴图的反射技术，能使物体反射出环境的倒影。
5.  菲涅尔反射能给物体添加边缘光，可以用菲涅尔近似等式来计算它。
6.  直接用法向量`normal`会出现一些小问题，需要进行一定的修复。

当你对光照模型有了充分的理解时，你就有能力去写出一些逼真的图形效果了。
