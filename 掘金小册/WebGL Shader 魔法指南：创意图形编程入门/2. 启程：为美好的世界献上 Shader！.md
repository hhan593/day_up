在正式开始之前，让我们先做一些准备工作吧。


## Shader 编程语言选择

首先，我们要选择一门`Shader`的编程语言。

目前`Shader`的主要编程语言有以下几种：

1. `GLSL`：主要用于`OpenGL`平台的图形`API`。
2. `HLSL`：由微软开发，主要用于`DirectX`平台的图形`API`。
3. `Cg`：由`Nvidia`开发，可被编译为`GLSL`和`HLSL`。
4. `WGSL`：主要用于`WebGPU`的图形`API`。

本小册主要是在`Web`环境下进行开发的，而`Web`环境有 2 种图形平台可供选择：`WebGL`和`WebGPU`。`WebGL`本身是基于`OpenGL ES`的一个`JS`图形`API`，而`OpenGL ES`又从属于`OpenGL`，因此我们可以在`GLSL`和`WGSL`中进行选择。（不考虑`Cg`，因为它主要被用于`Unity`平台。）

在小册写成的当下，`WebGPU`尽管较新，但浏览器的支持率并不是很高，这使得写出的效果只能在少数的浏览器下运行，用来初学不是很合适。而`WebGL`则是主流的浏览器几乎都支持，并且大部分网站的`Shader`效果也是基于这个图形平台实现的，因此我们将选择`WebGL`平台所对应的语言——`GLSL`——作为我们的`Shader`编程语言。



## WebGL 渲染管线流程

由于`GLSL`是属于`WebGL`平台的，因此我们有必要了解下`WebGL`渲染管线的整个流程是什么样的。说的通俗点，就是一个物体是如何被渲染到屏幕上的这一过程。

直接上图吧：

<p align=center><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0a729b8d1dae444c9799c7083aa84f65~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=600&s=26191&e=png&b=ffffff" alt=""  /></p>

还有其他的一些步骤（如深度测试等）我们可以暂时先不去了解，只要把上面的主要步骤理解即可。

整个流程是这样的：我们在`JS`中提供顶点的数据（通常是`Float32Array`类型的数组，包含了顶点的位置等信息），将这些数据传递给顶点着色器，让它计算每个顶点的位置，然后`WebGL`将顶点装配成图元（如三角形），图元再被转换成屏幕上的空像素（光栅化），让片元着色器来计算每个像素的颜色并填充上去，最终将物体渲染到屏幕上。

其中，图元装配和光栅化是`WebGL`自带的操作，我们无法进行额外的定制，但顶点着色器和片元着色器则是完全可通过编程来定制化的。我们学习这本小册的目的，就是要把对它们两者的定制化给做到极致！

本小册前面的章节，会专注于学习片元着色器，要渲染的物体仅仅是一个占满屏幕的平面。到后期，当我们接触顶点着色器时，我们就可以来玩`3D`的物体了。

## 第一个 Shader

撒，哈吉马路由！（好，让我们开始吧！）

`GLSL`语言的`Shader`文件后缀名就是`.glsl`，新建一个文件，命名为`first-shader.glsl`。

首先，编写`Shader`的主体。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){

}
```

我们定义了一个`mainImage`函数，它不返回任何值，故返回类型为`void`，它接受 2 个参数：一个是 4 维的`fragColor`，代表输出的像素颜色；另一个是 2 维的`fragCoord`，代表输入的像素坐标。

> 注意：从这一章到[《顶点着色器》](https://juejin.cn/book/7267462574734573604/section/7267463312315514921)章节之前，我们都是在`Shadertoy`的环境下进行的，主函数`main`需写成`mainImage`，输出颜色`gl_FragColor`需写成`fragColor`，输入坐标`gl_FragCoord`需写成`fragCoord`，《顶点着色器》章节后我们会换回原来的写法。

暂时先不用管`fragCoord`，直接向屏幕输出一个颜色吧，例如红色：

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec3 color=vec3(1.,0.,0.);
    fragColor=vec4(color,1.);
}
```

我们定义了一个名为`color`的 3 维变量，要将它的值设置为红色，红色的`RGB`颜色值为`(255,0,0)`，在`GLSL`中，我们需要先将颜色原先的值进行归一化操作（除以`255`）后才能将它正确地输出，因此将红色的值归一化后我们就得到了`(1,0,0)`这个值，将它转换为 3 维变量`vec3(1.,0.,0.)`赋给`color`变量。最后我们给输出颜色`fragColor`赋值一个 4 维变量，前 3 维就是`color`这个颜色变量，最后一维是透明度，由于纯红色并不透明，直接将其设为 1 即可。

按下`Ctrl+Shift+P`，输入`Shader Toy: Show GLSL Preview`，点击即可预览我们的结果，如果一切顺利的话，你应该能看到画面是一片红色。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4cccb6c12f4747f9ba8461d69cf86aa8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=3398&e=jpg&b=fe0000)

如果想要把结果变成黄色的话，改变`color`变量的值即可，黄色的颜色值是`(255,255,0)`，除以 255 归一化后便是`(1,1,0)`。

```glsl
vec3 color=vec3(1.,1.,0.);
```

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/03b01038ca8c4fc79bd880edd6d97433~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=3398&e=jpg&b=fefe00)

仅仅输出一种颜色也太单调了，让我们尝试连续输出四种颜色吧！

定义好四个颜色的变量（这里你既可以直接用下面选好的颜色，也可以选你自己喜欢的颜色）：

> 小贴士：`vscode`编辑器的 [glsl-canvas](https://marketplace.visualstudio.com/items?itemName=circledev.glsl-canvas) 插件有取色器的功能，用它选颜色更方便哦~

```glsl
vec3 color1=vec3(1.,0.,1.);
vec3 color2=vec3(1.,1.,0.);
vec3 color3=vec3(0.,0.,1.);
vec3 color4=vec3(1.,0.,0.);
```

先来输出第一种颜色吧，让它占画面整体的四分之一。

这里我们就要用到`fragCoord`变量了，它代表了输入的像素坐标，有 2 个维度`xy`，它们的大小取决于画面本身的大小。假设我们画面当前的大小为`1536x864`，那么每一个像素的`fragCoord`的`x`坐标值将会分布在`(0,1536)`之间，`y`坐标值则分布在`(0,864)`之间。

在当前的`Shader`开发环境内，还有个内置的变量`iResolution`，代表了画面整体的大小，使用它时一般会取它的`xy`维度。

我们取`fragCoord`的`x`轴维度，判断如果它小于四分之一的画面长度`iResolution.x*.25`，就填充第一种颜色`color1`。

```glsl
if(fragCoord.x<iResolution.x*.25){
    fragColor=vec4(color1,1.);
}
```

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/98b18ff3a8ea416e8e991c393bac8906~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=7205&e=jpg&b=000000)

然后是第二种颜色，如果`x`大于等于四分之一的画面长度`iResolution.x*.25`且小于二分之一的画面长度`iResolution.x*.5`，就填充第二种颜色`color2`。

```glsl
if(fragCoord.x<iResolution.x*.25){
    fragColor=vec4(color1,1.);
}else if(fragCoord.x>=iResolution.x*.25&&fragCoord.x<iResolution.x*.5){
    fragColor=vec4(color2,1.);
}
```

![](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c29b6c9521f244c0b2145218716ed38b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=11238&e=jpg&b=000000)

依此类推，剩余 2 种颜色也可以通过范围判断来填充。

```glsl
if(fragCoord.x<iResolution.x*.25){
    fragColor=vec4(color1,1.);
}else if(fragCoord.x>=iResolution.x*.25&&fragCoord.x<iResolution.x*.5){
    fragColor=vec4(color2,1.);
}else if(fragCoord.x>=iResolution.x*.5&&fragCoord.x<iResolution.x*.75){
    fragColor=vec4(color3,1.);
}else{
    fragColor=vec4(color4,1.);
}
```

![](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ecb865808c8046e0bad113438466738c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=15520&e=jpg&b=7f00fe)

我们得到了一个有着四种颜色的画面。（感觉有某种既视感？）

尽管它很简单，但也很好地说明了`Shader`的核心要点：**根据像素的坐标来计算出对应的颜色**。




## 小结

学到这里，让我们来简单地总结一下吧。

1. 本小册选择了基于`WebGL`平台的`GLSL`语言作为我们学习`Shader`的编程语言。
2. `WebGL`的渲染管线大致分成这几个部分：顶点着色器、图元装配、光栅化、片元着色器。其中我们可以通过编程来控制的是顶点着色器和片元着色器。
3. `Shader`的开发环境主要有 2 种：网站和编辑器，根据个人偏好，两种都可以选择。
4. 用编辑器写`Shader`需要预先安装一些插件，包括`Shader`语言支持、实时预览、语法高亮、格式化等。
5. `Shader`有一个主体函数`mainImage`，接受 2 个参数：输出颜色`fragColor`和输入坐标`fragCoord`。
6. 通过定义`vec`类型的变量赋值给`fragColor`，我们可以向屏幕输出颜色，颜色的值需要归一化。
7. 我们可以通过判断像素在屏幕上的哪个位置来填充不同的颜色，同时也是`Shader`的核心要点：根据像素的坐标来计算出对应的颜色。
