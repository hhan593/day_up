## 改变外观

小球默认的外观是一个`UV`坐标的图案，来尝试改变它吧。

先在片元着色器`frag.glsl`中把它改成白色。

```glsl
void main(){
    vec2 uv=vUv;
    vec3 col=vec3(1.);
    gl_FragColor=vec4(col,1.);
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/30a6d356933d437db2f59420c643454c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=15097&e=jpg&b=000000)

在顶点着色器`vert.glsl`中，在顶部声明`varying`变量`vNoise`，将扭曲函数`distort`中的噪声变量`noise`赋给它。

```glsl
varying float vNoise;

vec3 distort(vec3 p){
    float noise=cnoise(p+iTime);
    vNoise=noise;
    p+=noise*normal*.3*uDistort;
    return p;
}
```

在片元着色器`frag.glsl`的顶部也声明`varying`变量`vNoise`，这样我们就将顶点着色器里的噪声值传递给了片元着色器。

```glsl
varying float vNoise;
```

### 颜色

`Shader`使我们能够赋予图形各种各样的颜色，我们既可以直接指定颜色的`RGB`色值，也可以通过计算来动态生成颜色，这里我们选择后者。

`iq`（没错，又是这位图形学大咖）在早期的研究中提出了一种基于`cos`函数的染色方案。它的公式是这样的：

$$color(t)=a+b⋅cos[2π(c⋅t+d)$$

可以看出我们需要指定 4 种颜色`a,b,c,d`，以及一个变化参数`t`。

将它转换成`Shader`代码是这样的：

```glsl
vec3 palette(in float t,in vec3 a,in vec3 b,in vec3 c,in vec3 d)
{
    return a+b*cos(6.28318*(c*t+d));
}
```

当然，在我们的实战项目中，我们无需再复制一遍这份代码，只需引入`lygia`内置的`palette`函数即可。

4 种颜色在`iq`博客的[原文](https://iquilezles.org/articles/palettes/)上有列举，可以选一种自己喜欢的，变化参数`t`就用我们刚刚传入的噪声`vNoise`吧。

```glsl
#include "/node_modules/lygia/color/palette.glsl"

void main(){
    vec2 uv=vUv;
    vec3 col=vec3(1.);

    col=palette(vNoise,vec3(.5),vec3(.5),vec3(1.),vec3(0.,.10,.20));

    gl_FragColor=vec4(col,1.);
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9cf744a1e6fa4bb09f079fb390b3ce3c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=80923&e=jpg&b=000000)

图形呈现出了一种不一般的颜色来，这就是计算颜色的魅力。

### 光照

除了颜色以外，我们也不要忘了`Shader`的拿手绝活——光照！

> 注：在使用光照之前，别忘了在顶点着色器中将法向量`vNormal`（需要修正）和片元位置`vWorldPosition`传递给片元着色器，步骤这里就省略了，可以顺带复习下~

在[《光照模型》](https://juejin.cn/book/7267462574734573604/section/7267463407082291241)一章中，我们学到了几种最基础的光照手段，其中最常用于特效创作的手段是菲涅尔反射，因为它能给物体带上一层淡淡的边缘光，看上去就挺酷的。

在片元着色器`frag.glsl`中，注释掉刚刚的`palette`颜色，计算眼睛的方向向量`viewDir`，直接引入`lygia`内置的`fresnel`函数并调用它。

> 注：`fresnel`的调用方式不要照搬我们之前自己写的那个函数，要按照`lygia`所对应的[函数源码](https://github.com/patriciogonzalezvivo/lygia/blob/main/lighting/fresnel.glsl)来，参数顺序是初始色、法向量、眼睛的方向向量。

```glsl
#include "/node_modules/lygia/lighting/fresnel.glsl"

void main(){
    vec2 uv=vUv;

    vec3 normal=normalize(vNormal);

    vec3 col=vec3(1.);

    // col=palette(vNoise,vec3(.5),vec3(.5),vec3(1.),vec3(0.,.10,.20));

    vec3 viewDir=normalize(cameraPosition-vWorldPosition);
    col=fresnel(vec3(0.),normal,viewDir);

    gl_FragColor=vec4(col,1.);
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/958ad851e7aa47a6bfba7e8fb1c74235~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=23100&e=jpg&b=010101)

效果是有了，但看上去好像不太对，光亮并不在边缘，而是分布在了图形的后方。

为什么呢？是因为我们对顶点进行了扭曲的操作，仅仅改变了位置`position`，但未改变法线`normal`，就导致了光照依旧停留在原来的球体形状的位置，造成了上图的景象。

怎么修复呢？我们需要把法线也扭曲一下，但扭曲法线也并不仅仅是给法线应用扭曲函数就完事了，还需要一定的计算，不过好在已经有人把这种计算步骤放在了 [codepen](https://codepen.io/marco_fugaro/pen/xxZWPWJ) 上，我们将它封装成函数直接拿来用吧。

由于是一个通用的函数，我们可以在`Shaders`目录下创建一个通用函数的目录`Common`，再新建一个文件`fixNormal.glsl`来存放它，以便以后的项目能共用。

```sh
mkdir src/Experience/Shaders/Common
touch src/Experience/Shaders/Common/fixNormal.glsl
```

我们将下面的代码`CV`进`fixNormal.glsl`里。

```glsl
vec3 orthogonal(vec3 v){
    return normalize(abs(v.x)>abs(v.z)?vec3(-v.y,v.x,0.)
    :vec3(0.,-v.z,v.y));
}

vec3 fixNormal(vec3 position,vec3 distortedPosition,vec3 normal,float offset){
    vec3 tangent=orthogonal(normal);
    vec3 bitangent=normalize(cross(normal,tangent));
    vec3 neighbour1=position+tangent*offset;
    vec3 neighbour2=position+bitangent*offset;
    vec3 displacedNeighbour1=distort(neighbour1);
    vec3 displacedNeighbour2=distort(neighbour2);
    vec3 displacedTangent=displacedNeighbour1-distortedPosition;
    vec3 displacedBitangent=displacedNeighbour2-distortedPosition;
    vec3 displacedNormal=normalize(cross(displacedTangent,displacedBitangent));
    return displacedNormal;
}
```

在`TestObject.js`中，我们将半径和细分这 2 个变量单独提出来，将它们用作球体本身的属性的同时，也作为宏定义`defines`传入`Shader`，用于`fixNormal`函数的`offset`参数。

```js
// const geometry = new THREE.SphereGeometry(2, 64, 64);
const RADIUS = 2.001;
const SEGMENTS = 64.001;
const geometry = new THREE.SphereGeometry(RADIUS, SEGMENTS, SEGMENTS);
const material = new THREE.ShaderMaterial({
  vertexShader: testObjectVertexShader,
  fragmentShader: testObjectFragmentShader,
  defines: {
    RADIUS,
    SEGMENTS,
  },
});
```

> 注意：`defines`会根据传入的变量自动判断类型，由于`fixNormal`函数参数是浮点型，我们需要传入带小数点的数字，所以取了`2.001`这样的值。

在顶点着色器`vert.glsl`中，我们把`fixNormal`函数引进来（要放在`distort`函数的下方），在主函数里应用它即可。

这里我们把扭曲后的位置`dp`单独存成一个变量传入`fixNormal`函数，是因为这样可以不用在函数内部再扭曲一次（扭曲操作本身比较耗费性能，我们应该防止不必要的性能损耗）。

```glsl
#include "../Common/fixNormal.glsl"

void main(){
    vec3 p=position;
    vec3 dp=distort(p);
    gl_Position=projectionMatrix*modelViewMatrix*vec4(dp,1.);

    vUv=uv;
    // vNormal=(modelMatrix*vec4(normal,0.)).xyz;
    vec3 fNormal=fixNormal(p,dp,normal,RADIUS/SEGMENTS);
    vNormal=(modelMatrix*vec4(fNormal,0.)).xyz;
    vWorldPosition=vec3(modelMatrix*vec4(dp,1));
}
```

这样那个酷酷的边缘光效果就又回来啦~

```glsl
    float t=(p.y+offset)*PI*12.;
    float noise=(sin(t)*p.x+cos(t)*p.z)*2.;
    noise*=uDistort;
    vNoise=noise;
    p+=noise*normal*.01;
    return p;
}
```

## 调整画面

目前我们画面的主体色调是黑色，尽管这样也挺酷，但我们来试试把它换成别的颜色吧。

颜色的调试跟之前值的调试有点不太一样，我们声明一个`colorParams`的对象，里面定义好主色调`themeColor`的十六进制颜色值。

使用颜色时，我们要给颜色值包上`THREE.Color`，先把背景色`this.base.scene.background`设定好。

给材质新增`uThemeColor`的`uniform`变量，将颜色赋给它的`value`。

新增调试时要调用`addColor`方法，并且通过`onChange`方法来监听颜色值的改变，如果值改变了，那么我们更新材质颜色和背景的值。

```js
export default class TestObject extends kokomi.Component {
  constructor(base) {
    super(base);

    ...

    const colorParams = {
      themeColor: "#000000",
    };

    this.base.scene.background = new THREE.Color(colorParams.themeColor);

    ...
    material.uniforms = {
      ...material.uniforms,
      ...uj.shadertoyUniforms,
      ...params,
      uThemeColor: {
        value: new THREE.Color(colorParams.themeColor),
      },
    };

    const debug = this.base.debug;
    if (debug.active) {
      ...
      debugFolder.addColor(colorParams, "themeColor").onChange((val) => {
        material.uniforms.uThemeColor.value = new THREE.Color(val);
        this.base.scene.background = new THREE.Color(val);
      });
    }
  }
}
```

片元着色器`frag.glsl`中，我们将`uThemeColor`在顶部传入，将它作为`fresnel`函数的第一个参数。

> 注意：新版的`three.js`默认使用的是`sRGB`颜色空间，为了保持颜色的一致，我们需要给颜色加上伽玛校正函数`linear2gamma`，这个函数也是`lygia`内置的。

```glsl
#include "/node_modules/lygia/color/space.glsl"

uniform vec3 uThemeColor;

void main(){
    vec2 uv=vUv;
    vec3 col=vec3(1.);

    ...

    vec3 viewDir=normalize(cameraPosition-vWorldPosition);
    col=fresnel(uThemeColor,normal,viewDir);

    col=linear2gamma(col);

    ...

    gl_FragColor=vec4(col,1.);
}
```

给小球加上一束漫反射的光，可以回忆下[《光照模型》](https://juejin.cn/book/7267462574734573604/section/7267463407082291241)一章是怎么写的。

```glsl
void main(){
    vec2 uv=vUv;
    vec3 col=vec3(1.);

    ...

    vec3 lightColor=vec3(1.,0.,0.);
```

整个漫反射光照直接应用到了小球上，这样固然也不错，但我们可以做得更酷一点~

还记得混合函数`mix`吗？我们就用它来混合物体颜色和光照颜色，至于混合因子，我们用漫反射因子与菲涅尔反射因子的乘积。

```glsl

    col=mix(col,lightColor,diff*fres);
```

```glsl
#include "/node_modules/lygia/color/palette.glsl"
#include "/node_modules/lygia/lighting/fresnel.glsl"
#include "/node_modules/lygia/color/space.glsl"

uniform float iTime;
uniform vec3 iResolution;
uniform vec4 iMouse;

varying vec2 vUv;

varying float vNoise;
varying vec3 vNormal;
varying vec3 vWorldPosition;

uniform vec3 uThemeColor;
uniform vec3 uLightColor;
uniform float uFresnelIntensity;
uniform float uLightIntensity;
uniform vec3 uLight2Color;
uniform float uLight2Intensity;

void main(){
    vec2 uv=vUv;

    vec3 normal=normalize(vNormal);

    vec3 col=vec3(1.);

    // cos palette
    // col=palette(vNoise,vec3(.5),vec3(.5),vec3(1.),vec3(0.,.10,.20));

    // basic
    col=uThemeColor;

    // fresnel
    vec3 viewDir=normalize(cameraPosition-vWorldPosition);
    vec3 fres=fresnel(vec3(0.),normal,viewDir);
    col+=fres*uFresnelIntensity;

    // diffuse
    // vec3 lightColor=vec3(1.,0.,0.);
    vec3 lightPos=vec3(10.,5.,10.);
    float diff=max(dot(normal,normalize(lightPos-vWorldPosition)),0.);
    // col+=lightColor*diff;
    col=mix(col,uLightColor,diff*fres*uLightIntensity);

    vec3 light2Pos=vec3(-10.,-5.,10.);
    float diff2=max(dot(normal,normalize(light2Pos-vWorldPosition)),0.);
    col=mix(col,uLight2Color,diff2*fres*uLight2Intensity);

    // gamma
    col=linear2gamma(col);

    // debug noise
    // col=vec3(vNoise);

    gl_FragColor=vec4(col,1.);
}
```

## 优化建议

至此，我们已经基本完成了这个`3D`图形特效项目，下面是一些小小的优化建议：

1.  进一步地优化图形的颜色和光照。
2.  研究出更多的扭曲方式（可以试试`fbm`或其他噪声）。
3.  添加后期处理效果（如果想增强画面的颜色感，强烈推荐 [UnrealBloom](https://threejs.org/examples/webgl_postprocessing_unreal_bloom) 滤镜）。
4.  添加鼠标的互动效果。
5.  基于这个效果，玩一把音频可视化。（可以参考下官方的这个[demo](https://threejs.org/examples/#webaudio_visualizer)）
