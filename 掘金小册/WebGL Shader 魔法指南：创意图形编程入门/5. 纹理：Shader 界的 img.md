## 采样

引入纹理后，为了将它显示在屏幕上，我们需要对它进行采样操作。

`GLSL`内置了一个采样函数`texture`，我们直接用它将纹理给采样出来。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec2 uv=fragCoord/iResolution.xy;
    vec3 tex=texture(iChannel0,uv).xyz;
    fragColor=vec4(tex,1.);
}
```

采样函数`texture`接受 2 个参数：第一个是我们的纹理`iChannel0`，第二个是`UV`坐标变量`uv`，将采样的结果命名为`tex`，并且取它的前 3 维度（我们暂不考虑纹理的透明度），最后用变量`vec4(tex,1.)`输出到屏幕上。

如果你能看到屏幕上显示的图片，那就说明你成功地采样了纹理。

[jcode](https://code.juejin.cn/pen/7255175913963257892)

> 对于用编辑器开发的伙伴来说，可能有少数人的屏幕会是黑色的，并且控制台上报了一行红色的错误。原因很可能是你的浏览器不支持`WebGL2`，有 2 个选择：1. 升级你的浏览器；2. 将纹理采样函数`texture`改名为`texture2D`。

## 扭曲

第一种操作是扭曲，通过改变纹理的`UV`坐标来扭曲整个图片的形状。

让我们来创建一个扭曲函数，就叫`distort`吧。

```glsl
vec2 distort(vec2 p){
    return p;
}
```

在主函数`mainImage`中调用它。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec2 uv=fragCoord/iResolution.xy;
    uv=distort(uv);
    vec3 tex=texture(iChannel0,uv).xyz;
    fragColor=vec4(tex,1.);
}
```

图片没有任何变化，因为我们的扭曲函数仅仅是返回了原来的`UV`。

给扭曲函数内的`UV`坐标（也就是变量`p`）的`x`分量加上`sin`函数处理过的`y`分量。

```glsl
vec2 distort(vec2 p){
    p.x+=sin(p.y);
    return p;
}
```

## 转场

第二种操作是转场，简言之就是从一张纹理转变成另一张纹理。

这种操作需要两张图片，之前我们仅仅引入了一张图片，需要再引入一张，你可以自己找，也可以直接用本小册提供的图片链接：<https://s2.loli.net/2023/09/10/Jb8mIhZMBElPiuC.jpg>。

在`Shader`的上方加入如下的代码：

```glsl
#iChannel1 "https://s2.loli.net/2023/09/10/Jb8mIhZMBElPiuC.jpg"
```

如果你在`Shadertoy`上编写，只需点击`iChannel1`选择图片即可。

两张纹理都引入后，我们创建 2 个采样函数，用于将 2 个纹理采样出来。

```glsl
vec4 getFromColor(vec2 uv){
    return texture(iChannel0,uv);
}

vec4 getToColor(vec2 uv){
    return texture(iChannel1,uv);
}
```

然后就是我们的转场函数`transition`。

```glsl
vec4 transition(vec2 uv){
    float progress=iMouse.x/iResolution.x;
    return mix(getFromColor(uv),getToColor(uv),progress);
}
```

转场函数将 2 张纹理通过`mix`函数混合了起来，并且混合程度是用户鼠标归一化后的`x`轴位置。

在主函数`mainImage`中调用转场函数。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec2 uv=fragCoord/iResolution.xy;
    vec4 col=transition(uv);
    fragColor=col;
}
```

### 滑动转场

还记得`step`函数吗？如果用它会是什么样的效果呢？

```glsl
vec4 transition(vec2 uv){
    float progress=iMouse.x/iResolution.x;
    return mix(getFromColor(uv),getToColor(uv),1.-step(progress,uv.x));
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4ac7ab35717a451095a701c7ee557917~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=3328750&e=gif&f=65&b=817e65)

我们得到了一个像帘子一般的左右滑动的转场效果。原理见下图：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d4b37eb41d1944debd6853ea8d55f03c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=662&s=311902&e=jpg&b=f8f3f2)

将画布分成 2 个部分，位于鼠标左边的部分（左半部分）和位于鼠标右边的部分（右半部分），左半部分的`uv.x`是小于`progress`的，`step`函数会返回 0，混合程度则返回 1，表示第二张图片；右半部分的`uv.x`是大于`progress`的，`step`函数会返回 1，混合程度则返回 0，表示第一张图片。

### 遮罩转场

接下来，我们来换一种形式。在`progress`变量的下方新增一个`ratio`变量，表示画布的比例。

```glsl
float ratio=iResolution.x/iResolution.y;
```

把上节课“`UV`绘画基础”中圆形的`SDF`函数复制过来。

```glsl
float sdCircle(vec2 p,float r)
{
    return length(p)-r;
}
```

在转场函数中，用我们在上一章所学的知识画一个圆形的遮罩，半径取`progress`，不够大的话乘上一个数（如根号 2），然后将这个遮罩的取反（`1.-c`）作为混合程度。

```glsl
vec4 transition(vec2 uv){
    float progress=iMouse.x/iResolution.x;
    float ratio=iResolution.x/iResolution.y;

    vec2 p=uv;
    p-=.5;
    p.x*=ratio;
    float d=sdCircle(p,progress*sqrt(2.));
    float c=smoothstep(0.,.02,d);
    return mix(getFromColor(uv),getToColor(uv),1.-c);
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e8f708f3586a4d5884fea7f4403845b1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=415170&e=jpg&b=82b5e8)

我们能看到图片上出现了一个圆形遮罩，可以调整`smoothstep`函数的参数，使得遮罩看上去更加“朦胧”。

```glsl
float c=smoothstep(0.,.2,d);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/567bba4472bb446f91332bb82b6a329d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=422604&e=jpg&b=878469)

```glsl

```

### 置换转场

我们还可以用纹理本身来扭曲纹理的`UV`坐标，这样的操作称为“置换”（`displacement`），用来扭曲的纹理称为“置换纹理”。

先要找到一张“置换纹理”，我通常是在网站 [3d textures](https://3dtextures.me/) 上找的，这里选用了一张[沙砾的纹理](https://3dtextures.me/2022/04/27/gravel-001/)，在`Shader`的上方引入它。

```glsl
#iChannel2 "https://s2.loli.net/2023/07/17/3GDlwcvehqQjTPH.jpg"
```

编写转场函数`transition`。

```glsl
vec4 transition(vec2 uv){
    float progress=iMouse.x/iResolution.x;
    float ratio=iResolution.x/iResolution.y;

    vec2 dispVec=texture(iChannel2,uv).xy;
    vec2 uv1=vec2(uv.x-dispVec.x*progress,uv.y);
    vec2 uv2=vec2(uv.x+dispVec.x*(1.-progress),uv.y);
    return mix(getFromColor(uv1),getToColor(uv2),progress);
}
```

这里我们不用管混合程度`progress`了，我们需要做的是用置换纹理去扭曲 2 张图片的`UV`。

将置换纹理采样成`dispVec`，取前 2 个维度`xy`。

对于第一张图片，以原`UV`为基础，创建新的`UV`变量`uv1`，在上一章中说过，往正方向（右边）移动的话并非加上，而是减去，因此我们给`UV`的`x`坐标减去`x`轴上的偏移量`dispVec.x`，并给它乘上`progress`，让它能随鼠标的位置变化而变化；第二张图片同理也是这么操作，只不过要把方向取反，并且鼠标位置也要取反，变成`1.-progress`。最后将`getFromColor`和`getToColor`的参数分别设置成我们的新`UV`坐标。

这样基本是可以了，再加一个`strength`变量吧，用于调节置换效果的强度，因为默认的强度可能会过于凸显置换纹理本身。

```glsl
float strength=.6;
vec2 uv1=vec2(uv.x-dispVec.x*progress*strength,uv.y);
vec2 uv2=vec2(uv.x+dispVec.x*(1.-progress)*strength,uv.y);
```

我们得到了一个酷酷的置换转场特效。

## 小结

简要地总结下本文的要点吧：

1. 纹理有多种引入`Shader`程序的方式，根据需要选择合适的方式就行。
2. 利用`GLSL`内置函数`texture`可以把纹理采样出来，显示到画面上。
3. 扭曲操作的本质是通过改变纹理的`UV`坐标来扭曲纹理的形状，计算扭曲的坐标时，常常会使用一些`sin`之类的数学函数。
4. [Graphtoy](https://graphtoy.com/) 能对`Shader`函数进行可视化的调试。
5. 转场的本质是从一张纹理转变为另一张纹理，利用`mix`函数将 2 张纹理混合起来，用`progress`（用户鼠标的位置）这个变量来控制转场的过程。可以绘制遮罩来定制转场的形状。
6. 我们也可以用纹理来扭曲纹理的`UV`坐标，实现一种特殊的置换转场效果。

纹理是`Shader`中比较重要的一个部分，也是很容易玩出花的地方。多玩玩，或许你会有意想不到的收获~
