到目前为止，我们已经学习了很多的`Shader`知识要点，是时候来做一些实战的项目了。

在之前的章节中，我们写的所有代码都是在单文件的`.glsl`或`.html`文件里的，这样能方便我们学习某个特定的知识点。

然而，真正的项目肯定不能这么写，如果这么写的话，会有很多弊端，如代码量庞大、难以复用、查找困难、变量冲突、无智能提示、无法自动补全等等。因此，我们需要以一种特定的方式来组织我们的项目。

## vite

[vite](https://github.com/vitejs/vite)，可以说是目前最流行的构建工具之一，想必各位前端肯定都不陌生。它能够让我们以模块化的形式来组织代码，并且还有自带热重载功能的开发服务器，编译速度极快，可以说是项目开发的不二之选。

把之前“顶点着色器”章节的“初入`3D`世界”部分的物体创建代码直接拷贝过来，然后把这些代码里的`this`替换成`this.base`。

```js
export default class TestObject extends kokomi.Component {
  constructor(base) {
    super(base);

    const geometry = new THREE.SphereGeometry(2, 64, 64);
    // const geometry = new THREE.PlaneGeometry(4, 4);
    const material = new THREE.ShaderMaterial({
      vertexShader: /* glsl */ `...`,
      fragmentShader: /* glsl */ `...`,
    });
    const mesh = new THREE.Mesh(geometry, material);
    this.base.scene.add(mesh);

    const uj = new kokomi.UniformInjector(this.base);
    material.uniforms = {
      ...material.uniforms,
      ...uj.shadertoyUniforms,
    };
    this.base.update(() => {
      uj.injectShadertoyUniforms(material.uniforms);
    });
  }
}
```

`kokomi.Component`自带 2 个方法：`addExisting`和`update`，分别负责创建和更新物体，我们把代码的某些部分分别移动到这 2 个方法内（同时把某些对象也挂载到实例上），把`addExisting`内的`this.base.scene`改成`this.container`。

```js
export default class TestObject extends kokomi.Component {
  constructor(base) {
    super(base);

    const geometry = new THREE.SphereGeometry(2, 64, 64);
    // const geometry = new THREE.PlaneGeometry(4, 4);
    const material = new THREE.ShaderMaterial({
      vertexShader: /* glsl */ `...`,
      fragmentShader: /* glsl */ `...`,
    });
    this.material = material;
    const mesh = new THREE.Mesh(geometry, material);
    this.mesh = mesh;

    const uj = new kokomi.UniformInjector(this.base);
    this.uj = uj;
    material.uniforms = {
      ...material.uniforms,
      ...uj.shadertoyUniforms,
    };
  }
  addExisting() {
    this.container.add(this.mesh);
  }
  update() {
    this.uj.injectShadertoyUniforms(this.material.uniforms);
  }
}
```

### Shader

目前，我们的`Shader`是作为字符串直接写在`ShaderMaterial`的参数内的，为了项目的组织，也有必要把它们写进单独的`.glsl`文件内。

[vite](https://github.com/vitejs/vite) 暂不支持直接读取`.glsl`文件，需要借助一个额外的插件 [vite-plugin-glsl](https://github.com/UstymUkhman/vite-plugin-glsl) 才能读取，先安装它。

```sh
npm i -D vite-plugin-glsl
```

在`vite.config.js`中引入这个插件。

```js
import { defineConfig } from "vite";

import glsl from "vite-plugin-glsl";

export default defineConfig({
  server: {
    open: true,
  },
  plugins: [glsl()],
});
```

在`Experience`目录下新建`Shaders`目录，在`Shaders`目录下新建`TestObject`目录，在`TestObject`目录下新建 2 个`Shader`文件：顶点着色器`vert.glsl`和片元着色器`frag.glsl`。

```sh
mkdir src/Experience/Shaders
mkdir src/Experience/Shaders/TestObject
touch src/Experience/Shaders/TestObject/vert.glsl
touch src/Experience/Shaders/TestObject/frag.glsl
```

把`TestObject.js`里的 2 个着色器字符串分别复制进那 2 个`Shader`文件里，然后再把那 2 个文件的内容分别`import`过来，替换成`ShaderMaterial`的参数。

```js

import testObjectFragmentShader from "../Shaders/TestObject/frag.glsl";

export default class TestObject extends kokomi.Component {
  constructor(base) {

      vertexShader: testObjectVertexShader,

    ...
  }
  ...

```

### 调试

调试可以说是项目必不可少的一环。之前我们在写单文件`Shader`的时候，总是会对一些值进行各种微调，而且这些微调是纯粹靠手输，稍微有点繁琐，那能不能做成拖拽式的`GUI`呢？

当然可以，你需要安装一个调试专用的库——[lil-gui](https://github.com/georgealways/lil-gui)。

```sh
npm i lil-gui
```

在`Experience`目录下，新建`Debug.js`。

```sh
touch src/Experience/Debug.js
```

在`Debug.js`中，导出`Debug`类。

```js
export default class Debug {}
```

调试本身是可选的，因此我们可以用`#debug`这个路由哈希值来决定是否显示调试界面。

比如说你当前的`URL`地址是`http://127.0.0.1:5173/`，想要显示调试的话进入地址`http://127.0.0.1:5173/#debug`即可。

```js
export default class Debug {
  constructor() {
    this.active = window.location.hash === "#debug";
  }
}
```

引入 [lil-gui](https://github.com/georgealways/lil-gui)。

```js
import * as dat from "lil-gui";

export default class Debug {
  constructor() {
    this.active = window.location.hash === "#debug";

    if (this.active) {
      this.ui = new dat.GUI();
    }
  }
}
```

有的小伙伴可能会好奇引入时为什么会有个别名`dat`，那是因为`three.js`在早期使用的调试库名叫 [dat.gui](https://github.com/dataarts/dat.gui)，而它的引入方式就是`import * as dat from "dat.gui";`，现在的调试库则换成了 [lil-gui](https://github.com/georgealways/lil-gui)，为了保留这一传统就这么写了。

在`Experience.js`中，引入`Debug`类并初始化，把它放在偏前面的位置。

```js
    new kokomi.OrbitControls(this);

```

访问调试地址（在`URL`尾部加上`#debug`），如果看见了右上角的`GUI`，就说明调试引入成功了。

我们来给测试物体`TestObject`加上调试，先获取`debug`对象，如果它是`active`的，则创建一个调试组`testObject`。

```js
export default class TestObject extends kokomi.Component {
  constructor(base) {
    ...

    const debug = this.base.debug;
    if (debug.active) {
      const debugFolder = debug.ui.addFolder("testObject");
    }
  }
  ...
}
```

要调试的值通常都会存在一个对象里，我们在初始化`material`前创建一个`params`的对象来存放调试的值，创建一个`uDistort`的值，表明扭曲的程度，把调试值合并至`material.uniforms`里。

```js
export default class TestObject extends kokomi.Component {
  constructor(base) {
    super(base);

    const params = {
      uDistort: {
        value: 1,
      },
    };

    const geometry = new THREE.SphereGeometry(2, 64, 64);
    // const geometry = new THREE.PlaneGeometry(4, 4);
    const material = new THREE.ShaderMaterial({
      vertexShader: testObjectVertexShader,
      fragmentShader: testObjectFragmentShader,
    });
    this.material = material;
    const mesh = new THREE.Mesh(geometry, material);
    this.mesh = mesh;

    const uj = new kokomi.UniformInjector(this.base);
    this.uj = uj;
    material.uniforms = {
      ...material.uniforms,
      ...uj.shadertoyUniforms,
      ...params,
    };

    const debug = this.base.debug;
    if (debug.active) {
      const debugFolder = debug.ui.addFolder("testObject");
    }
  }
  ...
}
```

给`debugFolder`添加`uDistort`的值，并且设置好值的范围（这里是`[0,2]`），步长设为 0.01，属性名设为`distort`。

```js
debugFolder
  .add(params.uDistort, "value")
  .min(0)
  .max(2)
  .step(0.01)
  .name("distort");
```

### 素材管理

`kokomi.js`有一个`AssetManager`类，可以用于素材的统一管理。

先创建一个素材列表文件`resources.js`。

```sh
touch src/Experience/resources.js
```

我们可以在`resources.js`中配置好项目的所有素材。

在`public`目录下新建一个`textures`目录，用于存放纹理文件。

```sh
mkdir public/textures
```

这里提供了一张天空盒的纹理，[点击这里](https://s2.loli.net/2023/07/05/51vjRBsLCuiT2dW.png)把它下载到本地，然后放进`public/textures`目录下，命名为`skybox.png`。

在`resources.js`中引入这个纹理。

```js
export default [
  {
    name: "skybox",
    type: "texture",
    path: "textures/skybox.png",
  },
];
```

在`Experience.js`中，引入资源列表`resources`，实例化`AssetManager`，并给它传入资源列表。

```js
import * as kokomi from "kokomi.js";

import World from "./World/World";

import Debug from "./Debug";

import resources from "./resources";

export default class Experience extends kokomi.Base {
  constructor(sel = "#sketch") {
    super(sel);

    window.experience = this;

    this.debug = new Debug(this);

    this.am = new kokomi.AssetManager(this, resources);

    this.camera.position.set(0, 0, 5);
    new kokomi.OrbitControls(this);

    this.world = new World(this);
  }
}
```

在`World.js`中，我们要监听资源管理器的加载完成事件`ready`，触发后再通过`items`访问资源管理器的`skybox`资源，将它作为场景的天空盒。

```js
import * as kokomi from "kokomi.js";
import * as THREE from "three";

import TestObject from "./TestObject";

export default class World extends kokomi.Component {
  constructor(base) {
    super(base);

    this.base.am.on("ready", () => {
      const skybox = this.base.am.items["skybox"];
      skybox.mapping = THREE.EquirectangularReflectionMapping;
      this.base.scene.background = skybox;

      this.testObject = new TestObject(this.base);
      this.testObject.addExisting();
    });
  }
}
```

## 最后

恭喜你！完成了最基本的`Shader`项目模板的搭建！

从下一章开始，我们将正式进入项目的实战环节。

一个实战项目主要分为 3 个部分：`DOM`、`WebGL`和`Shader`。

`DOM`部分这点无需多言，就是大家平时打交道最多的页面。由于本小册的重点是`Shader`，这部分会非常简单地一笔带过。

`WebGL`部分对于`Shader`网站来说，是必不可少的。但是不用担心！`kokomi.js`已经将大部分的`WebGL`相关的`JS API`全都封装好了，这样我们就可以把重点放在`Shader`的编写上，`JS`部分仅仅是调用`API`，调整一些`uniform`的变量值，以及实现一些交互和缓动动画。

> 如果你之前研究过`WebGL`或`three.js`，也可以顺带看看 [kokomi.js](https://github.com/alphardex/kokomi.js) 是如何封装的。

`Shader`部分就是项目的重中之重了，你既可以按部就班地根据本小册的指引来一步步地完成项目，也可以自己充分发挥想象力，研究出别的玩法！

其实很多酷炫的效果，往往都是脑袋里那一瞬的灵感所带来的结晶。
