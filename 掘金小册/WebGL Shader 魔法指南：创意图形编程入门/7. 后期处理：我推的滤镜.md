**后期处理（`postprocessing`）**，是在图像渲染完成后，对它进行进一步的处理的过程。

这个词乍一听有点陌生，但是它的另一种说法你肯定知道——**滤镜**！

学过`CSS`的小伙伴应该知道`filter`这个属性，里面有各种各样的滤镜，包括模糊、亮度、对比度、灰度、反转等。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a85828416c24495bb65ca208deedaa57~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=491&s=287604&e=jpg&b=f9f3f2)

这些滤镜可以说是开箱即用，非常方便！但是，在`Shader`的世界中，并没有任何内置的滤镜函数可供我们使用，要想实现上面的那些滤镜效果，我们只能自己亲自动手去写出来。尽管这是个挑战，但它给我们的创造提供了极大的灵活性！我们可以用不着背一大堆已知函数的`API`，要什么效果自己写就完事了，而且`Shader`能实现的滤镜效果比`CSS`滤镜、`SVG`滤镜、`Canvas`滤镜等要多得多！

## RGB 位移

在学习这个滤镜前，我们先看下抖音的`logo`吧：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1d9af1d3aa54443a94f8678d97caf605~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=42812&e=jpg&b=000000)

抖音`logo`的中间是一个白色的音符，且白色音符的两侧有青色和红色的投影，这 2 个投影其实就是一种色调偏移效果的体现。

`RGB`位移滤镜也与之类似，它能将画面的 3 个颜色分量`R`、`G`、`B`进行一定的偏移，以产生一种色差的效果。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/11d33e61b5564b9986230aa878481b5a~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=54249&e=jpg&b=000000)

我们来尝试实现它吧。

开始编码前，我们先把原先的纹理替换成下面的纹理，它的色调相对较暗，会使得滤镜的效果更加明显一些。

```glsl
#iChannel0 "https://s2.loli.net/2023/09/10/63quVIA9xZLksDc.jpg"
```

让我们开始吧。将`UV`复制成 3 份，分别对应 `R`、`G`、`B` 3 个通道的`UV`。

```glsl
vec2 rUv=uv;
vec2 gUv=uv;
vec2 bUv=uv;
```

定义偏移量`offset`，对`R`通道和`B`通道分别应用不同大小（这里用了一正一负）的偏移量，`G`通道保持不变。

```glsl
float offset=.1;
rUv+=offset;
bUv-=offset;
```

用偏移过的`UV`采样 3 次纹理，最后的颜色分别取`UV`本身对应的通道值。

```glsl
vec4 rTex=texture(iChannel0,rUv);
vec4 gTex=texture(iChannel0,gUv);
vec4 bTex=texture(iChannel0,bUv);
vec4 col=vec4(rTex.r,gTex.g,bTex.b,gTex.a);
fragColor=col;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/76b1ac875dd84f269ca5b426d6794a00~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=811994&e=jpg&b=38351d)

偏移效果出现啦！还意外地挺酷？实际上这个偏移量太大了，将它调小一点吧。

```glsl
float offset=.0025;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cef2c0b8eac54f85a7f6821fdbcc28b8~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=684436&e=jpg&b=242321)

嗯，这样感觉好多了，也可以根据个人喜好进行进一步的调整。

除了一般的偏移外，我们还可以结合之前“[随机与噪声](https://juejin.cn/book/7267462574734573604/section/7267463265406418956)”一章中学到的随机函数，给偏移增加一些噪点。

把随机函数直接复制过来吧。

```glsl
highp float random(vec2 co)
{
    highp float a=12.9898;
    highp float b=78.233;
    highp float c=43758.5453;
    highp float dt=dot(co.xy,vec2(a,b));
    highp float sn=mod(dt,3.14);
    return fract(sin(sn)*c);
}
```

用随机函数制造噪点`noise`，再用`sin`函数和`cos`函数调整下偏移量位置，赋给`offset`。

```glsl
float noise=random(uv)*.5+.5;
vec2 offset=.05*vec2(cos(noise),sin(noise));
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/85d587195d564b079a726622608e26fa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=750758&e=jpg&b=282622)

我故意把噪点的值调大了点，这样就能看清楚那些噪点的分布了，然后把值调小。

```glsl
vec2 offset=.0025*vec2(cos(noise),sin(noise));
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d2de93540d65466685bf0076c8e63373~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=663610&e=jpg&b=242322)

这样我们就得到了一个微妙的、带噪点的`RGB`位移滤镜的效果。

## 膨胀

在`Photoshop`软件中，有一个叫“挤压”的滤镜。它的效果如下图所示：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/44bbe0262a3e42f1b4ec54e8200977f0~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=458&s=129474&e=jpg&b=3a3a3a)

我们可以设定一个数量值，使画面的中心根据这个值的大小进行一定的“凸起”变换。

当然，在`Shader`中，你也可以通过代码来做出类似的效果，比起“挤压”，我个人更偏向于用“膨胀”这个词来称呼这种效果，来尝试实现它吧。

创建膨胀函数`bulge`。

```glsl
vec2 bulge(vec2 p){
    return p;
}
```

在主函数`mainImage`中应用膨胀函数`bulge`。

```glsl
vec2 uv=fragCoord/iResolution.xy;
uv=bulge(uv);
vec3 tex=texture(iChannel0,uv).xyz;
fragColor=vec4(tex,1.);
```

接下来我们开始编写膨胀函数`bulge`。

用`length`函数计算`UV`上每个点到原点的距离，与`UV`相乘。

```glsl
float d=length(p);
p*=d;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/64b82e9e571346d6a060123f6f52f724~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=493670&e=jpg&b=817d64)

emmm，看上去不太对，这是因为`UV`的原点并不在画布中心，而是在画布左下角，我们需要对它进行居中操作。

定义一个中点，位置就取画布的正中心。

```glsl
vec2 center=vec2(.5);
```

在调用`length`函数前，我们先给`UV`减去这个中点，然后调用后再加回去。

```glsl
p-=center;

float d=length(p);
p*=d;

p+=center;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1a7eeead18e4d3882309f8be5b72753~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=343979&e=jpg&b=898369)

膨胀中心的位置对了。

我们还可以进一步地加强一下效果，对距离`d`应用`pow`函数，指数取 2。

```glsl
float dPow=pow(d,2.);
// p*=d;
p*=dPow;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/631273994bf743a589d484de7a01b37f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=269889&e=jpg&b=8a8369)

可以看到效果比之前更强烈了一些。

这种膨胀是外凸的，那么内凹的能实现吗？当然可以，我们需要给`UV`乘上距离的倒数。

```glsl
float dRev=1./dPow;
p*=dRev;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/88477ec27f854cb89f39b3976140eef6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=623774&e=jpg&b=7a765d)

呜哇，居然变成了万花筒！

为什么会这样呢？很简单，`dPow`有可能是一个非常小的值（假设 0.0001），那么倒数的值就会变得非常大（0.0001 的倒数是 10000），导致了上面万花筒现象的出现。

那么如何解决呢？给底数加上一个数字，让它不至于非常小，这个数字就取 1 吧。

```glsl
float dRev=1./(dPow+1.);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a5f6b334cd4243cb80673d0f5a829ea5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=414662&e=jpg&b=898368)

这样我们就实现了内凹的效果。

为了更好地微调这个效果，我们可以增加 2 个额外的参数：扭曲半径`radius`和扭曲强度`strength`。

```glsl
vec2 bulge(vec2 p){
    vec2 center=vec2(.5);

    float radius=.9;
    float strength=1.1;

    p-=center;

    float d=length(p);
    d/=radius;
    float dPow=pow(d,2.);
    float dRev=strength/(dPow+1.);

    // p*=d;
    // p*=dPow;
    p*=dRev;

    p+=center;

    return p;
}
```

其实，我们还可以把膨胀的中心指定为用户的鼠标位置（鼠标位置要归一化）！

```glsl
// vec2 center=vec2(.5);
vec2 center=iMouse.xy/iResolution.xy;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/57e06372f7bd409baf9f5a6c787a27b3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=4959825&e=gif&f=118&b=807b61)

## 像素化

你知道“马赛克”吗？它能将图片、视频等媒体的特定区域变成一坨方块，起到遮挡敏感信息的作用。

在`Shader`中，马赛克就是“像素”，将画面马赛克化，也就等于像素化，来尝试实现它吧。

我们先来认识一个新的内置函数——`floor`，这个函数的作用是向下取整，比如`floor(114.514)`就等于`114.`。

直接输出`UV`的`x`坐标。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec2 uv=fragCoord/iResolution.xy;
    float c=uv.x;
    fragColor=vec4(vec3(c),1.);
}
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b45109cf38fc4a77845a5cde0c7349e1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=37295&e=jpg&b=7f7f7f)

将`x`坐标乘上像素化的大小（这里取 10），向下取整，再除以像素化的大小。

```glsl
c=floor(c*10.)/10.;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/517580dae8334dff94b983e7f896af22~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=9962&e=jpg&b=262626)

乍一看可能会有点不明所以，把公式给可视化一下吧。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f57f3484b1474aedae6be7d8def1820f~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=451&s=167028&e=jpg&b=292929)

当`x`坐标位于`(0.,0.09)`时，`x*10`的值域为`(0.,0.9)`，而`floor`函数是向下取整的，因此`floor(x*10)`的值是`0`，除以`10`后还是`0`；但当`x`坐标位于`0.1`时，`x*10`就变成了`1`，此时向下取整的结果就从`0`变成了`1`，除以`10`后就变成了`0.1`；后面范围的值会以步长`0.1`递增，公式总体是阶梯的形状。

理解公式的含义后，我们回到原先的纹理采样。

```glsl
vec3 tex=texture(iChannel0,uv).xyz;
fragColor=vec4(tex,1.);
```

在采样之前，我们定义好像素化的大小`size`（这里取 50x50），对`UV`的`xy`轴分别应用`floor`函数。

```glsl
vec2 size=vec2(50.,50.);
uv.x=floor(uv.x*size.x)/size.x;
uv.y=floor(uv.y*size.y)/size.y;
```

两行`floor`函数的代码可以简写成一行。

```glsl
uv=floor(uv*size)/size;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/14d814e244c9452f90085057c1656369~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=199939&e=jpg&b=827e64)

画面瞬间变成了一幅难以看懂的抽象艺术画。

## 晕影

之前在逛国外摄影作品网站 [unsplash](https://unsplash.com/) 时，经常能看到这样的照片：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1572ce490e574cc29ed6f6efa0e89a90~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=640&s=616983&e=jpg&b=ecdacb)

照片的四周有一层淡淡的阴影，感觉挺酷的，后来查了下这种现象，发现是摄影学的一个专业名词——“晕影”。

我们尝试用`Shader`来实现这种效果吧。

为了更方便地观察这种效果，我们将画面变为纯白色。

```glsl
vec3 col=vec3(1.);

fragColor=vec4(col,1.);
```

在`vec3 col=vec3(1.);`的下面一行开始编写代码。

创建一个圆形的径向渐变，这一步骤就是之前“[`UV`绘画基础](https://juejin.cn/book/7267462574734573604/section/7267463291306246155)”一章中画圆的其中一步。

由于效果要拉伸覆盖至整个画面，因此无需进行画布比例的修正。

```glsl
vec2 p=uv;
p-=.5;
float d=length(p);
col=vec3(d);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/98b2f14c45cc448aa988adbf74b8a8e4~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=76717&e=jpg&b=5e5e5e)

看到径向渐变后，注释掉`col=vec3(d);`。

用`smoothstep`函数来将这个渐变“拉伸”到画面的四个边界，而且 2 个边界值的差稍微大点，能营造一种模糊的效果。

```glsl
float c=smoothstep(.4,.8,d);
col=vec3(c);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ac4ab4161046406f93d370abcf908121~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=32830&e=jpg&b=0a0a0a)

接下来就是关键的一步了：把渐变给反转过来！这样我们就能得到四周的遮罩了。

反转渐变有 2 种方式：第一种是 1 减去遮罩`c`的值，第二种是直接调换`smoothstep`函数的 2 个边界值，出于个人偏好，我选了第 2 种，当然你选第 1 种也完全没问题，得到的结果是一样的。

```glsl
float c=smoothstep(.8,.4,d);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/de4f718157a04676a637158d3dd7d161~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=36303&e=jpg&b=f4f4f4)

看到了四周的阴影了吗，这就是晕影效果~

我们把之前的纹理重新弄回来，再把阴影的遮罩与其相乘叠加，就得到了最终的效果。

```glsl
vec3 col=tex;
vec2 p=uv;
p-=.5;
float d=length(p);
float c=smoothstep(.8,.4,d);
col*=c;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fce22c161dd74f1ebbc0b8dcadda26f5~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=412717&e=jpg&b=868266)

## 小结

我们来简单地总结下本文滤镜的实现方式吧。

1. 染色滤镜实现非常简单，将图像的颜色值与一个颜色相乘即可。
2. `RGB`位移滤镜要将 3 个通道对应的`UV`进行一定的偏移，再用它们分别采样 3 次图像，取对应的通道值组合成最终的颜色即可。
3. 膨胀滤镜实现本质上是扭曲`UV`，将`UV`上每个点到原点的距离与`UV`相乘，可以用`pow`函数来进一步地增强效果，效果默认是外凸，内凹则要乘上距离加一个固定值的倒数。
4. 像素化滤镜的本质是利用`floor`函数来将`UV`阶梯化。
5. 晕影滤镜的本质是一个四周阴影的遮罩，它可以通过用`smoothstep`函数来模糊化圆形径向渐变来实现。

不知道上面所实现的滤镜中，你推（喜欢）的滤镜是哪一款呢？

对我来说，只要能让图像变得更酷，都是我推的滤镜！
