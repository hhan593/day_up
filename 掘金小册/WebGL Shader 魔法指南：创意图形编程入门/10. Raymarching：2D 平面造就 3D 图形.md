前面说过这样一句话：“顶点着色器可以说是`Shader`里必不可少的一环，跟片元着色器一样，两者缺一不可。”

这句话本身并没有错：我们在`3D`世界创建`3D`的图形，在顶点着色器里计算它的光照、形状等要素，最后在片元着色器里计算它的颜色，将它显示到画面上。

然而，你有没有想过，如果将这些操作统统交给片元着色器来做，会是什么样子呢？

先来看个例子吧：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ba5f19e5bf7a4edebada4ec7656b2d0e~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=274240&e=jpg&b=928c94)

很明显，这张图显示的是一个`3D`的场景，里面充满了`3D`的图形。然而，令人意想不到的是，它实际上仅仅是一个`2D`的平面，它里面的所谓`3D`图形是完完全全用片元着色器创造出来的！不信？你可以去看[它的源码](https://www.shadertoy.com/view/Xds3zN)。不论是图形，还是光影，都跟顶点着色器没有丝毫的关系！

究竟是什么技术，能让我们在一个`2D`的平面内描绘出`3D`的世界呢？它就是——`Raymarching`！中文叫“光线步进”。

接下来，我们将来一同探索这个神奇的技术。

## 第一个 Raymarching 场景

先来看一张示意图吧：

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2db6400f1c3b45ffa8bc80903026ec62~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=450&s=99200&e=jpg&b=fefefe)

`Raymarching`大致的流程是这样的：为相机指定一个位置，将光线从相机的位置射出来，光线会不断地行进，每次行进时我们会检测它是否和场景中的物体发生碰撞，如果撞到了的话，就对撞到的物体进行进一步的着色处理（包括赋予颜色、计算光照等步骤），反之没撞到的话，就让光线继续行进，直到整个流程结束为止。

### UV 居中

我们第一件要做的事，就是对`UV`进行居中处理（步骤在[《UV：大触级的绘图高手》一章](https://juejin.cn/book/7267462574734573604/section/7267463291306246155)中说明过），这样会更方便我们接下去的绘画。

```glsl
uv=(uv-.5)*2.;
uv.x*=iResolution.x/iResolution.y;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/86993ebd04464df0b783f1ccb05a1ccf~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=72523&e=jpg&b=f05800)

### 创建光线

创建一束光线，光线的起点`ro`设为`(0.,0.,1.)`，也就是我们相机的位置。

```glsl
vec3 ro=vec3(0.,0.,1.);
```

计算光线的方向向量`rd`，也就是光线行进时所朝向的方向。

```glsl
vec3 rd=normalize(vec3(uv,0.)-ro);
```

### SDF 函数

在[《UV：大触级的绘图高手》](https://juejin.cn/book/7267462574734573604/section/7267463291306246155)一章中，我们已经学习过了这个概念，但是，我们之前学习的`SDF`是`2D`的，而`Raymarching`的世界是一个`3D`的世界，因此我们要把`SDF`函数也变成`3D`的。

还记得圆的`SDF`函数吗？它的图像是这样的：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/61a40fe608b24cb6afe33c39a6cbb634~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=398476&e=jpg&b=ca8743)

下面是它的函数实现：

```glsl
float sdCircle(vec2 p,float r)
{
    return length(p)-r;
}
```

可以看到`2D`的`SDF`函数接收的是一个二维的位置变量`p`，那如何把它变成`3D`的函数呢？很简单，只要把二维的位置变量`p`改成三维的就行了。

> 注意：这种转换仅仅针对圆形有效，其他形状的`SDF`则需要用另外的公式，所有`3D`的`SDF`公式可查阅[这篇文章](https://iquilezles.org/articles/distfunctions/)。

```glsl
float sdSphere(vec3 p,float r)
{
    return length(p)-r;
}
```

这样我们就得到了一个`3D`的球体`SDF`函数。

定义一个`map`函数，用于存放`SDF`函数的结果。

在`map`函数中，用球体的`SDF`函数创建一个球体`d`，位置设定为`(0.,0.,-2.)`，半径设定为 1。

```glsl
float map(vec3 p){
    float d=sdSphere(p-vec3(0.,0.,-2.),1.);
    return d;
}
```

### 光线步进

光线未行进时，渲染结果里什么都没有，输出颜色`col`默认是黑色。

```glsl
vec3 col=vec3(0.);
```

设定行进距离`depth`为 0，创建一个`for`循环，让光线从相机的位置射出来，不断向前步进，当前前进的位置`p`等于原点位置加上光线方向向量与行进距离的乘积，前进的同时计算`SDF`函数输出的距离`d`，并累加到行进距离。当输出距离`d`无限趋向于 0 时，我们可以判断光线已经碰撞到了物体，此时终止光线的行进，并给光线撞到的物体赋予颜色（这里用了白色）。

```glsl
float depth=0.;
for(int i=0;i<64;i++){
    vec3 p=ro+rd*depth;
    float d=map(p);
    depth+=d;
    if(d<.01){
        col=vec3(1.);
        break;
    }
}

fragColor=vec4(col,1.);
```

![](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/64ddb5a1438640999acaf204d796bae1~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=10268&e=jpg&b=000000)

此时画面上会出现一个白色的圆形，其实它就是一个球体，只不过没有光照，看上去就像一个普通的圆。

### 光照

在[《光照模型》](https://juejin.cn/book/7267462574734573604/section/7267463407082291241)一章中，我们学习了如何给一个`3D`场景添加光照，而`Raymarching`场景也可以用同样的方法来进行，只不过稍微有些变化，因为我们放弃了顶点着色器（这里严谨点来说就是只用顶点着色器来固定了覆盖整个屏幕的平面的位置），自然也就没有了法向量这个变量，但是，在`Raymarching`中，我们可以主动把法向量给算出来，`iq`博客中的一篇文章给出了[法向量的计算公式](https://iquilezles.org/articles/normalsSDF/)，可以直接搬过来用（把`iq`文章原函数里的`f`函数替换成`map`函数）。

```glsl
vec3 calcNormal(in vec3 p)
{
    const float h=.0001;
    const vec2 k=vec2(1,-1);
    return normalize(k.xyy*map(p+k.xyy*h)+
    k.yyx*map(p+k.yyx*h)+
    k.yxy*map(p+k.yxy*h)+
    k.xxx*map(p+k.xxx*h));
}
```

在光线碰撞后计算出`normal`，注意这里没光照时要把颜色设为黑色。

```glsl
if(d<.01){
    // col=vec3(1.);
    col=vec3(0.);
    vec3 normal=calcNormal(p);

    break;
}
```

然后把[《光照模型》](https://juejin.cn/book/7267462574734573604/section/7267463407082291241)一章的“`Blinn-Phong`光照模型”部分的代码直接搬过来，再稍微修改一下部分参数。

```glsl
if(d<.01){
    // col=vec3(1.);
    col=vec3(0.);
    vec3 normal=calcNormal(p);

    vec3 objectColor=vec3(1.);
    vec3 lightColor=vec3(.875,.286,.333);

    // ambient
    float ambIntensity=.2;
    vec3 ambient=lightColor*ambIntensity;
    col+=ambient*objectColor;

    // diffuse
    vec3 lightPos=vec3(10.,10.,10.);
    vec3 lightDir=normalize(lightPos-p);
    float diff=dot(normal,lightDir);
    diff=max(diff,0.);
    vec3 diffuse=lightColor*diff;
    col+=diffuse*objectColor;

    // specular
    vec3 reflectDir=reflect(-lightDir,normal);
    vec3 viewDir=normalize(ro-p);
    vec3 halfVec=normalize(lightDir+viewDir);
    float spec=dot(normal,halfVec);
    spec=max(spec,0.);
    float shininess=32.;
    spec=pow(spec,shininess);
    vec3 specular=lightColor*spec;
    col+=specular*objectColor;

    break;
}
```

可以看到物体成功地受到了光照的影响，变成了一个实实在在的球体。

这样，我们就完成了我们的第一个`Raymarching`场景！

## 图形混合

还记得我们之前在[《UV》](https://juejin.cn/book/7267462574734573604/section/7267463291306246155)一章中学到的“`SDF`布尔运算”吗？

一个好消息：这些布尔运算不论是`2D`还是`3D`都是通用的！

我们把它们统统`CV`过来吧，具体代码就不贴了。

在`map`函数中，创建一个球体，比之前的稍微小一点。

```glsl

```

用平滑并函数`opSmoothUnion`来混合这 2 个球体。


可以看到图形整体变成了一个柚子的形状！


其实呢，我们可以把这种方式看作是一种新型的“建模”方法。

传统的建模，是用三角形网格来进行建模的。用`Raymarching`建模，相比之下有这些优势：

1.  你可以利用数学公式来创建复杂的图形，而不会受三角形网格的限制。
2.  `Raymarching`建模存储的是`SDF`函数的公式，而传统建模模型大小存储的是全部顶点的数据，公式的大小肯定比顶点数据的要小很多。

## 图形材质

此时，我们的`map`函数存储了`SDF`函数的距离。尽管这样能完成建模，但是物体的材质还暂时无法区分开来。

为了说明这个问题，我们来新增一个平面吧。

先查阅`iq`的[这篇文章](https://iquilezles.org/articles/distfunctions/)，从中可以找到平面的`SDF`函数。

```glsl
float sdPlane(vec3 p,vec3 n,float h)
{
    return dot(p,n)+h;
}
```

在`map`函数中创建平面`d3`。

```glsl
float d3=sdPlane(p-vec3(0.,-1.,0.),vec3(0.,1.,0.),.1);
d=opUnion(d,d3);
```

可以看到中间的部分有些怪，那是因为光线行进的还不够多，有的地方并没有被检测出来，我们可以增加光线步进的次数。


![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bc7a68a5fa4c45c9af2a484e9156ec44~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=960&h=540&s=74937&e=jpg&b=000000)

尽管可能还是有一丁点弯曲，但总体看上去已经可以了。

那么问题来了：地面和球体用的是同一个材质，那如果我要把它们的材质给区分开来呢？比如说把地面改成白色的材质。

我的做法是引入“材质编号”这个变量。

先在顶部`CV`一个二维的`opUnion`函数。

```glsl
vec2 opUnion(vec2 d1,vec2 d2)
{
    return(d1.x<d2.x)?d1:d2;
}
```

把`map`函数改造成二维的，第一个维度代表距离，第二个维度代表材质的编号。

给球体赋予材质编号 1，给地面赋予材质编号 2。

```glsl
vec2 map(vec3 p){
    vec2 d=vec2(1e10,0.);

    float d1=sdSphere(p-vec3(0.,0.,-2.),1.);

    float d2=sdSphere(p-vec3(0.,.8+abs(sin(iTime)),-2),.5);
    d1=opSmoothUnion(d1,d2,.5);
    d=opUnion(d,vec2(d1,1.));

    float d3=sdPlane(p-vec3(0.,-1.,0.),vec3(0.,1.,0.),.1);
    d=opUnion(d,vec2(d3,2.));

    return d;
}
```

`calcNormal`之前是直接获得`map`函数的距离的，而现在`map`函数的距离变成了第一维度，因此我们要取它的`x`维度来获取它的距离。

```glsl
vec3 calcNormal(in vec3 p)
{
    const float h=.0001;
    const vec2 k=vec2(1,-1);
    return normalize(k.xyy*map(p+k.xyy*h).x+
    k.yyx*map(p+k.yyx*h).x+
    k.yxy*map(p+k.yxy*h).x+
    k.xxx*map(p+k.xxx*h).x);
}
```

`mainImage`的`Raymarching`循环中，原先是直接获取`map`的距离`d`的，这次我们将`map`的结果存进`res`变量里，分别将距离和材质编号存入`d`和`m`变量。

```glsl
for(int i=0;i<128;i++){
    ...
    // float d=map(p);
    vec2 res=map(p);
    float d=res.x;
    float m=res.y;
    depth+=d;
    ...
}
```

有了材质编号`m`，我们便可以区分材质了。

上面我们将地面的材质编号设为了 2，在这里我们就可以用`if`条件分支来处理材质编号 2 的情况，将光照颜色设为白色。

```glsl
if(d<.01){
    // col=vec3(1.);
    col=vec3(0.);
    vec3 normal=calcNormal(p);

    vec3 objectColor=vec3(1.);
    vec3 lightColor=vec3(.875,.286,.333);

    // material
    if(m==2.){
        lightColor=vec3(1.);
    }

    // ambient
    ...

    break;
}
```

## 阴影

有没有感觉场景上好像少了一样东西？没错，是阴影！

在`Raymarching`中，阴影跟法向量`normal`一样，也是靠计算得出来的。而且这个公式在`iq`的[博客文章](https://iquilezles.org/articles/rmshadows/)上也有提供，我们直接`CV`过来吧。（注意函数里的`map`函数调用结果要取第一维度`x`。）

```glsl
float softshadow(in vec3 ro,in vec3 rd,float mint,float maxt,float k)
{
    float res=1.;
    float t=mint;
    for(int i=0;i<256&&t<maxt;i++)
    {
        float h=map(ro+rd*t).x;
        if(h<.001)
        return 0.;
        res=min(res,k*h/t);
        t+=h;
    }
    return res;
}
```

在漫反射光照部分，我们用`softshadow`函数计算出阴影`shadow`，并将它与下面的光照颜色相乘。

```glsl
vec3 diffuse=lightColor*diff;
float shadow=softshadow(p,lightDir,.01,10.,16.);
col+=diffuse*objectColor*shadow;
```

## 抗锯齿

```glsl
        ...
        if(d<.01){
            ...
            break;
        }

    return col;
}

void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec3 tot=vec3(0.);
    tot+=getSceneColor(fragCoord);
    fragColor=vec4(tot,1.);
}
```

在主函数内，我们要多次渲染场景，并且每次渲染的`UV`坐标带上一定的偏移量，这样就能起到抗锯齿的效果。

```glsl
void mainImage(out vec4 fragColor,in vec2 fragCoord){
    vec3 tot=vec3(0.);

    float AA_size=2.;
    float count=0.;
    for(float aaY=0.;aaY<AA_size;aaY++)
    {
        for(float aaX=0.;aaX<AA_size;aaX++)
        {
            tot+=getSceneColor(fragCoord+vec2(aaX,aaY)/AA_size);
            count+=1.;
        }
    }
    tot/=count;

    fragColor=vec4(tot,1.);
}
```

## 小结

我们来总结下本文的要点吧：

1.  `Raymarching`技术能在一个`2D`的平面内创造`3D`的世界。
2.  创建一个`Raymarching`场景的主要步骤：`UV`居中、光线创建、光线方向计算、创建`SDF`函数、光线步进、碰撞检测、计算法向量、应用光照。
3.  利用`SDF`布尔运算函数能实现图形混合的效果，同时也是一种新型的建模方式。
4.  给`map`函数新增另一个维度——材质编号，即可将不同物体的材质给区分开来。
5.  阴影能通过`softshadow`函数计算得出。
6.  抗锯齿的实现方式是将场景进行多次渲染，并且每次渲染要偏移`UV`坐标。

其实不止开头的那个例子，`Shadertoy`上的大部分酷炫`demo`都是用`Raymarching`技术做出来的。多多探索，或许你会有新的发现\~
