## 网格

我们来创建第一个`3D`物体吧，在`three.js`中，`3D`物体被称作网格`Mesh`，而网格是由两部分组成的：

1.  几何体`geometry`：定义了`3D`物体的形状；
1.  材质`material`：定义了`3D`物体的外观。

试着创建一个白色的球体吧。

```js
const geometry = new THREE.SphereGeometry(2, 64, 64);
const material = new THREE.MeshBasicMaterial({
  color: "#ffffff",
});
const mesh = new THREE.Mesh(geometry, material);
this.scene.add(mesh);
```

[SphereGeometry](https://threejs.org/docs/#api/en/geometries/SphereGeometry) 指定了几何体为球体，半径为 2，宽高的细分皆为 64；[MeshBasicMaterial](https://threejs.org/docs/#api/en/materials/MeshBasicMaterial) 制定了材质为基础材质（没有光照，只有颜色或纹理的材质），颜色为白色。效果如下：

说起`three.js`，它是一个`WebGL`封装程度非常高的框架。像我们刚刚用到的基础材质，包括它所有的内置材质，都是用`Shader`来实现的！如果之前你学过`three.js`，你就已经或多或少地用到了很多`Shader`！

尽管`three.js`内置材质本身非常好用，但也是有局限性的，如果你想实现一些自定义的效果，只用内置材质怕是难以实现。那么，有没有这样一种材质，能使你完完全全地自定义`Shader`呢？当然有！

## 自定义 Shader

**[ShaderMaterial](https://threejs.org/docs/#api/en/materials/ShaderMaterial) 使你能够自定义顶点着色器和片元着色器。**

将刚刚场景内的球体的材质替换为`ShaderMaterial`。

```js
const material = new THREE.ShaderMaterial();
```

将以下的`Shader`代码直接拷过来。

```js
const material = new THREE.ShaderMaterial({
  vertexShader: /* glsl */ `
void main(){
    vec4 modelPosition=modelMatrix*vec4(position,1.);
    vec4 viewPosition=viewMatrix*modelPosition;
    vec4 projectedPosition=projectionMatrix*viewPosition;
    gl_Position=projectedPosition;
}
  `,
  fragmentShader: /* glsl */ `
void main(){
    vec3 color=vec3(1.,0.,0.);
    gl_FragColor=vec4(color,1.);
}
  `,
});
```

效果如下：

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1b1f344d086744b78d0367d7ca55994d~tplv-k3u1fbpfcp-zoom-1.image)

`fragmentShader`是**片元着色器**，里面的代码看上去非常熟悉：就是你曾经写过的第一个`Shader`代码。（略微的差别是`mainImage`变成了`main`且参数消失了，`fragColor`变成了`gl_FragColor`。）

`vertexShader`是**顶点着色器**，是个新面孔，但同时也是这一章的主角。尽管它的代码稍微有点长，但表达的意思很简单：一个叫`position`的变量经过了 3 个矩阵的变换，被赋给了`gl_Position`，这 3 个矩阵被称作`MVP`矩阵，分别代表**模型、视图、投影**，它们的具体含义有一丁点复杂，这里就不详述了，实在想了解的话可以参考[这篇文章](https://zhuanlan.zhihu.com/p/343532009)。

其实，这 3 个矩阵的其中 2 个矩阵（模型和视图）可以合并为一个矩阵`modelViewMatrix`，这样我们可以把顶点着色器的代码给简化成仅仅一行。

```glsl
void main(){
    gl_Position=projectionMatrix*modelViewMatrix*vec4(position,1.);
}
```

`MVP`矩阵变换这个操作基本都是固定的，我们真正需要注意的是变量`position`，对它我们可以进行各种操作，将它单独作为一个变量`p`提取出来吧。

```glsl
void main(){
    vec3 p=position;
    gl_Position=projectionMatrix*modelViewMatrix*vec4(p,1.);
}
```

### varying

还记得我们之前的`UV`伙伴吗？它去哪了呢？其实，它就在顶点着色器里，变量名叫`uv`。然而，它是一个`attribute`变量，并不是`uniform`，这意味着它只存在于顶点着色器，而不在片元着色器内。那么有没有办法把它从一个着色器传递给另一个着色器呢？

在[《GLSL：Shader 语言的行动纲领》章节](https://juejin.cn/book/7267462574734573604/section/7267462924409520182)中，我们提到过“变量限定符”这个概念，而其中有一种能够在 2 个着色器之间传递变量，它就是`varying`。

先在顶点着色器顶部声明一个`varying`变量`vUv`。

```glsl
varying vec2 vUv;
```

在顶点着色器的`main`函数内部的末尾，将变量`uv`赋给`vUv`。

```glsl
void main(){
    ...

    vUv=uv;
}
```

在片元着色器中的顶部也声明`vUv`。

```glsl
varying vec2 vUv;
```

在片元着色器的`main`函数内，将`vUv`赋给`uv`变量，并直接输出颜色。

```glsl
void main(){
    vec2 uv=vUv;
    gl_FragColor=vec4(uv,0.,1.);
}
```

我们得到了一个绿油油的、底部又有点微微发红的球体。

光球体可能不怎么能看出来，为了验证我们是否得到了`UV`坐标，我们把几何体临时改成平面 [PlaneGeometry](https://threejs.org/docs/#api/en/geometries/PlaneGeometry)。

```js
// const geometry = new THREE.SphereGeometry(2, 64, 64);
const geometry = new THREE.PlaneGeometry(4, 4);
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3099de5382dd4779bc45d607a260b857~tplv-k3u1fbpfcp-zoom-1.image)

验证成功！我们确实得到了`UV`坐标。其实那个绿油油的球体可以看作是一个由`UV`包裹而成的球体，我们把球体给换回来吧。

### uniform

有的小伙伴可能会问了：之前章节用到的那些`uniform`变量（如`iTime`）还在吗？

答案当然是：没啦！因为之前我们是在`Shadertoy`环境内编写`Shader`的，而现在是在`three.js`环境下，默认是没有`Shadertoy`的那些`uniform`变量的，那怎么办呢？

放心，好在`kokomi.js`提供了这样一个类——`UniformInjector`，用它我们就能把`Shadertoy`的那些变量全部注入到`ShaderMaterial`的`uniform`里！

```js
const uj = new kokomi.UniformInjector(this);
material.uniforms = {
  ...material.uniforms,
  ...uj.shadertoyUniforms,
};
this.update(() => {
  uj.injectShadertoyUniforms(material.uniforms);
});
```

实例化一个`UniformInjector`对象，将`uj.shadertoyUniforms`并入`material.uniforms`，然后在动画循环`this.update`中，调用它的`injectShadertoyUniforms`方法，即可完成注入。还有值得注意的一点是，之前`Shadertoy`环境下我们没有显示声明那些`uniform`变量，但这次是要的，在 2 个着色器的最上方都加上`uniform`变量的声明吧。

```glsl
uniform float iTime;
uniform vec3 iResolution;
uniform vec4 iMouse;
```

## 顶点扭曲

对于顶点，我能想到的第一个操作自然是**扭曲——通过改变顶点的位置`position`来改变整个图形的形状**。

在顶点着色器中，创建一个扭曲函数`distort`。

```glsl
vec3 distort(vec3 p){
    return p;
}
```

在主函数`main`中调用它。

```glsl
void main(){
    vec3 p=position;
    p=distort(p);
    gl_Position=projectionMatrix*modelViewMatrix*vec4(p,1.);
}
```

在`distort`函数中，用`sin`函数来扭曲顶点位置`p`。

```glsl
p.x+=sin(p.y*10.+iTime)*.25;
```

这形状是不是有点像旋风土豆（一种有名的街头小吃）？

除了用一般的函数进行扭曲外，我们还可以用之前学到的“噪声”来扭曲它。

噪声函数的代码`cnoise`可以直接从“[随机与噪声](https://juejin.cn/book/7267462574734573604/section/7267463265406418956)”那章复制过来。这里的偏移量除了噪声本身外还乘了`normal`，`normal`表示的是法向量，乘上它后能使噪声在球体的所有方向都发挥作用。

## 粒子

除了网格`Mesh`外，`three.js`还支持另一种物体——粒子。

**网格，是由三角形组成的一个实体；而粒子，则是由许多微粒组成的一个较分散的结构。**

先回到最初的球体吧。

[jcode](https://code.juejin.cn/pen/7259621142494642191)

创建粒子，只需把之前的`Mesh`改为`Points`即可。

```js
const points = new THREE.Points(geometry, material);
this.scene.add(points);
```

粒子确实是创建出来了，但画面上依旧空空如也，因为我们还没有给它设定大小，在顶点着色器中，可以通过`gl_PointSize`变量来设置粒子的大小。

```glsl
gl_PointSize=10.;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0bb70d890fe143af92b92a286034591a~tplv-k3u1fbpfcp-zoom-1.image)

现在能看见粒子了，但是它默认的样子并不好看，我们可以根据已学过的知识（“[`UV`绘画基础](https://juejin.cn/book/7267462574734573604/section/7267463291306246155)”一章中的“发光效果”）来制作发光的粒子。

```glsl
void main(){
    vec2 uv=gl_PointCoord;
    uv=(uv-.5)*2.;

    float d=length(uv);
    float c=.05/d;
    c=pow(c,2.);
    gl_FragColor=vec4(vec3(1.),c);
}
```

注意，这里跟之前的步骤有点区别：

1.  `uv`从原来的`fragCoord/iResolution.xy`变成了`gl_PointCoord`，这个是`Points`专有的位置变量。
2.  去除了比例修正的一行代码`uv.x*=iResolution.x/iResolution.y;`，因为`Points`长宽是相等的，默认不用修正比例。
3.  一些数值进行了微调。
4.  最后得到的遮罩结果`c`放在了第四维度——透明度，前三维度变成了粒子的颜色。因为`3D`世界是可以受透明度影响的，将颜色和透明度分开，能防止因背景色改变而造成的视觉冲突。（你肯定不希望在白色的背景里看到黑色背景的粒子吧！）

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b4144afa9a224acb80de7b84fb28373e~tplv-k3u1fbpfcp-zoom-1.image)

emmm，好像还是哪里不太对，那是因为`three.js`的材质默认是不透明的，我们刚刚设置的透明值是无效的，为了让透明值有效化，我们要手动在材质中开启透明`transparent`。

```js
const material = new THREE.ShaderMaterial({
  vertexShader: /* glsl */ `...`,
  fragmentShader: /* glsl */ `...`,
  transparent: true,
});
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f682300c00f04c1c9096394c16688e56~tplv-k3u1fbpfcp-zoom-1.image)

透明度有了，但整体还是显得比较暗淡，再给材质加 2 个参数吧。

```js
const material = new THREE.ShaderMaterial({
  vertexShader: /* glsl */ `...`,
  fragmentShader: /* glsl */ `...`,
  transparent: true,
  blending: THREE.AdditiveBlending,
  depthWrite: false,
});
```

我们将材质的混合模式`blending`设为`AdditiveBlending`模式，并且关闭深度写入`depthWrite`，这个操作可以说是粒子特效固定的一种做法。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b0cd25c7ffe54b738b843ea5c7eb442a~tplv-k3u1fbpfcp-zoom-1.image)

开始有模有样了，只是这粒子还是太小，将`gl_PointSize`调大一点。

```glsl
gl_PointSize=20.;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f4faa881620649e7bc8a20809c719166~tplv-k3u1fbpfcp-zoom-1.image)

看着不错，但这里还有个小问题，把相机拉近一些后，我们就看不到近处的粒子了（仔细看还是能看见，就是特别小），因为尽管视角变了，粒子的大小依然是个固定值，并没有随着变化。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c6593e5bf3f443b0a860cdb6690563e7~tplv-k3u1fbpfcp-zoom-1.image)

理想的效果应该是靠近相机的粒子显得更大，而远离相机的粒子显得更小，为了实现这一点，我们需要给`gl_PointSize`来一个小小的矩阵变换操作，[这个操作](https://ycw.github.io/three-shaderlib-skim/dist/#/latest/points/vertex)在`three.js`封装的`Shader`里有写，我们可以稍作修改直接拿过来用。

```glsl
gl_PointSize=20.;
vec4 mvPosition=modelViewMatrix*vec4(p,1.);
gl_PointSize*=(1./-mvPosition.z);
```

改完后会发现粒子还是太小，继续调大些。

```glsl
gl_PointSize=50.;
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ee07bd1b5c040ae82d87fb8ed059522~tplv-k3u1fbpfcp-zoom-1.image)

把相机拉近后，我们就能看到近处的粒子了。

目前还有个小问题：粒子的大小是跟屏幕的像素比有关的，这会导致我们在不同尺寸的屏幕上看到粒子大小会不同。

为了抹平这一差异，我们需要获取像素比，作为一个`uniform`变量传进`Shader`内。

```js
const material = new THREE.ShaderMaterial({
  ...,
  uniforms: {
    uPixelRatio: {
      value: this.renderer.getPixelRatio(),
    },
  },
});
```

在顶点着色器中，顶部声明像素比的`uniform`变量，并将它与微粒大小相乘即可。

```glsl
uniform float uPixelRatio;

void main(){
    ...
    gl_PointSize=50.*uPixelRatio;
    ...
}
```

最后我们把相机移到合适的位置，是不是还挺酷的？

## 自定义形状

目前我们的粒子所形成的几何体是球体，那么能否把它改成自定义的形状呢？

还记得顶点着色器里的`position`变量吗？它其实是一个`attribute`变量。

`attribute`变量，是一个用于存储顶点信息的变量，顶点信息主要包含位置、颜色等。

假设我们要声明一个位置的`attribute`变量，我们可以这么写：

```glsl
attribute vec3 position;
```

不过呢，`three.js`的`ShaderMaterial`已经内置了这个声明，因此你不必再多写一次，但是，当你传入了其他的`attribute`变量时，你就有必要去声明一下它们。

回归正题，如果我们想要一个自定义的形状，那么也就意味着我们要创建一个新的几何体，并且要改变这个几何体的位置信息，也就是改变`position`这个`attribute`变量，那具体怎么做呢？

在`three.js`中，我们可以利用 [BufferGeometry](https://threejs.org/docs/index.html#api/en/core/BufferGeometry) 来创建自定义的几何体。

```js
const geometry = new THREE.BufferGeometry();
```

构造一个用于存储位置信息的`Array`数组，里面填充随机的数据。

```js
const count = 100;
let positions = Array.from({ length: count }, () =>
  [1, 1, 1].map(THREE.MathUtils.randFloatSpread)
);
console.log(positions);
```

创建了 100 个位置数据，每个数据都有 3 个维度，用`three.js`内置的数学函数`THREE.MathUtils.randFloatSpread(range)`生成随机分布于`(-range/2,range/2)`之间的值（这里是`(-0.5,0.5)`），打印下结果吧。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f295f7a7998643c8b73cb54f53c21c2f~tplv-k3u1fbpfcp-zoom-1.image)

实际需要的数组是一维的，但目前我们得到的是一个二维数组，因此还要把结果用`flat`函数扁平化一下。

```js
positions = positions.flat();
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4953f0d02b0b4d6b8fc61091660bb4f0~tplv-k3u1fbpfcp-zoom-1.image)

将数组转换为`Float32Array`类型，这个类型是`WebGL`缓冲区数据所支持的类型。

```js
positions = Float32Array.from(positions);
```

## 小结

让我们来简单归纳一下本文的要点吧：

1.  利用`three.js`和`kokomi.js`能快速搭建出一个`3D`世界。
2.  网格`Mesh`由几何体和材质组成，自定义材质可以用`ShaderMaterial`。
3.  `varying`变量限定符负责将变量从顶点着色器转给片元着色器。
4.  利用`kokomi.js`的`UniformInjector`能把`Shadertoy`的`uniform`变量注入自定义材质。
5.  可以通过改变`position`顶点位置变量来实现顶点扭曲，除了用一般的数学计算来扭曲外，也可以用噪声函数进行扭曲。
6.  粒子`Points`是`three.js`另一种支持的物体；能通过`gl_PointSize`来设定粒子的大小；通过改变材质的一些参数，能美化粒子效果的显示。
7.  利用`three.js`的`BufferGeometry`能自定义几何体的形状；通过创建`Float32Array`类型数组，并将它填充到新的`BufferAttribute`内，我们就能改变`attribute`顶点信息变量的值。

顶点着色器可以说是`Shader`里必不可少的一环，跟片元着色器一样，两者缺一不可。一旦掌握了它，你就可以创造出各种令人惊叹的`3D`图形效果。
