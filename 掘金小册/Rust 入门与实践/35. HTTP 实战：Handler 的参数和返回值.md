# 概念

[extractor](https://docs.rs/axum/0.6.0/axum/index.html#extractors)：是实现了 FromRequestParts 或 FromRequest 的类型，定义如何从 Request 中提取程序所需的部分，比如 Path/Query/Json/HeaderMap/。

[response](https://docs.rs/axum/0.6.0/axum/index.html#responses)：是实现了 IntoResponse 的类型，比如 StatusCode/String/Json/HeaderMap。

[handler](https://docs.rs/axum/0.6.0/axum/handler/index.html)：定义了如何处理请求，一般用于编写业务逻辑，可以添加到 Router 中使用。在本文中如果没有特殊说明“Handler trait”，都表示用于处理业务逻辑的函数。

[router](https://docs.rs/axum/latest/axum/struct.Router.html)：定义了访问哪些路径、方法，执行哪个 handler。

  


## handler

axum 中有 [Handler trait](https://docs.rs/axum/0.6.0/axum/handler/trait.Handler.html)，实现了 [Handler trait](https://docs.rs/axum/0.6.0/axum/handler/trait.Handler.html) 的东西可以传递给 [router](https://docs.rs/axum/latest/axum/struct.Router.html)。（一般不会手动实现 [Handler trait](https://docs.rs/axum/0.6.0/axum/handler/trait.Handler.html)）

如果存在一个函数，同时满足以下条件，axum 会自动为这个函数实现 [Handler trait](https://docs.rs/axum/0.6.0/axum/handler/trait.Handler.html)。

1.  函数是 async。
1.  所有参数都实现了 FromRequestParts/FromRequest （上面介绍的 extractor）
1.  返回值实现了 IntoResponse（上面介绍的 response）

  


**举个例子**

hello 函数满足以下条件：

1.  hello 函数是 async 函数。
1.  参数 Query 实现了 FromRequestParts，可以阅读源码（下图）或看[文档](https://docs.rs/axum/0.6.0/axum/extract/struct.Query.html#impl-FromRequestParts%3CS%3E-for-Query%3CT%3E)

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c05ca17ee8034c19a579cbfce5fa9617~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1602&h=678&e=png&b=2b2b2b)

  


3.  返回值 &'static str 实现了 IntoResponse，可以阅读源码（下图）或看[文档](https://docs.rs/axum/0.6.20/axum/response/trait.IntoResponse.html#impl-IntoResponse-for-%26'static+str)

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d418551a31744b22814fb014d375fcd3~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1362&h=166&e=png&b=2b2b2b)

```Rust

use std::collections::HashMap;

pub async fn hello(query: Query<HashMap<String, String>>) -> &'static str {
    tracing::info!("hello, query: {query:?}"); // 日志输出参数，比如 /hello?a=1&b=2
    "hello"
}
```

  


## extractor

extractor 有2种：

-   FromRequestParts 不会消耗所有权，通常用于从 head 提取信息（比如 Path/Query/HeaderMap）
-   FromRequest 会消耗所有权，通常用于从 body 提取信息（比如 String/Bytes/Json），参数中最多带有一个

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f5e0e8e27f8a4516a00b6e973dda4327~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1148&h=1454&e=png&b=2b2b2b)

  


例子：

> `Path(user_id): Path<u32>` 是模式匹配的语法

```Rust
use axum::extract::{Path, Query, Json};
use std::collections::HashMap;

// `Path` gives you the path parameters and deserializes them.
async fn path(Path(user_id): Path<u32>) {}

// `Query` gives you the query parameters and deserializes them.
async fn query(Query(params): Query<HashMap<String, String>>) {}

// Buffer the request body and deserialize it as JSON into a
// `serde_json::Value`. `Json` supports any type that implements
// `serde::Deserialize`.
async fn json(Json(payload): Json<serde_json::Value>) {}
```

  


建议阅读 [common extractors](https://docs.rs/axum/0.6.20/axum/extract/index.html#common-extractors) 了解常见 extractor

## response

IntoResponse：可以操作 header、status code 等，也可以操作 body（比如 String/Json）

  


例子：

```Rust
use axum::{
    body::Body,
    routing::get,
    response::Json,
    Router,
};
use serde_json::{Value, json};

// `&'static str` becomes a `200 OK` with `content-type: text/plain; charset=utf-8`
async fn plain_text() -> &'static str {
    "foo"
}

// `Json` gives a content-type of `application/json` and works with any type
// that implements `serde::Serialize`
async fn json() -> Json<Value> {
    Json(json!({ "data": 42 }))
}
```

  


### 哪些类型自动实现了 IntoResponse

  


**常用类型：**

1.  `&'static str` 实现了 `IntoResponse`
1.  `StatusCode` 实现了 `IntoResponse`
1.  `String` 实现了 `IntoResponse`
1.  `Json` 实现了 `IntoResponse`
1.  `HeaderMap` 实现了 `IntoResponse`
1.  如果 `T` 和 `E` 都实现了 `IntoResponse`，`Result<T, E>` 自动实现了 `IntoResponse`
1.  如果 `R` 实现了 `IntoResponse`，` (StatusCode, R)  `自动实现了 `IntoResponse`

以下 tuple 自动实现了 IntoResponse

-   `(StatusCode, T)`，其中 `T` 实现了 `IntoResponse`
-   `(Parts, T)`，其中 `T` 实现了 `IntoResponse`
-   `(Response<()>, T)`，其中 `T` 实现了 `IntoResponse`
-   `(T1, .., Tn, G)` ，其中 `T1` 到 `Tn` 都实现了 `IntoResponseParts`，`G` 实现了 `IntoResponse`
-   `(StatusCode, T1, .., Tn, G)`，其中 `T1` 到 `Tn` 都实现了 `IntoResponseParts`，`G` 实现了 `IntoResponse`
-   `(Parts, T1, .., Tn, impl IntoResponse)`，其中 `T1` 到 `Tn` 都实现了 `IntoResponseParts`
-   `(Response<()>, T1, .., Tn, G)` ，其中 `T1` 到 `Tn` 都实现了 `IntoResponseParts`，`G` 实现了 `IntoResponse`

  


> 简单来说就是 tuple 里面最多有一个操作 body 的，可以有多个操作 parts 的。

# 常见使用场景

1.  在自定义错误类型的时候，可以为自己的错误类型实现 `IntoResponse`，定义错误的状态码、响应体。
1.  自定义 User 结构体，为 User 实现 `FromRequestParts`，从 header 中获取 cookie/token 等信息，实现登录态验证。
1.  在前端 JS 中只支持 int53，不支持 int64。如果希望 Json 序列化时，自动转换 int64 变为 String，可以自定义 Json（替代 axum 提供的 Json），为自定义的 Json 实现 `IntoResponse` 和 `FromRequest`。
1.  需要使用其他格式进行传输，比如 ProtoBuf、Thrift 等。

  


  


# 总结

1.  handler 是异步函数

1.  handler 函数的参数需要实现 FromRequestParts 或 FromRequest

    1.  常见类型：Query、Path、Json、HeaderMap

1.  handler 函数的返回值需要实现 IntoResponse

    1.  常见类型：StatusCode、String、Json、HeaderMap
    1.  如果 `T` 和 `E` 都实现了 `IntoResponse`，`Result<T, E>` 自动实现了 `IntoResponse`
    1.  如果 `R` 实现了 `IntoResponse`，` (StatusCode, R)  `自动实现了 `IntoResponse`

1.  FromRequestParts/FromRequest 和 IntoResponseParts/IntoResponse 的区别

    1.  名字中带有“Parts”一般用于操作 header、url、query，可以有多个。
    1.  名字中不带“Parts”一般用于操作 body，只能有一个。
