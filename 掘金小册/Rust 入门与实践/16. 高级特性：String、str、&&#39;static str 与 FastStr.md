# 导读

在 Rust 中，你可能会遇到多种字符串类型，例如 `String`、`str`、`&'static str`和 `FastStr`。这么多的字符串类型常常让 Rust 初学者感到困惑，但是理解它们对于编写正确、高效的代码非常重要。本文将详细介绍这些字符串类型及其用法，帮助您更好地理解和掌握 Rust 字符串处理。

  


# str 类型

str 类型表示一个字符串切片（string slice），是 Rust 中表示字符串的最原始类型（most primitive type）， 它是一个不可变、不定长（`!Sized`)的类型，因此经常通过`&str` 的形式来使用，`str`存储的必须是一个有效合法的 UTF-8 字符串，由于 UTF-8 每个字符可能由1到4个字节表示（取决于字符本身），因此Rust的字符串不支持O(1)的索引操作。

  


在编写 Rust 代码的时候，`&str`经常用于作为函数的参数和返回值。下面是创建 `&str` 的一些示例：

```Rust
let s1: &str = "hello";
let s2: &str = &String::from("hello");
let s3: &str = &s2[0..2];
```

  


## &'static str 类型

`&'static str` 是一种特殊的 `&str` 类型，表示一个具有静态生命周期的字符串切片。这意味着它指向的字符串数据在整个程序运行期间都是有效的。这种类型的字符串切片通常用于存储字符串字面量，因为字符串字面量在编译后会被嵌入到程序的二进制文件中，不会在运行时被释放，也不能被修改。

  


下面的代码会创建一个 &'static str 类型的变量：

```Rust
static HELLO: &'static str = "hello";
```

  


## str 基本功能

作为 Rust 的内置字符串类型，`str`具有丰富的功能，这些功能可以通过 str的各种方法进行使用，下面简单介绍几种常见的 `str` 功能。

1.  ### 获取字符串的长度（以字节为单位）

```Rust
rustCopy code
let s = "Hello, world!";
println!("Length: {}", s.len());
```

2.  ### 检查字符串是否包含指定的子串

```Rust
let s = "Hello, world!";
println!("Contains 'world': {}", s.contains("world"));
```

3.  ### 查找子串在字符串中首次出现的位置（以字节为单位）

```Rust
let s = "Hello, world!";
match s.find("world") {
    Some(index) => println!("Found at index: {}", index),
    None => println!("Not found"),
}
```

4.  ### 根据指定的分隔符拆分字符串

```Rust
let s = "apple,banana,orange";
for fruit in s.split(',') {
    println!("{}", fruit);
}
```

5.  ### 去除字符串前后的空白字符（例如空格和换行符）

```Rust
let s = "  Hello, world!  ";
println!("Trimmed: '{}'", s.trim());
```

6.  ### chars：遍历字符串中字符。

```Rust
let s = "Hello, world!";
for c in s.chars() {
    println!("{}", c);
}
```

7.  ### 检查字符串是否以指定的子串开头和结尾

```Rust
let s = "Hello, world!";
println!("Starts with 'Hello': {}", s.starts_with("Hello"));
println!("Ends with 'world': {}", s.ends_with("world!"));
```

  


这些方法提供了基本的字符串处理功能，可以用于执行诸如查找、替换、拆分和遍历等操作。当然，`str`类型还有许多其他方法，你可以查阅 [Rust官方文档](https://doc.rust-lang.org/stable/std/primitive.str.html) 以了解更多细节。

  


# String 类型

通过上一小节可以看到，由于 `str` 表示一个不可变的切片，因此我们不能对 `str` 的内容做修改，为了能够动态的创建和修改字符串，Rust的标准库提供了 `String` 类型。

  


`String` 类型同样表示一个合法的 UTF-8 编码的字符串，在数据存储上，`String`类型的数据一定会存储到堆上，它自身只包含一个指向堆内存的指针，以及底层字符串的长度（length）和容量（capacity）。当字符串的长度超过其容量时，Rust会自动重新分配更大的内存空间，以便容纳更多的字符，并且将原本位置的字符串数据拷贝到新的位置。

  


## 创建 String

可以使用多种方法创建一个新的`String`实例。例如，你可以使用`String::new()`创建一个空字符串，或者使用`String::from()`从字面量字符串创建一个`String`。标准库还提供`format!`宏用来灵活组装一个字符串。

```Rust
let s1 = String::new();
let s2 = String::from("hello");
let name = "Alice";
let s3 = format!("Hello, {}!", name);
```

  


## 修改 String

`String`类型提供了各种方法来修改字符串的内容。例如，你可以使用`push_str()`和`push()`方法分别向字符串尾部附加字符串切片和字符，可以使用`pop()` 像stack一样从尾部弹出一个字符，或者使用`insert()` 向字符串的指定位置插入某个字符。

```Rust
let mut s = String::from("Hello");
s.push_str(",world!");
s.push('!');
s.pop();
s.insert(6, ' ');
```

  


## 何时使用 &str 或 String 类型

在编写 Rust 代码的时候，遇到需要动态生成，组装和修改字符串的需求，我们一般都会使用 String 类型，但是在函数传参和返回时，我们一般使用 &str 类型，原因如下：

1.  **通用性：** `&str`类型可以接受字符串字面量、`String`类型和其他`&str`类型的值。因此，使用`&str`作为参数类型可以使函数更加通用，同时`&str`兼容多种字符串表示形式。而使用`String`或`&String`作为参数类型将限制函数的使用场景，可能导致不必要的类型转换。
1.  **性能：** `&str`类型是一个对原始字符串数据的不可变引用，它不涉及所有权转移和内存分配。在大多数情况下，使用`&str`类型传递字符串数据的代价较小。相反，当使用`String`类型时，如果需要复制字符串数据，会产生额外的性能开销。即使使用`&String`类型，仍然可能需要额外的解引用操作来访问字符串数据。
1.  **安全性：** `&str`类型表示一个不可变的字符串切片，这意味着在函数中无法修改它所引用的字符串数据。这有助于提高代码的安全性和可维护性。当你看到一个函数参数为`&str`类型时，可以清楚地知道这个函数不会修改传入的字符串。

  


# FastStr

`FastStr`是 `faststr` crate中提供的字符串类型，针对 `String` 类型在拷贝时会产生额外的性能开销的问题，`faststr`提供了一种高性能的字符串类型，在兼容各种现有的字符串类型的同时，尽可能减小程序中由于字符串拷贝导致的内存开销，`FastStr`的clone方法对于底层实际存储的字符串数据来说是零拷贝的。

  


`String`、`Arc<String>`、`Cow`、`&'static str` 这些都可以直接 into 到 `FastStr`；`Bytes`、`BytesMut`、`Vec<u8>` 可以直接用 unsafe 方式转到 `FastStr` ，这些方法都可以避免数据拷贝。如果是普通的 `&str`，那么可以用 `FastStr::new(s)` 创建出 `FastStr`（会有一次内存分配和拷贝），但是之后在 `FastStr` 使用中的 clone 就是零开销了。

  


在使用上，`FastStr` 通过实现`Deref`` <Target =  ``str``>` triat ，将自己实现为`str`类型的智能指针，从而继承了大量`str`的方法，可以方便的在 `FastStr` 上使用。

  


大部分情况下，`str` 和 `String` 类型已经可以满足我们在 Rust 中操作和使用字符串的需求，在你的程序需要极致的性能优化的时候可以选择 `FastStr`。
