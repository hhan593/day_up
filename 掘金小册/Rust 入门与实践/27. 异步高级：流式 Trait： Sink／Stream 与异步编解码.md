Future 定义了一次性的可等待结果，对于异步数据流，我们需要额外的 Trait 定义，这里主要介绍 Sink 和 Stream 两个 Trait。

## Sink

Futures 库提供了 Sink 的定义([docs.rs](https://docs.rs/futures/latest/futures/sink/trait.Sink.html))：

```Rust
pub trait Sink<Item> {
    type Error;

    // Required methods
    fn poll_ready(
        self: Pin<&mut Self>,
        cx: &mut Context<'_>
    ) -> Poll<Result<(), Self::Error>>;
    fn start_send(self: Pin<&mut Self>, item: Item) -> Result<(), Self::Error>;
    fn poll_flush(
        self: Pin<&mut Self>,
        cx: &mut Context<'_>
    ) -> Poll<Result<(), Self::Error>>;
    fn poll_close(
        self: Pin<&mut Self>,
        cx: &mut Context<'_>
    ) -> Poll<Result<(), Self::Error>>;
}
```

**SinkExt 中提供了几个写入方法：**

-   send(item)：写入 Sink 并 flush
-   feed(item)：写入 Sink 但不 flush
-   flush()：将已经写入 Sink 的数据 flush
-   send_all()：将给定的 Stream 中的零或多个数据全部写入 Sink 并一次或多次 flush(自动决定何时flush)

  


## Stream

Doc: [Stream in futures::stream - Rust](https://docs.rs/futures/latest/futures/stream/trait.Stream.html)

```Rust
pub trait Stream {
    type Item;

    // Required method
    fn poll_next(
        self: Pin<&mut Self>,
        cx: &mut Context<'_>
    ) -> Poll<Option<Self::Item>>;

    // Provided method
    fn size_hint(&self) -> (usize, Option<usize>) { ... }
}
```

借助 StreamExt，我们可以使用 async 接口：

-   next()：迭代下一个元素
-   map(f)：将整个流应用 F 变为另一个流
-   collect()：持续迭代该流直至结束，并收集结果到某种 collection
-   take(n)：仅保留前 N 个元素
-   buffered(n): 提前迭代并缓存元素（对于异步爬取等需求可以大大提升效率）

  


### 实现一个计时器

我们可以借助 `tokio::time::sleep` 实现一个 Interval 结构，该结构是一个计时器，每隔固定的时间间隔会返回一条空消息。

```
use futures::{ready, Future, Stream, StreamExt};
use std::{ops::Add, pin::Pin, task::Poll, time::Duration};
use tokio::time::{sleep, Instant, Sleep};

#[tokio::main]
async fn main() {
    let mut i = Box::pin(Interval::new(Duration::from_millis(100)));
    let instant = Instant::now();
    while i.next().await.is_some() {
        println!("{}ms: interval triggered!", instant.elapsed().as_millis());
    }
}

struct Interval {
    period: Duration,
    sleep: Sleep,
}

impl Interval {
    fn new(period: Duration) -> Self {
        let sleep = sleep(period);
        Self { period, sleep }
    }
}

impl Stream for Interval {
    type Item = ();

    fn poll_next(
        self: Pin<&mut Self>,
        cx: &mut std::task::Context<'_>,
    ) -> Poll<Option<Self::Item>> {
        let period = self.period;
        unsafe {
            let this = self.get_unchecked_mut();
            ready!(Pin::new_unchecked(&mut this.sleep).poll(cx));
            Pin::new_unchecked(&mut this.sleep).reset(Instant::now().add(period));
        }
        Poll::Ready(Some(()))
    }
}

// Output:
// 101ms: interval triggered!
// 202ms: interval triggered!
// 303ms: interval triggered!
// 404ms: interval triggered!
// 505ms: interval triggered!
// 607ms: interval triggered!
// 708ms: interval triggered!
// ...
```

  


### 简便实现 Stream

实现 Stream 要面对裸的 poll 接口，这种接口会带来极大的编程复杂度。如果我们要基于 async fn 来实现 Stream 则难上加难。

对此，我们可以利用 async-stream 来快捷实现 Stream trait：https://docs.rs/async-stream/0.3.5/async_stream/

```
use async_stream::stream;

use futures_core::stream::Stream;
use futures_util::pin_mut;
use futures_util::stream::StreamExt;

fn zero_to_three() -> impl Stream<Item = u32> {
    stream! {
        for i in 0..3 {
            yield i;
        }
    }
}

#[tokio::main]
async fn main() {
    let s = zero_to_three();
    pin_mut!(s); // needed for iteration

    while let Some(value) = s.next().await {
        println!("got {}", value);
    }
}
```

  


## 异步编解码实践

在我们编写网络框架时，常用的一个辅助结构叫 Framed。借助这个结构，我们可以将 IO 逻辑与序列化/反序列化逻辑拆分开。

Framed 可以从 IO 和 Codec 创建出来，其对外暴露发送/接收结构体的接口，内部借助 Codec 完成序列化/反序列化，二进制流通过 IO 操作。Framed 对外暴露的异步发送/接收接口就是 Sink 和 Stream。

当我们可以低成本地拿到消息的总长度，使用 Framed 可以大大简化解码器的编写过程。

下面我以一个例子为大家讲解常用的异步编解码模式。

  


### 编码定义

我们假定有这么一种编码（使用大端序）：

```
[4byte message length]
[1byte field type][1byte field length][field data]
```

整个消息有一个 4byte 的长度头（以 byte 计算，含长度头自身）

Field type 假定有 2 种：

1.  List = 0，此时 field length 表示后续 field 个数
1.  String = 1，此时 field length 表示 String 长度（以 byte 计算）

例如，对于 `[a, [aa, bb, cc], d]` 我们可以将其编码为：

```
0, 0, 0, 26 // 总长度
0, 3, // 最外层是长度为 3 的 List
1, 1, 97, // 类型为 String，长度为 1，内容为 97(a)
0，3， // 长度为 3 的 List
1，2，97，97 // aa
1，2，98，98 // bb
1，2，99，99 // cc
1, 1, 100, // d
```

  


### 结构定义

我们现在要为该编码定义结构表示。通常我们有两种形式：

1.  对于固定 IDL 的 RPC 需求等，我们可以生成固定结构并生成其编解码实现
1.  对于非固定类型，我们可以提供通用表示

此处我们选择形式2。对于上述编码，我们有以下结构定义：

```
struct Message {
    field: Field,
}

enum Field {
    String(String),
    List(Vec<Field>),
}
```

  


### Encoder 实现

```
use bytes::{BufMut, BytesMut};
use tokio_util::codec::Encoder;

struct Message {
    field: Field,
}

enum Field {
    String(String),
    List(Vec<Field>),
}

struct CHEncoder;

impl Encoder<&Message> for CHEncoder {
    type Error = anyhow::Error;

    fn encode(&mut self, item: &Message, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let offset = dst.len();
        dst.put_u32(0);
        self.encode(&item.field, dst)?;
        let written: u32 = (dst.len() - offset).try_into()?;
        let mut buf = &mut dst[offset..offset + 4];
        buf.put_u32(written);
        Ok(())
    }
}

impl Encoder<&Field> for CHEncoder {
    type Error = anyhow::Error;

    fn encode(&mut self, item: &Field, dst: &mut BytesMut) -> Result<(), Self::Error> {
        match item {
            Field::String(s) => self.encode(s.as_str(), dst)?,
            Field::List(list) => self.encode(list.as_slice(), dst)?,
        }
        Ok(())
    }
}

impl Encoder<&str> for CHEncoder {
    type Error = anyhow::Error;

    fn encode(&mut self, item: &str, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.as_bytes().len().try_into()?;
        // 1 for string
        dst.put_u8(1);
        dst.put_u8(len);
        dst.extend_from_slice(item.as_bytes());
        Ok(())
    }
}

impl Encoder<&[Field]> for CHEncoder {
    type Error = anyhow::Error;

    fn encode(&mut self, item: &[Field], dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.len().try_into()?;
        // 0 for list
        dst.put_u8(0);
        dst.put_u8(len);
        for field in item {
            self.encode(field, dst)?;
        }
        Ok(())
    }
}

#[tokio::main]
async fn main() {
    let message = Message {
        field: Field::List(vec![
            Field::String("a".to_string()),
            Field::List(vec![
                Field::String("aa".to_string()),
                Field::String("bb".to_string()),
                Field::String("cc".to_string()),
            ]),
            Field::String("d".to_string()),
        ]),
    };
    let mut buffer = BytesMut::new();
    CHEncoder.encode(&message, &mut buffer).unwrap();
    println!("len: {}, encoded: {buffer:?}", buffer.len());
}

// Output:
// len: 26, encoded: b"\0\0\0\x1a\0\x03\x01\x01a\0\x03\x01\x02aa\x01\x02bb\x01\x02cc\x01\x01d"
```

该 Encoder 可以正确将结构序列化，与我们手动写的二进制一致。

  


### Decoder 实现

```
use anyhow::bail;
use bytes::{Buf, BufMut, BytesMut};
use tokio_util::codec::{Decoder, Encoder};

#[derive(Debug)]
struct Message {
    field: Field,
}

#[derive(Debug)]
enum Field {
    String(String),
    List(Vec<Field>),
}

struct CHEncoder;

impl Encoder<&Message> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &Message, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let offset = dst.len();

        let written: u32 = (dst.len() - offset).try_into()?;
        let mut buf = &mut dst[offset..offset + 4];
        buf.put_u32(written);
        Ok(())
    }
}

impl Encoder<&Field> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &Field, dst: &mut BytesMut) -> Result<(), Self::Error> {
        match item {
            Field::String(s) => self.encode(s.as_str(), dst)?,
            Field::List(list) => self.encode(list.as_slice(), dst)?,
        }
        Ok(())
    }
}

impl Encoder<&str> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &str, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.as_bytes().len().try_into()?;
        // 1 for string
        dst.put_u8(1);
        dst.put_u8(len);
        dst.extend_from_slice(item.as_bytes());
        Ok(())
    }
}

impl Encoder<&[Field]> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &[Field], dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.len().try_into()?;
        // 0 for list
        dst.put_u8(0);
        dst.put_u8(len);
        for field in item {
            self.encode(field, dst)?;
        }
        Ok(())
    }
}

struct CHMessageDecoder;
impl Decoder for CHMessageDecoder {
    type Item = Message;
    type Error = anyhow::Error;

    fn decode(&mut self, src: &mut BytesMut) -> Result<Option<Self::Item>, Self::Error> {
        if src.len() < 4 {
            // Not enough data to read length marker.
            return Ok(None);
        }
        // Read length marker.
        let mut length_bytes = [0u8; 4];
        length_bytes.copy_from_slice(&src[..4]);
        let length = u32::from_be_bytes(length_bytes) as usize;

        if src.len() < length {
            // The full string has not yet arrived.
            //
            // We reserve more space in the buffer. This is not strictly
            // necessary, but is a good idea performance-wise.
            src.reserve(length - src.len());

            // We inform the Framed that we need more bytes to form the next
            // frame.
            return Ok(None);
        }
        src.advance(4);
        let decoded = CHFieldCodec.decode(src)?;
        match decoded {
            None => Err(anyhow::anyhow!("invalid data")),
            Some(field) => Ok(Some(Message { field })),
        }
    }
}

struct CHFieldCodec;
impl Decoder for CHFieldCodec {
    type Item = Field;
    type Error = anyhow::Error;

    fn decode(&mut self, src: &mut BytesMut) -> Result<Option<Self::Item>, Self::Error> {
        Ok(Some(Self::read_field(src)?))
    }
}

impl CHFieldCodec {
    fn read_field(src: &mut BytesMut) -> anyhow::Result<Field> {
        Self::check(src, 2)?;
        let typ = src.get_u8();
        match typ {
            0 => {
                let len = src.get_u8();
                let mut items = Vec::with_capacity(len as usize);
                for _ in 0..len {
                    items.push(Self::read_field(src)?);
                }
                Ok(Field::List(items))
            }
            1 => {
                let len = src.get_u8() as usize;
                let buf = match src.get(0..len) {
                    Some(buf) => buf,
                    None => bail!("invalid string length"),
                };
                let s = String::from_utf8(buf.to_vec())?;
                src.advance(len);
                Ok(Field::String(s))
            },
            _ => bail!("invalid data type")
        }
    }

    #[inline]
    fn check(src: &BytesMut, n: usize) -> anyhow::Result<()> {
        if src.len() < n {
            return Err(anyhow::anyhow!("invalid data"));
        }
        Ok(())
    }
}

#[tokio::main]
async fn main() {
    let message = Message {
        field: Field::List(vec![
            Field::String("a".to_string()),
            Field::List(vec![
                Field::String("aa".to_string()),
                Field::String("bb".to_string()),
                Field::String("cc".to_string()),
            ]),
            Field::String("d".to_string()),
        ]),
    };
    println!("message: {message:?}");

    let mut buffer = BytesMut::new();
    CHEncoder.encode(&message, &mut buffer).unwrap();
    println!("len: {}, encoded: {buffer:?}", buffer.len());

    let message_decoded = CHMessageDecoder.decode_eof(&mut buffer).unwrap().unwrap();
    println!("decoded: {message_decoded:?}");
}

// Output:
// message: Message { field: List([String("a"), List([String("aa"), String("bb"), String("cc")]), String("d")]) }
// len: 26, encoded: b"\0\0\0\x1a\0\x03\x01\x01a\0\x03\x01\x02aa\x01\x02bb\x01\x02cc\x01\x01d"
// decoded: Message { field: List([String("a"), List([String("aa"), String("bb"), String("cc")]), String("d")]) }
```

Decoder trait 返回的是 `Result<Option<`*`Self`*` ::Item>,  `*`Self`*`::Error>`，其中 Option 表示数据不够，需要更多数据。

但是在这里我们直接使用了序列化好的 buffer，所以不存在数据不够的问题。

可以看出，我们序列化并反序列化后的结构与原始结构是一致的。

  


### 借助 Framed 组合 IO

Codec(Encoder+Decoder) 帮助我们在结构体与二进制之间转换，但是作为一个应用，最终二进制是要与文件 IO 或网络 IO 做交互的：

1.  将序列化好的结果写入 IO
1.  从 IO 读数据并反序列化

大家可能会好奇，这不就是读写嘛？看起来很简单呀。其实这里面临两个头疼的问题：

1.  写入 IO 可能需要 buffer 语义：可能单个消息很小，为了节省 syscall 可以先 buffer 起来，最后一起 flush
1.  读数据并反序列化时可能遇到数据不够的情况，这时要再次读并调用 decoder

Framed 就是帮助我们解决这两个问题的工具。

  


组合 Framed：

```
use anyhow::bail;
use bytes::{Buf, BufMut, BytesMut};
use futures::{SinkExt, StreamExt};
use tokio::net::{TcpStream, TcpListener};
use tokio_util::codec::{Decoder, Encoder, Framed};

#[derive(Debug)]
struct Message {
    field: Field,
}

#[derive(Debug)]
enum Field {
    String(String),
    List(Vec<Field>),
}

struct CHEncoder;

impl Encoder<&Message> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &Message, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let offset = dst.len();
        dst.put_u32(0);
        self.encode(&item.field, dst)?;
        let written: u32 = (dst.len() - offset).try_into()?;
        let mut buf = &mut dst[offset..offset + 4];
        buf.put_u32(written);
        Ok(())
    }
}

impl Encoder<&Field> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &Field, dst: &mut BytesMut) -> Result<(), Self::Error> {
        match item {
            Field::String(s) => self.encode(s.as_str(), dst)?,
            Field::List(list) => self.encode(list.as_slice(), dst)?,
        }
        Ok(())
    }
}

impl Encoder<&str> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &str, dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.as_bytes().len().try_into()?;
        // 1 for string
        dst.put_u8(1);
        dst.put_u8(len);
        dst.extend_from_slice(item.as_bytes());
        Ok(())
    }
}

impl Encoder<&[Field]> for CHEncoder {
    type Error = anyhow::Error;

    #[inline]
    fn encode(&mut self, item: &[Field], dst: &mut BytesMut) -> Result<(), Self::Error> {
        let len: u8 = item.len().try_into()?;
        // 0 for list
        dst.put_u8(0);
        dst.put_u8(len);
        for field in item {
            self.encode(field, dst)?;
        }
        Ok(())
    }
}

struct CHMessageDecoder;
impl Decoder for CHMessageDecoder {
    type Item = Message;
    type Error = anyhow::Error;

    fn decode(&mut self, src: &mut BytesMut) -> Result<Option<Self::Item>, Self::Error> {
        if src.len() < 4 {
            // Not enough data to read length marker.
            return Ok(None);
        }
        // Read length marker.
        let mut length_bytes = [0u8; 4];
        length_bytes.copy_from_slice(&src[..4]);
        let length = u32::from_be_bytes(length_bytes) as usize;

        if src.len() < length {
            // The full string has not yet arrived.
            //
            // We reserve more space in the buffer. This is not strictly
            // necessary, but is a good idea performance-wise.
            src.reserve(length - src.len());

            // We inform the Framed that we need more bytes to form the next
            // frame.
            return Ok(None);
        }
        src.advance(4);
        let decoded = CHFieldCodec.decode(src)?;
        match decoded {
            None => Err(anyhow::anyhow!("invalid data")),
            Some(field) => Ok(Some(Message { field })),
        }
    }
}

struct CHFieldCodec;
impl Decoder for CHFieldCodec {
    type Item = Field;
    type Error = anyhow::Error;

    fn decode(&mut self, src: &mut BytesMut) -> Result<Option<Self::Item>, Self::Error> {
        Ok(Some(Self::read_field(src)?))
    }
}

impl CHFieldCodec {
    fn read_field(src: &mut BytesMut) -> anyhow::Result<Field> {
        Self::check(src, 2)?;
        let typ = src.get_u8();
        match typ {
            0 => {
                let len = src.get_u8();
                let mut items = Vec::with_capacity(len as usize);
                for _ in 0..len {
                    items.push(Self::read_field(src)?);
                }
                Ok(Field::List(items))
            }
            1 => {
                let len = src.get_u8() as usize;
                let buf = match src.get(0..len) {
                    Some(buf) => buf,
                    None => bail!("invalid string length"),
                };
                let s = String::from_utf8(buf.to_vec())?;
                src.advance(len);
                Ok(Field::String(s))
            }
            _ => bail!("invalid data type"),
        }
    }

    #[inline]
    fn check(src: &BytesMut, n: usize) -> anyhow::Result<()> {
        if src.len() < n {
            return Err(anyhow::anyhow!("invalid data"));
        }
        Ok(())
    }
}

#[tokio::main]
async fn main() {
    let listener = TcpListener::bind("0.0.0.0:0").await.unwrap();
    let addr = listener.local_addr().unwrap();

    let message = Message {
        field: Field::List(vec![
            Field::String("a".to_string()),
            Field::List(vec![
                Field::String("aa".to_string()),
                Field::String("bb".to_string()),
                Field::String("cc".to_string()),
            ]),
            Field::String("d".to_string()),
        ]),
    };
    println!("message: {message:?}");

    // sender
    tokio::spawn(async move {
        let socket = TcpStream::connect(addr).await.unwrap();
        let mut framed_writer = Framed::new(socket, CHEncoder);
        framed_writer.send(&message).await.unwrap();
    });

    // receiver
    let (socket, _) = listener.accept().await.unwrap();
    let mut framed_reader = Framed::new(socket, CHMessageDecoder);
    let message_decoded = framed_reader.next().await.unwrap().unwrap();
    println!("decoded: {message_decoded:?}");
}

// Output:
// message: Message { field: List([String("a"), List([String("aa"), String("bb"), String("cc")]), String("d")]) }
// decoded: Message { field: List([String("a"), List([String("aa"), String("bb"), String("cc")]), String("d")]) }
```

在这个例子中，我们启动了一对 TCP Client 和 Server，在一端序列化并发送 Message，另一端接收并反序列化。

可以看出，我们利用 Codec trait 与 Framed 结构正确完成了该任务。
