## Rust 简介

在正式开启我们的 Rust 学习之前，我们先来简单了解一下这门语言，以及为什么要学习 Rust 吧！

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8fa0ee0b2def48978b832e3c055218fa~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3100&h=1284&e=png&b=ffffff" alt=""  width="50%"/></p>

了解一门语言呢，就要先了解它的发展史。

Graydon 是 Mozilla 的一位职业编程语言工程师，给其他语言开发编译器和工具集，在 2008 年私人研发了 Rust 这门语言，正值当时 Mozilla 要开发 firefox 的 Servo 引擎，想在保证安全的同时有着高性能，于是就瞄准了 Rust，开始赞助。

其实 2010 年到 2015 年早期的 Rust 是有 GC 的，不过还好 Rust 团队在 1.0 前夕做了一个英明的决定，把 GC 给干掉了，所以在 2015 年发布 1.0 版本的时候，确定了 Rust 的高性能+高安全性的特性。1.0 的发布，对 Rust 来说，意味着稳定性的保证。

Rust 的规划是以三年为一个节点的，在 2015 年到 2018 年的三年期间呢，Rust 主要解决的问题是“生产力”，在这段时间里 Rust 的工具链、文档、编译器都更加智能，对开发者更加友好了。

2018 到 2021 这三年，Rust 完善了异步的生态，也就是从这时候开始，Rust 开始适用于写业务，毕竟用同步写业务会遇到 C10k 问题。

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d93684965b484aba8ffbb91c91b4508d~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2032&h=1734&e=png&b=fefefe" alt=""  /></p>

2021 到 2024 年，Rust 的规划是“扩展授权”，为啥叫“扩展授权（empowerment）”呢，因为 Rust 的目标是“empower everyone to build reliable（可靠的） and efficient（高效的） software”。Rust 官方团队关注到了大家一直吐槽的“Rust 难学”的问题，在完善了 Rust 的基础特性之后，开始着重关注在易用性和项目落地上。

与此同时，我们也在经历着开源项目的爆炸性增长，可以说“一切能用 Rust 重写的项目都将或者正在用 Rust 重写”。

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/50f10953583a4d09a4e4d78016de4f12~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=3008&h=1728&e=png&b=202c44" alt=""  /></p>

虽然大家一直都在吐槽 Rust 难学，实际上还是口嫌体正直，最明显的就是 Rust 已经连续第七年获得 StackOverflow 最受开发者喜爱语言榜的榜首了。

一门语言的未来，光受开发者喜爱是不够的，关键还在于企业是否接受。

世界上很多知名公司都早早就瞄准了 Rust 语言，比如 Meta 从 2016 年就已经开始推动公司内 Rust 的应用，微软用 Rust 解决内存安全问题、Google 在 Chrome 和 Android 中使用 Rust，字节跳动和华为内部大量使用 Rust、Github 用 Rust 重写代码搜索引擎，蚂蚁金服用 Rust 重写时序数据库等等，同时也涌现出众多使用 Rust 的中小企业，在此不一一列举。

# Rust 特性介绍

接下来，我们简单介绍一下之前说的 Rust 的三个特性：性能、安全和协作。

## 性能

零成本抽象是 Rust 设计哲学中一个非常重要的指导规范，在 Rust 本身设计和社区里都会奉行这一原则，我们可以简单理解成两个要点：

1.  没有用到的功能，不需要编译和运行时开销，不用给不需要的东西付费；
1.  那如果是需要用的功能，你也不可能优化得更好，在最终机器码层面上，Rust 可以帮你做到最优；

如何体现呢，比如：Rust 没有 GC、采用的是 RAII 的方式来管理内存，同时 Rust 支持和 C/C++ 的零损耗 FFI。

这代表啥呢？代表着，所有 C 和 C++ 的生态，也同时是 Rust 的生态，Rust 都可以使用。

## 安全

接下来让我们看看，Rust 是如何做到内存安全的呢?

它有三个重要的概念：

1.  Ownership 所有权
1.  Borrowing 借用
1.  Liftime 生命周期

大家不用紧张，这三个概念和生活中的很多场景和概念非常类似，比如：我有一本书，那么书就是资源，我拥有这本书的所有权；你可以来问我借用这本书；但是这本书能借多久（生命周期）由我说了算。

### 所有权

我们来看一下这个代码

1.  fn main 是 rust 的入口函数；
1.  let 是 rust 申请变量的关键字；
1.  println! 是 rust 控制台打印宏函数。

我们申请了一个变量，它的类型是 String、值是 hi，然后我们用 println 打印出来，

那么这里我们认为变量 x 拥有值的所有权，也就是这里的 hi，最后在变量 x 出了作用域之后，hi 相关的内存就会被释放了。

不难看出，Rust 里内存的管理是自动的，不需要垃圾回收参与，申请和释放的时机都是在编译时就决定了。

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fb2eda5518dd4e7eba4df4ccf86a42e2~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1992&h=1000&e=png&a=1&b=000000" alt=""  /></p>

我们再来看一个例子，这里我们加入一个新变量 y，会发生什么？

程序报错了，因为在 Rust 里对所有权有一个重要约束：所有权是唯一的，`let y = x`实际上会把所有权从 x 转移到了 y，这个时候如果我们再去使用 x，Rust 就会报错，错误展示也很友好，所有权转移了，所以无法继续访问。

这就是 Rust 的第一个非常重要的概念——所有权，它有三条规则：

-   Rust 中的每一个值都有一个被称为其所有者（owner）的变量；
-   值在任意时刻有且只有一个所有者：这可以避免像**多次释放和错误释放**等内存安全问题；
-   当所有者（变量）离开作用域的时候，这个值将被丢弃（回收）：出作用域释放可以避免 memory leak 的问题。

是不是非常简单呢？

### 借用

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/36b97c9ea4f3445fa6f7cf353f2f03f7~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1972&h=1272&e=png&a=1&b=faffff" alt=""  /></p>

前面讲了唯一所有权带来的内存安全保障，但这不利于我们对值复杂的访问需求，这里我们介绍另外一个 Rust 概念——借用。

我们可以创建一个不可变借用来避免所有权转移，`let y = &x`，这样我们就能够在不打破单一所有权的同时，满足多个入口访问相同值的需求；换言之，借用只是共享了访问权限，不拥有值，它是只读的。

```Rust
fn main() {
    let mut x = String::from("Hi");
    let y = &mut x;
    y.push_str(" rust");
    let z = &x;
    println!("{z}");
    println!("{y}");
}
```

有些时候我们不仅需要共享只读权限，还需要去改变这个共享的值，这时候我们就需要一个可变的借用了`let y = &mut x`，我们这里创建了一个可变借用 y，然后通过 y 向 hi 后面追加了“ rust”字符串，最后我们打印到控制台，我们看到这里打印了“hi rust”，说明我们通过可变借用成功的改变了值。

那如果我们再创建一个不可变引用 z 呢，会发生什么？

<p align=center><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/eeb409bf85b1441e8499c0915e42dc2b~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1474&h=522&e=png&b=f9ffff" alt=""  /></p>

报错了，Rust 不允许可变引用和不可变引用同时存在，因为这样可能会存在数据竞争（data race）。

好了，这其实就是 Rust 的第二个重要概念——借用了，它只有一条规则：同时只能存在一个可变借用或者多个不可变借用。

是不是比所有权还简单呢？

### 生命周期

关于生命周期呢，就有那么亿点点复杂了，咱们在这先不讨论了。

不过原理其实很简单：一个资源能借用的时间，不能超过这个资源存在的时间（能借多久 Owner 说了算）。

### 无畏并发

这里再介绍一个 Rust 的特点：官方称之为无畏并发（Fearless Concurrency）。

经常接触并发编程的同学，我相信 Rust 的安全并发特性可以让你眼前一亮。并发编程有一大痛点：容易产生 data race。

但在 Rust 中情况恰恰相反，你甚至不容易写出并发不安全的代码，这主要得益于 Rust 的所有权、借用规则再加上了两个特殊的 Trait（这个是啥大家可以先无视）：Send 和 Sync。

1.  我们知道并发编程中最大的问题是 data race，前面我们介绍了 Rust 中无法同时存在多个可变借用或者可变和不可变借用共存，换言之 Rust 里无法同时读写值，这恰恰是数据竞争的充分必要条件；
1.  但同时读写数据对于并发编程是必要的，因此 Rust 提供了运行时可变性的概念。它们的原理就是把原本编译时需要检查的可变和不可变之间的互斥性，放在了运行时；
1.  我们实际使用的都是不可变引用，它们是可以共存的，写值的时候再做互斥性检查（或者使用原子操作）。这样就可以在不违背 Rust 借用检查的前提下，提供并发读写的能力，同时也强制了用户必须使用这些同步原语来读写数据。
1.  有的时候我们需要对数据有精细的并行访问控制，那我们可以通过 Send 和 Sync 两个标记来告诉编译器：比如本地线程相关的数据，我不希望其他线程访问，那么我们可以标记它非 Send 非 Sync，剩下的就交给 Rust 了。

Rust 通过借用检查、运行时可变性以及 Send/Sync 标记让安全并发非常容易，为开发者带来了极大的便利。

还记不记得我们刚才说过的，Rust 的原则是 Zero-Cost Abstraction，所有的这些 Ownership、Borrowing、Send/Sync 标记的检查全部都是在编译时完成的，运行时根本没有任何的性能损耗。

这一部分之后会详细为大家介绍，这里大家有个大致的概念就行了，只要知道：Rust 中，你甚至不容易写出并发不安全的代码，否则编译器会直接报错。

## 协作

接着，我们来介绍一下为什么说 Rust 很适合协作。

在项目协作中，我们经常要面临功能实现、code review、各种测试之后的代码反复修改。

一般来说，要开发某模块的功能，要如何去写出内存安全和并发安全的代码呢？对于传统语言，很可能需要对其它模块相对熟悉，并且要经历过仔细的 code review。

但是，大家也知道，对于如 C++ 这样的语言，code review 都是很难看出问题来的，特别是很多 codedump 都是隔了十万八千里在一个不起眼的地方莫名其妙出现的，甚至有的只是把某块内存踩坏了不会立刻 coredump 而是在运行了很久之后的某一天突然出现了错误数据……这种错误，遇到了只能——祝你好运。

那么 Rust 好在哪呢？你可以完全信任别人的代码！

怎么理解这句话呢？很简单：如果某个人写的代码有安全问题（无论是内存安全还是并发安全），编译都过不了。这也是为啥大家总听说 Rust 编译很难过，是真的很难过，编译很难过的时候程序员也很难过。

-   智能的编译器
-   完善的文档
-   齐全的工具链
-   成熟的包管理

编译器就好比是武侠小说里的扫地僧，有一个高人指点，功力提升肯定非常快速。并且文档、工具链也是非常的完善和齐全，这点我们在之后自己上手开发 Rust 的时候会有非常深刻的体会。

因此，我们只需要专注于逻辑功能的实现就行了，内存安全、并发安全这些都会由编译器帮你检查完成。

那我们要怎么样才能修成正果呢？这里我给大家一些学习 Rust 的建议：

首先，就是要夯实计算机基础知识，因为 Rust 语言实际上是门偏底层的语言，把很多控制权交给了程序员，所以程序员必须对于计算机的原理有一定了解，这会使得学习 Rust 更简单。比如，学 Rust 的时候，必须要能清楚地知道，一个对象是被分配在了堆上还是栈上。

然后，就是要学会知识屏蔽，那什么叫知识屏蔽呢？

知识屏蔽的意思是，把部分的知识当做黑盒，不要过于钻到细节里面去，也不要想着一次性把所有东西都学会。

比如，针对我们之后学习内部可变性的时候，可能会涉及很多底层的知识，这时候我们不需要了解到内部所有的细节以及相关的知识，只要知道我们这么干，能达到我们的目的就行。

这个对于很多计算机科学相关的知识学习也很有帮助，因为计算机科学相关的知识通常联系紧密且错综复杂，如果一直深究下去，精力完全不够用。

那么针对这部分我们当黑盒的知识怎么办呢？当然不是放过它们，而是可以在之后以“专题”的形式进行学习。

接下来，学会类比你已有的知识。计算机科学中的很多知识都是融会贯通的，比如学 Rust 中的 Vector 的时候，就可以和 C++ 中的 Vector 类比，这样能很大程度帮助大家理解。

第四点呢，就是多阅读源码，特别是标准库和优秀开源项目的源码。当然，因为标准库和优秀开源项目会用大量的 Rust 语言高级特性，甚至是 unsafe 特性，大家一定要记得我说的“知识屏蔽”，这时候反而就要有一点“不求甚解”，否则会在源码的海洋里面迷失自我。

最后一点呢，就是要学会以 Rust 的方式来思考。比如，要随时想着，这个变量是在哪里创建的，owner 是谁，谁会借用，什么时候销毁。久而久之，当这个成为了习惯，你会发现你写 Rust 会越来越得心应手，因为编译器也是这么思考的。

当然，还有最重要的一点：一万小时定律！一定要多写代码！

不用特意去死记硬背一些库，比如 std 库，你记住这些库的前提应该是你写得多用得多，而不是单纯死记硬背，不然你照样不会用。

孔子说过“学而不思则罔，思而不学则殆”，对应到 Rust 语言的学习来说也是一样的。

# 总结

我们最后做个简单总结，首先第一个问题：Rust 是什么？

-   新时代的全能型系统级语言：上得了业务，下得了内核。
-   可以和 C/C++ 无损 FFI：你的（生态）就是我的，我的还是我的。
-   安全、性能强、协作好、开发效率高。

第二个问题，为什么说开发效率高？实际上 Rust 的开发效率真的很高，特别是如果你把线上 debug 所需要的时间和成本算上。Rust 唯一的缺点是学习效率低，但这不代表开发效率低。

最后一个问题，学习 Rust 能带给你什么？

-   Rust 让你的编程经验更有价值，它足够底层提供了完全的掌控力，又提供了足够强的抽象能力，可以帮助你把你的能力发挥到极致
-   Rust 拥有丰富的生态和优秀的工程实践，可以成为你无论干什么都非常称手的工具

最后，我分享一点我自己的感受吧，Rust 会让你越用越享受，越用越想用。
