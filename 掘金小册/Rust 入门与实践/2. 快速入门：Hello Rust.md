# 安装 Rust

首先让我们来安装搭建好 Rust 开发环境。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3e847da2475049f39e097b549c37912c~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2014&h=1532&e=png&b=fdfdfd)

Rust 官方有提供一个叫做 rustup 的工具，专门用于 Rust 版本的管理，网址是：https://rustup.rs/ ，提供了很多实用的功能，推荐大家一定要使用这个工具来管理自己的 Rust 版本。

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d2181f073da44694994fd1d51759a956~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=2948&h=1906&e=png&b=f3f4f6)

而由于一些众所周知的特殊原因，rustup 官方源在中国大陆境内访问速度并不理想，因此我们推出了 https://rsproxy.cn/ 这个面向中国大陆的镜像源，通过该源为中国大陆的 Rust 用户提供快速稳定的服务。

接下来，我们先把这两行加入到你的 shell 配置文件（~/.zshrc 或者 ~/.bashrc）的末尾：

```Bash
export RUSTUP_DIST_SERVER="https://rsproxy.cn"
export RUSTUP_UPDATE_ROOT="https://rsproxy.cn/rustup"
```

然后，记得一定要 reload 或者 source 一下你的配置文件，让这两行配置生效（当然，最简单粗暴的方法就是关了这个终端再打开）。

然后，`mkdir -p ~/.cargo`创建一下文件夹，将以下几行放到你的`~/.cargo/config`文件中：

```Bash
[source.crates-io]
replace-with = 'rsproxy'

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"
[source.rsproxy-sparse]
registry = "sparse+https://rsproxy.cn/index/"

[registries.rsproxy]
index = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true 
```

然后，执行以下命令：

```Bash
curl --proto '=https' --tlsv1.2 -sSf https://rsproxy.cn/rustup-init.sh | sh
```

![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2fa1c01f87bb4a218c4e5eb507e493a6~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=810&h=428&e=png&b=06323d)

到这一步的时候，直接回车（默认）就好，然后执行一下：

```Bash
source "$HOME/.cargo/env"
```

接下来，我们来安装一下 VSCode 和 rust-analyzer，这两个是 Rust 社区中免费的主流 IDE。

VSCode 的安装方法大家直接参照官方说明进行：https://code.visualstudio.com/

打开 VSCode 后，我们按照 https://rust-analyzer.github.io/ 这里的 QuickStart 说明，在 MarketPlace 中下载并安装一下 rust-analyzer。

好了，大功告成，Rust 的开发环境安装就是这么简单又朴实无华，大家学会了么？

## Hello Rust

接下来，我们就要进入到真正的 Rust 实战中啦，是不是很期待呢～

Rust 社区使用 Cargo 来作为包管理和构建工具，因此我们之后的所有示例都会以 Cargo 作为我们使用的工具。

我们先输入以下命令，来创建我们的项目目录：

```Bash
cargo new chapter-02
```

然后，我们用 VSCode 打开这个目录，这时候 chapter-02 目录应当如下：

```Bash
chapter-02
├── Cargo.lock # Cargo 依赖版本锁定文件
├── Cargo.toml # Cargo 主要设置文件
└── src
    └── main.rs # Rust 程序入口
```

然后，我们在 main.rs 中加入以下代码（提醒一下，大家一定要跟着一起写哦）：

```Rust
use std::io; // 使用标准库中的 io 这个模块

fn main() {
    println!("Hello, world!");
    println!("Please input a number: ");
    let mut input = String::new(); // 在这里我们创建了一个新的 String，用来接收下面的输入
    io::stdin()
        .read_line(&mut input) // 读取一行
        .expect("Failed to read input!"); // 比较粗暴的错误处理
    println!("Your raw input is: {:?}.", input); // 打印输入的原始内容
    let number: i64 = input.trim().parse().expect("Input is not a number!"); // trim 把前后的空格、换行符这些空白字符都去掉，parse 将输入的字符串解析为 i64 类型，如果解析失败就报错
    println!("Your input is: {}.", number); // 打印 parse 之后的 i64 数字
}
```

好，接下来我们在 chapter-02 目录下输入 cargo run，然后输入 1024，看看输出是什么：

```Bash
➜  chapter-02 git:(main) cargo run
    Finished dev [unoptimized + debuginfo] target(s) in 0.00s
     Running `target/debug/chapter-02`
Hello, world!
Please input a number:
1024
Your raw input is: "1024\n".
Your input is: 1024.
```

好，接下来我们再试试看输入一个非数字，会发生什么：

```Bash
➜  chapter-02 git:(main) cargo run
    Finished dev [unoptimized + debuginfo] target(s) in 0.00s
     Running `target/debug/chapter-02`
Hello, world!
Please input a number:
abcd
Your raw input is: "abcd\n".
thread 'main' panicked at 'Input is not a number!: ParseIntError { kind: InvalidDigit }', src/main.rs:11:44
note: run with `RUST_BACKTRACE=1` environment variable to display a backtrace
```

完美，错误的情况我们也做好了防御！

## cargo 原理初探

刚才我们有提到过，cargo 是构建工具，那大家知道 cargo 背后的原理嘛？

我们执行一下命令试试：

```Bash
cargo clean
cargo build -v
```

这时候，可以看到一条类似这样的输出：

```Bash
Running `rustc --crate-name chapter_02 --edition=2021 ......
```

其实这才是真正编译用的命令，rustc 是 Rust 源代码真正的编译器。

我们也可以试试看手动使用 rustc 来进行编译：

```Bash
rustc src/main.rs
```

这时候，当前目录下应该会出现一个 main 的可执行文件，我们试试看直接执行：

```Bash
./main
Hello, world!
Please input a number:
1024
Your raw input is: "1024\n".
Your input is: 1024.
```

看起来和 cargo run 的效果是一样的。

其实在远古时期，那时候还没有 cargo，大家就是直接用 rustc 来编译的，这种在简单项目还可以，一旦遇到复杂项目嘛……所以这就是构建工具的意义。

这里为大家简单介绍一下这个知识点，虽然很有可能大家在以后的开发中永远不会用到，但是技多不压身嘛，而且万一以后大家 Rust 学成了，去做 Rustc 编译器开发了，也犹未可知。

# 总结

## rustup

我们来总结一下这个章节的内容，首先，我们学会了通过 rustup 工具来管理 Rust 版本，这里再教大家两个功能：

1.  可以使用 rustup default stable/beta/nightly 来切换 Rust 工具链的 channel；
1.  可以使用 rustup update 来更新 Rust 工具链版本。

## cargo

然后，我们学习了 cargo 的使用，同样的，这里也再介绍几个功能：

1.  cargo new 可以用来新建一个项目；
1.  cargo fmt 可以自动格式化代码；
1.  cargo check 可以用来检查是否编译通过；
1.  cargo clippy 用来做 lint（强烈推荐）；
1.  最后，cargo build （--release）用来编译产物，不加 --release 就是调试模式（不开优化，编译速度快），加了 --release 就是生产模式（优化火力开满，代价是编译慢）。

同时，我们还简单了解了一下 rustc 编译器，这是真正用来编译 Rust 代码的东西，一般我们通过 cargo 来调用。

## print

在这一章节里面，我们还学会了如何往控制台输出，这里也再介绍几种 print 方法：

-   打印到标准输出：print!、println!
-   打印到标准错误：eprint!、eprintln!
-   打印的两种常用格式：{} 代表 Display 格式、{:?} 代表 Debug 格式
-   更多格式参考：https://doc.rust-lang.org/std/fmt/

## 读取数据

除此之外，我们还学会了如何从标准输入（stdin）读取数据：

-   使用 use 语句引入标准库；
-   使用 stdin().read_line() 读取数据到变量；
-   使用 expect 处理错误。
