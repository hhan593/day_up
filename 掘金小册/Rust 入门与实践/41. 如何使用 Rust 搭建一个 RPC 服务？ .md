Volo 是字节跳动服务框架团队研发的轻量级、高性能、可扩展性强、易用性好的 Rust RPC 框架，使用了 Rust 最新的 GAT 特性（[Volo文档链接](https://www.cloudwego.io/zh/docs/volo/overview/)）。

## 框架代码结构

1.  `volo-gen` 目录下为代码生成配置，在项目编译时会依据它来把`thrift`或`protobuf` `idl` 文件编译为 Rust 代码。
1.  `src`目录下，`lib.rs`为**库单元包**的根节点，在`lib.rs`中，我们可以控制**模块**的私有性；
1.  `src/bin`目录下，存放了一个**二进制单元包**。`bin`目录下允许存在多个二进制单元包，每个二进制单元包都会生成单独的二进制文件。`/src/main.rs` 与 `/src/bin/` 共存时，运行时需要手动指定；
1.  `target`目录下会存放编译产物；
1.  `Cargo.toml`是配置文件，可以配置依赖、配置编译器等。
    ![](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0a775f0fd74442458ed172a6e0fe1018~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=810&h=848&e=png&b=3d3f41)

**基础知识复习：**

1. **模块**：除了`mod.rs`，每一个文件和目录都是一个模块，模块之间组成模块树，模块树帮我们把代码划分成可管理的部分，模块树外的代码或非公开的模块，将不能访问(**Err：File is not include in module tree, analysis is not available**)。对于文件夹来说，文件夹下哪些模块(文件/文件夹)需要纳入模块树，需要定义特殊文件`mod.rs`来指定。
1.  **包与单元包**：一个包中只能拥有最多一个库单元包(所以只有一个`lib.rs`)，其次包可以拥有任意多个二进制单元包(所以`/src/bin/`下可以定义多个二进制单元包)。最后包内必须存在一个单元包。

# 进阶 - 利用 Server 中间件进行错误转换

> 在第三节和第四节，我们将利用 Server 中间件和 Client 中间件，进行错误类型的传递&转换。流程：业务代码向框架抛出错误对象(Server) -> 中间件将错误对象转换为BaseResp(Server) -> 框架通信 -> 中间件将BaseResp转换为错误对象(Client) -> 业务代码消费错误对象。
>
> 本节我们将进行 Server 侧的代码实现。以中间件实现通用逻辑能够减少重复代码，利于项目维护。

  


在开始前，我们先定义一下业务函数和业务错误：

**`src/lib.rs`**

```Rust
 // func0 业务函数, 会固定返回Err(ServiceError::BizError)
fn func0() -> Result<(), ServiceError> {
   Err(TEST_ERROR_CODE.into())
}

// ServiceError 业务错误, 使用thiserror构建
#[derive(thiserror::Error, Debug)]
pub enum ServiceError {
   #[error("biz error: {0}")]
   BizError(&'static ErrCode),
   #[error(transparent)]
   Other(#[from] anyhow::Error),
}

// ErrCode 包含具体的错误码和错误信息
#[derive(Clone, Debug, Default)]
pub struct ErrCode {
   pub code: i32,
   pub message: &'static str,
}
impl std::fmt::Display for &ErrCode {
   fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
      write!(f, "biz error, code: {}, msg: {}", self.code, self.message)
   }
}
impl From<&'static ErrCode> for ServiceError {
   fn from(e: &'static ErrCode) -> Self {
      Self::BizError(&e)
   }
}
pub static TEST_ERROR_CODE: &ErrCode = &ErrCode {
   code: 12345678,
   message: "我是谁",
};
pub static INTERNAL_ERROR_CODE: &ErrCode = &ErrCode {
   code: -1,
   message: "系统错误",
};
```

**解：** 我们可以通过`metainfo`解决这个问题。

我们可以先定义一个需要人工添加的Response类型映射表。

请求进来时，我们可以利用`metainfo`注入埋点："biz_response" = "GetItemBackend"。后续我们可以就通过埋点找到真实的Response类型。

**`src/bin/server.rs`**

```Rust
impl<Cx, Req, S> volo::Service<Cx, Req> for ErrorService<S>
    where
        Req: std::fmt::Debug + Send + 'static, 
        S: Send + Sync + 'static + volo::Service<Cx, Req>,
        S::Response: std::fmt::Debug,
        S::Error: std::fmt::Debug,
        Cx: Send + 'static,
{
    type Response = S::Response;
    type Error = S::Error;
    type Future<'cx> = impl Future<Output=Result<Self::Response, Self::Error>> + Send + 'cx where S: 'cx;

    #[inline]
    fn call<'cx, 's>(&'s self, cx: &'cx mut Cx, req: Req) -> Self::Future<'cx>
        where
            's: 'cx,
    {
        async move {
            // 加入埋点
            metainfo::METAINFO. with (|metainfo| { 
 let method = cx. rpc_info ().method. clone (). unwrap_or_default (); 
 tracing::info!(  "Rust Demo method {:?}"  , method); 
 let mut info = metainfo. borrow_mut (); 
 info. set_persistent (  "biz_response"  , method); 
 }); 
                
            // 调用下游
            let resp = self.0.call(cx, req).await;
            
            // 读取埋点, 获取Response
            let biz_resp_key_option = metainfo::METAINFO. with (|metainfo| { 
 let mut info = metainfo. borrow_mut (); 
 let biz_resp_key_option = info. get_persistent (  "biz_response"  ); 
 return biz_resp_key_option
 }). unwrap (); 
 let biz_response_map_guard = BizResponseMap. read (). unwrap (); 
 let biz_resp = biz_response_map_guard. get (biz_resp_key. as_str ()). unwrap (); 
        }
    }
}
```

新问题：当`self.0.call(cx, req).await`返回`Result::Err<E>`时，`E`为关联类型`Self::Error`。怎么将`Self::Error`转为业务`ServiceError`？

**解：** 关联类型`Error`的实际类型为`anyhow::Error`，我们可以通过添加特征约束等方式，感知Error具体类型。

比如`self.0.call(cx, req).await;`的返回值是`Future::Output`，关联类型`Future::Output`被定义为`Result<Self::Response, Self::Error>`。因此，我们可以在第4行为`Error`指定类型为`volo_thrift::AnyhowError`(或者在第6行为关联类型`Error`加一个特征：`Into(volo_thrift::Error)`)，此时我们处理起关联类型`Error`就得心应手了。

**`src/bin/server.rs`**

```Rust
impl<Cx, Req, S> volo::Service<Cx, Req> for ErrorService<S>
    where
        Req: std::fmt::Debug + Send + 'static, 
        S: Send + Sync + 'static + volo::Service<Cx, Req, Error = volo_thrift::AnyhowError, Response = volo_gen::rust_demo_2::ItemBackendServiceResponseSend>, 
        S::Response: std::fmt::Debug,
        S::Error: std::fmt::Debug,
        Cx: Send + 'static,
{
    type Response = S::Response;
    type Error = S::Error;
    type Future<'cx> = impl Future<Output=Result<Self::Response, Self::Error>> + Send + 'cx where S: 'cx;

    #[inline]
    fn call<'cx, 's>(&'s self, cx: &'cx mut Cx, req: Req) -> Self::Future<'cx>
        where
            's: 'cx,
    {
        async move {
            // 加入埋点
            metainfo::METAINFO.with(|metainfo| {
                let method = cx.rpc_info().method.clone().unwrap_or_default();
                tracing::info!("Rust Demo method {:?}", method);
                let mut info = metainfo.borrow_mut();
                info.set_persistent("biz_response", method);
            });
                
            // 调用下游
            let resp = self.0.call(cx, req).await;
            match resp { 
 Err (e) => { 
  // 判断是不是ServiceError
 if let Some (service_error) = e.downcast_ref::<ServiceError>() {} 
 } 
 } 
                        
            // 读取埋点, 获取Response
            let biz_resp_key_option = metainfo::METAINFO.with(|metainfo| {
                let mut info = metainfo.borrow_mut();
                let biz_resp_key_option = info.get_persistent("biz_response");
                return biz_resp_key_option
            }).unwrap();
            let biz_response_map_guard = BizResponseMap.read().unwrap();
            let biz_resp = biz_response_map_guard.get(biz_resp_key.as_str()).unwrap();
        }
    }
```

新问题，`Response`的类型为`ItemBackendServiceResponseSend`，是一个枚举，怎么修改枚举值的`BaseResp`？

**解：** 我们可以为在生成Rust代码时，为我们关心的枚举、结构体增加一个`set_base_resp`方法，用来修改`BaseResp`。

```JSON
impl<Cx, Req, S> volo::Service<Cx, Req> for ErrorService<S>
    where
        Req: std::fmt::Debug + Send + 'static, // 特征约束
        S: Send + Sync + 'static + volo::Service<Cx, Req, Error = volo_thrift::AnyhowError, Response = volo_gen::rust_demo_2::ItemBackendServiceResponseSend>,
        S::Response: std::fmt::Debug,
        S::Error: std::fmt::Debug,
        Cx: Send + 'static + volo::context::Context,
{

    type Response = S::Response; // 关联类型
    type Error = S::Error;
    type Future<'cx> = impl Future<Output = Result<Self::Response, Self::Error>> + Send + 'cx where S: 'cx;

    #[inline]
    fn call<'cx, 's>(&'s self, cx: &'cx mut Cx, req: Req) -> Self::Future<'cx>
        where
            's: 'cx,
    {
        async move {
            metainfo::METAINFO
                .with(|metainfo| {
                    let method = cx.rpc_info().method.clone().unwrap_or_default();
                    tracing::info!("Rust Demo method {:?}", method);
                    let mut info = metainfo.borrow_mut();
                    info.set_persistent("biz_response", method);
                });

            let now = std::time::Instant::now();
            tracing::info!("Rust Demo {:?}", &req);
            let resp = self.0.call(cx, req).await;
            tracing::info!("Rust Demo response {:?}", &resp);
            tracing::info!("Rust Demo took {}ms", now.elapsed().as_millis());

            // 处理返回的结果
            return match resp {
                // 没错误不用管
                Ok(r) => {
                    Ok(r)
                }
                // 有错误, 处理错误
                Err(e) => {
                    tracing::info!("Rust Demo any error");
                    // 判断是不是ServiceError
                    if let Some(service_error) = e.downcast_ref::<ServiceError>() {
                        tracing::info!("Rust Demo service error");
                        match service_error {
                            // 业务错误, 可以转换为BaseResp
                            ServiceError::BizError(biz_error) => {
                                tracing::info!("Rust Demo biz_error {:?} {:?}", biz_error.code, biz_error.message);
                                // 查询业务Response类型
                                let biz_resp_key_option = metainfo::METAINFO
                                    .with(|metainfo| {
                                        let mut info = metainfo.borrow_mut();
                                        let biz_resp_key_option = info.get_persistent("biz_response");
                                        return biz_resp_key_option
                                    });

                                tracing::info!("Rust Demo resp type key {:?}", biz_resp_key_option);

                                match biz_resp_key_option {
                                    // 查找 Response Type Key
                                    Some(biz_resp_key) => {
                                        tracing::info!("Rust Demo find key");
                                        let biz_response_map_guard = BizResponseMap.read().unwrap();
                                        let biz_resp_option = biz_response_map_guard.get(biz_resp_key.as_str());
                                        // 查找 Response Type
                                        match biz_resp_option {
                                            Some(biz_resp) => {
                                                tracing::info!("Rust Demo begin transform!");
                                                let b = &mut biz_resp.clone();
                                                b.set_base_resp(biz_error.clone().code, biz_error.clone().message);
                                                return Ok(b.to_owned())
                                            }
                                            None => { tracing::info!("Rust Demo not found resp type"); }
                                        }
                                    }
                                    None => { tracing::info!("Rust Demo not found resp type key"); }
                                }
                            },
                            _ => tracing::info!("Rust Demo unknown error"),
                        }
                    }
                    // 无法处理的Error, 原样返回
                    return Err(e);
                }
            }
        }
    }
}
```

```Rust
impl<Cx, Req, S> volo::Service<Cx, Req> for ErrorFrontService<S>
    where
        Req: std::fmt::Debug + Send + 'static, // 特征约束 https://course.rs/basic/trait/trait.html
        S: Send + Sync + 'static + volo::Service<Cx, Req, Error = volo_thrift::Error, Response = Option<volo_gen::rust_demo_2::ItemBackendServiceResponseRecv>>,
        S::Response: std::fmt::Debug,
        S::Error: std::fmt::Debug,
        Cx: Send + 'static + volo::context::Context,
{

    type Response = S::Response; // 关联类型
type Error = S::Error;
    type Future<'cx> = impl Future<Output = Result<Self::Response, Self::Error>> + Send + 'cx where S: 'cx;

    #[inline]
    fn call<'cx, 's>(&'s self, cx: &'cx mut Cx, req: Req) -> Self::Future<'cx>
        where
            's: 'cx,
    {
        async move {
            let now = std::time::Instant::now();
            tracing::info!("Rust Front Layer Request {:?}", &req);
            let resp = self.0.call(cx, req).await;
            tracing::info!("Rust Front Layer response {:?}", &resp);
            tracing::info!("Rust Front Layer took {}ms", now.elapsed().as_millis());

            // 处理返回的结果
            return match resp {
                Err(e) => {
                    Err(e)
                }
                Ok(r) => {
                    tracing::info!("Rust Front Layer Find Resp!");
                    let r_option:Option<volo_gen::rust_demo_2::ItemBackendServiceResponseRecv> = r;
                    // Resp不见了, 默认成功
                    if r_option.is_none() {
                        return Ok(r_option)
                    }
                    let r_real = r_option.clone().unwrap();
                    let base_resp_option = r_real.get_base_resp();
                    // BaseResp不见了, 默认成功
                    if base_resp_option.is_none() {
                        return Ok(r_option)
                    }
                    let base_resp_real = base_resp_option.unwrap();
                    let status_code_option = base_resp_real.status_code;
                    // StatusCode不见了, 默认成功
                    if status_code_option.is_none() {
                        return Ok(r_option)
                    }
                    let status_code_real = status_code_option.unwrap();
                    return match status_code_real {
                        12345678 => {
                            tracing::info!("Rust Front Layer Transform!");
                            Err(volo_thrift::Error::Application(volo_thrift::ApplicationError{kind: ApplicationErrorKind::UNKNOWN_METHOD, message: "Good Morning".into()}))
                        },
                        _ => {
                            Err(volo_thrift::Error::Application(volo_thrift::ApplicationError{kind: ApplicationErrorKind::UNKNOWN_METHOD, message: "Good Evening".into()}))
                        }
                    };
                }
            }
        }
    }
}
```
