## 1. 概念

内部可变性指的是在拥有不可变引用的时候，依然可以获取到其内部成员的可变引用，进而对其数据进行修改。这时你的脑袋里是否有着大大的疑问，这不就是说可以存在多个可变引用吗，和Rust严格的借用规则相违背？其实不违背，使用内部可变性只是把编译期严格的借用检查推迟到了运行时，如果在运行时同时存在多个可变引用，程序依然会直接panic从而退出。

  


这时可能又会有一个疑问，那就是Rust一向所推崇的是把运行时的错误提前到编译期进行检查，但内部可变性显然是相反的。也许这就是语言设计上的折中，内部可变性一来是可以让我们写代码时更加灵活方便，比如当一个结构体只需要一个字段可变而其它字段不可变时，二来则是因为编译器的局限性，比如它有可能会拒绝掉原本正确的代码，这时内部可变性则可以帮助我们通过编译。

  


多说无益，下面就让我们实际来体验一下内部可变性的使用吧，看看它的效果到底如何。

  


## 2. 常见用法

Rust中通常使用Cell、RefCell以及Mutex来达到内部可变性的目的，其中Cell和RefCell用于单线程，Mutex则用于多线程中。

让我们来看看Cell和RefCell分别是如何使用的吧，在下面的代码中，对于不可变的结构体Foo，其中对不具备内部可变性的bar字段做修改时编译器会报错，而对具备内部可变性的cell_bar和refcell_bar字段则可以做修改。

```Rust
use std::cell::Cell;
use std::cell::RefCell;

struct Foo {
    bar: u8,
    cell_bar: Cell<u8>,
    refcell_bar: RefCell<String>,
}

fn main() {
    let foo = Foo {
        bar: 0,
        cell_bar: Cell::new(1),
        refcell_bar: RefCell::new(String::from("2")),
    };

    // error[E0594]: cannot assign to `foo.bar`, as `foo` is not declared as mutable
    // foo.bar = 100;

    foo.cell_bar.set(100);
    assert_eq!(foo.cell_bar.get(), 100);

    *foo.refcell_bar.borrow_mut() = String::from("200");
    assert_eq!(*foo.refcell_bar.borrow(), "200");
}
```

值得注意的是，Cell<T>需要类型T实现Copy，因此像Cell<String>这样的类型则会直接在编译期报错。因为Cell<T>直接操作的是T，每次get的时候都会直接copy一份出来，set的时候直接进行替换，使得其没有借用检查，因而不会有运行时panic。

  


相较于Cell<T>，RefCell<T>直接操作的则是&T和&mut T，通过borrow方法获得智能指针Ref<T>，实现了Deref可以当&T使用，borrow_mut方法获得智能指针RefMut<T>，实现了Deref和DerefMut可以当&T和&mut T使用，因而不会对类型T做约束，也不会有编译期问题。但在运行时RefCell<T>则会对借用进行记录和检查，保证任何时候只允许有多个不可变借用或一个可变借用存在，像下面的代码则会在运行时panic。

```Rust
use std::cell::RefCell;

// thread 'main' panicked at 'already borrowed: BorrowMutError', src/main.rs:6:16
fn main() {
    let s = RefCell::new(String::from("Hello, world"));
    let s1 = s.borrow();
    let s2 = s.borrow_mut();

    println!("{},{}", s1, s2);
}
```
