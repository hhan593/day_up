# 一、枚举(Enum 类型)

想象我们有一个装水果的箱子，这个箱子的空间是有限的，只能放进去一个水果。且我们现在有两种水果，苹果和梨，而且我们一定会往箱子里放一个水果。

那么，当我们打开这个箱子的时候，只有两种可能，要么是苹果，要么是梨。

这个模型可以由 Enum 类型来表达，我们在 rust 中这样表示 Enum 类型：

```Rust
enum FruitBox {
    Apple,
    Pear,
}
```

在前边的[章节](https://bytedance.feishu.cn/wiki/wikcniiKvPYA1kRDnBCZS5ThFve)，我们学习了结构体概念。有些同学可能会问，“我用结构体一样可以表达上述的模型，为什么要引入枚举这个概念呢？”。

那我们来看看，如果用结构体来描述上边的模型会不会有问题：

```Rust
struct FruitBox {
    pub contains_apple: bool,
    pub contains_pear: bool,
}
```

如果我要表达这个盒子里的是苹果，那么我会这么初始化：

```Rust
fn apple_in_the_box() {
    let _fruit_box = &FruitBox{contains_apple: true, contains_pear: false};
}
```

如果换成梨，则这么初始化：

```Rust
fn pear_in_the_box() {
    let _fruit_box = &FruitBox{contains_apple: false, contains_pear: true};
}
```

一切看起来都很美好，如果要判断盒子里是苹果，那么我只用判断 `_fruit_box.contains_apple == true` 如果要判断盒子里是梨，就判断 `_fruit_box.contains_pear == true`。

那么问题来了，如果这两个字段同时为 true，这个盒子里到底装的是苹果还是梨呢？

```Rust
fn what_is_in_the_box() {
    let _fruit_box = &FruitBox{contains_apple: true, contains_pear: true};
}
```

回忆我们之前的约定，盒子里只能装一个苹果，或是一个梨。所以这是一个未定义的状态，我们既不能说这个盒子里装了苹果，也不能说这个盒子里装了梨。**结构体无法在语法层面保证盒子里只能有一个水果成立。** 我们只能引入额外的校验逻辑来检查结构体的状态。

通过上述的例子，我们可以认识到引入 enum 类型的必要性，它可以用来表示变量只可能是 n 种状态中的一种的语义，且这个语义是结构体无法表示的。

  


## 基本操作

我们用 `::`操作符来初始化枚举值，下边的例子为创建一个放了苹果的箱子（第二行），以及创建一个放了梨的箱子(第三行）

```Rust
fn main() {
    let _ = FruitBox::Apple;
    let _ = FruitBox::Pear;
}
```

Rust 中的枚举是可以给各个枚举成员附上不同的信息，苹果和梨都有颜色，所以给他们俩都加上 Color 信息。苹果是非卖品，梨是可以卖出去的，所以给梨添加 Price 信息（是一个 u64 类型）：

```Rust
enum Color {
    Red,
    Yellow,
    Green,
}

struct Price(u64);

enum FruitBox {
    Apple(Color),
    Pear(Color, Price),
}
```

这个时候，我们需要这样初始化，红色的苹果，以及卖 50 块钱的绿色的梨：

```Rust
fn main() {
    let _ = FruitBox::Apple(Color::Red);
    let _ = FruitBox::Pear(Color::Green, Price(50));
}
```

这样也是 ok 的：

```Rust
enum Color {
    Red,
    Yellow,
    Green,
}

enum FruitBox {
    Apple(Color),
    Pear{color: Color, price: u64},
}

fn main() {
    let _ = FruitBox::Apple(Color::Red);
    let _ = FruitBox::Pear{color: Color::Green, price: 50};
}
```

  


# 二、match 语句 + if let + let else

在上边的章节中，我们定义了一个枚举。在这一章中，我们来学习一下如何“将水果从箱子中拿出来”。总共有三种方法，一种是 `match` 语句，一种是 `if let`,最后一种是 `let else`，每一种有每一种的优势。

```Rust
enum Color {
    Red,
    Yellow,
    Green,
}

struct Price(u64);

enum FruitBox {
    Apple(Color),
    Pear(Color, Price),
}
```

1.  ## match

考虑这样一种场景，如果箱子里的东西是苹果，那么就把它吃掉；如果是梨，就把它按照 price 来卖掉。

```Rust
fn eat_or_sell(fruit_box: FruitBox) {
    match fruit_box {
        FruitBox::Apple(_) => eat_apple(),
        FruitBox::Pear(_, price) => sell_pear(price),
    }
}
```

如果某个字段是我们不关心的，我们可以用 `_` 来代替。

需要注意一点，`match` 需要遍历所有的枚举值，否则是无法编译通过的，例如下述代码就无法编译通过：

```Rust
fn eat_or_sell(fruit_box: FruitBox) {
    match fruit_box {
        FruitBox::Apple(_) => eat_apple(),
    }
}
```

它会报错：`missing match arm: Pear(_, _) not covered`

如果枚举值的数量太多，而你只想处理关心的那几个，可以用 `_` 去做匹配：

```Rust
fn eat_or_sell(fruit_box: FruitBox) {
    match fruit_box {
        FruitBox::Apple(_) => eat_apple(),
        _ => do_nothing(),
    }
}
```

2.  ## if let

考虑这样一种场景，如果箱子里的东西是苹果，那么就把它吃掉，其他场景不做任何处理。这种场景下用 `if let`是非常合适的：

```Rust
fn eat_if_is_apple(fruit_box: FruitBox) {
    if let FruitBox::Apple(_) = fruit_box {
        eat_apple();
    }
}
```

3.  ## let else

还有一种代码中比较常见的场景，如果箱子里的东西是苹果，那么我会做一大段逻辑。这种场景用 `if let` 或者 `match` 都可以做，但是会多一层嵌套，这个时候用 `let else` 代码会好看很多：

```Rust
fn do_a_lot_thing_if_is_apple(fruit_box: FruitBox) {
    let FruitBox::Apple(apple_color) = fruit_box else {
        return;
    };
    
    do_things_1(apple_color);
    do_things_2();
    do_things_3();
    do_things_4();
}
```

  


4.  ## 小结

不知道同学们注意到没有，从枚举中取出枚举值是**安全**的。Rust 的语法确保了取出的值一定是箱子里的，且取出的东西没办法用在不该用的地方。

对于 `match` 语句来说，取出的东西只能在 `=>` 符号后边的 scope 中使用，不能在其他的 `match arm` 下，或者 `match` 的 scope 之外的地方使用。

对于 `if let` 来说，取出的值只能在后边的花括号中用。

对于 `let else` 来说，因为限定了 else 只能为 return 类型的语句，所以如果取出的值不是你想要的，函数会直接 return，无法被使用到。
