本节主要介绍两个 Rust 较新的特性，借助这些特性可以简化异步逻辑的编写。

# GAT

GAT 全称是 generic_associated_types，这个特性允许我们在关联类型上使用泛型。

```Rust
trait MakeShared {
    type Output<T>;
    fn make<T>(t: T) -> Self::Output<T>;
}

struct Local;
struct Global;

impl MakeShared for Local {
    type Output<T> = Rc<T>;

    fn make<T>(t: T) -> Self::Output<T> {
        Rc::new(t)
    }
}

impl MakeShared for Global {
    type Output<T> = Arc<T>;

    fn make<T>(t: T) -> Self::Output<T> {
        Arc::new(t)
    }
}
```

当我们需要让 Trait 实现时的关联类型定义包含泛型时，如上面的例子，就需要 GAT 这个特性。GAT 目前在 stable 中已可用。类型、生命周期和常数都可以作为泛型。

除了 fn 上的泛型类型外，GAT 也常被用作关联 `&self` 的生命周期。借助这一点，我们就可以摆脱常规异步 Trait 的 poll-style 函数定义 —— 因为我们可以返回捕获 `&self` 生命周期的 Future 了。

  


例如，在 Monoio 中，我们这么定义 Read Trait（类似 tokio 的 AsyncRead）：

```Rust
pub trait AsyncReadRent {
    type ReadFuture<'a, T>: Future<Output = BufResult<usize, T>>
    where
        Self: 'a,
        T: IoBufMut + 'a;
    type ReadvFuture<'a, T>: Future<Output = BufResult<usize, T>>
    where
        Self: 'a,
        T: IoVecBufMut + 'a;

    fn read<T: IoBufMut>(&mut self, buf: T) -> Self::ReadFuture<'_, T>;
    fn readv<T: IoVecBufMut>(&mut self, buf: T) -> Self::ReadvFuture<'_, T>;
}
```

除此之外，我们也可以定义 async 风格的 Tower Service trait：

```
pub trait Service<Request> {
    type Response;
    type Error;

    /// The future response value.
    type Future<'cx>: Future<Output = Result<Self::Response, Self::Error>>
    where
        Self: 'cx,
        Request: 'cx;

    fn call(&self, req: Request) -> Self::Future<'_>;
}
```

  


# TAIT

TAIT 全称是 type_alias_impl_trait，还有一个类似的特性叫 impl_trait_in_assoc_type。

前者允许我们 `type SomeType = impl SomeTrait` 来为某个匿名类型取别名；后者允许我们在 Trait 中做类似的事情。

这个特性让我们能够 “写出” opaque 类型和 async 语法糖生成的类型 —— 结合 GAT 我们就可以不借助 async-trait 做到它的能力。

  


例如，在 Monoio 中，我们这么实现 Read Trait：

```Rust
impl AsyncReadRent for TcpStream {
    type ReadFuture<'a, B> = impl std::future::Future<Output = crate::BufResult<usize, B>> where
        B: IoBufMut + 'a;
    type ReadvFuture<'a, B> = impl std::future::Future<Output = crate::BufResult<usize, B>> where
        B: IoVecBufMut + 'a;

    fn read<T: IoBufMut>(&mut self, buf: T) -> Self::ReadFuture<'_, T> {
        async move { ... }
    }
    fn readv<T: IoVecBufMut>(&mut self, buf: T) -> Self::ReadvFuture<'_, T> {
        async move { ... }
    }
}
```

相比 async-trait，这种写法可以避免 Box，节约运行时内存分配开销，同时允许编译器 inline 更多代码，进一步提升性能。

  


# Box<dyn Trait> 问题

对于使用了 GAT 的 trait，是不 Object Safe 的，此时不能将其封装为 Box<dyn Trait>。

但是我们仍旧可以利用 vtable 对外暴露为单一类型。

对于上述 async 版本的 Service trait，我们可以定义统一结构 BoxedService：

```Rust
pub struct BoxedService<Request, Response, E> {
    svc: *const (),
    type_id: TypeId,
    vtable: ServiceVtable<Request, Response, E>,
}
```

以及其中的 Vtable：

```Rust
struct ServiceVtable<T, U, E> {
    call: unsafe fn(raw: *const (), req: T) -> LocalBoxFuture<'static, Result<U, E>>,
    drop: unsafe fn(raw: *const ()),
}
```

之后需要为这个统一结构实现 Service trait：

```Rust
impl<Request, Response, E> Service<Request> for BoxedService<Request, Response, E> {
    type Response = Response;
    type Error = E;
    type Future<'cx> = LocalBoxFuture<'cx, Result<Response, E>>
    where
        Self: 'cx, Request: 'cx;

    #[inline]
    fn call(&self, req: Request) -> Self::Future<'_> {
        unsafe { (self.vtable.call)(self.svc, req) }
    }
}

impl<Request, Response, E> Drop for BoxedService<Request, Response, E> {
    #[inline]
    fn drop(&mut self) {
        unsafe { (self.vtable.drop)(self.svc) };
    }
}
```

最后暴露转换函数，来将任意的实现 Service 的类型包装为 BoxedSevice：

```Rust
pub trait BoxService<Request, Response, E> {
    fn into_boxed(self) -> BoxedService<Request, Response, E>;
}

impl<T, Request, Response, E> BoxService<Request, Response, E> for T
where
    T: Service<Request, Response = Response, Error = E> + 'static,
    Request: 'static,
{
    fn into_boxed(self) -> BoxedService<Request, Response, E> {
        BoxedService::new(self)
    }
}
```
