## 函数泛型

在我们编写代码时可能面临这样的需求，我们需要格式化输出`&[i32]`。

那我们可以实现一个这样的函数

```Rust
fn print_i32_slice(els: &[i32]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}
```

好的，问题解决了，去下一章吧……

但是突然第二天，我们又遇到了一个新的需求，需要我们以同样的格式来输出一个`&[&str]`

那么我们可以把上面的代码复制一遍，稍微改一改。

```Rust
fn print_str_slice(els: &[&str]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}
```

这两个功能明明有着相同的代码，仅仅只是参数类型不一样。是否存在什么办法能够让我们写一个这样的函数能够支持很多很多类型呢？

那么这个时候我们就可以使用泛型来解决这个问题了。

我们可以用一个泛型 `T` 来代替我们这里的 `i32` 和 `&str`，那么我们可以把我们的代码改成这样。

```Rust
fn print_slice<T>(els: &[T]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}
```

然后我们就会收到一个编译报错

```Rust
 --> src/main.rs:3:31
  |
3 |         println!("{:?} @ {}", el, i)
  |                               ^^ `T` cannot be formatted using `{:?}` because it doesn't implement `Debug`
  |
  = note: this error originates in the macro `$crate::format_args_nl` which comes from the expansion of the macro `println` (in Nightly builds, run with -Z macro-backtrace for more info)
help: consider restricting type parameter `T`
  |
1 | fn print_slice<T: std::fmt::Debug>(els: &[T]) {
  |                 +++++++++++++++++
```

编译器告诉我们类型 `T` 无法使用`{:?}` 来进行输出，而只有实现了 `std::fmt::Debug` 的类型才能使用`{:?}`进行输出。

那么我们只需要给 `T` 增加一下约束 `T: std::fmt::Debug` 就可以实现我们的功能了。

```Rust
fn print_slice<T: std::fmt::Debug>(els: &[T]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}
```

  


## 结构泛型

我们可以像函数那样给我们的结构体增加一些泛型，比如下面这样

```Rust
#[derive(Clone)]
struct Pair<X, Y>(X, Y);

let pair_a = Pair(1, 2);
let pair_b = Pair("hello", "world");
```

我们可以使用任意类型来构造结构`Pair`。现在我们需要实现一个函数能够把一个 `Pair` 数组转换成一个`HashMap` 那么我们可以如何实现呢？

首先按照我们上面学到的函数泛型和现在学到的结构泛型融合起来，我们就可以写出这样的一个函数了。

```Rust
fn pairs_to_map<X, Y>(pairs: &[Pair<X, Y>]) -> HashMap<X, Y> {
    let mut map = HashMap::with_capacity(pairs.len());

    for p in pairs {
        map.insert(p.0, p.1);
    }
    map
}
```

我们尝试编译一下，却收到了很多编译器错误。

```Rust
error[E0277]: the trait bound `X: Eq` is not satisfied
   --> src/main.rs:15:13
    |
15  |         map.insert(p.0, p.1);
    |             ^^^^^^ the trait `Eq` is not implemented for `X`
    
    
 error[E0277]: the trait bound `X: Hash` is not satisfied
   --> src/main.rs:15:13
    |
15  |         map.insert(p.0, p.1);
    |             ^^^^^^ the trait `Hash` is not implemented for `X`
```

原来`HashMap`的 `insert` 方法需要 `key: Hash + Eq`，这里的符号 `+` 的意思就是需要同时满足这些traits

那么我们就需要给我们的类型 `X` 增加一下约束了，`X: Hash + Eq`。

改好了之后，但是我们仍然收到了编译器的报错

```Rust
  --> src/main.rs:17:20
   |
17 |         map.insert(p.0, p.1);
   |                    ^^^ move occurs because `p.0` has type `X`, which does not implement the `Copy` trait
```

我们这里的 `p` 的类型是 `&Pair<X, Y>`，`p.0` 的类型也只是 `X`, 我们只有通过`clone` 才能得到类型`X`，所以我们这里还需要加上一些约束来进行Clone `X:Clone, Y:Clone`

```Rust
fn pairs_to_map<X: Eq + Hash + Clone, Y: Clone>(pairs: &[Pair<X, Y>]) -> HashMap<X, Y> {
    let mut map = HashMap::with_capacity(pairs.len());

    for p in pairs {
        map.insert(p.0.clone(), p.1.clone());
    }
    map
}
```

  


## 泛型是否会引入性能开销？

TLDR: 不会

首先我们需要理解 Rust 中的泛型是如何被编译器处理的。

比如我们存在这样的一个泛型函数

```Rust
fn print_slice<T>(els: &[T]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}

print_slice(&[1, 2, 3]);
print_slice(&["hello", "world"])
```

这里我们分别对 `&[i32]` 和 `&[&str]` 两种类型进行了 `print_slice` 调用, 而编译器会检察到这两种类型的调用。所以编译器的 `monomorphized` 阶段会生成对应的函数并且进行替换，最后我们的代码可能就被编译器转换成下面这样

```Rust
fn print_slice_i32(els: &[i32]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}

fn print_slice_str(els: &[str]) {
    for (i, el) in els.iter().enumerate() {
        println!("{:?} @ {}", el, i)
    }
}


print_slice_i32(&[1, 2, 3]);
print_slice_str(&["hello", "world"])
```

而这样就不会有任何额外的性能开销了，完全在编译时期做了替换转换成了直接调用。
