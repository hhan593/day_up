在这一章节中，我们介绍两种最为常用的类型，Vec<T> 和 HashMap<K, V>。通过对这两个样例的学习，相信同学们可以很容易地将知识迁移到其他的集合类型，例如 BTreeMap<K, V> 等。

# 动态数组: `Vec<T>`

## 1. 初始化

Rust 中的 `Vec<T>` 与 C++ 中的 `std::vector`, Go 中的 `[]T` **类似，都是长度可动态增长的列表。

`Vec<T>` 中的 `T` 是泛型参数，对应数组的元素类型。同学们可以在[13. 泛型](https://juejin.cn/book/7269676791348854839/section/7271121450592174141)中学习到关于泛型的知识。

`Vec<T>`有三种常见初始化方式，分别是：

```Rust
fn init_vec() {
    let _: Vec<i32> = Vec::new();
    let _: Vec<i32> = Vec::with_capacity(16);
    let _ = vec!["hello", "world"];
}
```

-   第一种为初始化一个长度为零的数组，且不进行内存分配。
-   第二种同样会初始化一个长度为零的数组，但是会提前分配好 16 个元素大小的内存(len = 0, capacity=16)。
-   第三种为直接用`vec!`宏的方式创建数组，例子中新建了一个拥有 2 个元素的数组，元素分别是 "hello" 和 "world"

因为 Rust 有类型推导的能力，所以这么写也是可以的：

```Rust
fn this_also_works() {
    let mut v = Vec::new(); // 这里可以先不指定类型
    v.push("str");          // 在这里，编译器会通过元素的类型确定 v 的类型，为 Vec<&str>
}
```

当然，这样写也是可以的，直接指定调用哪个泛型参数的方法：

```
fn this_also_works2() {
    let _ = Vec::<String>::new();
}
```

## 2. 访问&&修改数组元素

跟 C++ 和 Go 类似，我们可以通过下标`[i]`的方式访问数组元素，如果`i`超过数组长度，程序直接 panic。

```Rust
fn get_by_index() {
    let mut v = Vec::new();
    v.push(1);
    v.push(2);
    
    println!("{}", v[1]); // 输出 2
    
    println!("{}", v[2]); // 这里程序会直接退出
}
```

那有没有不会 panic 的方式呢？有的，我们可以通过`.get` 的方法，该函数的返回的是`Option<&T>`，如果数组越界了，会返回 None。

```Rust
fn get_by_index() {
    let mut v = Vec::new();
    v.push(1);
    v.push(2);
    
    println!("{:?}", v.get(1)); // 输出 Some(2)
    
    println!("{:?}", v.get(2)); // 输出 None
}
```

不过因为会多一次检查，这种方式性能会差一点，同学们可以根据使用场景自行抉择。

通过下标，我们也可以修改数组元素：

```Rust
fn modify_by_index() {
    let mut v = Vec::new();
    v.push(1);
    v.push(2);
    
    v[0] = 100;
    println!("{}", v[0]); // 会输出 100
}
```

## 3. 添加元素

我们可以通过`.push`方法添加一个元素。

```Rust
fn add_elem() {
    let mut v = Vec::new();
    v.push("hello");
    v.push("world");
}
```

## 4. 删除元素

我们可以通`.pop`将数组的最后一个元素弹出来，如果数组为空，则返回 None

```Rust
fn pop_elem() {
    let mut v = vec!["Hello", "Rust"];
    println!("{:?}", v.pop()); // 输出 Some("Rust")
}
```

我们还可以通过`.remove`移出指定下标的元素，且该方法会保持数组元素原来的顺序。也可以通过`.swap_remove`来移除，不过这个会将数组最后一个元素移到被删除的位置，不能保持数组元素原来的顺序。

```
fn remove_elem() {
    let mut v = vec!["Hello", "Rust"];
    v.remove(1);
    println!("{:?}", v); // 输出 ["Hello"]
}
```

## 遍历元素

遍历元素要特别注意所有权， `for`是会消耗变量的。

```Rust
fn will_consume_v() {
    let v = vec!["Hello", "World"];
    for elem in v {
        println!("{elem}");
    }
    
    println!("{:?}", v); // 这行将会报错，因为我们在第三行的 for 已经将 v 给消耗掉了
}
```

为了避免被消耗掉`v`，我们可以对`v`取引用，这样拿到的`elem`则会是`&T`类型：

```Rust
fn willnot_consume_v() {
    let v = vec!["Hello", "World"];
    for elem in &v {
        println!("{elem}");
    }
    
    println!("{:?}", v); // 这行会正常输出
}
```

如果想对元素进行修改，可以对`v`取可变引用

```Rust
fn modify_elem() {
    let mut v = vec!["Hello", "World"];
    for elem in &mut v {
        *elem = "elem"
    }
    
    println!("{:?}", v); // 这行会输出 ["elem", "elem"]
}
```

可能会有同学会好奇，`for ... in ...` 是只能用在`Vec<T>`上吗？其实不是的，通过后边的[19. 迭代器](https://juejin.cn/book/7269676791348854839/section/7271123557210390588) 章节，我们会学到，所有实现了`IntoIterator`trait 的数据结构都可以搭上`for ... in ...`的东风。

# HashMap<K, V>

## 1. 初始化

Rust 中的`HashMap<K, V>`与 C++ 中的`std::unordered_map`或者 Go 中的`map[K]V`类似。可以通过 key 找到对应的 value。

跟上边的`Vec<T>`类似，`HashMap<K, V>`也可以通过`new`和`with_capacity`来初始化，不同于`Vec<T>`，`HashMap`需要手动导入。

```Rust
use std::collections::HashMap;

fn init_hashmap() {
    let _: HashMap<String, i64> = HashMap::new();
    let _: HashMap<String, i64> = HashMap::with_capacity(16);
}
```

`HashMap`没有官方实现的，类似`vec!`一样的宏。不过 crates.io 上有类似的实现，而且自己实现一个也不难，在学习完成[24. 宏](https://juejin.cn/book/7269676791348854839/section/7271123043371384893)的声明宏章节之后，可以试着自己实现一个作为练习。

## 2. 添加元素

我们可以通过`.insert`来添加元素

```Rust
use std::collections::HashMap;
fn add_elem() {
    let mut map = HashMap::new();
    map.insert("I eat".to_string(), "apple".to_string());
    map.insert("You eat".to_string(), "pear".to_string());
}
```

## 3. 访问元素

我们可以通过`.get`方法来通过 key 访问到对应的 value。其返回值为`Option<&V>`，如果这个 key 不存在，value 为 None

```Rust
use std::collections::HashMap;
fn get_elem() {
    let mut map: HashMap<String, String> = HashMap::new();
    map.insert("I eat".to_string(), "apple".to_string());
    map.insert("You eat".to_string(), "pear".to_string());
    
    println!("{:?}", map.get("I eat")); // 会输出 Some(apple)
}
```

`.get`的输入为`&Q`，且需要满足 `K: Borrow<Q>`以及`Q: Hash + Eq`。上边的例子中，`.get`的入参为`&str`（即`&Q`类型)，`K`类型为 HashMap 的 key，即`String`。`Borrow`是一个 trait，trait 是什么可以在[14. 特性](https://juejin.cn/book/7269676791348854839/section/7271122195642843197) 这一章节了解到。`K: Borrow<Q>`限制了`Q`一定要为一个可以从`String`类型通过`Borrow`这个 trait 转换过去的类型，`str`满足要求。`Q: Hash + Eq`还限制了`Q`一定要满足可哈希以及可判断是否相等，`str`也满足要求。

## 4. 修改元素

当`.insert`的 key 存在时，会更新 value。

还有一种比较常用的场景，例如我们会先判断 value 是否为苹果，只有为苹果的时候，我才会更新为梨，这个时候，我们用 `.entry` 更合适：

```Rust
use std::collections::HashMap;

fn modify_elem_if_is_apple() {
    let mut map = HashMap::new();
    map.insert("I eat".to_string(), "apple".to_string());
    map.insert("You eat".to_string(), "pear".to_string());

    map.entry("I eat".to_string()).and_modify(|fruit| {
        if fruit == "apple" {
            *fruit = "pear".to_string();
        }
    });

    println!("{map:?}"); // 会输出：{"I eat": "pear", "You eat": "pear"}
}
```

## 5. 删除元素

我们可以通过`.remove`删除指定的 kv。

```Rust
use std::collections::HashMap;
fn remove_elem() {
    let mut map = HashMap::new();
    map.insert("I eat".to_string(), "apple".to_string());
    map.insert("You eat".to_string(), "pear".to_string());
    
    map.remove("You eat");
    println!("{:?}", map.get("You eat")); // 会输出 None
}
```

## 6. 遍历元素

如同上边的`Vec<T>`一样，我们通过 `for ... in ...`遍历hashmap（有一点要注意，hashmap 不保证遍历的顺序，如果需要保证遍历顺序，`可以用 BTreeMap<K, V>`）：

```Rust
use std::collections::HashMap;
fn iter_map() {
    let mut map = HashMap::new();
    map.insert("I eat".to_string(), "apple".to_string());
    map.insert("You eat".to_string(), "pear".to_string());
    
    for (k, v) in map {
        println!("{k} {v}");
    }
    // 将会输出
    // You eat pear
    // I eat apple
}
```

为了不消耗掉所有权，可以用`&map`或者`&mut map`去遍历。不过相应的，遍历元素的类型也会变成引用。
