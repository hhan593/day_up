结构体是一种常见的数据类型。它能够将多个相关的数据关联到一个组里，从而使得代码更加有组织。在这一节中，我们会给大家介绍一下Rust中的结构体。

  


# 基本操作

## 定义一个结构体

  


下面的代码中，我们定义了一个结构体：

```Rust
struct Point {
    x: f64,
    y: f64,
}
```

  


可以看到，要定义一个结构体，需要使用`struct`关键字。接着是结构体的名字`Point`，后面再跟着一个Block，里面定义结构体的字段。上例中，结构体`Point`包含两个字段`x`和`y`，都是`f64`类型。

  


## 创建结构体实例

定义好了结构体，要实际使用的话还需要创建一个结构体的实例。以上面的结构体`Point`为例，实例化这个结构体应该这样写：

```Rust
let point = Point { 
    x: 3.0, 
    y: 4.0 
};
```

  


通过这样的写法，就可以将`point`中的`x`初始化为3.0，`y`初始化为4.0。这里需要注意两点：

1.  初始化实例的时候，每个字段都需要初始化
1.  初始化时的字段顺序不需要要和结构体定义时保持一致

  


### 简化结构体创建

来看下面这个例子：

```Rust
fn create_point(x: f64, y: f64) -> Point {
    Point {
        x: x,
        y: y,
    }
}
```

  


注意到上面的例子中，`Point`的字段名和初始化使用的变量名刚好相同，都是`x`和`y`。在这种情况下，Rust为我们提供了一种更加简便的写法：

```Rust
fn create_point(x: f64, y: f64) -> Point {
    Point {
        x,
        y,
    }
}
```

  


变量名和字段名相同的时候，可以省略掉字段名。也即`x: x`可以简写成`x`。

  


## 访问结构体字段

要访问上面的结构体实例`point`中的字段，可以使用下面的写法：

```Rust
println!("The coordinates of the point are ({}, {})", point.x, point.y);
```

  


我们通过使用点号`.`来访问`point`中的字段。

  


## 更新字段的值

在默认情况下，结构体实例内的所有字段都是不可变的。如果想要更新字段的值，结构体实例必须是可变的。例如下面的代码：

```Rust
let mut point = Point { x: 3.0, y: 4.0 };
point.x = 5.0;
point.y = 6.0;
```

  


值得注意的是，Rust只能够将整个实例标记为可变的，而不支持将结构体内部某个字段标记成可变的。

  


## 从其他实例创建实例

有一种常见的场景是，我们需要利用一个已经存在的实例，来创建一个新的实例。例如下面这段代码：

```Rust
struct User {
    username: String,
    email: String,
    age: u8,
}

fn main() {
    let user1 = User {
        username: String::from("Alice"),
        email: String::from("alice@example.com"),
        age: 18,
    };
    
    let user2 = User {
        username: user1.username,
        email: String::from("alice2@example.com"),
        age: user1.age,
    };
}
```

  


在这段代码中，我们首先创建一个一个`user1`实例。接着我们需要创建一个`user2`实例，除了`email`字段之外，其他所有字段都和`user`相同。在创建`user2`的时候依旧还是像`user1`一样，一个一个字段在赋值。这样的写法有些繁琐，可以简化成下面的写法：

```Rust
fn main() {
    // --snip--

    let user2 = User {
        email: String::from("alice2@example.com"),
        ..user1
    };
}
```

  


通过`..`的语法，可以将`user2`没有显式赋值的字段值设置为和`user1`一样。

  


还记得上一节讲过的所有权吗？不管是采用这里的哪种方式来初始化`user2`，都认为发生了赋值操作。`user1`中的`username`的所有权会转移到`user2`当中。注意这里的`age`类型是`u8`，会发生自动拷贝。因此，在`user2`实例化之后，`user1`中的`username`字段不再可用，但是`email`字段和`age`字段依旧可用。

  


当然了，结构体当中也可存储引用类型`&str`而不是`String`。不过要使用引用类型，就需要用到一个我们目前还没有讲过的概念——生命周期。在后面的章节中，我们会学习关于生命周期的知识。

  


# 元组结构体

  


Rust支持结构体的字段没有名称，这种情况下会长得很像元组，因此叫做元组结构体。看下面的例子：

```Rust
struct Color(u8, u8, u8);

let red = Color(255, 0, 0);
```

  


可以通过`red.0`，`red.1`等这样的方式来访问字段的值。

  


# 单元结构体

  


Rust的中的结构体字段不但可以没有名称，甚至可以连字段都没有。这种结构体很像我们之前讲过的单元类型`()`，因此也被称为单元结构体。例如：

```Rust
struct Marker;

fn main() {
    let marker = Marker;
}
```

  


这里的`Marker`就是一个单元结构体。当对于一个类型我们只关心行为而不关心它的内容时，就可以用单元结构体。

  


# 函数与方法

  


在前面的章节中，我们已经学过了什么是函数。方法和函数有些类似的地方：它们都由关键字`fn`定义，都有一个名字，都可以有参数和返回值。方法与函数的不同之处在于，方法是定义在结构体的上下文中，第一个参数永远是`self`，用来表示方法被调用的结构体实例。

  


注：方法也可以用在枚举类型和特征中，这两点会在后面的章节介绍。

  


## 定义一个方法

  


我们通过一个例子来看一下方法是怎么定义的：

```Rust
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}
```

  


在上面的例子中，我们首先定义了一个结构体`Rectangle`。它包含两个字段`width`和`height`。接着我们通过`impl`关键字来为`Rectangle`结构体添加方法。我们添加了一个`area`方法，它唯一的参数是`&self`。注意这里的`&self`实际上是`self: &Self`的简写（注意s的大小写）。在`impl`块中，`Self`就代表与这个块相关联的类型，也即`Rectangle`类型。

  


注意到这里的`&self`实际上是一个`Rectangle`类型的不可变引用。这就意味着：

1.  我们没有获取`self`的所有权。关于方法的所有权我们等下会讲
1.  我们不能修改结构体当中的值

  


## 调用方法

  


方法定义好了之后需要调用。我们来看下这个例子：

```Rust
fn main() {
    let rect = Rectangle { width: 10, height: 20 };
    let area = rect.area();
    println!("The area of the rectangle is {}", area);
}
```

  


在上面的例子中，我们首先创建了一个`Rectangle`结构体的实例`rect`。接着在`rect`上直接调用`area`方法。有同学会问，`area`方法不是有一个参数`&self`吗，调用的时候怎么不需要传参？这是因为在Rust中，方法定义的时候第一个参数固定是`self`（当然可能是`self`, `&self`和`&mut self`这几种类型来表示所有权的情况）。那么在调用方法的时候，Rust就能根据方法定义时使用的具体形式，来将这里的结构体实例`rect`自动匹配成相应的类型（`rect`,`&rect`或者`&mut rect`）来进行调用。因此就不需要我们手动来写了。

  


## 所有权转移

  


上面已经提到，定义方法的时候第一个参数`self`有三种类型：`self`,`&self`和`&mut self`。我们这里再做一些详细的介绍。

  


当使用`self`时，结构体实例`rect`的所有权就转移到了方法内部。这也就意味着调用完成之后，`rect`将无法再使用。通常情况下我们不会使用这种形式来定义方法。

  


当使用`&self`和`&mut self`时，表示传入方法的是不可变借用或者可变借用。这种情况下方法内可以正常使用`rect`中的字段，同时也没有获取`rect`的所有权。

  


**我们来举一个完整的例子来说明这里提到的几种类型：**

```Rust
struct Person {
    name: String,
}

impl Person {
    // Immutable method
    fn greet(&self) {
        println!("Hello, my name is {}", self.name);
    }

    // Mutable method
    fn change_name(&mut self, new_name: String) {
        self.name = new_name;
    }

    // Consuming method
    fn get_name_and_consume(self) -> String {
        self.name
    }
}

fn main() {
    let mut person = Person {
        name: String::from("Alice"),
    };

    // Call the immutable method
    person.greet();

    // Call the mutable method
    person.change_name(String::from("Bob"));
    person.greet();

    // Call the consuming method
    let name = person.get_name_and_consume();
    println!("The person's name was: {}", name);

    // The following line would cause a compile error because `person` has been consumed
    // person.greet();
}
```

  


在上面的例子中，我们定义了一个结构体类型`Person`，它只有一个`String`类型的字段`name`。接着我们给`Person`添加了三个方法，分别是：

-   使用`&self`的`greet`
-   使用`&mut self`的`change_name`
-   使用`self`的`get_name_and_consume`

  


在`main`函数中，我们先是创建了可变的`person`实例，接着分别调用它的方法。可以看到`greet`和`change_name`方法可以交替调用，因为他们没有获取所有权。而当调用了`get_name_and_consume`之后，`person`实例就不再可用了。

  


## 关联函数

  


在`impl`块中，除了可以定义方法，还可以定义函数。这些函数的第一个参数不是`self`，因此就不能像`person.greet()`这样来调用。又因为函数定义在了`impl`块中，我们认为这个函数和结构体关系密切，因此称它为关联函数。

  


最常见的关联函数的例子就是结构体的构造方法。看下面的例子：

```Rust
impl Rectangle {
    fn new(w: u32, h: u32) -> Rectangle {
        Rectangle { width: w, height: h }
    }
}

fn main() {
    let sq = Rectangle::new(3,3);
}
```

  


这里我们就创建了一个名为`new`的构造函数。注意这里使用`new`只是一个约定俗成的用法，并不是强制的规定。因为`new`不是一个方法，因此不能像方法那样使用`.`来调用。要调用这个`new`，需要像上例中一样使用`::`语法。
