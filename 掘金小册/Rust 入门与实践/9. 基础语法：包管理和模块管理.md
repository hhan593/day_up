# 包管理器：Cargo

> Cargo 是Rust 语言的包管理器。Cargo 可以
>
> 1.  下载你的 Rust 包的依赖项
> 1.  编译你的包
> 1.  创建可分发的包，并将它们上传到crates.io（即Rust社区的包注册表）
>
> -- from [Cargo 官方文档](https://doc.rust-lang.org/cargo/)

现在让我们来通过实际操作感受一下如何使用 Cargo 吧！

  


 ## 1. 项目初始化

首先我们新建一个文件夹`my_project`，然后为这个文件夹初始化一个 Rust 项目吧！

```Bash
>> mkdir my_project
>> cd my_project
>> cargo init
```

执行完上述三个命令之后，Cargo 会为`my_project`文件夹初始化一个 bin 类型的 Rust 项目（即默认编译目标为一个可执行的二进制文件）。我们可以看一下`my_project`文件夹里的内容：

```Bash
my_project
├── Cargo.toml
└── src
    └── main.rs
```

其中`src/main.rs`中的`main`函数为整个程序的入口。`Cargo.toml`为这个项目的配置文件，可以配置包括依赖的包，编译选项等，但因为这个项目是一个新的项目，目前除了基本信息外没有其他内容。

我们用`cargo run`运行一下这个项目，成功输出 Hello, world!

```Bash
>> cargo run
   Compiling my_project v0.1.0 (/Users/bytedance/rust/my_project)
    Finished dev [unoptimized + debuginfo] target(s) in 0.99s
     Running `target/debug/my_project`
Hello, world!
```

如果想要初始化一个库类型的项目，供其他使用方使用（例如发布到 crates.io 上），在初始化的时候需要运行

```Shell
>> cargo init --lib
```

它会在`src`文件夹下新建一个`lib.rs`的文件作为整个库的入口。

  


## 2. 写一个小的程序

接下来让我们写一个小的程序：我们随机从 0,1 中取一个数，如果是 0，就输出 Hello，如果是 1 就输出 World。

我们首先去 [crates.io ](https://crates.io/)上找一下有没有随机相关的库可以用。搜索一下就可以发现，有一个使用量非常多的库 [rand](https://crates.io/crates/rand)。

我们有两种方式可以将这个库加入到我们的项目中来：

第一种是运行

```Bash
>> cargo add rand
```

第二种是直接编辑`Cargo.toml`，在`[dependencies]`这一栏下增加：

```Bash
rand = "0.8.5"
```

等于号后边是`rand`这个库的版本号。

引入了依赖之后，我们就可以写我们的代码了，编辑`src/main.rs`文件：

```Rust
use rand::Rng;

fn main() {
    let gen = rand::thread_rng().gen::<i64>() % 2;
    if gen == 0 {
        println!("Hello");
    } else {
        println!("World");
    }
}
```

其中第一行`use rand::Rng`在后边的小节会讲到，这个用来引入一个 trait，否则`.gen`方法是找不到的。

接着`cargo run`运行程序：

```Rust
>> cargo run
   Compiling cfg-if v1.0.0
   Compiling ppv-lite86 v0.2.17
   Compiling libc v0.2.147
   Compiling getrandom v0.2.10
   Compiling rand_core v0.6.4
   Compiling rand_chacha v0.3.1
   Compiling rand v0.8.5
   Compiling my_project v0.1.0 (/Users/bytedance/rust/my_project)
    Finished dev [unoptimized + debuginfo] target(s) in 2.41s
     Running `target/debug/my_project`
Hello
```

  


# 模块管理

当项目非常大的时候，我们需要一种将项目代码根据不同功能划分的方法，且各个功能之间的访问是可以自主控制的。这个方法在 Rust 中就是模块。

通过 Rust 的模块管理，我们可以做到：

1.  组织项目代码
1.  可见性控制

接下来我们通过一个小例子来熟悉一下如何操作吧，继续使用我们在上一节中的`my_project`。

## 1. 模块是一个树形结构

还记得我们上一节提到的`src/main.rs`和`src/lib.rs`吗？这两个其实就是包的根结点(crate root)，它们对应两个 crate。

模块构成的树是以包的根节点为根节点构成的树。将来同学们在开发的时候可能会遇到，明明添加了一个模块，但是不管加任何内容就是没有 IDE 提示。这个有可能是因为新添加的这个模块没有被加入到模块树中去（即被`src/main.rs`或`src/lib.rs`直接或间接依赖）。

## 2. 文件作为模块

拿出我们的`my_project`项目，我们现在想添加两个模块，其中一个叫`apple`模块，另一个叫`pear`模块。

我们需要两步:

第一步，在`src/`文件夹下添加两个文件，`apple.rs`和`pear.rs`

```Shell
>> cd src/
>> touch apple.rs
>> touch pear.rs
```

创建之后，我们的`src`文件夹结构为：

```Rust
src
├── main.rs
├── apple.rs
└── pear.rs
```

第二步，将这两个模块加入到模块树中，编辑`src/main.rs`，在文件开头添加这么两行：

```Rust
mod apple;
mod pear;
```

这样，模块树就构建好了，接下来我们可以给`apple`和`pear`模块添加内容，供`main`函数调用了：

我们分别给`apple`和`pear`添加：

```Rust
pub fn eat_apple() {
    println!("I eat apple");
}
```

```Rust
pub fn eat_pear() {
    println!("I eat pear");
}
```

其中，`pub`关键词控制这个东西能不能被该模块外的模块见到。

然后修改我们的`src/main.rs`为，我们通过`::`操作符拿模块中的东西：

```Rust
mod apple;
mod pear;
use rand::Rng;

fn main() {
    let gen = rand::thread_rng().gen::<i64>() % 2;
    if gen == 0 {
        apple::eat_apple();
    } else {
        pear::eat_pear();
    }
}
```

到项目根目录，运行`cargo run`

```Rust
>> cargo run
   Compiling my_project v0.1.0 (/Users/bytedance/rust/my_project)
    Finished dev [unoptimized + debuginfo] target(s) in 0.45s
     Running `target/debug/my_project`
I eat pear
```

## 3. 文件夹作为模块

如果把所有模块都平铺到`src/`文件夹下的话，项目一大文件就会非常多。所以我们需要一个通过文件夹组织模块的方法。如果要将文件夹作为模块，在文件夹下一定要有一个`mod.rs`的文件。这个文件类似`src/main.rs`或者`src/lib.rs`，该文件(`mod.rs`)控制该文件夹下所有其他文件模块的引入。

我们动手试试。

新建一个文件夹`orange`，在其下新建一个文件`mod.rs`

```Rust
>> cd src/
>> mkdir orange
>> cd orange
>> touch mod.rs
```

然后我们需要把这个新的文件夹模块添加到模块树中，编辑`src/main.rs`，在文件开头新增

```Rust
mod orange;
```

这样，文件夹模块就新增好了。

接下来我们趁热打铁，为`orange`添加一个子模块`eat`

```Rust
>> cd src/orange
>> touch eat.rs
```

别忘了给`mod.rs`添加一行(这里需要用`pub`关键词，否则`main.rs`无法“看到”`orange`内部的`eat`模块)

```Rust
pub mod eat;
```

接着，我们编辑`src/orange/eat.rs`

```Rust
pub fn eat_orange() {
    println!("I eat orange");
}
```

最后，我们就可以在`src/main.rs`中引用这个函数了：

```Rust
mod apple;
mod orange;
mod pear;
use rand::Rng;

fn main() {
    let gen = rand::thread_rng().gen::<i64>() % 2;
    if gen == 0 {
        apple::eat_apple();
    } else {
        pear::eat_pear();
    }

    orange::eat::eat_orange();
}
```

## 4. 文件内新建子模块

在文件内如果想新建模块的话，可以用`mod`关键字，且这个是可以嵌套的

```Rust
mod fruit {
    mod apple {
        pub fn eat_apple() {
            println!("I eat apple");
        }
    }
    
    mod pear {
        pub fn eat_pear() {
            println!("I eat pear");
        }       
    }
    
    mod orange {
        pub fn eat_orange() {
            println!("I eat orange");
        }               
    }
}
```
