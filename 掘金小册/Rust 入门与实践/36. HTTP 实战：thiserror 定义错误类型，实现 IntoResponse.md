3.  定义错误类型

因为暂时还没具体的业务逻辑，先随便定义几个错误类型 `NotImplemented`、`NotFound`、`IO` 用于演示

-   `#error` 用于设置 .to_string() 的内容
-   加上 `#[from]` 表示自动 `impl From<std::io::Error> for DemoError`，如果错误类型是 `srd::io::Error`，通过 `?` 返回错误时可以自动转换成 `HelloError`，也可以使用 `.map_err(Into::into)` 自动转换错误类型

  


src/error.rs

```Rust
use thiserror::Error;

// 为了使用方便，可以用 HelloResult<T> 代替 Result<T, HelloError>
pub type HelloResult<T> = Result<T, HelloError>;

#[derive(Error, Debug)]
pub enum HelloError {
    #[error("not implemented")]
    NotImplemented,
    #[error("not found")]
    NotFound,
    // #[from] 表示自动 impl From<std::io::Error> for HelloError
    // 如果错误类型是 srd::io::Error，通过 ? 返回错误时可以自动转换成 HelloError
    #[error("io error: {0}")]
    IO(#[from] std::io::Error),
}
```

  


4.  为 HelloError 实现 IntoResponse `impl IntoResponse for DemoError`

> 根据之前章节中介绍的规则：如果 `T` 和 `E` 都实现了 `IntoResponse`，`Result<T, E>` 自动实现了 `IntoResponse`
>
> `HelloError` 已经实现了 `IntoResponse`，所以只要 `T` 实现了 `IntoResponse`，`HelloResult<T>`就可以作为 handler 的返回类型

```Rust
use axum::http::StatusCode;
use axum::response::{IntoResponse, Response};
use thiserror::Error;

// 为了使用方便，可以用 HelloResult<T> 代替 Result<T, HelloError>
pub type HelloResult<T> = Result<T, HelloError>;

#[derive(Error, Debug)]
pub enum HelloError {
    #[error("not implemented")]
    NotImplemented,
    #[error("not found")]
    NotFound,
    // #[from] 表示自动 impl From<std::io::Error> for HelloError
    // 如果错误类型是 srd::io::Error，通过 ? 返回错误时可以自动转换成 HelloError
    #[error("io error: {0}")]
    IO(#[from] std::io::Error),
}

impl IntoResponse for HelloError {
    fn into_response(self) -> Response {
        let code = match self {
            Self::NotImplemented => StatusCode::NOT_IMPLEMENTED,
            Self::NotFound => StatusCode::NOT_FOUND,
            Self::IO(_) => StatusCode::INTERNAL_SERVER_ERROR,
        };
        let body = self.to_string();
        (code, body).into_response()
    }
}
```

6.  添加 handler 到 router

[.nest](https://docs.rs/axum/latest/axum/struct.Router.html#method.nest) 类似 routes group 概念

[.fallback](https://docs.rs/axum/latest/axum/struct.Router.html#method.fallback) 在其他所有路由都不匹配时使用

```Rust
use axum::routing::get;
use axum::{Router, Server};
use hello_axum::{hello, not_found_handler, not_implemented_handler, read_file_handler};
use std::net::SocketAddr;

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt::init();

    let app = Router::new()
        .route("/hello", get(hello))
        .nest(
            "/error",
            Router::new()
                .route("/not_implemented", get(not_implemented_handler))
                .route("/read_file", get(read_file_handler)),
        )
        .fallback(not_found_handler);

    // run it
    let addr = SocketAddr::from(([0, 0, 0, 0], 3000));
    tracing::info!("listening on {addr}");
    Server::bind(&addr)
        .serve(app.into_make_service())
        .await
        .unwrap();
}
```

# 总结

知识点：

-   如何使用 thiserror 自定义错误类型
-   为自定义类型实现 `IntoResponse`，定义错误码和返回 body
-   `Impl From<Error1> for Error2` 后，通过 `?` 返回自动转换错误类型
-   `.map_err(|err| { return CustomError })` 手动转换错误类型
-   `.nest` 定义 routes group
-   `.fallback` 定义路由不匹配的处理逻辑（404）
