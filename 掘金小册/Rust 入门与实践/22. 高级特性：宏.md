在 Rust 编程语言中，宏是一种定义可重用代码块的方式。它们可以被看作是一种函数，但是它们生成的不是值，而是代码，并且这是在编译期间完成的，而不是在程序运行期间。宏用感叹号表示，比如 `println!()`。

Rust 中有两种宏：声明式宏和过程式宏。

  


1.  **声明宏**：这是是最常用的宏形式，你可能会在如打印到控制台的 `println!()` 或者定义字面量的 vector `vec![]` 这样的地方看到。你定义模式和替换代码，Rust 将模式匹配到提供的输入并将其替换为代码。

```Rust
// 定义
#[macro_export]
macro_rules! vec {
    ( $( $x:expr ),* ) => {
        {
            let mut temp_vec = Vec::new();
            $(
                temp_vec.push($x);
            )*
            temp_vec
        }
    };
}

// 使用
vec![1, 2, 3];

// 展开后结果
let mut temp_vec = Vec::new();
temp_vec.push(1);
temp_vec.push(2);
temp_vec.push(3);
temp_vec
```

2.  **过程宏**：这些更复杂、更强大。过程式宏接受 rust 编译器解析得到的抽象语法树作为输入，操作这些代码，然后产生一些代码作为输出。它们在自己的 crate 中定义，使用特殊的 crate 类型 `proc-macro`。

过程式宏有三种形式：自定义派生，属性型宏和函数型宏。

1.  **自定义派生**：这些宏允许你编写可以自动为数据类型实现的特性。它们通过 `derive` 属性添加到结构体或枚举中。例如，Rust 的内置特性，如 `Clone` 和 `Debug`，可以与 derive 属性一起使用（`#[derive(Clone, Debug)]`），但你也可以实现你自己的特性来与 `derive` 一起使用。
1.  **属性宏**：这些与派生宏相似，但更灵活。它们也通过 `#[...]` 语法附加到项目上，但是它们不是为特性生成代码，而是可以添加、修改或抑制任何随意的项目，例如函数、模块或结构体。这允许你创建影响项目行为的注解。
1.  **函数宏**：这些宏类似于函数，就像声明式宏一样，但是它们更强大，因为它们可以处理原始语法标记。它们使用 `proc_macro::` 库声明，并且可以更灵活地操作输入代码。

所有这些过程宏都在其自己的 crate 中定义，crate 类型为 `proc-macro`。

  


这是一个简单的过程宏的例子：

```Rust
use proc_macro::TokenStream;
use quote::quote;
use syn;

fn impl_hello_macro(ast: &syn::DeriveInput) -> TokenStream {
    let name = &ast.ident;
    let gen = quote! {
        impl HelloMacro for #name {
            fn hello_macro() {
                println!("Hello, Macro! My name is {}!", stringify!(#name));
            }
        }
    };
    gen.into()
}

#[proc_macro_derive(HelloMacro)]
pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
    // Construct a representation of Rust code as a syntax tree
    // that we can manipulate
    let ast = syn::parse(input).unwrap();

    // Build the trait implementation
    impl_hello_macro(&ast)
}
```

  


这个过程宏将为其派生的类型生成 `HelloMacro` 特性的实现。最终，用户可以方便地将其修饰在自定义结构体上，为结构体自动实现 `hello_macro` 方法：

```Rust
use hello_macro::HelloMacro;
use hello_macro_derive::HelloMacro;

#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro();
}
```
