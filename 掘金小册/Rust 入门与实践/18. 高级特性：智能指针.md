在本节中，我们会学习智能指针的相关知识，并对标准库中一些常用的智能指针进行了解和使用，下面就让我们一起开始探索吧。

  


## 1. 概念

说到智能指针，大家可能马上会有这样一个疑问，那就是和平常接触过的普通指针有什么区别？实际上，智能指针就是对普通指针进行一层封装，使得它本身是一个对象，行为表现上却像指针。

  


那又为什么要这么设计呢？相信大家之前在学C语言的时候或多或少写出过内存泄漏的代码，而这通常是因为用完指针后，没有去手动释放指针所指向的内存所造成的。智能指针就可以很好的解决这个问题，结合上RAII（资源获取即初始化），智能指针的内存管理就可以由用户转交给它本身，其构造函数分配内存，析构函数释放内存，避免内存泄漏。

  


在Rust中，智能指针的两大特征就是实现了Deref和Drop这两个trait，Deref可以帮助我们像常规引用一样使用智能指针，Drop则可以帮助我们在值离开作用域后做资源清理的工作，下面就让我们一起动手尝试简单实现一个自己的智能指针吧。

  


## 2. 自定义智能指针

首先，我们可以定义一个智能指针的结构体，并为它实现构造的new方法。

```Rust
struct MyCustomSmartPointer<T> {
    data: T,
}

impl<T> MyCustomSmartPointer<T> {
    fn new(data: T) -> Self {
        MyCustomSmartPointer { data }
    }
}
```

  


在实现Deref之前，下面的这段代码是无法通过编译的，解引用操作只能发生在引用上。

```Rust
fn main() {
    let my_custom_smart_pointer = MyCustomSmartPointer::new(String::from("Hello, world!"));
    // error[E0614]: type `MyCustomSmartPointer<&str>` cannot be dereferenced
    assert_eq!("Hello, world!", *my_custom_smart_pointer);
    // error[E0308]: mismatched types
    print(&my_custom_smart_pointer);
}

fn print(s: &str) {
    println!("{}", s);
}
```

  


现在让我们来实现一下Deref，看看它是如何帮助通过编译的吧。我们只需要在deref方法里返回持有数据的引用即可，这样一来，在调用`*my_custom_smart_pointer`的时候编译器会自动转换为`*(my_custom_smart_pointer.deref())`，值得注意的是这个转换只会发生一次，也就是说不会出现无限递归替换；在调用print传参时会进行隐式deref强制转换，链路为`&MyCustomSmartPointer -> &String -> &str`。可以看到，Deref的实现可以让我们更好的把智能指针当成普通指针用，简化了编程。

```Rust
impl<T> core::ops::Deref for MyCustomSmartPointer<T> {
    type Target = T;

    fn deref(&self) -> &Self::Target {
        &self.data
    }
}
```

  


最后就是实现Drop，帮助我们在智能指针的生命周期结束后做一些资源释放操作。在这个例子中，当main函数结束后，控制台会打印出`Dropping MyCustomSmartPointer with data!`，模拟`MyCustomSmartPointer`的资源释放操作。

```
impl<T> Drop for MyCustomSmartPointer<T> {
    fn drop(&mut self) {
        println!("Dropping MyCustomSmartPointer with data!");
    }
}
```

  


到此为止，我们已经简单实现了一个[自己的智能指针](https://play.rust-lang.org/?version=stable&mode=debug&edition=2021&gist=5dc8a9245dfef7654e1e88a639607d1d)，但你可能更好奇它还能干些什么，所以让我们再来一起学习下Rust标准库中常见的一些智能指针吧。

  


## 3. 标准库中的智能指针

最简单直接的就是Box<T>，它可以帮助我们把数据T存放在堆上，自己本身则放在栈上，并指向堆上存放数据的那块内存空间。对于在编译期未知大小的类型，使用Box进行封装是一种常见的使用方式，比如当我们创建递归类型的时候，结构体本身Foo的成员又包含Foo，这时编译器就无法知道该给Foo在栈上分配多大的内存空间，引入Box后则可以帮助我们确定下来栈上空间的大小。

```Rust
enum Foo {
    // Bar(Foo), error[E0072]: recursive type `Foo` has infinite size
    Bar(Box<Foo>),
    Baz,
}

fn main() {
    _ = Foo::Bar(Box::new(Foo::Bar(Box::new(Foo::Baz))));
}
```

  


对于只关心是否实现某个trait而不关注具体类型的场景也适合使用Box进行封装，Box<dyn trait>是一种非常常见的使用方式，但一般会更推荐使用泛型做类型体操，性能更好。

```Rust
trait Foo {}

struct Bar;
struct Baz;

impl Foo for Bar {}
impl Foo for Baz {}

// 可以使用enum优化
fn return_bar_or_baz(is_bar: bool) -> Box<dyn Foo> {
    if is_bar {
        Box::new(Bar)
    } else {
        Box::new(Baz)
    }
}

fn main() {
    return_bar_or_baz(true);
}
```

  


当我们想让单个值有多个所有者的时候，比如在图结构中多条边拥有同一个节点，这时就需要Rc<T>和Arc<T>了，二者区别在于前者用于单线程，后者用于多线程。我们以Rc<T>为例介绍，其实现原理是Rc<T>在进行clone操作时，仅对其引用计数进行加1而不用去拷贝堆上数据T，当Rc<T>消亡时则对其引用计数进行减1，当减为0时则会释放堆上数据T所占据的内存空间。

```Rust
use std::rc::Rc;

fn main() {
    let a = Rc::new(1);
    println!("count after creating a = {}", Rc::strong_count(&a));
    {
        let b = Rc::clone(&a);
        println!("count after creating b = {}", Rc::strong_count(&b));
    }
    println!("count after b goes out of scope = {}", Rc::strong_count(&a));
    
    assert_eq!(Rc::try_unwrap(a), Ok(1));
}
```

值得注意的是，使用Rc<T>进行封装后我们只能共享拿到不可变引用，因而无法对其数据进行修改，这也符合Rust的借用规则。如果我们就是想对其数据进行修改呢？通常有两种方法，第一种是在修改的时候确保当前是最后使用到的地方，拿出所有权进行修改；第二种方法则是接着学习下一节的内部可变性。
