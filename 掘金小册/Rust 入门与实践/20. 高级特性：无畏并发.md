在本节中，我们会初步探索Rust中并发的世界，看看它到底是如何保证并发安全的，以及怎么使用常见的Mutex和Channel完成并发编程。

  


## 1. 并发安全

在并发的世界中，最常见的并发安全问题就是数据竞争，也就是两个线程同时对一个变量进行读写操作。但当你在Safe Rust中写出有数据竞争的代码时，编译器会直接拒绝编译。那么它是靠什么魔法做到的呢？这就不得不谈Send 和Sync这两个标记trait了，实现Send的类型可以在多线程间转移所有权，实现Sync的类型可以在多线程间共享引用 **。** 但它们内部都是没有任何方法声明以及方法体的，所以二者仅仅是作为一个类型约束的标记信息提供给编译器，帮助编译器拒绝线程不安全的代码。

```Rust
pub unsafe auto trait Send { }

pub unsafe auto trait Sync { }
```

  


下面让我们来尝试写一段多线程对共享数据进行修改的代码吧。我们在两个线程间同时操作Rc<T>并执行clone操作，意味着这时Rc<T>内部的引用计数有可能被同时修改进行加一的操作，从而引发数据竞争。幸运的是，这段代码是会被编译器拒绝掉的，因为Rc<T>没有实现Send trait，故而不能安全的在线程间转移所有权。

```Rust
use std::rc::Rc;
use std::thread;

// error[E0277]: `Rc<i32>` cannot be sent between threads safely
fn main() {
    let a = Rc::new(1);
    let a_clone = a.clone();
    thread::spawn(move || {
        a_clone.clone();
        println!("{}", a_clone);
    });
    a.clone();
}
```

  


让我们再来看下用于多线程的Arc，并在里面放一个RefCell进去会如何呢？这样我们是不是就可以在运行时得到多个可变数据所有者，再在多线程中修改就可以了。当我们还在沾沾自喜时，编译器又当头给了我们一棒槌，RefCell<T>没有实现Sync trait，故而不能安全的在线程间共享引用。

```Rust
use std::cell::RefCell;
use std::sync::Arc;
use std::thread;

// error[E0277]: `RefCell<i32>` cannot be shared between threads safely
fn main() {
    let a = Arc::new(RefCell::new(1));
    let a_clone = a.clone();
    thread::spawn(move || {
        *a_clone.borrow_mut() = 2;
    });
    *a.borrow_mut() = 3;
}
```

  


可以看出，在Rust中Send和Sync这两个标记可以帮助我们拒绝掉很多非并发安全的代码，那么正确的并发编程姿势是如何的呢？

  


## 2. 线程同步

通过线程间同步的方式，我们就可以正确的写出并发代码了，也就是说，多个线程对于数据的读写操作需要进行协商，排出一定的先后顺序，这样就不会有问题了。Rust中的两种同步方式分别是通过Mutex和Channel进行的，下面就让我们一起来学习下吧。

  


Mutex是通过共享状态完成线程间同步。也就是说，任意时刻，其只允许一个线程访问某些数据。为了访问Mutex中的数据，线程首先需要通过获取Mutex的锁来表明其希望访问数据，其中锁记录着谁有数据的排他访问权。下面是Mutex的一个简单使用方式，值得一说的是，在lock()方法后返回的是一个智能指针MutexGuard，在它的生命周期消亡时会自动对获得的锁进行释放，使得我们不会在锁和解锁上出错。

```Rust
use std::sync::Mutex;

fn main() {
    let m = Mutex::new(5);

    {
        let mut num = m.lock().unwrap();
        *num = 6;
    }

    println!("m = {:?}", m);
}
```

  


对于上面Arc<RefCell<T>>的例子，我们就可以把RefCell替换成Mutex来使用了，完成多线程对共享数据的修改需求。

```Rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let a = Arc::new(Mutex::new(1));
    let b = a.clone();
    let c = a.clone();
    
    let handle_b = thread::spawn(move || {
        *b.lock().unwrap() = 2;
    });
    let handle_c = thread::spawn(move|| {
        *c.lock().unwrap() = 3;
    });
    handle_b.join();
    handle_c.join();
    
    println!("{}", a.lock().unwrap());
}
```

  


Channel是通过通信完成线程间同步。通常，我们会使用Channel来获得发送者和接收者，然后把发送者和接收者分别放在两个不同的线程中，这样，通过调用发送者的send方法和接收者的recv方法，两个线程就可以完成通信同步，下面是一个多发送者单接收者的例子。

```Rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    let (tx, rx) = mpsc::channel();

    let tx1 = tx.clone();
    thread::spawn(move || {
        let vals = vec![
            String::from("hi"),
            String::from("from"),
            String::from("the"),
            String::from("thread"),
        ];

        for val in vals {
            tx1.send(val).unwrap();
            thread::sleep(Duration::from_secs(1));
        }
    });

    thread::spawn(move || {
        let vals = vec![
            String::from("more"),
            String::from("messages"),
            String::from("for"),
            String::from("you"),
        ];

        for val in vals {
            tx.send(val).unwrap();
            thread::sleep(Duration::from_secs(1));
        }
    });

    for received in rx {
        println!("Got: {}", received);
    }

}
```
