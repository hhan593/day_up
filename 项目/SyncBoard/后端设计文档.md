# SyncBoard 后端设计文档

## 1. 概述

SyncBoard 后端基于 NestJS 11 + TypeScript 构建，采用双协议架构：REST API 处理资源管理（认证、房间 CRUD），WebSocket Gateway 处理实时 CRDT 同步。数据层使用 PostgreSQL 持久化 + Redis 缓存/Pub-Sub。

---

### 5.2 实体定义

```typescript
// User 实体
@Entity('users')
export class UserEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ unique: true })
  username: string;

  @Column({ unique: true })
  email: string;

  @Column()
  password: string;           // bcrypt 哈希

  @Column({ nullable: true })
  avatar: string;

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;

  @OneToMany(() => RoomMemberEntity, (m) => m.user)
  memberships: RoomMemberEntity[];
}

// Room 实体
@Entity('rooms')
export class RoomEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column()
  name: string;

  @ManyToOne(() => UserEntity)
  @JoinColumn({ name: 'created_by' })
  creator: UserEntity;

  @Column({ default: false })
  isPublic: boolean;

  @Column({ nullable: true, unique: true })
  shareToken: string;         // 分享链接 token

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;

  @OneToMany(() => RoomMemberEntity, (m) => m.room)
  members: RoomMemberEntity[];
}

// RoomMember 实体 (多对多关系表)
@Entity('room_members')
export class RoomMemberEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @ManyToOne(() => UserEntity, (u) => u.memberships)
  @JoinColumn({ name: 'user_id' })
  user: UserEntity;

  @ManyToOne(() => RoomEntity, (r) => r.members)
  @JoinColumn({ name: 'room_id' })
  room: RoomEntity;

  @Column({ type: 'enum', enum: ['owner', 'editor', 'viewer'], default: 'editor' })
  role: 'owner' | 'editor' | 'viewer';

  @CreateDateColumn()
  joinedAt: Date;
}

// DocumentSnapshot 实体
@Entity('doc_snapshots')
export class DocumentSnapshotEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @ManyToOne(() => RoomEntity)
  @JoinColumn({ name: 'room_id' })
  room: RoomEntity;

  @Column({ type: 'bytea' })
  yjsState: Buffer;           // Yjs 文档编码后的二进制快照

  @Column()
  version: number;

  @ManyToOne(() => UserEntity)
  @JoinColumn({ name: 'created_by' })
  createdBy: UserEntity;

  @CreateDateColumn()
  createdAt: Date;
}
```

## 7. WebSocket Gateway 设计

### 7.1 连接流程

```
客户端                                       服务端
  │                                            │
  │── ws://host:3002/:roomId?token=JWT ───────►│
  │                                            │  验证 JWT
  │                                            │  检查房间权限
  │◄─── 连接确认 / 拒绝 ─────────────────────── │
  │                                            │  加入房间广播组
  │── sync step 1 (client state vector) ──────►│
  │◄── sync step 2 (missing updates) ──────────│
  │                                            │
  │◄═══ 双向实时同步通道已建立 ═══════════════►│
  │                                            │
  │── sync:update (Yjs 增量) ────────────────►│──► 广播给房间其他人
  │◄── sync:update (他人增量) ─────────────────│
  │                                            │
  │── awareness:update (光标/选区) ───────────►│──► 广播给房间其他人
  │◄── awareness:update (他人光标) ────────────│
  │                                            │
  │── disconnect ─────────────────────────────►│
  │                                            │  room:leave 广播
  │                                            │  保存最新快照
```

### 7.2 Gateway 实现

```typescript
@WebSocketGateway({
  port: 3002,
  cors: { origin: '*' },
})
export class SyncGateway implements OnGatewayConnection, OnGatewayDisconnect {

  // 房间管理: roomId → Set<WebSocket>
  private rooms = new Map<string, Set<WebSocket>>();

  // Yjs 文档: roomId → Y.Doc
  private docs = new Map<string, Y.Doc>();

  handleConnection(client: WebSocket, request: IncomingMessage) {
    // 1. 从 URL 解析 roomId 和 JWT token
    // 2. 验证 JWT，获取用户信息
    // 3. 检查用户对该房间的权限
    // 4. 加入房间广播组
    // 5. 加载或创建 Y.Doc
    // 6. 发送 sync step 1 初始同步
  }

  handleDisconnect(client: WebSocket) {
    // 1. 从房间广播组移除
    // 2. 广播 room:leave 事件
    // 3. 如果房间无人，定时保存快照并释放 Y.Doc
  }

  @SubscribeMessage(WS_EVENTS.SYNC_UPDATE)
  handleSyncUpdate(client: WebSocket, data: Uint8Array) {
    // 1. 应用增量到服务端 Y.Doc
    // 2. 广播给房间内其他客户端
  }

  @SubscribeMessage(WS_EVENTS.AWARENESS_UPDATE)
  handleAwarenessUpdate(client: WebSocket, data: Uint8Array) {
    // 广播给房间内其他客户端（不持久化）
  }
}
```

### 7.3 Yjs 文档生命周期

```
房间首次连接
     │
     ▼
从 PostgreSQL 加载最新 snapshot
     │
     ├── 有快照 → Y.applyUpdate(doc, snapshot)
     │
     └── 无快照 → 创建空 Y.Doc
     │
     ▼
内存中维护 Y.Doc，处理实时同步
     │
     ├── 定时保存快照 (每 5 分钟)
     │
     └── 房间无人时
          │
          ▼
     保存最终快照 → 释放内存中的 Y.Doc
```

## 8. 认证流程设计

### 8.1 JWT 策略

```typescript
// JWT Payload
interface JwtPayload {
  sub: string;       // userId
  email: string;
  username: string;
  iat: number;       // 签发时间
  exp: number;       // 过期时间
}

// 配置
{
  secret: process.env.JWT_SECRET,
  expiresIn: process.env.JWT_EXPIRES_IN  // 默认 7d
}
```

### 8.2 认证流程

```
注册流程:
  1. 接收 email + username + password
  2. 检查 email/username 唯一性
  3. bcrypt.hash(password, 10) 生成哈希
  4. 创建 User 记录
  5. 签发 JWT 并返回

登录流程:
  1. 接收 email + password
  2. 查找用户
  3. bcrypt.compare(password, user.password) 验证
  4. 签发 JWT 并返回

请求认证:
  1. 从 Authorization: Bearer <token> 提取 token
  2. jwt.verify(token, secret) 验证签名和过期
  3. 从 payload 中获取 userId
  4. 注入 request.user

WebSocket 认证:
  1. 从 ws URL query ?token=<JWT> 提取 token
  2. 同上验证流程
  3. 验证用户对 roomId 的访问权限
```

### 10.3 业务错误码

|--------|------|
| 400 | 请求参数验证失败 |
