## Phase 1: MVP（核心画布 + 离线 + 双人协作）

### 模块依赖关系

```
共享包(types/constants/utils)
    ├──► 后端: 数据库实体 + Auth 模块
    │        ├──► 后端: Room 模块
    │        └──► 后端: WebSocket Sync 网关
    │                 └──► 前后端联调
    └──► 前端: tldraw 画布
             ├──► 前端: Yjs + IndexedDB 离线
             │        └──► 前端: 协作组件 (光标/状态)
             └──► 前端: Auth 页面 + API 客户端
                      └──► 前端: Dashboard + Room 管理
```

## 关键建议

### 为什么从 Step 1（共享包）开始？
共享包定义了前后端的通信契约（类型 + 常量），是所有模块的基石。先完善它可以避免后续类型不一致的问题。

### 为什么 Step 2（画布）优先于 Step 4（后端 Auth）？
- 画布是产品核心价值，尽早验证 tldraw 集成方案
- 画布不依赖后端，可独立开发和演示
- Local-First 理念：先保证离线体验，再接入服务端

### 为什么 Step 3（Yjs）在 Step 6（WebSocket）之前？
- Yjs + IndexedDB 是纯前端能力，不需要后端
- 先实现离线存储再接入网络同步，符合 Local-First 渐进增强原则

### 何时引入 Docker？
在 Step 4（后端 Auth）开始前，需要启动 PostgreSQL 和 Redis：
```bash
docker compose up -d postgres redis
```
