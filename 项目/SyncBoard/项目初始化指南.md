# SyncBoard 项目初始化指南

## 前置要求

| 工具                    | 最低版本            | 验证命令                              |
| ----------------------- | ------------------- | ------------------------------------- |
| Node.js                 | 18.x+ (推荐 20 LTS) | `node -v`                             |
| pnpm                    | 8.x+                | `pnpm -v`                             |
| Git                     | 2.x+                | `git -v`                              |
| Docker & Docker Compose | 最新版              | `docker -v && docker compose version` |
| PostgreSQL              | 15+ (或通过 Docker) | `psql --version`                      |
| Redis                   | 7+ (或通过 Docker)  | `redis-cli --version`                 |

> 如果不想本地安装 PostgreSQL 和 Redis，后面会通过 Docker Compose 启动。

### 安装 pnpm（如未安装）

```bash
npm install -g pnpm
```

## 一、初始化 Monorepo 脚手架

### 1.1 初始化根项目

```bash
cd f:/project/SyncBoard

# 初始化 package.json
pnpm init

# 安装 Turborepo
pnpm add -D turbo
```

### 1.2 配置 pnpm workspace

创建 `pnpm-workspace.yaml`：

```yaml
packages:
  - "apps/*"
  - "packages/*"
```

### 1.3 配置 Turborepo

创建 `turbo.json`：

```json
{
  "$schema": "https://turbo.build/schema.json",
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {
      "dependsOn": ["^build"]
    },
    "test": {
      "dependsOn": ["build"]
    },
    "clean": {
      "cache": false
    }
  }
}
```

### 1.4 修改根 `package.json`

将根 `package.json` 改为以下内容：

```json
{
  "name": "syncboard",
  "version": "0.0.1",
  "private": true,
  "packageManager": "pnpm@9.15.4",
  "scripts": {
    "dev": "turbo run dev",
    "build": "turbo run build",
    "lint": "turbo run lint",
    "test": "turbo run test",
    "clean": "turbo run clean",
    "dev:web": "turbo run dev --filter=@syncboard/web",
    "dev:server": "turbo run dev --filter=@syncboard/server",
    "format": "prettier --write \"**/*.{ts,tsx,js,json,md}\""
  },
  "devDependencies": {
    "turbo": "^2.x",
    "prettier": "^3.x",
    "typescript": "^5.x"
  }
}
```

### 1.5 创建目录结构

```bash
mkdir -p apps/web/src/{components,canvas,collaboration,store,hooks,utils}
mkdir -p apps/server/src/{auth,room,sync,storage}
mkdir -p packages/shared/src/{types,constants}
```

## 三、初始化 React 前端 (apps/web)

### 3.1 使用 Vite 创建 React 项目

```bash
cd apps/web

# 用 Vite 初始化（选 React + TypeScript）
pnpm create vite . --template react-ts
```

> 如果提示目录非空，选择清空或忽略已有文件。

### 3.2 安装核心依赖

```bash
cd apps/web

# Yjs 及协作相关
pnpm add yjs y-indexeddb y-websocket

# 状态管理
pnpm add zustand

# 画布引擎（二选一，推荐先用 tldraw 快速起步，后续可替换）
# 方案 A：tldraw（开箱即用的画布）
pnpm add tldraw
# 方案 B：roughjs + 自研（更灵活，学习价值更高）
# pnpm add roughjs perfect-freehand

# 工具库
pnpm add nanoid

# 开发依赖
pnpm add -D @types/node
```

### 3.3 修改 `apps/web/package.json`

确保 name 和 workspace 依赖正确：

```json
{
  "name": "@syncboard/web",
  "private": true,
  "version": "0.0.1",
  "type": "module",
  "scripts": {
    "dev": "vite --port 3000",
    "build": "tsc -b && vite build",
    "lint": "eslint .",
    "preview": "vite preview",
    "clean": "rm -rf dist node_modules/.vite"
  },
  "dependencies": {
    "@syncboard/shared": "workspace:*",
    "react": "^18.x",
    "react-dom": "^18.x",
    "yjs": "^13.x",
    "y-indexeddb": "^9.x",
    "y-websocket": "^2.x",
    "zustand": "^5.x",
    "nanoid": "^5.x"
  }
}
```

### 3.4 配置 Vite

修改 `apps/web/vite.config.ts`：

```typescript

import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],

        target: "ws://localhost:3001",
```

## 四、初始化 NestJS 后端 (apps/server)

### 4.1 创建 NestJS 项目

```bash
# 全局安装 NestJS CLI（如未安装）
pnpm add -g @nestjs/cli

# 在 apps 目录下创建 server 项目
cd apps
nest new server --package-manager pnpm --skip-git
```

### 4.2 安装后端核心依赖

```bash
cd apps/server

# WebSocket 支持
pnpm add @nestjs/websockets @nestjs/platform-ws ws
pnpm add -D @types/ws

# 数据库
pnpm add @nestjs/typeorm typeorm pg

# Redis
pnpm add @nestjs/cache-manager cache-manager cache-manager-redis-store redis
pnpm add -D @types/cache-manager

# 认证
pnpm add @nestjs/jwt @nestjs/passport passport passport-jwt bcryptjs
pnpm add -D @types/passport-jwt @types/bcryptjs

# 配置管理
pnpm add @nestjs/config

# Yjs 服务端（用于文档持久化）
pnpm add yjs y-protocols lib0

# 数据校验
pnpm add class-validator class-transformer

# Swagger 文档
pnpm add @nestjs/swagger
```

### 4.3 修改 `apps/server/package.json`

确保 name 和 workspace 依赖正确：

```json
{
  "name": "@syncboard/server",
  "version": "0.0.1",
  "private": true,
  "scripts": {
    "dev": "nest start --watch",
    "build": "nest build",
    "start": "nest start",
    "start:prod": "node dist/main",
    "lint": "eslint \"{src,apps,libs,test}/**/*.ts\"",
    "test": "jest",
    "clean": "rm -rf dist"
  },
  "dependencies": {
    "@syncboard/shared": "workspace:*"
  }
}
```

### 4.5 配置 NestJS 数据库连接

修改 `apps/server/src/app.module.ts`：

```typescript
import { Module } from "@nestjs/common";
import { ConfigModule, ConfigService } from "@nestjs/config";
import { TypeOrmModule } from "@nestjs/typeorm";

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,
      envFilePath: ".env",
    }),
    TypeOrmModule.forRootAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: (config: ConfigService) => ({
        type: "postgres",
        host: config.get("DB_HOST"),
        port: config.get<number>("DB_PORT"),
        username: config.get("DB_USERNAME"),
        password: config.get("DB_PASSWORD"),
        database: config.get("DB_DATABASE"),
        autoLoadEntities: true,
        synchronize: config.get("NODE_ENV") === "development", // 生产环境禁用
      }),
    }),
  ],
})
export class AppModule {}
```

## 五、Docker Compose 配置

在项目根目录创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  postgres:
    image: postgres:16-alpine
    container_name: syncboard-postgres
    restart: unless-stopped
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: syncboard
      POSTGRES_PASSWORD: syncboard_dev_123
      POSTGRES_DB: syncboard
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    container_name: syncboard-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:
```

### 启动数据库服务

```bash
docker compose up -d
```

## 十一、初始化命令速查（按顺序执行）

```bash
# 1. 根目录初始化
cd f:/project/SyncBoard
pnpm init
pnpm add -D turbo prettier typescript eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-config-prettier

# 2. 创建目录
mkdir -p apps/web apps/server packages/shared/src/{types,constants}

# 3. 初始化 shared 包
cd packages/shared && pnpm init && cd ../..

# 4. 初始化前端
cd apps/web && pnpm create vite . --template react-ts && cd ../..

# 5. 前端依赖
cd apps/web && pnpm add yjs y-indexeddb y-websocket zustand nanoid && cd ../..

# 6. 初始化后端
cd apps && npx @nestjs/cli new server --package-manager pnpm --skip-git && cd ..

# 7. 后端依赖
cd apps/server && pnpm add @nestjs/websockets @nestjs/platform-ws ws @nestjs/typeorm typeorm pg @nestjs/jwt @nestjs/passport passport passport-jwt bcryptjs @nestjs/config yjs y-protocols lib0 class-validator class-transformer @nestjs/swagger && pnpm add -D @types/ws @types/passport-jwt @types/bcryptjs && cd ../..

# 8. 启动数据库
docker compose up -d

# 9. 安装所有依赖
pnpm install

# 10. 启动开发
pnpm dev
```

---

> **注意事项**：
>
> - `.env` 文件已加入 `.gitignore`，不会提交到 Git，请妥善保管
> - 开发环境下 TypeORM `synchronize: true` 会自动建表，生产环境务必关闭并使用 migrations
> - pnpm workspace 中子包之间通过 `"@syncboard/shared": "workspace:*"` 引用
