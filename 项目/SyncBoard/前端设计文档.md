# SyncBoard 前端设计文档

## 1. 概述

SyncBoard 前端是基于 React 19 + TypeScript + Vite 构建的 Local-First 实时协作白板应用。核心依赖 tldraw 提供画布渲染能力，Yjs (CRDT) 实现无冲突实时协作，Zustand 管理全局状态。

---

## 2. 技术栈

| 分类 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 框架 | React | 19.2.4 | UI 渲染 |
| 构建 | Vite | 8.0.1 | 开发服务器 & 打包 |
| 语言 | TypeScript | ~5.9.3 | 类型安全 |
| 画布引擎 | tldraw | 4.5.3 | 无限画布 & 绘图工具 |
| 协作同步 | Yjs | 13.6.30 | CRDT 无冲突数据同步 |
| WebSocket | y-websocket | 3.0.0 | Yjs 网络同步层 |
| 本地持久化 | y-indexeddb | 9.0.12 | 离线数据存储 |
| 状态管理 | Zustand | 5.0.12 | 全局状态管理 |
| ID 生成 | nanoid | 5.1.7 | 唯一标识符 |

## 5. 状态管理设计

### 5.1 Zustand Store 结构

```typescript
// authStore - 认证状态
interface AuthState {
  user: User | null;
  token: string | null;
  isAuthenticated: boolean;
  login: (email: string, password: string) => Promise<void>;
  logout: () => void;
  refreshToken: () => Promise<void>;
}

// canvasStore - 画布状态
interface CanvasState {
  activeTool: ToolType;              // 当前工具 (select, rectangle, text...)
  activeColor: string;               // 当前颜色
  strokeWidth: number;               // 描边宽度
  zoom: number;                      // 缩放比例
  selectedElements: string[];        // 选中元素 ID 列表
  setActiveTool: (tool: ToolType) => void;
  setZoom: (zoom: number) => void;
}

// roomStore - 房间状态
interface RoomState {
  rooms: Room[];                     // 房间列表
  currentRoom: Room | null;          // 当前房间
  onlineUsers: User[];               // 在线用户
  fetchRooms: () => Promise<void>;
  createRoom: (name: string) => Promise<Room>;
  joinRoom: (id: string) => Promise<void>;
  leaveRoom: () => void;
}

// uiStore - UI 状态
interface UIState {
  sidebarOpen: boolean;              // 左侧栏展开
  propertiesPanelOpen: boolean;      // 右侧属性面板
  theme: 'light' | 'dark';          // 主题
  shareModalOpen: boolean;           // 分享弹窗
  toggleSidebar: () => void;
  toggleTheme: () => void;
}
```

## 6. 核心模块设计

### 6.1 画布模块 (canvas/)

基于 tldraw SDK 封装，tldraw 提供完整的画布渲染和交互能力：

```
┌──────────────────────────────────────────────────────────────┐
│                    WhiteboardCanvas                          │
│  ┌──────┐ ┌────────────────────────────────────┐ ┌────────┐ │
│  │      │ │                                    │ │        │ │
│  │ Tool │ │         tldraw <Tldraw />           │ │  Prop  │ │
│  │ bar  │ │                                    │ │ Panel  │ │
│  │      │ │    (无限画布 + 绘图 + 交互)          │ │        │ │
│  │      │ │                                    │ │        │ │
│  └──────┘ └────────────────────────────────────┘ └────────┘ │
│            ┌────────────────────────────────────┐            │
│            │ PageTabs │ ConnectionStatus │ Zoom  │            │
│            └────────────────────────────────────┘            │
└──────────────────────────────────────────────────────────────┘
```

**关键集成点：**
- tldraw 的 `store` 与 Yjs Y.Doc 绑定，实现协作数据同步
- 自定义 tldraw UI（覆盖默认工具栏和菜单）
- 监听 tldraw store 变更 → 更新 Yjs → 广播到其他客户端

### 6.2 协作模块 (collaboration/)

```
┌─────────────────────────────────────────────────────────┐
│                    YjsProvider                          │
│                                                         │
│  ┌─────────────┐    ┌────────────────┐                  │
│  │  Y.Doc      │    │ WebsocketProvider                 │
│  │  (CRDT文档)  │◄──►│ (y-websocket)  │◄──► 服务端       │
│  └──────┬──────┘    └────────────────┘                  │
│         │                                               │
│  ┌──────▼──────┐    ┌────────────────┐                  │
│  │ IndexedDB   │    │ Awareness      │                  │
│  │ Provider    │    │ (光标/选区/在线) │                  │
│  │ (离线持久化) │    └────────────────┘                  │
│  └─────────────┘                                        │
└─────────────────────────────────────────────────────────┘
```

**离线同步流程：**
1. 用户编辑 → 写入本地 Yjs Doc → 自动持久化到 IndexedDB
2. 网络连接时 → y-websocket 自动将增量同步到服务端
3. 网络断开时 → 本地 IndexedDB 持续记录所有操作
4. 网络恢复时 → y-websocket 自动发送未同步的增量，CRDT 无冲突合并

### 6.3 连接状态指示器

```
状态枚举:
- CONNECTED    → 🟢 已连接     (绿色)
- DISCONNECTED → 🟡 离线模式   (黄色，顶部横幅提示)
- SYNCING      → 🔵 同步中...  (蓝色，显示进度)
- ERROR        → 🔴 连接错误   (红色，提示重试)
```

## 7. API 交互设计

### 7.1 HTTP API 调用 (REST)

```typescript
// 封装 fetch，自动携带 JWT
const api = {
  // 认证
  POST   /api/auth/login          // 登录
  POST   /api/auth/register       // 注册
  POST   /api/auth/refresh        // 刷新 Token
  GET    /api/auth/me             // 获取当前用户

  // 房间管理
  GET    /api/rooms               // 房间列表
  POST   /api/rooms               // 创建房间
  GET    /api/rooms/:id           // 房间详情
  PUT    /api/rooms/:id           // 更新房间信息
  DELETE /api/rooms/:id           // 删除房间

  // 分享 & 权限
  POST   /api/rooms/:id/share     // 生成分享链接
  GET    /api/rooms/:id/members   // 成员列表
  PUT    /api/rooms/:id/members/:uid  // 修改成员权限
  DELETE /api/rooms/:id/members/:uid  // 移除成员
}
```

### 7.2 WebSocket 连接 (Yjs 同步)

```
连接地址: ws://localhost:3002/:roomId?token=<JWT>

协议: y-websocket 标准协议
  - sync step 1/2: 文档初始同步
  - update: 增量更新广播
  - awareness: 用户感知 (光标位置、选区、在线状态)
```
