# 🐝 SwarmNote V2 桌面端详细 UI/UX 设计规范 (PRD & Design Spec)

> **文档状态:** Draft v2.0
> 
> **设计基调:** 暗黑极客 (Dark Amber)、无边框沉浸、底层透明化 (P2P/CRDT)
> 
> **技术上下文:** Tauri v2, React 19, CodeMirror 6, Tailwind CSS, yjs, libp2p

## 一、 全局设计系统 (Design Tokens & Rules)

### 1. 色彩规范 (Color System)

放弃纯黑，采用多层级的深灰构建“海拔” (Elevation)，用琥珀色作为系统生命力（网络连接/高亮）的象征。

- **应用底色 (App Background):**
    
    - `bg-layer-0`: `#0A0A0A` (主编辑器背景，最低海拔)
        
    - `bg-layer-1`: `#141414` (侧边栏、右侧面板背景)
        
    - `bg-layer-2`: `#1F1F1F` (悬浮卡片、下拉菜单背景，需配合极细边框使用)
        
- **品牌/交互主色 (Swarm Amber):**
    
    - `amber-primary`: `#F59E0B` (在线状态、主按钮、激活的 Tab)
        
    - `amber-hover`: `#D97706` (按钮悬停态)
        
    - `amber-pulse`: `rgba(245, 158, 11, 0.2)` (用于 P2P 同步时的呼吸灯光晕)
        
- **语义色 (Semantic):**
    
    - `status-synced`: `#10B981` (翠绿 - 状态完全一致)
        
    - `status-syncing`: `#F59E0B` (琥珀 - 数据传输中)
        
    - `status-offline`: `#525252` (中灰 - 本地隔离状态)
        
    - `text-diff-add`: `rgba(16, 185, 129, 0.15)` (合并新增文本的背景高亮)
        
- **排版色阶 (Typography):**
    
    - `text-h1`: `#FFFFFF` (大标题，极高对比度)
        
    - `text-body`: `#E5E5E5` (正文，降低刺眼感)
        
    - `text-muted`: `#737373` (占位符、时间戳、文件路径)
        

### 2. 布局与层级限制 (Layout & Z-Index)

- **无边框基调:** 禁用系统默认窗口，顶部自定义 `h-[40px]` 拖拽热区 (`app-region: drag`)。
    
- **Z-Index 规范:**
    
    - `z-0`: 编辑器正文
        
    - `z-10`: 侧边栏与标题栏
        
    - `z-20`: CodeMirror 远程光标气泡 (Remote Cursors)
        
    - `z-40`: 下拉菜单、Tooltip
        
    - `z-50`: ⌘K 命令面板、全屏配对遮罩
        

### 3. 动效规范 (Motion)

- **Micro-interactions (微交互):** 悬停、变色使用 `150ms ease-in-out`。
    
- **Panel Sliding (面板滑动):** Swarm 状态面板展开/折叠使用 `250ms cubic-bezier(0.4, 0, 0.2, 1)`。
    
- **Pulse (网络生命感):** 数据传输时，状态指示灯使用 `2s infinite` 的缓慢呼吸动画，绝不能高频闪烁。

## 三、 深度细节分解 (Deep Dive Details)

### 1. 左侧边栏 (Left Sidebar)

#### 1.1 工作区初始化/切换 (Workspace Switcher)

位于左上角。点击触发下拉菜单。

- **数据展示:** 当前选中的 `.swarmnote` 根目录别名。
    
- **下拉列表项:**
    
    - ✓ `[图标]` 个人知识库 `(/Users/xxx/Documents/Vault)`
        
    - `[图标]` 工作项目库 `(/Volumes/Data/Work)`
        
    - `--- 分割线 ---`
        
    - `+ 打开本地文件夹建立新工作区`
        

#### 1.2 树状文件系统 (File Tree)

完全映射本地系统，但屏蔽技术细节。

- **过滤规则:** 隐藏所有 `.md` 后缀。隐藏自动生成的 `.[文档名]_assets` 同名资源目录。隐藏 `.swarmnote` 配置文件夹。
    
- **缩进与辅助线:** 每一级文件夹向右缩进 `16px`。鼠标悬浮该层级时，左侧显现一条 `1px` 宽的 `border-neutral-700` 引导线。
    
- **交互状态机:**
    
    - _Default:_ 仅显示文件名与图标 (📁/📄)。
        
    - _Hover:_ 背景变为 `bg-layer-2`，右侧平滑显现 `[+新文件] [+新目录] [...]` 三个次级操作图标。
        
    - _Selected:_ 文本变为 `text-amber-primary`，左侧边缘出现 `2px` 宽的琥珀色激活条。
        
    - _Rename:_ 按 `Enter` 或 `F2`，文本原地替换为蓝色边框的 `<input>`，全选文本。
        
- **拖拽移动 (Drag & Drop):** 拖拽节点时，鼠标下方跟随半透明 ghost 元素；目标文件夹底色高亮。若拖入文件，其关联的 `_assets` 目录在底层必须同步静默移动。
    

#### 1.3 Swarm 状态面板 (The Network Radar)

固定在左侧边栏最底部，高度随状态动态变化。

- **折叠态 (高度 `56px`) - 三种网络枚举:**
    
    1. `[ 🟢 常亮 ] 集群同步完成 (3台)` -> 鼠标悬浮显示 Tooltip: "所有连接设备数据已对齐"
        
    2. `[ 🟡 呼吸 ] 正在同步 P2P 数据...` -> 底层 yjs 正在接收/发送 update 包。
        
    3. `[ ⚪️ 常亮 ] 仅本地运行 (离线)` -> mDNS 无发现，且未连接中继节点。
        
- **展开态 (点击折叠态区域触发，高度变为 `~300px` 抽屉)**
    
    - **模块 A: 本机身份:** 显示 `💻 [设备名] (本机) - IP: 192.168.x.x`。
        
    - **模块 B: 已连接网络 (Connected):** 列表显示已建立 libp2p 隧道的设备。
        
        - `📱 iPhone 15 Pro` -> 右侧跟一个微型 🟢 状态点。
            
        - 提供悬浮菜单：`[查看同步日志]` / `[断开连接]`。
            
    - **模块 C: 发现新设备 (Discovered - 高优先级 UI):**
        
        - 当 mDNS 广播收到未知 PeerId 响应时触发。
            
        - 该区域使用 `border border-amber-500/50 bg-amber-500/10` 突出显示。
            
        - 显示：`[未知设备] 💻 ThinkPad 请求配对`。
            
        - 包含两个强对比按钮：`[ 拒绝 (次级色) ]` / `[ 验证配对码 (主色) ]`。

### 2. 核心编辑区 (Main Editor)

#### 2.1 CodeMirror 6 渲染规范

放弃 Block 块编辑器，回归纯粹的 Markdown 极客体验，但赋予所见即所得的能力。

- **宽度约束:** `max-w-[760px] mx-auto`，无论窗口多大，保证最佳阅读视野。
    
- **排版渲染:**
    
    - `#` 输入后，该行自动被渲染为特大字号 (`text-3xl font-bold`)，并在下方绘制一条极细的分割线。
        
    - 图片拖拽入窗口：瞬间在光标处生成 `![Uploading...]()` 占位符。底层将图片拷入 `_assets` 目录，随后占位符静默更新为本地相对路径并渲染图片预览。
        

#### 2.2 CRDT 协同感知 (UI for yjs)

当多人/多设备同时编辑同一文档时触发。

- **无感合并高亮 (Merge Diff):**
    
    - **场景:** 电脑刚开机，连上网络，瞬间收到了手机端离线时写的三段话。
        
    - **UI表现:** 这三段话在编辑器中自动出现，其背景色被渲染为 `bg-emerald-500/20` (微弱的绿色高亮)。**高亮效果在 2.5 秒内通过 CSS 动画平滑过渡回透明。** 不打断用户当前输入。
        
- **实时远程光标 (Remote Cursors):**
    
    - 基于 CM6 的 `Decorations`。
        
    - 在远端设备的当前位置，绘制一条 `2px` 宽的绝对定位竖线，颜色根据 PeerId hash 取色（例如：蓝色）。
        
    - 光标上方附带一个倒立水滴状的气泡 `[📱 iPhone]`。为防止遮挡文本，该气泡仅在远端光标移动的 **2秒内** 显示，静止后自动隐藏气泡，仅留竖线。

### 3. 右侧面板：时空漫游 (Right Panel - History)

这是解决“文件冲突 (Conflict)”的终极形态。没有丑陋的冲突弹窗，一切皆可回溯。

- **Tabs:** `[ 📑 大纲 ]` `[ 🔗 链接 ]` `[ ⏱️ 时光机 (History) ]`
    
- **时光机视图视图结构:** 垂直时间轴 (Vertical Stepper)。
    
- **节点聚合逻辑 (Debounce):** 底层 yjs 记录了每次敲击，但 UI 不能展示上万个节点。前端需按“会话 (Session)”或“时间间隔 (如 5 分钟无操作)”将连续的本地修改打包为一个节点。当收到远端 P2P 同步包时，强制生成一个独立节点。
    
- **节点 UI 呈现:**
    
    Plaintext
    
    ```
    ▼ 今天
    │
    ◉ 14:30 · 💻 本机
    │  └ "添加了 3 个段落" (基于文本差异自动生成的摘要)
    │
    ◎ 11:15 · 📱 iPhone (来自同步)  <-- 绿色节点标示
    │  └ "修改了 标题 和 待办事项"
    │
    ▼ 昨天
    ◉ 22:00 · 💻 本机
    ```
    
- **查看差异 (Diff View) 交互:**
    
    - 鼠标点击 `11:15` 节点。
        
    - 左侧主编辑器立刻被一层极淡的蓝色半透明遮罩覆盖，进入**“时光机只读模式”**。
        
    - 主编辑器内容回滚至 11:15 的状态。新增内容底色变绿，删除内容底色变红加删除线。
        
    - 编辑器顶部浮现工具条：`[ 退出时光机 (ESC) ]` `[ 将此版本恢复为最新 ]` `[ 仅复制此版本内容 ]`。

### 4. 浮层与弹窗 (Overlays & Modals)

#### 4.1 ⌘K 全局指挥中心 (Command Palette)

应用内所有操作的快捷入口。

- **唤起动画:** 背景 `bg-black/40 backdrop-blur-sm` 渐显，面板从 `scale(0.98) opacity-0` 变为 `scale(1) opacity-100`。
    
- **布局:** 顶部无边框宽大输入框 (`text-xl`)。下方为带分组的虚拟滚动列表。
    
- **数据优先级结构:**
    
    1. _高亮动作:_ `⚡ 连接新设备` (当系统处于 mDNS 发现状态时置顶)。
        
    2. _最近文档:_ `📄 需求收集`、`📄 会议纪要`。
        
    3. _全文检索 (FTS5):_ 命中正文的文档。底部显示次级文本摘要，命中的关键词用 `text-amber-500` 高亮。
        
    4. _系统命令:_ `⚙️ 打开设置面板`、`🌙 切换暗色模式`。
        

#### 4.2 非局域网配对机制 (WAN Pairing Dialog)

解决“设备不在同一 Wi-Fi”时的连接问题。

- **入口:** Swarm 面板的 `[➕ 配对]` -> 选择 `[跨网络直连]`。
    
- **设备 A (发起端) UI:**
    
    - 生成并显示：“您的安全配对码为：**【 橘子 · 勇敢 · 海洋 】**” (基于 Hash 字典映射的易读词汇)。
        
    - 下方附带一个巨大的二维码（供移动端直接扫描）。
        
    - 状态提示：`"正在等待对方输入..." (带有转圈 loading)`。
        
- **设备 B (接收端) UI:**
    
    - 界面显示三个巨大的空白输入框，等待输入配对码。
        
    - 验证成功瞬间，输入框边框变绿，弹窗自动关闭，右下角弹出 Toast：`"成功与 [设备 A] 建立加密隧道"`。

## 四、 极端异常状态 UI 兜底 (Edge Cases)

- **1. 端口占用 / 网络组件崩溃 (P2P 严重错误):**
    
    - **表现:** 左下角 Swarm 面板图标变为 🔴 红色。
        
    - **UI:** 展开面板显示：“libp2p 端口 4001 被占用”。提供按钮 `[ 尝试更换随机端口并重启网络层 ]`。
        
- **2. 本地 SQLite 数据库损坏:**
    
    - **表现:** 启动 App 瞬间。
        
    - **UI:** 绝对全屏阻断。深红底色，白色大字。
        
    - **文案:** "本地存储仓库损坏。但这在 SwarmNote 不是世界末日。"
        
    - **操作:** `[ 导出本地受损数据 ]` / `[ 从 Swarm 集群 (如您的手机) 重新同步全量数据 ]`。
        
- **3. 外部程序修改文件 (外部冲突):**
    
    - **场景:** 用户用 VS Code 直接修改了 `.md` 文件，同时 SwarmNote 也在后台运行。
        
    - **UI:** 顶部标题栏下方滑出一个紧凑的黄色 Banner：
        
        `⚠️ "核心架构.md" 在磁盘上已被外部程序修改。 [ 重新加载磁盘内容 (推荐) ] [ 强制用当前编辑器内容覆盖磁盘 ]`
