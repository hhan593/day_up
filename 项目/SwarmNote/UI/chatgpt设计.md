# SwarmNote UI 设计稿 + PRD（Figma级结构）

---

# 一、设计目标

打造一个面向「技术用户 / 知识工作者」的本地优先 + 多设备同步的 Markdown 笔记工具。

设计风格参考：

- Obsidian（结构与信息密度）
    
- Anytype（现代感与卡片层级）
    
- VS Code（多面板布局 + 操作效率）
    

核心关键词：

> 克制 / 高效 / 可扩展 / 工具感

---

# 二、全局布局（Figma级结构）

## 2.1 主界面布局（Desktop）

```
┌──────────────────────────────────────────────────────────┐
│ 顶部栏 TopBar                                            │
│ SwarmNote | ⌘K | 同步状态 | 搜索 | 设置                  │
├────────┬────────────────┬───────────────────────────────┤
│ Icon   │ Side Panel     │ Editor Area                  │
│ Rail   │ (动态切换)      │ (Tab + 编辑器)                │
│        │                │                               │
│ 🗂     │ 文件树          │ [Tab1] [Tab2] [+]            │
│ 🔍     │ 搜索            │ ┌───────────────────────────┐ │
│ 🔄     │ 同步            │ │ 文档标题（可编辑）         │ │
│ ⚙     │ 快捷设置        │ │ BlockNote 编辑器          │ │
│        │                │ │                           │ │
│        │                │ └───────────────────────────┘ │
├────────┴────────────────┴───────────────────────────────┤
│ 状态栏 StatusBar                                        │
│ 字数 | 同步状态 | 最近保存 | 在线设备                    │
└──────────────────────────────────────────────────────────┘
```

## 2.2 布局说明（开发级）

# 三、核心模块设计（P0）

---

## 3.1 顶部栏 TopBar

### 结构

```
[Logo] [Workspace▼]     [⌘K 搜索]     [同步状态] [设置]
```

### 功能

|元素|功能|
|---|---|
|Workspace|切换工作区|
|⌘K|打开 Command Palette|
|同步状态|打开同步面板|
|设置|打开设置页|

---

## 3.2 Icon Rail（左侧导航栏）

```
│ 🗂 Files
│ 🔍 Search
│ 🔄 Sync
│ ⚙ Settings
```

### 交互

- 点击切换 Side Panel 内容
    
- 当前选中高亮
    
- Tooltip 提示

## 3.3 Side Panel（动态面板）

---

### A. 文件树（默认）

```
Workspace
├─ 📁 项目
│  ├─ 📄 设计
│  └─ 📄 文档
├─ 📄 README

+ New Note
+ New Folder
```

### 交互

- 右键菜单
    
- 拖拽排序
    
- 重命名（inline）
    
- hover 操作按钮
    

---

### B. 搜索面板

```
Search...
────────────
结果列表
- 匹配片段
```

支持：

- 文件名搜索
    
- 全文搜索
    

---

### C. 同步面板

```
Sync Status
────────────
✓ 已同步：12
⟳ 同步中：3
⚠ 冲突：1

最近活动：
- A.md → MacBook
- B.md → iPhone
```

# 十一、总结（设计原则）

1. 少按钮，多上下文
    
2. 所有操作可快捷键触达
    
3. 信息层级清晰
    
4. UI 不打断编辑体验
