## chat.service.ts（业务编排层）

它负责：

- 校验输入
    
- 选择模型
    
- 调用 runtime
    
- 保存消息（后期）
    
- 错误兜底
    

```ts
@Injectable()
export class ChatService {
  async streamChat(dto, res) {
    const runtime = this.runtimeService.create(dto);
    await runtime.run(res);
  }
}
```

# 五、AI Runtime 设计（核心中的核心）

## 1️⃣ Runtime 是什么？

你可以把 Runtime 理解为：

> **一次 AI 对话的“执行引擎”**

它串联：

```
Prompt 构建
→ LLM 调用
→ Token 流输出
→ Tool 调用（可选）
→ SSE 推送
```

---

## 2️⃣ runtime.service.ts（示意）

```ts
@Injectable()
export class ChatRuntimeService {
  async run(res: Response) {
    res.setHeader('Content-Type', 'text/event-stream');

    for await (const chunk of this.llm.chat(prompt)) {
      res.write(`data: ${JSON.stringify(chunk)}\n\n`);
    }

    res.write(`data: {"type":"done"}\n\n`);
    res.end();
  }
}
```

# 六、LLM 抽象层设计（你以后会非常感谢自己）

## 1️⃣ 统一接口（必须）

```ts
export interface LLMProvider {
  chat(messages: ChatMessage[]): AsyncIterable<LLMChunk>;
}
```

---

## 2️⃣ Qwen 本地模型 Adapter（示意）

```ts
@Injectable()
export class QwenAdapter implements LLMProvider {
  async *chat(messages) {
    const response = await axios.post(
      'http://localhost:11434/api/chat',
      {
        model: 'qwen2.5:3b',
        messages,
        stream: true
      },
      { responseType: 'stream' }
    );

    for await (const chunk of response.data) {
      yield parseChunk(chunk);
    }
  }
}
```

> 👉 你现在用的本地模型 **完美适配这个结构**

---

## 3️⃣ LLM Factory（模型路由）

```ts
@Injectable()
export class LLMFactory {
  get(model: string): LLMProvider {
    if (model === 'qwen') return this.qwen;
    if (model === 'openai') return this.openai;
  }
}
```

# 七、Prompt 构建层（别写在 Service 里）

```ts
export class PromptBuilder {
  build({ system, history, input }) {
    return [
      { role: 'system', content: system },
      ...history,
      { role: 'user', content: input }
    ];
  }
}
```

你未来加 RAG、用户画像，全在这里加。
